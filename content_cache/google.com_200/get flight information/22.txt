       <p><em>Editor’s note: This is the second in a series of posts about developing Alexa skills. Read the rest of the posts in this series to learn how to <a href="https://www.bignerdranch.com/blog/?q=alexa">build and deploy an Alexa skill</a>.</em></p>

       <h2 id="handling-launch">Handling Launch</h2> 
       <p>The first thing our Airport Info skill should do is respond to a launch request from Alexa. This is what will ultimately happen if a user says, “Alexa, launch Airport Info.”</p> 
       <p>Because we’re developing this skill using the alexa-app module, we have a nice shorthand syntax for defining both the launch request and the speech response. Add the following to the <code class="highlighter-rouge">index.js</code> file you’ve created in the <code class="highlighter-rouge">faa-info</code> directory.</p> 
       <div class="language-js highlighter-rouge">

       <p>Let’s take a moment to understand what output was just generated for us as a launch request through Alexa. If you refer to the <code class="highlighter-rouge">index.js</code> file we’ve just created, you’ll see an <code class="highlighter-rouge">app.launch</code> method. This method will be automatically triggered any time the skill is invoked, and it requires a response to be sent to the user. In this case, we’re calling a <code class="highlighter-rouge">response.say(prompt)</code>, which will output the text provided in the <code class="highlighter-rouge">var prompt</code> above it.</p> 
       <p>Afterwards, you will see a <code class="highlighter-rouge">reprompt(prompt)</code>. This is used any time a user is given the opportunity to say something, and will trigger approximately 8 seconds after the user is prompted the first time. Depending upon the design of your Alexa skill, you may want to shorten this to your liking, e.g. “Tell me an Airport code.”</p> 
       <p>Lastly, the <code class="highlighter-rouge">shouldEndSession(false)</code> method is the key determining factor as to whether your skill will continue listening for user interaction, or close the skill completely. When designing your skill, it’s important to weigh the benefits of keeping your stream open, as it may feel cumbersome to users if you expect them to continually interact with your skill. In this case, however, we want the user to be able to follow up after the welcome message with a desired airport code, so we’re choosing to leave this stream open (shouldEndSession(false)).</p> 
       <h2 id="intent-schema-and-utterances-for-airport-info">Intent Schema and Utterances for Airport Info</h2> 
       <p>We need to map utterances (training data of what the user may say) to intents (requests to perform functions on your skill service). Then in our web service, we take those intents and run any logic we need. If you are new to the concept of utterance and schema, you can learn more by reading the <a href="https://developer.amazon.com/public/solutions/alexa/alexa-skills-kit/docs/defining-the-voice-interface">Amazon documentation</a> for a detailed inquiry into those topics.</p> 
       <p>Here, we’ll get started on the Intent Schema.        <p>Add the following to <code class="highlighter-rouge">index.js</code> before the <code class="highlighter-rouge">module.exports</code>:</p> 
       <div class="language-js highlighter-rouge">

       <p>At this point, you’ve been given some useful tools and techniques for more easily developing an Alexa skill locally.
