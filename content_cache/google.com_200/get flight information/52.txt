              <p>How to find the version of an installed node.js/npm <strong>package</strong>?</p> 
              <p>This prints the version of npm itself:</p> 
              <pre><code>npm -v &lt;package-name&gt;
</code></pre> 
              <p>This prints a cryptic error:</p> 
              <pre><code>npm version &lt;package-name&gt;
</code></pre> 
              <p>This prints the package version <em>on the registry</em> (i.e. the latest version available):</p> 
              <pre><code>npm view &lt;package-name&gt; version
</code></pre> 
              <p>How do I get the <strong>installed version</strong>?</p>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p><code>npm list</code> for local packages or <code>npm list -g</code> for globally installed packages.</p> 
              <p>You can find the version of a specific package by passing its name as an argument. For example, <code>npm list grunt</code> will result in:</p> 
              <pre><code>projectName@projectVersion /path/to/project/folder
??? grunt@0.4.1
</code></pre> 
              <p>Alternatively, you can just run <code>npm list</code> without passing a package name as an argument to see the versions of all your packages:</p> 
              <pre><code>??? cli-color@0.1.6 
? ??? es5-ext@0.7.1 
??? coffee-script@1.3.3 
??? less@1.3.0 
??? sentry@0.1.2 
? ??? file@0.2.1 
? ??? underscore@1.3.3 
??? uglify-js@1.2.6 
</code></pre>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>Another quick way of finding out what packages are installed <strong>locally</strong> and <strong>without their dependencies</strong> is to use:</p> 
              <pre><code>npm list --depth=0
</code></pre> 
              <p>Which gives you something like </p> 
              <pre><code>??? bower@0.8.6
??? grunt@0.4.1
??? grunt-bower-requirejs@0.4.3
??? grunt-contrib-clean@0.4.1
??? grunt-contrib-coffee@0.7.0
??? grunt-contrib-copy@0.4.1
??? grunt-contrib-imagemin@0.1.4
??? grunt-contrib-jshint@0.1.1
??? grunt-contrib-livereload@0.1.2
??? grunt-contrib-requirejs@0.4.1
??? grunt-regarde@0.1.1
??? grunt-svgmin@0.1.0
</code></pre> 
              <p>Obviously, the same can be done globally with <code>npm list -g --depth=0</code>.</p> 
              <p>This method is clearer in case you have installed a lot of packages.</p> 
              <p>To find out which packages need to be updated, you can use <code>npm outdated -g --depth=0</code>.</p>

             <div class="post-text" itemprop="text"> 
              <p>If you agree to install <a href="http://stedolan.github.io/jq/">jq</a>, you can use the JSON output of <code>npm list</code>.</p> 
              <pre><code>npm -j ls &lt;package-name&gt; | jq -r .version
</code></pre> 
              <p>or, if you want to be verbose </p> 
              <pre><code>npm --json list &lt;package-name&gt; | jq --raw-output '.version'
</code></pre> 
              <p>For instance:</p> 
              <pre><code>$ npm -j ls ghost | jq -r .version
0.4.2
</code></pre> 
              <p>Also, the JSON format is slightly different for global packages, so you'll need to change the query.</p> 
              <p>For instance:</p> 
              <pre><code>$ npm -j -g ls | jq -r .dependencies.ghost.version
0.4.2
</code></pre>

              <p>Here's a portable Unix (using <code>grep</code> and <code>sed</code>) one-liner that returns the version string of a globally-installed npm package (remove the <code>g</code> from <code>-pg</code> to query local packages instead):</p> 
              <pre><code>$ npm ll -pg --depth=0 grunt | grep -o "@.*:" | sed 's/.$//; s/^.//'
0.4.5
</code></pre> 
              <ul> 
               <li>the <code>npm ll</code> outputs a parseable string formatted like: <code>/usr/lib/node_modules/npm:npm@2.14.8:</code>;</li> 
               <li>the <code>grep</code> command extracts the value between <code>@</code> and <code>:</code>, inclusive;</li> 
               <li>the <code>sed</code> command removes the surrounding characters.</li> 
              </ul>

             <div class="post-text" itemprop="text"> 
              <p>I added this to my .bashrc </p> 
              <pre><code>function npmv {
    case $# in # number of arguments passed
    0) v="$(npm -v)" ; #store output from npm -v in variable
        echo "NPM version is: $v"; #can't use single quotes 
                                   #${v} would also work
    ;;   
    1) s="$(npm list --depth=0 $1 | grep $1 | cut -d @ -f 2)";
       echo "$s";
    ;;
    2) case "$2" in # second argument
        g) #global|#Syntax to compare bash string to literal
             s="$(npm list --depth=0 -g $1 | grep $1 | cut -d @ -f 2)";
        echo "$s";
        ;;
        l) #latest
             npm view $1 version; #npm info $1 version does same thing
       ;;
       *) echo 'Invalid arguments';
       ;;
       esac;
    ;;
    *) echo 'Invalid arguments';
    ;;
    esac;
}
export -f npmv
</code></pre> 
              <p>Now all I have to do is type:</p> 
              <ul> 
               <li><strong>npmv</strong> for the version of npm eg: <code>NPM version is: 4.2.0</code></li> 
               <li><strong>npmv &lt;package-name&gt;</strong> for the local version eg: <code>0.8.08</code> </li> 
               <li><strong>npmv &lt;package-name&gt; g</strong> for global version eg: <code>0.8.09</code> </li> 
               <li><strong>npmv &lt;package-name&gt; l</strong> for latest version eg: <code>0.8.10</code> </li> 
              </ul> 
              <p>Note -d on cut command means delimit by, followed by @, then f means field the 2 means second field since there will be one either side of the @ symbol.</p>

              <h1>Access the <code>package.json</code></h1> 
              <p>You can access the <code>package.json</code> or <code>bower.json</code> of the package with:</p> 
              <p><code>notepad ./node_modules/:packageName/package.json</code></p> 
              <p>This will open the <code>package.json</code> in <code>notepad</code> which has the <code>version number</code> of the <code>:packageName</code> you included in the <code>command</code>.</p> 
              <h2>For example :</h2> 
              <p><code>notepad ./node_modules/vue-template-compiler/package.json</code></p>
