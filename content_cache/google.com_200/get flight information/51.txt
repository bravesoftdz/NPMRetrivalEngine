      <h1>How one developer just broke Node, Babel and thousands of projects in 11 lines of JavaScript</h1> 
      <h2>Code pulled from NPM – which everyone was using</h2> 
      <div class="byline_and_share"> 
       <div class="byline">

     <div id="body"> 
      <p><strong class="trailer">Updated</strong> Programmers were left staring at broken builds and failed installations on Tuesday after someone toppled the Jenga tower of JavaScript.</p> 
      <p>A couple of hours ago, Azer Koçulu unpublished more than 250 of his modules from <a target="_blank" href="https://www.npmjs.com/">NPM</a>, which is a popular package manager used by JavaScript projects to install dependencies.</p> 
      <p>Koçulu yanked his source code because, we're told, one of the modules was called Kik and that apparently <a target="_blank" href="https://medium.com/@azerbike/i-ve-just-liberated-my-modules-9045c06be67c#.rrf4u36oh">attracted the attention of lawyers</a> representing the instant-messaging <a target="_blank" href="http://www.kik.com/">app</a> of the same name.</p> 
      <p>According to Koçulu, Kik's briefs told him to rename the module, he refused, so the lawyers went to NPM's admins claiming brand infringement. When NPM took Kik away from the developer, he was furious and unpublished <i>all</i> of his NPM-managed modules. "This situation made me realize that NPM is someone’s private land where corporate is more powerful than the people, and I do open source because Power To The People," Koçulu blogged.</p> 
      <p>Unfortunately, one of those dependencies was <tt><a target="_blank" href="https://github.com/azer/left-pad/blob/master/index.js">left-pad</a></tt>. The code is below. It pads out the lefthand-side of strings with zeroes or spaces. And thousands of projects including Node and Babel relied on it.</p> 
      <p>With left-pad removed from NPM, these applications and widely used bits of open-source infrastructure were unable to obtain the dependency, and thus fell over during development and deployment. Thousands, worldwide. Left-pad was fetched 2,486,696 downloads in just the last month, according to NPM. It was that popular.</p> 
      <pre>
module.exports = leftpad;

function leftpad (str, len, ch) {
  str = String(str);

  var i = -1;

  if (!ch &amp;&amp; ch !== 0) ch = ' ';

  len = len - str.length;

  while (++i &lt; len) {
    str = ch + str;
  }

  return str;
}
</pre> 
      <p>You can witness some of the fallout <a target="_blank" href="https://github.com/azer/left-pad/issues/4">here</a>, <a target="_blank" href="https://news.ycombinator.com/item?id=11340510">here</a>, <a target="_blank" href="https://www.reddit.com/r/programming/comments/4bjss2/an_11_line_npm_package_called_leftpad_with_only/">here</a> and <a target="_blank" href="https://twitter.com/search?f=tweets&amp;vertical=default&amp;q=left-pad%20npm&amp;src=typd">here</a>.</p> 
      <p>To fix the internet, Laurie Voss, CTO and cofounder of NPM, took the "<a target="_blank" href="https://twitter.com/seldo/status/712414400808755200">unprecedented</a>" step of restoring the unpublished left-pad 0.0.3 that apps required. Normally, when a particular version is unpublished, it's gone and cannot be restored. Now NPM has forcibly resurrected that particular version to keep everyone's stuff building and running as expected.</p> 
      <p>"Un-un-publishing is an unprecedented action that we're taking given the severity and widespread nature of breakage, and isn't done lightly," Voss explained about an hour ago.</p> 
      <p>"This action puts the wider interests of the community of NPM users at odds with the wishes of one author; we picked the needs of the many. This whole situation sucks. We will be carefully considering the issues raised by and publishing a post-mortem later.</p> 
      <p>"In the meantime, several thousand open source projects have been repaired, and I'm sleeping fine tonight."</p> 
      <p>A new maintainer, who stepped forward to look after <a target="_blank" href="https://www.npmjs.com/package/left-pad">left-pad</a> on NPM, requested the restoration of version 0.0.3. Meanwhile, Oakland-based Koçulu has hosted his work on <a target="_blank" href="https://github.com/azer">GitHub</a>. If your code still won't build after the left-pad revival, try running <tt>npm cache clear</tt> to catch up with the changes.</p> 
      <p>And that's how JavaScript app development works in 2016. ®</p> 
      <h3>Updated to add at 1138 PT (1838 UTC) on March 23</h3> 
      <p>A spokesperson for Kik has been in touch to point us toward <a target="_blank" href="https://medium.com/@mproberts/a-discussion-about-the-breaking-of-the-internet-3d4d2a83aa4d#.ld8o5zqz7">a blog post</a> by Mike Roberts, the head of messenger at Kik, setting out its side of the story. It was published about 30 minutes ago.</p> 
      <p>Essentially, the instant-messaging biz says it was going to publish some open-source code including an NPM module also named Kik. That collided with Koçulu's Kik module, which is why the developer was asked to rename his software. As we know, Koçulu refused, and that led to Tuesday's cluster-fsck.</p> 
      <p>Referring to the emails sent by Kik's lawyer to Koçulu, Roberts said: "The wording we used here was not perfect. We’re sorry for creating any impression that this was anything more than a polite request to use the Kik package name on NPM for an open source project we have been working on that fits the name."</p>
