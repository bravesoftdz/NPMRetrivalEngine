             <section class="section" id="resource_limits">
              <div class="titlepage">
               <div>
                <div>
                 <h2 class="title">5.2.&nbsp;Resource Limits</h2>
                </div>
               </div>
              </div>
              <p> OpenShift <a class="link" href="https://access.redhat.com/documentation/en-us/openshift_container_platform/3.6/html/developer_guide/dev-guide-compute-resources">allows</a> administrators to set constraints to limit the number of objects or amount of compute resources that are used in each project. While these constraints apply to projects in the aggregate, <a class="link" href="https://access.redhat.com/documentation/en-us/openshift_container_platform/3.6/html/developer_guide/dev-guide-compute-resources#dev-compute-resources">each pod</a> may also request minimum resources and/or be constrained with limits on its memory and CPU use. </p>
              <p> The OpenShift template provided in the project repository uses this capability <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Zipkin/zipkin-mysql.yml#L171-L177">to request</a> that at least 20% of a CPU core and 200 megabytes of memory be made available to its container. Twice the processing power and four times the memory may be provided to the container, if necessary and available, but no more than that will be assigned. </p>
              <div class="informalexample">
               <p> resources:<br> &nbsp;&nbsp;limits:<br> &nbsp;&nbsp;&nbsp;&nbsp;cpu: "400m"<br> &nbsp;&nbsp;&nbsp;&nbsp;memory: "800Mi"<br> &nbsp;&nbsp;requests:<br> &nbsp;&nbsp;&nbsp;&nbsp;cpu: "200m"<br> &nbsp;&nbsp;&nbsp;&nbsp;memory: "200Mi"<br> </p>
              </div>
              <p> When the fabric8 Maven plugin is used to create the image and direct edits to the deployment configuration are not convenient, <a class="link" href="https://maven.fabric8.io/#configuration">resource fragments</a> can be used to provide the desired snippets. This application provides <a class="link" href="https://raw.githubusercontent.com/RHsyseng/spring-boot-msa-ocp/master/Airports/src/main/fabric8/deployment.yml"><span class="emphasis"><em>deployment.yml</em></span></a> files to leverage this capability and set resource requests and limits on Spring Boot projects: </p>
              <div class="informalexample">
               <p> spec:<br> &nbsp;&nbsp;&nbsp;replicas: 1<br> &nbsp;&nbsp;&nbsp;template:<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;spec:<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;containers:<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- resources:<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;requests:<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;cpu: '200m'<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;memory: '400Mi'<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;limits:<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;cpu: '400m'<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;memory: '800Mi'<br> </p>
              </div>
              <p> Control over the memory and processing use of individual services is often critical. Proper configuration of these values, as specified above, is seamless to the deployment and administration process. However, it can be helpful to set up <a class="link" href="https://access.redhat.com/documentation/en-us/openshift_container_platform/3.6/html/developer_guide/dev-guide-compute-resources#dev-quotas">resource quotas</a> in projects for the purpose of <a class="link" href="https://access.redhat.com/documentation/en-us/openshift_container_platform/3.6/html/developer_guide/dev-guide-compute-resources#dev-requests-vs-limits">enforcing</a> their inclusion in pod deployment configurations. </p>
             </section>

This application uses <a class="link" href="https://raw.githubusercontent.com/RHsyseng/spring-boot-msa-ocp/master/Airports/src/main/resources/application.yml">application.yml</a> files that begin with each projects name: </p>

                 </div>
                </div>

               <p> Each microservice project includes a <code class="literal"><span class="strong strong"><strong>Maven</strong></span></code> <a class="link" href="https://raw.githubusercontent.com/RHsyseng/spring-boot-msa-ocp/master/Airports/pom.xml"><span class="emphasis"><em>POM</em></span> file</a>, which in addition to declaring the module properties and dependencies, also includes a profile definition to use <a class="link" href="https://maven.fabric8.io/"><code class="literal"><span class="strong strong"><strong>fabric8-maven-plugin</strong></span></code></a> to create and deploy an OpenShift image. </p>
               <p> The <span class="emphasis"><em>POM</em></span> file uses a property to declare the base image containing the operating system and <code class="literal"><span class="strong strong"><strong>Java Development Kit (JDK)</strong></span></code>. All the services in this application build on top of a <code class="literal"><span class="strong strong"><strong>Red Hat Enterprise Linux (RHEL)</strong></span></code> base image, containing a supported version of <code class="literal"><span class="strong strong"><strong>OpenJDK</strong></span></code>: </p>
               <div class="informalexample">
                <p> &lt;properties&gt;<br> ...<br> &nbsp;&nbsp;&nbsp;&nbsp;&lt;fabric8.generator.from&gt;registry.access.redhat.com/redhat-openjdk-18/openjdk18-openshift&lt;/fabric8.generator.from&gt;<br> &lt;/properties&gt;<br> </p>
               </div>
               <p> To easily include the dependencies for a simple Spring Boot application that provides a REST service, declare the following two artifacts: </p>
               <div class="informalexample">
                <p> &lt;dependency&gt;<br> &nbsp;&nbsp;&nbsp;&nbsp;&lt;groupId&gt;org.springframework.boot&lt;/groupId&gt;<br> &nbsp;&nbsp;&nbsp;&nbsp;&lt;artifactId&gt;spring-boot-starter-web&lt;/artifactId&gt;<br> &lt;/dependency&gt;<br> &lt;dependency&gt;<br> &nbsp;&nbsp;&nbsp;&nbsp;&lt;groupId&gt;org.springframework.boot&lt;/groupId&gt;<br> &nbsp;&nbsp;&nbsp;&nbsp;&lt;artifactId&gt;spring-boot-starter-tomcat&lt;/artifactId&gt;<br> &lt;/dependency&gt;<br> </p>
               </div>
               <p> Every service in this application also declares a dependency on the <a class="link" href="https://github.com/spring-projects/spring-boot/tree/master/spring-boot-actuator"><code class="literal"><span class="strong strong"><strong>Spring Boot Actuator</strong></span></code></a> component, which includes a number of additional features to help monitor and manage your application. </p>
               <div class="informalexample">
                <p> &lt;dependency&gt;<br> &nbsp;&nbsp;&nbsp;&nbsp;&lt;groupId&gt;org.springframework.boot&lt;/groupId&gt;<br> &nbsp;&nbsp;&nbsp;&nbsp;&lt;artifactId&gt;spring-boot-starter-actuator&lt;/artifactId&gt;<br> &lt;/dependency&gt;<br> </p>
               </div>
               <p> When a dependency on the <span class="emphasis"><em>Actuator</em></span> is declared, <span class="emphasis"><em>fabric8</em></span> generates default OpenShift <a class="link" href="https://access.redhat.com/documentation/en-us/openshift_container_platform/3.6/html/developer_guide/dev-guide-application-health#container-health-checks-using-probes">health probes</a> that communicate with Actuator services to determine whether a service is running and ready to service requests: </p>
               <div class="informalexample">

               </div>
               <p> Declare a load balanced <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Flights/src/main/java/com/redhat/refarch/obsidian/brownfield/lambdaair/flights/service/Controller.java#L39-L47"><span class="emphasis"><em>RestTemplate</em></span></a> and use injection to assign it to a field: </p>
               <pre class="screen">@LoadBalanced
@Bean
RestTemplate restTemplate()
{
	return new RestTemplate();
}

@Autowired
private RestTemplate restTemplate;</pre>
               <p> For outgoing calls, simply <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Flights/src/main/java/com/redhat/refarch/obsidian/brownfield/lambdaair/flights/service/Controller.java#L64">use the restTemplate</a> field: </p>
               <pre class="screen">Airport[] airportArray = restTemplate.getForObject( "http://zuul/airports/airports", Airport[].class );</pre>
               <p> The service address provided as the host part of the URL is resolved through Ribbon, based on values provided in application <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Flights/src/main/resources/application.yml#L13-L15">properties</a>: </p>
               <div class="informalexample">
                <p> zuul:<br> &nbsp;&nbsp;ribbon:<br> &nbsp;&nbsp;&nbsp;&nbsp;listOfServers: zuul:8080<br> </p>
               </div>
               <p> In this case, Ribbon expects a list of statically defined service addresses, but a single one is provided with the hostname of <span class="emphasis"><em>zuul</em></span> with port 8080. Zuul uses the second part of the address, the root web context, to redirect the request through statically or dynamic routing, as explained <a class="link" href="design_and_development#zuul_service" title="5.9.&nbsp;Zuul">later</a> in this document. </p>
               <p> The provided hostname of <span class="emphasis"><em>zuul</em></span> is the OpenShift service name, and is resolved to the cluster IP address of the service, then routed to an internal OpenShift load balancer. The OpenShift service name is determined when a service is created using the <span class="emphasis"><em>oc</em></span> tool, or when deploying an image using the fabric8 Maven plugin, it is declared in the <a class="link" href="https://raw.githubusercontent.com/RHsyseng/spring-boot-msa-ocp/master/Zuul/src/main/fabric8/svc.yml">service yaml</a> file. </p>
               <p> Ribbon is effectively not load balancing requests, but rather sending them to an OpenShift internal load balancer, which is aware of replication and failure of service instances, and can redirect the request properly. </p>

             <section class="section" id="hystrix_2">
              <div class="titlepage">
               <div>
                <div>
                 <h2 class="title">5.6.&nbsp;Hystrix</h2>
                </div>
               </div>
              </div>
              <section class="section" id="overview_13">
               <div class="titlepage">
                <div>
                 <div>
                  <h3 class="title">5.6.1.&nbsp;Overview</h3>
                 </div>
                </div>
               </div>
               <p> The <span class="emphasis"><em>Presentation</em></span> service includes a second listening controller, this time a <a class="link" href="https://raw.githubusercontent.com/RHsyseng/spring-boot-msa-ocp/master/Presentation/src/main/java/com/redhat/refarch/obsidian/brownfield/lambdaair/presentation/service/API_GatewayController.java">REST controller</a>, that acts as an API gateway. The API gateway makes simple REST calls to the <span class="emphasis"><em>Airports</em></span> service, similar to the previously discussed <a class="link" href="design_and_development#flights_service">Flights service</a>, but also calls the <span class="emphasis"><em>Sales</em></span> service to get pricing information and uses a different pattern for this call. <span class="emphasis"><em>Hystrix</em></span> is used to avoid a large number of hung threads and lengthy timeouts when the <span class="emphasis"><em>Sales</em></span> service is down. Instead, flight information can be returned without providing a ticket price. The reactive interface of <span class="emphasis"><em>Hystrix</em></span> is also leveraged to implement parallel processing. </p>
              </section>
              <section class="section" id="circuit_breaker_2">
               <div class="titlepage">
                <div>
                 <div>
                  <h3 class="title">5.6.2.&nbsp;Circuit Breaker</h3>
                 </div>
                </div>
               </div>
               <p> <span class="emphasis"><em>Hystrix</em></span> provides multiple patterns for the use of its API. The <span class="emphasis"><em>Presentation</em></span> service uses <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Presentation/src/main/java/com/redhat/refarch/obsidian/brownfield/lambdaair/presentation/service/API_GatewayController.java#L184-L215"><span class="emphasis"><em>Hystrix</em></span> commands</a> for its outgoing calls to <span class="emphasis"><em>Sales</em></span>. This is implemented as a <span class="emphasis"><em>Hystrix</em></span> command: </p>
               <pre class="screen">private class PricingCall extends HystrixCommand&lt;Itinerary&gt;
{
	private Flight flight;

	PricingCall(Flight flight)
	{
		super( HystrixCommandGroupKey.Factory.asKey( "Sales" ), HystrixThreadPoolKey.Factory.asKey( "SalesThreads" ) );
		this.flight = flight;
	}

	@Override
	protected Itinerary run() throws Exception
	{
		try
		{
			return restTemplate.postForObject( "http://zuul/sales/price", flight, Itinerary.class );
		}
		catch( Exception e )
		{
			logger.log( Level.SEVERE, "Failed!", e );
			throw e;
		}
	}

	@Override
	protected Itinerary getFallback()
	{
		logger.warning( "Failed to obtain price, " + getFailedExecutionException().getMessage() + " for " + flight );
		return new Itinerary( flight );
	}
}</pre>
               <p> After being instantiated and provided a flight for pricing, the command takes one of two routes. When successful and able to reach the service being called, the <span class="emphasis"><em>run</em></span> method is executed which uses the now-familiar pattern of calling the service through Ribbon and the OpenShift service abstraction. However, if an error prevents us from reaching the <span class="emphasis"><em>Sales</em></span> service, <span class="emphasis"><em>getFallback()</em></span> provides a chance to recover from the error, which in this case involves returning the itinerary without a price. </p>
               <p> The fallback scenario can happen simply because the call has failed, but also in cases when the circuit is open (tripped). Configure Hystrix as part of the <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Presentation/src/main/resources/application.yml#L17-L24">service properties</a> to specify when a thread should time out and fail, as well as the queue used for concurrent processing of outgoing calls. </p>
               <p> To <a class="link" href="https://github.com/Netflix/Hystrix/wiki/configuration#execution.isolation.thread.timeoutInMilliseconds">configure</a> the command timeout for a specific command (and not globally), the <span class="emphasis"><em>HystrixCommandKey</em></span> is required. This defaults to the command class name, which is <span class="emphasis"><em>PricingCall</em></span> in this implementation. </p>
               <p> Configure <a class="link" href="https://github.com/Netflix/Hystrix/wiki/configuration#ThreadPool">thread pool properties</a> for this specific thread pool by using the specified thread pool key of <span class="emphasis"><em>SalesThreads</em></span>. </p>
               <div class="informalexample" id="hystrix_properties">
                <p> hystrix.command.PricingCall.execution.isolation.thread.timeoutInMilliseconds: 2000<br><br> hystrix:<br> &nbsp;&nbsp;threadpool:<br> &nbsp;&nbsp;&nbsp;&nbsp;SalesThreads:<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;coreSize: 20<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;maxQueueSize: 200<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;queueSizeRejectionThreshold: 200<br> </p>
               </div>
              </section>
              <section class="section" id="concurrent-execution">
               <div class="titlepage">
                <div>
                 <div>
                  <h3 class="title">5.6.3.&nbsp;Concurrent Reactive Execution</h3>
                 </div>
                </div>
               </div>
               <p> We assume technical considerations have led to the <span class="emphasis"><em>Sales</em></span> service accepting a single flight object in its API. To reduce lag time and take advantage of horizontal scaling, the service uses <a class="link" href="https://github.com/Netflix/Hystrix/wiki/How-To-Use#reactive-commands">Reactive Commands</a> for batch processing of pricing calls. </p>
               <p> The API gateway service queries and stores the configured thread pool size as a field: </p>
               <pre class="screen">@Value("${hystrix.threadpool.SalesThreads.coreSize}")
private int threadSize;</pre>
               <p> The thread size is later used as the batch size for the concurrent calls to calculate the price of a flight: </p>
               <pre class="screen">int batchLimit = Math.min( index + threadSize, itineraries.length );
for( int batchIndex = index; batchIndex &lt; batchLimit; batchIndex++ )
{
	observables.add( new PricingCall( itineraries[batchIndex] ).toObservable() );
}</pre>
               <p> The Reactive <a class="link" href="http://reactivex.io/documentation/operators/zip.html">zip</a> operator is used to <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Presentation/src/main/java/com/redhat/refarch/obsidian/brownfield/lambdaair/presentation/service/API_GatewayController.java#L142-L151">process the calls</a> for each batch concurrently and store results in a collection. The number of batches depends on the ratio of total flights found to the batch size, which is set to <a class="link" href="design_and_development#hystrix_properties">20</a> in this service configuration. </p>
              </section>
             </section>
             <section class="section" id="openshift_configmap">
              <div class="titlepage">
               <div>
                <div>
                 <h2 class="title">5.7.&nbsp;OpenShift ConfigMap</h2>
                </div>
               </div>
              </div>
              <section class="section" id="overview_14">
               <div class="titlepage">
                <div>
                 <div>
                  <h3 class="title">5.7.1.&nbsp;Overview</h3>
                 </div>
                </div>
               </div>
               <p> While considering the <a class="link" href="design_and_development#concurrent-execution" title="5.6.3.&nbsp;Concurrent Reactive Execution">concurrent execution</a> of pricing calls, it should be noted that the API gateway is itself multi-threaded, so the batch size is not the final determinant of the thread count. In this example of a batch size of 20, with a maximum queue size of 200 and the same threshold leading to rejection, receiving more than 10 concurrent <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Presentation/src/main/java/com/redhat/refarch/obsidian/brownfield/lambdaair/presentation/service/API_GatewayController.java#L80">query</a> calls can lead to errors. These values should be fine-tuned based on realistic expectations of load as well as the horizontal scaling of the environment. </p>
               <p> This configuration can be externalized by creating a <span class="emphasis"><em>ConfigMap</em></span> for each OpenShift environment, with overriding values provided in a properties file that is then provided to all future pods. </p>

                <p> This reference architecture demonstrates the use of <a class="link" href="https://access.redhat.com/documentation/en-us/openshift_container_platform/3.6/html/developer_guide/deployments#lifecycle-hooks">lifecycle hooks</a> to initialize a database after the pod has been created.                 <div class="informalexample">
                 <p> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;recreateParams:<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;post:<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;failurePolicy: Abort<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;execNewPod:<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;containerName: mysql<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;command:<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- /bin/sh<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- -c<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- hostname &amp;&amp; sleep 10 &amp;&amp; /opt/rh/rh-mysql57/root/usr/bin/mysql -h $DATABASE_SERVICE_NAME -u $MYSQL_USER -D $MYSQL_DATABASE -p$MYSQL_PASSWORD -P 3306 &lt; /docker-entrypoint-initdb.d/init.sql &amp;&amp; echo Initialized database<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;env:<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- name: DATABASE_SERVICE_NAME<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;value: ${DATABASE_SERVICE_NAME}<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;volumes:<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;- mysql-init-script<br> </p>
                </div>
                <p> Notice that the hook uses the command line <span class="emphasis"><em>mysql</em></span> utility to run the SQL script located <span class="emphasis"><em>/docker-entrypoint-initdb.d/init.sql</em></span>. Some database images standardize on this location for initialization, in which case a lifecycle hook is not required. </p>

                </div>
               </div>
               <p> The template uses the <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Zipkin/zipkin-mysql.yml#L145-L177">image</a> provided by OpenZipkin: </p>
               <div class="informalexample">
                <p> image: openzipkin/zipkin:1.19.2<br> </p>
               </div>
               <p> Required parameters for <span class="emphasis"><em>OpenZipkin</em></span> to access the associated <span class="emphasis"><em>MySQL</em></span> database are either configured or generated as part of the same template. Database passwords are <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Zipkin/zipkin-mysql.yml#L313-L324">randomly generated</a> by OpenShift as part of the template and stored in a <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Zipkin/zipkin-mysql.yml#L21-L30"><span class="emphasis"><em>secret</em></span></a>, which makes them inaccessible to users and administrators in the future. That is why a <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Zipkin/zipkin-mysql.yml#L5-L11">template message</a> is printed to allow a one-time access to the database password for monitoring and troubleshooting purposes. </p>
               <p> To create the <span class="emphasis"><em>Zipkin</em></span> service: </p>
               <pre class="screen"><span class="strong strong"><strong>$ oc new-app -f LambdaAir/Zipkin/zipkin-mysql.yml</strong></span>
--&gt; Deploying template "lambdaair/" for "zipkin-mysql.yml" to project lambdaair

              </section>
              <section class="section" id="spring_sleuth">
               <div class="titlepage">
                <div>
                 <div>
                  <h3 class="title">5.8.4.&nbsp;Spring Sleuth</h3>
                 </div>
                </div>
               </div>
               <p> While the <span class="emphasis"><em>Zipkin</em></span> service allows distributed tracing data to be aggregated, persisted and used for reporting, this application relies on <span class="emphasis"><em>Spring Sleuth</em></span> to correlate calls and send data to Zipkin. </p>
               <p> Integration with Ribbon and other framework libraries make it very easy to use Spring Sleuth in the application. Include the libraries by declaring a dependency in the project Maven file: </p>
               <div class="informalexample">
                <p> &lt;dependency&gt;<br> &nbsp;&nbsp;&nbsp;&nbsp;&lt;groupId&gt;org.springframework.cloud&lt;/groupId&gt;<br> &nbsp;&nbsp;&nbsp;&nbsp;&lt;artifactId&gt;spring-cloud-sleuth-zipkin&lt;/artifactId&gt;<br> &nbsp;&nbsp;&nbsp;&nbsp;&lt;version&gt;1.2.1.RELEASE&lt;/version&gt;<br> &lt;/dependency&gt;<br> </p>
               </div>
               <p> Also specify in the application properties the percentage of requests that should be traced, as well as the address to the zipkin server. Once again, we rely on the OpenShift service abstract to reach zipkin. </p>
               <div class="informalexample">
                <p> spring:<br> &nbsp;&nbsp;sleuth:<br> &nbsp;&nbsp;&nbsp;&nbsp;sampler:<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;percentage: 1.0<br> &nbsp;&nbsp;zipkin:<br> &nbsp;&nbsp;&nbsp;&nbsp;baseUrl: http://zipkin/<br> </p>
               </div>
               <p> The percentage value of 1.0 means 100% of requests will be captured. </p>
               <p> These two steps are enough to collect tracing data, but a <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Presentation/src/main/java/com/redhat/refarch/obsidian/brownfield/lambdaair/presentation/service/API_GatewayController.java#L63">Tracer</a> object can also be injected into the code for extended functionality. While every remote call can produce and store a trace by default, <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Presentation/src/main/java/com/redhat/refarch/obsidian/brownfield/lambdaair/presentation/service/API_GatewayController.java#L68">adding a tag</a> can help to better understand zipkin reports. The service also <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Presentation/src/main/java/com/redhat/refarch/obsidian/brownfield/lambdaair/presentation/service/API_GatewayController.java#L83">creates</a> and <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Presentation/src/main/java/com/redhat/refarch/obsidian/brownfield/lambdaair/presentation/service/API_GatewayController.java#L124">demarcates</a> tracing <span class="emphasis"><em>spans</em></span> of interest to collect more meaningful tracing data. </p>
               <section class="section" id="baggage_data">
                <div class="titlepage">
                 <div>
                  <div>
                   <h4 class="title">5.8.4.1.&nbsp;Baggage Data</h4>
                  </div>
                 </div>
                </div>
                <p> While <span class="emphasis"><em>Spring Sleuth</em></span> is primarily intended as a distributed tracing tool, its ability to correlate distributed calls can have other practical uses as well. Every created <span class="emphasis"><em>span</em></span> allows the attachment of arbitrary data, called a <span class="strong strong"><strong>baggage item</strong></span>, that will be automatically inserted into the HTTP header and seamlessly carried along with the business request from service to service, for the duration of the <span class="emphasis"><em>span</em></span>. This application is interested in making the original callers IP address available to every microservice. In an OpenShift environment, the calling IP address is stored in the HTTP header under a standard key. To retrieve and set this value on the <span class="emphasis"><em>span</em></span>: </p>
                <pre class="screen">querySpan.setBaggageItem( "forwarded-for", request.getHeader( "x-forwarded-for" ) );</pre>
                <p> This value will later be accessible from any service within the same call <span class="emphasis"><em>span</em></span> under the header key of <span class="emphasis"><em>baggage-forwarded-for</em></span>. It is <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Zuul/misc/ABTestingFilterBean.groovy#L32">used</a> by the <span class="emphasis"><em>Zuul</em></span> service in a <code class="literal"><span class="strong strong"><strong>Groovy</strong></span></code> script to perform dynamic routing. </p>
               </section>
              </section>
             </section>
             <section class="section" id="zuul_service">
              <div class="titlepage">
               <div>
                <div>
                 <h2 class="title">5.9.&nbsp;Zuul</h2>
                </div>
               </div>
              </div>
              <section class="section" id="overview_16">
               <div class="titlepage">
                <div>
                 <div>
                  <h3 class="title">5.9.1.&nbsp;Overview</h3>
                 </div>
                </div>
               </div>
               <p> This reference architecture uses <span class="emphasis"><em>Zuul</em></span> as a <a class="link" href="reference_architecture_environment#zuul_diagram" title="Figure&nbsp;3.5.&nbsp;Zuul Proxy">central proxy</a> for all calls between microservices. By default, the service uses static routing as defined in its <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Zuul/src/main/resources/application.yml#L8-L18">application properties</a>: </p>
               <div class="informalexample">
                <p> &nbsp;zuul:<br> &nbsp;&nbsp;&nbsp;routes:<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;airports:<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;path: /airports/**<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;url: http://airports:8080/<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;flights:<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;path: /flights/**<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;url: http://flights:8080/<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;sales:<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;path: /sales/**<br> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;url: http://sales:8080/<br> </p>
               </div>
               <p> The <span class="emphasis"><em>path</em></span> provided in the above rules uses the first part of the web address to determine the service to be called, and the rest of the address as the context. </p>
              </section>
              <section class="section" id="a_b_testing_2">
               <div class="titlepage">
                <div>
                 <div>
                  <h3 class="title">5.9.2.&nbsp;A/B Testing</h3>
                 </div>
                </div>
               </div>
               <p> To implement A/B testing, the Salesv2 service introduces a minor change in the algorithm for calculating fares. Dynamic routing is provided by <span class="emphasis"><em>Zuul</em></span> through a <a class="link" href="https://raw.githubusercontent.com/RHsyseng/spring-boot-msa-ocp/master/Zuul/misc/ABTestingFilterBean.groovy">filter</a> that filters some of the requests. </p>
               <p> Calls to other services are not filtered: </p>
               <pre class="screen">if( !RequestContext.currentContext.getRequest().getRequestURI().matches("/sales.*") )
{
    //Won't filter this request URL
    false
}</pre>
               <p> Only those calls to the <span class="emphasis"><em>Sales</em></span> service that originate from an IP address ending in an odd digit are filtered: </p>
               <pre class="screen">String caller = new HTTPRequestUtils().getHeaderValue("baggage-forwarded-for");
logger.info("Caller IP address is " + caller)
int lastDigit = caller.reverse().take(1) as Integer
if( lastDigit % 2 == 0 )
{
    //Even IP address won't be filtered
    false
}
else
{
    //Odd IP address will be filtered
    true
}</pre>
               <p> If the caller has an odd digit at the end of their IP address, the request is rerouted. That means the <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Zuul/misc/ABTestingFilterBean.groovy#L49-L52">run</a> method of the filter is executed, which changes the route host: </p>
               <pre class="screen">@Override
Object run() {
    println("Running filter")
    RequestContext.currentContext.routeHost = new URL("http://salesv2:8080")
}</pre>
               <p> To enable dynamic routing without changing application code, shared storage is made available to the OpenShift nodes and a persistent volume is <a class="link" href="https://raw.githubusercontent.com/RHsyseng/spring-boot-msa-ocp/master/Zuul/misc/zuul-pv.json">created</a> and <a class="link" href="https://raw.githubusercontent.com/RHsyseng/spring-boot-msa-ocp/master/Zuul/misc/zuul-pvc.json">claimed</a>. With the volume set up and the groovy filter in place, the OpenShift deployment config can be adjusted administratively to mount a directory as a volume: </p>
               <pre class="screen"><span class="strong strong"><strong>$ oc volume dc/zuul --add --name=groovy --type=persistentVolumeClaim --claim-name=groovy-claim --mount-path=/groovy</strong></span></pre>
               <p> This results in all groovy scripts under the groovy directory being found. The <span class="emphasis"><em>zuul</em></span> application code anticipates the introduction of dynamic routing filters by <a class="link" href="https://github.com/RHsyseng/spring-boot-msa-ocp/blob/master/Zuul/src/main/java/com/redhat/refarch/obsidian/brownfield/lambdaair/zuul/RestApplication.java#L27-L35">seeking and applying</a> any groovy script under this path: </p>
               <pre class="screen">for( Resource resource : new PathMatchingResourcePatternResolver().getResources( "file:/groovy/<span class="strong strong"><strong>.groovy" ) ) { logger.info( "Found and will load groovy script " + resource.getFilename() ); sources.add( resource ); } if( sources.size() == 1 ) { logger.info( "No groovy script found under /groovy/</strong></span>.groovy" );
}</pre>
               <p> </p>
              </section>
             </section>
            </section>
