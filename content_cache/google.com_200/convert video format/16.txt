     <section class="level1" id="section_doctype">
      <h1 id="doctype">DOCTYPE</h1> 
      <p>At the top of your page declare the HTML5 <a href="http://www.w3schools.com/tags/tag_doctype.asp">doctype</a>: </p>
      <div class="codebox">
       <figure class="code">
        <div class="highlight">
         <pre><span></span><span class="cp">&lt;!DOCTYPE html&gt;</span><br></pre>
        </div>
        <br>
        <figcaption>
         HTML
        </figcaption>
       </figure>
      </div>
      <p></p> 
      <p>This is mandatory. Otherwise HTML5 video will not work in some browsers, notably Internet Explorer 9.</p> 
     </section>
     <section class="level1" id="section_prerequisites">
      <h1 id="prerequisites">Prerequisites</h1> 
      <p>Load the required assets and declare the page title - the TITLE tag is mandatory for HTML5 pages - in the <a href="http://www.w3schools.com/tags/tag_head.asp">HEAD</a> section of your page: </p>
      <div class="codebox">
       <figure class="code">
        <div class="highlight">
         <pre><span></span><span class="p">&lt;</span><span class="nt">title</span><span class="p">&gt;</span>My Flowplayer video<span class="p">&lt;/</span><span class="nt">title</span><span class="p">&gt;</span><br>&nbsp;<br><span class="c">&lt;!-- 1. skin --&gt;</span><br><span class="p">&lt;</span><span class="nt">link</span> <span class="na">rel</span><span class="o">=</span><span class="s">"stylesheet"</span> <span class="na">href</span><span class="o">=</span><span class="s">"//releases.flowplayer.org/7.1.2/skin/skin.css"</span><span class="p">&gt;</span><br>&nbsp;<br><span class="c">&lt;!-- 2. jquery library - required for video tag based installs --&gt;</span><br><span class="p">&lt;</span><span class="nt">script</span> <span class="na">src</span><span class="o">=</span><span class="s">"//code.jquery.com/jquery-1.12.4.min.js"</span><span class="p">&gt;&lt;/</span><span class="nt">script</span><span class="p">&gt;</span><br>&nbsp;<br><span class="c">&lt;!-- 3. flowplayer --&gt;</span><br><span class="p">&lt;</span><span class="nt">script</span> <span class="na">src</span><span class="o">=</span><span class="s">"//releases.flowplayer.org/7.1.2/flowplayer.min.js"</span><span class="p">&gt;&lt;/</span><span class="nt">script</span><span class="p">&gt;</span><br></pre>
        </div>
        <br>
        <figcaption>
         HTML
        </figcaption>
       </figure>
      </div>
      <p></p> 
      <ul> 
       <li>These assets are served globally from a content delivery network (Amazon CloudFront). Free for you to use.</li> 
       <li>You can place the files on your own servers too, maybe combine them with your existing files for faster initial load. The latest commercial version is always available from your <a href="/account/#players">account page</a>, while the free version can be found on the <a href="/latest">free download page</a>.</li> 
       <li><a href="http://jquery.com/">jQuery</a> v1.7.2+ is required for <a href="/docs/setup.html#videotag-install">VIDEO tag based installations</a>. Make sure to load only one version of the library when integrating Flowplayer in an existing page.</li> 
       <li><a href="/docs/skinning.html">Alternate skins</a> are available.</li> 
       <li>In general it is recommended to load CSS stylesheets before JavaScript assets.</li> 
       <li>It is not recommended to load the Flowplayer and jQuery Javascripts in the BODY or at the end of the document.</li> 
      </ul> 
     </section>
     <section class="level1 has5" id="section_installation">
      <h1 id="installation">Installation</h1> 
      <p>For each player you need to prepare a html <a href="http://www.w3schools.com/tags/tag_div.asp">DIV element</a> as player <strong>container</strong> inside the <a href="http://www.w3schools.com/tags/tag_body.asp">BODY</a> of your page.</p> 
      <ul class="hilite"> 
       <li><span class="label">Note</span> Sections exclusively dedicated to <a href="/docs/setup.html#videotag-install">VIDEO tag based installations</a> are flagged with a <span style="color:pink">pink border</span>, sections exclusively dedicated to <a href="/docs/setup.html#javascript-install">pure JavaScript installations</a> with a <span style="color:green">green border</span>.<br> Advanced opions and functions are colored <span class="hilite">red</span>.</li> 
      </ul> 
      <section class="level2" id="section_3-ways-to-install">
       <h2 id="3-ways-to-install">3 ways to install</h2> 
       <ul> 
        <li><a href="/docs/setup.html#automatic">automatic installation</a>: quick, no fuss, no scripting required, and yet entirely customizable, including complete API access if the need arises</li> 
        <li><a href="/docs/setup.html#manual-install">manual installation</a>: install on demand, individual customization both in HTML and JSON syntax</li> 
        <li><a href="/docs/setup.html#javascript-install">pure JavaScript installation</a>: complete and consistent control via JavaScript, no dependence on external libraries, advanced configuration and seamless API integration</li> 
       </ul> 
      </section>

      <section class="level2 has2" id="section_videotag-install">
       <h2 id="videotag-install">Video tag based</h2> 
       <p>These installation methods require the <a href="http://jquery.com/">jQuery library</a> to be <a href="#prerequisites">loaded</a>.</p> 
       <p>Always use SOURCE tags inside the VIDEO tag and specify the video <code>type</code>. Flowplayer will then tweak the VIDEO tag as needed by platform or browser. Or, for the <a href="#flash">Flash</a> engine, replace it with a Flash OBJECT.</p> 
       <p>Advantages of video tag based installations:</p> 
       <ul> 
        <li>The video is represented in the page source code in a <strong>static</strong> fashion. For you, as the page creator: <strong>what you see is what you mean</strong>.</li> 
        <li>For many users the easiest way to get started.</li> 
       </ul> 
       <p>Restrictions of video tag based installations:</p> 
       <ul> 
        <li>Fine-tuning the configuration on the <a href="/docs/setup.html#clip-options">clip</a> and <a href="/docs/setup.html#source-options">source</a> level is not possible or only in a restricted way for <a href="playlists.html">playlists</a>.</li> 
       </ul> 
       <section class="level3" id="section_automatic">
        <h3 id="automatic">Automatic installation</h3> 
        <p>Flowplayer will be installed automatically into each container element for which the magic CSS class <code>"flowplayer"</code> is specified: </p>
        <div class="codebox">
         <figure class="code">
          <div class="highlight">
           <pre><span></span><span class="p">&lt;</span><span class="nt">div</span> <span class="na">class</span><span class="o">=</span><span class="s">"flowplayer"</span><span class="p">&gt;</span><br>   <span class="p">&lt;</span><span class="nt">video</span><span class="p">&gt;</span><br>      <span class="p">&lt;</span><span class="nt">source</span> <span class="na">type</span><span class="o">=</span><span class="s">"application/x-mpegurl"</span> <span class="na">src</span><span class="o">=</span><span class="s">"//mydomain.com/video.m3u8"</span><span class="p">&gt;</span><br>      <span class="p">&lt;</span><span class="nt">source</span> <span class="na">type</span><span class="o">=</span><span class="s">"video/mp4"</span>             <span class="na">src</span><span class="o">=</span><span class="s">"//mydomain.com/video.mp4"</span><span class="p">&gt;</span><br>   <span class="p">&lt;/</span><span class="nt">video</span><span class="p">&gt;</span><br><span class="p">&lt;/</span><span class="nt">div</span><span class="p">&gt;</span><br></pre>
          </div>
          <br>
          <figcaption>
           HTML
          </figcaption>
         </figure>
        </div>
        <p></p> 
        <p>Advantages of the automatic installation method:</p> 
        <ul> 
         <li>no further scripting required</li> 
         <li>players can be individually customized even with this automated installation: <a href="/docs/setup.html#configuration-summary">all options on all levels</a> are also available in <a href="#configuration">HTML notation</a>.</li> 
        </ul> 
        <p>Restriction of the automatic installation method:</p> 
        <ul> 
         <li>will and can only be run <strong>once</strong> at page load.</li> 
        </ul> 
        <p>The automatic method is used in the <a href="quickstart.html">quick start guide</a> and an example for the automatic method is shipped with each Flowplayer distribution as <a href="//releases.flowplayer.org/7.1.2/index.html">minimal example setup</a>.</p> 
        <ul class="hilite"> 
         <li><span class="label">Recommendation</span> The automatic installation method is ideal to get started with Flowplayer, and for quick no-fuss results.</li> 
        </ul> 
        <p class="help closure"> <a href="/standalone/basics/minimal.html">view standalone page</a> </p> 
       </section>
       <section class="level3" id="section_manual">
        <h3 id="manual">Manual installation with jQuery</h3> 
        <p></p>
        <div class="codebox">
         <figure class="code">
          <div class="highlight">
           <pre><span></span><span class="p">&lt;</span><span class="nt">div</span> <span class="na">class</span><span class="o">=</span><span class="s">"player"</span><span class="p">&gt;</span><br>   <span class="p">&lt;</span><span class="nt">video</span><span class="p">&gt;</span><br>      <span class="p">&lt;</span><span class="nt">source</span> <span class="na">type</span><span class="o">=</span><span class="s">"application/x-mpegurl"</span> <span class="na">src</span><span class="o">=</span><span class="s">"//mydomain.com/video.m3u8"</span><span class="p">&gt;</span><br>      <span class="p">&lt;</span><span class="nt">source</span> <span class="na">type</span><span class="o">=</span><span class="s">"video/mp4"</span>             <span class="na">src</span><span class="o">=</span><span class="s">"//mydomain.com/video.mp4"</span><span class="p">&gt;</span><br>   <span class="p">&lt;/</span><span class="nt">video</span><span class="p">&gt;</span><br><span class="p">&lt;/</span><span class="nt">div</span><span class="p">&gt;</span><br>&nbsp;<br><span class="p">&lt;</span><span class="nt">script</span><span class="p">&gt;</span><br><span class="c1">// run script after document is ready</span><br><span class="nx">$</span><span class="p">(</span><span class="kd">function</span> <span class="p">()</span> <span class="p">{</span><br>&nbsp;<br>   <span class="c1">// install flowplayer into all elements with CSS class="player"</span><br>   <span class="nx">$</span><span class="p">(</span><span class="s2">".player"</span><span class="p">).</span><span class="nx">flowplayer</span><span class="p">();</span><br>&nbsp;<br><span class="p">});</span><br><span class="p">&lt;/</span><span class="nt">script</span><span class="p">&gt;</span><br></pre>
          </div>
          <br>
          <figcaption>
           HTML
          </figcaption>
         </figure>
        </div>
        <p></p> 
        <ul> 
         <li>The player is installed using the <code>flowplayer</code> jQuery plugin.</li> 
         <li>Container elements are targeted with a <a href="http://api.jquery.com/category/selectors/">jQuery selector</a>.</li> 
        </ul> 
        <p>Advantages of the manual method over <a href="/docs/setup.html#automatic">automatic installation</a>:</p> 
        <ul> 
         <li>you control where and <strong>when</strong> to install the player - not confined to page load</li> 
         <li>you can specify a <a href="/docs/setup.html#local-configuration">player specific configuration in JavaScript</a> as argument to the <code>flowplayer</code> jQuery extension.</li> 
        </ul> 
        <p>Advantage of the manual method over <a href="/docs/setup.html#javascript-install">pure JavaScript installation</a>:</p> 
        <ul> 
         <li>jQuery allows you to select several container elements as install targets in one <code>flowplayer</code> call. The pure method can target only one element at a time, as it also must configure the individual <a href="#clip-options">video clip</a>. 
          <div class="codebox">
           <figure class="code">
            <div class="highlight">
             <pre><span></span><span class="nx">$</span><span class="p">(</span><span class="s2">".player"</span><span class="p">).</span><span class="nx">flowplayer</span><span class="p">({</span><br>    <span class="c1">// configuration common to all players in</span><br>    <span class="c1">// containers with class="player" goes here</span><br><span class="p">});</span><br></pre>
            </div>
            <br>
            <figcaption>
             JavaScript
            </figcaption>
           </figure>
          </div></li> 
        </ul> 
        <ul class="hilite">

     <section class="level1 has5" id="section_video">
      <h1 id="video">Video</h1> 
      <p>The Flowplayer <a href="#engines">engines</a> can play a slew of <a href="/docs/setup.html#video-formats">video formats</a>: all videos which can be played by a <a href="#html-video">HTML5 video</a> tag or by <a href="#flash-video">Flash</a>.</p> 
      <p>However, one of the main purposes of using Flowplayer is to achieve cross browser and cross device compatibility, so your videos can be viewed in all browsers and on all mobile devices. To achieve that goal, the videos must meet the these criteria:</p> 
      <ol> 
       <li>They must be correctly encoded for the purpose. See the <a href="encoding.html">encoding docs</a> if you want to do this yourself - or use <a href="drive.html">Flowplayer Drive</a>.</li> 
       <li>Not every format can be played in every browser or on every device. The trick is to find a combination of formats which meets the cross compatibility criterion with the least amount of fuss.</li> 
      </ol> 
      <p>The second point will be covered in this section.</p> 
      <section class="level2" id="section_clip-and-sources">
       <h2 id="clip-and-sources">Clip and sources</h2> 
       <p><em>clip</em> is how we call the sum of everything related to one video content.</p> 
       <p><em>sources</em> is how we call the format variants of this video, i.e., technically differing representations of the same one content.</p> 
       <p>The audience will watch <strong>one</strong> clip - the content -, but to grant an optimal viewing experience the player should be provided with <strong>several</strong> sources to <a href="/docs/setup.html#picking-order">choose from</a>:</p> 
      </section>

      <section class="level2 has4" id="section_hls">
       <h2 id="hls">HLS</h2> 
       <p>HLS stands for <a href="https://developer.apple.com/library/ios/documentation/networkinginternet/conceptual/streamingmediaguide/Introduction/Introduction.html#//apple_ref/doc/uid/TP40008332-CH1-SW1">Apple HTTP Live Streaming</a>, but can just as well be used for video on demand. The source is a so-called playlist with filename suffix <code>m3u8</code>. It can deliver <a href="/demos/qsel/#drive-qsel-demo">multiple resolutions</a> of a video presented via Adaptive Bit Rate and resolution streaming (<a href="http://en.wikipedia.org/wiki/Adaptive_bitrate_streaming">ABR</a>). HLS ABR lets your audience enjoy your content at the best quality available with the current connection speed and device capabilities. In almost all scenarios it should be the preferred source and <a href="/docs/setup.html#picking-order">listed first</a>.</p> 
       <p>Flowplayer is shipped with the most complete HLS support: Out of the box it plays HLS whenever technically possible: Either if native HTML5 playback of HLS <a href="#html5-engine">is supported by the client</a> or if the <a href="#flash-hls">Flash plugin</a> in the browser is enabled.</p> 
       <ul class="hilite"> 
        <li><span class="label">important</span> Make sure to have a <a href="#cross-domain">crossdomain.xml</a> file in your media server root for Flash HLS.</li> 
       </ul> 
       <ul class="hilite"> 
        <li><span class="label">tip</span> The <a href="plugins.html#hlsjs">hlsjs plugin</a> enables HLS playback in modern browsers and devices without requiring Flash. Its use is strongly recommended</li> 
       </ul> 
       <section class="level3" id="section_cost-effective">
        <h3 id="cost-effective">Cost effective</h3> 
        <p>Its streaming feature makes HLS the most server resource friendly way to deploy video: Only what the viewers see is retrieved from the server. Therefore it is also the most cost effective format: <strong>What they see, is what you pay for.</strong> No large files are downloaded in the background. And the ABR mechanism ensures not only the best quality per bandwidth, but also an optimal quality/cost ratio.</p> 
       </section>
       <section class="level3" id="section_live-streaming">
        <h3 id="live-streaming">Live streaming</h3> 
        <p>HLS is the only cross-device and cross-browser compatible <a href="#recommended-format-offering">format</a> for <a href="http://demos.flowplayer.org/basics/live.html">live streams</a> and <a href="http://demos.flowplayer.org/basics/dvr.html">live DVR</a>. Other solutions do not cover all scenarios:</p> 
        <ul> 
         <li>RTMP delivery requires <a href="#flash-video">Flash</a> and does not support DVR</li> 
         <li><a href="plugins.html#dashjs">DASH</a> requires a modern browser with MediaSource support, and for lack of the latter also does not work on iOS</li> 
        </ul> 
        <p>Both of the above are supported by Flowplayer, but you would still need to offer HLS for cross-compatibility.</p> 
        <p>Flowplayer supports HLS everywhere:</p> 
        <ul> 
         <li>with the <a href="plugins.html#hlsjs">hlsjs plugin</a> in modern browsers</li> 
         <li>native playback in browsers which can play HLS</li> 
         <li>with its <a href="#flash-hls">Flash engine</a> in older browsers</li> 
        </ul> 
        <p>As Flash is on the decline, we recommend to make your <a href="plugins.html#hlsjs-stream-compatibility">streams</a> and <a href="plugins.html#hlsjs-server-side">server</a> ready for hlsjs.</p> 
        <p>Then simply switch the player to live mode by setting the <code>live</code> option for <a href="/docs/setup.html#player-options">player</a> or <a href="/docs/setup.html#clip-options">clip</a> to <code>true</code>.</p> 
       </section>
       <section class="level3" id="section_hls-quality-selection">
        <h3 id="hls-quality-selection">HLS quality selection</h3> 
        <p>Manual HLS quality selection is available out of the box via the HD menu if the stream is delivered from a root playlist pointing to several variants.</p> 
        <p><strong>Note:</strong> Native HLS playback - for example on iOS - does not allow manual HLS level selection. This feature is available with <a href="plugins.html#hlsjs">hlsjs</a> and <a href="#flash-hls">Flash HLS</a>.</p> 
        <p>By default all HLS levels are shown in the HD menu. Manual selection can be disabled or a specific set of HLS levels specified with the <code>hlsQualities</code> <a href="/docs/setup.html#player-options">option</a> on the <a href="/docs/setup.html#global-configuration">global</a>, <a href="/docs/setup.html#player-options">player</a>, and <a href="/docs/setup.html#clip-options">clip</a> level.</p> 
       </section>
       <section class="level3" id="section_drm">
        <h3 id="drm">DRM</h3> 
        <p>HLS offers the best <a href="https://developer.apple.com/library/ios/documentation/NetworkingInternet/Conceptual/StreamingMediaGuide/UsingHTTPLiveStreaming/UsingHTTPLiveStreaming.html#//apple_ref/doc/uid/TP40008332-CH102-SW15">content protection</a> in HTML5 video. Flowplayer supports encrypted HLS also in Flash - check out the <a href="https://flowplayer.blacktrash.org/hls-crypt">demo</a>.</p>

      <section class="level2" id="section_picking-order">
       <h2 id="picking-order">Picking order</h2> 
       <p>The order in which you specify the sources matters. Flowplayer will cycle through the sources and choose the first source it can play.</p> 
       <p>If a source can be played by more than one <a href="#engines">engine</a> Flowplayer will try the html5 engine first. The main candidates for this scenario are the HLS and MP4 <a href="/docs/setup.html#video-formats">formats</a>: If the browser <a href="#html5-video">cannot play</a> the format and its Flash plugin is enabled, then this source will be played using the flash engine. - Exception for convenience: If HTML5 video is not supported by the browser, HLS is not on offer, but an RTMP stream, the RTMP stream takes precedence over MP4 via HTTP.</p> 
       <p>You can completely control the picking order by setting the <code>engine</code> <a href="/docs/setup.html#source-options">source option</a> explicitly.</p> 
       <p>This is the recommended order to provide the sources by <code>type</code>:</p> 
       <ol> 
        <li><code>application/x-mpegurl</code> - HLS - Adaptive Bitrate Streaming allows optimal quality according to bandwidth and device compatibility; reduces <a href="#cdn">Content Delivery Network</a> costs as some browsers will keep downloading a video <strong>file</strong> even while playback is paused</li> 
        <li><code>video/webm</code> - WebM - before MP4 because otherwise it won't be chosen.</li> 
        <li><code>video/mp4</code> - also required as Flash fallback if neither HLS nor an RTMP stream is available; for mobile device compatibility refer to our <a href="#encoding.html#mp4-for-ios-and-mobile-devices">encoding documentation</a></li> 
        <li><code>video/flash</code> - MP4 over RTMP if HLS is not available - allows <a href="#random-seeking">random seeking</a> if html5 video is not available</li> 
       </ol> 
       <ul class="hilite"> 
        <li><span class="label">Important</span> Engine plugins like <a href="plugins.html#hlsjs">hlsjs</a> or <a href="plugins.html#dashjs">dashjs</a> are usually inserted at the top of the above order and will take precedence.</li> 
       </ul> 
      </section>
     </section>

      <p>This section covers how the player is presented on the page before playback starts.</p> 
      <section class="level2" id="section_player-size">
       <h2 id="player-size">Player size</h2> 
       <p>The size of the player is determined by the dimensions of the container element. For example you can hard code them using CSS directives: </p>
       <div class="codebox">
        <figure class="code">
         <div class="highlight">
          <pre><span></span><span class="p">.</span><span class="nc">flowplayer</span> <span class="p">{</span><br>   <span class="k">width</span><span class="p">:</span> <span class="mi">600</span><span class="kt">px</span><span class="p">;</span><br>   <span class="k">height</span><span class="p">:</span> <span class="mi">338</span><span class="kt">px</span><span class="p">;</span><br><span class="p">}</span><br></pre>
         </div>
         <br>
         <figcaption>
          CSS
         </figcaption>
        </figure>
       </div>
       <p></p> 
       <p>However, hard coding width and height of the container element foregoes Flowplayer's <strong>responsive design</strong>: If you do not specify fixed dimensions for the container like above, the player will adapt itself to the dimensions of the parent element according to its <code>ratio</code> setting (see below) - very useful to make the player look good on small mobile screens for example. Therefore it is recommended to either just omit dimension directives, or you can use more flexible CSS rules like <code>max-width</code> and/or <code>max-height</code>: </p>
       <div class="codebox">
        <figure class="code">
         <div class="highlight">
          <pre><span></span><span class="p">.</span><span class="nc">flowplayer</span> <span class="p">{</span><br>   <span class="k">max-width</span><span class="p">:</span> <span class="mi">800</span><span class="kt">px</span><span class="p">;</span><br><span class="p">}</span><br></pre>
         </div>
         <br>
         <figcaption>
          CSS
         </figcaption>
        </figure>
       </div>

The height of the player is determined by the <code>aspectRatio</code> or <code>ratio</code> <a href="#configuration">configuration settings</a> which default to <code>"16:9"</code> or <code>9/16</code> respectively. Under the premise that letterboxing or empty sidebars on the video screen are to be avoided, this assumes a video of 16/9 aspect ratio. Note that the <code>ratio</code> notation is the inverse ratio of how one usually specifies the aspect ratio of a video: height/width (container) as opposed to width/height (video).</p>

      <section class="level2 has2" id="section_start-screen">
       <h2 id="start-screen">Start screen</h2> 
       <p>In a basic setup like the one in our <a href="quickstart.html#flowplayer">quick start guide</a> the video is loaded and the first frame is shown - unless <code>autoplay</code> is <a href="/docs/setup.html#player-options">configured</a>.</p> 
       <p>In most cases however, you want to customize the initial screen as what is often called a splash or poster screen. Flowplayer offers two techniques to implement a start screen which are also named "poster" and "splash".</p> 
       <p>Common features of "poster" and "splash":</p> 
       <ul> 
        <li>The visible result: the player area is filled with an image or coloring, and in the middle sits the play button to advertise a video player and invite the audience to play the video.</li> 
        <li>The implementation method: entirely via CSS, <code>background-image</code> or a <code>background-color</code> or both, as directives for the container element - not mandatory for "splash" because it also changes the initial player behaviour, see below.</li> 
       </ul> 
       <p>Where "poster" and "splash" differ is their behaviour:</p> 
       <ul> 
        <li><a href="/docs/setup.html#poster">poster</a>: the video is loaded in the background</li> 
        <li><a href="/docs/setup.html#splash">splash</a>: the video is loaded on demand, i.e. when the user starts playback with a click</li> 
       </ul> 
       <ul class="hilite"> 
        <li><span class="label">Important</span> A splash or poster setup is mandatory if the player is <em>hidden</em> at some point in its life cycle. Some browsers forbid hiding the Flash object, and thus errors are encountered when the <a href="#engines">flash engine</a> is in use. This notably concerns <a href="/demos/layout/#overlay-demo">modal window setups</a>.</li> 
       </ul> 
       <section class="level3" id="section_poster">
        <h3 id="poster">Poster</h3> 
        <p>The poster setup is recommended when you want to preload the video at startup while giving the player a custom look and feel.</p> 
        <p>Flowplayer features an extended and flexible concept of the generic <code>poster</code> <a href="/docs/setup.html#video-tag-attributes">VIDEO tag attribute</a>: If the <code>poster</code> attribute is given, it will also be used as CSS background image of the container element. Par consequence you can also choose to omit the poster attribute, and instead specify a background image or background color (treated as monochrome "poster") CSS directive to the same effect.</p> 
        <p>The advantages of this approach:</p> 
        <ol> 
         <li>works consistently in Flash and HTML5</li> 
         <li>does not break when browser is resized (very hard to do in Flash)</li> 
         <li>you can take advantage of CSS techniques such as positioning, animations, scaling and many more</li> 
         <li>you can use CSS sprites and thereby avoid multiple server calls</li> 
         <li>you get rid of browser issues, for example Internet Explorer 9 loads the first frame of the video on top of the poster image</li> 
         <li>you can use higher resolution images with Retina and similar devices</li> 
        </ol> 
        <p>To sum up, a poster setup requires that</p> 
        <ul> 
         <li>a <code>background-image</code> or <code>background-color</code> CSS rule applies to the container element or</li> 
         <li>the <code>poster</code> <a href="/docs/setup.html#player-options">configuration option</a> is set or</li> 
         <li>the VIDEO tag carries a <code>poster</code> attribute and</li> 
         <li>the <a href="/docs/setup.html#splash">splash</a> setup is not enforced.</li> 
        </ul> 
        <p>If both <code>background-image</code> for the container and <code>poster</code> attribute for the VIDEO tag are given, the poster image takes precedence and replaces the background image.</p> 
        <p>When the above conditions are met, the CSS "is-poster" <a href="/docs/skinning.html#states">state class</a> is assigned to the container at startup. For instance this allows you to create a loading animation for the entire player area with a few CSS rules.</p> 
        <p class="help closure"> <a href="/standalone/basics/poster.html">view standalone page</a> </p> 
       </section>
       <section class="level3" id="section_splash">
        <h3 id="splash">Splash</h3> 
        <p>Flowplayer has a <em>unique</em> feature called "splash screen" which is similar to the poster setup except that the nested VIDEO or Flash OBJECT tag initially is not present on the page, but is inserted <strong>on demand</strong>. The player is installed on the fly when the user clicks on the splash screen. This has the following benefits:</p> 
        <ol> 
         <li>You can have an unlimited amount of players on the page and they - or rather their splash screens - all render immediately, even in Flash mode.</li> 
         <li>There are no hidden Flash objects which interfere with your scripting or CSS layout dynamics.</li> 
         <li>Only one video can be played at a time. When the user clicks on a splash screen while another player instance is running, the latter is unloaded automatically.</li> 
        </ol> 
        <p>By design the splash setup also disables preloading of the video and therefore autoplay.</p> 
        <p>Please have a look at the poster and splash <a href="/standalone/basics/multiple.html">comparison page</a>.</p> 
        <p>&nbsp;</p> 
        <p>To set up splash screens you either add the <a href="/docs/skinning.html#configurable-states">state class</a> "is-splash" to the CSS class of the container element: </p>
        <div class="codebox">
         <figure class="code">
          <div class="highlight">
           <pre><span></span><span class="p">&lt;</span><span class="nt">div</span> <span class="na">class</span><span class="o">=</span><span class="s">"flowplayer is-splash"</span><br>     <span class="na">style</span><span class="o">=</span><span class="s">"background-color:#777; background-image:url(/path/to/splash.jpg);"</span><span class="p">&gt;</span><br>   <span class="p">&lt;</span><span class="nt">video</span><span class="p">&gt;</span><br>      <span class="p">&lt;</span><span class="nt">source</span> <span class="na">type</span><span class="o">=</span><span class="s">"application/x-mpegurl"</span> <span class="na">src</span><span class="o">=</span><span class="s">"//mydomain.com/video.m3u8"</span><span class="p">&gt;</span><br>      <span class="p">&lt;</span><span class="nt">source</span> <span class="na">type</span><span class="o">=</span><span class="s">"video/mp4"</span>             <span class="na">src</span><span class="o">=</span><span class="s">"//mydomain.com/video.mp4"</span><span class="p">&gt;</span><br>   <span class="p">&lt;/</span><span class="nt">video</span><span class="p">&gt;</span><br><span class="p">&lt;/</span><span class="nt">div</span><span class="p">&gt;</span><br></pre>
          </div>
          <br>
          <figcaption>
           HTML
          </figcaption>
         </figure>
        </div>
        <p></p> 
        <p>Or you configure the addition of the "is-splash" state class by setting the <code>splash</code> <a href="/docs/setup.html#player-options">option</a> to <code>true</code> or to the location of the splash image. For instance this <a href="/docs/setup.html#global-configuration">global configuration</a> applies the splash setup to all players on the page: </p>
        <div class="codebox">
         <figure class="code">
          <div class="highlight">
           <pre><span></span><span class="nx">flowplayer</span><span class="p">.</span><span class="nx">conf</span> <span class="o">=</span> <span class="p">{</span><br>  <span class="nx">splash</span><span class="o">:</span> <span class="kc">true</span><br><span class="p">};</span><br></pre>
          </div>
          <br>
          <figcaption>
           JavaScript
          </figcaption>
         </figure>
        </div>
        <p></p> 
        <p>The splash image is given in the CSS <code>background-image</code> directive for the container element or by setting the <code>splash</code> option to the location of the image. However, providing an image is not mandatory to set up a Flowplayer splash screen; in fact there is no CSS or image requirement for the splash setup. Usually you want at least a <code>background-color</code> to discern the player from its surroundings.</p> 
        <p>Here is how it works:</p> 
        <ul> 
         <li><a href="/docs/setup.html#videotag-install">Video tag based installations</a>: Upon recognition of the "is-splash" class name or the <code>splash</code> option the VIDEO tag is temporarily removed from the container element on page load.</li> 
         <li>When the splash screen is clicked the VIDEO or an OBJECT tag, depending on the engine picked, is placed inside the player container and the "is-splash" CSS class name is removed.</li> 
         <li>Unloading does the opposite: the VIDEO or OBJECT tag is removed and the "is-splash" class is re-added.</li> 
         <li>The player can be made to go back to splash state by hitting the <code>q</code> <a href="#keyboard">key</a> or by calling the <code>unload</code> <a href="api.html#methods">API method</a>.</li> 
         <li>Adding the "is-closeable" <a href="/docs/skinning.html#configurable-states">state class</a> displays an unload button.</li> 
        </ul> 
        <p>Splash screens are used heavily in our <a href="/demos/">demo area</a>. They are one of the main reasons why people use Flowplayer.</p> 
        <ul class="hilite"> 
         <li><span class="label">Tip</span> The cleanest and fastest way to implement a splash setup is to use a <a href="#javascript-install">pure JavaScript installation</a>, and, if desired, to set the splash image via CSS.</li> 
        </ul> 
        <p class="help closure"> <a href="/standalone/basics/splash.html">view standalone page</a> </p> 
       </section>

      <section class="level2 has4" id="section_commercial-configuration">
       <h2 id="commercial-configuration">Commercial configuration</h2> 
       <p>The <a href="/pricing/player.html#players">commercial version</a> allows you to get rid of the Flowplayer logo and optionally use your own. By default this will be displayed in the bottom/left corner of the player in the <a href="/docs/sharing.html">shared</a> players. When a user clicks the logo [s]he will be redirected to the originating page or the configured link.</p> 
       <p>The license key is tied to the domain name shown in the browser's location bar - in the case of IFRAMEs when the frame is viewed in its own window, i.e., the IFRAME source domain.</p> 
       <section class="level3" id="section_commercial-prerequisites">
        <h3 id="commercial-prerequisites">Commercial prerequisites</h3> 
        <p>Make sure to deploy and load the <strong>commercial</strong> release available at your <a href="/account/">Flowplayer account</a>, or simply load the commercial API script from <code>//releases.flowplayer.org/7.1.2/commercial/flowplayer.min.js</code>.</p> 
        <p>The following assets <em>must</em> be commercial for license validation:</p> 
        <ul> 
         <li>the API script: <code>flowplayer.min.js</code></li>

         </figcaption>
        </figure>

      <section class="level2 has2" id="section_flash-engine">
       <h2 id="flash-engine">Flash engine</h2> 
       <p>The Flash engine is chosen to play a source if</p> 
       <ol> 
        <li>the tested video source cannot be played as HTML5 video by the browser and</li> 
        <li>the tested video type can be played by the Flash engine and</li> 
        <li>the <code>engine</code> <a href="/docs/setup.html#source-options">source option</a> for this source is not set to a value other than "flash" and</li> 
        <li>the Flash plugin is enabled in the browser</li> 
       </ol> 
       <p>or if</p> 
       <ol> 
        <li>the <code>engine</code> <a href="/docs/setup.html#source-options">source option</a> for this source is set to "flash" and</li> 
        <li>the Flash plugin is enabled in the browser</li> 
       </ol> 
       <p>The Flowplayer Flash component requires Flash version 9.0.0. Flash supports playback of the <a href="/docs/setup.html#video-formats">MP4 format</a> since version 9.0.115. Consider it safe to neglect offering a FLV source as fallback for ancient Flash.</p> 
       <p>The Flash engine supports playback of the following <a href="/docs/setup.html#video-formats">video formats</a>:</p> 
       <ul> 
        <li><a href="#flash-hls">HLS</a></li> 
        <li>MP4</li> 
        <li>FLV</li> 
       </ul> 
       <section class="level3" id="section_flash-hls">
        <h3 id="flash-hls">Flash HLS</h3> 
        <ul class="hilite"> 
         <li><span class="label">Important</span> The <a href="plugins.html#hlsjs">hlsjs plugin</a> enables HLS playback in modern browsers and devices without requiring Flash. Compared to Flash HLS it features better playback performance and is more resource friendly. - Flash HLS will still be used in legacy browsers.</li> 
        </ul> 
        <p>Flowplayer supports <a href="/docs/setup.html#video-formats">HLS</a> playback with its <a href="#flash-engine">Flash engine</a>. Adaptive Bit Rate (ABR) switching is available not only in browsers which can play HLS in HTML5 video, but also in all other browsers, noteably most desktop browsers.</p> 
        <p>For live streams this means that the same range of clients can be reached with just one source of type <code>application/x-mpegurl</code> as with a combination of HLS for HTML5 video and RTMP for Flash (where RTMP does not offer the benefits of ABR).</p> 
        <p>Advantages of Flash HLS over RTMP:</p> 
        <ul> 
         <li>ABR is available</li> 
         <li>no additonal Flash source required</li> 
         <li>no RTMP server required to offer <a href="#random-seeking">random seeking</a> via the Flash engine</li> 
        </ul> 
        <ul class="hilite"> 
         <li><span class="label">compatibility</span> Flash HLS does not support <a href="https://developer.apple.com/library/content/technotes/tn2288/_index.html#//apple_ref/doc/uid/DTS40012238-CH1-BYTE_RANGE_SUPPORT_FOR_SEGMENTS">#EXT-X-BYTERANGE</a> playlists (#EXT-X-VERSION:4).</li> 
        </ul> 
        <p>Rare scenarios where RTMP delivery for Flash might be preferrable:</p> 
        <ul> 
         <li>live streams where synchronicity with the live event is of crucial importance; live stream delivery often is done over RTMP first, which then has to be transformed to HLS causing a slight delay</li> 
         <li>very fine seeking granularity is of crucial importance to the setup - reason: Flash HLS seeks only to keyframe positions <a href="#flashls">by default</a> and setting <a href="#flashls">flashls</a> <code>seekmode: "ACCURATE"</code> doesn't yield the desired experience. - Even in this scenario RTMP should only be 3rd choice after HLS via the <a href="plugins.html#hlsjs">hlsjs plugin</a> and native playback.</li> 
        </ul> 
       </section>
       <section class="level3" id="section_flash-video-rtmp">
        <h3 id="flash-video-rtmp">Flash video (RTMP)</h3> 
        <p>Flowplayer supports a special <code>video/flash</code> source type to target video specifically for Flash. The type is mostly and best used to offer an RTMP stream as Flash source if a <a href="#flash-hls">HLS stream</a> is not available: </p>
        <div class="codebox">
         <figure class="code">
          <div class="highlight">
           <pre><span></span><span class="c">&lt;!-- flowplayer with RTMP configuration option --&gt;</span><br><span class="p">&lt;</span><span class="nt">div</span> <span class="na">class</span><span class="o">=</span><span class="s">"flowplayer"</span> <span class="na">data-rtmp</span><span class="o">=</span><span class="s">"rtmp://myrtmpserver.com"</span><span class="p">&gt;</span><br>&nbsp;<br>  <span class="p">&lt;</span><span class="nt">video</span><span class="p">&gt;</span><br>    <span class="c">&lt;!-- consumed by the html5 engine --&gt;</span><br>    <span class="p">&lt;</span><span class="nt">source</span> <span class="na">type</span><span class="o">=</span><span class="s">"video/webm"</span>  <span class="na">src</span><span class="o">=</span><span class="s">"//mydomain.com/video.webm"</span><span class="p">&gt;</span><br>    <span class="p">&lt;</span><span class="nt">source</span> <span class="na">type</span><span class="o">=</span><span class="s">"video/mp4"</span>   <span class="na">src</span><span class="o">=</span><span class="s">"//mydomain.com/video.mp4"</span><span class="p">&gt;</span><br>&nbsp;<br>    <span class="c">&lt;!-- consumed by the flash engine --&gt;</span><br>    <span class="p">&lt;</span><span class="nt">source</span> <span class="na">type</span><span class="o">=</span><span class="s">"video/flash"</span> <span class="na">src</span><span class="o">=</span><span class="s">"mp4:path/to/video.mp4"</span><span class="p">&gt;</span><br>  <span class="p">&lt;/</span><span class="nt">video</span><span class="p">&gt;</span><br>&nbsp;<br><span class="p">&lt;/</span><span class="nt">div</span><span class="p">&gt;</span><br></pre>
          </div>
          <br>
          <figcaption>
           HTML
          </figcaption>
         </figure>
        </div>
        <p></p> 
        <p>RTMP delivery in Flash allows <a href="#random-seeking">seeking to unbuffered positions</a> in the timeline.</p> 
        <p><em>Important:</em></p> 
        <ul> 
         <li>The <code>rtmp</code> <a href="/docs/setup.html#player-options">option</a> for the server address - also called 'net connection url' - <em>must</em> be set. Please consult the documentation of your server for its exact value.</li> 
         <li>The <code>src</code> must be the path on the server, not a full URL. For video on demand it is often prefixed with <code>mp4:</code> for MP4 videos. Again the server documentation is your friend.</li> 
        </ul> 
        <p>Specifying a <code>video/flash</code> source delivered via HTTP for progressive download is rarely useful:</p> 
        <ul> 
         <li>If the video is in <a href="#video-formats">MP4 format</a>, the source of type <code>video/mp4</code> will be be <a href="#picking-order">automatically chosen</a> as Flash fallback format if no HLS or RTMP source is available.</li> 
         <li>The <a href="#video-formats">FLV format</a> has a lower quality/bitrate ratio compared to MP4 and cannot be played in HTML5 video.</li> 
        </ul> 
       </section>

      <section class="level2" id="section_cross-domain">
       <h2 id="cross-domain">Cross domain</h2> 
       <p>To make HLS work in Flash Flowplayer's <a href="#flash-engine">Flash engine</a> must have explicit permission to access the transport streams. The deployment of a a <a href="http://www.adobe.com/devnet/articles/crossdomain_policy_file_spec.html">cross domain policy file</a> at the <em>root</em> of the domain from which the TS files are served is <em>obligatory</em>. It must allow requests from the location of <code>flowplayerhls.swf</code> (see the <code>swfHls</code> <a href="/docs/setup.html#player-options">configuration option</a>).</p> 
       <p>Simple example of a <code>crossdomain.xml</code> file giving full access to requests from anywhere: </p>
       <div class="codebox">
        <figure class="code">
         <div class="highlight">
          <pre><span></span><span class="cp">&lt;?xml version="1.0"?&gt;</span><br><span class="cp">&lt;!DOCTYPE cross-domain-policy SYSTEM</span><br><span class="cp">"http://www.adobe.com/xml/dtds/cross-domain-policy.dtd"&gt;</span><br><span class="nt">&lt;cross-domain-policy&gt;</span><br>  <span class="nt">&lt;allow-access-from</span> <span class="na">domain=</span><span class="s">"*"</span> <span class="nt">/&gt;</span><br><span class="nt">&lt;/cross-domain-policy&gt;</span><br></pre>
         </div>
         <br>
         <figcaption>
          XML
         </figcaption>
        </figure>
       </div>
       <p></p> 
       <p>For a very loose policy which also allows retrieving assets via https to a http site it would look like this: </p>
       <div class="codebox">
        <figure class="code">
         <div class="highlight">
          <pre><span></span><span class="cp">&lt;?xml version="1.0" encoding="UTF-8" ?&gt;</span><br><span class="cp">&lt;!DOCTYPE cross-domain-policy SYSTEM</span><br><span class="cp">"http://www.adobe.com/xml/dtds/cross-domain-policy.dtd"&gt;</span><br><span class="nt">&lt;cross-domain-policy&gt;</span><br>  <span class="nt">&lt;allow-access-from</span> <span class="na">domain=</span><span class="s">"*"</span> <span class="na">secure=</span><span class="s">"false"</span><span class="nt">/&gt;</span><br>  <span class="nt">&lt;site-control</span> <span class="na">permitted-cross-domain-policies=</span><span class="s">"all"</span><span class="nt">/&gt;</span><br><span class="nt">&lt;/cross-domain-policy&gt;</span><br></pre>
         </div>
         <br>
         <figcaption>
          XML
         </figcaption>
        </figure>
       </div>
       <p></p> 
       <p>Refer to the Adobe Flash documentation for a <a href="http://www.adobe.com/devnet-docs/acrobatetk/tools/AppSec/CrossDomain_PolicyFile_Specification.pdf">full specification</a>.</p> 
       <ul class="hilite"> 
        <li><span class="label">Important</span> With Flash HLS it is especially important to load the Flowplayer assets (<code>flowplayerhls.swf</code>) and the media via the same <a href="#https">protocol</a>.</li> 
       </ul> 
      </section>
      <section class="level2" id="section_redirecting">
       <h2 id="redirecting">Redirecting</h2> 
       <p>Avoid redirects in the video source URLs. Some browsers may not be able to handle them in HTML5 video.</p> 
       <p><a href="#flash-hls">Flash HLS</a> does not allow for <code>302</code> redirecting of relative paths in the master variant HLS playlist.</p> 
      </section>
      <section class="level2" id="section_random-seeking">
       <h2 id="random-seeking">Random seeking</h2> 
       <p>Audiences expect to be able to skip back and forth in a video without inconvenience. They expect a "streamed" video. With HTML5 video the server side requirements are now lowered dramatically for offering this feature.</p> 
       <ol> 
        <li><a href="#video-formats">HLS</a> is genuinely made to order for this purpose. As Flowplayer's <a href="#flash-engine">Flash engine</a> plays HLS out of the box, most scenarios are covered, adaptive bitrate streaming included. It is also the most <a href="#cost-effective">cost effective</a> format.</li> 
        <li>Almost all modern HTTP servers support <a href="http://en.wikipedia.org/wiki/Byte_serving">byte-range requests</a>. Thus they fulfill the only requirement to "stream" HTML5 video on demand as it can be viewed in partial downloads.</li> 
        <li>Flash is only capable of progressive download over HTTP. If you cannot offer a HLS stream, consider serving from an RTMP server for Flash delivery. You do not necessarily have to run your own RTMP server for this purpose. Most <a href="#cdn">content delivery networks</a> (CDNs) offer this alternative and other advantages, see below.</li> 
        <li>Note on <a href="/docs/encoding.html#mp4-and-metadata">MP4 metadata</a>: make sure the MOOV atom is at the beginning of the file, or some browsers will not start playback before downlaoding the full file.</li> 
       </ol> 
      </section>
      <section class="level2" id="section_cdn">
       <h2 id="cdn">CDN</h2> 
       <p>In the interest of your audience, and to lower your worries regarding maintenance, reliability, and delivery speed, consider serving your videos from a <a href="http://en.wikipedia.org/wiki/Content_delivery_network">content delivery network</a>. The videos will be served from edge locations near your viewers, which can result in a noticeable speed gain, resulting in improved playback quality.</p> 
      </section>
     </section>

     <section class="level1" id="section_migration-from-version-5">
      <h1 id="migration-from-version-5">Migration from Version 5</h1> 
      <p>A lot of effort went into keeping the player compatible with existing setups. If you still encounter problems, please check also the migration notes in the <a href="skinning.html#migration-from-version-5">skinning</a> and <a href="api.html#migration-from-version-5">api</a> doc sections and turn to our <a href="/forum/">forum</a> for support.</p> 
      <p>Installation of a <code>playlist</code> of sources into an <a href="/v5docs/setup.html#empty-container">empty container element</a> has grown up to become a first class citizen in the form of the API friendly <a href="/docs/setup.html#javascript-install">JavaScript installation method</a> in Flowplayer 6 with an extensible <a href="#clip-object">clip object</a>. We recommend to embrace the more flexible full syntax which will give you even more control over each <a href="/docs/setup.html#clip-options">clip</a> and its <a href="/docs/setup.html#source-options">sources</a>.</p> 
      <p>Flowplayer 6 introduces a simpler source <a href="/docs/setup.html#picking-order">picking order</a>: Every source will be tried in the given order by all engines available. In the usual recommended setups this will not cause any difference in behaviour.</p> 
      <p>In the same vein, the <code>engine</code> option has moved from the <a href="/v5docs/setup.html#config-options">player level</a> to the <a href="/docs/setup.html#source-options">source level</a> to allow for complete fine-grained control of video source preferences.</p> 
      <p>Users of advanced RTMP options like <code>bufferTime</code>, <code>rtmpt</code>: The options will currently still work when set at player level directly. We recommend to start upgrading to the new <a href="#rtmp-options">rtmp configuration object</a> which is available both at <a href="/docs/setup.html#player-options">player</a> and <a href="/docs/setup.html#clip-options">clip</a> level and thereby offers increased flexibility.</p>
