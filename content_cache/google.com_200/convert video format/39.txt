          <div class="title">
           Prerequisites
          </div> 
          <blockquote style="padding-left: 0;">

         <div class="toc"> 
          <h3 class="title">Overview</h3> 
          <ul class="sectlevel2"> 
           <li><a href="#_screencast_the_neo4j_browser">Screencast: The Neo4j Browser</a></li> 
           <li><a href="#_presentation_svg_based_graph_interaction">Presentation: SVG-Based Graph Interaction</a></li> 
           <li><a href="#_library_alchemy_js_open_source_graph_visualization">Library: Alchemy.js Open Source Graph Visualization</a></li> 
           <li><a href="#_product_linkurious_graph_visualization">Product: Linkurious Graph Visualization</a></li> 
           <li><a href="#_product_tom_sawyer_perspectives">Product: Tom Sawyer Perspectives</a></li> 
           <li><a href="#_product_keylines_neo4j_graph_visualization">Product: Keylines Neo4j Graph Visualization</a></li> 
           <li><a href="#_howto_graph_visualization_step_by_step">HowTo: Graph Visualization Step By Step</a></li> 
          </ul> 
         </div> 
         <hr> 
         <div class="paragraph"> 
          <p>Graph visualisations are a powerful tool to convey the content of a graph. They can highlight patterns, and show clusters and connections. There are many excellent options for graph visualization, such as <a href="http://d3js.org/">D3.js</a>, <a href="http://threejs.org/">three.js</a>, <a href="http://sigmajs.org/">sigma.js</a> and <a href="http://graphalchemist.github.io/Alchemy">Alchemy.js</a>. A few of the more popular graph-visualization methods are discussed below.</p> 
         </div> 
         <div class="paragraph"> 
          <p>In all cases, the visualizer request JSON data for all or part of the graph data then dynamically creates an in-memory JavaScript visualization on the client side.</p> 
         </div>

           <li> <a href="http://chris.neoxygen.io/articles/easy-graph-visualization-with-vis-dot-js.html">Graph Visualization with Viz.js by Chris Willemsen</a> </li> 
           <li> <a href="http://www.popotojs.com/">Popoto.js visualization based graph search toolkit</a> </li> 
           <li> <a href="http://www.coppelia.io/2014/07/an-a-to-z-of-extra-features-for-the-d3-force-layout/">Article: A-Z features for D3 force layout</a> </li> 
           <li> <a href="//twitter.com/hashtag/showmeyourgraph?f=realtime&amp;src=hash">ShowMeYourGraph Visualization Contest</a> </li>

       <div id="_screencast_the_neo4j_browser" class="sect2 columns medium-12" style="padding-left: 0;"> 
        <h3>Screencast: The Neo4j Browser</h3> 
        <div class="paragraph"> 
         <p>The default <a href="http://neo4j.com/download">Neo4j Server</a> has a powerful, customizable data visualization tool based on the built-in D3.js library.</p> 
        </div> 
        <iframe src="//player.vimeo.com/video/97204829" width="500" height="313" frameborder="0" webkitallowfullscreen mozallowfullscreen allowfullscreen></iframe> 
        <div class="paragraph"> 
         <p>In the screencast, we demonstrate how to style nodes and relationships in the Neo4j’s Browser visualization, and how to set colors, sizes, and titles. We then discuss the Graph-Style-Sheet (GRASS) and how you can download, update, and reset the styling information.</p> 
        </div> 
       </div> 
       <div id="_presentation_svg_based_graph_interaction" class="sect2 columns medium-12" style="padding-left: 0;"> 
        <h3>Presentation: SVG-Based Graph Interaction</h3> 
        <div class="paragraph"> 
         <p><span class="image"><a class="image" href="http://www.apcjones.com/talks/2014-03-26_Neo4j_London/#slide-0"><img src="//s3.amazonaws.com/dev.assets.neo4j.com/wp-content/uploads/20160629125527/seening_pattern_in_data.jpg" alt="seening pattern in data" width="500"></a></span></p> 
        </div> 
        <div class="paragraph"> 
         <p>In this talk, Alistair Jones builds a very powerful graph web editing tool based on SVG, suited for easy integration into modern HTML frameworks.</p> 
        </div> 
       </div> 
       <div id="_library_alchemy_js_open_source_graph_visualization" class="sect2 columns medium-12" style="padding-left: 0;"> 
        <h3>Library: Alchemy.js Open Source Graph Visualization</h3> 
        <div class="paragraph"> 
         <p>As you’ve just seen, preparing, converting and rendering graph data with plain D3 force layout involves a bit of work. The open-source <a href="http://graphalchemist.github.io/Alchemy">Alchemy.js library</a> replaces this effort with a few lines of configuration. It uses the <a href="http://graphalchemist.github.io/Alchemy/#/docs">GraphJSON</a> format for interaction which is easier to handle than the raw D3 JSON format. Alchemy.js is mostly configuration based but can also be customized with javascript functions. It supports custom styling, highlighting, hoover effects, clustering and interactive exploration.</p> 
        </div> 
        <div class="paragraph"> 
         <p>Here is a <a href="http://graphalchemist.github.io/Alchemy/#/examples">simple example</a> of using Alchemy.js to render a simple graph:</p> 
        </div> 
        <div class="listingblock"> 
         <div class="content code-panel" style="width: 100%; height: auto; border: medium 1px;"> 
          <pre style="font-size: 16px; font-weight: 100;" class="code programlisting cm-s-neo" data-lang="javascript"><code>&lt;script src="http://cdn.graphalchemist.com/alchemy.min.js"&gt;&lt;/script&gt;
&lt;script type="text/javascript"&gt;
  alchemy.begin({
        dataSource: "actors.json",
        nodeCaption: 'name',
        nodeMouseOver: 'name',
        cluster: true,
        clusterColours: ["#1B9E77","#D95F02","#7570B3","#E7298A","#66A61E","#E6AB02"]})
&lt;/script&gt;</code></pre> 
         </div> 
        </div>

       <div id="_product_linkurious_graph_visualization" class="sect2 columns medium-12" style="padding-left: 0;"> 
        <h3>Product: Linkurious Graph Visualization</h3> 
        <div class="imageblock"> 
         <p class="content"> <a class="image" href="http://linkurio.us/"><img src="//s3.amazonaws.com/dev.assets.neo4j.com/wp-content/uploads/20170613104633/Linkurious_visualization1-e1497376040398.png" alt="Linkurious visualization1 e1497376040398"></a> </p> 
        </div> 
        <div id="_product_linkurious_enterprise" class="sect3 "> 
         <h4>Product: Linkurious Enterprise</h4> 
         <div class="paragraph"> 
          <p><a href="http://linkurio.us/">Linkurious</a> is an on-premise graph visualization and analysis software that allows you to detect suspicious patterns, understand connections and discover new insights in your data.</p> 
         </div> 
         <div class="paragraph"> 
          <p>Linkurious Enterprise leverages Neo4j’s graph database technology to offer you an easy solution to search, visualize, detect patterns and edit graph data. You can easily search for properties, nodes and relationships thanks to a built-in search engine or monitor specific Cypher query alerts. Linkurious also optimizes multi-users collaboration and make sharing and reporting available to teams. Linkurious is used in many different fields from fraud detection to enterprise architecture management or IT management. The solution provides a security framework compatible with the security requirements of Fortune 500 companies and government agencies.</p> 
         </div> 
        </div> 
        <div id="_product_linkurious_sdk" class="sect3 "> 
         <h4>Product: Linkurious SDK</h4> 
         <div class="paragraph"> 
          <p><a href="http://linkurio.us/product/#sdk">The Linkurious SDK</a> is a powerful toolkit to build graph visualization and analysis applications or add graph features to existing apps. The Linkurious Server REST API provides access to high-level services on top of Neo4j including data access (full text search or graph query), authentication and access rights management, alert management or visualization management (create, share, collaborate). The Ogma JavaScript graph visualizations library provides the ability to build interactive scalable graph visualization user interfaces. Display information with various graph layouts (Grid, Concentric, ForceLink, Continuous or Hierarchical) for a fast understanding of the data or customize the style (colors, icons, size, shape) to obtain easy-to-read graph visualizations.</p> 
         </div> 
        </div> 
       </div> 
       <div id="_product_tom_sawyer_perspectives" class="sect2 columns medium-12" style="padding-left: 0;"> 
        <h3>Product: Tom Sawyer Perspectives</h3> 
        <iframe src="//player.vimeo.com/video/90897466" width="500" height="281" frameborder="0" webkitallowfullscreen mozallowfullscreen allowfullscreen></iframe> 
        <div class="paragraph"> 
         <p><a href="//www.tomsawyer.com/pressreleases/pressrelease.php?release_id=164&amp;news=product">Tom Sawyer Perspectives 6.0</a> offers Cypher support for Neo4j graph databases and connects them to the broad visualization capabilities of the tool. Tom Sawyer Perspectives is integrated with the latest Neo4j version for easy visualization of complex results.</p> 
        </div> 
       </div> 
       <div id="_product_keylines_neo4j_graph_visualization" class="sect2 columns medium-12" style="padding-left: 0;"> 
        <h3>Product: Keylines Neo4j Graph Visualization</h3> 
        <div class="paragraph"> 
         <p><span class="image"><a class="image" href="http://cambridge-intelligence.com/keylines/neo4j"><img src="//s3.amazonaws.com/dev.assets.neo4j.com/wp-content/uploads/20160629100436/cambridge-intelligence-time-bar.png" alt="cambridge intelligence time bar" width="500"></a></span></p> 
        </div> 
        <div class="ulist"> 
         <ul> 
          <li> <a href="http://cambridge-intelligence.com/visualize-neo4j-time-graph">How to Visualize Time-Based Graphs with Neo4j</a> </li> 
         </ul> 
        </div> 
        <div class="paragraph"> 
         <p>KeyLines is an out-of-the-box JavaScript solution for visualizing networks. It works in all major browsers and all platforms, including the iPad.</p> 
        </div> 
        <div class="paragraph"> 
         <p>KeyLines integrates into existing web applications easily and with very little effort. How and where you get the data is up to you. KeyLines does the job of rendering it and responding to user interactions like clicking, touching, moving nodes, and more. You bind to these events to customize what happens, and, most importantly, your data stays under your control at all times: KeyLines is self-contained and needs no external connections.</p> 
        </div> 
        <iframe width="560" height="315" src="//www.youtube.com/embed/NcYJhzFKyfw?rel=0" frameborder="0" allowfullscreen></iframe> 
       </div> 
       <div id="_howto_graph_visualization_step_by_step" class="sect2 columns medium-12" style="padding-left: 0;"> 
        <h3>HowTo: Graph Visualization Step By Step</h3> 
        <div class="paragraph"> 
         <p>If you want to get started directly, here are the few steps you have to take to convert Neo4j’s graph query results into the format typically needed by the visualization libraries. This approach is also used in the <a href="../example-project">example project</a> which we implemented for each of the <a href="../language-guides">languages and drivers</a> listed. You can find the appropriate transformation code for your language and stack there.</p> 
        </div> 
        <div id="_target_json_datastructure" class="sect3 "> 
         <h4>Target JSON Datastructure</h4> 
         <div class="paragraph"> 
          <p>They visualization toolkits usually <strong>expect a JSON structure</strong> of node objects (their id, label(s) and properties) and a list of relationships (their properties, start- and end-node-id (or node-array-index for d3) and optionally relationship-id) like this. Here we see 3 nodes (2 people, one database) and 3 relationships in that datastructure.</p> 
         </div> 
         <div class="listingblock"> 
          <div class="content code-panel" style="width: 100%; height: auto; border: medium 1px;"> 
           <pre style="font-size: 16px; font-weight: 100;" class="code programlisting cm-s-neo" data-lang="javascript"><code>{"nodes":[{name:"Peter",label:"Person",id:1},{name:"Michael",label:"Person",id:2},
          {name:"Neo4j",label:"Database",id:3}],
 "links":[{source:0, target:1, type:"KNOWS", since:2010},{source:0, target:2, type:"FOUNDED"},
          {source:1, target:2, type:"WORKS_ON"}]}</code></pre> 
          </div> 
         </div> 
        </div> 
        <div id="_graph_rendering_javascript_code" class="sect3 "> 
         <h4>Graph Rendering Javascript Code</h4> 
         <div class="paragraph"> 
          <p>To render this list with D3, you just need the d3.js library dependency and a few lines of javascript.</p> 
         </div> 
         <div class="listingblock"> 
          <div class="content code-panel" style="width: 100%; height: auto; border: medium 1px;"> 
           <pre style="font-size: 16px; font-weight: 100;" class="code programlisting cm-s-neo" data-lang="javascript"><code>  var width = 800, height = 800;
  // force layout setup
  var force = d3.layout.force()
          .charge(-200).linkDistance(30).size([width, height]);

  // setup svg div
  var svg = d3.select("#graph").append("svg")
          .attr("width", "100%").attr("height", "100%")
          .attr("pointer-events", "all");

  // load graph (nodes,links) json from /graph endpoint
  d3.json("/graph", function(error, graph) {
if (error) return;

      force.nodes(graph.nodes).links(graph.links).start();

      // render relationships as lines
      var link = svg.selectAll(".link")
              .data(graph.links).enter()
              .append("line").attr("class", "link");

      // render nodes as circles, css-class from label
      var node = svg.selectAll(".node")
              .data(graph.nodes).enter()
              .append("circle")
              .attr("class", function (d) { return "node "+d.label })
              .attr("r", 10)
              .call(force.drag);

      // html title attribute for title node-attribute
      node.append("title")
              .text(function (d) { return d.title; })

      // force feed algo ticks for coordinate computation
      force.on("tick", function() {
          link.attr("x1", function(d) { return d.source.x; })
                  .attr("y1", function(d) { return d.source.y; })
                  .attr("x2", function(d) { return d.target.x; })
                  .attr("y2", function(d) { return d.target.y; });

          node.attr("cx", function(d) { return d.x; })
                  .attr("cy", function(d) { return d.y; });
      });
  });</code></pre> 
          </div> 
         </div> 
        </div> 
        <div id="_neo4j_query_result_format" class="sect3 "> 
         <h4>Neo4j Query Result Format</h4> 
         <div class="paragraph"> 
          <p>To generate the expected format from Neo4j you convert the results returned from the HTTP Cypher API endpoint. That endpoint already has an option (<code>{"resultDataContents":["graph"]}</code>) to return graph only results.</p> 
         </div> 
         <div class="paragraph"> 
          <p>For instance executing this request:</p> 
         </div> 
         <div class="listingblock"> 
          <div class="content code-panel" style="width: 100%; height: auto; border: medium 1px;"> 
           <pre style="font-size: 16px; font-weight: 100;" class="code programlisting cm-s-neo" data-lang="javascript"><code>:POST /db/data/transaction/commit
  {"statements":[{"statement":"MATCH path = (n)-[r]-&gt;(m) RETURN path",
                  "resultDataContents":["graph"]}]}</code></pre> 
          </div> 
         </div> 
         <div class="paragraph"> 
          <p>results in:</p> 
         </div> 
         <div class="listingblock"> 
          <div class="content code-panel" style="width: 100%; height: auto; border: medium 1px;"> 
           <pre style="font-size: 16px; font-weight: 100;" class="code programlisting cm-s-neo" data-lang="javascript"><code>var res  =
{ "results": [
    {
      "columns": ["path"],
      "data"   : [{
          "graph": {
            "nodes": [
              {"id": "1", "labels": ["Person"], "properties": {"name": "Peter"}},
              {"id": "2", "labels": ["Person"], "properties": {"name": "Michael"}}
             ],
            "relationships": [
              {"id": "0", "type": "KNOWS", "startNode": "1", "endNode": "2", "properties": {}}
             ]
          } // , {"graph": ...}, ...
      }]}
  ], "errors": []
}</code></pre> 
          </div> 
         </div> 
        </div> 
        <div id="_converting_neo4j_query_results_to_d3_json" class="sect3 "> 
         <h4>Converting Neo4j Query Results to D3 JSON</h4> 
         <div class="paragraph"> 
          <p>Which can be converted to our visualization format by collecting the nodes and relationships of the different rows into two lists. Just make sure that each node is unique in the list and that the relationships refer to the array-index (not the id) of the node.</p> 
         </div> 
         <div class="listingblock"> 
          <div class="content code-panel" style="width: 100%; height: auto; border: medium 1px;"> 
           <pre style="font-size: 16px; font-weight: 100;" class="code programlisting cm-s-neo" data-lang="javascript"><code>function idIndex(a,id) {
  for (var i=0;i&lt;a.length;i++) {
    if (a[i].id == id) return i;}
  return null;
}
var nodes=[], links=[];
res.results[0].data.forEach(function (row) {
   row.graph.nodes.forEach(function (n) {
     if (idIndex(nodes,n.id) == null)
       nodes.push({id:n.id,label:n.labels[0],title:n.properties.name});
   });
   links = links.concat( row.graph.relationships.map(function(r) {
     return {start:idIndex(nodes,r.startNode),end:idIndex(nodes,r.endNode),type:r.type};
   }));
});
viz = {nodes:nodes, links:links};</code></pre> 
          </div> 
         </div>
