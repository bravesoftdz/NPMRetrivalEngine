    <h1>Create a Node.js Lambda function and upload it in a deployment package</h1> 
    <!--
		#######################################
		#
		#		TWO COLUMN LAYOUT
		#
		#######################################
		--> 
    <div id="articleColumn" class="span-9"> 
     <p>This topic describes how to create an AWS Lambda Node.js function that sends data to HTTP Event Collector (HEC) using Splunk logging for JavaScript. You write the function, integrating Splunk logging for JavaScript and any other libraries you need, and then package everything in a ZIP file and upload it to the AWS Lambda console. To instead use the built-in console editor (where you will not have access to Splunk logging for JavaScript), see <a href="/view/event-collector/SP-CAAAE6Y">Create a Node.js Lambda function in the Console editor</a>.</p> 
     <p>This documentation guides you through our simple Node.js Lambda example function in detail.</p> 
     <blockquote> 
      <strong>Note: </strong>Splunk strongly suggests you familiarize yourself with 
      <a href="http://docs.aws.amazon.com/lambda/latest/dg/programming-model.html"> Programming Model (Node.js)</a> in the 
      <em>AWS Lambda Developer Guide</em> before continuing.
     </blockquote> 
     <br> 
     <h2>Example contents</h2> 
     <p>The custom code that implements our example is contained within two JavaScript files: </p> 
     <ul> 
      <li><tt>index.js</tt> contains the event handler based on the skeleton from the previous section. This is the code that actually sends events to HTTP Event Collector.</li> 
      <li><tt>loggerFactory.js</tt> contains code that assembles the logger events to send to HEC. </li> 
     </ul> 
     <p>In addition to these JavaScript files, we also customized the <tt>package.json</tt> file, which contains configuration and dependency information for your project in JSON format.</p> 
     <p>Our dependencies, which we must package up with our code in a ZIP file for upload to the AWS Lambda console, include the following two packages:</p> 
     <ul> 
      <li><strong><a href="https://www.npmjs.com/package/bunyan" target="_blank">bunyan</a></strong>, which is a JSON logging library for Node.js.</li> 
      <li><strong> <a href="https://www.npmjs.com/package/splunk-bunyan-logger" target="_blank"> splunk-bunyan-logger</a></strong>, which is Splunk's complement to Bunyan, and includes Splunk logging for Java as its own dependency.</li> 
     </ul> 
     <p>For information about how to package up these components, see "<a href="/view/SP-CAAAE6Z#create">Create the deployment package</a>," later in this topic.</p> 
     <h2>Example code</h2> 
     <p>This section contains the contents of each custom file, plus an overview and explanation of each file.</p>

     <p>First, we create a <tt>config</tt> variable to store the configuration information for the Splunk Cloud server. The values are as follows:</p> 
     <ul> 
      <li><tt>token</tt>: The HTTP Event Collector token with which to authenticate. For more information, see "<a href="http://docs.splunk.com/Documentation/Splunk/latest/Data/UsetheHTTPEventCollector#Configure_the_HTTP_Event_Collector_in_Splunk_Web">Create an Event Collector token</a>" in the <em>Getting Data In Manual</em>.</li> 
      <li><tt>url</tt>: The host including port for your Splunk Cloud instance.</li> 
      <li><tt>path</tt>: The REST endpoint for HTTP Event Collector. Do not change this value.</li> 
      <li><tt>protocol</tt>: The protocol to use. This value will be either <tt>http</tt> or <tt>https</tt>.</li> 
      <li><tt>port</tt>: The HEC port. By default, this value is 8088, though you can change it.</li> 
      <li><tt>level</tt>: The logging level to use. This value can be one of the following: <tt>trace</tt>, <tt>debug</tt>, <tt>info</tt>, <tt>warn</tt> or <tt>error</tt>.</li> 
      <li><tt>autoFlush</tt>: Determines whether to send events immediately. We've set this to <tt>false</tt> here, so that events are batched</li> 
     </ul> 
     <p>Next, we create a <tt>logger</tt> variable, passing in <tt>config</tt> to the <tt>loggerFactory</tt>. This calls the <tt>create()</tt> function that is defined within <a href="/view/event-collector/SP-CAAAE6Z#_loggerFactory.js"><tt>loggerFactory.js</tt></a>.</p> 
     <p>After logging an information message, we declare the event handler function. This is the function that actually sends the event data. The <tt>event</tt> and <tt>context</tt> parameters are passed to the function, three key values are logged to the <tt>logger</tt> variable, and then the <tt>flush</tt> function is called, which sends the event data to HEC.</p>

     <p>The <tt>create</tt> function is where we create a logger and a data stream to HEC. Event data is sent over this stream using the information in the <tt>config</tt> variable that was declared in <tt>index.js</tt>. Recall that this information includes the hostname for the Splunk Enterprise server or Splunk Cloud, plus the HEC token and other settings. The <tt>create</tt> function returns a logger.</p>

     <pre>{
  "name": "bunyanLambda",

  "dependencies": {
    "bunyan": "*",
    "splunk-bunyan-logger": "0.8.0"

     <p>The <tt>package.json</tt> file contains configuration and dependency information for the project in JSON format. To find out what the meaning of each key is, and whether it is optional, see <a href="https://docs.npmjs.com/files/package.json"> Specifics of npm's package.json handling</a> on the npmjs website.</p> 
     <h2 id="create">Create the deployment package</h2> 
     <p>The deployment package is a ZIP file that includes both your custom JavaScript code and any dependencies that your code needs to run. </p> 
     <p>The root level of the ZIP file contains any custom JavaScript code you've written, plus a directory called <tt>node_modules</tt>. Inside the <tt>node_modules</tt> directory is all the dependencies your code needs. For example, our deployment ZIP looks like this:</p> 
     <pre>index.js
loggerFactory.js
package.json
node_modules/bunyan
node_modules/splunk-bunyan-logger</pre> 
     <p>When you compress your Lambda function into a deployment ZIP, if you've been storing the code and dependencies in a directory, be sure to compress the contents of the directory, and not the directory itself.</p> 
     <p>For more information, see <a href="http://docs.aws.amazon.com/lambda/latest/dg/nodejs-create-deployment-pkg.html"> Creating Deployment Package (Node.js)</a> in the <em>AWS Lambda Developer Guide</em>.</p> 
     <h2>Upload the deployment package</h2> 
     <p>When you've got your deployment ZIP, you're ready to upload it to AWS Lambda. </p> 
     <ul> 
      <li>Log into the AWS Lambda Console, and then click <strong>Create a Lambda function</strong>. </li> 
      <li>On the Select blueprint page, click <strong>Skip</strong>.</li> 
      <li>On the Configure function page, enter a name for the function. This should be the same value as the name key's value in the <tt>package.json</tt> file. Enter a short description next to <strong>Description</strong>, and leave the <strong>Runtime</strong> pop-up menu set to <strong>Node.js</strong>.</li> 
      <li>Under <strong>Lambda function code</strong>, choose upload a ZIP file, and then click the <strong>Upload </strong>button. Upload your deployment ZIP.</li> 
      <li>Under <strong>Lambda function handler and role</strong>, enter the module-name.export value. If you've left the handler name as <tt>exports.handler</tt>, then leave the value as <tt>index.handler</tt>. In the <strong>Role</strong> pop-up menu, choose or create a role (likely <strong>lambda_basic_execution</strong>). </li> 
      <li>Finally, leave the <strong>Advanced settings </strong>section untouched, and then click <strong>Next</strong>.</li> 
      <li>On the <strong>Review</strong> page, review your settings. If you have to make any changes, click the <strong>Edit</strong> button. When you're done, click <strong>Create function</strong>.</li> 
     </ul> 
     <h2>Test the Lambda function</h2> 
     <p>To test your new Lambda function, click the <strong>Test</strong> button.</p> 
     <ul> 
      <li>You'll need to use sample event data for this function. Choose the "Hello World" sample event template from the <strong>Sample event template</strong> pop-up menu. You can change any of the values you want to in the edit window. Then click <strong>Submit</strong>. If the execution succeeded, you'll see a screen like the following:<br> <img alt="Screen shot showing successful test event transmission." src="http://coverall1.splunk.com/web_assets/developers/httpec/4_eventssent.png" width="700"></li> 
      <li>Now, log into Splunk Cloud and search for the data. For example, if you created a new token with default settings just for this walkthrough, search for source = &lt;<em>token_name</em>&gt;, where &lt;<em>token_name</em>&gt; is the name you gave the token when you created it. You'll see something like the following:<br> <img alt="Screen shot of Splunk Enterprise search app showing search results matching the events that were transmitted by AWS Lambda." src="http://coverall1.splunk.com/web_assets/developers/httpec/5_splunkresults.png" width="700pt"> </li> 
     </ul> 
     <p>Each event has been logged.</p>
