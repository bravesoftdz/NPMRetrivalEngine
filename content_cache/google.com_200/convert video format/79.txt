      <section class="blurb"> 
       <h1 class="teaser title">Basics</h1> 
       <p class="teaser text"> </p> 
       <p>With Flowplayer anyone can play videos no matter what their background is. We assume you want to play videos to users or visitors of your web site. If you only want to play videos locally, see the end of this document where <a href="#localinstall">local installation</a> is&nbsp;discussed.</p> 
       <p>The following steps are needed to display videos on your web&nbsp;site:</p> 
       <ul> 
        <li>Have a web server, or have your web site&nbsp;hosted</li> 
        <li>Place the Flowplayer distribution files on the&nbsp;server</li> 
        <li>Place your video files on the&nbsp;server</li> 
        <li>Include Flowplayer in your web&nbsp;pages</li> 
       </ul> 
       <h2 id="server">The web&nbsp;server</h2> 
       <p>The web server is a computer connected to the internet that handles requests for pages from the internet. In its simplest form it only sends pages that are stored as files on the hard disk of the web server. More complex web servers also run programs, such as a user registration application, that interacts with the web site’s&nbsp;users.</p> 
       <p>For Flowplayer you only need the simple form. If you have your web site hosted then this is important to know because there will be no extra costs for installing and running programs on the web server. Most people and many smaller companies have their website hosted so it is always available and they don’t need to bother with system administration and&nbsp;operations.</p> 
       <p>Video is a serious bandwidth hogger. <em>Bandwidth</em> is the number of bits per second you send. High quality videos require many bits per second to be sent (full <span class="caps">HD</span> quality is 8 mega-bit per second) and many videos sent simultaneously also means high bandwidth is required. Not all hosting providers can guarantee they can always send enough bits. Together with delays elsewhere on the internet, your users might sometimes receive chunky video. If you have only occasional video in your web pages, then you will be fine with a simple (hosted) server and you can go to next step. But if you want to get serious about serving videos then <a href="cdn.html">here</a> are some interesting possibilities, which Flowplayer fully&nbsp;supports.</p> 
      </section> 
      <h2 id="flowplayer-distribution">Flowplayer distribution</h2> 
      <p>You need to <a href="http://releases.flowplayer.org/flowplayer/flowplayer-3.2.18.zip">download</a> the Flowplayer distribution and place the following files in the zip file on your web server (so not the zip file itself). Choose an easily remembered place as you will place a path to these files later in your web pages:</p> 
      <ul> 
       <li><code>flowplayer-3.2.18.swf</code>, the player.</li> 
       <li><code>flowplayer.controls-3.2.16.swf</code>, the control bar.</li> 
       <li><code>flowplayer-3.2.13.min.js</code>, the api.</li> 
      </ul> 
      <p>That is all. Flowplayer is now installed on your web server.</p> 
      <p><strong>Important:</strong> You <strong>must</strong> load all SWF files from the same domain, and, for security reasons, do not load them from <code>flowplayer.org</code>!</p> 
      <p>When you download Flowplayer, you must choose between the free and commercial versions. The free version always displays the Flowplayer logo; with the commercial version you can display your own logo and further brand the player. We suggest that you start with the free version because it contains all of the same features as the commercial one does.</p> 
      <p>After you have downloaded the package, you need to decompress it with any of several programs available, for example, <a href="http://www.7-zip.org">7-Zip</a>, <a href="http://www.rarsoft.com">WinRAR</a>, or <a href="http://www.winzip.com">WinZip</a>. Decompressing the file should result in a single folder called "flowplayer". The quickest way to test your installation is to copy this folder to your web server root and point your browser to the following location:</p> 
      <p><code>http://your.server.com/flowplayer/example/index.html</code></p> 
      <p>If everything is working properly, a video playback should start.</p> 
      <h2 id="file">Place video files on your web server</h2> 
      <p>We assume you have video files available already and you want to play them. To make them available, you must place them on your web site so your web pages can refer to them. How exactly to upload files to your web server depends on the server and the hosting company, but a common way is to use FTP (File Transfer Protocol) to transfer the video files from your home or office computer to the web server. There are many good and free FTP clients available. A nice one is <a href="http://winscp.net">WinSCP</a> that lets you drag and drop files from your computer to the server.</p> 
      <p>Note that Flowplayer does not support all video formats. Flowplayer essentially supports all formats supported by Adobe Flash, as Flowplayer is a manager for a basic Adobe Flash player. See <a href="formats.html">here</a> for a discussion of video formats and conversion.</p> 
      <h2 id="player">Include Flowplayer in your web pages</h2> 
      <p>To have your web pages play videos with Flowplayer, you must embed Flowplayer in those pages. For each page there are the following steps:</p> 
      <ol> 
       <li> <p>Include file <code>flowplayer-3.2.13.min.js</code> in the web page</p> </li> 
       <li> <p>Load the player with one JavaScript statement</p> </li> 
       <li> <p>Place the link to your video file in a <code>href</code></p> </li> 
      </ol> 
      <p><strong>Step 1:</strong> To include file <code>flowplayer-3.2.13.min.js</code> in the web page you place the following tag in the HEAD section of your web page: </p>

        </div>

      <p>This minimal way to show video is discussed in more detail in the <a href="/demos/installation/index.html">minimal installation demo</a>. The <a href="/demos/index.html">demos</a> page has many more examples of installing and customizing Flowplayer.</p> 
      <h2 id="whats-next">What's next?</h2> 
      <p>If you have installed Flowplayer and it is working, you may want to do more advanced things. Then there are the following topics for you to study:</p> 
      <ul> 
       <li> <p><strong>Configuration:</strong> All aspects of Flowplayer can be customized to your liking. The key to that is the configuration. In the examples on this page the configuration is not used; the default behavior is just fine. But you can provide a third argument to the <code>flowplayer()</code> function that can contain a lot of information to tell Flowplayer how to behave. With the configuration you can change the background color, have a different control bar, place a clock on the screen, have a splash image displayed, load playlists, have overlays and many more options. See the section on <a href="/documentation/configuration/index.html">configuration</a> for that and see the <a href="/demos/index.html">demos</a>.</p> </li> 
       <li> <p><strong>JavaScript API:</strong> With the configuration you can customize Flowplayer to your liking, but that is <em>static</em>, that is, you cannot change it after the player has been loaded. If you want to control the behavior of Flowplayer after it has been loaded, you must use the functions of the JavaScript API. For example, in interactive learning you can tell the video to pause at a specific point and launch a question page or information page. The use of the API by end users (you) is simple in nature and you only require a basic JavaScript knowledge. See the <a href="/documentation/api/index.html">API documentation</a>.</p> </li> 
      </ul> 
      <h2 id="localinstall">Local Installation</h2> 
      <p>A <em>local installation</em> is where you run Flowplayer and your videos from your local hard disk.</p> 
      <p>The local installation works just as the installation for a web server; you place the distribution files on your hard disk, have your video files and embed the player in a web page from where you show your local videos.</p> 
      <p>There is just one thing different and that is the security settings of Flash. Normally Flash won't run videos from your hard disk and won't run the Flowplayer player object from your hard disk. So you must tell Flash it can trust your video location and trust the Flowplayer object. You do that using the <a href="http://www.adobe.com/support/documentation/en/flashplayer/help/settings_manager04.html">Flash security settings manager</a>, where your security settings are displayed. You can change them to allow the locations on your hard disk. (Note: your security settings are stored local by Flash and are not stored at the Adobe or Macromedia website; it is just that the software to manage your settings run there.) In the panel you can select files or directories to trust. It is advised to allow the directory where your video files are stored and where the Flowplayer .swf objects are stored.</p>
