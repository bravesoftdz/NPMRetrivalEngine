       <h1 itemprop="name" class="devsite-page-title"> Event Tracking </h1> 
       <nav class="devsite-page-nav-embedded devsite-nav"></nav> 
       <div class="devsite-article-body clearfix
            " itemprop="articleBody"> 
        <p>This guide explains how to implement event tracking with analytics.js.</p> 
        <h2 id="overview">Overview</h2> 
        <p>Events are user interactions with content that can be tracked independently from a web page or a screen load. Downloads, mobile ad clicks, gadgets, Flash elements, AJAX embedded elements, and video plays are all examples of actions you might want to track as Events.</p> 
        <p>If you're unfamiliar with events in Google Analytics you should first read the article <a href="https://support.google.com/analytics/answer/1033068">About Events</a> in the <a href="https://support.google.com/analytics">Analytics Help Center</a>.</p> 
        <h2 id="implementation">Implementation</h2> 
        <p>Event hits can be sent using the <a href="https://developers.google.com/analytics/devguides/collection/analyticsjs/command-queue-reference#send"><code>send</code></a> command and specifying a <a href="https://developers.google.com/analytics/devguides/collection/analyticsjs/field-reference#hitType">hitType</a> of <code>event</code>. The <code>send</code> command has the following signature for the <code>event</code> hit type:</p> 
        <pre class="prettyprint">
ga('send', 'event', [<a href="https://developers.google.com/analytics/devguides/collection/analyticsjs/field-reference#eventCategory">eventCategory</a>], [<a href="https://developers.google.com/analytics/devguides/collection/analyticsjs/field-reference#eventAction">eventAction</a>], [<a href="https://developers.google.com/analytics/devguides/collection/analyticsjs/field-reference#eventLabel">eventLabel</a>], [<a href="https://developers.google.com/analytics/devguides/collection/analyticsjs/field-reference#eventValue">eventValue</a>], [fieldsObject]);
</pre> 
        <h3 id="event_fields">Event fields</h3> 
        <p>The following table summarizes the event fields:</p> 
        <table> 
         <tbody>
          <tr> 
           <th>Field Name</th> 
           <th>Value Type</th> 
           <th>Required</th> 
           <th>Description</th> 
          </tr> 
          <tr> 
           <td><a href="https://developers.google.com/analytics/devguides/collection/analyticsjs/field-reference#eventCategory"><code>eventCategory</code></a></td> 
           <td>text</td> 
           <td><strong>yes</strong></td> 
           <td>Typically the object that was interacted with (e.g. <code>'Video'</code>)</td> 
          </tr> 
          <tr> 
           <td><a href="https://developers.google.com/analytics/devguides/collection/analyticsjs/field-reference#eventAction"><code>eventAction</code></a></td> 
           <td>text</td> 
           <td><strong>yes</strong></td> 
           <td>The type of interaction (e.g. <code>'play'</code>)</td> 
          </tr> 
          <tr> 
           <td><a href="https://developers.google.com/analytics/devguides/collection/analyticsjs/field-reference#eventLabel"><code>eventLabel</code></a></td> 
           <td>text</td> 
           <td>no</td> 
           <td>Useful for categorizing events (e.g. <code>'Fall Campaign'</code>)</td> 
          </tr> 
          <tr> 
           <td><a href="https://developers.google.com/analytics/devguides/collection/analyticsjs/field-reference#eventValue"><code>eventValue</code></a></td>

           <td>A numeric value associated with the event (e.g. <code>42</code>)</td> 
          </tr> 
         </tbody>
        </table> 
        <p>For a more in-depth description of each of these fields, see <a href="https://support.google.com/analytics/answer/1033068#Anatomy">Anatomy of an Event</a> in the Analytics Help Center.</p> 
        <h2 id="examples">Examples:</h2> 
        <p>The following command sends an event to Google Analytics indicating that the fall campaign promotional video was played:</p> 
        <pre class="prettyprint notranslate" translate="no"><code>ga('send', 'event', 'Videos', 'play', 'Fall Campaign');
</code></pre> 
        <p>Note that as with all <code>send</code> commands, the fields passed in the convenience parameters may also be specified in the <code>fieldsObject</code>. The above command could be rewritten as:</p> 
        <pre class="prettyprint notranslate" translate="no"><code>ga('send', {
  hitType: 'event',
  eventCategory: 'Videos',
  eventAction: 'play',
  eventLabel: 'Fall Campaign'
});
</code></pre> 
        <aside class="special">
          For more details, examples, and best practices for sending hits, see the 
         <a href="https://developers.google.com/analytics/devguides/collection/analyticsjs/sending-hits">Sending Data to Google Analytics</a> guide. For more details on the calling signature of the 
         <code>send</code> command, see the 
         <a href="https://developers.google.com/analytics/devguides/collection/analyticsjs/command-queue-reference#send">command queue reference</a>. 
        </aside> 
        <h3 id="outbound_link_and_form_tracking">Outbound link and form tracking</h3> 
        <p>When a user clicks a link that points to another page on your site, that page typically sends a pageview hit as the user arrives. Because there's a series of pageviews, Google Analytics can figure out on the back end where the user navigated to (and from). But if a user clicks a link or submits a form to an external domain, that action is not captured unless you specifically tell Google Analytics what happened.</p> 
        <p>Outbound link and form tracking can be accomplished by sending events and specifying the destination URL in one of the event fields. The following event handler function can be used to send outbound link click events to Google Analytics:</p> 
        <pre class="prettyprint notranslate" translate="no"><code>function handleOutboundLinkClicks(event) {
  ga('send', 'event', {
    eventCategory: 'Outbound Link',
    eventAction: 'click',
    eventLabel: event.target.href
  });
}
</code></pre> 
        <p>Tracking outbound links and forms can be tricky because most browsers will stop executing JavaScript on the current page once a new page starts to load. One solution to this problem is to set the <a href="https://developers.google.com/analytics/devguides/collection/analyticsjs/field-reference#transport"><code>transport</code></a> field to <code>beacon</code>:</p> 
        <pre class="prettyprint">
function handleOutboundLinkClicks(event) {
  ga('send', 'event', {
    eventCategory: 'Outbound Link',
    eventAction: 'click',
    eventLabel: event.target.href,
    <strong>transport: 'beacon'</strong>
  });
}
</pre> 
        <p>For browsers that don't support the beacon transport method, you have to postpone navigating to the next page until the event has finished sending. The <a href="https://developers.google.com/analytics/devguides/collection/analyticsjs/sending-hits#knowing_when_the_hit_has_been_sent">Knowing when a hit has been sent</a> section of the guide on <a href="https://developers.google.com/analytics/devguides/collection/analyticsjs/sending-hits">Sending data to Google Analytics</a> explains how to do this in detail.</p> 
        <aside> 
         <strong>Note:</strong> Developers wanting to track outbound links and forms can use 
         <a href="https://github.com/googleanalytics/autotrack">autotrack</a>, which includes the 
         <a href="https://github.com/googleanalytics/autotrack#outboundlinktracker">outboundLinkTracker</a> and 
         <a href="https://github.com/googleanalytics/autotrack#outboundformtracker">outboundFormTracker</a> plugins that handle these complexities for you. See the 
         <a href="https://github.com/googleanalytics/autotrack">autotrack documentation</a> for usage and installation instructions. 
        </aside> 
        <h3 id="non-interaction_events">Non-interaction events</h3> 
        <p>In some cases you might want to send an event as a <a href="https://support.google.com/analytics/answer/1033068#NonInteractionEvents">non-interaction event</a>. To do this, specify the <a href="https://developers.google.com/analytics/devguides/collection/analyticsjs/field-reference#nonInteraction"><code>nonInteraction</code></a> field as <code>true</code> in the <code>fieldsObject</code> of the <code>send</code> command:</p> 
        <pre class="prettyprint notranslate" translate="no"><code>ga('send', 'event', 'Videos', 'play', 'Fall Campaign', {
  nonInteraction: true
});
</code></pre> 
        <p>For more information on non-interaction hits and when to use them, read about <a href="https://support.google.com/analytics/answer/1033068#NonInteractionEvents">non-interaction events</a> in the Analytics Help Center</p>
