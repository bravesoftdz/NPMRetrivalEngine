       <h2>Overview</h2> 
       <p><span class="wysiwyg-color-black">Follow this workflow to make your Google Web Designer HTML5 Standard and Polite ads Sizmek compatible. You will be creating a Google Web Designer (GWD) ad, and then injecting Sizmek specific code into it. We'll introduce the following Sizmek features:&nbsp;</span></p> 
       <p class="note"><strong>Note:</strong> If you just need to convert your existing project to a Sizmek ad, see <a href="#apdx_convert">Appendix: Convert from DoubleClick</a>.</p> 
       <table style="height: 214px;" width="778"> 
        <tbody> 
         <tr> 
          <td>Feature</td> 
          <td>HTML5 Standard Ad</td> 
          <td>HTML5 Polite Ad</td> 
          <td>Description</td> 
         </tr> 
         <tr> 
          <td>Clickthrough</td> 
          <td><img src="https://support.sizmek.com/hc/en-us/article_attachments/201874086/tick.gif" alt=""></td> 
          <td><img src="https://support.sizmek.com/hc/en-us/article_attachments/201874086/tick.gif" alt=""></td> 
          <td>In most cases, you'll use the Sizmek clickthrough feature that is part of Sizmek's HTML5 API. You'll use it via javascript calls like this: <pre><code class="javascript">EB.clickthrough()</code></pre> .<br> Similar to <a href="/hc/en-us/articles/206676476">ClickTag</a> support, the platform will read the value of calls to EB.clickthrough() to automatically detect the clickthrough value for the ad, and register that for you during ad upload. For more information, see <a href="/hc/en-us/articles/201128145--reference-Sizmek-HTML5-API" target="_blank" rel="noopener noreferrer">Sizmek HTML5 API</a>.</td> 
         </tr> 
         <tr> 
          <td>Custom Interactions</td> 
          <td><img src="https://support.sizmek.com/hc/en-us/article_attachments/201874106/cross_blue.jpg" alt=""></td> 
          <td><img src="https://support.sizmek.com/hc/en-us/article_attachments/201874086/tick.gif" alt=""></td> 
          <td>A Custom Interaction is a tracking statistic that you can define. When you create an ad, the platform reads the files in an ad's workspace and discovers the custom interactions you've used in calls to functions like <pre><code class="javascript">EB.clickthrough('custom_clickthrough_name')</code></pre> and <pre><code class="javascript">EB.userActionCounter('uac_name')</code></pre> When the ad is served, the platform tracks these events for eventual display in reports. For more information, see <a href="/hc/en-us/articles/201128145--reference-Sizmek-HTML5-API" target="_blank" rel="noopener noreferrer">Sizmek HTML5 API</a>.</td> 
         </tr> 
         <tr> 
          <td>Video Interactions</td> 
          <td><img src="https://support.sizmek.com/hc/en-us/article_attachments/201874106/cross_blue.jpg" alt=""></td> 
          <td><img src="https://support.sizmek.com/hc/en-us/article_attachments/201874086/tick.gif" alt=""></td> 
          <td>Your HTML5 videos can be fully tracked quite easily. We'll discuss how in the <a href="#apdx_video">adding video appendix</a> at the end of the guide. For more information, see <a href="/hc/en-us/articles/201128615--mdx2-howto-Use-the-HTML5-Video-Module-to-Track-Your-Video-Interactions" target="_blank" rel="noopener noreferrer">HTML5 Video Module</a>.</td> 
         </tr> 
        </tbody> 
       </table> 
       <p class="important"><span class="wysiwyg-color-black"><strong>Important:</strong> We've created a starter-kit project that illustrates use of these features.<br> You can download it here: <a href="/hc/en-us/article_attachments/115008068946/sizmek_gwd_starter_kit.zip">sizmek_gwd_starter_kit.zip</a></span></p> 
       <p>The workflow for making your GWD ad Sizmek compatible is:</p> 
       <ol> 
        <li>Create a Non Google ad in GWD.</li> 
        <li>Deploy Sizmek API reference and polite init code.</li> 
        <li>Build your ad.</li> 
        <li>Add clickthroughs, custom interactions, and video tracking.</li> 
        <li>Prepare and package the ad as a Sizmek Workspace, then upload it.</li> 
       </ol> 
       <p class="note"><strong>Note:</strong> For a deeper understanding of some of the code referenced read our buildguides for Sizmek HTML5 Ads available by contacting <a href="mailto:email: info@sizmek.com" target="_blank" rel="noopener noreferrer">info@sizmek.com</a>.</p> 
       <h2>Step 1: Create Ad in GWD</h2> 
       <h5>Procedure</h5> 
       <ol> 
        <li>Open GWD, then create an ad from scratch by selecting "File &gt; New"</li> 
        <li>Set your Ad Environment to "Non-Google Ad"</li> 
       </ol> 
       <p>&nbsp;<img src="/hc/article_attachments/115007580283/gwdnga.png" alt="gwdnga.png"></p> 
       <p>The rest of the properties are up to you.<br> &nbsp;</p>

       <h2>Step 3: Build your Ad</h2> 
       <p><img style="width: 800px; height: 441px;" src="https://support.sizmek.com/hc/article_attachments/115007483746/work.png" alt=""></p>

       <h2>Step 4: Add Click-through and/or Custom Interactions</h2> 
       <p class="note"><strong>Note:</strong> If you only need to track impressions and clicks, you may be interested in <a href="/hc/en-us/articles/206676476">ClickTags</a>.</p> 
       <h3>Clickthrough</h3> 
       <p class="note"><strong>Note:</strong> Clickthrough using HTML5 API is supported for both HTML5 Standard and Polite Banners.</p> 
       <p>To add a clickthrough, create an event:</p> 
       <ol> 
        <li> 
         <div>
          Expand Events section and click (+) button:
          <br> 
          <img class="screenshot" style="width: 323px; height: 183px;" src="https://support.sizmek.com/hc/article_attachments/115007478626/events.png" alt="">
         </div> </li> 
        <li> 
         <div>
          Step through the event wizard:
         </div> 
         <ul> 
          <li> 
           <div>
            Target: select the region on ad you'd like to listen for clicks
           </div> </li> 
          <li> 
           <div>
            Event: expand Mouse, then select "click"
           </div> </li> 
          <li> 
           <div>
            Action: select Custom, then select +Add Custom Action
           </div> </li> 
          <li> 
           <div>
            In the Custom Code dialog, type "clickthrough" or some other name in the function name text field (see image below).
            <br> Then, in the actual custom code, enter Sizmek EB.clickthrough or EB.userActionCounter:
           </div> </li> 
         </ul> <img src="/hc/article_attachments/115008159083/ctgwd.png" alt="ctgwd.png"></li> 
       </ol> 
       <div>
        &nbsp;
       </div>

       <h3>&nbsp;</h3> 
       <h3>Setting up Video Tracking</h3> 
       <div>
        1. If you plan to use HTML5 videos in your project, please make sure that the following line - Video Module is present just above the reference to our API. 
        <pre><code class="html">&lt;script type="text/javascript"&gt;
&nbsp; &nbsp; EBModulesToLoad = ['Video'];
&nbsp; &lt;/script&gt;
&lt;script type="text/javascript" src="EBLoader.js"&gt; &lt;/script&gt;</code></pre> This is further ellaborated on in 
        <a href="#video delivery">Step 2</a> of this guide.
        <br> 
        <br> 2. Next and final step is declaring a new instance of EBG.VideoModule when your ad is ready. To do this we must navigate to our 
        <strong>checkInit </strong>function - delpoyed in 
        <a href="#checkInit">Step 2</a>,
        <br> and edit 
        <strong>onInit</strong>() function declaration so that it includes this new video tracking instance.
        <br> &nbsp; 
        <ol style="list-style-type: lower-alpha;" start="1"> 
         <li>Find <strong>onInit </strong>function declaration in the code from Step 2.</li>

function onInit() {
gwdAd.initAd();
var videoOne = new EBG.VideoModule(document.getElementById('<strong style="color: red;">video1'</strong>));

       </div>

        3. Make sure that the id in the selector matches the video component instance ID and that you dont have multiple videos with same ID.

       </div> 
       <h2>Step 5: Prepare Workspace and Upload</h2> 
       <p class="important"><span class="wysiwyg-color-black">Important: A Workspace is a directory that contains an index.html file which has a reference to EBLoader.js in it.</span></p> 
       <h5>Procedure</h5> 
       <ol> 
        <li> 
         <div>
          Save your project.
         </div> </li> 
        <li> 
         <div>
          Publish your project. Be sure to choose "Publish Locally".
          <br> 
          <img class="screenshot" style="width: 320px; height: 166px;" src="https://support.sizmek.com/hc/article_attachments/115007580143/gwd_exp.jpg" alt="">
          <br> Make sure that all checkboxes are are checked, apart from &lt;u&gt;Minify which is left unchecked&lt;/u&gt;, then click Publish button.
          <br> 
          <img class="screenshot" style="width: 600px; height: 391px;" src="https://support.sizmek.com/hc/article_attachments/115007483626/publish.png" alt="">
         </div> </li> 
        <li> 
         <div>
          Now you should have an archive that is packaged, optimized and 
          <em>almost</em> ready for upload. It will be in your project's directory (unless you changed its destination location).
          <br> 
          <img class="screenshot" style="width: 373px; height: 441px;" src="https://support.sizmek.com/hc/article_attachments/115007547386/ws1.png" alt="">
         </div> </li> 
        <li> 
         <div>
          You'll need to copy EBLoader.js if you are using it referenced relatively to the workspace - see Step 2. 
          <a href="/hc/en-us/article_attachments/202189493/EBLoader.js">Download the EBLoader.js script here</a>, and copy it into your published archive..
         </div> </li> 
        <li> 
         <div>
          Add a backup image to the archive. All ads served on Sizmek must have a backup image within the workspace archive.
         </div> <p class="important"><span class="wysiwyg-color-black">The backup image <strong>must</strong> be the <strong>exact</strong> dimensions of your ad. Once uploaded to the platform, the dimensions of the ad are determined by the dimensions of the backup image.</span></p> 
         <div>
          Copy your backup image into your published archive.
          <br> Your published Ad's Archive's contents should now look like:
          <br> 
          <img class="screenshot" style="width: 98px; height: 101px;" src="https://support.sizmek.com/hc/article_attachments/115007547406/arch.png" alt="">
         </div> </li> 
        <li> 
         <div>
          That's it, you're done. Now, zip up the published directory and upload!
         </div> </li> 
       </ol> 
       <p class="note"><strong>Note: </strong> For detailed information on how to upload workspaces <a href="/hc/en-us/articles/200722459--overview-HTML5-Workspaces#ManagingWorkspaces" target="_blank" rel="noopener noreferrer">please refer to our help section here</a>.</p> 
       <div>
        &nbsp;
       </div> 
       <div>
        Once your ad is uploaded, preview it and double check that all the Interactions are tracked as you would expect.
       </div>

       <h2>Appendix: Optimizing Your Ad's File Size</h2> 
       <p>Sizmek gives you the option of serving libraries related to your ad from our servers. This can greatly reduce your ad's published size, saving you overhead and bandwidth. For detailed information on how to use these shared libraries, see&nbsp;<a href="/hc/en-us/articles/206566866#gwd">How To Use Sizmek's Shared Libraries to Shrink Your Ad's Size</a>.</p> 
       <p><a name="apdx_convert"></a></p> 
       <h2>Appendix: Need to Convert an Existing DoubleClick Project?</h2> 
       <p>The process for doing this can be summed up briefly as: Replace `doubleclick` scripts and tags with `genericad`.</p> 
       <h5>Procedure</h5> 
       <p>This can be accomplished by either creating a blank Non-Google Ad and borrowing the gwdgenericad_min.js file from it, or reopening the modified html once the changes are made. Here we'll illustrate the latter, manually modifying code:</p> 
       <ol> 
        <li> 
         <div>
          Switch to code view.
          <br> 
          <img class="screenshot" src="https://support.sizmek.com/hc/en-us/article_attachments/202131836/19_convert_codeview.png" alt="">
         </div> </li> 
        <li> 
         <div>
          Now, let's find all instances of the string "doubleclick". Click cmd+f (mac) or ctrl+f (windows), and in the search text field, enter "doubleclick".
          <br> 
          <img class="screenshot" src="https://support.sizmek.com/hc/en-us/article_attachments/202131956/20_search.png" alt="">
         </div> </li> 
        <li> 
         <div>
          Wherever you find an instance of that word, replace it with "genericad". A text editor like Sublime Text will make this easier:
          <br> 
          <img class="screenshot" src="https://support.sizmek.com/hc/en-us/article_attachments/202299193/21_findreplace.png" alt="">
         </div> </li> 
        <li> 
         <div>
          Remove reference to Enabler.js script:
          <br> 
          <img class="screenshot" src="https://support.sizmek.com/hc/en-us/article_attachments/202140106/23_enabler.png" alt="">
         </div> </li> 
        <li> 
         <div>
          Now close the file, then reopen it. When it gets reopened, GWD will generate the necessary .js files for you. However, you'll need to handle the .css files manually. Go ahead and replace the word "doubleclick" with "genericad" in files with the .css extension:
          <br> 
          <img class="screenshot" src="https://support.sizmek.com/hc/en-us/article_attachments/202132396/22_rename.png" alt="">
         </div> </li> 
        <li> 
         <div>
          Finally, 
          <a href="#step3">head back up to step 4 </a>, then follow the process from there (add clickthrough and Sizmek delivery system).
         </div> </li> 
       </ol>
