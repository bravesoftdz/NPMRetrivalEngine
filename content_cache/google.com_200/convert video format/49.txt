       <h2 class="entry-title"><a href="https://blog.jetbrains.com/webstorm/2017/09/debugging-node-js-apps-in-webstorm/" rel="bookmark">Debugging Node.js apps in WebStorm</a></h2> 
       <div class="entry-meta"> 
        <span class="meta-prep meta-prep-author"> Posted on </span> 
        <a href="https://blog.jetbrains.com/webstorm/2017/09/debugging-node-js-apps-in-webstorm/" title="12:35 pm" rel="bookmark"><span class="entry-date">September 22, 2017</span></a> 
        <span class="meta-sep">by</span> 
        <span class="author vcard"><a class="url fn n" href="https://blog.jetbrains.com/webstorm/author/ekaterina-prigara/" title="View all posts by Ekaterina Prigara">Ekaterina Prigara</a></span> 
       </div>
       <!-- .entry-meta --> 
       <div class="entry-content"> 
        <p>WebStorm makes it easier to debug Node.js apps. You can put breakpoints right in your source code (no more <code>debugger</code> and <code>console.log()</code> statements!), which can be written in JavaScript or TypeScript.</p> 
        <p>You can do many things that will help you explore the code and understand where the bug is. In the debugging tool window, you can see the call stack and the variables in their current state, evaluate expressions in the editor, and step through the code. And you can start debugging your app very quickly. Here’s how.</p> 
        <p>We’ll use a simple <a href="http://expressjs.com/">Express app</a> as an example.</p> 
        <p>We need to do two things: first create a new Node.js run/debug configuration that will start the app, and then put a breakpoint.</p> 
        <ul>
         <li>To create a configuration, go to the <em>Run</em> menu and select <em>Edit configurations…</em> Click the <em>+</em> icon and select Node.js configuration type.</li> 
         <li>Now specify the path to the main file of the application that starts it. For the Express app it’s <em>bin/www</em></li> 
         <li>Save the configuration. Put the breakpoints in your code. Now press <em>Ctrl-D</em> on macOS or <em>Shift+F9</em> on Windows &amp; Linux to debug the app (or click the green bug-like icon).</li> 
        </ul>
        <p><img class="alignnone size-full wp-image-9245" src="https://d3nmt5vlzunoa1.cloudfront.net/webstorm/files/2017/09/debug-local.gif" alt="debug-local" width="600"></p> 
        <p>Now if you do the actions that trigger the code where the breakpoint is, the app will pause on the breakpoint. Explore the call stack and the variables in the debugging tool window, hover over the expressions in the editor to see their current state, step through the code, or put new breakpoints to help you debug further. <a href="https://blog.jetbrains.com/webstorm/2017/09/debugging-node-js-apps-in-webstorm/#more-9243" class="more-link">Continue reading <span class="meta-nav">?</span></a></p> 
       </div>

       <h2 class="entry-title"><a href="https://blog.jetbrains.com/webstorm/2017/09/webstorm-2017-3-eap-173-2463/" rel="bookmark">WebStorm 2017.3 EAP, 173.2463: multiple JavaScript versions in project, ESLint improvements</a></h2> 
       <div class="entry-meta"> 
        <span class="meta-prep meta-prep-author"> Posted on </span> 
        <a href="https://blog.jetbrains.com/webstorm/2017/09/webstorm-2017-3-eap-173-2463/" title="11:04 am" rel="bookmark"><span class="entry-date">September 21, 2017</span></a> 
        <span class="meta-sep">by</span> 
        <span class="author vcard"><a class="url fn n" href="https://blog.jetbrains.com/webstorm/author/ekaterina-prigara/" title="View all posts by Ekaterina Prigara">Ekaterina Prigara</a></span> 
       </div>
       <!-- .entry-meta --> 
       <div class="entry-content"> 
        <p>Please welcome the second <strong>EAP build of WebStorm 2017.3</strong> (173.2463.22).</p> 
        <p>To get it, update via <a href="https://www.jetbrains.com/toolbox/app/">Toolbox App</a>, or download it <a href="https://confluence.jetbrains.com/display/WI/WebStorm+EAP">here</a> and install side by side with your stable WebStorm version.</p> 
        <p align="center"><a class="jb-download-button" href="https://confluence.jetbrains.com/display/WI/WebStorm+EAP"><i class="download-icon"></i><br> Download WebStorm 2017.3 EAP</a></p> 
        <p>We started the WebStorm 2017.3 Early Access Program last week. If you’ve missed the announcement, check out this <a href="https://blog.jetbrains.com/webstorm/2017/09/webstorm-2017-3-eap/">blog post</a>.</p> 
        <p>Here are the new features you can try in this week’s EAP build (173.2463.22).</p> 
        <ul>
         <li><a href="#eslint-import">Advanced import of ESLint code style rules</a></li> 
         <li><a href="#emmet-in-jsx">New Emmet syntax for JSX</a></li> 
         <li><a href="#react-scripts">Support for script-versions in Create React App</a></li> 
         <li><a href="#multiple-js-versions">Using multiple JavaScript versions in a project</a></li> 
         <li><a href="#css-impovements">Improvements in CSS support</a></li> 
         <li><a href="#align-imports">Align ‘from’ in import statements</a></li> 
        </ul>
        <h2 id="eslint-import">Advanced import of ESLint code style rules</h2> 
        <p>Previously, in WebStorm 2017.2, you could only import code style rules that were explicitly written in ESLint JSON configuration files.</p> 
        <p>Now, it’s also possible to apply rules from more complex configuration files written in JavaScript and YAML, as well as from configurations that use <code>extends</code>. To do so, use the action <em>Apply ESLint code style rules</em>, which is available in the context menu of the config file.</p> 
        <p><img class="alignnone size-full wp-image-9223" src="https://d3nmt5vlzunoa1.cloudfront.net/webstorm/files/2017/09/eslint-apply.png" alt="eslint-apply" width="600"></p> 
        <p>WebStorm will send a special command to ESLint asking for a full summary of the applied rules in JSON. It will then parse the output and apply the rules to the IDE settings. You will see the list of applied rules in the IDE log.</p> 
        <p><img class="alignnone size-full wp-image-9224" src="https://d3nmt5vlzunoa1.cloudfront.net/webstorm/files/2017/09/eslint-log.png" alt="eslint-log" width="600"></p> 
        <p>Please note that this action requires ESLint to be installed in your project. It also means that the import won’t happen automatically, as it happens for the .eslintrc JSON configs.</p> 
        <h2 id="emmet-in-jsx">New Emmet syntax for JSX</h2> 
        <p>Now you can generate JSX tags with Emmet that will have text inside the tag or value in curly braces.</p> 
        <p><code>div{Title}</code> will expand into the tag with the content <em>Title</em>, and <code>div{{title}}</code> will expand into the tag with <em>title</em> in the curly braces.</p> 
        <p><img class="alignnone size-full wp-image-9225" src="https://d3nmt5vlzunoa1.cloudfront.net/webstorm/files/2017/09/emmet-in-jsx.gif" alt="emmet-in-jsx" width="600"></p> 
        <p>By the way, the <code>tag{}</code> syntax can also be very useful in HTML and other template files. <a href="https://blog.jetbrains.com/webstorm/2017/09/webstorm-2017-3-eap-173-2463/#more-9222" class="more-link">Continue reading <span class="meta-nav">?</span></a></p> 
       </div>

       <h2 class="entry-title"><a href="https://blog.jetbrains.com/webstorm/2017/09/webstorm-2017-3-eap/" rel="bookmark">WebStorm 2017.3 Early Access Preview: new and improved refactorings, better Vue.js support, new REST Client</a></h2> 
       <div class="entry-meta"> 
        <span class="meta-prep meta-prep-author"> Posted on </span> 
        <a href="https://blog.jetbrains.com/webstorm/2017/09/webstorm-2017-3-eap/" title="3:34 pm" rel="bookmark"><span class="entry-date">September 13, 2017</span></a> 
        <span class="meta-sep">by</span> 
        <span class="author vcard"><a class="url fn n" href="https://blog.jetbrains.com/webstorm/author/ekaterina-prigara/" title="View all posts by Ekaterina Prigara">Ekaterina Prigara</a></span> 
       </div>
       <!-- .entry-meta --> 
       <div class="entry-content"> 
        <p>Today we’re starting the Early Access Program for <strong>WebStorm 2017.3</strong>, the big update that will be released the end of November 2017.</p> 
        <p><a href="https://www.jetbrains.com/toolbox/app/">Toolbox App</a> is the easiest way to get the EAP builds. You can also subscribe to the EAP updates option in the IDE (<em>Preferences | Appearance &amp; Behavior | System Settings | Updates</em>) and get notifications when a new build is available.</p> 
        <p align="center"><a class="jb-download-button" href="https://confluence.jetbrains.com/display/WI/WebStorm+EAP"><i class="download-icon"></i><br> Download WebStorm 2017.3 EAP</a></p> 
        <p>If you’re not yet familiar with our Early Access Programs, here are some details:</p> 
        <ul>
         <li>Every week we publish a fresh build for the upcoming version of WebStorm.</li> 
         <li>EAP builds provide access to the newest features we’re currently developing.</li> 
         <li>EAP builds are free but expire 30 days after they’re built.</li> 
         <li>You can install an EAP build side-by-side with a stable WebStorm version.</li> 
         <li>We welcome your feedback in our issue tracker, youtrack.jetbrains.com/issues/WEB. Please don’t forget to mention the build number you’re using.</li> 
        </ul>
        <p><strong>Important: WebStorm EAP builds are not fully tested and may be unstable.</strong></p> 
        <p>WebStorm 2017.3 is going to bring many significant updates. Here are the highlights of what you can already try in this week’s EAP build (173.2290.2):</p> 
        <ul>
         <li><a href="#refactorings">Refactorings</a> – New Extract type alias refactoring in TypeScript, updated and improved Change signature and Extract parameter refactorings</li> 
         <li><a href="#vue">Improved Vue.js support</a></li> 
         <li><a href="#import-on-paste">Auto imports when you copy and paste from another file</a></li> 
         <li><a href="#rest-client">New REST Client</a></li> 
        </ul>
        <h2 id="refactorings">New and updated refactorings</h2> 
        <h3>Extract type alias in TypeScript</h3> 
        <p>The new <em>Extract type alias</em> refactoring allows you to create an alias for a complex type.</p> 
        <p>Put the cursor on the type and call the <em>Refactor this</em> popup (<em>Ctrl-T</em> on macOS / <em>Ctrl-Alt-Shift-T</em> on Windows &amp; Linux). Then select <em>Extract Type Alias</em> from the list and name the alias.</p> 
        <p><img class="alignnone size-full wp-image-9207" src="https://d3nmt5vlzunoa1.cloudfront.net/webstorm/files/2017/09/extract-type-alias.gif" alt="extract-type-alias" width="600"></p> 
        <p>And if the type is used multiple times, the IDE will spot this and suggest automatically replacing all the instances with the new name. <a href="https://blog.jetbrains.com/webstorm/2017/09/webstorm-2017-3-eap/#more-9205" class="more-link">Continue reading <span class="meta-nav">?</span></a></p> 
       </div>

       <h2 class="entry-title"><a href="https://blog.jetbrains.com/webstorm/2017/09/webstorm-2017-2-4/" rel="bookmark">WebStorm 2017.2.4 is now available</a></h2>

       <h2 class="entry-title"><a href="https://blog.jetbrains.com/webstorm/2017/08/how-to-configure-code-completion-in-full-stack-javascript-projects/" rel="bookmark">How to configure code completion in full-stack JavaScript projects?</a></h2> 
       <div class="entry-meta"> 
        <span class="meta-prep meta-prep-author"> Posted on </span> 
        <a href="https://blog.jetbrains.com/webstorm/2017/08/how-to-configure-code-completion-in-full-stack-javascript-projects/" title="1:15 pm" rel="bookmark"><span class="entry-date">August 7, 2017</span></a> 
        <span class="meta-sep">by</span> 
        <span class="author vcard"><a class="url fn n" href="https://blog.jetbrains.com/webstorm/author/ekaterina-prigara/" title="View all posts by Ekaterina Prigara">Ekaterina Prigara</a></span> 
       </div>
       <!-- .entry-meta --> 
       <div class="entry-content"> 
        <p>If you’re working on a full-stack JavaScript application in WebStorm, you’ve probably noticed that code completion suggests some Node.js APIs in your client-side code and DOM APIs in the Node.js code. Here’s how to fix that.</p> 
        <h2>What is Library in WebStorm?</h2> 
        <p>All the objects defined in your application are automatically added to the WebStorm completion suggestions, but for adding Node.js and DOM APIs as well as symbols from the project’s dependencies to the code completion WebStorm uses a mechanism called <em>External JavaScript Libraries</em> or just <em>Libraries</em>.</p> 
        <p>You can see a list of Libraries configured for your project in <em>Preferences | Languages &amp; Frameworks | JavaScript | Libraries</em>.</p> 
        <p>For every library, you can configure the scope where it works in the project. And that’s exactly what we need!</p> 
        <h2>Configuring HTML and Node.js Core Libraries</h2> 
        <p>The library called <em>HTML</em> contains all DOM APIs and by default is enabled for the whole project (hence the checked icon under Enabled). And so is the <em>Node.js Core</em> library with Node.js APIs (we enabled it before in <em>Preferences | Languages &amp; Frameworks | JavaScript | Node.js and npm</em>).</p> 
        <p><img class="alignnone size-full wp-image-9149" src="https://d3nmt5vlzunoa1.cloudfront.net/webstorm/files/2017/08/html-node-libs.png" alt="html-node-libs" width="600"></p> 
        <p>Uncheck <em>HTML</em> and <em>Node.js Core</em> libraries to disable them completely for now.</p> 
        <p>Now click on the <em>Manage Scopes…</em> button to start the configuration. Click + and select the folders with the client-side code – now select <em>HTML</em> library from the drop-down list next to them.</p> 
        <p>Click + again, select the folders or files with the server-side code and enable <em>Node.js Core</em> library for them.</p> 
        <p><img class="alignnone size-full wp-image-9150" src="https://d3nmt5vlzunoa1.cloudfront.net/webstorm/files/2017/08/manage-lib-scope.gif" alt="manage-lib-scope" width="600"></p> 
        <p>So now items from the <em>HTML</em> and <em>Node.js Core</em> libraries are resolved and shown in the code completion only in files in these specific project folders.</p> 
        <p>In the same way, we can configure the scope of the library that is automatically created for the <em>node_modules</em> folder.</p> 
        <p>For specific files, we can quickly manage the libraries with the <em>Use JavaScript Libraries</em> action in the file context menu in the editor. For example, we can enable <em>Node.js Core</em> library for our webpack configuration files.</p> 
        <p><img class="alignnone size-full wp-image-9151" src="https://d3nmt5vlzunoa1.cloudfront.net/webstorm/files/2017/08/enable-lib-for-file.png" alt="enable-lib-for-file" width="600"></p> 
        <p>By the way, the IDE inspections can be configured in a similar way using scopes in <em>Preferences | Editor | Inspections</em>.</p>
