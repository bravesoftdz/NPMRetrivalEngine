        <p>We’re happy to announce the second milestone release for Kotlin 1.2. The primary focus of this release is concerned with stability and bugfixes in Kotlin compiler and tooling as well as improvements of Kotlin standard library. It also includes many of the tooling features available in the upcoming release of Kotlin 1.1.4.</p> 
        <p>We appreciate your feedback regarding the new features or any problems that you may run into with this release.</p> 
        <p>The complete changelog since 1.2 M1 can be found <a href="https://github.com/JetBrains/kotlin/blob/1.2-M2/ChangeLog.md">here</a> with some major changes listed below.</p> 
        <p><span id="more-5160"></span></p> 
        <h2>Compiler</h2> 
        <h3>Breaking change: Java-default method calls</h3> 
        <p>Up to now Kotlin interface members overriding Java-default methods while targeting JVM 1.6 produced a warning on super calls: <code>Super calls to Java default methods are deprecated in JVM target 1.6. Recompile with '-jvm-target 1.8'</code>. In 1.2-M2 release we’ve replaced the warning with <strong>error</strong>, thus <strong>requiring</strong> any such code to be compiled with <code>-jvm-target 1.8</code> option</p> 
        <h2>Standard library</h2> 
        <h3>Revised windowed/pairwise operations</h3> 
        <p>We have been listening to your feedback about chunked/windowed/pairwise functions <a href="https://github.com/Kotlin/KEEP/blob/master/proposals/stdlib/window-sliding.md">KEEP-11</a>, which were released to preview in the previous milestone 1.2-M1, and based on it we decided to make some changes to these functions:</p> 
        <ul>
         <li><code>windowed</code> function now has its step parameter defaulting to 1.<br> It also gets an additional optional parameter <code>partialWindows</code>, which controls what to do with incomplete windows in the end. By default it is false, which means incomplete windows are dropped.</li> 
         <li><code>pairwise</code> function name was too confusing, it was unclear how it paired the elements together. Now it is called <code>zipWithNext</code>, so it is more clear that each element is zipped with the next element in a collection.</li> 
        </ul>
        <h3>Common math operations in the standard library</h3> 
        <p>There was a longstanding request to support math operations in the standard library <a href="https://youtrack.jetbrains.com/issue/KT-4900">KT-4900</a>. Until now one had to resort to math functions and constants from <code>java.lang.Math</code> class in JVM platform and to <code>kotlin.js.Math</code> which exposed native JS Math functions to Kotlin code in JS platform.<br> But now we’re introducing the following groups of API in the <code>kotlin.math</code> package:</p> 
        <ul>
         <li>constants: <code>PI</code> and <code>E</code>;</li> 
         <li>trigonometric: <code>cos</code>, <code>sin</code>, <code>tan</code> and inverse of them: <code>acos</code>, <code>asin</code>, <code>atan</code>, <code>atan2</code>;</li> 
         <li>hyperbolic: <code>cosh</code>, <code>sinh</code>, <code>tanh</code>;</li> 
         <li>exponentation: <code>pow</code> (an extension function), <code>sqrt</code>, <code>hypot</code>, <code>exp</code>, <code>expm1</code>;</li> 
         <li>logarithms: <code>log</code>, <code>log2</code>, <code>log10</code>, <code>ln</code>, <code>ln1p</code>;</li> 
         <li>rounding: 
          <ul>
           <li><code>ceil</code>, <code>floor</code>, <code>truncate</code>, <code>round</code> (half to even) functions;</li> 
           <li><code>roundToInt</code>, <code>roundToLong</code> (half to integer) extension functions;</li> 
          </ul></li> 
         <li>sign and absolute value: 
          <ul>
           <li><code>abs</code> and <code>sign</code> functions;</li> 
           <li><code>absoluteValue</code> and <code>sign</code> extension properties;</li> 
           <li><code>withSign</code> extension function;</li> 
          </ul></li> 
         <li><code>max</code> and <code>min</code> of two values;</li> 
         <li>binary representation: 
          <ul>
           <li><code>ulp</code> extension property;</li> 
           <li><code>nextUp</code>, <code>nextDown</code>, <code>nextTowards</code> extension functions;</li> 
           <li><code>toBits</code>, <code>toRawBits</code>, <code>Double.fromBits</code> (these are in the <code>kotlin</code> package).</li> 
          </ul></li> 
        </ul>
        <p>The same set of functions (but without constants) is also available for <code>Float</code> arguments.<br> Most of these functions (except the binary representation group) are also available in JS, thus solving the problem of writing the same calculations for both platforms.</p> 
        <h2>Pre-release notes</h2> 
        <blockquote>
         <p> As with other milestone releases, we give <b>no backwards compatibility guarantees</b> for new language and library features. Anything introduced in milestone releases of 1.2 is <b>subject to change </b>before the final 1.2 release. When we reach final RC, all binaries produced by pre-release versions will be outlawed by the compiler: you’ll be required to recompile everything that was compiled by 1.2?Mx.<br> However all the code compiled by 1.1.x and earlier releases is perfectly fine without recompilation. </p>
        </blockquote> 
        <h2>How to Try It</h2> 
        <p><strong>In Maven/Gradle</strong>: Add <code>http://dl.bintray.com/kotlin/kotlin-eap-1.2</code> as a repository for the build script and your projects; use <code>1.2-M2</code> as the version number for the compiler plugin and the standard library.</p> 
        <p><strong>In IntelliJ IDEA</strong>: Go to <em>Tools ? Kotlin ? Configure Kotlin Plugin Updates</em>, then select “Early Access Preview 1.2” in the <em>Update channel</em> drop-down list, then press <em>Check for updates</em>.</p> 
        <p>The command-line compiler can be downloaded from the <a href="https://github.com/JetBrains/kotlin/releases/tag/v1.2-M2">Github release page</a>.</p> 
        <p><strong>On <a href="https://try.kotlinlang.org">try.kotlinlang.org</a></strong>: Use the drop-down list at the bottom-right corner to change the compiler version to 1.2?M2 (coming soon).</p>
