    <h1> <a href="/t/javascript-syntax-highlight-quirk-with-fix/6036">JavaScript syntax highlight quirk (with fix)</a> </h1> 
    <div id="breadcrumbs"> 
     <div id="breadcrumb-0" itemscope itemtype="http://data-vocabulary.org/Breadcrumb"> 
      <a href="/c/general-discussion" itemprop="url"> <span itemprop="title">General Discussion</span> </a> 
     </div> 
    </div> 
    <hr> 
    <div itemscope itemtype="http://schema.org/DiscussionForumPosting"> 
     <div class="creator"> 
      <span> <a href="/u/zertosh"><b itemprop="author">zertosh</b></a> <time datetime="2012-06-24T16:23:21Z" itemprop="datePublished"> 2012-06-24 16:23:21 UTC </time> </span> 
      <span itemprop="position">#1</span> 
     </div> 
     <div class="post" itemprop="articleBody"> 
      <p>Some of the color schemes (like Monokai) will highlight function arguments. However, when an anonymous function is used as an argument, the anonymous function's argument will not be highlighted unless there is a space between the function keyword and the parentheses. Example:</p> 
      <p></p>
      <pre><code>// argument el should be highlighted but it isn't
Array.prototype.slice.call(arguments).forEach(function(el)  {
	//stuff
});
// argument el is highlighted
// notice the space between function and (el)
Array.prototype.slice.call(arguments).forEach(function (el)  {
	//stuff
});</code></pre> 
      <p>The bug seems to be in the regex to match regular functions. In JavaScript.tmLanguage line 260, you have:<br></p> 
      <p></p>
      <pre><code>&lt;string&gt;\b(function)\s+([a-zA-Z_$]\w*)?\s*(\()(.*?)(\))&lt;/string&gt;</code></pre> 
      <p>but it should be<br></p> 
      <p></p>
      <pre><code>&lt;string&gt;\b(function)\s*([a-zA-Z_$]\w*)?\s*(\()(.*?)(\))&lt;/string&gt;</code></pre> 
      <p>Hope this helps someone!</p> 
     </div>

     <div class="creator"> 
      <span> <a href="/u/agibsonsw"><b itemprop="author">agibsonsw</b></a> <time datetime="2012-06-24T16:31:38Z" itemprop="datePublished"> 2012-06-24 16:31:38 UTC </time> </span> 
      <span itemprop="position">#2</span> 
     </div> 
     <div class="post" itemprop="articleBody"> 
      <p>It is a <a href="http://javascript.crockford.com/code.html" rel="nofollow">coding convention</a> that there should be a space following the word function (for anonymous functions). So I thought the regex expression may even have been intentional.</p> 
     </div>

    <div itemscope itemtype="http://schema.org/DiscussionForumPosting"> 
     <div class="creator"> 
      <span> <a href="/u/zertosh"><b itemprop="author">zertosh</b></a> <time datetime="2012-06-24T16:38:55Z" itemprop="datePublished"> 2012-06-24 16:38:55 UTC </time> </span> 
      <span itemprop="position">#3</span> 
     </div> 
     <div class="post" itemprop="articleBody"> 
      <p>Maybe it is convention, but when you're using an anonymous function in a function expression, the arguments are highlighted with or without a space after the function keyword:</p> 
      <p></p>
      <pre><code>// min and max are highlighted with a space after function
Math.rand = function ( min, max ) {
	return Math.floor(Math.random() * (max - min + 1) + min);
}
//min and max are highlighted without space after function
Math.rand = function( min, max ) {
	return Math.floor(Math.random() * (max - min + 1) + min);
}</code></pre> 
      <p>So if the regex was done intentionally for convention, then it's not being used consistently.</p>
