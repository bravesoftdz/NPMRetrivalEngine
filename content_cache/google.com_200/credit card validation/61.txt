      <header class="dax-topic"> 
       <h2><a href="/docs/api/payments.billing-agreements/">Billing Agreements API</a></h2> 
       <div class="dax-preamble"> 
        <p>You use billing plans and billing agreements to create an agreement for a recurring credit card, bank card, or PayPal payment for goods or services.</p> 
        <p>To create an agreement, you reference an active <a href="/docs/api/payments.billing-plans/">billing plan</a> from which the agreement inherits information. You also supply customer and payment information and, optionally, can override the referenced plan's merchant preferences and shipping fee and tax information.</p> 
        <p>For more information, see <a href="/docs/integration/direct/billing-plans-and-agreements/">Billing Plans and Agreements</a>.</p> 
       </div> 
      </header> 
      <div class="dax-resource"> 
       <div class="dax-topic"> 
        <h3>Billing agreements <small>(resource group)</small></h3> 
        <div class="dax-preamble"> 
         <p>Use the <code>/billing-agreements</code> resource to create an agreement, update an agreement, show agreement details, and cancel agreements. You can also set or bill the balance for an agreement. You can re-activate an agreement, suspend an agreement, list agreement transactions, and execute agreements.</p> 
        </div> 
       </div> 
       <div class="dax-operation-list"> 
        <div class="dax-operation"> 
         <div class="dax-topic"> 
          <h4> <a href="/docs/api/payments.billing-agreements/#agreement_create">Create agreement</a> </h4> 
          <div class="dax-operation-verbpath"> 
           <span class="dax-operation-verb dax-operation-verb-post">POST</span> 
           <span class="dax-operation-path">/v1/payments/billing-agreements</span> 
          </div> 
          <div class="dax-preamble"> 
           <p>Creates a billing agreement.<br></p>
           <blockquote>
            <strong>Important:</strong> The use of the PayPal REST 
            <code>/payments</code> APIs to accept credit card payments is restricted. Instead, you can accept credit card payments with:
            <ul>
             <li><a href="/docs/classic/products/ap-guest-payments/">Guest payments</a> with a <a href="/docs/api/vault/#credit-card_create">credit card that is stored in the PayPal vault</a></li>
             <li><a href="https://www.braintreepayments.com/products/braintree-direct">Braintree Direct</a></li>
             <li><a href="https://www.paypal.com/us/webapps/mpp/merchant">PayPal business products</a></li>
            </ul>
           </blockquote>
           <br>In the JSON request body, include an 
           <code>agreement</code> object with the name, description, start date, ID of the plan on which to base the agreement, and customer and shipping address information. A successful request returns the HTTP 
           <code>201 Created</code> status code and a JSON response body that shows billing agreement details.
           <p></p> 
          </div> 
         </div> 
        </div> 
        <div class="dax-operation"> 
         <div class="dax-topic"> 
          <h4> <a href="/docs/api/payments.billing-agreements/#agreement_update">Update agreement</a> </h4> 
          <div class="dax-operation-verbpath"> 
           <span class="dax-operation-verb dax-operation-verb-patch">PATCH</span> 
           <span class="dax-operation-path">/v1/payments/billing-agreements/<wbr><code>agreement_id</code></span> 
          </div> 
          <div class="dax-preamble"> 
           <p>Updates details of a billing agreement, by ID.<br></p>
           <blockquote>
            <strong>Important:</strong> The use of the PayPal REST 
            <code>/payments</code> APIs to accept credit card payments is restricted. Instead, you can accept credit card payments with:
            <ul>
             <li><a href="/docs/classic/products/ap-guest-payments/">Guest payments</a> with a <a href="/docs/api/vault/#credit-card_create">credit card that is stored in the PayPal vault</a></li>

             <li><a href="https://www.paypal.com/us/webapps/mpp/merchant">PayPal business products</a></li>
            </ul>
           </blockquote>
           <br>Details include the description, shipping address, start date, and so on. A successful request returns the HTTP 
           <code>200 OK</code> status code with no JSON response body.
           <p></p> 
          </div> 
         </div> 
        </div>

        </div> 
        <div class="dax-operation"> 
         <div class="dax-topic"> 
          <h4> <a href="/docs/api/payments.billing-agreements/#agreement_bill-balance">Bill agreement balance</a> </h4> 
          <div class="dax-operation-verbpath"> 
           <span class="dax-operation-verb dax-operation-verb-post">POST</span> 
           <span class="dax-operation-path">/v1/payments/billing-agreements/<wbr><code>agreement_id</code>/<wbr>bill-balance</span> 
          </div> 
          <div class="dax-preamble"> 
           <p>Bills the balance for an agreement, by ID.<br></p>
           <blockquote>
            <strong>Important:</strong> The use of the PayPal REST 
            <code>/payments</code> APIs to accept credit card payments is restricted. Instead, you can accept credit card payments with:

             <li><a href="https://www.paypal.com/us/webapps/mpp/merchant">PayPal business products</a></li>
            </ul>
           </blockquote>
           <br>In the JSON request body, include an optional note that describes the reason for the billing action and the agreement amount and currency. A successful request returns the HTTP 
           <code>204 No Content</code> status code with no JSON response body.
           <p></p> 
          </div> 
         </div> 
        </div>

            </ul>
           </blockquote>
           <br>In the JSON request body, include an 
           <code>agreement_state_descriptor</code> object with an optional note that describes the reason for the cancellation and the agreement amount and currency. A successful request returns the HTTP

           <p></p> 
          </div>

     <div class="dax-api" id="payments.billing-plans"> 
      <header class="dax-topic"> 
       <h2><a href="/docs/api/payments.billing-plans/">Billing Plans API</a></h2> 
       <div class="dax-preamble"> 
        <p>You use billing plans and billing agreements to create an agreement for a recurring credit card, bank card, or PayPal payment for goods or services.</p> 
        <p>A billing plan includes payment definitions and other details. A plan must include at least one regular payment definition and, optionally, a trial payment definition. Each definition determines how often and for how long a customer is charged.</p> 
        <p>A plan can specify a type, which indicates whether the payment definitions in the plan have a fixed or infinite number of payment cycles. The plan also defines merchant preferences including how much it costs to set up the agreement, the URLs where a customer can approve or can cancel the agreement, and the action if the customer's initial payment fails.</p> 
        <p>By default, a plan is not active when you create it. To activate it, you update its <code>state</code> to <code>ACTIVE</code>.</p> 
        <p>For more information, see <a href="/docs/integration/direct/billing-plans-and-agreements/">Billing Plans and Agreements</a>.</p> 
       </div> 
      </header> 
      <div class="dax-resource"> 
       <div class="dax-topic">

        <div class="dax-preamble">

A payment definition can optionally specify shipping fee and tax amounts.<br><br> A successful request returns the HTTP <code>201 Created</code> status code and a JSON response body that shows plan details. The default state of a new plan is <code>CREATED</code>. Before you can create an agreement from a plan, you must activate the plan by updating its <code>state</code> to <code>ACTIVE</code>.</p>

       <h2><a href="/docs/api/invoicing/">Invoicing API</a></h2> 
       <div class="dax-preamble"> 
        <p>Use the Invoicing API to create, send, and manage invoices. You can also use the Invoicing API or webhooks to track invoice payments.</p> 
        <p>When you <a href="#invoices_send">send an invoice</a> to a customer, the invoice moves from draft to payable state. PayPal then emails the customer a link to the invoice on the PayPal website.</p> 
        <p>Customers with a PayPal account can log in and pay the invoice with PayPal. Alternatively, customers can pay as a guest with a debit card or credit card.</p>

       <div class="dax-preamble"> 
        <p>Use the Payments REST API to easily and securely accept online and mobile payments. The payments namespace contains resource collections for payments, sales, refunds, authorizations, captures, and orders.</p> 
        <p>You can enable customers to make PayPal and credit card payments with only a few clicks, depending on the country.       <div class="dax-resource"> 
       <div class="dax-topic"> 
        <h3>Payments <small>(resource group)</small></h3> 
        <div class="dax-preamble"> 
         <p>Use the <code>/payment</code> resource to create a <em><strong>sale</strong></em>, an <em><strong>authorized payment</strong></em>, or an <em><strong>order</strong></em>. A sale is a direct credit card payment, stored credit card payment, or PayPal payment. An authorized payment places funds on hold to be captured later. An order is a purchase that a customer has approved but for which the funds are not placed on hold. You can also use this resource to execute approved PayPal payments and show details for, update, and list payments. For more information, see also <a href="/docs/integration/direct/payments/">Payments</a>.</p> 
        </div> 
       </div> 
       <div class="dax-operation-list"> 
        <div class="dax-operation"> 
         <div class="dax-topic">

           <span class="dax-operation-path">/v1/payments/payment</span> 
          </div> 
          <div class="dax-preamble"> 
           <p>Creates a sale, an authorized payment to be captured later, or an order.<br><br>To create a sale, authorization, or order, include the payment details in the JSON request body.        <h2><a href="/docs/api/payments.payouts-batch/">Payouts API</a></h2> 
       <div class="dax-preamble"> 
        <p>Use the Payouts API to make PayPal payments to multiple PayPal accounts in a single API call. You can specify the recipients by using their PayPal email addresses, phone numbers, or encrypted PayPal account numbers.</p> 
        <p>The Payouts API is a fast, convenient way to send commissions, rebates, rewards, and general disbursements. Payouts appear as Mass Payments in the sender's PayPal account and are provided with the <a href="/docs/classic/mass-pay/integration-guide/MassPayReporting/">Mass Payment reports</a>.</p> 
        <blockquote> 
         <p><strong>Important:</strong> To use the Payouts API, request access through <a href="/developer/accountStatus/">My Account</a>. Alternatively, contact your account manager or <a href="https://www.paypal.com/contactus">PayPal Customer Support</a>. You must have a PayPal business account.</p> 
        </blockquote> 
        <p>The Payouts API uses the <a href="https://tools.ietf.org/html/rfc3339#section-5.6">ISO 8601 date and time format</a>.</p> 
        <p>For more information about the Payouts API, see <a href="/docs/integration/direct/payouts/">Payouts</a>.</p> 
        <blockquote> 
         <p><strong>Deprecation notice:</strong> <a href="#payout-modes">Synchronous mode</a> will soon be deprecated and is no longer available for new integrations but continues to be supported for existing integrations. A synchronous mode payout immediately returns the results of the payout.</p> 
        </blockquote> 
       </div>

     <div class="dax-api" id="webhooks"> 
      <header class="dax-topic"> 
       <h2><a href="/docs/api/webhooks/">Webhooks API</a></h2> 
       <div class="dax-preamble"> 
        <p>The PayPal REST APIs use <a href="/docs/integration/direct/webhooks/">webhooks</a> for event notification. Webhooks are HTTP callbacks that receive notification messages for events.</p> 
        <p>After you configure a webhook listener for your app, you can <a href="#webhooks_create">create a webhook</a>, which subscribes the webhook listener for your app to events.</p> 
        <p>The <code>notifications</code> namespace contains resource collections for webhooks.</p> 
       </div> 
      </header> 
      <div class="dax-resource"> 
       <div class="dax-topic"> 
        <h3>Webhooks <small>(resource group)</small></h3> 
        <div class="dax-preamble">
