    <section class="intro"> 
     <h2>What are Machines?</h2> 
     <p>The <a href="/spec/machine">machine specification</a> is an open standard for Javascript functions. Each machine has a single, clear purpose—whether it be sending an email, translating a text file, or fetching a web page. But whereas traditional functions are supposed to have only a single result, machines are aware of multiple possible outcomes. Machines are self-documenting, quick to implement, and simple to debug.</p> 
     <br> 
     <p>Machines are published to NPM through the use of <a href="/spec/machinepack">machinepacks</a>, or node modules that implement a standardized interface. In other words, when you <code>require()</code> a machinepack, you always know exactly which actions are available, what to provide as input values, and what you're going to get back.</p> 
     <br> Ready to get started? Check out the 
     <a href="/machinepacks">list of available machinepacks</a> or learn how to 
     <a href="/implementing">write your own</a>.

    <section class="three-column"> 
     <div class="column left-column"> 
      <h3>Self-documenting.</h3> 
      <p>Since every <a href="/spec/machine">machine</a> follows the same conventions as described in the <a href="/spec/machine">machine spec</a>, documentation can be created automatically simply by parsing their source file. All of the documentation on <a href="/">node-machine.org</a> is rendered via an automated process.</p> 
     </div> 
     <div class="column middle-column"> 
      <h3>Error-preventing.</h3> 
      <p>Say goodbye to unforseen consequences. Machines validate their inputs automatically, and by using the concept of <i>exits</i> to control program flow, they give developers an unprecedented degree of error-handling.</p> 
     </div> 
     <div class="column right-column"> 
      <h3>Universal.</h3> 
      <p>No more trial-and-error when integrating a new third-party module into your project: every machine follows the same easy-to-use interface.</p> 
     </div>
