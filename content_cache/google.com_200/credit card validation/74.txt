     <h1 class="document-title" id="doc-title">Stripe</h1> 
     <div class="entry"> 
      <p><script src="https://fast.wistia.com/assets/external/E-v1.js" async></script></p> 
      <div class="wistia_responsive_padding" style="padding:56.0% 0 0 0;position:relative;"> 
       <div class="wistia_responsive_wrapper" style="height:100%;left:0;position:absolute;top:0;width:100%;"> 
        <div class="wistia_embed wistia_async_mf975hx5de videoFoam=true" style="height:100%;width:100%">
         &nbsp;
        </div> 
       </div> 
      </div> 
      <p>&nbsp;</p> 
      <p>The <span style="text-decoration: underline;"><a href="http://woocommerce.com/products/stripe/" target="_blank" rel="noopener noreferrer">Stripe</a></span>&nbsp;payment gateway for&nbsp;WooCommerce lets you&nbsp;accept&nbsp;payments directly onsite, and includes the option to accept ApplePay and Bitcoin</p> 
      <p>Download it now at <a href="https://wordpress.org/plugins/woocommerce-gateway-stripe/">WordPress.org</a>. See <a href="https://github.com/woocommerce/woocommerce-gateway-stripe/blob/master/changelog.txt">Changelog</a>.</p> 
      <h2 id="section-1">Installation <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h2> 
      <ol> 
       <li><strong>Go to Plugins &gt; Add New</strong> on your WordPress site.</li> 
       <li><strong>Enter WooCommerce Stripe</strong>&nbsp;in the Search Plugins bar. Author is&nbsp;Automattic/WooCommerce.</li> 
       <li><strong>Click Install Now</strong>&nbsp;and <strong>Activate</strong>.</li> 
      </ol> 
      <p>To learn more, see <a href="http://codex.wordpress.org/Managing_Plugins#Installing_Plugins">Installing and Managing Plugins</a>.</p> 
      <h2 id="section-2">Setup and Configuration <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h2> 
      <p><img data-attachment-id="163877" data-permalink="https://docs.woocommerce.com/document/stripe/woocommerce-stripe/" data-orig-file="https://docs.woocommerce.com/wp-content/uploads/2012/09/woocommerce-stripe.png" data-orig-size="1410,814" data-comments-opened="0" data-image-meta="{&quot;aperture&quot;:&quot;0&quot;,&quot;credit&quot;:&quot;&quot;,&quot;camera&quot;:&quot;&quot;,&quot;caption&quot;:&quot;&quot;,&quot;created_timestamp&quot;:&quot;0&quot;,&quot;copyright&quot;:&quot;&quot;,&quot;focal_length&quot;:&quot;0&quot;,&quot;iso&quot;:&quot;0&quot;,&quot;shutter_speed&quot;:&quot;0&quot;,&quot;title&quot;:&quot;&quot;,&quot;orientation&quot;:&quot;0&quot;}" data-image-title="woocommerce-stripe" data-image-description="" data-medium-file="https://docs.woocommerce.com/wp-content/uploads/2012/09/woocommerce-stripe.png?w=550" data-large-file="https://docs.woocommerce.com/wp-content/uploads/2012/09/woocommerce-stripe.png?w=950" class="size-full wp-image-163877 alignnone" src="https://docs.woocommerce.com/wp-content/uploads/2012/09/woocommerce-stripe.png" alt="woocommerce-stripe" width="980" height="566"></p> 
      <ol> 
       <li>Go to:<strong> WooCommerce &gt; Settings &gt;&nbsp;Checkout &gt; Stripe</strong>.</li> 
       <li><strong>Tick</strong> the <strong>Enable Stripe</strong> checkbox. Untick to disable.</li> 
       <li><strong>Enter</strong> a Title (required) and Description (optional) seen by the customer during checkout.</li> 
       <li><strong>Tick</strong> the&nbsp;<strong>Enable&nbsp;Test Mode</strong> checkbox, if you wish to test before going live. <strong>Untick</strong> to begin accepting payments with Stripe.</li> 
       <li><strong>Enter</strong> your <strong>Live Secret Key</strong>&nbsp;and <strong>Live Publishable Key</strong> (<a title="Stripe API Keys" href="https://manage.stripe.com/account/apikeys" target="_blank" rel="noopener noreferrer">obtained&nbsp;from your Stripe account</a>). The Test Secret Key and Test Publishable Key are different and should only be used when Enable Test Mode selected in step 4.</li> 
       <li>Enter <strong>Statement Descriptor</strong>, a description that appears on a customer’s credit card statement.</li> 
       <li><b>Tick the checkbox&nbsp;</b>to <strong>Capture Charge Immediately</strong> (Authorize and Capture) or <strong>untick</strong> to only <strong>Authorize</strong>. If you untick the box, you need to capture funds manually for each order via the order screen.<img data-attachment-id="301172" data-permalink="https://docs.woocommerce.com/document/stripe/stripe-applepay-woocommerce/" data-orig-file="https://docs.woocommerce.com/wp-content/uploads/2012/09/stripe-applepay-woocommerce.png" data-orig-size="1404,1258" data-comments-opened="0" data-image-meta="{&quot;aperture&quot;:&quot;0&quot;,&quot;credit&quot;:&quot;&quot;,&quot;camera&quot;:&quot;&quot;,&quot;caption&quot;:&quot;&quot;,&quot;created_timestamp&quot;:&quot;0&quot;,&quot;copyright&quot;:&quot;&quot;,&quot;focal_length&quot;:&quot;0&quot;,&quot;iso&quot;:&quot;0&quot;,&quot;shutter_speed&quot;:&quot;0&quot;,&quot;title&quot;:&quot;&quot;,&quot;orientation&quot;:&quot;0&quot;}" data-image-title="stripe-applepay-woocommerce" data-image-description="" data-medium-file="https://docs.woocommerce.com/wp-content/uploads/2012/09/stripe-applepay-woocommerce.png?w=550" data-large-file="https://docs.woocommerce.com/wp-content/uploads/2012/09/stripe-applepay-woocommerce.png?w=950" class="aligncenter size-full wp-image-301172" src="https://docs.woocommerce.com/wp-content/uploads/2012/09/stripe-applepay-woocommerce.png" alt="" width="980" height="878"></li> 
       <li><strong>Enable Stripe Checkout</strong> if you prefer a popup to request details, the ability to accept <strong>Bitcoin</strong> and using a checkout image. See next section <strong>Stripe Checkout</strong> to understand how it appears for customers.</li> 
       <li><strong>Enable Payment Request API</strong> (optional) if you wish to give customers the ability to pay via&nbsp;the API (must be supported by their browser).&nbsp;For example, this will prompt Chrome Pay if you’re using this browser on Android or desktop and you’ve got cards saved.</li> 
       <li><strong>Enable Apple Pay</strong> if you wish to give customers with an iOS 10+ device and Safari-based web browser the option to pay using Apple Pay. 
        <ul> 
         <li>Select <strong>Black</strong> or <strong>White</strong> for <strong>Apple Pay Button Style</strong></li> 
         <li>Enter <strong>two-digit ISO code</strong> for Language. Default is en for English. Other languages, see:&nbsp;<a href="http://www.w3schools.com/tags/ref_language_codes.asp">http://www.w3schools.com/tags/ref_language_codes.asp</a></li> 
         <li><strong>Accept the Apple Pay Terms of Service</strong> (ToS). This is mandatory or you will receive an error.</li> 
        </ul> </li> 
       <li><strong>Saved Cards</strong>, if you want to give Stripe customers the ability to use a card saved on their account for future purchases.</li> 
       <li><strong>Log Debug Messages</strong>, if you wish to save logs that can help you/us troubleshoot your site. Can be disabled and enabled as needed.</li> 
       <li><strong>Save Changes</strong>.</li> 
      </ol> 
      <div class="woo-sc-box note   ">
       The same single user account credentials cannot be used to make purchases in&nbsp;both Live Mode and Test Mode. 
      </div> 
      <h2 id="section-3">Stripe Checkout <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h2> 
      <p>If you tick the box for <strong>Enable Stripe Checkout</strong>, more options will appear:</p> 
      <p><a href="https://docs.woocommerce.com/wp-content/uploads/2012/09/stripe-checkout.png"><img data-attachment-id="347394" data-permalink="https://docs.woocommerce.com/document/stripe/stripe-checkout-2/" data-orig-file="https://docs.woocommerce.com/wp-content/uploads/2012/09/stripe-checkout.png" data-orig-size="1356,460" data-comments-opened="0" data-image-meta="{&quot;aperture&quot;:&quot;0&quot;,&quot;credit&quot;:&quot;&quot;,&quot;camera&quot;:&quot;&quot;,&quot;caption&quot;:&quot;&quot;,&quot;created_timestamp&quot;:&quot;0&quot;,&quot;copyright&quot;:&quot;&quot;,&quot;focal_length&quot;:&quot;0&quot;,&quot;iso&quot;:&quot;0&quot;,&quot;shutter_speed&quot;:&quot;0&quot;,&quot;title&quot;:&quot;&quot;,&quot;orientation&quot;:&quot;0&quot;}" data-image-title="Stripe Checkout" data-image-description="" data-medium-file="https://docs.woocommerce.com/wp-content/uploads/2012/09/stripe-checkout.png?w=550" data-large-file="https://docs.woocommerce.com/wp-content/uploads/2012/09/stripe-checkout.png?w=950" class="aligncenter size-large wp-image-347394" src="https://docs.woocommerce.com/wp-content/uploads/2012/09/stripe-checkout.png?w=950" alt="" width="950" height="322"></a></p> 
      <p>&nbsp;</p> 
      <ul> 
       <li><strong>Stripe Checkout locale</strong>: English is default. Chinese, Dutch, French, German, Italian, Japanese and Spanish are also offered. Auto-displays user’s preferred language, if available.</li> 
       <li><strong>Enable Bitcoin Currency</strong>: Tick the box if you would like to accept Bitcoin via Stripe.</li> 
       <li><strong>Stripe Checkout Image</strong>: Enter the URL of a 128×128 px image/logo to be displayed in checkout. Use an image already uploaded to your site or upload one now at <strong>Media &gt; Add New</strong>.</li> 
      </ul> 
      <p><strong>Enable Stripe Checkout</strong>&nbsp;refers to this service: &nbsp;<a href="https://stripe.com/blog/stripe-checkout">https://stripe.com/blog/stripe-checkout</a>&nbsp;Instead of an inline payment form, which is the default setting, a pop-up is used to collect credit card details from the user.</p> 
      <p><a href="https://docs.woocommerce.com/wp-content/uploads/2012/09/stripe-checkout-modal-window.png"><img data-attachment-id="163369" data-permalink="https://docs.woocommerce.com/document/stripe/stripe-checkout-modal-window/" data-orig-file="https://docs.woocommerce.com/wp-content/uploads/2012/09/stripe-checkout-modal-window.png" data-orig-size="724,994" data-comments-opened="0" data-image-meta="{&quot;aperture&quot;:&quot;0&quot;,&quot;credit&quot;:&quot;&quot;,&quot;camera&quot;:&quot;&quot;,&quot;caption&quot;:&quot;&quot;,&quot;created_timestamp&quot;:&quot;0&quot;,&quot;copyright&quot;:&quot;&quot;,&quot;focal_length&quot;:&quot;0&quot;,&quot;iso&quot;:&quot;0&quot;,&quot;shutter_speed&quot;:&quot;0&quot;,&quot;title&quot;:&quot;&quot;,&quot;orientation&quot;:&quot;0&quot;}" data-image-title="Stripe Checkout Modal Window" data-image-description="" data-medium-file="https://docs.woocommerce.com/wp-content/uploads/2012/09/stripe-checkout-modal-window.png?w=401" data-large-file="https://docs.woocommerce.com/wp-content/uploads/2012/09/stripe-checkout-modal-window.png?w=692" class="wp-image-163369 size-medium aligncenter" src="https://docs.woocommerce.com/wp-content/uploads/2012/09/stripe-checkout-modal-window.png?w=401" alt="Stripe Checkout Modal Window" width="401" height="550"></a></p> 
      <h2 id="section-4">Apple Pay <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h2> 
      <p>Before you start using Apple Pay, you must accept the Terms of Service in your Stripe dashboard seen here:<br> <a href="https://docs.woocommerce.com/wp-content/uploads/2012/09/stripe-apple-pay-tos.png"><img data-attachment-id="324303" data-permalink="https://docs.woocommerce.com/document/stripe/stripe-apple-pay-tos/" data-orig-file="https://docs.woocommerce.com/wp-content/uploads/2012/09/stripe-apple-pay-tos.png" data-orig-size="2736,1066" data-comments-opened="0" data-image-meta="{&quot;aperture&quot;:&quot;0&quot;,&quot;credit&quot;:&quot;&quot;,&quot;camera&quot;:&quot;&quot;,&quot;caption&quot;:&quot;&quot;,&quot;created_timestamp&quot;:&quot;0&quot;,&quot;copyright&quot;:&quot;&quot;,&quot;focal_length&quot;:&quot;0&quot;,&quot;iso&quot;:&quot;0&quot;,&quot;shutter_speed&quot;:&quot;0&quot;,&quot;title&quot;:&quot;&quot;,&quot;orientation&quot;:&quot;0&quot;}" data-image-title="stripe-apple-pay-tos" data-image-description="" data-medium-file="https://docs.woocommerce.com/wp-content/uploads/2012/09/stripe-apple-pay-tos.png?w=550" data-large-file="https://docs.woocommerce.com/wp-content/uploads/2012/09/stripe-apple-pay-tos.png?w=950" class="alignnone size-full wp-image-324303" src="https://docs.woocommerce.com/wp-content/uploads/2012/09/stripe-apple-pay-tos.png" alt="" width="980" height="382"></a></p>

      <h2 id="section-6">FAQ <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h2> 
      <h3 id="section-7">The method is not showing. <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>In live mode, an SSL certificate must be installed on your site to use Stripe.&nbsp;In addition to SSL encryption, Stripe provides an <em>extra</em> JavaScript method to secure card data.</p> 
      <h3 id="section-8">Apple Pay is not showing. <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>The customer must have a device with iOS 10+ and be using a Safari-based web browser. Otherwise the option is unavailable.</p> 
      <h3 id="section-9">Apple Pay – I am getting this error “Error: Unable to copy domain association file to domain root” <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>That could mean your server configuration is not allowing files/paths to be written in that location. You would have to manually do this by following the step outlined <a href="https://stripe.com/docs/apple-pay/web#going-live" target="_blank" rel="noopener noreferrer">here</a></p> 
      <h3 id="section-10">Apple Pay – I am getting this error “Error: Unable to verify domain – Bad Request” <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>Try manually doing this by following the step outlined <a href="https://stripe.com/docs/apple-pay/web#going-live" target="_blank" rel="noopener noreferrer">here</a></p> 
      <h3 id="section-11">Can I test Apple Pay? <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>Yes, please follow the link here for instructions <a href="https://developer.apple.com/support/apple-pay-sandbox/" target="_blank" rel="noopener noreferrer">Apple Pay Testing</a></p> 
      <h3 id="section-12">Can I hide the Apple Pay button on the single product / cart / checkout pages? <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>Yes you can! As we do not recommend using CSS to do so, we encourage you to follow <a href="http://www.remicorson.com/remove-apple-pay-button-on-product-cart-checkout-pages-in-stripe-integration-for-woocommerce/" target="_blank" rel="noopener noreferrer">this tutorial</a>.</p> 
      <h3 id="section-13">When switching from the paid Stripe gateway to the free version, will my settings be saved? <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>Yes, after installing the free Stripe 3.X version, your settings from the paid Stripe 2.612 version will carry over.</p> 
      <h3 id="section-14">Can I use Apple Pay for recurring/subscription-based payments? <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>Not at this time. This feature is currently unsupported. The Apple Pay button stays hidden for recurring payments associated with a subscription.</p> 
      <h3 id="section-15">Can a single&nbsp;customer have multiple subscriptions? <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>Using the WooCommerce Stripe extension and WooCommerce Subscriptions, you can have multiple subscriptions in your WooCommerce store. It should be noted that WC Subscriptions and the WC Stripe extension don’t use Stripe’s Recurring Plan feature, but instead manage subscriptions in WooCommerce and use token billing for each recurring payment.</p> 
      <h3 id="section-16">Is it possible to override templates on the checkout page? <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>Yes. This extension only has one template, which is: saved-cards.php. To override it, copy the original template from wp-content/plugins/woocommerce-gateway-stripe/templates/ and place it in the wp-content/themes/your-theme-name/woocommerce-gateway-stripe/ folder. Open and edit as you wish.</p> 
      <h3 id="section-17">What version of the Stripe API does this plugin use? <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>This plugin uses v1 of the Stripe API.</p> 
      <h3 id="section-18">Does this extension support Bitcoin? <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>Yes, it does. As of version 2.4.0, Stripe accepts payment in Bitcoin (<strong>WooCommerce &gt; Settings &gt; Checkout &gt; Stripe &gt;&nbsp;Enable Bitcoin Currency</strong>). Refunds can be issued via the Stripe Dashboard&nbsp;and will require the customer’s&nbsp;Bitcoin address. See <span style="text-decoration: underline;"><a href="http://dzv365zjfbd8v.cloudfront.net/changelogs/woocommerce-gateway-stripe/changelog.txt" target="_blank" rel="noopener noreferrer">Changelog</a></span>.</p> 
      <p>Recurring and repeat payments with Bitcoin are not currently supported. Bitcoin may only be used for a one-time&nbsp;payment. More at: <a href="https://stripe.com/docs/bitcoin">Stripe – Bitcoin Guide</a>.</p> 
      <h3 id="section-19">Does this extension support Stripe Connect? <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>No, not currently; we’re unable to integrate at this time.</p> 
      <h3 id="section-20">Does Stripe use AVS – Address Verification System? <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>Yes! This is enabled in your Stripe account dashboard and will then work in WooCommerce. See Stripe’s documentation about <a href="https://support.stripe.com/questions/avoiding-fraud-and-disputes">Avoiding fraud and disputes</a>.</p> 
      <h3 id="section-21">Does Stripe Support Authorization and Capture? <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>Yes! This is an option in the extension settings. Charges can be captured from within the WooCommerce dashboard. Please note that un-captured authorizations will expire after 7 days.</p> 
      <p>When the Stripe payment gateway is set to <b>authorize,</b>&nbsp;orders made via Stripe will have the status <strong>on hold</strong> and the following note:</p> 
      <p><a href="https://docs.woocommerce.com/wp-content/uploads/2012/09/2016-08-01_10-03-30.png"><img data-attachment-id="163344" data-permalink="https://docs.woocommerce.com/document/stripe/2016-08-01_10-03-30/" data-orig-file="https://docs.woocommerce.com/wp-content/uploads/2012/09/2016-08-01_10-03-30.png" data-orig-size="271,190" data-comments-opened="0" data-image-meta="{&quot;aperture&quot;:&quot;0&quot;,&quot;credit&quot;:&quot;&quot;,&quot;camera&quot;:&quot;&quot;,&quot;caption&quot;:&quot;&quot;,&quot;created_timestamp&quot;:&quot;0&quot;,&quot;copyright&quot;:&quot;&quot;,&quot;focal_length&quot;:&quot;0&quot;,&quot;iso&quot;:&quot;0&quot;,&quot;shutter_speed&quot;:&quot;0&quot;,&quot;title&quot;:&quot;&quot;,&quot;orientation&quot;:&quot;0&quot;}" data-image-title="2016-08-01_10-03-30" data-image-description="" data-medium-file="https://docs.woocommerce.com/wp-content/uploads/2012/09/2016-08-01_10-03-30.png?w=271" data-large-file="https://docs.woocommerce.com/wp-content/uploads/2012/09/2016-08-01_10-03-30.png?w=271" class="alignnone size-full wp-image-163344" src="https://docs.woocommerce.com/wp-content/uploads/2012/09/2016-08-01_10-03-30.png" alt="2016-08-01_10-03-30" width="271" height="190"></a></p> 
      <p>Change the order status to <strong>processing </strong>or<strong> completed</strong>&nbsp;to take payment or cancel the order to remove the pre-authorization.</p> 
      <h3 id="section-22">Does Stripe allow refunds inside WooCommerce? <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>Yes. More info at: <a href="https://docs.woocommerce.com/document/woocommerce-refunds/#section-4">WooCommerce Refunds</a>.</p> 
      <h3 id="section-23">Does Stripe Support ACH? <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>The Stripe plugin from&nbsp;WooCommerce does not support ACH.</p> 
      <h3 id="section-24">Why aren’t my subscriptions displayed in the Stripe Dashboard? <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>WooCommerce Subscriptions does not use the <a href="https://stripe.com/subscriptions">Stripe Subscriptions</a> feature. Instead, it uses a Stripe Customer and Card token and controls the processing of recurring payments.</p> 
      <p>This avoids many potential issues that could occur by trying to keep two copies of each subscription’s data in sync on two different systems (one in your store and one at Stripe). It also and makes it possible for Stripe to support all of <a href="https://docs.woothemes.com/document/subscriptions/payment-gateways/#advanced-features">WooCommerce Subscriptions’ features</a>, like payment date changes, rather than only those the Stripe Subscriptions allows.</p> 
      <p>Because of this, you will not see subscriptions created with WooCommerce in your Stripe Dashboard as Subscriptions. This does not mean recurring payments will not be processed. As long as your store is running correctly, recurring payments will be processed correctly and will show up in the Stripe Dashboard as Payments.</p> 
      <h3 id="section-25">Do I need to enable “Saved Card” to process subscriptions? <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>Subscriptions have its own mechanism of saving credit card details. When a customer buys a subscription product, Stripe will create a special payment token that will use the same credit card details for future subscription payments. Saving the same card again on the customer’s account is not necessary, unless you want to allow the customer to reuse the card for other purchases in your store.</p> 
      <h2 id="section-26">Troubleshooting <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h2> 
      <h3 id="section-27">Root Relative URLs <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>There are known issues with the Root Relative URLs plugin. You will not be able to use this with Stripe.</p> 
      <h3 id="section-28">I get an error when placing an order telling me to enable JavaScript. <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>This error may&nbsp;be caused by non-standard/poorly coded themes and JavaScript (JS) issues. Common issues include:</p> 
      <ol> 
       <li>Javascript errors on the checkout page – To view the error, open up your browser error console (in Chrome: View &gt; developer &gt; javascript console) and look for red errors. This should indicate where the error is located and lead you to the problem, i.e., loading jquery wrong</li> 
       <li>Failing to load scripts – Stripe loads 2 JS files which it needs to function. If these are not loaded you will see errors. The most common reasons for not loading are: 
        <ul> 
         <li>A theme missing wp_head() or wp_footer() calls</li> 
         <li>Old overridden template files from WooCommerce inside your theme</li> 
         <li>Loading headers/footers in a non-standard way. WooCommerce uses get_header()’s get_header action to init the checkout and load scripts. If you are not using get_header() you either need to do so, or you need to trigger the get_header action manually using: do_action( ‘get_header ); in your custom header loader.</li> 
        </ul> </li> 
      </ol> 
      <h3 id="section-29">I get an error, “Please make sure your card details have been entered correctly and that your browser supports JavaScript.” <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>The solution is almost always a theme conflict or an issue with the placement of the public and secret keys. First, confirm you’ve placed the keys in the correct field. If that doesn’t help, try switching to the Twenty Seventeen&nbsp;theme to see if the issue disappears. If so, you likely&nbsp;have a JavaScript error or conflict in your theme.&nbsp;Try to <a href="http://stackoverflow.com/questions/2120093/how-to-find-javascript-syntax-errors" target="_blank" rel="noopener noreferrer">find the JavaScript error</a> or switch to a <a href="http://woocommerce.com/product-category/themes/?utm_source=support%20website&amp;utm_medium=woothemes%20referral&amp;utm_campaign=standard%20links&amp;utm_content=/knowledge-base/article/woocommerce/extensions/stripe-error-please-make-sure-your-card-details-have-been-entered-correctly-and-that-your-browser-supports-javascript&amp;utm_term=https://support.woothemes.com/hc/en-us/articles/203104417-Stripe-Error-Please-make-sure-your-card-details-have-been-entered-correctly-and-that-your-browser-supports-JavaScript-" target="_blank" rel="noopener noreferrer">higher quality theme</a>.</p> 
      <h3 id="section-30">I’m getting an API error. <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>Ensure that you have input the correct keys. If you are in live mode, you should have&nbsp;entered live keys (not test keys).</p> 
      <ul> 
       <li>If you are using test mode, keys are prepended with sk_test and pk_test.</li> 
       <li>If you are using live mode, keys are prepended with sk_live and pk_live.</li> 
      </ul> 
      <h3 id="section-31">I’m seeing the error “Could not find payment information.” <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>This usually means that test mode is still enabled while using live keys, or that test keys are being used in live mode.</p> 
      <h3 id="section-32">I get an error, “Developers: Please make sure that you are including jQuery and there are no JavaScript errors on the page.” <a href="#doc-title" class="woodocs-top-link" title="Back to top">? Back to top</a></h3> 
      <p>This is typically caused by entering your Secret and Publishable Stripe keys in the incorrect order. Please go to the&nbsp;<strong>WooCommerce &gt; Settings &gt; Checkout &gt; Stripe</strong> tab and update both the <em>Stripe Live Secret Key</em> and <em>Stripe Live Publishable Key</em>. Make sure that your enter the proper corresponding keys from your Stripe account into each field.</p>

		jQuery("input[name='s']:not('.no-autocomplete')").each(function(i) {

				});

				window.location.href = suggestion.permalink;

			$autocomplete.removeAttr('style');

				constraints: [

						to: 'window',
						attachment: 'none element'
