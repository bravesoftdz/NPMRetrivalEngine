        <h1 class="post__title">Stripe Elements – A set of pre-built components to easily accept credit cards</h1> 
       </header> 
       <p></p>
       <p>Stripe Elements is a recently launched set of pre-built UI components that allows you to securely collect credit card details. This is designed to make taking credit cards easier than ever. Here is a screenshot of the component:</p> 
       <p><img class="alignnone size-full wp-image-8278" src="https://i2.wp.com/wp.laravel-news.com/wp-content/uploads/2017/02/elements.png?resize=525%2C106" alt="" srcset="https://i2.wp.com/wp.laravel-news.com/wp-content/uploads/2017/02/elements.png?w=1424 1424w, https://i2.wp.com/wp.laravel-news.com/wp-content/uploads/2017/02/elements.png?resize=600%2C121 600w, https://i2.wp.com/wp.laravel-news.com/wp-content/uploads/2017/02/elements.png?resize=768%2C155 768w, https://i2.wp.com/wp.laravel-news.com/wp-content/uploads/2017/02/elements.png?resize=1400%2C283 1400w, https://i2.wp.com/wp.laravel-news.com/wp-content/uploads/2017/02/elements.png?w=1050 1050w" sizes="(max-width: 767px) 89vw, (max-width: 1000px) 54vw, (max-width: 1071px) 543px, 580px" data-recalc-dims="1"></p> 
       <p>Previously, with Stripe.js you had to build your form by including all the input fields, validation, and formatting manually. Now it’s all handled for you!</p> 
       <h2>Stripe Elements Setup</h2> 
       <p>First, include the JavaScript:</p> 
       <pre><code>&lt;script src="https://js.stripe.com/v3/"&gt;&lt;/script&gt;
</code></pre> 
       <p>Next, for your checkout form:</p> 
       <pre><code>&lt;form action="/charge" method="post" id="payment-form"&gt;
  &lt;div class="form-row"&gt;
    &lt;label for="card-element"&gt;
      Credit or debit card
    &lt;/label&gt;
    &lt;div id="card-element"&gt;
      &lt;!-- a Stripe Element will be inserted here. --&gt;
    &lt;/div&gt;

    &lt;!-- Used to display form errors --&gt;
    &lt;div id="card-errors"&gt;&lt;/div&gt;
  &lt;/div&gt;

  &lt;input type="submit" class="submit" value="Submit Payment"&gt;
&lt;/form&gt;

&lt;script&gt;
var stripe = Stripe('pk_test_6pRNASCoBOKtIshFeQd4XMUh');
var elements = stripe.elements();
var card = elements.create('card');

// Add an instance of the card UI component into the `card-element` &lt;div&gt;
card.mount('#card-element');
&lt;/script&gt;
</code></pre> 
       <p>Inside the <code>#card-element</code> Stripe will inject an iFrame with the UI components and all that is left up to you is adding the server side charge code.</p> 
       <p>Using this new component is not only easy, but it’s more secure for Stripe because it ensures credit card numbers are never sent to your server. In the previous version when you had to create your input fields they recommended to never use <code>name</code> fields for the inputs so the browser wouldn’t POST card data. I’ve seen on several occasions where developers missed that. Even though Stripe had a big warning:</p> 
       <blockquote>
        <p> Do not give names to the payment details form elements! This prevents that data from touching your server, which means you no longer need to worry about redacting logs, encrypting cardholder details, or other burdens of PCI compliance. </p>
       </blockquote> 
       <p>Elements is available for use now, and you can find more information on setup and usage from the <a href="https://stripe.com/docs/elements">official documentation</a>.</p>
