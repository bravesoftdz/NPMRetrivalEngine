           <p>In this quick tutorial we will show you how to create a simple credit card form. We'll build the whole thing from scratch, with a little help from <a href="http://getbootstrap.com/" target="_blank" rel="noreferrer noopener">Bootstrap 3</a> for the interface, and <a href="https://jondavidjohn.github.io/payform/" target="_blank" rel="noreferrer noopener">Payform.js</a> for client-side form validation.</p> 
           <blockquote class="warning">
            Keep in mind that storing credit card data on your servers is serious business. You need to be 
            <a href="https://en.wikipedia.org/wiki/Payment_Card_Industry_Data_Security_Standard" target="_blank" rel="noreferrer noopener">PCI</a> compliant and fulfill a vast number of requirements. &nbsp; Unless you know what you're doing, it is best to use the services of a third party payment processor like Stripe or PayPal that stores all payment information for you.
           </blockquote>

           <h2>Layout</h2> 
           <p>A credit card dialog needs to be simple, short, and straightforward. Here are the four input fields that every credit card form needs to have:</p> 
           <ul>
            <li>Credit card owner name</li> 
            <li>Card number</li> 
            <li>Secret code (also known as CVV/CVC/CID)</li> 
            <li>Expiration Date</li> 
           </ul> 
           <p>All we need to do is create a <code>&lt;form&gt;</code> and add all the required input fields. For the owner, card number, and CVV we will use simple text fields. For the expiration date we'll put a combination of two selects with predefined options.</p> 
           <p>Besides that our form will have a heading, a submit button, and images for popular credit card vendors. Since we are working with Bootstrap there is a little extra markup, but it helps keep the code organized and the layout responsive.</p>

           <p>Now that we have the needed input fields, we can setup the validation rules.</p> 
           <h2>Validation</h2> 
           <p>All of the validation we will show here is client side and done exclusively in the JavaScript. If it is HTML validation that you are interested in, check out <a href="https://tutorialzine.com/2014/12/quick-tip-easy-form-validation-with-html5/" target="_blank" rel="noreferrer noopener">this</a> article.</p> 
           <p>To kick things off we will define all the jQuery selectors we will need:</p> 
           <pre data-type="js">var owner = $('#owner'),
    cardNumber = $('#cardNumber'),
    cardNumberField = $('#card-number-field'),
    CVV = $("#cvv"),
    mastercard = $("#mastercard"),
    confirmButton = $('#confirm-purchase'),
    visa = $("#visa"),
    amex = $("#amex");</pre> 
           <p>Then, using Payform.js, we will turn our basic input fields into specialized input for credit card data. We simply need to call the right function and the library will automatically handle text formatting and maximum string length for us:</p> 
           <pre data-type="js">cardNumber.payform('formatCardNumber');
CVV.payform('formatCardCVC');</pre> 
           <p>Next, we want to be able to give real-time feedback to users while they are typing in their card number. To do so we will write a simple function that does two things:</p> 
           <ol>
            <li>Check if the current text in the field is ? valid card number or not. Add appropriate coloring to the text field.</li> 
            <li>Depending on the present input characters, see if the card is either Visa, MasterCard, or American Express. This is done using the <code>payform.parseCardType()</code> method.</li> 
           </ol> 
           <p>Since we want to execute the above actions every time a new character is typed in, we will use the jQuery <code><a href="https://api.jquery.com/keyup/" target="_blank" rel="noreferrer noopener">keyup()</a></code> event listener.</p> 
           <pre data-type="js">cardNumber.keyup(function() {
    amex.removeClass('transparent');
    visa.removeClass('transparent');
    mastercard.removeClass('transparent');

    if ($.payform.validateCardNumber(cardNumber.val()) == false) {
        cardNumberField.removeClass('has-success');
        cardNumberField.addClass('has-error');
    } else {
        cardNumberField.removeClass('has-error');
        cardNumberField.addClass('has-success');
    }

    if ($.payform.parseCardType(cardNumber.val()) == 'visa') {
        mastercard.addClass('transparent');
        amex.addClass('transparent');
    } else if ($.payform.parseCardType(cardNumber.val()) == 'amex') {
        mastercard.addClass('transparent');
        visa.addClass('transparent');
    } else if ($.payform.parseCardType(cardNumber.val()) == 'mastercard') {
        amex.addClass('transparent');
        visa.addClass('transparent');
    }
});</pre> 
           <p>There is one more thing we have to do and that is is check if all the field are holding valid data when the user tries to submit the form.</p> 
           <p>Name validation can be quite tricky. To keep this tutorial light, we won't be going into that subject, and we will only check if the input name is at least 5 characters long. Payform provides us with the needed methods for validating the rest of the form.</p> 
           <pre data-type="js">confirmButton.click(function(e) {
    e.preventDefault();

    var isCardValid = $.payform.validateCardNumber(cardNumber.val());
    var isCvvValid = $.payform.validateCardCVC(CVV.val());

    if(owner.val().length &lt; 5){
        alert("Wrong owner name");
    } else if (!isCardValid) {
        alert("Wrong card number");
    } else if (!isCvvValid) {
        alert("Wrong CVV");
    } else {
        // Everything is correct. Add your form submission code here.
        alert("Everything is correct");
    }
});</pre> 
           <blockquote class="warning">
            The above validation is for educational purposes only and shouldn't be used on commercial projects. Always include both client-side and server-side validation to your forms, especially when working with credit card data.
           </blockquote>
