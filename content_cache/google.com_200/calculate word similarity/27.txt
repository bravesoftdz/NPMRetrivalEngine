       <blockquote>
        <p>Keyword extraction is tasked with the automatic identification of terms that best describe the subject of a document.</p> 
        <p>Key phrases, key terms, key segments or just keywords are the terminology which is used for defining the terms that represent the most relevant information contained in the document. Although the terminology is different, function is the same: characterization of the topic discused in a document. Keyword extraction task is important problem in Text Mining, Information Retrieval and Natural Language Processing.</p>
       </blockquote> 
       <p><strong>1. <a href="https://github.com/aneesha/RAKE">RAKE</a> (A python implementation of the Rapid Automatic Keyword Extraction)</strong></p> 
       <p>Started with RAKE, a python implementation of the Rapid Automatic Keyword Extraction, I follow the document “<a href="https://www.airpair.com/nlp/keyword-extraction-tutorial">NLP keyword extraction tutorial with RAKE and Maui</a>“. As the document said:</p> 
       <blockquote>
        <p>A typical keyword extraction algorithm has three main components:</p> 
        <ul> 
         <li>Candidate selection: Here, we extract all possible words, phrases, terms or concepts (depending on the task) that can potentially be keywords.</li> 
         <li>Properties calculation: For each candidate, we need to calculate properties that indicate that it may be a keyword. For example, a candidate appearing in the title of a book is a likely keyword.</li> 
         <li>Scoring and selecting keywords: All candidates can be scored by either combining the properties into a formula, or using a machine learning technique to determine probability of a candidate being a keyword. A score or probability threshold, or a limit on the number of keywords is then used to select the final set of keywords..</li> 
        </ul> 
       </blockquote> 
       <p>RAKE follow the three steps strictly, and have a good design structure for keyword extraction. Follow the document example Rake tutorial, I tested RAKE on my mac os environment step-by-step:</p>

        <p>To summarize, RAKE is a simple keyword extraction library which focuses on finding multi-word phrases containing frequent words.        <p>5. <a href="http://www.nzdl.org/Kea/">KEA – Keyphrase Extraction Algorithm</a></p> 
       <p>We cannot ignore the KEA algorithm for keyword or keyphrase extraction:</p> 
       <blockquote>
        <p>Keywords and keyphrases (multi-word units) are widely used in large document collections. They describe the content of single documents and provide a kind of semantic metadata that is useful for a wide variety of purposes. The task of assigning keyphrases to a document is called keyphrase indexing. For example, academic papers are often accompanied by a set of keyphrases freely chosen by the author. In libraries professional indexers select keyphrases from a controlled vocabulary (also called Subject Headings) according to defined cataloguing rules. On the Internet, digital libraries, or any depositories of data (flickr, del.icio.us, blog articles etc.) also use keyphrases (or here called content tags or content labels) to organize and provide a thematic access to their data.</p> 
        <p>KEA is an algorithm for extracting keyphrases from text documents. It can be either used for free indexing or for indexing with a controlled vocabulary.</p> 
        <p>KEA is implemented in Java and is platform independent. It is an open-source software distributed under the GNU General Public License.</p>
       </blockquote> 
       <p>Google code project: <a href="https://code.google.com/p/kea-algorithm/">https://code.google.com/p/kea-algorithm/</a><br> Related Paper: <a href="http://www.cs.waikato.ac.nz/~ml/publications/2005/chap_Witten-et-al_Windows.pdf">KEA: Practical Automatic Keyphrase Extraction</a></p> 
       <p>Follow the Kea-5.0-Readme.txt, I compiled KEA and test it with an additional nlp.txt in the $KEAHome/testdocs/en/test which include the test text from the coursera nlp course. After run “java -Xmx526M TestKea”</p> 
       <p>Creating the model…<br> — Loading the Index…<br> — Building the Vocabulary index from SKOS file<br> — Reading the Documents…<br> Extracting keyphrases from test documents…<br> — Loading the Index…<br> — Building the Vocabulary index from SKOS file<br> — Extracting Keyphrases… </p> 
       <p>I got a nlp.key file which contains the extracted keywords for the nlp.txt:</p> 
       <p>Models<br> Computers<br> Data<br> Processing<br> Social groups<br> Methods<br> Uses</p> 
       <p>It seems not good, but give me a very good suggestion how to extract course text keywords.</p> 
       <p>6. <a href="https://github.com/turian/kea-service">kea-service</a>: KEA 5.0 (keyphrase extraction software), modified to be an XML-RPC service</p> 
       <p>This is XML-RPC service for Kea, and based the kea service server, you can use any code to communicate with KEA.</p> 
       <p>A related doc by the Author also provides a general introduction: <a href="http://metaoptimize.com/blog/2010/08/18/kea-keyphrase-extraction-as-an-xml-rpc-service/">KEA KEYPHRASE EXTRACTION AS AN XML-RPC SERVICE (CODE RELEASE)</a></p> 
       <p>7. <a href="https://pypi.python.org/pypi/topia.termextract/">topia.termextract</a></p> 
       <p>This package determines important terms within a given piece of content. It uses linguistic tools such as Parts-Of-Speech (POS) and some simple statistical analysis to determine the terms and their strength.</p> 
       <p>8. <a href="https://github.com/apresta/tagger">tagger</a>: A Python module for extracting relevant tags from text documents.</p> 
       <p>Here is a document to reference for me to find tagger and toipia.termextract: <a href="http://readwrite.com/2011/05/13/tagger">3 Open Source Tools for Auto-Generating Tags for Content</a></p>
