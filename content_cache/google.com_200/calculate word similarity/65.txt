          <h3><a name="semantic"></a>Semantic networks</h3> 
          <ul> 
           <li><a href="http://lcl.uniroma1.it/babelnet" target="_self">BabelNet&nbsp;</a>is a very large&nbsp;multilingual lexical database, developed by Roberto Navigli&nbsp;&nbsp;[<a href="mailto:navigli@di.uniroma1.it" target="_self">email</a>]&nbsp;, Simone&nbsp;Paolo Ponzetto&nbsp;and others at the University of Rome "La Sapienza" in&nbsp;the context of the&nbsp;<a href="http://lcl.uniroma1.it/multijedi" target="_self">ERC Starting Grant MultiJEDI</a>. BabelNet provides a wide-coverage&nbsp;knowledge repository in which WordNet is automatically aligned to the&nbsp;English Wikipedia, and lexicalizations for its concepts (i.e. English&nbsp;Wikipedia pages and WordNet synsets) are provided on the basis of&nbsp;Wikipedia cross-language links and the output of a machine&nbsp;translations system. The current version, which is freely available&nbsp;under a Creative Commons license, comprises a network of more than 3&nbsp;million synsets and 70 millions semantic relations, and it can also be&nbsp;browsed through an&nbsp;<a href="http://lcl.uniroma1.it/bnxplorer" target="_self">on-line interface</a>.</li> 
           <li>The <a href="http://www.globalwordnet.org/">Global WordNet Association</a> is a free, public and non-commercial organization that provides a platform for discussing, sharing and connecting wordnets for all languages in the world.</li> 
           <li>The <a href="http://gittens.nl/gittens/topics/SemanticNetworks.html" target="_self">Mimida Project</a>, developed by Maurice Gittens, is a WordNet-based mechanically-generated multilingual semantic network for more than 20 languages based on dictionaries found on the Web.</li> 
           <li><a href="http://multiwordnet.itc.it/">MultiWordNet</a>, developed by Luisa Bentivogli <a href="mailto:bentivo@ITC.IT">[email]</a> and others is a multilingual lexical database, developed at ITC-irst, in which the Italian WordNet is strictly aligned with Princeton WordNet 1.6. The current version includes around 44,400 Italian lemmas organized into 35,400 synsets which are aligned, whenever possible, with their corresponding English Princeton synsets. The MultiWordNet database can be freely browsed through its on-line interface, and is distributed both for research and commercial use. Information on the distribution licence is available at the web site.</li> 
           <li>The <a href="http://compling.hss.ntu.edu.sg/omw" target="_self">Open Multilingual WordNet</a> is a massively multilingual database&nbsp;that links many different WordNet projects, developed by Francis Bond [<a href="mailto:bond@ieee.org" target="_self">email</a>]&nbsp;at Nanyang Technological University. &nbsp;It currently links open-source&nbsp;wordnets in over 20 languages, and allows you to both look words up&nbsp;online or download the data.</li> 
          </ul> 
          <h3><a name="web"></a>Web Interfaces</h3> 
          <ul> 
           <li><a href="http://Wordvia.com" target="_self">Wordvia </a>is an online thesaurus is developed by <a href="mailto:sjp2e@yahoo.com" target="_self">Brandon Park</a>, based on Wordnet 3.0.</li> 
           <li><a href="http://www.omnilexica.com/" target="_self">Omnilexica</a>, developed by <a href="mailto:lucian.velea@lexica.ro" target="_self">Lucian Velea</a>, provides a complete interface for WordNet, search in almost all online and printed dictionaries, related photo/video content, pronunciation, examples, and other resources for every word and expression in English language.</li> 
           <li>As part of his research for his PhD thesis at the MAS laboratory, Ecole Centrale Paris, <a href="mailto:nicolas.james@gmail.com" target="_self">Nicolas James</a> has developed <a href="http://njames.trevize.net/wiki/projects:wnxplorer" target="_self">software for the visualization of WordNet</a>, exploring how WordNet may help for automatic image annotation. The software is based on Java and the JWI WordNet interface and works on Microsoft Windows and GNU/Linux systems and under GPL version 3. <a class="moz-txt-link-freetext" href="http://njames.trevize.net/wiki/projects:wnxplorer"><br> </a></li> 
           <li><a href="mailto:steven.vercruysse@nt.ntnu.no" target="_self">Steven Vercruysse</a>, from NTNU University in Trondheim, Norway, has developed an advanced <a href="http://wordvis.com" target="_self">web-interface</a> to browse the WordNet database. It looks and feels like the well-known Visual Thesaurus interface, but it is built on the latest web-technology (pure JavaScript, HTML5). It allows more user-interaction, and it is free to use.</li> 
           <li>A group of <a href="mailto:feedback@langua.de" target="_self">students in Hamburg</a>, Germany has created an <a href="http://langua.de/" target="_self">english-german dictionary</a>, featuring a intuitive interface with synonym, encyclopedia and related words search functions. It is written in PHP and uses a mySQL Database</li> 
           <li><a href="http://www.synonyms.net" target="_self">Synonyms.net</a> provides antonyms, translation to many languages, voice pronunciation, nearby terms and community related features that let users discuss language related issues and ask questions, as well as international versions of WordNet including Japanese, German and Arabic.</li> 
           <li><a href="http://www.beedictionary.com/" target="_self">Bee Dictionary</a> has created a Web 2.0 English Dictionary with audio, phonetics, usages and other informations using WordNet as base for definitions. Users can save the words in their personal wordlist and practice them as flash cards anytime.</li> 
           <li><a href="http://www.a2zDefined.com" target="_self">a2zDefined.com</a> is an online dictionary based on WordNet, developed by <a href="mailto:patrick@pjnation.com" target="_self">Patrick Phillips</a>. There is also an <a href="http://www.a2zDefined.com/iphone/" target="_self">iPhone dictionary</a> and a <a href="http://(http//www.google.com/ig/adde?moduleurl=www.a2zdefined.com/igoogle/dictionary.xml" target="_self">dictionary gadget</a> that can be added to igoogle.</li> 
           <li><a href="http://www.memidex.com/" target="_self">Mexidex</a> is a fast, simple, online dictionary and thesaurus based on WordNet 3.0. Added features include complete inflections, auto-suggest, adult-filtering, a browsable index, frequent updates, and support for small-screen, handheld/mobile devices. Developed by Serge Bohdjalian at Memidex [<a href="mailto:info@memidex.com" target="_self">email</a>].</li> 
           <li>Jack Dermody <a href="mailto:jack@dogbluesoftware.com" target="_self">[email]</a> has developed a new <a href="http://www.dogbluesoftware.com/wordnet" target="_self">web interface</a> that is quite fast, and has the following advantages  query expansion with automatic spelling corrections and the inclusion of the Princeton WordNet Gloss Corpus in the definitions.</li> 
           <li>Franz Chen <a href="mailto:franzchen@gmail.com">[email]</a> of OzyTek LLC has implemented a <a href="http://www.cozyenglish.com/dictionary">WordNet 3.0 interface</a> that ingtegrates with WordPress.</li> 
           <li>Szymanski Julian <a href="mailto:julszym@eti.pg.gda.pl">[email]</a> has developed an <a href="http://wordventure.eti.pg.gda.pl/">interactive visualization of WordNet</a> at Gdansk University of Technology.</li> 
           <li>A team of programmers at NIT Calicut, India have designed <a href="http://nalanda.nitc.ac.in/reference/general/wordnet/wordnet.html">a web interface to WordNet 2.0</a></li> 
           <li>Another simple <a href="http://www.golovchenko.org/cgi-bin/wnsearch">web interface</a> for WordNet 2.0 was developed by <a href="mailto:golovchenko@mail.ru">Nikolai Golovchenko</a>. It returns all of the search results on the same page, with hyperlinks on each word for easy browsing.</li> 
           <li>Philippe Martin <a href="mailto:philippe.martin@gu.edu.au">[email]</a> converted WordNet 1.7 nouns into an ontology browsable and exploitable for knowledge representation and knowledge sharing by the knowledge server <a href="http://www.webkb.org/">WebKB-2</a>.</li> 
           <li>A <a href="http://www.infres.enst.fr/%7Eyvon/Projets/Wapi/">WWW based Python interface</a> to WordNet developed by Francios Yvon at <a href="http://www.enst.fr/">ENST</a>.</li> 
           <li>A <a href="http://vancouver-webpages.com/wordnet/">"one-touch" interface</a> to WordNet 1.5 allows you to select multiple searches at one time. This was developed by Andrew Daviel at <a href="http://vancouver-webpages.com/">Vancouver Webpages</a>.</li> 
           <li>An <a href="http://www-inf.enst.fr/%7Emilc/wordnet.html">interactive CGI interface</a> from which you can select many searches at once was developed at E.N.S.T in France by Francois Yvon, <a href="http://www-inf.enst.fr/%7Everna/">Didier Verna</a> and several undergraduate students. This interface is in French.</li> 
           <li>A hyper-dictionary, <a href="http://www.hyperdic.net/">HyperDic Online</a>, was developed by Eric Kafe <a href="mailto:kafe@megadoc.net">[email]</a>.</li> 
           <li><a href="mailto:laplante@lookwayup.com">Martin Laplante [mailto]</a> has developed <a href="http://lookwayup.com/free/dictionary.htm">LookWAYup, an online dictionary based on WordNet and other soures</a>. It translates between English and French, Spanish, German, Dutch, or Portuguese. It uses morphology and corrects misspellings. It presents related terms as well as cousins and generates usage sentences from frames. Can be used online or plugs into Netscape, IE, Opera, Palm, various PDAs and mobile phones.</li> 
           <li><a href="http://worddetail.org" target="_self">WordDetail</a> is a collection of free and useful tools (also a thesaurus reference resource) for finding the definitions, synonyms and antonyms of a word. These tools are based on WordNet. The website also features abbreviation meaning finder and Scrabble word game solver.</li> 
           <li>Chris Cargile has developed three separate interfaces for retrieving WordNet data in <a href="http://chriscargile.com/dictionary/search.php?q=lemma" target="_self">HTML</a>, <a href="http://chriscargile.com/dictionary/toxml/lemma.xml" target="_self">XML</a> and <a href="http://chriscargile.com/dictionary/tojson/lemma.js" target="_self">JSON</a> (with links to example results shown). Further details on the interfaces and the structure of the results available through them is provided <a href="http://chriscargile.com/dictionary/develop.html" target="_self">here</a>.</li> 
           <li><a href="http://finedictionary.com" target="_self">FineDictionary.com</a> includes definitions from 4 dictionaries (WordNet,<br> Webster's Revised Unabridged Dictionary, Century Dictionary and<br> Cyclopedia and Chambers's Twentieth Century Dictionary), illustrations,<br> example sentences, interesting facts, quotations, idioms and etymology<br> as well as related words (synonyms, hypernyms, hyponyms).</li> 
          </ul> 
          <h4><a name=".NET"></a>.NET/C#</h4> 
          <ul> 
           <li><a href="mailto:gerber.matthew@gmail.com" target="_self">Matt Gerber</a> of Michgan State University has written a C# API for WordNet 3.0. You can read about it <a href="http://ptl.sys.virginia.edu/ptl/software" target="_self">here</a>. The API's most distinguishing feature is the option to use either in-memory or disk-based retrieval of the WordNet data. The former is essential for high-performance applications where plenty of memory is available (~200MB). The latter uses essentially no memory and is great for small-footprint applications (it is slower, of course).</li> 
           <li>François-Régis Chaumartin <a href="mailto:frc@proxem.com" target="_self">[email]</a> (frc at proxem.com) has developed at Proxem(<a class="moz-txt-link-abbreviated" href="http://www.proxem.com/">www.proxem.com</a>) the Antelope .NET linguistic framework. Antelope uses and extends WordNet; it also includes a subset of a French WordNet and several taggers and parsers. It also implements many out-of-the-box algorithms: similarity measures, Word Sense Disambiguation, Semantic Role Labeling and anaphora resolution. Antelope works under Windows (.NET) or Linux (MONO). It is free for research and education.</li> 
           <li>Troy Simpson <a href="mailto:troy@ebswift.com">[email]</a> has developed a C# interface to WordNet based on Malcolm Crowe's original port, but using WordNet 3.0. It requires the .NET framework and can be downloaded <a href="http://www.ebswift.com/wordnetnet.html" target="_self">here</a>.</li> 
           <li>Derik J. Palacino <a href="mailto:derik@palacino.net">[email]</a> has developed a C# interface. This project was built on the database files for WordNet 3.0 however the code was translated from version 2.1. It can be downloaded as a <a href="http://wordnet.princeton.edu/tools/WordNet_Binary.zip">Windows binary</a> or <a href="http://wordnet.princeton.edu/tools/WordNet_Source.zip">as source code</a>.</li> 
          </ul> 
          <h4><a name="COM"></a>COM</h4> 
          <ul> 
           <li><a href="http://wntw.sourceforge.net/">WordNet TreeWalk</a>, an interface to WordNet based on tree-views developed by Bernard Bou <a href="mailto:bbou@ac-toulouse.fr">[email]</a>, Lycée Champollion, France, has been updated to support WordNet 3.0. A <a href="http://wncom.sourceforge.net/">COM component object</a> that can also be accessed from COM-enabled languages (scripting languages, and C++).</li> 
          </ul> 
          <h4><a name="Java"></a>Java</h4> 
          <ul> 
           <li> 
            <div>
             extJWNL (Extended Java WordNet Library), developed by 
             <a href="mailto:avtaev@gmail.com" target="_self">Aliaksandr Autayeu</a>, is a Java API for creating, reading and updating dictionaries in WordNet format. extJWNL is an upgraded version of JWNL. Open source, Java, available at&nbsp;
             <a href="http://extjwnl.sourceforge.net/">http://extjwnl.sourceforge.net/</a>.
            </div> </li> 
           <li><a href="mailto:ferguson@cs.rochester.edu" target="_self">University of Rochester Computer Science Department</a> has released its <a href="http://www.cs.rochester.edu/research/cisd/wordnet" target="_self">URCS WordNet Browser</a>, a cross-platform browser for the WordNet 3.0 database. Implemented in Java, the browser runs on Windows, Mac OSX, and Linux (or any other platform that supports Java), and includes platform-specific customizations that make the user experience as natural as possible. The non-browser classes may have pedagogical value for others interested in using WordNet from Java.</li> 
           <li><a href="http://wnejb.sourceforge.net/" target="_self">WNEJB</a>, by Bernard Bou [<a href="mailto:bbou@ac-toulouse.fr" target="_self">email</a>], provides scalable, fault-tolerant, distributed access to WordNet in a Java Enterprise context. A session EJB is the favored interface for queries that return XML strings or serialized W3C documents. WNEJB comes with a deployment wizard, XML servlet and GUI browser, and must be hosted on a JBoss 5 or 6 application server.</li> 
           <li>Bernard Bou <a href="mailto:bbou@ac-toulouse.fr">[email]</a> has written <a href="http://wnjn.sourceforge.net/">WNJN</a>, a JNI-compliant Java Native library that interfaces the WordNet database (1.7.1 and 2.0) to Java programs. It offers Java classes that interface with binary code (written in C++) and provides query and search funcationality.</li> 
           <li>A new version of <a href="http://sourceforge.net/projects/jwordnet">JWNL (Java WordNet Library)</a>, written by John Didion <a href="mailto:jdidion@didion.net">[email]</a>, has been released. It is a Java API for accessing WordNet, and provides API-level access to WordNet data. It is pure Java (uses no native code), so it is completely portable, and all of the source code is available.</li> 
           <li><a href="http://projects.csail.mit.edu/jwi">JWI</a> (the MIT Java Interface to Wordnet) is a small, fast, full-featured, easily-extensible Java library that supports access to Wordnet 1.6 through 3.0 as well as other Wordnet variants. The library allows mounting dictionary files directly on-disk for a small memory footprint, or loading all of Wordnet into memory for lightning-quick access. The library is compatible with Java 1.5 and up, and is distributed under a non-commercial license with full source code and a short users' guide. JWI was developed by <a href="mailto:markaf@alum.mit.edu">Mark Finlayson</a>.</li> 
          </ul>

          <h4><a name="SQL"></a>SQL</h4> 
          <ul> 
           <li><a href="http://wnscope.sourceforge.net/" target="_self">WordNetScope</a>, by Bernard Bou [<a href="mailto:bbou@ac-toulouse.fr" target="_self">email</a>], is an easy-to-use cross-platform browser application for the WordNet SQL database (MySql, PostgreSql, SQlite,...) that displays results as trees. Standard query capabilities extend beyond words to other WordNet entities such as senses, synsets, semlinks, lexlinks, etc. Behind the scenes, the Hibernate framework maps relational data to objects.</li> 
           <li><a href="http://wnsql.sourceforge.net/" target="_self">WNSQL</a>, by Bernard Bou [<a href="mailto:bbou@ac-toulouse.fr" target="_self">email</a>], provides a ready-to-use WordNet SQL database in MySQL, PostgresSQL, Sqlite, HsqlDB (cached and memory tables) and Derby.</li> 
           <li><a href="http://wnpojo.sourceforge.net/" target="_self">WNPOJO</a>, by Bernard Bou [<a href="mailto:bbou@ac-toulouse.fr" target="_self">email</a>], is a library of Plain Old Java Objects to encapsulate WordNet entitites (word, sense, synset, etc.) and handle them transparently without having to manage SQL queries, using the Hibernate framework. Comes with a programming sample.</li> 
           <li>A companion project to WNPOJO, <a href="http://wntrans.sourceforge.net/" target="_self">WNTRANS</a> comes with a GUI wizard and currently supports MySQL, PostgresSQL, Sqlite, HsqDB, and Derby, but this set is extensible. WNTRANS provides portability to the WordNet SQL database through the Hibernate framework.</li> 
           <li>[MySQL, PostgreSQL] A <a href="http://sourceforge.net/project/showfiles.php?group_id=135112&amp;package_id=219735"> ready-to-use SQL database</a> that unifies WordNet 3.0, WordNet 2.0-2.1, 2.1-3.0, 2.0-3.0 sensemaps, VerbNet 2.3, XWordNet 1.1 compiled by Bernard Bou <a href="mailto:bbou@ac-toulouse.fr">[email]</a> supports both MySQL and PostgreSQL.</li> 
           <li>[MySQL, PostgreSQL] <a href="http://wnsqlbuilder.sourceforge.net/"> WNSQLBUILDER</a> is a Java tool by Bernard Bou <a href="mailto:bbou@ac-toulouse.fr"> [email] </a> to build a unified SQL database (see above) from the WordNet, WN sensemaps, VerbNet, XWordnet,, BNC and SUMO standard releases.</li> 
           <li>[MySQL] Android Technologies, Inc. provides <a href="http://www.androidtech.com/html/wordnet-mysql-20.php">a MySQL version of WordNet 2.0</a> converted from the Prolog files.</li> 
           <li>[MySQL, dBase] <a href="http://www.semantilog.org/wn2sql.html" target="_self">wordnet2sql</a>, developed by Richard Bergmair <a href="mailto:rb@semantilog.org" target="_self">[email]</a> is a system that converts WordNet 3.0 into SQL.</li> 
          </ul>

          <h3><a name="extensions"></a>Extensions</h3> 
          <div> 
           <ul> 
            <li><a href="http://wn-similarity.sourceforge.net/" name="Similarity">WordNet::Similarity</a>, developed by Ted Pedersen&nbsp;<a href="mailto:tpederse@d.umn.edu">[email]</a>, is an open source Perl module for measuring the semantic distance between words. It provides a number of measures of semantic similarity and semantic relatedness based on WordNet. Given two synsets, it will return a numeric score showing their degree of similarity/relatedness according to various measures that all rely on WordNet in different ways. It provides support for estimating information content values from untagged corpora, including plain text, the Penn Treebank, or the BNC. The mailing list is found at&nbsp;<a href="http://groups.yahoo.com/group/wn-similarity/">http://groups.yahoo.com/group/wn-similarity/</a>, and a<a href="http://www.d.umn.edu/%7Etpederse/wnsim-bib/">bibliography</a>&nbsp;of papers and publications that make use of WordNet::Similarity is maintained.</li> 
            <li><a href="https://chrome.google.com/webstore/detail/wordquake/kkgjeaicdkamcegnchfopjdflekijpbo" target="_blank">WordQuake</a> is a Chrome extension that transforms webpages using WordNet as a source of creative new words. From the author, <a href="mailto:aydu@princeton.edu" target="_self">Alan Du</a>, "Tired of what you see on a webpage? Want to spice things up? Or surprise your friends? Introducing WordQuake, an application that transforms entire webpages before your very eyes. Create new meanings from online news articles, editorials, classic literature...even your best friend's Facebook posts." Alan developed WordQuake with two other Princeton undergraduate students,&nbsp;Keith Gladstone (<a href="mailto:keithag@princeton.edu">keithag@princeton.edu</a>) and Derrick Xin (<a href="mailto:dxin@princeton.edu">dxin@princeton.edu</a>).</li> 
            <li>WordNet::Similarity requires a local copy of WordNet as well as the&nbsp;<a href="http://search.cpan.org/dist/WordNet-QueryData/">WordNet::QueryData</a>&nbsp;Perl module, which provides a direct Perl interface to the WordNet database. Author Jason Rennie also has&nbsp;<a href="http://www.ai.mit.edu/%7Ejrennie/WordNet/">other WordNet tools</a>&nbsp;on his site.</li> 
            <li>"<a href="http://www.certifiedchinesetranslation.com/openaccess/WordNet/" target="_self">A Complete&nbsp;Multilingual WordNet List by Language</a>" was compiled by&nbsp;<a href="mailto:schong2@go.pasadena.edu" target="_self">Samuel Chong</a>&nbsp;of&nbsp;Pasadena City College.</li> 
            <li>An&nbsp;<a href="http://kask.eti.pg.gda.pl/semagloss/index.html">API to Semantically annotaed gloss corpus</a>, developed by Julian Szymanski <a href="mailto:julian.szymanski@eti.pg.gda.pl">[email]</a> and others, is set of libraries in C# and Java to access <a href="http://wordnet.princeton.edu/glosstag.shtml"> corpus of Semantically Tagged glosses</a>.</li> 
            <li>Indio, by <a href="mailto:dirkholland@rocketmail.com" target="_self">Dirk Holland</a>, is a Microsoft Word Add In based on Wordnet 3.0. Simply highlight a word in a document to get its meaning. Words can also be typed in via the form. Indio also includes wild card searches and other advanced features such as the ability to click on lexical or semantic relations to get their definitions. Indio also includes definitions from Webster's 1913 dictionary. Availalbe via aemail directly from the author..</li> 
            <li>GeoWordNet is a semantic resource built from the full integration of WordNet, GeoNames and the Italian part of MultiWordNet. GeoWordNet Public Dataset contains 3,698,238 entities, 3,698,237 part-of relations between entities, 334 concepts, 182 relations between concepts, 3,698,238 relations between instances and concepts, and 13,562 (English and Italian) alternative entity names. Licensed as CC-BY and freely available at&nbsp;<a href="http://geowordnet.semanticmatching.org/">http://geowordnet.semanticmatching.org/.</a></li> 
            <li><a href="mailto:the.yangist@gmail.com" target="_self">Joshua Harwood</a> has&nbsp;produced a <a href="http://www.flashcardexchange.com/flashcards/view/1628943" target="_self">spreadsheet</a> from WordNet which makes a frequency list for some entries (selected from other frequency corpora), and the tokens that are contained within the most frequently occurring entries.&nbsp;</li> 
            <li><a href="http://www.embedplus.com/dictionary/how-to-pronounce-words-and-use-them.aspx">How do you pronounce English words and use them?</a> Check out the <a href="http://www.embedplus.com">EmbedPlus YouTube embed tool</a>, which uses WordNet definitions and word metadata for its video pronunciation API for dictionaries. Start here by watching videos of <a href="http://www.embedplus.com/dictionary/how-to-pronounce-gyro-and-use-it">how to pronounce gyro</a>, <a href="http://www.embedplus.com/dictionary/how-to-pronounce-salmon-and-use-it">salmon</a>, and <a href="http://www.embedplus.com/dictionary/how-to-pronounce-meme-and-use-it">meme</a>; three cases of often mispronounced words. <a href="mailto:ext@embedplus.com" target="_self">Contact</a> them for more info.</li> 
            <li><a href="mailto:adampease@earthlink.net" target="_self">Adam Pease</a> has announced that WordNet is now available in OWL with links to the <a href="http://www.ontologyportal.org" target="_self">Suggested Upper Merged Ontology</a> as a <a href="http://www.ontologyportal.org/WordNet.owl" target="_self">single large OWL file</a> and dynamically served as OWL pages comprising single synset at a time (e.g.<a href="http://sigma.ontologyportal.org:4010/sigma/OWL.jsp?kb=SUMO&amp;term=WN30-100002684" target="_self">physical_object</a>)</li>

            <li><a href="http://22frames.com/">22frames.com</a> provides an API that electronic dictionaries can use to embed video-based word usage examples in their definition pages. &nbsp; Given a word in WordNet, it finds one or more captioned video examples in which a speaker uses it. The idea extends that of audio pronunciations and sentence examples found in many dictionaries except it adds visual and situational context using social media. &nbsp; They have created a search interface for learning more about the idea <a href="http://www.22frames.com/dictionary" target="_self">here</a>. &nbsp; <a href="mailto:info@22frames.com" target="_self">Email</a>&nbsp;them for API access.</li> 
            <li><a href="http://artha.sourceforge.net/" target="_self">Artha</a> is a cross-platform open thesaurus with WordNet as its database for Linux/Unix-like systems. Once the application is launched it sits on the system tray monitoring for a set hot key combination. The user can select text on any window, and call Artha by pressing the set hot key combination. It can also show notification of the selected word's prime definition. Artha is developed and maintained by Sundaram Ramaswamy <a href="mailto:legends2k@yahoo.com" target="_self">[email]</a>.</li> 
            <li><a href="http://www.dsic.upv.es/grupos/nle/resources/geo-wn/download.html">Geo-WordNet</a> extends WordNet with geographic coordinates extracted from Wikipedia by Davide Buscaldi <a href="mailto:dbuscaldi@dsic.upv.es">[email]</a>.</li> 
            <li><a href="http://www.seas.gwu.edu/~simhaweb/software/jword/index.html" target="_self">JWord</a> is a WordNet-inspired tool that integrates other word sources such as Roget's Thesaurus. It includes a simple browser and, for developers, the ability to integrate other sources.</li> 
            <li>Panagiotis Toulis <a href="mailto:panefsky@gmail.com" target="_self">[email]</a> has developed a <a href="http://developeronline.blogspot.com/2008/04/perl-by-example-english-dictionary-in.html">cool little Perl interface</a> that does a WordNet lookup.</li> 
            <li><a href="http://www.defineitfast.com/" target="_self">DefineItFast</a> is an AJAX &amp; PHP Driven online dictionary that is powered by Wordnet 3.0.             <li><a href="http://www.cogs.susx.ac.uk/users/drh21/" target="_self">Java WordNet::Similarity</a>, developed by David Hope [<a href="mailto:davehope@gmail.com" target="_self">email]</a> at Sussex University, is a pure Java version of Perl WordNet::Similarity (developed by Ted Pedersen). The code supports all of the measures found in the Perl version.</li> 
            <li>Armando Stellato has written <a href="http://ai-nlp.info.uniroma2.it/software/OntoLing/">a plugin called OntoLing</a> for the Ontology editing tool Protégé It allows the user to: browse linguistic resources, enrich ontologies with elements from these resources, and build new ontologies starting from existing resources. The WordNetInterface for this tool is based on the <a href="http://wordnet.princeton.edu/links#Java">JWNL</a> library, and is compatible with WordNet 1.6 through 2.0.</li> 
            <li>Bernard Bou <a href="mailto:bbou@ac-toulouse.fr">[email]</a> has written <a href="http://wnwa.sourceforge.net/">WNWA</a>, a WordNet Web Application to be hosted on a Java Enterprise Bean Container (typically JBoss). It includes an Enterprise Java Bean that interfaces with the WordNet SQL database. Tomcat front-end servlets offer 4 renderings of query results: xml document, HTML, DHTML tree, hyperbolic tree. <a href="http://jws-champo.ac-toulouse.fr:8080/wordnet/">A demo site</a> with limited hardware/bandwidth capabilities has been deployed, but its continued existence is not guaranteed.</li> 
            <li>Nuno Seco <a href="mailto:nuno.seco@ucd.ie">[email]</a> of University College Dublin, has created a <a href="http://lucene.apache.org/">Lucene</a> index of WordNet. It allows extremely fast and flexible searches of the WordNet database. The <a href="http://eden.dei.uc.pt/%7Enseco/lwwn.tar.gz">index, related libraries, and a short readme</a> are available for download.</li> 
            <li>Tony Veale of University College Dublin is heading KNOW-BEST, which is attempting to use WordNet and other lexical ontologies as the generative basis for on-line language games. Playable samples of the games and information/papers related to their construction/research issues are available at <a href="http://afflatus.ucd.ie/">The Creative Language Systems Group web site</a>.</li> 
            <li>Keith Alcock has developed <a href="http://www.keithalcock.com/">the WordNet Relationship Browser</a>, a Windows program that allows a user to browse related words through the various links in WordNet. Each path thus created is given a score based on user-configurable weights. It is based on a version of WordNet contained in a .dll file, and is thus extremely fast.</li> 
            <li>Nuno Seco <a href="mailto:nuno.seco@ucd.ie">[email]</a> of University College Dublin, has developed <a href="http://eden.dei.uc.pt/%7Enseco/javasimlib.tar.gz">an Information Theoretic model for computing semantic similarity</a> that does not require any resources other than WordNet. Georgios Tagalakis also provides <a href="http://www.cs.ucd.ie/students/tagalakis/home/scientific_work.html">a Prolog implementation</a>.</li>

There is a MacOS X GUI application with Sicstus Prolog embedded in it. It generates graphs of the connections. There is also a TTY version with just the Prolog and C files. You do need Prolog for this version.</li>

WordNet is used as a thesaurus, to help users find synonyms and alternate words for their search terms. <a href="mailto:norm.friesen@ualberta.ca">Email Norm Friesen</a> for more information.</li>

          <h3><a name="mappings"></a>Mappings</h3> 
          <ul> 
           <li>For official mappings, see the <a href="/wordnet/download/" target="_self">WordNet download page</a>.</li> 
           <li>The <a href="http://www.lsi.upc.es/%7Enlp/">Natural Language Research Group</a> in Spain provides <a href="http://www.lsi.upc.edu/~nlp/web/index.php?option=com_content&amp;task=view&amp;id=21&amp;Itemid=57" target="_self">mappings for all-to-all pairs of WordNet versions from 1.5 to 3.0, both forward and backward</a> (follow Resources-&gt;Mapping).
