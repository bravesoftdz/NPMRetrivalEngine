           <p name="c14f" id="c14f" class="graf graf--p graf-after--h3">This is the first in a series of articles that I’ll be writing where we will learn how to develop and deploy Actions on Google. Actions power the Google Home and Google Assistant and allow users to interact with your application via voice or text. The goal is to make this process fun and easy for any developer level. Actions can currently be used on Google Home devices or in the Google Assistant app on Google Pixel phones. This tutorial will focus on the voice interface.</p>
           <p name="de9d" id="de9d" class="graf graf--p graf-after--p">We will be developing a simple random room generator called Three Doors. If you want to interact with it now and you have your Google Home nearby, simply say, “<strong class="markup--strong markup--p-strong">Hey Google I want to talk to Crazy House</strong>”.</p>
           <p name="816e" id="816e" class="graf graf--p graf-after--p">In this first article, the goal is to get you up and running in as little time as possible so that you can run your Action in test mode. In future articles, we will dive deeper into the code, learn how to test and deploy your action, and use other tools like API.ai. If you have any problems or suggestions for future articles, reach out to me <a href="https://twitter.com/eisenzopf" data-href="https://twitter.com/eisenzopf" class="markup--anchor markup--p-anchor" rel="noopener nofollow" target="_blank">@eisenzopf</a> on twitter. For now, time is ticking so let’s get started.</p>

           <h3 name="d347" id="d347" class="graf graf--h3 graf-after--blockquote">How Google Actions&nbsp;work</h3>
           <p name="d5c1" id="d5c1" class="graf graf--p graf-after--h3">Without getting too detailed, the way a Google Home device works is that it captures your voice intent on the Google Action Platform and translates this to text.</p>

           <ol class="postList">
            <li name="a21e" id="a21e" class="graf graf--li graf-after--figure">The Google Action Platform attempts to match the Intent to an Action in the directory and then to an Action endpoint. As a developer, when you’re ready to deploy an action, there is a Google registration process where you register your action intent name, like <strong class="markup--strong markup--li-strong">Three Doors</strong>. The Google Action Platform then know what Action to send the request to. Once registered, the Google Action Platform makes requests to your server side application via an HTTP POST request and sends the user’s input via a JSON document.</li>
            <li name="8d1a" id="8d1a" class="graf graf--li graf-after--li">Your application decodes the user’s inputs in a Node.js or other server side application, and sends back a JSON response, which will play content back to the user or ask for input. In the case above, we’re prompting the user to pick a door.</li>
            <li name="e639" id="e639" class="graf graf--li graf-after--li">The users says <strong class="markup--strong markup--li-strong">three</strong> which is sent to Google, translated to text, and sent on to the server side application, which takes the input and responds with another JSON document, which could be the next selection or end the dialog.</li>
           </ol>

           <h3 name="592c" id="592c" class="graf graf--h3 graf-after--p">Install the gcloud CLI and create a new Google Action&nbsp;project</h3>
           <p name="febc" id="febc" class="graf graf--p graf-after--h3">Now that you have a Google Cloud account, we are going to install the Google Cloud CLI called <code class="markup--code markup--p-code">gcloud</code> on our local machine and use it to create a new Google Action project.</p>
           <p name="1bbb" id="1bbb" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">1.</strong> To install the gcloud CLI, we must first install the Google Cloud SDK. There are several different installation options, which are all documented at <a href="https://cloud.google.com/sdk/downloads" data-href="https://cloud.google.com/sdk/downloads" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">https://cloud.google.com/sdk/downloads</a>. The easiest way is to run the following commands:</p>
           <pre name="9049" id="9049" class="graf graf--pre graf-after--p"><code class="markup--code markup--pre-code">curl https://sdk.cloud.google.com | bash<br>exec -l $SHELL<br>gcloud init</code></pre>
           <p name="e22a" id="e22a" class="graf graf--p graf-after--pre">You will be asked a number of questions that are straightforward and once done, the <code class="markup--code markup--p-code">gcloud</code> will be ready to go.</p>
           <p name="51bc" id="51bc" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">2.</strong> Now we will create a new Google Action project using the <code class="markup--code markup--p-code">gcloud</code> CLI from a terminal window from within your project directory. Run:</p>
           <pre name="7533" id="7533" class="graf graf--pre graf-after--p"><code class="markup--code markup--pre-code">gcloud alpha projects create three-doors-xxxx</code></pre>
           <blockquote name="ed8d" id="ed8d" class="graf graf--blockquote graf-after--pre">
            <em class="markup--em markup--blockquote-em">Note: Replace </em>
            <code class="markup--code markup--blockquote-code"><em class="markup--em markup--blockquote-em">xxxx</em></code>
            <em class="markup--em markup--blockquote-em"> with four random numbers of your choosing.</em>
           </blockquote>
           <p name="b3ab" id="b3ab" class="graf graf--p graf-after--blockquote">This will create a new project within your Google Cloud account. This is an alpha command and may change. You can alternatively create a new project in the <a href="https://console.cloud.google.com" data-href="https://console.cloud.google.com" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Google Cloud Console GUI</a>.</p>
           <p name="6e50" id="6e50" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">3.</strong> To confirm the project was created, run:</p>
           <pre name="9d1d" id="9d1d" class="graf graf--pre graf-after--p"><code class="markup--code markup--pre-code">gcloud projects list</code></pre>
           <p name="40e9" id="40e9" class="graf graf--p graf-after--pre">You should see your newly created project on the list in addition to a default project that was created when you ran <code class="markup--code markup--p-code">cloud init</code> command.</p>

           <h3 name="6ad0" id="6ad0" class="graf graf--h3 graf-after--p">Install and initialize gaction&nbsp;CLI</h3>
           <p name="0adb" id="0adb" class="graf graf--p graf-after--h3">Now its time to get <code class="markup--code markup--p-code">gaction</code> environment set up. This is a command line tool you’ll run from a terminal that deploys your <code class="markup--code markup--p-code">action.json</code> file. This is the file that, once published and approved by Google, enables <strong class="markup--strong markup--p-strong">Google Assistant</strong> enabled devices such as <strong class="markup--strong markup--p-strong">Google Home</strong> or <strong class="markup--strong markup--p-strong">Allo</strong> on Android or iOS to access your <strong class="markup--strong markup--p-strong">Google Action</strong> application.</p>
           <p name="e354" id="e354" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">1.</strong> First point your browser to <a href="https://developers.google.com/actions/tools/gactions-cli" data-href="https://developers.google.com/actions/tools/gactions-cli" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">https://developers.google.com/actions/tools/gactions-cli</a> and download the gactions command-line tool.</p>
           <blockquote name="71a9" id="71a9" class="graf graf--blockquote graf-after--p">
            <em class="markup--em markup--blockquote-em">Shortcut from a terminal window, in your project directory, you can download directly using </em>
            <code class="markup--code markup--blockquote-code"><em class="markup--em markup--blockquote-em">curl</em></code>
            <em class="markup--em markup--blockquote-em">.</em>
           </blockquote>

            <em class="markup--em markup--blockquote-em">On Linux:</em>

           <p name="53ea" id="53ea" class="graf graf--p graf-after--pre">This will create a default <code class="markup--code markup--p-code">action.json</code> file in your project directory. Now we need to edit it.</p>
           <h3 name="9973" id="9973" class="graf graf--h3 graf-after--p">Edit the action.json</h3>
           <p name="0dad" id="0dad" class="graf graf--p graf-after--h3">Open <code class="markup--code markup--p-code">action.json</code> in your favorite editor.</p>
           <p name="c0d7" id="c0d7" class="graf graf--p graf-after--p">Change the <strong class="markup--strong markup--p-strong">versionLable</strong> to 1.0 and change the <strong class="markup--strong markup--p-strong">projectId</strong> to three-doors-xxxx (replace with the project id you used when creating the project with <code class="markup--code markup--p-code">gcloud</code> CLI)</p>
           <pre name="9f92" id="9f92" class="graf graf--pre graf-after--p"><code class="markup--code markup--pre-code">{<br>  "versionLabel": "1.0",<br>  "agentInfo": {<br>    "languageCode": "en-US",<br>    "projectId": "three-doors-xxxx",<br>    "voiceName": "male_1"<br>  },</code></pre>
           <p name="8cbc" id="8cbc" class="graf graf--p graf-after--pre">When edited, it should look something like the above.</p>
           <p name="258b" id="258b" class="graf graf--p graf-after--p">Next we’re going to edit <strong class="markup--strong markup--p-strong">actions</strong> section of the <code class="markup--code markup--p-code">action.json</code> file, which contains the list of possible thinks that the application will recognize. Every Google Assistant <code class="markup--code markup--p-code">action.json</code> will have an <code class="markup--code markup--p-code">initialTrigger</code> section. This is the action that will always be called first, so it’s also the most important one to get correct.</p>
           <p name="fa5a" id="fa5a" class="graf graf--p graf-after--p">In this initial <code class="markup--code markup--p-code">action.json</code> file that was created with the <code class="markup--code markup--p-code">actions init</code> command, there are two <code class="markup--code markup--p-code">initialTrigger</code> sections. That is because an application can be called in multiple ways. We’re not going to worry about that in our first application so delete the second <code class="markup--code markup--p-code">initialTrigger</code> section.</p>
           <p name="cdad" id="cdad" class="graf graf--p graf-after--p">The <code class="markup--code markup--p-code">actions</code> portion of our <code class="markup--code markup--p-code">action.json</code> file should now look similar to the following:</p>
           <pre name="5cda" id="5cda" class="graf graf--pre graf-after--p"><code class="markup--code markup--pre-code">"actions": [<br>    {<br>      "description": "Launch intent",<br>      "initialTrigger": {<br>        "intent": "assistant.intent.action.MAIN"<br>      },<br>      "httpExecution": {<br>        "url": "https://us-central1-three-doors-xxxx.cloudfunctions.net/three_doors"<br>      }<br>    }<br>  ]</code></pre>
           <blockquote name="0a80" id="0a80" class="graf graf--blockquote graf-after--pre">
            <em class="markup--em markup--blockquote-em">Note: Replace </em>
            <code class="markup--code markup--blockquote-code"><em class="markup--em markup--blockquote-em">xxxx</em></code>
            <em class="markup--em markup--blockquote-em"> with the four numbers you chose above.</em>
           </blockquote>
           <h3 name="8043" id="8043" class="graf graf--h3 graf-after--blockquote">Deploy your application to Google Cloud Functions</h3>
           <pre name="737a" id="737a" class="graf graf--pre graf-after--h3"><code class="markup--code markup--pre-code">gcloud beta functions deploy three_doors --trigger-http --stage-bucket gs://three-doors-xxxx</code></pre>
           <blockquote name="8e9b" id="8e9b" class="graf graf--blockquote graf-after--pre">
            <em class="markup--em markup--blockquote-em">Note: Replace </em>
            <code class="markup--code markup--blockquote-code"><em class="markup--em markup--blockquote-em">xxxx</em></code>
            <em class="markup--em markup--blockquote-em"> with the four numbers you chose above.</em>
           </blockquote>
           <p name="64a5" id="64a5" class="graf graf--p graf-after--blockquote">This will deploy your Google Function, which is contained in the <code class="markup--code markup--p-code">index.js</code> source file (we’ll talk about that next time), into the Google Cloud. If this is the first time you’ve deployed the application, it will take some time; as much as 5 minutes. Don’t worry, updates don’t take nearly as long so it’s worth the wait of not having to manage your own servers.</p>
           <h3 name="d3d0" id="d3d0" class="graf graf--h3 graf-after--p">Deploy your action.json test&nbsp;file</h3>
           <p name="d2a8" id="d2a8" class="graf graf--p graf-after--h3">Next, it’s time to deploy your Action file (action.json). Use the command below. If this is the first time running the command, you may see instructions to enter a security token.</p>
           <pre name="a65c" id="a65c" class="graf graf--pre graf-after--p"><code class="markup--code markup--pre-code">./gactions preview --action_package action.json --invocation_name "three doors" --preview_mins 1234</code></pre>
           <p name="1035" id="1035" class="graf graf--p graf-after--pre">Running the <code class="markup--code markup--p-code">gactions</code> command above will turn on your new Action temporarily. Only you will be able to test it. This allows you to test the application in the command-line or web simulator or on your Google Home device. The google email you used to set up your cloud project will need to be the same account that manages your Google Home device.</p>
           <h3 name="f0a0" id="f0a0" class="graf graf--h3 graf-after--p">Test your&nbsp;action</h3>
           <pre name="1f1f" id="1f1f" class="graf graf--pre graf-after--h3"><code class="markup--code markup--pre-code">./gactions simulate</code></pre>
           <p name="88df" id="88df" class="graf graf--p graf-after--pre">In the simulator, type:</p>
           <pre name="1f2e" id="1f2e" class="graf graf--pre graf-after--p"><code class="markup--code markup--pre-code">talk to three doors</code></pre>
           <p name="ac15" id="ac15" class="graf graf--p graf-after--pre">This should start the dialog.</p>
           <p name="42ef" id="42ef" class="graf graf--p graf-after--p">When you run the command, there is also an option to start a web based simulator that will play the audio. You may be prompted to enable audio on your google gmail account.</p>
           <h3 name="1276" id="1276" class="graf graf--h3 graf-after--p">Congratulations!</h3>
           <p name="1cdc" id="1cdc" class="graf graf--p graf-after--h3">You just set up and deployed your first Action on Google. I’ve included a list of references below. In the next article in the series, we will look at the code and understand more how it works.</p>
