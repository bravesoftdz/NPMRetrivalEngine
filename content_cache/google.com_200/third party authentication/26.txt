        <hr>

        <hr> 
        <p>Livefyre Apps include rich interactive function like posting a comment, writing a review, or liking content. &nbsp;In order to have your users interact with Livefyre Apps, youll need to integrate Livefyre with an identity service using Livefyre.js Auth.</p> 
        <p>Livefyre.js Auth provides centralized authentication for all Livefyre Apps across your site and puts you in charge of exactly how users should log in and register. Add Auth globally to your sites template and use it on all pages, or add it once per page, and when your users have signed in Livefyre JS Auth will automatically broadcast the users information to all Apps on the page.</p> 
        <p>Whether youve built a custom identity service or are using a third-party identity service like Livefyre Enterprise Profiles (LFEP) or Janrain Capture, this section covers everything you need to know to integrate.</p> 
        <ul> 
         <li><a href="#AuthPackage">Auth Package Overview</a></li> 
         <li><a href="#AuthDelegateObject">Building an Auth Delegate</a></li> 
         <li><a href="#PostingUserPermissions">Posting User Permissions to&nbsp;External Systems (Optional)</a></li> 
        </ul> 
        <hr> 
        <h2><a id="AuthPackage"></a>Auth Package Overview</h2> 
        <p>Livefyre Apps use the global <span style="font-family: 'courier new', courier, monospace;">auth </span>package to associate users with App actions. The <span style="font-family: 'courier new', courier, monospace;">auth&nbsp;</span>package is available through <a href="/developers/reference/livefyre-js/">Livefyre.require</a>.</p> 
        <p>To enable authentication on your page, first add <span style="font-family: 'courier new', courier, monospace;">Livefyre.js</span> to the <span style="font-family: 'courier new', courier, monospace;">&lt;head&gt;</span> element of your webpage or website template.</p>

        <h3>Methods</h3> 
        <p>Once included as above using <span style="font-family: 'courier new', courier, monospace;">Livefyre.require</span>, the Auth module exposes the following methods that you can call to notify other Apps on the page about authentication-related events.</p> 
        <dl> 
         <dt>
          .login(callback)
         </dt> 
         <dd>
          Trigger the login flow as implemented by the registered AuthDelegate. Usually only auth-enabled Apps will call this, and not the host page itself.
         </dd> 
         <dt>
          .logout(callback)
         </dt> 
         <dd>
          Notify 
          <span style="font-family: 'courier new', courier, monospace;">auth </span>that the end-user has logged out by some external means, and that all relying Apps should clear their authentication state until the next login. This will clear the internal session maintained by Auth.
         </dd> 
         <dt>
          .authenticate(credentials)
         </dt> 
         <dd>
          Notify Auth that a user has authenticated by some external means, and a&nbsp;
          <a href="/developers/getting-started/tokens/auth/">Livefyre Authentication Token</a> has been procured for the end-user. Use this if you set a cookie with the Livefyre token, or have a token for the user and want to explicitly log the user in.&nbsp;For example:

        <h2><a id="AuthDelegateObject"></a>Building an Auth Delegate</h2> 
        <p>The <span style="font-family: 'courier new', courier, monospace;">auth </span>package doesnt do anything until it has been provided with a delegate. The AuthDelegate object implements your desired behavior for how to perform authentication actions and events. This allows for a flexible and customizable integration with a sites existing authentication system.</p> 
        <p>An auth delegate is any JavaScript object that implements the following methods:</p> 
        <p><strong>.login(finishLogin)</strong></p> 
        <p>Log in a valid user and invoke the <span style="font-family: 'courier new', courier, monospace;">finishLogin</span> function with either an <span style="font-family: 'courier new', courier, monospace;">Error</span> object if there was an error, or the users Livefyre credentials. Common implementations of this method redirect the user to a login page or open a new window or modal.</p>

        <h2><a id="PostingUserPermissions"></a>Posting User Permissions to External Systems (Optional)</h2> 
        <p>Livefyre uses a PUSH&nbsp;interface to send an external system information about changes to user permissions. These include the following states:</p> 
        <dl> 
         <dt>
          owner:
         </dt> 
         <dd>
          This user is an owner, and can both moderate content, and assign new moderators.
         </dd> 
         <dt>
          admin:
         </dt> 
         <dd>
          This user is a moderator, and can moderate content.
         </dd> 
         <dt>
          member:
         </dt> 
         <dd>
          This user is whitelisted. Posted content does not pass through spam or profanity filters, and does not&nbsp;require approval&nbsp;in pre-moderated streams.
         </dd> 
         <dt>
          none:
         </dt> 
         <dd>
          This user is a standard user, and has no special permissions.
         </dd> 
         <dt>
          outcast:
         </dt> 
         <dd>
          This user has been banned from participating in any conversations.
         </dd> 
         <dt></dt> 
        </dl> 
        <p>To post user permissions to external systems, you must register a URL that receives permissions data as POST requests.</p> 
        <p>&nbsp;</p>
