       <h1 class="entry-title">How WebStorm Works: Completion for JavaScript Libraries</h1> 
       <div class="entry-meta"> 
        <span class="meta-prep meta-prep-author"> Posted on </span> 
        <a href="https://blog.jetbrains.com/webstorm/2014/07/how-webstorm-works-completion-for-javascript-libraries/" title="10:39 am" rel="bookmark"><span class="entry-date">July 1, 2014</span></a> 
        <span class="meta-sep">by</span> 
        <span class="author vcard"><a class="url fn n" href="https://blog.jetbrains.com/webstorm/author/ekaterina-prigara/" title="View all posts by Ekaterina Prigara">Ekaterina Prigara</a></span>. Updated on 
        <span class="entry-date">March 8, 2017</span> 
       </div>
       <!-- .entry-meta --> 
       <div class="entry-content"> 
        <p>One of the essential features of WebStorm’s editor is code completion. The way it is implemented in WebStorm is that it takes all JavaScript files in the project and goes through them, adding functions and methods to an internal index used for code completion suggestions. The index is automatically updated whenever existing files are changed or new files are added.</p> 
        <p><img class="alignnone size-full wp-image-8880" src="https://d3nmt5vlzunoa1.cloudfront.net/webstorm/files/2014/07/completion-example.png" alt="completion-example" width="800"></p> 
        <p><a href="http://usejsdoc.org/">JSDoc</a>, used for documenting code, also plays an important role in making your coding experience better. It powers WebStorm autocompletion with additional details about parameters and their types.</p> 
        <p><img class="alignnone size-full wp-image-8881" src="https://d3nmt5vlzunoa1.cloudfront.net/webstorm/files/2014/07/jsdoc-example.png" alt="jsdoc-example" width="800"></p> 
        <p>But what if all the libraries are loaded dynamically, or we want to exclude the folder with all the library code from the project to eliminate this third-party code from search results and code analysis?</p> 
        <p>This is where we introduce the concept (or mechanism) of <strong>External libraries</strong> that we use in WebStorm.</p> 
        <h2>External libraries</h2> 
        <p><strong>External libraries</strong> are files explicitly added to project’s index to be used only in enhancing coding assistance (i.e. code completion, syntax highlighting, navigation, and documentation lookup). They are not to be edited in the scope of this project.</p> 
        <p>Let’s have a look at a couple of examples.</p> 
        <h3>The library is added to the project via a&nbsp;CDN link</h3> 
        <p>The quick-fix <em>‘Download library’</em> (call up with Alt+Enter) loads the file from the CDN link into the WebStorm cache (but not in your project).</p> 
        <p><img class="alignnone size-full wp-image-8875" src="https://d3nmt5vlzunoa1.cloudfront.net/webstorm/files/2014/07/download-lib.png" alt="download-lib" width="800"></p> 
        <p>To see the list of all libraries that power code completion for your project and enable specific aspects of code completion that you need&nbsp;to go to&nbsp;<em>Preferences | Languages &amp; Frameworks | JavaScript | Libraries.</em></p> 
        <p><img class="alignnone size-full wp-image-8876" src="https://d3nmt5vlzunoa1.cloudfront.net/webstorm/files/2014/07/list-of-libs.png" alt="list-of-libs" width="750"></p> 
        <h3>The library code is in a project folder or elsewhere</h3> 
        <p>You can also add libraries to this list manually. It’s not a very often case nowadays&nbsp;when most of the projects use npm and have a<em> node_modules</em> folder with all the dependencies. But it still might be useful when you have a JavaScript framework file somewhere in your project and you want the IDE to treat it as a library and not just as a source code.</p> 
        <p>Select <em>Preferences | Languages &amp; Frameworks |&nbsp;JavaScript | Libraries</em> and click <em>Add</em>. Enter the library name and the path to the library file or folder. If you select <em>Global</em> visibility level, you’ll be able to enable this library in your other projects.</p> 
        <p><img class="alignnone size-full wp-image-8879" src="https://d3nmt5vlzunoa1.cloudfront.net/webstorm/files/2014/07/add-ext-lib.png" alt="add-ext-lib" width="516"></p> 
        <p>Additionally, you can specify the path to library’s/framework’s documentation. This URL will be opened when you press Shift+F1 (<em>Show external documentation</em>) on methods from the library.</p> 
        <h3>All dependencies are in the node_modules folder</h3> 
        <p>We would like to have code completion for the modules and keep them in the project, but there is no need for us to run inspections against these files. That is why this folder should be marked as a library. In fact, WebStorm does that automatically for you.</p> 
        <p><img class="alignnone size-full wp-image-8877" src="https://d3nmt5vlzunoa1.cloudfront.net/webstorm/files/2014/07/node_modules_as_ext_lib.png" alt="node_modules_as_ext_lib" width="343"></p> 
        <h3>Completion for Node.js APIs</h3> 
        <p>Node.js core modules (like <em>http</em> or <em>fs</em>) are another example where External libraries mechanism is used. To enable code completion for Node.js APIs, go to <em>Preferences | Languages &amp; Frameworks | Node.js and </em>npm and click <em>Enable</em> below the selected Node.js version:</p> 
        <p><img class="alignnone size-full wp-image-8878" src="https://d3nmt5vlzunoa1.cloudfront.net/webstorm/files/2014/07/enable-node.png" alt="enable-node" width="709"></p> 
        <p>WebStorm will create and enable a new External library to provide code completion for Node.js APIs in the whole project.</p> 
        <h2 id="add-ts-def">Using TypeScript community stubs (TypeScript definition files)</h2> 
        <p>WebStorm provides one more workaround for adding coding assistance for some popular libraries, frameworks and modules when you don’t have the source code available in your project. It allows you to download <a href="https://github.com/borisyankov/DefinitelyTyped">DefinitelyTyped</a> stubs. These definition files for libraries and frameworks are added to the list of your project’s External libraries, thus enhancing code completion.</p> 
        <p>To add a TypeScript definition file for a library or framework you use, go to <em>Preferences | Languages &amp; Frameworks | JavaScript | Libraries</em> – click <em>Download</em> and search for the library you need by typing its name, then hit Download again.</p> 
        <p><img class="alignnone size-full wp-image-8874" src="https://d3nmt5vlzunoa1.cloudfront.net/webstorm/files/2014/07/download-d-ts.png" alt="download-d-ts" width="624"></p> 
        <p><em>Develop with pleasure!<br> JetBrains WebStorm Team</em></p> 
       </div>

          <div class="comment-body">
           <p>Very nice article! Couple of things to mention: 1) Webstorm will mark properties of unknown data structure as unresolved – that typically happen in ajax callbacks, so it might be a good idea to use @typefed or custom definition (*.d.ts) file to describe that structure 2) If you have a folder with minified sources it might be better to exclude this folder altogether from Directories dialog and then install library (or better community stubs).<br> Community stubs are definitely better because they contain type information, while it is not guarantied that unminified javascript files have jsdoc comments. Why is it important to have type information? Well there is another amazing part about webstorm – if it can guess type it will provide you type validation support, just like your favorite IDE for strictly typed language. Why would you care about types in javascript when you’re so smart to know how they are converted internally? Because we are all human beings and can make an error, so why not to catch that error as soon as possible then debug all the way back through callstack? Sometimes type validation in webstorm might fail due to more complex type definition of nested jsdoc structure, so if you would like to have better support please vote for these issues:<br> <a href="http://youtrack.jetbrains.com/issue/WEB-12553" rel="nofollow">http://youtrack.jetbrains.com/issue/WEB-12553</a><br> <a href="http://youtrack.jetbrains.com/issue/WEB-12554" rel="nofollow">http://youtrack.jetbrains.com/issue/WEB-12554</a><br> <a href="http://youtrack.jetbrains.com/issue/WEB-11553" rel="nofollow">http://youtrack.jetbrains.com/issue/WEB-11553</a><br> <a href="http://youtrack.jetbrains.com/issue/WEB-11760" rel="nofollow">http://youtrack.jetbrains.com/issue/WEB-11760</a></p>

            <div class="comment-body">
             <p>Hi, Maros.<br> To get relevant completion of imported AMD module, WebStorm have to correctly resolve path to module corresponding to function parameter. That path is now considered as file path, I’ve created issue <a href="http://youtrack.jetbrains.com/issue/WEB-12622" rel="nofollow">WEB-12622</a> for this.<br> Completion in Ekaterina’s example is working because we have heuristics to add type from parameter name matching to AMD module, and luckily the internal definition of Engine has such a name too.<br> By the way, WebStorm understands that /*Engine.prototype.constructor*/ notation specifies that a symbol has a constuctor type (to get static methods completion). Please see <a href="http://youtrack.jetbrains.com/issue/WEB-10214" rel="nofollow">WEB-10214</a>.</p>
