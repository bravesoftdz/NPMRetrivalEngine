            <h1 class="package-name-redundant">loopback-passport</h1> 
            <p>The module provides integration between <a href="http://loopback.io">LoopBack</a> and <a href="http://passportjs.org">Passport</a> to support third party login and account linking for LoopBack applications.</p> 
            <h1>Use cases</h1> 
            <h2>Third party login</h2> 
            <p>Social login becomes popular these days as our users don’t want to deal with so many identities. It would be nice to allow the use of a third party provider such as Facebook, Google, Twitter, or Github to log into LoopBack. The login profiles will be tracked and associated with corresponding LoopBack users.</p> 
            <h2>Linked accounts</h2> 
            <p>In LoopBack, most APIs will be built using models that are backed by data sources, which in turn uses connectors to interact with other systems or cloud services. Some of the backend systems require user-specific credentials to access the protected resources. For example, an e-commerce engine requires the user credential to see the order history. It’s also true to get pictures from one or more facebook accounts. One solution to this requirement is to link or pre-authorize a LoopBack user to other accounts.</p> 
            <h1>Key components</h1> 
            <p><img src="https://raw.githubusercontent.com/strongloop/loopback-passport/master/ids_and_credentials.png" alt="Key Components"></p> 
            <h2>UserIdentity model</h2> 
            <p>UserIdentity model keeps track of 3rd party login profiles. Each user identity is uniquely identified by provider and externalId. UserIdentity model comes with a 'belongsTo' relation to the User model.</p> 
            <p>Properties</p> 
            <ul> 
             <li>{String} provider: The auth provider name, such as facebook, google, twitter, linkedin</li> 
             <li>{String} authScheme: The auth scheme, such as oAuth, oAuth 2.0, OpenID, OpenID Connect</li> 
             <li>{String} externalId: The provider specific user id</li> 
             <li>{Object} profile: The user profile, see http://passportjs.org/guide/profile</li> 
             <li>{Object} credentials 
              <ul> 
               <li>oAuth: token, tokenSecret</li> 
               <li>oAuth 2.0: accessToken, refreshToken</li> 
               <li>OpenID: openId</li> 
               <li>OpenID Connect: accessToken, refreshToken, profile</li> 
              </ul> </li> 
             <li>{*} userId: The LoopBack user id</li> 
             <li>{Date} created: The created date</li> 
             <li>{Date} modified: The last modified date</li> 
            </ul> 
            <h2>UserCredential model</h2> 
            <p>UserCredential has the same set of properties as UserIdentity. It's used to store the credentials from a third party authentication/authorization provider to represent the permissions/authorizations from a user from the third party system.</p> 
            <h2>ApplicationCredential model</h2> 
            <p>Interacting with third party systems often require some client application level credentials. For example, you will need oAuth 2.0 client id and client secret to call facebook APIs. Such credentials can be supplied from a configuration file to your server globally. But if your server accepts API requests from multiple client applications, each client application should have its own credentials. To support the multi tenancy, this module provides the ApplicationCredential model to store credentials associated with a client application.</p> 
            <p>Properties</p> 
            <ul> 
             <li>{String} provider: The auth provider name, such as facebook, google, twitter, linkedin</li> 
             <li>{String} authScheme: The auth scheme, such as oAuth, oAuth 2.0, OpenID, OpenID Connect</li> 
             <li>{Object} credentials: The provider specific credentials 
              <ul> 
               <li>openId: {returnURL: String, realm: String}</li> 
               <li>oAuth2: {clientID: String, clientSecret: String, callbackURL: String}</li> 
               <li>oAuth: {consumerKey: String, consumerSecret: String, callbackURL: String}</li> 
              </ul> </li> 
             <li>{Date} created: The created date</li> 
             <li>{Date} modified: The last modified date</li> 
            </ul> 
            <p>ApplicationCredential model comes with a 'belongsTo' relation to the Application model.</p> 
            <h2>PassportConfigurator</h2> 
            <p>PassportConfigurator is the bridge between LoopBack and Passport.</p> 
            <ul> 
             <li>set up models with LoopBack</li> 
             <li>initialize passport</li> 
             <li>create Passport strategies from provider configurations</li> 
             <li>set up routes for auth and callback</li> 
            </ul> 
            <h1>Flows</h1> 
            <h2>Third party login flow</h2> 
            <p>The following steps use Facebook oAuth 2.0 login as an example.</p> 
            <ol> 
             <li>A visitor requests to log in using Facebook (or other providers), typically by clicking on a link or button backed by LoopBack to kick off oAuth 2.0 authorization code flow</li> 
             <li>LoopBack redirects the browser to Facebook's authorization endpoint so that the user can log into Facebook and grant permissions to LoopBack</li> 
             <li>Facebook redirects the browser to a callback URL hosted by LoopBack with the oAuth 2.0 authorization code</li> 
             <li>LoopBack makes a request to the Facebook token endpoint to get an access token using the authorization code</li> 
             <li>LoopBack uses the access token to retrieve the user's Facebook profile</li> 
             <li>LoopBack searches the UserIdentity model by (provider, externalId) to see there is an existing LoopBack user for the given Facebook id</li> 
             <li>If yes, set the LoopBack user to the current context</li> 
             <li>If not, create a LoopBack user from the profile and create a corresponding record in UserIdentity to track the 3rd party login. Set the newly created user to the current context.</li> 
            </ol> 
            <h2>Third party account linking flow</h2> 
            <p>The following steps use Facebook oAuth 2.0 login as an example.</p> 
            <ol> 
             <li>The user log into LoopBack first directly or through third party login</li> 
             <li>The user clicks on a link or button by LoopBack to kick off oAuth 2.0 authorization code flow so that the user can grant permissions to LoopBack</li> 
             <li>Perform the same steps 2-5 as third party login</li> 
             <li>LoopBack searches the UserCredential model by (provider, externalId) to see there is an existing LoopBack user for the given Facebook id</li> 
             <li>Link the Facebook account to the current user by creating a record in the UserCredential model to store the Facebook credentials, such as access token</li> 
             <li>Now the LoopBack user wants to get a list of pictures from the linked Facebook account(s). LoopBack can look up the Facebook credentials associated with the current user and use them to call Facebook APIs to retrieve the pictures.</li> 
            </ol> 
            <h1>Use the module with a LoopBack application</h1> 
            <p>A demo application is built with this module to showcase how to use the APIs with a LoopBack application. The code is available at:</p> 
            <p><a href="https://github.com/strongloop-community/loopback-example-passport">https://github.com/strongloop-community/loopback-example-passport</a></p> 
            <h2>Configure third party providers</h2> 
            <p>The following example shows two providers: facebook-login for login with facebook and google-link for linking your google accounts with the current LoopBack user.</p> 
            <pre><code class="language-json">{
  "facebook-login": {
    "provider": "facebook",
    "module": "passport-facebook",
    "clientID": "{facebook-client-id-1}",
    "clientSecret": "{facebook-client-secret-1}",
    "callbackURL": "http://localhost:3000/auth/facebook/callback",
    "authPath": "/auth/facebook",
    "callbackPath": "/auth/facebook/callback",
    "successRedirect": "/auth/account",
    "scope": ["email"]
  },
  ...
  "google-link": {
    "provider": "google",
    "module": "passport-google-oauth",
    "strategy": "OAuth2Strategy",
    "clientID": "{google-client-id-2}",
    "clientSecret": "{google-client-secret-2}",
    "callbackURL": "http://localhost:3000/link/google/callback",
    "authPath": "/link/google",
    "callbackPath": "/link/google/callback",
    "successRedirect": "/link/account",
    "scope": ["email", "profile"],
    "link": true
  }
}
</code></pre> 
            <p><strong>NOTE</strong></p> 
            <p>You'll need to register with facebook and google to get your own client id and client secret.</p> 
            <ul> 
             <li>Facebook: https://developers.facebook.com/apps</li> 
             <li>Google: https://console.developers.google.com/project</li> 
            </ul> 
            <h2>Add code snippets to app.js</h2> 
            <pre><code class="language-js">var loopback = require('loopback');
var path = require('path');
var app = module.exports = loopback();

// Create an instance of PassportConfigurator with the app instance
var PassportConfigurator = require('loopback-passport').PassportConfigurator;
var passportConfigurator = new PassportConfigurator(app);

app.boot(__dirname);

...

// Enable http session
app.use(loopback.session({ secret: 'keyboard cat' }));

// Load the provider configurations
var config = {};
try {
  config = require('./providers.json');
} catch(err) {
  console.error('Please configure your passport strategy in `providers.json`.');
  console.error('Copy `providers.json.template` to `providers.json` and replace the clientID/clientSecret values with your own.');
  process.exit(1);
}

// Initialize passport
passportConfigurator.init();

// Set up related models
passportConfigurator.setupModels({
  userModel: app.models.user,
  userIdentityModel: app.models.userIdentity,
  userCredentialModel: app.models.userCredential
});

// Configure passport strategies for third party auth providers
for(var s in config) {
  var c = config[s];
  c.session = c.session !== false;
  passportConfigurator.configureProvider(s, c);
}
</code></pre>
