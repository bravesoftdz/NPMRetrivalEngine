      <div class="content"> 
       <p>The Resource File Generator (Resgen.exe) converts text (.txt or .restext) files and XML-based resource format (.resx) files to common language runtime binary (.resources) files that can be embedded in a runtime binary executable or satellite assembly. (See <a href="../resources/creating-resource-files-for-desktop-apps" data-linktype="relative-path">Creating Resource Files</a>.) </p> 
       <p> Resgen.exe is a general-purpose resource conversion utility that performs the following tasks: </p> 
       <ul> 
        <li><p>Converts .txt or .restext files to .resources or .resx files. (The format of .restext files is identical to the format of .txt files. However, the .restext extension helps you identify text files that contain resource definitions more easily.) </p> </li> 
        <li><p>Converts .resources files to text or .resx files. </p> </li> 
        <li><p>Converts .resx files to text or .resources files. </p> </li> 
        <li><p>Extracts the string resources from an assembly into a .resw file that is suitable for use in a Windows 8.x Store app. </p> </li> 
        <li><p>Creates a strongly typed class that provides access to individual named resources and to the <a class="xref" href="../../api/system.resources.resourcemanager" data-linktype="relative-path">ResourceManager</a> instance. </p> </li> 
       </ul> 
       <p> If Resgen.exe fails for any reason, the return value is –1. </p> 
       <p> To get help with Resgen,exe, you can use the following command, with no options specified, to display the command syntax and options for Resgen.exe: </p> 
       <pre><code>resgen  
</code></pre>
       <p> You can also use the <code>/?</code> switch: </p> 
       <pre><code>resgen /?  
</code></pre>
       <p> If you use Resgen,exe to generate binary .resources files, you can use a language compiler to embed the binary files into executable assemblies, or you can use the <a href="al-exe-assembly-linker" data-linktype="relative-path">Assembly Linker (Al.exe)</a> to compile them into satellite assemblies. </p> 
       <p> This tool is automatically installed with Visual Studio. To run the tool, use the Developer Command Prompt (or the Visual Studio Command Prompt in Windows 7). For more information, see <a href="developer-command-prompt-for-vs" data-linktype="relative-path">Command Prompts</a>. </p> 
       <p> At the command prompt, type the following: </p> 
       <h2 id="syntax">Syntax</h2> 
       <pre><code>resgen  [/define:symbol1[,symbol2,...]] [/useSourcePath] filename.extension  | /compile filename.extension... [outputFilename.extension] [/r:assembly] [/str:lang[,namespace[,class[,file]]] [/publicclass]]   
</code></pre>
       <pre><code>resgen filename.extension [outputDirectory]  
</code></pre>

       <h2 id="resgenexe-and-resource-file-types">Resgen.exe and Resource File Types</h2> 
       <p> In order for Resgen.exe to successfully convert resources, text and .resx files must follow the correct format. </p> 
       <h3 id="text-txt-and-restext-files">Text (.txt and .restext) Files</h3> 
       <p> Text (.txt or .restext) files may contain only string resources. String resources are useful if you are writing an application that must have strings translated into several languages. For example, you can easily regionalize menu strings by using the appropriate string resource. Resgen.exe reads text files that contain name/value pairs, where the name is a string that describes the resource and the value is the resource string itself. </p> 
       <div class="NOTE">
        <h5>Note</h5>
        <p> For information about the format of .txt and .restext files, see the "Resources in Text Files" section of <a href="../resources/creating-resource-files-for-desktop-apps" data-linktype="relative-path">Creating Resource Files</a>. </p> 
       </div> 
       <p> A text file that contains resources must be saved with UTF-8 or Unicode (UTF-16) encoding unless it contains only characters in the Basic Latin range (to U+007F). Resgen.exe removes extended ANSI characters when it processes a text file that is saved using ANSI encoding. </p> 
       <p> Resgen.exe checks the text file for duplicate resource names. If the text file contains duplicate resource names, Resgen.exe will emit a warning and ignore the second value. </p> 
       <h3 id="resx-files">.resx Files</h3> 
       <p> The .resx resource file format consists of XML entries. You can specify string resources within these XML entries, as you would in text files. A primary advantage of .resx files over text files is that you can also specify or embed objects. When you view a .resx file, you can see the binary form of an embedded object (for example, a picture) when this binary information is a part of the resource manifest. As with text files, you can open a .resx file with a text editor (such as Notepad or Microsoft Word) and write, parse, and manipulate its contents. Note that this requires a good knowledge of XML tags and the .resx file structure. For more details on the .resx file format, see the "Resources in .resx Files" section of <a href="../resources/creating-resource-files-for-desktop-apps" data-linktype="relative-path">Creating Resource Files</a>. </p> 
       <p> In order to create a .resources file that contains embedded nonstring objects, you must either use Resgen.exe to convert a .resx file containing objects or add the object resources to your file directly from code by calling the methods provided by the <a class="xref" href="../../api/system.resources.resourcewriter" data-linktype="relative-path">ResourceWriter</a> class. </p> 
       <p> If your .resx or .resources file contains objects and you use Resgen.exe to convert it to a text file, all the string resources will be converted correctly, but the data types of the nonstring objects will also be written to the file as strings. You will lose the embedded objects in the conversion, and Resgen.exe will report that an error occurred in retrieving the resources. </p> 
       <h3 id="converting-between-resources-file-types">Converting Between Resources File Types</h3> 
       <p> When you convert between different resource file types, Resgen.exe may not be able to perform the conversion or may lose information about specific resources, depending on the source and target file types. The following table specifies the types of conversions that are successful when converting from one resource file type to another. </p>

       <h2 id="performing-specific-resgenexe-tasks">Performing Specific Resgen.exe Tasks</h2> 
       <p> You can use Resgen.exe in diverse ways: to compile a text-based or XML-based resource file into a binary file, to convert between resource file formats, and to generate a class that wraps <a class="xref" href="../../api/system.resources.resourcemanager" data-linktype="relative-path">ResourceManager</a> functionality and provides access to resources. This section provides detailed information about each task: </p> 
       <ul> 
        <li><p><a href="resgen-exe-resource-file-generator#Compiling" data-linktype="relative-path">Compiling Resources into a Binary File</a> </p> </li> 
        <li><p><a href="resgen-exe-resource-file-generator#Convert" data-linktype="relative-path">Converting Between Resource File Types</a> </p> </li> 
        <li><p><a href="resgen-exe-resource-file-generator#Multiple" data-linktype="relative-path">Compiling or Converting Multiple Files</a> </p> </li> 
        <li><p><a href="resgen-exe-resource-file-generator#Exporting" data-linktype="relative-path">Exporting Resources to a .resw File</a> </p> </li> 
        <li><p><a href="resgen-exe-resource-file-generator#Conditional" data-linktype="relative-path">Conditionally Compiling Resources</a> </p> </li> 
        <li><p><a href="resgen-exe-resource-file-generator#Strong" data-linktype="relative-path">Generating a Strongly Typed Resource Class</a> </p> </li> 
       </ul> 
       <p><a name="Compiling"></a> </p> 
       <h3 id="compiling-resources-into-a-binary-file">Compiling Resources into a Binary File</h3> 
       <p> The most common use of Resgen.exe is to compile a text-based resource file (a .txt or .restext file) or an XML-based resource file (a .resx file) into a binary .resources file. The output file then can be embedded in a main assembly by a language compiler or in a satellite assembly by <a href="al-exe-assembly-linker" data-linktype="relative-path">Assembly Linker (AL.exe)</a>. </p> 
       <p> The syntax to compile a resource file is: </p> 
       <pre><code>resgen inputFilename [outputFilename]   
</code></pre>
       <p> where the parameters are: </p> 
       <p> <code>inputFilename</code><br> The file name, including the extension, of the resource file to compile. Resgen.exe only compiles files with extensions of .txt, .restext, or .resx. </p> 
       <p> <code>outputFilename</code><br> The name of the output file. If you omit <code>outputFilename</code>, Resgen.exe creates a .resources file with the root file name of <code>inputFilename</code> in the same directory as <code>inputFilename</code>. If <code>outputFilename</code> includes a directory path, the directory must exist. </p> 
       <p> You provide a fully qualified namespace for the .resources file by specifying it in the file name and separating it from the root file name by a period. For example, if <code>outputFilename</code> is <code>MyCompany.Libraries.Strings.resources</code>, the namespace is MyCompany.Libraries. </p> 
       <p> The following command reads the name/value pairs in Resources.txt and writes a binary .resources file named Resources.resources. Because the output file name is not specified explicitly, it receives the same name as the input file by default. </p> 
       <pre><code>resgen Resources.txt   
</code></pre>
       <p> The following command reads the name/value pairs in Resources.restext and writes a binary resources file named StringResources.resources. </p> 
       <pre><code>resgen Resources.restext StringResources.resources  
</code></pre>
       <p> The following command reads an XML-based input file named Resources.resx and writes a binary .resources file named Resources.resources. </p> 
       <pre><code>resgen Resources.resx Resources.resources  
</code></pre>
       <p><a name="Convert"></a> </p> 
       <h3 id="converting-between-resource-file-types">Converting Between Resource File Types</h3> 
       <p> In addition to compiling text-based or XML-based resource files into binary .resources files, Resgen.exe can convert any supported file type to any other supported file type. This means that it can perform the following conversions: </p> 
       <ul> 
        <li><p>.txt and .restext files to .resx files. </p> </li> 
        <li><p>.resx files to .txt and .restext files. </p> </li> 
        <li><p>.resources files to .txt and .restext files. </p> </li> 
        <li><p>.resources files to .resx files. </p> </li> 
       </ul> 
       <p> The syntax is the same as that shown in the previous section. </p> 
       <p> In addition, you can use Resgen.exe to convert embedded resources in a .NET Framework assembly to a .resw file tor Windows 8.x Store apps. </p> 
       <p> The following command reads a binary resources file Resources.resources and writes an XML-based output file named Resources.resx. </p> 
       <pre><code>resgen Resources.resources Resources.resx  
</code></pre>
       <p> The following command reads a text-based resources file named StringResources.txt and writes an XML-based resources file named LibraryResources.resx. In addition to containing string resources, the .resx file could also be used to store non-string resources. </p> 
       <pre><code>resgen StringResources.txt LibraryResources.resx  
</code></pre>
       <p> The following two commands read an XML-based resources file named Resources.resx and write text files named Resources.txt and Resources.restext. Note that if the .resx file contains any embedded objects, they will not be accurately converted into the text files. </p> 
       <pre><code>resgen Resources.resx Resources.txt  
resgen Resources.resx Resources.restext  
</code></pre>
       <p><a name="Multiple"></a> </p> 
       <h3 id="compiling-or-converting-multiple-files">Compiling or Converting Multiple Files</h3> 
       <p> You can use the <code>/compile</code> switch to convert a list of resource files from one format to another in a single operation. The syntax is: </p> 
       <pre><code>resgen /compile filename.extension [filename.extension...]  
</code></pre>
       <p> The following command compiles three files, StringResources.txt, TableResources.resw, and ImageResources.resw, into separate .resources files named StringResources.resources, TableResources.resources, and ImageResources.resources. </p> 
       <pre><code>resgen /compile StringResources.txt TableResources.resx ImageResources.resx  
</code></pre>
       <p><a name="Exporting"></a> </p> 
       <h3 id="exporting-resources-to-a-resw-file">Exporting Resources to a .resw File</h3> 
       <p> If you're developing a Windows 8.x Store app, you may want to use resources from an existing desktop app. However, the two kinds of applications support different file formats. In desktop apps, resources in text (.txt or .restext) or .resx files are compiled into binary .resources files. In Windows 8.x Store apps, .resw files are compiled into binary package resource index (PRI) files. You can use Resgen.exe to bridge this gap by extracting resources from an executable or a satellite assembly and writing them to one or more .resw files that can be used when developing a Windows 8.x Store app. </p> 
       <div class="IMPORTANT">
        <h5>Important</h5>
        <p> Visual Studio automatically handles all conversions necessary for incorporating the resources in a portable library into a Windows 8.x Store app. Using Resgen.exe directly to convert the resources in an assembly to .resw file format is of interest only to developers who want to develop a Windows 8.x Store app outside of Visual Studio. </p> 
       </div> 
       <p> The syntax to generate .resw files from an assembly is: </p> 
       <pre><code>resgen filename.extension  [outputDirectory]  
</code></pre>
       <p> where the parameters are: </p> 
       <p> <code>filename.extension</code><br> The name of a .NET Framework assembly (an executable or .DLL). If the file contains no resources, Resgen.exe does not create any files. </p> 
       <p> <code>outputDirectory</code><br> The existing directory to which to write the .resw files. If <code>outputDirectory</code> is omitted, .resw files are written to the current directory. Resgen.exe creates one .resw file for each .resources file in the assembly. The root file name of the .resw file is the same as the root name of the .resources file. </p> 
       <p> The following command creates a .resw file in the Win8Resources directory for each .resources file embedded in MyApp.exe: </p> 
       <pre><code>resgen MyApp.exe Win8Resources  
</code></pre>
       <p><a name="Conditional"></a> </p> 
       <h3 id="conditionally-compiling-resources">Conditionally Compiling Resources</h3> 
       <p> Starting with the .NET Framework 4.5, Resgen.exe supports conditional compilation of string resources in text (.txt and .restext) files. This enables you to use a single text-based resource file in multiple build configurations. </p> 
       <p> In a .txt or .restext file, you use the <code>#ifdef</code>…<code>#endif</code> construct to include a resource in the binary .resources file if a symbol is defined, and you use the <code>#if !</code>... <code>#endif</code> construct to include a resource if a symbol is not defined. At compile time, you then define symbols by using the <code>/define:</code> option followed by a comma-delimited list of symbols. The comparison is cased-sensitive; the case of symbols defined by <code>/define</code> must match the case of symbols in the text files to be compiled. </p> 
       <p> For example, the following file named UIResources.rext includes a string resource named <code>AppTitle</code> that can take one of three values, depending on whether symbols named <code>PRODUCTION</code>, <code>CONSULT</code>, or <code>RETAIL</code> are defined. </p> 
       <pre><code>#ifdef PRODUCTION  
AppTitle=My Software Company Project Manager   
#endif  
#ifdef CONSULT  
AppTitle=My Consulting Company Project Manager  
#endif  
#ifdef RETAIL  
AppTitle=My Retail Store Project Manager  
#endif  
FileMenuName=File  
</code></pre>
       <p> The file can then be compiled into a binary .resources file with the following command: </p> 
       <pre><code>resgen /define:CONSULT UIResources.restext  
</code></pre>
       <p> This produces a .resources file that contains two string resources. The value of the <code>AppTitle</code> resource is "My Consulting Company Project Manager". </p> 
       <p><a name="Strong"></a> </p> 
       <h3 id="generating-a-strongly-typed-resource-class">Generating a Strongly Typed Resource Class</h3> 
       <p> Resgen.exe supports strongly typed resources, which encapsulates access to resources by creating classes that contain a set of static read-only properties. This provides an alternative to calling the methods of the <a class="xref" href="../../api/system.resources.resourcemanager" data-linktype="relative-path">ResourceManager</a> class directly to retrieve resources. You can enable strongly typed resource support by using the <code>/str</code> option in Resgen.exe, which wraps the functionality of the <a class="xref" href="../../api/system.resources.tools.stronglytypedresourcebuilder" data-linktype="relative-path">StronglyTypedResourceBuilder</a> class. When you specify the <code>/str</code> option, the output of Resgen.exe is a class that contains strongly typed properties that match the resources that are referenced in the input parameter. This class provides strongly typed read-only access to the resources that are available in the file processed. </p> 
       <p> The syntax to create a strongly typed resource is: </p> 
       <pre><code>resgen inputFilename [outputFilename] /str:language[,namespace,[classname[,filename]]] [/publicClass]  
</code></pre>
       <p> The parameters and switches are: </p> 
       <p> <code>inputFilename</code><br> The file name, including the extension, of the resource file for which to generate a strongly typed resource class. The file can be a text-based, XML-based, or binary .resources file; it can have an extension of .txt, .restext, .resw, or .resources. </p> 
       <p> <code>outputFilename</code><br> The name of the output file. If <code>outputFilename</code> includes a directory path, the directory must exist. If you omit <code>outputFilename</code>, Resgen.exe creates a .resources file with the root file name of <code>inputFilename</code> in the same directory as <code>inputFilename</code>. </p> 
       <p> <code>outputFilename</code> can be a text-based, XML-based, or binary .resources file. If the file extension of <code>outputFilename</code> is different from the file extension of <code>inputFilename</code>, Resgen.exe performs the file conversion. </p> 
       <p> If <code>inputFilename</code> is a .resources file, Resgen.exe copies the .resources file if <code>outputFilename</code> is also a .resources file. If <code>outputFilename</code> is omitted, Resgen.exe overwrites <code>inputFilename</code> with an identical .resources file. </p> 
       <p> <em>language</em><br> The language in which to generate source code for the strongly-typed resource class. Possible values are <code>cs</code>, <code>C#</code>, and <code>csharp</code> for C# code, <code>vb</code> and <code>visualbasic</code> for Visual Basic code, <code>vbs</code> and <code>vbscript</code> for VBScript code, and <code>c++</code>, <code>mc</code>, and <code>cpp</code> for C++ code. </p> 
       <p> <em>namespace</em><br> The namespace that contains the strongly typed resource class. The .resources file and the resource class should have the same namespace. For information about specifying the namespace in the <code>outputFilename</code>, see <a href="resgen-exe-resource-file-generator#Compiling" data-linktype="relative-path">Compiling Resources into a Binary File</a>. If <em>namespace</em> is omitted, the resource class is not contained in a namespace. </p> 
       <p> <em>classname</em><br> The name of the strongly typed resource class. This should correspond to the root name of the .resources file. For example, if Resgen.exe generates a .resources file named MyCompany.Libraries.Strings.resources, the name of the strongly typed resource class is Strings. If <em>classname</em> is omitted, the generated class is derived from the root name of <code>outputFilename</code>. If <code>outputFilename</code> is omitted, the generated class is derived from the root name of <code>inputFilename</code>. </p> 
       <p> <em>classname</em> cannot contain invalid characters such as embedded spaces. If <em>classname</em> contains embedded spaces, or if <em>classname</em> is generated by default from <em>inputFilename</em>, and <em>inputFilename</em> contains embedded spaces, Resgen.exe replaces all invalid characters with an underscore (_). </p> 
       <p> <em>filename</em><br> The name of the class file. </p> 
       <p> <code>/publicclass</code><br> Makes the strongly typed resource class public rather than <code>internal</code> (in C#) or <code>Friend</code> (in Visual Basic). This allows the resources to be accessed from outside the assembly in which they are embedded. </p> 
       <div class="IMPORTANT">
        <h5>Important</h5>
        <p> When you create a strongly typed resource class, the name of your .resources file must match the namespace and class name of the generated code. However, Resgen.exe allows you to specify options that produce a .resources file that has an incompatible name. To work around this behavior, rename the output file after it has been generated. </p> 
       </div> 
       <p> The strongly typed resource class has the following members: </p> 
       <ul> 
        <li><p>A parameterless constructor, which can be used to instantiate the strongly typed resource class.. </p> </li> 
        <li><p>A <code>static</code> (C#) or <code>Shared</code> (Visual Basic) and read-only <code>ResourceManager</code> property, which returns the <a class="xref" href="../../api/system.resources.resourcemanager" data-linktype="relative-path">ResourceManager</a> instance that manages the strongly typed resource. </p> </li> 
        <li><p>A static <code>Culture</code> property, which allows you to set the culture used for resource retrieval. By default, its value is <code>null</code>, which means that that the current UI culture is used. </p> </li> 
        <li><p>One <code>static</code> (C#) or <code>Shared</code> (Visual Basic) and read-only property for each resource in the .resources file. The name of the property is the name of the resource.- </p> </li> 
       </ul> 
       <p> For example, the following command compiles a resource file named StringResources.txt into StringResources.resources and generates a class named <code>StringResources</code> in a Visual Basic source code file named StringResources.vb that can be used to access the Resource Manager. </p> 
       <pre><code>resgen StringResources.txt /str:vb,,StringResources   
</code></pre>
