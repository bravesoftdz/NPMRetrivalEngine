      <h1 id="loopback-connector-soap">loopback-connector-soap</h1> 
      <p>The SOAP connector enables LoopBack applications to interact with <a href="http://www.w3.org/TR/soap">SOAP</a>-based web services described using <a href="http://www.w3.org/TR/wsdl">WSDL</a>.</p> 
      <p class="gh-only"> For more information, see the <a href="http://loopback.io/doc/en/lb2/SOAP-connector.html">LoopBack documentation</a>. </p> 
      <h2 id="installation">Installation</h2> 
      <p>In your application root directory, enter:</p> 
      <div class="language-shell highlighter-rouge">
       <pre class="highlight"><code><span class="gp">$ </span>npm install loopback-connector-soap --save
</code></pre> 
      </div> 
      <p>This will install the module from npm and add it as a dependency to the application’s&nbsp; <a href="http://loopback.io/doc/en/lb2/package.json.html">package.json</a>&nbsp;file.</p> 
      <h2 id="creating-a-data-source">Creating a data source</h2> 
      <p>Use the&nbsp;<a href="http://loopback.io/doc/en/lb2/Data-source-generator.html">Data source generator</a>&nbsp;to add a SOAP data source to your application.</p> 
      <p>With the API Connect toolkit:</p> 
      <div class="language-shell highlighter-rouge">
       <pre class="highlight"><code><span class="gp">$ </span>apic create --type datasource
</code></pre> 
      </div> 
      <p>With StrongLoop tools:</p> 
      <div class="language-shell highlighter-rouge">
       <pre class="highlight"><code><span class="gp">$ </span>slc loopback:datasource
</code></pre> 
      </div> 
      <p>Choose “SOAP webservices” as the data source type when prompted.</p> 
      <h2 id="soap-data-source-properties">SOAP data source properties</h2> 
      <p>The following table describes the SOAP data source properties you can set in <code class="highlighter-rouge">datasources.json</code>.</p> 
      <table> 
       <tbody> 
        <tr> 
         <th>Property</th> 
         <th>Type</th> 
         <th>Description</th> 
        </tr> 
        <tr> 
         <td>url</td> 
         <td>String</td> 
         <td> <p>URL to the SOAP web service endpoint. If not present, defaults to the <code>location</code> attribute of the SOAP address for the service/port from the WSDL document; for example:</p> <pre><code>&lt;wsdl:service name="Weather"&gt;
  &lt;wsdl:port name="WeatherSoap"
    binding="tns:WeatherSoap"&gt;
  &lt;soap:address
    location="http://wsf.cdyne.com/WeatherWS/Weather.asmx" /&gt;
  &lt;/wsdl:port&gt; ...
&lt;/wsdl:service&gt;</code></pre> </td> 
        </tr> 
        <tr> 
         <td>wsdl</td> 
         <td>String</td> 
         <td>HTTP URL or local file system path to the WSDL file. Default is <code>?wsdl</code>.</td> 
        </tr> 
        <tr> 
         <td>wsdl_options</td> 
         <td>Object</td> 
         <td>Indicates additonal options to pass to the SOAP connector, for example allowing self signed certificates. For example: <pre><code>wsdl_options: {
  rejectUnauthorized: false,
  strictSSL: false,
  requestCert: true,
}</code></pre></td> 
        </tr> 
        <tr> 
         <td>remotingEnabled</td> 
         <td>Boolean</td> 
         <td> <p>Indicates whether the operations are exposed as REST APIs.</p> <p>To expose or hide a specific method, override with:</p> <p><code>&lt;Model&gt;.&lt;method&gt;.shared = true | false;</code></p> </td> 
        </tr> 
        <tr> 
         <td>operations</td> 
         <td>Object</td> 
         <td>Maps WSDL binding operations to Node.js methods. Each key in the JSON object becomes the name of a method on the model. See <a href="#operations-property">operations property</a> below.</td> 
        </tr> 
        <tr> 
         <td>security</td> 
         <td>Object</td> 
         <td>security configuration. See <a href="#security-property">security property</a> below. </td> 
        </tr> 
        <tr> 
         <td>soapHeaders</td> 
         <td>Array of objects.</td> 
         <td>Custom SOAP headers. An array of header properties. For example: <pre><code>soapHeaders: [{
 element: {myHeader: 'XYZ'}, // The XML element in JSON object format
 prefix: 'p1', // The XML namespace prefix for the header
 namespace: 'http://ns1' // The XML namespace URI for the header
}]</code></pre> </td> 
        </tr> 
       </tbody> 
      </table> 
      <h3 id="operations-property">operations property</h3> 
      <p>The <code class="highlighter-rouge">operations</code> property value is a JSON object that has a property (key) for each method being defined for the model. The corresponding value is an object with the following properties:</p> 
      <table> 
       <thead> 
        <tr> 
         <th>Property</th> 
         <th>Type</th> 
         <th>Description</th> 
        </tr> 
       </thead> 
       <tbody> 
        <tr> 
         <td>service</td> 
         <td>String</td> 
         <td>WSDL service name</td> 
        </tr> 
        <tr> 
         <td>port</td> 
         <td>String</td> 
         <td>WSDL port name</td> 
        </tr> 
        <tr> 
         <td>operation</td> 
         <td>String</td> 
         <td>WSDL operation name</td> 
        </tr> 
       </tbody> 
      </table> 
      <p>Here is an example operations property for the stock quote service:</p> 
      <div class="language-javascript highlighter-rouge">
       <pre class="highlight"><code><span class="nx">operations</span><span class="err">:</span> <span class="p">{</span>
  <span class="c1">// The key is the method name</span>
  <span class="nl">stockQuote</span><span class="p">:</span> <span class="p">{</span>
    <span class="nl">service</span><span class="p">:</span> <span class="s1">'StockQuote'</span><span class="p">,</span> <span class="c1">// The WSDL service name</span>
    <span class="nx">port</span><span class="err">:</span> <span class="s1">'StockQuoteSoap'</span><span class="p">,</span> <span class="c1">// The WSDL port name</span>
    <span class="nx">operation</span><span class="err">:</span> <span class="s1">'GetQuote'</span> <span class="c1">// The WSDL operation name</span>
  <span class="p">},</span>
  <span class="nx">stockQuote12</span><span class="err">:</span> <span class="p">{</span>
    <span class="nl">service</span><span class="p">:</span> <span class="s1">'StockQuote'</span><span class="p">,</span>
    <span class="nx">port</span><span class="err">:</span> <span class="s1">'StockQuoteSoap12'</span><span class="p">,</span>
    <span class="nx">operation</span><span class="err">:</span> <span class="s1">'GetQuote'</span>
  <span class="p">}</span>
<span class="p">}</span>
</code></pre> 
      </div> 
      <h3 id="security-property">security property</h3> 
      <p>The <code class="highlighter-rouge">security</code> property value is a JSON object witha <code class="highlighter-rouge">scheme</code> property. The other properties of the object depend on the value of that property. For example:</p> 
      <div class="language-javascript highlighter-rouge">
       <pre class="highlight"><code><span class="nx">security</span><span class="err">:</span> <span class="p">{</span>
    <span class="nl">scheme</span><span class="p">:</span> <span class="s1">'WS'</span><span class="p">,</span>
    <span class="nx">username</span><span class="err">:</span> <span class="s1">'test'</span><span class="p">,</span>
    <span class="nx">password</span><span class="err">:</span> <span class="s1">'testpass'</span><span class="p">,</span>
    <span class="nx">passwordType</span><span class="err">:</span> <span class="s1">'PasswordDigest'</span>
<span class="p">}</span>
</code></pre> 
      </div> 
      <table> 
       <tbody> 
        <tr> 
         <th>Scheme</th> 
         <th>Description</th> 
         <th>Other properties</th> 
        </tr> 
        <tr> 
         <td>WS</td> 
         <td>WSSecurity scheme</td> 
         <td> 
          <ul> 
           <li>username: the user name</li> 
           <li>password: the password</li> 
           <li>passwordType: default is 'PasswordText'</li> 
          </ul> </td> 
        </tr> 
        <tr> 
         <td>BasicAuth</td> 
         <td>Basic auth scheme</td> 
         <td> 
          <ul> 
           <li>username: the user name</li> 
           <li>password: the password</li> 
          </ul> </td> 
        </tr> 
        <tr> 
         <td>ClientSSL</td> 
         <td>ClientSSL scheme</td> 
         <td> 
          <ul> 
           <li>keyPath: path to the private key file</li> 
           <li>certPath: path to the certificate file</li> 
          </ul> </td> 
        </tr> 
       </tbody> 
      </table>

      <h2 id="creating-a-model-from-a-soap-data-source">Creating a model from a SOAP data source</h2> 
      <p>The SOAP connector loads WSDL documents asynchronously. As a result, the data source won’t be ready to create models until it’s connected. The recommended way is to use an event handler for the ‘connected’ event; for example:</p> 
      <div class="language-javascript highlighter-rouge">
       <pre class="highlight"><code><span class="nx">ds</span><span class="p">.</span><span class="nx">once</span><span class="p">(</span><span class="s1">'connected'</span><span class="p">,</span> <span class="kd">function</span> <span class="p">()</span> <span class="p">{</span>
   <span class="c1">// Create the model</span>
   <span class="kd">var</span> <span class="nx">WeatherService</span> <span class="o">=</span> <span class="nx">ds</span><span class="p">.</span><span class="nx">createModel</span><span class="p">(</span><span class="s1">'WeatherService'</span><span class="p">,</span> <span class="p">{});</span>
   <span class="c1">//...</span>
<span class="p">}</span>
</code></pre> 
      </div> 
      <h2 id="extending-a-model-to-wrap-and-mediate-soap-operations">Extending a model to wrap and mediate SOAP operations</h2> 
      <p>Once you define the model, you can extend it to wrap or mediate SOAP operations and define new methods. The following example simplifies the&nbsp;<code class="highlighter-rouge">GetCityForecastByZIP</code>&nbsp;operation to a method that takes&nbsp;<code class="highlighter-rouge">zip</code>&nbsp;and returns an array of forecasts.</p> 
      <div class="language-javascript highlighter-rouge">
       <pre class="highlight"><code><span class="c1">// Refine the methods</span>
<span class="nx">WeatherService</span><span class="p">.</span><span class="nx">forecast</span> <span class="o">=</span> <span class="kd">function</span> <span class="p">(</span><span class="nx">zip</span><span class="p">,</span> <span class="nx">cb</span><span class="p">)</span> <span class="p">{</span>
    <span class="nx">WeatherService</span><span class="p">.</span><span class="nx">GetCityForecastByZIP</span><span class="p">({</span><span class="na">ZIP</span><span class="p">:</span> <span class="nx">zip</span> <span class="o">||</span> <span class="s1">'94555'</span><span class="p">},</span>
      <span class="kd">function</span> <span class="p">(</span><span class="nx">err</span><span class="p">,</span> <span class="nx">response</span><span class="p">)</span> <span class="p">{</span>
        <span class="nx">console</span><span class="p">.</span><span class="nx">log</span><span class="p">(</span><span class="s1">'Forecast: %j'</span><span class="p">,</span> <span class="nx">response</span><span class="p">);</span>
        <span class="kd">var</span> <span class="nx">result</span> <span class="o">=</span> <span class="p">(</span><span class="o">!</span><span class="nx">err</span> <span class="o">&amp;&amp;</span> <span class="nx">response</span><span class="p">.</span><span class="nx">GetCityForecastByZIPResult</span><span class="p">.</span><span class="nx">Success</span><span class="p">)</span> <span class="p">?</span>
        <span class="nx">response</span><span class="p">.</span><span class="nx">GetCityForecastByZIPResult</span><span class="p">.</span><span class="nx">ForecastResult</span><span class="p">.</span><span class="nx">Forecast</span> <span class="p">:</span> <span class="p">[];</span>
        <span class="nx">cb</span><span class="p">(</span><span class="nx">err</span><span class="p">,</span> <span class="nx">result</span><span class="p">);</span>
      <span class="p">});</span>
<span class="p">};</span>
</code></pre> 
      </div> 
      <p>The custom method on the model can be exposed as REST APIs. It uses the&nbsp;<code class="highlighter-rouge">loopback.remoteMethod</code>&nbsp;to define the mappings.</p> 
      <div class="language-javascript highlighter-rouge">
       <pre class="highlight"><code><span class="c1">// Map to REST/HTTP</span>
<span class="nx">loopback</span><span class="p">.</span><span class="nx">remoteMethod</span><span class="p">(</span>
  <span class="nx">WeatherService</span><span class="p">.</span><span class="nx">forecast</span><span class="p">,</span> <span class="p">{</span>
    <span class="na">accepts</span><span class="p">:</span> <span class="p">[{</span>
      <span class="na">arg</span><span class="p">:</span> <span class="s1">'zip'</span><span class="p">,</span>
      <span class="na">type</span><span class="p">:</span> <span class="s1">'string'</span><span class="p">,</span>
      <span class="na">required</span><span class="p">:</span> <span class="kc">true</span><span class="p">,</span>
      <span class="na">http</span><span class="p">:</span> <span class="p">{</span>
        <span class="na">source</span><span class="p">:</span> <span class="s1">'query'</span>
      <span class="p">}</span>
    <span class="p">}],</span>
    <span class="na">returns</span><span class="p">:</span> <span class="p">{</span>
      <span class="na">arg</span><span class="p">:</span> <span class="s1">'result'</span><span class="p">,</span>
      <span class="na">type</span><span class="p">:</span> <span class="s1">'object'</span><span class="p">,</span>
      <span class="na">root</span><span class="p">:</span> <span class="kc">true</span>
    <span class="p">},</span>
    <span class="na">http</span><span class="p">:</span> <span class="p">{</span>
      <span class="na">verb</span><span class="p">:</span> <span class="s1">'get'</span><span class="p">,</span>
      <span class="na">path</span><span class="p">:</span> <span class="s1">'/forecast'</span>
    <span class="p">}</span>
  <span class="p">}</span>
<span class="p">);</span>
</code></pre> 
      </div> 
      <h2 id="use-boot-script-to-create-model-and-expose-apis-to-api-explorer">Use boot script to create model and expose APIs to API Explorer</h2> 
      <p>The SOAP connector builds operations from WSDL asynchronously. To expose such methods over REST, you need to do the following with a boot script, such as <code class="highlighter-rouge">server/a-soap.js</code>:</p> 
      <div class="language-javascript highlighter-rouge">
       <pre class="highlight"><code><span class="nx">module</span><span class="p">.</span><span class="nx">exports</span> <span class="o">=</span> <span class="kd">function</span><span class="p">(</span><span class="nx">app</span><span class="p">,</span> <span class="nx">cb</span><span class="p">)</span> <span class="p">{</span>
  <span class="kd">var</span> <span class="nx">ds</span> <span class="o">=</span> <span class="nx">app</span><span class="p">.</span><span class="nx">dataSources</span><span class="p">.</span><span class="nx">WeatherServiceDS</span><span class="p">;</span>
  <span class="k">if</span> <span class="p">(</span><span class="nx">ds</span><span class="p">.</span><span class="nx">connected</span><span class="p">)</span> <span class="p">{</span>
    <span class="kd">var</span> <span class="nx">weather</span> <span class="o">=</span> <span class="nx">ds</span><span class="p">.</span><span class="nx">createModel</span><span class="p">(</span><span class="s1">'weather'</span><span class="p">,</span> <span class="p">{},</span> <span class="p">{</span><span class="na">base</span><span class="p">:</span> <span class="s1">'Model'</span><span class="p">});</span>
    <span class="nx">app</span><span class="p">.</span><span class="nx">model</span><span class="p">(</span><span class="nx">weather</span><span class="p">);</span>
    <span class="nx">process</span><span class="p">.</span><span class="nx">nextTick</span><span class="p">(</span><span class="nx">cb</span><span class="p">);</span>
  <span class="p">}</span> <span class="k">else</span> <span class="p">{</span>
    <span class="nx">ds</span><span class="p">.</span><span class="nx">once</span><span class="p">(</span><span class="s1">'connected'</span><span class="p">,</span> <span class="kd">function</span><span class="p">()</span> <span class="p">{</span>
      <span class="kd">var</span> <span class="nx">weather</span> <span class="o">=</span> <span class="nx">ds</span><span class="p">.</span><span class="nx">createModel</span><span class="p">(</span><span class="s1">'weather'</span><span class="p">,</span> <span class="p">{},</span> <span class="p">{</span><span class="na">base</span><span class="p">:</span> <span class="s1">'Model'</span><span class="p">});</span>
      <span class="nx">app</span><span class="p">.</span><span class="nx">model</span><span class="p">(</span><span class="nx">weather</span><span class="p">);</span>
      <span class="nx">cb</span><span class="p">();</span>
    <span class="p">});</span>
  <span class="p">}</span>
<span class="p">};</span>
</code></pre> 
      </div> 
      <h2 id="examples">Examples</h2> 
      <p>The&nbsp;[loopback-example-connector] (https://github.com/strongloop/loopback-example-connector/tree/soap) repository provides several examples in the <code class="highlighter-rouge">soap</code> branch:</p> 
      <p>Get stock quotes by symbols:&nbsp; [stock-ws.js] (https://github.com/strongloop/loopback-example-connector/blob/soap/stock-ws.js). Run with the command:</p> 
      <div class="language-shell highlighter-rouge">
       <pre class="highlight"><code><span class="gp">$ </span>node example/stock-ws
</code></pre> 
      </div> 
      <p>Get weather and forecast information for a given zip code:&nbsp; [weather-ws.js] (https://github.com/strongloop/loopback-connector-soap/blob/master/example/weather-ws.js). Run with the command:</p> 
      <div class="language-shell highlighter-rouge">
       <pre class="highlight"><code><span class="gp">$ </span>node example/weather-ws
</code></pre> 
      </div> 
      <p>Expose REST APIs to proxy the SOAP web services: [weather-rest.js] (https://github.com/strongloop/loopback-example-connector/blob/soap/weather-rest.js). Run with the command:</p> 
      <div class="language-shell highlighter-rouge">
       <pre class="highlight"><code><span class="gp">$ </span>node example/weather-rest
</code></pre> 
      </div> 
      <p>View the results at&nbsp; <a href="http://localhost:3000/explorer">http://localhost:3000/explorer</a>.</p>
