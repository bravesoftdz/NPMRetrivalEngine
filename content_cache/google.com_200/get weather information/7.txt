          <div class="single-right-big"> 
           <article id="post-1430" class="post-1430 post type-post status-publish format-standard has-post-thumbnail hentry category-past-weather category-seo-2 category-weather-2 category-weather-api category-website tag-api tag-api-guide tag-free-api tag-node-js tag-premium-api tag-weather-api"> 
            <div class="entry-content"> 
             <p>Were going to guide you through making a simple <a href="http://nodejs.org/">node.js</a> application that will check what the weather was like on Christmas Day in New York each year since 2008.</p> 
             <p>Did you know our <a href="http://www.worldweatheronline.com/api/historical-weather-api.aspx">historical API</a> provides data as far back as 1st July 2008. Were using this example to see how often New York has a white Christmas!</p> 
             <p><strong>Getting your API Key</strong></p> 
             <p>If youre new to using our online weather API, you should have a read through our previous guides on getting started.</p> 
             <p>We have a <a href="https://www.worldweatheronline.com/blog/2015/02/05/getting-started-world-weather-online-api-php/">PHP</a> and a <a href="https://www.worldweatheronline.com/blog/2015/02/19/getting-started-world-weather-online-api-node-js/">Node.js</a> guide which also shows you how to get your hands on and use your API key. This API key can be used for all of our weather APIs.</p> 
             <p><strong>Getting Started</strong></p> 
             <p>Firstly, well setup the basics of our node application, create our package.json file first:<br> Firstly, well setup the basics of our node application, create our package.json file first:</p> 
             <pre>{
  "name": "wwo-historical-api-with-node",
  "version": "1.0.0",
  "description": "How often do New York get snow on christmas day",
  "dependencies": {
    "express": "4.2.x",
    "request": "^2.53.0"
  },
  "private": true,
  "license": "MIT"
}
</pre> 
             <p>This tells node.js what packages/dependencies our application will need, well be using the express and request node libraries. Now lets start adding some code, create a new app.js file:</p> 
             <pre>// file: app.js
var express = require('express');
var app = express();
var request = require('request');
var apiKey = 'YOUR API KEY';

app.get('/', function (req, res) {
   console.log("Welcome to your node.js application");
});

var server = app.listen(3000, function () {
    var host = server.address().address;
    var port = server.address().port;
    console.log('Your app listening at http://%s:%s', host, port);
});
</pre> 
             <p>You can now run this application by running <code>node app.js</code> in a terminal window thats in your project directory. It wont do much at the minute, but weve got a good starting point.</p> 
             <p><strong>Making requests to our API</strong></p> 
             <p>The historic API works very similar to our Local weather API, except you can also provide a date (and even a range of dates) to get access to historical information.</p> 
             <p>The main difference is the base URL in which well make requests to. The below URLs are available for the historical API:</p> 
             <h3>Base URL</h3> 
             <p><b>Free API:</b><br> HTTP: <a href="http://api.worldweatheronline.com/free/v2/past-weather.ashx">http://api.worldweatheronline.com/free/v2/past-weather.ashx</a><br> HTTPS: <a href="https://api.worldweatheronline.com/free/v2/past-weather.ashx">https://api.worldweatheronline.com/free/v2/past-weather.ashx</a></p> 
             <p><b>Premium API:</b><br> HTTP: <a href="http://api.worldweatheronline.com/premium/v1/past-weather.ashx">http://api.worldweatheronline.com/premium/v1/past-weather.ashx</a><br> HTTPS: <a href="https://api.worldweatheronline.com/premium/v1/past-weather.ashx">https://api.worldweatheronline.com/premium/v1/past-weather.ashx</a></p> 
             <p>For our application, we want to pass a number of parameters to ensure that we get the correct response from our API, heres what we need:</p> 
             <p><strong>Date:</strong> We want to pass in the date for each Christmas day since 2008.<br> <strong>Search String:</strong> For our location we want information for New York.<br> <strong>Format:</strong> We want our information returned in JSON format.<br> <strong>API Key:</strong> Our API key for authorisation.</p> 
             <p>So to get information for Christmas day in 2008 our URL would look like this:</p> 
             <pre>https://api.worldweatheronline.com/premium/v1/past-weather.ashx?q=new+york&amp;tp=24&amp;format=json&amp;key=ourApiKey&amp;date=2008-12-25</pre> 
             <p><strong>Retrieving and displaying API Data</strong></p> 
             <p>Now lets update our app.get function; replace the console.log in the app.get function with the below:</p> 
             <pre>var requestUrl = 'https://api.worldweatheronline.com/premium/v1/past-weather.ashx?q=new+york&amp;tp=24&amp;format=json&amp;key=' + apiKey;

var years = [2008, 2009, 2010, 2011, 2012, 2013, 2014];
var dateString = '-12-25';

var results = [];

years.forEach(function(year, index) {
   var currentUrl = requestUrl + '&amp;date=' + year + dateString;

   request(currentUrl, function (error, response, body) {
        if (!error &amp;&amp; response.statusCode == 200) {
            var result = JSON.parse(body);
            results[index] = result.data.weather[0];
        } else {
            console.log(error, response);
        }
    });
});
</pre> 
             <p>Here were setting a base API URL that stores the parts of the URL which wont change for each year. As were querying a number of separate years, we need to make multiple requests to retrieve all of the information we require.</p> 
             <p>In the next few lines we setup some variables to hold all the years which we want information for and to hold the results.</p> 
             <p>We then repeat for each year we want information for, to build the date string for Christmas Day on each year and make the request to the API. We then store the results in the results array using the an index to keep them in the right order.</p> 
             <p>Because of the way node.js handles these connections asynchronously, the requests can finish in a different order than they were made.</p> 
             <p>Therefore we make sure we keep the results in order by using the year index but we also need to make sure all requests have finished before working with the data, but well do this next in our app.get function</p> 
             <pre>    var checkTimer = setInterval(function() {
        if (results.length == years.length) {
            console.log("we've got all of the results");
            clearInterval(checkTimer);
            displayResults(results);
        } else {
            console.log("Not finished yet, lets wait another 100ms");
        }
    }, 100);
</pre> 
             <p>Here were setting a 100ms timer to check the progress of the requests. We compare the length of the years array to the length of the results array and once they match we can then work on displaying this information.</p> 
             <p>Lets display our results in a tabular format, our displayResults function goes below the above code in the app.get function:</p> 
             <pre>    
    var displayResults = function(results) {

        // generate a HTML table
        var html = '&lt;table style="font-size: 10px; font-family: Arial, Helvetica, sans-serif"&gt;';

        // loop through each row
        for (var i = 0; i &lt; 3; i++) {
            html += "&lt;tr&gt;";

            results.forEach(function(result) {
                html += "&lt;td&gt;";
                switch (i) {
                    case 0:
                        html += result.date;
                        break;
                    case 1:
                        html += result.hourly[0].weatherDesc[0].value;
                        break;
                    case 2:
                        var imgSrc = result.hourly[0].weatherIconUrl[0].value;
                        html += '&lt;img src="'+ imgSrc + '" /&gt;';
                        break;
                }
                html += "&lt;/td&gt;";
            });
            html += "&lt;/tr&gt;";
        }

        res.end(html);
    };
</pre> 
             <p>This code just iterates over the results creating a HTML table to display the results. Once you have entered this you can run it again by running node app.js and you should see the following displayed</p> 
             <p><a href="https://scriptableltd.files.wordpress.com/2015/02/screen-shot-2015-02-12-at-20-24-17.png"><img class="alignnone wp-image-34 size-full" src="https://scriptableltd.files.wordpress.com/2015/02/screen-shot-2015-02-12-at-20-24-17.png" alt="Christmas snow fall in New York" width="1466" height="353"></a></p> 
             <p><strong>Summary</strong></p> 
             <p>This shows us that New York has not had a white Christmas for at least the last 7 years, which is quite disappointing for traditionalists!</p> 
             <p>However it does show how you can use our API to get historical data. You can play around with this, change the location to your home location and see the results.</p> 
             <p>The full code for this guide has been posted on <a href="http://jsfiddle.net/7ahj920y/">JsFiddle</a> for your reference.</p> 
             <p>This isjust a quick starter guide to give you ideas how can use our API in your own applications. Keep checking back for more useful guides!</p> 
             <p><strong>If you have any more questions concerning our API, visit <a href="http://stackoverflow.com/">StackOverflow</a> and post a question where our team is always looking to help World Weather Online customers get up and running.</strong></p>
