           <h2 name="dd12" id="dd12" class="graf graf--h4 graf-after--h3 graf--subtitle">If you have mailed a package or letter in the past, then you can understand closures in JavaScript.</h2>
           <p name="6621" id="6621" class="graf graf--p graf-after--h4">On your journey to becoming an intermediate or advanced JavaScript dev, you may have come across closures. After reading a technical resource on the subject… you also probably ran in the opposite direction.</p>
           <p name="59e5" id="59e5" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Here is the awesome thing about closures</strong>: they allow you to write functions with an intermediate step that can capture data from your site at a specific moment in time. It’s like adding a ‘pause’ button to your function. You can run your function and save the value of a variable at that particular point in time. Then, when you want to resume the function at a later point and use values of variables that have changed within your app… you can do that with a <strong class="markup--strong markup--p-strong">closure,</strong> or a function <strong class="markup--strong markup--p-strong">within the original function</strong>.</p>
           <p name="d199" id="d199" class="graf graf--p graf-after--p">This gets easier, I promise.</p>
           <p name="5ffe" id="5ffe" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">So, when the heck would you use a closure?</strong></p>
           <p name="af89" id="af89" class="graf graf--p graf-after--p">Let’s say you are building an interactive map of tourist landmarks in New York City using the Google Maps API. You have an array with a bunch of map markers that you want to add to the map- the Statue of Liberty, Empire State Building, Coney Island, you name it. You want to add all these markers to the map, but you also want to add a click event to each marker. When you click the marker, you want to show dynamic information about that marker, including live weather data.</p>
           <pre name="8fe5" id="8fe5" class="graf graf--pre graf-after--p">var touristPlaces= […];</pre>
           <pre name="388f" id="388f" class="graf graf--pre graf-after--pre">for(var i=0; i&lt; touristPlaces.length; i++){<br>  var marker= touristPlaces[i];<br>  $(marker).click(function(){<br>    showToolTip(i)<br>  });<br>}</pre>
           <p name="5d5f" id="5d5f" class="graf graf--p graf-after--pre">Here’s the issue- if you write it like this, <a href="http://stackoverflow.com/questions/2622421/what-are-the-use-cases-for-closures-callback-functions-in-javascript" data-href="http://stackoverflow.com/questions/2622421/what-are-the-use-cases-for-closures-callback-functions-in-javascript" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">it will not work</a>. The ‘for’ loop will finish before the callback in the click event can register the appropriate i value. You need to capture this intermediate point so you can call the function later with the appropriate i.</p>
           <p name="7293" id="7293" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">What do you need to know first?</strong></p>
           <ol class="postList">
            <li name="207c" id="207c" class="graf graf--li graf-after--p">Variable scoping</li>
            <li name="2a73" id="2a73" class="graf graf--li graf-after--li">The concept of callbacks (I wrote <a href="https://medium.freecodecamp.com/javascript-callbacks-explained-using-minions-da272f4d9bcd#.rf0y0hl26" data-href="https://medium.freecodecamp.com/javascript-callbacks-explained-using-minions-da272f4d9bcd#.rf0y0hl26" class="markup--anchor markup--li-anchor" rel="noopener" target="_blank">a guide on this too</a>!)</li>
           </ol>
           <p name="53eb" id="53eb" class="graf graf--p graf-after--li">If you are looking for a technical explanation of closures, <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Closures" data-href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Closures" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">the guide on MDN</a> is probably the best.</p>
           <h4 name="4092" id="4092" class="graf graf--h4 graf-after--p">Closures have the same process as mailing a&nbsp;package.</h4>
           <p name="c5af" id="c5af" class="graf graf--p graf-after--h4">Let’s look at some basic code that uses a closure to mail a package.</p>
           <figure name="dc8a" id="dc8a" class="graf graf--figure graf--iframe graf-after--p">

           <p name="3202" id="3202" class="graf graf--p graf-after--figure">The addressPackage() function is a <strong class="markup--strong markup--p-strong">closure</strong>! It can be called at any time after the packBox function has been called. It also has access to the variables and arguments from the time when packBox() was originally called.</p>
           <p name="fdc6" id="fdc6" class="graf graf--p graf-after--p">Notice how the console.log output does not show until lines 14 and 15? <strong class="markup--strong markup--p-strong">This is extremely important. </strong>If you ran this code after line 11, you would simply see ‘Put jersey in box’. There would be no error, but the closure, addressPackage(), would not run at that point.</p>
           <p name="310b" id="310b" class="graf graf--p graf-after--p">When you are mailing a package, you would probably agree that your job is not done until the package is filled and the address is written. Likewise, the packBox() function waits until the closure has also been called. Let’s go through this line-by-line.</p>
           <p name="470f" id="470f" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Line 11:</strong> You create the variable brotherGift, which is an<strong class="markup--strong markup--p-strong"> instance</strong> of the packBox() function. You are sending a jersey to your brother.</p>
           <p name="b5b4" id="b5b4" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Line 3:</strong> Your code logs a statement about the jersey.</p>
           <p name="184b" id="184b" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Line 8:</strong> The packBox() function returns… another function? Huh?</p>
           <p name="98f2" id="98f2" class="graf graf--p graf-after--p">Let’s stop here, and assume that line 13 has not run yet. Here is what is happening: The packBox() function will not return the “ready to send” line until you also call the addressPackage() function with an argument. Just like there are two steps to sending a package: first, filling it, and second, addressing it. Your package is worthless if it has no contents or it does not have an address! That being said, you do not necessarily need to address the package directly after you fill the contents. You can wait a few days before addressing it. You might need to go to your computer to look up the address. You might be waiting for your brother to officially change his address!</p>
           <p name="7ed1" id="7ed1" class="graf graf--p graf-after--p">Regardless, if you do not address the package immediately, this does not mean that the package will somehow magically empty itself. <strong class="markup--strong markup--p-strong">The contents will still be there when you return to address it!</strong> So, any time we call brotherGift, the first argument, jersey, will still be available.</p>
           <p name="a50b" id="a50b" class="graf graf--p graf-after--p">…Waiting…Waiting…Now let’s run line 13.</p>
           <p name="b898" id="b898" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Line 13:</strong> Alright, let’s finish off this<strong class="markup--strong markup--p-strong"> instance</strong>! You are ready to add the address, so you call brotherGift and offer the address as an argument. Remember from line 11, brotherGift is an <strong class="markup--strong markup--p-strong">instance</strong> of packBox with the ‘jersey’ argument. So when you call it, you are offering another argument, which will then be sent to the closure: addressPackage();</p>
           <p name="a688" id="a688" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Line 3: </strong>The console.log will show since we are now running the code from line 13.</p>
           <p name="149e" id="149e" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Line 4: </strong>We now offer the second argument to addressPackage();</p>
           <p name="f2a1" id="f2a1" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Line 6:</strong> addressPackage logs a statement related to the address argument.</p>
           <p name="cfcc" id="cfcc" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Line 8:</strong> The return statement can fire for this instance.</p>
           <p name="cf90" id="cf90" class="graf graf--p graf-after--p">Again, closures allow us to have this intermediate instance where one argument has been filled, but brotherGift is left unfulfilled until we add the second argument. <strong class="markup--strong markup--p-strong">If we wanted to do this in one line</strong>, we would write: packBox(‘jersey’)(‘123 Main Street, Anywhere USA 01234’);</p>

           <p name="5265" id="5265" class="graf graf--p graf-after--h4">Let’s say that you wanted to send a gift to each member of your family. You might pack each box before adding the addresses to each. This is what that looks like in code.</p>

           <p name="b38b" id="b38b" class="graf graf--p graf-after--figure">Another magical feature of closures! Each instance is able to use the correct gift item with the correct address, even thought we run the function with 4 separate gift/address pairs. In a traditional function, there is no concept of memory. You would need to explicitly restate the original gifts in lines 6–15 if you wanted to use a traditional function.</p>
