       <h1>How to Get Started with Code (JavaScript) on Zapier<a class="rel" href="#how-get-started-code-zapier">#</a></h1> 
       <blockquote> 
        <p>Looking for Python instead of JavaScript? Visit our <a href="/help/code-python/">Code (Python) documentation</a>.</p> 
       </blockquote> 
       <a name="introduction"></a>
       <h2>Introduction<a class="rel" href="#introduction">#</a></h2> 
       <p>With Zapier's Code action you can extend our platform to do <em>just about anything</em> using the most ubiquitous programming language on the web - JavaScript! Run code in response to any trigger that Zapier supports.</p> 
       <p>Some example use cases include:</p> 
       <ul> 
        <li>Transform incorrect dates or convert between other data types.</li> 
        <li>Use custom regular expressions to extract extra data like emails or tracking numbers from large text blobs.</li> 
        <li>Make an extra API call to a different service with <code>fetch</code> without building a full dev app.</li> 
        <li>Augment data from a trigger with extra data from some other source (either generated or external API).</li> 
        <li>Anything else you can dream up!</li> 
       </ul> 
       <blockquote> 
        <p><strong>Warning</strong> - this is advanced stuff! You probably need to be a programmer to use this - though you are welcome to play with it. Technical support for bugs in your code is not provided!</p> 
       </blockquote> 
       <a name="how-does-it-work"></a>
       <h2>How does it work?<a class="rel" href="#how-does-it-work">#</a></h2> 
       <p>The environment is vanilla node.js v4.3.2 which runs JavaScript. Your script is sandboxed and can only run for a limited amount of time and within a limited amount of memory. If you exceed those limits - your script will be killed (you can upgrade to a paid Zapier to increase your limits).</p> 
       <p>Since the amount of data that might feed into your script might be large or highly dynamic - you'll need to define an inputData mapping via our GUI. This is a really simple step - the screenshot below shows a very basic example:</p> 
       <p><img src="https://cdn.zapier.com/storage/photos/c30f2b170737ba3863c17b714662d9dd.png" alt=""></p> 
       <blockquote> 
        <p>Scroll to the examples section to see an assortment of starter scripts.</p> 
       </blockquote> 
       <p>In your code you will have access to a few variables:</p> 
       <a name="data-variables"></a>
       <h2>Data Variables<a class="rel" href="#data-variables">#</a></h2> 
       <ul> 
        <li><code>inputData</code> The mapping of data you did right above your code snippet (not available in triggers). All values will be strings.</li> 
        <li><code>output</code> An object or array of objects that will be the "return value" of this code. You can explicitly return early if you like.</li> 
       </ul> 
       <blockquote> 
        <p>Setting the <code>output</code> to an array of objects will run the subsequent steps multiple times — once for each object in the array. If Code by Zapier is the Zap's trigger and an empty array is returned, nothing happens.</p> 
       </blockquote> 
       <a name="utilities"></a>
       <h2>Utilities<a class="rel" href="#utilities">#</a></h2> 
       <ul> 
        <li><code>callback(err, output)</code> (optional) A callback if you need to do async work - whatever you set to the <code>output</code> data variable is ignored since you provide it directly here. We inspect your code and make our best guess if you are using <code>callback</code> or not!</li> 
        <li><code>fetch</code> An easy to use HTTP client - <a href="https://github.com/bitinn/node-fetch/tree/32b60634434a63865ea3f79edb33d17e40876c9f#usage">read the documentation here</a>.</li> 
        <li><code>console.log</code> Super helpful if you want to debug your function - you'll need to test your zap to see the values (the logs are returned to you in a <code>runtime_meta</code> added automatically to your <code>output</code>).</li> 
       </ul> 
       <a name="testingdebugging"></a>
       <h2>Testing/Debugging<a class="rel" href="#testingdebugging">#</a></h2> 
       <p>Running your zap via the dashboard is the canonical way to confirm the behavior you expect - your Task History will have all relevant details around the ran code's inputData, output and logs. The test step in the editor can be used for a tighter feedback loop.</p>

       <h1>Code (Javascript) Examples<a class="rel" href="#code-examples">#</a></h1> 
       <blockquote> 
        <p><strong>Important!</strong> Every example depends on specific <code>inputData</code> - which is provided under the "Input Data" field when setting up your - <a href="https://cdn.zapier.com/storage/photos/07a37cd88d3c9de7949dd9fcf5070b13.png">this example screenshot</a> shows three demonstration inputs you can use in your code like this: <code>inputData.body</code>, <code>inputData.receivedDate</code>, and <code>inputData.subject</code>. Be sure you read the code examples and provide the right inputs otherwise nothing will work as expected!</p> 
       </blockquote> 
       <p>In this section we provide some examples - it is important to note that javascript is an advanced programming language - if you get lost it might make sense to ask a programmer friend or <a href="https://www.codecademy.com/tracks/javascript" rel="nofollow" target="_blank">learn javascript</a> yourself.</p> 
       <a name="introductory-examples"></a>
       <h2>Introductory Examples<a class="rel" href="#introductory-examples">#</a></h2> 
       <p>Each of the four examples below expects a <code>name</code> in the "Input Data" field.</p> 
       <p>A synchronous example might be something as trivial as:</p> 
       <div class="highlight">
        <pre><span class="k">return</span> <span class="p">{</span><span class="nx">id</span><span class="o">:</span> <span class="mi">1234</span><span class="p">,</span> <span class="nx">hello</span><span class="o">:</span> <span class="s1">'world!'</span><span class="p">,</span> <span class="nx">name</span><span class="o">:</span> <span class="nx">inputData</span><span class="p">.</span><span class="nx">name</span><span class="p">};</span>
</pre>
       </div> 
       <p>You can also bind the result to <code>output</code> - the code above has <strong>exactly the same</strong> behavior as the code below:</p> 
       <div class="highlight">
        <pre><span class="nx">output</span> <span class="o">=</span> <span class="p">{</span><span class="nx">id</span><span class="o">:</span> <span class="mi">1234</span><span class="p">,</span> <span class="nx">hello</span><span class="o">:</span> <span class="s1">'world!'</span><span class="p">,</span> <span class="nx">name</span><span class="o">:</span> <span class="nx">inputData</span><span class="p">.</span><span class="nx">name</span><span class="p">};</span>
</pre>
       </div> 
       <blockquote> 
        <p>You'll notice that when it comes to code - you can solve the same problems in hundreds of unique ways!</p> 
       </blockquote> 
       <p>A synchronous example with an early empty return might be something as trivial as:</p> 
       <div class="highlight">
        <pre><span class="k">if</span> <span class="p">(</span><span class="nx">inputData</span><span class="p">.</span><span class="nx">name</span> <span class="o">===</span> <span class="s1">'Larry'</span><span class="p">)</span> <span class="p">{</span>
  <span class="k">return</span> <span class="p">[];</span> <span class="c1">// we don't work for Larry!</span>
<span class="p">}</span>
<span class="k">return</span> <span class="p">{</span><span class="nx">id</span><span class="o">:</span> <span class="mi">1234</span><span class="p">,</span> <span class="nx">hello</span><span class="o">:</span> <span class="s1">'world!'</span><span class="p">,</span> <span class="nx">name</span><span class="o">:</span> <span class="nx">inputData</span><span class="p">.</span><span class="nx">name</span><span class="p">};</span>
</pre>
       </div> 
       <blockquote> 
        <p>If Code by Zapier is the Zap's trigger and you return an empty array <code>[]</code>, we will not trigger any actions downstream — it is as if you said "nevermind" in code.</p> 
       </blockquote> 
       <p>An asynchronous example might be something as trivial as:</p> 
       <div class="highlight">
        <pre><span class="nx">callback</span><span class="p">(</span><span class="kc">null</span><span class="p">,</span> <span class="p">{</span><span class="nx">id</span><span class="o">:</span> <span class="mi">1234</span><span class="p">,</span> <span class="nx">hello</span><span class="o">:</span> <span class="s1">'world!'</span><span class="p">,</span> <span class="nx">name</span><span class="o">:</span> <span class="nx">inputData</span><span class="p">.</span><span class="nx">name</span><span class="p">});</span>
</pre>
       </div>

       <p>It isn't uncommon to want to stash some data away for the next run, our <code>StoreClient</code> can do exactly that. For example - this is a counter that counts how many times it is ran:</p> 
       <div class="highlight">
        <pre><span class="kd">var</span> <span class="nx">store</span> <span class="o">=</span> <span class="nx">StoreClient</span><span class="p">(</span><span class="s1">'your secret here'</span><span class="p">);</span>
<span class="kd">var</span> <span class="nx">outCount</span><span class="p">;</span>
<span class="nx">store</span>
  <span class="p">.</span><span class="nx">get</span><span class="p">(</span><span class="s1">'some counter'</span><span class="p">)</span>
  <span class="p">.</span><span class="nx">then</span><span class="p">(</span><span class="kd">function</span><span class="p">(</span><span class="nx">count</span><span class="p">)</span> <span class="p">{</span>
    <span class="nx">count</span> <span class="o">=</span> <span class="p">(</span><span class="nx">count</span> <span class="o">||</span> <span class="mi">0</span><span class="p">)</span> <span class="o">+</span> <span class="mi">1</span><span class="p">;</span>
    <span class="nx">outCount</span> <span class="o">=</span> <span class="nx">count</span><span class="p">;</span>
    <span class="k">return</span> <span class="nx">store</span><span class="p">.</span><span class="nx">set</span><span class="p">(</span><span class="s1">'some counter'</span><span class="p">,</span> <span class="nx">count</span><span class="p">);</span>
  <span class="p">})</span>
  <span class="p">.</span><span class="nx">then</span><span class="p">(</span><span class="kd">function</span><span class="p">()</span> <span class="p">{</span>
    <span class="nx">callback</span><span class="p">(</span><span class="kc">null</span><span class="p">,</span> <span class="p">{</span><span class="s1">'the count'</span><span class="o">:</span> <span class="nx">outCount</span><span class="p">});</span>
  <span class="p">})</span>
  <span class="p">.</span><span class="k">catch</span><span class="p">(</span><span class="nx">callback</span><span class="p">);</span>
</pre>

       <a name="storeclient-javascript"></a> 
       <h1>StoreClient (Javascript)<a class="rel" href="#storeclient-javascript">#</a></h1> 
       <p>Storing and retrieving data with <code>StoreClient</code> is very simple.</p> 
       <p>There is no need to require it - it comes pre-imported in your Code environment.</p> 
       <blockquote> 
        <p>You will need to provide a secret that will protect your data. I recommend using <a href="https://www.random.org/passwords/?num=2&amp;len=16&amp;format=html&amp;rnd=new">Random.org's excellent password generator</a>. Be very sure you pick a complex secret - <strong>if anyone guesses it they'll be able to read and write your data!</strong></p> 
       </blockquote> 
       <p>Instantiating a client is very simple:</p> 
       <div class="highlight">
        <pre><span class="kd">var</span> <span class="nx">store</span> <span class="o">=</span> <span class="nx">StoreClient</span><span class="p">(</span><span class="s1">'your secret here'</span><span class="p">);</span>
</pre>
       </div> 
       <blockquote> 
        <p><code>StoreClient</code> is a promise based library - get ready for some <code>then()</code> and <code>catch()</code> action.</p> 
       </blockquote> 
       <p>Most likely you just want to <code>get</code> and <code>set</code> some values - this is the simplest possible example:</p> 
       <div class="highlight">
        <pre><span class="kd">var</span> <span class="nx">store</span> <span class="o">=</span> <span class="nx">StoreClient</span><span class="p">(</span><span class="s1">'your secret here'</span><span class="p">);</span>
<span class="nx">store</span>
  <span class="p">.</span><span class="nx">set</span><span class="p">(</span><span class="s1">'hello'</span><span class="p">,</span> <span class="s1">'world'</span><span class="p">)</span>
  <span class="p">.</span><span class="nx">then</span><span class="p">(</span><span class="kd">function</span><span class="p">()</span> <span class="p">{</span>
    <span class="k">return</span> <span class="nx">store</span><span class="p">.</span><span class="nx">get</span><span class="p">(</span><span class="s1">'hello'</span><span class="p">);</span>
  <span class="p">})</span>
  <span class="p">.</span><span class="nx">then</span><span class="p">(</span><span class="kd">function</span><span class="p">(</span><span class="nx">value</span><span class="p">)</span> <span class="p">{</span>
    <span class="c1">// value === 'world'</span>
    <span class="k">return</span> <span class="nx">store</span><span class="p">.</span><span class="k">delete</span><span class="p">(</span><span class="s1">'hello'</span><span class="p">);</span>
  <span class="p">})</span>
  <span class="p">.</span><span class="nx">then</span><span class="p">(</span><span class="kd">function</span><span class="p">()</span> <span class="p">{</span>
    <span class="nx">callback</span><span class="p">();</span>
  <span class="p">})</span>
  <span class="p">.</span><span class="k">catch</span><span class="p">(</span><span class="nx">callback</span><span class="p">);</span>
</pre>
       </div> 
       <p>If the value doesn't exist during your <code>get()</code> call - it will return a <code>null</code> value.</p> 
       <a name="bulk-operations"></a>
       <h2>Bulk Operations<a class="rel" href="#bulk-operations">#</a></h2> 
       <p>You can also save and retrieve multiple keys and values - a slightly more complex example:</p> 
       <div class="highlight">
        <pre><span class="kd">var</span> <span class="nx">store</span> <span class="o">=</span> <span class="nx">StoreClient</span><span class="p">(</span><span class="s1">'your secret here'</span><span class="p">);</span>
<span class="nx">store</span>
  <span class="p">.</span><span class="nx">setMany</span><span class="p">({</span><span class="nx">hello</span><span class="o">:</span> <span class="s1">'world'</span><span class="p">,</span> <span class="nx">foo</span><span class="o">:</span> <span class="s1">'bar'</span><span class="p">})</span>
  <span class="p">.</span><span class="nx">then</span><span class="p">(</span><span class="kd">function</span><span class="p">()</span> <span class="p">{</span>
    <span class="k">return</span> <span class="nx">store</span><span class="p">.</span><span class="nx">getMany</span><span class="p">(</span><span class="s1">'hello'</span><span class="p">,</span> <span class="s1">'foo'</span><span class="p">);</span>
  <span class="p">})</span>
  <span class="p">.</span><span class="nx">then</span><span class="p">(</span><span class="kd">function</span><span class="p">(</span><span class="nx">values</span><span class="p">)</span> <span class="p">{</span>
    <span class="c1">// value === {hello: 'world', foo: 'bar'}</span>
    <span class="k">return</span> <span class="nx">store</span><span class="p">.</span><span class="nx">deleteMany</span><span class="p">(</span><span class="s1">'hello'</span><span class="p">,</span> <span class="s1">'foo'</span><span class="p">);</span>
  <span class="p">})</span>
  <span class="p">.</span><span class="nx">then</span><span class="p">(</span><span class="kd">function</span><span class="p">()</span> <span class="p">{</span>
    <span class="c1">// or, if you want to wipe everything</span>
    <span class="k">return</span> <span class="nx">store</span><span class="p">.</span><span class="nx">clear</span><span class="p">();</span>
  <span class="p">})</span>
  <span class="p">.</span><span class="nx">then</span><span class="p">(</span><span class="kd">function</span><span class="p">()</span> <span class="p">{</span>
    <span class="nx">callback</span><span class="p">();</span>
  <span class="p">})</span>
  <span class="p">.</span><span class="k">catch</span><span class="p">(</span><span class="nx">callback</span><span class="p">);</span>
</pre>
       </div> 
       <p>Note, you can call <code>getMany</code> and <code>deleteMany</code> in a few different ways:</p> 
       <div class="highlight">
        <pre><span class="nx">store</span><span class="p">.</span><span class="nx">getMany</span><span class="p">(</span><span class="s1">'hello'</span><span class="p">,</span> <span class="s1">'foo'</span><span class="p">);</span> <span class="c1">// as arguments</span>
<span class="nx">store</span><span class="p">.</span><span class="nx">getMany</span><span class="p">([</span><span class="s1">'hello'</span><span class="p">,</span> <span class="s1">'foo'</span><span class="p">]);</span> <span class="c1">// as array</span>
<span class="nx">store</span><span class="p">.</span><span class="nx">getMany</span><span class="p">({</span><span class="nx">hello</span><span class="o">:</span> <span class="kc">null</span><span class="p">,</span> <span class="nx">foo</span><span class="o">:</span> <span class="kc">null</span><span class="p">});</span> <span class="c1">// as object</span>
</pre>

       <a name="limitations"></a>
       <h2>Limitations<a class="rel" href="#limitations">#</a></h2> 
       <ul> 
        <li>Any JSON serializable value can be saved.</li> 
        <li>The secret must be under 32 chars in length.</li> 
        <li>Every key must be under 32 chars in length.</li> 
        <li>Every value must be under 2500 bytes.</li> 
        <li>Only 500 keys may be saved per secret.</li> 
        <li>Keys will expire if you do not touch them in 3 months.</li> 
       </ul>

       <a name="common-problems-code-zapier"></a> 
       <h1>Common Problems with Code (Javascript) on Zapier<a class="rel" href="#common-problems-code-zapier">#</a></h1> 
       <a name="i-need-help"></a>
       <h2>I need help!<a class="rel" href="#i-need-help">#</a></h2> 
       <p>Try asking for help by tagging questions as <a href="http://stackoverflow.com/questions/tagged/zapier">Zapier on Stack Overflow</a>!</p> 
       <a name="i-keep-getting-quotprocess-exited-before-completing-requestquot-what-gives"></a>
       <h2>I keep getting "Process exited before completing request" - what gives?<a class="rel" href="#i-keep-getting-quotprocess-exited-before-completing-requestquot-what-gives">#</a></h2> 
       <p>9 times out of 10 this happens if your script completes without calling the <code>callback</code> - the most common case is if your <code>fetch().then()</code> doesn't have a <code>.catch()</code> for errors. A simple <code>.catch(callback)</code> will suffice because the error will be passed as the first argument.</p> 
       <a name="39nonetype39-object-does-not-support-item-assignment"></a>
       <h2>'NoneType' object does not support item assignment<a class="rel" href="#39nonetype39-object-does-not-support-item-assignment">#</a></h2> 
       <p>This shows up when you redefine some of the important variables in the function, namely <code>callback</code>. Lambda expects <code>callback</code> to be there to complete an async function, so if you've redefined it you'll get some strange errors. </p> 
       <a name="requiring-or-using-external-libraries"></a>
       <h2>Requiring or Using External Libraries<a class="rel" href="#requiring-or-using-external-libraries">#</a></h2> 
       <p>Unfortunately you cannot require external libraries or install or import libraries commonly referred to as "npm modules". Only the standard node.js library and the <a href="https://www.npmjs.com/package/fetch"><code>fetch</code> package</a> are available in the Code app. <code>fetch</code> is already included in the namespace.</p> 
       <a name="time-amp-memory-limits"></a>
       <h2>Time &amp; Memory Limits<a class="rel" href="#time-amp-memory-limits">#</a></h2> 
       <p>Free users are limited to <code>1 second</code> and <code>128mb of RAM</code>. Paid users get well over <strong>double</strong> that at <code>10 seconds</code> and <code>256mb of RAM</code>. Your Zap will hit an error if you exceed these limits.</p> 
       <a name="i-am-getting-various-syntaxerror39s-and-other-weird-bugs"></a>
       <h2>I am getting various SyntaxError's and other weird bugs.<a class="rel" href="#i-am-getting-various-syntaxerror39s-and-other-weird-bugs">#</a></h2> 
       <p>Javascript is a fairly complex language and you should probably be a programmer or willing to dedicate some serious time to learn to code - while we can't help you fix your code (try asking for by tagging questions as <a href="http://stackoverflow.com/questions/tagged/zapier">Zapier on Stackoverflow</a>) - here are some helpful tools:</p> 
       <a name="use-a-linter"></a>
       <h3>Use a Linter<a class="rel" href="#use-a-linter">#</a></h3> 
       <p>Linters do an amazing job of letting you know when things are broken - for example - we love <a href="http://jshint.com/">http://jshint.com/</a> and you can paste your code directly there! For example, let's say our code looks like this and we keep getting an error about <code>response</code> not being defined:</p> 
       <div class="highlight">
        <pre><span class="c1">// this is bad code - but we don't know why...</span>
<span class="nx">fetch</span><span class="p">(</span><span class="s1">'http://example.com/'</span><span class="p">)</span>
  <span class="p">.</span><span class="nx">then</span><span class="p">(</span><span class="kd">function</span><span class="p">(</span><span class="nx">res</span><span class="p">)</span> <span class="p">{</span>
    <span class="k">return</span> <span class="nx">res</span><span class="p">.</span><span class="nx">text</span><span class="p">();</span>
  <span class="p">}).</span><span class="nx">then</span><span class="p">(</span><span class="kd">function</span><span class="p">(</span><span class="nx">body</span><span class="p">)</span> <span class="p">{</span>
    <span class="nx">callback</span><span class="p">(</span><span class="kc">null</span><span class="p">,</span> <span class="p">{</span><span class="nx">id</span><span class="o">:</span> <span class="mi">1234</span><span class="p">,</span> <span class="nx">rawHTML</span><span class="o">:</span> <span class="nx">response</span><span class="p">});</span>
  <span class="p">}).</span><span class="k">catch</span><span class="p">(</span><span class="kd">function</span><span class="p">(</span><span class="nx">error</span><span class="p">)</span> <span class="p">{</span>
    <span class="nx">callback</span><span class="p">(</span><span class="nx">error</span><span class="p">);</span>
  <span class="p">});</span>
</pre>
       </div> 
       <p>The error is driving us crazy! Now in <a href="http://jshint.com/">http://jshint.com/</a> we paste this (note the extra globals definition so jshint knows about Zapier's provided variables):</p> 
       <div class="highlight">
        <pre><span class="cm">/* globals inputData, output: true, fetch, callback */</span>

<span class="c1">// this is bad code - but we don't know why...</span>
<span class="nx">fetch</span><span class="p">(</span><span class="s1">'http://example.com/'</span><span class="p">)</span>
  <span class="p">.</span><span class="nx">then</span><span class="p">(</span><span class="kd">function</span><span class="p">(</span><span class="nx">res</span><span class="p">)</span> <span class="p">{</span>
    <span class="k">return</span> <span class="nx">res</span><span class="p">.</span><span class="nx">text</span><span class="p">();</span>
  <span class="p">}).</span><span class="nx">then</span><span class="p">(</span><span class="kd">function</span><span class="p">(</span><span class="nx">body</span><span class="p">)</span> <span class="p">{</span>
    <span class="nx">callback</span><span class="p">(</span><span class="kc">null</span><span class="p">,</span> <span class="p">{</span><span class="nx">id</span><span class="o">:</span> <span class="mi">1234</span><span class="p">,</span> <span class="nx">rawHTML</span><span class="o">:</span> <span class="nx">response</span><span class="p">});</span>
  <span class="p">}).</span><span class="k">catch</span><span class="p">(</span><span class="kd">function</span><span class="p">(</span><span class="nx">error</span><span class="p">)</span> <span class="p">{</span>
    <span class="nx">callback</span><span class="p">(</span><span class="nx">error</span><span class="p">);</span>
  <span class="p">});</span>
</pre>
       </div> 
       <p>If we look at the results we see that <code>response</code> is indeed incorrect! We should have used <code>body</code>:</p>
