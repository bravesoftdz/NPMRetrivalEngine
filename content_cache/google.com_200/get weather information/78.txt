           <h3 name="0c0f" id="0c0f" class="graf graf--h3 graf-after--p">Making the API&nbsp;call</h3>
           <p name="4f31" id="4f31" class="graf graf--p graf-after--h3">To make our API call, we’ll be using a popular npm module called <a href="https://www.npmjs.com/package/request" data-href="https://www.npmjs.com/package/request" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">request</a>. request has millions of downloads and is a module that simplifies the code needed to make an http request in node.</p>
           <p name="65d2" id="65d2" class="graf graf--p graf-after--p">Install request by running:</p>
           <pre name="0dac" id="0dac" class="graf graf--pre graf-after--p">npm install request --save</pre>
           <p name="1f18" id="1f18" class="graf graf--p graf-after--pre">As I said, request is pretty easy to use. We just need to pass in our target url, and request returns a callback function. Our starter code looks like this:</p>
           <pre name="ec1b" id="ec1b" class="graf graf--pre graf-after--p">const request = require('request');</pre>
           <pre name="5d37" id="5d37" class="graf graf--pre graf-after--pre">request(url, function (err, response, body) {<br>  if(err){<br>    console.log('error:', error);<br>  } else {<br>    console.log('body:', body);<br>  }<br>});</pre>
           <p name="cb68" id="cb68" class="graf graf--p graf-after--pre">Lets break this code down:</p>
           <ol class="postList">
            <li name="49f0" id="49f0" class="graf graf--li graf-after--p">We require the request package</li>
            <li name="5140" id="5140" class="graf graf--li graf-after--li">We pass in a url, and request returns a callback function with three arguments: <code class="markup--code markup--li-code">err</code>, <code class="markup--code markup--li-code">response</code>, and <code class="markup--code markup--li-code">body</code>.</li>
            <li name="145f" id="145f" class="graf graf--li graf-after--li">We check for an error in our request. If there is one, we log the error and are done.</li>
            <li name="f999" id="f999" class="graf graf--li graf-after--li">If there is no error, we log the entire contents of the response body.</li>
           </ol>
           <p name="8222" id="8222" class="graf graf--p graf-after--li"><strong class="markup--strong markup--p-strong">Awesome, so what is the url we’re making the request to?</strong></p>
           <p name="2725" id="2725" class="graf graf--p graf-after--p">By reading the OpenWeatherMap <a href="https://openweathermap.org/current" data-href="https://openweathermap.org/current" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Documentation</a>, we are able to determine that this is url we should make our requests to: <code class="markup--code markup--p-code"><a href="http://api.openweathermap.org/data/2.5/weather" data-href="http://api.openweathermap.org/data/2.5/weather" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">http://api.openweathermap.org/data/2.5/weather</a></code> The URL also has two required query parameters. <strong class="markup--strong markup--p-strong">Query parameters are key/value pairs that allow us to pass in data to a URL</strong>. In this instance we need to include the city we’re searching for, and our API Key.</p>
           <p name="9664" id="9664" class="graf graf--p graf-after--p">Here’s what the code will look like:</p>
           <pre name="b1a2" id="b1a2" class="graf graf--pre graf-after--p">let apiKey = '****************************';<br>let city = 'portland';<br>let url = `<a href="http://api.openweathermap.org/data/2.5/weather?q=$%7Bcity%7D&amp;units=imperial&amp;appid=$%7BapiKey%7D`" data-href="http://api.openweathermap.org/data/2.5/weather?q=${city}&amp;units=imperial&amp;appid=${apiKey}`" class="markup--anchor markup--pre-anchor" rel="nofollow noopener" target="_blank">http://api.openweathermap.org/data/2.5/weather?q=${city}&amp;appid=${apiKey}`</a></pre>
           <p name="41b7" id="41b7" class="graf graf--p graf-after--pre">We’ve done 3 things above:</p>
           <ol class="postList">
            <li name="5058" id="5058" class="graf graf--li graf-after--p">Create a variable named <code class="markup--code markup--li-code">apiKey</code> and assigned it the string value of our API Key (note: your api key will not be ‘***’s)</li>
            <li name="e9a8" id="e9a8" class="graf graf--li graf-after--li">Create a <code class="markup--code markup--li-code">city</code> and assigned it a string value of the city we’d like to test with</li>
            <li name="d6b3" id="d6b3" class="graf graf--li graf-after--li">Create a variable named <code class="markup--code markup--li-code">url</code> and assigned it the OpenWeatherMap url with our two required query parameters. Notice that query params start with a&nbsp;<code class="markup--code markup--li-code">?</code> question mark. They are then indicated with key/value pairs separated by an <code class="markup--code markup--li-code">=</code> equal sign. Different key/value pairs are separated with an <code class="markup--code markup--li-code">&amp;</code> ampersand.</li>
           </ol>
           <p name="5d10" id="5d10" class="graf graf--p graf-after--li"><strong class="markup--strong markup--p-strong">At this point, if we put everything together, here’s what our code should look like:</strong></p>

           <p name="a27d" id="a27d" class="graf graf--p graf-after--h3">The application isn’t really helpful as is. The jumbled mess of data that’s returned is just annoying. Lets clean everything up.</p>
           <p name="f4f4" id="f4f4" class="graf graf--p graf-after--p">The first thing we need to do is convert the jumbled text that’s returned (called JSON) into a JavaScript Object. (JavaScript Object Notation) is a way to store information in an organized and easy to access manner?—?and it looks like a JavaScript Object, but if you noticed, there are some extra quotation marks involved.</p>
           <p name="05a9" id="05a9" class="graf graf--p graf-after--p">We can make this conversion with one line of code:</p>

           <p name="82d5" id="82d5" class="graf graf--p graf-after--pre">Now that we have a JavaScript object, we can access data within the object with dot or bracket notation. Below, we construct a message string by accessing data within our weather object:</p>
           <pre name="cb10" id="cb10" class="graf graf--pre graf-after--p">let message = `It's ${weather.main.temp} degrees in<br>               ${weather.name}!`;<br>console.log(message);</pre>

           <h3 name="bb95" id="bb95" class="graf graf--h3 graf-after--figure">Adding in interactivity</h3>
           <p name="9255" id="9255" class="graf graf--p graf-after--h3">Our app is still boring. We can only access the weather for Portland, Oregon. Lets add some interactivity. For this, we’ll use <a href="https://www.npmjs.com/package/yargs" data-href="https://www.npmjs.com/package/yargs" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">yargs</a>. Yargs is a pirate themed interactive command line interface tool. Or more simply put, <strong class="markup--strong markup--p-strong">it allows us to define variables from the command line.</strong></p>
           <p name="cef9" id="cef9" class="graf graf--p graf-after--p">Install yargs with:</p>
           <pre name="ca2c" id="ca2c" class="graf graf--pre graf-after--p">npm install yargs --save</pre>
           <p name="d1a8" id="d1a8" class="graf graf--p graf-after--pre">Yargs works by exposing any variables we use in the console onto the <code class="markup--code markup--p-code">argv</code> object. We set up and access this object like so:</p>
           <pre name="2814" id="2814" class="graf graf--pre graf-after--p">const argv = require('yargs').argv;</pre>
           <p name="e90e" id="e90e" class="graf graf--p graf-after--pre">We’ll use the <strong class="markup--strong markup--p-strong">flag </strong>of <code class="markup--code markup--p-code">c</code> for city:</p>
           <p name="bbf1" id="bbf1" class="graf graf--p graf-after--p">Lets have our city variable equal either <code class="markup--code markup--p-code">argv.c</code> OR if no variable is input, we’ll have a default city value of <code class="markup--code markup--p-code">Portland</code>:</p>
           <pre name="a1dc" id="a1dc" class="graf graf--pre graf-after--p">let city = argv.c || 'portland';</pre>
           <p name="7b3c" id="7b3c" class="graf graf--p graf-after--pre">Now, to run the app, instead of just saying:</p>
           <pre name="997b" id="997b" class="graf graf--pre graf-after--p">node index.js</pre>
           <p name="e8c4" id="e8c4" class="graf graf--p graf-after--pre">We need to pass in a variable named <code class="markup--code markup--p-code">c</code> like this:</p>
           <pre name="f4b1" id="f4b1" class="graf graf--pre graf-after--p">node index.js -c Boston</pre>
           <pre name="a033" id="a033" class="graf graf--pre graf-after--pre">// It's 85 degrees in Boston</pre>
           <p name="57e9" id="57e9" class="graf graf--p graf-after--pre">We use a <em class="markup--em markup--p-em">flag </em>to indicate we’re passing in the variable. Since we set up our variable to be the letter <code class="markup--code markup--p-code">c</code> we pass in our variable with <code class="markup--code markup--p-code">-c</code>. After a space, we can use any city name we’d like!</p>
           <pre name="9ff7" id="9ff7" class="graf graf--pre graf-after--p">node index.js -c Anchorage</pre>
           <pre name="48c1" id="48c1" class="graf graf--pre graf-after--pre">// It's 47 degrees in Anchorage</pre>
