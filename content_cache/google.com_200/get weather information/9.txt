           <p>The weather. It impacts our life, our health, our mood but also businesses.&nbsp;Last month, IBM <a href="/blogs/bluemix/2015/10/23/ibm-insights-weather-available-in-bluemix/" target="_blank">announced the availability</a> of the Insights for Weather service in the Bluemix catalog providing developers with an easy way to tap into weather data:</p> 
           <p><img src="/blogs/bluemix/wp-content/uploads/2015/11/weather-catalog.png" al="Insights for Weather"></p> 
           <p>The <a href="http://insights-weather.mybluemix.net/" target="_blank">Insights Weather sample</a>&nbsp;is an application that shows you how to&nbsp;integrate the Insights for Weather service into your applications:</p> 
           <p><img src="/blogs/bluemix/wp-content/uploads/2015/12/weather-home_edited.png" alt="Insights Weather sample - home"></p> 
           <p>Once you select a city, it presents a familiar weather app showing the current weather and the forecast:</p> 
           <p><img src="/blogs/bluemix/wp-content/uploads/2015/12/weather-antibes_edited.png" alt="Antibes weather" style="border: 1px solid lightgray;"></p> 
           <p>This sample gives a good opportunity to exercise all service APIs.&nbsp;This post looks at the code of this sample, highlighting the&nbsp;service API calls and&nbsp;the format of their responses, together with providing tips on how to display the weather data in your application.</p> 
           <h2>How to Follow Along the Sample Review</h2> 
           <p>To follow along in&nbsp;the source code as we go through it in this post, you have several&nbsp;options:</p> 
           <ul> 
            <li>Deploy your own instance of the sample application by clicking the<em> Deploy to Bluemix&nbsp;</em>button.&nbsp;Then you can clone the Bluemix DevOps Git repository created by this process on your local machine <p><a href="https://bluemix.net/deploy?repository=https://github.com/IBM-Bluemix/insights-weather"><img src="http://bluemix.net/deploy/button.png" width="155" height="32" alt="Deploy to Bluemix"></a></p></li> 
            <li>Directly clone the GitHub repository to your local computer: <pre class="brush: plain; title: ; notranslate" title="">git clone&nbsp;https://github.com/IBM-Bluemix/insights-weather.git</pre> <p>Then later follow <a href="https://github.com/IBM-Bluemix/insights-weather/blob/master/README.md" target="_blank">these steps</a> to push your own instance of the application</p></li> 
            <li>Simply&nbsp;<a href="https://github.com/IBM-Bluemix/insights-weather" target="_blank">browse the GitHub repository</a> when necessary.</li> 
           </ul> 
           <h2>Sample Architecture Flow</h2> 
           <p>The sample uses a Node.js back-end and Angular/Bootstrap for the client. It is decomposed as follows:</p> 
           <p><img src="/blogs/bluemix/wp-content/uploads/2015/12/weather-architecture1_edited.png" alt="weather architecture"></p> 
           <ul> 
            <li><a href="https://github.com/IBM-Bluemix/insights-weather/blob/master/public/index.html" target="_blank">public/index.html</a> – the entry point for the angular application.</li> 
            <li><a href="https://github.com/IBM-Bluemix/insights-weather/blob/master/public/js/app.js" target="_blank">public/js/app.js</a> – configures the angular app, declares its dependencies and implements the routing between the home page and the search results</li> 
            <li><a href="https://github.com/IBM-Bluemix/insights-weather/blob/master/public/js/search.controller.js" target="_blank">public/js/search.controller.js</a> – holds the model for the view and supports the user interface interactions</li> 
            <li><a href="https://github.com/IBM-Bluemix/insights-weather/blob/master/public/js/search.service.js" target="_blank">public/js/search.service.js</a> – implements the call to the back-end API</li> 
            <li><a href="https://github.com/IBM-Bluemix/insights-weather/blob/master/app.js" target="_blank">app.js</a> – serves the html files and implements the back-end API used by the client. It delegates the call to the Insights for Weather service to <em>lib/weather.js</em></li> 
            <li><a href="https://github.com/IBM-Bluemix/insights-weather/blob/master/lib/weather.js" target="_blank">lib/weather.js</a> – In our sample, we don’t call the Insights for Weather service directly from the client user interface&nbsp;but hide it behind our own back-end API. This avoids putting the credentials of the service into the client app (this would be a bad idea) and it allows us to expose a slightly different API, such as giving the ability to look up weather conditions not only with geocode but also with plain text (and using a suggestion engine).</li> 
           </ul> 
           <h2>Obtaining Service Credentials</h2> 
           <p>When the Weather service is provisioned and bound to an application, the service credentials are exposed through <code>VCAP_SERVICES</code> such as:</p> 
           <pre class="brush: plain; title: ; notranslate" title="">  "weatherinsights": [
    {
      "name": "insights-weather-weatherinsights",&lt;br /&gt;      "label": "weatherinsights",&lt;br /&gt;      "plan": "Free",&lt;br /&gt;      "credentials": {&lt;br /&gt;        "host": "twcservice.mybluemix.net",&lt;br /&gt;        "password": "iwaT3D0BBQ",&lt;br /&gt;        "port": 443,&lt;br /&gt;        &lt;strong&gt;"url": "https://7af5b79d-5434-6f76-a0a0-d280c153537d:iwaT3D0BBQ@twcservice.mybluemix.net",&lt;/strong&gt;&lt;br /&gt;        "username": "7af5b79d-5434-6f76-a0a0-d280c153537d"&lt;br /&gt;      }&lt;br /&gt;    }&lt;br /&gt;  ]</pre> 
           <p>For most apps, the <code>url</code> property will be all that&nbsp;you need to call&nbsp;the service because it includes the <code>host</code>, <code>username</code> and <code>password</code> – the port is not included because&nbsp;443 is the default port for HTTPS so this would be redundant.</p> 
           <h2>Overview of the Insights for Weather Service API</h2> 
           <p>The Insights for Weather service comes with four easy-to-use&nbsp;REST API endpoints:</p> 
           <ul> 
            <li><code>/api/weather/v2/observations/current</code> – Real-time weather updates for current weather conditions, including temperature, wind direction and speed, humidity, pressure, visibility and more! Updates include a sensible weather phrase and a matching weather icon.</li> 
            <li><code>/api/weather/v2/forecast/hourly/24hour</code> – Hourly forecasts for the next 24 hours, including descriptive forecast text.</li> 
            <li><code>/api/weather/v2/forecast/daily/10day</code> – Daily forecasts for the next 10-day period, including forecast narrative text string of up to 256 characters with appropriate units of measure for the location and in the language requested.</li> 
            <li><code>/api/weather/v2/observations/timeseries/24hour</code> – Historical weather data for the previous 24 hours, based on weather data from site-based observation stations.</li> 
           </ul> 
           <p>All four endpoints expect the same input parameters to be passed in the query:</p> 
           <ul> 
            <li><code>geocode</code> – A comma-delimited latitude and longitude to retrieve data for (for example, 34.53,84.50).&nbsp;The latitude and longitude need to have at most two decimals.</li> 
            <li><code>language</code> – The language to return the response in (for example, en-US, es, es-MX, fr-FR)</li> 
            <li><code>units</code> – The units of measure to return the data in (for example, e=Imperial(English), m=Metric, h=Hybrid)</li> 
           </ul>
           <p></p> 
           <p>We really only need one function with the endpoint as a parameter. Here is an extract from <code>lib/weather.js</code> that implements this logic:</p> 
           <p>[gist&nbsp;https://gist.github.com/l2fprod/c974750e439e784857a2]</p> 
           <ul> 
            <li>line 2 – default options will set the language to English and the unit to imperial. The language impacts the narratives and weather condition descriptions provided by the service. In some cases, the <em>units&nbsp;</em>parameter impacts the format of the JSON output. For the sample, we use&nbsp;imperial and convert values on the client when needed.</li> 
            <li>line 7 – the&nbsp;<code>callByGeolocation</code> function is the key. It uses the&nbsp;<code>request</code> Node.js package to call the Weather service. The&nbsp;<code>url</code> has been retrieved from the environment variables in&nbsp;<code>app.js</code>.</li> 
            <li>line 11 –&nbsp;we ensure the latitude and longitude have only 2 decimals – the service API requires this.</li> 
           </ul> 
           <h2>Rendering the Weather Data</h2> 
           <p>The service API returns JSON data we can easily process on the client. In the sample:</p> 
           <ul> 
            <li><span><span><span><em><a href="https://github.com/IBM-Bluemix/insights-weather/blob/master/public/partials/search.html" target="_blank">public/partials/search.html</a></em> takes care of the rendering of the search results, current observation, hourly forecast and 10-day forecast;</span></span></span></li> 
            <li><span><span><span><em><a href="https://github.com/IBM-Bluemix/insights-weather/blob/master/public/js/search.service.js" target="_blank">public/js/search.service.js</a></em> calls the nodejs back-end;</span></span></span></li> 
            <li><span><span><span><em><a href="https://github.com/IBM-Bluemix/insights-weather/blob/master/public/js/search.controller.js" target="_blank">public/js/search.controller.js</a></em> performs the search by using the search service and holds the results for the angular&nbsp;rendering.<br></span></span></span></li> 
           </ul> 
           <p><span><span>In our back-end, we requested the units as&nbsp;<em>imperial</em>. This impacts the structure of the returned data as shown below where information such as the low/high temperature are found under the&nbsp;<em>imperial</em> node.</span></span></p> 
           <p><img src="/blogs/bluemix/wp-content/uploads/2015/12/weather-antibes-currentjson_edited.png"></p> 
           <p>The user interface then processes the values and displays them in Celsius or Fahrenheit based on the user preference. This is achieved with an angular filter:</p> 
           <pre class="brush: plain; title: ; notranslate" title="">{{controller.data.current.observation.imperial.temp | formatTemperature: controller.data.temperatureMode }}</pre> 
           <p>defined in <code>public/js/search.controller.js</code>:</p> 
           <pre class="brush: plain; title: ; notranslate" title="">.filter('formatTemperature', [&lt;br /&gt;  function () {&lt;br /&gt;    return function (input, scale) {&lt;br /&gt;    // input is assumed in Fahrenheit, as specified in lib/weather.js&lt;br /&gt;    // conversion to Celsius occurs in the display&lt;br /&gt;    if (scale === 'C') {&lt;br /&gt;      return Math.round((input - 32) * 5.0 / 9.0);&lt;br /&gt;    } else {&lt;br /&gt;      return input;&lt;br /&gt;    }&lt;br /&gt;  };&lt;br /&gt;}])</pre> 
           <p>As homework, I’ll let you explore the search.html file. You can also use the <em>View JSON source</em> button in each section of the user interface to drill down into the data behind the view. For the full specification of the data returned by the Weather service, refer to the <a href="https://twcservice.mybluemix.net/rest-api/">service API documentation</a>.</p> 
           <h2>Rendering the Weather Condition Icons</h2> 
           <p>When working with observation or forecast, you can choose to display an icon to illustrate the conditions. The service provides <a href="https://twcdocs.mybluemix.net/download/weatherinsightsicons.zip" target="_blank">a default icon set</a> you can use to map icon codes to an image. The images in the set are named after the icon code returned by the API.</p> 
           <p>As example, if we look&nbsp;for the current observation in Antibes, France, we get:</p> 
           <p><img src="/blogs/bluemix/wp-content/uploads/2015/12/weather-icon-code_edited.png"></p> 
           <p>Looking at the JSON returned by the server for the current observation, the <code>icon_code</code> is <strong>34</strong>. Using the icon named <em>icon<strong>34</strong>.png</em>&nbsp;from the default icon set, you can quickly show a visual hint for the observation. The same applies to forecast data. Everywhere you see this&nbsp;<code>icon_code</code> in the service response you can do this mapping.</p> 
           <p>In the sample application, look for references to&nbsp;<code>icon_code</code> in <a href="https://github.com/IBM-Bluemix/insights-weather/blob/master/public/partials/search.html" target="_blank">public/partials/search.html</a>. The weather icon displayed above uses this code:</p> 
           <pre class="brush: plain; title: ; notranslate" title="">&amp;lt;img class="media-object weather-current-icon" ng-src="&lt;strong&gt;images/weathericons/icon{{controller.data.current.observation.icon_code}}.png&lt;/strong&gt;" /&amp;gt; {{controller.data.current.observation.phrase_32char}}</pre> 
           <h2>Rendering the Wind Direction Icons</h2> 
           <p>In observation and forecast, the Daytime average wind direction is given by the&nbsp;<em>wdir</em> value. In the previous example,&nbsp;<em>wdir&nbsp;</em>is 230.&nbsp;It is expressed in magnetic notation from 0 to 359.&nbsp;You can use the wind icon from the default set to render the wind direction with a simple CSS rotation, as shown in&nbsp;<span> </span><a href="https://github.com/IBM-Bluemix/insights-weather/blob/master/public/partials/search.html" target="_blank">public/partials/search.html</a>:</p> 
           <pre class="brush: plain; title: ; notranslate" title="">&amp;lt;img class="weather-forecast-icon" src="&lt;strong&gt;images/weathericons/icon23.png&lt;/strong&gt;" title="{{forecast.wdir_cardinal}}" &lt;strong&gt;style="-webkit-transform: rotate({{forecast.wdir+90}}deg); -moz-transform: rotate({{forecast.wdir+90}}deg); -ms-transform: rotate({{forecast.wdir+90}}deg); -o-transform: rotate({{forecast.wdir+90}}deg); transform: rotate({{forecast.wdir+90}}deg);"&lt;/strong&gt; bs-tooltip/&amp;gt; {{forecast.wspd}} mph</pre> 
           <h2>Conclusion</h2> 
           <p>This sample should give you a good starting point to implement the Insights for Weather service into your application. To dig deeper in the Insights for Weather service, you can also check out the <a href="https://github.com/IBM-Bluemix/places-insights-lab" target="_blank">Places Insights hands-on lab</a> we delivered at IBM Insight in Las Vegas in October this year. It mixes the Insights for Weather and Insights for Twitter services and gives you some additional coding challenges to improve the resulting application.</p>
