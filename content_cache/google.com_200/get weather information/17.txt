       <p><em>Editor’s note: This is the first in a series of posts about developing Alexa skills. Read the rest of the posts in this series to learn how to <a href="https://www.bignerdranch.com/blog/?q=alexa">build and deploy an Alexa skill</a>.</em></p> 
       <p>If you want to build <a href="https://developer.amazon.com/public/solutions/alexa/alexa-skills-kit">Alexa Skills</a>, where should you start? You could begin with building one of the <a href="https://developer.amazon.com/public/solutions/alexa/alexa-skills-kit/docs/using-the-alexa-skills-kit-samples">sample skills</a> like the color picker or the trivia game. But when you’ve already tackled “Hello, World,” you’re ready to dive in.</p> 
       <p>Not quite yet. First, let’s set up a local development environment. Why should you use a local development environment over testing on a live server? There are many benefits. Chief among them are the fact that you gain access to the debugger and the stack trace, and you can quickly test changes without uploading files to a remote server, cutting down your iteration time.</p> 
       <p>In addition to time considerations, there are other concerns: what if the network is running slowly, you’re on a plane, or the Wi-Fi isn’t working? With a local dev environment, you can still get work done.</p> 
       <p>That’s where this post comes in: it will guide you through setting up a local development environment so that you can work more efficiently, enabling you to rapidly test your skills as you develop them. We will first set up a working environment with Node.js, and then we will build a model for our Alexa Skill. This skill—Airport Info—will track airport flight delays and weather conditions, and will give us a chance to try developing a more complex Alexa Skill locally.</p>
