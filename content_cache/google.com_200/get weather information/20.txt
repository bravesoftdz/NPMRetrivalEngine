      <div id="life-the-universe-and-everything"> 
       <h2><a href="#id10" rel="nofollow">Life, the Universe, and Everything</a></h2> 
       <p>A Plone package to display the current weather at selected locations inside a portlet or viewlet.</p> 
      </div> 
      <div id="mostly-harmless"> 
       <h2><a href="#id11" rel="nofollow">Mostly Harmless</a></h2> 
       <a href="http://travis-ci.org/collective/collective.weather" rel="nofollow"><img src="https://secure.travis-ci.org/collective/collective.weather.png?branch=master"></a> 
       <a href="https://coveralls.io/r/collective/collective.weather" rel="nofollow"><img src="https://coveralls.io/repos/collective/collective.weather/badge.png?branch=master"></a> 
       <p>Got an idea? Found a bug? Let us know by <a href="https://github.com/collective/collective.weather/issues" rel="nofollow">opening a support ticket</a>.</p> 
      </div> 
      <div id="don-t-panic"> 
       <h2><a href="#id12" rel="nofollow">Don’t Panic</a></h2> 
       <div id="installation"> 
        <h3><a href="#id13" rel="nofollow">Installation</a></h3> 
        <p>To enable this package on a buildout-based installation:</p> 
        <ol> 
         <li><p>Edit your buildout.cfg and add <tt>collective.weather</tt> to the list of eggs to install:</p> <pre>[buildout]
...
eggs =
    collective.weather
</pre> </li> 
         <li><p>If you are using Plone 4.1, you may need to extend a Dexterity known good set (KGS) to make sure that you get the right versions of the packages that make up Dexterity:</p> <pre>[buildout]
...
extends =
    https://good-py.appspot.com/release/dexterity/1.2.1?plone=4.1.6
</pre> </li> 
        </ol> 
        <p>After updating the configuration you need to run ”bin/buildout”, which will take care of updating your system.</p> 
        <p>Go to the ‘Site Setup’ page in a Plone site and click on the ‘Add-ons’ link.</p> 
        <p>Check the box next to ”collective.weather” and click the ‘Activate’ button.</p> 
        <div> 
         <p>Note</p> 
         <p>You may have to empty your browser cache and save your resource registries in order to see the effects of the product installation.</p> 
        </div> 
       </div> 
       <div id="usage"> 
        <h3><a href="#id14" rel="nofollow">Usage</a></h3> 
        <p>Go to Site Setup and select ‘Weather’.</p> 
        <p>Enter the list of locations that are going to be available on the site in the following format:</p> 
        <pre>location_id|name
</pre> 
        <p>Where <em>location_id</em> should be a unique value and not repeated among any of the locations; <em>name</em> is the name to be shown in the drop down (this doesn’t need to be unique). Some examples could be:</p> 
        <pre>455827|São Paulo (for Yahoo! Weather)
-23.548871,-46.638814|São Paulo (for Forecast.io and Weather Underground)
</pre> 
        <p>Select the sistem of units: Metric or Imperial. Metric system uses degrees Celsius. Imperial system uses degrees Fahrenheit.</p> 
       </div> 
       <div id="finding-locations"> 
        <h3><a href="#id15" rel="nofollow">Finding locations</a></h3> 
        <p>Different weather service providers need different location ids to get weather conditions.</p> 
        <p>You’ll have to change the <strong>Available locations</strong> setting depending on your selection. Please, refer to providers documentation for more information:</p> 
        <ul> 
         <li><a href="http://developer.yahoo.com/weather/" rel="nofollow">Yahoo Weather</a> needs a <a href="http://developer.yahoo.com/geo/geoplanet/guide/concepts.html#woeids" rel="nofollow">WOEID</a>. There’s a convenient <a href="http://woeid.rosselliot.co.nz/lookup" rel="nofollow">online tool to get WOEID</a>.</li> 
         <li><a href="https://developer.forecast.io/docs/v2" rel="nofollow">Forecast.io API</a> just needs a <tt>latitude, longitude</tt> coordinate.</li> 
         <li><a href="http://www.wunderground.com/weather/api/d/docs?d=data/index&amp;MR=1" rel="nofollow">Weather Underground</a> accepts many options (check the <tt>query</tt> option).</li> 
        </ul> 
       </div> 
       <div id="portlet"> 
        <h3><a href="#id16" rel="nofollow">Portlet</a></h3> 
        <p>The package includes a portlet that you can add in your site.</p> 
        <ul> 
         <li>Open the ‘Manage portlets’ screen and select ‘Weather portlet’</li> 
         <li>Set the title of the portlet</li> 
         <li>Select a city from the list</li> 
        </ul> 
        <img src="https://raw.github.com/collective/collective.weather/master/portlet.png"> 
       </div> 
       <div id="viewlet"> 
        <h3><a href="#id17" rel="nofollow">Viewlet</a></h3> 
        <div> 
         <p>Warning</p> 
         <p>The weather viewlet, as originaly planned, has proven to be pretty complex to implement. Current implementation is still buggy. Use it at your own risk.</p> 
        </div> 
        <p>The package also includes a viewlet that will display the weather in one of the locations defined in the settings. The viewlet will be displayed on top of the site inside the IPortalHeader viewlet manager.</p> 
        <p>To use the viewlet you need to activate it: go to Site Setup and select ‘Weather’ again. Select the ‘Show weather viewlet?’ box.</p> 
        <p>The user will be able to select from one of the locations (this information will be stored inside a cookie to show this location the next time the user visits the site).</p> 
        <p>The current weather condition of the selected location will be displayed at the viewlet.</p> 
        <img src="https://raw.github.com/collective/collective.weather/master/viewlet.png"> 
        <div id="internals"> 
         <h4><a href="#id18" rel="nofollow">Internals</a></h4> 
         <div> 
          <p>Note</p> 
          <p>This section could be outdated.</p> 
         </div> 
         <p>The weather viewlet uses Javascript in order to change cities, so this will only work for Javascript enabled browsers.</p> 
         <p>To render the city weather, there is a “current-weather” view that will render the latest info it has on weather conditions for a given city. A cookie is used to get the latest chosen city, and you can override it by passing a “city” argument to the view.</p> 
         <p>To update the city weather, there is a “update-weather”, that, when called without parameters, it will update all cities from the list. You can pass a “city” argument to the view, to only update the given city.</p> 
         <p>There’s an internal cache for each city (30 minutes), that if not enough time has passed, then it will assume the current weather is updated, and it will not do anything.</p> 
         <p>The Javascript that changes the city, will call this “update-weather” for the chosen city to update it first. Thanks to this internal cache, this view will return fast, if not enough time has passed.</p> 
         <p>In order to make it really fast for visitors of your site, you can set-up a clockserver job to call this “update-weather” view with no params, once every 30 minutes, so weather information for all your cities are ready for when the visitor changes it from the drop-down.</p> 
        </div> 
       </div> 
       <div id="extending-the-package-with-new-weather-providers"> 
        <h3><a href="#id19" rel="nofollow">Extending the package with new weather providers</a></h3> 
        <p>In case you want to contribute with new weather providers for this package or if you want to add a new one in a custom package for your site you’ll just need to create a new named utility that should implement <tt>collective.weather.interfaces.IWeatherInfo</tt>.</p> 
        <p><strong>Weather</strong> control panel will automatically learn about the new utility and it will be listed as an option in the provider’s drop-down.</p> 
        <p>This package already comes with some utilities you can check to get a quick idea of how to create yours:</p> 
        <ul> 
         <li><a href="https://github.com/collective/collective.weather/blob/master/src/collective/weather/utilities/yahoo.py" rel="nofollow">yahoo</a></li> 
         <li><a href="https://github.com/collective/collective.weather/blob/master/src/collective/weather/utilities/forecastio.py" rel="nofollow">forecast.io</a></li> 
         <li><a href="https://github.com/collective/collective.weather/blob/master/src/collective/weather/utilities/wunderground.pyweather" rel="nofollow">wunderground</a></li> 
        </ul> 
        <p><a href="https://github.com/collective/collective.weather/blob/master/src/collective/weather/interfaces.py#L21" rel="nofollow">The API for this utility</a> is very simple.</p> 
        <p>In case your utility needs <a href="https://github.com/collective/collective.weather/blob/master/src/collective/weather/utilities/forecastio.py#L114" rel="nofollow">an API key you can pass it on initialization</a>.</p> 
        <p>Here’s and example you can copy and paste to start your custom utility:</p> 
        <pre>"""Example of a named utility for IWeatherInfo.
"""
from collective.weather.interfaces import IWeatherInfo
from zope.interface import implements


class DummyProvider(object):
    """Dummy weather implementation of IWeatherInfo
    """
    implements(IWeatherInfo)

    def __init__(self, key=None):
        self.key = key

    def getWeatherInfo(self, location, units='metric', lang='en'):
        """Dummy implementation of getWeatherInfo as an example
        """
        return {
            'summary': u'What a lovely day!',
            'temperature': 20,
            'icon': u'lovely-day-icon.png',
        }
</pre>
