       <h2 id="whatshttp">What's HTTP?</h2> 
       <p>HTTP stands for <em>Hypertext Transfer Protocol</em>. HTTP functions as a request–response protocol in the client–server computing model.</p> 
       <h2 id="httpstatuscodes">HTTP Status Codes</h2> 
       <p>Before diving into the communication with other APIs, let's review the HTTP status codes we may encounter during the process. They describe the outcome of our requests and are essential for error handling.</p> 
       <ul> 
        <li> <p>1xx - Informational</p> </li> 
        <li> <p>2xx - Success: These status codes indicate that our request was received and processed correctly. The most common success codes are <code>200 OK</code>, <code>201 Created</code> and <code>204 No Content</code>.</p> </li> 
        <li> <p>3xx - Redirection: This group shows that the client had to do an additional action to complete the request. The most common redirection codes are <code>301 Moved Permanently</code>, <code>304 Not Modified</code>.</p> </li> 
        <li> <p>4xx - Client Error: This class of status codes is used when the request sent by the client was faulty in some way. The server response usually contains the explanation of the error. The most common client error codes are <code>400 Bad Request</code>, <code>401 Unauthorized</code>, <code>403 Forbidden</code>, <code>404 Not Found</code>, <code>409 Conflict</code>.<br> ?</p> </li> 
        <li> <p>5xx - Server Error: These codes are sent when the server failed to fulfill a valid request due to some error. The cause may be a bug in the code or some temporary or permanent incapability. The most common server error codes are <code>500 Internal Server Error</code>, <code>503 Service Unavailable</code>.<br> ?<br> If you'd like to learn more about HTTP status codes, you can find a detailed explanation about them <a href="http://www.restapitutorial.com/httpstatuscodes.html">here</a>.<br> ?</p> </li> 
       </ul> 
       <h2 id="sendingrequeststoexternalapis">Sending Requests to External APIs</h2> 
       <p>Connecting to external APIs is easy in Node. You can just require the core <a href="https://nodejs.org/api/http.html">HTTP module</a> and start sending requests.</p> 
       <p>Of course, there are much better ways to call an external endpoint. On NPM you can find multiple modules that can make this process easier for you. Fo?r example, the two most popular ones are the <a href="https://www.npmjs.com/package/request">request</a> and <a href="https://www.npmjs.com/package/superagent">superagent</a> modules.</p> 
       <p>Both of these modules have an error-first callback interface that can lead to some issues (I bet you've heard about Callback-Hell), but luckily we have access to the promise-wrapped versions.</p> 
       <div class="card"> 
        <img src="https://trace.risingstack.com/images/trace-by-risingstack-logo.svg">

       <h2 id="usingthenodejsrequestmodule">Using the Node.js Request Module</h2> 
       <p>Using the <a href="https://www.npmjs.com/package/request-promise">request-promise module</a> is simple. After installing it from NPM, you just have to require it:</p> 
       <pre><code class="language-javascript">const request = require('request-promise')
</code></pre> 
       <p>?<br> Sending a GET request is as simple as:</p> 
       <pre><code class="language-javascript">const options = {
  method: 'GET',
  uri: 'https://risingstack.com'
}
?
request(options)
  .then(function (response) {
    // Request was successful, use the response object at will
  })
  .catch(function (err) {
    // Something bad happened, handle the error
  })
</code></pre> 
       <p>If you are calling a JSON API, you may want the request-promise to parse the response automatically. In this case, just add this to the request options:</p> 
       <pre><code class="language-javascript">json: true
</code></pre> 
       <p>?<br> POST requests work in a similar way:</p> 
       <pre><code class="language-javascript">const options = {
  method: 'POST',
  uri: 'https://risingstack.com/login',
  body: {
    foo: 'bar'
  },
  json: true 
    // JSON stringifies the body automatically
}
?
request(options)
  .then(function (response) {
    // Handle the response
  })
  .catch(function (err) {
    // Deal with the error
  })
</code></pre> 
       <p>?<br> To add query string parameters you just have to add the <code>qs</code> property to the options object:</p> 
       <pre><code class="language-javascript">const options = {
  method: 'GET',
  uri: 'https://risingstack.com',
  qs: {
    limit: 10,
    skip: 20,
    sort: 'asc'
  }
}
</code></pre> 
       <p>This will make your request URL: <code>https://risingstack.com?limit=10&amp;skip=20&amp;sort=asc</code>.<br> ?<br> You can also define any header the same way we added the query parameters:</p> 
       <pre><code class="language-javascript">const options = {
  method: 'GET',
  uri: 'https://risingstack.com',
  headers: {
    'User-Agent': 'Request-Promise',
    'Authorization': 'Basic QWxhZGRpbjpPcGVuU2VzYW1l'
  }
}
</code></pre> 
       <h2 id="errorhandling">Error handling</h2> 
       <p>Error handling is an essential part of making requests to external APIs, as we can never be sure what will happen to them. Apart from our client errors the server may respond with an error or just send data in a wrong or inconsistent format. Keep these in mind when you try handling the response. Also, using <code>catch</code> for every request is a good way to avoid the external service crashing our server.</p> 
       <h2 id="puttingittogether">Putting it together</h2> 
       <p>As you have already learned how to spin up a Node.js HTTP server, how to render HTML pages, and how to get data from external APIs, it is time to put them together!</p> 
       <p>In this example, we are going to create a small Express application that can render the current weather conditions based on? city names.</p> 
       <p><em>(To get your AccuWeather? API key, please visit their <a href="http://apidev.accuweather.com/developers/samples">developer site</a>)</em></p> 
       <pre><code class="language-javascript">const express = require('express')
const rp = require('request-promise')
const exphbs = require('express-handlebars')

const app = express()

app.engine('.hbs', exphbs({
  defaultLayout: 'main',
  extname: '.hbs',
  layoutsDir: path.join(__dirname, 'views/layouts')
}))
app.set('view engine', '.hbs')
app.set('views', path.join(__dirname, 'views'))

app.get('/:city', (req, res) =&gt; {
  rp({
    uri: 'http://apidev.accuweather.com/locations/v1/search',
    qs: {
      q: req.params.city,
      apiKey: 'api-key'
         // Use your accuweather API key here
    },
    json: true
  })
    .then((data) =&gt; {
      res.render('index', data)
    })
    .catch((err) =&gt; {
      console.log(err)
      res.render('error')
    })
})

app.listen(3000)
</code></pre> 
       <p>The example above does the following:</p> 
       <ul> 
        <li>creates an Express server</li> 
        <li>sets up the handlebars structure - for the <code>.hbs</code> file please refer to the <a href="https://blog.risingstack.com/your-first-node-js-http-server/">Node.js HTTP tutorial</a></li> 
        <li>sends a request to the external API 
         <ul> 
          <li>if everything is ok, it renders the page</li> 
          <li>otherwise, it shows the error page and logs the error</li> 
         </ul> </li> 
       </ul>
