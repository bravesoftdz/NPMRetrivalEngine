<h2>NEW (March 2005) - GFDL CM2.0 and CM2.1 Climate Experiments:</h2>

<p>
Model output and documentation from a set of multi-century experiments
performed using GFDL's CM2.0 and CM2.1 models are being made available
to the public.  Data sets accessible from the GFDL Data Portal include
those from experiments associated with the Intergovernmental Panel on
Climate Change's 4th Assessment Report (<a href="http://www.ipcc.ch/"
target="extl">IPCC</a>) and <a href="http://www.climatescience.gov/"
target="extl">the US Climate Change Science Program</a>.
</p>
<p>
Some key points to consider:
  <ul>
    <li>There are 2 new GFDL models (CM2.0 and CM2.1) that differ
        in significant ways (for example, entirely different
        atmospheric dynamical cores are used in the two models).
    </li>
    <li>There are 2 places one can visit to download GFDL CM2.0
        and CM2.1 model output. Model output is available to the
        public from this GFDL Data Portal.  Additionally,
        registered IPCC Working Group 1 (WG1) users can download
        files from servers maintained by the Program for Climate
        Model Diagnosis and Intercomparison (<a
        href="http://www-pcmdi.llnl.gov/ipcc/about_ipcc.php"
        target="extl">PCMDI</a>) at the Lawrence Livermore National
        Laboratory.
    </li>
    <li>There are 2 broad categories of CM2.x model output:
      <ol>
        <li>atmosphere and land model output, and</li>
        <li>ocean and sea ice model output.</li>
      </ol>
        Atmosphere and land files were the first to be processed
        and made available for download, and can be found on both
        the GFDL and the PCMDI servers.  Preparation of ocean and
        sea ice output files has lagged a bit, and most will be
        available only from the GFDL Data Portal and not from PCMDI.
    </li>
    <li>There are two grid issues one should be aware of when using
        GFDL CM2.x ocean and sea ice files. One is that north of
        65N the ocean and sea ice models' "tripolar" horizontal grid
        is not a regular latitude longitude grid.  Rather, to avoid
        a singularity at the North Pole, the grid has "poles" located
        in northern Canada and northern Russia, in addition to one at
        the South Pole.  Secondly, when using 3-D ocean data files,
        one needs to consider the ocean model's use of partial bottom
        cells.
    </li>
  </ul>
</p>

<p>
More information is available on the <b><a
href="/nomads/forms/deccen/">deccen coupled climate models web
page</a></b>.
</p>

    <h2>Our Data Portal Services 
     <!--
<IMG style="FLOAT: right;" title="image of a temperature field from GFDL's eddy-resolving ocean model" alt="image of the temperature field at a depth of 55 meters in the North Atlantic ocean from GFDL's eddy-resolving ocean model" hspace=8 
--> </h2> 
    <p> Public data sets from GFDL are made available through the GFDL Data Portal. The Data Portal is designed to allow one to download files, display data files attributes, and graphically display the data. One can download complete files via "http" and "ftp" access. One can also display data file attributes that provide information about file contents without having to download the file itself. Continued development of the Live Access Server is ongoing to enhance the ability of a remote user to graphically display the data. </p> 
    <h2>Our Public Data Files</h2> 
    <p>Registration for the GFDL Data Files is free. Users are requested to complete the Registration Form for Public Data Files (found on the right hand menu) when they first begin using the data portal. Information from this form will be used to provide registered users with news on when additional public data becomes available and when corrections are made to existing public data. The information gathered will not be used for any purposes other than to send notices about GFDL Data Portal Services. </p> 
    <h2>Data Storage</h2> 
    <p>The data files on the data portal are stored in netCDF (network Common Data Form), and can be identified by the suffix ".nc". The files are intended to be compliant with the COARDS conventions for the standardization of netCDF files. More information about netCDF, a machine independent format for representing array-oriented scientific data, can be found at the Unidata Program Center netCDF website. </p> 
    <h2>Software</h2> 
    <p> This site implements the following: </p>
    <ul> 
     <li>the Distributed Oceanographic Data System (<a href="http://www.unidata.ucar.edu/packages/dods/">DODS</a>) package from <a href="http://www.unidata.ucar.edu/">Unidata</a>/ <a href="http://www.ucar.edu/">UCAR</a> </li>
     <li>the Grid Analysis and Display System - DODS Server (<a href="http://grads.iges.org/grads/gds/">GDS</a>) package from <a href="http://grads.iges.org/cola.html">COLA</a>/ <a href="http://grads.iges.org/">IGES</a> </li>
     <li>Live Access Server (<a href="http://ferret.pmel.noaa.gov/LAS">LAS</a>) package from <a href="http://www.pmel.noaa.gov/">PMEL</a>/ <a href="http://www.noaa.gov/">NOAA</a>. (Please note: Users of LAS require a web browser that supports Java and JavaScript.) </li> 
    </ul> 
    <h2>Data Download Statistics</h2> To get information about amount of data downloaded from GFDL Data Portal use web application at 
    <a href="http://nomads.gfdl.noaa.gov:8080/LogStats/">http://nomads.gfdl.noaa.gov:8080/LogStats</a>

    <a href="http://nomads.gfdl.noaa.gov/contacts.html"> MIP Contact Information </a>. We highly recommend that you use

    <h2>Questions?</h2> 
    <p> Questions concerning the availability of GFDL Data Portal data files and their content may be directed to: <a href="mailto:gfdl.climate.model.info@noaa.gov">GFDL.Climate.Model.Info@noaa.gov</a> <br> Other questions related to GFDL Data Portal functionality, development, system administration, etc., may be directed to: <a href="mailto:oar.gfdl.webmaster-data1@noaa.gov">oar.gfdl.webmaster-data1@noaa.gov</a> </p>

    <h4>Spotlight on<br>NOMADS</h4> 
    <p>NOMADS is being developed as a Unified Climate and Weather Archive to provide Web access to model information so that users can make decisions about their specific needs. This spans time scales from days (weather), to months (El Nino), to decades (global warming). For more, see NCDC's <a href="http://www.ncdc.noaa.gov/data-access/model-data">nomads</a> page. </p> 
    <h4>Spotlight on<br>GO-ESSP</h4> 
    <p>The Global Organization for Earth System Science Portal (GO-ESSP) is a collaboration designed to build the infrastructure needed to create web portals to provide observed and simulated data to the climate and weather communities. The infrastructure created within GO-ESSP will provide a flexible framework of front-end and back-end software components. For more, see the <a href="http://go-essp.gfdl.noaa.gov/">GO-ESSP</a> home page. </p>
