     <h1> Get started with Ionic 2 apps in Visual Studio </h1> 
     <p> Updated: 1/16/2017 </p> 
     <div class="wa-navigationScroll fixed-element" data-top-spacing="100" data-resting-position="absolute"> 
      <span>In this article:</span> 
      <ul class="scrollspy"> 
       <li><a href="#requirements">Requirements</a></li> 
       <li><a href="#getTemplates">Installing the Ionic 2 templates </a></li> 
       <li><a href="#creating-an-ionic-project-in-visual-studio">Creating an Ionic Project in Visual Studio</a></li>

       <li><a href="#creating-the-weather-app">Creating the Weather App</a></li>

     <p><a href="http://www.ionicframework.com">Ionic</a> is a popular front-end JavaScript framework for developing cross-platform mobile apps using <a href="http://cordova.io">Apache Cordova</a>. The Ionic Framework gives Cordova applications a native look and feel, and automatically adjusts that look across platforms. You can use Visual Studio 2015 to easily create and debug cross-platform Ionic apps. The <a href="../get-started-first-mobile-app/">Get started with Visual Studio Tools for Apache Cordova</a> guide showed how to create a simple Weather app using jQuery and jQuery Mobile. In this article, you'll learn how to configure a Visual Studio 2015 development environment for Ionic 2, and create the Ionic 2 version of the Weather App as shown below: </p> 
     <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/tutorial-ionic2/20170216102556/figure-01.png" alt="Finished Weather application"> </p> 
     <h2><a id="requirements" name="requirements" href="#requirements"></a>Requirements</h2> 
     <p>To manage, code, run and debug Ionic 2 applications using Visual Studio, you must install the following:</p> 
     <ul> 
      <li><a href="http://go.microsoft.com/fwlink/?LinkID=533794">Visual Studio 2015</a></li> 
      <li><a href="http://taco.visualstudio.com/en-us/docs/install-vs-tools-apache-cordova/">Visual Studio Tools For Apache Cordova</a></li> 
      <li>Visual Studio Ionic 2 Templates (described below)</li> 
      <li>Ionic Template dependencies (described below)</li> 
     </ul> 
     <h2>Installing the Ionic 2 templates <a name="getTemplates" id="getTemplates" href="#getTemplates"></a></h2> 
     <ol> 
      <li><p>If you haven't already, <a href="http://go.microsoft.com/fwlink/?LinkID=533794">install Visual Studio 2015</a>.</p> <p>Under the covers, Ionic apps are Apache Cordova apps, you'll need a functional Cordova development environment before you can use with Ionic. </p> <p>Verify that you can create and run the default <strong>Cordova Blank App</strong>. In Visual Studio, open the <strong>File</strong> menu, select <strong>New</strong>, then <strong>Project</strong>. In the new project dialog, expand the <strong>JavaScript</strong> templates section, select <strong>Apache Cordova Apps</strong> then pick the <strong>Blank App (Apache Cordova)</strong> template. Give the new project a name and location then click the <strong>OK</strong> button. Press <strong>F5</strong> to build and run the new project in the <strong>Ripple Emulator</strong>. The Chrome browser should open and render the app's content. If you have any issues running the application, see these <a href="../tips-and-workarounds-general-readme/">troubleshooting steps</a>.</p></li> 
      <li><p>Install the Ionic template in Visual Studio by selecting the <strong>Tools</strong> menu, then <strong>Extensions and Updates...</strong>. In the Extensions and Updates dialog box, select <strong>Online</strong>. In the search box located in the upper-right corner of the dialog, type <strong>Ionic 2</strong>. Select the <strong>Ionic 2 RC Templates</strong> option in the list. Click the <strong>Download</strong> button to start the installation.</p> <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/tutorial-ionic2/20170216102556/figure-02.png" alt="Getting the templates"> </p> <p>The template files will download, then Visual Studio will automatically launch the installation process. When prompted, click the <strong>Install</strong> button to begin the installation.</p> <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/tutorial-ionic2/20170216102556/figure-03.png" alt="Installing the templates"> </p></li> 
     </ol> 
     <blockquote> 
      <p><strong>Note:</strong> The Ionic templates may not appear in Visual Studio until it's restarted. Close, then re-open Visual Studio before continuing.</p> 
     </blockquote> 
     <h2><a id="creating-an-ionic-project-in-visual-studio" name="creating-an-ionic-project-in-visual-studio" href="#creating-an-ionic-project-in-visual-studio"></a>Creating an Ionic Project in Visual Studio</h2> 
     <ol> 
      <li><p>Create a new Ionic project. Open the <strong>File</strong> menu, select <strong>New</strong>, then <strong>Project</strong>. In the new project dialog, expand the <strong>TypeScript</strong> templates section, and then select <strong>Apache Cordova Apps</strong>. Ionic offers three default app styles: <strong>Blank</strong>, <strong>Sidemenu</strong>, and <strong>Tabs</strong>; this is a simple project, so select the Blank template. </p> <p>Give the new project a <strong>Name</strong> and <strong>Location</strong>, and then click the <strong>OK</strong> button. This app will later become our Weather App project, so you should probably call it <strong>Weather App</strong> or <strong>Weather App Ionic 2</strong> to save time.</p> <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/tutorial-ionic2/20170216102556/figure-04.png" alt="New Project dialog"> </p></li> 
      <li><p>Check the new Ionic project's readme file for any additional tools that must be installed to use the template. </p> <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/tutorial-ionic2/20170216102556/figure-05.png" alt="Dependencies node"> </p> <p>At the time of this writing, you'll need to install the following Visual Studio extensions:</p> 
       <ul> 
        <li>NPM Task Runner</li> 
        <li>TypeScript 2.0.6 editor</li> 
        <li>Microsoft ASP.NET and Web Tools Preview 2</li> 
       </ul> <p>The requirements may change as updates are made to the templates. To install the required extensions, open the <strong>Tools</strong> menu, and select <strong>Extensions and Updates</strong>. Use search to locate and install the required extensions.</p></li> 
      <li><p>Ionic is a heavy framework, it requires a lot of tools and libraries to operate. Once Visual Studio creates the new project, it kicks off a process to download and install the required components using the Node Package Manager (npm). Wait a few minutes as Ionic's npm packages are installed.</p> 
       <blockquote> 
        <p><strong>Note</strong> This process will take several minutes depending on your system and internet connection speeds.</p> 
       </blockquote> <p>To check progress, open <strong>Solution Explorer</strong> and look for the <strong>Dependencies</strong> node. You should see <strong>Restoring...</strong>, if you don't, right-click on the Dependencies node in Solution Manager then select <strong>Restore Packages</strong>.</p> <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/tutorial-ionic2/20170216102556/figure-06.png" alt="Dependencies node"> </p> 
       <blockquote> 
        <p><strong>Note:</strong> After Visual Studio finishes installing dependencies, the Dependencies node may show <strong>not installed</strong>. This is a known issue; the Ionic project will be OK at this point.</p> 
        <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/tutorial-ionic2/20170216102556/figure-07.png" alt="Dependencies node"> </p> 
       </blockquote> <p>You can monitor the package installation process through Visual Studio's Output window. Open the <strong>View</strong> menu, then select <strong>Output</strong> or use the keyboard shortcut <strong>Ctrl-W</strong> + <strong>O</strong>:</p> <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/tutorial-ionic2/20170216102556/figure-08.png" alt="Task Runner"> </p></li> 
      <li><p>At this point, you have a complete Ionic application project ready to go. To test and debug the application in Visual Studio, select a <strong>target platform</strong> in the Standard Toolbar, select a <strong>target device</strong>, then pres <strong>F5</strong> to run the application on the selected target. Developing iOS applications requires some extra configuration; refer to the <a href="../vs-taco-2017-ios-guide/">Visual Studio Tools for Apache Cordova: iOS Guide</a> for additional information.</p> <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/tutorial-ionic2/20170216102556/figure-09.png" alt="Ionic application running on Android"> </p></li> 
     </ol> 
     <h2><a id="enabling-live-reload-in-ripple" name="enabling-live-reload-in-ripple" href="#enabling-live-reload-in-ripple"></a>Enabling Live Reload in Ripple</h2> 
     <p>Ionic uses Node Package Manager (<a href="https://www.npmjs.com">npm</a>) scripts to compile your application's TypeScript and SASS files into JavaScript and CSS, then copies the output into the project's <code>www</code> folder. By default, Visual Studio uses the <code>ionic:build</code> command in the project's <code>package.json</code> to build the project. Visual Studio executes this script every time you launch the Ionic application. </p> 
     <p>While developing your app using the Ripple emulator, you may want to setup your project to live reload changes into the browser as you work on your files. To do this:</p> 
     <ol> 
      <li><p>Open the <strong>Task Runner Explorer</strong>: In Visual Studio, open the <strong>View</strong> menu, select <strong>Other Windows</strong> then Task Runner Explorer. You can also use the keyboard shortcut: <strong>Ctrl</strong> + <strong>Alt</strong> + <strong>Backspace</strong>.</p></li> 
      <li><p>Right click the ionic:build task and choose the <strong>Bindings</strong> -&gt; <strong>Project Open</strong> menu to setup the watch task to run when you load your project.</p></li> 
      <li><p>Run your Ionic app in the Ripple browser.</p></li> 
     </ol> 
     <p>After following the above steps, you'll now be able to make edits to the files in your <code>src</code> folder and your changes will automatically show up in the Ripple emulator. The status of the watch task will appear in a separate window in the Task Runner Explorer.</p> 
     <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/tutorial-ionic2/20170216102556/figure-10.png" alt="Task Runner - Watch"> </p> 
     <h2><a id="creating-the-weather-app" name="creating-the-weather-app" href="#creating-the-weather-app"></a>Creating the Weather App</h2> 
     <p>Now, lets do something with this app you created. You'll take the blank application project and turn it into the Weather App shown earlier. In the following sections, you'll start by adding a <strong>Current Conditions</strong> page, then wire in a <strong>Search Box</strong>, and finally add an area to display <strong>Forecast</strong> data. Lets get started.</p> 
     <p>The app uses the free <a href="http://openweathermap.org/">OpenWeatherMap</a> service to retrieve weather conditions for a location. Before you can use the service, you must setup an account and request an API key for their Current Conditions API. Point your browser to <a href="https://home.openweathermap.org/">OpenWeatherMap</a> and setup an account. Once you have a valid login, go to the service's <a href="http://openweathermap.org/api">API page</a> and subscribe to the <strong>Current Weather data</strong> service, and then generate an API key.</p> 
     <blockquote> 
      <p><strong>Note:</strong> Make note of the API key as you'll need it later in the app's TypeScript code. </p> 
     </blockquote> 
     <h3><a id="configuring-the-application-project" name="configuring-the-application-project" href="#configuring-the-application-project"></a>Configuring the Application Project</h3> 
     <ol> 
      <li><p>Start by setting some configuration options for the application. In Visual Studio Solution Explorer, double-click the project's <code>config.xml</code> file. Visual Studio will open a custom editor similar to the one shown in the following figure:</p> <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/tutorial-ionic2/20170216102556/figure-11.png" alt="Modifying the application's config.xml file"> </p> <p>At a minimum, set the project's <strong>Display Name</strong> and <strong>Package Name</strong> values; you may want to update the <strong>Author</strong> and <strong>Description</strong> fields as well. </p></li> 
      <li><p>By default, the application displays weather data based on the device's current location, so we'll need to use the <a href="https://cordova.apache.org/docs/en/latest/reference/cordova-plugin-geolocation/">Cordova Geolocation</a> plugin to add that capability to our application. private makeDataURL(loc: any, command: string): string {
  //Build a weather service URL using the command string and
  //location data that we have.
  let uri = this.weatherEndpoint + command;

  //Do we have a location?
  if (loc.long) {
    //then use the 'grographical coordinates' version of the API
    uri += '?lat=' + loc.lat + '&amp;lon=' + loc.long;
  } else {
    //Otherwise, use the zip code
    uri += '?zip=' + loc.zip;
  }

  //Configure output for imperial (English) measurements
  uri += '&amp;units=imperial';
  //Use the following instead for metric
  //uri += '&amp;units=metric';

  //Append the API Key to the end of the URI
  uri += '&amp;APPID=' + this.weatherKey;
  //Return the value
  return uri;
}

//'Borrowed' from //https://angular.io/docs/ts/latest/guide/server-communication.html
private extractData(res: Response) {
  //Convert the response to JSON format  
  let body = res.json();
  //Return the data (or nothing)
  return body || {};
}

//'Borrowed' from //https://angular.io/docs/ts/latest/guide/server-communication.html
private handleError(res: Response | any) {
  console.error('Entering handleError');
  console.dir(res);
  return Promise.reject(res.message || res);
}
</pre> <p>On start up, the application passes the current location (longitude and latitude) to the <code>getCurrent</code> function. private formatWeatherData(data): any {
  //create a blank array to hold our results
  let tmpArray = [];
  //Add the weather data values to the array
  if (data.name) {
    //Location name will only be available for current conditions
    tmpArray.push({ 'name': 'Location', 'value': data.name });
  }
  tmpArray.push({ 'name': 'Temperature', 'value': data.main.temp + this.degreeStr });
  tmpArray.push({ 'name': 'Low', 'value': data.main.temp_min + this.degreeStr });
  tmpArray.push({ 'name': 'High', 'value': data.main.temp_max + this.degreeStr });
  tmpArray.push({ 'name': 'Humidity', 'value': data.main.humidity + '%' });
  tmpArray.push({ 'name': 'Pressure', 'value': data.main.pressure + ' hPa' });
  tmpArray.push({ 'name': 'Wind', 'value': data.wind.speed + ' mph' });
  //Do we have visibility data?
  if (data.visibility) {
    tmpArray.push({ 'name': 'Visibility', 'value': data.visibility + ' meters' });
  }
  //do we have sunrise/sunset data?
  if (data.sys.sunrise) {
    var sunriseDate = new Date(data.sys.sunrise * 1000);
    tmpArray.push({ 'name': 'Sunrise', 'value': sunriseDate.toLocaleTimeString() });
  }
  if (data.sys.sunset) {
    var sunsetDate = new Date(data.sys.sunset * 1000);
    tmpArray.push({ 'name': 'Sunset', 'value': sunsetDate.toLocaleTimeString() });
  }
  //Do we have a coordinates object? only if we passed it in on startup
  if (data.coord) {
    //Then grab long and lat
    tmpArray.push({ 'name': 'Latitude', 'value': data.coord.lat });
    tmpArray.push({ 'name': 'Longitude', 'value': data.coord.lon });
  }
  //Return the new array to the calling function
  return tmpArray;
}

this.displayMode = this.currentMode
</pre> <p>This sets <code>this.displayMode</code> to <code>current</code>. Add the code to the <code>resume</code> event listener, the resulting code should look like this:</p> <pre class="prettyprint copy-to-clipboard lang-typescript">document.addEventListener('resume', () =&gt; {

  this.displayMode = this.currentMode
  //then update it with local weather conditions

</pre></li>

    this.showCurrent();
  } else {

      <li><p>The <code>setZipCode</code> function defaults to loading current conditions when a Zip Code is entered, so it must switch to the Current tab before getting results. this.displayMode = this.currentMode;
</pre> <p>The complete function should look like this:</p> <pre class="prettyprint copy-to-clipboard lang-typescript">setZipCode() {
  //whenever the user enters a zip code, replace the current location
  //with the entered value, then show current weather for the selected
  //location.
  //Hide the keyboard if it's open, just in case
  Keyboard.close();
  //Populate the currentLoc variable with the city name
  this.currentLoc = { 'zip': this.searchInput };
  //Clear the Zip Code input field
  this.searchInput = '';
  //Switch to the 'current' tab
  this.displayMode = this.currentMode;
  //get the weather for the specified city
  this.showCurrent();
}
</pre></li> 
      <li><p>Finally, add the <code>showForecast</code> function to the <code>HomePage</code> class:</p> <pre class="prettyprint copy-to-clipboard lang-typescript">showForecast() {
  //clear out the previous array contents
  this.f_items = [];
  //Create the loading indicator
  let loader = this.loadingCtrl.create({
    content: "Retrieving forecast..."
  });
  //Show the loading indicator
  loader.present();
  this.weather.getForecast(this.currentLoc).then(
    data =&gt; {
      //Hide the loading indicator
      loader.dismiss();
      //Now, populate the array with data from the weather service
      //Do we have some data?
      if (data) {
        //Then lets build the results array we need
        //Process each forecast period in the array
        for (let period of data.list) {
          //Create a 'record' consisting of a time period's results
          let weatherValues: any = this.formatWeatherData(period);
          //Append this, along with the time period information, into the forecast
          //items array.          
          //Get the forecast date as a date object                     
          let d = new Date(period.dt_txt);
          //Determe the day of the week
          let day = this.days[d.getDay()];
          //And the time
          let tm = d.toLocaleTimeString();
          //Create a new object in the results array for this period          
          this.f_items.push({ 'period': day + ' at ' + tm, 'values': weatherValues });
        }
      } else {
        //This really should never happen
        console.error('Error displaying weather data: Data object is empty');
      }
    },
    error =&gt; {
      //Hide the loading indicator
      loader.dismiss();
      console.error("Error retrieving weather data");
      console.dir(error);
      this.showAlert(error);
    }
  );
}   
</pre> <p>This function gets the forecast data, then formats it for display on two pages. First, it builds an array of time periods that represent the different forecast periods. Then, for each period, it appends an object containing the forecast data for that period. The page's forecast segment displays the list of forecast periods, then, when you tap on an entry, the application will eventually (you'll see how in the next section) open a separate page to display the details.</p></li> 
     </ol>

     <h3><a id="no-templates" name="no-templates" href="#no-templates"></a>I installed the Ionic templates, but they don't appear as an option in Visual Studio</h3> 
     <p>Did you close, then restart Visual Studio? After the installation, the templates aren't available until Visual Studio restarts. </p> 
     <h3><a id="node" name="node" href="#node"></a> Did you receive an error message indicating that some node modules won't restore?</h3> 
     <p>The installed version of Node.js may not support Promises. To fix this, you must install the latest Visual Studio 2015 update (Update 3 or later) and Microsoft ASP.NET and Web Tools. Check for any notifications or warning icons in the upper-right corner of the Visual Studio IDE and install these components, if instructed.</p> 
     <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/tutorial-ionic2/20170216102556/figure-17.png" alt="Update your credentials"> </p> 
     <p>If you are unable to install the Web Tools:</p> 
     <ol> 
      <li><p><a href="https://visualstudiogallery.msdn.microsoft.com/c94a02e9-f2e9-4bad-a952-a63a967e3935">Download the latest version of the Web Tools here</a>.</p></li> 
      <li><p>Run the following command from the folder containing the download:</p> <pre class="prettyprint copy-to-clipboard ">DotNetCore.1.0.0-VS2015Tools.Preview2.exe SKIP_VSU_CHECK=1
</pre></li> 
      <li><p>Create a new Ionic 2 project from the templates that you downloaded previously.</p></li> 
     </ol> 
     <h3><a id="node_modules" name="node_modules" href="#node_modules"></a> Did you receive an error message indicating that some node modules won't restore?</h3> 
     <p>If you can't restore some of the Node.js modules, such as ionic-angular, make sure that you have installed the Update 3 or later of Visual Studio and Microsoft ASP.NET and Web Tools (for more instructions, see the <a href="#node">previous issue</a>).</p> 
     <p>If that doesn't resolve the issue, open a command line and go to the project folder, then run this command:</p> 
     <pre class="prettyprint copy-to-clipboard ">npm install
</pre> 
     <p>and then this command:</p> 
     <pre class="prettyprint copy-to-clipboard ">npm rebuild node-sass
</pre> 
     <p>Then create a new Ionic 2 project from the templates you downloaded previously.</p> 
     <h3><a id="wwahost" name="wwahost" href="#wwahost"></a> Did you receive a WWAHost runtime error?</h3> 
     <p>When debugging on a Windows 8.1 dev machine, you may get a WWAHost runtime error when navigating between pages in Ionic apps. You can work around this by:</p> 
     <ul> 
      <li><p>Closing DOM Explorer before navigating pages, or</p></li> 
      <li><p>Upgrading to Windows 10 on your dev machine (the platform issue is fixed in Windows 10).</p></li> 
     </ul> 
     <h3><a id="partialpages" name="partialpages" href="#partialpages"></a> Did you receive an error message asking you to install a new app?</h3> 
     <p>When you are using the AngularJS routing module (Ionic templates use this module) on Windows, you may need to include a call to <code>aHrefSanitizationWhitelist</code>. This will correct errors that occur when loading partial pages.</p> 
     <p>If you see the dialog box shown here, you have likely run into this issue.</p> 
     <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/tutorial-ionic2/20170216102556/figure-18.png" alt="partial pages don't load"> </p> 
     <p>Typically, you include the code fix in app.js or wherever you are calling your module configuration code (inside <code>angular.module.config</code>):</p> 
     <pre class="prettyprint copy-to-clipboard ">$compileProvider.aHrefSanitizationWhitelist(/^\s*(https?|ftp|mailto|file|ghttps?|ms-appx|ms-appx-web|x-wmapp0):/);

$compileProvider.imgSrcSanitizationWhitelist(/^\s*(https?|ftp|file|ms-appx|ms-appx-web|x-wmapp0):|data:image\//);
</pre> 
     <p>If you use the preceding code, then add a reference to $compileProvider in the following line of code.</p> 
     <pre class="prettyprint copy-to-clipboard ">.config(function ($stateProvider, $urlRouterProvider) {
</pre> 
     <p>so that it looks like this:</p> 
     <pre class="prettyprint copy-to-clipboard ">.config(function ($compileProvider, $stateProvider, $urlRouterProvider) {
</pre> 
     <h3><a id="unhandled" name="unhandled" href="#unhandled"></a> Did you receive an 'Unhandled exception' running on Windows?</h3> 
     <p>If you see the following unhandled exception when targeting Win/WinPhone 8.1, follow the <a href="#configWindows">earlier steps</a> to call platformOverrides.js to fix this issue.</p> 
     <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/tutorial-ionic2/20170216102556/figure-19.png" alt="unhandled exception"> </p> 
     <p>If you see the same error when targeting Windows 10, make sure you set <strong>Windows 10.0</strong> as the target in the Windows tab of the configuration designer.</p> 
     <h3><a id="appxrecipe" name="appxrecipe" href="#appxrecipe"></a> Did you receive an error messaging indicating that the <code>appxrecipe</code> file is missing?</h3> 
     <p>If you see this error when targeting Windows 10, make sure you set <strong>Windows 10.0</strong> as the target in the Windows tab of the configuration editor (config.xml). Then delete the platforms\windows folder and rebuild the project.</p> 
     <h3><a id="csp" name="csp" href="#csp"></a> Did you receive an error message saying that the Content Security Policy is missing?</h3> 
     <p>Cordova projects automatically add the Whitelist plugin to every project, so you must add the following <code>meta</code> element to the project's <code>index.html</code> file:</p> 
     <pre class="prettyprint copy-to-clipboard ">&lt;meta http-equiv="Content-Security-Policy" content="default-src 'self' data: gap: https://ssl.gstatic.com 'unsafe-eval'; style-src 'self' 'unsafe-inline'; media-src *"&gt;
</pre> 
     <h3><a id="certificate" name="certificate" href="#certificate"></a> Did you receive a certificate error on Windows?</h3> 
     <p>Make sure your user credentials in Visual Studio are up to date. Check for any notifications or warning icons in the upper-right corner of the Visual Studio IDE:</p> 
     <p><img src="https://tacovs.blob.core.windows.net/vstaco-media/taco.visualstudio.com/en-us/documentation/articles/tutorial-ionic2/20170216102556/figure-17.png" alt="Update your credentials"> </p> 
     <p>You may need to re-enter your credentials. If the notifications indicate that you need to update Cordova tooling, please click on the notifications and follow the provided instructions.</p> 
     <h3><a id="vsAndroidEmu" name="vsAndroidEmu" href="#vsAndroidEmu"></a> Visual Studio Emulator for Android won't run?</h3> 
     <p>The VS Emulator for Android requires Hyper-V and is not supported when running on a VM. For more info, see <a href="https://msdn.microsoft.com/en-us/library/mt228282.aspx#NoStart2">this information</a>.</p> 
     <p>If you have previously run the VS Emulator for Android successfully but now the emulator won't run, try deleting the emulator VM instance in the Hyper-V Manager. For more info, see <a href="https://msdn.microsoft.com/en-us/library/mt228282">Troubleshooting</a>.</p>
