       <header class="trailer-1"> 
        <h1 class="trailer-1">A quick tour of sharing geoprocessing workflows</h1> 
       </header> 
       <main class="content-area"> 
        <p id="GUID-257ED762-C8F0-42A7-8367-DDC17007070C">A workflow is any multiple-step procedure that creates a result. A geoprocessing workflow includes geoprocessing tools and geographic data that produce meaningful results in the form of new datasets, maps, charts, and/or reports. For example, you may have a workflow that predicts landslides based on soil, slope, and weather data that use a variety of geoprocessing tools to produce a map of high-risk areas. The list of workflows you can create is endless—crime analysis, site suitability, hydrologic analysis, wildfire management, and so on. Workflows can be written as models or scripts that contain the tools needed to perform your analysis or management process.</p>
        <p id="GUID-4EFFD6DF-8183-42D5-A54A-04B28824AAB4">Your workflows can be conveniently shared with others by creating a</p>
        <ul class="bulleted" id="UL_F9BBF8E2F6264BC7A62689E76EFD3C8B">
         <li id="LI_C9D57407BAC74A56BF1DE85C1A88B509"><b>Geoprocessing package</b> (<span class="usertext">.gpk</span>) that can be uploaded to ArcGIS Online, e-mailed to others, or shared through your local network</li>
         <li id="LI_58E92A85F4004B0D92F9D9555E09BBD3"><b>Geoprocessing service</b> that can be accessed via the web by web-enabled clients, such as <span class="ph">ArcGIS for Desktop</span>; <span class="ph">ArcGIS Explorer Desktop</span>; ArcGIS Online; and custom web applications built with web-based APIs such JavaScript, Silverlight, and Flex </li>
        </ul>
        <p></p> 
        <p id="GUID-A15EB53E-0DEB-47F2-8F25-0F353851A810"> To create a package or a service, start with a geoprocessing result found in the <span class="wintitle">Results</span> window. A geoprocessing result contains all the information needed to share your workflow in its entirety. This includes</p>
        <ul class="bulleted" id="UL_DC7CDB8DB53449D8804E539DFA1FAFCC">
         <li id="LI_40A409208D9E45209BCE2D1B5B0701A5">The tool or tools used (usually a model or script tool you developed)</li>
         <li id="LI_EFFEDE2AB79A4092AD123448CC3E7774">The input and output data used to create the result</li>
         <li id="LI_E0E0DDBF2982461BAC5F678E702F2B88">The environment settings in effect when the tool was run</li>
        </ul>
        <p></p> 
        <p id="GUID-F114918D-4D24-45F8-8362-1309C0AC256E"> To create a package or service, right-click a result in the <span class="wintitle">Results</span> window and select <span class="uicontrol">Share As <span class="greaterthan">&gt;</span> Geoprocessing Package</span> or <span class="uicontrol">Share As <span class="greaterthan">&gt;</span> Geoprocessing Service</span>, as illustrated below. </p>
        <p id="GUID-5C2B6CCB-5CC7-4AD7-BC14-EE773F1451E9"></p>
        <figure class="fig">
         <img src="GUID-CCD0E365-E52A-4012-AB77-9B485B483818-web.png" alt="Sharing a geoprocessing result" title="Sharing a geoprocessing result">
        </figure>
        <p></p>
        <p id="GUID-5B3D50C1-FAD7-4434-8BFD-C30ED0A23B21">If you're creating a package, the <span class="wintitle">Geoprocessing Package</span> window opens and guides you through the process of creating your package. From this window, you can add additional files to the package and choose whether to include the data used or just the schema of the data (data schema is the definition of the tables and feature classes but not the actual rows or features). Recipients of your package can drag and drop it directly into ArcMap, and the contents of the package will be unpacked. Data in the package will be written to disk and then added to the ArcMap table of contents. A new result will be added to the <span class="wintitle">Results</span> window beneath the <span class="uicontrol">Shared</span> node. Recipients can then examine your tools and data; rerun your tools on the unpacked data; or, by supplying their own input data, add your tools to their own models. In short, they can replicate your entire workflow on their computer.</p> 
        <p id="GUID-6D7CEBCB-CA55-460F-883D-A43D802D3D06"> If you're creating a service, the <span class="wintitle">Share as Service</span> wizard opens. After choosing from among a few standard publishing options in the <span class="wintitle">Share as Service</span> wizard, the <span class="wintitle">Service Editor</span> window opens. The <span class="wintitle">Service Editor</span> window is where you define your geoprocessing service. </p>
        <p id="GUID-210DB78E-BBC6-4C2B-8290-2234471F60DF"> The <span class="uicontrol">Share As</span> menu choice will be disabled for a result when:</p>
        <ul class="bulleted" id="UL_3FA9FFC5F31E4510A1614BB54DC60F8A">
         <li id="LI_7951E30202104697A92168D23A02B523">The result is invalid—a result that has an execution error <img src="GUID-8D5ED5A4-D38D-4453-AEB7-5E8E5E84C727-web.png" alt="Error" title="Error">.</li>
         <li id="LI_6D8567E898F144B19849793D2BB5C434">The <span class="wintitle">Geoprocessing Package</span> window or the <span class="wintitle">Service Editor</span> is already open.</li>
         <li id="LI_4C7E80A64E4F427988DA7E9991F2CA18">The result was created by executing a task within a geoprocessing service—only tools that execute on your computer can be shared.</li>
         <li id="LI_14540D5DE35F46A8AFB13C3A508FA441">The result is from an execution of the <span>Stage Service</span> or <span>Upload Service Definition</span> tool—these tools cannot be packaged or made into service tasks.</li>
         <li id="LI_8363365CBCB44BBCA5E14B87DA22FAD2">The result uses a model tool that has <a class="xref xref" esrisubtype="extrel" hrefpfx="/en/arcmap/10.3/analyze" href="/en/arcmap/10.3/analyze/modelbuilder/using-lists.htm">list variables</a> within a model—model tools that use list variables cannot be included in packages or made into service tasks.</li>
        </ul>
        <p></p>
