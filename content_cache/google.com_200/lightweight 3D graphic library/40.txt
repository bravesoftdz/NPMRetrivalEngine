     <p><b>Libraries</b>, also known as <b>frameworks</b>, <b>components</b>, <b>modules</b>, or <b>software development kits</b> (SDKs), contain reusable code that helps developers integrate OpenStreetMap into their websites or software applications without reinventing the wheel. Developers have written libraries for accessing and analyzing data, rendering maps, geocoding, and routing. These libraries are compatible with the Web and a wide variety of desktop and mobile platforms. </p>
     <p>A library is primarily included in a larger application, rather than being run as a standalone tool. However, some of the libraries listed here also provide command-line tools to facilitate development. A standalone tool may be just as convenient as a library in some environments, particularly on the command line; see <a href="/wiki/Software" title="Software">Software</a> for a full listing. </p>
     <p>The libraries listed here are actively maintained or are considered to be mature and complete. Libraries of historical interest can be found at <a href="/wiki/Category:Discontinued_software" title="Category:Discontinued software">Category:Discontinued software</a>. </p>

     <h2><span class="mw-headline" id="Accessing_raw_OpenStreetMap_data">Accessing raw OpenStreetMap data</span></h2>

     <h2><span class="mw-headline" id="Displaying_interactive_maps">Displaying interactive maps</span></h2>

     <p>Interactive maps, also known as <a href="/wiki/Slippy_map" class="mw-redirect" title="Slippy map">slippy maps</a> on the Web, are the most common way of integrating OpenStreetMap into a website or application.      <h2><span class="mw-headline" id="Other_developer_tools">Other developer tools</span></h2> 
     <p><i>Main article:</i> <a href="/wiki/GIS_software" title="GIS software">GIS software</a> </p>
     <p>General-purpose GIS libraries are often used in conjunction with OpenStreetMap-specific libraries to work with geographic data: </p> 
     <ul>
      <li> <a rel="nofollow" class="external text" href="http://trac.osgeo.org/proj/">PROJ.4</a> - Geo Projection Conversion lib with multiple bindings</li> 
      <li> <a rel="nofollow" class="external text" href="http://www.gdal.org/">GDAL</a> - Geo Raster image formats lib</li> 
      <li> <a rel="nofollow" class="external text" href="http://www.gdal.org/ogr/">OGR</a> - Geo Vector data formats lib</li> 
      <li> <a href="/wiki/PostGIS" title="PostGIS">PostGIS</a> - The leading Geo enabled Database</li> 
      <li> <a rel="nofollow" class="external text" href="http://www.gaia-gis.it/gaia-sins/">spatialite</a> - sqlite extension, fully fledged spatial db</li> 
      <li> <a rel="nofollow" class="external text" href="http://www.mapbender.org">Mapbender</a> - Another more GIS focused web map lib</li> 
      <li> <a rel="nofollow" class="external text" href="http://www.geotools.org/">Geotools</a> -Java GIS Toolkit</li> 
      <li> <a rel="nofollow" class="external text" href="http://trac.gispython.org/lab">GIS-Python lab</a> - Python GIS Toolkit</li> 
      <li> <a rel="nofollow" class="external text" href="http://github.com/k3b/k3b-geoHelper">k3b-geoHelper </a> - java lib to parse gpx and kml files and en/decode geo-uris. Works with Android, too</li> 
      <li> <a rel="nofollow" class="external text" href="https://framagit.org/PanierAvide/Pic4Carto.js">Pic4Carto.js</a> - JavaScript library for open-licensed street pictures retrieval (which can be used for <a href="/wiki/Photo_mapping" title="Photo mapping">photo mapping</a>, <a class="external text" href="http://www.openstreetmap.org/user/PanierAvide/diary/40298">tutorial</a>)</li>
     </ul>
