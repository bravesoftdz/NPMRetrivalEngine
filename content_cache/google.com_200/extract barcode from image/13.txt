   <h2><a name="JavaScript_Integration"></a>JavaScript Integration</h2> 
   <p>The JavaScript Font Encoders listed in <a href="#Available_JavaScript_Font_Encoders">Chart 1</a> are provided with a developer license to any of the IDAutomation <a href="http://www.idautomation.com/barcode-fonts/"> Barcode Font Packages</a>.</p> 
   <ol> 
    <li>Extract the files from the compressed zip file.</li> 
    <li>Several files are provided for different barcode types (Symbologies) and purposes. Use the file that <a href="#Barcode_Properties">represents</a> the symbology that is needed. This tutorial will use the Code 128 symbology, which is in the file named <span style="font-family: Courier New; color: green"> IDAutomation_Uni_C128.js</span>. HTML example files are also provided and may be referred to as necessary. <br>Two files are also provided for various purposes:
     <ol> 
      <li><em>symbology</em>.js - the primary compressed file to use in production.</li> 
      <li><em>symbology</em>.uncompressed.js - the uncompressed source code of the primary file.</li> 
     </ol> </li> 
    <li>Open the application or IDE where the barcode is to be integrated.</li> 
    <li>Add the following line at the top of the file just before the <span style="font-family: Courier New; color: green">&lt;/head&gt;</span> line to load the appropriate JavaScript file:<br> <pre class="brush: js; gutter: false;">&lt;script language="JavaScript" src="IDAutomation_Uni_C128.js"&gt;&lt;/script&gt;</pre></li> 
    <li>Add a script where the barcode is to appear, which is the file name without the ".js":<br> <pre class="brush: js; gutter: false;">&lt;script type="text/javascript"&gt;
document.write(IDAutomation_Uni_C128(DataToEncode))
&lt;/script&gt;</pre> </li> 
    <li>Modify "DataToEncode" to equal what is to be encoded in the barcode. This may be accomplished with a variable, a <a href="http://www.idautomation.com/barcode-components/java/servlet-demo.html"> dynamically generated webpage</a> or another product such as JQuery, which is used in this code example: <br> <pre class="brush: js; gutter: false;">&lt;div id="Barcode_Font"&gt; &lt;/div&gt;
&lt;script type="text/javascript"&gt;
&nbsp;&nbsp; $(document).ready(function() {&nbsp;&nbsp;&nbsp; 
   $("#encode").click(function(e) {&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 
   barcodetext = $("#datatoencode").val();&nbsp;&nbsp;&nbsp;&nbsp; 
   $("#Barcode_Font").html(IDAutomation_Uni_C128(barcodetext));&nbsp;&nbsp; 
}); 
&nbsp;&nbsp; });
&lt;/script&gt;</pre> JQuery is also used with the HRI object from this product in the <a href="http://www.idautomation.com/barcode-properties/definitions/gs1-element-string-hri-tool.html"> GS1 Element String HRI Verification Tool</a>.</li> 
    <li>Display the result with the <a href="#Required_Font">required barcode font</a> to generate the correct barcode. This may be accomplished using CSS and Web Fonts or using CSS with locally installed fonts. For example, the following CSS and HTML format data to the font:<br> <pre class="brush: js; gutter: false;">&lt;script language="JavaScript" src="IDAutomation_Uni_C128.js"&gt;&lt;/script&gt;
&lt;style type="text/css"&gt;
.barcodefont1 {font-family: "IDAutomation Uni M", "IDAutomationGS1 46"; font-size: 200%; }
&lt;/style&gt;
...
&lt;span class="barcodefont1"&gt; &lt;script type="text/javascript"&gt;
document.write(IDAutomation_Uni_C128("www.idautomation.com"))
&lt;/script&gt; &lt;/span&gt;
...</pre></li> 
    <li>Print or display the result to verify that the barcode is generated. If a scanner is needed to verify barcodes, IDAutomation offers several <a href="http://www.idautomation.com/barcode-scanners/">barcode scanners</a>. It is normal to view some distortion of the barcode on the screen because of low screen resolution. However, the barcodes should be accurate when printed or viewed on high resolution screens.</li> 
   </ol>

   <h2><a name="Barcode_Properties"></a>Barcode Properties</h2> 
   <p>Barcode properties may be modified within the top of the referenced .js file or by a function parameter. </p> 
   <p><strong>Top of the .js file Example:</strong><br> <img alt="Top of the .js file Modification Example" src="img/javascript-function-modify-top.png" class="whitespace"></p> 
   <p> <strong>Function Parameter Example:</strong><br> <img alt="Function Parameter Modification Example" src="img/javascript-function-modify-ide.png" class="whitespace"></p> 
   <p><strong>The following is an explanation of popular properties:</strong></p> 
   <ul> 
    <li><strong>ApplyTilde</strong> - Processes the tilde character for special functions such as encoding functions, encoding ASCII characters directly or generating <a href="http://www.idautomation.com/barcode-faq/gs1-128/"> GS1-128</a>, <a href="http://www.idautomation.com/barcode-faq/2d/gs1-datamatrix/index.html"> GS1-DataMatrix</a> or <a href="https://www.idautomation.com/barcode-faq/2d/qr-code/aim-gs1-fnc1.html"> GS1-QRCode</a> symbols.</li> 
    <li><strong>IDElement</strong> - is an optional parameter that can place the returned image into the referenced div id element.<br> <pre class="brush: js; gutter: false;">&lt;div id="html5_barcode_font"&gt;&lt;/div&gt;
...
&lt;script type="text/javascript"&gt;
&nbsp;&nbsp; document.write(IDAutomation_Uni_C128.js("123456789012",'html5_barcode_font'));
&lt;/script&gt;</pre></li> 
    <li><strong>HTML</strong> - returns &lt;br&gt; for carriage returns in 2D symbologies when set to "1", this is the default setting. Set to "0" to use CR &amp; LF for carriage returns.</li> 
    <li><strong><a name="GS1_Encoding_Options">GS1 Encoding Options</a></strong> - The FNC1 may be encoded with ~202 or the ASCII 202 character Ê in <a href="http://www.idautomation.com/barcode-faq/gs1-128/">GS1-128</a> and with ~1 in <a href="http://www.idautomation.com/barcode-faq/2d/gs1-datamatrix/index.html"> GS1-DataMatrix</a> or <a href="https://www.idautomation.com/barcode-faq/2d/qr-code/aim-gs1-fnc1.html"> GS1-QRCode</a> when the tilde option is enabled. The FNC1 may also be encoded by surrounding the AI in parentheses.</li> 
    <li><strong>HRI Options</strong> - When using IDAutomation_GS1_HRI.js, multiple GS1 fixed length element strings are automatically displayed. <br>Examples:<br> <a href="http://www.idautomation.com/barcode-faq/gs1-128/">GS1-128</a> encoding: <strong>~202</strong>01095011010209171719050810ABCD1234<br> <img alt="GS1-128 JavaScript Example" height="92" src="img/gs1-128-javascript-example.png" width="552" class="whitespace"><br> <a href="http://www.idautomation.com/barcode-faq/2d/gs1-datamatrix/index.html"> GS1-DataMatrix</a> encoding: <strong>~1</strong>01095011010209171719050810ABCD1234<br> <img alt="GS1-DataMatrix JavaScript Example" height="240" src="img/gs1-datamatrix--javascript-example.png" width="182" class="whitespace"><br></li> 
    <li><strong><a name="Tilde_Options">Tilde Options</a></strong> - enabling ApplyTilde or ProcessTilde will allow the following calculation options where <strong>??</strong> is the number of digits before the tilde to include in the calculation:
     <ul> 
      <li><strong>~m??</strong> calculates a MOD10 check character, commonly used for GS1.<br>Example: ~2020000801234999999999<strong>~m17</strong>&nbsp; <img alt="HIBC JavaScript Example" height="132" src="img/hibc-example-javascript.png" style="float: right" width="497" class="whitespace"><br>Encodes: (00) 00801234999999999<strong>7</strong></li> 
      <li><strong>~f??</strong> calculates a MOD43 check character, commonly used for HIBC.<br>Example: +A123BJC5D6E71<strong>~f14 </strong><br> Encodes: +A123BJC5D6E71<strong>G</strong></li> 
      <li><strong>~i??</strong> calculates a MOD37-2 check character, commonly used for ISBT.<br>Example: G123412654321<strong>~i13 </strong><br>Encodes: G123412654321<strong>L</strong></li> 
      <li><strong>~k??</strong> calculates a MOD37-2 check character in brackets, commonly used for ISBT manual entry. This option will not encode the check character within the barcode.<br>Example: G123412654321~k13 <br>Encodes: G123412654321 [L]</li> 
     </ul> </li> 
   </ul>
