        <p>Web scraping is a common and effective way of collecting data for projects and for work. In this guide, well be touring the essential stack of Python web scraping libraries.</p> 
        <div class="image-caption" style="width:640px;margin-top:0px;margin-bottom:px;margin-right:auto;margin-left:auto;">
         <img alt="Python Web Scraping Libraries" src="https://datonauts.com/wp-content/uploads/2016/10/pyhon-web-scraping-feature-image-small-1.jpg" border="0" class="full-width">
        </div> 
        <p><strong>Why only 5 libraries?</strong></p> 
        <p>There are dozens of packages for web scraping out there... but you only need a handful to be able to scrape almost any site. This is an opinionated guide. We've decided to feature the 5 Python libraries for web scraping that we love most. Together, they cover all the important bases, and they are well-documented.</p> 
        <p><strong>Do I need to learn every library below?</strong></p> 
        <p>No, but everyone&nbsp;will need Requests, because it's how you communicate with websites. The rest depend on your use case. Here's a rule of thumb:</p> 
        <ul> 
         <li>You should learn at least one of BeautifulSoup or lxml. Pick&nbsp;depending on which is more intuitive for you (more on this below).</li> 
         <li>Learn Selenium if you need to scrape sites with data tucked away by JavaScript.</li> 
         <li>Learn Scrapy if you need to build a real spider or web-crawler, instead of just scraping a few pages here and there.</li> 
        </ul> 
        <p><strong>Why are they tasty?</strong></p> 
        <p>Because they are yummy! So without further ado...</p> 
        <ul> 
         <li><a href="#requests">The Farm: Requests</a></li> 
         <li><a href="#beautiful-soup">The Stew: Beautiful Soup 4</a></li> 
         <li><a href="#lxml">The Salad: lxml</a></li> 
         <li><a href="#selenium">The Restaurant: Selenium</a></li> 
         <li><a href="#scrapy">The Chef: Scrapy</a></li> 
        </ul> 
        <h2 id="requests">The Farm: Requests</h2> 
        <p>The Requests library is vital to add to your data science toolkit. It's a simple yet powerful HTTP library, which means you can use it to access web pages.</p> 
        <p>We call it The Farm because you'll be using it to get the raw ingredients (i.e. raw HTML) for your dishes (i.e. usable data).</p> 
        <p>Its simplicity is definitely its greatest strength. It's so easy use that you could jump right in without reading documentation.</p>

        <p>But that's not all that Requests can do. It can access API's, post to forms, and much more.</p> 
        <p>Plus, it's got character... It's the only library that calls itself Non-GMO, organic, and grass-fed. You gotta love that.</p>

        <ul> 
         <li><a href="http://docs.python-requests.org/en/master/user/quickstart/" target="_blank" rel="noopener">Requests Quickstart Guide</a>&nbsp;- Official documentation. Covers practical&nbsp;topics like passing parameters, handling responses, and configuring headers.</li> 
        </ul> 
        <h2 id="beautiful-soup">The Stew: Beautiful Soup 4</h2> 
        <p>After you have your ingredients, now what? Now you make them into a stew... a beautiful stew.</p> 
        <p>Beautiful Soup (BS4)&nbsp;is a parsing library that can use different <strong>parsers</strong>. A parser is simply a program that can extract&nbsp;data from HTML and XML documents.</p> 
        <p>Beautiful Soup's default parser comes from Python's standard library. It's flexible and forgiving, but a little slow. The good news is that you can swap out its parser with a faster one if you need the speed.</p> 
        <p>One advantage of BS4&nbsp;is its ability to automatically&nbsp;detect encodings. This allows it to&nbsp;gracefully handle HTML documents with special characters.</p> 
        <p>In addition, BS4 can help you&nbsp;navigate a parsed document and find what you need. This makes it quick&nbsp;and painless&nbsp;to build common applications. For example, if you wanted to find all the links in the web page we pulled down earlier, it's only a few&nbsp;lines:</p>

        <p><strong>Resources</strong></p> 
        <ul> 
         <li><a href="https://www.crummy.com/software/BeautifulSoup/bs4/doc/" target="_blank" rel="noopener">Beautiful Soup Documentation</a> -&nbsp;Includes convenient quickstart guide.</li> 
         <li><a href="https://gist.github.com/bradmontgomery/1872970" target="_blank" rel="noopener">Really Short Example</a>&nbsp;- Short example of using Beautiful Soup and Requests together.</li> 
        </ul> 
        <h2 id="lxml">The Salad: lxml</h2> 
        <p>Lxml is a high-performance, production-quality HTML and XML parsing library. We call it The Salad&nbsp;because you can rely on it to be good for you, no matter which diet you're following.</p> 
        <p>Among all the Python web scraping libraries, we've enjoyed using lxml the most. It's straightforward, fast, and feature-rich.</p> 
        <p>Even so, it's quite easy to pick up if you have experience with either XPaths or CSS. Its raw speed and power has also helped it become widely adopted in the industry.</p> 
        <p><strong>Beautiful Soup vs lxml</strong></p> 
        <p><em>Historically,</em> the rule of thumb was:</p> 
        <ul> 
         <li>If you need speed, go for lxml.</li> 
         <li>If you need to handle messy documents, choose Beautiful Soup.</li> 
        </ul> 
        <p>Yet, this distinction no longer holds. Beautiful Soup now supports using the lxml parser, and vice-versa. It's also pretty easy to learn the other once you've learned one.</p> 
        <p>So to start, we recommend trying both and picking the one that feels more intuitive for you. We prefer lxml, but many swear by Beautiful Soup.</p> 
        <p><strong>Resources</strong></p> 
        <ul> 
         <li><a href="http://lxml.de/index.html#introduction" target="_blank" rel="noopener">lxml Documentation</a> - Official documentation.</li> 
         <li><a href="http://docs.python-guide.org/en/latest/scenarios/scrape/" target="_blank" rel="noopener">HTML Scraping with lxml and Requests</a> - Short and sweet tutorial on pulling a webpage with Requests and then using XPath selectors to mine the desired data. This is more beginner-friendly than the official documentation.</li> 
        </ul> 
        <h2 id="selenium">The Restaurant: Selenium</h2> 
        <p>Sometimes, you do need to go to a restaurant to eat certain dishes. The farm is great, but you can't find everything there.</p> 
        <p>Likewise, sometimes the Requests library is not enough to scrape a website. Some sites out there use JavaScript to serve content. For example, they might wait until you scroll down on the page or click a button before loading certain content.</p> 
        <p>Other sites may require you to click through forms before seeing their content. Or select options from a dropdown. Or perform a tribal rain dance...</p> 
        <p>For these sites, you'll need something more powerful. You'll need Selenium (which can handle everything except tribal rain dancing).</p> 
        <p>Selenium is a tool that automates browsers, also known as a web-driver. With it, you can actually open a Google Chrome window, visit a site, and click on links. Pretty cool, right?</p> 
        <p>It also comes with Python bindings for controlling it right from your application. This makes it a breeze to integrate with your chosen parsing library.</p> 
        <p><strong>Resources</strong></p> 
        <ul> 
         <li><a href="http://selenium-python.readthedocs.io/" target="_blank" rel="noopener">Selenium with Python</a> -&nbsp;Documentation for Selenium's Python bindings.</li> 
         <li><a href="http://thiagomarzagao.com/2013/11/12/webscraping-with-selenium-part-1/" target="_blank" rel="noopener">Webscraping with Selenium</a> - Excellent, thorough 3-part&nbsp;tutorial for scraping websites with Selenium.</li> 
         <li><a href="https://www.scrapehero.com/tutorial-web-scraping-hotel-prices-using-selenium-and-python/" target="_blank" rel="noopener">Scraping Hotel Prices</a>&nbsp;- Code snippet for scraping hotel prices using Selenium and lxml.</li> 
        </ul> 
        <h2 id="scrapy">The Chef:&nbsp;Scrapy</h2> 
        <p>Ok, we covered a lot just now. You've got Requests and Selenium for fetching HTML/XML from web pages. Then, you can use Beautiful Soup or lxml to parse it into useful data.</p> 
        <p>But what if you need more than that? What if you need a complete spider that can crawl through entire websites in a systematic way?</p> 
        <p>Introducing: Scrapy! Scrapy is technically not even a library... it's a complete web scraping framework. That means you can use it to manage requests, preserve user sessions, follow redirects, and handle output pipelines.</p> 
        <p>It also means you can swap out individual modules with other Python web scraping libraries. For instance, if you need to insert Selenium for scraping dynamic web pages, you can do that (<a href="http://stackoverflow.com/questions/17975471/selenium-with-scrapy-for-dynamic-page" target="_blank" rel="noopener">see example</a>).</p> 
        <div class="image-caption frame-style-1" style="width:640px;margin-top:0px;margin-bottom:px;margin-right:auto;margin-left:auto;">
         <img alt="Scrapy architecture" src="https://datonauts.com/wp-content/uploads/2016/10/scrapy_architecture.png" border="0" class="full-width">
         <p class="wp-caption-text">Scrapy architecture, image borrowed from <a href="https://doc.scrapy.org/en/latest/topics/architecture.html" target="_blank">official documentation</a></p>
        </div> 
        <p>So if you need to reuse your crawler, scale it, manage complex data pipelines, or cook up some other sophisticated spider, then Scrapy was made for you.</p> 
        <p><strong>Resources</strong></p> 
        <ul> 
         <li><a href="https://scrapy.org/" target="_blank" rel="noopener">Scrapy Documentation</a> - Official site with links to many other resources.</li> 
         <li><a href="https://medium.com/@kaismh/extracting-data-from-websites-using-scrapy-e1e1e357651a#.sw7c9ycio" target="_blank" rel="noopener">Extracting data from websites with Scrapy</a> - Detailed tutorial for scraping an e-commerce site using Scrapy.</li> 
         <li><a href="https://scrapinghub.com/" target="_blank" rel="noopener">Scrapinghub</a> - Cloud-based crawling service by the creators of Scrapy.&nbsp;The first cloud unit is free.</li> 
        </ul>
