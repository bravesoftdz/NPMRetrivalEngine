           <h3 name="e3fc" id="e3fc" class="graf graf--h3 graf-after--figure">What is web scraping?</h3>
           <p name="dced" id="dced" class="graf graf--p graf-after--h3">Web scraping is <strong class="markup--strong markup--p-strong">extracting data from a website</strong>. Why would someone want to scrape the web? Here are four examples:</p>
           <ul class="postList">
            <li name="81d3" id="81d3" class="graf graf--li graf-after--p">Scraping social media sites to find trending data</li>
            <li name="9d26" id="9d26" class="graf graf--li graf-after--li">Scraping email addresses from websites that publish public emails</li>
            <li name="2930" id="2930" class="graf graf--li graf-after--li">Scraping data from another website to use on your own site</li>
            <li name="febc" id="febc" class="graf graf--li graf-after--li">Scraping online stores for sales data, product pictures, etc.</li>
           </ul>
           <h3 name="8c8f" id="8c8f" class="graf graf--h3 graf-after--li">Warnings.</h3>
           <p name="095f" id="095f" class="graf graf--p graf-after--h3">Web scraping is against most website’s terms of service. Your IP address may be banned from a website if you scrape too frequently or maliciously.</p>
           <h3 name="45d1" id="45d1" class="graf graf--h3 graf-after--p">What will we&nbsp;need?</h3>
           <p name="7d50" id="7d50" class="graf graf--p graf-after--h3">For this project we’ll be using <a href="https://nodejs.org/en/" data-href="https://nodejs.org/en/" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Node.js</a>. If you’re not familiar with Node, check out my <a href="https://codeburst.io/3-best-node-js-courses-dd9983edb32c" data-href="https://codeburst.io/3-best-node-js-courses-dd9983edb32c" class="markup--anchor markup--p-anchor" target="_blank"><strong class="markup--strong markup--p-strong">3 Best Node.JS Courses</strong></a>.</p>
           <p name="9774" id="9774" class="graf graf--p graf-after--p">We’ll also be using two open-sourced <a href="https://www.npmjs.com/" data-href="https://www.npmjs.com/" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank"><strong class="markup--strong markup--p-strong">npm</strong></a><strong class="markup--strong markup--p-strong"> </strong>modules to make today’s task a little easier:</p>
           <ul class="postList">
            <li name="ccce" id="ccce" class="graf graf--li graf-after--p"><a href="https://github.com/request/request-promise" data-href="https://github.com/request/request-promise" class="markup--anchor markup--li-anchor" rel="noopener" target="_blank"><strong class="markup--strong markup--li-strong">request-promise?</strong></a>—?Request is a simple HTTP client that allows us to make quick and easy HTTP calls.</li>
            <li name="28b4" id="28b4" class="graf graf--li graf-after--li"><a href="https://github.com/cheeriojs/cheerio" data-href="https://github.com/cheeriojs/cheerio" class="markup--anchor markup--li-anchor" rel="noopener" target="_blank"><strong class="markup--strong markup--li-strong">cheerio</strong></a>?—?jQuery for Node.js. Cheerio makes it easy to select, edit, and view DOM elements.</li>
           </ul>
           <h3 name="b931" id="b931" class="graf graf--h3 graf-after--li">Project Setup.</h3>
           <p name="7069" id="7069" class="graf graf--p graf-after--h3">Create a new project folder. Within that folder create an <code class="markup--code markup--p-code">index.js</code> file. We’ll need to install and require our dependencies. Open up your command line, and install and save:<strong class="markup--strong markup--p-strong"> request, request-promise, and cheerio</strong></p>
           <pre name="0cad" id="0cad" class="graf graf--pre graf-after--p">npm install --save request request-promise cheerio</pre>
           <p name="06d6" id="06d6" class="graf graf--p graf-after--pre">Then require them in our <code class="markup--code markup--p-code">index.js</code> file:</p>
           <pre name="8d4b" id="8d4b" class="graf graf--pre graf-after--p">const rp = require('request-promise');<br>const cheerio = require('cheerio');</pre>
           <h3 name="1732" id="1732" class="graf graf--h3 graf-after--pre">Setting up the&nbsp;Request</h3>
           <p name="c7bc" id="c7bc" class="graf graf--p graf-after--h3"><code class="markup--code markup--p-code">request-promise</code> accepts an object as input, and returns a promise. The <code class="markup--code markup--p-code">options</code> object needs to do two things:</p>
           <ol class="postList">
            <li name="ca80" id="ca80" class="graf graf--li graf-after--p">Pass in the url we want to scrape.</li>
            <li name="8af1" id="8af1" class="graf graf--li graf-after--li">Tell Cheerio to load the returned HTML so that we can use it.</li>
           </ol>
           <p name="83b9" id="83b9" class="graf graf--p graf-after--li">Here’s what that looks like:</p>
           <pre name="0133" id="0133" class="graf graf--pre graf-after--p">const options = {<br>  uri: `<a href="https://www.google.com/search?num=10&amp;q=$%7Bsearch%7D`" data-href="https://www.google.com/search?num=10&amp;q=${search}`" class="markup--anchor markup--pre-anchor" rel="nofollow noopener" target="_blank">https://www.yourURLhere.com`</a>,<br>  transform: function (body) {<br>    return cheerio.load(body);<br>  }<br>};</pre>
           <p name="b5d2" id="b5d2" class="graf graf--p graf-after--pre">The <code class="markup--code markup--p-code">uri</code> key is simply the website we want to scrape.</p>
           <p name="87d3" id="87d3" class="graf graf--p graf-after--p">The <code class="markup--code markup--p-code">transform</code> key tells <code class="markup--code markup--p-code">request-promise</code> to take the returned body and load it into Cheerio before returning it to us.</p>
           <p name="80fe" id="80fe" class="graf graf--p graf-after--p">Awesome. We’ve successfully set up our HTTP request options! Here’s what your code should look like so far:</p>
           <pre name="544f" id="544f" class="graf graf--pre graf-after--p">const rp = require('request-promise');<br>const cheerio = require('cheerio');</pre>
           <pre name="0d16" id="0d16" class="graf graf--pre graf-after--pre">const options = {<br>  uri: `<a href="https://www.google.com/search?num=10&amp;q=$%7Bsearch%7D`" data-href="https://www.google.com/search?num=10&amp;q=${search}`" class="markup--anchor markup--pre-anchor" rel="nofollow noopener noopener" target="_blank">https://www.yourURLhere.com`</a>,<br>  transform: function (body) {<br>    return cheerio.load(body);<br>  }<br>};</pre>
           <h3 name="24e5" id="24e5" class="graf graf--h3 graf-after--pre">Make the&nbsp;Request</h3>
           <p name="9675" id="9675" class="graf graf--p graf-after--h3">Now that the options are taken care of, we can actually make our request. The boilerplate in the documentation for that looks like this:</p>
           <pre name="f13f" id="f13f" class="graf graf--pre graf-after--p">rp(<strong class="markup--strong markup--pre-strong">OPTIONS</strong>)<br>    .then(function (data) {<br>        // <strong class="markup--strong markup--pre-strong">REQUEST SUCCEEDED:</strong> <strong class="markup--strong markup--pre-strong">DO SOMETHING</strong><br>    })<br>    .catch(function (err) {<br>        // <strong class="markup--strong markup--pre-strong">REQUEST FAILED: ERROR OF SOME KIND</strong><br>    });</pre>
           <p name="b27a" id="b27a" class="graf graf--p graf-after--pre">We pass in our <code class="markup--code markup--p-code">options</code> object to <code class="markup--code markup--p-code">request-promise</code>, then wait to see if our request succeeds or fails. Either way, we do something with the returned data.</p>
           <p name="743d" id="743d" class="graf graf--p graf-after--p">Knowing what the documentation says to do, lets create our own version:</p>
           <pre name="8d77" id="8d77" class="graf graf--pre graf-after--p">rp(options)<br>  .then(($) =&gt; {<br>    console.log($);<br>  })<br>  .catch((err) =&gt; {<br>    console.log(err);<br>  });</pre>
           <p name="5a78" id="5a78" class="graf graf--p graf-after--pre">The code is pretty similar. The big difference is I’ve used arrow functions. I’ve also logged out the returned data from our HTTP request. We’re going to test to make sure everything is working so far.</p>
           <p name="dc5e" id="dc5e" class="graf graf--p graf-after--p">Replace the placeholder <code class="markup--code markup--p-code">uri</code> with the website you want to scrape. Then, open up your console and type:</p>
           <pre name="dbd0" id="dbd0" class="graf graf--pre graf-after--p"><strong class="markup--strong markup--pre-strong">node index.js</strong></pre>
           <pre name="bb3a" id="bb3a" class="graf graf--pre graf-after--pre">// LOGS THE FOLLOWING:<br>{ [Function: initialize]<br>  fn:<br>   initialize {<br>     constructor: [Circular],<br>     _originalRoot:<br>      { type: 'root',<br>        name: 'root',<br>        namespace: '<a href="http://www.w3.org/1999/xhtml%27" data-href="http://www.w3.org/1999/xhtml'" class="markup--anchor markup--pre-anchor" rel="nofollow noopener" target="_blank">http://www.w3.org/1999/xhtml'</a>,<br>        attribs: {},<br>        ...</pre>
           <p name="dc48" id="dc48" class="graf graf--p graf-after--pre">If you don’t see an error, then everything is working so far?—?and you just made your first scrape!</p>

           <h3 name="666e" id="666e" class="graf graf--h3 graf-after--figure">Using the&nbsp;Data</h3>
           <p name="7089" id="7089" class="graf graf--p graf-after--h3">What good is our web scraper if it doesn’t actually return any useful data? This is where the fun begins.</p>
           <p name="911a" id="911a" class="graf graf--p graf-after--p">There are numerous things you can do with Cheerio to extract the data that you want. First and foremost, Cheerio’s selector implementation is nearly identical to jQuery’s. So if you know jQuery, this will be a breeze. If not, don’t worry, I’ll show you.</p>
           <h4 name="b639" id="b639" class="graf graf--h4 graf-after--p">Selectors</h4>
           <p name="b9b4" id="b9b4" class="graf graf--p graf-after--h4">The selector method allows you to traverse and select elements in the document. You can get data and set data using a selector. Imagine we have the following HTML in the website we want to scrape:</p>
           <pre name="213a" id="213a" class="graf graf--pre graf-after--p"><strong class="markup--strong markup--pre-strong">&lt;ul id="cities"&gt;<br>  &lt;li class="large"&gt;New York&lt;/li&gt;<br>  &lt;li id="medium"&gt;Portland&lt;/li&gt;<br>  &lt;li class="small"&gt;Salem&lt;/li&gt;<br>&lt;/ul&gt;</strong></pre>
           <p name="06b4" id="06b4" class="graf graf--p graf-after--pre">We can select id’s using (<code class="markup--code markup--p-code">#</code>), classes using (<code class="markup--code markup--p-code">.</code>), and elements by their tag names, ex: <code class="markup--code markup--p-code">div</code>.</p>
           <pre name="b137" id="b137" class="graf graf--pre graf-after--p">$('.large').text()<br>// New York</pre>
           <pre name="24c0" id="24c0" class="graf graf--pre graf-after--pre">$('#medium').text()<br>// Portland</pre>
           <pre name="aeab" id="aeab" class="graf graf--pre graf-after--pre">$('li[class=small]').html()<br>// &lt;li class="small"&gt;Salem&lt;/li&gt;</pre>
           <h4 name="b0ae" id="b0ae" class="graf graf--h4 graf-after--pre">Looping</h4>
           <p name="88db" id="88db" class="graf graf--p graf-after--h4">Just like jQuery, we can also iterate through multiple elements with the <code class="markup--code markup--p-code">each()</code> function. Using the same HTML code as above, we can return the inner text of each <code class="markup--code markup--p-code">li</code> with the following code:</p>
           <pre name="ca2d" id="ca2d" class="graf graf--pre graf-after--p">$('li').each(function(i, elem) {<br>  cities[i] = $(this).text();<br>});</pre>
           <pre name="3313" id="3313" class="graf graf--pre graf-after--pre">// New York Portland Salem</pre>
           <h4 name="6dc1" id="6dc1" class="graf graf--h4 graf-after--pre">Finding</h4>
           <p name="139f" id="139f" class="graf graf--p graf-after--h4">Imagine we have two lists on our web site:</p>
           <pre name="88dd" id="88dd" class="graf graf--pre graf-after--p"><strong class="markup--strong markup--pre-strong">&lt;ul id="cities"&gt;<br>  &lt;li class="large"&gt;New York&lt;/li&gt;<br>  &lt;li id="c-medium"&gt;Portland&lt;/li&gt;<br>  &lt;li class="small"&gt;Salem&lt;/li&gt;<br>&lt;/ul&gt;<br>&lt;ul id="towns"&gt;<br>  &lt;li class="large"&gt;Bend&lt;/li&gt;<br>  &lt;li id="t-medium"&gt;Hood River&lt;/li&gt;<br>  &lt;li class="small"&gt;Madras&lt;/li&gt;<br>&lt;/ul&gt;</strong></pre>
           <p name="b810" id="b810" class="graf graf--p graf-after--pre">We can select each list using their respective ID’s, then find the <em class="markup--em markup--p-em">small </em>city/town within each list:</p>
           <pre name="ee08" id="ee08" class="graf graf--pre graf-after--p">$('#cities').find('.small').text()<br>// Salem</pre>
           <pre name="de33" id="de33" class="graf graf--pre graf-after--pre">$('#towns').find('.small').text()<br>// Madras</pre>
           <blockquote name="53c0" id="53c0" class="graf graf--blockquote graf-after--pre">
            Finding will search all descendant DOM elements, not just immediate children as shown in this example.
           </blockquote>
           <h4 name="3adf" id="3adf" class="graf graf--h4 graf-after--blockquote">Children</h4>
           <p name="21d0" id="21d0" class="graf graf--p graf-after--h4">Children is similar to find. The difference is that children <strong class="markup--strong markup--p-strong">only </strong>searches for immediate children of the selected element.</p>
           <pre name="d68c" id="d68c" class="graf graf--pre graf-after--p">$('#cities').children('#c-medium').text();<br>// Portland</pre>
           <h4 name="6865" id="6865" class="graf graf--h4 graf-after--pre">Text &amp;&nbsp;HTML</h4>
           <p name="e549" id="e549" class="graf graf--p graf-after--h4">Up until this point, all of my examples have included the&nbsp;<code class="markup--code markup--p-code">.text()</code> function. Hopefully you’ve been able to figure out that this function is what gets the text of the selected element. You can also use&nbsp;<code class="markup--code markup--p-code">.html()</code> to return the html of the given element:</p>
           <pre name="4000" id="4000" class="graf graf--pre graf-after--p">$('.large')<strong class="markup--strong markup--pre-strong">.text()</strong><br>// Bend</pre>
           <pre name="76e6" id="76e6" class="graf graf--pre graf-after--pre">$('.large')<strong class="markup--strong markup--pre-strong">.html()</strong><br>// &lt;li class="large"&gt;Bend&lt;/li&gt;</pre>

           <h3 name="fe6c" id="fe6c" class="graf graf--h3 graf-after--p">Limitations</h3>
           <p name="7bae" id="7bae" class="graf graf--p graf-after--h3">As Jaye Speaks points out:</p>
           <blockquote name="10fe" id="10fe" class="graf graf--blockquote graf-after--p">
            MOST websites modify the DOM using JavaScript. Unfortunately Cheerio doesn’t resolve parsing a modified DOM. Dynamically generated content from procedures leveraging AJAX, client-side logic, and other async procedures are not available to Cheerio.
           </blockquote>
