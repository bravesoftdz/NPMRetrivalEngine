         <h2>Mais quel rapport entre un datepicker javascript et un scraper python ?</h2> 
         <p>Si vous ne savez ni ce quest un <em>datepicker</em> en Javascript ou un <em>scraper python</em>, je vous ai probablement déjà perdu. Pourtant, dans cet article ultra fun (ok, je suis bizarre), je vous montrerai comme jai automatisé via un scraper développé en python la recherche de dates libres dans un calendrier Javascript sur une page web :</p> 
         <ul> 
          <li>en scriptant la génération de cette page via une ligne de commande</li> 
          <li>en simulant un clic pour passer au mois suivant</li> 
          <li>et enfin en récupérant les bonnes valeurs (pour me les envoyer par email par exemple)</li> 
         </ul> 
         <h2>Commençons donc par un peu de storytelling</h2> 
         <p>Depuis quelques mois, je vais régulièrement sur un site web un peu artisanal sur lequel je fais des réservations. Pour se faire, ce site web propose une page web simple dans lequel a été intégré un composant calendrier :</p> 
         <p><a href="https://blog.zwindler.fr/wp-content/uploads/2017/05/calendrier1.png"><img class="wp-image-3043 aligncenter" src="https://blog.zwindler.fr/wp-content/uploads/2017/05/calendrier1.png" alt="" width="194" height="257"></a></p> 
         <p>Rien de particulièrement extraordinaire jusque là.</p> 
         <p>Mais (car bien sûr il y a un mais), ce site à linconvénient dêtre victime de son succès. Pour vous donner une idée, cest le genre de site web qui tombe 30 secondes après louverture des réservations à 19h00. Et que dès que ladministrateur système le relance entre 20 et 50 minutes plus tard, les réservations sont complètes en quelques minutes.</p> 
         <h2>Méthode 1 : F5</h2> 
         <p>La première méthode, probablement plébiscité par la plupart des utilisateurs, est à base de refresh frénétique du site web, jusquà ce quil devienne inopérant Une fois que le site est down, on peut continuer à appuyer sur F5 en espérant être le premier quand il reviendra.</p> 
         <p><a href="https://blog.zwindler.fr/wp-content/uploads/2017/05/F5.gif"><img class="wp-image-3044 aligncenter" src="https://blog.zwindler.fr/wp-content/uploads/2017/05/F5.gif" alt="" width="192" height="189"></a></p> 
         <p>Le souci de cette méthode, cest que je manque cruellement de patience et que je lâche laffaire assez vite. Et donc tous les mois, les plus hargneux (ou les plus chanceux) me passent toujours devant car jai loupé le moment où le site redevient accessible.</p> 
         <h2>Méthode 2 : cURL</h2> 
         <p>En bon administrateur système que je suis, je ne pouvais pas me résoudre à en rester là. Mon problème principal étant dêtre prévenu lorsque le site devenait à nouveau opérationnel, jai sorti ma boîte à outils habituelle : cURL !</p> 
         <p><b>cURL</b> (abréviation de <i>client URL request library</i>), est un outil à tout faire quand on veut jouer avec des URL sur un serveur Linux. Cest simple et efficace.</p> 
         <p>Dans le cas où le site est tombé suite à la connexion massive des internautes avides de réservations, jai donc créé un petit script basé sur ce <em>oneliner</em></p> 
         <pre class="brush: plain; title: ; notranslate" title="">
curl http://le_fqdn_du_site/le_chemin_de_la_page/ -I 2&gt;/dev/null | head -n 1
HTTP/1.1 200 OK
</pre> 
         <p>Dès que le code retour passe de 503 à 200, cest que le site web est de nouveau accessible et je peux me remettre à faire F5 en espérant que les réservations ouvrent.</p> 
         <h2>Méthode 2bis : aller plus loin avec cURL</h2> 
         <p>Bon La méthode précédente fonctionne mais ce nest pas encore ça !</p> 
         <p>Typiquement, il arrive que louverture des dates soit reportée au lendemain, quand le site est vraiment trop longtemps dans les choux. Du coup le site répond (et mon test me revoie que le site est opérationnel), mais les dates ne sont pas disponibles pour autant</p> 
         <p>Un rapide coup dil aux sources me donne linfo que je cherche. Il ny a plus quà automatiser ça.</p> 
         <figure id="attachment_3045" style="width: 640px" class="wp-caption aligncenter">
          <a href="https://blog.zwindler.fr/wp-content/uploads/2017/05/calendrier2.png"><img class="wp-image-3045 size-medium" src="https://blog.zwindler.fr/wp-content/uploads/2017/05/calendrier2-640x371.png" alt="" width="640" height="371" srcset="https://blog.zwindler.fr/wp-content/uploads/2017/05/calendrier2-640x371.png 640w, https://blog.zwindler.fr/wp-content/uploads/2017/05/calendrier2-768x446.png 768w, https://blog.zwindler.fr/wp-content/uploads/2017/05/calendrier2-1024x594.png 1024w, https://blog.zwindler.fr/wp-content/uploads/2017/05/calendrier2.png 1361w" sizes="(max-width: 640px) 100vw, 640px"></a>
          <figcaption class="wp-caption-text">
           Fonction «&nbsp;Examiner lélément&nbsp;» dans Firefox
          </figcaption>
         </figure> 
         <p>Mais évidemment ça nest pas si simple</p> 
         <p>Firefox interprète le Javascript, mais pas cURL, qui ne sait pas <a href="http://stackoverflow.com/questions/20554113/how-to-get-webcontent-that-is-loaded-by-javascript-using-curl">interpréter le Javascript (cf ce post de Stack Overflow)</a>. Le retour de ma commande contient donc le code source HTML avec le Javascript brut, mais pas les données que je recherche</p> 
         <h2>Méthode 3 : Un scraper ?</h2> 
         <p>Je dois écrire un <em>scraper</em> qui va aller simuler laction de lhumain qui va ouvrir la page dans un navigateur. Le post de Stack Overflow cite <a href="http://phantomjs.org/">PhantomJS</a> qui est probablement une bonne solution.</p> 
         <blockquote>
          <p>PhantomJS is a headless WebKit scriptable with a JavaScript API. It has <strong>fast</strong> and <strong>native</strong> support for various web standards: DOM handling, CSS selector, JSON, Canvas, and SVG.</p>
         </blockquote> 
         <p>Cependant je ne connais pas bien JS (ça fait longtemps on va dire) et je préfère capitaliser sur Python. Jai donc cherché un module Python pour régler mon problème, et jen ai trouvé 2 : un module python <strong>selenium</strong> et un plus simple appelé <strong>dryscrape</strong>.</p> 
         <p>Jai installé dryscrape sur un de mes serveurs (un CentOS 7 minimal, sans environnement graphique). La documentation de dryscrape <a href="http://dryscrape.readthedocs.io/en/latest/installation.html">est disponible ici</a> et <a href="https://dryscrape.readthedocs.io/en/stable/usage.html">ici (stable, documentation plus fournie)</a>.</p>
         <div class="mailmunch-forms-in-post-middle" style="display: none !important;"></div> 
         <pre class="brush: plain; title: ; notranslate" title="">
yum install -y qt5-qtwebkit-devel gcc-c++
sed -i 's/PATH=$PATH:$HOME\/bin/PATH=$PATH:$HOME\/bin:\/usr\/lib64\/qt5\/bin/' .bash_profile
source .bash_profile
pip install dryscrape
</pre> 
         <p>Et jai tout de suite voulu tester un bout de code sans trop chercher à comprendre.</p> 
         <pre class="brush: plain; title: ; notranslate" title="">
cat &lt;&lt; EOF &gt; test.py
import dryscrape
session = dryscrape.Session()
session.visit('http://le_fqdn_du_site/le_chemin_de_la_page/')
response = session.body()
print response
EOF
</pre> 
         <p> je me suis donc pris un bon gros Traceback :</p> 
         <pre class="brush: plain; title: ; notranslate" title="">
Traceback (most recent call last):
File "test.py", line 3, in &lt;module&gt;;
session = dryscrape.Session()
[...]raise NoX11Error("Could not connect to X server. "
webkit_server.NoX11Error: Could not connect to X server. Try calling dryscrape.start_xvfb() before creating a session.
</pre> 
         <p>Ahhhhh oui ça !</p> 
         <blockquote>
          <p>xvfb_ (necessary only if no other X server is available)</p>
         </blockquote> 
         <p>En fait il suffit dajouter un test pour exécuter dans xvfb si nécessaire.</p> 
         <h3>Un premier exemple</h3> 
         <p>Le bout de code suivant ne fait que récupérer le code source de la page et lafficher à lécran.</p> 
         <pre class="brush: plain; title: ; notranslate" title="">
cat &lt;&lt; EOF &gt; simple_scraper.py
#!/usr/bin/python
# -*- coding: utf-8 -*-
import sys
import dryscrape
if 'linux' in sys.platform:
# start xvfb in case no X is running. Make sure xvfb
# is installed, otherwise this won't work!
  dryscrape.start_xvfb()
session = dryscrape.Session()
session.visit('http://le_fqdn_du_site/le_chemin_de_la_page/')
response = session.body()
print response
EOF
</pre> 
         <p>Mais la grosse différence avec le cURL précédent, cest que dans le cas présent, <strong>ici le Javascript est interprété</strong>. Cette fois ci dans le code source retourné, je retrouve bien mon calendrier</p> 
         <p><a href="https://blog.zwindler.fr/wp-content/uploads/2017/05/datepicky.png"><img class="size-medium wp-image-3048 aligncenter" src="https://blog.zwindler.fr/wp-content/uploads/2017/05/datepicky-640x378.png" alt="" width="640" height="378" srcset="https://blog.zwindler.fr/wp-content/uploads/2017/05/datepicky-640x378.png 640w, https://blog.zwindler.fr/wp-content/uploads/2017/05/datepicky-768x454.png 768w, https://blog.zwindler.fr/wp-content/uploads/2017/05/datepicky.png 821w" sizes="(max-width: 640px) 100vw, 640px"></a></p> 
         <p>A partir de là, il ne me reste donc en théorie quà terminer décrire un petit bout de script pour extraire les informations qui mintéressent et à me notifier quand des places sont libres ! On progresse !</p> 
         <h2>Méthode 3 : Scraper comme jamais</h2> 
         <figure id="attachment_3053" style="width: 640px" class="wp-caption aligncenter">
          <a href="https://blog.zwindler.fr/wp-content/uploads/2017/05/maitre-gims-scrape-comme-jamais2.jpg"><img class="size-medium wp-image-3053" src="https://blog.zwindler.fr/wp-content/uploads/2017/05/maitre-gims-scrape-comme-jamais2-640x276.jpg" alt="" width="640" height="276" srcset="https://blog.zwindler.fr/wp-content/uploads/2017/05/maitre-gims-scrape-comme-jamais2-640x276.jpg 640w, https://blog.zwindler.fr/wp-content/uploads/2017/05/maitre-gims-scrape-comme-jamais2-768x331.jpg 768w, https://blog.zwindler.fr/wp-content/uploads/2017/05/maitre-gims-scrape-comme-jamais2-1024x441.jpg 1024w, https://blog.zwindler.fr/wp-content/uploads/2017/05/maitre-gims-scrape-comme-jamais2.jpg 1160w" sizes="(max-width: 640px) 100vw, 640px"></a>
          <figcaption class="wp-caption-text">
           Ce montage pourri dun clip de Maître Gimms avec des logos dans la main vous est offert par Zwindler
          </figcaption>
         </figure> 
         <p>Mais je nétais pas encore satisfait. Mon but est dêtre prévenu lorsque des places sont disponibles. Or, les places souvrent le 1er du mois, <strong>pour le mois suivant</strong>. En revanche, le composant Javascript datepicker a la mauvaise idée de donner les disponibilités <strong>pour le mois en cours</strong>. Mon script <strong>simple_scraper.py</strong> ne récupère donc pas les bonnes dates</p> 
         <p>Pour contourner le problème, il faut aller encore un peu plus loin dans lautomatisation des actions. Il faut simuler le clic de lutilisateur sur la flèche vers la droite, qui passe au mois suivant.</p> 
         <p>Cest possible avec <strong>dryscrape</strong>, qui propose notamment une fonction pour sélectionner des nodes XPATH et interagir avec. Pour ceux qui ne connaissent pas XPATH, vous pouvez aller sur la doc de dryscrape qui donne quelques exemples, et aussi sur <a href="https://www.w3schools.com/xml/xpath_intro.asp">le tutoriel de la W3School</a>.</p> 
         <p>Comme jai examiné le code HTML retourné par le datepicker, je sais quoi chercher :<br> <a href="https://blog.zwindler.fr/wp-content/uploads/2017/05/datepicker3.png"><img class="size-medium wp-image-3055 aligncenter" src="https://blog.zwindler.fr/wp-content/uploads/2017/05/datepicker3-640x70.png" alt="" width="640" height="70" srcset="https://blog.zwindler.fr/wp-content/uploads/2017/05/datepicker3-640x70.png 640w, https://blog.zwindler.fr/wp-content/uploads/2017/05/datepicker3-768x84.png 768w, https://blog.zwindler.fr/wp-content/uploads/2017/05/datepicker3.png 987w" sizes="(max-width: 640px) 100vw, 640px"></a></p> 
         <p>Pour changer de mois en cours, dans un premier temps jai essayé dutiliser le label «&nbsp;»&nbsp;» mais le module dryscrape ne supporte pas les caractères non ASCII ! Une solution qui fonctionne est de cliquer sur le lien qui contient <em>javascript:void(0)</em> contenu dans le div de classe <em>datepick-next </em>:</p> 
         <pre class="brush: plain; title: ; notranslate" title="">
cat &lt;&lt; EOF &gt; datepick_scraper.py
#!/usr/bin/python
# -*- coding: iso-8859-15 -*-
import sys
import dryscrape
if 'linux' in sys.platform:
  # start xvfb in case no X is running. Make sure xvfb 
  # is installed, otherwise this won't work!
  dryscrape.start_xvfb()
session = dryscrape.Session()
session.visit('http://le_fqdn_du_site/le_chemin_de_la_page/')

datepicknext = session.at_xpath('//*[@class="datepick-next"]/a')
datepicknext.click()

response = session.body()
print response
EOF
</pre> 
         <p>Après vérification, cette version du script renvoie bien le tableau avec les date de Juin (quand on est en Mai) !</p> 
         <h2>Le mot de la fin</h2> 
         <p>A partir du moment où vous en êtes là, il ny a plus vraiment de limites à ce que vous êtes capable de faire. Dans mon exemple, jaurai pu aller encore plus loin, extraire les dates disponibles, et même pourquoi pas aller jusquà remplir automatiquement le formulaire pour réserver automatiquement une date prédéfinie (voire même toutes les dates disponibles).</p> 
         <p><a href="https://blog.zwindler.fr/wp-content/uploads/2017/05/datepickx.png"><img class="size-medium wp-image-3047 aligncenter" src="https://blog.zwindler.fr/wp-content/uploads/2017/05/datepickx-640x103.png" alt="" width="640" height="103" srcset="https://blog.zwindler.fr/wp-content/uploads/2017/05/datepickx-640x103.png 640w, https://blog.zwindler.fr/wp-content/uploads/2017/05/datepickx.png 728w" sizes="(max-width: 640px) 100vw, 640px"></a></p> 
         <p>Ce nest pas mon but : je vais me contenter de menvoyer un email dès quune date se libère. Mais cest possible !</p> 
         <p>Si vous voulez explorer les possibilité de dryscrape, la documentation officielle donne quelques exemples, et notamment celui de lenvoi dun email depuis Gmail, entièrement automatisé comme si cétait un humain (!)</p> 
         <p>De quoi donner des idées, pour <em><strong>scraper comme jamais</strong></em> !</p>
