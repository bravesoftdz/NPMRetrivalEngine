             <p><a href="http://facebook.github.io/jest" target="_blank" rel="nofollow">Jest</a> is a test runner for testing Javascript code.</p> 
             <p>A test runner is a piece of software that looks for tests on your codebase and runs them. It also takes care of displaying the result on the CLI interface. And when it comes to Jest, it boasts of having <strong>very fast</strong> test execution as it runs tests in parallel.</p> 
             <h2 id="key-features"><a class="header-anchor" href="#key-features" aria-hidden="true"><i class="fa fa-link" aria-hidden="true"></i></a> Key Features</h2> 
             <ul> 
              <li><strong>Easy configuration</strong>: For trivial cases, <code>jest</code> can be run without any configuration. And if needed, Jest also provides a variety of ways to customize your tests by providing relevant parameters in <code>package.json</code> or as a separate <code>jest.config.js</code> file.</li> 
             </ul> 
             <pre><code class="language-javascript">jest --config jest.config.js
</code></pre> 
             <ul> 
              <li> <p><strong>Snapshot testing</strong>:<br> Much has been written about this kind of testing. Jest provides a way to take "snapshots" of your DOM components, which is essentially a file with the HTML of the component's render stored as a string. The snapshots are human readable and act as an indicator of any DOM change due to component code changes.</p> </li> 
              <li> <p><strong>Mocking</strong>:<br> Jest provides easy ways to auto-mock or manually mock entities. You can mock functions and modules which are irrelevant to your test. Jest also provides fake timers for you to control the <code>setTimeout</code> family of functions in tests.</p> </li> 
              <li> <p><strong>Interactive CLI</strong>:<br> Jest has a watch mode which watches for any test or relevant code changes and re-runs the respective tests. The watch has an interactive feature to it wherein it provides options to run all tests or fix snapshots while in watch mode itself. This is incredibly convenient.</p> </li> 
              <li> <p><strong>Coverage metrics</strong>: Jest provides some awesome coverage metrics right out of the box.</p> <p>It can be configured to show different levels of these metrics. Also, it is possible to cap the coverage to a threshold and bail or error out if the threshold is breached. Your tests, however, will complete very slowly if this feature is enabled.</p> </li> 
             </ul> 
             <pre><code class="language-javascript">jest --coverage
</code></pre> 
             <h3 id="setting-up-jest"><a class="header-anchor" href="#setting-up-jest" aria-hidden="true"><i class="fa fa-link" aria-hidden="true"></i></a> Setting Up Jest</h3> 
             <p>In this post, we will look at installing Jest and getting started with DOM testing a React application. Theoretically, these techniques could be used with other view frameworks like <code>deku</code>, <code>preact</code> and so on. But there may be challenges with finding proper rendering utils which are mature and feature rich.</p> 
             <p>Install <code>jest</code> for your project</p> 
             <pre><code class="language-javascript">npm install --save-dev jest
</code></pre> 
             <p>Add <code>npm test</code> script</p> 
             <p><strong>package.json</strong></p> 
             <pre><code class="language-javascript">{
    ...
    scripts: {
        ...
        test: <span class="hljs-string">'jest --watch --verbose'</span>
    }
    ...
}
</code></pre> 
             <h3 id="dom-testing"><a class="header-anchor" href="#dom-testing" aria-hidden="true"><i class="fa fa-link" aria-hidden="true"></i></a> DOM Testing</h3> 
             <p>In the following examples, we will mostly test trivial methods of general testing areas for a front-end application.</p> 
             <p>This involves:</p> 
             <ul> 
              <li>Basic DOM testing</li> 
              <li>Event simulation and testing</li> 
              <li>Window events simulation</li> 
             </ul> 
             <p>The sample code can be found at <a href="https://github.com/pksjce/jest-blog-samples" target="_blank" rel="nofollow">jest-blog-samples</a></p>

             <p>The app itself consists of a React component <code>App</code> which prints out a <code>div</code> with text "Hello jest from react".</p>

<span class="hljs-keyword">export</span> <span class="hljs-keyword">default</span> <span class="hljs-class"><span class="hljs-keyword">class</span> <span class="hljs-title">App</span> <span class="hljs-keyword">extends</span> <span class="hljs-title">Component</span> </span>{

             <p>Let's break this down.</p> 
             <ul> 
              <li>We need to import <code>react</code> (for the purpose of using <a href="https://jsx.github.io/" target="_blank" rel="nofollow">JSX</a>) and the <code>App</code> (to instantiate and test) source.</li> 
              <li><a href="https://github.com/airbnb/enzyme" target="_blank" rel="nofollow">Enzyme</a> is JavaScript test utility that helps render React components for testing. Funny enough, a lot of people think that they should pick between React and Enzyme, and this <a href="https://www.codementor.io/vijayst/tutorials/unit-testing-react-components-jest-or-enzyme-du1087lh8" target="_blank" rel="nofollow">tutorial</a> will clarify this use. But for this tutorial, we will use it to <code>shallow render</code> our <code>App</code> component and inspect the resulting tree.</li> 
              <li>We describe a test suite named "The main app."</li> 
              <li>Our first test is named "the app should have text."</li> 
              <li>We <a href="https://github.com/airbnb/enzyme/blob/master/docs/api/shallow.md" target="_blank" rel="nofollow">shallow render</a> our component and store it in a <code>const</code>.</li> 
              <li>Add an <code>expect</code> assertion for the app to contain the expected <code>div</code>.</li> 
             </ul>

             <h4 id="test-an-event-execution"><a class="header-anchor" href="#test-an-event-execution" aria-hidden="true"><i class="fa fa-link" aria-hidden="true"></i></a> Test an event execution</h4> 
             <p>Using Enzyme, we can simulate events in our rendered components.</p> 
             <p>We intend to build an App Component that is clickable. It should change the text displayed after the body is clicked on.</p> 
             <p>To test this component, we need to:</p> 
             <ul> 
              <li>Test for the default content on the first load.</li> 
              <li>Test for the first toggle of click and expect the change in text.</li> 
              <li>Test for the second toggle of click and again expect the change in text from the default.</li> 
             </ul> 
             <p><a href="https://github.com/pksjce/jest-blog-samples/blob/master/event-testing/__tests__/index.test.js" target="_blank" rel="nofollow"><strong>tests</strong>/index.test.js</a></p> 
             <pre><code class="language-javascript"><span class="hljs-keyword">import</span> React <span class="hljs-keyword">from</span> <span class="hljs-string">'react'</span>;
<span class="hljs-keyword">import</span> App, {DEFAULT_TEXT, CLICKED_TEXT} <span class="hljs-keyword">from</span> <span class="hljs-string">'../index'</span>;
<span class="hljs-keyword">import</span> {shallow} <span class="hljs-keyword">from</span> <span class="hljs-string">'enzyme'</span>;

describe(<span class="hljs-string">'The main app'</span>, () =&gt; {
    <span class="hljs-keyword">let</span> app;
    beforeEach(<span class="hljs-function"><span class="hljs-params">()</span> =&gt;</span> {
        app  = shallow(<span class="xml"><span class="hljs-tag">&lt;<span class="hljs-name">App</span>/&gt;</span></span>);
    })

    it(<span class="hljs-string">'the app should have text'</span>, () =&gt; {
        expect(app.text()).toBe(DEFAULT_TEXT);
    })
    it (<span class="hljs-string">'should change text on click'</span>, () =&gt; {
        app.simulate(<span class="hljs-string">'click'</span>);
        expect(app.text()).toBe(CLICKED_TEXT);
        app.simulate(<span class="hljs-string">'click'</span>);
        expect(app.text()).toBe(DEFAULT_TEXT);

})
</code></pre> 
             <p>Jest provides us with the following <a href="http://facebook.github.io/jest/docs/api.html" target="_blank" rel="nofollow">global testing methods</a></p> 
             <pre><code class="language-javascript">describe(name,fn), it(name, fn), test(name, fn)
</code></pre> 
             <p>These are the basic tools in a JavaScript testing environment's <a href="https://en.wikipedia.org/wiki/Arsenal_F.C." target="_blank" rel="nofollow">arsenal</a>.</p> 
             <ul> 
              <li><code>describe</code> is used to group a set of related tests together into a <strong>test suite</strong>.</li> 
              <li><code>it</code> or <code>test</code> is used to write your actual test where you can build an independent environment and test particular units or behavior.</li> 
             </ul> 
             <pre><code class="language-javascript">afterEach(fn)/beforeEach(fn)/afterAll(fn)/beforeAll(fn)
</code></pre> 
             <p>These functions run before/after each test or before/after the test suite. You can group all your initialization and tear down the code in these hooks which are called during a test cycle.</p> 
             <p>Keeping the above info in context, we write a <code>beforeEach</code> function which creates a shallow rendering of the Button <code>App</code>.<br> After this, we write our tests for each of the behavior we expect out of the component. Enzyme has a <a href="http://airbnb.io/enzyme/docs/api/shallow.html" target="_blank" rel="nofollow">bunch of helper APIs</a> to inspect a rendered object. We can use these to check rendered DOM for desired changes.</p> 
             <p>The corresponding React component would look like this:</p> 
             <p><a href="https://github.com/pksjce/jest-blog-samples/blob/master/event-testing/index.js" target="_blank" rel="nofollow"><strong>index.js</strong></a></p> 
             <pre><code class="language-javascript"><span class="hljs-keyword">import</span> React, {Component} <span class="hljs-keyword">from</span> <span class="hljs-string">'react'</span>;

<span class="hljs-keyword">export</span> <span class="hljs-keyword">default</span> <span class="hljs-class"><span class="hljs-keyword">class</span> <span class="hljs-title">App</span> <span class="hljs-keyword">extends</span> <span class="hljs-title">Component</span> </span>{
    <span class="hljs-keyword">constructor</span>() {
        <span class="hljs-keyword">super</span>();

    }
    render() {
        <span class="hljs-keyword">return</span> <span class="xml"><span class="hljs-tag">&lt;<span class="hljs-name">div</span> <span class="hljs-attr">onClick</span>=<span class="hljs-string">{this.handleClick}</span>&gt;</span>{this.state.clicked ? CLICKED_TEXT : DEFAULT_TEXT}<span class="hljs-tag">&lt;/<span class="hljs-name">div</span>&gt;</span></span>;
    }
    handleClick() {
        <span class="hljs-keyword">this</span>.setState({
            <span class="hljs-attr">clicked</span>: !<span class="hljs-keyword">this</span>.state.clicked

}

render(<span class="xml"><span class="hljs-tag">&lt;<span class="hljs-name">App</span>/&gt;</span></span>, <span class="hljs-built_in">document</span>.body)
</code></pre> 
             <h4 id="window-event-testing"><a class="header-anchor" href="#window-event-testing" aria-hidden="true"><i class="fa fa-link" aria-hidden="true"></i></a> <code>window</code> event testing</h4> 
             <p>Many times, our apps have the functionality to respond to outer stimuli like <code>window</code> events. We would like to test if our app behaves correctly under these conditions.<br> For this case, we will have to mock the <code>document</code> and <code>document.window</code> object.</p> 
             <p>We will use <a href="https://github.com/tmpvar/jsdom#for-the-hardcore-jsdomjsdom" target="_blank" rel="nofollow">JSDOM</a> API which provides a way of faking <code>document</code> and <code>window</code> objects.</p> 
             <pre><code class="language-bash">npm install --save-dev jsdom
</code></pre> 
             <p>We are now going to use JSDOM to wire up our fake objects. Since this activity needs to happen before every test as any of them could want the window object, we will write this code at a <code>setupFile</code> which can be configured in Jest.</p> 
             <p>Go to your <code>package.json</code> or <code>jest.config.js</code> file and add the following <a href="http://facebook.github.io/jest/docs/configuration.html#setupfiles-array" target="_blank" rel="nofollow">option</a></p> 
             <p><a href="https://github.com/pksjce/jest-blog-samples/blob/master/window-testing/package.json" target="_blank" rel="nofollow"><strong>package.json</strong></a></p> 
             <pre><code class="language-json">jest: {
    "setupFiles" : ["&lt;rootDir&gt;/setupFile.js"]
}
</code></pre> 
             <p><a href="https://github.com/pksjce/jest-blog-samples/blob/master/window-testing/setupFile.js" target="_blank" rel="nofollow"><strong>setupFile.js</strong></a></p> 
             <pre><code class="language-javascript"><span class="hljs-keyword">import</span> {jsdom} <span class="hljs-keyword">from</span> <span class="hljs-string">'jsdom'</span>;

<span class="hljs-keyword">const</span> documentHTML = <span class="hljs-string">'&lt;!doctype html&gt;&lt;html&gt;&lt;body&gt;&lt;div id="root"&gt;&lt;/div&gt;&lt;/body&gt;&lt;/html&gt;'</span>;
global.document = jsdom(documentHTML);
global.window = <span class="hljs-built_in">document</span>.parentWindow;

global.window.resizeTo = <span class="hljs-function">(<span class="hljs-params">width, height</span>) =&gt;</span> {
  global.window.innerWidth = width || global.window.innerWidth;
  global.window.innerHeight = width || global.window.innerHeight;
  global.window.dispatchEvent(<span class="hljs-keyword">new</span> Event(<span class="hljs-string">'resize'</span>));
};

</code></pre> 
             <p>Note that the above code also gives us a <code>div</code> element with the ID <code>root</code>. We can now <code>mount</code> our react component onto this element for inclusion in the DOM.</p> 
             <p>Also, <code>resizeTo</code> will not be defined by default in this fake <code>window</code> object. It will return <code>undefined</code>. Hence, we need to mock up the whole function as well. You will need to do this to trigger any event on the <code>window</code> from your test.<br> Currently, I'm only bothered with the <code>innerWidth</code> and <code>innerHeight</code> parameters. Hence, I will only update them as part of the mock.</p> 
             <p>Our App code for this test is a React component that responds to <code>window</code>'s resize event. If the resize results in the <code>innerWidth &gt; 1024</code> then it changes its <code>width</code> to <code>350</code>. If not <code>width = 300</code>.</p> 
             <p><a href="https://github.com/pksjce/jest-blog-samples/blob/master/window-testing/src/App.js" target="_blank" rel="nofollow"><strong>src/App.js</strong></a></p> 
             <pre><code class="language-javascript">
<span class="hljs-keyword">import</span> React, {Component} <span class="hljs-keyword">from</span> <span class="hljs-string">'react'</span>;

<span class="hljs-keyword">export</span> <span class="hljs-keyword">const</span> WIDTH_ABOVE_1024 = <span class="hljs-number">350</span>;
<span class="hljs-keyword">export</span> <span class="hljs-keyword">const</span> WIDTH_BELOW_1024 = <span class="hljs-number">300</span>;
<span class="hljs-keyword">export</span> <span class="hljs-keyword">const</span> THRESHOLD_WIDTH = <span class="hljs-number">1024</span>;

<span class="hljs-keyword">export</span> <span class="hljs-keyword">default</span> <span class="hljs-class"><span class="hljs-keyword">class</span> <span class="hljs-title">App</span> <span class="hljs-keyword">extends</span> <span class="hljs-title">Component</span> </span>{
    <span class="hljs-keyword">constructor</span>() {
        <span class="hljs-keyword">super</span>();
        <span class="hljs-keyword">this</span>.state = {
            <span class="hljs-attr">width</span>: <span class="hljs-built_in">window</span>.innerWidth &gt; THRESHOLD_WIDTH ? WIDTH_ABOVE_1024: WIDTH_BELOW_1024
        }
        <span class="hljs-keyword">this</span>.handleResize = <span class="hljs-keyword">this</span>.handleResize.bind(<span class="hljs-keyword">this</span>);
    }
    componentDidMount() {
        <span class="hljs-built_in">window</span>.addEventListener(<span class="hljs-string">'resize'</span>, <span class="hljs-keyword">this</span>.handleResize)
    }
    componentWillUnmount() {
        <span class="hljs-built_in">window</span>.removeEventListener(<span class="hljs-string">'resize'</span>, <span class="hljs-keyword">this</span>.handleResize)
    }
    render() {
        <span class="hljs-keyword">return</span> <span class="xml"><span class="hljs-tag">&lt;<span class="hljs-name">div</span> <span class="hljs-attr">style</span>=<span class="hljs-string">{{width:this.state.width}}</span>&gt;</span>My width is {this.state.width}<span class="hljs-tag">&lt;/<span class="hljs-name">div</span>&gt;</span></span>;
    }
    handleResize() {
        <span class="hljs-keyword">this</span>.setState({
            <span class="hljs-attr">width</span>: <span class="hljs-built_in">window</span>.innerWidth &gt; THRESHOLD_WIDTH ? WIDTH_ABOVE_1024 : WIDTH_BELOW_1024
        })
    }
}
</code></pre> 
             <p>For the above component, we again need to test if:</p> 
             <ul> 
              <li>Default behaviour.</li> 
              <li>Resize to <code>width &gt; 1024</code></li> 
              <li>Resize to <code>width &lt;=1024</code></li> 
             </ul> 
             <p><a href="https://github.com/pksjce/jest-blog-samples/blob/master/window-testing/src/App.test.js" target="_blank" rel="nofollow"><strong>src/App.test.js</strong></a></p> 
             <pre><code class="language-javascript"><span class="hljs-keyword">import</span> React <span class="hljs-keyword">from</span> <span class="hljs-string">'react'</span>;
<span class="hljs-keyword">import</span> App, {WIDTH_ABOVE_1024, WIDTH_BELOW_1024, THRESHOLD_WIDTH} <span class="hljs-keyword">from</span> <span class="hljs-string">'./App'</span>;
<span class="hljs-keyword">import</span> {mount} <span class="hljs-keyword">from</span> <span class="hljs-string">'enzyme'</span>;

describe(<span class="hljs-string">'The main app'</span>, () =&gt; {
    <span class="hljs-keyword">let</span> app;
    beforeEach(<span class="hljs-function"><span class="hljs-params">()</span> =&gt;</span> {
        app = mount(<span class="xml"><span class="hljs-tag">&lt;<span class="hljs-name">App</span>/&gt;</span></span>, {<span class="hljs-attr">attachTo</span>: <span class="hljs-built_in">document</span>.getElementById(<span class="hljs-string">'root'</span>)});
    })

    it(<span class="hljs-string">'the app should have text'</span>, () =&gt; {
        <span class="hljs-keyword">var</span> width = <span class="hljs-built_in">window</span>.innerWidth &gt; THRESHOLD_WIDTH ? WIDTH_ABOVE_1024 : WIDTH_BELOW_1024;
        expect(app.text()).toBe(<span class="hljs-string">"My width is "</span> + width);
        app.unmount();
    })
    it (<span class="hljs-string">'should change text on click'</span>, () =&gt; {
        <span class="hljs-built_in">window</span>.resizeTo(<span class="hljs-number">1000</span>, <span class="hljs-number">1000</span>);
        expect(app.text()).toBe(<span class="hljs-string">"My width is "</span> + WIDTH_BELOW_1024);
        <span class="hljs-built_in">window</span>.resizeTo(<span class="hljs-number">1025</span>, <span class="hljs-number">1000</span>)
        expect(app.text()).toBe(<span class="hljs-string">"My width is "</span> + WIDTH_ABOVE_1024);
        app.unmount();
    })
})
</code></pre> 
             <p>Things to note are:</p> 
             <ul> 
              <li>We can totally use the component's constants for testing.</li> 
              <li>Once you <code>mount</code> your app using Enzyme, you will also need to <code>unmount()</code> it at the end of each test.</li> 
              <li><code>window.resizeTo</code> will trigger a change in <code>window.innerWidth</code>. The app code rerenders as per app logic and we can test for printed width.</li> 
             </ul> 
             <h3 id="key-takeaways"><a class="header-anchor" href="#key-takeaways" aria-hidden="true"><i class="fa fa-link" aria-hidden="true"></i></a> Key Takeaways</h3> 
             <ul> 
              <li>Testing is now way easier than before with Jest.</li> 
              <li>DOM testing with React needs Enzyme as a renderer util and JSDOM as a DOM helper.</li> 
              <li>Mocking can be tricky. It's fine to have hacky mocks.</li> 
             </ul> 
             <p>I hope this article has inspired you to take up testing your application with Jest and you will be motivated to add it to your dev workflow!</p>

It should change the text displayed after the body is clicked on.\r\n\r\nTo test this component, we need to:\r\n\r\n - Test for the default content on the first load.\r\n - Test for the first toggle of click and expect the change in text.\r\n - Test for the second toggle of click and again expect the change in text from the default.\r\n\r\n[__tests__/index.test.js](https://github.com/pksjce/jest-blog-samples/blob/master/event-testing/__tests__/index.test.js)\r\n\r\n```javascript\r\nimport React from 'react';\r\nimport App, {DEFAULT_TEXT, CLICKED_TEXT} from '../index';\r\nimport {shallow} from 'enzyme';\r\n\r\ndescribe('The main app', () =\u003e {\r\n    let app;\r\n    beforeEach(() =\u003e {\r\n        app  = shallow(\u003cApp/\u003e);\r\n    })\r\n\r\n    it('the app should have text', () =\u003e {\r\n        expect(app.text()).toBe(DEFAULT_TEXT);\r\n    })\r\n    it ('should change text on click', () =\u003e {\r\n        app.simulate('click');\r\n        expect(app.text()).toBe(CLICKED_TEXT);\r\n        app.simulate('click');\r\n        expect(app.text()).toBe(DEFAULT_TEXT);\r\n    })\r\n})\r\n```\r\n\r\nJest provides us with the following [global testing methods](http://facebook.github.io/jest/docs/api.html)\r\n\r\n```javascript\r\ndescribe(name,fn), it(name, fn), test(name, fn)\r\n```\r\n\r\nThese are the basic tools in a JavaScript testing environment's [arsenal](https://en.wikipedia.org/wiki/Arsenal_F.C.).\r\n\r\n- `describe` is used to group a set of related tests together into a __test suite__.\r\n- `it` or `test` is used to write your actual test where you can build an independent environment and test particular units or behavior.\r\n\r\n```javascript\r\nafterEach(fn)/beforeEach(fn)/afterAll(fn)/beforeAll(fn)\r\n```\r\n\r\nThese functions run before/after each test or before/after the test suite. We can use these to check rendered DOM for desired changes.\r\n\r\nThe corresponding React component would look like this:\r\n\r\n[__index.js__](https://github.com/pksjce/jest-blog-samples/blob/master/event-testing/index.js)\r\n\r\n```javascript\r\nimport React, {Component} from 'react';\r\nimport {render} from 'react-dom';\r\n\r\nexport const DEFAULT_TEXT = \"Hello jest from react\";\r\nexport const CLICKED_TEXT = \"This has been clicked\";\r\n\r\nexport default class App extends Component {\r\n    constructor() {\r\n        super();\r\n        this.state = {\r\n            clicked: false\r\n        }\r\n        this.handleClick = this.handleClick.bind(this);\r\n    }\r\n    render() {\r\n        return \u003cdiv onClick={this.handleClick}\u003e{this.state.clicked ? We would like to test if our app behaves correctly under these conditions.\r\nFor this case, we will have to mock the `document` and `document.window` object.\r\n\r\nWe will use [JSDOM](https://github.com/tmpvar/jsdom#for-the-hardcore-jsdomjsdom) API which provides a way of faking `document` and `window` objects.\r\n\r\n```bash\r\nnpm install --save-dev jsdom\r\n```\r\n\r\nWe are now going to use JSDOM to wire up our fake objects. Since this activity needs to happen before every test as any of them could want the window object, we will write this code at a `setupFile` which can be configured in Jest.\r\n\r\nGo to your `package.json` or `jest.config.js` file and add the following [option](http://facebook.github.io/jest/docs/configuration.html#setupfiles-array)\r\n\r\n[__package.json__](https://github.com/pksjce/jest-blog-samples/blob/master/window-testing/package.json)\r\n\r\n```json\r\njest: {\r\n    \"setupFiles\" : [\"\u003crootDir\u003e/setupFile.js\"]\r\n}\r\n```\r\n\r\n[__setupFile.js__](https://github.com/pksjce/jest-blog-samples/blob/master/window-testing/setupFile.js)\r\n\r\n```javascript\r\nimport {jsdom} from 'jsdom';\r\n\r\nconst documentHTML = '\u003c!doctype html\u003e\u003chtml\u003e\u003cbody\u003e\u003cdiv id=\"root\"\u003e\u003c/div\u003e\u003c/body\u003e\u003c/html\u003e';\r\nglobal.document = jsdom(documentHTML);\r\nglobal.window = document.parentWindow;\r\n\r\nglobal.window.resizeTo = (width, height) =\u003e {\r\n\tglobal.window.innerWidth = width || global.window.innerWidth;\r\n\tglobal.window.innerHeight = width || global.window.innerHeight;\r\n\tglobal.window.dispatchEvent(new Event('resize'));\r\n};\r\n\r\n```\r\n\r\nNote that the above code also gives us a `div` element with the ID `root`. We can now `mount` our react component onto this element for inclusion in the DOM.\r\n\r\nAlso, `resizeTo` will not be defined by default in this fake `window` object. It will return `undefined`. Hence, we need to mock up the whole function as well. You will need to do this to trigger any event on the `window` from your test.\r\nCurrently, I'm only bothered with the `innerWidth` and `innerHeight` parameters. Hence, I will only update them as part of the mock.\r\n\r\nOur App code for this test is a React component that responds to `window`'s resize event. If the resize results in the `innerWidth \u003e 1024` then it changes its `width` to `350`. If not `width = 300`.\r\n\r\n[__src/App.js__](https://github.com/pksjce/jest-blog-samples/blob/master/window-testing/src/App.js)\r\n\r\n```javascript\r\n\r\nimport React, {Component} from 'react';\r\n\r\nexport const WIDTH_ABOVE_1024 = 350;\r\nexport const WIDTH_BELOW_1024 = 300;\r\nexport const THRESHOLD_WIDTH = 1024;\r\n\r\nexport default class App extends Component {\r\n    constructor() {\r\n        super();\r\n        this.state = {\r\n            width: window.innerWidth \u003e THRESHOLD_WIDTH ? WIDTH_ABOVE_1024 : WIDTH_BELOW_1024\r\n        })\r\n    }\r\n}\r\n```\r\n\r\nFor the above component, we again need to test if:\r\n\r\n - Default behaviour.\r\n - Resize to `width \u003e 1024`\r\n - Resize to `width \u003c=1024`\r\n\r\n [__src/App.test.js__](https://github.com/pksjce/jest-blog-samples/blob/master/window-testing/src/App.test.js)\r\n\r\n```javascript\r\nimport React from 'react';\r\nimport App, {WIDTH_ABOVE_1024, WIDTH_BELOW_1024, THRESHOLD_WIDTH} from './App';\r\nimport {mount} from 'enzyme';\r\n\r\ndescribe('The main app', () =\u003e {\r\n    let app;\r\n    beforeEach(() =\u003e {\r\n        app = mount(\u003cApp/\u003e, {attachTo: document.getElementById('root')});\r\n    })\r\n\r\n    it('the app should have text', () =\u003e {\r\n        var width = window.innerWidth \u003e THRESHOLD_WIDTH ?
