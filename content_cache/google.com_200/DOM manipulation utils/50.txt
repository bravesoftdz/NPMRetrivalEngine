           <h1 name="3273" id="3273" class="graf graf--h3 graf--leading graf--title">Into the Great Unknown?—?Migrating from Mocha to&nbsp;Jest</h1>
           <h2 name="b569" id="b569" class="graf graf--h4 graf-after--h3 graf--subtitle">A Tale of Adventure, Ambition and JavaScript Unit&nbsp;Tests</h2>

          <div class="section-inner sectionLayout--insetColumn">
           <p name="7422" id="7422" class="graf graf--p graf-after--figure">Facebook’s unit test framework <a href="https://facebook.github.io/jest/" data-href="https://facebook.github.io/jest/" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Jest</a> has a lot going for it: it makes writing tests for React components a breeze with its <a href="https://facebook.github.io/jest/docs/snapshot-testing.html" data-href="https://facebook.github.io/jest/docs/snapshot-testing.html" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">snapshot testing feature</a>, it runs tests very quickly through parallel test execution and it is very easy to set up.</p>
           <p name="fc53" id="fc53" class="graf graf--p graf-after--p">Inspired by <a href="http://2017.jsconf.eu/speakers/chris-pojer-building-high-quality-javascript-tools.html" data-href="http://2017.jsconf.eu/speakers/chris-pojer-building-high-quality-javascript-tools.html" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Chris Pojer’s talk at JSConf.eu 2017</a>, I decided to overcome my unease about relying on yet another tool from the big company from Menlo Park and give it a shot.</p>
           <p name="ea19" id="ea19" class="graf graf--p graf-after--p">When I started my current project “Polly” earlier this year, that unease led me to rely on proven technology we’d been using for years: <a href="https://mochajs.org" data-href="https://mochajs.org" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Mocha</a>, and the various tools that are usually used in combination with it.</p>
           <p name="c12d" id="c12d" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">How to migrate an existing code base to Jest? Let’s put on our sun helmets, zip up our sturdy boots and prepare to venture into the great unknown!</strong></p>
           <h3 name="147b" id="147b" class="graf graf--h3 graf-after--p">Travel Preparations</h3>
           <p name="240e" id="240e" class="graf graf--p graf-after--h3"><a href="https://facebook.github.io/jest/docs/getting-started.html" data-href="https://facebook.github.io/jest/docs/getting-started.html" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Getting started with Jest</a> is easy?—?developer experience is very important to Facebook, and it shows. Jest follows the <a href="https://en.wikipedia.org/wiki/Convention_over_configuration" data-href="https://en.wikipedia.org/wiki/Convention_over_configuration" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">convention over configuration</a> paradigm and requires you to set up very little. Installing the <a href="https://www.npmjs.com/package/jest" data-href="https://www.npmjs.com/package/jest" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">jest npm package</a>, changing the “test“ command of our project’s <em class="markup--em markup--p-em">package.json</em> to simply “jest” and creating a “Hello World” test was a matter of minutes, almost seconds.</p>
