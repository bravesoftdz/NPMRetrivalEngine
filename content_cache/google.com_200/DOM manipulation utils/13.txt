             <div class="post-text" itemprop="text"> 
              <p>Yes you can, using a library I created called nodeQuery <a href="https://github.com/tblobaum/nodeQuery" rel="noreferrer">https://github.com/tblobaum/nodeQuery</a></p> 
              <pre><code>var Express = require('express')
    , dnode = require('dnode')
    , nQuery = require('nodeQuery')
    , express = Express.createServer();

var app = function ($) {
    $.on('ready', function () {
        // do some stuff to the dom in real-time
        $('body').append('Hello World');
        $('body').append('&lt;input type="text" /&gt;');
        $('input').live('click', function () {
            console.log('input clicked');
            // ...
        });
    });
};

nQuery
    .use(app);

express
    .use(nQuery.middleware)
    .use(Express.static(__dirname + '/public'))
    .listen(3000);

dnode(nQuery.middleware).listen(express);
</code></pre>

              <h1>A simple crawler using Cheerio</h1> 
              <p>This is my formula to make a simple crawler in Node.js. It is the main reason for wanting to do DOM manipulation on the server side and probably it's the reason why you got here.</p> 
              <p>First, use <a href="//github.com/request/request" rel="noreferrer"><code>request</code></a> to download the page to be parsed. When the download is complete, handle it to <a href="//github.com/cheeriojs/cheerio" rel="noreferrer"><code>cheerio</code></a> and begin DOM manipulation just like using jQuery.</p> 
              <p>Working example:</p> 
              <pre class="lang-js prettyprint-override"><code>var
    request = require('request'),
    cheerio = require('cheerio');

function parse(url) {
    request(url, function (error, response, body) {
        var
            $ = cheerio.load(body);

        $('.question-summary .question-hyperlink').each(function () {
            console.info($(this).text());
        });
    })
}

parse('http://stackoverflow.com/');
</code></pre> 
              <p>This example will print to the console all top questions showing on SO home page. This is why I love Node.js and its community. It couldn't get easier than that :-)</p> 
              <p>Install dependencies:</p> 
              <blockquote> 
               <p>npm install request cheerio</p> 
              </blockquote> 
              <p>And run (assuming the script above is in file <code>crawler.js</code>):</p> 
              <blockquote> 
               <p>node crawler.js</p> 
              </blockquote> 
              <hr> 
              <h2>Encoding</h2> 
              <p>Some pages will have non-english content in a certain encoding and you will need to decode it to <code>UTF-8</code>. For instance, a page in brazilian portuguese (or any other language of latin origin) will likely be encoded in <code>ISO-8859-1</code> (a.k.a. "latin1"). When decoding is needed, I tell <code>request</code> not to interpret the content in any way and instead use <a href="//github.com/ashtuchkin/iconv-lite" rel="noreferrer"><code>iconv-lite</code></a> to do the job.</p> 
              <p>Working example:</p> 
              <pre class="lang-js prettyprint-override"><code>var
    request = require('request'),
    iconv = require('iconv-lite'),
    cheerio = require('cheerio');

var
    PAGE_ENCODING = 'utf-8'; // change to match page encoding

function parse(url) {
    request({
        url: url,
        encoding: null  // do not interpret content yet
    }, function (error, response, body) {
        var
            $ = cheerio.load(iconv.decode(body, PAGE_ENCODING));

        $('.question-summary .question-hyperlink').each(function () {
            console.info($(this).text());
        });
    })
}

parse('http://stackoverflow.com/');
</code></pre> 
              <p>Before running, install dependencies:</p> 
              <blockquote> 
               <p>npm install request iconv-lite cheerio</p> 
              </blockquote> 
              <p>And then finally:</p> 
              <blockquote> 
               <p>node crawler.js</p> 
              </blockquote> 
              <hr> 
              <h2>Following links</h2> 
              <p>The next step would be to follow links. Say you want to list all posters from each top question on SO. You have to first list all top questions (example above) and then enter each link, parsing each question's page to get the list of involved users.</p> 
              <p>When you start following links, a <a href="https://strongloop.com/strongblog/node-js-callback-hell-promises-generators/" rel="noreferrer">callback hell</a> can begin. To avoid that, you should use some kind of promises, futures or whatever. I always keep <a href="https://github.com/caolan/async" rel="noreferrer">async</a> in my toolbelt. So, here is a full example of a crawler using async:</p> 
              <pre class="lang-js prettyprint-override"><code>var
    url = require('url'),
    request = require('request'),
    async = require('async'),
    cheerio = require('cheerio');

var
    baseUrl = 'http://stackoverflow.com/';

// Gets a page and returns a callback with a $ object
function getPage(url, parseFn) {
    request({
        url: url
    }, function (error, response, body) {
        parseFn(cheerio.load(body))
    });
}

getPage(baseUrl, function ($) {
    var
        questions;

    // Get list of questions
    questions = $('.question-summary .question-hyperlink').map(function () {
        return {
            title: $(this).text(),
            url: url.resolve(baseUrl, $(this).attr('href'))
        };
    }).get().slice(0, 5); // limit to the top 5 questions

    // For each question
    async.map(questions, function (question, questionDone) {

        getPage(question.url, function ($$) {

            // Get list of users
            question.users = $$('.post-signature .user-details a').map(function () {
                return $$(this).text();
            }).get();

            questionDone(null, question);
        });

    }, function (err, questionsWithPosters) {

        // This function is called by async when all questions have been parsed

        questionsWithPosters.forEach(function (question) {

            // Prints each question along with its user list
            console.info(question.title);
            question.users.forEach(function (user) {
                console.info('\t%s', user);
            });
        });
    });
});
</code></pre> 
              <p>Before running:</p> 
              <blockquote> 
               <p>npm install request async cheerio</p> 
              </blockquote> 
              <p>Run a test:</p> 
              <blockquote> 
               <p>node crawler.js</p> 
              </blockquote> 
              <p>Sample output:</p> 
              <pre class="lang-none prettyprint-override"><code>Is it possible to pause a Docker image build?
    conradk
    Thomasleveil
PHP Image Crop Issue
    Elyor
    Houston Molinar
Add two object in rails
    user1670773
    Makoto
    max
Asymmetric encryption discrepancy - Android vs Java
    Cookie Monster
    Wand Maker
Objective-C: Adding 10 seconds to timer in SpriteKit
    Christian K Rider
</code></pre> 
              <p>And that's the basic you should know to start making your own crawlers :-)</p>

http.get(options, function(res) {
res.on('data', function(data) {
    // collect the data chunks to the variable named "html"
    html += data;
}).on('end', function() {
    // the whole of webpage data has been collected. parsing time!

 });

      });
