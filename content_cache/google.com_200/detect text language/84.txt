      <h1>langdetect 1.0.7</h1> 
      <div id="download-button"> 
       <a class="button green" style="float:right;" href="https://pypi.python.org/packages/59/59/4bc44158a767a6d66de18c4136c8aa90491d56cc951c10b74dd1e13213c9/langdetect-1.0.7.zip">Download<br> <span style="font-size: 75%">langdetect-1.0.7.zip</span> </a> 
      </div> 
      <p style="font-style: italic">Language detection library ported from Google's language-detection.</p> 
      <div id="langdetect"> 
       <h2>langdetect</h2> 
       <p>Port of Google’s [language-detection](<a href="https://code.google.com/p/language-detection/" rel="nofollow">https://code.google.com/p/language-detection/</a>) library (version from 03/03/2014) to Python.</p> 
      </div> 
      <div id="installation"> 
       <h2>Installation</h2> 
       <blockquote>
         $ pip install langdetect
       </blockquote> 
       <p>Supported Python versions 2.6, 2.7, 3.x.</p> 
      </div> 
      <div id="languages"> 
       <h2>Languages</h2> 
       <p><tt>langdetect</tt> supports 55 languages out of the box ([ISO 639-1 codes](<a href="https://en.wikipedia.org/wiki/List_of_ISO_639-1_codes" rel="nofollow">https://en.wikipedia.org/wiki/List_of_ISO_639-1_codes</a>)):</p> 
       <blockquote>
         af, ar, bg, bn, ca, cs, cy, da, de, el, en, es, et, fa, fi, fr, gu, he, hi, hr, hu, id, it, ja, kn, ko, lt, lv, mk, ml, mr, ne, nl, no, pa, pl, pt, ro, ru, sk, sl, so, sq, sv, sw, ta, te, th, tl, tr, uk, ur, vi, zh-cn, zh-tw
       </blockquote> 
      </div> 
      <div id="basic-usage"> 
       <h2>Basic usage</h2> 
       <p>To detect the language of the text:</p> 
       <p><tt>`python &gt;&gt;&gt; from langdetect import detect &gt;&gt;&gt; <span class="pre">detect("War</span> doesn't show who's right, just who's <span class="pre">left.")</span> 'en' &gt;&gt;&gt; <span class="pre">detect("Ein,</span> zwei, drei, vier") 'de' `</tt></p> 
       <p>To find out the probabilities for the top languages:</p> 
       <p><tt>`python &gt;&gt;&gt; from langdetect import detect_langs &gt;&gt;&gt; <span class="pre">detect_langs("Otec</span> matka <span class="pre">syn.")</span> [sk:0.572770823327, pl:0.292872522702, cs:0.134356653968] `</tt></p> 
       <p><strong>NOTE</strong></p> 
       <p>Language detection algorithm is non-deterministic, which means that if you try to run it on a text which is either too short or too ambiguous, you might get different results everytime you run it.</p> 
       <p>To enforce consistent results, call following code before the first language detection:</p> 
       <p><tt>`python from langdetect import DetectorFactory DetectorFactory.seed = 0 `</tt></p> 
      </div> 
      <div id="how-to-add-new-language"> 
       <h2>How to add new language?</h2> 
       <p>You need to create a new language profile. The easiest way to do it is to use the [langdetect.jar](<a href="https://github.com/shuyo/language-detection/raw/master/lib/langdetect.jar" rel="nofollow">https://github.com/shuyo/language-detection/raw/master/lib/langdetect.jar</a>) tool, which can generate language profiles from Wikipedia abstract database files or plain text.</p> 
       <p>Wikipedia abstract database files can be retrieved from “Wikipedia Downloads” ([<a href="http://download.wikimedia.org/](http://download.wikimedia.org/" rel="nofollow">http://download.wikimedia.org/](http://download.wikimedia.org/</a>)). They form ‘(language code)wiki-(version)-abstract.xml’ (e.g. ‘enwiki-20101004-abstract.xml’ ).</p> 
       <p>usage: <tt>java <span class="pre">-jar</span> langdetect.jar <span class="pre">--genprofile</span> <span class="pre">-d</span> [directory path] [language codes]</tt></p> 
       <ul> 
        <li>Specify the directory which has abstract databases by -d option.</li> 
        <li>This tool can handle gzip compressed file.</li> 
       </ul> 
       <p>Remark: The database filename in Chinese is like ‘zhwiki-(version)-abstract-zh-cn.xml’ or zhwiki-(version)-abstract-zh-tw.xml’, so that it must be modified ‘zh-cnwiki-(version)-abstract.xml’ or ‘zh-twwiki-(version)-abstract.xml’.</p> 
       <p>To generate language profile from a plain text, use the genprofile-text command.</p> 
       <p>usage: <tt>java <span class="pre">-jar</span> langdetect.jar <span class="pre">--genprofile-text</span> <span class="pre">-l</span> [language code] [text file path]</tt></p> 
       <p>For more details see [language-detection Wiki](<a href="https://code.google.com/archive/p/language-detection/wikis/Tools.wiki" rel="nofollow">https://code.google.com/archive/p/language-detection/wikis/Tools.wiki</a>).</p> 
      </div> 
      <div id="original-project"> 
       <h2>Original project</h2> 
       <p>This library is a direct port of Google’s [language-detection](<a href="https://code.google.com/p/language-detection/" rel="nofollow">https://code.google.com/p/language-detection/</a>) library from Java to Python. All the classes and methods are unchanged, so for more information see the project’s website or wiki.</p> 
       <p>Presentation of the language detection algorithm: [<a href="http://www.slideshare.net/shuyo/language-detection-library-for-java](http://www.slideshare.net/shuyo/language-detection-library-for-java" rel="nofollow">http://www.slideshare.net/shuyo/language-detection-library-for-java](http://www.slideshare.net/shuyo/language-detection-library-for-java</a>).</p> 
      </div>
