           <h2 class="header-scroll">
            <div class="anchor waypoint" id="section-localize-initialize"></div>Localize.initialize<a class="fa fa-anchor" href="#section-localize-initialize"></a></h2> 
           <pre><code>Localize.initialize(options);
</code></pre>
           <p><strong>Available options:</strong></p> 
           <ul> 
            <li><code>key</code> Required. Your project key. Find it here.</li>
            <li><code>targetLanguage</code> Language to translate your website to.</li>
            <li><code>rememberLanguage</code> Defaults to false. If true, Localize will translate your website to the last selected language on subsequent page views.</li>
            <li><code>fetchTranslations</code> Defaults to true. If true, translations will be fetched from Localize if not bootstrapped.</li>
            <li><code>saveNewPhrases</code> Defaults to true. If true, unrecognized phrases will be added to your Localize account. Disable this in development.</li>
            <li><code>translateAlt</code> Defaults to true. If true, "alt" attributes will be translated.</li>
            <li><code>prefetch</code> Defaults to false. Set to true to prefetch all active languages, or pass a language code or an array of codes to.</li>
            <li><code>blockedClasses</code> Array of class names for which Localize will ignore.</li>
            <li><code>translateClasses</code> Array of class names for which Localize will translate. If you use this option, Localize will <em>only</em> translate content contained in these classes and will ignore all other content in the body of the page.</li>
            <li><code>autodetectLanguage</code> Defaults to true. Automatically default the page language to the user's preferred language. The first path segment in the URL is used to check to detect the language, ie. <code>www.localize.com/fr</code>. If no language dictionary exists for that segment then the language setting in their browser is used.</li>
            <li><code>translateBody</code> Defaults to false. When true, Localize will attempt to translate the entire body of the page. If false, Localize will only translate content contained with a "localizejs" class name.</li>
            <li><code>defaultLanguage</code> The default language your website will be in when no language has been selected. Defaults to the source language of your website.</li>
            <li><code>basePath</code> The base path will be stripped from the URL of the phrase as seen in the "Filter by pages" feature.</li>
            <li><code>translateTitle</code> Defaults to true. If true, the &lt;title&gt; of the page will translate.</li>
            <li><code>translateMetaTags</code> Defaults to false. Allows users to turn on meta tag translation. This optimizes your site for SEO.</li>
            <li><code>saveNewPhrasesFromSource</code> Defaults to false. If true, Localize will detect phrases only when the page is not translated. Please contact support@localizejs.com prior to updating this option.</li>
            <li><code>retranslateOnNewPhrases</code> Defaults to false. Automatically translate content that is added dynamically to your webpage. For example, if your webpage dynamically adds html into the source of the page, our library will translate it once the translations have been generated. Behind the scenes this means the dictionary file with all your translated content is available for use with <code>Localize.translate()</code>. However, translations are not generated instantly, so use with our library event <code>updatedDictionary</code> is recommended. </li>
            <li><code>enhancedContentSecurity</code> Defaults to false. If true, the Localize library will not send additional metadata to our servers. This metadata includes the surrounding HTML of the phrases detected on your website.</li>
            <li><code>translateTimeElement</code> Defaults to false. If true, the Localize library will pick up phrases in the &lt;time&gt; elements. </li>
            <li><code>translateNumbers</code> Defaults to false. If true, the Localize library will pick up numbers as phrases.</li>
           </ul>

           <h2 class="header-scroll">
            <div class="anchor waypoint" id="section-localize-translate"></div>Localize.translate<a class="fa fa-anchor" href="#section-localize-translate"></a></h2> 
           <p>Translates text or text within html. </p> 
           <pre><code>Localize.translate(input, variables, callback)
</code></pre>
           <ul> 
            <li><code>input</code> Required. Can be text, html or native DOM elements</li>
            <li><code>variables</code> Optional. Object of variables that will be replaced in the input, if it's a string</li>
            <li><code>callback</code> Optional. Callback will trigger once translations have been fetched from Localize.</li>
           </ul> 
           <p>If the Localize.translate() input is a string, instances of <code>%{variable}</code> will be replaced with the given value in the <code>variables</code> object. You may also use HTML <code>&lt;var&gt;</code> tags in the string For example, all of the following variations are supported:</p> 
           <pre><code>Localize.translate('My name is %{name}', { name: 'Joe' }, callback);
Localize.translate('My name is &lt;var name&gt;&lt;/var&gt;', { name: 'Joe' }, callback);
Localize.translate('My name is &lt;var name&gt;Joe&lt;/var&gt;', callback);
</code></pre>
           <p>If the active language is the source language of the page, <code>Localize.translate</code> will return the untranslated phrase.</p> 
           <p><code>Localize.translate</code> can be used with or without a callback. We highly recommend using the callback approach if you're calling <code>Localize.translate</code> in the first 10 seconds of page load to ensure that the latest translations are available. The callback will allow the translation to delay until translations have been fully loaded into the browser. If the translations are already loaded, the callback is executed immediately.</p> 
           <pre><code>Localize.translate('My name is %{name}', { name: 'Joe' }, function(translation) {
  console.log(translation);
});
</code></pre>

    }
    return null;
}
