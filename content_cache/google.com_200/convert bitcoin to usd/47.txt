        <h1>bitcoin-receive-payments</h1> 
        <p>Allow your service to receive/accept bitcoin payments <strong>directly from the bitcoin P2P network</strong>.</p> 
        <p>Payment notifications are received <strong>live</strong>, directly from the bitcoin network in less than 4 seconds.</p> 
        <p>An unique bitcoin address is used for each payment, and all funds go to the same wallet, using a <a href="/redirect?url=https%3A%2F%2Fen.bitcoin.it%2Fwiki%2FDeterministic_wallet&amp;hash=ae1864abb8d12ed99ab3da7fe28a5be4">Deterministic Wallet</a> public key. </p> 
        <h2>Motivation</h2> 
        <p>This module eliminates the need of a <a href="/redirect?url=https%3A%2F%2Fblockchain.info%2Fes%2Fapi%2Fapi_receive&amp;hash=026d051a093ff50b1e08ab5d4a34e7dc">payment API</a> like the one provided by <a href="/redirect?url=https%3A%2F%2Fblockchain.info%2F&amp;hash=43c27afc0b1a8d59e8218151ad8f6ef9">blockchain.info</a> making bitcoin payments even more private and secure.</p> 
        <p>Bitcoin is designed to remove intermediaries, this module intends exactly that.</p> 
        <h2>Security</h2> 
        <p>No private key is stored in the server, so that <strong>funds cannot be stolen in case of the server being hacked</strong>.</p> 
        <h2>Backend</h2> 
        <p>This module uses <a href="/redirect?url=https%3A%2F%2Fwww.npmjs.com%2Fpackage%2Fbitcoin-live-transactions&amp;hash=6cb876937870df9026f327c836cda70b">bitcoin-live-transactions</a> to connect in real time to the <strong>Bitcoin P2P network</strong>, using <a href="/redirect?url=https%3A%2F%2Fgithub.com%2Fbitpay%2Finsight-api&amp;hash=53c9e57c88771496b62d96a74faee37e">insight</a> public instances that are running online (<a href="/redirect?url=https%3A%2F%2Fwww.npmjs.com%2Fpackage%2Fsocket.io&amp;hash=143113d3b85f75f1910383e0ab6ad535">socket.io</a>).</p> 
        <h2>Install</h2> 
        <blockquote> 
         <p>npm install bitcoin-receive-payments --save</p> 
        </blockquote> 
        <h2>Initialization</h2> 
        <pre><code class="language-javascript">const pub_key = 'xpub6CV... extended public key (xPub)'
const openexchangerates_key = 'd1c95b7b... key from openexchangerates.org' // to automatically convert USD amounts to BTC at real time rates</code></pre> 
        <p>Initialize the payment gateway inside in your nodejs app:</p> 
        <pre><code class="language-javascript">const BitcoinGateway = require('bitcoin-receive-payments')
const gateway = new BitcoinGateway(pub_key, openexchangerates_key)</code></pre> 
        <h2>Obtaining an Extended Public Key (xPub) </h2> 
        <p>xPubs can be created any bitcoin wallet that supports <a href="/redirect?url=https%3A%2F%2Fgithub.com%2Fbitcoin%2Fbips%2Fblob%2Fmaster%2Fbip-0032.mediawiki&amp;hash=482e762e2b618d4381c825cf4c0a7a62">BIT32</a> you can also <a href="/redirect?url=http%3A%2F%2Fbip32.org%2F&amp;hash=bcd87ab862babec374a3f07c08134bd7">create one online</a> but I wouldn't recommend it.</p> 
        <p>I do recommend visiting <a href="/redirect?url=http%3A%2F%2Fbip32.org%2F&amp;hash=bcd87ab862babec374a3f07c08134bd7"></a><a href="/redirect?url=http%3A%2F%2Fbip32.org%2F&amp;hash=bcd87ab862babec374a3f07c08134bd7">http://bip32.org/</a> to better understand how Deterministic Wallets work.</p> 
        <p>For example, the <strong>bitpay</strong> wallet (desktop version) automatically creates one for you, to get it go to: </p> 
        <blockquote> 
         <p><strong>Settings -&gt; Personal Wallet -&gt; More Options -&gt; Wallet Information -&gt; Copayer 0</strong></p> 
        </blockquote> 
        <p>On <strong>Copay</strong> (iOS) wallet:</p> 
        <blockquote> 
         <p><strong>Settings -&gt; Advanced -&gt; Wallet Information -&gt; Copayer 0</strong></p> 
        </blockquote> 
        <p>This public key (xPub) will not allow the gateway to use any funds in your wallet, but will allow the gateway to create as many child public addresses as needed, to receive all the payments from your customers.</p> 
        <h2>Accept payments</h2> 
        <p>Every time you want to allow the user to make you a Bitcoin payment, all you need is an unique_ID for that user in your database, and use that unique_ID to create an <strong><em>bitcoin address</em></strong> for him to pay at, for example:</p> 
        <pre><code class="language-javascript">var unique_ID = 5554555 // get this from your database

gateway.createAddress(unique_ID)
.then(function(address) {

    console.log('got new address', address.address, 'and it has', address.seconds_left / 60, 'minutes left before it expires.')

    var amount = 3.99

    console.log('ask user to pay ', amount, 'USD in it as', gateway.USDtoBIT(amount) + ' bits, using HTML, preferably as a QR code')

}).catch(function() {
    console.log('limit reached! cant get a new address :(')
})</code></pre> 
        <p>Would output:</p> 
        <pre><code>created new address 1K2xWPtGsvg5Sa2X7URZ5VfU8xS62McbXz and it has 14 minutes left before it expires.
ask user to pay 3.99 USD (3763.63610805 bits) using HTML, preferably as a QR code</code></pre> 
        <h2>Process payments</h2> 
        <p>With that address (<strong>1K2xWPtGsvg5Sa2X7URZ5VfU8xS62McbXz</strong>) you can now ask the user to make the payment, using a QR code, and a <strong><em>fancy user interface</em></strong>.</p> 
        <p>On your service backend, any payment done to that address sent to your wallet will trigger a <strong>payment</strong> event that must be handled as follows:</p> 
        <pre><code class="language-javascript">gateway.events.on('payment', function(payment) {
  console.log('got a payment!.', payment)
})</code></pre> 
        <p>So that if someone makes a payment to that address, you will receive a notification it in matter of seconds:</p> 
        <pre><code>got a payment!. 
{ address: '1K2xWPtGsvg5Sa2X7URZ5VfU8xS62McbXz',
  amount: 380600,
  id: '5554555',
  usd_amount: 4.034911868211425 }</code></pre> 
        <p>With this information you can perfectly know that the user with the unique_id <strong>5554555</strong> is the one who made the payment of the <strong>$3.99</strong> (usd_amount field) and process it accordingly (some cents difference will always happen depending on exchange rate of the moment). Preferably updating the user interface on the fly, using some real-time communitacion like <a href="/redirect?url=https%3A%2F%2Fsocket.io%2F&amp;hash=8f3ae8fd5c2ce2d2572ddb40d6dfa8aa">socket.io</a>.</p> 
        <p><strong>Note:</strong></p> 
        <p>The <strong><em>payment</em></strong> events will also be triggered at <strong>gateway.connect()</strong> if there are payments that happened when the server was reloading/restarting or just turned off. So make sure to handle the <strong><em>payment</em></strong> event, before starting the gateway with:</p> 
        <pre><code class="language-javascript">gateway.connect()</code></pre> 
        <p>The <strong>initialized</strong> event means that the gateway is ready and monitoring the bitcoin network so to detect any payment made to any of the newly created addresses generated by the gateway. </p> 
        <pre><code class="language-javascript">gateway.events.on('initialized', function() {
  console.log('gateway has been intialized.')
  // start creating addresses after the initialized event
  // gateway.createAddress(unique_id)...
});</code></pre> 
        <h2>Address expiration timer</h2> 
        <p>All newly created addresses have a 15 minutes countdown, it is a consequence of a limitation imposed by <a href="/redirect?url=https%3A%2F%2Fgithub.com%2Fbitcoin%2Fbips%2Fblob%2Fmaster%2Fbip-0044.mediawiki&amp;hash=25c5f83b662a7cca2d081ae3e71bbd2f">BIP0044</a>.</p> 
        <p><strong>It is important to show the user the time it has left to pay to a given bitcoin address, and to handle it's expiration (address renewal) appropriately, preferably showing a live timer/countdown</strong></p> 
        <h2>Timer needed because of BIP44</h2> 
        <p><a href="/redirect?url=https%3A%2F%2Fgithub.com%2Fbitcoin%2Fbips%2Fblob%2Fmaster%2Fbip-0044.mediawiki&amp;hash=25c5f83b662a7cca2d081ae3e71bbd2f">BIP0044</a> defines that when recovering a <a href="/redirect?url=https%3A%2F%2Fen.bitcoin.it%2Fwiki%2FDeterministic_wallet&amp;hash=ae1864abb8d12ed99ab3da7fe28a5be4">Deterministic Wallet</a>, all child addresses are re-created to recover all the funds, if 20 consecutive child addresses are checked and no funds are found, it finishes recovering and no more are checked, assuming there is no more funds in any more child-addresses.</p> 
        <p>In other words, regardless that a <strong>Extended Public Key (xPub)</strong> can create any number of public addresses to receive funds at the same time, if there is a gap of more than 20 addresses between an address that received funds and the next one, the wallet recovery algorithm will not be able to recover all the funds when recovering that wallet, transfering the keys to another platform, etc.</p> 
        <p>So this is solved assigning addresses an expiration date in minutes and re-using them if no payment have been received in them after that period of time, limiting the number of unused addresses created simultaneously.</p> 
        <p>The number of minutes before expiration can be modified towards improving user experience.</p> 
        <h2>Batteries included</h2> 
        <p>This module automatically takes care of all the timing and proper address reuse to comply with <a href="/redirect?url=https%3A%2F%2Fgithub.com%2Fbitcoin%2Fbips%2Fblob%2Fmaster%2Fbip-0044.mediawiki&amp;hash=25c5f83b662a7cca2d081ae3e71bbd2f">BIP0044</a>. All payments made to addresses created using this module, will reach your wallet without problem.</p>
