      <h1>ccxt 1.7.101</h1> 
      <div id="download-button"> 
       <a class="button green" style="float:right;" href="#downloads">Downloads ?</a> 
      </div> 
      <p style="font-style: italic">A library for cryptocurrency trading and e-commerce with support for many bitcoin/ether/altcoin exchange markets and merchant APIs</p> 
      <p> <strong>Latest Version: </strong><a title="Latest Version URL" href="https://pypi.python.org/pypi/ccxt/1.9.19"> <span>1.9.19</span> </a> </p> 
      <a href="https://travis-ci.org/kroitor/ccxt" rel="nofollow"><img src="https://travis-ci.org/kroitor/ccxt.svg?branch=master"></a> 
      <a href="https://npmjs.com/package/ccxt" rel="nofollow"><img src="https://img.shields.io/npm/v/ccxt.svg"> </a> 
      <a href="https://pypi.python.org/pypi/ccxt" rel="nofollow"><img src="https://img.shields.io/pypi/v/ccxt.svg"> </a> 
      <a href="https://www.npmjs.com/package/ccxt" rel="nofollow"><img src="https://img.shields.io/npm/dm/ccxt.svg"> </a> 
      <a href="https://npm.runkit.com/ccxt" rel="nofollow"><img src="https://badge.runkitcdn.com/ccxt.svg"> </a> 
      <a href="https://github.com/kroitor/ccxt/wiki/Exchange-Markets" rel="nofollow"><img src="https://img.shields.io/badge/exchanges-85-blue.svg"> </a> 
      <p>A JavaScript / Python / PHP library for cryptocurrency trading and e-commerce with support for many bitcoin/ether/altcoin exchange markets and merchant APIs.</p> 
      <p>The <strong>CCXT</strong> library is used to connect and trade with cryptocurrency / altcoin exchanges and payment processing services worldwide. It provides quick access to market data for storage, analysis, visualization, indicator development, algorithmic trading, strategy backtesting, bot programming, webshop integration and related software engineering.</p> 
      <p>It is intended to be used by <strong>coders, developers, technically-skilled traders, data-scientists and financial analysts</strong> for building trading algorithms on top of it.</p> 
      <p>Current featurelist:</p> 
      <ul> 
       <li>support for many exchange markets, even more upcoming soon</li> 
       <li>fully implemented public and private APIs for all exchanges</li> 
       <li>all currencies, altcoins and symbols, prices, order books, trades, tickers, etc…</li> 
       <li>optional normalized data for cross-exchange or cross-currency analytics and arbitrage</li> 
       <li>an out-of-the box unified all-in-one API extremely easy to integrate</li> 
       <li>works in Node 7.6+, Python 2 and 3, PHP 5.3+, web browsers</li> 
      </ul> 
      <p><a href="https://github.com/ccxt-dev/ccxt" rel="nofollow">ccxt on GitHub</a> | Install | Usage | <a href="https://github.com/ccxt-dev/ccxt/wiki" rel="nofollow">Manual</a> | <a href="https://github.com/ccxt-dev/ccxt/tree/master/examples" rel="nofollow">Examples</a> | <a href="https://github.com/ccxt-dev/ccxt/blob/master/CHANGELOG.md" rel="nofollow">Changelog</a> | <a href="https://github.com/ccxt-dev/ccxt/blob/master/CONTRIBUTING.md" rel="nofollow">Contributing</a></p>

      <div id="install"> 
       <h2>Install</h2> 
       <p>The easiest way to install the ccxt library is to use builtin package managers:</p> 
       <ul> 
        <li><a href="http://npmjs.com/package/ccxt" rel="nofollow">ccxt in **NPM**</a> (JavaScript / Node)</li> 
        <li><a href="https://pypi.python.org/pypi/ccxt" rel="nofollow">ccxt in **PyPI**</a> (Python 2 and 3)</li> 
       </ul> 
       <p>This library is shipped as an all-in-one module implementation with minimalistic dependencies and requirements:</p> 
       <ul> 
        <li><tt>`ccxt.js</tt> &lt;<a href="https://github.com/ccxt-dev/ccxt/blob/master/ccxt.js" rel="nofollow">https://github.com/ccxt-dev/ccxt/blob/master/ccxt.js</a>&gt;`__ in JavaScript</li> 
        <li><tt>`ccxt/</tt> &lt;<a href="https://github.com/ccxt-dev/ccxt/blob/master/ccxt/" rel="nofollow">https://github.com/ccxt-dev/ccxt/blob/master/ccxt/</a>&gt;`__ in Python (generated from JS)</li> 
        <li><tt>`build/ccxt.php</tt> &lt;<a href="https://github.com/ccxt-dev/ccxt/blob/master/build/ccxt.php" rel="nofollow">https://github.com/ccxt-dev/ccxt/blob/master/build/ccxt.php</a>&gt;`__ in PHP (generated from JS)</li> 
       </ul> 
       <p>You can also clone it into your project directory from <a href="https://github.com/ccxt-dev/ccxt" rel="nofollow">ccxt GitHub repository</a>:</p> 
       <pre>git clone https://github.com/ccxt-dev/ccxt.git
</pre> 
       <p>An alternative way of installing this library into your code is to copy a single file manually into your working directory with language extension appropriate for your environment.</p> 
       <div id="javascript-npm"> 
        <h3>JavaScript (NPM)</h3> 
        <p>JavaScript version of ccxt works both in Node and web browsers.</p> 
        <p><a href="http://npmjs.com/package/ccxt" rel="nofollow">ccxt crypto trading library in npm</a></p> 
        <pre>npm install ccxt
</pre> 
        <pre><span class="kd">var</span> <span class="nx">ccxt</span> <span class="o">=</span> <span class="nx">require</span> <span class="p">(</span><span class="s1">'ccxt'</span><span class="p">)</span>

<span class="nx">console</span><span class="p">.</span><span class="nx">log</span> <span class="p">(</span><span class="nx">ccxt</span><span class="p">.</span><span class="nx">exchanges</span><span class="p">)</span> <span class="c1">// print all available exchanges</span>
</pre> 
       </div> 
       <div id="javascript-for-use-with-the-script-tag"> 
        <h3>JavaScript (for use with the <tt>&lt;script&gt;</tt> tag):</h3> 
        <p><a href="https://unpkg.com/ccxt" rel="nofollow">All-in-one browser bundle</a> (dependencies included), served from <a href="https://unpkg.com/" rel="nofollow">unpkg CDN</a>, which is a fast, global content delivery network for everything on NPM.</p> 
        <pre><span class="p">&lt;</span><span class="nt">script</span> <span class="na">type</span><span class="o">=</span><span class="s">"text/javascript"</span> <span class="na">src</span><span class="o">=</span><span class="s">"https://unpkg.com/ccxt"</span><span class="p">&gt;&lt;/</span><span class="nt">script</span><span class="p">&gt;</span>
</pre> 
        <p>Creates a global <tt>ccxt</tt> object:</p> 
        <pre><span class="nx">console</span><span class="p">.</span><span class="nx">log</span> <span class="p">(</span><span class="nx">ccxt</span><span class="p">.</span><span class="nx">exchanges</span><span class="p">)</span> <span class="c1">// print all available exchanges</span>
</pre> 
       </div> 
       <div id="python"> 
        <h3>Python</h3> 
        <p><a href="https://pypi.python.org/pypi/ccxt" rel="nofollow">ccxt algotrading library in PyPI</a></p> 
        <pre>pip install ccxt
</pre> 
        <pre><span class="kn">import</span> <span class="nn">ccxt</span>
<span class="k">print</span><span class="p">(</span><span class="n">ccxt</span><span class="o">.</span><span class="n">exchanges</span><span class="p">)</span> <span class="c1"># print a list of all available exchange classes</span>

      <div id="usage"> 
       <h2>Usage</h2> 
       <div id="intro"> 
        <h3>Intro</h3> 
        <p>The ccxt library consists of a public part and a private part. Anyone can use the public part out-of-the-box immediately after installation. Public APIs open access to public information from all exchange markets without registering user accounts and without having API keys.</p> 
        <p>Public APIs include the following:</p> 
        <ul> 
         <li>market data</li> 
         <li>instruments/trading pairs</li> 
         <li>price feeds (exchange rates)</li> 
         <li>order books</li> 
         <li>trade history</li> 
         <li>tickers</li> 
         <li>OHLC(V) for charting</li> 
         <li>other public endpoints</li> 
        </ul> 
        <p>For trading with private APIs you need to obtain API keys from/to exchange markets. It often means registering with exchanges and creating API keys with your account. Most exchanges require personal info or identification. Some kind of verification may be necessary as well. If you want to trade you need to register yourself, this library will not create accounts or API keys for you. Some exchange APIs expose interface methods for registering an account from within the code itself, but most of exchanges don’t. You have to sign up and create API keys with their websites.</p> 
        <p>Private APIs allow the following:</p> 
        <ul> 
         <li>manage personal account info</li> 
         <li>query account balances</li> 
         <li>trade by making market and limit orders</li> 
         <li>deposit and withdraw fiat and crypto funds</li> 
         <li>query personal orders</li> 
         <li>get ledger history</li> 
         <li>transfer funds between accounts</li> 
         <li>use merchant services</li> 
        </ul> 
        <p>This library implements full public and private REST APIs for all exchanges. WebSocket and FIX implementations in JavaScript, PHP, Python and other languages coming soon.</p> 
        <p>The ccxt library supports both camelcase notation (preferred in JavaScript) and underscore notation (preferred in Python and PHP), therefore all methods can be called in either notation or coding style in any language.</p> 
        <pre>// both of these notations work in JavaScript/Python/PHP
exchange.methodName ()  // camelcase pseudocode
exchange.method_name () // underscore pseudocode
</pre> 
        <p>Read the <a href="https://github.com/ccxt-dev/ccxt/wiki" rel="nofollow">Manual</a> for more details.</p>
