        <p>Display value of crypto-currency hodlings. Prices fetched from <a href="https://api.coinmarketcap.com/v1/ticker/">coinmarketcap API</a>.</p> 
        <p>Sample output:</p> 
        <p><a href="https://github.com/davidhq/hodlings/blob/master/output.png?raw=true" target="_blank"><img src="https://github.com/davidhq/hodlings/raw/master/output.png?raw=true" alt="Sample Output" style="max-width:100%;"></a></p> 
        <h2> <a id="user-content-prerequisites" class="anchor" href="#prerequisites" aria-hidden="true">
          <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
           <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
          </svg></a>Prerequisites</h2> 
        <p><strong>Summary:</strong> <code>node.js</code> and <code>git</code>. You can skip to the next section if you already have these.</p> 
        <ul> 
         <li> <p><a href="https://nodejs.org/en/download/">node.js</a> spectacular platform. LTS (Long-term support) is the right choice for most.</p> </li> 
         <li> <p><a href="https://git-scm.com/downloads">git</a> incredible version control (needed in the background to install some dependencies on first install).</p> </li> 
        </ul> 
        <p><strong>Optional:</strong> if you want easy switching between different <code>node.js</code> versions in the future and you are using <strong>unix based systems</strong> (macOS, Linux), you can install <code>node.js</code> via fantastic <a href="https://github.com/tj/n">n</a> (don't download it from <em>nodejs.org</em> in this case).</p> 
        <p>You can also install both <code>node.js</code> and <code>git</code> via <a href="https://brew.sh">Homebrew</a> friendly package manager for macOS. <a href="https://github.com/tj/n">n</a> is still recommended for <code>node.js</code> though.</p> 
        <p><a href="https://www.npmjs.com">npm</a> (Node Package Manager) is installed automatically with <code>node.js</code> install (just FYI).</p> 
        <h2> <a id="user-content-install-hodlings-project" class="anchor" href="#install-hodlings-project" aria-hidden="true">
          <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
           <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
          </svg></a>Install hodlings project</h2> 
        <p>In <strong>Terminal</strong> (Linux, macOS) or <strong>Command Prompt</strong> (Windows) type:</p> 
        <p><code>npm install -g hodlings</code></p> 
        <h2> <a id="user-content-run" class="anchor" href="#run" aria-hidden="true">

        <p><code>hodl</code></p> 
        <h2> <a id="user-content-options" class="anchor" href="#options" aria-hidden="true">
          <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
           <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
          </svg></a>Options</h2> 
        <p>Try <code>hodl -h</code> for complete list and here are a few examples:</p> 
        <p><code>hodl -x eur</code></p> 
        <p><code>hodl -x cny</code></p> 
        <p>Add <code>--eth</code> to focus on <strong>Ethereum</strong> or <code>--btc</code> to focus on <strong>Bitcoin</strong> and save space for two unneeded columns.</p> 
        <h2> <a id="user-content-configure" class="anchor" href="#configure" aria-hidden="true">
          <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
           <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
          </svg></a>Configure</h2> 
        <p>Create or edit the file <code>~/.hodlings</code> (Linux/MacOS: <em>/home/[user]</em>, Windows: <em>c:\Users\[user]</em>) containing your portfolio. Use <code>#</code> to put in comments.</p> 
        <h3> <a id="user-content-example" class="anchor" href="#example" aria-hidden="true">
          <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
           <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
          </svg></a>Example</h3> 
        <p>Specifying the entire portfolio is simple:</p> 
        <pre><code># Hodlings
BTC: 0.3123
ETH: 14.99858897
GNT: 148
REP: 623
</code></pre> 
        <p>Just use token ticker symbols or (even better) the token ids because multiple tokens can have the same ticker :/</p> 
        <p>Same file but using token ids:</p> 
        <pre><code># Hodlings
bitcoin: 0.3123
ethereum: 14.99858897
golem-network-tokens: 148
augur: 623
</code></pre> 
        <p>Read token ids from <a href="https://api.coinmarketcap.com/v1/ticker">CoinMarketCap API</a>.</p> 
        <p><strong>Tip:</strong> it is much easier to read the json output in Chrome if you install the <a href="https://chrome.google.com/webstore/detail/jsonview/chklaanhfefbnpoihckbnefhakgolnmc?hl=en">JSONView Chrome extension</a> first. There are equivalent extensions for other browsers.</p> 
        <h2> <a id="user-content-notes" class="anchor" href="#notes" aria-hidden="true">
          <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
           <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
          </svg></a>Notes</h2> 
        <p>If you run <code>hodl</code> without this file present, it will create an example file for your convenience.</p> 
        <p>To use a different hodlings file for portfolio check the <code>-f</code> option (basically just use <code>hodl -f ~/path_to/hodlings2</code>).</p> 
        <h2> <a id="user-content-complete-options" class="anchor" href="#complete-options" aria-hidden="true">
          <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
           <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
          </svg></a>Complete options</h2> 
        <pre><code>  Usage: hodl [options]


  Options:

    -w, --watch               refresh data periodically every 10 min
    -f, --file &lt;f&gt;            file to use for hodlings [~/.hodlings]

    --locale &lt;locale&gt;         locale to use for formatting [en]

    --no-color                don't display colors

        <h2> <a id="user-content-updates-and-attribution" class="anchor" href="#updates-and-attribution" aria-hidden="true">
          <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
           <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
          </svg></a>Updates and attribution</h2> 
        <p><strong>hodlings</strong> is based on <a href="https://github.com/Talljoe/crytrac">crytrac</a>.</p> 
        <p>Improvements over original project (I wrote this list in the beginning, there were more improvements later):</p> 
        <ul> 
         <li>show percentage change vs USD since last run in addition to 1h, 24h and 7d changes</li> 
         <li>show 7d change vs ETH and BTC as well</li> 
         <li>show each hodling value in terms of ETH and BTC in addition to chosen fiat currency USD, EUR... (<em>-x</em> option)</li> 
         <li>coins in <em>~/.hodlings</em> file can be specified with ids (<em>basic-attention-token</em> instead of symbols, eg. <em>BAT</em>). Some coins were ambiguous (same symbol) which caused problems. Using ids is recommended.</li> 
         <li>all columns are shown by default because users mostly don't know how to turn them on before reading <em>--help</em> and besides mostly all columns are useful and nicely laid out</li> 
         <li>footer shows total value in ETH and BTC in addition to chosen fiat currency</li> 
         <li>footer shows ETH market cap percentage of total crypto market cap</li> 
         <li>footer shows the <em>flippening ratio</em> </li> 
         <li>add 24h volume (as percentage of bitcoin's volume) column</li> 
         <li>add rank column</li> 
         <li>copy <em>.hodlings-example</em> file to <em>~/.hodlings</em> on first run for easier onboarding</li> 
         <li>adjust a few colors in the portfolio table for a nicer look</li> 
         <li>remove broken export to csv feature and a few deprecated or unneeded config options</li> 
        </ul> 
        <p>I changed the project's repo/name because:</p> 
        <ul> 
         <li>easy install over npm using a new package name</li> 
         <li>possible rewrite in regular JavaScript instead of LiveScript and further expansion of features so regular fork would not be suitable</li> 
        </ul> 
        <p>Thanx again to <a href="http://talljoe.com">Joe Wasson</a> for a great project which persuaded with simplicity and nice look.</p>
