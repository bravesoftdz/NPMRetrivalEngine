      <h2 id="introduction">Introduction</h2> 
      <p>SoChain's <em>fast</em> blockchain API is the easiest, most cost-effective way to build applications on Dogecoin, Bitcoin, and Litecoin. We also offer Test Networks for developers to get started in a sandbox environment. Currency is just the first application of the Blockchain, and there's alot more to come. We're helping you make the future happen.</p> 
      <p>This API is fast, free, and unlimited. If you'd like to see more features, or if you need an obscenely large number of API calls, <a href="mailto:support@chain.so">talk to us</a>. We're happy to help.</p> 
      <p>To get started, check out our basic <a href="#code-examples">code examples</a>. Using this API is simple: no login or API key required. How quickly can you get started? Here's a Javascript example to get basic network information for the Dogecoin Blockchain:</p> 
      <pre><code>
	  $.get( "https://chain.so/api/v2/get_info/DOGE", function( response ) {
     	       // success! use the data any way you like
	  });
      </code></pre>

     <h2 id="code-examples">Code Examples</h2> 
     <p>Accessing the SoChain API is simple. Whichever programming language you use, you can call our REST API using any HTTP adapter. Below are some basic examples of how to access our API through Javascript, Ruby/Ruby on Rails, and Python.</p> 
     <p>In the examples below, we get the name, and number of blocks on the Dogecoin network (DOGE). For more networks, see <a href="#networks-supported">Networks Supported</a>.</p> 
     <h3>Using Javascript</h3> For accessing this API using Javascript, we recommend using 
     <a href="https://jquery.com/" target="_blank">jQuery</a>. Here is a simple call to get network information for Dogecoin: 
     <h4>Pre-requisites</h4> 
     <p>In the example code, we will use jQuery to access the API.</p> 
     <pre><code class="html">
&lt;script src="http://ajax.googleapis.com/ajax/libs/jquery/1.11.0/jquery.min.js"&gt;&lt;/script&gt;
</code></pre> 
     <h4>Example Code</h4> 
     <p>The following code is an HTML example of the Javascript code. It requires inclusion of the jQuery javascript, as shown above.</p> 
     <pre><code>
// get network info for Dogecoin, and display network name and total number of blocks
&lt;script&gt;
$.get( "https://chain.so/api/v2/get_info/DOGE", function( response ) {
  $( "body" )
    .append( "Name: " + response.data.name + "&lt;br/&gt;" ) // Dogecoin
    .append( "Total Blocks: " + response.data.blocks + "&lt;/br&gt;"); //  current block count
}, "json" );
&lt;/script&gt;
</code></pre> 
     <h3>Using Ruby</h3> 
     <p>For accessing this API using Ruby or Ruby on Rails, we recommend using the <a href="https://rubygems.org/gems/httpclient" target="_blank">HTTPClient</a> gem. Furthermore, you will need the JSON gem to properly interpret the JSON response from our API.</p> 
     <h4>Pre-requisites</h4> 
     <p>In the example code, we use HTTPClient and JSON gems. Here's how to install them if using Ruby:</p> 
     <pre><code>$ gem install httpclient</code></pre> 
     <pre><code>$ gem install json</code></pre> 
     <p><em>If you are using Ruby on Rails instead</em>, add the following lines to your Rails project's <em>Gemfile</em> in the root of your Rails project's directory.</p> 
     <pre><code>
gem 'httpclient'
gem 'json'
</code></pre> 
     <p>Next, execute the following on the command-line in project's root directory, and you're all set for Ruby on Rails.</p> 
     <pre><code>$ bundle install</code></pre> in the same directory.
     <p></p> 
     <h4>Example Code</h4> 
     <p>Here is an example script to retrieve basic network information for Dogecoin:</p> 
     <pre><code class="ruby">
require 'httpclient'
require 'json'

client = HTTPClient.new

response = client.get('https://chain.so/api/v2/get_info/DOGE')

if response.status_code == 200 then
   # everything went swimmingly

  content = JSON.parse response.content

  puts "Name: "+content['data']['name']
  puts "Total Blocks: "+content['data']['blocks'].to_s

end
</code></pre> 
     <h3>Using Python</h3> 
     <p>For accessing the API through Python, we recommend the <a href="http://docs.python-requests.org/en/latest/" target="_blank">python-requests</a> package.</p> 
     <h4>Pre-requisites</h4> 
     <p>Install the <em>python-requests</em> package by executing the following code on your command prompt.</p> 
     <pre><code>
$ pip install requests
</code></pre> 
     <p>If you prefer using <em>apt-get</em> instead to install Python packages, exceute the following code on your command prompt.</p> 
     <pre><code>$ apt-get install python-requests</code></pre> 
     <h4>Example Code</h4> 
     <pre><code class="python">
import requests

response = requests.get('https://chain.so/api/v2/get_info/DOGE')

if response.status_code == 200:
    # everything went swimmingly                                                                                                                   
    # parse the response as JSON
    content = response.json()

    print "Name:", content['data']['name']
    print "Total Blocks:", content['data']['blocks']

</code></pre>

     <h2 id="get-balance">Get Balance </h2> 
     <p>Returns basic balance details for a Dogecoin, Bitcoin, or Litecoin address.</p> 
     <h3>ENDPOINT</h3> 
     <pre><code>GET /api/v2/get_address_balance/{NETWORK}/{ADDRESS}[/{MINIMUM CONFIRMATIONS}]
</code></pre> 
     <h3>PARAMETERS</h3>
     <h3> 
      <blockquote>

     <h2 id="get-network-confidence">Get Network Confidence <small><span class="label label-success">New</span></small></h2> 
     <p>Returns a percentage confidence in the transaction as mirrored by the returned number of nodes in the network. SoChain listens to large parts of the Bitcoin/Dogecoin/etc. networks, and provides a numerical value (0.0 to 0.99) of its confidence in a new, unconfirmed transaction. This dramatically speeds up transactions in retail environments, and reliably bypasses the long confirmation times (e.g., 10min for Bitcoin) if required. Confirmed transactions get a confidence rating of 1.0.</p> 
     <p>If the network detects an attempted double spend of the coins used in a given transaction, the returned <em>is_double_spend</em> field is set to <em>true</em> for unconfirmed transactions, and the <em>confidence</em> field is set to <em>0.0</em>. When a transaction is confirmed, the <em>is_double_spend</em> and <em>nodes</em> fields return <em>null</em> since they become irrelevant.</p> 
     <p><strong>Note</strong>: There is a margin of error of 1%.</p> 
     <h3>ENDPOINT</h3> 
     <pre><code>GET /api/v2/get_confidence/{NETWORK}/{TXID}
</code></pre> 
     <h3>PARAMETERS</h3>
     <h3> 
      <blockquote> 
       <p><em>network: string</em><br>The acronym of the network you're querying. <em>Required</em>.</p> 
       <p><em>txid: string</em><br>The transaction hash (id) on the network you're querying. <em>Required</em>.</p> 
      </blockquote> </h3>
     <h3>REQUEST</h3> 
     <p>No request body required.</p> 
     <h3>200 - SUCCESSFUL RESPONSE</h3> 
     <p>A JSON object which contains the following fields:</p> 
     <blockquote> 
      <p><em>network: string</em><br> The acronym of the network.</p> 
      <p><em>txid: string</em><br> The transaction id.</p> 
     </blockquote> 
     <h3>404 - FAILED RESPONSE</h3> 
     <p>The NETWORK is unsupported, or TXID is invalid.</p>

     <h2 id="get-prices">Get Prices </h2> 
     <p>Returns prices of coins in multiple base pairs. This API supports the exchanges Cryptsy, Bitfinex, BitStamp, Prelude, BTCChina, Coinbase, and BTC-e. The API will return prices for all coins supported by the above exchanges.</p> 
     <h3>ENDPOINT</h3> 
     <pre><code>GET /api/v2/get_price/{COIN}/{BASE_CURRENCY}
</code></pre> 
     <h3>PARAMETERS</h3>
     <h3> 
      <blockquote> 
       <p><em>coin: string</em><br>The acronym of the coin you want prices for, e.g., DOGE, BTC, LTC. <em>Required</em>.</p> 
       <p><em>base_currency: string</em><br>The denomination currency of the prices, e.g., USD, BTC, LTC. <em>Optional</em>.</p> 
      </blockquote> </h3>
     <h3>REQUEST</h3> 
     <p>No request body required.</p> 
     <h3>200 - SUCCESSFUL RESPONSE</h3> 
     <p>A JSON object which contains the following fields:</p> 
     <blockquote> 
      <p><em>network: string</em><br> The acronym of the network.</p> 
      <p><em>prices: array</em><br> An array containing hashes/dictionary with prices of coins in price_base currencies, the exchange names, and the time the price was last updated.</p> 
     </blockquote> 
     <h3>404 - FAILED RESPONSE</h3> 
     <p>If no COIN is specified, the API will default to BTC. There will be no 404 error.</p>

     <h2 id="realtime-balance-updates">Realtime Balance Updates</h2> 
     <p>Gives realtime information on balance changes for a given address on a given network. Data received includes details on the transaction that caused the balance update trigger.</p> 
     <p>Our realtime updates use the Pusher Websocket protocol, and will need necessary dependancies (packages) installed. See <a href="http://pusher.com/docs/client_libraries" target="_blank">Pusher Client libraries</a>. We provide examples for Ruby and Javascript below.</p> 
     <h3>ENDPOINTS</h3> 
     <pre><code>
Pusher Channel: 'address_{NETWORK}_{ADDRESS}'
Update Event: 'balance_update'
</code></pre> 
     <h3>PARAMETERS</h3>
     <h3> 
      <blockquote> 
       <p><em>network: string</em><br>The acronym of the coin (in lower case) you want prices for, e.g., doge, btc, ltc. <em>Required</em>.</p> 
       <p><em>address: string</em><br>The wallet address you wish to monitor for the given network. <em>Required</em>.</p> 
      </blockquote> </h3>
     <h3>ON BALANCE_UPDATE EVENT</h3> 
     <p>A JSON object which contains the following fields:</p> 
     <blockquote> 
      <p><em>type: string</em><br> The type of information, i.e., it is for an 'address'.</p> 
      <p><em>value: json object</em><br>The object containing the updates.</p> 
      <p><em>value.address: string</em><br>The address to which this update refers.</p> 
      <p><em>value.value_sent: string</em><br>The total amount sent by this address in this transaction.</p> 
      <p><em>value.value_received: string</em><br>The total amount received by this address in this transaction.</p> 
      <p><em>value.balance_change: string</em><br>The net amount received by this address in this transaction.</p> 
      <p><em>value.tx: json</em><br>A JSON object containing transaction ID, inputs, and outputs.</p> 
     </blockquote>

     <h2 id="realtime-latest-transactions">Realtime Latest Transactions</h2>

     <h2 id="realtime-latest-prices">Realtime Price Updates</h2> 
     <p>Sends updates for up-to-date prices. Our realtime updates use the Pusher Websocket protocol, and will need necessary dependancies (packages) installed. See <a href="http://pusher.com/docs/client_libraries" target="_blank">Pusher Client libraries</a>. We provide examples for Ruby and Javascript below.</p> 
     <p> This API supports the exchanges Cryptsy, Bitfinex, BitStamp, Prelude, BTCChina, Coinbase, and BTC-e. The API will return prices for all coins supported by the above exchanges. </p> 
     <h3>ENDPOINTS</h3> 
     <pre><code>
Pusher Channel: 'ticker_{NETWORK}_{BASE_CURRENCY}'
Update Event: 'price_update'
</code></pre> 
     <h3>PARAMETERS</h3>
     <h3> 
      <blockquote> 
       <p><em>network: string</em><br>The acronym of the coin you want prices for, e.g., DOGE, BTC, LTC. <em>Required</em>.</p> 
       <p><em>base_currency: string</em><br>The denomination currency of the prices, e.g., USD, BTC, LTC. <em>Required</em>.</p> 
      </blockquote> </h3>
     <h3>ON PRICE_UPDATE EVENT</h3> 
     <p>A JSON object which contains the following fields:</p> 
     <blockquote> 
      <p><em>type: string</em><br> The nature of the data, i.e., 'price'.</p> 
      <p><em>value: json object</em><br>The object containing the updates.</p> 
      <p><em>value.time: integer</em><br>The time at which the price was updated.</p> 
      <p><em>value.price: string</em><br>The price of the coin in base_pair denomination.</p> 
      <p><em>value.price_base: string</em><br>The denominating currency.</p> 
      <p><em>value.symbol: string</em><br>The acronym of the currency the price is for.</p> 
      <p><em>value.exchange: string</em><br>The name of the exchange which published the price of the coin.</p> 
     </blockquote>
