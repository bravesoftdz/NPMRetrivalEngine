      <h3>Improved Support for Windows Modern</h3> 
      <p>MobiControl 13.3 augmented our support for the Windows Modern platform in the following areas:</p> 
      <h4>Advanced Security</h4> 
      <h5>Windows Information Protection</h5> 
      <p>MobiControl v13.3 adds support for Windows Information Protection (WIP), a transparent containment solution that reduces the inconvenience on end users when protecting corporate data by limiting the ability to share data with unapproved applications and network destinations configured by the administrator.</p> 
      <h5>Windows Defender / Advanced Threat Protection</h5> 
      <p>MobiControl v13.3 adds support for ad-hoc and scheduled Windows Defender scanning, and ensuring devices have the latest definitions to ensure threat protection against managed devices. Additionally, support has been added for onboarding of devices to the Advanced Threat Protection Service (ATP). ATP is a subscription cloud-based monitoring service offered by Microsoft to help enterprises detect, investigate, and respond to advanced attacks on their networks. It supplements the security provided by Windows Defender to bring an additional layer of protection to your desktop devices.</p> 
      <h5>Custom Health Policy</h5> 
      <p>Custom health policies expand on the Health Attestation Report feature introduced in MobiControl v13.0 by adding support for defining the security conditions that are vital to your organization and generating alerts whenever a device triggers a warning or failure of those conditions.</p> 
      <h4>Application Deployment</h4> 
      <h5>Broader Application Deployment Compatibility</h5> 
      <p>The Application Catalog rule has been extended to support classic Windows desktop applications in *.msi format, to Windows Modern desktop devices.</p> 
      <h5>Windows Companion for MobiControl</h5> 
      <p>Extract application package information necessary for various Windows Modern profile configurations using the Windows Companion utility available for download from the <a href="/products/mobicontrol/download" target="_blank">MobiControl Download</a> page.</p> 
      <h4>Secure Communication and Access</h4> 
      <h5>Certificate Distribution</h5> 
      <p>The Certificates profile configuration supports distribution of private-key client certificates. For root certificate management on desktop devices, you can now also explicitly target various user or device certificate stores.</p> 
      <h5>Assigned Access</h5> 
      <p>Assigned access is now available for Windows Modern desktop or tablets to restrict user accounts to a single application chosen by the administrator – suitable for kiosk use cases.</p> 
      <h4>Convergence of the Windows Classic Desktop Features on the Windows Modern Platform</h4> 
      <p>The following Windows Classic features are now available for Windows Modern device enrollments leveraging a device agent that is deployed silently during enrollment.</p> 
      <ul> 
       <li>Remote Control</li> 
       <li>File Sync Rule</li> 
       <li>Package Deployment via Profiles</li> 
       <li>Custom Data and Custom Attributes</li> 
      </ul> 
      <h3>Federated Authentication</h3> 
      <p>With the release of MobiControl 13.3, we now integrate with Identity Providers (IdP) using SAMLv2 for authentication to the MobiControl administrative console, Self Service Portal, iOS Profile Catalog, and device enrollment for Android and iOS devices.</p> 
      <h3>Zebra StageNow</h3> 
      <p>Support for out of the box staging as well as in field configuration of Zebra devices via Zebra StageNow - a Windows application that helps organizations configure their latest Zebra devices running Android.</p> 
      <h3>Improved Support for Android for Work</h3> 
      <h4>Remote Control and View</h4> 
      <p>You can now Remote Control Android for Work Managed Devices for certain Android OEMs with the installation of a Remote Control plugin, and Remote View any Android for Work Managed Profile devices from the MobiControl Web Console.</p> 
      <h4>New Device Provisioning Method</h4> 
      <p>Adds a simple method of provisioning Android for Work managed devices without individual Google accounts by leveraging the “afw#mobicontrol” token.</p> 
      <h4>New Feature Control option</h4> 
      <p>Disable Multi-User Profiles - available on Managed Devices only</p> 
      <h4>Remote Device Reboot</h4> 
      <p>You can now reboot Android for Work Managed Devices from the MobiControl Web Console.</p> 
      <h3>Support for NetMotion VPN</h3> 
      <p>MobiControl 13.3 adds support for the NetMotion VPN using profile configurations on Android+, Android for Work, and iOS devices.</p> 
      <h3>iOS Improvements</h3> 
      <ul> 
       <li>Devices enrolled in the Apple Device Enrollment Program can now be assigned to specific Add Devices Rules to support different enrollment destinations and Apple DEP enrollment behavior.</li> 
       <li>Lost Mode locks and tracks missing iOS devices from the MobiControl Web Console</li> 
      </ul> 
      <h3>SOTI surf Enhancements</h3> 
      <p>As of MobiControl 13.3.0.3217</p> 
      <p>The latest release of MobiControl improves existing functionality and adds new features to the SOTI surf profile configuration and app. You can now restrict web browsing on different network types, set a website as the app home screen, and designate ‘corporate bookmarks’. Improvements to kiosk mode, website filtering, and privacy settings provide increased regulation of the app.</p> 
      <p>The SOTI surf app now supports download pausing and resumption, bookmark and history searches, and miscellaneous design modifications to improve user experience. On Android devices only, custom URL schemes (File://) and external certificates are now supported. Download the SOTI surf app from the Apple App Store or the Google Play Store.</p> 
      <p>It is recommended that you also update the ERG component for SOTI surf using the latest SOTI apps installer (<a href="/products/mobicontrol/download" target="_blank">available here</a>) whenever you update your SOTI surf app.</p> 
      <h3>SOTI hub Enhancements</h3> 
      <p>As of MobiControl 13.3.0.3623</p> 
      <p>You can now designate certain files or folders in your content repository as "Mandatory" downloads. These files and folders are automatically downloaded by the SOTI hub app when it connects to the content repository. Mandatory files and folders appear in a separate menu item in the SOTI hub app, allowing device users to easily and swiftly access these important documents. Device users will be unable to remove these files from the app.</p> 
      <p>Download the SOTI hub app from the Apple App Store or the Google Play store.</p> 
      <h3>General MobiControl Features</h3> 
      <p>MobiControl v13.3 adds the ability to restrict which networks can be used when upgrading a MobiControl agent.</p>

      <p>MobiControl 13.3 will not support devices running Android 2.3.</p>

      <p class="SubHeading1"><span class="heading1"><strong>Enhanced Apple Volume Purchase Program</strong></span></p> 
      <p>The Apple Volume Purchase Program (VPP) is designed to simplify application licensing across a large number of devices. It enables organizations to purchase application licenses in bulk, saving on cost, time, and effort. In version 13.1 of MobiControl, we are pleased to announce increased support of VPP with the addition of the following new features:</p> 
      <ul> 
       <li>VPP Managed Distribution - Device Based Assignment</li> 
       <li>Manual Reconciliation of Licenses</li> 
      </ul> 
      <p class="SubHeading1"><span class="heading1"><strong>VPP Managed Distribution - Device Based Assignment</strong></span></p> 
      <p>Device-based assignment adds significant flexibility to the Managed Distribution program. Previously, application licenses were tied to an Apple ID which could make license management difficult in certain circumstances - such as when there are multiple users over a device's lifetime. With the introduction of device-based assignment, you can now assign application licenses directly to devices. Using device-based assignment requires no additional effort and switching between Apple ID based and Device-based assignment is quick and easy.</p> 
      <p class="SubHeading1"><span class="heading1"><strong>Manual Reconciliation of Licenses</strong></span></p> 
      <p>MobiControl 13.1 adds the ability to manually reconcile any licenses that are technically unattached to a device but remain unavailable until after the completion of nightly maintenance. Now you can force an audit on the statuses of your licenses at any time with the click of a button.</p> 
      <p class="SubHeading1"><span class="heading1"><strong>Increased API Functionality<br></strong></span></p> 
      <p>MobiControl API now supports the ability to locate devices that are assigned to a specific user and the ability to set custom attribute values at the device level.</p> 
      <p class="SubHeading1"><strong>Server Configurations</strong></p> 
      <p>VPP Account Ownership</p> 
      <p>An <a href="http://www.apple.com/business/vpp/">Apple VPP account</a> allows you to easily manage your application purchases and license distribution. Some organizations find it useful to share a single Apple VPP account across multiple MDM deployments to satisfy their unique requirements. VPP Account Ownership alleviates any confusion such a division of accounts can cause. With Ownership, any changes regarding your application licensing can only be completed by one MDM at a time. MobiControl 13.1 provides a clear indication of Ownership status, allows you to easily regain Ownership and updates you whenever Ownership is transferred. You'll never have to worry about license distribution conflicts again.</p>

      <h2>SOTI hub</h2> 
      <p>SOTI hub is a new MobiControl feature that enables employees to use their Android devices to access corporate files from outside the organization's internal network. SOTI hub has two parts:</p> 
      <ul> 
       <li>A SOTI hub app that is installed on the mobile device to enable the device user to access corporate files.</li> 
       <li>A SOTI hub profile payload that is pushed to the device and that a MobiControl administrator can configure to control the SOTI hub app settings.</li> 
      </ul> 
      <p>Files that you want to make available to the SOTI hub must be hosted in a content repository (an NTFS file server) that is accessible via an IIS WebDAV server.</p> 
      <p>Some of the key benefits of SOTI hub are:</p> 
      <ul> 
       <li>SOTI hub ensures that access to the corporate files is available only to employees' Android devices that are secured and managed by MobiControl.</li> 
       <li>Access to corporate files can be immediately revoked, and cached copies of files within the SOTI hub can be wiped, either on-demand or based on predefined criteria (such as violation of the IT compliance policy).</li> 
       <li>SOTI hub can be configured to require employees to log in using their LDAP credentials before they can gain access to corporate files.</li> 
      </ul> 
      <p><strong>SOTI hub App</strong></p> 
      <p>The SOTI hub app is what you use on a mobile device to access corporate files. The SOTI hub app must be packaged using MobiControl Package Studio and pushed to mobile devices.</p> 
      <p>The SOTI hub app enables you to perform the following tasks on your mobile device:</p> 
      <ul> 
       <li>Navigate up and down content repository folders and view folder contents.</li> 
       <li>Download and cache files.</li> 
       <li>Open, edit, and delete files.</li> 
       <li>Open files in third-party apps.</li> 
       <li>View file details.</li> 
       <li>Add files to favorite groups.</li> 
       <li>Search for files by file name.</li> 
       <li>Sort and filter files.</li> 
       <li>Control the ability to download files over a cellular network.</li> 
       <li>Control the ability to download files while roaming.</li> 
       <li>Delete cached files to free space on the device.</li> 
      </ul> 
      <p>For information about how to use the SOTI hub app, refer to the&nbsp;<a href="http://www.soti.net/mc/help/v12/en/default.htm#Web/IntegratedApplication/SotiHub.htm" target="_blank">MobiControl online help</a>.</p> 
      <p><strong>SOTI hub Payload</strong></p> 
      <p>The SOTI hub payload provides the configuration settings that are used by the SOTI hub app on the mobile device. You add the SOTI hub payload to a profile, then push the profile to a mobile device on which the SOTI hub app has been, or will be, installed.</p> 
      <p>The SOTI hub payload tells the SOTI hub app everything it needs to know to access the content repository, including:</p> 
      <ul> 
       <li>The URL of the content repository server that the SOTI hub app will contact.</li> 
       <li>The method of authentication the SOTI hub app will use when contacting the content repository server.</li> 
       <li>The username and password used to authenticate the SOTI hub application to the content repository server.</li> 
       <li>How often the SOTI hub app will communicate with the content repository to refresh the contents.</li> 
       <li>The amount of time that will elapse, during which the user has not interacted with SOTI hub, before the user is logged out of SOTI hub.</li> 
       <li>Whether devices are required to communicate with a MobiControl Enterprise Resource Gateway to access the content repository.</li> 
       <li>Whether devices are able to download content over a cellular network.</li> 
       <li>Whether devices are able to download content while roaming.</li> 
      </ul> 
      <p>For information about the configuration settings available in the SOTI hub payload, refer to the&nbsp;<a href="http://www.soti.net/mc/help/v12/en/default.htm#Web/Profiles/Sotihub_Payload/ApplicationSettings.htm" target="_blank">MobiControl online help</a>.</p> 
      <p><a name="entresgateway" class="mceItemAnchor"></a></p> 
      <h2>Enterprise Resource Gateway</h2> 
      <p>The Enterprise Resource Gateway is a new MobiControl component that enables you to control Internet traffic using a proxy server. The proxy server acts as a single point of contact – a gateway – serving client requests. The proxy server authenticates each request and forwards it to the desired destination server.</p> 
      <p><strong>Enterprise Resource Gateway Properties</strong></p> 
      <p>After you have installed the Enterprise Resource Gateway on a server, you can set various properties for it, such as:</p> 
      <ul> 
       <li>The name and URL of the Enterprise Resource Gateway instance.</li> 
       <li>The type of filtering you want the Enterprise Resource Gateway to do: Exchange Server for emails, or Content Repositories for files.</li> 
       <li>Whether to enable Secure Email Access.</li> 
      </ul> 
      <p>For information about the steps required to set up the Enterprise Resource Gateway, refer to the&nbsp;<a href="http://www.soti.net/mc/help/v12/en/default.htm#Setup/EnterpriseResourceGateway/ERG_setup_intro.html" target="_blank">MobiControl online help</a>.</p> 
      <p><a name="settingsmanager" class="mceItemAnchor"></a></p> 
      <h2>Settings Manager</h2> 
      <p>The Settings Manager is an application that enables a MobiControl administrator to provide controlled access to a subset of device settings when a device is in lockdown mode. The types of device settings that can be controlled via the Settings Manager are:</p> 
      <ul> 
       <li>Display</li> 
       <li>Sound</li> 
       <li>WiFi</li>

      <h2 class="SubHeading2">ELM Agent for Samsung Android</h2> 
      <p>After upgrade, Samsung Android devices with MDMv4 capabilities will receive a new type of Device Agent during enrollment, referred to as the “ELM Agent”. Without compromising management functionality, Samsung’s Enterprise License Manager (ELM) allows SOTI to deliver timelier updates of the device agent in order to serve our customers better. The following observations however should be considered before upgrading MobiControl:</p> 
      <ul> 
       <li>The ELM Agent requires Internet connectivity during enrollment, and periodically thereafter, to validate MDM licensing against Samsung servers.</li> 
       <li>End Users will be required to accept a privacy dialog during enrollment to acknowledge that non-identifying device information will be used to perform MDM license validation.</li> 
       <li>Migration to the ELM agent for devices with MDMv4 capabilities is advised for all devices currently enrolled in the system. A&nbsp;<strong>Right-Click (on device) &gt; Agent Update &gt; Migrate to ELM Agent</strong>&nbsp;option has been added to the Web Console to initiate the migration process. Migration may temporarily roll back policies and will require end user action as described above.</li> 
       <li>The Web Console will show the agent type installed on each device under the&nbsp;<strong>Android+ &gt; Information Panel</strong>. “ELM” represents the new agent while “Signed” is used to represent the older agent.</li> 
       <li>The “Signed” agent is still available for download and manual installation under&nbsp;<strong>Android+ &gt; Rules &gt; Add Devices &gt; Right Click (on rule) &gt; Download Device Agent</strong>&nbsp;but is deprecated for Samsung devices with MDMv4 and higher, and may not be included in future releases.</li> 
      </ul> 
      <h2 class="SubHeading2">Virtual Group Behavior Modification</h2> 
      <p>Virtual Groups created in v11 will only include devices that reside in the parent Device Group(s) for which the Virtual Group also resides. That is to say, if you nest a Virtual Group in a Device Group, the scope of the Virtual Group is limited to the parent Device Group(s). Existing Virtual Groups will maintain the old functionality until deleted.</p>
