       <div id="_neo4j_for_javascript_developers" class="sect2 columns medium-12" style="padding-left: 0;"> 
        <h3>Neo4j for JavaScript Developers</h3> 
        <div class="paragraph"> 
         <p>Neo4j can be installed on any system and then accessed via its binary and HTTP APIs.</p> 
        </div> 
        <div class="paragraph"> 
         <p>You can use the official binary driver for JavaScript (neo4j-driver) or connect via HTTP using the <code>request</code> module or any of our community drivers.</p> 
        </div> 
       </div> 
       <div id="neo4j-javascript-driver" class="sect2 columns medium-12" style="padding-left: 0;"> 
        <h3><span class="icon"><i class="fa fa-check-circle-o"></i></span> Neo4j Javascript Driver</h3> 
        <script src="https://cdnjs.cloudflare.com/ajax/libs/codemirror/5.13.4/mode/javascript/javascript.min.js"></script> 
        <div class="paragraph"> 
         <p>The Neo4j JavaScript driver is <strong>officially supported</strong> by Neo4j and connects to the database using the binary protocol. It aims to be minimal, while being idiomatic to JavaScript, allowing you to subscribe to a stream of responses, errors and completion events.</p> 
        </div> 
        <div class="listingblock"> 
         <div class="content code-panel" style="width: 100%; height: auto; border: medium 1px;"> 
          <pre style="font-size: 16px; font-weight: 100;" class="code programlisting cm-s-neo" data-lang="shell"><code>npm install neo4j-driver@1.2.0</code></pre> 
         </div> 
        </div> 
        <div class="listingblock"> 
         <div class="content code-panel" style="width: 100%; height: auto; border: medium 1px;"> 
          <pre style="font-size: 16px; font-weight: 100;" class="code programlisting cm-s-neo" data-lang="javascript"><code>const neo4j = require('neo4j-driver').v1;

const driver = neo4j.driver(uri, neo4j.auth.basic(user, password));
const session = driver.session();

const personName = 'Alice';
const resultPromise = session.run(
  'CREATE (a:Person {name: $name}) RETURN a',
  {name: personName}
);

resultPromise.then(result =&gt; {
  session.close();

  const singleRecord = result.records[0];
  const node = singleRecord.get(0);

  console.log(node.properties.name);

  // on application exit:
  driver.close();
});</code></pre>

       <div id="_the_example_project" class="sect2 columns medium-12" style="padding-left: 0;"> 
        <h3>The Example Project</h3> 
        <div class="paragraph"> 
         <p>The Neo4j example project is a small, one page webapp for the movies database built into the Neo4j tutorial. The front-end page is the same for all drivers: movie search, movie details, and a graph visualization of actors and movies. Each backend implementation shows you how to connect to Neo4j from each of the different languages and drivers.</p> 
        </div> 
        <div class="paragraph"> 
         <p>You can learn more about our small, consistent example project across many different language drivers <a href="../example-project">here</a>. You will find the implementations for all drivers as <a href="//github.com/neo4j-examples?query=movies">individual GitHub repositories</a>, which you can clone and deploy directly.</p> 
        </div> 
       </div>

       <div id="node-neo4j" class="sect2 columns medium-12" style="padding-left: 0;"> 
        <h3>node-neo4j (Thingdom)</h3> 
        <div class="paragraph"> 
         <p>Most widely used Node.js driver for Neo4j. This driver aims to be the most robust, comprehensive, and battle-tested community driver available. It’s run in production by <a href="//www.fiftythree.com/">FiftyThree</a> to power <a href="//www.fiftythree.com/paper">Paper</a> and <a href="//mix.fiftythree.com/">Mix</a>.</p> 
        </div> 
        <div id="_features" class="sect3 "> 
         <h4>Features</h4> 
         <div class="ulist"> 
          <ul> 
           <li> Cypher queries, parameters, batching, and transactions </li> 
           <li> Arbitrary HTTP requests, for custom Neo4j plugins </li> 
           <li> Custom headers, for high availability, application tracing, query logging, and more </li> 
           <li> Precise errors, for robust error handling from the start </li> 
           <li> Configurable connection pooling, for performance tuning &amp; monitoring </li> 
           <li> Thorough test coverage with &gt;100 tests </li> 
           <li> Continuously integrated against multiple versions of Node.js and Neo4j </li> 
          </ul>

       <div id="_cypher_stream" class="sect2 columns medium-12" style="padding-left: 0;"> 
        <h3>cypher-stream</h3> 
        <div class="paragraph"> 
         <p>Neo4j cypher queries as node object streams.</p> 
        </div> 
        <div class="paragraph"> 
         <p>Transactions are duplex streams that allow you to write query statements then commit or roll back the written queries.</p> 
        </div> 
        <div class="paragraph"> 
         <p>To get a stream per statement, just pass a callback function with the statement object. This works for regular cypher calls and transactions.</p> 
        </div>

         <ul> 
          <li> <a href="//github.com/brian-gates/streaming-stack">Simple example of an ember/node/neo4j stack</a> </li>

        <h3>Seraph</h3>

         <p><strong>Seraph</strong> is a comprehensive, well documented driver for Neo4j, which supports all Neo4j features and provides not only Neo4j connection support, but extensive lifecycle handling.        <div id="_using_the_http_endpoint_directly" class="sect2 columns medium-12" style="padding-left: 0;"> 
        <h3>Using the HTTP-Endpoint directly</h3> 
        <div class="paragraph"> 
         <p>You can use something as simple as the <code>request</code> node-module to send queries to and receive responses from Neo4j. The endpoint protocol and formats are explained in detail in the <a href="http://neo4j.com/docs/developer-manual/current/http-api/#http-api-transactional">Neo4j Manual</a>. It enables you do to much more, e.g. sending many statements per request or keeping transactions open across multiple requests.</p> 
        </div> 
        <div class="paragraph"> 
         <p>Here is a very simple example:</p> 
        </div> 
        <div class="listingblock"> 
         <div class="title">
          Simple Function Accessing the Remote Endpoint
         </div> 
         <div class="content code-panel" style="width: 100%; height: auto; border: medium 1px;"> 
          <pre style="font-size: 16px; font-weight: 100;" class="code programlisting cm-s-neo" data-lang="javascript"><code>var r=require("request");
var txUrl = "http://localhost:7474/db/data/transaction/commit";
function cypher(query,params,cb) {
  r.post({uri:txUrl,
          json:{statements:[{statement:query,parameters:params}]}},
         function(err,res) { cb(err,res.body)})
}</code></pre> 
         </div> 
        </div> 
        <div class="listingblock"> 
         <div class="title">
          Running the Function
         </div> 
         <div class="content code-panel" style="width: 100%; height: auto; border: medium 1px;"> 
          <pre style="font-size: 16px; font-weight: 100;" class="code programlisting cm-s-neo" data-lang="javascript"><code>var query="MATCH (n:User) RETURN n, labels(n) as l LIMIT {limit}"
var params={limit: 10}
var cb=function(err,data) { console.log(JSON.stringify(data)) }

cypher(query,params,cb)

{"results":[
  {"columns":["n","l"],
   "data":[
     {"row":[{"name":"Aran"},["User"]]}
    ]
  }],
 "errors":[]}</code></pre> 
         </div> 
        </div>
