        <h1 data-animated="fadeIn">How-To: Create an image recognition app with Vuforia and HoloLens <span class="divider"></span></h1>
       </div>
       <div class="clearfix"></div>
       <div class="post">
        <div class="col-md-9">
         <div class="img-body"> 
          <img src="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Android-Octocat.png" alt="" class="img-responsive">
         </div>
         <div class="post-meta">
           Friday, May 26, 2017 by 
          <a href="http://southworks.com/blog/author/sgambolati/" title="Posts by sgambolati" rel="author">sgambolati</a> 
          <span><a href="http://southworks.com/blog/2017/05/26/how-to-create-an-image-recognition-app-with-vuforia/#respond" class="comments-link"><i class="fa fa-comments-o"></i> 0</a></span>
         </div>
         <p></p>
         <p>Image recognition and interpretation is a very requested task into modern application. Either you want to identify a specific image or object or either you want to&nbsp;interpret what the users get in front their smartphone camera.</p>
         <p>A&nbsp;technology that is&nbsp;becoming more and more popular among&nbsp;developers with this task is <a href="https://www.vuforia.com" target="_blank" rel="noopener noreferrer">Vuforia</a>. This framework can recognize a specific image and track it while the image is in the scene. The same goes to Custom 3D object, Cylinders, boxes, etc.</p>
         <p>Moreover, Vuforia is widely accepted because we can use it with&nbsp;almost any device, from older and cheap Android smartphone to&nbsp;the most recent VR/AR/MX devices. In this post, we will use the Vuforia capabilities from Unity3D because it is also multi-platform.</p>
         <p><span id="more-97871"></span></p>
         <h4>You can access this final project code in <a href="https://github.com/sGambolati/VuforiaImageRecognition" target="_blank" rel="noopener noreferrer">github repo</a>.</h4>
         <h2>Prerequisites</h2>
         <p>First, we need to download Unity3D from <a href="https://store.unity.com/" target="_blank" rel="noopener noreferrer">here</a>. You may try the personal (and free) plan. In the meantime, it is installing, review <em>What is Unity3D</em>.</p>
         <h5>When installing Unity, be sure to install the Windows Store platform, and the .Net Scripting Backend</h5>
         <h3>What is Unity3d?</h3>
         <p>Unity3D is a videogame engine, which have support for 3D and 2D games. It has excellent physics, collision, light systems and much more. It has a very simple but powerful IDE and we can include scripts in JavaScript or C#.</p>
         <p>Unity3D have a great and complete community with great tutorials with various levels of knowledge.</p>
         <p>We can use the built-in editor to code or any version of Visual Studio. If we install the Visual Studio Tools for Unity, we can attach it to Unity and debug our code. Additionally, you can run your game inside Unity3D to test and debug your game.</p>
         <p>We can generate the output for Windows, Linux, MacOS, many mobile OS (Android, iOS, BlackBerry, etc.) and finally, we also can create executable for Xbox, Wii and PS3/PS4/PSP platforms.</p>
         <p>We also have an enormous, great integrated, <a href="https://www.assetstore.unity3d.com/" target="_blank" rel="noopener noreferrer">Assets Store</a>&nbsp;(paid and free) to don’t need to design buildings, characters, effects, code snippets, and all we can need.</p>
         <h3>What is Vuforia?</h3>
         <p>Basically, Vuforia is a framework for image and object recognition and tracking. It has a <a href="https://developer.vuforia.com/" target="_blank" rel="noopener noreferrer">developer portal</a> where we can upload images to a database that later can be used for tracking other images and objects.</p>
         <h2>Step 1: Setup Vuforia Database</h2>
         <ol>
          <li>Download from <a href="https://developer.vuforia.com/downloads/sdk" target="_blank" rel="noopener noreferrer">here</a> the latest Vuforia SDK for Unity.</li>
          <li>Navigate to the <a href="https://developer.vuforia.com/" target="_blank" rel="noopener noreferrer">Vuforia Developer Portal</a>, Log in or Register (if this is your first time). And click on the <strong>Develop</strong> top section and next click on the <strong>License Manager</strong>&nbsp;and click on the <strong>Add License Key</strong> button. Licenses will determine your final target for your project. By now, we will choose the <strong>Development</strong> option. Complete with an <strong>Application Name</strong> like <em>VuforiaImageSample</em>.<a href="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Vuforia-AddLicenseKey.png"><img class="aligncenter wp-image-98071 size-medium" src="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Vuforia-AddLicenseKey-275x300.png" alt="" width="275" height="300" srcset="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Vuforia-AddLicenseKey-275x300.png 275w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Vuforia-AddLicenseKey-768x838.png 768w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Vuforia-AddLicenseKey.png 796w" sizes="(max-width: 275px) 100vw, 275px"></a></li>
          <li>Confirm your choice and notice your Usage Quotes.</li>
          <li>Click on the name of the recent created License. And save the <em>License Key</em> for later usage.</li>
          <li>Now, let’s create our database to upload our image. Click on the <strong>Target Manager</strong> on the top sub-menu. And then click on the <strong>Create Database</strong> button. Choose a Database name (like <em>VuforiaImageDataBase</em>) and select <em>Device</em> as Database <strong>Type</strong>. Accept to dialog clicking on the <strong>Create</strong> button.<a href="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Vuforia-CreateDatabase.png"><img class="aligncenter wp-image-98081 size-medium" src="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Vuforia-CreateDatabase-300x192.png" alt="" width="300" height="192" srcset="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Vuforia-CreateDatabase-300x192.png 300w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Vuforia-CreateDatabase-768x491.png 768w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Vuforia-CreateDatabase.png 778w" sizes="(max-width: 300px) 100vw, 300px"></a></li>
          <li>Now, we will need an image to upload. Take in consideration that you can upload any image that you want, but you must think that Vuforia will works better with a high quality and contrasted image. For our sample, we will choose <a href="https://drive.google.com/open?id=0B2BGTvBwYnPWWmZEMEhvd09yRDA" target="_blank" rel="noopener noreferrer">this Octocat image</a>.</li>
          <li>To finally upload our image that will be recognized, click on the <em>Add Target</em> button. And a popup like&nbsp;the next image open. Choose the Single Image as the Type, pick up the image you want to upload and enter the Width for the image. This is not the image width, instead is a relative unity which allow to scale in relation with others image you have in the same Database. At this moment, we don’t plan to have another image so let’s enter 50. Click on the <strong>Add</strong> button to process to upload the image.<a href="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Vuforia-AddImageTarget.png"><img class="aligncenter wp-image-98091 size-medium" src="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Vuforia-AddImageTarget-239x300.png" alt="" width="239" height="300" srcset="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Vuforia-AddImageTarget-239x300.png 239w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Vuforia-AddImageTarget.png 716w" sizes="(max-width: 239px) 100vw, 239px"></a></li>
          <li>Almost finish here. Now you should see a grid with the image and the most important property here is the <em>Rating</em>. Higher Rating make easier to detect the image. Click on the <strong>Download Database</strong> button and a new popup will show. Choose <strong>Unity Editor</strong>&nbsp;and click on the <strong>Download</strong> button and save the given file.</li>
         </ol>
         <h2>Step 2: Create Unity3D project and import Vuforia asset</h2>
         <ol>
          <li>Now, we need a new Unity3D project. So, let’s create one starting Unity3D and click on the <strong>New</strong> button. Complete the project name with any of yours choose, <em>VuforiaImageSample</em> may be a good one, and a location folder as you consider best.</li>
         </ol>
         <p><a href="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-CreateProject.png"><img class="aligncenter wp-image-97941 size-medium" src="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-CreateProject-300x173.png" alt="" width="300" height="173" srcset="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-CreateProject-300x173.png 300w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-CreateProject-768x442.png 768w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-CreateProject-1024x590.png 1024w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-CreateProject.png 1252w" sizes="(max-width: 300px) 100vw, 300px"></a></p>
         <ol>
          <li>Import the Vuforia SDK. Go to the menu <strong>Asserts -&gt; Import Package -&gt; Custom Package</strong> and locate the <em>.unitypackage</em> file you have downloaded in the previous step. After a few moments, a dialog will show to confirm the import process. Click on the <strong>Import</strong> button.</li>
          <li>We will do the same with the Vuforia Database we downloaded at last in the previous step.</li>
          <li>Now, let’s do some magic here. In the Hierarchy tab <em>delete</em> the <strong>Main camera</strong> component. On the Project tab, navigate to the <em>Vuforia\Prefab</em> folder and drag the ARCamera component to the <em>Hierarchy</em> tab. This component will be the camera which your app will constantly scan and trying to identify your image. With the ARCamera selected, go to the <em>Inspector</em> tab and click on the <strong>Open Vuforia configuration</strong>. Paste your App License Key you have saved from last step.</li>
          <li>Locate the Datasets node and ensure the Load is checked and next, check the child Activate. This tells to Vuforia to load and activate the Database when the scene starts. Otherwise, we need to load and activate programmatically.<a href="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-VuforiaAppLicenseKey.png"><img class="wp-image-97961 size-full aligncenter" src="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-VuforiaAppLicenseKey.png" alt="" width="420" height="731" srcset="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-VuforiaAppLicenseKey.png 420w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-VuforiaAppLicenseKey-172x300.png 172w" sizes="(max-width: 420px) 100vw, 420px"><br> </a></li>
          <li>Go back to the <em>Vuforia\Prefab</em> folder in the <em>Project</em> tab and drag the <strong>ImageTarget</strong> component to the <strong>Hierarchy</strong> tab. This component is the responsible for&nbsp;recognizing your image from what the ARCamera capture. Maintain selected this component and go to the <strong>Inspector</strong> tab and select your <em>Database name</em> from the <strong>DataBase</strong> options and next select the <strong>Image Target</strong> you uploaded to the Vuforia Portal. After that you should note that the ImageTarget is bigger than before. It was scaled to the magic number you entered in the Width image when uploaded.<a href="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageTargetDatabase.png"><img class="aligncenter wp-image-97971 size-large" src="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageTargetDatabase-1024x724.png" alt="" width="1000" height="707" srcset="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageTargetDatabase-1024x724.png 1024w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageTargetDatabase-300x212.png 300w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageTargetDatabase-768x543.png 768w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageTargetDatabase.png 1155w" sizes="(max-width: 1000px) 100vw, 1000px"></a></li>
          <li>Now, you can say that you have an AR app that recognize an image. But if you run the app in Unity3D it will route to your Webcam (if there is one) and however you show the image or not, nothing happens. More specific, you can’t see if the image is recognized or not.</li>
          <li>First, you will put a basic 3D object as a flag to get notice if the image was or not recognized. Ensure the ImageTarget is still selected and go to the <strong>GameObject -&gt; 3D Object -&gt; Cube</strong> menu this should&nbsp;add a cube as a child of your ImageTarget. Now, play around with the <strong>Transform</strong> properties of the cube to make more accurate&nbsp;to your image. I set the properties as you can see in the next image.<a href="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageTargetCube.png"><img class="aligncenter wp-image-97981 size-large" src="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageTargetCube-1024x724.png" alt="" width="1000" height="707" srcset="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageTargetCube-1024x724.png 1024w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageTargetCube-300x212.png 300w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageTargetCube-768x543.png 768w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageTargetCube.png 1155w" sizes="(max-width: 1000px) 100vw, 1000px"></a></li>
          <li>Now, you can play the project and show to the camera the image you upload. You can print the image or load from another&nbsp;device as a smartphone.<a href="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageRecognized.png"><img class="aligncenter wp-image-97991 size-large" src="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageRecognized-1024x549.png" alt="" width="1000" height="536" srcset="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageRecognized-1024x549.png 1024w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageRecognized-300x161.png 300w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageRecognized-768x412.png 768w" sizes="(max-width: 1000px) 100vw, 1000px"></a></li>
         </ol>
         <p>Amazing, huh? You may notice that this process could be confusing but once you have done this a few times, it’s become very easy to do.</p>
         <h5>Notice the Vuforia word and logo. That is because you are using a Development License. This will be removed if you use a paid License Key.</h5>
         <p>Ensure to save the scene you are working with any name you desired, it not important to what we are doing here the name chosen.</p>
         <h2>Step 3: More 3D Models</h2>
         <p>Now, let make it a bit more pretty. Let’s change the boring cube to something more adequate to the image we are tracking. However, the Unity3D’s Asset Store has a lot of material you can import without exit the app, sometimes you can find what you really need. So, I have found a better 3D model to use, <a href="https://sketchfab.com/models/cad2ffa5d8a24423ab246ee0916a7f3e#" target="_blank" rel="noopener noreferrer">the Octocat itself</a>&nbsp;(thanks <a href="https://sketchfab.com/sillyking" target="_blank" rel="noopener noreferrer">sillyking</a> for this model). Sadly, to be enabled to download the model you must register in the sketchfab page.</p>
         <ol>
          <li>Go back to Unity3D and click on the <strong>Assets -&gt; Import new Asset</strong> menu. Pick up the <strong>octocat.obj</strong> file from the <em>twice</em> zipped file.</li>
          <li>Delete the ugly cube and drag the <em>Octocat</em> model as a child of the <em>ImageTarget</em>.</li>
          <li>In the Project tab, navigate to the <em>Assets\Editor\QCAR\ImageTargetTexture\\</em> and select the image that&nbsp;is there. In the <em>Inspector</em> tab, change texture type to sprite and click apply. Now, you can see the original image you have uploaded and recognizing. With that in mind, you can set the transform properties of the Octocat 3D object as you want to base on the right image position. You may use the same position info as I:<a href="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-PositionOctocatModel.png"><img class="aligncenter wp-image-98001 size-large" src="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-PositionOctocatModel-1024x724.png" alt="" width="1000" height="707" srcset="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-PositionOctocatModel-1024x724.png 1024w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-PositionOctocatModel-300x212.png 300w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-PositionOctocatModel-768x543.png 768w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-PositionOctocatModel.png 1155w" sizes="(max-width: 1000px) 100vw, 1000px"></a></li>
          <li>Try to play again and show you image to your cam. Moreover, to see the 3D model in the <em>Game</em>&nbsp;panel, we can see how it shows/hides likely the same in in the <em>Scene</em> and<em> Game</em> tabs<em>.</em><a href="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageTarget3DModel.png"><img class="aligncenter wp-image-98011 size-large" src="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageTarget3DModel-1024x724.png" alt="" width="1000" height="707" srcset="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageTarget3DModel-1024x724.png 1024w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageTarget3DModel-300x212.png 300w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageTarget3DModel-768x543.png 768w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ImageTarget3DModel.png 1155w" sizes="(max-width: 1000px) 100vw, 1000px"></a></li>
         </ol>
         <h2>Step 4: Deploy to Android</h2>
         <p>You can deploy this application to many-many platforms. First, deploy to an Android device (you can choose to deploy to an iOS device, too). You may not have the required Unity3D package for that platform. In that case, Unity3D will display a button to download what is necessary and you only need to install it.</p>
         <ol>
          <li>Go to the <strong>File -&gt; Build Settings</strong>. In the platform list, select the <em>Android</em> one and click on the <strong>Switch platform button</strong>.</li>
          <li>Click on the Player Settings, which will open the properties of the Android application.</li>
          <li>Click on the <strong>Add Open Scenes</strong>&nbsp;button to include the scene you have been working out in the application.<a href="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-BuildAndroid.png"><img class="aligncenter wp-image-98021 size-medium" src="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-BuildAndroid-300x291.png" alt="" width="300" height="291" srcset="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-BuildAndroid-300x291.png 300w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-BuildAndroid.png 630w" sizes="(max-width: 300px) 100vw, 300px"></a></li>
          <li>Click on the <strong>Player Settings</strong> button and a new tab should appear. Search the section <strong>Other Settings</strong> and then the subsection <strong>Identification</strong>. Change the <strong>Package Name</strong> to something with the format: <em>com..</em>. Ensure to save your scene after you made the change. This is required to identify your application property.</li>
          <li>Click on the <strong>Build</strong> button and you will be prompted to save the <em>.apk</em> that will be generated. Choose any folder of your convenience and accept.</li>
          <li>Now, you can copy the file to your device and run it native.<a href="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Android-Octocat.png"><img class="aligncenter wp-image-97931 size-large" src="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Android-Octocat-1024x576.png" alt="" width="1000" height="563" srcset="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Android-Octocat-1024x576.png 1024w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Android-Octocat-300x169.png 300w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Android-Octocat-768x432.png 768w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Android-Octocat.png 1280w" sizes="(max-width: 1000px) 100vw, 1000px"></a></li>
         </ol>
         <h2>Step 5: Deploy to HoloLens</h2>
         <p>To Deploy to HoloLens, you should do several tweaks&nbsp;to your project.</p>
         <ol>
          <li>Select ImageTarget component and Check&nbsp;the <strong>Enable Extended Tracking</strong> property.</li>
          <li>Download <a href="https://github.com/Microsoft/HoloToolkit-Unity/raw/master/External/Unitypackages/HoloToolkit-Unity-v1.5.7.0.unitypackage" target="_blank" rel="noopener noreferrer">HoloToolkit Package</a>. HoloToolkit is a set of utilities for Unity3D development for HoloLens. It has a cool samples and Prefabs to import (We will see in next posts).<a href="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-HoloLensConfiguration-EnableExtendedTracking.png"><img class="aligncenter size-full wp-image-98031" src="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-HoloLensConfiguration-EnableExtendedTracking.png" alt="" width="470" height="250" srcset="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-HoloLensConfiguration-EnableExtendedTracking.png 470w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-HoloLensConfiguration-EnableExtendedTracking-300x160.png 300w" sizes="(max-width: 470px) 100vw, 470px"></a></li>
          <li>Go to the menu <strong>Asserts -&gt; Import Package -&gt; Custom Package</strong> and pick&nbsp;the <em>.unitypackage</em> file you have just downloaded. When a dialog with the details of the files that will be imported is shown, click on <strong>Import</strong>.</li>
          <li>In the <em>Project</em>&nbsp;tab, locate the <strong>Assets\HoloToolkit\Input\Prefabs\HoloLensCamera.prefab</strong> and drag to the <em>Hierarchy</em> tab.</li>
          <li>Select the <strong>ARCamera</strong> component in the <em>Hierarchy</em> tab and at <strong>Inspector</strong> Tab set the <strong>Central Anchor point</strong> to <em>HoloLensCamera</em> component.<a href="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-HoloLensConfiguration-SetHoloLensCamera.png"><img class="aligncenter wp-image-98041 size-full" src="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-HoloLensConfiguration-SetHoloLensCamera.png" alt="" width="1538" height="833" srcset="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-HoloLensConfiguration-SetHoloLensCamera.png 1538w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-HoloLensConfiguration-SetHoloLensCamera-300x162.png 300w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-HoloLensConfiguration-SetHoloLensCamera-768x416.png 768w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-HoloLensConfiguration-SetHoloLensCamera-1024x555.png 1024w" sizes="(max-width: 1538px) 100vw, 1538px"></a></li>
          <li>You may noticed already that HoloToolkit &nbsp;has added a new menu name <em>HoloToolkit</em> (D’oh!) in Unity3D. &nbsp;Inside this menu there is a sub-menu that will C<em>onfigure</em> some parts to enabled to work or work fine in side <em>HoloLens</em>. Click <strong>Apply HoloLens Scene Settings</strong> and a popup should be displayed. Ensure all checks are checked and click <strong>Apply</strong>. Save your scene.<a href="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ApplyHoloLensSceneSettings.png"><img class="aligncenter size-full wp-image-98042" src="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ApplyHoloLensSceneSettings.png" alt="" width="437" height="332" srcset="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ApplyHoloLensSceneSettings.png 437w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ApplyHoloLensSceneSettings-300x228.png 300w" sizes="(max-width: 437px) 100vw, 437px"></a></li>
          <li>Next, do the same with the <strong>Apply HoloLens Capability Settings</strong>. Save your scene.<a href="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ApplyHoloLensCapabilitySettings.png"><img class="aligncenter size-full wp-image-98061" src="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ApplyHoloLensCapabilitySettings.png" alt="" width="437" height="351" srcset="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ApplyHoloLensCapabilitySettings.png 437w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ApplyHoloLensCapabilitySettings-300x241.png 300w" sizes="(max-width: 437px) 100vw, 437px"></a></li>
          <li>Do the same with the <strong>Apply HoloLens Project Settings</strong>. This popup will requires reload your entire project to force new settings take effect.<a href="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ApplyHoloLensProjectSettings.png"><img class="aligncenter size-full wp-image-98051" src="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ApplyHoloLensProjectSettings.png" alt="" width="437" height="357" srcset="http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ApplyHoloLensProjectSettings.png 437w, http://southworks.com/blog/wp-content/uploads/sites/81/2017/05/Unity-ApplyHoloLensProjectSettings-300x245.png 300w" sizes="(max-width: 437px) 100vw, 437px"></a></li>
          <li>You may do an extra-check for the <strong>AppKey</strong> and the <strong>Load Database </strong>and<strong> Activate</strong> on the <strong>VuforiaConfiguration</strong>.</li>
          <li>Save your scene, before continuing.</li>
          <li>Finally, go back to the Build window and click the <strong>Build</strong> button and choose a directory where you would like to export the project.</li>
          <li>Once the build process has ended, open the <em>.sln</em> in the folder you set with Visual Studio. Change the <strong>Configuration Setting</strong> from <em>ARM</em> to <em>x86</em> and now you can deploy to your <em>HoloLens</em> or run in the <a href="https://developer.microsoft.com/en-us/windows/mixed-reality/install_the_tools" target="_blank" rel="noopener noreferrer"><em>HoloLens Emulator</em></a>.</li>
         </ol>
         <h2>Conclusion</h2>
         <p>In this post, Upload images to a database that later can be used for tracking other images and objects. Moreover, we imported it to Unity3D and setup the ARCamera and ImageTarget component. And we imported an external 3D model to our scene. At the end, we build the project for&nbsp;different platform. And we didn’t make&nbsp;a single line of code!</p>
         <p>We will continue in a next post showing how to recognize multiple images and how to distinguish each one showing different 3D models or specific data.</p>
