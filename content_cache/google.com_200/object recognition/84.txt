         <div title=".NET WinForms">
           This section provides an overview of possible reasons for a limited object recognition with Ranorex tools and recommends ways to correct a malfunction. 
          <br> 
          <br> 
          <div class="toc-macro client-side-toc-macro  conf-macro output-block" data-headerelements="H1,H2" data-hasbody="false" data-macro-name="toc"> 
          </div>

         <div> 
          <h2 class="first-headline" id="GeneralTroubleshooting-.NETWinForms">.NET WinForms</h2> 
          <p>Ranorex failed to load the assemblies containing the definition of this control. UI element identification capabilities for this control are limited. There might be several reasons for limited support of .NET WinForms based applications or controls:</p> 
          <h3 id="GeneralTroubleshooting-Reason#1:">Reason #1:</h3> 
          <p>The automating process (Ranorex executable) and the automated application (AUT) are not started with the same security rights, i.e. not as the same Windows user.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Try starting both applications as the same user.</p> 
          <h3 id="GeneralTroubleshooting-Reason#2:">Reason #2:</h3> 
          <p>Either the automating executable or the AUT are started from a network drive or encrypted folder and consequently do not have the required security rights.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Try copying both applications to a folder on your hard disk.</p> 
          <h3 id="GeneralTroubleshooting-Reason#3:">Reason #3:</h3> 
          <p>The .NET Framework 4 (or higher) got installed after Ranorex was already installed and the AUT uses .NET 4.0. Consequently, as the .NET Framework 4 was not present when Ranorex was installed, the Ranorex setup could not install the DLLs required to recognize and automate .NET 4.0 applications.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Uninstall Ranorex completely and then re-install Ranorex.</p> 
          <h3 id="GeneralTroubleshooting-Reason#4:">Reason #4:</h3> 
          <p>The controls that cannot be identified are implemented in a mixed-mode EXE file (not DLL). This can be the case with some obfuscating applications or assembly merging utilities that create mixed-mode EXE files. The .NET Framework does not support loading such mixed-mode executables in other processes and that's why Ranorex cannot recognize controls implemented within them.</p> 
          <p><strong>Solution:</strong></p> 
          <p>As a workaround, you can try to automate the non-obfuscated version of your application.</p> 
          <h3 id="GeneralTroubleshooting-Reason#5:">Reason #5:</h3> 
          <p>The controls that cannot be identified are implemented in an assembly that targets a particular platform/processor and the automating executable targets a different platform/processor. This can cause problems on 64 bit operating systems, since such assemblies cannot be loaded by the automating process.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Try changing the 'Target CPU' (Ranorex Studio) or 'Platform Target' (Visual Studio) option in the automation project settings to match the target of the automated application as described in <a data-macro-name="sp-plaintextbody-link" href="/help/latest/64-bit-platforms" class="conf-macro output-inline" data-hasbody="true">this section</a>. If you don't know the target CPU/platform of the automated application, first try to change the option to 'Any processor' or 'Any CPU', respectively, then try the remaining options.</p> 
          <h3 id="GeneralTroubleshooting-Reason#6:">Reason #6:</h3> 
          <p>The <a rel="nofollow" href="http://www.microsoft.com/download/en/details.aspx?displaylang=en&amp;id=17718" title="Opens external link in new window" class="external-link">.NET Framework 4.0 Extended</a>&nbsp;package is needed but not installed. An indication for this problem typically is that controls can correctly be identified by the standalone Ranorex Spy application, but not by the integrated Ranorex Spy or Recorder in Ranorex Studio.The application under test uses the .NET Framework 4.0 and you may get Technology Limitation pop-ups saying that specific assemblies cannot be loaded.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Either install the <a rel="nofollow" href="http://www.microsoft.com/download/en/details.aspx?displaylang=en&amp;id=17718" title="Opens external link in new window" class="external-link">.NET Framework 4.0 Extended</a> or if that is not possible always use the standalone Ranorex Spy and Recorder tools instead of the integrated ones in Ranorex Studio.</p> 
          <h3 id="GeneralTroubleshooting-Reason#7:">Reason #7:</h3> 
          <p>If the application under test is built with the .NET Framework 4.0, a&nbsp;&lt;<em>executableName</em>&gt;.exe.config file (where &lt;<em>executableName</em>&gt; has to be replaced by the name of the Ranorex executable) needs to reside in the same folder as the automating Ranorex executable and it needs to contain the configuration below.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Create and add an app.config file with the below content to your project. Make sure that after compiling and when executing the executable there is a &lt;<em>executableName</em>&gt;.exe.config file in the same folder as the Ranorex executable with following content:</p> 
          <pre class="prettyprint lang-$params.language conf-macro output-block" data-hasbody="true" data-macro-name="sp-macrooverride-plaintextbody-block">&lt;configuration&gt;  
  &lt;startup&gt;  
    &lt;supportedRuntime version="v4.0" /&gt;  
  &lt;/startup&gt;  
&lt;/configuration&gt;  



</pre> 
         </div> 
         <div> 
          <h2 class="first-headline" id="GeneralTroubleshooting-.NETWPF">.NET WPF</h2> 
         </div> 
         <div> 
          <p>For compatibility with existing tests, Ranorex 5.3+ shows WPF windows twice in its tree; once using the WPF-specific plug-in, another time using UIAutomation. When selecting a specific element, the UIAutomation tree was chosen, although the native WPF plug-in is available and active.</p> 
          <h3 id="GeneralTroubleshooting-Reason#1:.1">Reason #1:</h3> 
          <p>While selecting an element of a WPF application in the element-tree of Spy, you selected the UIAutomation tree of that application.</p> 
          <p><strong>Solution:</strong></p> 
          <p>The preceding sibling of the selected window will identify the same window, but provide the "WpfElement" capability instead of "UiAutomation". When browsing elements in Spy, select elements below the "WpfElement"&nbsp;window.</p> 
         </div> 
         <div> 
          <h3 id="GeneralTroubleshooting-Reason#2:.1">Reason #2:</h3> 
          <p>Instrumentation of the WPF application has failed, because the application runs on a .NET 3.0 runtime, and .NET 3.5 is not installed.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Upgrade the .NET installation to version 3.5. No changes are required for the application under test.<br><br></p> 
         </div> 
         <div> 
          <h2 class="first-headline" id="GeneralTroubleshooting-.NETSilverlight">.NET Silverlight</h2> 
          <h3 id="GeneralTroubleshooting-Reason#1:.2">Reason #1:</h3> 
          <p>Ranorex is not able to access UI objects inside your Silverlight application.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Please make sure that the Windowless property of your Silverlight application is set to 'False'.<br><br></p> 
         </div> 
         <div> 
          <h2 class="first-headline" id="GeneralTroubleshooting-Flash/FlexApplications">Flash/Flex Applications</h2> 
          <p>After activating the Flash/Flex Plug-In for your application using the instrumentation wizard Ranorex is still not able to recognize Flash/Flex elements within your application.</p> 
          <h3 id="GeneralTroubleshooting-Reason#1:.3">Reason #1:</h3> 
          <p>The AllowScriptAccess parameter in the HTML code that loads a SWF file controls the ability to perform outbound URL access from within the SWF file.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Set this parameter inside the PARAM or EMBED tag to 'always'.</p> 
          <h3 id="GeneralTroubleshooting-Reason#2:.2">Reason #2:</h3> 
          <p>Your Flash/Flex application was instrumented twice using different versions of Ranorex Flash components.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Ensure that only one of the recommended <a data-macro-name="sp-plaintextbody-link" href="/help/latest/technology-instrumentation/flash-flex-testing" class="conf-macro output-inline" data-hasbody="true">instrumentation</a> approaches for Flash/Flex is used.<br><br></p> 
         </div> 
         <div> 
          <h2 class="first-headline" id="GeneralTroubleshooting-Win32BasedApplications">Win32 Based Applications</h2> 
          <p>There might be several reasons for limited support of Win32 based applications:</p> 
          <h3 id="GeneralTroubleshooting-Reason#1:.4">Reason #1:</h3> 
          <p>The automating process is not being run with&nbsp;the required security permissions to access the process under test. UI element identification capabilities for controls in that process are limited.</p> 
          <p><strong>Solution:</strong></p> 
          <p>The automating process (Ranorex executable) and the automated application (AUT) are not started with the same security rights, i.e. not as the same Windows user. Ensure that both processes are started with the same rights.</p> 
          <h3 id="GeneralTroubleshooting-Reason#2:.3">Reason #2:</h3> 
          <p>The automating and the automated process do not have the same bit width and Ranorex was unable to start the 32/64 Bit Bridge. UI element identification capabilities for controls in that process are limited.</p> 
          <p><br></p> 
          <p><strong>Solution:</strong></p> 
          <p>Please contact Ranorex&nbsp;<a rel="nofollow" class="external-link" href="/cdn-cgi/l/email-protection#d3a0a6a3a3bca1a793a1b2bdbca1b6abfdb0bcbe">support team</a> for further assistance.</p> 
          <h3 id="GeneralTroubleshooting-Reason#3:.1">Reason #3:</h3> 
          <p>The automating and the automated process do not have the same bit width and the 32/64 Bit Bridge is disabled. UI element identification capabilities for controls in that process are limited.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Enable the 32/64 Bit Bridge in the <a data-macro-name="sp-plaintextbody-link" href="/help/latest/ranorex-settings/settings-overview" class="conf-macro output-inline" data-hasbody="true">Ranorex Settings</a>.<br><br></p> 
         </div> 
         <div> 
          <h2 class="first-headline" id="GeneralTroubleshooting-Browser(MozillaFirefox,GoogleChrome,AppleSafari,MicrosoftInternetExplorer)">Browser (Mozilla Firefox, Google Chrome, Apple Safari, Microsoft Internet Explorer)</h2> 
          <h3 id="GeneralTroubleshooting-MozillaFirefox:">Mozilla Firefox:</h3> 
          <p>UI element identification capabilities for Mozilla Firefox are limited.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Please make sure that you have installed and enabled the Ranorex Addon in your Mozilla browser. Please refer to the <a data-macro-name="sp-plaintextbody-link" href="/help/latest/instrumentation-wizard/mozilla-firefox" class="conf-macro output-inline" data-hasbody="true">Mozilla Firefox instrumentation wizard documentation</a> for more information.</p> 
          <h3 id="GeneralTroubleshooting-GoogleChrome:">Google Chrome:</h3> 
          <p>UI element identification capabilities for Google Chrome are limited.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Please make sure that you have&nbsp;installed and activated the Ranorex Extension in your Google browser. Please refer to the <a data-macro-name="sp-plaintextbody-link" href="/help/latest/instrumentation-wizard/google-chrome" class="conf-macro output-inline" data-hasbody="true">Google Chrome instrumentation wizard documentation</a> for more information.</p> 
          <h3 id="GeneralTroubleshooting-AppleSafari:">Apple Safari:</h3> 
          <p>UI element identification capabilities for Apple Safari are limited.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Please make sure that you have&nbsp;installed and activated the Ranorex Extension in your Apple browser. Please refer to the <a data-macro-name="sp-plaintextbody-link" href="/help/latest/instrumentation-wizard/apple-safari" class="conf-macro output-inline" data-hasbody="true">Apple Safari instrumentation wizard documentation</a> for more information.</p> 
          <h3 id="GeneralTroubleshooting-MicrosoftInternetExplorer:">Microsoft&nbsp;Internet Explorer:</h3> 
          <p>UI element identification capabilities for&nbsp;Microsoft Internet Explorer&nbsp;are slow.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Please make sure that you have&nbsp;enabled the Ranorex Extension in your Internet Explorer (Manage Add-ons).<br><br></p> 
          <div class="rxtip conf-macro output-block" data-hasbody="true" data-macro-name="sp-macrooverride-richtextbody-block"> 
           <p><strong>Note</strong> The "Enhanced Protected Mode" needs&nbsp;to be disabled.</p> 
          </div> 
          <strong><br></strong> 
          <div class="rxtip conf-macro output-block" data-hasbody="true" data-macro-name="sp-macrooverride-richtextbody-block"> 
           <p><strong>Note</strong> The "Internet Explorer&nbsp;Enhanced Security Configuration" has to be disabled for the add-on to work on Windows Server machines.</p> 
          </div> 
          <br> 
          <br> 
         </div> 
         <div> 
          <h2 class="first-headline" id="GeneralTroubleshooting-ImageRelatedAutomationorValidation">Image Related Automation or Validation</h2> 
          <h3 id="GeneralTroubleshooting-Reason#1:.5">Reason #1:</h3> 
          <p>Ranorex's image based recording ability is limited because the asynchronous dispatching of input events is disabled.</p> 
          <p><strong>Solution:</strong></p> 
          <p>If you experience problems when making image based recordings, please enable asynchronous dispatching of mouse and keyboard events using the <a data-macro-name="sp-plaintextbody-link" href="/help/latest/ranorex-settings/settings-overview#SettingsOverview-AdvancedSettings" class="conf-macro output-inline" data-hasbody="true">Advanced Settings dialog</a>.<br><br></p> 
         </div> 
         <div> 
          <h2 class="first-headline" id="GeneralTroubleshooting-GeometryRelatedIssues">Geometry Related Issues</h2> 
          <h3 id="GeneralTroubleshooting-Reason#1:.6">Reason #1:</h3> 
          <p>Controls being displaced, staying empty, or being scrolled out of view during automation.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Set the 'Use Ensure Visible' property of the corresponding repository items (and their parent folders) to 'False' as described in the chapter <a data-macro-name="sp-plaintextbody-link" href="/help/latest/lesson-6-ui-mapping-with-ranorex-repository/repository-item-properties" class="conf-macro output-inline" data-hasbody="true">Repository Item Properties</a>.<br><br></p> 
         </div> 
         <div> 
          <h2 class="first-headline" id="GeneralTroubleshooting-WindowsApps(akaWindows8UIApps)">Windows Apps (aka Windows 8 UI Apps)</h2> 
          <p>There are a few reasons causing Windows 8 UI&nbsp;based applications not to be recognized correctly:</p> 
          <h3 id="GeneralTroubleshooting-Reason#1:.7">Reason #1:</h3> 
          <p>Test execution is started using a Ranorex tool and the setting 'Use UiaLauncher to elevate privileges for processes started by tools' ('General Settings' dialog -&gt; 'Advanced' tab) is&nbsp;disabled.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Enable the mentioned setting.</p> 
          <h3 id="GeneralTroubleshooting-Reason#2:.4">Reason #2:</h3> 
          <p>Ranorex is not installed in a secure location.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Install Ranorex to 'Program Files' or 'Program Files (x86)'.</p> 
          <h3 id="GeneralTroubleshooting-Reason#3:.2">Reason #3:</h3> 
          <p>The test executable&nbsp;is not started by a Ranorex Tool.</p> 
          <p><strong>Solution #1:</strong></p> 
          <p>Start the test suite using the TestSuiteRunner.</p> 
          <p><strong>Solution #2:</strong></p> 
          <p>Use the Ranorex UiaLauncher (&lt;Ranorex Installation Folder&gt;\bin\Ranorex.UiaLauncher32.exe) to start the test executable by passing the executable name as the first argument.</p> 
          <p>E.g. <em>Ranorex.UiaLauncher32.exe &lt;Solution Folder&gt;\bin\debug\&lt;Project Executable&gt;.exe</em>.</p> 
          <p><strong>Solution #3:</strong></p> 
          <p>Add the following lines of code to the Program.cs/Program.vb of the main (executable) project:<br><br></p> 
          <div class="dynamic-tab-pane-control tab-pane">

          <h3 id="GeneralTroubleshooting-Reason#4:.1">Reason #4:</h3> 
          <p>You checked the "Run this program as an administrator" in the Compatibility settings of a Ranorex Tool EXE file, e.g. for the RanorexStudio.exe file. Setting this option disables access to Windows 8 UI apps.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Do not check the mentioned setting for any of the Ranorex EXE files. Alternatively, if you want to always run the tool using administrator rights, you can create a shortcut to the EXE file and check the "Run as administrator" option in the "Shortcut" -&gt; "Advanced..." settings of the shortcut properties.<br><br></p>

          <h2 class="first-headline" id="GeneralTroubleshooting-OtherSoftware">Other Software</h2> 
          <p>Running some antivirus or security software might&nbsp;lead to a limited object recognition with Ranorex tools.</p> 
          <h3 id="GeneralTroubleshooting-Reason#1:.8">Reason #1:</h3> 
          <p>The antivirus or security software blocks certain Ranorex functionality.</p> 
          <p><strong>Solution:</strong></p> 
          <p>Add an exception for the appropriate Ranorex process or temporarily switch off the specific security application.</p>

         <h2 class="first-headline" id="GeneralTroubleshooting-DifferentDisplayDPIScalingValues">Different Display DPI Scaling Values</h2> 
         <p>By default, Microsoft Windows is automatically adapting the DPI scaling value for each of your physical displays connected to your machine. This ensures a uniform experience over all displays, even if the displays have a different physical size.</p> 
         <p>When using Ranorex, all physical displays have to be set to the same DPI scaling value.</p> 
         <h3 id="GeneralTroubleshooting-SolutionforWindows8.1">Solution&nbsp;for Windows 8.1</h3> 
         <ol> 
          <li>On your Windows 8.1 machine, open the Control Panel and switch to section 'Display'. Do not use the application 'PC settings'.<br><br> Make sure the option 'Let me choose one scaling level for all my Displays' is checked.<br><img alt="Activating the option 'Let me choose one scaling level for all my Displays'" width="700" data-macro-name="sp-image" class="confluence-embedded-image conf-macro output-inline" src="/help/latest/files/2720743/3868900/1/1494496032476/03-DPI-Scaling_Values_Windows_81_Settings.png" data-hasbody="false"><br>&nbsp;</li> 
          <li>After checking this option, choose the size category you prefer.<br><img alt="Choosing the scaling size" width="700" data-macro-name="sp-image" class="confluence-embedded-image conf-macro output-inline" src="/help/latest/files/2720743/3868901/1/1494496032482/04-DPI-Scaling_Values_Windows_81_Settings_One_Scaling.png" data-hasbody="false"><br><br>Press 'Apply', sign-out and sign-in again to your machine to activate the new settings.</li> 
         </ol> 
         <p>The technology limitation warning should not appear again.</p> 
         <h3 id="GeneralTroubleshooting-SolutionforWindows10">Solution&nbsp;for Windows 10</h3> 
         <ol> 
          <li>On your Windows 10 machine, open the Windows start menu and choose 'Settings'.</li> 
          <li>Click on 'System'.<br><img alt="The System icon in the Settings window" data-macro-name="sp-image" class="confluence-embedded-image conf-macro output-inline" src="/help/latest/files/2720743/3868898/1/1494496032466/01-DPI_Scaling_Values_Windows_10_Settings.png" data-hasbody="false"><br>&nbsp;</li> 
          <li><p>The section 'Display' should be shown automatically – if not, open it manually.&nbsp;<br>&nbsp;<br> Make sure that all your physically connected displays have the same setting at 'Change the size of text, apps, and other items'.&nbsp;&nbsp;<br>Move through the list of displays (labelled with numbers) one by one, and check the setting.<br> When changing a display value, press 'Apply' before moving on to the next display.<br><img alt="Customizing the scaling for multiple displays" data-macro-name="sp-image" class="confluence-embedded-image conf-macro output-inline" src="/help/latest/files/2720743/3868899/1/1494496032471/02-DPI_Scaling_Values_Windows_10_Settings_Display.png" data-hasbody="false"></p></li> 
          <li><p>Once all changes have been applied, you can close the 'Settings' window.</p></li> 
         </ol> 
         <p>The technology limitation warning should not appear again.</p>
