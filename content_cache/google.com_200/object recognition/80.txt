      <p>**</p> 
      <p><span class="bbcode-size-200">JavaScript Ultimate</span></p> 
      <p>**</p> 
      <p>JavaScript Ultimate is a <strong>new syntax definition for JavaScript</strong>, which will **replace your default **one (also, this is considered a complete upgrade from <a href="https://packagecontrol.io/packages/Better%20JavaScript" rel="nofollow">Better JavaScript</a>).</p> 
      <p>(Read the official <a href="https://github.com/JoshuaWise/javascript-ultimate" rel="nofollow">Readme on Github</a>)</p> 
      <p>Here are some of the more <strong>obvious insufficiencies</strong> of every other JavaScript syntax out there:</p> 
      <ul> 
       <li>dollar signs ($) are not treated as word-characters. This means that in patterns like <em>$true</em><br>, the <em>true</em> part is treated as a constant, instead of part of the identifier</li> 
       <li><p>unicode identifiers are not supported</p></li> 
       <li><p>they often don't detect current reserved words</p></li> 
       <li><p>no support for modern DOM APIs (canvas, SVG, etc.)</p></li> 
       <li><p>they often don't follow the ECMAScript 5 spec accurately, if at all</p></li> 
       <li><p>in patterns like <em>myObject.true</em><br>, they think that <em>true</em> is a constant, as opposed to an identifier</p></li> 
       <li><p>no JavaDoc support</p></li> 
       <li><p>regular expression literals aren't detected in some valid contexts</p></li> 
       <li><p>many, many more</p></li> 
      </ul> 
      <p>Built from the ground up, this syntax definition fixes all of those things, which no JavaScript syntax does right now:</p> 
      <ul> 
       <li>all valid JavaScript identifiers are detected, with <strong>full Unicode support</strong><br>(and the dollar sign is treated as a word-character)</li> 
       <li><p>updated DOM library; obsolete items removed, added support for Canvas, SVG functions, and more</p></li> 
       <li><p>Includes all features of ECMAScript 5, and some of ES6 (will make an ES6 version when the spec if finalized)</p></li> 
       <li><p>added JavaDoc support</p></li> 
       <li><p>many, many more</p></li> 
      </ul> 
      <p>Until this package, <a href="https://github.com/Benvie/JavaScriptNext.tmLanguage" rel="nofollow">JavaScript Next</a> was probably the best JavaScript syntax available. That one is still better for you <strong><em>if</em></strong> you actually use ES6 features (i.e., you code in <a href="https://github.com/google/traceur-compiler" rel="nofollow">traceur</a>).<br>Otherwise, this one is more desirable, as it's <em>much</em> more accurate.</p> 
      <p>Additionally, this syntax actually comes with two syntaxes:</p> 
      <ul> 
       <li><p><em>JavaScript</em></p></li> 
       <li><p><em>JavaScript (DOM)</em></p></li> 
      </ul> 
      <p>So when you're coding server-side JS, you don't need to falsely highlight DOM functions.</p> 
      <p>There's a <em>JavaScript (Node)</em> on the way, so all your <em>__dirname</em>s and <em>require()</em>s can be highlighted too.</p> 
      <p><span class="bbcode-size-150">Install through Package Control</span></p> 
      <p><span class="bbcode-size-150">Github: <a href="https://github.com/JoshuaWise/javascript-ultimate" rel="nofollow">https://github.com/JoshuaWise/javascript-ultimate</a></span></p>

      <p>There are only so many scopes that are recognized by most color schemes. Unless you used one of the color schemes that comes with JavaScript Ultimate (or modified your own), either custom functions or built-in functions would have to share the same color as something else. I may look into this further.</p> 
      <p>[quote="iamntz"]<br>Also, could you quickly explain this kind of sorcery?<br></p> 
      <p></p>
      <pre><code>(?&amp;lt;!\p{L}\p{Nl}$\p{Mn}\p{Mc}\p{Nd}\p{Pc}_\x{200C}\x{200D}])\p{L}\p{Nl}$_]\p{L}\p{Nl}$\p{Mn}\p{Mc}\p{Nd}\p{Pc}_\x{200C}\x{200D}]*+(?=\s*+\()</code></pre> 
      <p>[/quote]</p> 
      <p>That is the sorcery of Unicode regular expressions in Oniguruma (the regexp engine used by Sublime Text). <br>The <a href="http://www.ecma-international.org/ecma-262/5.1/#sec-7.6" rel="nofollow">JavaScript spec</a> specifies what characters are legal as identifier names (i.e. variable names, etc.). Most language syntaxes only support the valid characters that lie within the ASCII range. JavaScript Ultimate detects all valid identifiers in JavaScript.</p> 
      <p><strong>Translations</strong><br><em>(?&lt;!\p{L}\p{Nl}$\p{Mn}\p{Mc}\p{Nd}\p{Pc}_\x{200C}\x{200D}])</em><br>Preceding text mustn't contain a valid identifier character.</p> 
      <p><em>\p{L}\p{Nl}$<em>]\p{L}\p{Nl}$\p{Mn}\p{Mc}\p{Nd}\p{Pc}</em>\x{200C}\x{200D}]</em>+*<br>Text must be a valid identifier.</p> 
      <p><em>(?=\s</em>+()*<br>Text must be followed by any number of white space, and then an open parenthesis.</p>
