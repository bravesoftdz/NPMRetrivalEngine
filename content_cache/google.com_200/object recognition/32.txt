The blog announcement can be found <a class="jive-link-external-small" href="https://communities.ca.com/external-link.jspa?url=https%3A%2F%2Fwww.ca.com%2Fen%2Fblog-agile-requirements-designer%2Fautomate-test-automation-script-creation-for-packaged-apps.html" target="_blank">here</a>.</p>
         <p><a href="https://communities.ca.com/servlet/JiveServlet/showImage/2-242010900-135738/Ranorex+Short.jpg"><img alt="" class="image-1 jive-image j-img-centered j-img-original" height="89" src="https://communities.ca.com/servlet/JiveServlet/downloadImage/2-242010900-135738/Ranorex+Short.jpg" style="height: auto; display: block; margin-left: auto; margin-right: auto;" width="240"></a></p>
         <h1>Technical Integration</h1>
         <p>Continuous Delivery isn’t possible without automation, especially test automation. If test cases and scripts are constantly being designed and rewritten, teams are not realizing their full, agile potential. CA Agile Requirements Designer provides a powerful platform to automatically generate and manage optimized test cases and test automation scripts directly from defined requirements. Ranorex is a powerful UI object recognition and test automation engine built tackle the challenges of manual testing. Together these solutions shift testing left like never before. Instead of designing test case strategies, users can focus on designing rich requirements. Instead of writing automation code, users can scan an application’s objects and actions to generate automation script without typing a single command. When it comes to bug fixes and new versions, both test cases and automation scripts are identified as effected by changes and regenerated. Not only are QA cycles shortened, processes withheld until after code is released to QA are taking place in the design and planning stages allowing for more parallel, in-sprint work streams.</p>
         <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
         <h2>How it works</h2>
         <p>Ranorex allows users to automate UI testing for web and mobile applications as well as packaged apps like SAP, Oracle EBS, and the like. Users scan an application’s UI to identify and track objects and build their associated action scripts. The result is automated workflows. This allows customers to automate their regression testing suites while saving manual testing for exploratory testing only.&nbsp;</p>
         <p>With CA Agile Requirements Designer users are focused on the requirements before the code is written. In using a visual model, teams are aligned with requirements, test cases, expected results, and acceptance criteria. So, by linking Ranorex objects and actions to the nodes in a CA Agile Requirements Designer model we are continuing to reinforce Test Design Automation by <span style="font-size: 14.0pt;"><strong>eliminating the need to manually build and maintain automation code!</strong></span></p>
         <h2>Architectural Overview</h2>
         <p>&nbsp;</p>
         <p><a href="https://communities.ca.com/servlet/JiveServlet/showImage/2-242010900-135749/Ranorex+Integration+Architecture.png"><img alt="" class="jive-image image-2 j-img-centered j-img-original" height="878" src="https://communities.ca.com/servlet/JiveServlet/downloadImage/2-242010900-135749/Ranorex+Integration+Architecture.png" style="width: 620px; height: 349px; display: block; margin-left: auto; margin-right: auto;" width="1560"></a></p>
         <p>&nbsp;</p>
         <p>This new integration adds a critical piece to a much bigger automation story.</p>
         <ol>
          <li><strong>Manage your user stories through CA Agile Central</strong> or any other widely used agile application lifecycle management tools.</li>
          <li><strong>Import the user story into CA Agile Requirements Designer</strong> and model the requirements. The model itself reduces up to half of all defects by eliminating communication gaps found in relying solely on written and spoken word.</li>
          <li><strong>Link Ranorex objects</strong> along with their actions directly from the repository to the model’s nodes or blocks, all within CA Agile Requirements Designer.</li>
          <li><strong>Create additional test assets</strong>. CA Agile Requirements Designer can be used to create virtualized endpoints, test data, and the expected results needed for automated testing.</li>
          <li><strong>Sync optimized test cases back to the user story</strong>. CA Agile Requirements Designer optimizes the paths through the model to give you the fewest number of test cases for the maximum functional coverage.</li>
          <li><strong>Import the automation scripts</strong>. Based of the linked Ranorex objects and actions CA Agile Requirements Designer compiles the automation scripts necessary to run each test case. These are then imported and run through Ranorex</li>
          <li><strong>Sync and analyze the results</strong>. Immediately identify where failures occur and pass the results back to CA Agile Central for further review.</li>
          <li><strong>Rinse and Repeat</strong> - Reuse or update test artifacts when applications change. CA Agile Requirements Designer actively adapts to changes in a requirements model, saving hours, if not days, of manual effort when compared to traditional methods.</li>
          <li><strong>Automate it all</strong>. With CA Continuous Delivery Director manual handoffs and kickoffs can be orchestrated to run automatically without human interaction.</li>
         </ol>
         <p style="padding-left: 60px;">Once the code is checked in, CA CDD can:</p>
         <p style="padding-left: 90px;">Promote the code to the QA environment</p>
         <p style="padding-left: 90px;">Spin up virtual services</p>
         <p style="padding-left: 90px;">Provision test data</p>
         <p style="padding-left: 90px;">Pick up the automation scripts</p>
         <p style="padding-left: 90px;">Run them through Ranorex</p>
         <p style="padding-left: 90px;">Pass feedback through to the user story</p>
         <p style="padding-left: 90px;">Send alerts to the dashboard</p>
         <p style="padding-left: 90px;">Deploy successes to production</p>
         <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
         <p><strong>Now that’s Continuous Delivery!!!</strong></p>
         <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
         <h2>Added Value</h2>
         <p>CA Agile Requirements Designer gives users the ability to truly “shift left” their QA activities, all the way back to requirements gathering. Testers and developers now have the advantage of knowing exactly what will be tested before coding begins. This strengthens the coding by enabling Test-Driven Development (TDD) and Acceptance Test-Driven Development (ATDD) adoption, and it accelerates functional testing.</p>
         <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
         <p>Only occasionally are teams working with net-new applications. In most cases, the application already exists and so does the Ranorex repository containing its tracked objects. All that’s missing is the model to link everything and everyone together and manage testing effectively. Once requirements are modeled, CA Agile Requirements Designer produces the optimal set of test cases, reducing over/under-testing, and maximizes test coverage with the minimum amount of test cases possible. Because CA Agile Requirements designer is active requirements modeling tool, both test cases and automation scripts dynamically adapt to new versions. The result is full-functional automation testing driven by intentional software design – fewer defects, less rework, lower costs, a shorter time to market, and increased quality.</p>
