      <h2>1. Overview</h2> 
      <p>ViSearch is an API that provides accurate, reliable and scalable image search. ViSearch API provides endpoints that let developers index their images and perform image searches efficiently. ViSearch API can be easily integrated into your web and mobile applications. More details about ViSearch API can be found in the <a href="http://www.visenze.com/docs/overview/introduction">documentation</a>.</p> 
      <p>The ViSearch Javascript SDK is an open source software for easy integration of ViSearch Search API with your application server. It provides three search methods based on the ViSearch Search API - pre-indexed search, color search and upload search. For source code and references, visit the github <a href="https://github.com/visenze/visearch-sdk-javascript">repository</a>.</p> 
      <p>Our latest stable version: <img src="https://img.shields.io/npm/v/visearch-javascript-sdk.svg?style=flat" alt="npm version"></p> 
      <h2>2. Setup and initialization</h2> 
      <p>Paste the following snippet into the header of your site.</p> 
      <p>This snippet will load visearch.js onto the page asynchronously, so it won뭪 affect your page load speed.</p> 
      <p>Replace "YOUR_APP_KEY" with your ViSearch Client-side App Key. It is recommended to initiate the client when the SDK is loaded into the page.</p> 
      <p>Your credentials can be found in <a href="https://dashboard.visenze.com">ViSearch Dashboard</a></p> 
      <pre><code class="language-html">&lt;script type="text/javascript"&gt;

      <pre><code class="language-shell">npm install visearch-javascript-sdk --save

      <h2>3. Searching Images</h2> 
      <p>Any search api request should follow the pattern below.</p> 
      <pre><code class="language-javascript">visearch.[API_METHOD](parameters, // parameter objects.
  function(res) {
    /**
    this is response handler, will callback this function after receiving the http response.
    please add your code here to process the http response.
    **/
});
</code></pre> 
      <ul> 
       <li><code>parameters</code></li> 
      </ul> 
      <blockquote> 
       <p>Please refer <a href="https://developers.visenze.com/http/#Search-API">Parameters In ViSearch</a> for the detail of each supported parameters.</p> 
      </blockquote> 
      <ul> 
       <li><code>callback</code> function</li> 
      </ul> 
      <blockquote> 
       <p>A JSON object of the search response will be passed into this callback function. please refer <a href="https://developers.visenze.com/http/#Search-Responses">Search Responses</a> for details of the object.</p> 
      </blockquote> 
      <ul> 
       <li><code>error</code> function</li> 
      </ul> 
      <blockquote> 
       <p>This function will be callback if any client exception raised when proceed API calling. an <code>error</code> parameter will be passed in, the structure of this <code>error</code> parameter will be like this:</p> 
      </blockquote> 
      <pre><code>{
  "message": "Failed to fetch"
}
</code></pre> 
      <h3>3.1 Pre-indexed Search</h3> 
      <p>Pre-index search is to search similar images based on the your indexed image by its unique identifier (im_name). It should be a valid ID that is used to index your images in the database.</p> 
      <pre><code class="language-javascript">var imageId = "yoox-38377218";
visearch.idsearch({
  im_name: imageId,
  fl: ["im_url"]
}, function(res) {
  // @TODO
  // some response handler
});
</code></pre> 
      <h3>3.2 Color Search</h3> 
      <p>Color search is to search images with similar color by providing a color code. The color code should be in Hexadecimal and passed to the colorsearch service.</p> 
      <pre><code class="language-javascript">var colorHexFormat = "fa4d4d";
visearch.idsearch({
  color: colorHexFormat,
  fl: ["im_url"]
}, function(res) {
  // @TODO
  // some response handler
});
</code></pre> 
      <h3>3.3 Upload Search</h3> 
      <p>Upload search is used to search similar images by uploading an image or providing an image url.</p> 
      <ul> 
       <li>Using an image from a local file path:</li> 
       <li><code>image</code> parameter must be a html file Object. you can get this object with follow method: 
        <ul> 
         <li>Raw Javascript: <code>document.getElementById("fileInputId")</code></li> 
         <li>JQuery style: <code>$("#fileInputId")[0]</code></li> 
        </ul> </li> 
      </ul> 
      <p>Sample code for upload search with html form:</p> 
      <pre><code class="language-html">&lt;form&gt;
  Upload image: &lt;input type="file" id="fileUpload" name="fileInput"&gt;&lt;br&gt;
  &lt;input type="submit" value="Submit"&gt;
&lt;/form&gt;
</code></pre> 
      <pre><code class="language-javascript">var inputImageFile = document.getElementById("fileUpload");
visearch.uploadsearch({
  image: inputImageFile, //html file input object
  fl: ["im_url"]
}, function(res) {
  // @TODO
  // some response handler
});
</code></pre> 
      <ul> 
       <li>Alternatively, you can pass an image url directly to uploadsearch to start the search.</li> 
      </ul> 
      <pre><code class="language-javascript">visearch.uploadsearch({
  im_url: "", // image url string
  fl: ["im_url"]
}, function(res) {
  // @TODO
  // some response handler
});
</code></pre> 
      <p>You can also search multiple objects within an image:</p> 
      <pre><code class="language-javascript">visearch.similarproducts({
  im_url: "", // image url string
  fl: ["im_url"]
}, function(res) {
  // @TODO
  // some response handler
});
</code></pre> 
      <h4>3.3.1 Selection Box</h4> 
      <p>If the object you wish to search for takes up only a small portion of your image, or other irrelevant objects exists in the same image, chances are the search result could become inaccurate. Use the Box parameter to refine the search area of the image to improve accuracy. Noted that the box coordinated needs to be set with respect to the original size of the image passed.</p> 
      <pre><code class="language-javascript">visearch.uploadsearch({
  im_url: "", // image url string
  // input a string array of [x1, y1, x2, y2]
  // where (0,0) is the top-left corner
  // of the image, (x1, y1) is the top-left corner of the box,
  // and (x2, y2) is the bottom-right corner of the box.
  box: [x1, y1, x2, y2],
  fl: ["im_url"]
}, function(res) {
  // @TODO
  // some response handler
});
</code></pre> 
      <h2>4. Search Results</h2> 
      <p>ViSearch returns a maximum number of 1000 most relevant image search results. You can provide pagination parameters to control the paging of the image search results.</p> 
      <p>Pagination parameters:</p> 
      <table> 
       <thead> 
        <tr> 
         <th>Name</th> 
         <th>Type</th> 
         <th>Description</th> 
        </tr> 
       </thead> 
       <tbody> 
        <tr> 
         <td>page</td> 
         <td>Integer</td> 
         <td>Optional parameter to specify the page of results. The first page of result is 1. Defaults to 1.</td> 
        </tr> 
        <tr> 
         <td>limit</td> 
         <td>Integer</td> 
         <td>Optional parameter to specify the result per page limit. Defaults to 10.</td> 
        </tr> 
       </tbody> 
      </table> 
      <pre><code class="language-javascript">visearch.uploadsearch({
  im_url: "", // image url string,
  page: 1,
  limit: 25,
  fl: ["im_url"]
}, function(res) {
  // @TODO
  // some response handler
});
</code></pre> 
      <h2>5. Advanced Search Parameters</h2> 
      <h3>5.1 Retrieving Metadata</h3> 
      <p>To retrieve metadata of your image results, provide the list of metadata keys for the metadata value to be returned in the <code>fl</code> (field list) property:</p> 
      <pre><code class="language-javascript">visearch.uploadsearch({
  im_url: "", // image url string,
  fl: ["price","brand","title","im_url"] //list of field list to be returned
}, function(res) {
  // @TODO
  // some response handler
});
</code></pre> 
      <p>To retrieve all metadata of your image results, specify <code>get_all_fl</code> parameter and set it to <code>True</code>:</p> 
      <pre><code class="language-javascript">visearch.uploadsearch({
  im_url: "", // image url string,
  get_all_fl: true
}, function(res) {
  // @TODO
  // some response handler
});
</code></pre> 
      <blockquote> 
       <p>Only metadata of type string, int, and float can be retrieved from ViSearch. Metadata of type text is not available for retrieval.</p> 
      </blockquote> 
      <h3>5.2 Filtering Results</h3> 
      <p>To filter search results based on metadata values, provide a string array of metadata key to filter value in the <code>fq</code> (filter query) property:</p> 
      <pre><code class="language-javascript">var filterQuery = ['description : bag', 'brand : nike'];
visearch.uploadsearch({
  im_url: "", // image url string,
  fq: filterQuery
}, function(res) {
  // @TODO
  // some response handler
});
</code></pre> 
      <p>Querying syntax for each metadata type is listed in the following table:</p> 
      <table> 
       <thead> 
        <tr> 
         <th>Type</th> 
         <th>FQ</th> 
        </tr> 
       </thead> 
       <tbody> 
        <tr> 
         <td>string</td> 
         <td>Metadata value must be exactly matched with the query value, e.g. "Vintage Wingtips" would not match "vintage wingtips" or "vintage"</td> 
        </tr> 
        <tr> 
         <td>text</td> 
         <td>Metadata value will be indexed using full-text-search engine and supports fuzzy text matching, e.g. "A pair of high quality leather wingtips" would match any word in the phrase</td> 
        </tr> 
        <tr> 
         <td>int</td> 
         <td>Metadata value can be either: 
          <ul>
           <li>exactly matched with the query value</li>
           <li>matched with a ranged query <code>minValue,maxValue</code>, e.g. int value <code>1, 99</code>, and <code>199</code> would match ranged query <code>0,199</code> but would not match ranged query <code>200,300</code></li>
          </ul></td> 
        </tr> 
        <tr> 
         <td>float</td> 
         <td>Metadata value can be either 
          <ul>
           <li>exactly matched with the query value</li>
           <li>matched with a ranged query <code>minValue,maxValue</code>, e.g. float value <code>1.0, 99.99</code>, and <code>199.99</code> would match ranged query <code>0.0,199.99</code> but would not match ranged query 200.0,300.0</li>
          </ul></td> 
        </tr> 
       </tbody> 
      </table> 
      <h3>5.3 Result Score</h3> 
      <p>ViSearch image search results are ranked in descending order i.e. from the highest scores to the lowest, ranging from 1.0 to 0.0. By default, the score for each image result is not returned. You can turn on the <code>score</code> property to retrieve the scores for each image result:</p> 
      <pre><code class="language-javascript">visearch.uploadsearch({
  im_url: "", // image url string,
  score: true
}, function(res) {
  // @TODO
  // some response handler
});
</code></pre> 
      <p>If you need to restrict search results from a minimum score to a maximum score, specify the <code>score_min</code> and/or <code>score_max</code> parameters:</p> 
      <table> 
       <thead> 
        <tr> 
         <th>Name</th> 
         <th>Type</th> 
         <th>Description</th> 
        </tr> 
       </thead> 
       <tbody> 
        <tr> 
         <td>score_min</td> 
         <td>Float</td> 
         <td>Minimum score for the image results. Default is 0.0.</td> 
        </tr> 
        <tr> 
         <td>score_max</td> 
         <td>Float</td> 
         <td>Maximum score for the image results. Default is 1.0.</td> 
        </tr> 
       </tbody> 
      </table> 
      <pre><code class="language-javascript">visearch.uploadsearch({
  im_url: "", // image url string,
  score: true,
  min_score: 0.2,
  max_score: 0.5
}, function(res) {
  // @TODO
  // some response handler
});
</code></pre> 
      <h3>5.4 Automatic Object Recognition Beta</h3> 
      <p>With Automatic Object Recognition, ViSearch /uploadsearch API is smart to detect the objects present in the query image and suggest the best matched product type to run the search on.</p> 
      <p>You can turn on the feature in upload search by setting the API parameter "detection=all". We are now able to detect various types of fashion items, including <code>Top</code>, <code>Dress</code>, <code>Bottom</code>, <code>Shoe</code>, <code>Bag</code>, <code>Watch</code> and <code>Indian Ethnic Wear</code>. The list is ever-expanding as we explore this feature for other categories.</p> 
      <p>Notice: This feature is currently available for fashion application type only. You will need to make sure your app type is configurated as "fashion" on <a href="https://developers.visenze.com/setup/#Choose-Your-Application-Type">ViSenze dashboard</a>.</p> 
      <pre><code class="language-javascript">visearch.uploadsearch({
  im_url: "", // image url string,
  score: true,
  detection: 'all'
}, function(res) {
  // @TODO
  // some response handler
});
</code></pre> 
      <p>You could also recognize objects from a paticular type on the uploaded query image through configuring the detection parameter to a specific product type as "detection={type}". Our API will run the search within that product type.</p> 
      <p>Sample request to detect <code>bag</code> in an uploaded image:</p> 
      <pre><code class="language-javascript">visearch.uploadsearch({
  im_url: "", // image url string,
  score: true,
  detection: 'bag'
}, function(res) {
  // @TODO
  // some response handler
});
</code></pre> 
      <p>The detected product types are listed in <code>product_types</code> together with the match score and box area of the detected object. Multiple objects can be detected from the query image and they are ranked from the highest score to lowest. The full list of supported product types by our API will also be returned in <code>product_types_list</code>.</p> 
      <h2>6. Event Tracking</h2> 
      <p>ViSearch Javascript SDK provides methods to understand how your customer interact with the search results.</p> 
      <p>In addition, to improve subsequent search quality, it is recommended to send user actions when they interact with the results.</p> 
      <h3>6.1 Event handler</h3> 
      <p>User action can be sent through an event handler. Register an event handler to the element in which the user will interact.</p> 
      <pre><code class="language-javascript">visearch.send({
  reqid: "xxxxxx",
  im_name: "xxxxx",
  action: "action type"
});
</code></pre> 
      <ul> 
       <li><code>reqid</code> The request id of the search request. This <code>reqid</code> can be obtained from the all search responses.</li> 
      </ul> 
      <pre><code class="language-javascript">visearch.idsearch({
  // request parameters
}, function(res) {
  // some response handler
  var reqid=res.reqid
});
</code></pre> 
      <ul> 
       <li><code>action</code> The action type of this event. We are currently able to support "click". More actions will be supported in the future.</li> 
      </ul> 
      <pre><code class="language-javascript">visearch.send({
  reqid: "xxxxxx",
  im_name: "xxxxx",
  action: "click"
});
</code></pre> 
      <ul> 
       <li><code>im_name</code> The im_name of the image which the user has clicked on. im_name is the unique identifier of the indexed image, in this case the searched result.</li> 
      </ul> 
      <h3>6.2 Tracking via url</h3> 
      <p>The SDK provides another alternative to event tracking which is via url. Append the following url requests into the links where the user will interact.</p> 
      <pre><code class="language-html">?reqid={reqid}&amp;im_name={im_name}
</code></pre> 
      <p>When the link is clicked, the system will detect the user interaction.</p> 
      <pre><code class="language-html">&lt;a href="domain/path?reqid={reqid}&amp;im_name={im_name}"&gt;text&lt;/a&gt;
</code></pre> 
      <h2>7 FAQ</h2> 
      <ul> 
       <li>When performing upload search, you may notice the increased search latency with increased image file size. This is due to the increased time spent in network transferring your images to the ViSearch server, and the increased time for processing larger image files in ViSearch.</li> 
      </ul>
