        <p>In this post, we will interface an Ultrasonic sensor –&nbsp;HC-SR04 and Raspberry Pi B+. And then we will write a Node.js program to detect the distances between an object and the Ultrasonic sensor. We will be using an awesome Node.js module named&nbsp;<a href="https://www.npmjs.org/package/r-pi-usonic" onclick="__gaTracker('send', 'event', 'outbound-article', 'https://www.npmjs.org/package/r-pi-usonic', 'r-pi-usonic');" title="r-pi-usonic" target="_blank">r-pi-usonic</a>&nbsp;to interact with the Ultrasonic sensor from our Pi.</p> 
        <p>We will be implementing 2 variations of measuring distances using the Ultrasonic sensor. One variation is to get the distance on demand. And the second one is to continuously measure the distance.</p> 
        <p>Below is a quick demo on</p>

        <p style="text-align: center;"> 
         <!-- iframe plugin v.4.2 wordpress.org/plugins/iframe/ --> <iframe width="640" height="360" src="//www.youtube.com/embed/tlt7-GPowuU?rel=0" frameborder="0" allowfullscreen scrolling="yes" class="iframe-class"></iframe> </p> 
        <p style="text-align: left;">You can find the completed code <a href="https://github.com/arvindr21/pi_ultraSonicSensor" onclick="__gaTracker('send', 'event', 'outbound-article', 'https://github.com/arvindr21/pi_ultraSonicSensor', 'here');" title="code" target="_blank">here</a>.</p>

        <p>If you are new to Raspberry pi and have not yet installed Node.js on it, I would recommend going through <a title="Getting Started with Raspberry pi and Node.js" href="http://thejackalofjavascript.com/getting-started-raspberry-pi-node-js/" target="_blank">Getting Started with Raspberry pi and Node.js</a>.</p> 
        <p>If you are new to electronics devices and circuits, I would recommend going through the&nbsp;<a href="http://www.allaboutcircuits.com/videos/index.html" onclick="__gaTracker('send', 'event', 'outbound-article', 'http://www.allaboutcircuits.com/videos/index.html', 'video lectures');" title="Video Lectures" target="_blank">video lectures</a> from All About Circuits.</p>

        <ol> 
         <li>1 – Raspberry pi B+</li> 
         <li>1 – Breadboard</li> 
         <li>1 – 1K Ohm resistor</li> 
         <li>1 –&nbsp;HC-SR04 Ultrasonic sensor</li> 
         <li>4 – Jumper wires</li> 
        </ol> 
        <p>If you are new to Raspberry Pi GPIO, please refer <a title="GPIO" href="http://thejackalofjavascript.com/raspberry-pi-node-js-led-emit-morse-code/#gpio" target="_blank">this</a>.</p> 
        <h2>Understanding HC-SR04 Ultrasonic Sensor</h2> 
        <p>HC-SR04 sensor works with a power of 5v. It has a range of approximately 2cm to 4m. It outputs a frequency of 40khz, twice the human ear can hear. (<em>more info on the sensor <a href="http://www.micropik.com/PDF/HCSR04.pdf" onclick="__gaTracker('send', 'event', 'download', 'http://www.micropik.com/PDF/HCSR04.pdf');" title="HCSR04" target="_blank">here</a></em>)</p> 
        <p><a href="http://i0.wp.com/thejackalofjavascript.com/wp-content/uploads/2014/11/hcsr04-e1416573917986.png" data-rel="lightbox-0" title=""><img class="wp-image-2513 size-medium aligncenter" src="http://i0.wp.com/thejackalofjavascript.com/wp-content/plugins/lazy-load/images/1x1.trans.gif" data-lazy-src="http://i2.wp.com/thejackalofjavascript.com/wp-content/uploads/2014/11/hcsr04-e1416573917986-451x297.png?resize=451%2C297" alt="" data-recalc-dims="1">
          <noscript>
           <img class="wp-image-2513 size-medium aligncenter" src="http://i2.wp.com/thejackalofjavascript.com/wp-content/uploads/2014/11/hcsr04-e1416573917986-451x297.png?resize=451%2C297" alt="" data-recalc-dims="1">
          </noscript></a></p> 
        <p>As you can see from the above diagram, the HC-SR04 sensor has 4 pins.</p> 
        <ol> 
         <li>VCC&nbsp;&nbsp;: +5v power supply</li> 
         <li>Trigger :&nbsp;A high input to this pin for a duration of 10 micro seconds, activates the sensor</li> 
         <li>Echo :&nbsp;This pin returns the time taken&nbsp;for the transmitted signal to come back after hitting an object.</li> 
         <li>Ground :&nbsp;Connected to the&nbsp;ground</li> 
        </ol> 
        <p>So, this is how the sensor works. First, we send&nbsp;a high signal to the trigger pin for a duration of 10 micro seconds. This dispatches a sound wave of 40khz frequency via the the Transmitter (<em>speaker on the left hand side of the sensor</em>) and when the sound wave hits an object, it will return an “echo” to the Receiver (<em>mic</em><em>&nbsp;on the right</em><em>&nbsp;hand side of the sensor</em>).</p> 
        <p>When the echo comes back, the echo pin&nbsp;will output a high signal for a duration that is equal to the time taken for the soundwave to originate and echo back. So, the duration of the high signal&nbsp;is directly proportional to twice the distance of the obstacle from the sensor.</p> 
        <p>Now, we will monitor the duration of the high signal on the echo pin and using this value, we will calculate the distance.</p> 
        <p>We know that</p> 
        <p style="text-align: center;"> <span id="crayon-59c3359f5dc98125131209" class="crayon-syntax crayon-syntax-inline  crayon-theme-arduino-ide crayon-theme-arduino-ide-inline crayon-font-droid-sans-mono" style="font-size: 12px !important; line-height: 15px !important;font-size: 12px !important;"><span class="crayon-pre crayon-code" style="font-size: 12px !important; line-height: 15px !important;font-size: 12px !important; -moz-tab-size:4; -o-tab-size:4; -webkit-tab-size:4; tab-size:4;"><span class="crayon-v">speed</span><span class="crayon-h"> </span><span class="crayon-o">=</span><span class="crayon-h"> </span><span class="crayon-sy">(</span><span class="crayon-e">distance </span><span class="crayon-v">travelled</span><span class="crayon-sy">)</span><span class="crayon-h"> </span><span class="crayon-o">/</span><span class="crayon-h"> </span><span class="crayon-sy">(</span><span class="crayon-e">time </span><span class="crayon-v">taken</span><span class="crayon-sy">)</span></span></span></p> 
        <p>Here the distance will be equal to twice the distance between the obstacle and the sensor. Because, the time we are using is equal to the time taken for the signal to originate and echo back.</p> 
        <p>We also know that speed of sound&nbsp;is&nbsp;<em>340 m/s</em>. So, the&nbsp;above equation will become</p> 
        <p style="text-align: center;"> <span id="crayon-59c3359f5dcb3985830309" class="crayon-syntax crayon-syntax-inline  crayon-theme-arduino-ide crayon-theme-arduino-ide-inline crayon-font-droid-sans-mono" style="font-size: 12px !important; line-height: 15px !important;font-size: 12px !important;"><span class="crayon-pre crayon-code" style="font-size: 12px !important; line-height: 15px !important;font-size: 12px !important; -moz-tab-size:4; -o-tab-size:4; -webkit-tab-size:4; tab-size:4;"><span class="crayon-cn">340</span><span class="crayon-h"> </span><span class="crayon-o">=</span><span class="crayon-h"> </span><span class="crayon-cn">2d</span><span class="crayon-h"> </span><span class="crayon-o">/</span><span class="crayon-h"> </span><span class="crayon-v">time</span></span></span></p> 
        <p style="text-align: center;">or</p> 
        <p style="text-align: center;"> <span id="crayon-59c3359f5dcc0323414078" class="crayon-syntax crayon-syntax-inline  crayon-theme-arduino-ide crayon-theme-arduino-ide-inline crayon-font-droid-sans-mono" style="font-size: 12px !important; line-height: 15px !important;font-size: 12px !important;"><span class="crayon-pre crayon-code" style="font-size: 12px !important; line-height: 15px !important;font-size: 12px !important; -moz-tab-size:4; -o-tab-size:4; -webkit-tab-size:4; tab-size:4;"><span class="crayon-v">d</span><span class="crayon-h"> </span><span class="crayon-o">=</span><span class="crayon-h"> </span><span class="crayon-cn">170</span><span class="crayon-h"> </span><span class="crayon-o">*</span><span class="crayon-h"> </span><span class="crayon-v">time</span></span></span></p> 
        <p>So, if we plug in the time from the echo pin, we can calculate the distance between the obstacle and the sensor.</p>

        <h2>Setup&nbsp;HC-SR04 Ultrasonic Sensor</h2>

        <p>As you can see from the above diagram, we will connect the VCC pin of the sensor to +5v and GND to GND. We will connect the Trigger pin to GPIO pin 17 and finally we will connect a 1K Ohm resistor between the Echo pin and GPIO pin 18.</p> 
        <p>The Echo pin outputs a 5V output, which will damage the GPIO pin, if interfaced directly. This is why we connect a 1K Ohm resistor between the Echo and GPIO pin 18.</p>

var statistics = require('math-statistics');
var usonic = require('r-pi-usonic');
var init = function(config) {
var sensor = usonic.sensor(config.echoPin, config.triggerPin, config.timeout);
//console.log(config);
var distances;
(function measure() {
if (!distances || distances.length === config.rate) {
if (distances) {
print(distances);
}
distances = [];
}
setTimeout(function() {
distances.push(sensor());
measure();
}, config.delay);
}());
};
var print = function(distances) {
var distance = statistics.median(distances);
process.stdout.clearLine();
process.stdout.cursorTo(0);
if (distance &lt; 0) {
process.stdout.write('Error: Measurement timeout.\n');
} else {
process.stdout.write('Distance: ' + distance.toFixed(2) + ' cm');
}
};
init({

timeout: 1000, //Measurement timeout in µs
