        <article class="markdown-body entry-content" itemprop="text">
         <h1><a id="user-content-nodejs-wrapper-for-yolodarknet-recognition-framework" class="anchor" href="#nodejs-wrapper-for-yolodarknet-recognition-framework" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Node.js Wrapper for YOLO/Darknet recognition framework</h1> 
         <p>Darknet is <a href="https://github.com/pjreddie/darknet">an open source neural network framework written in C and CUDA</a>. This project wraps this framework in a Node.js native addon.</p> 
         <p><a href="https://camo.githubusercontent.com/dff60b365a732dc65155c9a4e1f0fcb6522ef806/68747470733a2f2f6c61622e6d6f6f76656c2e636f6d2f636f6e74656e742f75706c6f6164732f796f6c6f2d6d6963726f2d7072762e676966" target="_blank"><img src="https://camo.githubusercontent.com/dff60b365a732dc65155c9a4e1f0fcb6522ef806/68747470733a2f2f6c61622e6d6f6f76656c2e636f6d2f636f6e74656e742f75706c6f6164732f796f6c6f2d6d6963726f2d7072762e676966" width="458px" height="258px" data-canonical-src="https://lab.moovel.com/content/uploads/yolo-micro-prv.gif" style="max-width:100%;"></a></p> 
         <ul> 
          <li><a href="http://pjreddie.com/darknet/">darknet</a></li> 
          <li><a href="http://pjreddie.com/darknet/yolo/">YOLO (You only look once)</a></li> 
         </ul> 
         <h2><a id="user-content-status" class="anchor" href="#status" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Status</h2> 
         <p>Currently, the wrapper implements a demo method to run a recognition from the webcam/video or an image file. Equivalent to</p> 
         <div class="highlight highlight-source-shell">
          <pre>./darknet detector demo cfg/coco.data cfg/yolo.cfg yolo.weights
./darknet detect cfg/yolo.cfg yolo.weights data/dog.jpg</pre>
         </div> 
         <h2><a id="user-content-prerequisites" class="anchor" href="#prerequisites" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Prerequisites</h2> 
         <p>Requires:</p> 
         <ul> 
          <li>Modern Node and NPM versions (tested with latest LTS release Node v6.<em>.</em>, npm 3.<em>.</em>, supposed to work with the newer versions as well)</li> 
          <li>OpenCV (version 2, for example, 2.4.9.1) to be installed on your system</li> 
          <li>If you are on a mac: macOS 10.12 Sierra or newer, Apple LLVM version 8.0.0 (xcode 8.2, check version with <code>clang -v</code>). For GPU support, Nvidia <a href="https://developer.nvidia.com/cuda-downloads">CUDA Toolkit</a> <em>(you need to have Nvidia CUDA GPU graphic card)</em></li> 
         </ul> 
         <p>First, you need to compile <a href="https://github.com/OrKoN/darknet">this fork</a> of darknet with OpenCV support (optionally, with CUDA support):</p> 
         <div class="highlight highlight-source-shell">
          <pre>git clone https://github.com/OrKoN/darknet
<span class="pl-c1">cd</span> darknet
make OPENCV=1 <span class="pl-c"><span class="pl-c">#</span> optionally GPU=1</span>
make install <span class="pl-c"><span class="pl-c">#</span> by default installed to /usr/local</span></pre>
         </div> 
         <p>After that you can process with the installation via NPM.</p> 
         <p>The fork is required because it contains <a href="https://github.com/pjreddie/darknet/compare/master...OrKoN:master">a few important changes</a> to the origin source code which allow using darknet with NodeJS:</p> 
         <ul> 
          <li>Makefile is extended to build a static library (darknet.a).</li> 
          <li>Makefile is extended with <code>install</code> and <code>uninstall</code> commands which install the library globally so that this module can easily find and link it.</li> 
          <li>All darknet functions in header files are marked with <code>extern "C"</code> if they are included in a C++ program (such as a NodeJS module).</li> 
         </ul> 
         <h2><a id="user-content-installation" class="anchor" href="#installation" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Installation</h2> 
         <div class="highlight highlight-source-shell">
          <pre>npm install @moovel/yolo --save</pre>

           </svg></a>Usage</h2> 
         <p>Either download your own <code>cfg</code>, <code>data</code> folders and <code>.weight</code> files from the darknet project or use the ones included in <code>test</code> folder (see also <a href="/moovel/node-yolo/blob/master/test/README.md">test/readme.md</a>).
