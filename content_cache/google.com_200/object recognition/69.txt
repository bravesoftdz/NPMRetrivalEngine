    <h1> <a name="usage" class="anchor" href="#usage"><span class="octicon octicon-link"></span></a>USAGE</h1> 
    <h2> <a name="include-the-library-gestjs" class="anchor" href="#include-the-library-gestjs"><span class="octicon octicon-link"></span></a>Include the library (gest.js)</h2> 
    <p>You will need to include the 'gest.js' library using something like:</p> 
    <p><code>&lt;script type="text/javascript" src="gest.min.js"&gt;&lt;/script&gt;</code></p> 
    <h2> <a name="start-gesture-detection" class="anchor" href="#start-gesture-detection"><span class="octicon octicon-link"></span></a>Start gesture detection</h2> 
    <p>You can start gest.js by calling: <code>gest.start();</code></p> 
    <h2> <a name="listen-for-recognised-gestures" class="anchor" href="#listen-for-recognised-gestures"><span class="octicon octicon-link"></span></a>Listen for recognised gestures</h2> 
    <h3> <a name="method-1-recommended" class="anchor" href="#method-1-recommended"><span class="octicon octicon-link"></span></a>Method 1 (recommended)</h3> 
    <p>Use the <code>.options.subscribeWithCallback(...)</code> function to listen for gestures:</p> 
    <pre><code>gest.options.subscribeWithCallback(function(gesture) {
    //handle gesture .direction .up .down .left .right .error
});
</code></pre> 
    <p>This method is recommended because it will automatically handle cross-browser event handling.</p> 
    <h3> <a name="method-2" class="anchor" href="#method-2"><span class="octicon octicon-link"></span></a>Method 2</h3> 
    <p>You can register an event listener on the <code>document</code> for <code>gest</code> using:</p> 
    <pre><code>document.addEventListener('gest', function(gesture) {
    //handle gesture .direction .up .down .left .right .error
}, false);
</code></pre> 
    <h2> <a name="how-to-handle-recognised-gestures" class="anchor" href="#how-to-handle-recognised-gestures"><span class="octicon octicon-link"></span></a>How to handle recognised gestures</h2> 
    <p>On every event, you will be passed a <code>gesture</code> object that contains:</p> 
    <ul> 
     <li> <code>.direction</code> the recognised gesture in words as a string</li> 
     <li> <code>.up</code> boolean, true if the recognised gesture is up</li> 
     <li> <code>.down</code> boolean, true if the recognised gesture is down</li> 
     <li> <code>.left</code> boolean, true if the recognised gesture is left</li> 
     <li> <code>.right</code> boolean, true if the recognised gesture is right</li> 
     <li> <code>.error</code> an error object with... 
      <ul> 
       <li> <code>.code</code> a code as an int</li> 
       <li> <code>.message</code> and a message as a string</li> 
      </ul> </li> 
    </ul>
    <h2> <a name="stop-gesture-detection" class="anchor" href="#stop-gesture-detection"><span class="octicon octicon-link"></span></a>Stop gesture detection</h2> 
    <p>You can stop gest.js at any time by calling: <code>gest.stop();</code></p> 
    <h2> <a name="using-options" class="anchor" href="#using-options"><span class="octicon octicon-link"></span></a>Using Options</h2> 
    <h3> <a name="sensitivity-80-by-default" class="anchor" href="#sensitivity-80-by-default"><span class="octicon octicon-link"></span></a>Sensitivity (80 by default)</h3> 
    <p>You can specify the degree of colour change that should be used to determine whether or not a pixel has changed. The specified value should range from 0-100, with 100 implying that the slightest change should be enough. You can specify sensitivity using: <code>gest.options.sensitivity(85);</code></p> 
    <h3> <a name="skin-filtering-off-by-default" class="anchor" href="#skin-filtering-off-by-default"><span class="octicon octicon-link"></span></a>Skin Filtering (off by default)</h3> 
    <p>To improve recognition in bright lighting conditions, you can enable HSV skin filtering using: <code>gest.options.skinFilter(true);</code></p> 
    <h3> <a name="debugging-off-by-default" class="anchor" href="#debugging-off-by-default"><span class="octicon octicon-link"></span></a>Debugging (off by default)</h3> 
    <p>In order to view the video stream and enable <code>console.log(…)</code> messages, you will need to toggle debugging using: <code>gest.options.debug(true);</code></p> 
    <h1> <a name="whats-changed" class="anchor" href="#whats-changed"><span class="octicon octicon-link"></span></a>WHAT'S CHANGED</h1> 
    <h2> <a name="v050-latest" class="anchor" href="#v050-latest"><span class="octicon octicon-link"></span></a>v0.5.0 (latest)</h2> 
    <ul> 
     <li>Firefox support (woohoo!)</li> 
     <li>Error handling, all the way back to the stone-age (IE7)</li> 
     <li>Better event handling</li> 
     <li>Ability for the developer to define sensitivity</li> 
     <li>A new <code>.options.subscribeWithCallback(...)</code> function</li> 
     <li>On screen messages is now DEPRECATED. I decided that this is something that the front-end developer should build - it shouldn't be part of the library. If you really want it back, check demo/simple.</li> 
     <li>Locking is now DEPRECATED. It wasn't an elegant enough solution. It might come back in future releases...</li> 
    </ul>
