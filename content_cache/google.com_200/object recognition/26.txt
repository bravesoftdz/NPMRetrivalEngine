       <h1>JavaScript</h1>
       <a name="d1185e18"></a>
       <p><a name="d1185e19"></a><span class="emphasis">This feature is supported in the Ultimate edition only.</span></p>
       <a name="d336504e18"></a>
       <p> With IntelliJ IDEA, you can develop modern web, mobile, and desktop applications with <a name="d336504e20"></a><span class="emphasis">JavaScript</span> and <a name="d336504e23"></a><span class="emphasis">Node.js</span>. IntelliJ IDEA supports <a name="d336504e26"></a><span class="emphasis">JavaScript</span> and <a name="d336504e29"></a><span class="emphasis">TypeScript</span> programming languages, <a name="d336504e32"></a><span class="emphasis">React</span> and <a name="d336504e36"></a><span class="emphasis">Angular</span> frameworks and provides tight integration with various tools for web development.</p>
       <a name="d336504e43"></a>
       <p>On this page:</p>
       <ul data-swiftype-index="false">
        <li class="list__item"><a href="#prereq">Prerequisites</a></li>
        <li class="list__item"><a href="#d336504e55">JavaScript support</a></li>
        <li class="list__item"><a href="#d336504e296">Developing an application that contains JavaScript: general overview</a></li>
        <li class="list__item"><a href="#d336504e478">In this section</a></li>
       </ul>
       <a name="prereq"></a>
       <h2>Prerequisites</h2> 
       <a name="d14640e60"></a>
       <p>Before you start working with JavaScript, make sure that the <a name="d14640e62"></a><span class="control">JavaScript Support</span> plugin is enabled. The plugin is bundled with IntelliJ IDEA and is activated by default. If the plugin is not activated, enable it on the <a name="d14640e65"></a><a href="plugins-2.html"><span>Plugins</span></a> page of the <a name="d14640e67"></a><a href="settings-preferences-dialog.html"><span>Settings / Preferences Dialog</span></a> as described in <a name="d14640e69"></a><a href="enabling-and-disabling-plugins.html"><span>Enabling and Disabling Plugins</span></a>.</p> 
       <a name="d336504e55"></a>

          <li class="list__item"><a name="d336504e78"></a>Error and syntax highlighting.</li>
          <li class="list__item"><a name="d336504e81"></a>Code <a name="d336504e83"></a><a href="code-style.html">formatting</a> and <a name="d336504e86"></a><a href="folding-code-elements.html">folding</a>.</li>

          <li class="list__item"><a name="d336504e121"></a>Inserting, expanding, and generating JavaScript code blocks using <a name="d336504e123"></a><a href="live-templates.html">live templates</a>.</li>

       <a name="d336504e296"></a>
       <h2>Developing an application that contains JavaScript: general overview</h2> 
       <a name="d336504e298"></a>
       <p>Developing an application that contains JavaScript, generally, includes performing the following steps:</p> 
       <ol class="list _decimal">
        <li class="list__item"><a name="d336504e305"></a> <a name="d336504e307"></a><a href="creating-and-managing-projects.html">Create a project</a> to implement your application. On the first page of the <a name="d336504e312"></a><a href="new-project-wizard.html">New Project wizard</a>, select <a name="d336504e315"></a><span class="control">Static Web</span>. </li>
        <li class="list__item"><a name="d336504e320"></a> On the <a name="d336504e322"></a><a href="javascript-3.html"><span>JavaScript</span></a> page of the <a name="d336504e324"></a><a href="settings-preferences-dialog.html">Settings</a> dialog box, choose the JavaScript language version that you are going to use in your project. </li>
        <li class="list__item"><a name="d336504e328"></a> Install the project dependencies via the <a name="d336504e330"></a><span class="emphasis">Node Package Manager</span> by doing one of the following: 
         <ul class="list _bullet">
          <li class="list__item"><a name="d336504e335"></a> 
           <ol class="list _decimal">
            <li class="list__item"><a name="d68164e5549"></a> Open the <a name="d1595e8206"></a><a href="settings-preferences-dialog.html"><span>Settings / Preferences Dialog</span></a> by pressing <kbd data-swiftype-index="false" data-primary_default="Ctrl+Alt+S" data-primary_default_for_gnome="Ctrl+Alt+S" data-primary_default_for_kde="Ctrl+Alt+S" data-primary_default_for_xwin="Ctrl+Alt+S" data-primary_emacs="Ctrl+Alt+S" data-primary_jbuilder="N/A" data-primary_visual_studio="Alt+F7" data-primary_netbeans_6.5="N/A" data-primary_eclipse="Ctrl+Alt+S" data-secondary_mac_os_x_10.5_="?," data-secondary_mac_os_x="?," data-secondary_eclipse_mac_os_x="?," data-secondary_intellij_idea_classic_os_x="N/A" data-secondary_xcode="N/A" data-secondary_visual_studio="?F7" data-secondary_resharper="N/A" data-secondary_resharper_osx="N/A" data-secondary_emacs="??S">Ctrl+Alt+S</kbd> or by choosing <a name="d1595e8210"></a><span data-swiftype-index="false" class="menupath">File | Settings</span> for Windows and Linux or <a name="d1595e8213"></a><span data-swiftype-index="false" class="menupath">IntelliJ IDEA | Preferences</span> for macOS, and click <a name="d1595e8237"></a><span class="control">Node.js and NPM</span> under <a name="d1595e8240"></a><span class="control">Languages &amp; Frameworks</span>. </li>
            <li class="list__item"><a name="d68164e5561"></a> On the <a name="d68164e143"></a><span class="control">Node.js and NPM</span> page that opens, the <a name="d68164e146"></a><span class="control">Packages</span> area shows all the Node.js-dependent packages that are currently installed on your computer, both at the <a name="d68164e149"></a><span class="emphasis">global</span> and at the <a name="d68164e152"></a><span class="emphasis">project</span> level. Click <img alt="/help/img/idea/2017.2/new.png" title="/help/img/idea/2017.2/new.png" src="/help/img/idea/2017.2/new.png" width="12" height="12" class="inline-icon-12">. </li>
            <li class="list__item"><a name="d68164e5566"></a> In the <a name="d68164e5568"></a><span class="control">Available Packages</span> dialog box that opens, select the required package. </li>
            <li class="list__item"><a name="d68164e5572"></a> Optionally specify the product version and click <a name="d68164e5574"></a><span class="control">Install Package</span> to start installation. </li>
           </ol> </li>
          <li class="list__item"><a name="d336504e350"></a> Right-click the <code class="code" data-lang="Java">package.json</code> file in the editor or in the <a name="d336504e355"></a><span class="control">Project</span> tool window and choose <a name="d336504e358"></a><span class="control">Run 'npm install'</span> on the context menu. </li>
          <li class="list__item"><a name="d336504e362"></a> Open the built-in <a name="d336504e364"></a><span class="control">Terminal</span> (<a name="d336504e367"></a><span data-swiftype-index="false" class="menupath">View | Tool Windows | Terminal</span>) and run the <code class="code" data-lang="Java">npm install &lt;required_package&gt;</code> commands. </li>
          <li class="list__item"><a name="d336504e374"></a> Install the required packages on the <a name="d336504e376"></a><a href="node-js-and-npm.html"><span>Node.js and NPM</span></a> page of the <a name="d336504e378"></a><a href="settings-preferences-dialog.html"><span>Settings / Preferences Dialog</span></a>. </li>
         </ul> <a name="d336504e382"></a><p> For details, see <a name="d336504e384"></a><a href="npm.html"><span>NPM</span></a>. </p> </li>
        <li class="list__item"><a name="d336504e390"></a> <a name="d336504e392"></a><a href="populating-projects.html">Populate the project</a>. Use the following IntelliJ IDEA facilities, where applicable: 
         <ul class="list _ul">
          <li class="list__item"><a name="d336504e397"></a> <a name="d336504e399"></a><a href="#coding_assistance">Coding assistance</a>.</li>
          <li class="list__item"><a name="d336504e403"></a> <a name="d336504e405"></a><a href="generating-code.html">Code Generation</a>.</li>
          <li class="list__item"><a name="d336504e409"></a>Various types of <a name="d336504e411"></a><a href="#navigation">navigation</a> and <a name="d336504e414"></a><a href="searching-through-the-source-code.html">search</a> through the source code.</li>
          <li class="list__item"><a name="d336504e418"></a> <a name="d336504e420"></a><a href="viewing-reference-information.html">Viewing reference</a> and <a name="d336504e423"></a><a href="creating-jsdoc-comments.html">generating documentation comments</a>.</li>
          <li class="list__item"><a name="d336504e427"></a> <a name="d336504e429"></a><a href="documentation-look-up-in-external-javascript-libraries.html">Look-up in external libraries</a>.</li>
         </ul> </li>
        <li class="list__item"><a name="d336504e436"></a> Improve the quality and maintainability of your code using various types of <a name="d336504e438"></a><a href="refactoring-source-code.html">refactorings</a>, built-in code quality tools, and integration with other linters, see <a name="d336504e441"></a><a href="linters.html"><span>JavaScript Linters</span></a>. </li>
        <li class="list__item"><a name="d336504e446"></a> Run your application by <a name="d336504e448"></a><a href="html.html">opening its starting HTML page in the IntelliJ IDEA default browser</a>. </li>
        <li class="list__item"><a name="d336504e452"></a> <a name="d336504e454"></a><a href="debugging.html">Debug</a> your application in the <a name="d336504e457"></a><a href="http://www.google.com/chrome/" data-bypass="yes"><span>Google Chrome</span></a> browser. <a name="d336504e460"></a><p>The JavaScript debugging functionality is incorporated in IntelliJ IDEA, so just <a name="d336504e462"></a><a href="debugging-javascript-in-firefox.html">configure the debugger</a>, whereupon you can start the <a name="d336504e465"></a><a href="debugging-javascript-in-chrome.html">debugging session</a> and <a name="d336504e468"></a><a href="debugging.html">proceed as usual</a>: set the breakpoints, step through them, stop and resume the program, and examine it when suspended.</p> </li>
