             <h2>Introduction</h2> 
             <p>The purpose of this post is to present a possible animation of <em>Quicksort</em> algorithm using JavaScript. Quicksort is a practical and fast sorting algorithm. It is a fundamental topic in the standard algorithm course in computer science. The algorithm for Quicksort serves as a good example of the <em>divide-and-conquer</em> algorithm-design technique; a technique for which the resulting algorithm can be compactly and elegantly expressed using recursion. As an aid to understanding the algorithm, an educator might show a trace of the algorithm's execution on some specific input. The trace normally shows the content of the input array and other key parameters at various stages during execution. This is essentially what our JavaScript code does but in a vivid way.</p> 
             <p> The animated Quicksort algorithm can be run from <a href="http://faculty.kfupm.edu.sa/ics/darwish/JS_Quicksort/">here</a>.</p> 
             <p>The interface provides one of two views (Display list-box). In the "Shrunk" view, the array is shown for every call of the algorithm with the values for the <em>lo</em>,<em>hi</em>, and <em>mid</em> parameters shown to the left. The line-display for the current array dynamically changes with every swap. In the "Expanded" view, the array is shown on a separate line for every swap of elements with the swapped elements highlighted using a distinct color. In both views, the elements from <em>lo</em> to <em>hi</em> are shown with a gray background, while other elements will have a white background. The pivot is identified with red foreground color. </p> 
             <p> The approach used for animation uses the same technique presented by the author in the <a href="http://www.codeproject.com/Articles/679651/">Tower-of-Hanoi</a> article. As noted by the author, placing the <code>setInterval()</code> calls directly within the procedure to be animated is a recipe for chaos. An effective alternative is to run the normal unanimated procedure until completion whilst using a stack to save the parameters of interest. Then process the stack and utilize JavaScript setInterval() function combined with some visual elements to show the algorithm progress. </p> 
             <h2>How does Quicksort work?</h2> 
             <p>The recursive algorithm for Quicksort can be expressed by the following <code>Quicksort(A, lo, hi)</code> JavaScript function:</p> 
             <pre lang="jscript"><span class="code-keyword">function</span> Quicksort(A, lo, hi) 
{  <span class="code-keyword">if</span> (lo &lt; hi) 
   {  <span class="code-keyword">var</span> mid = Partition(A,lo,hi);
      Quicksort(A,lo,mid-1);
      Quicksort(A,mid+1,hi);
   } 
} </pre> 
             <p> The input to the function is <em>an array A</em> of the items to be sorted. The <em>lo</em> and <em>hi</em> parameters denote, respectively, the lowest index and highest index of some unsorted section within the array. For the initial call of the above function, the <em>lo</em> and <em>hi</em> parameters are respectively set (by caller) to the first (leftmost) and last (rightmost) indices of the array. </p> 
             <p><img src="/KB/recipes/779771/Quicksort.png" alt="Quicksort Image" width="520" height="273"></p> 
             <p> As illustrated by the preceding figure, Quicksort uses a divide-and-conquer strategy, dividing the input array into two parts (sections) and sorting each part recursively. Quicksort uses an auxiliary procedure to perform in-place partitioning of its input (in-place means without using an additional working array). The Partition procedure works as follow: </p> 
             <blockquote>
               it picks an element 
              <em>P</em> and arranges the elements so that elements ? 
              <em>P</em> are moved to the left side of the sequence and the elements &gt; 
              <em>P</em> are moved to the right side. The element 
              <em>P</em> used for partitioning is referred to as the 
              <em>pivot</em> element. The pivot can be any of the elements in the input sequence to be partitioned; however, quite often, the first (leftmost) element is chosen as the pivot. 
             </blockquote> 
             <pre lang="jscript"><span class="code-keyword">function</span> Partition(A, lo, hi)
{<span class="code-comment">//</span><span class="code-comment"> Input: An integer array A[lo..hi]  </span>
 <span class="code-comment">//</span><span class="code-comment"> Output: The array A arranged as: elements &lt;= pivot, pivot, elements &gt; pivot</span>
 <span class="code-comment">//</span><span class="code-comment"> Return the final position of pivot </span>
 <span class="code-comment">//</span><span class="code-comment"> The pivot is A[lo] but it can be any of the elements A[lo], ..., A[hi];</span>
 <span class="code-comment">//</span><span class="code-comment"> If pivot is other than A[lo] then (at the start) swap it with A[lo]</span>
 
   <em><strong>paramStack.push([-lo,hi]); <span class="code-comment">//</span><span class="code-comment"> push lo and hi</span></strong></em>
   
   <span class="code-keyword">var</span> pivot = A[lo];   
   <span class="code-keyword">var</span> left = lo+1; 
   <span class="code-keyword">var</span> right = hi;
   <span class="code-keyword">while</span> (left &lt;= right)
   {  <span class="code-keyword">while</span> ((left &lt;= right) &amp;&amp; (A[left] &lt;= pivot)) left++;
      <span class="code-keyword">while</span> ((left &lt;= right) &amp;&amp; (A[right] &gt; pivot)) right--;
      <span class="code-keyword">if</span> (left &lt; right)
      {  <span class="code-comment">//</span><span class="code-comment"> save left and right parameters to paramStack array </span>
         <em><strong>paramStack.push([left,right]); </strong></em>
         <span class="code-keyword">var</span> t = A[left]; A[left] = A[right]; A[right] = t; left++; right--; 
      }
   }
  
   <span class="code-comment">//</span><span class="code-comment"> Final swap: swap pivot with A[right] and return right (final pivot location) </span>
   A[lo] = A[right]; A[right] = pivot;
   <em><strong>paramStack.push([lo,right]); <span class="code-comment">//</span><span class="code-comment"> push lo and right</span></strong></em> 
   <span class="code-keyword">return</span> right;
}</pre> 
             <p> One simple algorithm for partitioning the input array is given by the preceding <code>Partition()</code> function. </p> 
             <p><em>Note:</em> The function includes modifications in three places (the lines in bold) that use <code>paramStack.push()</code> to save the <em>lo</em> and <em>hi</em> values as well as the locations of the array elements being swapped. </p> 
             <p> This uses two pointers (<em>left</em> and <em>right</em>) that are, respectively, set to the start and end of the array to be partitioned. The left pointer moves toward the right, skipping elements ? <em>pivot</em>, whereas the right pointer moves toward the left, skipping elements &gt; <em>pivot</em>, but where the left and right pointers stop and <em> left &lt; right</em>, the corresponding elements are swapped. The process ends when the left and right pointers meet or pass each other. Then a final swap involving the pivot element and the element pointed to by the right pointer is executed to put the pivot into its proper sorted location. </p>
