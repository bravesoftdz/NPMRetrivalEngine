        <h1 class="wikititle">Quicksort</h1> 
        <p></p> 
        <a name="Description"></a>
        <h1>Description</h1> 
        <p>The quicksort example is a concurrent implementation of the well-known <a href="http://en.wikipedia.org/wiki/Quicksort" class="wiki_ext_link">quicksort</a> sorting algorithm developed by computer scientist <a href="http://en.wikipedia.org/wiki/C._A._R._Hoare" class="wiki_ext_link">C. A. R. Hoare</a>. Quicksort uses a "divide and conquer" strategy to sort a structure. It applies a basic algorithm to the structure which leads to a division of the elements into to two substructures. Then it applies the same algorithm to each of the substructures, and so on, until the whole structure is sorted. Because of the repetitive application of the same algorithm to evolving parts of the structure, the quicksort is often used in computer science classes to provide students with experience in <a href="http://en.wikipedia.org/wiki/Recursion_(computer_science)" class="wiki_ext_link">recursive</a> computation. </p> 
        <p>In the SCOOP example, instead of recursive calls, substructures are handled (within limits) by separate <a href="/doc/solutions/Concurrent%20programming%20with%20SCOOP" class="wiki_link">SCOOP processors</a> running concurrently. </p> 
        <a name="Highlights"></a>
        <h1>Highlights</h1> 
        <p>The quicksort example sorts a randomly generated container of integers. The set-up for this example is done in the root class. It is interactive in the sense that when you run the example, you get to to choose how many elements will be sorted (within certain limits) and you get to provide a seed for the random number generator which will be used to produce the unsorted structure. </p> 
        <p>The quicksort algorithm is embodied in the class <code class="inline">QUICKSORTER</code>, primarily in its routine <code class="inline">sort</code>. Instances of <code class="inline">QUICKSORTER</code> declared as <code class="inline">separate</code> are spawned to sort substructures as the algorithm progresses. </p> 
        <p>The structures acted upon by <code class="inline">QUICKSORTER</code> are managed in instances of class <code class="inline">DATA</code>. <code class="inline">DATA</code> is a class designed specifically to support the quicksort example. </p> 
        <p>When the example runs, separate <code class="inline">QUICKSORTER</code> processes are used for the recursive sorts up until a certain depth of recursion is reached. The limit is defined by the <code class="inline">NATURAL</code> constant <code class="inline">{QUICKSORTER}.max_recursion_depth</code>. </p>
