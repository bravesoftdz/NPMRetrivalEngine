            <li>If you don’t have a Nexmo account, <a href="https://dashboard.nexmo.com/sign-up" target="_blank" rel="noopener noreferrer">sign up for one</a>.</li> 
            <li>Login to <a href="https://dashboard.nexmo.com/sign-in" target="_blank" rel="noopener noreferrer">Nexmo Dashboard</a>. Scroll to the API Settings section to get your API credentials. You will need the <em>API Key</em> and <em>API Secret</em> for this tutorial.<img class="aligncenter size-large wp-image-86597" src="https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/nexmo-credentials-1024x683.png" alt="Screen shot of Nexmo API Settings" width="680" height="454" srcset="https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/nexmo-credentials-1024x683.png 1024w, https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/nexmo-credentials-800x533.png 800w" sizes="(max-width: 680px) 100vw, 680px"></li> 
           </ol> 
           <p>This step-by-step instruction uses <a href="https://nodejs.org/" target="_blank" rel="noopener noreferrer">Node.js</a>, so you’ll need to install it on your machine. Once you are ready, let’s get started!</p> 
           <h3 class="ibm-h3">Creating a New Application</h3> 
           <ol> 
            <li>Go to your <a href="https://console.ng.bluemix.net/dashboard/apps/" target="_blank" rel="noopener noreferrer">Bluemix Dashboard</a>, and click the Create App button (see below):<img class="aligncenter size-large wp-image-86599" src="https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/01-create-app-1024x680.png" alt="Screen shot of Create a Bluemix app window" width="680" height="452" srcset="https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/01-create-app-1024x680.png 1024w, https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/01-create-app-800x531.png 800w" sizes="(max-width: 680px) 100vw, 680px"></li> 
            <li>Next, scroll down to Cloud Foundry Apps, and choose SDK for Node.js (see below):<img class="aligncenter size-large wp-image-86601" src="https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/02-create-app-js-1024x680.png" alt="Cloud Foundry Apps - SDK for Node.js" width="680" height="452" srcset="https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/02-create-app-js-1024x680.png 1024w, https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/02-create-app-js-800x531.png 800w" sizes="(max-width: 680px) 100vw, 680px"></li> 
            <li>Then, give your new application a unique name. In this tutorial, you are going to build a very simple app that just sends an SMS message to your phone number, so let’s name it something like “send-SMS”. Then click Create (as shown below):<img class="aligncenter size-large wp-image-86605" src="https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/03-name-app-1024x680.png" alt="Create a Cloud Foundry app" width="680" height="452" srcset="https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/03-name-app-1024x680.png 1024w, https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/03-name-app-800x531.png 800w" sizes="(max-width: 680px) 100vw, 680px"></li> 
           </ol> 
           <p>Now, you’ll go back to the dashboard. You should see some progress indicators that shows you the new application has been created.</p> 
           <h3 class="ibm-h3">Using Nexmo with the New Application</h3> 
           <p>As was mentioned, you are going to send a simple SMS message.</p> 
           <ol> 
            <li>Click Catalog at the top menu bar (see image below):<img class="aligncenter size-large wp-image-86609" src="https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/04-go-to-catalog-1024x680.png" alt="Cloud Foudry apps - Go to Catalog" width="680" height="452" srcset="https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/04-go-to-catalog-1024x680.png 1024w, https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/04-go-to-catalog-800x531.png 800w" sizes="(max-width: 680px) 100vw, 680px"></li> 
            <li>At the catalog, search Nexmo under Application Services, then click it:<img class="aligncenter size-large wp-image-86611" src="https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/05-add-nexmo-1024x678.png" alt="Add Nexmo from the Bluemix Catalog" width="680" height="450" srcset="https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/05-add-nexmo-1024x678.png 1024w, https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/05-add-nexmo-800x530.png 800w" sizes="(max-width: 680px) 100vw, 680px"></li>

           <h3 class="ibm-h3">Downloading Your Template Files</h3> 
           <p>After you connected the service to your “send-sms” application, click the app, then Getting Started from the side menu, then click “Download the Sample Code” link to get the zip file to your computer. Now unzip it. The folder should contain the template code files. Now you are ready to move on!</p> 
           <h3 class="ibm-h3">Using the Cloud Foundry Command Line interface</h3> 
           <p>To download, modify, and redeploy your Cloud Foundry applications and service instances, you need to download and install the <a href="https://console.ng.bluemix.net/docs/starters/install_cli.html" target="_blank" rel="noopener noreferrer">Cloud Foundry command line interface</a> (CLI).</p> 
           <ol> 
            <li>After the installation, use the cf command to login and connect to the Bluemix environment: <pre class="brush: plain; title: ; notranslate" title="">$ cf api https://api.ng.bluemix.net</pre> <p>Or use the API endpoint for your region, <a href="https://api.eu-gb.bluemix.net">https://api.eu-gb.bluemix.net</a> (United Kingdom) and <a href="https://api.au-syd.bluemix.net">https://api.au-syd.bluemix.net</a> (Sydney).</p></li> 
            <li>Then login: <pre class="brush: plain; title: ; notranslate" title="">$ cf login</pre> </li> 
           </ol> 
           <p>Now you got your Bluemix app set up, so you can get started creating your Node.js application.</p> 
           <h3 class="ibm-h3">Setting up Your Nexmo Phone Number</h3> 
           <p>In this tutorial, we are going to create a very basic app that sends an SMS message from your Nexmo phone number to your mobile phone number.</p> 
           <p>You can get your own number by going to your <a href="https://dashboard.nexmo.com/sign-in" target="_blank" rel="noopener noreferrer">Nexmo dashboard</a> and click Numbers from the top menu bar. Although you can purchase any number with a country code of your choice, to avoid the international fee, you should choose the same country code as your mobile phone number where you will be sending the SMS message.<img class="aligncenter size-large wp-image-86615" src="https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/nexmo-buy-numbers-1024x680.png" alt="Nexmo Buy numbers window" width="680" height="452" srcset="https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/nexmo-buy-numbers-1024x680.png 1024w, https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/nexmo-buy-numbers-800x531.png 800w" sizes="(max-width: 680px) 100vw, 680px"></p> 
           <h3 class="ibm-h3">Using the Nexmo REST API Client for Node.js</h3> 
           <p>Now, let’s code! Make sure that node.js is installed on your machine to start coding.</p> 
           <p>Take a look at the project folder that you previously downloaded from your Bluemix dashboard. It should contain files like this:</p> 
           <pre class="brush: plain; title: ; notranslate" title="">
.
??? LICENSE
??? README.md
??? app.js
??? manifest.yml
??? package.json
??? public
    ??? images
    ??? index.html
    ??? stylesheets
...

</pre> 
           <p>On terminal, cd to the project root, then use the npm command to install dependencies:</p> 
           <pre class="brush: plain; title: ; notranslate" title="">$ npm install</pre> 
           <p>Then install Nexmo Node.js client library:</p> 
           <pre class="brush: plain; title: ; notranslate" title="">$ npm install nexmo --save</pre> 
           <p>This will add nexmo as the dependencies in your package.json file automatically. Also, check your manifest.yml file, which is used by Bluemix for the deployment. It should look like this:</p> 
           <pre class="brush: plain; title: ; notranslate" title="">
applications:
- path: .
services
- Nexmo-yl
name: send-sms
host: send-sms
...

</pre> 
           <p>For the service name, you have an automatically generated one for your application. It usually looks like this, nexmo-, and two random characters, such as nexmo-yl. You will need your service name at the next step.</p> 
           <p>Now, open the app.js file. At the end of the code template, assign your Nexmo API credentials. You can access your Nexmo credentials within the VCAP_SERVICES environment variable with cfenv that parses Cloud Foundry-provided environment variables:</p> 
           <pre class="brush: plain; title: ; notranslate" title="">

// The next two lines have been declared already in the template
const cfenv = require('cfenv');
const appEnv = cfenv.getAppEnv();

</pre> 
           <pre class="brush: plain; title: ; notranslate" title="">

// Copy and paste these lines to your app.js
const nexmoApiKey = appEnv.getServiceCreds('Nexmo-j1')['api_key'];
const nexmoApiSecret = appEnv.getServiceCreds('Nexmo-j1')['api_secret'];

</pre> 
           <p>Again, the name must match your generated service name. To find out all the VCAP_SERVICES info, use cf env [APP_NAME] command on terminal.</p> 
           <p>Now, initialize a Nexmo instance with the credentials:</p> 
           <pre class="brush: plain; title: ; notranslate" title="">
const Nexmo = require('nexmo');

const nexmo = new Nexmo({
  apiKey: nexmoApiKey,
  apiSecret: nexmoApiSecret
});
</pre> 
           <p>&nbsp;</p> 
           <h3 class="ibm-h3">Sending SMS Messages with Node.js</h3> 
           <p>Now, let’s send a SMS message from your Nexmo virtual number to your real mobile phone! To send a message, use the nexmo.sms.sendSms function and pass your virtual number (YOUR_VIRTUAL_NUMBER) you are sending the message from, a recipient number, and the message to be sent. Also, you can pass <a href="https://docs.nexmo.com/messaging/sms-api/api-reference#request" target="_blank" rel="noopener noreferrer">optional params</a> and a callback.</p> 
           <p>For this exercise, let’s simply hard-code your mobile real phone number (which should start with a country code, e.g. “15105551234”) and a message to try the API:</p> 
           <pre class="brush: plain; title: ; notranslate" title="">
nexmo.message.sendSms(
 YOUR_VIRTUAL_NUMBER, '15105551234', 'yo',
   (err, responseData) =&gt; {
     if (err) {
       console.log(err);
     } else {
       console.dir(responseData);
     }
   }
);

</pre> 
           <p>Now, let’s deploy this app to Bluemix.</p> 
           <h3 class="ibm-h3">Deploying Your Application</h3> 
           <p>To deploy your application, you will be using the Cloud Foundry CLI again. You can push your code with cf push command:</p> 
           <pre class="brush: plain; title: ; notranslate" title="">$ cf push</pre> 
           <p>Once your application is deployed to Bluemix and run correctly, Nexmo will send the message to your phone. The time of the actual delivery depends on your mobile phone carrier, so it can be instant, or it might take a few seconds.<img class="aligncenter size-large wp-image-86617" src="https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/sms-android-1024x576.png" alt="Nexmo SMS Android example" width="680" height="383" srcset="https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/sms-android-1024x576.png 1024w, https://www.ibm.com/blogs/bluemix/wp-content/uploads/2017/04/sms-android-800x450.png 800w" sizes="(max-width: 680px) 100vw, 680px"></p> 
           <p>While this example is a simple SMS application,&nbsp; it should provide you with a good idea of how to use the Nexmo SMS API to build better applications! To create something more practical using the Nexmo APIs, such as an app to make and receive voice calls or setting up two-factor authentications, visit the <a href="https://www.nexmo.com/blog/category/developers-2/tutorial/" target="_blank" rel="noopener noreferrer">Nexmo Tutorial Blog</a>.</p>
