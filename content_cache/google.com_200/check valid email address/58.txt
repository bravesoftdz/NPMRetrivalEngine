      <div class="entry-content"> 
       <p>Magento uses Prototype library to manage form validation. This comes in handy, because all you need to do when writing custom form is to assign a valid class names to your input fields, and pass the form id to <strong>VarienForm </strong>object.</p> 
       <p><span id="more-745"></span></p>

       <p>In case you would want to add your custom validation rule, just create a javascript file with any name you want in <strong>js</strong> folder of your Magento installation. Ill show you how to do this in the following example.</p> 
       <p>Say you want to modify one of the validation rules, the best way would be to <strong>create a method with a same name and load it after the original method</strong>. This way, all calls would be redirected to <strong>your method instead of the original</strong>. Just be careful if youre overriding validation rules like this, because <strong>javascript is only client-side validation</strong>. Your may have your server validate the field as well  which can cause some errors even though the value in field was entered the way you wanted.<br> So, be sure to test your validation rule.</p> 
       <p>For our purposes, well override Magentos <strong>validate-email</strong> method. We want it to accept only email addresses ending with <strong>@gmail.com</strong>.</p> 
       <p>Lets create our javascript file and place it in the <strong>js</strong> folder.</p> 
       <p><strong>js/inchoo.js</strong></p> 
       <pre class="ish"><code class="language-javascript">Validation.<span class="me1">add</span><span class="br0">(</span><span class="st0">'validate-email'</span><span class="sy0">,</span> <span class="st0">'Please enter a valid Gmail address. For example johndoe@gmail.com.'</span><span class="sy0">,</span> <span class="kw1">function</span><span class="br0">(</span>v<span class="br0">)</span> <span class="br0">{</span>
    <span class="kw1">return</span> Validation.<span class="kw1">get</span><span class="br0">(</span><span class="st0">'IsEmpty'</span><span class="br0">)</span>.<span class="me1">test</span><span class="br0">(</span>v<span class="br0">)</span> <span class="sy0">||</span> <span class="co2">/^([a-zA-Z0-9]+[a-zA-Z0-9._%-]*@gmail\.com)$/i</span>.<span class="me1">test</span><span class="br0">(</span>v<span class="br0">)</span>
<span class="br0">}</span><span class="br0">)</span></code></pre> 
       <p>All we need to do now is include our validation script. Well do that using Magentos <strong>addJs</strong> method which will add the javascript file to our websites head.</p> 
       <pre class="ish"><code class="language-xml"><span class="sc3"><span class="re1">&lt;default<span class="re2">&gt;</span></span></span>
    <span class="sc3"><span class="re1">&lt;reference</span> <span class="re0">name</span>=<span class="st0">"head"</span><span class="re2">&gt;</span></span>
        <span class="sc3"><span class="re1">&lt;action</span> <span class="re0">method</span>=<span class="st0">"addJs"</span><span class="re2">&gt;</span><span class="re1">&lt;script<span class="re2">&gt;</span></span></span>inchoo.js<span class="sc3"><span class="re1">&lt;/script<span class="re2">&gt;</span></span><span class="re1">&lt;/action<span class="re2">&gt;</span></span></span>
    <span class="sc3"><span class="re1">&lt;/reference<span class="re2">&gt;</span></span></span>
<span class="sc3"><span class="re1">&lt;/default<span class="re2">&gt;</span></span></span></code></pre> 
       <p>You can do this from any layout file (preferably your modules layout file), Magento did it in their themes layout page called <strong>page.xml</strong> located in <strong>app/design/frontend/base/default/layout/page.xml</strong></p> 
       <p>Hope this clears some things up for you. At the very least, you now have a list of all validation classes in Magento.</p>
