    <h2 class="entry-title"> <a href="https://www.webdigi.co.uk/blog/2009/how-to-check-if-an-email-address-exists-without-sending-an-email/" class="no-link" title="Permalink to How to check if an email address exists without sending an email?" rel="bookmark">How to check if an email address exists without sending an email?</a> </h2> 
    <div class="entry-content"> 
     <p>We have all been doing email address validation for a very long time to make sure that the email is correctly formatted. This is to avoid users entering wrongly formatted email address but still they can accidentally give us a wrong email address.</p> 
     <p style="text-align: left;">Example of a correctly formatted email address but still wrong:</p> 
     <p style="text-align: center;"><strong>mailbox.does.not.exist@webdigiapps.com</strong> [VALID format but does not exist]</p> 
     <p style="text-align: left;">Above case specifically happens when you take important customer email on phone and you type in the wrong email. So is there a QUICK solution to really check the email without sending a test message to the user? Yes.<br> <span id="more-19"></span></p> 
     <h3>How to test if the email address actually exists</h3> 
     <p>To check if user entered email <strong>mailbox.does.not.exist</strong><strong>@<strong>webdigiapps</strong>.com</strong> really exists go through the following in command prompt on windows / terminal on mac. The commands you type in are in <span style="color: #339966;">green</span> and the server response is in <span style="color: #3366ff;">blue</span>. Please refer to MAC &amp; PC screenshots towards the end of this post.</p> 
     <p><strong>Step 1&nbsp;</strong> Find mail exchanger or mail server of webdigiapps.com</p> 
     <p style="padding-left: 30px;">COMMAND:<br> <span style="color: #339966;">nslookup -q=mx webdigiapps.com</span><br> RESPONSE:<br> <span style="color: #3366ff;">Non-authoritative answer:</span><br> <span style="color: #3366ff;">webdigiapps.com mail exchanger = 0 mx2.sub3.homie.mail.dreamhost.com.</span><br> <span style="color: #3366ff;">webdigiapps.com mail exchanger = 0 mx1.sub3.homie.mail.dreamhost.com.</span></p> 
     <p style="text-align: left;"><strong>Step 2&nbsp;</strong> Now we know the mail server address so let us connect to it. You can connect to one of the exchanger addresses in the response from Step 1.</p> 
     <p style="text-align: left; padding-left: 30px;">COMMAND:<br> <span style="color: #339966;">telnet mx2.sub3.homie.mail.dreamhost.com&nbsp;25</span><br> RESPONSE:<br> <span style="color: #3366ff;">Connected to mx2.sub3.homie.mail.dreamhost.com.</span><br> <span style="color: #3366ff;">Escape character is ^].</span><br> <span style="color: #3366ff;">220 homiemail-mx7.g.dreamhost.com ESMTP</span></p> 
     <p style="text-align: left; padding-left: 30px;">COMMAND:<br> <span style="color: #339966;">helo hi</span><br> RESPONSE:<br> <span style="color: #3366ff;">250 homiemail-mx8.g.dreamhost.com</span></p> 
     <p style="padding-left: 30px;">COMMAND:<br> <span style="color: #339966;">mail from: &lt;youremail@gmail.com&gt;</span><br> RESPONSE:<br> <span style="color: #3366ff;">250 2.1.0 Ok</span></p> 
     <p style="padding-left: 30px;">COMMAND:<br> <span style="color: #339966;">rcpt to: &lt;mailbox.does.not.exist@webdigiapps.com&gt;</span><br> RESPONSE:<br> <span style="color: #3366ff;">550 5.1.1 &lt;mailbox.does.not.exist@webdigiapps.com&gt;: Recipient address rejected: User unknown in virtual alias table</span></p> 
     <p style="padding-left: 30px;">COMMAND:<br> <span style="color: #339966;">quit</span><br> RESPONSE:<br> <span style="color: #3366ff;">221 2.0.0 Bye</span></p> 
     <h3>Screenshots &nbsp;MAC Terminal &amp; Windows</h3> 
     <p><img class="aligncenter size-full wp-image-14" src="https://www.webdigi.co.uk/blog/wp-content/uploads/2009/01/MAC-Terminal-Verify-Email.png" alt="MAC email verification"><br> <img class="aligncenter size-full wp-image-14" src="https://www.webdigi.co.uk/blog/wp-content/uploads/2009/01/Windows-Email-Verification.png" alt="Windows email verification"></p> 
     <h3>NOTES:</h3> 
     <p>1) the <strong>550 </strong>response indicates that the email address is not valid and you have caught a valid but wrong email address. This code can be on the server and called on AJAX when user tabs out of the email field.&nbsp; The entire check will take less than 2 seconds to run and you can make sure that the email is correct.<br> 2) If email was present the server will respond with a 250 instead of 550<br> 3) There are certain servers with a CATCH ALL email and this means all email address are accepted as valid on their servers (RARE but some servers do have this setting).<br> 4)&nbsp;Please do not use this method to continuously to check for&nbsp;availability&nbsp;of&nbsp;gmail / yahoo /&nbsp;msn accounts etc as this may cause your IP to be added to a blacklist.<br> 5) This is to supplement the standard email address javascript validation.</p>
