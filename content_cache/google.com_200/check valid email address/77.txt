        <h1 class="main_title">RSV – Standalone version</h1> 
        <div class="post-content clearfix"> 
         <div class="entry_content"> 
          <p>This small but powerful script lets you add javascript validation to your forms quickly and with very little effort. No programming experience is necessary, but it’s fully extensible to allow programmers to augment the built-in validation rules with their own. </p> 
          <h3>Contents</h3> 
          <ul> 
           <li><a href="#features">Features</a></li> 
           <li><a href="#getting_started">Getting Started</a></li> 
           <li><a href="#available_validation_rules">Available Validation Rules</a></li> 
           <li><a href="#configuration_options">Configuration Options</a></li> 
           <li><a href="#custom_validation_rules">Custom Validation Rules</a></li> 
           <li><a href="#demo_forms">Demo forms</a></li> 
           <li><a href="#other_comments">Other Comments</a></li> 
           <li><a href="#download">Download</a></li> 
          </ul> 
          <p></p> 
          <hr size="1"> 
          <p><a name="features"></a></p> 
          <h3>Features</h3> 
          <p>RSV comes with a number of validation rules for the more common tasks, including:</p> 
          <ul> 
           <li>Checking required fields</li> 
           <li>Email validation</li> 
           <li>Checking input field lengths: exact minimum or maximum lengths</li> 
           <li>Date validation</li> 
           <li>Integers: checking ranges, in between, greater than, less than</li> 
           <li>Regular Expressions or “custom alpha” (layman’s regexps)</li> 
           <li>Alphanumeric or letters only</li> 
           <li>Nested conditional testing (if field A == something, then validate field B)</li> 
          </ul> 
          <p>But also, the script is fully extensible to allow you to write your own custom validation routines to be integrated with the script.</p> 
          <p>The configuration options include:</p> 
          <ul> 
           <li>The choice of displaying the errors to the user sequentially (a single alert box showing the first error, which re-appears whenever the user resubmits the form), all at once (an single alert box containing all errors) or by displaying the errors as HTML in the page, via DHTML.</li> 
           <li>An option to highlight (via a custom CSS class) all offending fields and auto-focus on the first field for ease of use.</li> 
           <li>For advanced users, there is the option of passing all error information to a custom function to manage the errors as you see fit.</li> 
           <li>Also for advanced users is the option to define a custom onComplete function which gets called when the form validation is successful.</li> 
          </ul> 
          <p>To see a demonstration of various configuration patterns, take a look at the demo pages.</p> 
          <p><a name="getting_started"></a></p> 
          <h3>Getting Started</h3> 
          <p>Okay, let’s get started! To my mind, the quickest way to learn how to implement this script is to take a look at the demo forms. Personally, I always learn much quicker from viewing examples. But for those of a different stripe, here’s some step by step instructions. </p> 
          <ol> 
           <li> First, download the <i>compressed</i> javascript validation file from <a href="https://github.com/benkeen/rsv/tree/master/standalone" target="_blank">github</a> and upload it to the same folder containing your webpage. Import the code in between your webpage’s &lt;head&gt;…&lt;/head&gt;, like so:<p></p> 
            <div class="codecolorer-container javascript blackboard" style="overflow:auto;white-space:nowrap;">
             <table cellspacing="0" cellpadding="0">
              <tbody>
               <tr>
                <td class="line-numbers">
                 <div>
                  1
                  <br>
                 </div></td>
                <td>
                 <div class="javascript codecolorer">
                  <span class="sy0">&lt;</span>script src
                  <span class="sy0">=</span>
                  <span class="st0">"rsv.js"</span>
                  <span class="sy0">&gt;&lt;/</span>script
                  <span class="sy0">&gt;</span>
                 </div></td>
               </tr>
              </tbody>
             </table>
            </div> 
            <div class="comment">
             <b>Note:</b> You can either use the compressed or uncompressed version (rsv-uncompressed.js). But unless you’re planning on modifying the code, I would suggest using the compressed version since it’s smaller for browsers to download.
            </div> </li>

          <div class="codecolorer-container javascript blackboard" style="overflow:auto;white-space:nowrap;">

               <div>
                1
                <br>
               </div></td>

               <div class="javascript codecolorer">

                <span class="st0">"function,your_function_name"</span>
