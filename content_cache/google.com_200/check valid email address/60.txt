       <h2 id="download">How to add JavaScript Form Validation quickly</h2> 
       <p>First, <a href="http://javascript-coder.com/files/form-validation/javascript_form.zip">download the JavaScript form validation script here</a>.<br> The zip file contains the javascript file, examples.</p> 
       <p>The script has a catalog of almost all the common validation types built-in.</p> 
       <p>The idea is to create a set of validation descriptors associated with each element in a form. The validation descriptor is nothing but a string specifying the type of validation to be performed.</p> 
       <p>Each field in the form can have zero one or more validations. For example, you can have an input field that should not be empty, should be less than 25 chars and should be alpha-numeric.</p> 
       <p>In other words, in order to validate a field, you just associate a set of validation descriptors for each input field in the form.</p>

        <li>Include gen_validatorv4.js in your html file just before closing the HEAD tag</li>

        <p>The format of the addValidation() function is:</p>

       <h2>Some Additional Notes</h2> 
       <ul> 
        <li>The form validators should be created only after defining the HTML form (only after the &lt;/form&gt; tag. ) </li> 
        <li>Your form should have a distinguished name. If there are more than one form in the same page, you can add validators for each of them. The names of the forms and the validators should not clash.</li> 
        <li>You cant use the javascript onsubmit event of the form if it you are using this validator script. It is because the validator script automatically overrides the onsubmit event. If you want to add a custom validation, see the section below</li> 
       </ul> 
       <h2 id="custom">Adding a custom validation</h2> 
       <p>If you want to add a custom validation, which is not provided by the validation descriptors, you can do so. Here are the steps:</p> 
       <ol> 
        <li>Create a javascript function which returns true or false depending on the validation</li> 
        <div> 
         <pre class="brush: jscript; title: ; notranslate" title="">
function DoCustomValidation()
{
  var frm = document.forms["myform"];
  if(frm.pwd1.value != frm.pwd2.value)
  {
    sfm_show_error_msg('The Password and verified password does not match!',frm.pwd1);
    return false;
  }
  else
  {
    return true;
  }
}
</pre> 
        </div> 
        <p>sfm_show_error_msg() function displays the error message in your chosen style. The first parameter is the error message and the second parameter is the input object.</p> 
        <li>Associate the validation function with the validator object.</li> 
        <div> 
         <pre class="brush: jscript; title: ; notranslate" title="">
frmvalidator.setAddnlValidationFunction("DoCustomValidation");
</pre> 
        </div> 
        <p>The custom validation function will be called automatically after other validations.</p> 
        <p>If you want to do more than one custom validations, you can do all those validations in the same function.</p> 
        <div> 
         <pre class="brush: jscript; title: ; notranslate" title="">
function DoCustomValidation()
{
  var frm = document.forms["myform"];
  if(false == DoMyValidationOne())
  {
    sfm_show_error_msg('Validation One Failed!');
    return false;
  }
  else
  if(false == DoMyValidationTwo())
  {
    sfm_show_error_msg('Validation Two Failed!');
    return false;
  }
  else
  {
    return true;
  }
}
</pre> 
        </div> 
        <p>where DoMyValidationOne() and DoMyValidationTwo() are custom functions for validation.</p> 
        <h2>Clear All Validations</h2> 
        <p>In some dynamically programmed pages, it may be required to change the validations in the form at run time. For such cases, a function is included which clears all validations in the validator object.</p> 
        <div> 
         <pre class="brush: jscript; title: ; notranslate" title="">
frmvalidator.clearAllValidations();
</pre> 
        </div> 
        <p>This function call clears all validations you set.</p> 
        <h2>Set focus on validation failure</h2> 
        <p>By default, if there is a validation error, the focus is set on the input element having the error. You can disable this behavior by calling:</p> 
        <div> 
         <pre class="brush: jscript; title: ; notranslate" title="">
frmvalidator.EnableFocusOnError(false);
</pre> 
        </div> 
        <h2 id="descriptors">Table of Validation Descriptors</h2> 
        <table class="datatable" cellspacing="0"> 
         <thead> 
          <tr> 
           <td>Validation Descriptor</td> 
           <td>Usage</td> 
          </tr> 
         </thead> 
         <tbody> 
          <tr> 
           <td> required or<br> req </td> 
           <td> The field should not be empty.<p></p> <p>Note that this validation if for fields like Textbox and multi-line text box. For selections like drop down and radio group, use an appropriate validation like dontselect or selone_radio. </p></td> 
          </tr> 
          <tr> 
           <td> maxlen=??? or<br> maxlength=??? </td> 
           <td>Limits the length of the input.<br> For example, if the maximum size permitted is 25, give the validation descriptor as maxlen=25 </td> 
          </tr> 
          <tr> 
           <td> minlen=??? or<br> minlength=??? </td> 
           <td>Checks the length of the entered string to the required minimum.<br> Example minlen=5 </td> 
          </tr> 
          <tr> 
           <td> alphanumeric or<br> alnum </td> 
           <td>The input can contain alphabetic or numeric characters only.<br> (Note that space or punctuation also are not allowed since those characters are not alpha numeric) </td> 
          </tr> 
          <tr> 
           <td> alphanumeric_space<br> alnum_s </td> 
           <td> Allows only alphabetic, numeric and space characters </td> 
          </tr> 
          <tr> 
           <td>num<br> numeric </td> 
           <td>Allow numbers only </td> 
          </tr> 
          <tr> 
           <td>alpha<br> alphabetic </td>
