          <h1>Email Validation<a class="headerlink" href="#email-validation" title="Permalink to this headline">¶</a></h1> 
          <p>This API endpoint is an email address validation service. We will validate the given address based on:</p> 
          <ul class="simple"> 
           <li>Mailbox detection</li> 
           <li>Syntax checks (RFC defined grammar)</li> 
           <li>DNS validation</li> 
           <li>Spell checks</li> 
           <li>Email Service Provider (ESP) specific local-part grammar (if available).</li> 
          </ul> 
          <p>Pricing details for Mailgun’s email validation service can be found on our <a class="reference external" href="https://www.mailgun.com/pricing">pricing page</a>.</p> 
          <p>Mailgun’s email validation service is intended to validate email addresses submitted through forms like newsletters, online registrations and shopping carts. Refer to our <a class="reference external" href="http://mailgun.com/aup">Acceptable Use Policy (AUP)</a> for more information about how to use the service appropriately.</p> 
          <p>Different email validation rates are given based on the API endpoint. Both the public and private API endpoints are limited to a burst per minute rate. The public endpoints have a default limit of calls per month, which can be changed, to prevent abuse of the public API key. The use of private API endpoints for email validation is encouraged and there is no limit past the initial burst per minute rate. It is highly suggested that the private key is used whenever possible.</p> 
          <div class="admonition warning"> 
           <p class="first admonition-title">Warning</p> 
           <p class="last">Do not use your Mailgun private API key on publicly accessible code. Instead, use your Mailgun public key, available in the My Account tab of the Control Panel.</p> 
          </div>

The parsed addresses are a list of addresses that are syntactically valid (and optionally pass DNS and ESP specific grammar checks). The unparsable list is a list of character sequences that could not be parsed (or optionally failed DNS or ESP specific grammar checks). Delimiter characters are comma (,) and semicolon (;).</p>
