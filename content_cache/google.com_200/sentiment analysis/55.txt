         <h1 class="entry-title">How to implement Sentiment Analysis in Tableau using R?</h1> 
        </header>
        <!-- .entry-header --> 
        <div class="entry-content">

         <p>In my previous post I highlighted <a href="http://alexloth.com/2016/01/14/how-to-perform-text-mining-at-the-speed-of-thought-directly-in-tableau/">Tableau’s text mining capabilities</a>, resulting in fancy visuals such as word clouds:</p> 
         <p><script src="https://public.tableau.com/javascripts/api/viz_v1.js" type="text/javascript"></script></p> 
         <div class="tableauPlaceholder" style="width: 100%; height: 400px;">

         <p>Today I’d like to follow up on this and show how to implement sentiment analysis in Tableau using <a href="http://alexloth.com/2013/11/24/data-science-toolbox-use-r-tableau/">Tableau’s R integration</a>. Some of the many uses of social media analytics is sentiment analysis where we evaluate whether posts on a specific issue are positive, neutral, or negative (polarity), and which emotion in predominant.</p> 
         <p><strong>What do customers like or dislike about your products? How do people perceive your brand compared to last year? </strong></p> 
         <p>In order to answer such questions in Tableau, we need to install an R package that is capable of performing the sentiment analysis. In the following example we use an <a href="https://github.com/aloth/sentiment" target="_blank">extended version of the <em>sentiment</em> package</a>, which was <a href="https://github.com/timjurka/sentiment" target="_blank">initiated by Timothy P. Jurka</a>.</p> 
         <p>The <em>sentiment</em> package requires the <a href="https://cran.r-project.org/web/packages/tm/index.html" target="_blank"><em>tm</em></a> and <a href="https://cran.r-project.org/src/contrib/Archive/Rstem/" target="_blank"><em>Rstem</em></a> packages, so make sure that they are installed properly. Execute these commands in your R console to install <em>sentiment</em> from GitHub (see alternative way to install at the end of this blog post):</p> 
         <p></p>
         <div class="embedpress-wrapper ose-github ose-uid-0452d82a552f27e48b8f379f95ee729e responsive">
          <script type="text/javascript" src="https://gist.github.com/aloth/a9eaa1cc477a947c50447968e2217329.js"></script>
         </div>
         <p></p> 
         <p>The <em>sentiment</em> package offers two functions, which can be easily called from calculated fields in Tableau:</p> 
         <p><img class="alignnone size-full wp-image-926" src="https://i2.wp.com/alexloth.com/wp-content/uploads/2016/01/Screenshot-2016-01-31-15.25.24-crop.png?resize=545%2C320" alt="Screenshot 2016-01-31 15.25.24 crop" srcset="https://i2.wp.com/alexloth.com/wp-content/uploads/2016/01/Screenshot-2016-01-31-15.25.24-crop.png?w=545 545w, https://i2.wp.com/alexloth.com/wp-content/uploads/2016/01/Screenshot-2016-01-31-15.25.24-crop.png?resize=300%2C176 300w" sizes="(max-width: 545px) 85vw, 545px" data-recalc-dims="1"></p> 
         <p>The function <code>get_polarity</code> returns “positive”, “neutral”, or “negative”:</p> 
         <p></p>
         <div class="embedpress-wrapper ose-github ose-uid-35854eb5ace33a7ef1a172ebbbb8a9bd responsive">
          <script type="text/javascript" src="https://gist.github.com/aloth/3d425afc264c299f5880.js"></script>
         </div>
         <p></p> 
         <p>The function <code>get_emotion</code> returns “anger”, “disgust”, “fear”, “joy”, “sadness”, “surprise”, or “NA”:</p> 
         <p></p>
         <div class="embedpress-wrapper ose-github ose-uid-264e733e8f3d172d536cbe4fff483ba2 responsive">
          <script type="text/javascript" src="https://gist.github.com/aloth/eb4e59d1b45826db5e8a.js"></script>
         </div>
         <p></p> 
         <p>The <em>sentiment</em> package follows a lexicon based approach and comes with two files <code>emotions_english.csv.gz</code> (<a href="https://github.com/aloth/sentiment/blob/master/sentiment/man/emotions.Rd" target="_blank">source and structure</a>) and <code>subjectivity_english.csv.gz</code> (<a href="https://github.com/aloth/sentiment/blob/master/sentiment/man/subjectivity.Rd" target="_blank">source and structure</a>). Both files contain word lists in English and are stored in the R package library under <code>/sentiment/data</code> directory.</p> 
         <p>If text is incorrectly classified, you could easily fix this issue by extending these two files. If your aim is to analyze text other than English, you need to <a href="http://alexloth.com/2016/07/21/crowdsourcing-help-build-multi-language-sentiment-analysis/">create word lists for the target language</a>. Kindly share them in the comments!</p> 
         <p>Feel free to download the <em>Packaged Workbook</em> (twbx) <a href="http://alexloth.com/assets/twbx/Tweets-featuring-tableau.twbx" target="_blank">here</a>.</p>
