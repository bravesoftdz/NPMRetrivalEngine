             <h1 itemprop="name" class="entry-title">Analysing Sentiments with NLTK</h1> 
             <div class="hidden" itemscope itemtype="http://schema.org/Thing"> 
              <span itemprop="name">Analysing Sentiments with NLTK</span> 
             </div> 
             <div class="meta-info"> 
              <!--<span itemprop="author" itemscope itemtype="http://schema.org/Person" class="entry-author">

              <a href="http://opensourceforu.com/author/shravan-i-v/" title="Posts by Shravan I.V" class="author url fn" rel="author">Shravan I.V</a> 
              <time itemprop="datePublished" class="entry-date" content="2016-12-01" datetime="2016-12-01T17:08:36+00:00"> December 1, 2016 </time> 
              <div class="count-data right"> 
               <span class="meta-info-viewer">2.78K<i class="tm-view"></i></span> 
               <span class="meta-info-comments"><a href="http://opensourceforu.com/2016/12/analysing-sentiments-nltk/"><span class="dsq-postid" data-dsqidentifier="26475 http://opensourceforu.com/?p=26475">0</span></a><i class="tm-comment"></i></span>

            <div class="text-content"> 
             <p><em><img class="alignnone wp-image-26476" src="https://i0.wp.com/opensourceforu.com/wp-content/uploads/2016/12/NTLK.jpg?resize=700%2C442" alt="ntlk" srcset="https://i0.wp.com/opensourceforu.com/wp-content/uploads/2016/12/NTLK.jpg?resize=350%2C221 350w, https://i0.wp.com/opensourceforu.com/wp-content/uploads/2016/12/NTLK.jpg?resize=768%2C485 768w, https://i0.wp.com/opensourceforu.com/wp-content/uploads/2016/12/NTLK.jpg?resize=590%2C372 590w, https://i0.wp.com/opensourceforu.com/wp-content/uploads/2016/12/NTLK.jpg?resize=300%2C189 300w, https://i0.wp.com/opensourceforu.com/wp-content/uploads/2016/12/NTLK.jpg?resize=850%2C536 850w, https://i0.wp.com/opensourceforu.com/wp-content/uploads/2016/12/NTLK.jpg?resize=150%2C95 150w, https://i0.wp.com/opensourceforu.com/wp-content/uploads/2016/12/NTLK.jpg?w=900 900w" sizes="(max-width: 700px) 100vw, 700px" data-recalc-dims="1"></em></p> 
             <p><em>In this article, we explore ways to analyse sentiments from a given text and how some machine learning techniques can help in the process.</em></p> 
             <p>Sentiment analysis is used in opinion mining, business analytics and reputation monitoring. It helps businesses understand the customers experience with a particular service or product by analysing their emotional tone from the product reviews they post, the online recommendations they make, their survey responses and other forms of social media text. Businesses can get feedback on how happy or dissatisfied the customer is, and use this insight to gain a competitive edge.<br> In this article, we explore how to conduct sentiment analysis on a piece of text using some machine learning techniques. Python happens to be one of the best programming language choices when it comes to machine learning and textual analytics as it is easy to learn, is open source, and is effective in catering to machine learning requirements like processing large data sets and performing mathematical computations. Natural Language ToolKit (NLTK) is one of the popular packages in Python that can aid in sentiment analysis.</p> 
             <p><strong>About NLTK</strong><br> NLTK is an open source natural language processing (NLP) platform available for Python. It is capable of textual tokenisation, parsing, classification, stemming, tagging, semantic reasoning and other computational linguistics. NLTK is a community driven project and is available for use on Linux, Mac OS X and Windows.<br> Lets first get started by installing NLTK to glue with Python using the following steps.<br> 1. NLTK can be installed using Pip, a package management tool that Python users might be familiar with. Pip comes, by default, on Python version 2.7.9 and later. However, if you are using an older version of Python and dont have Pip already installed, use the following command to do so.<br> On Ubuntu:</p> 
             <pre class="brush: bash; gutter: false; first-line: 1">sudo apt-get install python-pip</pre> 
             <p>On Fedora Linux:</p> 
             <pre class="brush: bash; gutter: false; first-line: 1">sudo yum install python-pip</pre> 
             <p>2. With Pip, install NLTK using the following command:</p> 
             <pre class="brush: bash; gutter: false; first-line: 1">sudo pip install U nltk</pre> 
             <p>This completes the NLTK download and installation, and you are all set to import and use it in your Python programs.<br> In this article, we will analyse sentiments from a piece of text using the NLTK sentiment analyser and the Naïves Bayes Classifier. As a sample, Ive taken some user reviews on restaurants in Bengaluru from <em>www.zomato.com</em> as shown below.</p> 
             <pre class="brush: bash; gutter: false; first-line: 1">Great place to be when you are in Bangalore.
The place was being renovated when I visited so the seating was limited.
Loved the ambience, loved the food
The food is delicious but not over the top.
Service - Little slow, probably because too many people.
The place is not easy to locate
Mushroom fried rice was tasty</pre> 
             <p><strong>Analysis using NLTK Vader SentimentAnalyser</strong><br> NLTK comes with an inbuilt sentiment analyser module  <em>nltk.sentiment.vader</em>that can analyse a piece of text and classify the sentences under positive, negative and neutral polarity of sentiments. A code snippet of how this could be done is shown below:</p> 
             <pre class="brush: bash; gutter: false; first-line: 1">import nltk
from nltk.sentiment.vader import SentimentIntensityAnalyzer 
&nbsp;
hotel_rev = [Great place to be when you are in Bangalore.,
The place was being renovated when I visited so the seating was limited.,
Loved the ambience, loved the food,
The food is delicious but not over the top.,
Service - Little slow, probably because too many people.,
The place is not easy to locate,
Mushroom fried rice was tasty]
&nbsp;
sid = SentimentIntensityAnalyzer()
for sentence in hotel_rev:
&nbsp;&nbsp;&nbsp;&nbsp; print(sentence)
&nbsp;&nbsp;&nbsp;&nbsp; ss = sid.polarity_scores(sentence)
&nbsp;&nbsp;&nbsp;&nbsp; for k in ss:
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; print({0}: {1}, .format(k, ss[k]), end=)
&nbsp;&nbsp;&nbsp;&nbsp; print()</pre> 
             <p>When running the above Python script, the different sentiment proportions for individual sentences are obtained as shown below:</p> 
             <pre class="brush: bash; gutter: false; first-line: 1">Great place to be when you are in Bangalore.
neg: 0.0, neu: 0.661, compound: 0.6249, pos: 0.339, 
&nbsp;
The place was being renovated when I visited so the seating was limited.
neg: 0.147, neu: 0.853, compound: -0.2263, pos: 0.0, 
&nbsp;
Loved the ambience, loved the food
neg: 0.0, neu: 0.339, compound: 0.8316, pos: 0.661, 
&nbsp;
The food is delicious but not over the top.
neg: 0.168, neu: 0.623, compound: 0.1184, pos: 0.209, 
&nbsp;
Service - Little slow, probably because too many people.
neg: 0.0, neu: 1.0, compound: 0.0, pos: 0.0, 
&nbsp;
The place is not easy to locate
neg: 0.286, neu: 0.714, compound: -0.3412, pos: 0.0, 
&nbsp;
Mushroom fried rice was tasty
neg: 0.0, neu: 1.0, compound: 0.0, pos: 0.0,</pre> 
             <p>The compound value here conveys the overall positive or negative user experience.</p> 
             <p><strong>Analysis using Naïves Bayes Classifier</strong><br> Apart from Vader, one can create ones own classification model using Naïves Bayes Classifier. In the machine learning context, Naïves Bayes Classifier is a probabilistic classifier based on Bayes theorem that constructs a classification model out of training data. This classifier learns to classify the reviews to positive or negative using the supervised learning mechanism. The learning process starts by feeding in sample data that aids the classifier to construct a model to classify these reviews.</p> 
             <pre class="brush: bash; gutter: false; first-line: 1">import nltk
from nltk.tokenize import word_tokenize
&nbsp;
# Step 1  Training data
train = [("Great place to be when you are in Bangalore.", "pos"),
&nbsp; ("The place was being renovated when I visited so the seating was limited.", "neg"),
&nbsp; ("Loved the ambience, loved the food", "pos"),
&nbsp; ("The food is delicious but not over the top.", "neg"),
&nbsp; ("Service - Little slow, probably because too many people.", "neg"),
&nbsp; ("The place is not easy to locate", "neg"),
&nbsp; ("Mushroom fried rice was spicy", "pos"),
]
&nbsp;
# Step 2 
dictionary = set(word.lower() for passage in train for word in word_tokenize(passage[0]))
&nbsp;
# Step 3
t = [({word: (word in word_tokenize(x[0])) for word in dictionary}, x[1]) for x in train]
&nbsp;
# Step 4  the classifier is trained with sample data
classifier = nltk.NaiveBayesClassifier.train(t)
&nbsp;
test_data = "Manchurian was hot and spicy"
test_data_features = {word.lower(): (word in word_tokenize(test_data.lower())) for word in dictionary}
&nbsp;
print (classifier.classify(test_data_features))</pre> 
             <p>The output for the above code can be pos, denoting positive. The training data here is an array of sentences with corresponding class types  positive (pos) or negative (neg) to train the classifier. The dictionary formed in Step 2 consists of all the words obtained by tokenising or breaking this list of sentences. Step 3 starts constructing the data to be fed to the Naïve Bayes Classifier and Step 4 feeds the data to the classifier. With these steps, you might try out testing the classifier with different sentences.</p>
