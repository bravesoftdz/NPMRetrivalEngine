      <h2><a href="https://unbareablelightness.tumblr.com/post/115530471149/sentiment-analysis-in-nodejs">Sentiment Analysis in Node.js</a></h2> 
      <p>I’ve been working on a few projects that require Natural Language Processing (NLP) and have done a lot &nbsp;of research on libraries that implement sentiment analysis. There are actually quite a few out there (I found about 15) but not all of them are created equally. I am using one now in production but there were 3 of them that caught my attention. I focused on libraries that were open source and not tied to any API that would have costs associated with them(Like IBM Watson). </p>
      <h2><a href="http://t.umblr.com/redirect?z=https%3A%2F%2Fgithub.com%2Fhiteshjoshi%2Fnode-stanford-corenlp&amp;t=OTNjYTBjZGRiYmY4ODRlMzU1ZDljMTcxYzQ0NzEzN2FiY2QzZDZiOSxoeG5BY2ttUw%3D%3D&amp;b=t%3A60j9f0QC4jskPTwZ7f54Mg&amp;p=https%3A%2F%2Funbareablelightness.tumblr.com%2Fpost%2F115530471149%2Fsentiment-analysis-in-nodejs&amp;m=1" target="_blank">Node-Stanford-CoreNLP</a></h2>
      <p> This library is a wrapper around the Stanford Core NLP project and it has a very robust set of NLP methods. From a performance perspective, it wasn’t the fastest of all the modules I tested, but it was by far the library with the best classifications. The sentiment analysis was spot on. This is partly because the core &nbsp;package is trained on a lot larger of a corpus than anything else I used. With that said, the performance issues were too much for what I needed to do. If getting a more accurate sentiment tag was important this would have been the no brainer. I did mess around with trying to re-write part of the library to add threading, but I ran out of time and needed to work on the core project.</p>
      <h2><a href="http://t.umblr.com/redirect?z=https%3A%2F%2Fgithub.com%2Fthisandagain%2Fsentiment&amp;t=OWU2MDNhZTNhNDk0MmEyMDBiZWU0MjFiYTkzYjg2ODZkYTI0YmM4YSxoeG5BY2ttUw%3D%3D&amp;b=t%3A60j9f0QC4jskPTwZ7f54Mg&amp;p=https%3A%2F%2Funbareablelightness.tumblr.com%2Fpost%2F115530471149%2Fsentiment-analysis-in-nodejs&amp;m=1" target="_blank">Sentiment</a></h2>
      <p> Sentiment is easy to implement and very fast. It uses the </p>
      <p><a href="http://t.umblr.com/redirect?z=http%3A%2F%2Fwww2.imm.dtu.dk%2Fpubdb%2Fviews%2Fpublication_details.php%3Fid%3D6010&amp;t=OWEwMGQ3MTE2ZjI3MTYxNGYyYTcxOGIwNzg0MTlhYjY0MTIyZmQ4NixoeG5BY2ttUw%3D%3D&amp;b=t%3A60j9f0QC4jskPTwZ7f54Mg&amp;p=https%3A%2F%2Funbareablelightness.tumblr.com%2Fpost%2F115530471149%2Fsentiment-analysis-in-nodejs&amp;m=1" target="_blank">AFINN</a></p>
      <p> as it’s classification methodology and scores accordingly. It also has features for manual override. The AFINN is pretty good, but in the world of sentiment analysis, simple rule based classifications can become a bit unwieldy to mange. That being said, the manual override feature comes in handy. It’s a quick and dirty way to implement sentiment analysis in your application.</p>
      <h2><a href="http://t.umblr.com/redirect?z=https%3A%2F%2Fgithub.com%2Fwooorm%2Fretext-sentiment&amp;t=ZTM2NWJjOTFmNjYxYmFiOWRhZjk2ODUyZmEyZjQ4ZGVhOGZmYTMxZCxoeG5BY2ttUw%3D%3D&amp;b=t%3A60j9f0QC4jskPTwZ7f54Mg&amp;p=https%3A%2F%2Funbareablelightness.tumblr.com%2Fpost%2F115530471149%2Fsentiment-analysis-in-nodejs&amp;m=1" target="_blank">Retext-Sentiment</a></h2>
      <p> This library is probably my favorite. It leverages the AFINN as well, but is a bit more robust when it comes to parsing strings, emoji and other character classifications. As such, it works great with sentiment analysis on social networks and other data. Performance wise it holds up to what I found out of </p>
      <p><i>Sentiment </i></p>
      <p>and it was quite frankly fun to use because of the API design. It’s not as good at classifying as the Stanford Core-NLP but nothing I’ve used in Node.js is. This library gets most of the work done and there exists functionality to do a manual override when needed. </p>
      <p> Overall, I’m excited to see the Node ecosystem grow in this way. With the addition of ES6 coming up, plus library like </p>
      <p><a href="http://t.umblr.com/redirect?z=https%3A%2F%2Fgithub.com%2Fmateogianolio%2Fvectorious&amp;t=YmU5YzIyYWZhMTg5ZDlmN2Q5NDdkYzJmNjhlM2FhNDRjMDFiYzUxMCxoeG5BY2ttUw%3D%3D&amp;b=t%3A60j9f0QC4jskPTwZ7f54Mg&amp;p=https%3A%2F%2Funbareablelightness.tumblr.com%2Fpost%2F115530471149%2Fsentiment-analysis-in-nodejs&amp;m=1" target="_blank">Vectorious</a>&nbsp;and&nbsp;<a href="http://t.umblr.com/redirect?z=http%3A%2F%2Fmathjs.org&amp;t=MWZjNjA3NTg4NGFjYWM0NGMwOTRkMzM0ZTllZGZkN2I5OGI2ZTA5ZSxoeG5BY2ttUw%3D%3D&amp;b=t%3A60j9f0QC4jskPTwZ7f54Mg&amp;p=https%3A%2F%2Funbareablelightness.tumblr.com%2Fpost%2F115530471149%2Fsentiment-analysis-in-nodejs&amp;m=1" target="_blank">Math.js</a>, writing good sentiment analysis and other NLP libraries will become much easier.</p>
