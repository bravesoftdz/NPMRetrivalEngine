           <h1 name="bf72" id="bf72" class="graf graf--h3 graf--leading graf--title">Javascript Sentiment Analysis with Google’s Prediction API</h1>
           <figure name="8631" id="8631" class="graf graf--figure graf-after--h3">
            <div class="aspectRatioPlaceholder is-locked" style="max-width: 300px; max-height: 279px;">
             <div class="aspectRatioPlaceholder-fill" style="padding-bottom: 93%;"></div>
             <div class="progressiveMedia js-progressiveMedia graf-image" data-image-id="1*PTx13XaWhUjxCernfCfFOg.jpeg" data-width="300" data-height="279">
              <img src="https://cdn-images-1.medium.com/freeze/max/60/1*PTx13XaWhUjxCernfCfFOg.jpeg?q=20" crossorigin="anonymous" class="progressiveMedia-thumbnail js-progressiveMedia-thumbnail">
              <canvas class="progressiveMedia-canvas js-progressiveMedia-canvas"></canvas>
              <img class="progressiveMedia-image js-progressiveMedia-image" data-src="https://cdn-images-1.medium.com/max/1600/1*PTx13XaWhUjxCernfCfFOg.jpeg">
              <noscript class="js-progressiveMedia-inner">
               <img class="progressiveMedia-noscript js-progressiveMedia-inner" src="https://cdn-images-1.medium.com/max/1600/1*PTx13XaWhUjxCernfCfFOg.jpeg">
              </noscript>
             </div>
            </div>
            <figcaption class="imageCaption">
             How I felt when first playing with the Google Prediction API
            </figcaption>
           </figure>
           <p name="414f" id="414f" class="graf graf--p graf-after--figure">Let me start off by saying I am a complete newb when it comes to machine learning, advanced algorithms, and things of that nature.</p>
           <p name="9120" id="9120" class="graf graf--p graf-after--p">I’m a web developer for Pete’s sake. (Sorry for taking your name in vain Pete.) I don’t do that sort of thing. I make sure my widgets wobble and my gidgets do whatever those do.</p>
           <p name="3305" id="3305" class="graf graf--p graf-after--p">Sure, I do plenty of backend programming, but my heart is in the Javascript world where algorithms and science-y things aren’t as common.</p>
           <p name="ca68" id="ca68" class="graf graf--p graf-after--p">But I was recently tasked with doing some comparisons between Google’s Prediction API and IBM’s Alchemy APIs for sentiment analysis. Since they both seemed relatively easy to play with, I jumped in head first.</p>
           <p name="965b" id="965b" class="graf graf--p graf-after--p">First, the verdict. With Google’s API I could train my own models (which meant that I could have more contextual results). With Alchemy, I was forced to use their models. Which is fine if you don’t want to do any sort of classification training things.</p>
           <p name="00cd" id="00cd" class="graf graf--p graf-after--p">In short, because I had the data already, Google’s API was far more accurate. I think I had about an 86% accuracy rating if I recall.</p>
           <p name="b267" id="b267" class="graf graf--p graf-after--p">Anyway, that said, let’s dive into what I did.</p>
           <p name="6996" id="6996" class="graf graf--p graf-after--p">As mentioned, I had my own dataset to work with. It looked like this:</p>
           <figure name="36b4" id="36b4" class="graf graf--figure graf--iframe graf-after--p">
            <div class="aspectRatioPlaceholder is-locked">
             <div class="aspectRatioPlaceholder-fill" style="padding-bottom: 35.699999999999996%;"></div>
             <div class="iframeContainer">
              <iframe width="700" height="250" src="/media/5997f9bff17a9e3198526caa94bd279f?postId=b7d154675216" data-media-id="5997f9bff17a9e3198526caa94bd279f" allowfullscreen frameborder="0"></iframe>
             </div>
            </div>
           </figure>
           <p name="839c" id="839c" class="graf graf--p graf-after--figure">Just a simple CSV file.</p>
           <p name="81e9" id="81e9" class="graf graf--p graf-after--p">The first column is the result and the second is what you’re classifying.</p>
           <p name="08b5" id="08b5" class="graf graf--p graf-after--p">You’re basically saying?—?<strong class="markup--strong markup--p-strong">classifierFunction(column[1]) === column[0]</strong></p>
           <p name="700b" id="700b" class="graf graf--p graf-after--p">Keep in mind that the larger the dataset, the better your results. But, as it turns out, the API has limits. I tried to train it on a 1.3 million row file today and it timed out&nbsp;:(</p>
           <p name="f2f5" id="f2f5" class="graf graf--p graf-after--p">C’est la vie.</p>
           <p name="7c2e" id="7c2e" class="graf graf--p graf-after--p">Once you’ve got your model ready to rock, go install this npm package.</p>
           <div name="43d4" id="43d4" class="graf graf--mixtapeEmbed graf-after--p">
            <a href="https://www.npmjs.com/package/google-cloud-prediction-cli" data-href="https://www.npmjs.com/package/google-cloud-prediction-cli" class="markup--anchor markup--mixtapeEmbed-anchor" title="https://www.npmjs.com/package/google-cloud-prediction-cli" rel="nofollow"><strong class="markup--strong markup--mixtapeEmbed-strong">google-cloud-prediction-cli</strong><br><em class="markup--em markup--mixtapeEmbed-em">A small CLI tool that will allow you to easily insert and check the status of training models with the Google Cloud…</em>www.npmjs.com</a>
            <a href="https://www.npmjs.com/package/google-cloud-prediction-cli" class="js-mixtapeImage mixtapeImage u-ignoreBlock" data-media-id="f4f196333feb95c46c4cd768ea24effd" data-thumbnail-img-id="0*CPnTkS34yUBaMJsB." style="background-image: url(https://cdn-images-1.medium.com/fit/c/320/320/0*CPnTkS34yUBaMJsB.);"></a>
           </div>
           <p name="505e" id="505e" class="graf graf--p graf-after--mixtapeEmbed">It’s a little CLI tool I built to make training models simpler.</p>
           <h3 name="dfed" id="dfed" class="graf graf--h3 graf-after--p"><strong class="markup--strong markup--h3-strong">Setting Up Your Model &amp; CLI&nbsp;Tool</strong></h3>
           <p name="aa03" id="aa03" class="graf graf--p graf-after--h3">So, you’ve got your file.</p>
           <p name="dbfb" id="dbfb" class="graf graf--p graf-after--p">Next step is to upload your file to Google Cloud Storage.</p>
           <figure name="9e20" id="9e20" class="graf graf--figure graf--iframe graf-after--p">
            <div class="aspectRatioPlaceholder is-locked">
             <div class="aspectRatioPlaceholder-fill" style="padding-bottom: 175%;"></div>
             <div class="iframeContainer">
              <iframe data-width="552" data-height="966" width="552" height="966" src="/media/7cf6c23724276bb6f8556002cbb5eef0?postId=b7d154675216" data-media-id="7cf6c23724276bb6f8556002cbb5eef0" allowfullscreen frameborder="0"></iframe>
             </div>
            </div>
            <figcaption class="imageCaption">
             Uploading your model file to Google Cloud Storage
            </figcaption>
           </figure>
           <p name="74ae" id="74ae" class="graf graf--p graf-after--figure">Once you’ve gotten to storage, you’ll want to create a bucket and upload the file. No special permissions are required.</p>
           <p name="a57d" id="a57d" class="graf graf--p graf-after--p">One last step and you can train your model.</p>
           <p name="0813" id="0813" class="graf graf--p graf-after--p">For the CLI tool to work, you need to set an environment variable called GOOGLE_APPLICATION_CREDENTIALS that’s just the path to your Google credentials&nbsp;.json file. You can create one of those when activating the Google Prediction API in your console.</p>
           <p name="f7fd" id="f7fd" class="graf graf--p graf-after--p">(In linux/osx, add something like the following to ~/.bash_profile)</p>
           <p name="f548" id="f548" class="graf graf--p graf-after--p">export GOOGLE_APPLICATION_CREDENTIALS=~/.creds/mycredfile.json</p>
           <p name="05ad" id="05ad" class="graf graf--p graf-after--p">Now you’re ready to rock.</p>
           <h3 name="b652" id="b652" class="graf graf--h3 graf-after--p"><strong class="markup--strong markup--h3-strong">Training Your&nbsp;Model</strong></h3>
           <p name="55a2" id="55a2" class="graf graf--p graf-after--h3">I’ll be frank. Google’s APIs are the pits. The docs are incomplete and figuring out what to pass to the methods in their SDK is not intuitive in the least.</p>
           <p name="2bab" id="2bab" class="graf graf--p graf-after--p">So, let the CLI tool do the heavy lifting for you.</p>
           <p name="7ec9" id="7ec9" class="graf graf--p graf-after--p">Once you’ve set everything up, just run the following to train your model.</p>
           <figure name="85f8" id="85f8" class="graf graf--figure graf--iframe graf-after--p">
            <div class="aspectRatioPlaceholder is-locked">
             <div class="aspectRatioPlaceholder-fill" style="padding-bottom: 35.699999999999996%;"></div>
             <div class="iframeContainer">
              <iframe width="700" height="250" src="/media/3368b7506d2c284b424ff4992fb955a7?postId=b7d154675216" data-media-id="3368b7506d2c284b424ff4992fb955a7" allowfullscreen frameborder="0"></iframe>
             </div>
            </div>
           </figure>
           <figure name="154c" id="154c" class="graf graf--figure graf--iframe graf-after--figure">
            <div class="aspectRatioPlaceholder is-locked">
             <div class="aspectRatioPlaceholder-fill" style="padding-bottom: 18%;"></div>
             <div class="iframeContainer">
              <iframe data-width="1332" data-height="240" width="700" height="126" src="/media/91d7e090ebfb90d4a6d278643c100b70?postId=b7d154675216" data-media-id="91d7e090ebfb90d4a6d278643c100b70" allowfullscreen frameborder="0"></iframe>
             </div>
            </div>
           </figure>
           <p name="969a" id="969a" class="graf graf--p graf-after--figure">This uses four flags.</p>
           <ol class="postList">
            <li name="88de" id="88de" class="graf graf--li graf-after--p">insert<br>This tells the tool you’re wanting to?—?well?—?insert and train a new model</li>
            <li name="6128" id="6128" class="graf graf--li graf-after--li">project<br>This is the project name in Google Cloud</li>
            <li name="880d" id="880d" class="graf graf--li graf-after--li">model-id<br>This is what you’re naming the model. Make it easy to remember.</li>
            <li name="f0f8" id="f0f8" class="graf graf--li graf-after--li">model<br>This is a path to the file you uploaded earlier. It’s the bucket + the filename.</li>
           </ol>
           <p name="4dea" id="4dea" class="graf graf--p graf-after--li">Once you’ve inserted it, you can view the status of the training at any time.</p>
           <figure name="3c45" id="3c45" class="graf graf--figure graf--iframe graf-after--p">
            <div class="aspectRatioPlaceholder is-locked">
             <div class="aspectRatioPlaceholder-fill" style="padding-bottom: 35.699999999999996%;"></div>
             <div class="iframeContainer">
              <iframe width="700" height="250" src="/media/e6daa96aa953152e72db2f9056e3a80c?postId=b7d154675216" data-media-id="e6daa96aa953152e72db2f9056e3a80c" allowfullscreen frameborder="0"></iframe>
             </div>
            </div>
           </figure>
           <figure name="d668" id="d668" class="graf graf--figure graf--iframe graf-after--figure">
            <div class="aspectRatioPlaceholder is-locked">
             <div class="aspectRatioPlaceholder-fill" style="padding-bottom: 15.8%;"></div>
             <div class="iframeContainer">
              <iframe data-width="1332" data-height="210" width="700" height="110" src="/media/47740725346ddb2d4b6cb95d64ed84d9?postId=b7d154675216" data-media-id="47740725346ddb2d4b6cb95d64ed84d9" allowfullscreen frameborder="0"></iframe>
             </div>
            </div>
           </figure>
           <p name="b659" id="b659" class="graf graf--p graf-after--figure">That’s it! Now you’ve got a trained model that you can use to predict classifications with.</p>
           <p name="71f8" id="71f8" class="graf graf--p graf-after--p">Let’s use it for sentiment analysis now.</p>
           <p name="4d1e" id="4d1e" class="graf graf--p graf-after--p">Let’s imagine we have a tweet we want to analyze to see if it’s actionable.</p>
           <p name="17eb" id="17eb" class="graf graf--p graf-after--p">We’re going to need to make a request to the Google Prediction API and provide them with our model data and the text we want to use.</p>
           <figure name="a63d" id="a63d" class="graf graf--figure graf--iframe graf-after--p">
            <div class="aspectRatioPlaceholder is-locked">
             <div class="aspectRatioPlaceholder-fill" style="padding-bottom: 35.699999999999996%;"></div>
             <div class="iframeContainer">
              <iframe width="700" height="250" src="/media/22f55f1dd29b495f318a5b023f0aa810?postId=b7d154675216" data-media-id="22f55f1dd29b495f318a5b023f0aa810" allowfullscreen frameborder="0"></iframe>
             </div>
            </div>
           </figure>
           <p name="6ed5" id="6ed5" class="graf graf--p graf-after--figure">If you’re fancy like me, you’ll stuff the nasty callbacks here into promises, but I think this will give you the idea of what’s going on.</p>
           <p name="63e9" id="63e9" class="graf graf--p graf-after--p">And that’s it! Note that your results will live in the ‘outputLabel’ property on the res object.</p>
           <p name="d9fb" id="d9fb" class="graf graf--p graf-after--p">They don’t really make that clear, but that’s where your results are.</p>
           <p name="3b61" id="3b61" class="graf graf--p graf-after--p">Doing machine learning with cloud services like these is pretty damn easy. Thanks Google!</p>
           <p name="84c7" id="84c7" class="graf graf--p graf-after--p">Also to note, I’m considering adding functionality for the last bit into the CLI tool. I just haven’t gotten around to that yet. If anyone wants to contribute to making it better, feel free to toss in a pull request and we can be on our way!</p>
           <p name="768b" id="768b" class="graf graf--p graf-after--p graf--trailing">Thanks for reading and leave a comment if you’ve got any feedback or questions.</p>
