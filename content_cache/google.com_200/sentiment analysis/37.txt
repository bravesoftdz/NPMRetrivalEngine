       <h1 class="entry-title">Day 9: TextBlob–Finding Sentiments in Text</h1> 
       <time class="published" datetime="2013-11-06T00:00:00+00:00">November 6, 2013</time> 
       <p class="byline author vcard">By <a href="https://blog.openshift.com/author/shekhargulati/" rel="author" class="fn">Shekhar Gulati</a></p> 
       <script>

       <p class="addthis_toolbox addthis_default_style" addthis:url="https://blog.openshift.com/day-9-textblob-finding-sentiments-in-text/" addthis:title="Day 9: TextBlob–Finding Sentiments in Text"> <a class="addthis_button_tweet" data-via="openshift" tw:count="none"></a> <a class="addthis_button_google_plusone" g:plusone:count="false" g:plusone:size="medium"></a> <a class="addthis_button_facebook_like" fb:like:layout="button"></a> <a class="addthis_counter addthis_pill_style"></a> </p>

      <div class="entry-content"> 
       <p>Today for <a href="/learning-30-technologies-in-30-days-a-developer-challenge">my 30 day challenge</a>, I decided to take a break from JavaScript and learn about text processing using the Python programming language. I will be focusing on Sentiment Analysis in this blog. My interest in sentiment analysis is few years old when I wanted to write an application which will process a stream of tweets about a movie, and then output the overall sentiment about the movie. Having this information would help me decide if I wanted to watch a particular movie or not.</p> 
       <p>I googled around, and found that <a href="http://en.wikipedia.org/wiki/Naive_Bayes_classifier">Naive Bayes classifier</a> can be used to solve this problem. The only programming language that I knew at the time was Java, so I wrote a custom implementation and used the application for some time. I was lazy to commit the code, so when my machine crashed, I lost the code and application. Now I commit all my code to <a href="https://github.com/shekhargulati">github</a>, and I have close to 200 public repositories :)</p> 
       <p>In this blog, I will talk about a Python package called TextBlob which can help developers solve this problem. We will first cover some basics, and then we will develop a simple Flask application which will use the TextBlob API.</p> 
       <p><img src="/wp-content/uploads/imported/textblob-logo.png" width="250" height="250"></p> 
       <h2>What is TextBlob?</h2> 
       <p><a href="https://github.com/sloria/textblob">TextBlob</a> is an open source text processing library written in Python. It can be used to perform various natural language processing tasks such as part-of-speech tagging, noun-phrase extraction, sentiment analysis, text translation, and many more. You can read about all the features supported by TextBlog in the official <a href="https://textblob.readthedocs.org/en/latest/#features">documentation</a>.</p> 
       <h2>Why should I care?</h2> 
       <p>The reason I decided to learn TextBlob are as follows:</p> 
       <ol> 
        <li> <p>I wanted to develop applications which require text processing. When we add text processing capabilities to the application, the application becomes more human in that it can understand behavior better. Text processing is very hard to get it right. TextBlob stands on strong shoulders of <a href="http://nltk.org/">NTLK</a>, which is the leading platform for building Python programs to work with human language data.</p> </li> 
        <li> <p>I wanted to learn how text processing can be done in Python.</p> </li> 
       </ol> 
       <h2>Install TextBlob</h2> 
       <p>Before we can get started with TextBlob, we need to install <a href="http://www.python.org/download/">Python</a> and <a href="http://www.virtualenv.org/en/latest/">virtualenv</a> on the machine. The Python version I am using in this blog post is 2.7.</p> 
       <p>There are various ways to install TextBlob on the machine as mentioned in the <a href="https://textblob.readthedocs.org/en/latest/install.html">official documentation</a>. We will use the <strong><em>pip install</em></strong> way. For developers unaware of pip, it is Python package manager. We can install pip from the <a href="https://pypi.python.org/pypi/pip">official website</a>. Go to any convenient directory on your file system, and run following commands.</p> 
       <div class="geshifilter"> 
        <pre class="text geshifilter-text">$ mkdir myapp
$ cd myapp
$ virtualenv venv --python=python2.7
$ . venv/bin/activate
$ pip install textblob
$ curl <a href="https://raw.github.com/sloria/TextBlob/master/download_corpora.py" title="https://raw.github.com/sloria/TextBlob/master/download_corpora.py">https://raw.github.com/sloria/TextBlob/master/download_corpora.py</a> | python</pre> 
       </div> 
       <p>The commands above will create a <em>myapp</em> directory on the local machine, then activate <em>virtualenv</em> with Python version 2.7, then install the textblob package, and then finally download the necessary NTLK corpora.</p> 
       <h2>Github Repository</h2> 
       <p>The code for today’s demo application is available on <a href="https://github.com/shekhargulati/day9-textblob-demo-openshift">github: day9-textblob-demo-openshift</a>.</p> 
       <h2>Application</h2> 
       <p>The demo application is running on OpenShift <a href="http://showmesentiments-t20.rhcloud.com/" title="http://showmesentiments-t20.rhcloud.com/">http://showmesentiments-t20.rhcloud.com/</a>. It is a very simple example of using TextBlob sentiment analysis API. As user types, he will see whether the message is positive(Green), negative(Red), or neutral(Orange).</p> 
       <p><img src="/wp-content/uploads/imported/Sentiment_Analysis_Demo.png" width="600" alt="Sentiment Analysis Demo app running on OpenShift"></p> 
       <p>We will develop a simple Flask application which will expose a REST API. If you are not aware of Flask, you can refer to my <a href="/day-3-flask-instant-python-web-development-with-python-and-openshift">earlier post on it</a>.</p> 
       <p>Next we will install the Flask framework. To install the Flask framework, we will run first activate the <em>virtualenv</em> and then use pip to install Flask.</p> 
       <div class="geshifilter"> 
        <pre class="text geshifilter-text">$ . venv/bin/activate
$ pip install flask</pre> 
       </div> 
       <p>As I mentioned in my earlier blog post on Flask, it is awesome for writing REST based web services. Create a new file called <em>app.py</em> under the <em>myapp</em> folder.</p> 
       <div class="geshifilter"> 
        <pre class="text geshifilter-text">$ touch app.py</pre> 
       </div> 
       <p>Copy the following code and paste it in the <em>app.py</em> source file</p> 
       <div class="geshifilter"> 
        <pre class="text geshifilter-text">from flask import Flask , jsonify, render_template
from textblob import TextBlob
&nbsp;
app = Flask(__name__)
&nbsp;
@app.route('/')
@app.route('/index')
def index():
    return render_template('index.html')
&nbsp;
@app.route('/api/v1/sentiment/&lt;message&gt;')
def sentiment(message):
    text = TextBlob(message)
    response = {'polarity' : text.polarity , 'subjectivity' : text.subjectivity}
    return jsonify(response)
&nbsp;
if __name__ == "__main__":
    app.run(debug=True)</pre> 
       </div> 
       <p>The code shown above does the following:</p> 
       <ol> 
        <li>It imports the Flask class, jsonify function, and render_template function from flask package.</li> 
        <li>It imports the TextBlob class from textblob package.</li> 
        <li>It defines a route to ‘/’ and ‘index’ url. So, if a user makes a GET request to either ‘/’ or ‘/index’, then the <strong><em>index.html</em></strong> will be rendered.</li> 
        <li>It defines a route to ‘/api/v1/sentiment/’ url. The is a placeholder and will contain the text message the user want to run sentiment analysis on. We create an instance of TextBlob passing it the message. Next, we get polarity and subjectivity of the message, and then create a json object and return it back.</li> 
        <li>Finally, we start the development server to run the application using the python app.py command. We also enabled debugging by passing Debug=True. Debugging provides an interactive debugger in the browser when an unexpected exceptions occur. Another benefit of the debugger is that it will automatically reload the changes. We can keep the debugger running in the background and work through our application. This provides a highly productive environment.</li> 
       </ol> 
       <p>The index() function renders an html file. Create a new folder called templates in the <em>myapp</em> directory and then create new file named <em>index.html</em>.</p> 
       <div class="geshifilter"> 
        <pre class="text geshifilter-text">$ mkdir templates
$ touch templates/index.html</pre> 
       </div>

       <h2>Deploy to the cloud</h2> 
       <p>Before we can deploy the application to our cloud environment, we’ll have to do few setup tasks :</p> 
       <ol>
