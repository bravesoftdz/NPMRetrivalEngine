           <h3 name="373c" id="373c" class="graf graf--h3 graf-after--p">How the demo&nbsp;works</h3>
           <p name="a030" id="a030" class="graf graf--p graf-after--h3">On the backend, I wrote a Node server that streams tweets using the Twitter Streaming API. It sends the text of each tweet to the Natural Language API and writes the NL API response to both Firebase for our realtime dashboard and BigQuery for analysis when we’re done collecting data. The architecture looks something like this:</p>

           <h4 name="0b77" id="0b77" class="graf graf--h4 graf-after--figure">Streaming tweets in&nbsp;realtime</h4>
           <p name="6550" id="6550" class="graf graf--p graf-after--h4">Let’s look at a few code snippets from each step in the diagram starting with the Node server. Using the <a href="https://www.npmjs.com/package/twitter" data-href="https://www.npmjs.com/package/twitter" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">twitter npm package</a> I create a client with my API credentials and stream tweets containing a few search terms:</p>
           <pre name="4b7b" id="4b7b" class="graf graf--pre graf-after--p">const searchTerms = 'googleio,googlecloud,firebase';</pre>
           <pre name="ee32" id="ee32" class="graf graf--pre graf-after--pre">client.stream('statuses/filter', {track: searchTerms, language: 'en'}, function(stream) {<br>  stream.on('data', function(event) {<br>    callNlApi(event);<br>  });<br>});</pre>
           <p name="53ae" id="53ae" class="graf graf--p graf-after--pre">When I new tweet comes in the API returns data for each tweet including the text, user, hashtags, time zone, and more.</p>
           <h4 name="884c" id="884c" class="graf graf--h4 graf-after--p">Calling the Natural Language&nbsp;API</h4>
           <p name="4657" id="4657" class="graf graf--p graf-after--h4">With the text of the tweet, I’m ready to call the NL API using the npm <a href="https://www.npmjs.com/package/request" data-href="https://www.npmjs.com/package/request" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">request</a> module. My request JSON looks like this:</p>
           <pre name="868d" id="868d" class="graf graf--pre graf-after--p">{<br>  "document": {<br>    "type": "PLAIN_TEXT",<br>    "content": tweet.text<br>  },<br>  "features": {<br>    "extractSyntax": true,<br>    "extractEntities": true,<br>    "extractDocumentSentiment": true<br>  }<br>}</pre>
           <p name="948e" id="948e" class="graf graf--p graf-after--pre">I’m making use of all 3 features of the NL API: syntax annotation, entity extraction, and sentiment analysis (more on those in the next section). Once I get the response, I format it to write the results to Firebase and BigQuery?—?it’ll be a slightly different format for each.</p>
           <h4 name="23d2" id="23d2" class="graf graf--h4 graf-after--p">Saving NL data to&nbsp;Firebase</h4>
           <p name="372f" id="372f" class="graf graf--p graf-after--h4">I combine data returned from the Twitter API (tweet text and location) with data from the NL API (syntax, entities, sentiment) and save it in an object to write to Firebase:</p>
           <pre name="0b5e" id="0b5e" class="graf graf--pre graf-after--p">let tweetDataForFirebase = {<br>  id: tweet.id_str,<br>  text: tweet.text,<br>  user: tweet.user.screen_name,<br>  user_time_zone: tweet.user.time_zone,<br>  user_followers_count: tweet.user.followers_count,<br>  hashtags: tweet.entities.hashtags,<br>  tokens: body.tokens,<br>  score: body.documentSentiment.score,<br>  magnitude: body.documentSentiment.magnitude,<br>  entities: body.entities<br>}</pre>
           <p name="0084" id="0084" class="graf graf--p graf-after--pre">Saving it to Firebase is easy. First we need to initialize the <code class="markup--code markup--p-code">firebase-admin</code> module for our app for writing to our Firebase database:</p>
           <pre name="dbfc" id="dbfc" class="graf graf--pre graf-after--p">var admin = require("firebase-admin");<br>var serviceAccount = require("./service-acct.json");<br>admin.initializeApp({<br>  credential: admin.credential.cert(serviceAccount),<br>  databaseURL: "<a href="https://io-nlp-demo.firebaseio.com" data-href="https://io-nlp-demo.firebaseio.com" class="markup--anchor markup--pre-anchor" rel="nofollow noopener" target="_blank">https://your-firebase-project-id.firebaseio.com</a>"<br>});</pre>
           <p name="11af" id="11af" class="graf graf--p graf-after--pre">Then we’ll create a reference to the path in our database where we want to store the tweet:</p>
           <pre name="50ab" id="50ab" class="graf graf--pre graf-after--p">const db = admin.database();<br>const latestTweetRef = db.ref('latest');</pre>
           <p name="2785" id="2785" class="graf graf--p graf-after--pre">I’ll only store the full JSON data for the latest tweet. Writing that to Firebase is just one line of code:</p>
           <pre name="b186" id="b186" class="graf graf--pre graf-after--p">latestTweetRef.set(tweetDataForFirebase);</pre>
           <p name="f4aa" id="f4aa" class="graf graf--p graf-after--pre">I’ve got a listener at that <code class="markup--code markup--p-code">/latest</code> path, and whenever a new tweet is added I check for adjectives and hashtags, count them, and keep a total count of each adjective and hashtag seen in Firebase:</p>

           <h4 name="7378" id="7378" class="graf graf--h4 graf-after--figure">Saving NL data to&nbsp;BigQuery</h4>
           <p name="c513" id="c513" class="graf graf--p graf-after--h4">We’ll write the same data to BigQuery as we did to Firebase, but we’ll <code class="markup--code markup--p-code">JSON.stringify()</code> hashtags, tokens, and entities so they are strings in our BigQuery table. With BigQuery initialized (docs on how to do that <a href="https://googlecloudplatform.github.io/google-cloud-node/#/docs/google-cloud/v0.55.0/bigquery" data-href="https://googlecloudplatform.github.io/google-cloud-node/#/docs/google-cloud/v0.55.0/bigquery" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">here</a>), we can write the tweet data to our table:</p>
           <pre name="e852" id="e852" class="graf graf--pre graf-after--p">table.insert(bqRow, function(error, insertErr, apiResp) {<br>  if (error) {<br>    console.log('err', error);<br>  } else {<br>     console.log('successfully added the row!');<br>  }<br>});</pre>
           <p name="6063" id="6063" class="graf graf--p graf-after--pre">Now we can query the data to find, for example, the most common adjectives used to describe machine learning. Since our token data is stored as JSON strings, we’ll make use of BigQuery’s <a href="https://cloud.google.com/bigquery/docs/reference/standard-sql/user-defined-functions" data-href="https://cloud.google.com/bigquery/docs/reference/standard-sql/user-defined-functions" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">UDF feature</a> to parse them:</p>
           <pre name="7f69" id="7f69" class="graf graf--pre graf-after--p">SELECT COUNT(*) as adj_count, adjective<br>FROM <br> JS(<br> (SELECT tokens FROM [sara-bigquery:googlenext_17.tweets] ),<br> tokens,<br> "[{ name:'adjective', type: 'string'}]",<br> "function(row, emit) { <br>   try {<br>     x = JSON.parse(row.tokens);<br>     x.forEach(function(token) {<br>       if (token.partOfSpeech.tag === 'ADJ') {<br>         emit({ adjective: token.lemma.toLowerCase() });<br>       }<br>     });<br>   } catch (e) {}<br> }" <br> )<br>GROUP BY adjective<br>ORDER BY adj_count DESC<br>LIMIT 100Visualizing tweet data in realtime</pre>

           <h3 name="045d" id="045d" class="graf graf--h3 graf-after--p">What is the Natural Language&nbsp;API?</h3>
           <p name="ea78" id="ea78" class="graf graf--p graf-after--h3">If you’ve gotten this far you might be wondering: what is this <a href="https://cloud.google.com/natural-language/" data-href="https://cloud.google.com/natural-language/" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">NL API</a> and what can I do with it? The NL API helps you understand the meaning of text with a single API call. More specifically, it provides three methods:</p>
           <ul class="postList">
            <li name="6cad" id="6cad" class="graf graf--li graf-after--p"><a href="https://cloud.google.com/natural-language/docs/analyzing-entities" data-href="https://cloud.google.com/natural-language/docs/analyzing-entities" class="markup--anchor markup--li-anchor" rel="noopener" target="_blank"><strong class="markup--strong markup--li-strong">Analyze entities</strong></a>: extract entities like people, landmarks, and organizations from text. This method returns data on the type of entity, its importance in the context of the document, and a Wikipedia URL for the entity if it exists.</li>
            <li name="ef2f" id="ef2f" class="graf graf--li graf-after--li"><a href="https://cloud.google.com/natural-language/docs/analyzing-sentiment" data-href="https://cloud.google.com/natural-language/docs/analyzing-sentiment" class="markup--anchor markup--li-anchor" rel="noopener" target="_blank"><strong class="markup--strong markup--li-strong">Analyze sentiment</strong></a>: find out if your text is positive or negative, and how strong the overall sentiment of the text. There’s also a new beta feature that provides <a href="https://cloud.google.com/natural-language/docs/analyzing-entity-sentiment" data-href="https://cloud.google.com/natural-language/docs/analyzing-entity-sentiment" class="markup--anchor markup--li-anchor" rel="noopener" target="_blank">entity-based sentiment</a> to get the sentiment around specific entities in your text. Here’s an example for the sentence <em class="markup--em markup--li-em">“I liked the sushi but the service was terrible”</em>:</li>
           </ul>

This is useful for analyzing how people talk about a specific topic.</li>
