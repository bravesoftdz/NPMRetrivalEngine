     <h1><a href="http://varianceexplained.org/r/yelp-sentiment/" rel="bookmark" title="Does sentiment analysis work? A tidy analysis of Yelp reviews">Does sentiment analysis work? A tidy analysis of Yelp reviews</a></h1> 
    </div>
    <!--/ .headline-wrap --> 
    <div class="article-wrap"> 
     <p>This year <a href="http://juliasilge.com/">Julia Silge</a> and I released the <a href="https://github.com/juliasilge/tidytext">tidytext</a> package for text mining using tidy tools such as <a href="https://cran.r-project.org/package=dplyr">dplyr</a>, <a href="https://cran.r-project.org/package=tidyr">tidyr</a>, <a href="https://cran.r-project.org/package=ggplot2">ggplot2</a> and <a href="https://cran.r-project.org/package=broom">broom</a>. One of the canonical examples of tidy text mining this package makes possible is <a href="https://en.wikipedia.org/wiki/Sentiment_analysis">sentiment analysis</a>.</p> 
     <p>Sentiment analysis is often used by companies to quantify general social media opinion (for example, using tweets about several brands to compare customer satisfaction). One of the simplest and most common sentiment analysis methods is to classify words as “positive” or “negative”, then to average the values of each word to categorize the entire document. (See <a href="https://cran.r-project.org/web/packages/tidytext/vignettes/tidytext.html">this vignette</a> and <a href="http://juliasilge.com/blog/Life-Changing-Magic/">Julia’s post</a> for examples of a tidy application of sentiment analysis). But does this method actually <em>work</em>? Can you predict the positivity or negativity of someone’s writing by counting words?</p> 
     <p>To answer this, let’s try sentiment analysis on a text dataset where we know the “right answer”- one where each customer also quantified their opinion. In particular, we’ll use the <a href="https://www.yelp.com/dataset_challenge">Yelp Dataset</a>: a wonderful collection of millions of restaurant reviews, each accompanied by a 1-5 star rating. We’ll try out a specific sentiment analysis method, and see the extent to which we can predict a customer’s rating based on their written opinion. In the process we’ll get a sense of the strengths and weaknesses of sentiment analysis, and explore another example of tidy text mining with tidytext, dplyr, and ggplot2.</p> 
     <h3 id="setup">Setup</h3> 
     <p>I’ve downloaded the <code class="highlighter-rouge">yelp_dataset_challenge_academic_dataset</code> folder from <a href="https://www.yelp.com/dataset_challenge">here</a>.<sup id="fnref:termsofuse"><a href="#fn:termsofuse" class="footnote">1</a></sup> First I read and process them into a data frame:</p>

     <h3 id="tidy-sentiment-analysis">Tidy sentiment analysis</h3> 
     <p>Right now, there is one row for each review. To analyze in the <a href="http://github.com/juliasilge/tidytext">tidy text</a> framework, we need to use the <code class="highlighter-rouge">unnest_tokens</code> function and turn this into one-row-per-term-per-document:</p> 
     <figure class="highlight">
      <pre><code class="language-r" data-lang="r"><span class="n">library</span><span class="p">(</span><span class="n">tidytext</span><span class="p">)</span><span class="w"> </span><span class="n">review_words</span><span class="w"> </span><span class="o">&lt;-</span><span class="w"> </span><span class="n">reviews</span><span class="w"> </span><span class="o">%&gt;%</span><span class="w"> </span><span class="n">select</span><span class="p">(</span><span class="n">review_id</span><span class="p">,</span><span class="w"> </span><span class="n">business_id</span><span class="p">,</span><span class="w"> </span><span class="n">stars</span><span class="p">,</span><span class="w"> </span><span class="n">text</span><span class="p">)</span><span class="w"> </span><span class="o">%&gt;%</span><span class="w"> </span><span class="n">unnest_tokens</span><span class="p">(</span><span class="n">word</span><span class="p">,</span><span class="w"> </span><span class="n">text</span><span class="p">)</span><span class="w"> </span><span class="o">%&gt;%</span><span class="w"> </span><span class="n">filter</span><span class="p">(</span><span class="o">!</span><span class="n">word</span><span class="w"> </span><span class="o">%in%</span><span class="w"> </span><span class="n">stop_words</span><span class="o">$</span><span class="n">word</span><span class="p">,</span><span class="w"> </span><span class="n">str_detect</span><span class="p">(</span><span class="n">word</span><span class="p">,</span><span class="w"> </span><span class="s2">"^[a-z']+$"</span><span class="p">))</span><span class="w"> </span><span class="n">review_words</span></code></pre>
     </figure>

     <p>Notice that there is now one-row-per-term-per-document: the tidy text form. In this cleaning process we’ve also removed “stopwords” (such as “I”, “the”, “and”, etc), and removing things things that are formatting (e.g. “—-“) rather than a word.</p> 
     <p>Now let’s perform sentiment analysis on each review. We’ll use the <a href="http://www2.imm.dtu.dk/pubdb/views/publication_details.php?id=6010">AFINN</a> lexicon, which provides a positivity score for each word, from -5 (most negative) to 5 (most positive). This, along with several other lexicons, are stored in the <code class="highlighter-rouge">sentiments</code> table that comes with tidytext. (I’ve tried some other lexicons on this dataset and the results are pretty similar.)</p>

     <p>Now as described <a href="http://juliasilge.com/blog/Life-Changing-Magic/">in Julia’s post</a>, our sentiment analysis is just an inner-join operation followed by a summary:</p> 
     <figure class="highlight">
      <pre><code class="language-r" data-lang="r"><span class="n">reviews_sentiment</span><span class="w"> </span><span class="o">&lt;-</span><span class="w"> </span><span class="n">review_words</span><span class="w"> </span><span class="o">%&gt;%</span><span class="w"> </span><span class="n">inner_join</span><span class="p">(</span><span class="n">AFINN</span><span class="p">,</span><span class="w"> </span><span class="n">by</span><span class="w"> </span><span class="o">=</span><span class="w"> </span><span class="s2">"word"</span><span class="p">)</span><span class="w"> </span><span class="o">%&gt;%</span><span class="w"> </span><span class="n">group_by</span><span class="p">(</span><span class="n">review_id</span><span class="p">,</span><span class="w"> </span><span class="n">stars</span><span class="p">)</span><span class="w"> </span><span class="o">%&gt;%</span><span class="w"> </span><span class="n">summarize</span><span class="p">(</span><span class="n">sentiment</span><span class="w"> </span><span class="o">=</span><span class="w"> </span><span class="n">mean</span><span class="p">(</span><span class="n">afinn_score</span><span class="p">))</span><span class="w"> </span><span class="n">reviews_sentiment</span></code></pre>

     <h3 id="next-time-machine-learning">Next time: Machine learning</h3> 
     <p>In this post I’ve focused on basic exploration of the Yelp review dataset, and an evaluation of one sentiment analysis method for predicting review positivity. (Our conclusion: it’s good, but far from perfect!) But what if we want to create our own prediction method based on these reviews?</p> 
     <p>In my next post on this topic, I’ll show how to train LASSO regression (with the <a href="https://cran.r-project.org/web/packages/glmnet/index.html">glmnet</a> package) on this dataset to create a predictive model. This will serve as an introduction to machine learning methods in text classification. It will also let us create our own new “lexicon” of positive and negative words, one that may be more appropriate to our context of restaurant reviews.</p>
