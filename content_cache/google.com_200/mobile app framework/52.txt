          <p>Keep an eye on <a href="https://github.com/trueadm/inferno" target="_blank">inferno</a>, <a href="https://svelte.technology/guide" target="_blank">Svelte</a>, and <a href="https://github.com/RisingStack/nx-framework" target="_blank">NX</a> in 2017 for building component based UI applications.</p> 
          <hr> 
          <h6 id="advice">ADVICE:</h6> 
          <p><sup><a href="#fn_1" id="reffn_1">1</a></sup> If you are new to front-end/JavaScript application development I'd start with <a href="http://riotjs.com/" target="_blank">Riot</a> or <a href="http://vuejs.org/" target="_blank">Vue.js</a>. Then I'd work my way to <a href="http://facebook.github.io/react/" target="_blank">React</a>. Then I'd look at <a href="https://angular.io/" target="_blank">Angular 2</a>, <a href="http://emberjs.com/" target="_blank">Ember</a>, or <a href="http://aurelia.io/" target="_blank">Aurelia</a>.</p> 
          <p>If you are building a simple website that has minimal interactions with data (i.e. mostly a static content web site), you should avoid a front-end framework. A lot of work can be done with a task runner like <a href="https://github.com/vigetlabs/gulp-starter" target="_blank">Gulp and jQuery</a>, while avoiding the unnecessary complexity of learning and use an app framework tool.</p> 
          <p>Want something smaller than React, consider <a href="https://preactjs.com/" target="_blank">Preact</a>. Preact is an attempt to recreate the core value proposition of React (or similar libraries like Mithril) using as little code as possible, with first-class support for ES2015. Currently the library is around 3kb (minified &amp; gzipped).</p> 
          <p>Can't decide between React or Angluar 2, read, <a href="https://medium.com/javascript-scene/angular-2-vs-react-the-ultimate-dance-off-60e7dfbc379c#.j0r1xdvzl" target="_blank">"Angular 2 vs React: The Ultimate Dance Off"</a></p>
