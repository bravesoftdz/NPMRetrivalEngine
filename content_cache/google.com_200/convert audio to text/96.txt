       <p>When the service for your skill returns a response to a user’s request, you provide text that the Alexa service converts to speech. Alexa automatically handles normal punctuation, such as pausing after a period, or speaking a sentence ending in a question mark as a question.</p> 
       <p>However, in some cases you may want additional control over how Alexa generates the speech from the text in your response. For example, you may want a longer pause within the speech, or you may want a string of digits read back as a standard telephone number. The Alexa Skills Kit provides this type of control with <em>Speech Synthesis Markup Language (SSML)</em> support.</p> 
       <p><a href="http://www.w3.org/TR/speech-synthesis11/" title="W3C Speech Synthesis Markup Language (SSML) Version 1.1">SSML</a> is a markup language that provides a standard way to mark up text for the generation of synthetic speech. The Alexa Skills Kit supports a <em>subset</em> of the tags defined in the SSML specification. The specific tags supported are listed in <a href="#ssml-supported">Supported SSML Tags</a>.</p>

       <h2 id="using-ssml-in-your-response">Using SSML in Your Response</h2> 
       <p>To use SSML, construct your output speech using the <a href="#ssml-supported">supported SSML tags</a>. When sending back a response from your service, you must indicate that it is using SSML rather than plain text:</p> 
       <ul> 
        <li>When using the Java library, use the <code class="highlighter-rouge">SsmlOutputSpeech</code> class. Call the <code class="highlighter-rouge">setSsml()</code> method and pass in the output speech marked up with the tags.</li> 
        <li>When not using the Java library, provide the marked-up text in the <code class="highlighter-rouge">outputSpeech</code> property, but set the <code class="highlighter-rouge">type</code> to <code class="highlighter-rouge">SSML</code> instead of <code class="highlighter-rouge">PlainText</code>. Use the <code class="highlighter-rouge">ssml</code> property instead of <code class="highlighter-rouge">text</code> for the marked-up text:</li> 
       </ul> 
       <div class="language-json highlighter-rouge">
        <pre class="highlight"><code><span class="s2">"outputSpeech"</span><span class="err">:</span><span class="w"> </span><span class="p">{</span><span class="w"> </span><span class="nt">"type"</span><span class="p">:</span><span class="w"> </span><span class="s2">"SSML"</span><span class="p">,</span><span class="w"> </span><span class="nt">"ssml"</span><span class="p">:</span><span class="w"> </span><span class="s2">"&lt;speak&gt;This output speech uses SSML.&lt;/speak&gt;"</span><span class="w"> </span><span class="p">}</span><span class="w"> </span></code></pre> 
       </div> 
       <ul> 
        <li>You can use SSML with both the normal output speech response and any re-prompt included in the response.</li> 
       </ul> 
       <p>The SSML you provide must be wrapped within <code class="highlighter-rouge">&lt;speak&gt;</code> tags. For example:</p> 
       <div class="language-xml highlighter-rouge">
        <pre class="highlight"><code><span class="nt">&lt;speak&gt;</span>
    Here is a number <span class="nt">&lt;w</span> <span class="na">role=</span><span class="s">"amazon:VBD"</span><span class="nt">&gt;</span>read<span class="nt">&lt;/w&gt;</span> 
    as a cardinal number: 
    <span class="nt">&lt;say-as</span> <span class="na">interpret-as=</span><span class="s">"cardinal"</span><span class="nt">&gt;</span>12345<span class="nt">&lt;/say-as&gt;</span>. 
    Here is a word spelled out: 
    <span class="nt">&lt;say-as</span> <span class="na">interpret-as=</span><span class="s">"spell-out"</span><span class="nt">&gt;</span>hello<span class="nt">&lt;/say-as&gt;</span>. 
<span class="nt">&lt;/speak&gt;</span>
</code></pre> 
       </div> 
       <h2 id="ssml-supported">Supported SSML Tags</h2> 
       <p>The Alexa Skills Kit supports the following SSML tags (listed in alphabetic order):</p> 
       <ul> 
        <li><a href="#amazon-effect"><code class="highlighter-rouge">amazon:effect</code></a></li> 
        <li><a href="#audio"><code class="highlighter-rouge">audio</code></a></li> 
        <li><a href="#break"><code class="highlighter-rouge">break</code></a></li> 
        <li> <p><a href="#emphasis"><code class="highlighter-rouge">emphasis</code></a></p> </li> 
        <li><a href="#p"><code class="highlighter-rouge">p</code></a></li> 
        <li><a href="#phoneme"><code class="highlighter-rouge">phoneme</code></a></li> 
        <li><a href="#prosody"><code class="highlighter-rouge">prosody</code></a></li> 
        <li><a href="#s"><code class="highlighter-rouge">s</code></a></li> 
        <li><a href="#say-as"><code class="highlighter-rouge">say-as</code></a></li> 
        <li><a href="#speak"><code class="highlighter-rouge">speak</code></a></li> 
        <li><a href="#sub"><code class="highlighter-rouge">sub</code></a></li>

       <p>Note that the Alexa service strips out any unsupported SSML tags included in the text you provide.</p>

       <h2 id="audio">audio</h2> 
       <p>The <code class="highlighter-rouge">audio</code> tag lets you provide the URL for an MP3 file that the Alexa service can play while rendering a response. You can use this to embed short, pre-recorded audio within your service’s response. For example, you could include sound effects alongside your text-to-speech responses, or provide responses using a voice associated with your brand. For more information, see <a href="https://developer.amazon.com/public/solutions/alexa/alexa-skills-kit/docs/handling-requests-sent-by-alexa#audio">Including Short Pre-Recorded Audio in your Response</a>.</p> 
       <table class="grid"> 
        <colgroup> 
         <col width="25%"> 
         <col width="75%"> 
        </colgroup> 
        <thead> 
         <tr class="header"> 
          <th align="left">Attribute</th> 
          <th align="left">Possible Values</th> 
         </tr> 
        </thead> 
        <tbody> 
         <tr> 
          <td align="left"> <p><code class="highlighter-rouge">src</code></p> </td> 
          <td align="left"> <p>Specifies the URL for the MP3 file. Note the following requirements and limitations:</p> 
           <ul> 
            <li>The MP3 must be hosted at an Internet-accessible HTTPS endpoint. HTTPS is required, and the domain hosting the MP3 file must present a valid, <em>trusted</em> SSL certificate. Self-signed certificates cannot be used.</li> 
            <li>The MP3 must not contain any customer-specific or other sensitive information.</li> 
            <li>The MP3 must be a valid MP3 file (<em>MPEG version 2</em>).</li> 
            <li>The audio file cannot be longer than ninety (90) seconds.</li> 
            <li>The bit rate must be 48 kbps. Note that this bit rate gives a good result when used with spoken content, but is generally not a high enough quality for music.</li> 
            <li>The sample rate must be 16000 Hz.</li> 
           </ul> <p>You may need to <a href="#h3_converting_mp3">use converter software</a> to convert your MP3 files to the required codec version (MPEG version 2) and bit rate (48 kbps).</p> </td> 
         </tr> 
        </tbody> 
       </table> 
       <p>Include the <code class="highlighter-rouge">audio</code> tag within your text-to-speech response within the <code class="highlighter-rouge">speak</code> tag. Alexa plays the MP3 at the specified point within the text to speech. For example:</p> 
       <div class="language-xml highlighter-rouge">
        <pre class="highlight"><code><span class="nt">&lt;speak&gt;</span>
    Welcome to Car-Fu. 
    <span class="nt">&lt;audio</span> <span class="na">src=</span><span class="s">"https://carfu.com/audio/carfu-welcome.mp3"</span> <span class="nt">/&gt;</span> 
    You can order a ride, or request a fare estimate. 
    Which will it be?
<span class="nt">&lt;/speak&gt;</span> 
</code></pre> 
       </div> 
       <p>When Alexa renders this response, it would sound like this:</p> 
       <p><span style="color:blue"><strong>Alexa:</strong> Welcome to Car-Fu.</span><br> <span style="color:gray"><em>(the specified <code class="highlighter-rouge">carfu-welcome.mp3</code> audio file plays)</em></span><br> <span style="color:blue"><strong>Alexa:</strong> You can order a ride, or request a fare estimate. Which will it be?</span><br></p> 
       <p>A single response sent by your service can include multiple <code class="highlighter-rouge">audio</code> tags according to the following limits:</p> 
       <ul> 
        <li>No more than <em>five</em> audio files can be used in a single response.</li> 
        <li>The combined total time for all audio files in a single response cannot be more than <em>ninety (90) seconds</em>.</li> 
       </ul> 
       <h3 id="h3_converting_mp3">Converting Audio Files to an Alexa-Friendly Format</h3> 
       <p>You may need to use converter software to convert your MP3 files to the required codec version (<em>MPEG version 2</em>) and bit rate (<em>48 kbps</em>). One option for this is a command-line tool, <a href="https://www.ffmpeg.org/" title="FFmpeg">FFmpeg</a>. The following command converts the provided <code class="highlighter-rouge">&lt;input-file&gt;</code> to an MP3 file that works with the <code class="highlighter-rouge">audio</code> tag.</p> 
       <div class="highlighter-rouge">
        <pre class="highlight"><code>ffmpeg -i &lt;input-file&gt; -ac 2 -codec:a libmp3lame -b:a 48k -ar 16000 &lt;output-file.mp3&gt;
</code></pre> 
       </div> 
       <p>Another option is <a href="http://www.audacityteam.org/" title="Audacity">Audacity</a>:</p> 
       <ol> 
        <li>Open the file to convert.</li> 
        <li>Set the <strong>Project Rate</strong> in the lower-left corner to <code class="highlighter-rouge">16000</code>.</li> 
        <li>Click <strong>File &gt; Export Audio</strong> and change the <strong>Save as type</strong> to <code class="highlighter-rouge">MP3 Files</code>.</li> 
        <li>Click <strong>Options</strong>, set the <strong>Quality</strong> to <code class="highlighter-rouge">48 kbps</code> and the <strong>Bit Rate Mode</strong> to <code class="highlighter-rouge">Constant</code>.</li> 
       </ol> 
       <p>This requires the Lame library, which can be found at: <a href="http://lame.buanzo.org/#lamewindl">http://lame.buanzo.org/#lamewindl</a>.</p> 
       <h3 id="hosting-the-audio-files-for-your-skill">Hosting the Audio Files for Your Skill</h3> 
       <p>The MP3 files you use to provide audio must be hosted on an endpoint that uses HTTPS. The endpoint must provide an SSL certificate signed by an <a href="https://www.mozilla.org/en-US/about/governance/policies/security-group/certs/included/">Amazon-approved certificate authority</a>. Many content hosting services provide this. For example, you could host your files at a service such as <a href="https://aws.amazon.com/s3/" title="Amazon S3">Amazon Simple Storage Service (Amazon S3)</a> (an <a href="http://aws.amazon.com/" title="Amazon Web Services">Amazon Web Services</a> offering).</p> 
       <p>We don’t require that you authenticate the requests for the audio files. Therefore, you <em>must not</em> include any customer-specific or sensitive information in these audio files. For example, building a custom MP3 file in response to a user’s request, and including sensitive information within the audio, is not allowed.</p> 
       <h2 id="break">break</h2> 
       <p>Represents a pause in the speech. Set the length of the pause with the <code class="highlighter-rouge">strength</code> or <code class="highlighter-rouge">time</code> attributes.</p> 
       <table class="grid"> 
        <colgroup> 
         <col width="25%"> 
         <col width="75%"> 
        </colgroup> 
        <thead> 
         <tr class="header"> 
          <th align="left">Attribute</th> 
          <th align="left">Possible Values</th> 
         </tr> 
        </thead> 
        <tbody> 
         <tr> 
          <td align="left"> <p><code class="highlighter-rouge">strength</code></p> </td> 
          <td align="left"> 
           <ul> 
            <li><em><code class="highlighter-rouge">none</code></em>: No pause should be outputted. This can be used to remove a pause that would normally occur (such as after a period).</li> 
            <li><em><code class="highlighter-rouge">x-weak</code></em>: No pause should be outputted (same as <code class="highlighter-rouge">none</code>).</li> 
            <li><em><code class="highlighter-rouge">weak</code></em>:?Treat adjacent words as if separated by a single comma (equivalent to <em><code class="highlighter-rouge">medium</code></em>).</li> 
            <li><em><code class="highlighter-rouge">medium</code></em>: Treat adjacent words as if separated by a single comma.</li> 
            <li><em><code class="highlighter-rouge">strong</code></em>: Make a sentence break (equivalent to using the <code class="highlighter-rouge">&lt;s&gt;</code> tag).</li> 
            <li><em><code class="highlighter-rouge">x-strong</code></em>: Make a paragraph break (equivalent to using the <code class="highlighter-rouge">&lt;p&gt;</code> tag).</li> 
           </ul> </td> 
         </tr> 
         <tr> 
          <td align="left"> <p><code class="highlighter-rouge">time</code></p> </td> 
          <td align="left"> <p>Duration of the pause; up to 10 seconds (<code class="highlighter-rouge">10s</code>) or 10000 milliseconds (<code class="highlighter-rouge">10000ms</code>). Include the unit with the time (<code class="highlighter-rouge">s</code> or <code class="highlighter-rouge">ms</code>).</p> </td> 
         </tr> 
        </tbody> 
       </table> 
       <p>The default is <em><code class="highlighter-rouge">medium</code></em>. This is used if you don’t specify any attributes, or if you provide any unsupported attribute values.</p> 
       <div class="language-xml highlighter-rouge">
        <pre class="highlight"><code><span class="nt">&lt;speak&gt;</span>
    There is a three second pause here <span class="nt">&lt;break</span> <span class="na">time=</span><span class="s">"3s"</span><span class="nt">/&gt;</span> 
    then the speech continues.
<span class="nt">&lt;/speak&gt;</span> 
</code></pre> 
       </div> 
       <h2 id="emphasis">emphasis</h2> 
       <p>Emphasize the tagged words or phrases. Emphasis changes <em>rate</em> and <em>volume</em> of the speech. More emphasis is spoken <em>louder</em> and <em>slower</em>. Less emphasis is quieter and faster.</p> 
       <table class="grid"> 
        <colgroup> 
         <col width="25%"> 
         <col width="75%"> 
        </colgroup> 
        <thead> 
         <tr class="header"> 
          <th align="left">Attribute</th> 
          <th align="left">Possible Values</th> 
         </tr> 
        </thead> 
        <tbody> 
         <tr> 
          <td align="left"> <p><code class="highlighter-rouge">level</code></p> </td> 
          <td align="left"> 
           <ul> 
            <li><em><code class="highlighter-rouge">strong</code></em>: Increase the volume and slow down the speaking rate so the speech is louder and slower.</li> 
            <li><em><code class="highlighter-rouge">moderate</code></em>: Increase the volume and slow down the speaking rate, but not as much as when set to <code class="highlighter-rouge">strong</code>. This is used as a default if <code class="highlighter-rouge">level</code> is not provided.</li> 
            <li><em><code class="highlighter-rouge">reduced</code></em>: Decrease the volume and speed up the speaking rate. The speech is softer and faster.</li> 
           </ul> </td> 
         </tr> 
        </tbody> 
       </table> 
       <div class="language-xml highlighter-rouge">
        <pre class="highlight"><code><span class="nt">&lt;speak&gt;</span>
    I already told you I 
    <span class="nt">&lt;emphasis</span> <span class="na">level=</span><span class="s">"strong"</span><span class="nt">&gt;</span>really like<span class="nt">&lt;/emphasis&gt;</span> 
    that person.
<span class="nt">&lt;/speak&gt;</span> 
</code></pre> 
       </div> 
       <h2 id="p">p</h2> 
       <p>Represents a paragraph. This tag provides <em>extra-strong</em> breaks before and after the tag. This is equivalent to specifying a pause with <code class="highlighter-rouge">&lt;break strength="x-strong"/&gt;</code>.</p> 
       <div class="language-xml highlighter-rouge">
        <pre class="highlight"><code><span class="nt">&lt;speak&gt;</span>                                         
    <span class="nt">&lt;p&gt;</span>This is the first paragraph. There should be a pause after this text is spoken.<span class="nt">&lt;/p&gt;</span>       
    <span class="nt">&lt;p&gt;</span>This is the second paragraph.<span class="nt">&lt;/p&gt;</span> 
<span class="nt">&lt;/speak&gt;</span>                                        
</code></pre> 
       </div> 
       <h2 id="phoneme">phoneme</h2> 
       <p>Provides a phonemic/phonetic pronunciation for the contained text. For example, people may pronounce words like “pecan” differently.</p> 
       <table class="grid"> 
        <colgroup> 
         <col width="25%"> 
         <col width="75%"> 
        </colgroup> 
        <thead> 
         <tr class="header"> 
          <th align="left">Attribute</th> 
          <th align="left">Possible Values</th> 
         </tr> 
        </thead> 
        <tbody> 
         <tr> 
          <td align="left"> <p><code class="highlighter-rouge">alphabet</code></p> </td> 
          <td align="left"> <p>Set to the phonetic alphabet to use:</p> 
           <ul> 
            <li><em><code class="highlighter-rouge">ipa</code></em>:?The International Phonetic Alphabet (IPA).</li> 
            <li><em><code class="highlighter-rouge">x-sampa</code></em>:?The Extended Speech Assessment Methods Phonetic Alphabet (X-SAMPA).</li> 
           </ul> </td> 
         </tr> 
         <tr> 
          <td align="left"> <p><code class="highlighter-rouge">ph</code></p> </td> 
          <td align="left"> <p>The phonetic pronunciation to speak.</p> <p>See below for a list of <a href="#supported-symbols">supported symbols</a> in each of the supported skill languages (English (US), English (UK), and German). .</p> </td> 
         </tr> 
        </tbody> 
       </table> 
       <p>When using this tag, Alexa uses the pronunciation provided in the <code class="highlighter-rouge">ph</code> attribute rather than the text contained within the tag. However, you should still provide human-readable text within the tags. In the following example, the word “pecan” shown within the tags is never spoken. Instead, Alexa speaks the text provided in the <code class="highlighter-rouge">ph</code> attribute:</p> 
       <div class="language-xml highlighter-rouge">
        <pre class="highlight"><code><span class="nt">&lt;speak&gt;</span>
    You say, <span class="nt">&lt;phoneme</span> <span class="na">alphabet=</span><span class="s">"ipa"</span> <span class="na">ph=</span><span class="s">"p??k??n"</span><span class="nt">&gt;</span>pecan<span class="nt">&lt;/phoneme&gt;</span>. 
    I say, <span class="nt">&lt;phoneme</span> <span class="na">alphabet=</span><span class="s">"ipa"</span> <span class="na">ph=</span><span class="s">"?pi.kæn"</span><span class="nt">&gt;</span>pecan<span class="nt">&lt;/phoneme&gt;</span>.
<span class="nt">&lt;/speak&gt;</span> 
</code></pre>

       <h2 id="prosody">prosody</h2> 
       <p>Modifies the volume, pitch, and rate of the tagged speech.</p> 
       <table class="grid"> 
        <colgroup> 
         <col width="25%"> 
         <col width="75%"> 
        </colgroup> 
        <thead> 
         <tr class="header"> 
          <th align="left">Attribute</th> 
          <th align="left">Possible Values</th> 
         </tr> 
        </thead> 
        <tbody> 
         <tr> 
          <td align="left"> <p><code class="highlighter-rouge">rate</code></p> </td> 
          <td align="left"> <p>Modify the rate of the speech:</p> 
           <ul> 
            <li><em><code class="highlighter-rouge">x-slow</code></em>, <em><code class="highlighter-rouge">slow</code></em>, <em><code class="highlighter-rouge">medium</code></em>, <em><code class="highlighter-rouge">fast</code></em>, <em><code class="highlighter-rouge">x-fast</code></em>: Set the rate to a predefined value.</li> 
            <li><em><code class="highlighter-rouge">n%</code></em>: specify a percentage to increase or decrease the speed of the speech: 
             <ul> 
              <li><code class="highlighter-rouge">100%</code> indicates no change from the normal rate.</li> 
              <li>Percentages greater than <code class="highlighter-rouge">100%</code> increase the rate.</li> 
              <li>Percentages below <code class="highlighter-rouge">100%</code> decrease the rate.</li> 
              <li>The minimum value you can provide is <code class="highlighter-rouge">20%</code>.</li> 
             </ul> </li> 
           </ul> </td> 
         </tr> 
         <tr> 
          <td align="left"> <p><code class="highlighter-rouge">pitch</code></p> </td> 
          <td align="left"> <p>Raise or lower the tone (pitch) of the speech:</p> 
           <ul> 
            <li><em><code class="highlighter-rouge">x-low</code></em>, <em><code class="highlighter-rouge">low</code></em>, <em><code class="highlighter-rouge">medium</code></em>, <em><code class="highlighter-rouge">high</code></em>, <em><code class="highlighter-rouge">x-high</code></em>: Set the pitch to a predefined value.</li> 
            <li><em><code class="highlighter-rouge">+n%</code></em>: Increase the pitch by the specified percentage. For example: <em><code class="highlighter-rouge">+10%</code></em>, <em><code class="highlighter-rouge">+5%</code></em>. The maximum value allowed is <code class="highlighter-rouge">+50%</code>. A value higher than <code class="highlighter-rouge">+50%</code> is rendered as <code class="highlighter-rouge">+50%</code>.</li> 
            <li><em><code class="highlighter-rouge">-n%</code></em>: Decrease the pitch by the specified percentage. For example: <em><code class="highlighter-rouge">-10%</code></em>, <em><code class="highlighter-rouge">-20%</code></em>. The smallest value allowed is <code class="highlighter-rouge">-33.3%</code>. A value lower than <code class="highlighter-rouge">-33.3%</code> is rendered as <code class="highlighter-rouge">-33.3%</code>.</li> 
           </ul> </td> 
         </tr> 
         <tr> 
          <td align="left"> <p><code class="highlighter-rouge">volume</code></p> </td> 
          <td align="left"> <p>Change the volume for the speech:</p> 
           <ul> 
            <li><em><code class="highlighter-rouge">silent</code></em>, <em><code class="highlighter-rouge">x-soft</code></em>, <em><code class="highlighter-rouge">soft</code></em>, <em><code class="highlighter-rouge">medium</code></em>, <em><code class="highlighter-rouge">loud</code></em>, <em><code class="highlighter-rouge">x-loud</code></em>:?Set volume to a predefined value for current voice.</li> 
            <li><em><code class="highlighter-rouge">+ndB</code></em>: Increase volume relative to the current volume level. For example, <code class="highlighter-rouge">+0dB</code> means no change of volume. <code class="highlighter-rouge">+6dB</code> is approximately twice the current amplitude. The maximum positive value is about <code class="highlighter-rouge">+4.08dB</code>.</li> 
            <li><em><code class="highlighter-rouge">-ndB</code></em>: Decrease the volume relative to the current volume level. For example, <code class="highlighter-rouge">-6dB</code> means approximately half the current amplitude.</li> 
           </ul> </td> 
         </tr> 
        </tbody> 
       </table> 
       <div class="language-xml highlighter-rouge">
        <pre class="highlight"><code><span class="nt">&lt;speak&gt;</span>
    Normal volume for the first sentence.
    <span class="nt">&lt;prosody</span> <span class="na">volume=</span><span class="s">"x-loud"</span><span class="nt">&gt;</span>Louder volume for the second sentence<span class="nt">&lt;/prosody&gt;</span>.
    When I wake up, <span class="nt">&lt;prosody</span> <span class="na">rate=</span><span class="s">"x-slow"</span><span class="nt">&gt;</span>I speak quite slowly<span class="nt">&lt;/prosody&gt;</span>.
    I can speak with my normal pitch, 
    <span class="nt">&lt;prosody</span> <span class="na">pitch=</span><span class="s">"x-high"</span><span class="nt">&gt;</span> but also with a much higher pitch <span class="nt">&lt;/prosody&gt;</span>, 
    and also <span class="nt">&lt;prosody</span> <span class="na">pitch=</span><span class="s">"low"</span><span class="nt">&gt;</span>with a lower pitch<span class="nt">&lt;/prosody&gt;</span>.
<span class="nt">&lt;/speak&gt;</span>

</code></pre> 
       </div> 
       <h2 id="s">s</h2> 
       <p>Represents a sentence. This tag provides <em>strong</em> breaks before and after the tag.</p> 
       <p>This is equivalent to:</p> 
       <ul> 
        <li>Ending a sentence with a period (.).</li> 
        <li>Specifying a pause with <code class="highlighter-rouge">&lt;break strength="strong"/&gt;</code>.</li> 
       </ul> 
       <div class="language-xml highlighter-rouge">
        <pre class="highlight"><code><span class="nt">&lt;speak&gt;</span>
    <span class="nt">&lt;s&gt;</span>This is a sentence<span class="nt">&lt;/s&gt;</span>
    <span class="nt">&lt;s&gt;</span>There should be a short pause before this second sentence<span class="nt">&lt;/s&gt;</span> 
    This sentence ends with a period and should have the same pause.
<span class="nt">&lt;/speak&gt;</span>

</code></pre> 
       </div> 
       <h2 id="say-as">say-as</h2> 
       <p>Describes how the text should be interpreted. This lets you provide additional context to the text and eliminate any ambiguity on how Alexa should render the text. Indicate how Alexa should interpret the text with the <code class="highlighter-rouge">interpret-as</code> attribute.</p> 
       <table class="grid"> 
        <colgroup> 
         <col width="25%"> 
         <col width="75%"> 
        </colgroup> 
        <thead> 
         <tr class="header"> 
          <th align="left">Attribute</th> 
          <th align="left">Possible Values</th> 
         </tr> 
        </thead> 
        <tbody> 
         <tr> 
          <td align="left"> <p><code class="highlighter-rouge">interpret-as</code></p> </td> 
          <td align="left"> 
           <ul> 
            <li><em><code class="highlighter-rouge">characters</code>, <code class="highlighter-rouge">spell-out</code></em>:?Spell out each letter.</li> 
            <li><em><code class="highlighter-rouge">cardinal</code>, <code class="highlighter-rouge">number</code></em>: Interpret the value as a cardinal number.</li> 
            <li><em><code class="highlighter-rouge">ordinal</code></em>:?Interpret the value as an ordinal number.</li> 
            <li><em><code class="highlighter-rouge">digits</code></em>: Spell each digit separately .</li> 
            <li><em><code class="highlighter-rouge">fraction</code></em>:?Interpret the value as a fraction. This works for both common fractions (such as 3/20) and mixed fractions (such as 1+1/2).</li> 
            <li><em><code class="highlighter-rouge">unit</code></em>:?Interpret a value as a measurement. The value should be either a number or fraction followed by a unit (with no space in between) or just a unit.</li> 
            <li><em><code class="highlighter-rouge">date</code></em>:?Interpret the value as a date. Specify the format with the <code class="highlighter-rouge">format</code> attribute.</li> 
            <li><em><code class="highlighter-rouge">time</code></em>:?Interpret a value such as <code class="highlighter-rouge">1'21"</code> as duration in minutes and seconds.</li> 
            <li><em><code class="highlighter-rouge">telephone</code></em>:?Interpret a value as a 7-digit or 10-digit telephone number. This can also handle extensions (for example, 2025551212x345).</li> 
            <li><em><code class="highlighter-rouge">address</code></em>:?Interpret a value as part of street address.</li> 
            <li><em><code class="highlighter-rouge">interjection</code></em>: Interpret the value as an interjection. Alexa speaks the text in a more expressive voice. For optimal results, only use the <em>supported interjections</em> and surround each one with a pause. For example: <code class="highlighter-rouge">&lt;say-as interpret-as="interjection"&gt;Wow.&lt;/say-as&gt;</code>. Speechcons are supported for <a href="https://developer.amazon.com/public/solutions/alexa/alexa-skills-kit/docs/speechcon-reference">English (US)</a>, <a href="https://developer.amazon.com/public/solutions/alexa/alexa-skills-kit/docs/speechcon-reference-uk">English (UK)</a>, and <a href="https://developer.amazon.com/public/solutions/alexa/alexa-skills-kit/docs/speechcon-reference-de">German</a>.</li> 
            <li><em><code class="highlighter-rouge">expletive</code></em>: “Bleep” out the content inside the tag.</li> 
           </ul> </td> 
         </tr> 
         <tr> 
          <td align="left"> <p><code class="highlighter-rouge">format</code></p> </td> 
          <td align="left"> <p>Only used when <code class="highlighter-rouge">interpret-as</code> is set to <code class="highlighter-rouge">date</code>. Set to one of the following to indicate format of the date:</p> 
           <ul> 
            <li><code class="highlighter-rouge">mdy</code></li> 
            <li><code class="highlighter-rouge">dmy</code></li> 
            <li><code class="highlighter-rouge">ymd</code></li> 
            <li><code class="highlighter-rouge">md</code></li> 
            <li><code class="highlighter-rouge">dm</code></li> 
            <li><code class="highlighter-rouge">ym</code></li> 
            <li><code class="highlighter-rouge">my</code></li> 
            <li><code class="highlighter-rouge">d</code></li> 
            <li><code class="highlighter-rouge">m</code></li> 
            <li><code class="highlighter-rouge">y</code></li> 
           </ul> <p>Alternatively, if you provide the date in YYYYMMDD format, the <code class="highlighter-rouge">format</code> attribute is ignored. You can include question marks (?) for portions of the date to leave out. For instance, Alexa would speak <code class="highlighter-rouge">&lt;say-as interpret-as="date"&gt;????0922&lt;/say-as&gt;</code> as “September 22nd”.</p> </td> 
         </tr> 
        </tbody> 
       </table> 
       <p>Note that the Alexa service attempts to interpret the provided text correctly based on the text’s formatting even without this tag. For example, if your output speech includes “202-555-1212”, Alexa speaks each individual digit, with a brief pause for each dash. You don’t need to use <code class="highlighter-rouge">&lt;say-as interpret-as="telephone"&gt;</code> in this case. However, if you provided the text “2025551212”, but you wanted Alexa to speak it as a phone number, you would need to use <code class="highlighter-rouge">&lt;say-as interpret-as="telephone"&gt;</code>.</p> 
       <div class="language-xml highlighter-rouge">
        <pre class="highlight"><code><span class="nt">&lt;speak&gt;</span>
    Here is a number spoken as a cardinal number: 
    <span class="nt">&lt;say-as</span> <span class="na">interpret-as=</span><span class="s">"cardinal"</span><span class="nt">&gt;</span>12345<span class="nt">&lt;/say-as&gt;</span>.
    Here is the same number with each digit spoken separately:
    <span class="nt">&lt;say-as</span> <span class="na">interpret-as=</span><span class="s">"digits"</span><span class="nt">&gt;</span>12345<span class="nt">&lt;/say-as&gt;</span>.
    Here is a word spelled out: <span class="nt">&lt;say-as</span> <span class="na">interpret-as=</span><span class="s">"spell-out"</span><span class="nt">&gt;</span>hello<span class="nt">&lt;/say-as&gt;</span>
<span class="nt">&lt;/speak&gt;</span>
</code></pre> 
       </div> 
       <h2 id="speak">speak</h2> 
       <p>This is the root element of an SSML document. When using SSML with the Alexa Skills Kit, surround the text to be spoken with this tag.</p> 
       <div class="language-xml highlighter-rouge">
        <pre class="highlight"><code><span class="nt">&lt;speak&gt;</span>
    This is what Alexa sounds like without any SSML.
<span class="nt">&lt;/speak&gt;</span>
</code></pre> 
       </div> 
       <h2 id="sub">sub</h2> 
       <p>Pronounce the specified word or phrase as a different word or phrase. Specify the pronunciation to substitute with the <code class="highlighter-rouge">alias</code> attribute.</p> 
       <table class="grid"> 
        <colgroup> 
         <col width="25%"> 
         <col width="75%"> 
        </colgroup> 
        <thead> 
         <tr class="header"> 
          <th align="left">Attribute</th> 
          <th align="left">Possible Values</th> 
         </tr> 
        </thead> 
        <tbody> 
         <tr> 
          <td align="left"> <p><code class="highlighter-rouge">alias</code></p> </td> 
          <td align="left"> <p>The word or phrase to speak in place of the tagged text.</p> </td> 
         </tr> 
        </tbody> 
       </table> 
       <p>This example replaces the abbreviated chemical elements with the full words:</p> 
       <div class="language-xml highlighter-rouge">
        <pre class="highlight"><code><span class="nt">&lt;speak&gt;</span>
    My favorite chemical element is <span class="nt">&lt;sub</span> <span class="na">alias=</span><span class="s">"aluminum"</span><span class="nt">&gt;</span>Al<span class="nt">&lt;/sub&gt;</span>,
    but Al prefers <span class="nt">&lt;sub</span> <span class="na">alias=</span><span class="s">"magnesium"</span><span class="nt">&gt;</span>Mg<span class="nt">&lt;/sub&gt;</span>. 
<span class="nt">&lt;/speak&gt;</span> 
</code></pre> 
       </div> 
       <h2 id="w">w</h2> 
       <p>Similar to <code class="highlighter-rouge">&lt;say-as&gt;</code>, this tag customizes the pronunciation of words by specifying the word’s part of speech.</p> 
       <table class="grid"> 
        <colgroup> 
         <col width="25%"> 
         <col width="75%"> 
        </colgroup> 
        <thead> 
         <tr class="header"> 
          <th align="left">Attribute</th> 
          <th align="left">Possible Values</th> 
         </tr> 
        </thead> 
        <tbody> 
         <tr> 
          <td align="left"> <p><code class="highlighter-rouge">role</code></p> </td> 
          <td align="left"> <p>Set to one of the following</p> 
           <ul> 
            <li><em><code class="highlighter-rouge">amazon:VB</code></em>: Interpret the word as a verb (present simple).</li> 
            <li><em><code class="highlighter-rouge">amazon:VBD</code></em>: Interpret the word as a past participle.</li> 
            <li><em><code class="highlighter-rouge">amazon:NN</code></em>: Interpret the word as a noun.</li> 
            <li><em><code class="highlighter-rouge">amazon:SENSE_1</code></em>:?Use the non-default sense of the word. For example, the noun “bass” is pronounced differently depending on meaning. The “default” meaning is the lowest part of the musical range. The alternate sense (which is still a noun) is a freshwater fish. Specifying <code class="highlighter-rouge">&lt;speak&gt;&lt;w role="amazon:SENSE_1"&gt;bass&lt;/w&gt;"&lt;/speak&gt;</code> renders the non-default pronunciation (freshwater fish).</li> 
           </ul> </td> 
         </tr> 
        </tbody> 
       </table> 
       <div class="language-xml highlighter-rouge">
        <pre class="highlight"><code><span class="nt">&lt;speak&gt;</span>
    The word <span class="nt">&lt;say-as</span> <span class="na">interpret-as=</span><span class="s">"characters"</span><span class="nt">&gt;</span>read<span class="nt">&lt;/say-as&gt;</span> may be interpreted 
    as either the present simple form <span class="nt">&lt;w</span> <span class="na">role=</span><span class="s">"amazon:VB"</span><span class="nt">&gt;</span>read<span class="nt">&lt;/w&gt;</span>, 
    or the past participle form <span class="nt">&lt;w</span> <span class="na">role=</span><span class="s">"amazon:VBD"</span><span class="nt">&gt;</span>read<span class="nt">&lt;/w&gt;</span>.
<span class="nt">&lt;/speak&gt;</span> 
</code></pre> 
       </div> 
       <p>Note that these tags previously used the <code class="highlighter-rouge">ivona</code> namespace in the attribute names. The tags are backwards compatible, so existing SSML written with the <code class="highlighter-rouge">ivona</code> namespace continues to work.</p>
