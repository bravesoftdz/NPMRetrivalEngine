     <section class="docs-section"> 
      <h1 class="section-title">Pick a file using the Filestack v2 library</h1> 
      <p>Using our simple yet powerful Javascript API, you can programatically open the Uploader Dialog by calling the pick() function. The dialog will not launch without an API key. To set your key use <code>filepicker.setKey("your_API_KEY");</code>. Filestack's live code examples include an API key in the background, but if you want to test something using your own key, you can always add filepicker.setKey("YOUR_API_KEY") to the live code examples.</p> 
     </section> 
     <section class="docs-section"> 
      <h2 id="pickSyntax" class="section-title">Syntax</h2> 
      <p>The javascript pick() function can be called anytime after the apikey has been set in order to display the Picker dialog. You may pass in an optional dictionary of key-value pairs to customize the options of the Upload dialog. Handler functions are also optional, and can be passed in as callback functions for the events onProgress, onSuccess, and onError.</p> 
      <div class="code__pane"> 
       <div class="pane__body"> 
        <pre class="prettyprint">&lt;script src="https://api.filestackapi.com/filestack.js"&gt;&lt;/script&gt;</pre> 
        <pre class="prettyprint">
&lt;script type="text/javascript"&gt;
&nbsp;&nbsp;filepicker.setKey("your_API_KEY");
&nbsp;&nbsp;filepicker.pick(picker_options, onSuccess(Blob){}, onError(FPError){}, onProgress(FPProgress){});
&lt;/script&gt;</pre> 
       </div> 
      </div> 
     </section> 
     <section class="docs-section">

      <div class="row">

      <h3>Client side image compression and resize</h3> 
      <div class="hot-tips margin-top--2x"> 
       <span class="small-text">Note:</span> 
       <p>Client side image compression and resizing concerns local &amp; webcam images and is available for mobile and desktop devices. These functions also rely on the dialog being open to work. If a user utilizes the drop-pane functionality to upload their images, meaning no modal dialog window opens, their images will not be modified.</p>

      <h3>Crop UI</h3> 
      <div class="hot-tips margin-top--2x"> 
       <span class="small-text">Note:</span> 
       <p>To include the crop UI in a widget add the 'CONVERT' service to the services list. 'CONVERT' is included by default.</p> 
       <a href="/docs/javascript-api/crop-ui" class="btn-tip">See more</a>

      <div class="panel panel-info margin-top--2x"> 
       <div class="panel-heading"> 
        <span id="conversions">Conversions</span> 
        <div class="parameter-type">
         Array
        </div> 
       </div> 
       <div class="panel-body"> 
        <div class="col-md-4"> 
         <code>conversions: ['crop', 'rotate', 'filter']</code> 
        </div> 
        <div class="col-md-8"> 
         <p>When used in conjunction with the 'CONVERT' service, the conversions array allows you to specify what functions are available in the UI for the user. Include 'crop' to allow the user to crop an image, include 'rotate' to allow a user to rotate the uploaded image left or right, include filter to allow user to blur or sharpen the uploaded image. You can include all the services or a selection of the three. If 'CONVERT' is specified, but no conversions are set, the default behavior is to display the cropping tool only.</p> 
        </div> 
       </div> 
      </div>
