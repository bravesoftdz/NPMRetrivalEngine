          <h1>Converting books to MP3 audio (text-to-speech)</h1> 
          <div class="row meta"> 
           <div class="column grid_8"> 
            <div class="author">
             Posted on February 21, 2016
            </div> 
            <div class="tags"> 
             <ul> 
              <li><a href="https://ericheikes.com/tag/linux/" rel="tag">linux</a></li> 
              <li><a href="https://ericheikes.com/tag/mp3/" rel="tag">mp3</a></li> 
              <li><a href="https://ericheikes.com/tag/ocr/" rel="tag">ocr</a></li> 
              <li><a href="https://ericheikes.com/tag/os-x/" rel="tag">os x</a></li> 
              <li><a href="https://ericheikes.com/tag/pdf/" rel="tag">pdf</a></li> 
              <li><a href="https://ericheikes.com/tag/text-to-speech/" rel="tag">text to speech</a></li> 
              <li><a href="https://ericheikes.com/tag/windows/" rel="tag">Windows</a></li> 
             </ul>

          <div class="row body"> 
           <div class="column grid_8"> 
            <p>My wife recently (re)started school at Iowa State University, which is an hour-long drive from where we live. To make the commute more productive, I looked into converting her paper and digital textbooks into audiobooks.</p> 
            <p>Most of the tools I used are run from the command line (<a href="http://www.computerhope.com/issues/chusedos.htm">Windows</a>, <a href="http://blog.teamtreehouse.com/introduction-to-the-mac-os-x-command-line">OSX</a>, or <a href="http://linuxcommand.org/">Linux</a>), but the process is fairly straightforward and should be mostly copy-and-paste.</p> 
            <h2>Scanning pages</h2> 
            <p><img src="https://ericheikes.com/wp-content/uploads/2016/02/chapter7-04-240x400.jpg" alt="Example of scanned page" width="240" height="400" class="alignright size-medium wp-image-5241" srcset="https://ericheikes.com/wp-content/uploads/2016/02/chapter7-04-240x400.jpg 240w, https://ericheikes.com/wp-content/uploads/2016/02/chapter7-04-174x290.jpg 174w, https://ericheikes.com/wp-content/uploads/2016/02/chapter7-04-614x1024.jpg 614w, https://ericheikes.com/wp-content/uploads/2016/02/chapter7-04.jpg 654w" sizes="(max-width: 240px) 100vw, 240px">The first part is the most annoying, since most of us do not have <a href="http://www.treventus.com/automatic-book-scanner-scanrobot.html">an easy way to scan bound books</a>. A flatbed scanner provides the best quality, but you can try a phone camera, as long as it is high resolution, fairly flat, and in focus. (In my experiments, my Samsung Galaxy S5 phone took a perfectly fine photo at its highest camera setting — 5312×2988 pixels, around 4.5MB — but the light and focus wasn’t as good as a scanner.) Save the scanned pages as image files; the software that comes with your scanner should make this easy.</p> 
            <h2>Converting images to text</h2> 
            <p><a href="https://github.com/tesseract-ocr/tesseract">Tesseract</a> is great <a href="https://en.wikipedia.org/wiki/Optical_character_recognition">OCR</a> software that will convert your images into text. For Windows, you can download the installer from their homepage. OSX and Linux usually have packages available in Homebrew, apt, etc.</p> 
            <p>To convert an image file, run the following in the command line:</p> 
            <pre><code class="shell">tesseract imagefile.jpg output -psm 1
</code></pre> 
            <p>…where <code>imagefile.jpg</code> is your image file, and <code>output</code> is your desired name for the text file (<em>without</em> the file extension — it will be added automatically).</p> 
            <p>Tesseract understands many image formats, so you can convert JPG, PNG, TIF, and more. Its character recognition is very good; the only time I run into problems is with text that is very close to the book binding (and hard to scan), laid over background images, or italicized.</p> 
            <p>Tesseract has multiple “page segmentation modes” (PSMs) — how it detects and segments the page. In the command above, I’m specifying a PSM of <code>1</code>, which automatically detects the page orientation and segments the page into its constituent blocks of text. This is great if your page was scanned upside-down, or if it contains columns or other visually-separate blocks of text.</p> 
            <p>Tesseract’s page segmentation is pretty great, but be warned that even if it detects the sidebars, captions, and other areas of text, it will probably insert them in the middle of the main text; you may want to leave out that supplemental content by covering them up or deleting them from the image.</p> 
            <h2>Converting PDFs to text</h2> 
            <p><img src="https://ericheikes.com/wp-content/uploads/2016/02/xpdf.png" alt="Xpdf logo" width="100" class="alignright size-full wp-image-5243">If you have e-books and other digital materials, you’ll want to convert those files into plain text (i.e., without all the layout and text formatting).</p> 
            <p>You can always “select all” and copy+paste into a document, but why go to all that work? A free tool called “pdftotext” can do that for you:</p> 
            <pre><code class="shell">pdftotext -enc UTF-8 -nopgbrk -layout chapter2.pdf chapter2.txt
</code></pre> 
            <p>The pdftotext tool is part of the <a href="http://www.foolabs.com/xpdf/download.html">Xpdf software suite</a>. If you are using OSX or Linux, it is also usually available in a utility package (search for “pdftotext”, “poppler”, or “xpdf”).</p> 
            <h2>Side note: Textract</h2> 
            <p>If you want an all-in-one solution for extracting text from files, take a look at <a href="https://github.com/dbashford/textract">textract</a>. You’ll need <a href="https://nodejs.org/en/">NodeJS</a> to run it, but once you have that, you can <a href="https://github.com/dbashford/textract#commmand-line">install textract globally so you can run it from the command line</a>. It uses the same tools like Tesseract and pdftotext under the hood, but also supports extraction for HTML, Word documents, CSV, PowerPoint, and more, while wrapping it all into a single tool. Unfortunately, there is little ability to customize the underlying tools if you want to tweak them.</p> 
            <h2>Fixing the text</h2> 
            <p>Unfortunately, Tesseract and pdftotext will generate some special characters that text-to-speech programs won’t understand, as well as including other cruft. We need to fix up the text so it creates the best audio.</p> 
            <p>First, use <a href="http://billposer.org/Software/uni2ascii.html">uni2ascii</a> to convert fancy characters to their simple equivalents (7-bit ASCII, if you’re curious):</p> 
            <pre><code class="shell">uni2ascii -B chapter2.txt &gt; chapter2-fixed.txt
</code></pre> 
            <p>Finally, you’ll probably want to scan the text (with your eyes) just to catch any mistakes made by the OCR software. Some fonts, like heavily italicized text, are more difficult for it to recognize. Other things, like headings, benefit from adding a period or other punctuation to separate them from paragraphs. Most of these can’t be automated, but here are some common fixes that can be run in on the command line or in a script (these require <a href="https://en.wikipedia.org/wiki/Sed">sed</a> and <a href="https://en.wikipedia.org/wiki/Perl">Perl</a>):</p> 
            <pre><code class="shell"># OCR often interprets "w" as "vv",
# and "vv" is an uncommon combination in English.
sed -ri s/vv/w/ig chapter2.txt

# OCR sometimes interprets "&amp;" as other characters.
sed -ri 's/:5:/\&amp;/ig' chapter2.txt
sed -ri 's/8c/\&amp;/ig' chapter2.txt

# Some observed PDFs use "greek question mark" instead of semicolon.
sed -ri s/0x037E/;/ig chapter2.txt

# Combine words that are broken (with hyphens) across lines.
perl -0777 -i -pe 's/(\S+)-\n\s*(\S+)/$1$2/ig' chapter2.txt
</code></pre> 
            <h2>Converting text to speech</h2> 
            <p><img src="https://ericheikes.com/wp-content/uploads/2016/02/audio-wave-inverted-290x115.jpg" alt="Audio wave" width="290" height="115" class="alignright size-thumbnail wp-image-5248" srcset="https://ericheikes.com/wp-content/uploads/2016/02/audio-wave-inverted-290x115.jpg 290w, https://ericheikes.com/wp-content/uploads/2016/02/audio-wave-inverted.jpg 414w" sizes="(max-width: 290px) 100vw, 290px"><a href="http://www.cstr.ed.ac.uk/projects/festival/">Festival</a> is the <em>de facto</em> free software for converting text into speech. You can download it and some voices from <a href="http://festvox.org/festival/">festvox.org</a>. It’s not the easiest software to install. If you’re using Linux, using a package manager is the easiest method. Installing voices is even more difficult; I used <a href="http://ubuntuforums.org/showthread.php?t=751169">this old forum post</a> to install some extra ones on my Linux machine.</p> 
            <p>Once you have Festival installed, you can convert the text file to speech using this command (using the “voice_us2_mbrola” voice):</p> 
            <pre><code class="shell">text2wave chapter2.txt -o chapter2.wav -eval "(voice_us2_mbrola)"
</code></pre> 
            <p>Unfortunately, even the best free voices will still sound like <a href="https://www.youtube.com/watch?v=OTmPw4iy0hk#t=0m22s">Stephen Hawking</a>. If you want something that sounds even close to <a href="https://en.wikipedia.org/wiki/Siri">Siri</a>, you’ll have to pay for it. I use <a href="https://www.cepstral.com/en/personal">Cepstral</a>, which for $35-$45 will get you a nice-sounding voice of your choice for use with their Windows/OSX/Linux software.</p> 
            <h2>Encoding to MP3</h2> 
            <p>Festival and Cepstral encodes the speech as raw WAV files, which are quite large. To make them more portable, you’ll want to convert them to MP3. Luckily there’s a free tool to do that called <a href="http://lame.sourceforge.net/">LAME</a>. There are pre-built packages available for OSX and Linux; Windows users or those wanting a graphical interface (GUI) can check out their <a href="http://lame.sourceforge.net/links.php">list of software links</a>. (I use WinLAME myself.)</p> 
            <p>Because speech has less fidelity than music, you can use fairly low-quality settings. Mono (instead of stereo) should be fine too. LAME’s <code>-V5</code> option uses a variable bit rate quality of “medium”, or around 130kbps, which should be fine for speech:</p> 
            <pre><code class="shell">lame -V5 chapter2.wav chapter2.mp3
</code></pre> 
            <h2>Putting it all together</h2> 
            <p>Depending on exactly which tools you use, your usage may vary from mine, but here’s the full process that you can customize.</p> 
            <h3>Installation</h3> 
            <ul> 
             <li>Ubuntu/Debian: <code>apt-get install tesseract-ocr poppler-utils perl sed festival festvox-us2 lame</code></li> 
             <li>OSX: <code>brew install tesseract poppler perl speech-tools lame</code>, then install Festival, the festlex packages, and any voices from the <a href="http://www.cstr.ed.ac.uk/projects/festival/download.html">download page</a>.</li> 
             <li>Windows: Most of the tools mentioned have Windows installers.</li> 
            </ul> 
            <h3>Scripts</h3> 
            <p>If you’re using OSX or Linux, here are some shell scripts for the process: <a href="https://gist.github.com/eheikes/b738a8dc89833129b2b3">textify.sh</a> and <a href="https://gist.github.com/eheikes/a51697bc63c4cabf0a3c">tts.sh</a>. Save those to your computer and run <code>chmod a+x textify.sh tts.sh</code>.</p> 
            <p>Usage:</p> 
            <pre><code class="shell"># (1) Scan your images. Remove sidebars and supplemental text.
# (2) Run the textify script:
./textify.sh *.tif  # assumes TIF images
# (3) Look over the text files. Make any necessary fixes.
# (4) Run the tts script:
./tts.sh *.txt
</code></pre> 
            <p>After the <code>tts.sh</code> script runs, you’ll have your final <code>out.mp3</code> audio file with your text as speech.</p>
