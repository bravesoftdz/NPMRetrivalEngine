         <p>This week has certainly been an exciting one, especially in the cloud IT market! <a href="https://aws.amazon.com" target="_blank">Amazon Web Services (AWS)</a> just recently announced a really cool, new service called <a href="https://aws.amazon.com/polly/" target="_blank">Amazon Polly</a>! AWS Polly is a <a href="https://en.wikipedia.org/wiki/Speech_synthesis" target="_blank">speech synthesis</a> (text-to-speech) service, aimed at software developers and architects, that enables you to generate speech from text, and play it directly inside your application. Does that sound really awesome? If so, then youre in the right place!</p> 
         <p>Amazon Polly can be used in many different ways. For example, if youre writing an automated call management system, or perhaps youre writing a monitoring application that periodically announces status information, or maybe youre writing an announcement system for an airport. Who knows, maybe youre even a hobby developer whos excited about home automation, and wants an easy way to get information, without looking at a screen! All of these scenarios can benefit from integrating with speech synthesis services, like Amazon Polly, to dynamically generate rich, life-like content.</p> 
         <p>Speaking of life-like, one of the really cool features about Amazon Polly is that you can select from a variety of different voices, including languages such as: English, German, Polish, Italian, French, and others. This helps give some variety and flexibility to the applications that you integrate with Polly. In fact, during my testing, I noticed that if I provided English text to Polly, the non-English voices would still pronounce the English words, just with a heavy accent. This helps to give an element of realism to the results that Amazon Polly gives.</p> 
         <p><a href="https://twitter.com/jeffbarr" target="_blank">Jeff Barr</a>, a Developer Evangelist at AWS, has already <a href="https://aws.amazon.com/blogs/aws/polly-text-to-speech-in-47-voices-and-24-languages/">published an article</a> that describes how to use the <a href="https://aws.amazon.com/cli/">AWS CLI tool</a> to generate an audio file with Polly. Using the AWS CLI is a great way for system administrators and junior developers to integrate Polly with build scripts and systems automation scripts, but its not the right interface to build a scalable, enterprise application. In this article, were going to take a look at how to integrate AWS Polly with a <a href="https://nodejs.org">Node.js</a> application, using the using the AWS SDK. You can access the <a href="http://docs.aws.amazon.com/AWSJavaScriptSDK/latest/AWS/Polly.html">Node.js SDK documentation</a> and the <a href="http://docs.aws.amazon.com/polly/latest/dg/what-is.html">Developer Guide / API Reference documentation</a> for Amazon Polly.</p> 
         <p><strong>NOTE</strong>: As of this writing, the latest version of the <a href="https://www.powershellgallery.com/packages/AWSPowerShell/3.3.24.0">AWS PowerShell module</a> does not seem to support Amazon Polly. To verify this, I updated the <strong>AWSPowerShell</strong> module to the latest version and ran <strong>Get-Command -Module AWSPowerShell -Name <em>polly</em></strong>. Hopefully support for this is added in the future.</p>

         <h2>Create AWS IAM User Account + Access Credentials</h2> 
         <p>One of the first things we need to do, in any cloud project, is make sure that were authenticated to the cloud platform. In Amazon Web Services, the authentication and authorization service is called <a href="https://aws.amazon.com/iam/">Identity and Access Management (IAM)</a>. Well create a new user account in IAM, and IAM will automatically generate an <strong>Access Key ID</strong> and <strong>Secret Access Key</strong>. These are the username/password-like values that we use from various client tools, to access AWS features, including Polly.</p> 
         <ol> 
          <li>Login to the <a href="https://console.aws.amazon.com">AWS Console</a></li> 
          <li>Navigate to the IAM feature</li> 
          <li>Create a new user account for <strong>programmatic access</strong></li> 
          <li>Attach the <strong>AmazonPollyFullAccess</strong> built-in policy</li> 
          <li>Finish creating the user account</li> 
         </ol> 
         <p><span style="color: #ff6600;"><strong>IMPORTANT:</strong></span> Make sure that you copy and paste the <strong>Access Key ID</strong> and <strong>Secret Access Key</strong> values after creating the new user account. You will need these in order to authenticate from your Node.js application. If you lose these, youll have to regenerate them again. I recommend using a password manager, such as <a href="https://lastpass.com">LastPass</a>, to keep track of your various AWS IAM credentials.</p>

         <h2>Authenticate to AWS from Node</h2> 
         <p>Now that weve generated our access credentials, we need to add them to our project. To do this, well <a href="http://docs.aws.amazon.com/sdk-for-javascript/v2/developer-guide/loading-node-credentials-json-file.html">create a simple JSON file</a> that the AWS SDK knows how to read. There are <a href="http://docs.aws.amazon.com/sdk-for-javascript/v2/developer-guide/setting-credentials-node.html">other options for configuring AWS authentication</a>, but I prefer this method, since it keeps everything in the same logical area on the filesystem.</p> 
         <p><span style="color: #ff6600;"><strong>WARNING:</strong></span> These AWS credentials will be included in a separate JSON file that <span style="text-decoration: underline;"><strong>must not</strong></span> be checked into your projects source control. You should probably add the file to <strong>.gitignore</strong>, if youre using Git, to make sure that you dont accidentally commit the file to a Git repository. If someone obtains your AWS credentials, they may be able to cause a significant amount of harm by running up your AWS account bill, as <a href="https://awsinsider.net/articles/2015/09/08/aws-bug-bill.aspx">has happened in the past</a>.</p> 
         <p>If you practice the <a href="https://en.wikipedia.org/wiki/Principle_of_least_privilege">least privilege model</a>, as we did in the previous step, by granting limited permissions to user accounts, you can help limit the impact that leaked credentials might otherwise have. Our IAM user account only has access to Amazon Polly, so compromised credentials for this account would not impact any of our other AWS cloud resources. I cant stress how important this is, especially for production environments.</p> 
         <ol> 
          <li>In Visual Studio Code, add a new file to the root of your project, called <strong>awscreds.json</strong>. The actual name you give the file is irrelevant, but I will be referring to it by this name, in my example.</li> 
          <li>Inside the file, define your AWS credentials, that were generated in the previous section of this article. Use the example template below, and fill in your own details.</li> 
          <li>For the <strong>region</strong> JSON property, define one of the supported <a href="https://aws.amazon.com/polly/faqs/">AWS Polly regions</a></li> 
         </ol> 
         <pre>{
 "accessKeyId": "&lt;YourAccessKeyID&gt;",
 "secretAccessKey": "&lt;YourSecretAccessKey&gt;",
 "region": "us-east-2"
}</pre> 
         <p>After creating the JSON file with your AWS IAM credentials, go ahead and create a new Node.js module to handle authentication. Well reference this module inside of our main Node.js application file later on.</p> 
         <ol> 
          <li>In the root of your project, create a new file called <strong>awsauth.js</strong></li> 
          <li>Place the contents below into the file. If you customized the name of your JSON credentials file, make sure you change it in this code snippet.</li> 
         </ol> 
         <pre>var AWS = require('aws-sdk');

// Load AWS credentials
AWS.config.loadFromPath('./awscreds.json');</pre>

           <noscript>
            <img class="alignright size-full wp-image-4209" src="https://trevorsullivan.net/wp-content/uploads/2016/12/PowerShell-AWS-Polly-Voices.png" alt="List AWS Polly Voices with Node.js" width="466" height="404" srcset="https://trevorsullivan.net/wp-content/uploads/2016/12/PowerShell-AWS-Polly-Voices.png 466w, https://trevorsullivan.net/wp-content/uploads/2016/12/PowerShell-AWS-Polly-Voices-300x260.png 300w, https://trevorsullivan.net/wp-content/uploads/2016/12/PowerShell-AWS-Polly-Voices-173x150.png 173w, https://trevorsullivan.net/wp-content/uploads/2016/12/PowerShell-AWS-Polly-Voices-150x130.png 150w" sizes="(max-width: 466px) 100vw, 466px">
           </noscript></a>As we discussed towards the beginning of the article, one of the cool features in AWS Polly is that you can choose from a variety of voices. While the AWS Polly documentation enumerates a list of them, you can also dynamically obtain a list of voice metadata from the Polly REST API. The SDK method you want to call is named <strong><a href="http://docs.aws.amazon.com/AWSJavaScriptSDK/latest/AWS/Polly.html#describeVoices-property" target="_blank">describeVoices()</a></strong>.</p> 
         <p>When you call this API, youll receive a JSON array, containing a list of the supported voices. Each voice has a name / ID, which is how youll actually identify the desired voice later on (a very easy, nice, and personal interface, in my opinion), and it also specifies the voices gender (male or female), and the language name and code (eg. en-US, de-DE).</p> 
         <ol> 
          <li>In the root of your project folder, create a new file called <strong>app.js</strong></li> 
          <li>Add the following text to <strong>app.js</strong></li> 
         </ol> 
         <pre>var AWS = require('aws-sdk');
require('./awsauth.js');

// Create a new AWS Polly object
var polly = new AWS.Polly();

polly.describeVoices(function (err, data) {
 if (err) console.log(err, err.stack); // an error occurred
 else console.log(data); // successful response
})</pre> 
         <p>After saving the file, go ahead and run <strong>node app.js</strong>, and you should see&nbsp;some JSON results printed out to your terminal.</p> 
         <pre>node app.js</pre> 
         <p>As you can see from the code above, the Amazon Polly API is pretty straightforward. All we need to do is authenticate to AWS, create a new Polly object, and then start calling the Polly APIs. While the <strong>describeVoices()</strong> API is not exactly network intensive, as it doesnt return a large result set, you can filter the results by passing in a <strong>LanguageCode</strong> that you want to target. To do that, before specifying the callback function, pass in a JSON object with a <strong>LanguageCode</strong> property. See the example below, and refer to the <a href="http://docs.aws.amazon.com/AWSJavaScriptSDK/latest/AWS/Polly.html#describeVoices-property">Amazon Polly Node.js Reference</a>, for a comprehensive list of supported language codes.</p> 
         <pre>polly.describeVoices({ "LanguageCode": "de-DE" }, function (err, data) {
 if (err) console.log(err, err.stack); // an error occurred
 else console.log(data); // successful response
})</pre> 
         <p>Lets move on to the next step, which is to generate our first audio file using Amazon Polly!</p> 
         <h2>Generate an Audio File with Polly</h2> 
         <p>Now that weve chosen an AWS Polly voice to use, lets actually go out and generate some audio files with it. Generating text-to-speech is done using a method called <strong><a href="http://docs.aws.amazon.com/AWSJavaScriptSDK/latest/AWS/Polly.html#synthesizeSpeech-property" target="_blank">synthesizeSpeech()</a></strong>. This API allows us to specify the desired voice name / ID, the text that we want to be converted to speech audio, and the output format. There are a few key input parameters that you need to specify, as a JSON object, when you call <strong>synthesizeSpeech()</strong>. Well define those as a JSON object below. Go ahead and add this to <strong>app.js</strong>.</p> 
         <pre>var params = {
 OutputFormat: 'mp3',               // You can also specify pcm or ogg_vorbis formats.
 Text: 'Good morning, Trevor.',     // This is where you'll specify whatever text you want to render.
 VoiceId: 'Carla'                   // Specify the voice ID / name from the previous step.
};</pre> 
         <p>After defining the input parameters, well define a callback function. This callback will be invoked after the synthesizeSpeech() API completes. The callback function will write the generated data to a file, otherwise the result would simply reside in a Node buffer. The buffer isnt really useful to us during development and testing.</p> 
         <pre>var synthCallback = function (err, data) {
  if (err) console.log(err, err.stack); // an error occurred
  else console.log(data); // successful response

fs.writeFile('testing.mp3', data.AudioStream, function (err) {
  if (err) { 
  console.log('An error occurred while writing the file.');
  console.log(err);
  }
  console.log('Finished writing the file to the filesystem')
  });
};</pre> 
         <p>As you can see, the callback above references the built-in Node.js filesystem module, aliased as&nbsp;<strong>fs</strong>. Youll need to make sure you add a require statement to <strong>app.js</strong>, at the top, so that we can properly reference this module. Go ahead and add that now.</p> 
         <pre>// Import the built-in Node.js filesystem module
var fs = require('fs');</pre> 
         <p>Now that weve defined the input parameters and the callback function, all you need to do is call the <strong>synthesizeSpeech()</strong> API, passing in the JSON parameter object and the callback function.</p> 
         <pre>// Call the synthesizeSpeech() API, with the user-defined parameters, and write the result to a file
polly.synthesizeSpeech(params, synthCallback);</pre> 
         <p>At this point, save the <strong>app.js</strong> file, and in your terminal run this command to invoke your application.</p> 
         <pre>node app.js</pre> 
         <p>You should now have an MP3 audio file in the root directory of your project named <strong>testing.mp3</strong>. Use your favorite MP3 player to listen to this, and make sure it sounds alright. To be honest, the quality of the Amazon Polly results are pretty impressive. During my own testing, I tried out quite a few different phrases, both English and foreign, along with a few different voices, and was pleased with nearly all of the results.</p> 
         <p>Congratulations, youve just used <a href="https://aws.amazon.com/polly" target="_blank">Amazon Polly</a> for your first time!</p>

         <h2>Play the Audio File with PowerShell</h2> 
         <p>While you can certainly use your operating systems built-in media player, or an alternative like <a href="https://videolan.org" target="_blank">VLC</a>, to test out the audio file locally, you can actually automate the process of testing out your media files using PowerShell. This simple PowerShell script will play back a media file.</p> 
         <p>Feel free to customize this script / function for your own needs, however keep in mind that this currently will only work on the Windows platform. The .NET&nbsp;Core media APIs are highly unlikely to work on the PowerShell Core Edition, which works cross-platform on Mac OS X and Linux systems. If youre using Mac OS X or Linux as your development workstation operating system, then youll probably want to substitute this PowerShell script with a different media playback utility, until the .NET Core media APIs mature.</p> 
         <pre>function Invoke-MediaFile {
    [CmdletBinding()]
    param (
        [ValidateScript({ 
            if (!(Test-Path -Path $PSItem)) { throw 'Invalid file path.' }
            else { $true }
        })]
        [string] $Path
    )
    $MediaPlayer = New-Object -TypeName System.Windows.Media.MediaPlayer
    $MediaPlayer.Volume = 1
    $MediaPlayer.Open($Path)
    $MediaPlayer.Play()
    Start-Sleep -Milliseconds 100
    while ($MediaPlayer.Position -lt $MediaPlayer.NaturalDuration.TimeSpan) {
        Start-Sleep -Milliseconds 200
    }
    $MediaPlayer.Close()
    $MediaPlayer = $null
}

Invoke-MediaFile -Path C:\ArtofShell\polly\testing2.mp3</pre> 
         <p>To run this script, simply fire up the PowerShell Integrated Scripting Editor (ISE) on your Windows system, paste the code in, and run it. Of course, youll need to modify the fully-qualified filesystem path to the audio file that was generated, unless youre using the exact same directory structure that I am. Chances are youre not. ? If PowerShell ISE isnt your thing, you can also use Visual Studio Code (hey, its cross-platform and works with any language!) in conjunction with the open source&nbsp;<a href="https://github.com/PowerShell/vscode-powershell" target="_blank">PowerShell Extension</a> for VSCode.</p> 
         <h2>Scaling Content Generation with Docker</h2> 
         <p>At this point, we have successfully built an application that is capable of dynamic audio content generation, based on textual input. If your application requires a significant amount of text-to-speech translation, then Amazon Polly is ready and able to satisfy your applications demands. As one of many infinitely scalable cloud services, Amazon Polly scales along with your applications needs. In case a single instance of your application is hindered by blocked threads,&nbsp;<a href="https://docker.com" target="_blank">Docker</a> can help you scale even more. By incorporating your application code into a custom Docker container image, your client code is capable of infinitely scaling along with Amazons global infrastructure.</p> 
         <p>You can use clustering and orchestration tools like <a href="https://www.docker.com/products/docker-swarm" target="_blank">Docker Swarm</a> or <a href="http://kubernetes.io/" target="_blank">Kubernetes</a> to build clusters of container hosts on <a href="https://aws.amazon.com/ec2/" target="_blank">Amazon EC2 cloud infrastructure</a>, or you can use <a href="https://aws.amazon.com/ecs/" target="_blank">AWS EC2 Container Service (ECS)</a> to further simplify the setup process. Whatever route you choose, I strongly encourage you to investigate Docker containers for all of your application scaling, portability, and consistent, rapid deployment needs.</p> 
         <p><em>Are you brand new to Docker? Still not entirely clear on what Docker is, and how it works, and why youd want to use it?</em> You can read more about Docker container fundamental concepts on <a href="https://trevorsullivan.net/docker/" target="_blank">my Docker documentation page</a>! If you have further questions, please reach out to me, and Ill do my best to help.</p>

AWS is the world leader of cloud technologies, and its global data centers are trusted by small start-ups, all the way up to large enterprises, for reliability, cost savings, and its feature-rich offerings.</p>
