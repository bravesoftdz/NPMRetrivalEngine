       <h2>Discover the Strengths and Weaknesses of Google Cloud Speech API in this Special Report by Cloud Academys Roberto Turrin</h2> 
       <p><span style="font-weight: 400;">Google recently opened its brand new </span><a href="https://cloud.google.com/speech/"><span style="font-weight: 400;">Cloud Speech API</span></a><span style="font-weight: 400;">  announced at the </span><a href="https://cloudplatformonline.com/NEXT2016.html"><span style="font-weight: 400;">NEXT event</span></a><span style="font-weight: 400;"> in San Francisco  for a limited preview. </span></p> 
       <p><span style="font-weight: 400;">This speech recognition technology has been developed and already used by several Google products for some time, such as the Google search engine where there is the option to make voice search.</span></p> 
       <p><span style="font-weight: 400;">The capability to convert voice to text is based on <strong>deep neural networks</strong>, state-of-the-art machine learning algorithms recently demonstrated to be particularly effective for pattern detection in video and audio signals. The neural network is updated as new speech samples are collected by Google, so that new terms are learned and the recognition accuracy keeps on increasing. </span></p> 
       <h3>Speech-to-Text in the Cloud</h3> 
       <p><span style="font-weight: 400;"><strong>Speech-to-text</strong> features are used in a multitude of use cases including voice-controlled smart assistants on mobile devices, home automation, audio transcription, and automatic classification of phone calls. </span></p> 
       <p><span style="font-weight: 400;">Now that such technology will be accessible as a cloud service to developers, it will allow any application to integrate speech-to-text recognition, representing a valuable alternative to the common </span><a href="https://developer.nuance.com/public/index.php?task=home"><span style="font-weight: 400;">Nuance technology</span></a><span style="font-weight: 400;"> (used by Apples Siri and Samsungs S-Voice, for instance) and challenging other solutions such as the </span><a href="http://www.ibm.com/watson/developercloud/speech-to-text.html"><span style="font-weight: 400;">IBM Watson speech-to-text</span></a><span style="font-weight: 400;"> and the </span><a href="https://www.microsoft.com/cognitive-services/en-us/speech-api"><span style="font-weight: 400;">Microsoft Bing Speech API</span></a><span style="font-weight: 400;">. </span></p> 
       <h3>An Outline of the Google Cloud Speech API</h3> 
       <p><span style="font-weight: 400;">The API, still in alpha, </span><span style="font-weight: 400;">exposes a RESTful interface that can be accessed via common POST HTTP requests. </span></p> 
       <p><span style="font-weight: 400;">The batch processing is very straightforward; just by providing the audio file to process and describing its format the API returns the best-matching text, together with the recognition accuracy. Optionally, it can be requested to return multiple alternatives in addition to the best-matching, each one with the estimated accuracy. </span></p> 
       <p><span style="font-weight: 400;">The file to recognise can be provided both by including the audio signal into the HTTP request payload (encoded with Base64) or by giving the URI of the file (currently, only Google Storage can be used). Supported formats are <strong>raw audio</strong> and <strong>FLAC</strong> format, while MP3 and AAC are not accepted. </span></p> 
       <p><span style="font-weight: 400;">In order to improve the accuracy of the system, words or sentences can be attached to the request as text. This is particularly useful in the case of noisy audio signals or when uncommon, domain-specific words are present. </span></p> 
       <p><span style="font-weight: 400;">Additional, interesting options are the filter for profanities  which allow&nbsp;to mask profanities with asterisks  and the possibility to receive interim results, i.e., partial results marked as non-final.</span></p> 
       <p><span style="font-weight: 400;">A few clients are provided for common programming languages (e.g., Python, Java, iOS, Node.js), both for <strong>batch</strong> and <strong>real-time requests</strong> (with asynchronous responses). </span></p> 
       <h3>My Initial Experience and Code Samples</h3> 
       <p><span style="font-weight: 400;">My quick experience with the API has revealed quite an accurate technology. Regardless the APIs do not accept MP3 as input audio, I took the chance&nbsp;to&nbsp;stress the system and I tried to&nbsp;experiment with&nbsp;an&nbsp;MP3 file containing an <a href="http://tinyurl.com/gkrfb6k">online English lesson</a>. I converted&nbsp;the first 15 seconds of the file to a 200-KB FLAC format that I submitted to the Google Speech APIs with the following Python script:</span></p>

    tokens = word_tokenize(sentence)
    # detect the PoS of each token
    for tag in pos_tag(tokens):
        print '%s: %s' % (tag[0], tag[1])</textarea>

       <p><span style="font-weight: 400;">Anyway, you can immediately note that the lack of any key punctuation (e.g., question marks, full stops, etc.) makes the work of the PoS tagger particularly challenging, as not even sentences are not recognised. </span></p> 
       <p><span style="font-weight: 400;">In my opinion, the integration of punctuation and PoS tagging directly in the speech-to-text service might take advantage of&nbsp;the properties of the speech signal &nbsp; such as intonation, gaps, etc.  which provide useful information to identify particular elements of the whole speech and of the single sentences. </span></p> 
       <p><span style="font-weight: 400;">Finally, the voice recognition still does not provide&nbsp;the capability to identify the source of the voice  in our&nbsp;case&nbsp;multiple subjects are speaking. This&nbsp;feature&nbsp;could be used, for instance, to transcript&nbsp;dialogs.</span></p> 
       <h3>More Information</h3> 
       <p><span style="font-weight: 400;">Waiting for the API to be publicly released to all developers, if your curiosity has been aroused and you wish to play with the speech-to-text service, just go to </span><a href="https://www.google.com/intl/it/chrome/demos/speech.html"><span style="font-weight: 400;">https://www.google.com/intl/it/chrome/demos/speech.html</span></a><span style="font-weight: 400;"> for a live demo (or&nbsp;enrol for the </span><a href="https://services.google.com/fb/forms/speech-api-alpha/"><span style="font-weight: 400;">limited preview</span></a><span style="font-weight: 400;">). You will be surprised by the number of supported languages  80  way more than the 38 supported by Nuance, the 28 by Microsoft Bing Speech, and the 8 by IBM Watson.</span></p>
