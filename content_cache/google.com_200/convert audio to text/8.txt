          <div class="article__body"> 
           <p>In this tutorial we are going to experiment with the <a href="https://developer.mozilla.org/en-US/docs/Web/API/Web_Speech_API" target="_blank" rel="noreferrer noopener">Web Speech API</a>. It's a very powerful browser interface that allows you to record human speech and convert it into text. We will also use it to do the opposite - reading out strings in a human-like voice. </p> 
           <p>Let's jump right in!</p> 
           <h2>The App</h2> 
           <p>To showcase the ability of the API we are going to build a simple voice-powered note app. It does 3 things:</p> 
           <ul>
            <li>Takes notes by using voice-to-text or traditional keyboard input.</li> 
            <li>Saves notes to localStorage.</li> 
            <li>Shows all notes and gives the option to listen to them via Speech Synthesis. </li> 
           </ul> 
           <figure>
            <a href="https://demo.tutorialzine.com/2017/08/converting-from-speech-to-text-with-javascript/" target="_blank" rel="noreferrer noopener"> <img src="https://tutorialzine.com/media/2017/08/demo2.png" alt="demo2.png"></a> 
            <figcaption>
             Our App for Taking Notes Using Voice Input.
            </figcaption>
           </figure> 
           <p>We won't be using any fancy dependencies, just good old <a href="https://jquery.com/" target="_blank" rel="noreferrer noopener">jQuery</a> for easier DOM operations and <a href="https://shoelace.style/index.html" target="_blank" rel="noreferrer noopener">Shoelace</a> for CSS styles. We are going to include them directly via CDN, no need to get NPM involved for such a tiny project.</p> 
           <p>The HTML and CSS are pretty standard so we are going to skip them and go straight to the JavaScript. To view the full source code go to the <strong>Download</strong> button near the top of the page.</p> 
           <h2>Speech to Text</h2> 
           <p>The Web Speech API is actually separated into two totally independent interfaces. We have <a href="https://developer.mozilla.org/en-US/docs/Web/API/SpeechRecognition" target="_blank" rel="noreferrer noopener">SpeechRecognition</a> for understanding human voice and turning it into text (Speech -&gt; Text) and <a href="https://developer.mozilla.org/en-US/docs/Web/API/SpeechSynthesis" target="_blank" rel="noreferrer noopener">SpeechSynthesis</a> for reading strings out loud in a computer generated voice (Text -&gt; Speech). We'll start with the former.</p> 
           <p>The Speech Recognition API is surprisingly accurate for a free browser feature. It recognized correctly almost all of my speaking and knew which words go together to form phrases that make sense. It also allows you to dictate special characters like full stops, question marks, and new lines. </p> 
           <p>The first thing we need to do is check if the user has access to the API and show an appropriate error message. Unfortunately, the speech-to-text API is supported only in Chrome and Firefox (with a flag), so a lot of people will probably see that message.</p> 
           <pre data-type="js">try {
  var SpeechRecognition = window.SpeechRecognition || window.webkitSpeechRecognition;
  var recognition = new SpeechRecognition();
}
catch(e) {
  console.error(e);
  $('.no-browser-support').show();
  $('.app').hide();
}</pre> 
           <p>The <code>recognition</code> variable will give us access to all the API's methods and properties. There are various options available but we will only set <code>recognition.continuous</code> to true. This will enable users to speak with longer pauses between words and phrases. </p> 
           <p>Before we can use the voice recognition, we also have to set up a couple of event handlers. Most of them simply listen for changes in the recognition status:</p> 
           <pre data-type="js">recognition.onstart = function() { 
  instructions.text('Voice recognition activated. Try speaking into the microphone.');
}

recognition.onspeechend = function() {
  instructions.text('You were quiet for a while so voice recognition turned itself off.');
}

recognition.onerror = function(event) {
  if(event.error == 'no-speech') {
    instructions.text('No speech was detected. Try again.');  
  };
}</pre> 
           <p>There is, however, a special <code>onresult</code> event that is very crucial. It is executed every time the user speaks a word or several words in quick succession, giving us access to a text transcription of what was said.</p> 
           <p>When we capture something with the <code>onresult</code> handler we save it in a global variable and display it in a textarea:</p> 
           <pre data-type="js">recognition.onresult = function(event) {

  // event is a SpeechRecognitionEvent object.
  // It holds all the lines we have captured so far. 
  // We only need the current one.
  var current = event.resultIndex;

  // Get a transcript of what was said.
  var transcript = event.results[current][0].transcript;

  // Add the current transcript to the contents of our Note.
  noteContent += transcript;
  noteTextarea.val(noteContent);
}</pre> 
           <p>The above code is slightly simplified. There is a very weird bug on Android devices that causes everything to be repeated twice. There is no official solution yet but we managed to solve the problem without any obvious side effects. With that bug in mind the code looks like this:</p> 
           <pre data-type="js">var mobileRepeatBug = (current == 1 &amp;&amp; transcript == event.results[0][0].transcript);

if(!mobileRepeatBug) {
  noteContent += transcript;
  noteTextarea.val(noteContent);
}</pre> 
           <p>Once we have everything set up we can start using the browser's voice recognition feature. To start it simply call the <code>start()</code> method:</p> 
           <pre data-type="js">$('#start-record-btn').on('click', function(e) {
  recognition.start();
});</pre> 
           <p>This will prompt users to give permission. If such is granted the device's microphone will be activated. </p> 
           <blockquote class="info">
            Most APIs that require user permission don't work on non-secure hosts. Make sure you are serving your Web Speech apps over HTTPS.
           </blockquote> 
           <p>The browser will listen for a while and every recognized phrase or word will be transcribed. The API will stop listening automatically after a couple seconds of silence or when manually stopped.</p> 
           <pre data-type="js">$('#pause-record-btn').on('click', function(e) {
  recognition.stop();
});</pre> 
           <p>With this, the speech-to-text portion of our app is complete! Now, let's do the opposite!</p> 
           <h2>Text to Speech</h2> 
           <p>Speech Synthesys is actually very easy. The API is accessible through the <a href="https://developer.mozilla.org/en-US/docs/Web/API/SpeechSynthesis" target="_blank" rel="noreferrer noopener">speechSynthesis</a> object and there are a couple of methods for playing, pausing and other audio related stuff. It also has a couple of cool options that change the pitch, rate, and even the voice of the reader. </p> 
           <p>All we will actually need for our demo is the <code>speak()</code> method. It expects one argument, an instance of the beautifully named <code>SpeechSynthesisUtterance</code> class. </p> 
           <p>Here is the entire code needed to read out a string.</p> 
           <pre data-type="js">function readOutLoud(message) {
  var speech = new SpeechSynthesisUtterance();

  // Set the text and voice attributes.
  speech.text = message;
  speech.volume = 1;
  speech.rate = 1;
  speech.pitch = 1;

  window.speechSynthesis.speak(speech);
}</pre> 
           <p>When this function is called, a robot voice will read out the given string, doing it's best human impression. </p> 
           <h2>Conclusion</h2> 
           <p>In an era where voice assistants are more popular then ever, an API like this gives you a quick shortcut to building bots that understand and speak human language.</p> 
           <p>Adding voice control to your apps can also be a great form of accessibility enhancement. Users with visual impairment can benefit from both speech-to-text and text-to-speech user interfaces. </p> 
           <p>The speech synthesis and speech recognition APIs work pretty well and handle different languages and accents with ease. Sadly, they have limited browser support for now which narrows their usage in production. If you need a more reliable form of speech recognition, take a look at these third-party APIs:</p> 
           <ul>
            <li><a href="https://cloud.google.com/speech/" target="_blank" rel="noreferrer noopener">Google Cloud Speech API</a> </li> 
            <li><a href="https://azure.microsoft.com/en-us/services/cognitive-services/speech/" target="_blank" rel="noreferrer noopener">Bing Speech API</a></li> 
            <li><a href="https://cmusphinx.github.io/" target="_blank" rel="noreferrer noopener">CMUSphinx</a> and it's JavaScript version <a href="https://syl22-00.github.io/pocketsphinx.js/" target="_blank" rel="noreferrer noopener">Pocketsphinx</a> (both open-source).</li> 
            <li><a href="https://api.ai/" target="_blank" rel="noreferrer noopener">API.AI</a> - Free Google API powered by Machine Learning</li> 
           </ul>
