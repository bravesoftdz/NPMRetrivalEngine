            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>The software you can use is CMUSphinx. Unlike suggested in another answer Julius is not suitable because it requires models. Models for large vocabulary speech recognition are not available for Julius.</p> 
              <p>You can use pocketsphinx to convert audio file. Those two commands must do the work. First you convert the file to the required format and then you recognize it:</p> 
              <pre><code>ffmpeg -i file.mp3 -ar 16000 -ac 1 file.wav
</code></pre> 
              <p>The run pocketsphinx </p> 
              <pre><code>pocketsphinx_continuous -infile file.wav 2&gt; pocketsphinx.log &gt; result.txt
</code></pre> 
              <p>Result will be stored in result.txt.</p>

             <div class="post-text" itemprop="text"> 
              <p>I know this is old, but to expand on Nikolay's answer and hopefully save someone some time in the future, in order to get an up-to-date version of pocketsphinx working you need to compile it from the github or sourceforge repository (not sure which is kept more up to date).</p> 
              <pre><code>git clone https://github.com/cmusphinx/sphinxbase.git
cd sphinxbase
./audogen.sh
./configure
make -j8
make -j8 check
sudo make install
cd ..
git clone https://github.com/cmusphinx/pocketsphinx.git
cd pocketsphinx
./autogen.sh
./configure
make -j8
make -j8 check
sudo make install
cd ..
</code></pre> 
              <p>Then, from: <a href="https://sourceforge.net/projects/cmusphinx/files/Acoustic%20and%20Language%20Models/US%20English/" rel="nofollow noreferrer">https://sourceforge.net/projects/cmusphinx/files/Acoustic%20and%20Language%20Models/US%20English/</a> download the newest versions of <code>cmusphinx-en-us-....tar.gz</code> and <code>en-70k-....lm.gz</code></p> 
              <pre><code>tar -xzf cmusphinx-en-us-....tar.gz
gunzip en-70k-....lm.gz
</code></pre> 
              <p>Then you can finally proceed with the steps from Nikolay's answer:</p> 
              <pre><code>ffmpeg -i book.mp3 -ar 16000 -ac 1 book.wav
pocketsphinx_continuous -infile book.wav \
    -hmm cmusphinx-en-us-8khz-5.2 -lm en-70k-0.2.lm \
    2&gt;pocketsphinx.log &gt;book.txt
</code></pre> 
              <p>Sphinx works alright. I wouldn't rely on it to make a readable version of the text, but it's good enough that you can search it if you're looking for a particular quote. That works especially well if you use a search algorithm like Xapian (<a href="http://www.lesbonscomptes.com/recoll/" rel="nofollow noreferrer">http://www.lesbonscomptes.com/recoll/</a>) which accepts wildcards and doesn't require exact search expressions.</p> 
              <p>Hope this helps.</p>
