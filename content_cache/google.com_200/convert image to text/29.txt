       <h1> Build an Image Generator with html2canvas </h1> 
       <br> 
      </div> 
      <div id="flash_container"> 
      </div> 
      <div>
       March 18, 2016 
      </div> 
      <hr> 
      <p><img src="https://d3n8a8pro7vhmx.cloudfront.net/veracitymedia/pages/2002/attachments/original/1456946943/marco-mag-blog.jpg?1456946943" alt="marco-mag-blog.jpg"></p> 
      <p>&nbsp;</p> 
      <p>From the <a href="http://mashable.com/2015/10/09/jeb-bush-pointing/">countless riffs on a photo of Jeb Bush pointing in front of a green screen</a> during a campaign stop late last year to an image of Donald Trump's lips Photoshopped over his eyes that went viral last month&nbsp;<a href="http://www.buzzfeed.com/juliareinstein/someone-swapped-donald-trumps-eyes-for-his-lips-and-its-just">(disturbingly, he looks exactly the same)</a>, there's huge potential in image editing for sharing messages and poking fun at political opponents.</p>
      <p>With the enormous ecosystem of photo-editing tools available, it's no surprise that photo manipulation has become a weapon of choice for aspiring political satirists this election cycle. But the high cost of professional tools like Photoshop and the prohibitive learning curve of graphic design raise the barriers to entry to produce images like these. Many developers, sensing opportunity in this knowledge gap, have built tools that make it easier to caption images and produce commonly shared memes.</p> 
      <p>When <em>Time Magazine</em> put Marco Rubio on the cover in February, calling him "The Republican Savior," the image was just begging for snarky photoshop shenanigans. <strong><a title="Make Your Own Marco Generator - Veracity Media" href="http://snip.ly/rlfaa" target="_blank">I put together an image generator in CodePen</a></strong>&nbsp;allowing users to make their own Marco cover. If you've got some front-end development skill, you can build an image generator, too (this tutorial assumes some familiarity with HTML, CSS, JavaScript, and jQuery). A jQuery plugin called <a href="https://html2canvas.hertzen.com/">html2canvas</a> makes them relatively straightforward to build.</p> 
      <p>Here's how.</p> 
      <hr> 
      <h2>Step 1: Preparation and dependencies</h2> 
      <p>We'll need to create an HTML file, a CSS stylesheet, and a JavaScript script. We'll need to link the stylesheet and script in our HTML, along with two dependencies—jQuery and html2canvas.</p> 
      <p>Fortunately, CloudFlare's cdnjs.com provides both dependencies as CDNs. So we can put them right into our HTML:</p> 
      <pre>  <code>
    
    &lt;script&nbsp;type="text/javascript"&nbsp;src="//cdnjs.cloudflare.com/ajax/libs/jquery/2.1.3/jquery.min.js"&gt;&lt;/script&gt;
    &nbsp;&lt;script&nbsp;type="text/javascript"&nbsp;src="//cdnjs.cloudflare.com/ajax/libs/html2canvas/0.4.1/html2canvas.js"&gt;&lt;/script&gt;
    &nbsp;&lt;script&nbsp;type="text/javascript"&nbsp;src="script.js"&gt;&lt;/script&gt;
    &nbsp;&lt;link&nbsp;rel="stylesheet"&nbsp;type="text/css"&nbsp;href="style.css"&gt;
  </code>
</pre> 
      <p>Now we're ready to start building the HTML structure.</p> 
      <hr> 
      <h2>Step 2: HTML</h2> 
      <p>Before we dive into the jQuery, let's think about all the pieces we'll need for our generator and have them ready-built so they're good to go when we start implementing the script.</p> 
      <p>We want the user to be able to type a message, watch as that message is dynamically appended to the background image, and then press a button to take a 'screenshot' that turns the message and the background image into a new image the user can save.</p> 
      <p>So, we're going to need:</p> 
      <ul> 
       <li>An input where the user can type in their message and submit it to generate the image</li> 
       <li>An image preview section where text from the input is appended as the user types, so the user can see what their image will look like</li> 
       <li>A method of delivering the newly created image to the user—either in a new browser tab or in a section of the current tab</li> 
      </ul> 
      <p>For that last one, in order to avoid dealing with security restraints involved in opening a new tab on a user's browser, we'll use a simple lightbox overlay that shows the new image.</p> 
      <pre>  <code>
    
  &lt;form&nbsp;class="message-form"&gt;
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&lt;input&nbsp;type="text"&nbsp;class="message-box"/&gt;
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&lt;button&nbsp;type="button"&nbsp;class="message-submit-button"&gt;
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Save&nbsp;PNG&nbsp;Image
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&lt;/button&gt;
  &nbsp;&nbsp;&nbsp;&nbsp;&lt;/form&gt;
  &nbsp;&nbsp;&nbsp;&nbsp;&lt;div&nbsp;class="image-preview-wrap"&gt;
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&lt;div&nbsp;class="image-preview"&gt;
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&lt;p&nbsp;class="image-message"&gt;
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Default&nbsp;text&nbsp;here
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&lt;/p&gt;
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&lt;/div&gt;
  &nbsp;&nbsp;&nbsp;&nbsp;&lt;/div&gt;
  &nbsp;&nbsp;&nbsp;&nbsp;&lt;div&nbsp;class="lightbox"&gt;
  &nbsp;&nbsp;&nbsp;&nbsp;&lt;a&nbsp;href="#"&nbsp;class="close-lightbox"&gt;&amp;times;&lt;/a&gt;
  &nbsp;&nbsp;&nbsp;&nbsp;&lt;img&nbsp;src=""&nbsp;class="new-image"/&gt;
  &nbsp;&nbsp;&nbsp;&nbsp;&lt;/div&gt;
  </code>
</pre> 
      <p>The CSS to make this work is fairly simple. We'll need a <code>background-image</code> on the and some font and color styles on the to make the generated image look interesting.</p> 
      <p>We'll also need to set the <code>.lightbox</code> div to <code>display: none; position: fixed; z-index: 99</code> to make sure that it's hidden by default and that, when we use jQuery to show it, it appears on top of the rest of the page.</p> 
      <p>A word of warning: certain styles can mess with html2canvas's ability to render the image preview properly. For example, setting <code>background-size: cover</code> on a background image on or within the preview element can cause that background image to appear transparent or solid black in the image html2canvas creates. If you're getting similar errors, try playing with the styles, and especially consider testing with a simpler version of any dynamic or experimental styles, which may be less likely to render.</p> 
      <hr> 
      <h2>Step 3: jQuery</h2> 
      <p>Now that we have our HTML structured, let's think about what we need our script to accomplish. There are 6 primary tasks:</p> 
      <ul> 
       <li>Appending text from our message box to our image preview in real time</li> 
       <li>Preventing our form from triggering the default submit event so we can hijack it with jQuery to build our new image instead</li> 
       <li>Using a listener on the submit button to tell html2canvas to construct our image</li> 
       <li>Displaying our lightbox</li> 
       <li>Outputting the image html2canvas gives us as a source for the blank image in our lightbox</li> 
       <li>And finally, attaching a listener on the lightbox's close button so users can return to the app after they've created an image</li> 
      </ul> 
      <p>To accomplish that, we'll need a couple blocks of jQuery. We'll use this to append text to the image preview:</p> 
      <pre>  <code>
  $('.message-box').keyup(
    function(){
      var value = $(this).val();
      $('.image-message').text(value);
    }
  );
  </code>
</pre> 
      <p>jQuery is smart enough to know whether we intend to use its functions to grab an element's content or replace it with new content. So all we need to do is attach a listener to the input box that triggers on every keystroke. Each time a user hits a key, jQuery will look at the contents of the input and append them as text inside the element in our image preview.</p> 
      <p>Next, we'll need a code snippet that cancels the default response when a user hits the submit button:</p> 
      <pre>  <code>
  $('.message-form').submit(function(e){
    e.preventDefault;
    return false;
  });
  </code>
</pre> 
      <p>By default, browsers will assume you want to send the contents of the form's inputs somewhere and load a new page when a form's submit button is triggered. But html2canvas actually doesn't look at our input boxes at all. It's entirely focused on making a copy of the image preview area. So we want to override the default response and replace it with our own script.</p> 
      <p>Our final two code blocks involve attaching an <code>.on('click')</code> event listener to the submit button and lightbox close button. Because we need to wait until the page has loaded until we can do so, we need to wrap both of these blocks in a <code>$(document).ready</code> function like this:</p> 
      <pre>  <code>
  $(document).ready(function(){
    [ First code block goes here ]
    [ Second code block goes here ]
  });
  </code>
</pre> 
      <p>First, we need to make the image, swap it in for the missing image in the lightbox, and display the lightbox.</p> 
      <p>When the submit button is clicked, we'll trigger the html2canvas function, which we set as an argument in jQuery's <code>.on('click')</code> function. The html2canvas function takes two arguments: the element we want to convert into a <code>canvas</code>, and an object that allows us to configure options. The <code>onrendered</code> key allows us to set as its value a callback function that waits until the rendering is complete and then triggers, with the <code>canvas</code> that html2canvas has created passed through as its argument:</p> 
      <pre>  <code>
  $('.message-submit-button').on('click', function(){
    html2canvas($('.image-preview-wrap'), {
      onrendered: function(canvas) {
        var myImage = canvas.toDataURL('image/png');
        $('.lightbox').fadeIn(200);
        $('.new-image').attr('src', myImage).fadeIn(200);
      }
    });
  });
  </code>
</pre> 
      <p>Now we have the element as a <code>canvas</code>, and although it's not yet an image, it's simple to parse it with JavaScript and return a data URI, which the user will see as a regular png image file they can save like any other. We just need to use the <code>canvas.toDataURL</code> function, with the file type as our argument(you can also pass through <code>'image/jpeg'</code> for a jpg instead of a png.)</p> 
      <p>Then, we'll use jQuery's <code>.fadeIn()</code> function, which flips our <code>lightbox</code> element's <code>display</code> style on and also animates it so it fades in over a period of milliseconds we can specify. We'll also feed our new image the data URI we just created as its source by calling jQuery's <code>.attr()</code> function and passing in the <code>src</code> attribute and our data URI as arguments.</p> 
      <p>Finally, we'll add one more block to put on a listener on the lightbox's close button so the user can close the lightbox.</p> 
      <pre>  <code>
  $('.closebox').on('click', function(){
    $('.lightbox').css('display', 'none');
  });
  </code>
</pre> 
      <p>And that's it! Our image generator is now ready to go.</p>
