              <p>If you have a working example in any language, then it should not be had to translate that into processing/java. But if you don't understand the code, that is maybe not the best way.</p> 
              <p>I don't know how far you are yet. But i would start with a static image and if that works, you can proceed to an animated/interactive version.</p> 
              <p>With letters instead of words, that's kind of easy, you can find a good start in <br> <strong>Examples -&gt; Libraries -&gt; Video -&gt; Capture -&gt; AsciiVideo.</strong></p> 
              <p>The basic idea is to break down the image into smaller areas and find the average brightness of these areas. Then you place a letter that has a similiar brightness-value in this area. (i.e: a "M" should appear darker than an "i").</p> 
              <p>With words this gets more complicated, you would have to consider regions of different sizes. Depending on the desired output, you may have to think about how much words may overlap, how much the size of the words can differ, etc.</p>

              <p>Ok, i suppose you did not look into the AsciiVideo-example? Did not see any code yet, so i can only guess your level of code-comprehension. Here is another very basic example, from there it's still a lot of work until you get results like you posted above.</p> 
              <pre><code>PImage src;
int threshold = 150;

void setup(){
  // basic settings
  size(800, 600);
  textAlign(CENTER);
  background(255);
  fill(0);

  // load an image
  src = loadImage("http:"+"//images.techtimes.com/data/images/full/161832/snoopy-dance.png");
  // resize
  src.resize(80, 60);


  // load piyels of source-image
  src.loadPixels();

  // iterate over image-pixels
  for(int i=0; i&lt;src.pixels.length; i++ ){
    // get the color
    int col = src.pixels[i];

    // draw a word if pixels is darker than threshold
    if(brightness(col) &lt; threshold){
      int x = i%src.width * 10;
      int y = i/src.width * 10 +10;
      text("snoopy", x, y );
    }
  }
  src.updatePixels();
}
</code></pre>

              <p>Sure you can use an image instead of a video. A video is actually just an sequence of images.</p> 
              <p>The resize in the example above is to get a low-resolution image, where each pixel represents an area of 10px*10px from the original image. You could as well calculate the average brightness of these 10x10 squares yourself, but the resize-function already does that.</p>
