       <h1>babel-sublime</h1> 
       <blockquote> 
        <p>Only supports <strong>Sublime Text 3</strong>. Sublime Text 2 is no longer supported.</p> 
       </blockquote> 
       <p>Language definitions for <a href="http://kangax.github.io/compat-table/es6/">ES6+ JavaScript</a> with <a href="http://facebook.github.io/react/docs/jsx-in-depth.html">React JSX syntax</a> extensions.</p> 
       <h2>Installation</h2> 
       <p>Find it as <a href="https://packagecontrol.io/packages/Babel"><strong>Babel</strong></a> through <a href="https://packagecontrol.io/">Package Control</a>.</p> 
       <h4>Setting as the default syntax</h4> 
       <p>To set it as the default syntax for a particular extension: 1. Open a file with that extension, 2. Select <code>View</code> from the menu, 3. Then <code>Syntax</code> <code>-&gt;</code> <code>Open all with current extension as...</code> <code>-&gt;</code> <code>Babel</code> <code>-&gt;</code> <code>JavaScript (Babel)</code>. 4. Repeat this for each extension (e.g.: <code>.js</code> and <code>.jsx</code>).</p> 
       <h4>Setting a Color Scheme</h4> 
       <p><code>Babel</code> comes bundled with <code>Next</code> and <code>Monokai</code> from <a href="https://github.com/Benvie/JavaScriptNext.tmLanguage">Benvie/JavaScriptNext.tmLanguage</a>. Select one from <code>Preferences</code> <code>-&gt;</code> <code>Color Scheme</code> <code>-&gt;</code> <code>Babel</code></p> 
       <h4>Advanced usage</h4> 
       <p>It's possible to set <code>Babel</code> as the <em>only</em> JavaScript package by disabling the stock one. To do so, add <code>"ignored_packages": ["JavaScript"]</code> to your <code>Preferences.sublime-settings</code>. The benefits include: * extension-less node scripts will automatically be recognized as <code>JavaScript (Babel)</code>, * and reduced clutter in the syntax menu.</p> 
       <p><strong>Keep in mind</strong>, the stock snippets will no longer work (you may still use your own), and other third-party packages that depend on the stock package may break (no known ones so far).</p> 
       <h2>Screenshots</h2> 
       <p><img src="/readmes/img/332987a8b13a70c86fc9c3b81e031d237b875aee.png" alt="babel-sublime-vs-sublime-react--react-class"></p> 
       <ul>
        <li><code>babel-sublime</code> supports modern JavaScript syntax, including <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Functions/Arrow_functions">arrow functions</a>, <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Destructuring_assignment">destructuring</a>, <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Functions/Method_definitions">shorthand methods</a>, <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/template_strings">template strings</a>, and more.</li> 
       </ul>
       <p><img src="/readmes/img/8e99bbdc99a285708a1495a6e3dd68916753906e.png" alt="babel-sublime-vs-sublime-react--jsx-comments-etc"></p> 
       <ul>
        <li><code>babel-sublime</code> correctly matches JSX comments between attributes, namespaced components, and non-alpha characters in tag/attribute names.</li> 
       </ul>
       <p><img src="/readmes/img/67d28905a384bdf47c252a75e40a3363ea99dfe2.png" alt="babel-sublime-vs-sublime-react--jsx-tight"></p> 
       <ul>
        <li>Greater-/less-than comparisons are correctly identified as such and not mistaken for JSX tags.</li> 
       </ul>

       <h2>About</h2> 
       <p>Under the hood, <em>babel-sublime</em> is based on the excellent <a href="https://github.com/Benvie/JavaScriptNext.tmLanguage">Benvie/JavaScriptNext.tmLanguage</a> with JSX syntax support built on top. The initial definitions for JSX came from <a href="https://github.com/reactjs/sublime-react">reactjs/sublime-react</a> via <a href="https://github.com/yungsters/sublime.git">yungters/sublime</a> - special thanks go to <a href="https://github.com/jgebhardt">@jgebhardt</a> and <a href="https://github.com/zpao">@zpao</a>.</p> 
       <h2>Contributing</h2> 
       <p>Pull Requests should include your changes to the respective <code>YAML-tmKittens</code> file as well as the converted <code>tmKittens</code> file. Use <a href="https://github.com/SublimeText/AAAPackageDev">AAAPackageDev</a> to convert the <code>YAML-tmKittens</code> files.</p>
