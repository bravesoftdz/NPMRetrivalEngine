        <h1 class="entry-title" itemprop="headline">5 Brilliant Ways to Lazy Load Images For Faster Page Loads</h1> 
        <p class="entry-meta"><time class="entry-time" itemprop="datePublished" datetime="2017-09-20T18:27:46+00:00">September 20, 2017</time> by <span class="entry-author" itemprop="author" itemscope itemtype="http://schema.org/Person"><a href="http://blog.dynamicdrive.com/author/dynamic_drive/" class="entry-author-link" itemprop="url" rel="author"><span class="entry-author-name" itemprop="name">Dynamic Drive</span></a></span> <span class="entry-comments-link"><a href="http://blog.dynamicdrive.com/5-brilliant-ways-to-lazy-load-images-for-faster-page-loads/#comments">5 Comments</a></span> </p>
       </header>
       <div class="entry-content" itemprop="text">
        <p><a href="http://httparchive.org/interesting.php?a=All&amp;l=Dec%2015%202016&amp;s=Top1000" target="_blank" rel="noopener">At last check</a> (Dec 2016), images make up over 50% of the top 1000 web sites’ total size, and dissecting your own site probably yields similar numbers. When it comes to dramatically improving page loading times then (which has countless benefits, from decreased bounce rates, improved conversations, to higher Google rankings), it makes sense to start with your site’s images first. Tools such as <a href="http://tools.dynamicdrive.com/imageoptimizer/" target="_blank" rel="noopener">Online Image Optimizer</a> help reduce your large images’ size without any noticeable degradation in quality, though the images are still loaded indiscriminately nonetheless. That’s where lazy loading images come in- as the name suggests, the idea is to asynchronously or even conditionally load images on a page only when they appear in the browser’s viewport, so some images are not loaded at all if the user doesn’t get that far on the page. You’ve most likely seen this in action on many blogs already. In this post we’ll look at 5 of the best scripts to lazy load images, simply and automatically without much afterthought required from you. They are listed in increasing order of features depending on your needs.</p> 
        <h2>#1 Lazy Load Images after page has loaded using just 7 lines of JavaScript</h2> 
        <p><img class="lazy lazy-hidden alignnone size-full wp-image-173" style="float: right;" src="//blog.dynamicdrive.com/wp-content/plugins/a3-lazy-load/assets/images/lazy_placeholder.gif" data-lazy-type="image" data-src="http://blog.dynamicdrive.com/wp-content/uploads/2017/01/js.png" alt="" width="129" height="129">
         <noscript>
          <img class="alignnone size-full wp-image-173" style="float: right;" src="http://blog.dynamicdrive.com/wp-content/uploads/2017/01/js.png" alt="" width="129" height="129">
         </noscript>The first method of lazy loading images consist of just 7 lines of JavaScript, and is sufficient in optimizing the initial download time of the page while still loading all images automatically. Simply modify the images you wish to lazy load so their “<code>src</code>” property points to a blank or “loading spinner”, and use a custom attribute such as <code>data-src</code> to store the reference to the real image that should be lazily loaded:</p> 
        <pre><code>&lt;img src="data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7" data-src="realimage.gif" /&gt;

          <!-- Load animated gif by default -->
&lt;img src="spinner.gif" data-src="realimage2.gif" /&gt;</code></pre> 
        <p>In case you’re wondering, the original “src” property in the first img tag above simply points to a Base 64 transparent gif (<strong>tip:</strong> You can use a tool such as <a href="http://tools.dynamicdrive.com/imagetobase64/" target="_blank" rel="noopener">Image to Base 64 Converter</a> to convert any image into Base 64 format). Then at the bottom of your page add the following JavaScript to scan through all images on the page and lazy load those with a <code>data-src</code> attribute defined <strong>after</strong> the page has fully loaded:</p> 
        <pre><code>&lt;script&gt;
window.addEventListener('load', function(){
    var allimages= document.getElementsByTagName('img');
    for (var i=0; i&lt;allimages.length; i++) {
        if (allimages[i].getAttribute('data-src')) {
            allimages[i].setAttribute('src', allimages[i].getAttribute('data-src'));
        }
    }
}, false)
&lt;/script&gt;</code></pre> 
        <p>The key action that makes this optimization work is loading the real images after the page has loaded. By relegating the target images to load asynchronously after the page has loaded, the page, well, loads faster, and actions that depend on the page having finished loading to fire such as Google Ads, many third party scripts, etc will fire sooner. Google most likely will also evaluate the page as having a decreased loading time, which helps with your SEO.</p> 
        <div class="summarybox"> 
         <h3>In Summary:</h3> 
         <p><strong>Pros of this technique:</strong></p> 
         <ul> 
          <li>Simple as pie to implement</li> 
          <li>Only 7 lines of JavaScript unminified</li> 
         </ul> 
         <p><strong>Cons:</strong></p> 
         <ul> 
          <li>Still loads every image on the page whether or not they’re seen.</li> 
          <li>If your initial page contents take a long time to load, there is a long waiting time before users can see the lazily loaded images.</li> 
         </ul> 
        </div> 
        <h2>#2 <a href="https://github.com/toddmotto/echo" target="_blank" rel="noopener">Echo.js Lazy Image Loading JavaScript</a></h2> 
        <p><img class="lazy lazy-hidden alignnone size-full wp-image-175" style="float: right;" src="//blog.dynamicdrive.com/wp-content/plugins/a3-lazy-load/assets/images/lazy_placeholder.gif" data-lazy-type="image" data-src="http://blog.dynamicdrive.com/wp-content/uploads/2017/01/echojs.png" alt="" width="180" height="71">
         <noscript>
          <img class="alignnone size-full wp-image-175" style="float: right;" src="http://blog.dynamicdrive.com/wp-content/uploads/2017/01/echojs.png" alt="" width="180" height="71">
         </noscript>Echo.js is the next logical candidate to consider if you’re looking for a compact, vanilla JavaScript that actually lazily loads images on demand, when they become visible in the user’s screen. Only 1kb minified, the script focuses squarely on lazy loading images, without any bells or whistles.</p> 
        <p>Here’s an example implementation of the script:</p> 
        <pre><code>&lt;img src="img/blank.gif" alt="Photo" data-echo="img/photo.jpg" /&gt;

&lt;script src="dist/echo.js"&gt;&lt;/script&gt;
&lt;script&gt;
echo.init({
    callback: function (element, op) {
        console.log(element, 'has been', op + 'ed')
    }
});

// echo.render(); is also available for non-scroll callbacks
&lt;/script&gt;</code></pre> 
        <div class="summarybox"> 
         <h3>In Summary:</h3> 
         <p><strong>Pros of this technique:</strong></p> 
         <ul> 
          <li>No Library dependencies such as jQuery</li> 
          <li>Only 1kb minified in size</li> 
          <li>Can also lazy load background images</li> 
          <li>Supports callback after an image has been lazily loaded</li> 
         </ul> 
         <p><strong>Cons:</strong></p> 
         <ul> 
          <li>No built in “loading gif” or image transition when revealing the images, though you can easily implement one yourself using the callback function.</li> 
          <li>Doesn’t work with images inside a scrollable DIV</li> 
         </ul> 
        </div> 
        <h2>#3 <a href="http://dinbror.dk/blazy/" target="_blank" rel="noopener">BLazy.js Lazy Image Loading JavaScript</a></h2> 
        <p><img class="lazy lazy-hidden alignnone size-full wp-image-178" style="float: right;" title="BLazy.js Lazy Image Loading JavaScript" src="//blog.dynamicdrive.com/wp-content/plugins/a3-lazy-load/assets/images/lazy_placeholder.gif" data-lazy-type="image" data-src="http://blog.dynamicdrive.com/wp-content/uploads/2017/01/belazy.png" alt="" width="246" height="69">
         <noscript>
          <img class="alignnone size-full wp-image-178" style="float: right;" title="BLazy.js Lazy Image Loading JavaScript" src="http://blog.dynamicdrive.com/wp-content/uploads/2017/01/belazy.png" alt="" width="246" height="69">
         </noscript>At 1.6kb minified, BLazy is slightly larger than Echo.js, though you get a lot of added features from those extra bytes. Apart from being able to lazy load images and background images, you can also serve up different lazy images based on device sizes, such as one image for 480px or less devices, and another for 720px or less etc. You can also load high resolution images for retina display capable devices. And finally, lazy loading inside scrollable containers also works. An example basic implementation:</p> 
        <pre><code>&amp;lt;!- image src points to a base64 transparent gif to start --&amp;gt;
&lt;img class="b-lazy" src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==" alt="alt-text" data-src="image.jpg" /&gt;

&lt;script src="blazy.js"&gt;&lt;/script&gt;
&lt;script&gt;
    // Initialize
    var bLazy = new Blazy();
&lt;/script&gt;</code></pre> 
        <div class="summarybox"> 
         <h3>In Summary:</h3> 
         <p><strong>Pros of this technique:</strong></p> 
         <ul> 
          <li>A manageable 1.6kb in size minified</li> 
          <li>No Library dependencies such as jQuery</li> 
          <li>Apart from lazy loading images, can also serve up different images based on device width and for retina devices.</li> 
          <li>Works with images inside a scrollable container</li> 
         </ul> 
         <p><strong>Cons:</strong></p> 
         <ul> 
          <li>No built in “loading” or image transition when revealing the images, though you can easily implement one yourself using the callback function or by defining a specific CSS class containing the CSS3 transition effects to be applied.</li> 
         </ul> 
        </div> 
        <h2>#4 <a href="http://jquery.eisbehr.de/lazy/" target="_blank" rel="noopener">jQuery Lazy Image Loading Plugin</a></h2> 
        <p><img class="lazy lazy-hidden alignnone size-full wp-image-180" style="float: right;" title="JQuery Lazy Image Loading Plugin" src="//blog.dynamicdrive.com/wp-content/plugins/a3-lazy-load/assets/images/lazy_placeholder.gif" data-lazy-type="image" data-src="http://blog.dynamicdrive.com/wp-content/uploads/2017/01/jquerylazy.png" alt="" width="300" height="54">
         <noscript>
          <img class="alignnone size-full wp-image-180" style="float: right;" title="JQuery Lazy Image Loading Plugin" src="http://blog.dynamicdrive.com/wp-content/uploads/2017/01/jquerylazy.png" alt="" width="300" height="54">
         </noscript>We now venture into jQuery plugins territory. If your site already uses jQuery, then there’s no reason not to consider a jQuery plugin to lazy load your images; otherwise bear in mind the 32kb hit that jQuery itself (minified) adds to your page (although it most likely will be cached). With that said, <a href="http://jquery.eisbehr.de/lazy/" target="_blank" rel="noopener">jQuery Lazy</a> is a compact jQuery Lazy Image plugin (1.27kb minified) that differentiates from the above scripts in a few ways. Most notable about this script is the ability to lazy load images in both a vertical and horizontal layout, which is useful if you have a scrollable horizontal “carousel of images”, for example. Also, the plugin comes with additional, optional plugins (another 2.27kb in size total) you can add on to lazy load other types of content, such as IFRAME content, JavaScripts, AUDIO and VIDEOS. And finally, there are CDN hosted versions of the plugins available so you don’t have to upload any files to your server. Here’s a sample implementation:</p> 
        <pre><code>&lt;script src="https://ajax.googleapis.com/ajax/libs/jquery/1.9.1/jquery.min.js"&gt;&lt;/script&gt;
&lt;script type="text/javascript" src="http://cdnjs.cloudflare.com/ajax/libs/jquery.lazy/1.7.4/jquery.lazy.min.js"&gt;&lt;/script&gt;

&lt;img class="lazy" src="img/blank.gif" alt="Photo" data-src="http://pipsum.com/800x500.jpg" /&gt;
&lt;img class="lazy" src="img/blank.gif" alt="Photo" data-src="http://pipsum.com/400x500.jpg" /&gt;
&lt;img class="lazy" src="img/blank.gif" alt="Photo" data-src="http://pipsum.com/700x500.jpg" /&gt;

&lt;script&gt;
    $(function() {
        $('.lazy').lazy();
    });
&lt;/script&gt;</code></pre> 
        <div class="summarybox"> 
         <h3>In Summary:</h3> 
         <p><strong>Pros of this technique:</strong></p> 
         <ul> 
          <li>Supports CDN hosting so you don’t have to upload any files to your server</li> 
          <li>Lazy loads images in both vertical and horizontal layouts, plus inside scrollable containers</li> 
          <li>Supports a loading animation and simple jQuery transitions such as Fade In out of the box</li> 
          <li>With an additional plugin can lazy load other media types such as IFRAME content, videos etc</li> 
         </ul> 
         <p><strong>Cons:</strong></p> 
         <ul> 
          <li>Requires jQuery as a dependency</li> 
         </ul> 
        </div> 
        <h2>#5 <a href="https://github.com/ressio/lazy-load-xt" target="_blank" rel="noopener">Lazy Load XT jQuery plugin</a></h2> 
        <p><img class="lazy lazy-hidden alignnone size-full wp-image-182" style="float: right;" src="//blog.dynamicdrive.com/wp-content/plugins/a3-lazy-load/assets/images/lazy_placeholder.gif" data-lazy-type="image" data-src="http://blog.dynamicdrive.com/wp-content/uploads/2017/01/lazyloadxt.png" alt="" width="250" height="72">
         <noscript>
          <img class="alignnone size-full wp-image-182" style="float: right;" src="http://blog.dynamicdrive.com/wp-content/uploads/2017/01/lazyloadxt.png" alt="" width="250" height="72">
         </noscript>Last but certainly not least, we arrive at <a href="https://github.com/ressio/lazy-load-xt" target="_blank" rel="noopener">Lazy Load XT</a>. Get past the unusual name, and herein lies a lazy loading plugin packed to the rim with features, by way of add-ons (not to be confused with additional plugins). The stock script (at around 3kb minified) already supports lazy loading images inside scrollable containers, both vertical and horizontal layouts, and “infnite scroll” set ups. Where things really get interesting is when you throw in an add on (either another .js or .css file). Want to show a default animated spinner while images are being loaded? Just reference another .css file. Want a fancy transition to introduce the lazily loaded image with style? <a href="https://github.com/daneden/animate.css" target="_blank" rel="noopener">Animate.css</a> works with Lazy Load XT to provide a myriad of effects. How about serve up different images based on device size? The “response image” add-on will enable that. As always, here’s an example basic implementation:</p> 
        <pre><code>&lt;script src="https://ajax.googleapis.com/ajax/libs/jquery/1.12.4/jquery.min.js"&gt;&lt;/script&gt;
&lt;script src="//cdnjs.cloudflare.com/ajax/libs/jquery.lazyloadxt/1.0.0/jquery.lazyloadxt.min.js"&gt;&lt;/script&gt;

&lt;img src="img/blank.gif" alt="Photo" width="800px" height="500px" data-src="http://pipsum.com/800x500.jpg" /&gt;
&lt;img src="img/blank.gif" alt="Photo" width="800px" height="500px" data-src="http://pipsum.com/800x500.jpg" /&gt;
&lt;img src="img/blank.gif" alt="Photo" width="800px" height="500px" data-src="http://pipsum.com/800x500.jpg" /&gt;</code></pre> 
        <div class="summarybox"> 
         <h3>In Summary:</h3> 
         <p><strong>Pros of this technique:</strong></p> 
         <ul> 
          <li>Supports CDN hosting of the core and add on files, so you don’t have to upload any files to your server</li> 
          <li>Lazy loads images in scrollable containers, horizontal and infinite scroll layouts</li> 
          <li>Supports a loading animation and Animate.css transitions such as Bounce In via “add-ons”</li> 
          <li>With an additional add-on can also lazy load other media types such as IFRAME content, videos etc</li> 
         </ul> 
         <p><strong>Cons:</strong></p> 
         <ul> 
          <li>Requires jQuery 1.7+ as a dependency</li> 
          <li>Depending on the features you need, you may need to reference multiple “add-on” files.</li> 
         </ul> 
        </div> 
        <h2>#6 <a href="https://en-ca.wordpress.org/plugins/a3-lazy-load/" target="_blank" rel="noopener">a3 Lazy Load WordPress Plugin</a> <span style="color: #ff0000;">Bonus</span></h2> 
        <p>A lot of you have asked me for a good image lazy load plugin for WordPress, and for those users, I definitely recommend <a href="https://en-ca.wordpress.org/plugins/a3-lazy-load/" target="_blank" rel="noopener">a3 Lazy Load</a>. It’s what I’m using on this blog, and it works great out of the box. It’s free, lightweight, and doesn’t require any configuration to get up and running. What else can you ask for?</p> 
        <p>To install it, search for the plugin by name “<strong>a3 lazy load</strong>” inside your WordPress dashboard, or click on the link above to manually download it first.</p> 
        <h2>Conclusion</h2> 
        <p>With images constituting the biggest factor bogging down webpages’ download times, lazy loading images is an approach that should pay dividends in spades for many a sites. The 5 solutions we just covered provides you with a stepping stone of increasingly feature rich, plug-and-play lazy loading solutions to cover all the basis. Enjoy!</p>
