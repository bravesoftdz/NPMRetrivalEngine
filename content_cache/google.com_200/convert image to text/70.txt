      <div class="l-grid-10 m-grid-9 s-grid12 content-column">
       <h1 class="">Picking the Right MIME Type for JavaScript</h1>
      </div> 
     </div> 
     <div class="no-overflow row"> 
      <div class="l-grid-2 m-grid-3 s-grid-12"> 
       <div class="article-info"> 
        <p class="author-name">Ish Sookun</p> 
        <p class="timestamp">January 18, 2016</p> 
       </div> 
       <div class="edited-date"> 
        <p class="no-bottom-margin">Updated</p> 
        <p>January 22, 2016</p> 
       </div> 
      </div> 
      <div class="l-grid-10 m-grid-9 s-grid-12 content-column post-content"> 
       <p>MIME stands for Multi-purpose Internet Mail Extensions. Though, the use of the word MIME has evolved to encompass data transactions outside the realm of web mail. Back in the day, MIME types simply defined content delivered between mailing servers and clients (Outlook) for <em>1-on-1 communication</em>. But now they are used by clients like browsers (Chrome) to interpret <em>global communication</em>.</p> 
       <div class="orange-box">
        <strong>NOTE:</strong> MIME types are also called media types and content types.
       </div> 
       <h2>A Simple Explanation of MIME Types</h2> 
       <p>A MIME type consists of two parts. The first part defines what web content is at a high level (image, video, application, etc.); the second part defines what web content is at a more detailed level (png, avi, javascript, etc.). This two-part definition is needed so clients like web browsers can translate content that servers give them into terms users like us can experience.</p> 
       <p>Here are some basic examples of how MIME types are displayed:</p> 
       <ul> 
        <li><code>image/png</code></li> 
        <li><code>video/avi</code></li> 
        <li><code>application/javascript</code></li> 
       </ul> 
       <p>Using the last example: The MIME value of <code>application/javascript</code> is for identifying JavaScript. The “type” is <code>application</code> and the “subtype” is <code>javascript</code>. We’ll get into more terminology and details below, but first let’s go over why we’re bringing MIME types up in the first place.</p> 
       <h2>Why We’re Talking About MIME Types</h2> 
       <p>In <a href="https://github.com/MaxCDN/bootstrap-cdn/pull/603" target="_blank">November 2015</a>, MaxCDN’s Justin Dorfman and his partner Josh Mervine ran into a dilemma when rewriting <a href="https://github.com/MaxCDN/bootstrap-cdn/blob/develop/tests/test_helper.js#L10" target="_blank"><code>content-type</code> tests</a> for <a href="http://www.bootstrapcdn.com/" target="_blank">BootsrapCDN</a>, an open source CDN powered by MaxCDN that delivers JavaScript, CSS, and Fonts for open source projects.</p> 
       <p>The dilemma came about because there is currently more than one variation of MIME type for JavaScript. And as a CDN trying to stay on top of new best practices, the BootstrapCDN team had to find out what the best practice was for naming JavaScript MIME types. After some research, they found that the standard is registered as <span style="background-color: #FFFF00"><code>application/javascript</code></span> by the Internet Engineering Task Force (IETF).</p> 
       <p>Some JavaScript MIME types considered “obsolete” by the IETF include:</p> 
       <ul> 
        <li><code>text/javascript</code></li> 
        <li><code>text/x-javascript</code></li> 
        <li><code>application/x-javascript</code></li> 
       </ul> 
       <p>Although the IETF only considers <code>application/javascript</code> as the standard MIME type for JavaScript, variations do need to be accepted. For instance, Google uses <code>text/javascript</code>. Regardless, Google web properties still continue to function. So even though using the standard MIME type doesn’t really matter, it does, which is why we’re talking about MIME types. </p> 
       <p><img src="https://www.maxcdn.com/one/media/how-standards-proliferate.png" alt="a comical sketch on how web standards proliferate"><br> <small>Source: <a href="https://xkcd.com/927/" target="_blank">xkcd</a></small></p> 
       <h2>A Technical Explanation of MIME Types</h2> 
       <p>Most operating systems like Microsoft Windows, Mac OS X, or the various Linux distributions identify file types through file name extensions – usually the three characters following the dot ending a file name (example: .doc).</p> 
       <p>Likewise, an operating system associates applications that should read and edit certain file types. For example, in Microsoft Windows we typically would see Microsoft Word opening a file ending with the .doc extension (<code>application/msword</code>), and an image viewer would open a file with the .jpeg extension (<code>image/jpeg</code>).</p> 
       <div class="orange-box">
        <strong>NOTE:</strong> By default, Microsoft Windows hides file name extensions from the user’s view for convenience. They can be enabled by following instructions in 
        <a href="http://windows.microsoft.com/en-us/windows/show-hide-file-name-extensions#show-hide-file-name-extensions=windows-7" target="_blank">this how-to</a> by Microsoft.
       </div> 
       <p>The equivalence of “filename extensions” on the Internet is provided by MIME types. They tell web browsers and email clients what type of files they are dealing with so the application knows how to process the file. Just the way operating systems associate default programs and actions to file name extensions, applications on the Internet make use of the MIME types. </p> 
       <p>MIME types are defined in two parts, having a “type” and a “sub-type” as described in the first section of this article. If you look at the HTTP header of a PNG image, you’ll see the MIME type defined in the <code>Content-Type</code> field. Here is the HTTP header of an image file from the MaxCDN Blog:</p> 
       <pre>HTTP/1.1 200 OK
Date: Sun, 10 Jan 2016 12:56:04 GMT
<span style="background-color: #FFFF00">Content-Type: image/png</span>
Content-Length: 2075</pre> 
       <p>The MIME type is <code>image/png</code>. That tells the browser the resource is of type “image” and sub-type “png”, which we usually refer to as the image format. The browser can then decide on how to display the image, whether it can display or does not have the required technology to display such type of image. This is more common with audio and video types.</p> 
       <p>The ability to categorize resources in types and sub-types gives applications a unique advantage to provision when displaying web content. For example, if a browser receives a file from a web server of type <code>audio/xyz</code>, it might not have the required plugin to play. But because the browser at least understands it’s an audio file, it can prompt the user accordingly.</p> 
       <p>The <code>Content-Type</code> field may carry additional information called “arguments” and there may be several arguments for the MIME type. One example of an argument is the character-set information for the <code>text/html</code> type. Here is a part of the HTTP header from the MaxCDN Blog showing the <code>Content-Type</code> field:</p> 
       <pre>HTTP/1.1 200 OK
Date: Sun, 10 Jan 2016 13:07:47 GMT
<span style="background-color: #FFFF00">Content-Type: text/html; charset=UTF-8</span></pre> 
       <p>The <code>Content-Type</code> field accepts several arguments and their order is not important. In the above header extract, the argument is “charset=UTF-8”. The browser thus interprets the resource as text of sub-type <code>html</code> and the character set required is <code>UTF-8</code>.</p> 
       <p>There is an <a href="http://www.iana.org/assignments/media-types/media-types.xhtml" target="_blank">official list of MIME Types</a> published by the Internet Assigned Names and Numbers Association (IANA), which is regularly updated. The list contains MIME types that have been reviewed by the association.</p> 
       <h4>Experimental MIME Types</h4> 
       <p>Applications sometimes would denote MIME types that aren’t “official,” or experimental, by prefixing the sub-type with “x-”. For example, for some time the JavaScript MIME Type <code>application/x-javascript</code> was referred to as <code>text/javascript</code> until <a href="http://tools.ietf.org/html/rfc4329" target="_blank">RFC 4329</a> rendered <code>text/javascript</code> as obsolete (giving it the official status of <code>application/javascript</code>).</p> 
       <h2>MIME Type Resources</h2> 
       <p>Technical standards by the IETF have lengthily described the MIME format. I recommend that Internet geeks be familiar with the following RFCs:</p> 
       <ul> 
        <li><a href="https://www.ietf.org/rfc/rfc2045.txt" target="_blank">RFC 2045 – MIME</a>, Format of Internet Message Bodies</li> 
        <li><a href="https://www.ietf.org/rfc/rfc2046.txt" target="_blank">RFC 2046 – MIME</a>, Media Types</li> 
        <li><a href="https://www.ietf.org/rfc/rfc2047.txt" target="_blank">RFC 2047 – MIME</a>, Message Header Extensions for Non-ASCII Text</li> 
        <li><a href="https://www.ietf.org/rfc/rfc4289.txt" target="_blank">RFC 4289 – MIME</a>, Registration Procedures</li> 
        <li><a href="https://www.ietf.org/rfc/rfc2049.txt" target="_blank">RFC 2049 – MIME</a>, Conformance Criteria and Examples</li> 
       </ul> 
       <p>And here are some MIME types lists:</p> 
       <ul> 
        <li><a href="http://www.freeformatter.com/mime-types-list.html" target="_blank">MIME Types List</a> (Free for Matter)</li> 
        <li><a href="http://www.sitepoint.com/web-foundations/mime-types-complete-list/" target="_blank">The Complete List of MIME Types</a> (Sitepoint)</li> 
       </ul>
