        <h2 id="description-header">Description</h2> 
        <p>Mammoth is designed to convert .docx documents, such as those created by Microsoft Word, and convert them to HTML. Mammoth aims to produce simple and clean HTML by using semantic information in the document, and ignoring other details. For instance, Mammoth converts any paragraph with the style <code>Heading1</code> to <code>h1</code> elements, rather than attempting to exactly copy the styling (font, text size, colour, etc.) of the heading. This allows you to paste from Word documents without the usual mess.</p> 
        <p>There’s a large mismatch between the structure used by .docx and the structure of HTML, meaning that the conversion is unlikely to be perfect for more complicated documents. Mammoth works best if you only use styles to semantically mark up your document.</p> 
        <p>The following features are currently supported:</p> 
        <ul> 
         <li> <p>Headings.</p> </li> 
         <li> <p>Lists.</p> </li> 
         <li> <p>Tables. The formatting of the table itself, such as borders, is currently ignored, but the formatting of the text is treated the same as in the rest of the document.</p> </li> 
         <li> <p>Footnotes and endnotes.</p> </li> 
         <li> <p>Images.</p> </li> 
         <li> <p>Bold, italics, superscript and subscript.</p> </li> 
         <li> <p>Links.</p> </li> 
         <li> <p>Text boxes. The contents of the text box are treated as a separate paragraph that appears after the paragraph containing the text box.</p> </li> 
        </ul> 
        <h4>Embedded style maps</h4> 
        <p>By default, Mammoth maps some common .docx styles to HTML elements. For instance, a paragraph with the style name <code>Heading 1</code> is converted to a <code>h1</code> element. If you have a document with your own custom styles, you can use an embedded style map to tell Mammoth how those styles should be mapped. For instance, you could convert paragraphs with the style named <code>WarningHeading</code> to <code>h1</code> elements with <code>class="warning"</code> with the style mapping:</p> 
        <pre><code>p[style-name='WarningHeading'] =&gt; h1.warning:fresh
</code></pre> 
        <p><a href="http://mike.zwobble.org/projects/mammoth/embed-style-map/" rel="nofollow">An online tool</a> can be used to embed style maps into an existing document. Details of <a href="https://github.com/mwilliamson/mammoth.js#writing-style-maps" rel="nofollow">how to write style maps can be found on the mammoth.js documentation</a>.</p>

        <ul> 
         <li> <p>Allow MAMMOTH_OPTIONS to override idPrefix.</p> </li> 
         <li> <p>Update mammoth.js to 1.4.0. This includes improved handling of hyperlinks, and converts table headers into thead elements.</p> </li> 
        </ul>

        <ul> 
         <li> <p>Handle unsuccessful image uploads where the HTTP request succeeds, but WordPress rejects the file. Fixes an issue where documents with EMF images couldn’t be imported.</p> </li> 
         <li> <p>Update mammoth.js to 1.3.2. This includes a fix for documents where images are referenced by a URI relative to the base URI.</p> </li> 
        </ul>

        <ul> 
         <li> <p>Update mammoth.js to 1.3.1. This includes new ways to map styles, such as style name prefixes.</p> </li> 
         <li> <p>Improve styling of preview to match the editor.</p> </li> 
         <li> <p>Fix a bug where images wouldn’t upload on certain server configurations.</p> </li> 
         <li> <p>Allow options to be passed to mammoth.js through a MAMMOTH_OPTIONS global variable.</p> </li> 
        </ul> 
        <h4>1.3.0</h4> 
        <ul> 
         <li>Update mammoth.js to 1.2.5. This includes better support for image alt text and boolean run properties (bold, italic, underline and strikethrough).</li> 
        </ul> 
        <h4>1.2.0</h4> 
        <ul> 
         <li> <p>Include wp-image-* class when inserting images. This allows the WordPress editor to correctly identify the image and show appropriate editing options.</p> </li> 
         <li> <p>If an image has an alt text description in the original document, set the alt text in the media library when uploading that image.</p> </li> 
         <li> <p>If an image has an alt text description in the original document, use it to generate the filename.</p> </li> 
         <li> <p>Set image filename extension based on the image content type.</p> </li> 
         <li> <p>Show a message while the document is being inserted.</p> </li> 
        </ul> 
        <h4>1.1.0</h4> 
        <ul> 
         <li>Update mammoth.js to 1.1.0. This includes support for merged table cells and content controls, such as bibliographies. This should also improve performance when converting larger documents.</li> 
        </ul> 
        <h4>1.0.0</h4> 
        <ul> 
         <li>Update mammoth.js to 0.3.33. This includes better support for reading documents that use undefined styles, and generates simpler HTML in some cases.</li> 
        </ul> 
        <h4>0.1.25</h4> 
        <ul> 
         <li> <p>Update mammoth.js to 0.3.30. This includes better support for lists made in LibreOffice.</p> </li> 
         <li> <p>Fix JavaScript error on admin pages without editors.</p> </li> 
        </ul> 
        <h4>0.1.24</h4> 
        <ul> 
         <li>Update mammoth.js to 0.3.29. This improves support for mc:AlternateContent elements.</li> 
        </ul> 
        <h4>0.1.23</h4> 
        <ul> 
         <li>Update mammoth.js to 0.3.28. This improves support for reading images.</li> 
        </ul> 
        <h4>0.1.22</h4> 
        <ul> 
         <li>Update mammoth.js to 0.3.28-pre.1. Fixes newlines being inserted around inline elements when the editor is in text mode.</li> 
        </ul> 
        <h4>0.1.21</h4> 
        <ul> 
         <li>Update mammoth.js to 0.3.27. Fixes recursive collapsing of HTML elements.</li> 
        </ul> 
        <h4>0.1.20</h4> 
        <ul> 
         <li>Update mammoth.js to 0.3.26. Improves the collapsing of HTML elements, such as allowing collapsing elements generated by different runs.</li> 
        </ul>
