</code></pre> 
         <p>An <code>.apk</code> file should appear at <code>$HOME/watson-calorie-counter/mobile/platforms/android/build/outputs/apk/android-debug.apk</code>, which contains the Android application.</p> 
         <p>You can then either manually transfer the <code>.apk</code> to your device and run it yourself, or if your device is tethered (as described in the previous step), then you can run:</p> 
         <pre><code>$ cordova run android
</code></pre> 
         <p>However, a Docker container does not have access to your host's USB devices, unless you explicitly allow them to be passed through. You can expose your device to the Docker container, and allow Cordova to do the transfer for you. To accomplish that, you'll need to know which USB device to pass through. Discover your USB devices with <code>lsusb</code>.</p> 
         <p>For example, in this case, I know that my Android device is <code>Bus 001, Device 002</code>:</p> 
         <pre><code>$ lsusb
Bus 002 Device 001: ID 1d6b:0003 Linux Foundation 3.0 root hub
Bus 001 Device 003: ID 046d:082d Logitech, Inc. HD Pro Webcam C920
Bus 001 Device 002: ID 18d1:4ee6 Google Inc.
Bus 001 Device 005: ID 046d:c085 Logitech, Inc.
Bus 001 Device 004: ID 045e:02e6 Microsoft Corp.
Bus 001 Device 001: ID 1d6b:0002 Linux Foundation 2.0 root hub
</code></pre> 
         <p>I can then pass my device through to the container using <code>--device=/dev/bus/usb/&lt;bus-number&gt;/&lt;device-number&gt;</code> and allow Cordova to access it. The complete Docker command would then be:</p> 
         <pre><code>$ docker run \
  --volume="$HOME/watson-calorie-counter/mobile:/mobile" \
  --device=/dev/bus/usb/001/002 \
  calorie-counter \
  cordova run android
</code></pre> 
         <p>At this point, the app named <code>Calorie Counter</code> should be on your mobile device. Use the camera icon to take a photo of a food item, and allow Watson to analyze the image and fetch the calorie results.</p>
