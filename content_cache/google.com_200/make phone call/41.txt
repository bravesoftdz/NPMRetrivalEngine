      <div id="articleBody" class="col-md-12 row"> 
       <p>If you are building an application, eventually at some point, you need a data store. This could be an in-memory data store or a physical database like SQL Server, MySQL, Oracle or even a <a href="http://www.dotnetcurry.com/nodejs/1161/connect-mongodb-nodejs-mongodb-driver">NoSQL database like MongoDB</a> or CouchDB. Node.js applications are no different and require a data store at some point. The Node community as well as vendors have built bridges and drivers to connect to almost any data store out there.</p> 
       <p id="jquery" class="articlead">&nbsp;</p> 
       <p>In this article, we will explore how to <b>use Node.js to connect to SQL Server database</b> and perform read/write operations. There are some packages available to connect to SQL Server database from Node.js. We can grab information about these packages from <a href="http://www.npmjs.com">http://www.npmjs.com</a>.</p> 
       <p>Two popular packages are:</p> 
       <p>· <b>msnodesql - </b>this is Microsoft’s driver for Node.js for SQL Server. It allows Node.js applications on Windows platform and Azure platform to connect to Sql Server and SQL Azure. More information about this package can be read from <a href="https://www.npmjs.com/package/msnodesql">this link</a>.</p> 
       <p>· <b>mssql </b>is a popular third-party easy-to-use SQL Server connector for Node.js. It supports Stored Procedures, Sql Statements, Transactions and Connection Pooling. This package uses <b>Promises </b>to perform database operations and Callback mechanisms too. More information about this package can be read from <a href="https://www.npmjs.com/package/mssql">this link</a>.</p> 
       <p>Since mssql is production ready, we will use this in our application. The article uses a local instance of SQL Server database with Mixed Mode authentication. Before implementing the application, we will make sure that the following Sql environment configuration are done successfully.</p> 
       <h3>Features of mssql package</h3> 
       <p>Here are some features of mssql package as mentioned in its homepage</p> 
       <ul> 
        <li>unified interface for multiple TDS drivers.</li> 
        <li>built-in connection pooling.</li> 
        <li>Support for built-in JSON serialization (SQL Server 2016) as well as serialization of Geography and Geometry CLR types</li> 
        <li>Support for Stored Procedures, Transactions, Prepared Statements, Bulk Load and Table-Valued Parameters (TVP).</li> 
        <li>Mapping of JS data type to SQL data type.</li> 
        <li>Support for Promises, Streams and standard callbacks.</li> 
        <li>Production ready and well documented.</li> 
       </ul> 
       <p><b>Note 1:</b> mssql is a wrapper around other clients for MS-SQL like <b>tedious. It uses tedious under the hood, </b>is simpler to use, and contains additional functionality like connection pooling.</p> 
       <p><b>Note 2:</b> Since we are using SQL Server, we will be using a Windows environment. I am assuming you have already installed SQL Server on your machine.</p> 
       <h3>Connecting to SQL Server using Node.js</h3> 
       <p><b>Step 1: </b>Run Services.msc from Start &gt; Run, and make sure that following services are running:</p> 
       <ul> 
        <li>SQL Server</li> 
        <li>SQL Server Agent</li> 
        <li>SQL Server Browser</li> 
       </ul> 
       <p><img title="sql-services" border="0" alt="sql-services" src="http://www.dotnetcurry.com/images/nodejs/Connect-to-SQL-Server-using-Node.js_7CB0/sql-services.png" width="859" height="100"></p> 
       <p><b>Step 2: </b>Open Sql Server Configuration Manager, and enable TCP/IP protocol from SQL Server Network Configuration as shown in the following image</p> 
       <p><img title="sql-server-network-config" border="0" alt="sql-server-network-config" src="http://www.dotnetcurry.com/images/nodejs/Connect-to-SQL-Server-using-Node.js_7CB0/sql-server-network-config.png" width="1153" height="230"></p> 
       <p>This application is implemented using <a href="http://bitly.com/dncvscode">Visual Studio Code</a>. This is a free new IDE for building and debugging modern Web and Cloud applications. This IDE can be downloaded from <a href="http://bitly.com/dncvscode">here</a>. This application uses Node.js tools. They can be downloaded from <a href="https://github.com/Microsoft/nodejstools#readme">here</a>.</p> 
       <p><b>Step 3: </b>For this application, we will be using a fictional Company Database with Employee and EmployeeInfo tables. The script for Database and Tables are as following:</p>

Create database Company</pre>

       <p><b>Step 5: </b>In the app.js, add the following code:</p> 
       <pre class="brush: js;">
//1.
var sql = require('mssql');
//2.
var config = {
    server: 'localhost',
    database: 'Company',
    user: 'sa',
    password: 'sa',
    port: 1433
};
//3.
function loadEmployees() {
    //4.
    var dbConn = new sql.Connection(config);
    //5.
    dbConn.connect().then(function () {
        //6.
        var request = new sql.Request(dbConn);
        //7.
        request.query("select * from EmployeeInfo").then(function (recordSet) {
            console.log(recordSet);
            dbConn.close();
        }).catch(function (err) {
            //8.
            console.log(err);
            dbConn.close();
        });
    }).catch(function (err) {
        //9.
        console.log(err);
    });
}
//10.
loadEmployees();</pre> 
       <p>The above code does the following:</p> 
       <p>(Note: Following numbering matches with Comment numbers given for the above code.)</p> 
       <p>1. Load the <b>mssql </b>module so that Sql server can be connected for performing Read/Write operations.</p> 
       <p>2. The JavaScript <b>config </b>object with Sql Server connection properties e.g. server name, database name, user name, password and port. In this application, we are connecting to the localhost instance. localhost can be replaced by IP address or a Sql Server named instance. The default port 1433 must be set.</p> 
       <p>3. The <b>loadEmployees () </b>function. This is used to read all employees from EmployeeInfo table of the Company Database.</p> 
       <p>4. A Connection instance is created of the name <b>dbConn </b>using <b>Connection() </b>function of the sql object defined in Line 1. The <b>config </b>object is passed to this function.</p> 
       <p>5. The <b>connect() </b>function of the dbConn instance uses the promise object, (this is provided by the mssql package), to make sure that the Read/Write operation with Sql Server can be performed only when the connection is successfully established.</p> 
       <p>6. If the connection is successful, then the <b>Request </b>object is created over the Connection. This object is used to query the table.</p> 
       <p>7. The <b>query () </b>function is called by passing a Sql Statement to in (in this is case it is SELECT statement), this again returns the <b>promise </b>object. If the query is successful, then data is returned using <b>recordSet </b>object.</p> 
       <p>8. If there is problem with query then <b>catch() </b>function will be executed with error.</p> 
       <p>9. If the connection fails with the database then <b>catch() </b>function will be executed with an error.</p> 
       <p>10. Call the <b>loadEmployees() </b>function.</p>

       <p><b>Step 7: </b>In app.js, add the following function</p> 
       <pre class="brush: js;">
//1
function executeStoredProc() {
    //2. 
    var dbConn = new sql.Connection(config);
    dbConn.connect().then(function () {
        
        //3.
        var request = new sql.Request(dbConn);
        request.input('Salary', sql.Int, 50000)
        .execute("GetAllEmployeesBySalary").then(function (recordSet) {
            //4.
            console.log(recordSet);
            dbConn.close();
        }).catch(function (err) {
            //5.
            console.log(err);
            dbConn.close();
        });
    }).catch(function (err) {
        //6.
        console.log(err);
    });
}
//7.
executeStoredProc();</pre> 
       <p>The above code does the following:</p> 
       <p>(Note: Following numbering matches with Comment numbers given for the above code.)</p> 
       <p>1. The function executeStoredProc () contains logic executing stored procedure on Sql Server.</p> 
       <p>2. The <b>dbConn</b> is the connection instance using the <b>Connection() </b>function of the <b>sql </b>object of the <b>mssql </b>package. This function accepts the <b>config </b>JavaScript object which contains Sql Server information.</p> 
       <p>3. The <b>Request </b>object is created using the <b>dbConn </b>instance of the Connection object. This is used to define the input parameters to stored procedure using <b>input() </b>function. This function accepts three parameters representing parameter name (Salary), parameter type (sql.Int) and value (50000). The request object is used to execute stored procedure using <b>execute() </b>function. This function accept the stored procedure name (GetAllEmployeesBySalary). This function returns the promise object.</p> 
       <p>4. If the stored procedure is executed successfully then the recordSet parameter will be returned and will be printed on console.</p> 
       <p>5. If the stored procedure fails then <b>catch() </b>will be executed with error.</p> 
       <p>6. If the connection with database fails, then <b>catch() </b>will be executed with the error details.</p> 
       <p>7. Call the executeStoredProc() function.</p>

       <h3>Inserting a New Row in the Table</h3> 
       <p><b>Step 9: </b>In app.js, add the following code</p> 
       <pre class="brush: js;">
//1.
function insertRow() {
    //2.
    var dbConn = new sql.Connection(config);
    //3.
    dbConn.connect().then(function () {
        //4.
        var transaction = new sql.Transaction(dbConn);
        //5.
        transaction.begin().then(function () {
            //6.
            var request = new sql.Request(transaction);
            //7.
            request.query("Insert into EmployeeInfo (EmpName,Salary,DeptName,Designation) values ('T.M. Sabnis',13000,'Accounts','Lead')")
        .then(function () {
                //8.
                transaction.commit().then(function (recordSet) {
                    console.log(recordSet);
                    dbConn.close();
                }).catch(function (err) {
                    //9.
                    console.log("Error in Transaction Commit " + err);
                    dbConn.close();
                });
            }).catch(function (err) {
                //10.
                console.log("Error in Transaction Begin " + err);
                dbConn.close();
            });
            
        }).catch(function (err) {
            //11.
            console.log(err);
            dbConn.close();
        });
    }).catch(function (err) {
        //12.
        console.log(err);
    });
}
//13.
insertRow();</pre> 
       <p>The above code does the following:</p> 
       <p>(Note: Following numbering matches with Comment numbers given for the above code.)</p> 
       <p>1. The <b>insertRow() </b>function contains logic for performing an insert operation with Sql Server.</p> 
       <p>2. The Connection instance of name <b>dbConn</b> is declared using <b>Connection() </b>function of the sql object. sql is an instance of the mssql package. The Connection() function accepts the <b>config </b>object. This object defines the Sql Server information.</p> 
       <p>3. The <b>connect() </b>function call returns a promise object.</p> 
       <p>4. If the database connection is successful, then the <b>Transaction </b>object is created using dbConn.</p> 
       <p>5. The Transaction <b>begin() </b>function is used to start the transaction.</p> 
       <p>6. The Request object is created over the transaction.</p> 
       <p>7. The <b>query() </b>function of the request object accepts a SQL query (in this case it is insert query).</p> 
       <p>8. If the query is executed, then the transaction <b>commit() </b>function is called which will commit INSERT on the table specified in the insert query.</p> 
       <p>9. If commit fails, then <b>catch() </b>is executed with an error.</p> 
       <p>10. If query fails, then <b>catch() </b>is executed with an error.</p> 
       <p>11. If transaction is not initiated, then <b>catch() </b>is executed with an error.</p> 
       <p>12. If database connection fails then <b>catch() </b>is executed with an error.</p> 
       <p>13. Call the insertRow () function.</p>
