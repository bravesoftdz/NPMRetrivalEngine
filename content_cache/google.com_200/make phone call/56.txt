           </svg></a>Browser Calls</h1> 
         <p><a href="https://travis-ci.org/TwilioDevEd/browser-calls-node"><img src="https://camo.githubusercontent.com/c074294e7a37e7e1a34ef6784b8739df1fbb5671/68747470733a2f2f7472617669732d63692e6f72672f5477696c696f44657645642f62726f777365722d63616c6c732d6e6f64652e7376673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.org/TwilioDevEd/browser-calls-node.svg?branch=master" style="max-width:100%;"></a></p> 
         <p>Twilio Client allows your users to make and receive phone calls in their browsers. This tutorial will show you the front-end and backend code necessary to make browser-to-phone and browser-to-browser calls with Twilio Client.</p> 
         <p><a href="https://www.twilio.com/docs/tutorials/walkthrough/browser-calls/node/express">See step-by-step tutorial.</a></p> 
         <h3><a href="#create-a-twiml-app" aria-hidden="true" class="anchor" id="user-content-create-a-twiml-app">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Create a TwiML App</h3> 
         <p>This project is configured to use a <strong>TwiML App</strong>, which allows us to easily set the voice URLs for all Twilio phone numbers we purchase in this app.</p> 
         <p><a href="https://www.twilio.com/console/phone-numbers/dev-tools/twiml-apps/add">Create a new TwiML app</a> and use its <code>Sid</code> as the <code>TWILIO_APP_SID</code> environment variable wherever you run this app.</p> 
         <p><a href="https://camo.githubusercontent.com/7a88d54e62dd74bb8cd5e79ee8f1868b2d7f7c55/687474703a2f2f686f77746f646f63732e73332e616d617a6f6e6177732e636f6d2f63616c6c2d747261636b696e672d7477696d6c2d6170702e676966" target="_blank"><img src="https://camo.githubusercontent.com/7a88d54e62dd74bb8cd5e79ee8f1868b2d7f7c55/687474703a2f2f686f77746f646f63732e73332e616d617a6f6e6177732e636f6d2f63616c6c2d747261636b696e672d7477696d6c2d6170702e676966" alt="Creating a TwiML App" data-canonical-src="http://howtodocs.s3.amazonaws.com/call-tracking-twiml-app.gif" style="max-width:100%;"></a></p> 
         <p>See the end of the "Local development" section for details on the exact URL to use in your TwiML app.</p> 
         <p>Once you have created your TwiML app, <a href="https://www.twilio.com/help/faq/twilio-client/how-do-i-create-a-twiml-app">configure your Twilio phone number</a>. If you don't have a Twilio phone number yet, you can purchase a new number in your <a href="https://www.twilio.com/console/phone-numbers/search">Twilio Account Dashboard</a>.</p>

           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Local development</h2> 
         <p>First you need to install</p> 
         <ul> 
          <li><a href="http://nodejs.org/">Node.js</a> which should also install <a href="https://www.npmjs.com/">npm</a>.</li> 
          <li><a href="https://www.mongodb.org/">MongoDB</a></li> 
         </ul> 
         <ol> 
          <li> <p>First clone this repository and <code>cd</code> into its directory:</p> 
           <div class="highlight highlight-source-shell">
            <pre>git clone git@github.com:TwilioDevEd/browser-calls-node.git

<span class="pl-c1">cd</span> browser-calls-node</pre>
           </div> </li> 
          <li> <p>Install dependencies:</p> <pre><code>npm install
</code></pre> </li> 
          <li> <p>Copy the sample configuration file and edit it to match your configuration</p> 
           <div class="highlight highlight-source-shell">
            <pre>$ cp .env.example .env</pre>
           </div> <p>You can find your <code>TWILIO_ACCOUNT_SID</code> and <code>TWILIO_AUTH_TOKEN</code> in your <a href="https://www.twilio.com/console">Twilio Account Settings</a>. You will also need a <code>TWILIO_PHONE_NUMBER</code>, which you may find <a href="https://www.twilio.com/console/phone-numbers/incoming">here</a>.</p> <p>Run <code>source .env.local</code> to export the environment variables</p> </li> 
          <li> <p>Run the application.</p> 
           <div class="highlight highlight-source-shell">
            <pre>node ./bin/www</pre>
           </div> <p>Alternatively you might also consider using <a href="https://github.com/remy/nodemon">nodemon</a> for this. It works just like the node command, but automatically restarts your application when you change any source code files.</p> 
           <div class="highlight highlight-source-shell">
            <pre>npm install -g nodemon
nodemon ./bin/www</pre>
           </div> </li> 
          <li> <p>Run the application.</p> 
           <div class="highlight highlight-source-shell">
            <pre>$ npm start</pre>
           </div> </li> 
          <li> <p>To actually forward incoming calls, your development server will need to be publicly accessible. <a href="https://www.twilio.com/blog/2015/09/6-awesome-reasons-to-use-ngrok-when-testing-webhooks.html">We recommend using ngrok to solve this problem</a>.</p> </li> 
          <li> <p>Once you have started ngrok, update your TwiML app's voice URL setting to use your ngrok hostname, so it will look something like this:</p> <p><code>http://88b37ada.ngrok.io/call/connect</code></p> </li> 
         </ol>
