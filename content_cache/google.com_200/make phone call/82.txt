   <h1 id="firstHeading" class="firstHeading" lang="en">Foreign function interface</h1>

      <p>A <b>foreign function interface</b> (<b>FFI</b>) is a mechanism by which a program written in one <a href="/wiki/Programming_language" title="Programming language">programming language</a> can call routines or make use of services written in another.</p> 
      <p></p>

      <h2><span class="mw-headline" id="Naming">Naming</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Foreign_function_interface&amp;action=edit&amp;section=1" title="Edit section: Naming">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>The term comes from the specification for <a href="/wiki/Common_Lisp" title="Common Lisp">Common Lisp</a>, which explicitly refers to the language features for inter-language calls as such;<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> the term is also used officially by the <a href="/wiki/Haskell_(programming_language)" title="Haskell (programming language)">Haskell</a><sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> and <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a> programming languages.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup> Other languages use other terminology: the <a href="/wiki/Ada_programming_language" class="mw-redirect" title="Ada programming language">Ada programming language</a> talks about "<a href="/wiki/Language_binding" title="Language binding">language bindings</a>", while <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> refers to its FFI as the JNI (<a href="/wiki/Java_Native_Interface" title="Java Native Interface">Java Native Interface</a>) or JNA (<a href="/wiki/Java_Native_Access" title="Java Native Access">Java Native Access</a>). Foreign function interface has become generic terminology for mechanisms which provide such services.</p> 
      <h2><span class="mw-headline" id="Operation">Operation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Foreign_function_interface&amp;action=edit&amp;section=2" title="Edit section: Operation">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>The primary function of a FFI is to mate the semantics and <a href="/wiki/Calling_convention" title="Calling convention">calling conventions</a> of one programming language (the <i>host</i> language, or the language which defines the FFI), with the semantics and conventions of another (the <i>guest</i> language). This process must also take into consideration the <a href="/wiki/Runtime_environment" class="mw-redirect" title="Runtime environment">runtime environments</a> and/or <a href="/wiki/Application_binary_interface" title="Application binary interface">application binary interfaces</a> of both. This can be done in several ways:</p> 
      <ul> 
       <li>Requiring that guest-language functions which are to be host-language callable be specified or implemented in a particular way; often using a compatibility library of some sort.</li> 
       <li>Use of a tool to automatically "wrap" guest-language functions with appropriate <a href="/wiki/Glue_code" title="Glue code">glue code</a>, which performs any necessary translation.</li> 
       <li>Use of <a href="/wiki/Wrapper_library" title="Wrapper library">wrapper libraries</a></li> 
       <li>Restricting the set of host language capabilities which can be used cross-language. For example, C++ functions called from C may not (in general) include reference parameters or throw exceptions.</li> 
      </ul> 
      <p>FFIs may be complicated by the following considerations:</p> 
      <ul> 
       <li>If one language supports <a href="/wiki/Garbage_collection_(computer_science)" title="Garbage collection (computer science)">garbage collection</a> (GC) and the other does not; care must be taken that the non-GC language code does nothing to cause GC in the other to fail. In JNI, for example, C code which "holds on to" object references that it receives from Java must "register" this fact with the <a href="/wiki/Java_virtual_machine" title="Java virtual machine">Java runtime environment</a> (JRE); otherwise, Java may delete objects before C has finished with them. (The C code must also explicitly release its link to any such object, as soon as there is no further need, by C, of that object.)</li> 
       <li>Complicated or non-trivial objects or datatypes may be difficult to map from one environment to another.</li> 
       <li>It may not be possible for both languages to maintain references to the same instance of a mutable object, due to the mapping issue above.</li> 
       <li>One or both languages may be running on a <a href="/wiki/Virtual_machine" title="Virtual machine">virtual machine</a> (VM); moreover, if both are, these will probably be different VMs.</li> 
       <li>Cross-language <a href="/wiki/Inheritance_(computer_science)" class="mw-redirect" title="Inheritance (computer science)">inheritance</a> and other differences, such as between <a href="/wiki/Type_system" title="Type system">type systems</a> or between <a href="/wiki/Object_composition" title="Object composition">object-composition models</a>, may be especially difficult.</li> 
      </ul> 
      <h2><span class="mw-headline" id="By_language">By language</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Foreign_function_interface&amp;action=edit&amp;section=3" title="Edit section: By language">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Examples of FFIs include:</p> 
      <ul> 
       <li><a href="/wiki/Ada_(programming_language)" title="Ada (programming language)">Ada</a> language bindings, allowing not only to call foreign functions but also to export its functions and methods to be called from non-Ada code.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup></li> 
       <li><a href="/wiki/C%2B%2B" title="C++">C++</a> has a trivial FFI with <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>, as the languages share a significant common subset. The primary effect of the <tt><a href="/wiki/Extern_%22C%22" class="mw-redirect" title="Extern &quot;C&quot;">extern "C"</a></tt> declaration in C++ is to disable C++ <a href="/wiki/Name_mangling" title="Name mangling">name mangling</a>.</li> 
       <li><a href="/wiki/Compiled_Native_Interface" class="mw-redirect" title="Compiled Native Interface">CNI</a>, alternative to JNI used in the GNU compiler environment.</li> 
       <li><a href="/wiki/D_(programming_language)" title="D (programming language)">D</a> does it the same way as <a href="/wiki/C%2B%2B" title="C++">C++</a> does, with <a href="/wiki/Extern_%22C%22" class="mw-redirect" title="Extern &quot;C&quot;">extern "C"</a> through extern (C++)</li> 
       <li><a href="/wiki/Dynamic_language" class="mw-redirect" title="Dynamic language">Dynamic languages</a>, such as <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, <a href="/wiki/Perl" title="Perl">Perl</a>, <a href="/wiki/Tcl" title="Tcl">Tcl</a>, and <a href="/wiki/Ruby_programming_language" class="mw-redirect" title="Ruby programming language">Ruby</a>, all provide easy access to native code written in C/C++ (or any other language obeying C/C++ calling conventions).</li> 
       <li><a href="/wiki/Factor_(programming_language)" title="Factor (programming language)">Factor</a> has FFIs for <a rel="nofollow" class="external text" href="http://fun-factor.blogspot.com/2007/10/getting-started-with-factor-easy-ffi.html">C</a>, <a rel="nofollow" class="external text" href="http://article.gmane.org/gmane.comp.lang.factor.general/2790">Fortran</a>, <a rel="nofollow" class="external text" href="http://docs.factorcode.org/content/vocab-cocoa.html">Objective-C</a>, and <a rel="nofollow" class="external text" href="http://docs.factorcode.org/content/vocab-windows.com.html">Windows COM</a>; all of these enable importing and calling arbitrary shared libraries dynamically.</li> 
       <li>The FFIs of <a href="/wiki/Common_Lisp" title="Common Lisp">Common Lisp</a> and <a href="/wiki/Haskell_(programming_language)" title="Haskell (programming language)">Haskell</a></li> 
       <li><a href="/wiki/Fortran" title="Fortran">Fortran</a> 2003 has a module ISO_C_BINDING which provides interoperable data types (both intrinsic types and POD structs), interoperable pointers, interoperable global data stores, and mechanisms for calling C from Fortran and for calling Fortran from C.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[5]</a></sup></li> 
       <li><a href="/wiki/Perl_6" title="Perl 6">Perl 6</a> can call <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a>, <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, <a href="/wiki/Perl_(programming_language)" class="mw-redirect" title="Perl (programming language)">Perl 5</a>, <a href="/wiki/Brainfuck_(programming_language)" class="mw-redirect" title="Brainfuck (programming language)">Brainfuck</a>, <a href="/wiki/Lua_(programming_language)" title="Lua (programming language)">Lua</a>, <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>, <a href="/wiki/C%2B%2B" title="C++">C++</a> and Scheme <a href="/wiki/GNU_Guile" title="GNU Guile">Guile</a>/<a href="/wiki/Gambit_(scheme_implementation)" title="Gambit (scheme implementation)">Gambit</a> <sup id="cite_ref-6" class="reference"><a href="#cite_note-6">[6]</a></sup> <sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup></li> 
       <li><a href="/wiki/Go_(programming_language)" title="Go (programming language)">Go</a> can call C code directly via the <code>"C"</code> pseudo-package.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[8]</a></sup></li> 
       <li><a href="/wiki/Google_Web_Toolkit" title="Google Web Toolkit">GWT</a>, in which Java is compiled to JavaScript, has a FFI called JSNI which allows Java source to call arbitrary JavaScript functions, and for JavaScript to call back into Java.</li> 
       <li><a href="/wiki/Java_Native_Interface" title="Java Native Interface">JNI</a>, which provides an interface between <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> and C/C++, the preferred systems languages on most systems where Java is deployed. <a href="/wiki/Java_Native_Access" title="Java Native Access">JNA</a> provides an interface with native libraries without having to write <a href="/wiki/Glue_code" title="Glue code">glue code</a>. Another example is <a rel="nofollow" class="external text" href="https://github.com/jnr/jnr-ffi">JNR</a></li> 
       <li>Python provides the <a rel="nofollow" class="external text" href="https://docs.python.org/3/library/ctypes.html">ctypes</a> and <a rel="nofollow" class="external text" href="https://cffi.readthedocs.io/en/latest/">cffi</a> modules. For example, the ctypes module can load C functions from <a href="/wiki/Shared_library" class="mw-redirect" title="Shared library">shared libraries</a>/<a href="/wiki/Dynamic-link_library" title="Dynamic-link library">DLLs</a> on-the-fly and translate simple data types automatically between Python and C semantics as follows: 
        <div class="mw-highlight mw-content-ltr" dir="ltr"> 
         <pre>
<span class="kn">import</span> <span class="nn">ctypes</span>
<span class="n">libc</span> <span class="o">=</span> <span class="n">ctypes</span><span class="o">.</span><span class="n">CDLL</span><span class="p">(</span> <span class="s1">'/lib/libc.so.6'</span> <span class="p">)</span>   <span class="c1"># under Linux/Unix</span>
<span class="n">t</span> <span class="o">=</span> <span class="n">libc</span><span class="o">.</span><span class="n">time</span><span class="p">(</span><span class="bp">None</span><span class="p">)</span>                      <span class="c1"># equivalent C code: t = time(NULL)</span>
<span class="k">print</span> <span class="n">t</span>
</pre>
        </div> </li> 
       <li><a href="/wiki/P/Invoke" class="mw-redirect" title="P/Invoke">P/Invoke</a>, which provides an interface between the Microsoft <a href="/wiki/Common_Language_Runtime" title="Common Language Runtime">Common Language Runtime</a> and native code.</li> 
       <li><a href="/wiki/Racket_(programming_language)" title="Racket (programming language)">Racket</a> has a native FFI based heavily on macros that enables importing arbitrary shared libraries dynamically.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">[9]</a></sup><sup id="cite_ref-10" class="reference"><a href="#cite_note-10">[10]</a></sup></li> 
       <li><a href="/wiki/Rust_(programming_language)" title="Rust (programming language)">Rust</a> also defines a foreign function interface.<sup id="cite_ref-11" class="reference"><a href="#cite_note-11">[11]</a></sup></li> 
       <li><a href="/wiki/Visual_Basic" title="Visual Basic">Visual Basic</a> has a declarative syntax that allows it to call non-Unicode C functions.</li> 
       <li>One of the bases of the <a href="/wiki/Component_Object_Model" title="Component Object Model">Component Object Model</a> is a common interface format, which natively uses the same types as Visual Basic for strings and arrays.</li> 
       <li>LuaJIT, a <a href="/wiki/Just-in-time_compilation" title="Just-in-time compilation">just-in-time</a> implementation of <a href="/wiki/Lua_(programming_language)" title="Lua (programming language)">Lua</a>, has a FFI that allows "calling external C functions and using C data structures from pure Lua code".<sup id="cite_ref-12" class="reference"><a href="#cite_note-12">[12]</a></sup></li> 
       <li><a href="/wiki/PhoneGap" class="mw-redirect" title="PhoneGap">PhoneGap</a> (was called by the name Apache Callback, but now Apache Cordova) is a platform for building native mobile applications using HTML, CSS and JavaScript. Additionally has FFIs via JavaScript callback functions for access to methods and properties of mobile phone's native features including Accelerometer, Camera (also PhotoLibrary and SavedPhotoAlbum), Compass, Storage (SQL database and localStorage), Notification, Media and Capture (playing and recording or audio and video), File, Contacts (address book), Events, Device and Connection information.<a rel="nofollow" class="external autonumber" href="http://phonegap.com/">[1]</a>,<a rel="nofollow" class="external autonumber" href="http://cordova.apache.org/">[2]</a>.</li> 
       <li><a href="/wiki/Wolfram_Language" title="Wolfram Language">Wolfram Language</a> provides a technology called WSTP (Wolfram Symbolic Transfer Protocol) which enables bidirectional calling of code between other languages with bindings for C++, Java, .NET and other languages.</li> 
      </ul> 
      <p>In addition, many FFIs can be generated automatically: for example, <a href="/wiki/SWIG" title="SWIG">SWIG</a>. However, in the case of an <a href="/wiki/Extension_language" class="mw-redirect" title="Extension language">extension language</a> a semantic inversion of the relationship of guest and host can occur, when a smaller body of extension language is the guest invoking services in the larger body of host language, such as writing a small plugin <sup id="cite_ref-13" class="reference"><a href="#cite_note-13">[13]</a></sup> for GIMP.<sup id="cite_ref-14" class="reference"><a href="#cite_note-14">[14]</a></sup></p> 
      <p>Some FFIs are restricted to free standing <a href="/wiki/Subroutine" title="Subroutine">functions</a>, while others also allow calls of functions embedded in an object or class (often called <a href="/wiki/Method_call" class="mw-redirect" title="Method call">method calls</a>); some even permit migration of complex datatypes and/or objects across the language boundary.</p> 
      <p>In most cases, a FFI is defined by a "higher-level" language, so that it may employ services defined and implemented in a lower level language, typically a systems language like <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a> or <a href="/wiki/C%2B%2B" title="C++">C++</a>. This is typically done to either access OS services in the language in which the OS' API is defined, or for performance considerations.</p> 
      <p>Many FFIs also provide the means for the called language to invoke services in the host language as well.</p> 
      <p>The term foreign function interface is generally not used to describe multi-lingual runtimes such as the Microsoft <a href="/wiki/Common_Language_Runtime" title="Common Language Runtime">Common Language Runtime</a>, where a common "substrate" is provided which enables any CLR-compliant language to use services defined in any other. (However, in this case the CLR does include an FFI, <a href="/wiki/P/Invoke" class="mw-redirect" title="P/Invoke">P/Invoke</a>, to call outside the runtime.) In addition, many distributed computing architectures such as the <a href="/wiki/Java_remote_method_invocation" title="Java remote method invocation">Java remote method invocation</a> (RMI), RPC, <a href="/wiki/CORBA" class="mw-redirect" title="CORBA">CORBA</a>, <a href="/wiki/SOAP" title="SOAP">SOAP</a> and <a href="/wiki/D-Bus" title="D-Bus">D-Bus</a> permit different services to be written in different languages; such architectures are generally not considered FFIs.</p> 
      <h2><span class="mw-headline" id="Special_cases">Special cases</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Foreign_function_interface&amp;action=edit&amp;section=4" title="Edit section: Special cases">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>There are some special cases, in which the languages compile into the same bytecode VM, like <a href="/wiki/Clojure_(programming_language)" class="mw-redirect" title="Clojure (programming language)">Clojure</a> and <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, as well as <a href="/wiki/Elixir_(programming_language)" title="Elixir (programming language)">Elixir</a> and <a href="/wiki/Erlang_(programming_language)" title="Erlang (programming language)">Erlang</a>. Since there is no interface, it is strongly spoken not a FFI, while it offers the same functionality to the user.</p>
