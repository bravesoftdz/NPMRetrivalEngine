           <h1 name="662e" id="662e" class="graf graf--h3 graf--leading graf--title">How to never miss, or even answer, a buzzer call from friends ever again&nbsp;(1/2)</h1>
           <p name="e5c4" id="e5c4" class="graf graf--p graf-after--h3">In this two-part post I will explain how to build this fancy tool to let friends into your apartment easily.</p>

           <h3 name="1bcf" id="1bcf" class="graf graf--h3 graf-after--figure">Problem statement</h3>
           <p name="b22f" id="b22f" class="graf graf--p graf-after--h3">I don’t always pay attention to my phone, so when friends come over and try to buzz-in to my apartment, I sometimes miss the calls and they are stuck at the door until I answer or they give up and leave.</p>
           <h3 name="d2dc" id="d2dc" class="graf graf--h3 graf-after--p">Constraints</h3>
           <p name="289e" id="289e" class="graf graf--p graf-after--h3">I don’t want to have to pay a lot of money for a solution because I don’t really care about it enough, it’s just a fun nice-to-have. I also only have one day to do it?—?we do an “Expo jam” where we kick the tires of a new release by building something with it. I could realistically spend more time on it but it’s nice to have something shippable by the end of the day.</p>
           <h3 name="03d5" id="03d5" class="graf graf--h3 graf-after--p">Proposed solution</h3>
           <p name="f8c3" id="f8c3" class="graf graf--p graf-after--h3">While I could just be more considerate, this requires human behavioural change, which is much more difficult than writing an app. So instead I will build an app that lets my friends unlock the door without any intervention on my behalf.</p>
           <p name="d2cb" id="d2cb" class="graf graf--p graf-after--p">The app will allow them to sign in with Facebook, and if their user id is on a whitelist, they will be presented with a big “Unlock” button. To enter the door, they need to press that button and then dial my extension. For 30 seconds after they press unlock, the buzzer call should be automatically answered by some kind of robot whose sole life purpose is emitting the dialtone for the number 6. Welcome to the club, pal.</p>

           <h3 name="9f2c" id="9f2c" class="graf graf--h3 graf-after--figure">Technology</h3>
           <ul class="postList">
            <li name="d332" id="d332" class="graf graf--li graf-after--h3"><a href="http://twilio.com/" data-href="http://twilio.com/" class="markup--anchor markup--li-anchor" rel="noopener" target="_blank">Twilio</a>: none of this would be possible without Twilio. For $1/month I can register a local phone number that hits a webhook on incoming voice calls in order to determine how it should respond. As you will see, the API is very powerful.</li>
            <li name="8447" id="8447" class="graf graf--li graf-after--li"><a href="http://now.sh/" data-href="http://now.sh/" class="markup--anchor markup--li-anchor" rel="noopener" target="_blank">now.sh</a>: I need a server process running somewhere to handle the webhooks and store knowledge of the user whitelist, maybe other things too. With now.sh I get this for free. But I already have a subscription which I use for other things (economies of hobby-project-scale pays off again), which makes it so I can permanently allocate a minimum of 1 instance to the server, rather than have it sleep when inactive like with the free plan.</li>
            <li name="1951" id="1951" class="graf graf--li graf-after--li"><a href="https://expo.io/" data-href="https://expo.io/" class="markup--anchor markup--li-anchor" rel="noopener" target="_blank">Expo</a>: there needs to be some kind of frontend to this for my friends to sign in and press a button. You could use anything that is capable of running on a phone and firing http requests and performing OAuth. Expo works particularly well here, though, because I can use React and native platform components and it has nice primitives for social authentication. I can also share my app with friends without putting it on the app and play store, which is always time consuming and annoying to do. Alternatively I could build a web app, that would be fine too.</li>
           </ul>
           <h3 name="6ec5" id="6ec5" class="graf graf--h3 graf-after--li">Time to set up&nbsp;Twilio</h3>
           <p name="ac90" id="ac90" class="graf graf--p graf-after--h3">The first thing I need to do is register a phone number. If you don’t have a Twilio account, go ahead and register one, throw some credit in there, then go to the <a href="https://www.twilio.com/console/phone-numbers/search" data-href="https://www.twilio.com/console/phone-numbers/search" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Buy a Phone Number screen</a> and search for one with your area code, then buy it.</p>
           <p name="6355" id="6355" class="graf graf--p graf-after--p">Your phone number will now show up in your <a href="https://www.twilio.com/console/phone-numbers/incoming" data-href="https://www.twilio.com/console/phone-numbers/incoming" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Active numbers dashboard</a>. Let’s make it actually do something</p>
           <h4 name="d6f6" id="d6f6" class="graf graf--h4 graf-after--p">Create a TwiML&nbsp;Bin</h4>
           <p name="6a48" id="6a48" class="graf graf--p graf-after--h4">A TwiML Bin is basically a canned response for phone calls. TwiML is Twilio’s markup language for describing how it should respond to a phone call or SMS. Go to your <a href="https://www.twilio.com/console/dev-tools/twiml-bins" data-href="https://www.twilio.com/console/dev-tools/twiml-bins" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">TwiML Bins dashboard</a> and create a new bin, name it “Call Me”, then put this in the contents:</p>
           <pre name="a8ab" id="a8ab" class="graf graf--pre graf-after--p"><code class="markup--code markup--pre-code">&lt;?xml version="1.0" encoding="UTF-8"?&gt;<br>&lt;Response&gt;<br>  &lt;Dial&gt;your-phone-number-including-country-code-here&lt;/Dial&gt;<br>&lt;/Response&gt;</code></pre>
           <p name="06d7" id="06d7" class="graf graf--p graf-after--pre">Save it now. Well done.</p>

           <p name="7c9a" id="7c9a" class="graf graf--p graf-after--h4">Go back to your <a href="https://www.twilio.com/console/phone-numbers/incoming" data-href="https://www.twilio.com/console/phone-numbers/incoming" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Active numbers dashboard</a>, click on the phone number that you registered, and set the properties under “Voice” for “A call comes in” and “Primary handler fails” as TwiML with the “Call Me” bin that you just created.</p>

Once you're in, create a new project and run <code class="markup--code markup--p-code">npm init</code> then <code class="markup--code markup--p-code">npm install express twilio --save</code> and <code class="markup--code markup--p-code">npm install nodemon --save-dev</code> and add these to scripts in package.json:</p>
           <pre name="290c" id="290c" class="graf graf--pre graf-after--p">"scripts": {<br>    "start": "node index.js",<br>    "dev": "NODE_ENV=development nodemon index.js",<br>    "deploy": "now -e NODE_ENV=production"<br> },</pre>

           <h3 name="0d2e" id="0d2e" class="graf graf--h3 graf-after--p">Uh oh, real world&nbsp;stuff</h3>
           <p name="29e2" id="29e2" class="graf graf--p graf-after--h3">This is where you might be asking yourself?—?cool, I can get a call to a phone number to reply with “Hello world” but how does this help me get into my apartment? We need to actually change the mapping of your extension on your building’s buzzer system to point to your Twilio phone number. Fear not, if your server goes down or whatever, we already configured it to fall back to the TwiML Bin that calls your phone number instead. The instructions for how to change this really depend on your building, probably you need to ask your building manager to do it for you. Depending on their level of technophobia, they may or may not want to let you do this (thankfully not a problem in my case), so maybe don’t tell them what it’s for. They probably won’t ask anyways. Back to the bit flipping.</p>
           <h3 name="e077" id="e077" class="graf graf--h3 graf-after--p">First, let’s just let everybody in automatically</h3>
           <p name="1558" id="1558" class="graf graf--p graf-after--h3">This is precisely what your building manager would be concerned of, so let’s just do it and get it out of the way. Go back to your index.js and change the <code class="markup--code markup--p-code">app.post('/',&nbsp;...)</code> handler to the following:</p>
           <pre name="0be5" id="0be5" class="graf graf--pre graf-after--p"><code class="markup--code markup--pre-code">app.post('/', (request, response) =&gt; {<br>  let twiml = new twilio.twiml.VoiceResponse();<br>  twiml.play({ digits: 'ww666ww666ww666' });</code></pre>
           <pre name="0852" id="0852" class="graf graf--pre graf-after--pre"><code class="markup--code markup--pre-code">  response.type('text/xml');<br>  response.send(twiml.toString());<br>});</code></pre>
           <p name="926e" id="926e" class="graf graf--p graf-after--pre">Deploy it again with <code class="markup--code markup--p-code">now</code> and update the webhook url in Twilio (each deploy is immutable with now, so the url will be new -- you can use <code class="markup--code markup--p-code">now</code> with your own domain and alias new deploys so the url is persistent if you like). Call the phone number again and you'll notice it pauses for 1 second, then presses 6 three times, and repeats that two more times. If your building requires that you press 9, swap out 6 for 9, or whatever the number is.</p>
           <h3 name="4bc7" id="4bc7" class="graf graf--h3 graf-after--p">Next, let’s make it a little bit more&nbsp;secure</h3>
           <p name="745a" id="745a" class="graf graf--p graf-after--h3">Open up index.js again and replace it with this:</p>
           <pre name="db65" id="db65" class="graf graf--pre graf-after--p">const twilio = require('twilio');<br>const express = require('express');</pre>
           <pre name="8bf9" id="8bf9" class="graf graf--pre graf-after--pre">let app = express();</pre>
           <pre name="4e62" id="4e62" class="graf graf--pre graf-after--pre">// State that persists across requests<br>let _unlockedAt = null;</pre>
           <pre name="7780" id="7780" class="graf graf--pre graf-after--pre">app.post('/unlock', (request, response) =&gt; {<br>  _unlockedAt = new Date();<br>  response.type('application/json');<br>  response.send({ unlockedAt: _unlockedAt });<br>});</pre>
           <pre name="761f" id="761f" class="graf graf--pre graf-after--pre">app.post('/lock', (request, response) =&gt; {<br>  _unlockedAt = null;<br>  response.type('application/json');<br>  response.send({ locked: true });<br>});</pre>
           <pre name="c20a" id="c20a" class="graf graf--pre graf-after--pre">app.post('/', (request, response) =&gt; {<br>  let twiml = new twilio.twiml.VoiceResponse();<br>  if (_isUnlocked()) {<br>    twiml.play({ digits: 'ww666ww666ww666' });<br>  } else {<br>    // Have Twilio fallback to our "Call Me" bin<br>    throw new Error('Not permitted to unlock');<br>  }</pre>
           <pre name="9e53" id="9e53" class="graf graf--pre graf-after--pre">  response.type('text/xml');<br>  response.send(twiml.toString());<br>});</pre>
           <pre name="43f5" id="43f5" class="graf graf--pre graf-after--pre">function _isUnlocked() {<br>  return _unlockedAt &amp;&amp; new Date().getTime() - _unlockedAt.getTime() &lt; 30000;<br>}</pre>
           <pre name="a476" id="a476" class="graf graf--pre graf-after--pre">app.listen(3000);<br>console.log(`server running (${process.env.NODE_ENV})`);</pre>
           <p name="73aa" id="73aa" class="graf graf--p graf-after--pre">Cool, this is actually the part where we get into building an Expo app. Let’s just do this part on <a href="https://snack.expo.io/" data-href="https://snack.expo.io/" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Snack</a> for now. I went ahead and built a really simple app that interacts with this endpoint on Snack: <a href="https://snack.expo.io/rJAlRIH8-" data-href="https://snack.expo.io/rJAlRIH8-" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Basic Buzzer</a>.</p>
           <p name="2097" id="2097" class="graf graf--p graf-after--p">The app has a single button in the middle: “Unlock”. When you press it, it sends a POST request to <code class="markup--code markup--p-code">{your-server}/unlock</code> and then updates the unlocked at date and starts a countdown. When the countdown expires, we switch back to the locked state. Alternatively, is the user presses "Lock" then we send a POST request to <code class="markup--code markup--p-code">{your-server}/lock</code> and reset to the locked state. In order to show a bit of feedback to the user, we display an <code class="markup--code markup--p-code">ActivityIndicator</code> while the http requests are in flight.</p>
           <p name="600d" id="600d" class="graf graf--p graf-after--p">To test it out, replace the url at the top of the Basic Buzzer example with the url for your server, then make a post request to <code class="markup--code markup--p-code">/</code> on your server from your http client and it should throw an error. Next, press "Unlock" in the app, then send that post request again. You should see something like this.</p>
