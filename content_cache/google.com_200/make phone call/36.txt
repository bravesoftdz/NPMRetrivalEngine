            <h1 data-reactid="94" itemprop="headline"><a href="/post/5-best-libraries-for-making-ajax-calls-in-react-cis8x5f7k0jl7th53z68s41k1" data-reactid="95">5 best libraries for making AJAX calls in React</a></h1>
           </div>
           <a href="/post/5-best-libraries-for-making-ajax-calls-in-react-cis8x5f7k0jl7th53z68s41k1" data-reactid="96" class="story-cover-photo blurImage" style="background-image:url(&quot;https://res.cloudinary.com/hashnode/image/upload/w_80,fl_lossy,q_10/v1472044278/yycj511xzdtpltu7getd.jpg&quot;)" alt=""></a>
           <div data-reactid="97" class="post-meta-desc">
            <div data-reactid="98" itemprop="articleBody mainEntityOfPage">
             <p>I have been working with React for the last two years. One of the questions many React beginners ask: "What's the React way to fetch data from the server" or "How should I make AJAX calls in React"? To answer your question, as many developers would tell you, React is just a view library and you are free to use whatever you are comfortable with. However, this doesn't help much - especially when the JavaScript landscape is changing so rapidly. So, in this article I will try to answer this basic question and list down 5 simple libraries for making AJAX calls in React. </p> 
             <h2 id="jquery-ajax">jQuery $.ajax</h2> 
             <p>This is a quick &amp; dirty way to make AJAX calls. In the former, <a href="http://chenglou.github.io/react/docs/tutorial.html" rel="noopener noreferrer" target="_blank">official React tutorial</a>, they use jQuery to fetch data from the server. If you are just starting out and are playing around with React, this can save you a lot of time. Many of us are already familiar with jQuery. So, it won't take a lot of time to understand and use it in React. Here is how a simple API call is made, with jQuery: </p> 
             <pre><code>loadCommentsFromServer: <span class="hljs-function"><span class="hljs-keyword">function</span>(<span class="hljs-params"></span>) </span>{
      $.ajax({
      url: <span class="hljs-keyword">this</span>.props.url,
      dataType: <span class="hljs-string">'json'</span>,
      cache: <span class="hljs-literal">false</span>,
      success: <span class="hljs-function"><span class="hljs-keyword">function</span>(<span class="hljs-params">data</span>) </span>{
        <span class="hljs-keyword">this</span>.setState({data: data}); <span class="hljs-comment">// Notice this</span>
      }.bind(<span class="hljs-keyword">this</span>),
      error: <span class="hljs-function"><span class="hljs-keyword">function</span>(<span class="hljs-params">xhr, status, err</span>) </span>{
        <span class="hljs-built_in">console</span>.error(<span class="hljs-keyword">this</span>.props.url, status, err.toString());
      }.bind(<span class="hljs-keyword">this</span>)
    });
  }
</code></pre>
             <p><em>P.S. Snippet is from React's former <a href="http://chenglou.github.io/react/docs/tutorial.html" rel="noopener noreferrer" target="_blank">official tutorial</a>.</em></p> 
             <p>It's the same old jQuery's <code>$.ajax</code> used inside a React component. Notice how <code>this.setState()</code> is called inside the <code>success</code> callback. This is how you update the state with data obtained from the API call.</p> 
             <p>However, jQuery is a big library with many functionalities - So, it doesn't make sense to use it just for making API calls (Unless you are already using it for a bunch of other tasks). So, what's the alternative? What should we use? The answer is <code>fetch</code> API.</p> 
             <h2 id="fetch-api">Fetch API</h2> 
             <p>Fetch is a new, simple and standardised API that aims to unify fetching across the web and replace <code>XMLHttpRequest</code>. It has a <a href="https://github.com/github/fetch" rel="noopener noreferrer" target="_blank">polyfill</a> for older browsers and should be used in modern web apps. If you are making API calls in Node.js you should also check out <a href="https://github.com/bitinn/node-fetch" rel="noopener noreferrer" target="_blank">node-fetch</a> which brings <code>fetch</code> to Node.js. </p> 
             <p>Here is what the modified API call looks like :</p> 
             <pre><code>loadCommentsFromServer: <span class="hljs-function"><span class="hljs-keyword">function</span>(<span class="hljs-params"></span>) </span>{
    fetch(<span class="hljs-keyword">this</span>.props.url).then(<span class="hljs-function"><span class="hljs-keyword">function</span>(<span class="hljs-params">response</span>)</span>{
        <span class="hljs-comment">// perform setState here</span>
    });
}
</code></pre>
             <p>In most modern React tutorials you will find <code>fetch</code> being used. To know more about <code>fetch</code>, check out the following links :</p> 
             <ul> 
              <li><a href="https://developer.mozilla.org/en/docs/Web/API/Fetch_API" rel="noopener noreferrer" target="_blank">Mozilla</a></li> 
              <li><a href="https://davidwalsh.name/fetch" rel="noopener noreferrer" target="_blank">David Walsh Blog</a></li> 
              <li><a href="https://developers.google.com/web/updates/2015/03/introduction-to-fetch?hl=en" rel="noopener noreferrer" target="_blank">Google Developers </a></li> 
              <li><a href="https://fetch.spec.whatwg.org/" rel="noopener noreferrer" target="_blank">WHATWG</a></li> 
             </ul> 
             <h2 id="superagent">Superagent</h2> 
             <p><a href="https://visionmedia.github.io/superagent/" rel="noopener noreferrer" target="_blank">Superagent</a> is a light weight AJAX API library created for better readability and flexibility. If due to some reason, you don't want to use <code>fetch</code>, you should definitely check this out. Here is a snippet to demonstrate its usage : </p> 
             <pre><code>loadCommentsFromServer: <span class="hljs-function"><span class="hljs-keyword">function</span>(<span class="hljs-params"></span>) </span>{
    request.get(<span class="hljs-keyword">this</span>.props.url).end(<span class="hljs-function"><span class="hljs-keyword">function</span>(<span class="hljs-params">err,res</span>)</span>{
        <span class="hljs-comment">// perform setState here</span>
    });
}
</code></pre>
             <p>Superagent also has a Node.js module with the same API. If you are building isomorphic apps using Node.js and React, you can bundle <code>superagent</code> using something like webpack and make it available on the client side. As the APIs for client and server are the same, no code change is required in order to make it work in the browser.</p> 
             <h2 id="axios">Axios</h2> 
             <p><a href="https://github.com/mzabriskie/axios" rel="noopener noreferrer" target="_blank">Axios</a> is a promise based HTTP client for Node.js and browser. Like <code>fetch</code> and <code>superagent</code>, it can work on both client and server. It has many other useful features which you can find on their GitHub page. </p> 
             <p>Here is how you make an API call using Axios : </p> 
             <pre><code>loadCommentsFromServer: <span class="hljs-function"><span class="hljs-keyword">function</span>(<span class="hljs-params"></span>) </span>{
    axios.get(<span class="hljs-keyword">this</span>.props.url).then(<span class="hljs-function"><span class="hljs-keyword">function</span>(<span class="hljs-params">response</span>)</span>{
      <span class="hljs-comment">// perform setState here</span>
    }).catch(<span class="hljs-function"><span class="hljs-keyword">function</span>(<span class="hljs-params">error</span>)</span>{
      <span class="hljs-comment">//Some error occurred</span>
    });
}
</code></pre>
             <h2 id="request">Request</h2> 
             <p>This list will be incomplete without <a href="https://github.com/request/request" rel="noopener noreferrer" target="_blank">request</a> library which was designed with simplicity in mind. With more that 12k GitHub stars, it's also one of the most popular Node.js modules. You can find more about <code>request</code> module on their <a href="https://github.com/request/request" rel="noopener noreferrer" target="_blank">GitHub page</a>. </p> 
             <p>Sample usage : </p> 
             <pre><code>loadCommentsFromServer: <span class="hljs-function"><span class="hljs-keyword">function</span>(<span class="hljs-params"></span>) </span>{
    request(<span class="hljs-keyword">this</span>.props.url, <span class="hljs-function"><span class="hljs-keyword">function</span>(<span class="hljs-params">err, response, body</span>)</span>{
          <span class="hljs-comment">// perform setState here</span>
    });
}
</code></pre>
             <h1 id="my-take">My Take</h1> 
             <p>As <code>fetch</code> is the new standardised API to interact with remote resources, I would suggest using it for all your AJAX needs (not only in React, but all types of JavaScript apps).</p>

               <pre><code class="lang-javascript">qwest.get(<span class="hljs-keyword">this</span>.props.url)
     .then(<span class="hljs-function"><span class="hljs-keyword">function</span>(<span class="hljs-params">xhr, response</span>) </span>{

</code></pre> 
               <p>When components unmount/unload, call <code>abort()</code> to avoid React spamming the console when Ajax ?Promises resolve after the component has unmounted/unloaded from the DOM.</p>
