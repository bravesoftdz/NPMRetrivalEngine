         <div id="content_container_2554"> 
          <p>The E-commerce package provides an interface where <em>Site</em> visitors can navigate through an online store, ordering items through a customised E-commerce system. These systems can provide different ordering methods as well as a number of payment and delivery options.</p>
          <p class="caption"><img title="The general format of an E-Commerce system" alt="The general format of an E-Commerce system" height="383" width="400" src="https://matrix.squiz.net/__data/assets/image/0019/4843/4-0-0_general-format-ecom-system.png"><br>The general format of an E-Commerce system</p>
          <p>The <em>E-commerce</em> package consists of the following components:</p>
          <ul>
           <li><strong>Ecommerce Form Page</strong>: the <em>Ecommerce Form Page</em> allows the creation of a form used for e-commerce transactions. Questions are added to the form to collect different information from the user, such as a name and delivery address. The form page is then linked to a payment method.</li>
           <li><strong>Cart</strong>: items ordered can be added to a <em>Cart</em>, allowing users to modify and confirm their purchases before proceeding with the transaction. The <em>Cart</em> is then linked to a <em>Checkout Page</em>.</li>
           <li><strong>Checkout Page</strong>: the <em>Checkout Page</em> provides users with information allowing them to select a delivery method and input any required information for that method. Other information on an order can be displayed on this page, including ordered items and tax.</li>
           <li><strong>Ecommerce Default Delivery Method</strong>: a number of delivery methods can be configured to allow users to select how they will acquire the items they are purchasing. These delivery methods include e-commerce forms specific to the delivery method selected.</li>
           <li><strong>E-commerce Payment Gateways</strong>: payment gateways can be attached to an <em>Ecommerce Form Page</em> or <em>Checkout Page</em> to provide users with a secure system to pay for their items. The payment gateway assets available in the <em>E-commerce </em>package are: 
            <ul>
             <li><strong><a href="https://matrix.squiz.net/manuals/e-commerce/chapters/securepay-direct-post-payment-gateway">SecurePay</a></strong></li>
             <li><strong><a href="https://matrix.squiz.net/manuals/e-commerce/chapters/datacash-payment-gateway">Datacash</a></strong></li>
             <li><strong><a href="https://matrix.squiz.net/manuals/e-commerce/chapters/sagepay-payment-gateway">SagePay</a></strong></li>
             <li><strong><a href="https://matrix.squiz.net/manuals/e-commerce/chapters/dps-payment-gateway">Payment Express</a></strong></li>
             <li><strong><a href="https://matrix.squiz.net/manuals/e-commerce/chapters/paypal-payment-gateway">PayPal</a></strong></li>
             <li><strong><a href="https://matrix.squiz.net/manuals/e-commerce/chapters/braintree-payment-gateway">Braintree</a> </strong></li>
            </ul></li>
           <li><strong>PayPal Payment Gateway: </strong>the PayPal payment gateway assets can be used to create an e-commerce system with PayPalpayment buttons, allowing users to purchase items and make donations with Paypal. The PayPal payment gateway is configured using the following assets: 
            <ul>
             <li><strong>Paypal Configuration: </strong>the <em>Paypal Configuration</em> asset stores the information regarding the security and encryption of the PayPal transactions.</li>
             <li><strong>Paypal Business Account: </strong>the <em>Paypal Business Account</em> asset stores information regarding the Merchant Account that will receive payments from the PayPal transactions.</li>
             <li><strong>Paypal Payment Button</strong>: the <em>Paypal Payment Button</em>asset configures a variety of PayPal payment options available through buttons on your e-commerce <strong><em>Site</em></strong>. The button types available are: 
              <ul>
               <li><strong>Buy Now: </strong>this button allows users to make single-item purchases on an e-commerce store.</li>
               <li><strong>Donate: </strong>this button allows users to make donations to a specified organisation or service.</li>
               <li><strong>Add to Cart: </strong>this button allows users to add items to a Shopping Cart maintained on the PayPal website. Once an item has been added to their cart, a user can choose to continue shopping on the e-commerce store or proceed to the checkout process, purchasing the item(s) in their cart.</li>
               <li><strong>View Cart: </strong>this button allows users to view the items currently stored in their PayPal Shopping Cart. There they can remove items from their cart, update item quantities and/or proceed to the checkout process.</li>
              </ul></li>
             <li><strong>Paypal IPN Receiver: </strong>the <em>Paypal IPN Receiver </em>asset receives an <em>Instant Payment Notification (IPN)</em> for each successful payment from the PayPal payment gateway.</li>
            </ul></li>
           <li><strong>Product</strong>: products are added to a <em>Site</em> for users to purchase. Name, value and tax options are included in the <em>Product</em> details as well as descriptions and specifications. Further information can be assigned to a <em>Product</em> with applied metadata.</li>
           <li><strong>Donation</strong>:<em> Donation </em>assets allow you to create donation items, allowing site users to donate money using an e-commerce system.</li>
           <li><strong>Promotional Code</strong>: the <em>Promotional Code </em>asset allows customers to receive discounts on items when entering store promotional codes.</li>
          </ul>
          <p>The E-commerce assets can be used for a variety of features such as:</p>
          <ul>
           <li>Order logging</li>
           <li>Taxation</li>
           <li>Delivery fees</li>
           <li>Flat fees</li>
           <li>Quantity tracking</li>
           <li>Delivery Emails for both the buyer and seller</li>
          </ul>
          <p>These functions are outlined in this manual.</p>
