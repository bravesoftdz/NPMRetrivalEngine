      <h1 class="page-title">Node.js E-Commerce with Harp JS &amp; Snipcart</h1> 
      <span class="post__date">August 25, 2016</span> 
      <div class="page-top__image--normal"> 
       <img src="https://snipcartweb-10f3.kxcdn.com/media/1256/cart.png" data-small="https://snipcartweb-10f3.kxcdn.com/media/1256/cart.png" data-medium="https://snipcartweb-10f3.kxcdn.com/media/1256/cart.png" data-large="https://snipcartweb-10f3.kxcdn.com/media/1256/cart.png" alt="" class="post__image src-change"> 
      </div> 
     </div> 
     <div class="page-top__overlay"></div> 
    </div> 
    <div class="normal-content"> 
     <div class="post__body stretchable-section"> 
      <div class="content-wrapper--small"> 
       <div class="post__rte rte"> 
        <p>If we've learned one thing these last few years, it's that developers are getting tired of putting up with closed ecosystems, clunky backends, and heavy databases. This frustration, coupled with the rise of client-side possibilities, made way for <a href="https://jamstack.org/">the JAMstack revolution</a> (JavaScript, APIs, &amp; Markup). As a result, an exponential number of developers are now using static site generators.</p> 
        <p>At Snipcart, we've also played around with them, a lot. Whether with <a href="https://snipcart.com/blog/static-site-e-commerce-part-2-integrating-snipcart-with-jekyll">Jekyll</a>, <a href="https://snipcart.com/blog/hugo-tutorial-static-site-ecommerce">Hugo</a>, or many others, we've created <a href="https://github.com/snipcart">a bunch of demo sites and cart integrations</a> on top of static sites.</p> 
        <p><em>Note: If you're new to the whole static/JAMstack development phenomenon, we strongly advise giving <a href="https://www.smashingmagazine.com/2015/11/modern-static-website-generators-next-big-thing/">this Smashing Magazine post</a> a read.</em></p> 
        <p>Recently, we published a post <a href="https://snipcart.com/blog/learn-vanilla-javascript-before-using-js-frameworks">telling developers to learn vanilla JS before frameworks</a>. In it, we touched on the awesomeness of <strong>Node.js</strong> a bit. A good friend of ours, knowing how much we liked static site generators, pointed us to a Node.js-powered tool he thought we'd enjoy: <strong>Harp JS</strong>.</p> 
        <p>I immediately took it upon myself to craft a Harp JS tutorial for our blog. So in this post, I'm going to show how easily Node.js e-commerce can be handled on a static site with Harp JS.</p> 
        <p>First, a few definitions.</p> 
        <h2>What is Node.js?</h2> 
        <p>Basically, <a href="https://nodejs.org/en/">Node.js</a> is an event-driven JavaScript runtime environment—so, yeah, server-side, browser-less JS execution. It's lightweight, efficient, and boasts the most popular open source package ecosystem in the world: <code>npm</code>.</p> 
        <p>Why is it awesome, you ask? To quote Isaac Lewis in <a href="https://www.ike.io/blog/make-your-own-static-site-with-harp.html">his cool post on using Harp JS / Node.js</a>:</p> 
        <blockquote> 
         <p>"This is why I love Node.js: Instead of having to install PHP and MySQL and Apache, you just install Node + a few packages, and we're running a local server in minutes, instead of days."</p> 
        </blockquote> 
        <p>The community around Node is huge; it's become the #1 choice for many developers putting together killer server-side applications.</p> 
        <h2>What is Harp JS?</h2> 
        <p><a href="https://harpjs.com/">Harp JS</a> is a Node.js open source static web server with built-in preprocessing. Since it serves Jade, Markdown, EJS, CoffeeScript, Sass, LESS and Stylus as HTML, CSS &amp; JavaScript without requiring any configuration, it has become very popular in the static web dev community.</p> 
        <p>Oh, and it also <a href="http://charliegleason.com/articles/harp-gulp-and-browsersync">plays well with BrowserSync &amp; Gulp</a>.</p> 
        <p>All right, so now, I'm going to walk you through a Harp static shopping cart set up. By the end of this post, we'll have a nice Node.js e-commerce set up thanks to Harp &amp; Snipcart.</p> 
        <h3>Precondition</h3> 
        <p>I'll assume you've already played around with Harp JS a bit. If not, you can read the in-depth post developer Matt Lambert wrote a while ago about <a href="http://mattlambert.ca/blog/how-to-use-the-harp-js-static-site-generator/">how to use Harp's static web server</a>.</p> 
        <p><strong>You'll also need a Snipcart account. Development is forever free in test mode; you can <a href="https://app.snipcart.com/register">sign up here</a>.</strong></p> 
        <h2>1. Installing Harp JS and initializing your e-commerce site</h2> 
        <p>First, you will have to install Harp and initialize a project. Of course, there's a handy little Node.js <code>npm</code> package to do just that:</p> 
        <pre><code class="language-bash">npm install -g harp
harp init snipcartdemo
</code></pre> 
        <p>The second command will generate a basic Harp project that you could already serve if you wanted to.</p> 
        <h2>2. Adding the Snipcart API Key into your Harp project</h2> 
        <p>We will add our Snipcart API key (found under Account &gt; API Keys in your merchant dashboard) in the global config before we start the local server.</p> 
        <p>To do so, you will create a <code>harp.json</code> file in your project's root directory.</p> 
        <p>Here's what we will put in the new JSON file:</p> 
        <pre><code class="language-json">{
    "globals": {
        "snipcartApiKey": "YjdiNWIyOTUtZTIyMy00MWMwLTkwNDUtMzI1M2M2NTgxYjE0"
    }
}
</code></pre> 
        <p>By declaring the API key this way, Harp will make this variable public so any page can access it.</p> 
        <p>You can now boot up the server so we can play around a little! This command, using your project name, will do the trick:</p> 
        <pre><code class="language-bash"> harp server snipcartdemo
</code></pre> 
        <h2>3. Creating products and appropriate layouts</h2> 
        <p>Now that everything is up and running, let's put some meat on this e-commerce bone!</p> 
        <p>We will create a <code>products</code> folder that will contain both the data and the layout templates related to the products.</p> 
        <p>Now, create a <code>_data.json</code> file in this new products folder.</p> 
        <p>We put the low dash in front of the file so it won't be served publicly—a Harp convention. This is where any information regarding the products will reside.</p> 
        <p>Here's our file for the demo:</p> 
        <pre><code class="language-json">{
  "product1": {
    "name": "Ski Goggles",
    "id": "1",
    "image": "https://snipcartweb-10f3.kxcdn.com/media/9896/harpjs-shopping-cart-ski-goggles.jpg",
    "description": "Clear lens, perfect for night runs in the park.",
    "url": "/",
    "price": "120.00"
  },
  "product2": {
    "name": "Lightweight Helmet",
    "id": "2",
    "description": "Subtle style with secure, tested shock protection.",
    "image": "https://snipcartweb-10f3.kxcdn.com/media/9897/harpjs-shopping-cart-ski-helmet.jpg",

    "price": "100.00"
  },
  "product3": {

    "url": "/",
    "price": "80.00"
  }
}  
</code></pre> 
        <p>Okay, so let's create a template to show this product data.</p> 
        <p>We will stay in the same folder (<code>products</code>) and create a <code>_product.jade</code> file.</p> 
        <p>In the following examples, we're going to use the <code>jade</code> templating engine; we're big fans here at Snipcart. ;) But if you feel more comfortable with more HTML-like templating engines, we suggest you take a look at <a href="http://www.embeddedjs.com/">EJS</a>. It can be used with Harp out of the box.</p> 
        <p>Here's the template we will be using:</p> 
        <pre><code>div.product-details
  img(src=image)
  div.product-description 
    p #{ content } 
    
    button.snipcart-add-item(
      data-item-name=name
      data-item-id=id
      data-item-image=image 
      data-item-url=url
      data-item-price=price
      data-item-description=description)
      Buy it for #{ price } 
</code></pre> 
        <p>You'll see further down how this simple products layout looks!</p> 
        <h2>4. Injecting the actual frontend shopping cart</h2> 
        <p>Now, let's generate a partial <a href="http://docs.snipcart.com/getting-started/installation">including all the necessary scripts for Snipcart</a>.</p> 
        <p>We will create a new <code>Snipcart</code> folder, and nest a <code>_snipcart.jade</code> file in it.</p> 
        <p>Here's ours:</p> 
        <pre><code>     script(src="https://cdnjs.cloudflare.com/ajax/libs/jquery/3.1.0/jquery.js")
     
     script(src="https://cdn.snipcart.com/scripts/2.0/snipcart.js" data-api-key=snipcartApiKey id="snipcart")
     
     link(href="https://cdn.snipcart.com/themes/2.0/base/snipcart.min.css" rel="stylesheet" type="text/css")

</code></pre> 
        <p>As you can see, for the <code>data-api-key</code>, we are not hard coding the key there, but rather using the global variable we defined earlier.</p> 
        <h2>5. Putting the final touches on your Harp JS e-commerce site</h2> 
        <p>Almost there, folks! What we want to do now is use everything we created to bring our little demo store to life. We'll keep everything simple and use the default index page to do so.</p> 
        <p>First, we'll put our partial that contains the scripts, and then iterate through our products data and render a product template for each.</p> 
        <p>Here's what it looks like in code:</p> 
        <pre><code class="language-js">h1 Harp store powered by Snipcart

!= partial("./snipcart/_snipcart.jade")

div.products
    for product in public.products._data
        != partial("./products/_product", product)
</code></pre> 
        <p>And <em>voilà</em>:</p> 
        <p><img src="https://snipcartweb-10f3.kxcdn.com/media/9905/nodejs-ecommerce-harpjs-snipcart.png" alt="nodejs-ecommerce-harpjs-snipcart"></p> 
        <blockquote> 
         <p>Go ahead, <a href="https://snipcart.github.io/snipcart-harp/index">check out the live demo!</a></p> 
        </blockquote> 
        <p>And also, all of the code used for this integration is available <a href="https://github.com/snipcart/snipcart-harp">in this GitHub repo</a>.</p> 
        <h2>Conclusion</h2> 
        <p>I truly had a blast crafting this lean Node.js e-commerce site with Harp JS &amp; our shopping cart platform.</p> 
        <p>Altogether, reading the Harp JS docs + creating the templates and data took me about an hour or two. Yeah, it took me quite some time to understand that you had to restart the server to access newly created globals. ;)</p> 
        <p>The Harp JS <a href="https://harpjs.com/docs/">documentation</a> was helpful and went straight to the point. And preprocessing out-of-the-box was a bliss (setting up the stack with a bash command only).</p> 
        <p>Of course, we could have pushed the integration a bit further. For instance: creating a custom layout <a href="http://docs.snipcart.com/getting-started/the-cart">showing a cart icon</a> with products count, or displaying a single page for every product.</p> 
        <p>I sincerely hope this post convinced you to start playing with Harp JS and/or explore the modern static web dev and JAMstack possibilities around you. And if you ever use Snipcart to enable e-commerce on your static project, shoot us an email at geeks@snipcart.com. We'd like to check it out, and maybe feature it somewhere if it's good, inspiring stuff for our community!</p>
