             <p>If you’re familiar with Shopify Themes and the traditional workflow of building (and customizing) a Shopify store, you’ll know there’s a strong focus on both Liquid and Ruby developing languages — perfect for working with Shopify, on the Shopify platform.</p> 
             <p>However, a lot of web designers and developers also do projects for other major frameworks, like WordPress, Drupal, or client-hosted websites. And, common to these frameworks, is one of the most frequently used developing languages on the web: JavaScript.</p> 
             <p>All of this begs the question: how do you combine ecommerce with a developing language that has such a large community following?</p> 
             <p>Recently, we sat down with Tessa Thornton — front end developer from Shopify’s Buy Button Team— for a <a target="_blank" href="/partners/blog/partner-webinars">Shopify Partner Session webinar</a>, where we discussed how to use <a target="_blank" href="/partners/blog/introducing-buybutton-js-shopifys-new-javascript-library">BuyButton.js</a> and <a target="_blank" href="/partners/blog/introducing-buybutton-js-shopifys-new-javascript-library">JS Buy SDK</a> to create ecommerce experiences on any platform.</p> 
             <p><a target="_blank" href="/partners/blog/introducing-buybutton-js-shopifys-new-javascript-library">We’ve written about Shopify’s two Buy Button libraries before</a>, but this article will dig a little deeper into the technical nuances of both, and give you a better understanding of how to use Shopify’s Buy Button functionality in your current, or upcoming, client project.</p> 
             <h2>Why Buy Button?</h2> 
             <p>Ever wanted to add Shopify to a client’s website, but couldn’t because they weren’t on Shopify’s Online Store channel (a store hosted on Shopify servers)? This is the pain point we aimed to solve when we created the Buy Button.</p> 
             <p>You can use the Buy Button Channel to benefit your clients in the following ways:</p> 
             <ul> 
              <li>Turn non-ecommerce pages into product pages</li> 
              <li>Prototype and validate design ideas</li> 
              <li>Add purchasing functionality to client apps</li> 
              <li>Integrate products into existing landing pages</li> 
              <li>Monetize your client’s blog</li> 
             </ul> 
             <p class="marketing-block marketing-block--light marketing-block--padded"><strong>You might also like:</strong> <a target="_blank" href="/partners/blog/four-smart-reasons-to-embed-a-shopify-store-in-your-client-websites" rel="noopener noreferrer">Four Smart Reasons to Embed a Shopify Store in Your Client Websites</a></p> 
             <h2>The Buy Button channel</h2> 
             <p>First things first. Regardless of how you wish to approach Buy Button in your web design and development process, you’ll need to <a target="_blank" href="https://apps.shopify.com/shopify-widgets">install the Buy Button channel</a> on whichever client store you’re working on. This will allow you to generate the authentication token needed to initialize the <code class="marketing-code">buildClient</code>, which in turn, gives you access to your client’s product data.</p> 
             <p>It’s important to note that, though you don’t need the Online Store channel to use Buy Button, your client does need to be on a paid Shopify plan — the basic $9/month (Shopify Lite) will do just fine.</p> 
             <p>On Shopify Lite, your client will get access to everything they need to implement a simple ecommerce solution on their website, including unlimited products, order tracking, and access to various payment gateways — so don’t worry about losing functionality in lieu of the smaller price tag.</p> 
             <h2>JS Buy SDK</h2> 
             <p>So, what exactly is the JS Buy SDK?</p> 
             <p>For starters, it’s an API client that allows freelancers and agencies to add Shopify ecommerce functionality to non-Shopify hosted stores, like Wix, SquareSpace, Drupal, WordPress, self-hosted websites, and websites hosted on cloud-based solutions.</p> 
             <p><strong>TL;DR you can add ecommerce functionality to any platform that supports JavaScript.</strong></p> 
             <p>The JS Buy SDK is built on top of Shopify’s API and uses the Fetch API to make HTTP requests. It’s purpose is to give you access to product listing data outside of the online store environment (typically associated with the Online Store channel, or a Shopify storefront), allows you to manage your client's cart experience, and makes consuming a REST API with JavaScript a breeze.</p> 
             <p>Say you wanted to write a JavaScript app or script to consume Shopify’s API, there are a couple of things to consider: you’ll need to write some boilerplate code, worry about authentication, and make AJAX requests across platforms. The JS Buy SDK takes care of these nitty gritty details for you, so you can focus on building customized ecommerce solutions for your clients.</p> 
             <p>You can learn more about JS Buy SDK, and how to get it up and running on your client’s website, with this <a rel="nofollow noopener noreferrer" target="_blank" href="https://shopify.github.io/js-buy-sdk/">helpful documentation</a> created by the Shopify Buy Button Team.</p> 
             <p><strong>Knowledge of JavaScript required to use JS Buy SDK: </strong>Intermediate to Advanced.</p> 
             <h2>Features of JS Buy SDK</h2> 
             <p>There are many features of the JS Buy SDK that make it an attractive option for web designers and developers.</p> 
             <ul> 
              <li> <strong>It’s created on a promise-based interface</strong>. This ensures that all operations are completed before moving on to the next step. For example, with a promise-based interface, someone is unable to purchase a product before the product has finished loading.</li> 
             </ul> 
             <ul> 
              <li> <strong>It has polyfills</strong>. This gives the JS Buy SDK browser support back to IE 9.</li> 
             </ul> 
             <ul> 
              <li> <strong>It features models with dynamic properties</strong>. No longer will you need to actively remember the changes you’ve made to certain data points. Dynamic properties means that if you update the <code class="marketing-code">selectedVariant</code> of a product model, the next time you go to access the <code class="marketing-code">selectedVariant</code> property, it will already be set to the new one.</li> 
             </ul> 
             <ul> 
              <li> <strong>It has a tiny footprint.</strong> No, really. It’s under 30kb minified.</li> 
             </ul> 
             <h2>What can you build with JS Buy SDK?</h2> 
             <h4>The world is your oyster</h4> 
             <p>If we’re being completely honestly, you can build whatever your heart desires (as long as you have the skillset to implement it, of course). This includes any kind of interface, as long as it includes product, variant, and cart concepts.</p> 
             <h4>Create your own libraries</h4> 
             <p>Is your client looking for a very specific ecommerce customization for their website? Luckily, you can use the JS Buy SDK as a foundation to build your own JS and JQuery libraries. This is extremely helpful if, for example, you want to create a library that allows for detailed product galleries on your existing (or upcoming) client project.</p> 
             <h4>Integrate with other frameworks</h4> 
             <p>The JS Buy SDK offers various JavaScript framework integrations. You can create product or cart components for React, Ember, and Angular. We encourage you to&nbsp;experiment as much as you can with JS Buy SDK — it plays nicely with others!</p>

             <h2>BuyButton.js</h2> 
             <p>Before we get into what BuyButton.js is, it’s important to understand why it was created in the first place.</p> 
             <p>Following the release of JS Buy SDK, Shopify developers started to see commonalities in partner, web designer, and developer work in the wild. From these observations, patterns were established between what people were already building with the JS Buy SDK, and what they ultimately wanted to build with it.</p> 
             <p>Shopify Developers then noticed that a lot of developers were writing repetitive code, and more generalist developers didn’t really know where to start with the JS Buy SDK. If you’re not familiar with more advanced JavaScript coding, that’s understandable — it may be difficult to envision how to create an actual interface from the objects in your console.</p> 
             <p>And so BuyButton.js was born.</p> 
             <p>Built on top of the JS Buy SDK, BuyButton.js is an out-of-the-box UI/component library that allows web designers and developers to create interactive ecommerce components, using only a few lines of code.</p> 
             <p>BuyButton.js can be used for:</p> 
             <ul> 
              <li>Product embeds</li> 
              <li>Collection embeds</li> 
              <li>Carts</li> 
              <li>Product details modal</li> 
             </ul> 
             <p>All BuyButton.js embeds are sandboxed in iFrames — so they won’t inherit the CSS from your client’s webpage, and will always look the same, regardless of which page you place them on. However, if you want complete control over the CSS of your elements, you can opt out of this sandboxing by passing iFrames false, then starting with the default BuyButton.css, or starting from scratch.</p> 
             <p>You can find more details and implementation walkthroughs for BuyButton.js in this <a rel="nofollow noopener noreferrer" target="_blank" href="https://shopify.github.io/buy-button-js/">helpful documentation</a>, maintained by the Shopify Buy Button team.</p> 
             <p><strong>Knowledge of JavaScript required to use BuyButton.js: </strong>Beginner to Advanced, depending on the complexity of your customization.</p>

             <h2>Customization API</h2> 
             <p>BuyButton.js also gives you access to a flexible <a rel="nofollow noopener noreferrer" target="_blank" href="http://shopify.github.io/buy-button-js/customization/">Customization API</a> — a powerful design resource that isn't too difficult to learn. Essentially, each component is customized by its contents, which all have unique keywords. For example, the product component has a “price” element. To customize this element, you would use the keyword “price.”</p> 
             <p>In BuyButton.js logic, everything is customized with big custom objects, very similar to how you would configure a JQuery plugin. For each element that gets created by the Buy Button UI library, you can customize the following:</p> 
             <ul> 
              <li>The contents of each element</li> 
              <li>The order in which the elements appear in</li> 
              <li>Styles and text</li> 
              <li>The templates that change the HTML components (Mustache templates)</li> 
             </ul>
