        <p>In the <a data-macro-name="sp-richtextbody-link" href="/k10/developing-websites/developing-sites-using-asp-net-mvc/starting-with-mvc-development/installing-kentico-integration-packages" class="conf-macro output-inline" data-hasbody="true"><strong>Kentico.Ecommerce</strong> integration package</a>, you can find supporting models and services to build a shopping cart in your on-line store. Visitors can put products into shopping carts and after their browsing through your store, they can continue with the <a data-macro-name="sp-plaintextbody-link" href="/k10/developing-websites/developing-sites-using-asp-net-mvc/developing-on-line-stores-in-mvc/checkout-process-in-mvc" class="conf-macro output-inline" data-hasbody="true">checkout process</a> to create an <a data-macro-name="sp-plaintextbody-link" href="/k10/e-commerce-features/managing-your-store/orders" class="conf-macro output-inline" data-hasbody="true">order</a> and become a <a data-macro-name="sp-plaintextbody-link" href="/k10/e-commerce-features/managing-your-store/customers" class="conf-macro output-inline" data-hasbody="true">customer</a>.</p> 
        <h2 id="UsingashoppingcartonMVCsites-CreatingashoppingcartonMVCwebsites">Creating a shopping cart on MVC websites</h2> 
        <p>When a visitor opens your website, you need to create a shopping cart instance and assign the instance to the customer. To create a shopping cart, use the <strong style="line-height: 1.42857;">Kentico.Ecommerce</strong> integration package.</p> 
        <ol> 
         <li><p>Open your MVC project in Visual Studio.</p></li> 
         <li><p>Create a controller for the shopping cart.</p> 
          <ul> 
           <li><p>In all documentation examples regarding the checkout process, the <strong>CheckoutController</strong> is shared by all checkout process steps.</p> 
            <div class="code panel pdl conf-macro output-block" data-hasbody="true" data-macro-name="sp-macrooverride-plaintextbody-block"> 
             <div class="codeContent panelContent pdl"> 
              <pre><code class="language-csharp">public class CheckoutController : Controller
{
}</code></pre> 
             </div> 
            </div></li> 
          </ul></li> 
         <li><p>Initialize the <strong>ShoppingService, PricingService, KenticoPaymentMethodRepository</strong><span>,</span><strong> KenticoCustomerAddressRepository</strong>,<span> and <strong>Kentico</strong></span><strong>ShippingOptionRepository</strong><span> classes</span> that connects to the shopping cart in Kentico.</p> 
          <div class="box-general box-tip conf-macro output-block" data-hasbody="true" data-macro-name="sp-macrooverride-richtextbody-block"> 
           <div class="box-icon-tip"> 
           </div> 
           <div class="box-content"> 
            <p>We recommend using a <a data-macro-name="sp-plaintextbody-link" href="/k10/developing-websites/developing-sites-using-asp-net-mvc/developing-mvc-applications/initializing-kentico-services-with-dependency-injection" class="conf-macro output-inline" data-hasbody="true">dependency injection container</a> to initialize service instances. When configuring the lifetime scope for the services and repositories, create a separate instance for each request.</p> 
           </div> 
          </div><p></p> 
          <div class="code panel pdl conf-macro output-block conf-macro output-inline" data-hasbody="true" data-macro-name="code-pro"> 
           <div class="codeContent panelContent pdl"> 
            <pre><code class="language-csharp">        private readonly IShoppingService shoppingService;
        private readonly IPricingService pricingService;
        private readonly IPaymentMethodRepository paymentRepository;
        private readonly ICustomerAddressRepository addressRepository;
        private readonly IShippingOptionRepository shippingOptionRepository;
        
        /// &lt;summary&gt;
        /// Constructor.
        /// You can use a dependency injection container to initialize the services and repositories.
        /// &lt;/summary&gt;
        public CheckoutController()
        {
            shoppingService = new ShoppingService();
            pricingService = new PricingService();
            paymentRepository = new KenticoPaymentMethodRepository();
            addressRepository = new KenticoCustomerAddressRepository();
            shippingOptionRepository = new KenticoShippingOptionRepository();
        }
</code></pre> 
           </div> 
          </div><p></p></li> 
        </ol> 
        <p>After initializing the services and repositories, your MVC application can use the shopping cart and other steps of the <a data-macro-name="sp-plaintextbody-link" href="/k10/developing-websites/developing-sites-using-asp-net-mvc/developing-on-line-stores-in-mvc/checkout-process-in-mvc" class="conf-macro output-inline" data-hasbody="true">checkout process</a> from Kentico. The shopping cart processes actions that <a data-macro-name="sp-plaintextbody-link" href="/k10/e-commerce-features/managing-your-store/customers" class="conf-macro output-inline" data-hasbody="true">customers</a> require when using your on-line store. Namely, <a data-macro-name="sp-plaintextbody-link" href="/k10/e-commerce-features/managing-your-store/customers" class="conf-macro output-inline" data-hasbody="true">customers</a> can expect the following actions that you need to implement:</p> 
        <ul> 
         <li><a data-macro-name="sp-plaintextbody-link" href="/k10/developing-websites/developing-sites-using-asp-net-mvc/developing-on-line-stores-in-mvc/checkout-process-in-mvc/using-a-shopping-cart-on-mvc-sites#UsingashoppingcartonMVCsites-Displayingtheshoppingcart" class="conf-macro output-inline" data-hasbody="true">Display the shopping cart content</a></li> 
         <li><a data-macro-name="sp-plaintextbody-link" href="/k10/developing-websites/developing-sites-using-asp-net-mvc/developing-on-line-stores-in-mvc/checkout-process-in-mvc/using-a-shopping-cart-on-mvc-sites#UsingashoppingcartonMVCsites-Addingaproducttotheshoppingcart" class="conf-macro output-inline" data-hasbody="true">Add a product to the shopping cart</a></li> 
         <li><a data-macro-name="sp-plaintextbody-link" href="/k10/developing-websites/developing-sites-using-asp-net-mvc/developing-on-line-stores-in-mvc/checkout-process-in-mvc/using-a-shopping-cart-on-mvc-sites#UsingashoppingcartonMVCsites-Updatingashoppingcartitem" class="conf-macro output-inline" data-hasbody="true">Update an item (a product) already in the shopping cart</a></li> 
         <li><a data-macro-name="sp-plaintextbody-link" href="/k10/developing-websites/developing-sites-using-asp-net-mvc/developing-on-line-stores-in-mvc/checkout-process-in-mvc/using-a-shopping-cart-on-mvc-sites#UsingashoppingcartonMVCsites-Usingmethodsandpropertiesfromtheclassrepresentingitemsintheshoppingcart" class="conf-macro output-inline" data-hasbody="true">Remove an item (a product) from the shopping cart</a></li>

        <h3 id="UsingashoppingcartonMVCsites-Displayingtheshoppingcart">Displaying the shopping cart</h3> 
        <p>To the checkout controller, add a method that retrieves the current visitor's shopping cart and displays it.</p> 
        <div class="code panel pdl conf-macro output-block conf-macro output-inline" data-hasbody="true" data-macro-name="code-pro"> 
         <div class="codeContent panelContent pdl"> 
          <pre><code class="language-csharp">        /// &lt;summary&gt;
        /// Displays the current site's shopping cart.
        /// &lt;/summary&gt;
        public ActionResult ShoppingCart()
        {
            // Initializes the shopping cart model
            ShoppingCartViewModel model = new ShoppingCartViewModel();
            
            // Gets the current user's shopping cart
            model.Cart = shoppingService.GetCurrentShoppingCart();
            model.RemainingAmountForFreeShipping = pricingService.CalculateRemainingAmountForFreeShipping(model.Cart);
            
            // Displays the shopping cart
            return View(model);
        }
</code></pre> 
         </div> 
        </div> 
        <p></p> 
        <div class="box-general box-tip conf-macro output-block" data-hasbody="true" data-macro-name="sp-macrooverride-richtextbody-block"> 
         <div class="box-icon-tip"> 
         </div> 
         <div class="box-content"> 
          <p>If you do not want to use coupon codes, you do not need to create a model for your shopping cart. You can just display the shopping cart and adjust the view accordingly:</p> 
          <div class="code panel pdl conf-macro output-block" data-hasbody="true" data-macro-name="sp-macrooverride-plaintextbody-block"> 
           <div class="codeContent panelContent pdl"> 
            <pre><code class="language-csharp">return View(cart);</code></pre> 
           </div> 
          </div> 
         </div> 
        </div> 
        <p>In the shopping cart's view, you can then use the shopping cart's properties to display the content of the shopping cart. For example:</p> 
        <div class="code panel pdl conf-macro output-block conf-macro output-inline" data-hasbody="true" data-macro-name="code-pro"> 
         <div class="codeContent panelContent pdl"> 
          <pre><code class="language-csharp">@if (Model.Cart.IsEmpty)
{
    &lt;span&gt;Your shopping cart is empty.&lt;/span&gt;
}
else
{
    &lt;ul&gt;
        @* Loops through all shopping cart items *@
        @foreach (Kentico.Ecommerce.ShoppingCartItem cartItem in Model.Items)
        {
            @* Displays the shopping cart items' properties *@
            &lt;li&gt;
                @cartItem.Units&amp;times; @cartItem.Name ... @currency.FormatPrice(cartItem.Subtotal)
&nbsp;
                @using (Html.BeginForm("RemoveItem", "Checkout", FormMethod.Post))
                {
                    @Html.Hidden("ItemId", cartItem.ID)
                    &lt;input type="submit" value="Remove" /&gt;
                }
            &lt;/li&gt;
        }
    &lt;/ul&gt;
}
</code></pre> 
         </div> 
        </div>

        <h3 id="UsingashoppingcartonMVCsites-Addingaproducttotheshoppingcart">Adding a product to the shopping cart</h3> 
        <p>To the checkout controller, add a POST method that adds a <a data-macro-name="sp-plaintextbody-link" href="/k10/e-commerce-features/managing-your-store/products" class="conf-macro output-inline" data-hasbody="true">product</a> to the shopping cart. For example, the system can trigger the method after a visitor clicks the <em>Add to cart</em> button.</p> 
        <div class="code panel pdl conf-macro output-block conf-macro output-inline" data-hasbody="true" data-macro-name="code-pro"> 
         <div class="codeContent panelContent pdl"> 
          <pre><code class="language-csharp">        /// &lt;summary&gt;
        /// Adds products to the current site's shopping cart.
        /// &lt;/summary&gt;
        /// &lt;param name="itemSkuId"&gt;ID of the added product (its SKU object).&lt;/param&gt;
        /// &lt;param name="itemUnits"&gt;Number of added units.&lt;/param&gt;
        [HttpPost]
        public ActionResult AddItem(int itemSkuId, int itemUnits)
        {
            // Gets the current user's shopping cart
            ShoppingCart cart = shoppingService.GetCurrentShoppingCart();
            
            // Adds a specified number of units of a specified product
            cart.AddItem(itemSkuId, itemUnits);
            
            // Displays the shopping cart
            return RedirectToAction("ShoppingCart");
        }
</code></pre>

        <h3 id="UsingashoppingcartonMVCsites-Updatingashoppingcartitem">Updating a shopping cart item</h3> 
        <p>To the checkout controller, add a POST method that changes the number of units of a shopping cart item. For example, the system can trigger the method after a visitor clicks the <em>Update</em> button in the shopping cart.</p> 
        <div class="code panel pdl conf-macro output-block conf-macro output-inline" data-hasbody="true" data-macro-name="code-pro"> 
         <div class="codeContent panelContent pdl">

        /// &lt;/summary&gt;
        /// &lt;param name="itemID"&gt;ID of the updated shopping cart item.&lt;/param&gt;
        /// &lt;param name="itemUnits"&gt;Result number of units of the shopping cart item.&lt;/param&gt;
        [HttpPost]
        public ActionResult UpdateItem(int itemID, int itemUnits)
        {
            // Gets the current user's shopping cart
            ShoppingCart cart = shoppingService.GetCurrentShoppingCart();
            
            // Updates a specified product with a specified number of units
            cart.UpdateQuantity(itemID, itemUnits);
            
            // Displays the shopping cart
            return RedirectToAction("ShoppingCart");
        }

        <p>To the checkout controller, add a POST method that removes a shopping cart item from the shopping cart. For example, the system can trigger the method after a visitor clicks the <em>Remove</em> button in the shopping cart.</p> 
        <div class="code panel pdl conf-macro output-block conf-macro output-inline" data-hasbody="true" data-macro-name="code-pro">

        /// &lt;/summary&gt;
        /// &lt;param name="itemID"&gt;ID of the removed shopping cart item.&lt;/param&gt;
        [HttpPost]
        public ActionResult RemoveItem(int itemID)
        {
            // Gets the current user's shopping cart
            ShoppingCart cart = shoppingService.GetCurrentShoppingCart();

            // Displays the shopping cart

        <h3 id="UsingashoppingcartonMVCsites-Checkingout">Checking out</h3> 
        <p>To the checkout controller, add a method that enables the visitor to continue with the checkout process to the next step to make an order. You need to validate the shopping cart first.</p> 
        <p><span>The next step is usually a step where the visitor/customer types their details, see <a data-macro-name="sp-nobody-link" href="/k10/developing-websites/developing-sites-using-asp-net-mvc/developing-on-line-stores-in-mvc/checkout-process-in-mvc/creating-the-customer-details-step-in-mvc-checkout-processes" class="conf-macro output-inline" data-hasbody="false">Creating the customer details step in MVC checkout processes</a>.</span></p> 
        <div class="code panel pdl conf-macro output-block conf-macro output-inline" data-hasbody="true" data-macro-name="code-pro"> 
         <div class="codeContent panelContent pdl"> 
          <pre><code class="language-csharp">        /// &lt;summary&gt;
        /// Validates the shopping cart and proceeds to the next checkout step with customer details.
        /// &lt;/summary&gt;
        [HttpPost]
        [ActionName("ShoppingCart")]
        public ActionResult ShoppingCartCheckout()
        {
            // Gets the current user's shopping cart
            ShoppingCart cart = shoppingService.GetCurrentShoppingCart();
            
            // Validates the shopping cart
            ShoppingCartCheckResult checkResult = cart.ValidateContent();
            
            // If the validation is successful, redirects to the next step of the checkout process
            if (!checkResult.CheckFailed)
            {
                return RedirectToAction("DeliveryDetails");
            }
            
            // If the validation fails, redirects back to shopping cart
            return RedirectToAction("ShoppingCart");
        }
</code></pre> 
         </div>
