      <h2 class="entry-header">What JavaScript Means for Ecommerce Merchants</h2> 
     </div>
     <!-- end .row --> 
     <div class="row content-pad"> 
      <div class="column col8"> 
       <div class="entry-meta-con"> 
        <div class="entry-meta meta-detail"> 
         <h6>April 19, 2016 • <a rel="author" title="Posts by Eric Davis" href="http://www.practicalecommerce.com/author/eric-davis">Eric Davis</a></h6> 
        </div>
        <!-- end .entry-meta --> 
        <div class="pec-social-post"> 
         <ul> 
          <li><a data-via="practicalecomm" data-url="" target="_blank" data-size="small" data-dir="left" href="http://twitter.com/share?text=What JavaScript Means for Ecommerce Merchants&amp;url=http://pec-ly.com/?vMN&amp;via=practicalecomm" title="Click to share this post on Twitter"><i class="fa fa-twitter"></i></a></li> 
          <li><a target="_blank" href="http://www.facebook.com/sharer.php?u=http://www.practicalecommerce.com/What-JavaScript-Means-for-Ecommerce-Merchants&amp;t=What JavaScript Means for Ecommerce Merchants"><i class="fa fa-facebook"></i></a></li> 
          <li><a href="mailto:?subject=What%20JavaScript%20Means%20for%20Ecommerce%20Merchants&amp;body=http%3A%2F%2Fwww.practicalecommerce.com%2FWhat-JavaScript-Means-for-Ecommerce-Merchants" title="Send this article to a friend!"><i class="fa fa-envelope"></i></a></li> 
          <li class="print-article"><a rel="nofollow" href="javascript:void(0)" onclick="(['Click','Print Page','Page: Article']);window.print()"><i class="fa fa-print"></i></a></li> 
          <li> </li> 
         </ul> 
        </div>

       </div>
       <!-- end .entry-meta-con --> 
       <div class="entry-content"> 
        <p>Web developers use three core tools when building ecommerce sites: HTML for content and organization, CSS for the design and presentation, and JavaScript for the interaction with backend servers.</p> 
        <p>This article, addressing JavaScript, is the third installment in my series on those tools, following “<a href="http://www.practicalecommerce.com/articles/97359-What-HTML5-Means-for-Ecommerce-Merchants">What HTML5 Means for Ecommerce Merchants</a>” and “<a href="http://www.practicalecommerce.com/articles/97997-What-CSS-Means-for-Ecommerce-Merchants">What CSS Means for Ecommerce Merchants</a>.”</p> 
        <p>JavaScript is a complete programming language that is primarily used on the web. It’s powerful enough to use for servers and other applications, but until recently those uses were rare. In this article, I’ll focus on JavaScript’s role in websites, which is called client-side JavaScript. In web development, the browser is called the client. JavaScript runs in the web browser.</p> 
        <p>There are two main roles for JavaScript within ecommerce stores:</p> 
        <ul> 
         <li>Adding interactivity to a web page;</li> 
         <li>Connecting to backend servers.</li> 
        </ul> 
        <p>There is also a supplementary role: customer tracking and analytics.</p> 
        <h3>Why Interactivity?</h3> 
        <p>Since its introduction on websites, JavaScript has been adding dialog between humans and computer programs. Back in 2005, for example, I used JavaScript for flyout menus. It was interactive in that the flyout didn’t occur without a human first moving her mouse over, say, a category name.</p> 
        <p>The point of adding interactivity to a web page is so that users can see changes based on their actions. Without it, there isn’t much happening on a page: a user can only scroll, click, and fill out forms. It makes it difficult to engage users — shoppers — and keep their attention.</p> 
        <h3>Adding Interactivity with JavaScript</h3> 
        <p>Using JavaScript, merchants can engage their visitors.</p> 
        <p>Product images can change when the mouse hovers on them. Clicking on an image could zoom in and show a higher-resolution version, or slowly rotate the product 360 degrees.</p> 
        <p>As a shopper is typing into a search form, common search terms can appear automatically and help guide the shopper’s search to one that has more results, or correct common typos.</p> 
        <p>Something as simple as having multilevel navigational flyout menus can add interactivity, which improves a visitor’s experience by helping her to quickly drill down to appropriate product categories. (Interestingly, recent versions of CSS have added some JavaScript-like features for interactivity.)</p> 
        <p>While using JavaScript for a more interactive web experience is popular, there is another use that has become common with ecommerce stores: backend integration.</p> 
        <h3>Better Backend Integration Using AJAX</h3> 
        <p>Using JavaScript, a visitor could interact with a page. But if he wanted to load another page or complete a new form, he’d have to wait for the server to respond with a new page or a new form.</p> 
        <p>While this worked okay, it wasn’t very smooth. It was frustrating, for example, to change shopping cart quantities for five items and remove a few others. Each change would take a few seconds to load. Shoppers would often become annoyed and leave.</p> 
        <p>Enter AJAX — Asynchronous JavaScript and XML — which was developed around 2005. AJAX allowed web applications to send data to and retrieve from a server asynchronously (in the background) without interfering with the display and behavior of the existing page. Instead of the user performing an action and then waiting for a new page to appear, she could perform an action and AJAX would communicate with the server in the background and then update the existing page automatically.</p> 
        <p>This opened up many interactions that are now common:</p> 
        <ul> 
         <li>Updating an item in a cart and seeing the change right away;</li> 
         <li>Search suggestions and results that appear as you type;</li> 
         <li>New email messages appearing automatically in Gmail;</li> 
         <li>Ability to drag and drop elements of a web page;</li> 
         <li>Forms that notify you of problems as you fill them out.</li> 
        </ul> 
        <p>AJAX enabled JavaScript to expand its capabilities, so interactions were more than just visual changes and hints. They affected how a store functions.</p> 
        <p>Now, because of AJAX, modern ecommerce applications are full of these dynamic, backend-supported interactions.</p> 
        <h3>Web Tracking and Analytics</h3> 
        <p>A third use of JavaScript is web tracking and analytics. Not only does JavaScript provide real-time tracking of page views, but it also allows you to see how far down a page people are reading (heatmaps), where their mouse is, and what they try to click on (click tracking). JavaScript can track on-site events — notifying backend servers about a visitor’s behavior and taking some action around it, such as loading an abandoned cart.</p> 
        <p>Most ecommerce sites will not build their own tracking and analytics system, given that there are so many powerful ones already. But many merchants do use JavaScript to customize and log specific events.</p> 
        <h3>Disadvantages, Costs of JavaScript</h3> 
        <p>JavaScript provides ecommerce sites a powerful tool to improve their shoppers’ experiences. But it’s not without a cost.</p> 
        <p>For example, seemingly simple interactions on the frontend can become complex underneath. Say you want your search form to automatically show the first five results. Simple, right? Here’s what is actually involved.</p> 
        <ul> 
         <li>Add JavaScript to wait for a user to start typing in a field.</li> 
        </ul> 
        <ul> 
         <li>After three characters, make an AJAX request to the server with the search term.</li> 
        </ul> 
        <ul> 
         <li>On the server, perform your search function with that search term.</li> 
        </ul> 
        <ul> 
         <li>Format the search results as an HTML snippet or a JSON (a method of exchanging data) structure so it can be returned back to the search form’s JavaScript.</li> 
        </ul> 
        <ul> 
         <li>Once the search form gets the search results, display them nicely using CSS and make sure they appear on top of the page.</li> 
        </ul> 
        <ul> 
         <li>Cancel the AJAX requests and any server processing when a user continues to type more characters — four, five, six, and so on — in the search field.</li> 
        </ul> 
        <ul> 
         <li>Facilitate and communicate instances when there are no search results found on the server.</li> 
        </ul> 
        <ul> 
         <li>Facilitate and communicate server errors.</li> 
        </ul> 
        <ul> 
         <li>Add suggestions to the search, so the user can see popular searches.</li> 
        </ul> 
        <ul> 
         <li>Correct the user’s typos and misspellings.</li> 
        </ul> 
        <p>I’ll stop there. There are many&nbsp;more steps that even a simple search would need to do. Good developers understand this and are able to describe these additional parts to build interactivity.</p> 
        <p>Also, you might have noticed that while we were building a JavaScript component, it required:</p> 
        <ul> 
         <li>Changes on the server to perform the search;</li> 
         <li>HTML to display the search;</li> 
         <li>CSS to present the results in a visually appealing manner.</li> 
        </ul> 
        <p>Coordinating all of these means JavaScript development can become more difficult&nbsp;than it appears.</p> 
        <h3>JavaScript Libraries Help and Hurt</h3> 
        <p>To address the complexity, there has been a huge growth in code libraries and reusable JavaScript components. While building a live search would take all of the work described above, if a developer uses a live search code library, she might be able to cut 75 percent of that work.</p> 
        <p>While code libraries are appealing, they come with their own disadvantages. Some can slow down pages when they are used. Some libraries provide hundreds of functions, even though you only need one or two. Even knowing which library to use can be difficult as there are so many choices.</p> 
        <h3>JavaScript: Powerful Tool in the Right Hands</h3> 
        <p>For the modern ecommerce store, using JavaScript is an easy decision. Most merchants are already using it, and rightly so. It’s a powerful tool and it provides valuable improvements to websites.</p> 
        <p>Just be careful with how much you use it. It’s easy to overuse and damage your site. To get the most out of JavaScript:</p> 
        <ul> 
         <li>Hire seasoned developers who know JavaScript well;</li> 
         <li>Carefully weigh the full costs and impacts of any changes made with JavaScript.</li> 
        </ul> 
        <p>As you use the web, note the interactions that surprise and delight you. Some will be subtle and some not. But JavaScript will power almost all of them. Some of them, in fact, you might be able to use in your own ecommerce site.</p>
