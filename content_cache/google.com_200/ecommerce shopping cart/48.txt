             <h2>Introduction</h2> 
             <p>If you want to create any kind of on-line store, you will probably need a shopping cart.</p> 
             <p>A shopping cart is basically a list that contains products selected by the user while he shops. When the user is finished shopping, he will usually examine the list to double-check that the items, quantities, and prices are correct. If he finds any errors he should be able to edit the list. Once he is ready, he should be able to check out. The checkout process involves an exchange of information that results in a sale.</p> 
             <p>Sounds simple, right? And it actually is. The only challenge is performing the checkout, because that involves personal information and money. Fortunately, there are services that handle this type of transaction and you can leverage them. Some of the most popular are <strong>PayPal</strong> and <strong>Google Wallet</strong>.</p> 
             <p>This article describes the implementation of a shopping cart using JavaScript. The cart uses <strong>PayPal</strong> and <strong>Google Wallet</strong> payment services. Adding other providers is fairly easy. If you have your own payment infrastructure for example, you can extend the shopping cart to use that in addition to the <strong>PayPal</strong> and <strong>Google Wallet</strong> options. Offering more payment options should increase sales.</p> 
             <p>The article includes a sample application called “Angular Store” that demonstrates how to use the shopping cart in <strong>AngularJS</strong> applications.&nbsp;</p> 
             <h2>Shopping Cart Requirements</h2> 
             <p>When I started developing the shopping cart, I had the following requirements in mind:</p> 
             <ul> 
              <li>Must be 100% pure JavaScript (so it is easy to integrate into any site)</li> 
              <li>Must follow the MVVM architecture (so it is easy to customize its look and feel)</li> 
              <li>Must be safe (we don't want to be responsible for storing people's credit card numbers etc.)</li> 
              <li>Must be fast and reliable (we don’t want users to give up before they checkout!)</li> 
              <li>Must be flexible (it should allow payments to be processed using different services)&nbsp;</li> 
              <li>Must be extensible (adding new payment methods should be easy)</li> 
              <li>Must be easy to use (because there's no reason for it to be complicated)</li> 
             </ul> 
             <p>I believe the “shoppingCart” class described below addresses all these requirements. It uses <strong>jQuery</strong> and integrates well with <strong>AngularJS</strong> applications. The “shoppingCart” class contains all the logic and provides the object model needed to create flexible and attractive views.</p> 
             <h2>The Angular Store Sample Application</h2> 
             <p>To understand how the cart works, let’s take a quick look at a typical application. The Angular Store app has three main views:</p> 
             <p><strong>Store</strong>: This is the main view. It presents a list of the products available. Users can search for items using a filter, get detailed information about specific products by clicking their names, add products to the shopping cart, and see a quick summary of what is in their cart. Clicking the summary navigates to the cart. This is what the store view looks like:</p> 
             <p><img height="405" src="/KB/E-commerce/576246/shoppingcart1.jpg" width="550"></p> 
             <p><strong>Product Details:</strong> This view shows details about a product and allows users to add or remove the product to/from the shopping cart. The view also presents a quick summary of the cart so users can tell whether this product is already in the cart. This is what the product details view looks like:</p> 
             <p><img height="405" src="/KB/E-commerce/576246/shoppingcart2.jpg" width="550"></p> 
             <p><strong>Shopping Cart</strong>: This view shows the shopping cart. Users can edit the cart and checkout using <strong>PayPal</strong> or <strong>Google Wallet</strong>. Offering more payment options tends to increase sales, because some users may have accounts with one service or the other. This is what the shopping cart view looks like:</p>

             <h2>AngularJS Infrastructure</h2> 
             <p>The sample application starts with the definition of an <strong>AngularJS</strong> module that represents the application. The module <strong>AngularStore</strong> module is defined in the <strong>app.js</strong> file as follows:</p> 
             <pre lang="jscript"><span class="code-comment">//</span><span class="code-comment"> App Module: the name AngularStore matches the ng-app attribute</span>
<span class="code-comment">//</span><span class="code-comment"> in the main &lt;html&gt; tag. The route provides parses the URL and</span>
<span class="code-comment">//</span><span class="code-comment"> injects the appropriate partial page</span>
<span class="code-keyword">var</span> storeApp = angular.module(<span class="code-string">'</span><span class="code-string">AngularStore'</span>, []).
  config([<span class="code-string">'</span><span class="code-string">$routeProvider'</span>, <span class="code-keyword">function</span>($routeProvider) {
  $routeProvider.
    when(<span class="code-string">'</span><span class="code-string">/store'</span>, { 
      templateUrl: <span class="code-string">'</span><span class="code-string">partials/store.htm'</span>,
      controller: storeController }).
    when(<span class="code-string">'</span><span class="code-string">/products/:productSku'</span>, {
      templateUrl: <span class="code-string">'</span><span class="code-string">partials/product.htm'</span>,
      controller: storeController }).
    when(<span class="code-string">'</span><span class="code-string">/cart'</span>, { 
      templateUrl: <span class="code-string">'</span><span class="code-string">partials/shoppingCart.htm'</span>,
      controller: storeController }).
    otherwise({
      redirectTo: <span class="code-string">'</span><span class="code-string">/store'</span> });
}]);</pre> 
             <p>This first block of code defines the <strong>storeApp</strong> object that represents the application. It contains a <strong>routeProvider</strong> that specifies which view should be displayed based on the URL.</p> 
             <p>For example, when the URL ends with “/cart”, the app should display the view defined in the “partials/shoppingCart.htm” file. The view should be bound to a controller of type “storeController”.</p> 
             <p>When the URL ends with “/product/:productSku”, the app should display the view defined in the “partials/product.htm” file. The view should be bound to a controller of the same type “storeController”. In this case, the “/:productSku” represents a variable parameter used to identify the product being shown. It will be replaced at runtime with an actual product code.</p> 
             <p>In this case, all views have the same type of controller, a class that contains a “store” and a “cart”.</p> 
             <p>Because in this case all views refer to the same store and cart, it makes sense to create these data objects once, at the app level, and allow all controllers to use them. This will improve performance because it eliminates the need to re-load the store and cart items whenever a new view is displayed.</p> 
             <p>The easiest way to share data between controllers in <strong>AngularJS</strong> is by defining an app-level “service”, and later using this service to initialize the controllers that need them. This links shows a simple example that illustrates the concept: <a href="http://egghead.io/video/angularjs-sharing-data-between-controllers/"> http://egghead.io/video/angularjs-sharing-data-between-controllers/</a></p> 
             <p>Here is the definition of the “DataService” that provides data shared by all views in the Angular Store application:</p> 
             <pre lang="jscript"><span class="code-comment">//</span><span class="code-comment"> create a data service that provides a store and a shopping</span>
<span class="code-comment">//</span><span class="code-comment"> cart that will be shared by all views</span>
<span class="code-comment">//</span><span class="code-comment"> (instead of creating fresh ones for each view).</span>
storeApp.factory(<span class="code-string">"</span><span class="code-string">DataService"</span>, <span class="code-keyword">function</span>() {

  myCart.addCheckoutParameters(<span class="code-string">"</span><span class="code-string">Google"</span>, <span class="code-string">"</span><span class="code-string"><em>your Google merchant account id</em> "</span>, {

    ship_method_price_1: <span class="code-string">"</span><span class="code-string">20.00"</span>,
    ship_method_currency_1: <span class="code-string">"</span><span class="code-string">USD"</span>,
    ship_method_name_2: <span class="code-string">"</span><span class="code-string">UPS Ground"</span>,
    ship_method_price_2: <span class="code-string">"</span><span class="code-string">15.00"</span>,
    ship_method_currency_2: <span class="code-string">"</span><span class="code-string">USD"</span>
  });
  <span class="code-keyword">return</span> {

    cart: myCart
  };
});</pre> 
             <p>The service creates a “store” object that contains a list of the products available and a “shoppingCart” object that represents the shopping cart.</p> 
             <p>When the “shoppingCart” object is created, it automatically loads its contents from local storage, so users can add items to the cart, close the application, and continue shopping later on.</p> 
             <p>After creating the cart, the service configures the cart’s checkout parameters. In this example, the cart provides two checkout options:</p> 
             <ol> 
              <li>The “PayPal” option specifies the merchant account to use for checking out, and has no additional options. To use the shopping cart with PayPal, you have to create a merchant account with PayPal. You can do that here: <a href="https://www.paypal.com/webapps/mpp/merchant"> https://www.paypal.com/webapps/mpp/merchant</a></li> 
              <li>The “Google” option specified the merchant account and additional options related to shipping charges. To use the shopping cart with Google Wallet, you have to create a merchant account with Google.              </ol> 
             <p>Once the data service has been created, it can be used by the “storeController” objects that will drive all the views in the application. This is done in the <strong>controller.js</strong> file:</p> 
             <pre lang="jscript"><span class="code-keyword">function</span> storeController($scope, $routeParams, DataService) {

  <span class="code-comment">//</span><span class="code-comment"> get store and cart from service</span>
  $scope.store = DataService.store;
  $scope.cart = DataService.cart;

  <span class="code-comment">//</span><span class="code-comment"> use routing to pick the selected product</span>

  }

             <p>The “storeController” function retrieves the store and cart from the “DataService” discussed earlier and adds them to the AngularJS <strong>$scope</strong> object. The <strong>$scope</strong> object works as a data context for the view.</p> 
             <p>The <strong>app.js</strong> and <strong>controller.js</strong> files contain all the <strong>AngularJS</strong> part of the application code.              <h2>The ‘shoppingCart’ class</h2> 
             <p>The “shoppingCart” class is the most interesting class in the project. It is defined in the <strong>shoppingCart.js</strong> file and implements an object model as follows:</p> 
             <h3>shoppingCart(cartName)</h3> 
             <p>This is the constructor.</p> 
             <p>The <strong>cartName</strong> parameter identifies the cart when saving it to or loading it from local storage.&nbsp; Before you can actually use the cart for checkout operations, you must initialize it by adding one or more payment providers. This is done with the <strong>addCheckoutParameters</strong> method.</p> 
             <h3>addCheckoutParameters(serviceName, merchantID, [options])</h3> 
             <p>This method defines a set of checkout parameters.</p> 
             <p>The <strong>serviceName</strong> parameter defines the name of the payment provider to use. In the current implementation, this must be set to either “PayPal” or “Google”.</p> 
             <p>The <strong>merchantID</strong> parameter specifies the merchant account associated with the service. You can create PayPal and Google merchant accounts using these links:<br> <strong>PayPal</strong>: <a href="https://www.paypal.com/webapps/mpp/merchant"> https://www.paypal.com/webapps/mpp/merchant</a><br> <strong>Google</strong>: <a href="https://developers.google.com/commerce/wallet/digital/training/getting-started/merchant-setup"> https://developers.google.com/commerce/wallet/digital/training/getting-started/merchant-setup</a>&nbsp;</p> 
             <p>The <strong>options</strong> parameter defines additional provider-specific fields. In our example, we used this parameter to specify custom shipping methods associated with the Google checkout. Both PayPal and Google support a large number of optional parameters that you can use to customize the checkout process.</p> 
             <h3>addItem(sku, name, price, quantity)</h3> 
             <p>This method adds or removes items from the cart.</p> 
             <p>If the cart already contains items with the given <strong>sku</strong>, then the quantity of that item is modified. If the quantity reaches zero, the item is automatically removed from the cart.</p> 
             <p>If the cart does not contain items with the given <strong>sku</strong>, then a new item is created and added to the cart using the specified <strong>sku</strong>, <strong>name</strong>, <strong>price</strong>, and <strong>quantity</strong>.</p> 
             <p>After the cart has been updated, it is automatically saved to local storage.</p> 
             <h3>clearItems()</h3> 
             <p>This method clears the cart by removing all items. It also saves the empty cart to local storage.</p> 
             <h3>getTotalCount([sku])</h3> 
             <p>This method gets the quantity of items or a given type or for all items in the cart.</p> 
             <p>If the <strong>sku</strong> is provided, then the method returns the quantity of items with that <strong>sku</strong>. It the <strong>sku</strong> is omitted, then the method returns the quantity of all items in the cart.</p> 
             <h3>getTotalPrice([sku])</h3> 
             <p>This method gets the total price (unit price * quantity) for one or all items in the cart.</p> 
             <p>If the <strong>sku</strong> is provided, then the method returns the price of items with that <strong>sku</strong>. It the <strong>sku</strong> is omitted, then the method returns the total price of all items in the cart.</p> 
             <h3>checkout([serviceName], [clearCart])</h3> 
             <p>This method initiates a checkout transaction by building a form object and submitting it to the specified payment provider.</p> 
             <p>If provided, the <strong>serviceName</strong> parameter must match one of the service names registered with calls to the <strong>addCheckoutParameters</strong> method. If omitted, the cart will use the first payment service registered. The <strong>clearCart</strong> parameter specifies whether the cart should be cleared after the <strong>checkout</strong> transaction is submitted.</p> 
             <p>The <strong>checkout</strong> method is the most interesting in this class, and is listed below:</p> 
             <pre lang="jscript">shoppingCart.prototype.checkout = <span class="code-keyword">function</span> (serviceName, clearCart) {
  <span class="code-comment">//</span><span class="code-comment"> select service</span>
  <span class="code-keyword">if</span> (serviceName == <span class="code-keyword">null</span>) {
    <span class="code-keyword">var</span> p = <span class="code-keyword">this</span>.checkoutParameters[Object.keys(<span class="code-keyword">this</span>.checkoutParameters)[<span class="code-digit">0</span>]];
    serviceName = p.serviceName;
  }
  <span class="code-keyword">if</span> (serviceName == <span class="code-keyword">null</span>) {
    <span class="code-keyword">throw</span> <span class="code-string">"</span><span class="code-string">Define at least one checkout service."</span>;
  }
  <span class="code-keyword">var</span> parms = <span class="code-keyword">this</span>.checkoutParameters[serviceName];
  <span class="code-keyword">if</span> (parms == <span class="code-keyword">null</span>) {
    <span class="code-keyword">throw</span> <span class="code-string">"</span><span class="code-string">Cannot get checkout parameters for '"</span> + serviceName + <span class="code-string">"</span><span class="code-string">'."</span>;
  }

  <span class="code-comment">//</span><span class="code-comment"> invoke service</span>
  <span class="code-keyword">switch</span> (parms.serviceName) {
    <span class="code-keyword">case</span> <span class="code-string">"</span><span class="code-string">PayPal"</span>:
      <span class="code-keyword">this</span>.checkoutPayPal(parms, clearCart);
      <span class="code-keyword">break</span>;
    <span class="code-keyword">case</span> <span class="code-string">"</span><span class="code-string">Google"</span>:
      <span class="code-keyword">this</span>.checkoutGoogle(parms, clearCart);
      <span class="code-keyword">break</span>;
    <span class="code-keyword">default</span>:
      <span class="code-keyword">throw</span> <span class="code-string">"</span><span class="code-string">Unknown checkout service: "</span> + parms.serviceName;
  }
}</pre> 
             <p>The method starts by making sure it has a valid payment service, and then defers the actual work to the <strong>checkoutPayPal</strong> or <strong>checkoutGoogle</strong> methods. These methods are very similar but are service-specific. The <strong>checkoutPayPal</strong> method is implemented as follows:</p> 
             <pre lang="jscript"><span class="code-comment">//</span><span class="code-comment"> check out using PayPal; for details see:</span>
<span class="code-comment">//</span><span class="code-comment"> http://www.paypal.com/cgi-bin/webscr?cmd=p/pdn/howto_checkout-outside</span>
shoppingCart.prototype.checkoutPayPal = <span class="code-keyword">function</span> (parms, clearCart) {

  <span class="code-comment">//</span><span class="code-comment"> global data</span>
  <span class="code-keyword">var</span> data = {
    cmd: <span class="code-string">"</span><span class="code-string">_cart"</span>,
    business: parms.merchantID,
    upload: <span class="code-string">"</span><span class="code-string">1"</span>,
    rm: <span class="code-string">"</span><span class="code-string">2"</span>,
    charset: <span class="code-string">"</span><span class="code-string">utf-8"</span>
  };

  <span class="code-comment">//</span><span class="code-comment"> item data</span>
  <span class="code-keyword">for</span> (<span class="code-keyword">var</span> i = <span class="code-digit">0</span>; i &lt; <span class="code-keyword">this</span>.items.length; i++) {
    <span class="code-keyword">var</span> item = <span class="code-keyword">this</span>.items[i];
    <span class="code-keyword">var</span> ctr = i + <span class="code-digit">1</span>;
    data[<span class="code-string">"</span><span class="code-string">item_number_"</span> + ctr] = item.sku;
    data[<span class="code-string">"</span><span class="code-string">item_name_"</span> + ctr] = item.name;
    data[<span class="code-string">"</span><span class="code-string">quantity_"</span> + ctr] = item.quantity;
    data[<span class="code-string">"</span><span class="code-string">amount_"</span> + ctr] = item.price.toFixed(<span class="code-digit">2</span>);
  }

  <span class="code-comment">//</span><span class="code-comment"> build form</span>
  <span class="code-keyword">var</span> form = $(<span class="code-string">'</span><span class="code-string">&lt;form&gt;&lt;/form&gt;'</span>);
  form.attr(<span class="code-string">"</span><span class="code-string">action"</span>, <span class="code-string">"</span><span class="code-string">https://www.paypal.com/cgi-bin/webscr"</span>);
  form.attr(<span class="code-string">"</span><span class="code-string">method"</span>, <span class="code-string">"</span><span class="code-string">POST"</span>);
  form.attr(<span class="code-string">"</span><span class="code-string">style"</span>, <span class="code-string">"</span><span class="code-string">display:none;"</span>);
  <span class="code-keyword">this</span>.addFormFields(form, data);
  <span class="code-keyword">this</span>.addFormFields(form, parms.options);
  $(<span class="code-string">"</span><span class="code-string">body"</span>).append(form);

  <span class="code-comment">//</span><span class="code-comment"> submit form</span>
  <span class="code-keyword">this</span>.clearCart = clearCart == <span class="code-keyword">null</span> || clearCart;
  form.submit();
  form.remove();
}</pre> 
             <p>The <strong>checkoutPayPal</strong> method builds a form, populates it with hidden input fields that contain the cart data, and submits the form to the <strong>PayPal</strong> servers. The whole process is described here: <a href="https://www.paypal.com/cgi-bin/webscr?cmd=p/pdn/howto_checkout-outside"> https://www.paypal.com/cgi-bin/webscr?cmd=p/pdn/howto_checkout-outside</a></p> 
             <p>The <strong>checkoutGoogle</strong> method is very similar. It also builds and submits a form, the only difference being the name and content of the fields. Details are available here: <a href="https://developers.google.com/checkout/developer/Google_Checkout_Custom_Cart_How_To_HTML"> https://developers.google.com/checkout/developer/Google_Checkout_Custom_Cart_How_To_HTML</a>.</p> 
             <p>Both checkout methods allow you to add custom fields specified in the <strong>options</strong> parameter of the cart’s <strong>addCheckoutParameters</strong> method. These custom fields can be used to specify things like return URLs, custom images for the cart on the server’s site, custom shipping rules and prices, etc.</p> 
             <p>When the <strong>checkout</strong> method submits the form, the user is taken to the appropriate site (PayPal or Google Wallet), where he can review the information about the items, update his own personal and credit card information, and finalize the transaction. All this happens outside the scope of the application. The payment provider will then use the information associated with the merchant id provided by the form to notify you of the transaction so you can collect the payment and ship the goods to the customer.</p> 
             <p>If you wanted to add more payment options to the cart, you would have to:</p> 
             <ol> 
              <li>Modify the <strong>addCheckoutParameters</strong> method to accept the new service name.</li> 
              <li>Create a new <strong>checkout&lt;ServiceName&gt;</strong> method to handle the checkouts using the new service. This would probably be similar to the existing <strong>checkoutPayPal</strong> and <strong>checkoutGoogle</strong> methods.</li> 
              <li>Modify the <strong>checkout</strong> method to call the new method depending on the service name specified by the user.</li> 
             </ol> 
             <p>For example, if you wanted to leverage an existing payment infrastructure you have on your site, you could create a method similar to <strong>checkoutPayPal</strong>, but with a URL on your site. The server would receive the form with all the information encoded as hidden fields, and would have access to the current session, user, etc. At this point, you would have all the information required by your payment infrastructure (cart and user).</p> 
             <h2>AngularJS Views</h2> 
             <p>Now that we have covered the AngularJS infrastructure and the controller classes, let’s turn our attention to the views.</p>

             <p>The view uses <strong>Bootstrap’s</strong> built-in icons, in this case the “icon-shopping-cart” class to enhance the view with simple and attractive icons. <strong>Bootstrap</strong> includes a set of 140 icons that cover a lot of common scenarios (see the complete list here: <a href="http://twitter.github.io/bootstrap/base-css.html#icons"> http://twitter.github.io/bootstrap/base-css.html#icons</a>).</p> 
             <p>The body of the table uses an <strong>ng-repeat</strong> attribute to show a sorted, filtered list of all products.              <p>The body of the table is generated with an <strong>“ng-repeat”</strong> attribute that loops through the items in the <strong>cart.items</strong> array. For each item, the table shows the item name, followed the item quantity and price.</p> 
             <p>The item quantity is shown using a composite element made up of an input field bound to the <strong>item.quantity</strong> property and two buttons used to increment or decrement the quantity.</p> 
             <p>Notice how the <strong>“ng-change”</strong> attribute is used to save the cart contents when the quantity changes. Notice also how the decrement button is disabled when the item quantity reaches one. At this point, decrementing the quantity would remove the item from the cart, and we don’t want users to do that by accident.</p> 
             <p>After the quantity field, the table shows the total price of the item (unit price times quantity) and a button that allows users to remove the item from the cart.</p> 
             <p>The table footer shows a summary of the cart contents, and is automatically updated as the user edits quantities or removes items from the cart. The updates are handled automatically by AngularJS.</p>

             <h2>Conclusion</h2> 
             <p>The “shoppingCart” class presented here fulfills the requirements outlined in the beginning of the article. It is 100% JavaScript, and has no requirements on the server, so it should be easy to add to existing projects. The cart supports <strong>PayPal</strong> and <strong>Google Wallet</strong>, which are popular payment services. Many applications will probably want to extend this to support their own custom payment services, and that should be easy to do.</p> 
             <p>The MVVM pattern allows the same cart object to be exposed in multiple views, which contain very simple markup and virtually no logic. The sample application for example has a view that shows the whole cart, and allows users to edit it; but it also shows cart summaries on the store and product pages. These views are easy to create and customize, and there is no impact on the application logic.</p> 
             <p>I am a big fan of <strong>AngularJS</strong>. In addition to the MVVM support it provides, which is great, it has an amazing list of features that include routing and partial views, filters, custom directives, and more.</p> 
             <p>I especially like the fact that <strong>AngularJS</strong>’s data binding features work with plain JavaScript objects. Some MVVM libraries (like <strong>KnockoutJS</strong>) require special “observable” properties, which are declared and accessed using a syntax that is different from plain properties.</p> 
             <p>The one aspect of <strong>AngularJS</strong> I do not like is the lack of documentation. You can find a lot of information about the details of pretty much any aspect of <strong>AngularJS</strong>, but I have not found a good reference that presents an overall conceptual view of the framework. My favorite source of documentation on AngularJS is a series of videos created by John Lindquist which you can find here: <a href="http://www.youtube.com/user/johnlindquist"> http://www.youtube.com/user/johnlindquist</a>.</p> 
             <p>I also like <strong>Bootstrap</strong>, because it makes it easy to create attractive, responsive HTML layouts. In addition to a nice set of styles and icons, <strong>Bootstrap</strong> also provides some JavaScript components that you can use to enhance your UIs with things like tooltips, pop-overs, menus, etc. You can learn about <strong>Bootstrap</strong> here: <a href="http://twitter.github.io/bootstrap/"> http://twitter.github.io/bootstrap/</a>.</p>

$(window).on(<span class="code-string">'</span><span class="code-string">storage'</span>, function (e) {
    <span class="code-keyword">if</span> (e.originalEvent.key == self.cartName + <span class="code-string">'</span><span class="code-string">_items'</span> &amp;&amp; !self.savingItems) {
        self.loadItems();
    }
});</pre> <br> Added code to the loadItems function to notify listeners (e.g. Angular) that the items list has changed:<br> &nbsp; <pre>        <span class="code-comment">//</span><span class="code-comment"> load items</span>
        <span class="code-keyword">this</span>.items.length = <span class="code-digit">0</span>;
        <span class="code-comment">//</span><span class="code-comment"> ...</span>
&nbsp;
        <span class="code-comment">//</span><span class="code-comment"> notify listeners of change</span>
        <span class="code-keyword">if</span> (<span class="code-keyword">this</span>.itemsChanged) {
            <span class="code-keyword">this</span>.itemsChanged();
        }</pre> <br> <b>2) Controller.js:</b><br> <br> Added code to the controller to notify AngularJS when items are updated:<br> &nbsp; <pre><span class="code-comment">//</span><span class="code-comment"> apply changes when cart items change</span>
$scope.cart.itemsChanged = function (e) {
    <span class="code-keyword">if</span> (!$scope.$$phase) {
        $scope.$apply();
    }
}</pre> <br> This seems to work just fine.
