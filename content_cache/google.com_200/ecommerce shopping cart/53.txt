          <h2 id="overview">Overview</h2> 
          <p>Module package is a distributable piece of software that can be installed in VirtoCommerce Platform. Technically, module package is a ZIP archive containing module manifest and other module content:</p> 
          <p><img src="../../../assets/images/docs/image2015-5-29_12-13-41.png" alt=""></p> 
          <p>Modules can be&nbsp;unmanaged (JavaScript only) or managed (containing .NET assemblies).</p> 
          <h2 id="create-an-unmanaged-module-package">Create an unmanaged module package</h2> 
          <p>For pure JavaScript modules you can use your favorite ZIP archiver: navigate to the folder containing <em>module.manifest</em> and archive all the necessary files / folders.</p> 
          <h2 id="create-a-managed-module-package">Create a managed module package</h2> 
          <p>Managed modules should be created by using a special Powershell command in Visual Studio (VS).</p> 
          <h2 id="preparing-the-project">Preparing the project</h2> 
          <p>The following steps are needed only once per module.&nbsp;Skip this paragraph if your module has the reference to VirtoCommerce.Module library.</p> 
          <ul> 
           <li>Open Package Manager Console in VS having your new module project loaded (Tools -&gt; NuGet Package Manager -&gt; Package Manager Console);</li> 
           <li>In the "Default project" list select the module project (the one with module.manifest file);</li> 
           <li>Install VirtoCommerce.Module&nbsp;package (execute command:&nbsp;Install-Package VirtoCommerce.Module). More info on the package: <a href="https://www.nuget.org/packages/VirtoCommerce.Module" rel="nofollow">https://www.nuget.org/packages/VirtoCommerce.Module</a></li> 
           <li>readme.txt is displayed after the package installation is completed. <img src="../../../assets/images/docs/image2016-6-1_10-21-32.png" alt=""></li> 
          </ul> 
          <p>The module is ready to be build as a package.</p> 
          <h2 id="building-the-module-package">Building the module package</h2> 
          <p>Execute these steps every time when you need the module package to be created / updated:</p> 
          <p>Open and build the module&nbsp;project in VS;If need, update the module version (in module.manifest)&nbsp;and build again;Open Package Manager Console (Tools -&gt; NuGet Package Manager -&gt; Package Manager Console);In the "Default project" list select the module project (the one with module.manifest file);Execute command:&nbsp;Compress-Module</p> 
          <p><img src="../../../assets/images/docs/image2016-6-1_10-44-13.png" alt=""></p> 
          <p>The module package is created in the project directory.</p>
