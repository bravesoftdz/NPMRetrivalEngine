     <h2>Release Notes</h2> 
     <p>Release Notes for Prince 11<br><br>November 2016<br>Rasterisation<br><br>Support for rasterising documents directly to a PNG or JPEG image.<br>Filters<br><br>Support for filters in SVG and CSS.<br>CSS Support<br><br>Support for CSS4 Selectors.<br><br>Support for CSS4 Media Queries.<br><br>Support for CSS rem units.<br><br>Support for CSS @supports rule and @import conditions.<br><br>New CSS parser with improved conformance and helpful error messages.<br><br>Support for the new CSS attr() function syntax.<br><br>Support for CSS @charset rule for Latin1 and Windows-1252 encodings.<br><br>Support for CSS style sheets encoded in UTF-16.<br>SVG Support<br><br>Support for the SVG dominant-baseline property.<br><br>Support for the SVG baseline-shift property.<br><br>Support for preserveAspectRatio="defer" for nested SVG images.<br><br>Support for global defs in SVG, eg. references to elements outside of the current SVG fragment, either in the same document or other documents.<br><br>Support for internal links into and out of SVG content.<br><br>Improved support for SVG clip paths.<br><br>Fixed bug affecting viewports on nested SVG elements.<br>Images Support<br><br>Added new values to prince-image-magic property: jpeg-verbatim, convert-to-jpeg, and recompress-jpeg (replacing jpeg-quality).<br><br>Support for detecting intrinsic JPEG image resolution specified in Exif metadata.<br><br>ICC profiles and Exif metadata are stripped from embedded JPEG images.<br><br>16-bit images are converted to 8-bit automatically when the specified PDF profile does not support them.<br><br>PNG images are color corrected according to their gamma or chromacity chunks.<br><br>Progressive/interlaced JPEGs are automatically re-encoded as they can fail to display correctly in Acrobat.<br>Color Support<br><br>Added new option --convert-colors to convert all the colors and images to the specified PDF output intent ICC profile.<br><br>Added --fallback-cmyk-profile option.<br><br>Added a new property prince-pdf-color-options to control the conversion between RGB and grayscale color spaces.<br><br>Added a new property prince-pdf-page-colorspace for specifying the color space used to composite pages with transparency.<br><br>Added new CSS properties prince-fallback-cmyk-profile and prince-filter-resolution for the @prince-pdf rule.<br><br>Support for opacity with CMYK colors using optional alpha component for cmyk() color function.<br>Hyphenation &amp; Line Breaking<br><br>Support for Thai line-breaking.<br><br>Support for hyphenation of words containing punctuation, like "quell'impossibilità" in Italian.<br><br>Support for the hyphenate-character property. (Prefixed as prince-hyphenate-character for now).<br><br>Support for hyphenating words within hyphenated phrases.<br><br>Avoids line-breaks after hyphens at the beginning of words, eg. "-word".<br><br>Updated hyphenation pattern dictionaries.<br>Integration<br><br>New --control interface to use a persistent Prince process for multiple consecutive document conversions to reduce latency and increase throughput.<br><br>Support for creating PDF files larger than 4GB (for 64-bit Prince packages).<br><br>Added new option to control SSL / TLS version for HTTPS connections.<br><br>Disabled XInclude by default (it can be enabled with the --xinclude option).<br><br>Disabled XML external entity loading (XXE) by default (it can be enabled with the --xml-external-entities or --xxe options).<br><br>Removed support for "prince input.html output.pdf" usage without -o/--output option.<br>PDF Features<br><br>Support for tooltip popups in generated PDF files via a new prince-tooltip CSS property that can take text from the HTML title attribute.<br><br>Support for new URL scheme for PDF actions, eg. pdf-action:GoBack.<br>PDF Profiles<br><br>Support for PDF/X-1a and PDF/X-3 profiles.<br><br>Support for the PDF/A-3b profile.<br>JavaScript Support<br><br>Fixed issue where calling console.log() from JavaScript could cause deadlocks when Prince is being run from a wrapper library.<br><br>Support for the JavaScript DOM onload/onerror events for script elements.<br><br>Support for the JSON object in the JavaScript DOM, eg. stringify() and parse().<br><br>Extended Prince JavaScript shell to provide API for document conversions.<br><br>Support for next/previousElementSibling properties on JavaScript DOM nodes.<br><br>Support for the DOM window.top, window.parent, and document.defaultView properties.<br><br>Optimised getElementsByTagName and getElementsByClassName by caching live node lists.<br><br>Changed JavaScript DOM method window.setTimeout so that the second argument is optional.<br><br>Fixed bug affecting setting individual values for the CSS clip property in JavaScript.<br><br>Fixed JavaScript DOM bug where innerHTML string included unnecessary end tags for empty elements like br and img.<br><br>Support for the JavaScript Object.getPrototypeOf method.<br><br>Fixed bug where XMLHttpRequest object wasn't an event target, causing spurious error messages.<br><br>Support for JavaScript DOM properties for URLs on anchor elements (needed for AngularJS).<br><br>Support for JavaScript accessor properties with getter/setter functions (needed for mark.js).<br><br>Support for JavaScript Array.every and Array.some methods (needed for mark.js).<br><br>Support for JavaScript DOM method Element.matches aka matchesSelector. (needed for mark.js).<br><br>Changed DOMContentLoaded event to fire on the window object as well as the document.<br><br>Support for JavaScript Array.reduce method (needed for Char.js).<br><br>Support for JavaScript DOM TypedArray.subarray method needed for Japanese line-breaking script.<br><br>Fixed bug where JavaScript expression "c = [c, c, c]" can be evaluated incorrectly as an array of arrays not array of undefined values.<br><br>Support for the JavaScript method Number.toPrecision.<br><br>Support for the JavaScript DOM signed integer typed array objects Int{8,16,32}Array.<br><br>Support for indented JSON output (via third argument of JSON.stringify DOM method).<br><br>Checks type attribute of script elements to ensure JSON is not executed.<br><br>Fixed JavaScript bug where accessing undefined global variables would not throw a reference error.<br><br>Support for the DOMImplementation.createHTMLDocument method.<br><br>Support for the HTMLDocument interface.<br><br>Support for the document.title property.<br><br>Changed the JavaScript console.log method to insert spaces between its arguments.<br><br>Support for the JavaScript DOM TypedArray.set method.<br><br>Support for the Array.of and Array.from methods in JavaScript.<br><br>Support for the Number.MIN_VALUE and MAX_VALUE constants in JavaScript.<br><br>Support for the JavaScript "debugger" statement as a no-op.<br><br>Fixed escaping of text returned from innerHTML DOM property for unescaped elements like script and style.<br>Bug Fixes &amp; Improvements<br><br>Support for relative base URLs in the HTML base element.<br><br>Added Korean font names to default font style sheet.<br><br>Use the PostScript name for TrueType fonts if it is present, and add an experimental --force-identity-encoding option.<br><br>Changed handling of fonts with positive descent metrics to match the behaviour on Windows.<br><br>Changed XML processing so that XHTML, SVG, and MathML elements must be in their respective namespaces in order to be recognised.<br><br>Support for the zero-width non-joiner (zwnj) character, commonly used in Arabic text.<br><br>Added new property prince-mark-width to control crop/cross mark thickness.<br><br>Support for HTML input[type="number"] and date consistently with other form inputs.<br><br>Changed "page-break-before: left | right" to control whether the first page of the document is treated as a left or right facing page.<br><br>Optimised conversion of documents with many repeated style rules.<br><br>Changed prince-text-replace to apply before prince-linebreak-magic.<br><br>Added an auto value to the prince-pdf-page-layout property.<br><br>Added new footnote-display property for a more convenient way of specifying whether footnotes should be block or inline.<br><br>Support for new syntax for the border-radius shorthand property.<br><br>Changed default value for widows/orphans property to 1 instead of 2.<br><br>Fixed issues affecting multiple comma-separated values for the background properties.<br><br>Fixed bug where backgrounds applied to table cells could repeat on the second page even if they are specified as no-repeat.<br><br>Fixed issue where page sizes were rounded to the nearest point (1/72").<br><br>Fixed issue with :blank pages and page-break-after property.<br><br>Fixed various conformance issues affecting CSS stacking contexts for tables and transformed elements.<br><br>Fixed border-radius so that it applies correctly to boxes split across pages.<br><br>Fixed bug affecting @page:blank selector combined with page-break-before and multiple input documents.<br><br>Fixed issue affecting relative positioning on table cells.<br><br>Fixed bug where table cells with large specified height could lead to unnecessary page breaking.<br><br>Fixed bug where box-sizing: padding-box was not correct.<br><br>Fixed bug where long floating blocks could be "interleaved" across page breaks.<br><br>Fixed bug affecting page breaking on content that spans multiple columns.<br><br>Fixed absolute positioning for over-constrained situations in right-to-left contexts.<br><br>Fixed float margin resolution in right-to-left contexts.<br><br>Fixed bug affecting horizontal margins for spans and inline-block elements in right-to-left contexts.<br><br>Fixed margin calculation so that fixed-width blocks end up right aligned in right-to-left contexts.<br><br>Fixed bug where prince-shrink-to-fit could cause severe slowdown.</p>
