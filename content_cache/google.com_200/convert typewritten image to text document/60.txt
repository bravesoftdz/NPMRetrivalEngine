           <p pid="2">In this post, we will see how we can upload files or images to the server using <a href="http://sibeeshpassion.com/category/Node-JS/" rel="nofollow" target="_blank">Node.js</a>. Here we are going to use <a href="http://sibeeshpassion.com/category/visual-studio/" target="_blank" rel="nofollow">Visual Studio</a> for our development and the following NPM packages for our easy development:</p> 
           <li>Express</li> 
           <li>Multer</li> 
           <li>Body-parser</li> 
           <p pid="3">We will briefly explain the use of these packages. As you all know, Node.js is a runtime environment built on Chromes V8 JavaScript engine for server-side and networking application. And it is an open source which supports cross-platforms. Node.js applications are written in pure <a href="http://sibeeshpassion.com/category/javascript/" target="_blank" rel="nofollow">JavaScript</a>. If you are new to Node.js, I strongly recommend you read my previous posts about Node.js&nbsp;<a href="http://sibeeshpassion.com/category/node-js/" rel="nofollow" target="_blank">here</a>.</p> 
           <h2 pid="4"><strong>Download the Source Code for This Post</strong></h2> 
           <li><a href="https://code.msdn.microsoft.com/Upload-Files-Or-To-Server-15f69aaa" rel="nofollow" target="_blank">Upload Files Or Images To Server Using Node.js</a></li> 
           <h2 pid="5"><strong>Background</strong></h2> 
           <p pid="6">A few years back, if you needed to upload any files or images to a server, you were completely dependent on server side languages like <a href="http://sibeeshpassion.com/category/csharp/" target="_blank" rel="nofollow">C#</a> and <a href="http://sibeeshpassion.com/category/PHP/" target="_blank" rel="nofollow">PHP</a>. Everything has changed after the&nbsp;<a href="http://sibeeshpassion.com/category/node-js/" rel="nofollow" target="_blank">Node.js</a>&nbsp;revolution. Here I will show you how to upload the files to a server using Node.js, without writing even a single line of server-side code. Enjoy!</p> 
           <h3 pid="7"><strong>Create a Blank Node.js Web Application</strong></h3> 
           <p pid="8">Create a blank Node JS web application.</p> 
           <p pid="9">new_node_js_web_application</p> 
           <h2 pid="10"><strong>Set Up Dependencies in package.json</strong></h2> 
           <p pid="11">To get started, we will set up our dependencies first. To do so, please open your package.json file and paste the preceding code.</p> 
           <pre><code lang="text/javascript"> {
     "name": "node_js_file_upload",
     "version": "0.0.1",
     "description": "Node_JS_File_Upload",
     "main": "server.js",
     "dependencies": {
         "body-parser": "^1.15.2",
         "express": "^4.14.0",
         "multer": "^1.2.0"
     },
     "author": {
         "name": "Sibeesh"
     }
 }</code></pre> 
           <p pid="55"><br></p> 
           <p pid="12">Now, run the NPM install command as follows.</p> 
           <pre><code lang="csharp; title: ; notranslate"> npm install </code></pre> 
           <p pid="13">Once you run the command, you can see that the dependencies are installed in the solution.</p> 
           <h3 pid="14">npm-Packages</h3> 
           <p pid="15">Now we can understand what are these dependencies used for.</p> 
           <h4 pid="52">Express</h4> 
           <p pid="16">As per the <a href="https://expressjs.com/" target="_blank" rel="nofollow">Express Team</a>, Express is a minimal and flexible Node.js web application framework that provides a robust set of features for web and mobile applications. Express provides a thin layer of fundamental web application features, without obscuring Node.js features that you know and love. You can always learn more about Express Package <a href="https://expressjs.com/" target="_blank" rel="nofollow">here</a>.</p> 
           <h4 pid="53">Multer</h4> 
           <p pid="17">Multer is a Node.js middleware for handling multipart/form-data, which is primarily used for uploading files. It is written on top of busboy for maximum efficiency. You can find out more about the&nbsp;multer package <a href="https://github.com/expressjs/multer" target="_blank" rel="nofollow">here.</a></p> 
           <h2 pid="18"><strong>Start Using Our Dependencies</strong></h2> 
           <p pid="19">You can create the instances of our dependencies as follows.</p> 
           <pre><code lang="text/javascript"> var Express = require('express');
 var multer = require('multer');
 var bodyParser = require('body-parser');
 var app = Express();
 app.use(bodyParser.json());</code></pre> 
           <p pid="54"><br></p> 
           <p pid="20">Then, it is time to create a storage which says where and how the files/images should be saved.</p> 
           <pre><code lang="text/javascript"> var Storage = multer.diskStorage({
     destination: function(req, file, callback) {
         callback(null, "./Images");
     },
     filename: function(req, file, callback) {
         callback(null, file.fieldname + "_" + Date.now() + "_" + file.originalname);
     }
 });</code></pre> 
           <p pid="56"><br></p> 
           <p pid="21">Each file contains the following information:</p> 
           <p pid="22"><em>fieldname:</em> Field name specified in the form.<br><em>originalname:</em> Name of the file on the users computer.<br><em>encoding:</em> Encoding type of the file.<br><em>mimetype:</em> Mime type of the file.<br><em>size:</em> Size of the file in bytes.<br><em>destination:</em> The folder to which the file has been saved.<br><em>filename:</em> The name of the file in the destination.<br><em>path:</em> The full path to the uploaded file.<br><em>buffer:</em> A Buffer of the entire file.</p> 
           <p pid="23">Now, create a&nbsp;multer object as follows.</p> 
           <pre><code lang="text/javascript"> var upload = multer({
     storage: Storage
 }).array("imgUploader", 3); //Field name and max count</code></pre> 
           <p pid="57"><br></p> 
           <p pid="24">Here,&nbsp;multer accepts the storage we created in our previous step as the parameter. The function</p> 
           <pre><code lang="text/javascript">array(fieldname[, maxCount])</code></pre> 
           <p pid="58"><br></p> 
           <p pid="25">accepts an array of files, all with the name fieldname.</p> 
           <p pid="26">Now it is time to write our post and get action.</p> 
           <pre><code lang="text/javascript"> app.get("/", function(req, res) {
     res.sendFile(__dirname + "/index.html");
 });
 app.post("/api/Upload", function(req, res) {
     upload(req, res, function(err) {
         if (err) {
             return res.end("Something went wrong!");
         }
         return res.end("File uploaded sucessfully!.");
     });
 });</code></pre> 
           <p pid="59"><br></p> 
           <p pid="27">Here <em>/api/Upload</em> is the action name we are going to set in our HTML page which we will create soon. And last but not the least, we need to make sure that the app is listening to our particular port, in this case, it is port 2000.</p> 
           <pre><code lang="text/javascript"> app.listen(2000, function(a) {
     console.log("Listening to port 2000");
 });</code></pre> 
           <p pid="60"><br></p> 
           <h2 pid="28"><strong>Create an HTML Page and Set Up Uploading</strong></h2> 
           <p pid="29">You can create the page as follows with the references of the <em>jquery-3.1.1.min.js</em> and <em>jquery.form.min.js</em>.</p> 
           <pre><code lang="text/html">&lt;!DOCTYPE html&gt;
&lt;html xmlns="http://www.w3.org/1999/xhtml"&gt;

&lt;head&gt;
    &lt;meta charset="utf-8" /&gt;
    &lt;title&gt;Upload images to server using Node JS&lt;/title&gt;
    &lt;script src="Scripts/jquery-3.1.1.min.js"&gt;&lt;/script&gt;
    &lt;script src="Scripts/jquery.form.min.js"&gt;&lt;/script&gt;
&lt;/head&gt;

&lt;body&gt;
    &lt;form id="frmUploader" enctype="multipart/form-data" action="api/Upload/" method="post"&gt;
        &lt;input type="file" name="imgUploader" multiple /&gt;
        &lt;input type="submit" name="submit" id="btnSubmit" value="Upload" /&gt; &lt;/form&gt;
&lt;/body&gt;

&lt;/html&gt;</code></pre> 
           <p pid="61"><br></p> 
           <p pid="30">Please be noted that the <em>ecctype </em>for your form must be <em>multipart/form-data</em> and the action must be same as we set in our API.</p> 
           <h2 pid="31"><strong>Create an Ajax Submit Event</strong></h2> 
           <p pid="32">Now it is time to create our ajax event where we are going to call our API.</p> 
           <pre><code lang="text/javascript">&lt;script&gt;
    $(document).ready(function() {
                var options = {
                        beforeSubmit: showRequest, // pre-submit callback success: showResponse // post-submit callback }; // bind to the form's submit event $('#frmUploader').submit(function () { $(this).ajaxSubmit(options); // always return false to prevent standard browser submit and page navigation return false; }); }); // pre-submit callback function showRequest(formData, jqForm, options) { alert('Uploading is starting.'); return true; } // post-submit callback function showResponse(responseText, statusText, xhr, $form) { alert('status: ' + statusText + '\n\nresponseText: \n' + responseText ); }
&lt;/script&gt;</code></pre> 
           <p pid="62"><br></p> 
           <p pid="33"><em>ajaxSubmit</em> function is part of the plugin <em>jquery.form.min.js</em>, so please make sure that you have included it.</p> 
           <h2 pid="34"><strong>Run Your Application</strong></h2> 
           <p pid="35">Now, run your application. Before running your application, you can always set your script file as your startup file, to set it, just&nbsp;right-click on your project and click on properties.</p> 
           <p pid="36"><em>set-start-up-file</em></p> 
           <p pid="37">Now you can open your command prompt, and manually locate your project in the command prompt, or you can use the Open command prompt here option. To select, right-click on your project and select the option as follows:</p> 
           <p pid="38"><em>open_command_prompt_here_option</em></p> 
           <p pid="39">Now type <em>nodeserver.js</em> in your command prompt which will make sure that your server is running. And if everything is fine, you will see this:&nbsp;</p> 
           <p pid="40"><em>node_server_js_output</em></p> 
           <p pid="41">Our server is now ready, and we can run our web page! Now, go to your browser and type the URL as <a href="http://localhost:2000" target="_blank" rel="nofollow">http://localhost:2000</a>. Select few files using the file uploader we have created.</p> 
           <p pid="42"><em>select_fiels_in_file_upload</em></p> 
           <p pid="43">If you click submit, you can see we are calling our method action and the files are uploaded.</p> 
           <p pid="44"><em>file_uploaded_successfully_</em></p> 
           <p pid="45"><em>solution_explorer_window_after_saving_files</em></p> 
           <p pid="46">You can always download the source code attached to see the complete code and application. Happy coding!</p>

                    article.isLimited = true;
                    TH$Dialog.success('You have limited comments for this Article. Now all comments will go through moderation.');
                } else {
                    TH$Dialog.error('error', 'Your requested was denied')
                }
            });
        } else {

            });
        }
