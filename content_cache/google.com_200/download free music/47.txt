         <h2>Description</h2> 
         <div class="title__star"> 
          <iframe src="http://ghbtns.com/github-btn.html?user=IonDen&amp;repo=ion.sound&amp;type=watch&amp;count=true&amp;size=large" allowtransparency="true" frameborder="0" scrolling="0" width="170" height="30"></iframe>

        <ul> 
         <li>Ion.Sound — JavaScript-plugin for playing sounds based on Web Audio API.</li> 
         <li>Plugin is working on most popular desktop and mobile browsers and can be used everywhere, from common web sites to browser games.</li> 
         <li>For not so modern browsers plugin falls back to HTML5 audio.</li> 
         <li>Audio-sprites support included.</li> 
         <li><a href="https://github.com/IonDen/ion.sound">Project on GitHub</a>.</li> 
         <li>Ion.Sound freely distributed under terms of <a href="http://ionden.com/a/plugins/licence-en.html" target="_blank">MIT licence</a>.</li> 
         <li>25 free sounds included</li> 
        </ul> 
        <p>Today websites are full of events (new mail, new chat-message, content update etc.). Often it is not enough to indicate this events only visually to get user attention. You need sounds! This library, made for playing small sounds, will help you with this task.</p> 
        <p>Also, new version of Ion.Sound is capable to handle browser games audio. It has full control of loading, playing and removing audio files. And audio-sprites support of course.</p>

         <h2>Supported browsers</h2> 
        </div> 
        <h3>Desktop Windows, OS X, Linux:</h3> 
        <ul> 
         <li>Google Chrome</li> 
         <li>Mozilla Firefox</li> 
         <li>Microsoft Internet Explorer 9.0+</li> 
         <li>Opera 12.16+</li> 
         <li>Safari 5.1+ (Safari on Windows requires QuickTime to play sounds)</li> 
        </ul> 
        <h3>Mobile:</h3> 
        <ul> 
         <li>iOS Safari and others (with some <a href="https://developer.apple.com/library/safari/documentation/audiovideo/conceptual/using_html5_audio_video/device-specificconsiderations/device-specificconsiderations.html" target="_blank">restrictions</a>)</li> 
         <li>Android Google Chrome and others (with some restrictions also)</li> 
         <li>WP8 Internet Explorer</li> 
        </ul> 
        <p>Can i use <a href="http://caniuse.com/#feat=audio-api" target="_blank">Web Audio API</a> and <a href="http://caniuse.com/audio" target="_blank">HTML5 Audio</a>?</p>

         <h2>Usage</h2> 
        </div> 
        <h5>Add the following library to the page:</h5> 
        <ul> 
         <li>ion.sound.min.js</li> 
        </ul> 
        <h5>Prepare sound-files (25 sounds are included) and put them in some folder (eg. "sounds"):</h5> 
        <ul> 
         <li>my_cool_sound.mp3</li> 
         <li>my_cool_sound.ogg</li> 
         <li>my_cool_sound.aac</li> 
        </ul> 
        <p> It is not enough to have only MP3-file, you should make OGG and AAC-file too, because not all browsers support MP3.<br> You can easily convert you MP3-s to OGG-s and AAC-s at <a href="http://media.io/" target="_blank">Media.io</a> or at <a href="https://cloudconvert.org/formats#audio" target="_blank">CloudConvert.org</a>.<br> <i>AAC support was added to improve cross browser support of iOS 8.x devices (iPhone, iPad)</i> </p> 
        <div class="title"> 
         <h2>Install with bower</h2> 
        </div> 
        <ul> 
         <li>bower install ionsound</li> 
        </ul>

        <ul> 
         <li>spm install ion-sound</li> 
        </ul> 
        <div class="title"> 
         <h2>Initialisation</h2> 
        </div> 
        <p>To initialise plugin call this method:</p> 
        <pre>
ion.sound({
    sounds: [
        {
            name: "my_cool_sound"
        },
        {
            name: "notify_sound",
            volume: 0.2
        },
        {
            name: "alert_sound",
            volume: 0.3,
            preload: false
        }
    ],
    volume: 0.5,
    path: "sounds/",
    preload: true
});
</pre> 
        <p>And play sound!</p> 
        <pre>
// Simple
ion.sound.play("my_cool_sound");
</pre>

         <h2>General settings</h2> 
        </div> 
        <table class="options"> 
         <thead> 
          <tr> 
           <th>Option</th> 
           <th>Defaults</th> 
           <th>Type</th> 
           <th>Description</th> 
          </tr> 
         </thead> 
         <tbody> 
          <tr class="options__step"> 
           <td>sounds</td> 
           <td>-</td> 
           <td>array</td> 
           <td>Collection of sound objects. Each object contains information about sound file and (optional) individual settings</td> 
          </tr> 
          <tr> 
           <td>path</td> 
           <td>-</td> 
           <td>string</td> 
           <td>Path to file</td> 
          </tr> 
          <tr> 
           <td>preload</td> 
           <td>false</td> 
           <td>boolean</td> 
           <td>Preloading sounds</td> 
          </tr> 
          <tr> 
           <td>multiplay</td> 
           <td>false</td> 
           <td>boolean</td> 
           <td>Sound multi play. If set, will allow to play multiple instances of one sound at once</td> 
          </tr> 
          <tr> 
           <td>loop</td> 
           <td>false</td> 
           <td>boolean/number</td> 
           <td>If set to true will enable infinite loop. Or paste a number to set loop limit</td> 
          </tr> 
          <tr class="options__step"> 
           <td>volume</td> 
           <td>1.0</td> 
           <td>number</td> 
           <td>Playback volume from 0 to 1</td> 
          </tr> 
          <tr> 
           <td>scope</td> 
           <td>null</td> 
           <td>object</td> 
           <td>Callbacks will be called in that object's scope</td> 
          </tr> 
          <tr> 
           <td>ready_callback</td> 
           <td>null</td> 
           <td>function</td> 
           <td>Called after sound file is fully uploaded (or ready to play for HTML5 audio)</td> 
          </tr> 
          <tr> 
           <td>ended_callback</td> 
           <td>null</td> 
           <td>function</td> 
           <td>Called each time then sound file will reach it's end</td> 
          </tr> 
         </tbody> 
        </table> 
        <div class="title"> 
         <h2>Sound object</h2> 
        </div> 
        <table class="options"> 
         <thead> 
          <tr> 
           <th>Option</th> 
           <th>Defaults</th> 
           <th>Type</th> 
           <th>Description</th> 
          </tr> 
         </thead> 
         <tbody> 
          <tr> 
           <td>name</td> 
           <td>-</td> 
           <td>string</td> 
           <td>File name. Plugin will choose file extension automatically (.mp3, .ogg, .aac, .mp4 and etc.)</td> 
          </tr> 
          <tr> 
           <td>alias</td> 
           <td>-</td> 
           <td>string</td> 
           <td>Alias for sound call, optional. Normally is used to shorten ion.sound calls</td> 
          </tr> 
          <tr> 
           <td>sprite</td> 
           <td>-</td> 
           <td>object</td> 
           <td> Mark that sound is audio-sprite. This is an object. Example: <code>{"part_name_1": [0, 2], "part_name_2": [2, 2]}</code><br> Part_name is a name of sprite piece (it is used instead of name to play a sound). And array with time marks: [start, duration] in seconds. </td> 
          </tr> 
          <tr> 
           <td colspan="4">And also individual: path, preload, multiplay, loop, volume, scope and callbacks</td> 
          </tr> 
         </tbody> 
        </table>
