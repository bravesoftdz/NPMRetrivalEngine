           <h1 name="39f5" id="39f5" class="graf graf--h3 graf--leading graf--title">How to Run Framer JS on&nbsp;Windows</h1>
           <h2 name="72a2" id="72a2" class="graf graf--h4 graf-after--h3 graf--subtitle">2 (Mostly) hassle free ways to compile and&nbsp;run</h2>
           <figure name="551a" id="551a" class="graf graf--figure graf-after--h4">
            <div class="aspectRatioPlaceholder is-locked" style="max-width: 700px; max-height: 463px;">
             <div class="aspectRatioPlaceholder-fill" style="padding-bottom: 66.2%;"></div>
             <div class="progressiveMedia js-progressiveMedia graf-image" data-image-id="1*t1-jGpX3fUz3Qm48eAzz5g.png" data-width="3667" data-height="2427" data-action="zoom" data-action-value="1*t1-jGpX3fUz3Qm48eAzz5g.png">
              <img src="https://cdn-images-1.medium.com/freeze/max/60/1*t1-jGpX3fUz3Qm48eAzz5g.png?q=20" crossorigin="anonymous" class="progressiveMedia-thumbnail js-progressiveMedia-thumbnail">
              <canvas class="progressiveMedia-canvas js-progressiveMedia-canvas"></canvas>
              <img class="progressiveMedia-image js-progressiveMedia-image" data-src="https://cdn-images-1.medium.com/max/1600/1*t1-jGpX3fUz3Qm48eAzz5g.png">
              <noscript class="js-progressiveMedia-inner">
               <img class="progressiveMedia-noscript js-progressiveMedia-inner" src="https://cdn-images-1.medium.com/max/1600/1*t1-jGpX3fUz3Qm48eAzz5g.png">

            <figcaption class="imageCaption">
             Framer running on Windows using VS Code, Microsoft Edge, and&nbsp;Gulp
            </figcaption>

           <p name="6bfa" id="6bfa" class="graf graf--p graf-after--figure">Since releasing the <a href="https://docs.microsoft.com/en-us/windows/uwp/design-downloads/index" data-href="https://docs.microsoft.com/en-us/windows/uwp/design-downloads/index" class="markup--anchor markup--p-anchor" rel="noopener nofollow" target="_blank">Fluent Design Toolkit</a>, there’s been a lot of interest in the <a href="https://github.com/Microsoft/windows-framer-toolkit" data-href="https://github.com/Microsoft/windows-framer-toolkit" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Windows Framer Toolkit</a>. Framer is a powerful tool that we use every day here at Microsoft, which may lead some to wonder <em class="markup--em markup--p-em">How do you run Framer on Windows?</em></p>
           <p name="1b91" id="1b91" class="graf graf--p graf-after--p">Framer Studio is only available for Mac. But since <a href="https://github.com/koenbok/Framer" data-href="https://github.com/koenbok/Framer" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Framer’s code</a> is open source, you can compile and run on Windows without much hassle (ok, a little bit). There are two options to do this. Option 1 is based on node.js, uses VS Code, and Gulp. We use this method a lot, especially since we create and use a lot of modules, and it offers full module support. Option 2 is easier, uses Atom, but has some limitations, especially with modules.</p>
           <h4 name="738f" id="738f" class="graf graf--h4 graf-after--p">Option 1— <a href="https://github.com/joetheday/framer-boilerplate" data-href="https://github.com/joetheday/framer-boilerplate" class="markup--anchor markup--h4-anchor" rel="nofollow noopener" target="_blank">Framer Boilerplate</a></h4>
           <ol class="postList">
            <li name="4f33" id="4f33" class="graf graf--li graf-after--h4">Install <a href="https://nodejs.org/en/" data-href="https://nodejs.org/en/" class="markup--anchor markup--li-anchor" rel="noopener nofollow" target="_blank">node.js</a></li>
            <li name="e163" id="e163" class="graf graf--li graf-after--li">Install <a href="https://git-scm.com/" data-href="https://git-scm.com/" class="markup--anchor markup--li-anchor" rel="nofollow noopener" target="_blank">Git</a> (if you haven’t already)</li>
            <li name="2f8a" id="2f8a" class="graf graf--li graf-after--li">Install <a href="http://gulpjs.com/" data-href="http://gulpjs.com/" class="markup--anchor markup--li-anchor" rel="noopener nofollow" target="_blank">Gulp</a>?—?Open up Command Prompt and type <code class="markup--code markup--li-code">npm install -g gulp-cli</code>. Gulp enables compiling a Framer project and viewing it in any browser.</li>
            <li name="3078" id="3078" class="graf graf--li graf-after--li">Download <a href="https://github.com/joetheday/framer-boilerplate" data-href="https://github.com/joetheday/framer-boilerplate" class="markup--anchor markup--li-anchor" rel="noopener nofollow" target="_blank">Framer Boilerplate</a> to a place that you’ll remember. You’ll be coming back to this for each Framer project you create. Framer Boilerplate is what does all the Framery magic.</li>
            <li name="5003" id="5003" class="graf graf--li graf-after--li">Copy the Framer Boilerplate files to your working directory. I put mine at <code class="markup--code markup--li-code">C:\src\[project-name]</code></li>
            <li name="e257" id="e257" class="graf graf--li graf-after--li">If you are using any Modules, put them in the <code class="markup--code markup--li-code">C:\src\[project-name]\app\modules\</code> directory. You’ll see we’ve already included some basic modules from the Windows Framer Toolkit.</li>
            <li name="1d57" id="1d57" class="graf graf--li graf-after--li">In Command Prompt, browse to your project <code class="markup--code markup--li-code">cd \src\[project-name]</code>and type <code class="markup--code markup--li-code">npm install</code>.</li>
            <li name="9f03" id="9f03" class="graf graf--li graf-after--li">Open up your project in <a href="https://code.visualstudio.com/" data-href="https://code.visualstudio.com/" class="markup--anchor markup--li-anchor" rel="nofollow noopener" target="_blank">VS Code</a>. Because CoffeeScript files are whitespace dependent, it’s important to make sure your code editor has the correct whitespace and tab settings.&nbsp;<br>&nbsp;* Tab space: 4<br>&nbsp;* Tab type: hard</li>
            <li name="fc0e" id="fc0e" class="graf graf--li graf-after--li">In Command Prompt, run a gulp server by typing <code class="markup--code markup--li-code">gulp</code> from the root of your project.</li>
            <li name="822c" id="822c" class="graf graf--li graf-after--li">Browse to <a href="http://localhost:3000" data-href="http://localhost:3000" class="markup--anchor markup--li-anchor" target="_blank">http://localhost:3000</a> and witness the magic!</li>
           </ol>
           <h4 name="167f" id="167f" class="graf graf--h4 graf-after--li">Option 2?—?<a href="https://atom.io/packages/atom-live-server" data-href="https://atom.io/packages/atom-live-server" class="markup--anchor markup--h4-anchor" rel="nofollow noopener" target="_blank">atom-live-server</a></h4>
           <ol class="postList">
            <li name="5889" id="5889" class="graf graf--li graf-after--h4">Install <a href="https://atom.io/" data-href="https://atom.io/" class="markup--anchor markup--li-anchor" rel="noopener nofollow" target="_blank">Atom</a></li>
            <li name="e4db" id="e4db" class="graf graf--li graf-after--li">Type <code class="markup--code markup--li-code">control +&nbsp;,</code> to get into Atom’s settings</li>
            <li name="8791" id="8791" class="graf graf--li graf-after--li">Click on <code class="markup--code markup--li-code">+ Install</code></li>
            <li name="918d" id="918d" class="graf graf--li graf-after--li">Search for <code class="markup--code markup--li-code">atom-live-server</code> and install it</li>
            <li name="7392" id="7392" class="graf graf--li graf-after--li">Open your Framer project’s <code class="markup--code markup--li-code">app.coffee</code> file in Atom</li>
            <li name="fc6d" id="fc6d" class="graf graf--li graf-after--li">In Atom do <code class="markup--code markup--li-code">ctrl + alt +l</code> to run a server (or, find your <code class="markup--code markup--li-code">Packages</code> menu, and select <code class="markup--code markup--li-code">atom-live-server &gt; start server</code>.             <li name="af5d" id="af5d" class="graf graf--li graf-after--li">To stop a server, in Atom do <code class="markup--code markup--li-code">ctrl + alt + q</code> (or, find your <code class="markup--code markup--li-code">Package</code> menu, and select <code class="markup--code markup--li-code">atom-live-server &gt; stop server</code>)</li>
           </ol>
