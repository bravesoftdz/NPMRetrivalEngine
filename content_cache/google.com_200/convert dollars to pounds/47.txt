       <h1><span class="mw-headline" id="Overview"><span class="mw-headline-number">1</span> Overview</span></h1> 
       <p>Administrators can use currency and price fields in scripts. </p>
       <p>You can obtain currency values using: </p> 
       <ul>
        <li> <b>getReference</b> methods: apply to the system's base currency, when the item was created for the service catalog.</li> 
        <li> <b>getCurrency</b> methods: apply to the currency used by the customer who ordered the item.</li> 
        <li> <b>getSession</b> methods: apply to the currency of the currently logged-in user viewing the item, for example, a member of a fulfillment group processing the order.</li>
       </ul> 
       <h1><span class="mw-headline" id="Obtaining_Currency_Values"><span class="mw-headline-number">2</span> Obtaining Currency Values</span></h1> 
       <p>The following methods allow you to obtain currency values. </p>
       <p>The <b>Example value returned</b> column shows details for an item&nbsp;: </p> 
       <ul>
        <li> Created with an item price of $600 in a system base currency of USD (US dollars)</li> 
        <li> Ordered by a British user in the GBP (British pounds) currency</li> 
        <li> Viewed and fulfilled by a Spanish user in the EUR (Euro) currency</li>
       </ul> 
       <table> 
        <tbody>
         <tr> 
          <th> Method Name </th> 
          <th> Description </th> 
          <th> Example value returned </th>
         </tr>

       <h1><span class="mw-headline" id="Examples"><span class="mw-headline-number">3</span> Examples</span></h1> 
       <pre>current.price=1000</pre> 
       <p>This sets the current price = 1,000 of whatever the current session currency is. For example, for a Japanese user, the price is JPY (Japanese Yen) 1,000. </p> 
       <pre>current.price='USD;1000'</pre> 
       <p>Sets the current price = 1,000 USD. </p> 
       <h1><span class="mw-headline" id="Working_with_Values"><span class="mw-headline-number">4</span> Working with Values</span></h1> 
       <h2><span class="mw-headline" id="Altering_Values"><span class="mw-headline-number">4.1</span> Altering Values</span></h2> 
       <p>You cannot simply add an integer to the return values, as they are returned as string values. </p>
       <p>For example, with a current.price of 708.32, <b>current.price + 100</b> returns a value of 708.32100. </p>
       <p>You can use a <b>parseFloat</b> function in your script to provide the correct result. For example, <b>parseFloat(current.price) + 100</b> will return 808.32. </p> 
       <h2><span class="mw-headline" id="Returning_Predictable_Values"><span class="mw-headline-number">4.2</span> Returning Predictable Values</span></h2> 
       <p>Adding values to prices with scripts can be unpredictable. For example, consider a script which adds 100 to the price of our item. </p> 
       <pre>var newPrice = parseFloat(current.price) + 100;</pre> 
       <p>Running that script in the UK returns a value of <b>var newPrice = parseFloat(708.32) + 100</b> = 808.32. </p>
       <p>However, if you run it in the US, this returns a value of <b>var newPrice = parseFloat(1000) + 100</b> = 1100. </p>
       <p>To get the predictable values back into our item, regardless of the session under which the script runs, you can write your scripts to run in the system's base currency. </p>
       <p>For example: </p> 
       <pre>var newPrice = parseFloat(current.price.getReferenceValue()) + 100&nbsp;;
current.price
 = current.price.getReferenceCurrencyCode()
 + ';' + newPrice&nbsp;; </pre>
