        <h1 class="entry-title" itemprop="name">Format and un-format money/currency in JavaScript</h1> 
       </header> 
       <!-- .entry-header --> 
       <div class="entry-content" itemprop="mainContentOfPage"> 
        <p>Here’s a couple of simple JavaScript money-formattin’ snippets I use all the time where currencies are handled.</p> 
        <p class="message">The code in this post has been expanded into <a title="accounting.js" href="http://josscrowcroft.github.com/accounting.js/" target="_blank"><strong>accounting.js</strong></a>, a tiny JS library with some handy methods for number, money and currency formatting. Check it out!</p> 
        <p class="message error">Also: need to do real-time currency conversion in your apps and sites? Take a look at the <a title="Open Exchange Rates - free currency data JSON API service" href="http://openexchangerates.org/" target="_blank"><strong>Open Exchange Rates</strong></a> API!</p> 
        <p>The first is a function to format money/currency in JavaScript, turning an integer or float into a formatted string (with customisable decimal precision, currency symbol, decimal- and thousands-separators).</p> 
        <p>The second is a regular expression that removes currency formatting, so that <code>parseInt</code> / <code>parseFloat</code> can be used to extract the value from a currency-formatted string.</p> 
        <h2>JavaScript Money Format</h2> 
        <p>This extends the native Number object, so that all new numbers (integers/floats) can have the <code>moneyformat()</code> method called on them. If you prefer not to extend JavaScript’s native Number object for any reason, this also works fine as a standalone function or a library method (examples below.)</p> 
        <pre class="prettyprint lang-js">// Extend the default Number object with a formatMoney() method:
// usage: someVar.formatMoney(decimalPlaces, symbol, thousandsSeparator, decimalSeparator)
// defaults: (2, "$", ",", ".")
Number.prototype.formatMoney = function(places, symbol, thousand, decimal) {
	places = !isNaN(places = Math.abs(places)) ? places : 2;
	symbol = symbol !== undefined ? symbol : "$";
	thousand = thousand || ",";
	decimal = decimal || ".";
	var number = this, 
	    negative = number &lt; 0 ? "-" : "",
	    i = parseInt(number = Math.abs(+number || 0).toFixed(places), 10) + "",
	    j = (j = i.length) &gt; 3 ? j % 3 : 0;
	return symbol + negative + (j ? i.substr(0, j) + thousand : "") + i.substr(j).replace(/(\d{3})(?=\d)/g, "$1" + thousand) + (places ? decimal + Math.abs(number - i).toFixed(places).slice(2) : "");
};</pre> 
        <p><em>NB: minified version below.</em></p> 
        <h3>Some JS currency-formatting examples:</h3> 
        <pre class="prettyprint lang-js">// Default usage and custom precision/symbol :
var revenue = 12345678;
console.log(revenue.formatMoney()); // $12,345,678.00
console.log(revenue.formatMoney(0, "HK$ ")); // HK$ 12,345,678

// European formatting:
var price = 4999.99;
console.log(price.formatMoney(2, "€", ".", ",")); // €4.999,99

// It works for negative values, too:
console.log( (-500000).formatMoney(0, "£ ") ); // £ -500,000</pre> 
        <h2>Currency to number – removing money formatting:</h2> 
        <p>Say you have a currency-formatted string, eg. <code>"HK$ -100,000.50"</code> and you want to get the value from that, removing all currency formatting and <em>thousand-separators</em>. Default methods won’t get you very far: <code>parseInt</code> and <code>parseFloat</code> both return <code>NaN</code> (“not a number”, as the string had non-numeric characters in it.)</p> 
        <p>No problemo. A simple <code>regex</code> will take care of this:</p> 
        <pre class="prettyprint lang-js">var price = (12345.99).formatMoney(); // "$12,345.99"

// Remove non-numeric chars (except decimal point/minus sign):
priceVal = parseFloat(price.replace(/[^0-9-.]/g, '')); // 12345.99</pre> 
        <p>NB: This only works where the decimal separator is a point (<code>.</code>) – if the separator is a comma (<code>,</code>), put a comma in the regex instead of a point, eg: <code>/[^0-9-,]/g</code></p> 
        <p>Hope somebody finds this useful!</p> 
        <h4>Non-Number.prototype version:</h4> 
        <pre class="prettyprint lang-js">// To set it up as a global function:
function formatMoney(number, places, symbol, thousand, decimal) {
	number = number || 0;
	places = !isNaN(places = Math.abs(places)) ? places : 2;
	symbol = symbol !== undefined ? symbol : "$";
	thousand = thousand || ",";
	decimal = decimal || ".";
	var negative = number &lt; 0 ? "-" : "",
	    i = parseInt(number = Math.abs(+number || 0).toFixed(places), 10) + "",
	    j = (j = i.length) &gt; 3 ? j % 3 : 0;
	return symbol + negative + (j ? i.substr(0, j) + thousand : "") + i.substr(j).replace(/(\d{3})(?=\d)/g, "$1" + thousand) + (places ? decimal + Math.abs(number - i).toFixed(places).slice(2) : "");
}

// To create it as a library method:
myLibrary.formatMoney = function(number, places, symbol, thousand, decimal) {
	/* as above */
}

// Example usage:
formatMoney(54321); // $54,321
myLibrary.formatMoney(12345, 0, "£ "); // £ 12,345</pre>

            <div class="comment-content ">
             <p>Hey! Yeah that’s an interesting one. </p> 
             <p><code>symbol = symbol !== undefined ? symbol : "$";</code></p> 
             <p>This one says: If <code>symbol</code> is anything except <code>undefined</code>, use it; if not, use the default of <code>"$"</code>. The <code>!==</code> check is strict and checks the variable type (which means that symbol can be an empty string, <code>""</code>, but needs to be explicitly defined as such.)</p> 
             <p><code>thousand = thousand || ",";</code></p> 
             <p>This one says: Use <code>thousand</code> if it’s <em>truthy</em>, or anything that coerces to a boolean true (not <em>falsey</em>, which would include <code>null</code>, <code>undefined</code>, <code>""</code> and <code>0</code>) – otherwise, use <code>","</code>. Logically this makes sure that thousands are always separated, otherwise there’d be no sense using this method, but you can separate them with space or point (<code>" "</code> or <code>"."</code>)</p> 
             <p>Check out this fantastic <a href="http://james.padolsey.com/javascript/truthy-falsey/" title="Turthy and Falsey Values in JavaScript" target="_blank" rel="nofollow">primer on truthy and falsey values</a> if you’re interested in how the mechanics work.</p> 
             <p>Also – check out <a href="http://josscrowcroft.github.com/accounting.js/" title="accounting.js" target="_blank" rel="nofollow"><strong>accounting.js</strong></a> which is more up-to-date and easy to use!</p>
