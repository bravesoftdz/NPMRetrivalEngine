   <section id="own_rates_provider" class="anchor"> 
    <h2>Deploy own currency rate provider</h2> 
    <p> By default, currency conversion plugin is using 3rd party services to pull currency rates, but it can affect the performance of the widget, because no one can guarantee the trouble-free operation of external APIs. To avoid this, widget comes with own PHP API.<br> The pros of using own server: </p>
    <ul> 
     <li>You can run widget over https</li> 
     <li>You can get rid of dependence using third-party services</li> 
    </ul> Here is the instructions how to deploy your own server: 
    <ol> 
     <li>Make sure PHP version is greater than <strong>5.0</strong> and <strong>mysqli</strong> extension is enabled.</li> 
     <li>Upload <strong>php_api/api_currency.php</strong> on FTP (For instance in root folder: http://example.com/api_currency.php)</li> 
     <li>Create database and import dump: <strong>php_api/dump.sql</strong></li> 
     <li> Setup MySQL credentials. You can find them at the top of <strong>php_api/api_currency.php</strong> file. <pre class="brush: js;">
defined('CURRENCY_API_DB_HOST') or define("CURRENCY_API_DB_HOST", "localhost"); // Mysql host
defined('CURRENCY_API_DB_USER') or define("CURRENCY_API_DB_USER", "root"); // Mysql user
defined('CURRENCY_API_DB_PASSWORD') or define("CURRENCY_API_DB_PASSWORD", ""); // Mysql password
defined('CURRENCY_API_DB_DBNAME') or define("CURRENCY_API_DB_DBNAME", "api_currency"); // Database name
</pre> </li> 
     <li> Tell widget to use own PHP API <pre class="brush: js;">
$('#currency-widget').currency({
	localRateProvider: 'api_currency.php', // This is the same as http(s)://example.com/api_currency.php
});
</pre> </li> 
     <p></p> 
    </ol>

   <section id="localization" class="anchor"> 
    <h2>Localization</h2> 
    <p> Currency converter plugin contains only <strong>en_US</strong> localization, but you can easy create your own. Let's take a look how it must be done.</p> 
    <h3>Step #1</h3> 
    <p>Create new file <strong>jquery.currency.localization.your_LOCALE.js</strong></p> 
    <h3>Step #2</h3> 
    <p>Open <strong>jquery.currency.localization.en_US.js</strong> file, copy all and paste into your newly created file.</p> 
    <h3>Step #3</h3> Change object's keys with chosen locale. 
    <pre class="brush: js;">

$(function () {
	$.currency.interface['your_LOCALE'] = {
		...
	},
	$.currency.isocode['your_LOCALE'] = {
		...
	}
});
</pre> and begin translation. 
    <h3>Step #4</h3> Pass 
    <strong>locale</strong> parameter at widget initialization. 
    <pre class="brush: js;">

$(document).ready(function(){
	$('#currency-widget').currency({
		locale: 'your_LOCALE', // Set widget locale
	});
});
</pre> 
    <h3>All in one place</h3> 
    <p>At the end of the localization process the code should look like this.</p> 
    <pre class="brush: xhtml;">

&lt;!DOCTYPE HTML&gt;
&lt;html lang="en"&gt;

	$('#currency-widget').currency({
