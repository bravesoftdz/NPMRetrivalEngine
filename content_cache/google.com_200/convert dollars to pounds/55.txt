     <h1 class="article-title">Setting Up Enhanced Conversion Tracking</h1> 
     <div class="article-meta"> 
      <div class="article-updated meta">
       <time datetime="2017-09-18T13:22:58Z" title="2017-09-18T13:22:58Z" data-datetime="calendar">September 18, 2017 13:22</time>
      </div> 
     </div> 
     <div class="article-body markdown"> 
      <p>These are general instructions for dynamically passing back revenue recorded for each sale. If you work with a particular e-commerce platform, check if we have your platform listed in&nbsp;<a href="/hc/en-us/categories/201557907" target="_blank" rel="noopener noreferrer">Platform Integrations</a>.&nbsp;</p> 
      <h2>Before&nbsp;You Start</h2> 
      <p>To pass back revenue values dynamically, make sure that you don't specify a conversion value when creating your conversion audience. If you previously specified a conversion value, please edit your conversion audience.</p> 
      <p>&nbsp;</p> 
      <h2>Basic Conversion Tracking</h2> 
      <p>Your AdRoll&nbsp;Pixel and dashboard provide an easy way to set up basic conversion tracking through URL-based rules.&nbsp;Any time a visitor lands on&nbsp;a page that represents completion of a&nbsp;desired action, for example&nbsp;<a href="http://www.example.com/thankyou" target="_blank" rel="noopener noreferrer">http://www.example.com/thankyou</a>,&nbsp;a conversion event can&nbsp;be triggered and attributed to your advertising.</p> 
      <p>If you want&nbsp;granular attribution insight, you'll need to add our enhanced conversion tracking JavaScript code to your conversion page(s). This code will instruct your AdRoll&nbsp;Pixel to pass back additional variables, such as order value or ID.</p> 
      <p>To implement enhanced conversion tracking, you'll&nbsp;need to know:</p> 
      <ul> 
       <li>How to access your conversion page HTML.</li> 
       <li>Variable names for&nbsp;the information you'd like to track.</li> 
      </ul> 
      <p>&nbsp;</p> 
      <h2>Conversion Revenue</h2> 
      <p>We've implemented a specific conversion variable for order amounts&nbsp;that is recognized by your AdRoll Pixel. When a value is assigned to this variable, it will show up in your AdRoll dashboard.</p> 
      <p>In the HTML of your success page, &nbsp;add additional JavaScript below your AdRoll&nbsp;Pixel<em>. </em>The script will instruct your pixel to recognize the variable&nbsp;that logs&nbsp;the order amount assigned to the conversion.</p> 
      <p> <script src="https://gist.github.com/adroll-isabelle/50a11252d4826e3a9299.js"></script> </p> 
      <p>The revenue variable name (<em>X</em>) is unique to each website and e-commerce platform. Make sure to refer to the proper documentation from your respective platform.</p> 
      <p>To&nbsp;verify that you've implemented the revenue variable correctly, trigger a test transaction. Then, check the HTML of your conversion page. The revenue value (<em>X)</em> should be replaced with the actual order amount&nbsp;logged.</p> 
      <p> <script src="https://gist.github.com/adroll-isabelle/7b56a4a9e0393c29e944.js"></script> </p> 
      <p>&nbsp;</p> 
      <h2>Custom Currencies</h2> 
      <p>AdRoll defaults to reporting revenue values in USD. To report revenue values&nbsp;in a custom currency, replace&nbsp;"USD" with <a href="/hc/en-us/articles/213429827" target="_blank" rel="noopener noreferrer">the three-letter code that represents your desired currency</a>&nbsp;in the script.</p> 
      <p><strong>Additional considerations:</strong></p> 
      <ul> 
       <li>We can currently set up your dashboard to report in US Dollars (USD), Euro (EUR), British Pound Sterling (GBP), Australian Dollars (AUD), and Japanese Yen (JPY).</li> 
       <li>If you specify a different currency for enhanced conversion tracking than what is set on your dashboard, revenue tracked will be converted to&nbsp;your dashboard currency setting based on daily exchange rates.</li> 
       <li>If no currency code is specified, reporting defaults to USD.</li> 
      </ul> 
      <h4 style="color: #2e3640; font-size: 16px; font-family: 'Helvetica Neue', Helvetica, Arial, Verdana, sans-serif;">&nbsp;</h4> 
      <h2>Additional Custom Values</h2> 
      <p>Set up&nbsp;a JavaScript object with the name <em>adroll_custom_data</em> to&nbsp;encapsulate custom variables and values.</p> 
      <p> <script src="https://gist.github.com/adroll-isabelle/f2ac78a129aeab5735c6c2007c642c32.js"></script> </p> 
      <p>You can continue to define other information you want captured, as long as you know the corresponding variable name. Simply follow the format:</p> 
      <pre><code>adroll_custom_data = {"ORDER_ID": W, "USER_ID": Z, "custom_name": A, "custom_name" : B}</code></pre> 
      <p>Where "custom_name" is your name for the data you are trying to capture&nbsp;and&nbsp;<em>A&nbsp;</em>and&nbsp;<em>B</em>&nbsp;are replaced&nbsp;by the corresponding variable names.</p> 
      <p>&nbsp;</p> 
      <h2>Revenue&nbsp;and Custom Values</h2> 
      <p>If you’d like to pass AdRoll both types of data, you can set both variables for revenue and custom values at the same time. Simply set&nbsp;<em>adroll_conversion_value</em>,&nbsp;<em>adroll_currency</em>,&nbsp;and&nbsp;<em>adroll_custom_data</em>&nbsp;variables consecutively in JavaScript.</p>
