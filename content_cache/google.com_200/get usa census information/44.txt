        <h1 class="entry-title">Seventeen Ways to Map Data in Kaggle Kernels: Tutorials for Python and R Users</h1> 
        <p class="p-meta"><span><a href="http://blog.kaggle.com/author/mrisdal/">Megan Risdal</a></span>|<span><time class="entry-date" datetime="2016-11-30T10:04:33+00:00">11.30.2016</time></span></p> 
       </header> 
       <div class="entry-featured"> 
        <div class="entry-thumb">
         <img width="1000" height="280" src="http://5047-presscdn.pagely.netdna-cdn.com/wp-content/uploads/2016/11/blog_mapping_data_python_r.png" class="attachment-entry size-entry wp-post-image" alt="Mapping data in Kaggle Kernels: Tutorials for Python and R Users" srcset="http://5047-presscdn.pagely.netdna-cdn.com/wp-content/uploads/2016/11/blog_mapping_data_python_r.png 1000w, http://5047-presscdn.pagely.netdna-cdn.com/wp-content/uploads/2016/11/blog_mapping_data_python_r-300x84.png 300w, http://5047-presscdn.pagely.netdna-cdn.com/wp-content/uploads/2016/11/blog_mapping_data_python_r-768x215.png 768w, http://5047-presscdn.pagely.netdna-cdn.com/wp-content/uploads/2016/11/blog_mapping_data_python_r-100x28.png 100w" sizes="(max-width: 1000px) 100vw, 1000px">

       <div class="entry-content content"> 
        <p>Kaggle users have created nearly <a href="https://www.kaggle.com/kernels" target="_blank">30,000 kernels</a> on our open data science platform so far which represents an impressive and growing amount of reproducible knowledge. I've personally found our repository of code and data to be a great place to learn about new techniques and libraries for Python and R that I otherwise would have never found.</p> 
        <p>In this blog post, I feature some great user kernels as mini-tutorials for getting started with mapping using datasets published on Kaggle. You’ll learn about several ways to wrangle and visualize geospatial data in Python and R including real code examples. I've also included resources so you can learn more about each of the packages highlighted in each tutorial as well as further user analyses for more inspiration.</p> 
        <p><img src="http://5047-presscdn.pagely.netdna-cdn.com/wp-content/uploads/2016/10/data_growth_small.png" alt="data_growth_small" width="75" height="75" class="aligncenter size-full wp-image-6269"></p> 
        <h2 id="the-basics">The basics</h2> 
        <p>Creating a simple map for exploratory purposes doesn’t require you to learn how to manipulate shapefiles or fancy projections. And there are quick and easy ways to put your data on the map whether you prefer to do it in R or Python.</p> 
        <h3 id="r-maps">R Maps</h3> 
        <p>For the R users out there, Kaggler <a href="https://www.kaggle.com/umeshnarayanappa" target="_blank">Umesh</a> shows that all you need are the <code>ggplot2</code> and <code>maps</code> packages by Hadley Wickham to visualize which US states have the highest percentage of daily smokers using data from the CDC published on Kaggle.</p> 
        <p>The <code>map_data</code> function is first used to create a dataframe containing map data which is then merged with state-level data. You can find its <a href="http://docs.ggplot2.org/current/map_data.html" target="_blank">documentation here</a>. (Note that you can also load the <code>mapsdata</code> package for, you guessed it, more map data if <code>maps</code> alone doesn’t have what you need.)</p> 
        <pre class="brush: r; title: ; notranslate" title="">

library(ggplot2)
library(dplyr)
library(tidyr)
library(DT)
library(maps)

smoking &lt;- smoking[smoking$State!="district of columbia",]
smoking$State &lt;- tolower(smoking$State)
all_states &lt;- map_data("state")
smoking$region &lt;- smoking$State
Total &lt;- merge(all_states, smoking, by="region")
</pre> 
        <p>Then creating the map itself is almost as familiar as creating any other <code>ggplot</code> visualization.</p> 
        <pre class="brush: r; title: ; notranslate" title="">
ggplot() +
  geom_polygon(data=Total, aes(x=long, y=lat, group = group, fill=Total$Smoke.everyday),colour="white") +
     scale_fill_continuous(low = "thistle2", high = "darkred", guide="colorbar") +
        labs(fill = "Every Day Smokers", title = "Statewise Every Day Smokers", x="", y="") +
          theme(legend.position = "top") +
        theme_bw() +
          scale_y_continuous(breaks=c()) + 
              scale_x_continuous(breaks=c()) + 
                  theme(panel.border =  element_blank()) +
                    theme(legend.position = "top") 
</pre> 
        <p>The end product is a clear picture of which US states have the most daily smokers.</p> 
        <div id="attachment_6318" style="width: 910px" class="wp-caption aligncenter">

        <p>Here are some further excellent resources for working with <code>maps</code>, <code>mapsdata</code>, and <code>ggplot2</code>:</p> 
        <ul> 
         <li><a href="http://eriqande.github.io/rep-res-web/lectures/making-maps-with-R.html" target="_blank">Making maps in R</a></li> 
         <li><a href="http://zevross.com/blog/2014/07/16/mapping-in-r-using-the-ggplot2-package/" target="_blank">Mapping in R using the <code>ggplot2</code> package</a></li>

        <p>Note that you can’t currently use <code>ggmaps</code> in Kernels. For the most part, you unfortunately can’t do things like make calls to an API from within our environment.</p> 
        <h3 id="python-maps">Python Maps</h3> 
        <p>A good place to start for Python users is the <code>matplotlib basemap</code> toolkit for plotting 2D maps. You can read more in the <code>basemap</code> <a href="http://matplotlib.org/basemap/" target="_blank">documentation here</a> and there are <a href="http://matplotlib.org/basemap/users/examples.html" target="_blank">various examples shown here</a>.</p> 
        <p>There are a lot of great kernels written by users, but Kaggler <a href="https://www.kaggle.com/dotman" target="_blank">Dotman</a> shows how easy it is to <a href="https://www.kaggle.com/dotman/d/fivethirtyeight/uber-pickups-in-new-york-city/data-exploration-and-visualization" target="_blank">visualize data from nearly one million Uber trips</a> in New York City using <code>basemap</code>:</p> 
        <pre class="brush: python; title: ; notranslate" title="">
import pandas as pd
import matplotlib.pyplot as plt
from mpl_toolkits.basemap import Basemap
from matplotlib import cm
%matplotlib inline

west, south, east, north = -74.26, 40.50, -73.70, 40.92

fig = plt.figure(figsize=(14,10))
ax = fig.add_subplot(111)

m = Basemap(projection='merc', llcrnrlat=south, urcrnrlat=north,
            llcrnrlon=west, urcrnrlon=east, lat_ts=south, resolution='i')
x, y = m(uber_data['Lon'].values, uber_data['Lat'].values)
m.hexbin(x, y, gridsize=1000,
         bins='log', cmap=cm.YlOrRd_r);

</pre>

        <h2 id="interactive-maps">Interactive maps</h2> 
        <p>With interactive maps (and interactive data visualization in general) you can limit the ink spilled to only what you believe is more broadly relevant to your audience, but also empower users to drill down in areas where they want more information. Here I've highlighted user-created maps made using Plotly, Leaflet, and Highcharter.</p> 
        <h3 id="plotly">Plotly</h3> 
        <p>In a dataset made available by <a href="https://www.kaggle.com/fivethirtyeight" target="_blank">FiveThirtyEight</a>, users can examine <a href="https://www.kaggle.com/fivethirtyeight/police-officer-deaths-in-the-us/" target="_blank">causes of police officer deaths</a> in the United States going back to 1971. Given location information, Kaggler <a href="https://www.kaggle.com/abigaillarion" target="_blank">Abigail Larion</a> compared maps of police deaths by state using Python and Plotly. Her code demonstrates how simple it is to create a clean-looking, interactive map with counts normalized by state populations:</p> 
        <pre class="brush: python; title: ; notranslate" title="">
import numpy as np
import pandas as pd

import plotly.plotly as py
import plotly.graph_objs as go
from plotly import tools
from plotly.offline import iplot, init_notebook_mode
init_notebook_mode()

# state population estimates for July 2015 from US Census Bureau
# www.census.gov/popest/data/state/totals/2015/tables/NST-EST2015-01.csv
state_population = np.asarray([738432, 4858979, 2978204, 6828065, 39144818, 5456574,\
                               3590886, 672228, 945934, 20271272, 10214860, 1431603,\
                               3123899, 1654930, 12859995, 6619680, 2911641, 4425092,\
                               4670724, 6794422, 6006401, 1329328, 9922576, 5489594,\
                               6083672, 2992333, 1032949, 10042802, 756927, 1896190,\
                               1330608, 8958013, 2085109, 2890845, 19795791, 11613423,\
                               3911338, 4028977, 12802503, 1056298, 4896146, 858469,\
                               6600299, 27469114, 2995919, 8382993, 626042, 7170351,\
                               5771337, 1844128, 586107])

# police officer deaths per 100,000 people in state
police_percapita = police_perstate / state_population * 100000

# District of Columbia outlier (1 law enforcement death per 500 people) adjusted
police_percapita[7] = police_percapita[7] / 10

# plotly code for choropleth map
police_scale = [[0, 'rgb(229, 239, 245)'],[1, 'rgb(1, 97, 156)']]

data = [ dict(
        type = 'choropleth',
        colorscale = police_scale,
        autocolorscale = False,
        showscale = False,
        locations = us_states,
        z = police_percapita,
        locationmode = 'USA-states',
        marker = dict(
            line = dict (
                color = 'rgb(255, 255, 255)',
                width = 2
            ) ),
        ) ]

layout = dict(
        title = 'Police Officer Deaths per 100,000 People in United States (1791-2016)',
        geo = dict(
            scope = 'usa',
            projection = dict( type='albers usa' ),
            showlakes = True,
            lakecolor = 'rgb(255, 255, 255)',
            countrycolor = 'rgb(255, 255, 255)')
             )

figure = dict(data=data, layout=layout)

iplot(figure)
</pre>

        <h3 id="leaflet">Leaflet</h3> 
        <p>Another option for creating interactive maps in Kaggle Kernels is <a href="http://leafletjs.com/" target="_blank">Leaflet</a>, an open-source JavaScript library for mobile-friendly interactive maps. There is a great R package called, well, <a href="https://rstudio.github.io/leaflet/" target="_blank"><code>leaflet</code></a> which “makes it easy to integrate and control Leaflet maps in R.” You can read about the <code>leaflet</code> map widget and how to manipulate its attributes in <a href="https://rstudio.github.io/leaflet/map_widget.html" target="_blank">their tutorial</a>.</p> 
        <p>A fantastic kernel by <a href="https://www.kaggle.com/ewenhen" target="_blank">Ewen Henderson</a> examines neighborhood listings and “superhosts” in Airbnb data from Boston using super concise <code>leaflet</code> code.</p> 
        <pre class="brush: r; title: ; notranslate" title="">
require(tidyverse)
require(leaflet)
require(ggmap)

In another of <a href="https://www.kaggle.com/ewenhen" target="_blank">Ewen Henderson’s</a> kernels, he analyzes the 2016 polls data published as a Kaggle dataset by FiveThirtyEight making <code>highcharter</code> look super easy to use, too.                      to = c(0, 1),

                                      format = '{point.name}')) %&gt;%
  hc_colorAxis(dataClasses = dclass) %&gt;%

        <h2 id="animated-maps">Animated maps</h2> 
        <p>Interactive maps can be great for when you want to give the reader control over exploring details in the data at their leisure. If your goal is to illustrate a particular story, convey change over time as a new dimension in the data, or simply add some eye-catching drama, you may choose animation instead. And yes, you can visualize animated gifs in Kernels.</p> 
        <p>One user, <a href="https://www.kaggle.com/pavelevap" target="_blank">pavelevap</a>, used data recording historical global temperatures to create a stunning animation of average temperatures in cities across the world over time. As you watch the animation unfold, you anxiously hope for more blue orbs to appear. This makes pavelevap’s visualization, using <code>basemap</code>, quite effective.</p> 
        <pre class="brush: python; title: ; notranslate" title="">
import numpy as np 
import pandas as pd
import seaborn as sns
import matplotlib.pyplot as plt

%matplotlib inline

# using Basemap for map visualization. Installed it with "conda install basemap"
from mpl_toolkits.basemap import Basemap
from matplotlib import animation, rc
from IPython.display import HTML

fig = plt.figure(figsize=(10, 10))
cmap = plt.get_cmap('coolwarm')

map = Basemap(projection='cyl')
map.drawmapboundary()
map.fillcontinents(color='lightgray', zorder=1)

START_YEAR = 1950
LAST_YEAR = 2013

n_cities = 500
random_cities = city_means.sample(n_cities).index
year_text = plt.text(-170, 80, str(START_YEAR),fontsize=15)

temp_markers = get_temp_markers(random_cities, START_YEAR)
xs, ys = map(temp_markers['lon'], temp_markers['lat'])
scat = map.scatter(xs, ys, s=temp_markers['size'], c=temp_markers['color'], cmap=cmap, marker='o', 
                   alpha=0.3, zorder=10)

def update(frame_number):
    current_year = START_YEAR + (frame_number % (LAST_YEAR - START_YEAR + 1))
    
    temp_markers = get_temp_markers(random_cities, current_year)
    xs, ys = map(temp_markers['lon'], temp_markers['lat'])

    scat.set_offsets(np.dstack((xs, ys)))
    scat.set_color(cmap(temp_markers['color']))
    scat.set_sizes(temp_markers['size'])
    
    year_text.set_text(str(current_year))

# # # Construct the animation, using the update function as the animation
# # # director.
ani = animation.FuncAnimation(fig, update, interval=500, frames=LAST_YEAR - START_YEAR + 1)

cbar = map.colorbar(scat, location='bottom')
cbar.set_label('0.0 -- min temperature record for the city   1.0 -- max temperature record for the city')
plt.title('Mean year temperatures for {} random cities'.format(n_cities))
plt.show()
</pre>
