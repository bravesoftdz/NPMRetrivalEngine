    <h1 id="introduction">Introduction</h1> 
    <blockquote> 
     <p><i class="fa fa-chevron-circle-up"></i> Pick your favorite programming language</p> 
    </blockquote> 
    <p>Geocodio’s RESTful API allows you to perform forward and reverse geocoding lookups. We support both batch requests as well as individual lookups.</p> 
    <p>You can also optionally ask for data appends such as timezone, congressional districts or similar things of that nature.</p> 
    <p>The base API url is <code class="prettyprint">https://api.geocod.io/v1/</code>.</p> 
    <p>You can also use Geocodio over plain HTTP, but it’s not recommended.</p> 
    <p>All HTTP responses (including errors) are returned with <a href="http://www.json.org">JSON-formatted</a> output.</p> 
    <p>We may add additional properties to the output in the future, but existing properties will never be changed or removed without a new API version release.</p> 
    <aside class="notice">
      Note the versioning prefix in the base url, which is required for all requests. 
    </aside>

    <p>All requests require an API key. You can <a href="https://dash.geocod.io">register here</a> to get your own API key.</p> 
    <p>The API key must be included in all requests using the <code class="prettyprint">?api_key=YOUR_API_KEY</code> query parameter.</p> 
    <p>Accounts can have multiple API keys. This can be useful if you’re working on several projects and want to be able to revoke access using the API key for a single project in the future or if you want to keep track of usage per API key.</p> 
    <p>You can also download a CSV of usage and fees per API key.</p> 
    <aside class="warning">
      Make sure to replace YOUR_API_KEY with your personal API key found on the 
     <a href="https://dash.geocod.io" target="_blank">Geocodio dashboard</a>. 
    </aside> 
    <h1 id="geocoding">Geocoding</h1> 
    <p>Geocoding (also known as forward geocoding) allows you to convert one or more addresses into geographic coordinates (i.e. latitude and longitude). Geocoding will also parse the address and append additional information (e.g. if you specify a zip code, Geocodio will return the city and state corresponding the zip code as well)</p> 
    <p>Geocodio supports geocoding of addresses, cities and zip codes in various formats.</p> 
    <aside class="notice">
      Make sure to check the 
     <a href="#address-formats">address formats</a> section for more information on the different address formats supported. 
    </aside> 
    <p>You can either geocode a single address at a time or collect multiple addresses in batches in order to geocode up to 10,000 addresses at the time.</p> 
    <p>Whenever possible, batch requests are recommended since they are significantly faster due to reduced network overhead.</p>

    <p>If you have several addresses that you need to geocode, batch geocoding is a much faster option since it removes the overhead of having to perform multiple <code class="prettyprint">HTTP</code> requests.</p> 
    <p>Batch geocoding requests are performed by making a <code class="prettyprint">POST</code> request to the <em>geocode</em> endpoint, suppliying a <code class="prettyprint">JSON</code> array or <code class="prettyprint">JSON</code> object in the body with any key of your choosing.</p> 
    <aside class="warning">
      You can batch geocode up to 10,000 addresses at the time. Geocoding 10,000 addresses takes about 300 seconds, so please make sure to adjust your timeout value accordingly. 
    </aside>

    <h3 id="json-array-object">JSON array/object</h3> 
    <p>When making a batch geocoding request, you can <code class="prettyprint">POST</code> queries as either a JSON array or a JSON object. If a JSON object is posted, you can specify a custom key for each element of your choice, this can be useful to match queries up with your existing data, after the request is complete.</p>

    <h1 id="reverse-geocoding">Reverse Geocoding</h1> 
    <p>Reverse geocoding is the process of turning geographic coordinates (i.e. latitude and longitude) into a human-readable address.</p> 
    <p>Geocodio will find matching street(s) and determine the correct house number based on the location. Note that Geocodio does not guarantee to return a valid house number; it is our closest approximation.</p> 
    <p>As with forward geocoding, you can either geocode a single set of coordinates at the time or collect multiple coordinates in batches and reverse geocode up to 10,000 coordinates at the time.</p> 
    <p>This endpoint can return up to 5 possible matches ranked and ordered by an <a href="#accuracy-score">accuracy score</a>.</p> 
    <aside class="success">
      A geographic coordinate consists of latitude followed by longitude separated by a comma, e.g. `38.9002898,-76.9990361` 
    </aside>

    <h2 id="congressional-districts">Congressional Districts</h2> 
    <pre class="highlight json"><code><span class="err">...</span><span class="w"> </span><span class="s2">"fields"</span><span class="err">:</span><span class="w"> </span><span class="p">{</span><span class="w"> </span><span class="nt">"congressional_district"</span><span class="p">:</span><span class="w"> </span><span class="p">{</span><span class="w"> </span><span class="nt">"name"</span><span class="p">:</span><span class="w"> </span><span class="s2">"Congressional District 36"</span><span class="p">,</span><span class="w"> </span><span class="nt">"district_number"</span><span class="p">:</span><span class="w"> </span><span class="mi">36</span><span class="p">,</span><span class="w"> </span><span class="nt">"congress_number"</span><span class="p">:</span><span class="w"> </span><span class="s2">"115th"</span><span class="p">,</span><span class="w"> </span><span class="nt">"congress_years"</span><span class="p">:</span><span class="w"> </span><span class="s2">"2017-2019"</span><span class="w"> </span><span class="p">}</span><span class="w"> </span><span class="p">}</span><span class="w"> </span><span class="err">...</span><span class="w"> </span></code></pre> 
    <p>You can retrieve the congressional district for an address or coordinate using <code class="prettyprint">cd</code>, <code class="prettyprint">cd113</code>, <code class="prettyprint">cd114</code>, or <code class="prettyprint">cd115</code> in the <code class="prettyprint">fields</code> query parameter. <code class="prettyprint">cd</code> will always return the congressional district for the current progress while e.g. <code class="prettyprint">cd113</code> will continue to show the congressional district for the 113th congress.</p> 
    <p>The field returns the full name of the congressional district, the district number the congress number and the range of years the congress are covering.</p> 
    <aside class="warning">
      It might be tempting to look congressional districts up by zip code, but this is discouraged — primarily because zip codes are postal routes and not areas which can cause imprecise results. Also note that some zip codes span over multiple congressional districts and Geocodio will only return one district per result. 
    </aside>

    <pre class="highlight json"><code><span class="err">...</span><span class="w"> </span><span class="s2">"fields"</span><span class="err">:</span><span class="w"> </span><span class="p">{</span><span class="w"> </span><span class="nt">"state_legislative_districts"</span><span class="p">:</span><span class="w"> </span><span class="p">{</span><span class="w"> </span><span class="nt">"house"</span><span class="p">:</span><span class="w"> </span><span class="p">{</span><span class="w"> </span><span class="nt">"name"</span><span class="p">:</span><span class="w"> </span><span class="s2">"Assembly District 42"</span><span class="p">,</span><span class="w"> </span><span class="nt">"district_number"</span><span class="p">:</span><span class="w"> </span><span class="mi">42</span><span class="w"> </span><span class="p">},</span><span class="w"> </span><span class="nt">"senate"</span><span class="p">:</span><span class="w"> </span><span class="p">{</span><span class="w"> </span><span class="nt">"name"</span><span class="p">:</span><span class="w"> </span><span class="s2">"State Senate District 28"</span><span class="p">,</span><span class="w"> </span><span class="nt">"district_number"</span><span class="p">:</span><span class="w"> </span><span class="mi">28</span><span class="w"> </span><span class="p">}</span><span class="w"> </span><span class="p">}</span><span class="w"> </span><span class="p">}</span><span class="w"> </span><span class="err">...</span><span class="w"> </span></code></pre> 
    <p>You can retrieve the state legislative districts for an address or coordinate using <code class="prettyprint">stateleg</code> in the <code class="prettyprint">fields</code> query parameter.</p> 
    <p>The field will return both the <em>house</em> and <em>senate</em> state legislative district (also known as <em>lower</em> and <em>upper</em>) with the full name and district numbe for each. For areas with a <a href="http://en.wikipedia.org/wiki/Unicameralism">unicameral legislature</a> (such as Washington DC or Nebraska), only the <code class="prettyprint">senate</code> key is returned.</p>

    <h1 id="accuracy-score">Accuracy score</h1> 
    <p>Each geocoded result is returned with an accuracy score, which is a decimal number ranging from 0.00 to 1.00. This score is generated by the internal Geocodio engine based on how accurate the result is believed to be. The higher the score the better the result. Results are always returned ordered by accuracy score.</p> 
    <p>For example, if against all odds an address simply can’t be found — instead of returning no results, Geocodio will return a geocoded point based on the zip code or city but with a much lower accuracy score and accuracy type set to “place”.</p> 
    <p>Generally, accuracy scores that are larger than or equal to <code class="prettyprint">0.8</code> are usually helpful, whereas results with lower accuracy scores might be very rough matches.</p> 
    <p>An accuracy type is also returned with all results. The accuracy types are different for forward and reverse geocoding results.</p> 
    <p>We recommend using a combination of the accuracy score and accuracy type to evaluate and filter the returned results.</p>

    <h1 id="address-formats">Address formats</h1> 
    <p>Geocodio supports geocoding the following entities:</p> 
    <ul> 
     <li>Streets with or without house numbers (requires a city or a zip in conjuction)</li> 
     <li><a href="#intersections">Intersections</a></li> 
     <li>Cities</li> 
     <li>Zip codes</li> 
     <li>States</li> 
    </ul> 
    <p>If a city is provided without a state, Geocodio will automatically guess and add the state based on what it most likely might be. Geocodio also understands shorthands for both streets and cities, e.g. <em>NYC</em>, <em>SF</em>, etc. are acceptable city names.</p> 
    <p>Geocoding queries can be formatted in various ways. Here are some examples of valid queries:</p> 
    <ul> 
     <li><a href="https://api.geocod.io/v1/geocode?q=1109+N+Highland+St%2c+Arlington+VA&amp;api_key=YOUR_API_KEY" target="_blank">1109 N Highland St, Arlington VA</a></li>

     <li><a href="https://api.geocod.io/v1/geocode?q=1109+North+Highland+Street%2c+Arlington+VA&amp;api_key=YOUR_API_KEY" target="_blank">1109 North Highland Street, Arlington VA</a></li> 
     <li><a href="https://api.geocod.io/v1/geocode?q=1109+N+Highland+St%2c+Arlington+VA&amp;api_key=YOUR_API_KEY" target="_blank">1109 N Highland St, Arlington VA</a></li>

    <h1 id="errors">Errors</h1> 
    <blockquote> 
     <p>Here is an example of a 422 Unprocessable Entity response:</p> 
    </blockquote> 
    <pre class="highlight json"><code><span class="p">{</span><span class="w"> </span><span class="nt">"error"</span><span class="p">:</span><span class="w"> </span><span class="s2">"Could not geocode address, zip code or city/state are required"</span><span class="w"> </span><span class="p">}</span><span class="w"> </span></code></pre> 
    <p>The Geocodio API employs semantic HTTP status codes. This is some of the status codes you can expect to see:</p> 
    <table>
     <thead> 
      <tr> 
       <th>Error Code</th> 
       <th>Meaning</th> 
      </tr> 
     </thead>
     <tbody> 
      <tr> 
       <td>200 OK</td> 
       <td>Hopefully you will see this most of the time. Note that this status code will also be returned even though no geocoding results were available</td> 
      </tr> 
      <tr> 
       <td>403 Forbidden</td> 
       <td>Invalid API key or other reason why access is forbidden</td> 
      </tr> 
      <tr> 
       <td>422 Unprocessable Entity</td> 
       <td>A client error prevented the request from executing succesfully (e.g. invalid address provided). A JSON object will be returned with an error key containing a full error message</td> 
      </tr> 
      <tr> 
       <td>500 Server Error</td> 
       <td>Hopefully you will never see this…it means that something went wrong in our end. Whoops.</td> 
      </tr> 
     </tbody>
    </table> 
    <p>Please check <a href="http://status.geocod.io">status.geocod.io</a> for latest platform status updates, if encounter see any unexpected errors.</p>
