           <p name="ce3b" id="ce3b" class="graf graf--p graf-after--h3"><a href="http://geojson.org" data-href="http://geojson.org" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">GeoJSON</a> has been increasingly becoming a standard format to store geospatial (geographical) information, especially for Web applications. Tools like ‘leaflet’ is making it super easy to visualize data in GeoJSON format with Java Script. And, tools like <a href="https://blog.exploratory.io/visualizing-geospatial-data-with-your-own-geojson-f96dde0f6296#.4j31lhsts" data-href="https://blog.exploratory.io/visualizing-geospatial-data-with-your-own-geojson-f96dde0f6296#.4j31lhsts" class="markup--anchor markup--p-anchor" target="_blank">Exploratory</a> is making it easier to visualize the data even without programming.</p>
           <p name="d7b5" id="d7b5" class="graf graf--p graf-after--p">If you want to know more about GeoJSON, here is a great article, “<a href="http://www.macwright.org/2015/03/23/geojson-second-bite.html" data-href="http://www.macwright.org/2015/03/23/geojson-second-bite.html" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">More than you ever wanted to know about GeoJSON</a>”, from <a href="https://twitter.com/tmcw" data-href="https://twitter.com/tmcw" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Tom MacWright</a>, which explains GeoJSON in a comprehensive and easy to digest way.</p>
           <p name="7e21" id="7e21" class="graf graf--p graf-after--p">The problem is though, most of the geospatial data publicly available today are still in something called ‘Shapefile’ format, which is defined by <a href="http://www.esri.com" data-href="http://www.esri.com" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Esri</a>?—?a comapany behind one of the most popular GIS (Geographical Information System) product?—?and used to exchange the geospatial data among GIS softwares. It contains pretty much the same thing you find in GeoJSON, such as Polygons, Lines, Points, and their attribute information.</p>
           <p name="e5b5" id="e5b5" class="graf graf--p graf-after--p">The good news is, converting those shapefile format files to GeoJSON format files has become much easier in R, thanks to ‘<a href="https://cran.r-project.org/web/packages/rgdal/index.html" data-href="https://cran.r-project.org/web/packages/rgdal/index.html" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">rgdal</a>’ package from <a href="https://www.nhh.no/en/employees/faculty/roger-bivand/" data-href="https://www.nhh.no/en/employees/faculty/roger-bivand/" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Roger Bivand</a> and others, and the recent advancement in GeoJSON data handling with many amazing packages from <a href="http://ropensci.org" data-href="http://ropensci.org" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">ROpenSci</a> folks.</p>
           <p name="2f96" id="2f96" class="graf graf--p graf-after--p">In this post, I’m going to walk you through how you can produce your GeoJSON files out of shapefile format files and manipulate the data to fit your needs, so that you can use tools like leaflet or <a href="https://blog.exploratory.io/visualizing-geospatial-data-with-your-own-geojson-f96dde0f6296#.4j31lhsts" data-href="https://blog.exploratory.io/visualizing-geospatial-data-with-your-own-geojson-f96dde0f6296#.4j31lhsts" class="markup--anchor markup--p-anchor" target="_blank">Exploratory</a>, which uses leaflet internally, to quickly visualize the spatial data.</p>
           <p name="6087" id="6087" class="graf graf--p graf-after--p">Here is the basic flow.</p>
           <ol class="postList">
            <li name="8e82" id="8e82" class="graf graf--li graf-after--p">Obtain Shapefile?—?Geospatial (Map) Data</li>
            <li name="8954" id="8954" class="graf graf--li graf-after--li">Import Shapefile to R</li>
            <li name="46cb" id="46cb" class="graf graf--li graf-after--li">Manipulate Attributes Data in Spatial Data Frame</li>
            <li name="6140" id="6140" class="graf graf--li graf-after--li">Convert to GeoJSON</li>
            <li name="73ae" id="73ae" class="graf graf--li graf-after--li">Manipulate Geometry Data in GeoJSON?—?Simplify and Clip / Erase</li>
            <li name="76f0" id="76f0" class="graf graf--li graf-after--li">Export as GeoJSON file</li>
           </ol>
           <h3 name="1fd7" id="1fd7" class="graf graf--h3 graf-after--li">Install R packages for spatial and GeoJSON data&nbsp;handling</h3>
           <p name="2052" id="2052" class="graf graf--p graf-after--h3">To do all the operations below, you will need to install the following R packages first.</p>
           <ul class="postList">
            <li name="3d0d" id="3d0d" class="graf graf--li graf-after--p"><strong class="markup--strong markup--li-strong">rgdal</strong>?—?for importing Shapefiles to R as Spatial data frame.</li>
            <li name="606a" id="606a" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">spdplyr</strong>?—?for manipulating the attribute data inside the spatial data frame.</li>
            <li name="643e" id="643e" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">geojsonio</strong>?—?for converting the spatial data frame to GeoJSON and saving to file systems.</li>
            <li name="abb5" id="abb5" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">rmapshaper</strong>?—?for manipulating the geometry (polygon, line, marker) part of the GeoJSON data.</li>
           </ul>
           <pre name="8573" id="8573" class="graf graf--pre graf-after--li">install.packages(rgdal)<br>install.packages(spdplyr)<br>install.packages(geojsonio)<br>install.packages(rmapshaper)</pre>

One is a data frame that stores the attribute information like ID, Name, or other information. And the other part stores geometry information like polygons, which draw boundaries, lines, etc.</p>

           <p name="2fde" id="2fde" class="graf graf--p graf-after--h4">To import the shapefile data into R, you can use ‘<strong class="markup--strong markup--p-strong">readOGR</strong>’ function from ‘<strong class="markup--strong markup--p-strong">rgdal</strong>’ package, which is basically an R interface for <a href="http://www.gdal.org" data-href="http://www.gdal.org" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">gdal</a> (Geospatial Data Abstraction Library).</p>
           <pre name="8d68" id="8d68" class="graf graf--pre graf-after--p">library(<strong class="markup--strong markup--pre-strong">rgdal</strong>)</pre>
           <pre name="e85e" id="e85e" class="graf graf--pre graf-after--pre">county &lt;- <strong class="markup--strong markup--pre-strong">readOGR</strong>(dsn = "/User/kan/Data/cb_2015_us_county_500k"),<br>                  layer = "cb_2015_us_county_500k", verbose = FALSE)</pre>
           <p name="63af" id="63af" class="graf graf--p graf-after--pre">The first argument is to set the path to the folder that contains all the shape files like ‘.shp’, etc. And the second is to set the name of the files without the file extension. Note that this function doesn’t like a relative path (e.g. ‘~/Data’) for ‘dsn’ argument, so you want to set an absolute path (‘/User/kan/Data’) here.</p>

If your original shape files included Polygons (Area) then you would end up with <strong class="markup--strong markup--p-strong">SpatialPolygonsDataFrame</strong>. If they were Lines then it would be <strong class="markup--strong markup--p-strong">SpatialLinesDataFrame</strong>.</p>

           <h3 name="18d3" id="18d3" class="graf graf--h3 graf-after--p">3. Manipulate Attribute Data, if necessary</h3>
           <p name="fc34" id="fc34" class="graf graf--p graf-after--h3">This is not a mandatory step. If you are fine with the existing attribute information and want to generate GeoJSON with all the data, then you can skip to the next step. Otherwise, I’m listing a few scenarios where you might want to manipulate the data.</p>
           <p name="6476" id="6476" class="graf graf--p graf-after--p">As mentioned above, thanks to ‘<strong class="markup--strong markup--p-strong">spdplyr</strong>’ package, you can easily manipulate the attribute part of the spatial data frame, just like you would do with dplyr commands with good ‘ol data frame.</p>
           <h4 name="258c" id="258c" class="graf graf--h4 graf-after--p">Filtering to get only California geospatial information</h4>
           <p name="0afe" id="0afe" class="graf graf--p graf-after--h4">Let’s say we are interested in only the counties in California. We can use filter command and set California (06) by using STATEFP column.</p>
           <pre name="9664" id="9664" class="graf graf--pre graf-after--p">library(spdplyr)</pre>
           <pre name="adb0" id="adb0" class="graf graf--pre graf-after--pre">county_ca &lt;- county %&gt;% <br>  <strong class="markup--strong markup--pre-strong">filter</strong>(STATEFP == "06")</pre>
           <p name="9451" id="9451" class="graf graf--p graf-after--pre">In case, if you are not familiar with the dplyr or R basic, ‘&lt;-’ is to store the result of the right hand side to the left hand side (county_ca). And, ‘%&gt;%’ is to get the data of the left hand side (county) and pass it to the operation at right hand side.</p>
           <h4 name="d9c9" id="d9c9" class="graf graf--h4 graf-after--p">Renaming the column&nbsp;names</h4>
           <p name="a528" id="a528" class="graf graf--p graf-after--h4">If you just want to rename attribute names in the data, you can use ‘rename’ command like below.</p>
           <pre name="d1ab" id="d1ab" class="graf graf--pre graf-after--p">county_id &lt;- county %&gt;% <strong class="markup--strong markup--pre-strong">rename</strong>(county_id = GEOID)</pre>
           <h4 name="5e5f" id="5e5f" class="graf graf--h4 graf-after--pre">Selecting or Dropping&nbsp;columns</h4>
           <p name="2cbf" id="2cbf" class="graf graf--p graf-after--h4">There might be too many columns in your imported spatial data frame. You can select only the columns you need with ‘select’ command.</p>
           <pre name="6f2d" id="6f2d" class="graf graf--pre graf-after--p">county_select &lt;- county %&gt;% <strong class="markup--strong markup--pre-strong">select</strong>(STATEFP, COUNTYFP, GEOID, NAME)</pre>
           <p name="8166" id="8166" class="graf graf--p graf-after--pre">As you have seen at the above examples, it’s super easy to manipulate the attribute part of the spatial data frame. Notice that the result of the manipulation is still in the spatial data frame with all the geometry information. Kudo to spdplyr! ?</p>
           <h3 name="6646" id="6646" class="graf graf--h3 graf-after--p">5. Manipulate Geoemtry Data in&nbsp;GeoJSON</h3>
           <p name="16af" id="16af" class="graf graf--p graf-after--h3">This is also not mandatory step. If you are fine with the one you have, then skip to the next step.</p>
           <p name="5a1e" id="5a1e" class="graf graf--p graf-after--p">There are two common scenarios why you might want to manipulate the geometry part of the data (e.g. Polygons, Lines, etc).</p>
           <ul class="postList">
            <li name="3eeb" id="3eeb" class="graf graf--li graf-after--p">Simplfy the geometry (Polygons, Lines) information to reduce the data size while keeping the integrity of the geometry intact.</li>
            <li name="c08d" id="c08d" class="graf graf--li graf-after--li">Keep or remove certain area of the geometry.</li>
           </ul>
           <p name="1ab2" id="1ab2" class="graf graf--p graf-after--li">Here’s how.</p>
           <h4 name="e912" id="e912" class="graf graf--h4 graf-after--p">5.1. Convert to&nbsp;GeoJSON</h4>
           <p name="e085" id="e085" class="graf graf--p graf-after--h4">Before manipulating the geometry data, we want to convert the spatial data frame to GeoJSON first. Though you can manipulate it directly agasint the spatial data frame, I’m finding it much easier to do with GeoJSON today, thanks to amazing R packages like ‘<a href="https://cran.r-project.org/web/packages/geojsonio/index.html" data-href="https://cran.r-project.org/web/packages/geojsonio/index.html" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank"><strong class="markup--strong markup--p-strong">geojsonio</strong></a>’ from <a href="https://twitter.com/sckottie" data-href="https://twitter.com/sckottie" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Scott Chamberlain</a> and ‘<a href="https://cran.r-project.org/web/packages/rmapshaper/index.html" data-href="https://cran.r-project.org/web/packages/rmapshaper/index.html" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank"><strong class="markup--strong markup--p-strong">rmapshaper</strong></a>’ from <a href="https://twitter.com/andyteucher" data-href="https://twitter.com/andyteucher" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Andy Teucher</a>.</p>
           <p name="8492" id="8492" class="graf graf--p graf-after--p">We can convert from Spatial Data Frame to GeoJSON with a function called <strong class="markup--strong markup--p-strong">geojson_json</strong> from <strong class="markup--strong markup--p-strong">geojsonio</strong> package, which makes it easy to make the geospatial data in and out of GeoJSON format.</p>
           <pre name="be50" id="be50" class="graf graf--pre graf-after--p">county_json &lt;- <strong class="markup--strong markup--pre-strong">geojson_json</strong>(county)</pre>

           <h4 name="3019" id="3019" class="graf graf--h4 graf-after--figure">5.2. Simplify GeoJSON&nbsp;file</h4>
           <p name="7292" id="7292" class="graf graf--p graf-after--h4">GeoJSON file size can easily get bigger because it just has a lot of information in order to draw the polygons or lines. And as you would imagine, the bigger the file size is, the slower it gets for loading, processing, and drawing the data on Map.</p>
           <p name="4e4c" id="4e4c" class="graf graf--p graf-after--p">The good news is, you can ’simplify’ the geometry data (polygons and lines). ‘Simplify’ means that it will remove the detail information of the geometry that might not make much of the difference especially when you are visualizing the data at high level.</p>
           <p name="a389" id="a389" class="graf graf--p graf-after--p">And even better, you can perform ‘simplify’ operation in R against GeoJSON data easily, thanks to this function called ‘<strong class="markup--strong markup--p-strong">ms_simpify</strong>’ from ‘<a href="https://cran.r-project.org/web/packages/rmapshaper/index.html" data-href="https://cran.r-project.org/web/packages/rmapshaper/index.html" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank"><strong class="markup--strong markup--p-strong">rmapshaper</strong></a>’ package. ‘rmapshaper’ is an R interface for mapshaper library in JavaScript. It does an amazing job to reduce the data size without losing the integrity of the data. Here is how it looks before and after the ‘simplify’ operation with the default setting, which is to retain 0.05% of the original data with <a href="https://bost.ocks.org/mike/simplify/" data-href="https://bost.ocks.org/mike/simplify/" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank"><strong class="markup--strong markup--p-strong">Visvalingam</strong> simplification algorithm</a>.</p>
