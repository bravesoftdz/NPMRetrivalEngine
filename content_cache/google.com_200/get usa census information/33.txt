          <h1 id="article-headline" itemprop="headline">ABS hobbles census data downloaders</h1> 
          <div id="article-details-mobile">
            By 
           <a href="/author/juha-saarinen-224495"> <span itemprop="author" itemscope itemtype="http://schema.org/Person"> <span itemprop="name">Juha Saarinen</span> </span> </a> on

            <h2 id="article-intro" itemprop="description">DVDs charged at $250 a pop.</h2> 
            <div id="article-body" itemprop="articleBody"> 
             <p> The Australian Bureau of Statistics has released the latest census data for free under a Creative Commons license but appears to be steering people towards a $250 mailed out DVD rather than making it easy to download the information directly over the internet.</p> 
             <p> Programmer and freelance journalist Grahame Bowland who first noticed it, said the government agency is going to great lengths to discourage people from downloading the files directly by dint of &nbsp;a convoluted site layout and Javascript functions that obfuscate file paths.</p> 
             <p> Bowland discovered how difficult it was to download the data packs for free when he tried to do it, with the ABS forcing him to jump through several hoops before accessing the data.</p> 
             <p> First, people have to register on a hard-to-find <a href="http://www.abs.gov.au/websitedbs/D3310114.nsf/Home/Registration+Centre?opendocument&amp;navpos=290">page</a>, Bowland said, after which they are redirected to another page with a big matrix of data packs.</p> 
             <div class="ArticleImage" style="text-align:center"> 
              <img src="https://i.nextmedia.com.au/Utils/ImageResizer.ashx?n=http%3a%2f%2fi.nextmedia.com.au%2fNews%2fabs+screenshot.JPG&amp;w=460&amp;c=0" style="border-width:0px;"> 
              <div class="ArticleImageCaption"> 
               <em>ABS census datapacks download page</em>
              </div> 
             </div> 
             <p> "You have to click to download each pack individually, and they've set the site up deliberately to make it difficult to use a browser plugin to download everything that is contained on the released DVD image," Bowland told <em>iTNews</em>.</p> 
             <p> <span style="font-size: 12px;">Javascript that obfuscates the file paths has been added to the site by the ABS developers, including comments. Bowland </span><span style="font-size: 12px;">provided the below commented code sample directly from the agency site to <em>iTNews</em>:</span></p> 
             <blockquote> 
              <p> // Function: guidGenerator<br> // Description:returns a pseudo-random GUID<br> //This is appended to a url for 2 reasons<br> //1. to make the URL unique, so that the browser always gets it and doesn't use a cached version<br> //2. to make a URL look like its got a unique key, in a naive attempt to fool a not-so-wily hacker<br> //into thinking they can't download a datapack directly if they know the URL pattern, because they<br> //need a unique key.<br> &nbsp;</p> 
             </blockquote> 
             <p> "The ABS is trying to obfuscate paths to make it hard for people to bulk download the data, and labelling people who want to 'hackers'," Bowland said.</p> 
             <p> <span style="font-size: 12px;">Bowland said on closer inspection of the code it appears you don't actually have to register at all to get the data packs, as you can use a&nbsp;</span><span style="font-size: 12px;">Javascript code snippet to generate the URLs of all the data packs.</span></p> 
             <p> To prevent this, the ABS Javascript coders have documented what they call a "pathetic attempt" at stopping the direct data pack downloads by using a randomly generated number appended to the URLs so as to make them appear as if a complex key is required as per below:</p> 
             <blockquote> 
              <p> // Function: getZip<br> // Parameters:fileName - the file to be downloaded.<br> //There are basically 2 formats, normal DataPacks and boundary files. For example:<br> //* 2010_BCP_SA1_for_Vic_short-header.zip<br> //* 2010_SA4_POW_shape.zip&nbsp; or without the POW eg. 2010_SA4_shape.zip<br> // Descrition:<br> // This function is ultimately fired when a user clicks on a download link on the DataPacks download page.<br> // It does 4 things:<br> //<br> // Step 1. Get some dynamic values from the page. These were substituted when the page was created.<br> //'dpserver' is the full domino path, as seen in the dominoserver.properties value DataPacks.DominoServerExt<br> //&nbsp; eg. http://www.censusdata.idev.abs.gov.au/CensusOutput/copsubdatapacks.nsf/All%20docs%20by%20catNo,<br> //&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Also, generate a random number, which we append to the URL, to make it appear as if a complex<br> //key is required. This is a pathetic attempt to discourage someone from downloading the ZIPs<br> //directly (ie. without having to login), if they deduce the URL pattern.<br> //It's also used to make every URL unique, so that the browser always sends the request to the server,<br> //(ie. doesn't use its cached version), because we want to know about every click.</p> 
             </blockquote> 
             <p> Despite having got this far, Bowland noted that the some additional geometry files for DVD 3 couldn't be found on the ABS website, so he decided to stump up $250 for all the releases to be mailed to him.</p> 
             <p> According to Bowland, the high cost set to post out an optical disc means "in reality, they're subsidising internal admin roles by selling DVDs".</p> 
             <p> For those who don't wish to tangle with obfuscating Javascript or pay hefty charges for DVDs, Bowland has made the census data available for <a href="http://blog.angrygoats.net/2013/04/12/2011-australian-census-release-3/">download via Bittorrent</a> on his website.</p> 
             <p> <strong>ABS responds</strong></p> 
             <p> A spokesperson for the ABS said the $250 charge for the DVDs was to "recover administration costs" but pointed out that this was the first time its census data had been made free and available to everyone via its&nbsp;<a href="http://abs.gov.au/census">website</a>.</p> 
             <p> According to the spokesperson, the ABS has worked hard to reduce the costs since 2006, when similar datapacks cost $805.</p> 
             <p> As for the convoluted download site layout with registration and obfuscated file paths, the spokesperson said there was room for improvement.</p> 
             <p> "The ABS is constantly looking at ways it can simplify the website and enhance the user experience," <em>iTnews</em> was told via email.</p> 
             <p> "We will shortly be conducting a review of all census products and services and will engage users of census data to better understand their needs," the spokesperson added.</p>
