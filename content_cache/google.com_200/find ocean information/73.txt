         <p>This is one of the common development cycles of any developer. </p> 
         <p>Most of the time testing and deployment steps do not change frequently and in order to keep the developer focus on writing code, we do the automation of testing and deployment. </p> 
         <p>This automation is called <strong>“continuous integration and deployment”.</strong></p> 
         <p>In this tutorial, we will learn how to set up the continuous integration and deployment infrastructure for the Node.js project.</p> 
         <h2>Prerequistics</h2> 
         <p>I am going to use <strong>Jenkins</strong> as the automation tool and <strong>Github</strong> as source code management system. </p> 
         <p><strong>Jenkins</strong> is free for use but we require public server in order to integrate the <strong>Github</strong> repository.</p> 
         <p>For the server, I will be using <strong>$5</strong> droplet on Digitalocean. In case you wish to buy it, you can use <a href="https://m.do.co/c/31513addbd1b" target="_blank" rel="nofollow">this</a> link to get the <strong>$10</strong> off for the first month.</p> 
         <p><strong>Related learning : </strong> <a href="https://codeforgeek.com/2016/03/hosting-node-js-app-to-digitalocean-server/" target="_blank">Hosting Node.js app to Digitalocean Server</a></p> 
         <h2>How it’s going to work</h2> 
         <p>Here is the complete flow of the system.</p> 
         <ul> 
          <li>You made some changes in your project. </li> 
          <li>You push those changes in Github on master or any branch. </li> 
          <li>Github will notify Jenkins about the new push. ( We will configure it ) </li> 
          <li> Jenkins will then run the commands you ask it to run.</li> 
         </ul> 
         <p>Those commands will contain following.</p> 
         <ul> 
          <li>Test script. </li> 
          <li>Deployment script.</li> 
         </ul> 
         <p>Deployment script will be added to Project only and Jenkins will use that to communicate to Server and perform the push.</p> 
         <p>Basically, what you do manually such as running test command, login to Server, performing Git pull and then restarting build server, we will do all of them automatically by writing the script.</p>

         <h3>Adding Github webhook to push events to Jenkins</h3> 
         <p>When any changes i.e commits are pushed to Github repository, we need a mechanism to notify that event to our Jenkins Server which we going to configure in next section.</p> 
         <p>Github allows us to add Webhook services to achieve same. Open GitHub repository, go to settings and click <strong>“Webhooks and Services”</strong> from side panel. Click on <strong>“Add Services”</strong> and search Jenkins, click on <strong>“Jenkins Github Plugin”.</strong></p> 
         <p>Note: Don’t choose Git one.</p> 
         <p><img src="https://i1.wp.com/codeforgeek.com/wp-content/uploads/2016/04/Screen-Shot-2016-08-30-at-12.52.18-PM.png?resize=640%2C361&amp;ssl=1" alt="Adding Github webhook" class="aligncenter size-full wp-image-2567" srcset="https://i1.wp.com/codeforgeek.com/wp-content/uploads/2016/04/Screen-Shot-2016-08-30-at-12.52.18-PM.png?w=1003&amp;ssl=1 1003w, https://i1.wp.com/codeforgeek.com/wp-content/uploads/2016/04/Screen-Shot-2016-08-30-at-12.52.18-PM.png?resize=300%2C169&amp;ssl=1 300w, https://i1.wp.com/codeforgeek.com/wp-content/uploads/2016/04/Screen-Shot-2016-08-30-at-12.52.18-PM.png?resize=768%2C433&amp;ssl=1 768w, https://i1.wp.com/codeforgeek.com/wp-content/uploads/2016/04/Screen-Shot-2016-08-30-at-12.52.18-PM.png?resize=640%2C361&amp;ssl=1 640w" sizes="(max-width: 640px) 100vw, 640px" data-recalc-dims="1"></p>

         </div> 
         <div class="codecolorer-container text twitlight" style="overflow:auto;white-space:nowrap;width:99%;">
          <div class="text codecolorer">

         <h3>Configuring deployment process</h3> 
         <p>We have Github communicating to Jenkins, it’s time to perform the deployment to our Server. One thing, Jenkins will not directly communicate to your web server. All it can do is to run instruction which you want it to run.</p> 
         <p>So let’s first list down the steps we manually perform to do the deployment to any Server.</p> 
         <ul> 
          <li>Login to Server using SSH.</li> 
          <li>Switching to project directory.</li> 
          <li>Pulling code from Github.</li> 
          <li>Restarting process manager ( say pm2 ).</li> 
         </ul> 
         <p>We will write down all of these commands in one file and tell Jenkins to execute it. </p> 
         <p>But, what about SSH username and password ? Of course, Jenkins won’t enter it like we do. We need to make sure Jenkins can perform SSH login without using the password to our Server.</p> 
         <p>To achieve that, we will generate SSH key manually from Jenkins Server and add it into our Development Server. </p> 
         <p>Let’s do it.</p> 
         <p>Login to Jenkins Server using Terminal and switch to root user using this command.</p> 
         <div class="codecolorer-container text twitlight" style="overflow:auto;white-space:nowrap;width:99%;">
          <div class="text codecolorer">
           sudo su
          </div>
         </div> 
         <p>Jenkins automatically creates a new user after installation. Switch to it using this command.</p> 
         <div class="codecolorer-container text twitlight" style="overflow:auto;white-space:nowrap;width:99%;">
          <div class="text codecolorer">
           su jenkins
          </div>
         </div> 
         <p>Let’s generate RSA key, run following command.</p> 
         <div class="codecolorer-container text twitlight" style="overflow:auto;white-space:nowrap;width:99%;">
          <div class="text codecolorer">
           ssh-keygen -t rsa
          </div>
         </div> 
         <p>Press Enter for the location and do not type any password when it asks to, just hit enter.</p> 
         <p>Once the process is completed, print the public key information using this command.</p> 
         <div class="codecolorer-container text twitlight" style="overflow:auto;white-space:nowrap;width:99%;">
          <div class="text codecolorer">
           cat ~/.ssh/id_rsa.pub
          </div>
         </div> 
         <p>It should start with <strong>ssh-rsa</strong> and ends with <strong><a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="305a555e5b595e437054425f405c55441d5940">[email&nbsp;protected]</a></strong>. Copy the key.</p> 
         <p>Now login to your development server and switch to <strong>~/.ssh</strong> directory using following command.</p> 
         <div class="codecolorer-container text twitlight" style="overflow:auto;white-space:nowrap;width:99%;">
          <div class="text codecolorer">
           cd ~/.ssh
          </div>
         </div> 
         <p>If it’s not present, create one using this command.</p> 
         <div class="codecolorer-container text twitlight" style="overflow:auto;white-space:nowrap;width:99%;">
          <div class="text codecolorer">
           mkdir ~/.ssh
          </div>
         </div> 
         <p>Now open the file named <strong>authorized_keys</strong> if its present, else create one.</p> 
         <div class="codecolorer-container text twitlight" style="overflow:auto;white-space:nowrap;width:99%;">
          <div class="text codecolorer">
           nano authorized_keys
          </div>
         </div> 
         <p>Paste the key in the file, if there is already some information is present, just append the key in the new line and do not change anything else.</p> 
         <p>Once done save the file.</p> 
         <p>In order to validate whether keys are properly configured or not, switch to Jenkins Server and try to login to development server using SSH.</p> 
         <div class="codecolorer-container text twitlight" style="overflow:auto;white-space:nowrap;width:99%;">
          <div class="text codecolorer">
           ssh 
           <a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="562325332418373b3316323320333a39263b3338227b3f26">[email&nbsp;protected]</a>
          </div>
         </div> 
         <p>If it’s not asking any password and log in successfully, you are good to go.</p> 
         <p>Now we have both the Server connecting each other. Let’s write the deployment script and finish this job.</p> 
         <p>Create a new file in your project directory with no extension. I name it <strong>deploy</strong>. </p> 
         <p>Paste following code in it.</p> 
         <div class="codecolorer-container text twitlight" style="overflow:auto;white-space:nowrap;width:99%;">
          <div class="text codecolorer">
           -#!/bin/sh&nbsp; &nbsp; &nbsp; 
           <br> ssh 
           <a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="384d4b5d4a7659555d785c5d4e5d545748555d564c154b5d4a4e5d4a155148">[email&nbsp;protected]</a> &lt;&lt;EOF&nbsp; &nbsp; &nbsp; &nbsp; 
           <br> &nbsp;cd /your-project-path&nbsp; 
           <br> &nbsp;git pull &nbsp; &nbsp; &nbsp; 
           <br> &nbsp;npm install --production &nbsp; &nbsp; &nbsp; 
           <br> &nbsp;pm2 restart all
           <br> &nbsp;exit &nbsp; &nbsp; &nbsp; 
           <br> EOF
          </div>
         </div> 
         <p>Save the file. Each line of the code in this file is what you actually do manually, I don’t think so I need to explain this.</p> 
         <p>Make the file executable using the following command. This is important for Shell script to execute.</p> 
         <div class="codecolorer-container text twitlight" style="overflow:auto;white-space:nowrap;width:99%;">
          <div class="text codecolorer">
           chmod +x deploy
          </div>
         </div> 
         <p>Go to Jenkins project page, from left navigation click on <strong>“Configure”</strong> and scroll down to <strong>“Build”</strong> section.</p> 
         <p>Add <strong>“./deploy”</strong> in the build commands. Here is screenshot for same.</p> 
         <p><img src="https://i1.wp.com/codeforgeek.com/wp-content/uploads/2016/04/screenshot-45.55.135.204-8080-2016-04-18-17-40-45.png?resize=640%2C140&amp;ssl=1" alt="Adding deployment script" class="aligncenter size-full wp-image-2216" srcset="https://i1.wp.com/codeforgeek.com/wp-content/uploads/2016/04/screenshot-45.55.135.204-8080-2016-04-18-17-40-45.png?w=1073&amp;ssl=1 1073w, https://i1.wp.com/codeforgeek.com/wp-content/uploads/2016/04/screenshot-45.55.135.204-8080-2016-04-18-17-40-45.png?resize=300%2C66&amp;ssl=1 300w, https://i1.wp.com/codeforgeek.com/wp-content/uploads/2016/04/screenshot-45.55.135.204-8080-2016-04-18-17-40-45.png?resize=768%2C168&amp;ssl=1 768w, https://i1.wp.com/codeforgeek.com/wp-content/uploads/2016/04/screenshot-45.55.135.204-8080-2016-04-18-17-40-45.png?resize=1024%2C224&amp;ssl=1 1024w, https://i1.wp.com/codeforgeek.com/wp-content/uploads/2016/04/screenshot-45.55.135.204-8080-2016-04-18-17-40-45.png?resize=640%2C140&amp;ssl=1 640w" sizes="(max-width: 640px) 100vw, 640px" data-recalc-dims="1"></p> 
         <p>Click on Save.</p> 
         <p>Now in the project directory, add the deploy file, commit and push it to Github.</p> 
         <p>Check the second build in Jenkins.</p>

         <p>Refresh your project page and you shall see the changes updated in the project.</p> 
         <p>It Works. We have automated the process to do the testing and deployment every time we make some changes in the project.</p> 
         <h2>Tips for integrating Github private repository or organization with Jenkins</h2> 
         <p>Setting up Jenkins for the private repository is similar to what we did above. I personally configured the continuous integration and deployment environment for my organization using same steps I mentioned above except some little extra efforts due to Server configuration ( This is my guess ).</p> 
         <p>When Jenkins was trying to pull the code, it was throwing an error due to HTTPS method used by Github. I stumbled upon various methods and nothing worked for me. I changed the HTTPS origin to SSH origin and updated the repository origin too in Development server.</p> 
         <p>I also configured my account to use the SSH key in order to perform the Git operation without authentication and that you may need in order to let Jenkins do the code update.</p> 
         <p>Here is what I did.</p> 
         <p>Go to this official doc of Github to generate and add the SSH key in Github. You need to generate SSH key in your development Server and add the Public key in Github.</p> 
         <p><a href="https://help.github.com/articles/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent/" target="_blank">Link to official doc.</a></p> 
         <p>Once that is done, change the origin of Github repository from HTTPS to SSH.</p> 
         <p>Then log-in to development Server and switch to project directory and run this command to change the origin to SSH.</p> 
         <div class="codecolorer-container text twitlight" style="overflow:auto;white-space:nowrap;width:99%;">
          <div class="text codecolorer">
           &nbsp;git remote set-url origin &lt;copy it from github&gt;
          </div>
         </div> 
         <p>I have solved the issue using this technique, if there is still a better way and I am sure there is, let me know in the comment or email me.</p>

In this case you can have the a step to promote build to QA or STG. I have promotions of DevDeployed, ReadyForQA, QADeployed, PassedQA, and ReleaseCandidate and they all have dependencies to ensure that each promotion is followed. So once build promoted to ReleaseCandidate that is the build that is then deployed to Production with no rebuild.</p>
