         <p><span class="Apple-style-span" style="font-size: 18px; font-weight: bold; line-height: 28px;">What is Mongoose?</span></p> 
         <p style="text-align: justify;">Mongoose is an “<em>elegant mongodb object modeling for node.js</em>“. If you have used MongoDB before and tried basic database operations, you might have noticed that MongoDB is &nbsp;“schema&nbsp;less”. When you are looking to implement a more structured database and want to leverage the power of MongoDB, Mongoose is one of the ODM (<strong>O</strong>bject <strong>D</strong>ata <strong>M</strong>apping) solutions.</p> 
         <p>To quickly demonstrate, you run an insert command into a collection named users like:</p> 
         <pre class="brush: jscript; title: ; notranslate" title="">

db.users.insert({ name : 'Arvind', gender : 'male'});
</pre> 
         <p>And right after that you can run</p> 
         <pre class="brush: jscript; title: ; notranslate" title="">

db.users.insert({ name : 'Arvind', gender : 'male', password : '!@#$'});
</pre> 
         <p style="text-align: justify;">and MongoDB will never complain about the variation in the number of columns (key value pairs). This is very&nbsp;flexible. But when you want to keep your data more organized and structured, you would need to maintain that in your server code, writing validation, making sure nothing irrelevant is stored in a collection. And this is where Mongoose makes life easy.</p> 
         <p><em>“Mongoose provides a straight-forward, schema-based solution to modeling your application data and includes built-in type casting, validation, query building, business logic hooks and more, out of the box.”</em></p> 
         <h2>Install Node js &amp; MongoDB</h2> 
         <p>To use Mongoose, we need to have Node js installed, you can find info <a title="Install Node" href="http://nodejs.org/" target="_blank">here</a>.</p> 
         <h2>Start Developing</h2> 
         <p>Let us first create a small playground, where we can have fun. Create a new folder named myMongooseApp. And open terminal/prompt here and run</p> 
         <pre class="brush: jscript; title: ; notranslate" title="">npm init
</pre> 
         <p>This will help us in initializing a new node project. Fill it up as required. Next, we will install Mongoose as a dependency to our project. Run</p> 
         <pre class="brush: jscript; title: ; notranslate" title="">npm install mongoose --save-dev</pre> 
         <p>then&nbsp;start the MongoDB service by running</p> 
         <pre class="brush: jscript; title: ; notranslate" title="">mongod
</pre> 
         <p>Next, create a new file named index.js at the root of the and then open it up in your favorite editor. And add the below code.</p> 
         <pre class="brush: jscript; title: ; notranslate" title="">var mongoose = require('mongoose');
mongoose.connect('mongodb://localhost/myTestDB');

var db = mongoose.connection;

db.on('error', function (err) {
console.log('connection error', err);
});
db.once('open', function () {
console.log('connected.');
});
</pre> 
         <p>Here, we require the mongoose package to connect to the DB, and initialize the connection. The name of our Database is myTestDB.</p> 
         <p>Then run</p> 
         <pre class="brush: jscript; title: ; notranslate" title="">node index.js
</pre> 
         <p style="text-align: justify;">and you should see the connected message. You can also use a node package named <a title="Nodemon" href="http://nodemon.io/" target="_blank">nodemon</a> for automatically restarting the node server on changes.<br> Now, our sandbox is ready to play!</p> 
         <h2>Mongoose Schemas</h2> 
         <p style="text-align: justify;">Schemas are like skeletons. The bare bones of how your data collection will look like. If you are dealing with a collection of users, your schema would look something like this.</p> 
         <pre class="brush: jscript; title: ; notranslate" title="">
Name - String
Age - Number
Gender - String
Date of Birth - Date
</pre> 
         <p>And if you are dealing with a collection of products, your schema will look something like this</p> 
         <pre class="brush: jscript; title: ; notranslate" title="">
SKU - String
Name - String
Price - Number
InStock - Boolean
Quantity - Number
</pre> 
         <p style="text-align: justify;">You can see the drift. When our data is guarded with a schema like this, the possibility of storing garbage data reduces drastically.</p> 
         <p style="text-align: justify;">Now we got an understanding of schemas, lets try and build a user schema using Mongoose. Back to index.js and add the below code</p> 
         <pre class="brush: jscript; title: ; notranslate" title="">var Schema = mongoose.Schema;
var userSchema = new Schema({
name : String,
age : Number,
DOB : Date,
isAlive : Boolean
});
</pre> 
         <p>Basic User related fields and their schema types. You can find a list of schema types <a title="Schema Types" href="http://mongoosejs.com/docs/schematypes.html" target="_blank">here</a>.</p> 
         <p>Next, we will create a model from the schema. Add</p> 
         <pre class="brush: jscript; title: ; notranslate" title="">var User = mongoose.model('User', userSchema);
</pre> 
         <p style="text-align: justify;">Thats it, our User model is ready. We will use this as our base schema to insert users into the database. This way we know that every document in a User collection will have the fields listed on the schema. Lets create a new user instance and save it to DB add</p> 
         <pre class="brush: jscript; title: ; notranslate" title="">var arvind = new User({
name : 'Arvind',
age : 99,
DOB : '01/01/1915',
isAlive : true
});

arvind.save(function (err, data) {
if (err) console.log(err);
else console.log('Saved : ', data );
});
</pre> 
         <p>And you should see something like:</p> 
         <pre class="brush: jscript; title: ; notranslate" title="">Saved : { __v: 0,
name: 'Arvind',
age: 99,
DOB: Fri Jan 01 1915 00:00:00 GMT+0530 (IST),
isAlive: true,
_id: 536a4866dba434390d728216 }
</pre> 
         <p>No hassles, no issues. Simple and easy API to interact with Models.</p> 
         <p style="text-align: justify;">Lets say that we want each model to have a method named isYounger. This method will return true if age is less than 50, false if greater. We can do this by querying the DB for the current user, then checking the conditioning and the returning true or false.</p> 
         <p style="text-align: justify;">But what if we want to implement this method to all the models of User schema? This is how we do it in Mongoose</p> 
         <pre class="brush: jscript; title: ; notranslate" title="">var mongoose = require('mongoose');
mongoose.connect('mongodb://localhost/myTestDB');

var db = mongoose.connection;

db.on('error', function (err) {
console.log('connection error', err);
});
db.once('open', function () {
console.log('connected.');
});

var Schema = mongoose.Schema;
var userSchema = new Schema({
name : String,
age : Number,
DOB : Date,
isAlive : Boolean
});

userSchema.methods.isYounger = function () {
return this.model('User').age &lt; 50 ? true : false;
}

var User = mongoose.model('User', userSchema);

var arvind = new User({
name : 'Arvind',
age : 99,
DOB : '01/01/1915',
isAlive : true
});

arvind.save(function (err, data) {
if (err) console.log(err);
else console.log('Saved ', data );
});

console.log('isYounger : ',arvind.isYounger());
</pre> 
         <p style="text-align: justify;">On line 21 we add the method definition. And the result on line 39 will be false. This is a simple handy way of adding methods to your Schema, making it more Object Oriented-ish.</p> 
         <p style="text-align: justify;">In case you have a password field, you can add a method like encryptPassword(), to encrypt password and comparePassword(), to compare the passwords at login to the userSchema itself. You can read more about password Authentication <a title="Password Authentication" href="http://blog.mongodb.org/post/32866457221/password-authentication-with-mongoose-part-1" target="_blank">here</a>.</p> 
         <p style="text-align: justify;">Out of box Mongoose also provides a few options when the schema is created. For example if you take a look at the below schema declaration, we are passing an option</p> 
         <pre class="brush: jscript; title: ; notranslate" title="">
var userSchema = new Schema({
name : String,
age : Number,
DOB : Date,
isAlive : Boolean
}, {strict : false});
</pre> 
         <p>strict:false. Strict option is true by default and it does not allow the ‘non-schema’ key value pairs to be saved. Example:</p> 
         <pre class="brush: jscript; title: ; notranslate" title="">
var arvind = new User({
name : 'Arvind',
age : 99,
DOB : '01/01/1915',
isAlive : true
});
</pre> 
         <p>will be saved, where as</p> 
         <pre class="brush: jscript; title: ; notranslate" title="">
var arvind = new User({
name : 'Arvind',
age : 99,
DOB : '01/01/1915',
isAlive : true,
bucketList : [{...}, {...}, {...} ]
});
</pre> 
         <p style="text-align: justify;">All of the above will be saved minus the bucketList array, because it was not declared as part of schema. So, no client who consumes your services will be able to dump invalid data into your collections.</p> 
         <p style="text-align: justify;">Another cool option is collection. If you don’t want your model name to be the same as collection name, you can pass the name as an option like</p> 
         <pre class="brush: jscript; title: ; notranslate" title="">
var userSchema = new Schema({
name : String,
age : Number,
DOB : Date,
isAlive : Boolean
}, {collection : 'appusers'});
</pre> 
         <p>You can find a list of other options <a title="Options" href="http://mongoosejs.com/docs/guide.html#options" target="_blank">here</a>.</p> 
         <p>With Mongoose, You can also add events to your schemas like pre save or post save, where you can perform validations or process data or run other queries in the respective events. These methods are called as Middlewares.</p> 
         <p>A simple example can be a parallel middleware like</p> 
         <pre class="brush: jscript; title: ; notranslate" title="">
var userSchema = new Schema({
name : String,
age : Number,
DOB : Date,
isAlive : Boolean
})
schema.pre('save', true, function (next, done) {
// calling next kicks off the next middleware in parallel
next();
doAsync(done);
});
</pre> 
         <p>You can read more about middlewares <a title="Middleware" href="http://mongoosejs.com/docs/middleware.html" target="_blank">here</a>. In our next post we will talk more about using Yeoman to <a title="Yeoman, Mongoose and MongoDB" href="https://scalegrid.io/blog/yeoman-mongoose-and-mongodb/">automatically generate scaffolding for your mongodb projects</a>.</p>
