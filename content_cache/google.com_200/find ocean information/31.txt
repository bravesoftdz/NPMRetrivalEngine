function debounce(func, wait, immediate) {
  var timeout;
  return function() {
    var context = this, args = arguments;
    var later = function() {
      timeout = null;
      if (!immediate) func.apply(context, args);
    };
    var callNow = immediate && !timeout;
    clearTimeout(timeout);
    timeout = setTimeout(later, wait);
    if (callNow) func.apply(context, args);
  };
};
</script>

        <h1>Build and Deploy Node.js Web Application to DigitalOcean</h1> 
        <div class="main-style"> 
         <p><br> Node.js is a powerful runtime environment for developing web applications. It’s widely used throughout the software development world to create I/O intensive web applications, such as video streaming or single page sites. Deploying these apps can take time, and this is where Distelli comes in. Distelli automates the entire Build/Deploy process enabling developers to spend their time doing what they do best, creating cool apps. Today, we’ll focus on getting your Distelli account set up and deploying your first Node.js application.</p> 
         <h3 id="prerequisites:3837ab80a627b46aef532ddbefc39933">Prerequisites</h3> 
         <p>For this tutorial you should have a working knowledge of Javascript, Basic Shell Commands, and a basic understanding of Node.js. You will also need to have a GitHub/BitBucket account. Please make sure you have your Source Control account set up before we begin the tutorial. I will be using GitHub for this specific example.</p> 
         <h3 id="set-up:3837ab80a627b46aef532ddbefc39933">Set-Up</h3> 
         <p>For this tutorial we will be using a basic “Hello World” Node.js app, but you can use your own custom application if you like. To start you are going to need to open your terminal and navigate to the folder where you would like to store your files. For myself, I am going to navigate to my WorkSpace (ws) and further more to my Source (src) directory. Then I am going to make a directory called nodeTutorial to store my Node.js app and create the actual app file.</p> 
         <div class="code"> 
          <pre>
$ cd ws/src
$ mkdir nodeTutorial
$ cd nodeTutorial
$ touch app.js
</pre> 
         </div> 
         <p>Now that we have everything all set up we can start by creating a basic Node.js “Hello World” app. Open app.js in the text editor of your choice and paste in the below code:</p> 
         <div class="code"> 
          <pre>
var express = require('express');
var app = express();

app.get('/', function(req, res){
  res.send('Hello World');
});

var server = app.listen(3000, function(){
  console.log('Magic is happening on port 3000')
});
</pre> 
         </div> 
         <p>You will notice that I have included “express” as a Node Module. Make sure you have express installed by going to your terminal, making sure you are still in the directory where you are storing your files, and type <code>npm install express</code>. Once you have done that save app.js and run it in your terminal. As a reminder, you run a Node.js app by using the command <code>node app.js</code>. Open your browser and point it towards ‘localhost:300’. This is what you should see:</p> 
         <p><img src="/docs/assets/images/NodeTutorial/nodeHelloWorld.png" alt="Node.js Example App"></p> 
         <p>See wasn’t that easy? Now that we have a working Node application all we have to do is deploy it to our server. Distelli makes this process incredibly easy and painless. Instead of having to write custom scripts to deploy your application, Distelli will do it all for you. With just a couple clicks your deployment will be live and ready for your customers! Below are the simple steps you need to follow to get your application from being hosted locally to being hosted on your production server.</p> 
         <p>Before we move on, make sure you have set up a repository in GitHub or BitBucket.</p> 
         <p><img src="/docs/assets/images/NodeTutorial/gitHubRepo.png" alt="GitHub Repo"></p> 
         <h2 id="step-1-push-your-code-to-github:3837ab80a627b46aef532ddbefc39933">Step 1. Push your Code to GitHub</h2> 
         <p>For this tutorial I am using GitHub as my repository, but you can use BitBucket as well without any issue. First you will need to create a repo in GitHub and push your code to that repository. Once created, open your terminal and make sure you are in the directory that your app code is in.</p> 
         <p>Initialize the local directory as a Git Repo </p>
         <div class="code"> 
          <pre>
$ git init
</pre> 
         </div> 
         <p></p> 
         <p>Add the files in your new local repo </p>
         <div class="code"> 
          <pre>
$ git add .
</pre> 
         </div> 
         <p></p> 
         <p>Commit the files you have just staged </p>
         <div class="code"> 
          <pre>
$ git commit -m "First Commit. Created app.js"
</pre> 
         </div> 
         <p></p> 
         <p>Navigate to your GitHub or BitBucket Repo and find your remote repository URL. Once you have your remote repository URL use the following commands to finish setting up your repo.</p> 
         <div class="code"> 
          <pre>
$ git remote add origin "Your Remote Repository URL"
# Sets the new URL
$ git remote -v
# Verifies the URL
</pre> 
         </div> 
         <p>Now push the changes to your repo </p>
         <div class="code"> 
          <pre>
$git push origin master
</pre> 
         </div> 
         <p></p> 
         <p>Now we have our GitHub/BitBucket repo all set up. Yet there is still one more thing we have to do before we set up our app on Distelli and get it deployed into production. Go back to your terminal and enter the command <code>npm init</code>. Follow the prompts and afterwards you should now have a package.json file in your directory. Once that file is created, push the update into your GitHub Rep.</p> 
         <div class="code"> 
          <pre>
$ git add .
$ git commit -m "Added package.json"
$ git push origin master
</pre> 
         </div> 
         <h2 id="step-2-add-your-application-to-distelli:3837ab80a627b46aef532ddbefc39933">Step 2. Add your Application to Distelli</h2> 
         <p>Now that we have installed the Distelli Agent on your server, we are ready to deploy our app. Navigate to <a href="https://www.distelli.com/" target="_blank">Distelli.com</a> and make sure you are logged in. Once you are logged in, go the Applications Page and click the ‘New App’ button.</p> 
         <p><img src="/docs/assets/images/NodeTutorial/distelliCreateNewApp.png" alt="Create New Distelli App"></p> 
         <p>Name your app and then select the Repository you would like to connect to. As a reminder, we have been using GitHub as our repository</p> 
         <p><img src="/docs/assets/images/NodeTutorial/distelliCreateNewAppBlank.png" alt="Name Your Distelli App"></p> 
         <p>After we click the button to connect to our Repository, we then select the appropriate repo that contains our code. In this example I store my code in the repo name ‘node-tutorial’. Make sure to select the repo we set up earlier in the tutorial.</p> 
         <p><img src="/docs/assets/images/NodeTutorial/distelliChooseRepo.png" alt="Choose Your Distelli Repo"></p> 
         <p>Then select the appropriate branch for your deployment. I only have a master branch, but you can deploy any branch from your repo!</p> 
         <p><img src="/docs/assets/images/NodeTutorial/distelliChooseBranch.png" alt="Choose Your Distelli Repo's Branch"></p> 
         <p>Next Distelli will ask you set your build steps. For this particular application we will use the PkgInclude section. The Build section is where you would want to include any tests or tasks you would like to run while your application is building. Please make sure your PkgInclude section looks identical to the one below.</p> 
         <p><img src="/docs/assets/images/NodeTutorial/deployNodePkgInclude.png" alt="Node.js Distelli PkgInclude"></p> 
         <p>The final step is select our Build Image. In this case we are going to select “Distelli Javascript (Docker)” for our Node.js Application. Select the “Auto Build” checkbox and the the “Looks good. Start Build!”.</p> 
         <div class="call-out co-warning clearfix"> 
          <div class="flag"> 
           <i class="fa fa-exclamation-circle"></i> 
          </div> 
          <p>The Auto-Build feature builds your application everytime you commit code to your repository</p> 
         </div> 
         <p><img src="/docs/assets/images/NodeTutorial/deployNodeBuildStart.png" alt="Start Distelli Build"></p> 
         <p>Now our application should be automatically building. You can watch its progress by navigating to the builds tab on your Distelli account and clicking on your build.</p> 
         <h2 id="step-3-deploy-steps:3837ab80a627b46aef532ddbefc39933">Step 3. Deploy Steps</h2> 
         <div class="call-out co-info clearfix"> 
          <div class="flag"> 
           <i class="fa fa-info-circle"></i> 
          </div> 
          <p>This tutorial assumes that the server is out of the box and needs to have your application dependencies installed &amp; configured. Once your application dependencies are installed &amp; configured you can remove those commands from your <b>Manifest</b>.</p> 
         </div> 
         <p>Once your build has completed navigate back to your application page and open your application.</p> 
         <p><img src="" alt=""></p> 
         <p>Select the Manifest tab</p> 
         <p><img src="/docs/assets/images/NodeTutorial/distelliManifestTab.png" alt="Manifest Tab for Distelli App"></p> 
         <p>Expand the “Deployment Manifest” section by clicking on the plus sign. This is where we are going to tell Distelli what commands to run during our Deployment.</p> 
         <h3 id="preinstall:3837ab80a627b46aef532ddbefc39933">PreInstall</h3> 
         <p>Scroll down to your PreInstall section and enter in the following</p> 
         <p><img src="/docs/assets/images/NodeTutorial/deployNodeUbuntuPreInstall.png" alt="Distelli Node.js PreInstall Commands"></p> 
         <p>Copy/Paste </p>
         <div class="code"> 
          <pre>
echo "Starting PreInstall"
sudo apt-get update
sudo apt-get install nodejs -y
sudo apt-get install npm -y
</pre> 
         </div> 
         <p></p> 
         <h3 id="postinstall:3837ab80a627b46aef532ddbefc39933">PostInstall</h3> 
         <p>Scroll down to your PostInstall section and enter in the following</p> 
         <p><img src="/docs/assets/images/NodeTutorial/deployNodeUbuntuPostInstall.png" alt="Distelli Node.js PostInstall Commands"></p> 
         <p>Copy/Paste </p>
         <div class="code"> 
          <pre>
echo "Starting PostInstall"
npm install
</pre> 
         </div> 
         <p></p> 
         <h3 id="exec:3837ab80a627b46aef532ddbefc39933">Exec</h3> 
         <p>Scroll down to your Exec section and enter in the following</p> 
         <p><img src="/docs/assets/images/NodeTutorial/deployNodeUbuntuExec.png" alt="Distelli Node.js Exec Commands"></p> 
         <p>Copy/Paste </p>
         <div class="code"> 
          <pre>
echo "Executing"
/usr/bin/nodejs app.js
</pre> 
         </div> 
         <p></p> 
         <h3 id="re-build-application:3837ab80a627b46aef532ddbefc39933">Re-Build Application</h3> 
         <p>Since we have changed our deployment steps we need to rebuild our application. First save your changes by clicking the “Publish Manifest” button and then rebuild your application. To trigger a new build just click the wrench icon in the right hand corner! That button triggers a build of your latest code, build steps, and deployment steps.</p> 
         <p><img src="/docs/assets/images/NodeTutorial/deployTriggerNewBuild.png" alt="Trigger new build of Node.js Application"></p> 
         <h2 id="step-4-set-up-your-server:3837ab80a627b46aef532ddbefc39933">Step 4. Set Up Your Server</h2> 
         <p>The next step in deploying our app is to make sure you have a server to run your application on. You can use any type of server - public cloud, private cloud or physical. Distelli will let you deploy your code to Vagrant VMs or servers in a VPC. You can even deploy to the server under your desk. For this tutorial I am going to be deploying to a server hosted by Digital Ocean. If you already have your server set up you can skip to the next step.</p> 
         <p>To get your server set up on Digital Ocean, you will need to navigate to <a href="https://www.digitalocean.com/" target="_blank">digitalocean.com</a> to get set up. If you already have an account please Log In, if not Sign Up for an account now. Once you have created an account, or logged in, you will need to “Create Droplet”.</p> 
         <p>Please select the settings for your droplet that best fit your needs. I am using an Ubuntu image set up in New York. You can choose any options that you please as Distelli is able to deploy to any server, anywhere!</p> 
         <p><img src="/docs/assets/images/NodeTutorial/digitalOceanSetUp.png" alt="Set Up Digital Ocean Server"></p> 
         <p>Select your <b>Image</b>, <b>Size</b>, and <b>Region</b>. In the <b>Additional Options</b> section check the box next to <b>User Data</b></p> 
         <p><img src="/docs/assets/images/NodeTutorial/digitalOceanEnableUserData.png" alt="Digital Ocean enable user data"></p> 
         <p>This will open a text area for you to enter in your <b>User Data</b>. To automate the install of the Distelli Agent, use the script below:</p> 
         <div class="code"> 
          <pre>
#!/bin/bash

export DISTELLI_ACCESS_TOKEN='<b>YOUR DISTELLI ACCESS TOKEN</b>'
export DISTELLI_SECRET_KEY='<b>YOUR DISTELLI SECRET KEY</b>'

echo "DistelliAccessToken: $DISTELLI_ACCESS_TOKEN" &gt; /etc/distelli.yml
echo "DistelliSecretKey: $DISTELLI_SECRET_KEY" &gt;&gt; /etc/distelli.yml
echo "Environments:" &gt;&gt; /etc/distelli.yml
echo "  - <b>YOUR DISTELLI ENVIRONMENT</b>" &gt;&gt; /etc/distelli.yml
apt-get -y install wget
wget -qO- https://www.distelli.com/download/client | sh
/usr/local/bin/distelli agent install -readyml /etc/distelli.yml
/usr/local/bin/distelli login -conf /etc/distelli.yml
</pre> 
         </div> 
         <p>You will need to insert your own Distelli <b>Access Token</b> and <b>Secret Key</b>. These fields are tied to a Distelli account. If you are using teams, ensure you are <a href="/docs/user-guides/using-a-team/">Using a Team</a> account when retrieving these values. </p>
         <h4>Retrieving Credentials</h4>
         <p></p> 
         <ol> 
          <li>Click the <b>gear</b> icon from the top right.</li> 
          <li>Click the <b>Agent</b> link on the left.</li> 
          <li>Under <i>Agent Credentials</i> click the <b>Show</b> button.</li> 
         </ol> 
         <p>You will see your <b>Access Token</b> and <b>Secret Key</b> here. Please insert them into your <b>User Data</b> script, and insert the Distelli environment you want to add your server to.</p> 
         <h2 id="step-5-deploy-application:3837ab80a627b46aef532ddbefc39933">Step 5. Deploy Application</h2> 
         <p>Now that we have successfully built our application we are ready to deploy to our server. On the builds page select the “New Deployment” button in the right hand corner.</p> 
         <p><img src="/docs/assets/images/NodeTutorial/deployNewDeployment.png" alt="Node.js Distelli Build"></p> 
         <h3 id="deploy-your-node-js-application:3837ab80a627b46aef532ddbefc39933">Deploy your Node.js Application</h3> 
         <p>Once you click the button, you should be directed towards Distelli’s deployment page. The first step is to select the “Deploy a Release” option.</p> 
         <p><img src="/docs/assets/images/NodeTutorial/distelliDeployRelease.png" alt="Deploy Distelli Release"></p> 
         <p>Distelli will now prompt you to select the application you want to deploy. Please select the application we created earlier in the tutorial.</p> 
         <p><img src="/docs/assets/images/NodeTutorial/deployNodeUbuntuSelectApp.png" alt="Select Distelli Application for Deployment"></p> 
         <p>Then you will be asked to select the release you would like to deploy. For now there should only be one release for deployment.</p> 
         <p><img src="/docs/assets/images/NodeTutorial/deployNodeUbuntuSelectRelease.png" alt="Choose Your Distelli Release"></p> 
         <p>The last step in our deployment is to select the environment you wish to deploy in. First you will need to create an environment by enter a name and selecting the “Add Environment” button.</p> 
         <p><img src="/docs/assets/images/NodeTutorial/deployCreateEnvironment.png" alt="Add Distelli Deployment Environment"></p> 
         <p>Next select the Environment you just created and click “All Done”.</p> 
         <p><img src="/docs/assets/images/NodeTutorial/deploySelectEnvironment.png" alt="Select Distelli Deployment Environment"></p> 
         <p>Now Distelli will ask you to add your server. Click the “Add Servers” button to get started. Select the server you configured earlier in the tutorial and add it to your account. Once you have added your server, you can close the “Add Servers” panel and continue with your deployment. You will see a final option to set your delay between deployments on your servers and a “Deploy!” button.</p> 
         <p><img src="/docs/assets/images/NodeTutorial/deployNodeUbuntuStartDeployment.png" alt="Start Your Distelli Deployment"></p> 
         <p>Click the “Deploy!” button to begin your deployment. You will redirected to a page where you can view the progress of your deployment. Simply clicking the “log” button on the left side that enables you to view realtime streaming logs for the deployment.</p> 
         <p><img src="/docs/assets/images/NodeTutorial/deployNodeUbuntuSuccess.png" alt="Distelli Deployment Successful"></p> 
         <h3 id="view-your-application:3837ab80a627b46aef532ddbefc39933">View Your Application</h3> 
         <p>And that’s it! You should now be able to point your browser to “http://&lt;-You Server’s IP Address-&gt;:3000” and see your Node.js Application!</p> 
         <p><img src="/docs/assets/images/NodeTutorial/deployNodeUbuntuAppDeployed.png" alt="Node.js Successful Deployment"></p> 
         <h2 id="step-6-enable-auto-deploy-change-application:3837ab80a627b46aef532ddbefc39933">Step 6. Enable Auto deploy &amp; Change Application</h2> 
         <p>If you would like your application to be automatically deployed after each sucessful Build, you can do so by enabling the Auto-Deploy feature in your Application’s settings. Start by navigating to your applications page in Distelli and you should see an App Pipeline section on the right hand side of the screen</p> 
         <p><img src="/docs/assets/images/NodeTutorial/deployNodeUbuntuPipeline.png" alt="Distelli App Pipeline"></p> 
         <p>Click the “Add Step” button and select your environment</p> 
         <p><img src="/docs/assets/images/NodeTutorial/deployNodeUbuntuPipelineEnv.png" alt="Distelli Pipeline Add Step"></p> 
         <p>Press Add and then check the Auto Deploy checkbox with the condition of “If the Build succeeds”</p> 
         <p><img src="/docs/assets/images/NodeTutorial/deployNodeUbuntuAutoDeploy.png" alt="Distelli Pipeline Auto Deploy"></p> 
         <h3 id="change-application:3837ab80a627b46aef532ddbefc39933">Change Application</h3> 
         <p>Once you have set up Automatic Deployment after a successful build of your application, please make a change to your Node.js application. I will be changing what my app outputs onto the page. Once you have made your changes, push your code into your repository and watch Distelli automatically start Building your application and then Deploying your application. Once the Deployment has completed your can see the updated application by visiting “http://&lt;-You Server’s IP Address-&gt;:3000”</p> 
         <p><img src="/docs/assets/images/NodeTutorial/deployNodeUpdate.png" alt="Updated Node.js Application"></p> 
         <p>And that’s it! You have now set up a Node.js Application with Coninuous Integraion and Continuous Deployment! Thanks for following along and happy coding! Thank you for following along, and I hope that you have learned how Distelli can simplify the process behind deploying your Node.js applications to your servers. If you liked this tutorial, like our <a href="https://www.digitalocean.com/community/projects/distelli" target="_blank">project on DigitalOcean</a>.</p>
