      <h3> <a href="/yaml-vs-json">YAML vs JSON</a> </h3> 
      <p> when to prefer one over the other </p> 
      <h4> <a href="http://json.org/">JSON</a> </h4> 
      <p> a subset of the JavaScript object notation syntax </p> 
      <ul> 
       <li> data stored in name/value pairs </li> 
       <li> records separated by commas </li> 
       <li> field names &amp; strings are wrapped by double quotes </li> 
      </ul> 
      <h4> <a href="http://www.yaml.org/spec/1.1/">YAML</a> </h4> 
      <p> stands for YAML ain't markup language and is a superset of JSON - <a href="/convert-yaml-to-json">Convert YAML to JSON</a> </p> 
      <ul> 
       <li> .yml files begin with '---', marking the start of the document </li> 
       <li> key value pairs are separated by colon </li> 
       <li> lists begin with a hyphen </li> 
      </ul>
