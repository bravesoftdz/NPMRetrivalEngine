           <h3 name="bf33" id="bf33" class="graf graf--h3 graf-after--figure">What is&nbsp;npm?</h3>
           <p name="96ae" id="96ae" class="graf graf--p graf-after--h3">npm makes it easy for JavaScript developers to share and reuse code, and it makes it easy to update the code that you’re sharing. The <a href="http://dailyjs.com/2015/06/26/npm-3/" data-href="http://dailyjs.com/2015/06/26/npm-3/" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">beta of npm v3</a> has been released recently.</p>
           <h3 name="aa4a" id="aa4a" class="graf graf--h3 graf-after--p">Pre-reqs</h3>
           <ul class="postList">
            <li name="c625" id="c625" class="graf graf--li graf-after--h3">Sign up for a <a href="https://github.com/" data-href="https://github.com/" class="markup--anchor markup--li-anchor" rel="nofollow noopener" target="_blank">GitHub</a> account</li>
            <li name="df57" id="df57" class="graf graf--li graf-after--li">Install <a href="https://nodejs.org/" data-href="https://nodejs.org/" class="markup--anchor markup--li-anchor" rel="nofollow noopener" target="_blank">Node.js</a> (npm comes with Node, but it’s often an old version)</li>
            <li name="f5d7" id="f5d7" class="graf graf--li graf-after--li">Update to the latest npm version (Node’s release cycle is far slower than npm’s, which is weekly)</li>
           </ul>
           <pre name="d7e2" id="d7e2" class="graf graf--pre graf-after--li">npm install npm@latest -g</pre>
           <pre name="aa6d" id="aa6d" class="graf graf--pre graf-after--pre"><em class="markup--em markup--pre-em">If you get an EACCES error, don’t worry!            <h3 name="d75e" id="d75e" class="graf graf--h3 graf-after--p">Why add a&nbsp;license?</h3>
           <p name="18e2" id="18e2" class="graf graf--p graf-after--h3">It’s your right not to include a license with your code or project, but the absence of a license means that the default copyright laws apply. This means that you retain all rights to your source code and that nobody else may reproduce, distribute, or create derivative works from your work. This might not be what you intend.</p>

           <h3 name="8539" id="8539" class="graf graf--h3 graf-after--p">Tell npm who you&nbsp;are</h3>

           <pre name="edb0" id="edb0" class="graf graf--pre graf-after--pre">npm set init.author.email “<a href="mailto:magic.mike@gmail.com" data-href="mailto:magic.mike@gmail.com" class="markup--anchor markup--pre-anchor" rel="nofollow" target="_blank">magic.mike@gmail.com</a>”</pre>

           <p name="0c4f" id="0c4f" class="graf graf--p graf-after--pre">Your credentials will be now saved to a <strong class="markup--strong markup--p-strong">~/.npmrc</strong> file and used as defaults whenever you initialize a npm package, so you don’t have to enter them each time.</p>
           <p name="b8ca" id="b8ca" class="graf graf--p graf-after--p">To authenticate on your machine, log in with your npm <strong class="markup--strong markup--p-strong">username</strong>, <strong class="markup--strong markup--p-strong">password</strong>, and <strong class="markup--strong markup--p-strong">email</strong>.</p>

           <h3 name="1289" id="1289" class="graf graf--h3 graf-after--pre">Initialize a npm&nbsp;package</h3>
           <p name="8546" id="8546" class="graf graf--p graf-after--h3">When you create a new module, you want to describe your package with the <strong class="markup--strong markup--p-strong">package.json</strong> file.</p>
           <pre name="be90" id="be90" class="graf graf--pre graf-after--p">npm init</pre>
           <p name="676b" id="676b" class="graf graf--p graf-after--pre">If you want to <a href="https://docs.npmjs.com/getting-started/scoped-packages#initializing-a-scoped-package" data-href="https://docs.npmjs.com/getting-started/scoped-packages#initializing-a-scoped-package" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">namespace your modules</a>, do this instead</p>
           <pre name="90f5" id="90f5" class="graf graf--pre graf-after--p">npm init --scope=username</pre>
           <p name="c177" id="c177" class="graf graf--p graf-after--pre">Answer all the questions you’re prompted.</p>
           <p name="8b82" id="8b82" class="graf graf--p graf-after--p">For the <strong class="markup--strong markup--p-strong">version</strong>, let’s start with</p>

           <p name="e905" id="e905" class="graf graf--p graf-after--pre">When you make changes to your library in the future, remember to update the version according to <a href="http://semver.org/" data-href="http://semver.org/" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">semantic versioning</a> (<strong class="markup--strong markup--p-strong">SemVer</strong>).</p>

           <p name="5010" id="5010" class="graf graf--p graf-after--figure">SemVer isn’t used by all libraries. Some maintainers <a href="https://gist.github.com/jashkenas/cbd2b088e20279ae2c8e" data-href="https://gist.github.com/jashkenas/cbd2b088e20279ae2c8e" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">strongly believe that it makes no sense</a>. They feel that any change to a library could potentially break backwards compatibility. In my experience, libraries that don’t follow SemVer cause more problems for the consumers, so I encourage you to follow it.</p>
           <p name="127e" id="127e" class="graf graf--p graf-after--p">Answer the <strong class="markup--strong markup--p-strong">test command</strong> question with this</p>
           <pre name="a2a7" id="a2a7" class="graf graf--pre graf-after--p">mocha --reporter spec</pre>
           <p name="b605" id="b605" class="graf graf--p graf-after--pre">You should now see a <a href="http://browsenpm.org/package.json" data-href="http://browsenpm.org/package.json" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank"><strong class="markup--strong markup--p-strong">package.json</strong></a> file in your project folder.</p>
           <h3 name="9f9f" id="9f9f" class="graf graf--h3 graf-after--p">Specify a minimum Node&nbsp;version</h3>
           <p name="7c9f" id="7c9f" class="graf graf--p graf-after--h3">By adding the following (optional) to your <strong class="markup--strong markup--p-strong">package.json</strong>, you can specify the minimum Node version that your package requires to ensure your library will work for your users.</p>
           <pre name="4587" id="4587" class="graf graf--pre graf-after--p">"engines": {<br>  "node": "&gt;=4.2.4"<br>},</pre>
           <h3 name="56f1" id="56f1" class="graf graf--h3 graf-after--pre">Create a node&nbsp;module</h3>
           <p name="f6a3" id="f6a3" class="graf graf--p graf-after--h3">A Node/npm module is just a regular JavaScript file, but it must follow the <a href="http://wiki.commonjs.org/wiki/Modules/1.1" data-href="http://wiki.commonjs.org/wiki/Modules/1.1" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">CommonJS module spec</a>. <a href="https://github.com/umdjs/umd" data-href="https://github.com/umdjs/umd" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">UMD</a>, which is CommonJS compatible, also works.</p>
           <p name="132c" id="132c" class="graf graf--p graf-after--p">Now, let’s create a very tiny module and name the file <strong class="markup--strong markup--p-strong">index.js</strong>.</p>
           <pre name="af83" id="af83" class="graf graf--pre graf-after--p">'use strict';<br><br><em class="markup--em markup--pre-em">/**<br> * Adds commas to a number<br> * </em><strong class="markup--strong markup--pre-strong"><em class="markup--em markup--pre-em">@param </em></strong><em class="markup--em markup--pre-em">{number} number<br> * </em><strong class="markup--strong markup--pre-strong"><em class="markup--em markup--pre-em">@param </em></strong><em class="markup--em markup--pre-em">{string} locale<br> * </em><strong class="markup--strong markup--pre-strong"><em class="markup--em markup--pre-em">@return </em></strong><em class="markup--em markup--pre-em">{string}<br> */<br></em>module.exports = <strong class="markup--strong markup--pre-strong">function</strong>(number, locale) {<br>    <strong class="markup--strong markup--pre-strong">return </strong>number.toLocaleString(locale);<br>};</pre>
           <p name="26e2" id="26e2" class="graf graf--p graf-after--pre"><strong class="markup--strong markup--p-strong"><em class="markup--em markup--p-em">Confession</em></strong>: I know this module is pretty pointless since browser support is already there via the native <em class="markup--em markup--p-em">toLocaleString</em> method. However, I wanted to get this working in Node by passing in different locales, and I just couldn’t figure it out. Hints are welcome!</p>
           <p name="877b" id="877b" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong"><em class="markup--em markup--p-em">Side note</em></strong>: Sindre Sorhus is the author of over <a href="https://www.npmjs.com/~sindresorhus" data-href="https://www.npmjs.com/~sindresorhus" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">600</a> modules on npm.            <h3 name="6793" id="6793" class="graf graf--h3 graf-after--pre">Publish to&nbsp;npm</h3>
           <pre name="00b0" id="00b0" class="graf graf--pre graf-after--h3">npm publish</pre>
           <p name="b723" id="b723" class="graf graf--p graf-after--pre">If you run into errors with the above command, it could mean that the package name is already taken. In that case, you might want to either rename or scope your package.</p>
           <p name="3770" id="3770" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Scoped packages are private by default.</strong> To publish private modules, you need to be a paid <a href="https://www.npmjs.com/private-modules" data-href="https://www.npmjs.com/private-modules" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">private modules</a> user.</p>
           <p name="270e" id="270e" class="graf graf--p graf-after--p">However, <strong class="markup--strong markup--p-strong">public</strong> scoped modules are free and don’t require a paid subscription. To publish a public scoped module, <strong class="markup--strong markup--p-strong">set the access option when publishing it. This option will remain set for all subsequent publishes.</strong></p>
           <pre name="c715" id="c715" class="graf graf--pre graf-after--p">npm publish --access=public</pre>
           <p name="5d6a" id="5d6a" class="graf graf--p graf-after--pre"><strong class="markup--strong markup--p-strong"><em class="markup--em markup--p-em">Pro tip!</em></strong> When creating a scoped module, it’s a good idea to check your <strong class="markup--strong markup--p-strong">package.json</strong> before publishing just to make sure that the name of your package is “@username/project-name”?—?otherwise it won’t be scoped and if it’s meant to be private, it will end up being public by accident.</p>
           <p name="90c2" id="90c2" class="graf graf--p graf-after--p">Log in to your npm account and your module should show up there. <strong class="markup--strong markup--p-strong">Congrats! You’ve just published your first Node module.</strong></p>

           <h3 name="a111" id="a111" class="graf graf--h3 graf-after--figure">Add code coverage statistics</h3>
           <p name="8b94" id="8b94" class="graf graf--p graf-after--h3"><a href="http://gotwarlost.github.io/istanbul/" data-href="http://gotwarlost.github.io/istanbul/" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Istanbul</a> is a JavaScript code coverage library that will check statement, branch, and function coverage statistics by inserting lines of code into your JavaScript as your tests run and then report back how many of its inserted lines were called.</p>
           <p name="1ec2" id="1ec2" class="graf graf--p graf-after--p"><a href="https://coveralls.io/" data-href="https://coveralls.io/" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Coveralls</a> is a test coverage reporter that you can integrate with Travis. It formats the coverage data in a nice way. You can get pretty much everything from the reports from Istanbul, but only locally and less pretty.</p>
           <pre name="b024" id="b024" class="graf graf--pre graf-after--p">npm i coveralls -D<br>npm i istanbul -D</pre>
           <p name="7ccc" id="7ccc" class="graf graf--p graf-after--pre">Log in to Coveralls with your GitHub account, click the “<strong class="markup--strong markup--p-strong">ADD REPO</strong>” button, and <strong class="markup--strong markup--p-strong">toggle the switch</strong> to enable the repo for which you want code coverage statistics. To use Coveralls, your code must be hosted on GitHub or <a href="https://bitbucket.org" data-href="https://bitbucket.org" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Bitbucket</a>.</p>
           <p name="aaed" id="aaed" class="graf graf--p graf-after--p">Configure your <strong class="markup--strong markup--p-strong">package.json</strong> file for Istanbul and Coveralls by adding a new script named “<strong class="markup--strong markup--p-strong">cover</strong>”:</p>
           <pre name="4702" id="4702" class="graf graf--pre graf-after--p">"scripts": {<br>  "test": "node_modules/.bin/mocha --reporter spec",<br>  <strong class="markup--strong markup--pre-strong">"cover": "node_modules/istanbul/lib/cli.js cover node_modules/mocha/bin/_mocha -- -R spec test/*"</strong><br>},</pre>
           <p name="9308" id="9308" class="graf graf--p graf-after--pre">And update your&nbsp;<strong class="markup--strong markup--p-strong">.travis.yml </strong>file to this</p>
           <pre name="48d8" id="48d8" class="graf graf--pre graf-after--p">language: node_js</pre>
           <pre name="385f" id="385f" class="graf graf--pre graf-after--pre">node_js:<br>  - stable</pre>
           <pre name="59ac" id="59ac" class="graf graf--pre graf-after--pre">install:<br>  - npm install<br><br>script:<br>  - <strong class="markup--strong markup--pre-strong">npm run cover</strong><br><br><strong class="markup--strong markup--pre-strong"><em class="markup--em markup--pre-em"># Send coverage data to Coveralls<br></em>after_script: "cat coverage/lcov.info | node_modules/coveralls/bin/coveralls.js"</strong></pre>
           <p name="74da" id="74da" class="graf graf--p graf-after--pre">Now you can run your tests and code coverage stats from your command line with</p>
           <pre name="51aa" id="51aa" class="graf graf--pre graf-after--p">npm run cover</pre>
           <p name="a76f" id="a76f" class="graf graf--p graf-after--pre">Now, <strong class="markup--strong markup--p-strong">commit</strong> and <strong class="markup--strong markup--p-strong">push</strong> to GitHub.</p>
           <p name="1271" id="1271" class="graf graf--p graf-after--p">Travis will now invoke Istanbul, and Istanbul will run an lcovonly code coverage report, save that info to coverage/lcov.info, and pipe that data to the Coveralls library.</p>
           <p name="70dd" id="70dd" class="graf graf--p graf-after--p">Log in to Coveralls to check and see if everything executed smoothly.</p>

           <h3 name="0693" id="0693" class="graf graf--h3 graf-after--p"><strong class="markup--strong markup--h3-strong">Release a new&nbsp;version</strong></h3>
           <p name="605d" id="605d" class="graf graf--p graf-after--h3">Now that we have our sweet badges on GitHub, let’s get them on npm as well.</p>
           <p name="8ff5" id="8ff5" class="graf graf--p graf-after--p">The only way to update npm is to release a new version, even if you’re just fixing a typo. npm will not allow you to re-publish to the same number.</p>
           <p name="03b2" id="03b2" class="graf graf--p graf-after--p">So let’s create a new release by bumping the version in your <strong class="markup--strong markup--p-strong">package.json</strong>. To do that, you can use edit the file manually or use</p>
           <pre name="88c2" id="88c2" class="graf graf--pre graf-after--p">npm version &lt;update_type&gt; -m "&lt;message&gt;"</pre>
           <p name="55fc" id="55fc" class="graf graf--p graf-after--pre">where <strong class="markup--strong markup--p-strong">update_type</strong> is one of the <a href="http://semver.org/" data-href="http://semver.org/" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">semantic versioning</a> release types:<br><strong class="markup--strong markup--p-strong">patch</strong>, <strong class="markup--strong markup--p-strong">minor</strong>, or <strong class="markup--strong markup--p-strong">major</strong>.</p>
           <p name="489d" id="489d" class="graf graf--p graf-after--p">So let’s do</p>
           <pre name="0a8c" id="0a8c" class="graf graf--pre graf-after--p">npm version patch -m "Version %s - add sweet badges"</pre>
           <p name="faa0" id="faa0" class="graf graf--p graf-after--pre"><strong class="markup--strong markup--p-strong">%s</strong> = the new version number.</p>
           <p name="ddf1" id="ddf1" class="graf graf--p graf-after--p">This command will bump the version number in <strong class="markup--strong markup--p-strong">package.json</strong>, add a new commit, and tag it with this release number.</p>
           <p name="9752" id="9752" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong"><em class="markup--em markup--p-em">Note</em></strong>: Your Git working directory has to be <strong class="markup--strong markup--p-strong">clean</strong> before you can run <strong class="markup--strong markup--p-strong">npm version</strong>.</p>
           <p name="e989" id="e989" class="graf graf--p graf-after--p">After bumping the version number</p>
           <pre name="d028" id="d028" class="graf graf--pre graf-after--p">git push &amp;&amp; git push --tags (or git push origin master --tags)</pre>
           <pre name="91d8" id="91d8" class="graf graf--pre graf-after--pre">npm publish</pre>
           <p name="bf24" id="bf24" class="graf graf--p graf-after--pre">Now if you go to your published module on npm, you should see your new release with the two sweet badges!</p>
           <h3 name="52d2" id="52d2" class="graf graf--h3 graf-after--p">Test out your&nbsp;module</h3>
           <p name="9848" id="9848" class="graf graf--p graf-after--h3">Create a new directory and a <strong class="markup--strong markup--p-strong">package.json</strong> file.</p>
           <pre name="5927" id="5927" class="graf graf--pre graf-after--p">{<br>  "dependencies": {<br>    "project-name": "*"<br>  }<br>}</pre>
           <p name="8602" id="8602" class="graf graf--p graf-after--pre">Then from the command line</p>
           <pre name="fae7" id="fae7" class="graf graf--pre graf-after--p">npm i</pre>
           <p name="239f" id="239f" class="graf graf--p graf-after--pre">Create an <strong class="markup--strong markup--p-strong">index.js</strong> file with the following content</p>
           <pre name="dca1" id="dca1" class="graf graf--pre graf-after--p">var<strong class="markup--strong markup--pre-strong"> </strong>numFormmater = require('project-name');<br><br>var<strong class="markup--strong markup--pre-strong"> </strong>formattedNum = numFormmater(234324234);<br><br>console.log(formattedNum);</pre>
           <p name="3065" id="3065" class="graf graf--p graf-after--pre"><strong class="markup--strong markup--p-strong"><em class="markup--em markup--p-em">Reminder</em></strong>: If you published a scoped module, update the project name to</p>
           <pre name="0f2d" id="0f2d" class="graf graf--pre graf-after--p">@username/project-name</pre>
           <p name="1c1b" id="1c1b" class="graf graf--p graf-after--pre">Then from the command line</p>
           <pre name="9994" id="9994" class="graf graf--pre graf-after--p">node index</pre>
