          <h1 class="entry-title">Google Chrome: How to Use the Web Speech API</h1> 
          <div class="entry-meta"> 
           <span class="post-author">Posted by <a class="post-author__link" href="https://stiltsoft.com/blog/author/alexkuznetsov/">Alexander Kuznetsov</a></span>
           <span class="post__date"><i class="material-icons">event</i><a class="post-date__link" href="https://stiltsoft.com/blog/2013/05/google-chrome-how-to-use-the-web-speech-api/"><time class="post-date__time" datetime="2013-05-02T07:45:11+00:00">May 2, 2013</time></a></span> 
          </div>
          <!-- .entry-meta --> 
         </header>
         <!-- .entry-header --> 
         <figure class="post-thumbnail"> 
         </figure>
         <!-- .post-thumbnail --> 
         <div class="entry-content"> 
          <div style="padding: 10px; background-color: #ffe0b2; line-height: 1; border-radius: 5px;">
           <em>Check our <a target="_blank" href="http://stiltsoft.com/plugins.html?from=blog-adv">add-ons</a> for Atlassian Products: Confluence, JIRA, Bamboo.</em>
          </div> 
          <p>This February Google released Chrome version 25. One of the newest and most interesting features introduced in this version was&nbsp;<a target="_blank" href="https://dvcs.w3.org/hg/speech-api/raw-file/tip/speechapi.html" rel="nofollow">Web Speech API</a>&nbsp;support.&nbsp;Web Speech API is the JavaScript library that allows speech recognition and speech-to-text conversion.&nbsp;Conversely,&nbsp;Web Speech API enables you to transform text to speech.</p> 
          <p>This helped us develop the <a target="_blank" href="https://marketplace.atlassian.com/plugins/com.stiltsoft.confluence.speech.handsfree">Handsfree plugin for Confluence</a> that <a target="_blank" href="https://stiltsoft.com/blog/2013/04/handsfree-add-on-for-confluence-create-confluence-content-by-voice/">creates pages</a> and page comments with voice in Chrome 25+.</p> 
          <p>Speech recognition supports several popular languages and is quite effective. Currently, developers have two options of implementing speech recognition on web-pages.</p> 
          <p><span id="more-349"></span></p> 
          <h5>The First Method</h5> 
          <p>The easiest way to use this technology is to use the already implemented functionality for the html tag&nbsp;<strong>&lt;input&gt;</strong>. You only need to add the attribute&nbsp;<strong>x-webkit-speech</strong>:</p> 
          <pre class="brush: xml; title: ; notranslate" title="">
&lt;input x-webkit-speech&gt;
</pre> 
          <p>And you get a text box that allows you to dictate a text.</p> 
          <p><a target="_blank" href="https://stiltsoft.com/blog/wp-content/uploads/2013/05/web-speech-example-x-webkit-speech-attribute.png"><img class="alignnone size-full wp-image-350 colorbox-349" src="//stiltsoft.com/blog/wp-content/uploads/2013/05/web-speech-example-x-webkit-speech-attribute.png" alt="" width="600" height="278" srcset="https://stiltsoft.com/blog/wp-content/uploads/2013/05/web-speech-example-x-webkit-speech-attribute.png 600w, https://stiltsoft.com/blog/wp-content/uploads/2013/05/web-speech-example-x-webkit-speech-attribute-300x139.png 300w" sizes="(max-width: 600px) 100vw, 600px"></a></p> 
          <p>By default the recognition language will be the same as that set in your browser. But you can change it in two ways:</p> 
          <p>1) By adding the attribute<strong>&nbsp;lang=”en”</strong>&nbsp;where the attribute value defines the language to be recognized:</p> 
          <pre class="brush: xml; title: ; notranslate" title="">
&lt;input lang=”en” x-webkit-speech&gt;
</pre> 
          <p>2) By using the&nbsp;<strong>&lt;meta&gt;</strong>&nbsp;tag on your html page:</p> 
          <pre class="brush: xml; title: ; notranslate" title="">
&lt;meta http-equiv = 'Content-Language' content = 'en'/&gt;
</pre> 
          <p>The&nbsp;<strong>x-webkit-speech&nbsp;</strong>attribute’s advantages are:</p> 
          <ul> 
           <li>It’s easy to implement</li> 
           <li>The browser doesn’t request the user to allow to use the microphone.</li> 
          </ul> 
          <p>However, there’re significant disadvantages:</p> 
          <ul> 
           <li>Speech recognition is stopped after a pause</li> 
           <li>When you resume speech recognition in the same box, the old value is substituted by a new one, so you can’t add data.</li> 
           <li>It’s supported only by the&nbsp;<strong>&lt;input&gt;</strong>&nbsp;tag.</li> 
           <li>The interim results are not displayed which contributes to poor feedback as the user sees the recognition result only after they stop talking and the recognition process is finished.</li> 
          </ul> 
          <p>You can see how it works&nbsp;<a target="_blank" href="https://stiltsoft.com/blog/wp-content/demo/web-speech-api/input/input-x-webkit-speech.html" rel="nofollow noopener">here</a>.</p> 
          <h5 id="GoogleChrome:HowtoUsetheWebSpeechAPI-TheSecondMethod-usingWebSpeechAPIonJavaScript">The Second Method – using Web Speech API on JavaScript</h5> 
          <p>This method is based on the interaction with Web Speech API with the help of JavaScript (<a target="_blank" href="https://stiltsoft.com/blog/wp-content/demo/web-speech-api/textarea/wsapi-textarea.html" rel="noopener">demo</a>). To start&nbsp;using API, you need to create a new object that will be employed for recognition:</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
var recognition = new webkitSpeechRecognition();
</pre> 
          <p>Further you can set the following speech recognition parameters:</p> 
          <p>1) Set the continious recognizing that enable the user to make long pauses and dictate large texts. By default this property is set to&nbsp;<strong>false</strong>&nbsp;(i.e. a speech pause will stop the recognition process).</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
recognition.continuous = true;
</pre> 
          <p>2) Enable interim results fetching. Thus you have access to interim recognition results and can display them in the text box immediately after receiving them. The user will see a constantly refreshing text, otherwise the recognized text will be available only after a pause. The default value is&nbsp;<strong>false</strong>.</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
recognition.interimResults = true;
</pre> 
          <p>3) Set the recognition language. By default it corresponds to the browser language.</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
recognition.lang = “en”;
</pre> 
          <p>To start recognizing you need to call the function:</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
recognition.start();
</pre> 
          <p>The following function is called to stop recognition:</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
recognition.stop();
</pre> 
          <p>Now you need to initialize the recognition results handler:</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
recognition.onresult = function (event) {};
</pre> 
          <p>Add the result handling logic inside this function. The&nbsp;<strong>event</strong>&nbsp;object has the following fields:</p> 
          <ul> 
           <li><strong>event.results[i]</strong>&nbsp;– the array containing recognition result objects. Each array element corresponds to a recognized word on the&nbsp;<strong>i</strong>&nbsp;recognition stage.</li> 
           <li><strong>event.resultIndex</strong>&nbsp;– the current recognition result index.</li> 
           <li><strong>event.results[i][j]</strong>&nbsp;– the&nbsp;<strong>j</strong>-th alternative of a recognized word. The first element is a mostly probable recognized word.</li> 
           <li><strong>event.results[i].isFinal</strong>&nbsp;– the&nbsp;Boolean&nbsp;value that shows whether this result is final or&nbsp;interim.</li> 
           <li><strong>event.results[i][ j].transcript</strong>&nbsp;– the text representation of a word.</li> 
           <li><strong>event.results[i][ j].confidence</strong>&nbsp;– the probability of the given word correct decoding (value from 0 to 1).</li> 
          </ul> 
          <p>Now let’s write an elementary function that adds only final results to a text box (<strong>&lt;input&gt;&nbsp;</strong>as well as&nbsp;<strong>&lt;textarea&gt;</strong>&nbsp;can be used):</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
recognition.onresult = function (event) {
	for (var i = event.resultIndex; i &lt; event.results.length; ++i) {
		if (event.results[i].isFinal) {
			insertAtCaret(textAreaID, event.results[i][0].transcript);
		}
	}
};
</pre> 
          <p>This function contains a loop that iterates over all objects of recognized words. If the result is final, it’s displayed in the text box.</p> 
          <p>Here&nbsp;<strong>insertAtCaret()</strong>&nbsp;is the function that inserts a text (the 2nd argument ) into<strong>&nbsp;&lt;input&gt;</strong>&nbsp;or&nbsp;<strong>&lt;textarea&gt;</strong>&nbsp;with the&nbsp;<strong>textAreaID&nbsp;</strong>identificator.</p> 
          <p>Now let’s consider a more complex example that outputs interim results to a text box. The implementation of final results output is the same, but we added a code that outputs interim results.</p> 
          <pre class="brush: jscript; title: ; notranslate" title="">
recognition.onresult = function (event) {
	// Calculating and saving the cursor position where the text will be displayed
	var pos = textArea.getCursorPosition() - interimResult.length;
	// Deleting an interim result from the textArea field
	textArea.val(textArea.val().replace(interimResult, ''));
	interimResult = '';
	// Restoring the cursor position
	textArea.setCursorPosition(pos);
	for (var i = event.resultIndex; i &lt; event.results.length; ++i) {
		if (event.results[i].isFinal) {
			insertAtCaret(textAreaID, event.results[i][0].transcript);
		} else {
			// Outputting the interim result to the text field and adding
			// an interim result marker - 0-length space
			insertAtCaret(textAreaID, event.results[i][0].transcript + '\u200B');
			interimResult += event.results[i][0].transcript + '\u200B';
		}
	}
};
</pre> 
          <p>The advantages of JavaScript Web Speech API:</p> 
          <ul> 
           <li>the possibility to&nbsp;continuously&nbsp;dictate a text</li> 
           <li>the possibility to implement a multi-session recognition and save the result.</li> 
           <li>the possibility to insert the recognized speech anywhere in a text</li> 
           <li>it can be used for any (and not only) html element (you can implement voice commands)</li> 
           <li>the possibility to display interim recognition results</li> 
          </ul> 
          <p>The disadvantages are:</p> 
          <ul> 
           <li>the user should allow the browser to use the microphone before starting the session</li> 
           <li>the session length is limited to 60 seconds.</li> 
          </ul> 
          <p>The most important disadvantage of the whole implementation (including the <strong>x-webkit-speech</strong> attribute in the <strong>&lt;input&gt;</strong> tag) of Web Speech API in Chrome is that recognition is performed on the server and not locally in the browser. This is the reason why recognition has time limits. Currently popular browsers as Firefox, Internet Explorer and Safari don’t support Web Speech API, but it’s expected that these browsers also implement speech recognition shortly.</p>
