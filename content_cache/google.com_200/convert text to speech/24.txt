        <p>Imagine you&nbsp;are on&nbsp;a vacation to Paris&nbsp;and you do not know how to speak French! Wouldn’t that be a big bummer? What if, there was an App that can speak french on your behalf?</p> 
        <p>Sounds intriguing right?</p> 
        <p>I got this idea from Eric Bidelman’s demo on the Web Speech API</p> 
        <p style="text-align: center;"> 
         <!-- iframe plugin v.4.2 wordpress.org/plugins/iframe/ --> <iframe width="480" height="360" src="//www.youtube.com/embed/N_wTBKMuJis?start=1695&amp;rel=0" frameborder="0" allowfullscreen scrolling="yes" class="iframe-class"></iframe> </p> 
        <p><em>Note : The Web Speech API starts @ 30 minutes into the video</em></p> 
        <p>So, I wanted to replicate this idea as a hybrid mobile app. But because of the poor webview support for Web Speech API, I was looking around for a polyfill.</p> 
        <p>I found a polyfill&nbsp;named&nbsp;<a href="https://github.com/janantala/speech-synthesis" onclick="__gaTracker('send', 'event', 'outbound-article', 'https://github.com/janantala/speech-synthesis', 'speech-synthesis');" title="speech-synthesis" target="_blank">speech-synthesis</a>&nbsp;which works awesome on browsers, but unfortunately it does not on webviews!</p> 
        <p>So I forked the repo and patched it with a fix, so it works for webviews as well.</p> 
        <p>The issue was with the audio tag, which was not working&nbsp;on the device. Tried a couple of hacks but nothing seemed to work. I added a patch so that the polyfill uses Cordova Media API instead of the HTML5 Audio tag and Bam!! it started working.</p> 
        <p>The patched repo is called&nbsp;<a href="https://github.com/arvindr21/speech-synthesis-cordova" onclick="__gaTracker('send', 'event', 'outbound-article', 'https://github.com/arvindr21/speech-synthesis-cordova', 'speech-synthesis-cordova');" title="speech-synthesis-cordova" target="_blank">speech-synthesis-cordova</a>. And we are going to use the same in this post.</p> 
        <p>Below is quick peak of the app</p> 
        <p style="text-align: center;"> 
         <!-- iframe plugin v.4.2 wordpress.org/plugins/iframe/ --> <iframe width="640" height="360" src="//www.youtube.com/embed/FFM31ZS0LNY?rel=0" frameborder="0" allowfullscreen scrolling="yes" class="iframe-class"></iframe> </p> 
        <p>Awesome right! Getting one more step closer to my robot butler Jarvis!</p> 
        <p>You can find the completed code <a href="https://github.com/arvindr21/smartMouth" onclick="__gaTracker('send', 'event', 'outbound-article', 'https://github.com/arvindr21/smartMouth', 'here');" title="Code" target="_blank">here</a>.</p> 
        <p>So let us get started.</p> 
        <h2>Prerequisites</h2> 
        <p>If you are new to PhoneGap and have not set up or used it before I would recommend going through the below</p> 
        <ul> 
         <li><a title="PhoneGap 3 CLI Setup on Mac &amp; Windows" href="http://thejackalofjavascript.com/phonegap-3-cli-setup-mac-windows/" target="_blank">PhoneGap 3 CLI Setup on Mac &amp; Windows</a></li> 
         <li><a title="PhoneGap Quick Start" href="http://thejackalofjavascript.com/phonegap-quick-start/" target="_blank">PhoneGap Quick Start</a></li> 
         <li><a title="PhoneGap 3 – Contacts API Tutorial" href="http://thejackalofjavascript.com/phonegap-3-contacts-api-tutorial/" target="_blank">PhoneGap 3 – Contacts API Tutorial</a></li> 
        </ul> 
        <p>If you are new to Web Speech API, you can check out <a title="Getting Started with Web Speech API" href="http://thejackalofjavascript.com/getting-started-web-speech-api/" target="_blank">Getting Started with Web Speech API</a>.</p> 
        <h3>Understanding the polyfill</h3> 
        <p>If the Web Speech API is available in your browser, the Web Speech API will be used to convert text to speech. Else, the polyfill will make a request to Google Translate and then play that media.</p> 
        <p>In our case, we will be playing the response from the Google Translate using Cordova’s&nbsp;Media API.</p> 
        <h2>Setup Project</h2> 
        <p>Make sure you have followed&nbsp;<a title="PhoneGap 3 CLI Setup on Mac &amp; Windows" href="http://thejackalofjavascript.com/phonegap-3-cli-setup-mac-windows/" target="_blank">PhoneGap 3 CLI Setup on Mac &amp; Windows</a>&nbsp;and set up Phonegap on your machine.</p> 
        <p>Next, create a new folder named&nbsp;<em>smartMouthApp</em>. Open a new terminal/prompt here and run</p> 
        <p class="p1" style="text-align: center;"> <span id="crayon-59c35fc30f0e7810806749" class="crayon-syntax crayon-syntax-inline  crayon-theme-arduino-ide crayon-theme-arduino-ide-inline crayon-font-droid-sans-mono" style="font-size: 12px !important; line-height: 15px !important;font-size: 12px !important;"><span class="crayon-pre crayon-code" style="font-size: 12px !important; line-height: 15px !important;font-size: 12px !important; -moz-tab-size:4; -o-tab-size:4; -webkit-tab-size:4; tab-size:4;"><span class="crayon-e">phonegap </span><span class="crayon-e">create </span><span class="crayon-v">smartMouth</span></span></span>&nbsp;&nbsp;<span class="Apple-converted-space">&nbsp; &nbsp; &nbsp;</span></p> 
        <p class="p1">This will create a new Phonegap project. Once the setup is done, we need to CD into the&nbsp;<em>smartMouth</em> folder. And then we will&nbsp;add the <a href="https://github.com/apache/cordova-plugin-media" onclick="__gaTracker('send', 'event', 'outbound-article', 'https://github.com/apache/cordova-plugin-media', 'Media API');" title="Media API" target="_blank">Media API</a> plugin. Run</p> 
        <p class="p1" style="text-align: center;"> <span id="crayon-59c35fc30f111381450300" class="crayon-syntax crayon-syntax-inline  crayon-theme-arduino-ide crayon-theme-arduino-ide-inline crayon-font-droid-sans-mono" style="font-size: 12px !important; line-height: 15px !important;font-size: 12px !important;"><span class="crayon-pre crayon-code" style="font-size: 12px !important; line-height: 15px !important;font-size: 12px !important; -moz-tab-size:4; -o-tab-size:4; -webkit-tab-size:4; tab-size:4;"><span class="crayon-e">cd </span><span class="crayon-v">smartMouth</span></span></span></p> 
        <p class="p1" style="text-align: center;"> <span id="crayon-59c35fc30f119886508569" class="crayon-syntax crayon-syntax-inline  crayon-theme-arduino-ide crayon-theme-arduino-ide-inline crayon-font-droid-sans-mono" style="font-size: 12px !important; line-height: 15px !important;font-size: 12px !important;"><span class="crayon-pre crayon-code" style="font-size: 12px !important; line-height: 15px !important;font-size: 12px !important; -moz-tab-size:4; -o-tab-size:4; -webkit-tab-size:4; tab-size:4;"><span class="crayon-e">phonegap </span><span class="crayon-e">plugin </span><span class="crayon-e">add </span><span class="crayon-v">org</span><span class="crayon-sy">.</span><span class="crayon-v">apache</span><span class="crayon-sy">.</span><span class="crayon-v">cordova</span><span class="crayon-sy">.</span><span class="crayon-v">media</span></span></span></p> 
        <p class="p1">This should take care of plugin dependency.</p> 
        <p class="p1">Next, we need to move the <em>config.xml</em> from the root of&nbsp;<em>smartMouth</em> folder to inside&nbsp;<em>www</em> folder. This is more of a Phonegap build dependency.</p> 
        <p class="p1">Once you move the <em>config.xml&nbsp;</em>into the&nbsp;<em>www</em> folder, open it up in your favorite editor. And update the essential details like</p>

        <p><strong>Line 43</strong> : When the user clicks on translate button, we check if the text is valid on line 48 and then make an Ajax call to translate.google.com to get the translation for the text.</p> 
        <p><strong>Line 46</strong> : In case the text is empty we warn the user.</p> 
        <p><strong>Line 56</strong> : Once the data is received, we eval it, parse it and get the translated text.</p> 
        <p><strong>Line 58</strong> :&nbsp;The translated text is show on the UI</p> 
        <p><strong>Line 60</strong> :&nbsp; <span id="crayon-59c35fc30f184044149508" class="crayon-syntax crayon-syntax-inline  crayon-theme-arduino-ide crayon-theme-arduino-ide-inline crayon-font-droid-sans-mono" style="font-size: 12px !important; line-height: 15px !important;font-size: 12px !important;"><span class="crayon-pre crayon-code" style="font-size: 12px !important; line-height: 15px !important;font-size: 12px !important; -moz-tab-size:4; -o-tab-size:4; -webkit-tab-size:4; tab-size:4;"><span class="crayon-v">fallbackSpeechSynthesisUtterance</span></span></span>&nbsp;is called passing in the translated text to be read out loud along with the lang</p> 
        <p><strong>Line 67</strong> : Welcomes the user when the app launches.</p> 
        <p class="p1">Bam! That is it! Our app is ready!</p> 
        <h2>Test the app</h2> 
        <p class="p1">You can test the app on an emulator or by adding android/ios platforms.</p> 
        <p class="p1" style="text-align: center;"> <span id="crayon-59c35fc30f18e065281040" class="crayon-syntax crayon-syntax-inline  crayon-theme-arduino-ide crayon-theme-arduino-ide-inline crayon-font-droid-sans-mono" style="font-size: 12px !important; line-height: 15px !important;font-size: 12px !important;"><span class="crayon-pre crayon-code" style="font-size: 12px !important; line-height: 15px !important;font-size: 12px !important; -moz-tab-size:4; -o-tab-size:4; -webkit-tab-size:4; tab-size:4;"><span class="crayon-e">phonegap </span><span class="crayon-i">install</span>&nbsp;<span class="crayon-v">android</span></span></span></p> 
        <p class="p1">If you are facing issues on the device, I highly recommend using <a href="https://developer.chrome.com/devtools/docs/remote-debugging" onclick="__gaTracker('send', 'event', 'outbound-article', 'https://developer.chrome.com/devtools/docs/remote-debugging', 'chrome inspect tool');" title="Chrome Inspect" target="_blank">chrome inspect tool</a>.</p> 
        <h2 class="p1">Phonegap Build</h2> 
        <p class="p1">Now that the app is tested, we will issue a Phonegap build to generate installer.</p> 
        <p class="p1">Navigate to <a href="https://build.phonegap.com/apps" onclick="__gaTracker('send', 'event', 'outbound-article', 'https://build.phonegap.com/apps', 'Phonegap Build');" title="Phonegap build" target="_blank">Phonegap Build</a>&nbsp;and create a new app.</p> 
        <p class="p1"><a href="http://i1.wp.com/thejackalofjavascript.com/wp-content/uploads/2014/10/Screen-Shot-2014-10-28-at-4.43.19-pm.png" data-rel="lightbox-0" title=""><img class="size-medium wp-image-2444 aligncenter" src="http://i0.wp.com/thejackalofjavascript.com/wp-content/plugins/lazy-load/images/1x1.trans.gif" data-lazy-src="http://i1.wp.com/thejackalofjavascript.com/wp-content/uploads/2014/10/Screen-Shot-2014-10-28-at-4.43.19-pm.png?resize=451%2C142" alt="Screen Shot 2014-10-28 at 4.43.19 pm" data-recalc-dims="1">
          <noscript>
           <img class="size-medium wp-image-2444 aligncenter" src="http://i1.wp.com/thejackalofjavascript.com/wp-content/uploads/2014/10/Screen-Shot-2014-10-28-at-4.43.19-pm.png?resize=451%2C142" alt="Screen Shot 2014-10-28 at 4.43.19 pm" data-recalc-dims="1">
          </noscript></a></p> 
        <p class="p1">You can either upload the&nbsp;<em>www</em> folder (only <em>www</em> folder is required for a phonegap build) or&nbsp;push the <em>www</em> folder to a git repo and then provide the details in the above screen.</p> 
        <p class="p1">Once your code is uploaded/synced successfully, you will see a Ready to Build button.&nbsp;<a href="http://i1.wp.com/thejackalofjavascript.com/wp-content/uploads/2014/10/Screen-Shot-2014-10-28-at-4.47.36-pm.png" data-rel="lightbox-1" title=""><img class="size-medium wp-image-2445 aligncenter" src="http://i0.wp.com/thejackalofjavascript.com/wp-content/plugins/lazy-load/images/1x1.trans.gif" data-lazy-src="http://i1.wp.com/thejackalofjavascript.com/wp-content/uploads/2014/10/Screen-Shot-2014-10-28-at-4.47.36-pm.png?resize=451%2C111" alt="Screen Shot 2014-10-28 at 4.47.36 pm" data-recalc-dims="1">
          <noscript>
           <img class="size-medium wp-image-2445 aligncenter" src="http://i1.wp.com/thejackalofjavascript.com/wp-content/uploads/2014/10/Screen-Shot-2014-10-28-at-4.47.36-pm.png?resize=451%2C111" alt="Screen Shot 2014-10-28 at 4.47.36 pm" data-recalc-dims="1">
          </noscript></a></p> 
        <p class="p1">And once you click it, the build process will start.</p> 
        <p class="p1">Once the build is completed, you should see something like</p> 
        <p class="p1"><a href="http://i0.wp.com/thejackalofjavascript.com/wp-content/uploads/2014/10/Screen-Shot-2014-10-28-at-4.49.32-pm.png" data-rel="lightbox-2" title=""><img class="size-medium wp-image-2446 aligncenter" src="http://i0.wp.com/thejackalofjavascript.com/wp-content/plugins/lazy-load/images/1x1.trans.gif" data-lazy-src="http://i0.wp.com/thejackalofjavascript.com/wp-content/uploads/2014/10/Screen-Shot-2014-10-28-at-4.49.32-pm.png?resize=451%2C243" alt="Screen Shot 2014-10-28 at 4.49.32 pm" data-recalc-dims="1">
          <noscript>
           <img class="size-medium wp-image-2446 aligncenter" src="http://i0.wp.com/thejackalofjavascript.com/wp-content/uploads/2014/10/Screen-Shot-2014-10-28-at-4.49.32-pm.png?resize=451%2C243" alt="Screen Shot 2014-10-28 at 4.49.32 pm" data-recalc-dims="1">
          </noscript></a>You can either QR the image or click on install to download the installer for your device.</p> 
        <p class="p1">This app may not be using much of the Web Speech API, but this can be a start. As&nbsp;more and more updates keep coming in, we can run Web Speech API without any polyfills and achieve the same results.</p>
