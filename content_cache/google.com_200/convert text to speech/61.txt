           <h4 name="427f" id="427f" class="graf graf--h4 graf-after--p">Step 2?—?Tone&nbsp;Analyzer</h4>
           <p name="92af" id="92af" class="graf graf--p graf-after--h4">Tone analyzer is one of Watson’s unique offerings. It’s based on the theory of psycholinguistics, a field of research focused on the relationship between linguistic behaviors and psychological theories. You can read more about the science behind the API <a href="https://www.ibm.com/watson/developercloud/doc/tone-analyzer/science.shtml" data-href="https://www.ibm.com/watson/developercloud/doc/tone-analyzer/science.shtml" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">here</a>. The results include emotional tone, social tone, and language tone but we’ll only be using the emotional tone in this project.</p>
           <p name="2e33" id="2e33" class="graf graf--p graf-after--p">The possible emotions returned include joy, sadness, fear, disgust, and anger, along with their respective likelihood score (between 0 and 1). For simplicity’s sake, TJBot will choose the emotion with the highest likelihood score, but better heuristics should be chosen in the future since a sentence can also contain a mix of emotions or the most likely emotion might not be the dominant one.</p>

           <p name="a951" id="a951" class="graf graf--p graf-after--figure">You simply use the <strong class="markup--strong markup--p-strong">synthesize</strong> function and pipe the result to fs to create the file <em class="markup--em markup--p-em">output.wav</em>. We then use the <a href="https://www.npmjs.com/package/play-sound" data-href="https://www.npmjs.com/package/play-sound" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">play-sound</a> module to play the file through the speakers.</p>
           <p name="454b" id="454b" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Important Note</strong>: The<em class="markup--em markup--p-em"> </em><a href="https://www.npmjs.com/package/play-sound" data-href="https://www.npmjs.com/package/play-sound" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">play-sound</a> module requires you having one of the <a href="https://www.npmjs.com/package/play-sound#options" data-href="https://www.npmjs.com/package/play-sound#options" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">players</a> installed. I installed <strong class="markup--strong markup--p-strong">mplayer</strong> by simply doing ‘<strong class="markup--strong markup--p-strong"><em class="markup--em markup--p-em">brew install mplayer’</em>.</strong></p>
           <h4 name="58f3" id="58f3" class="graf graf--h4 graf-after--p"><strong class="markup--strong markup--h4-strong">Step 5?—?Putting It All&nbsp;Together</strong></h4>
           <p name="6a3a" id="6a3a" class="graf graf--p graf-after--h4">Finally it’s time get all the APIs to play together. This is done in <a href="https://github.com/boxcarton/tjbot-raspberrypi-nodejs/blob/master/run.js" data-href="https://github.com/boxcarton/tjbot-raspberrypi-nodejs/blob/master/run.js" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank"><strong class="markup--strong markup--p-strong"><em class="markup--em markup--p-em">run.js</em></strong></a><em class="markup--em markup--p-em">. </em>At this point, you should be familiar with most of the code in this file. However, there are few changes I’d like to point out.</p>
           <ul class="postList">
            <li name="a837" id="a837" class="graf graf--li graf-after--p"><strong class="markup--strong markup--li-strong">Only start the conversation upon hearing “Watson” (or whatever attention word you’ve set in your config file)</strong></li>
           </ul>
           <p name="4b5b" id="4b5b" class="graf graf--p graf-after--li">Since the microphone is on all the time, we don’t want TJBot to respond to every sound it hears. After converting user speech to text, we make sure the word “watson” is actually spoken.</p>

           <ul class="postList">
            <li name="d204" id="d204" class="graf graf--li graf-after--p"><strong class="markup--strong markup--li-strong">Prevent TJBot from talking to himself</strong></li>
           </ul>
           <p name="e2fd" id="e2fd" class="graf graf--p graf-after--li">Notice that we’ve put the text-to-speech code into the <strong class="markup--strong markup--p-strong">speakResponse</strong> function. More importantly, notice on line 108 that we pause the microphone right before we play the file. This is to prevent the audio from being heard by our speech-to-text function, resulting in a infinite feedback loop.</p>

           <p name="7f56" id="7f56" class="graf graf--p graf-after--figure">But how we know when to turn the microphone back on?            <p name="a6d2" id="a6d2" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong"><em class="markup--em markup--p-em">Important Note</em></strong>: node-ffprobe requires <strong class="markup--strong markup--p-strong">ffmpeg</strong> to be installed (see <a href="https://www.npmjs.com/package/node-ffprobe" data-href="https://www.npmjs.com/package/node-ffprobe" class="markup--anchor markup--p-anchor" rel="noopener nofollow" target="_blank">official package homepage</a>):</p>
           <p name="bcd8" id="bcd8" class="graf graf--p graf-after--p"><code class="markup--code markup--p-code">sudo apt-get install ffmpeg<br>npm install node-ffprobe</code></p>
           <p name="6e5a" id="6e5a" class="graf graf--p graf-after--p">We also add on another 0.5 seconds to the duration just to be safe.</p>

            <li name="8e12" id="8e12" class="graf graf--li graf-after--p"><strong class="markup--strong markup--li-strong">Using Tone Analyzer requires </strong><a href="https://www.promisejs.org/" data-href="https://www.promisejs.org/" class="markup--anchor markup--li-anchor" rel="nofollow noopener" target="_blank"><strong class="markup--strong markup--li-strong">promise</strong></a></li>
