        <div class="section" id="textract"> 
         <h1>textract<a class="headerlink" href="#textract" title="Permalink to this headline">¶</a></h1> 
         <p>As undesireable as it might be, more often than not there is extremely useful information embedded in Word documents, PowerPoint presentations, PDFs, etc—so-called “dark data”—that would be valuable for further textual analysis and visualization. While <a class="reference internal" href="#supporting"><span class="std std-ref">several packages</span></a> exist for extracting content from each of these formats on their own, this package provides a single interface for extracting content from any type of file, without any irrelevant markup.</p> 
         <p>This package provides two primary facilities for doing this, the <a class="reference internal" href="command_line_interface.html#command-line-interface"><span class="std std-ref">command line interface</span></a></p> 
         <div class="highlight-bash">
          <div class="highlight">
           <pre><span></span>textract path/to/file.extension
</pre>
          </div> 
         </div> 
         <p>or the <a class="reference internal" href="python_package.html#python-package"><span class="std std-ref">python package</span></a></p> 
         <div class="highlight-python">
          <div class="highlight">
           <pre><span></span><span class="c1"># some python file</span>
<span class="kn">import</span> <span class="nn">textract</span>
<span class="n">text</span> <span class="o">=</span> <span class="n">textract</span><span class="o">.</span><span class="n">process</span><span class="p">(</span><span class="s2">"path/to/file.extension"</span><span class="p">)</span>
</pre>
          </div> 
         </div> 
         <div class="section" id="currently-supporting"> 
          <span id="supporting"></span>
          <h2>Currently supporting<a class="headerlink" href="#currently-supporting" title="Permalink to this headline">¶</a></h2> 
          <p>textract supports a growing list of file types for text extraction. If you don’t see your favorite file type here, Please recommend other file types by either mentioning them on the <a class="reference external" href="https://github.com/deanmalmgren/textract/issues">issue tracker</a> or by <a class="reference internal" href="contributing.html#contributing"><span class="std std-ref">contributing a pull request</span></a>.</p> 
          <ul class="simple"> 
           <li><code class="docutils literal"><span class="pre">.csv</span></code> via python builtins</li> 
           <li><code class="docutils literal"><span class="pre">.doc</span></code> via <a class="reference external" href="http://www.winfield.demon.nl/">antiword</a></li> 
           <li><code class="docutils literal"><span class="pre">.docx</span></code> via <a class="reference external" href="https://github.com/ankushshah89/python-docx2txt">python-docx2txt</a></li> 
           <li><code class="docutils literal"><span class="pre">.eml</span></code> via python builtins</li> 
           <li><code class="docutils literal"><span class="pre">.epub</span></code> via <a class="reference external" href="https://github.com/aerkalov/ebooklib">ebooklib</a></li> 
           <li><code class="docutils literal"><span class="pre">.gif</span></code> via <a class="reference external" href="https://code.google.com/p/tesseract-ocr/">tesseract-ocr</a></li> 
           <li><code class="docutils literal"><span class="pre">.jpg</span></code> and <code class="docutils literal"><span class="pre">.jpeg</span></code> via <a class="reference external" href="https://code.google.com/p/tesseract-ocr/">tesseract-ocr</a></li> 
           <li><code class="docutils literal"><span class="pre">.json</span></code> via python builtins</li> 
           <li><code class="docutils literal"><span class="pre">.html</span></code> and <code class="docutils literal"><span class="pre">.htm</span></code> via <a class="reference external" href="http://beautiful-soup-4.readthedocs.org/en/latest/">beautifulsoup4</a></li> 
           <li><code class="docutils literal"><span class="pre">.mp3</span></code> via <a class="reference external" href="http://sox.sourceforge.net/">sox</a>, <a class="reference external" href="https://pypi.python.org/pypi/SpeechRecognition/">SpeechRecognition</a>, and <a class="reference external" href="https://github.com/cmusphinx/pocketsphinx/">pocketsphinx</a></li> 
           <li><code class="docutils literal"><span class="pre">.msg</span></code> via <a class="reference external" href="https://github.com/mattgwwalker/msg-extractor">msg-extractor</a></li> 
           <li><code class="docutils literal"><span class="pre">.odt</span></code> via python builtins</li> 
           <li><code class="docutils literal"><span class="pre">.ogg</span></code> via <a class="reference external" href="http://sox.sourceforge.net/">sox</a>, <a class="reference external" href="https://pypi.python.org/pypi/SpeechRecognition/">SpeechRecognition</a>, and <a class="reference external" href="https://github.com/cmusphinx/pocketsphinx/">pocketsphinx</a></li> 
           <li><code class="docutils literal"><span class="pre">.pdf</span></code> via <a class="reference external" href="http://poppler.freedesktop.org/">pdftotext</a> (default) or <a class="reference external" href="https://github.com/goulu/pdfminer">pdfminer.six</a></li> 
           <li><code class="docutils literal"><span class="pre">.png</span></code> via <a class="reference external" href="https://code.google.com/p/tesseract-ocr/">tesseract-ocr</a></li> 
           <li><code class="docutils literal"><span class="pre">.pptx</span></code> via <a class="reference external" href="https://python-pptx.readthedocs.org/en/latest/">python-pptx</a></li> 
           <li><code class="docutils literal"><span class="pre">.ps</span></code> via <a class="reference external" href="http://pages.cs.wisc.edu/~ghost/doc/pstotext.htm">ps2text</a></li> 
           <li><code class="docutils literal"><span class="pre">.rtf</span></code> via <a class="reference external" href="http://www.gnu.org/software/unrtf/">unrtf</a></li> 
           <li><code class="docutils literal"><span class="pre">.tiff</span></code> and <code class="docutils literal"><span class="pre">.tif</span></code> via <a class="reference external" href="https://code.google.com/p/tesseract-ocr/">tesseract-ocr</a></li> 
           <li><code class="docutils literal"><span class="pre">.txt</span></code> via python builtins</li> 
           <li><code class="docutils literal"><span class="pre">.wav</span></code> via <a class="reference external" href="https://pypi.python.org/pypi/SpeechRecognition/">SpeechRecognition</a> and <a class="reference external" href="https://github.com/cmusphinx/pocketsphinx/">pocketsphinx</a></li> 
           <li><code class="docutils literal"><span class="pre">.xlsx</span></code> via <a class="reference external" href="https://pypi.python.org/pypi/xlrd">xlrd</a></li> 
           <li><code class="docutils literal"><span class="pre">.xls</span></code> via <a class="reference external" href="https://pypi.python.org/pypi/xlrd">xlrd</a></li> 
          </ul> 
         </div> 
         <div class="section" id="related-projects"> 
          <span id="id2"></span>
          <h2>Related projects<a class="headerlink" href="#related-projects" title="Permalink to this headline">¶</a></h2> 
          <p>Of course, textract isn’t the first project with the aim to provide a simple interface for extracting text from any document. But this is, to the best of my knowledge, the only project that is written in python (a language commonly chosen by the natural language processing community) and is <a class="reference internal" href="contributing.html#contributing"><span class="std std-ref">method agnostic about how content is extracted</span></a>. I’m sure that there are other similar projects out there, but here is a small sample of similar projects:</p> 
          <ul class="simple"> 
           <li><a class="reference external" href="http://tika.apache.org/">Apache Tika</a> has <a class="reference external" href="https://github.com/deanmalmgren/textract/issues/12">very similar, if not identical, aims as textract</a> and has impressive coverage of a wide range of file formats. It is written in java.</li> 
           <li><a class="reference external" href="https://github.com/dbashford/textract">textract (node.js)</a> has similar aims as this textract package (including an identical name! great minds...). It is written in node.js.</li> 
           <li><a class="reference external" href="http://johnmacfarlane.net/pandoc/">pandoc</a> is intended to be a document conversion tool (a much more difficult task!), but it does have <a class="reference external" href="http://johnmacfarlane.net/pandoc/demos.html">the ability to convert to plain text</a>. It is written in Haskell.</li> 
          </ul>
