        <h1>Introduction to the Drone Journey</h1> 
        <p><a href="https://i2.wp.com/www.marktorr.com/wp-content/uploads/2017/02/pablo-37.png"><img class="aligncenter size-medium wp-image-1028" src="https://i2.wp.com/www.marktorr.com/wp-content/uploads/2017/02/pablo-37.png?resize=300%2C150" alt="Final Installment" srcset="https://i2.wp.com/www.marktorr.com/wp-content/uploads/2017/02/pablo-37.png?resize=300%2C150 300w, https://i2.wp.com/www.marktorr.com/wp-content/uploads/2017/02/pablo-37.png?resize=768%2C384 768w, https://i2.wp.com/www.marktorr.com/wp-content/uploads/2017/02/pablo-37.png?w=1024 1024w, https://i2.wp.com/www.marktorr.com/wp-content/uploads/2017/02/pablo-37.png?w=948 948w" sizes="(max-width: 300px) 100vw, 300px" data-recalc-dims="1"></a></p> 
        <p>This post is the&nbsp;eleventh, and final, post&nbsp;in documenting the steps I went through on my journey to build an autonomous, voice-controlled, face recognizing drone. There are 10 other posts building up to this one which you can find at the end of this post.</p> 
        <h2>Focus of this post</h2> 
        <p>In this post I will share a video of the complete end-to-end demo and share details of the architecture which sits behind it. I will also share information on what I bought/used to bring this all together and relist all the different software, services and node packages in a single place.</p> 
        <h2>Pulling It All Together</h2> 
        <p>A lot of what we have been doing with this project is humanizing the way we communicate with machines/computers/things. That means talking and observing to drive intelligent interaction&nbsp;rather than using a mouse, keyboard or touch screen.</p> 
        <p>Our Autonomous Voice-Controlled, Face Recognizing, Drone is a smart drone which showcases, albeit crudely, how interaction with services filled with intelligence is going to evolve. It highlights the importance of cognitive services to the success of organizations in the future.</p>

        <h1>Introduction to the Drone Journey</h1> 
        <p><a href="https://i0.wp.com/www.marktorr.com/wp-content/uploads/2017/02/pablo-36.png"><img class="aligncenter size-medium wp-image-1018" src="https://i0.wp.com/www.marktorr.com/wp-content/uploads/2017/02/pablo-36.png?resize=300%2C150" alt="Telemetry Data" srcset="https://i0.wp.com/www.marktorr.com/wp-content/uploads/2017/02/pablo-36.png?resize=300%2C150 300w, https://i0.wp.com/www.marktorr.com/wp-content/uploads/2017/02/pablo-36.png?resize=768%2C384 768w, https://i0.wp.com/www.marktorr.com/wp-content/uploads/2017/02/pablo-36.png?w=1024 1024w, https://i0.wp.com/www.marktorr.com/wp-content/uploads/2017/02/pablo-36.png?w=948 948w" sizes="(max-width: 300px) 100vw, 300px" data-recalc-dims="1"></a></p> 
        <p>This post is the&nbsp;tenth post&nbsp;in documenting the steps I went through on my journey to build an autonomous, voice-controlled, face recognizing drone. There are&nbsp;9 other posts building up to this one which you can find at the end of this post.</p> 
        <h2>Focus of this post</h2> 
        <p>Up until now we have been mostly working on controlling the drone, using the Microsoft Cognitive Services&nbsp;Face API to identify people and lastly making use of the Microsoft Cognitive Services API to convert text to speech and speech to text.</p> 
        <p>In this blog the focus will move to:</p> 
        <ul> 
         <li>How we can take some of the telemetry information from the&nbsp;drone and pass it to the <a href="https://azure.microsoft.com/en-us/services/iot-hub/">Microsoft Azure IoT Hub</a>.</li> 
         <li>Using <a href="https://azure.microsoft.com/en-us/services/stream-analytics/">Microsoft Azure Stream Analytics</a> to grab data in real-time</li> 
         <li>Visualizing the acquired data in real-time using <a href="https://azure.microsoft.com/en-us/services/iot-hub/">Microsoft PowerBI</a>.</li> 
        </ul> 
        <p>Ultimately we will have built an intelligent end-to-end IoT solution featuring analytics and visualization. The instructions here can be used to also understand how to get data in from other devices as well!</p>

        <h1>Introduction to the Drone Journey</h1> 
        <p><a href="https://i2.wp.com/www.marktorr.com/wp-content/uploads/2017/02/pablo-25.png"><img class="aligncenter size-medium wp-image-1000" src="https://i2.wp.com/www.marktorr.com/wp-content/uploads/2017/02/pablo-25.png?resize=300%2C150" alt="Face Recognition" srcset="https://i2.wp.com/www.marktorr.com/wp-content/uploads/2017/02/pablo-25.png?resize=300%2C150 300w, https://i2.wp.com/www.marktorr.com/wp-content/uploads/2017/02/pablo-25.png?resize=768%2C384 768w, https://i2.wp.com/www.marktorr.com/wp-content/uploads/2017/02/pablo-25.png?w=1024 1024w, https://i2.wp.com/www.marktorr.com/wp-content/uploads/2017/02/pablo-25.png?w=948 948w" sizes="(max-width: 300px) 100vw, 300px" data-recalc-dims="1"></a></p> 
        <p>This post is the&nbsp;seventh post&nbsp;in documenting the steps I went through on my journey to build an autonomous, voice-controlled, face recognizing drone. There are 6 other posts building up to this one which you can find at the end of this post.</p> 
        <h2>Focus of this post</h2> 
        <p>In this post I am going to pick-up where we left off and look at :</p> 
        <ul> 
         <li>How to use the Microsoft Cognitive Services Face API to recognize a specific face. Specifically&nbsp;we will explore&nbsp;the&nbsp;face.identify approach.</li> 
         <li>How to build the identify approach into your DroneWebServer.js file such that the drone will land when it sees a&nbsp;named person.</li> 
        </ul> 
        <h2>My issues</h2> 
        <p>I have to admit to facing quite some challenges getting <strong>face.identify</strong> to work. Using the <strong>face.identify</strong> capability requires a large number of things to be completed, in a very specific order, before it will work.</p> 
        <p>Node.js does not make doing things in a specific order easy without nested functions, callbacks&nbsp;and other things.&nbsp; The truth is that I wasted a large number of hours before I realized that the asynchronous nature of node.js was a major&nbsp;source of many of&nbsp;&nbsp;the issues I was&nbsp; running into rather than me misusing the Cognitive Services APIs o any issue with the APIs themselves.</p> 
        <p>I also found that the online examples sometimes do not really give you the step by step help you might need. this looked to mostly be the case in the Node.JS world as other programming languages&nbsp; seemed much more comprehensive. In the node.js world you essentially get the code to look at and try to understand which does not help with ordering issues.</p> 
        <p>When you add to that it is entirely possible to create things, people for example,&nbsp;with the same name multiple times (without realizing it)&nbsp;you can see where my issues came from.</p> 
        <p>I did speak with Lukas again at this stage. He told me that he set up his target faces using the&nbsp;SDK console&nbsp;rather than through code so I hit a dead end there as I was determined to try avoid that (I must admit I thought about it ;)..).</p> 
        <p>This is where my first outreach to someone at Microsoft came. I contacted Chris Thrasher who very kindly sent me some code samples he had worked on. He too found the challenge of things not being called in order which he solved using the bluebird package.</p> 
        <p>Thanks to his pointer and&nbsp;his sample code I identified a spot I was not doing something right which allowed me to move forwards.&nbsp; I want to be sure to say thanks to Chris because without his help I would have stopped at <strong>face.similar</strong>.</p> 
        <p><strong>A warning: This is a monster blog post. I decided to put it all in one rather than break it up. Grab a coffee, give yourself sometime and enjoy the ride!</strong></p>
