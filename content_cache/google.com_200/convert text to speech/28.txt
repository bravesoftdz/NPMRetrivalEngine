        <section property="articleBody"> 
         <p>In general, speech synthesis isn’t easy.&nbsp; You can’t just assume that when an application reads each letter of a sentence the output will make sense. A few common challenges for text-to-speech applications include:</p> 
         <ul> 
          <li>Words that are written the same way, but that are pronounced differently: I <strong><u>live</u></strong> in Las Vegas. <em>vs.</em> This presentation broadcasts <strong><u>live</u></strong> from Las Vegas.</li> 
         </ul> 
         <ul> 
          <li>Text normalization. Disambiguating abbreviations, acronyms, and units:&nbsp;<strong><u>St.</u></strong>, which can be expanded as street or saint.</li> 
         </ul> 
         <ul> 
          <li>Converting text to phonemes in languages with complex mapping, such as, in English, t<strong><u>ough</u></strong>, thr<strong><u>ough</u></strong>, th<strong><u>ough</u>. </strong>In this example, similar parts of different words can be pronounced differently depending on the word and context.</li> 
         </ul> 
         <ul> 
          <li>Foreign words (<strong><u>déjà vu</u></strong>), proper names (<strong><u>François Hollande</u></strong>), slang (<strong><u>ASAP</u></strong>, <strong><u>LOL</u></strong>), etc.</li> 
         </ul> 
         <p><a href="https://aws.amazon.com/polly/" target="_blank">Amazon Polly</a> provides speech synthesis functionality that overcomes those challenges, allowing you to focus on building applications that use text-to-speech instead of addressing interpretation challenges.</p> 
         <p>Amazon Polly turns text into lifelike speech. It lets you create applications that talk naturally, enabling you to build entirely new categories of speech-enabled products. Amazon Polly is an Amazon AI service that uses advanced deep learning technologies to synthesize speech that sounds like a human voice. It currently includes 47 lifelike voices in 24 languages, so you can select the ideal voice and build speech-enabled applications that work in many different countries.</p> 
         <p>In addition, Amazon Polly delivers the consistently fast response times required to support real-time, interactive dialog. You can cache and save Polly’s audio files for offline replay or redistribution. (In other words, what you convert and save is yours. There are no additional text-to-speech charges for using the speech.) And Polly is easy to use. You simply send the text you want to convert into speech to the Amazon Polly API. Amazon Polly immediately returns the audio stream to your application so that your application can play it directly or store it in a standard audio file format such as an MP3.</p> 
         <p>In this blog post, we create a basic, serverless application that uses Amazon Polly to convert text to speech. The application has a simple user interface that accepts text in many different languages and then converts it to audio files which you can play from a web browser. We’ll use blog posts, but you can use any type of text. For example, you can use the application to read recipes while you are preparing a meal, or news articles or books while you’re driving or riding a bike.</p> 
         <h2>The application’s architecture</h2> 
         <p>The following diagram shows the application architecture. It uses a <em>serverless approach</em>, which means that we don’t need to work with servers – no provisioning, no patching, no scaling. The Cloud automatically takes care of this, allowing us to focus on our application.</p> 
         <p><img class="alignnone size-full wp-image-315" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/23/text-to-speech_1.gif" alt="" width="1000" height="675"></p> 
         <p>The application provides two methods – one for sending information about a new post, which should be converted into an MP3 file, and one for retrieving information about the post (including a link to the MP3 file stored in an S3 bucket). Both methods are exposed as RESTful web services through Amazon API Gateway. Let’s look at how the interaction works in the application.</p> 
         <p><span id="more-314"></span></p> 
         <p>When the application sends information about new posts:</p> 
         <ol> 
          <li>The information is received by the RESTful web service exposed by Amazon API Gateway. In our scenario, this web service is invoked by a static webpage hosted on Amazon Simple Storage Service (Amazon S3).</li> 
          <li>Amazon API Gateway sets off a dedicated Lambda function, “New Post,” which is responsible for initializing the process of generating MP3 files.</li> 
          <li>The Lambda function inserts information about the post into a DynamoDB table, where information about all posts is stored.</li> 
          <li>To run the whole process asynchronously, we use Amazon SNS to decouple the process of receiving information about new posts and starting their conversion.</li> 
          <li>Another Lambda function, “Convert to Speech,” is subscribed to our SNS topic whenever a new message appears (which means that a new post should be converted into an audio file). This is the trigger.</li> 
          <li>The “Convert to Speech” Lambda function uses Amazon Polly to convert the text into an audio file in the specified language (the same as the language of the text).</li> 
          <li>The new MP3 file is saved in a dedicated S3 bucket.</li> 
          <li>Information about the post is updated in the DynamoDB table. Then, the reference (URL) to the S3 bucket is saved with the previously stored data.</li> 
         </ol> 
         <p>When the application retrieves information about posts:</p> 
         <ol> 
          <li>The RESTful web service is deployed using Amazon API Gateway. Amazon API Gateway exposes the method for retrieving information about posts. These methods contain the text of the post and the link to the S3 bucket where the MP3 file is stored. In our scenario, this web service is invoked by a static webpage hosted on Amazon S3.</li> 
          <li>Amazon API Gateway invokes the “Get Post” Lambda function, which deploys the logic for retrieving the post data.</li> 
          <li>The “Get Post” Lambda function retrieves information about the post (including the reference to Amazon S3) from the DynamoDB table.</li> 
         </ol> 
         <p>If you will want to replicate the following steps yourself, please be sure to choose a <a href="https://aws.amazon.com/about-aws/global-infrastructure/regional-product-services/" target="_blank">region where Amazon Polly service is available</a>.</p> 
         <h2>Creating a DynamoDB table</h2> 
         <p>We store information about posts, including the text and URL for the MP3 file, on DynamoDB. From the DynamoDB console we create a single table, which we call “posts.” Our primary key (id) is a string, which the “New Post” Lambda function creates when new records (posts) are inserted into a database.</p> 
         <p><img class="alignnone size-full wp-image-316" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/23/text-to-speech_2.gif" alt="" width="1000" height="327"></p> 
         <p>We won’t define the whole structure of the table now, but let’s see how it will look with some records.</p> 
         <p><img class="alignnone size-full wp-image-317" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/23/text-to-speech_3.gif" alt="" width="1000" height="311"></p> 
         <p>The columns provide the following information:</p> 
         <ul> 
          <li>id – The ID of the post</li> 
          <li>status – UPDATED or PROCESSING, depending on whether an MP3 file has already been created</li> 
          <li>text – The post’s text, for which an audio file is being created</li> 
          <li>url – A link to an S3 bucket where an audio file is being stored</li> 
          <li>voice – The Amazon Polly voice that was used to create audio file</li> 
         </ul> 
         <h2>Creating an S3 bucket</h2> 
         <p>We also need to create an S3 bucket to store all audio files created by the application. To do this, you need to go to the S3 console where you will find an option to create a new bucket.&nbsp; You can choose any name for the bucket as long as it’s globally unique.</p> 
         <p><img class="alignnone size-full wp-image-318" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/23/text-to-speech_4.gif" alt="" width="1000" height="490"></p> 
         <h2>Creating an SNS topic</h2> 
         <p>As you probably noticed in our architecture diagram, we have split the logic of converting a post (text) into an audio file into two Lambda functions. We did this for a couple of reasons. First, it allows our application to use asynchronous calls so that the user who sends a new post to the application receives the ID of the new DynamoDB item; so it knows what to ask for later; and to eliminate waiting for the conversion to finish. With small posts, the process of converting to audio files can take milliseconds, but with bigger posts (100,000 words or more), converting the text can take a bit longer. In other use cases, when we want to do real-time streaming, size isn’t a problem, because Amazon Polly starts to stream speech back as soon as the first bytes are available.</p> 
         <p>The second reason is that we use a Lambda function, which allows a single execution to run as long as 5 minutes. This should be more than enough time to convert our posts. In the future we might want to convert something bigger. In that case, we might want to use AWS Batch instead of Lambda. Decoupling these two parts of the application makes this change much easier.</p> 
         <p>When we have two components (in our case two Lambda functions) we need to integrate them. In other words, the second one needs to know when to start. You could do this in many different ways. In our case, we will use Amazon SNS. It sends the message about the new post from the first function to the second one.</p> 
         <p>So let’s create a simple SNS topic. We can do it from the SNS console, where you will find a button for creating a new topic. Let’s call it <tt>new_posts</tt>.</p> 
         <p><img class="alignnone size-full wp-image-319" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/23/text-to-speech_5.gif" alt="" width="1000" height="358"></p> 
         <h2>Creating an IAM role</h2> 
         <p>Before we dive into creating Lambda functions, we need to create an IAM role for the functions. The role specifies which AWS services (APIs) the functions can interact with. We will create one role for all three functions.</p> 
         <p>In the IAM console find <strong>Roles</strong> tab and then press <strong>Create New Role</strong> button to open a wizard for creating a new role.&nbsp; Let’s name the role <tt>LambdaPostsReaderRole</tt>.</p> 
         <p><img class="alignnone size-full wp-image-320" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/23/text-to-speech_6.gif" alt="" width="1000" height="253"></p> 
         <p>In the second step of the wizard you will need to assign new role to AWS Lambda service. &nbsp;Don’t configure any other parameters in the wizard and just create it.</p> 
         <p><img class="alignnone size-full wp-image-323" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/23/text-to-speech_7.gif" alt="" width="800" height="416"></p> 
         <p>After the role is created, on the <strong>Permissions </strong>tab, choose <strong>Inline Policies </strong>and then click on <strong>click here</strong> button – this will allow us to specify permissions for our new role.</p> 
         <p><img class="alignnone size-full wp-image-324" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/23/text-to-speech_8.gif" alt="" width="1000" height="522"></p>

         <h2>Creating the “New Post” Lambda function</h2> 
         <p>The first Lambda function that we create is the entry point for our application. It receives information about new posts that should be converted into audio files.</p> 
         <p>In the Lambda console you will see a button for creating a new Lambda function. Let’s call it <tt>PostReader_NewPost</tt>. For <strong>Runtime</strong>, we choose <strong>Python 2.7</strong>. For now, we don’t configure any triggers.</p> 
         <p><img class="alignnone size-full wp-image-325" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/23/text-to-speech_9.gif" alt="" width="1000" height="275"></p> 
         <p>As shown in the following code, this Lambda function does the following:</p> 
         <ol> 
          <li>Retrieves two input parameters: 
           <ul> 
            <li>Voice – one of dozens of voices that are supported by Amazon Polly</li> 
            <li>Text – the text of the post that we want to convert into an audio file</li> 
           </ul> </li> 
          <li>Creates a new record in the DynamoDB table with information about the new post</li> 
          <li>Publishes information about the new post to SNS (the ID of the DynamoDB item/post ID is published there as a message)</li> 
          <li>Returns the ID of the DynamoDB item to the user</li> 
         </ol> 
         <pre><code class="lang-python">import boto3
import os
import uuid

def lambda_handler(event, context):
    
    recordId = str(uuid.uuid4())
    voice = event["voice"]
    text = event["text"]

    print('Generating new DynamoDB record, with ID: ' + recordId)
    print('Input Text: ' + text)
    print('Selected voice: ' + voice)
    
    #Creating new record in DynamoDB table
    dynamodb = boto3.resource('dynamodb')
    table = dynamodb.Table(os.environ['DB_TABLE_NAME'])
    table.put_item(
        Item={
            'id' : recordId,
            'text' : text,
            'voice' : voice,
            'status' : 'PROCESSING'
        }
    )
    
    #Sending notification about new post to SNS
    client = boto3.client('sns')
    client.publish(
        TopicArn = os.environ['SNS_TOPIC'],
        Message = recordId
    )
    
    return recordId
</code></pre> 
         <p>In addition, the New Post Lambda function needs to know the name of the DynamoDB table and the SNS topic. To provide these values, we use the following environment variables:</p> 
         <ul> 
          <li>SNS_TOPIC – the Amazon Resource Name (ARN) of the SNS topic we created</li> 
          <li>DB_TABLE_NAME – the name of the DynamoDB table (in our case, it’s posts)</li> 
         </ul> 
         <p>You will find <strong>Environment variables</strong> section just below your code.</p>

         <h2>Creating the “Convert to Audio” Lambda function</h2> 
         <p>Now let’s create the Lambda function that converts text that is stored in a DynamoDB table into an audio file, “Convert to Audio.”</p> 
         <p>In the first step of the wizard, we specify the SNS topic that we created. This time, we configure and enable a trigger. Whenever our SNS topic receives a new message, it executes this function.</p> 
         <p><img class="alignnone size-full wp-image-328" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/23/text-to-speech_12.gif" alt="" width="1000" height="432"></p> 
         <p>Let’s call our new function <code class="lang-tt">PostReader_ConvertToAudio</code>. As before, we use Python 2.7 for <strong>Runtime</strong>.</p> 
         <p><img class="alignnone size-full wp-image-331" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/23/text-to-speech_13-1.gif" alt="" width="1000" height="282"></p> 
         <p>This Lambda function does the following:</p> 
         <ol> 
          <li>Retrieves the ID of the DynamoDB item (post ID) which should be converted into an audio file from the input message (SNS event)</li> 
          <li>Retrieves the item from DynamoDB</li> 
          <li>Converts the text into an audio stream</li> 
          <li>Places the audio (MP3) file into an S3 bucket</li> 
          <li>Updates the DynamoDB table with a reference to the S3 bucket and the new status</li> 
         </ol> 
         <p>The following code is mostly self explanatory, but let’s dive into the part that invokes Amazon Polly. The <tt>synthesize_speech</tt> method receives the text that should be converted and the voice that should be used. In return, it provides the audio stream. The catch is that there is a size limit of 1,500 characters on the text that can be provided as input. Because our posts can be big, we need to divide them into blocks of about 1,000 characters, depending where the final word in the block ends. After converting the blocks into an audio stream, we join them together again.</p> 
         <pre><code class="lang-python">import boto3
import os
from contextlib import closing
from boto3.dynamodb.conditions import Key, Attr

def lambda_handler(event, context):

    postId = event["Records"][0]["Sns"]["Message"]
    
    print "Text to Speech function. Post ID in DynamoDB: " + postId
    
    #Retrieving information about the post from DynamoDB table
    dynamodb = boto3.resource('dynamodb')
    table = dynamodb.Table(os.environ['DB_TABLE_NAME'])
    postItem = table.query(
        KeyConditionExpression=Key('id').eq(postId)
    )
    

    text = postItem["Items"][0]["text"]
    voice = postItem["Items"][0]["voice"] 
    
    rest = text
    
    #Because single invocation of the polly synthesize_speech api can 
    # transform text with about 1,500 characters, we are dividing the 
    # post into blocks of approximately 1,000 characters.
    textBlocks = []
    while (len(rest) &gt; 1100):
        begin = 0
        end = rest.find(".", 1000)

        if (end == -1):
            end = rest.find(" ", 1000)
            
        textBlock = rest[begin:end]
        rest = rest[end:]
        textBlocks.append(textBlock)
    textBlocks.append(rest)            

    #For each block, invoke Polly API, which will transform text into audio
    polly = boto3.client('polly')
    for textBlock in textBlocks: 
        response = polly.synthesize_speech(
            OutputFormat='mp3',
            Text = textBlock,
            VoiceId = voice
        )
        
        #Save the audio stream returned by Amazon Polly on Lambda's temp 
        # directory. If there are multiple text blocks, the audio stream
        # will be combined into a single file.
        if "AudioStream" in response:
            with closing(response["AudioStream"]) as stream:
                output = os.path.join("/tmp/", postId)
                with open(output, "a") as file:
                    file.write(stream.read())



    s3 = boto3.client('s3')
    s3.upload_file('/tmp/' + postId, 
      os.environ['BUCKET_NAME'], 
      postId + ".mp3")
    s3.put_object_acl(ACL='public-read', 
      Bucket=os.environ['BUCKET_NAME'], 
      Key= postId + ".mp3")

    location = s3.get_bucket_location(Bucket=os.environ['BUCKET_NAME'])
    region = location['LocationConstraint']
    
    if region is None:
        url_begining = "https://s3.amazonaws.com/"
    else:
        url_begining = "https://s3-" + str(region) + ".amazonaws.com/" \
    
    url = url_begining \
            + str(os.environ['BUCKET_NAME']) \
            + "/" \
            + str(postId) \
            + ".mp3"

    #Updating the item in DynamoDB
    response = table.update_item(
        Key={'id':postId},
          UpdateExpression=
            "SET #statusAtt = :statusValue, #urlAtt = :urlValue",                   
          ExpressionAttributeValues=
            {':statusValue': 'UPDATED', ':urlValue': url},
        ExpressionAttributeNames=
          {'#statusAtt': 'status', '#urlAtt': 'url'},
    )
        
    return</code></pre>

         <ul> 
          <li><strong>DB_TABLE_NAME</strong> – The name of the DynamoDB table (in our case, it’s posts )</li> 
          <li><strong>BUCKET_NAME </strong>– The name of the S3 bucket that we created to store MP3 files</li> 
         </ul> 
         <p><img class="alignnone size-full wp-image-332" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/23/text-to-speech_14.gif" alt="" width="1000" height="113"></p>

         <h2>Creating a “Get Post” Lambda function</h2> 
         <p>The third Lambda function provides a method for retrieving information about posts from our database. From the Lambda console we will create a new function. We’ll call this function <code class="lang-tt">PostReader_GetPost</code>. As before, we will use Python 2.7 as the runtime, but we won’t specify any triggers.</p> 
         <p><img class="alignnone size-full wp-image-335" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/23/text-to-speech_17.gif" alt="" width="1000" height="288"></p> 
         <p>This time the code is very short. This function expects to get the post ID (the DynamoDB item ID) and, on the basis of this ID, it retrieves all information (including the S3 link to the audio file if it exists), and then returns it. To make it a little more user friendly if the input parameter is and asterisk (*), our Lambda function returns all items from the database. (For a database with a lot of items, avoid this approach because it can degrade performance and might take a long time.)</p> 
         <p>&nbsp;</p> 
         <pre><code class="lang-python">import boto3
import os
from boto3.dynamodb.conditions import Key, Attr

def lambda_handler(event, context):
    
    postId = event["postId"]
    
    dynamodb = boto3.resource('dynamodb')
    table = dynamodb.Table(os.environ['DB_TABLE_NAME'])
    
    if postId=="*":
        items = table.scan()
    else:
        items = table.query(
            KeyConditionExpression=Key('id').eq(postId)
        )
    
    return items["Items"]</code></pre> 
         <p>Again, we provide the name of the DynamoDB table as the environment variable for our function.</p> 
         <p><img class="alignnone size-full wp-image-336" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/23/text-to-speech_18.gif" alt="" width="1000" height="65"></p>

         <h2>Exposing the Lambda function as a RESTful web service</h2> 
         <p>The last thing we need to do is expose our application logic as a RESTful web service so it can be invoked easily using a standard HTTP protocol. To do this, we use Amazon API Gateway. From API Gateway console, we choose <strong>Create API</strong> option. Let’s call the API PostReaderAPI.</p> 
         <p><img class="alignnone size-full wp-image-338" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/23/text-to-speech_20.gif" alt="" width="1000" height="338"></p> 
         <p>After our API is created, we create two HTTP methods (from <strong>Actions</strong> button and then <strong>Create Method</strong>). The POST method invokes the <code class="lang-tt">PostReader_NewPost</code>&nbsp;Lambda function.&nbsp; For the <code class="lang-tt">GET</code>&nbsp;method, our API invokes the <code class="lang-tt">PostReader_GetPost</code>&nbsp;Lambda function.</p> 
         <p><img class="alignnone size-full wp-image-339" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/23/text-to-speech_21.gif" alt="" width="1000" height="465"></p> 
         <p><img class="alignnone size-full wp-image-340" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/23/text-to-speech_22.gif" alt="" width="1000" height="486"></p> 
         <p>The last method is <code class="lang-tt">CORS</code>&nbsp;<em>(</em><em>cross-origin resource sharing). </em>This method enables invoking the API from a website with a different hostname.</p> 
         <p><img class="alignnone size-full wp-image-341" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/23/text-to-speech_23.gif" alt="" width="500" height="521"></p> 
         <p>Now, configure the <code class="lang-tt">GET </code>method for a query parameter, <code class="lang-tt">postId</code>, which provides information about the id of the post that should be returned. To do this, click on &nbsp;<code class="lang-tt">GET</code> method, modify the configuration of the <code class="lang-tt">Method Request, </code>and add information about the new URL query string parameter (the second screen shot below). Just provide the name of the parameter,&nbsp; in our case <code class="lang-tt">postId</code>.</p> 
         <p><img class="alignnone size-full wp-image-343" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/24/text-to-speech_24.gif" alt="" width="1000" height="475"></p> 
         <p><img class="alignnone size-full wp-image-344" src="https://d2908q01vomqb2.cloudfront.net/f1f836cb4ea6efb2a0b1b99f41ad8b103eff4b59/2017/03/24/text-to-speech_25.gif" alt="" width="1000" height="361"></p>
