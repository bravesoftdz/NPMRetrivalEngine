      <section class="step-container">
       <h2 id="intro" class="step-title intro-title">Introduction: TJBot - Build a Talking Robot With Watson Conversation</h2>
       <div class="photoset">
        <img alt="Picture of TJBot - Build a Talking Robot With Watson Conversation" src="https://cdn.instructables.com/FG9/4SAB/IUYD1976/FG94SABIUYD1976.MEDIUM.jpg">
       </div>
       <div class="step-body">
        <p>This instructable guides you through connecting a Raspberry Pi to Watson conversation services and making a talking robot. You will use (1) <a href="https://www.ibm.com/watson/developercloud/speech-to-text.html">Watson Speech to Text</a> to convert your voice to text, (2) <a href="http://www.ibm.com/watson/developercloud/conversation.html">Watson Conversation</a> to process the text and calculate a response, and (3) <a href="http://www.ibm.com/watson/developercloud/text-to-speech.html">Watson Text to Speech</a> to talk the response back.</p>
        <img alt="" class="lazy-img-onload" data-src="https://www.instructables.com/files/deriv/F9O/KIYI/J0MORWWX/F9OKIYIJ0MORWWX.LARGE.jpg" src="/intl_static/img/pixel.png" style="width: 700.0px;">

      <section class="step-container">
       <h2 id="step2" class="step-title">Step 2: Prepare Your Pi</h2>
       <div class="step-body">
        <p>If you have used Raspberry Pi before, install Node.js and go to the next step. Otherwise, follow the instructions below to set up your Pi:</p>
        <p><strong><br><u>Getting Started with Your Pi</u></strong></p>
        <p>Raspberry Pi is similar to a full computer, which means you need a monitor, mouse, and keyboard for it. If you have a TV around, you can connect your Pi to your TV via a HDMI cable. In most of the Pi kits, the SD card is already preloaded with an image of the Raspberry Pi Operating System. You need to put the SD card in the Pi, turn the Pi ON and follow the instructions on screen to complete the installation of the operating system. If you have problems setting up your Pi, you can troubleshoot <a href="https://www.raspberrypi.org/learning/hardware-guide/" rel="nofollow">here</a>.</p>
        <p><a href="https://www.raspberrypi.org/documentation/configuration/wireless/pixel-desktop.md">Here is an easy way to get your Pi connected to wifi.</a></p>
        <p><br><strong><u>Install Packages</u></strong></p>
        <p>Open a terminal application on the Pi and execute the following commands to install the latest version of Node.js and npm (Node Package Manager). You need these packages later to run your code. </p>
        <pre>sudo apt-get update

sudo apt-get dist-upgrade

curl -sL https://deb.nodesource.com/setup_6.x | sudo -E bash -

sudo apt-get install -y nodejs

sudo apt-get install alsa-base alsa-utils libasound2-dev
</pre>
       </div>
      </section>
      <section class="step-container">
       <h2 id="step3" class="step-title">Step 3: Assemble TJBot</h2>
       <div class="step-body">
        <p>Here is a quick video of how to fold the kit. The detailed instructions are available on another instructable: <a href="https://www.instructables.com/id/Build-TJ-Bot-Out-of-Cardboard/">Build TJBot out of Cardboard</a></p>
        <iframe allowfullscreen frameborder="0" height="315" src="https://www.youtube.com/embed/bLt3Cf2Ui3o" width="560"></iframe>
        <p>Once your TJBot is ready, plug in your USB microphone and the speaker.</p>
        <p><img alt="" class="lazy-img-onload" data-src="https://www.instructables.com/files/deriv/FPJ/5TRU/IVA4GT54/FPJ5TRUIVA4GT54.LARGE.jpg" src="/intl_static/img/pixel.png"></p>
        <p>Depending on which audio output source you are using with your Pi (HDMI, 3.5mm audio jack, Bluetooth, USB speaker), you may need to set the audio config.</p>
        <p><strong><u>HDMI/ 3.5mm Audio Jack</u></strong></p>
        <p>If you use HDMI or 3.5mm audio jack, you may need to set the audio config. To do this, go to the terminal and open up raspi-config.</p>
        <pre> sudo raspi-config  </pre>
        <p>This will open up the Raspberry Pi configuration screen:</p>
        <p><img alt="" class="lazy-img-onload" data-src="https://www.instructables.com/files/deriv/FSH/XNCL/IVA4GT7M/FSHXNCLIVA4GT7M.LARGE.jpg" src="/intl_static/img/pixel.png"></p>
        <p>Select "Advanced Options" and press Enter, then select "Audio" and press Enter. Choose the correct channel for the output audio. If you have connected an external speaker to the audio jack, you should select 3.5mm jack.</p>
        <p><img alt="" class="lazy-img-onload" data-src="https://www.instructables.com/files/deriv/FRJ/BWPT/IVA4GT7O/FRJBWPTIVA4GT7O.LARGE.jpg" src="/intl_static/img/pixel.png"></p>
        <p><strong><u>USB Speaker</u></strong></p>
        <p>If you have a USB audio, you need to update your /usr/share/alsa/alsa.config to set the USB audio as the default device. Begin with running the following command to make sure your USB is connected and listed there.</p>
        <pre> lsusb </pre>
        <p>Next is to detect the card number of your USB audio.</p>
        <pre>aplay -l </pre>
        <p>Take a note of the card number associated with your USB Audio. Then go to the alsa.config file to set it as default.</p>
        <pre>sudo nano /usr/share/alsa/alsa.conf</pre>
        <p>Look for </p>
        <pre>defaults.ctl.card 0
defaults.pcm.card 0</pre>
        <p>and update the card number (0 here) to the card number of your USB audio.</p>
        <p>Different versions of Raspberry Pi OS may need a different setup. If you have problem with your USB setup, check out <a href="https://learn.adafruit.com/usb-audio-cards-with-a-raspberry-pi">this guide</a> to troubleshoot.</p>
        <p><u><strong>Bluetooth Speaker</strong></u></p>
        <p>If you plan to use a Bluetooth speaker, you may need to install the Raspberry Pi Bluetooth software. Here is a good tutorial to set it up: <a href="https://www.raspberrypi.org/magpi/bluetooth-audio-raspberry-pi-3/">Play audio on a bluetooth speaker with Raspberry Pi 3</a></p>
       </div>

      <section class="step-container">
       <h2 id="step5" class="step-title">Step 5: Update Your Credentials</h2>
       <div class="step-body">
        <p>In this step, we help you get API access to three Watson conversation services.</p>
        <p><img alt="" class="lazy-img-onload" data-src="https://www.instructables.com/files/deriv/FCW/7ZPJ/IVA4GTLS/FCW7ZPJIVA4GTLS.LARGE.jpg" src="/intl_static/img/pixel.png"></p>
        <p>You will use three types of Watson services in this recipe: (1) <a href="http://www.ibm.com/watson/developercloud/speech-to-text.html">Watson Speech to Text</a>, (2) <a href="http://www.ibm.com/watson/developercloud/conversation.html">Watson Conversation</a>, (3) <a href="http://www.ibm.com/watson/developercloud/text-to-speech.html">Watson Text to Speech</a>. You need to copy your credentials for all these services.<br> Let's start <a href="https://console.ng.bluemix.net/catalog/services/conversation">with creating a Watson Conversation instance</a> (If you don't have a Bluemix account, you need to create one).</p>
        <p>You can leave the default values and select 'Create'.</p>
        <p><img alt="" class="lazy-img-onload" data-src="https://cdn.instructables.com/FD5/C50R/J6FDYHUX/FD5C50RJ6FDYHUX.LARGE.jpg" src="/intl_static/img/pixel.png" style="width: 642.0px;"></p>
        <p>Now we need to copy your credentials from Bluemix. Click on 'Sevice Credentials' on the left menu and copy your credentials into clipboard.</p>
        <p><br><img alt="" class="lazy-img-onload" data-src="https://cdn.instructables.com/FYR/UVBQ/J6FDYHXT/FYRUVBQJ6FDYHXT.LARGE.jpg" src="/intl_static/img/pixel.png" style="width: 638.0px;"></p>
        <p>Now make a copy of the default config file.</p>
        <pre>cp config.default.js config.js</pre>
        <p>Open <strong>config.js</strong> file using your favorite text editor and update it with your username and password.</p>
        <p><img alt="" class="lazy-img-onload" data-src="https://www.instructables.com/files/deriv/FQ0/US1R/J0MORHVT/FQ0US1RJ0MORHVT.LARGE.jpg" src="/intl_static/img/pixel.png" style="width: 400.0px;"></p>
        <p>The Conversation service needs one more ID to configure. It's called the Workspace ID (exports.conversationWorkspaceId). Don't worry about it for now. In the next step, we will walk you through creating a conversation workspace.</p>

      <section class="step-container">
       <h2 id="step6" class="step-title">Step 6: Create a Conversation Flow</h2>
       <div class="step-body">
        <p>You need to train your robot with what to say and when to say it. For that, we use <a href="https://www.ibm.com/watson/developercloud/conversation.html">Watson Conversation</a>. Open a browser and go to <a href="https://www.ibmwatsonconversation.com/instances">IBM Watson Conversation</a>.</p>
        <p>Log in, and in the the top left menu, select the name of your conversation service. Click on 'Import' icon to import the sample conversation.</p>
        <p><img alt="" class="lazy-img-onload" data-src="https://www.instructables.com/files/deriv/FKJ/MYE9/J6FDYLR9/FKJMYE9J6FDYLR9.LARGE.jpg" src="/intl_static/img/pixel.png"></p>
        <p>In the dialog that opens, click 'Choose a file' and select 'workspace-sample.json' from the 'tjbot/recipes/conversation' folder. Make sure 'Everything (Intents, Entities, and Dialog)' is selected.</p>
        <p><img alt="" class="lazy-img-onload" data-src="https://www.instructables.com/files/deriv/FGO/OZMS/J0MORJ20/FGOOZMSJ0MORJ20.LARGE.jpg" src="/intl_static/img/pixel.png" style="width: 600.0px;"></p>
        <p>After the workspace has been imported, click the dotted menu and select 'View details'. Copy your Workspace ID as shown.</p>
        <p><img alt="" class="lazy-img-onload" data-src="https://www.instructables.com/files/deriv/FGU/QQ43/J6FDYM7U/FGUQQ43J6FDYM7U.LARGE.jpg" src="/intl_static/img/pixel.png" style="width: 750.0px;"></p>
        <p><img alt="" class="lazy-img-onload" data-src="https://www.instructables.com/files/deriv/F1F/ZBBQ/J6FDYMFQ/F1FZBBQJ6FDYMFQ.LARGE.jpg" src="/intl_static/img/pixel.png" style="width: 800.0px;"></p>
        <p>Next, update your config.js file with the workspace ID.</p>
        <p><img alt="" class="lazy-img-onload" data-src="https://www.instructables.com/files/deriv/FZ5/NVBD/J0MORU43/FZ5NVBDJ0MORU43.LARGE.jpg" src="/intl_static/img/pixel.png" style="width: 800.0px;"></p>
        <p>Congratulations, your conversation is ready!</p>
       </div>

      <section class="step-container">
       <h2 id="step7" class="step-title">Step 7: Run the Code</h2>
       <div class="step-body">
        <p>Now, you are ready to talk to your TJBot! </p>
        <p>Open a terminal and execute the following command:</p>
        <pre>sudo node conversation.js</pre>
        <p>Watson conversation uses intents to label the purpose of a sentence. For example when you ask TJBot "Please introduce yourself", the intent is to make an introduction. You can add your own new intents in the Conversation editor, but for now, we have started you off with two intents:</p>
        <ul>
         <li><strong>Introduction</strong>. You can say phrases such as "Watson, please introduce yourself", "Watson, who are you", and "Watson, can you introduce yourself" </li>
         <li><strong>Joke</strong>. You can ask "Watson, please tell me a joke" or "Watson, I would like to hear a joke".</li>
        </ul>
        <p>If you would like to create your own intents, we recommend following <a href="https://www.ibm.com/blogs/watson/2017/03/bot-yourself/">this guide</a>.</p>
        <p>An attention word is used so TJBot knows you are talking to him. The default attention word is 'Watson', but you can change it in <strong>conversation.js</strong> as follows. Update <strong>conversation.js</strong> and change tjConfig to add the 'robot' section:</p>
        <pre>// turn on debug logging to the console
var tjConfig = {
    verboseLogging: true,
    robot: {
        name: 'Watson',
        gender: 'male'
    }
};
</pre>
        <p>You can change the 'name' to whatever you would like to call your TJBot. In addition, if you change the gender to 'female', TJBot will use a female voice to speak to you!</p>
        <p>Enjoy! Don't forget to share a picture/video of your setup #TJBot! :-)</p>
        <p><font color="red"><strong>Troubleshoot</strong></font></p>
        <p>If you see TJBot's response on the terminal but don't hear TJBot talking, there is a good chance that one of these two things has happened: (1) The audio output is directed to a wrong channel (you can fix it from raspi-config), (2) your sound modules are blocked. In that case, go to /etc/modprobe.d/ and remove <strong>blacklist-rgb-led.conf</strong></p>
        <p> Then run the following command:</p>
        <pre>sudo update-initramfs -u</pre>
        <p><strong>Reboot </strong>and confirm the "snd" modules are running by executing the command "lsmod". This should solve the problem.</p>
        <pre>lsmod</pre>
