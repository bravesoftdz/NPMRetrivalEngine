        <h2><a href="https://blog.getfirebug.com/2016/10/11/firebug-2-0-18/" rel="bookmark" title="Permanent Link to Firebug 2.0.18">Firebug 2.0.18</a></h2> 
        <div class="postDate">
         11 October 2016 | 11:36 am by Jan Honza Odvarko 
        </div> 
        <div class="entry"> 
         <p>The Firebug team released <a title="Firebug on addons.mozilla.org" href="https://addons.mozilla.org/firefox/addon/firebug/">Firebug 2.0.18</a>. This is a maintenance release ensuring compatibility with latest Firefox releases.</p> 
         <p>&nbsp;</p> 
         <p><em>The beta channel on AMO is also updated</em>.</p> 
         <p>&nbsp;<br> <strong>Firebug 2.0.18</strong> is compatible with <strong>Firefox 30 – 52</strong></p> 
         <p>&nbsp;<br> <strong>Firebug 2.0.18</strong> fixes issue <a href="https://github.com/firebug/firebug/issues/8030">8030</a>. The extension is marked as multi-process (e10s) compatible so, it isn’t blacklisted and users can continue installing it. This way Firebug can help users to easily migrate into built-in Firefox developer tools – in case it’s running in multi-process enabled browser. When opened in a multi-process Firefox, Firebug’s tools cannot be used. Firebug will then only assist the user with migration to Firefox’s built-in tools.</p> 
         <p>&nbsp;</p> 
         <p>You might also want to read about <a href="https://blog.getfirebug.com/2016/06/07/unifying-firebug-firefox-devtools/">Unifying Firebug &amp; Firefox DevTools</a>.</p> 
         <p>&nbsp;</p> 
         <p>Please post feedback in the <a title="Newsgroup post for Firebug 2.0.18" href="https://groups.google.com/forum/#!topic/firebug/t3quNcR3D7s">newsgroup</a>, thanks.</p> 
         <p>Jan ‘Honza’ Odvarko</p>
