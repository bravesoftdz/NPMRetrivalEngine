        <div class="conf-macro output-block" data-hasbody="true" data-macro-name="sp-pagelayout"> 
         <p>This guide shows you how to use Bitbucket Pipelines for building and testing a Node.js software project in a Docker container.</p> 
         <p>Most of the configuration happens in the&nbsp;<code>bitbucket-pipelines.yml</code> file that Pipelines uses to define the build.</p> 
         <p></p> 
         <div class="toc-macro client-side-toc-macro  conf-macro output-block" data-headerelements="H2,H3" data-hasbody="false" data-macro-name="toc"> 
         </div>&nbsp; 
         <p></p> 
         <h2 id="Javascript(Node.js)withBitbucketPipelines-SpecifyyourNode.jsversionwithDocker">Specify your Node.js version with Docker</h2> 
         <p>Bitbucket Pipelines runs all your builds in Docker containers using an image that you provide at the beginning of your configuration file. You can easily use Node.js with Bitbucket Pipelines by using one of the&nbsp;<a rel="nofollow" class="external-link" href="https://hub.docker.com/_/node/">official Node.js Docker images</a>&nbsp;on Docker Hub.&nbsp;</p> 
         <p>For instance, you can use Node.js 6.9.4 by specifying it at the beginning of your bitbucket-pipelines.yml file.</p> 
         <div class="code-block conf-macro output-block" data-hasbody="true" data-macro-name="sp-macrooverride-plaintextbody-block"> 
          <pre><code class="language-none">image: node:6.9.4
pipelines:
  default:
    - step:
        script:
          - node -v</code></pre> 
         </div> 
         <p>If you wanted to use a different version of Node.js you simply need to change the tag of the Node.js Docker image. The example below would start a container with Node 4.7.2:</p> 
         <div class="code-block conf-macro output-block" data-hasbody="true" data-macro-name="sp-macrooverride-plaintextbody-block"> 
          <pre><code class="language-none">image: node:4.7.2</code></pre> 
         </div> 
         <p>To find a list of all supported Node.js versions and corresponding image tags, please refer to&nbsp;<a rel="nofollow" class="external-link" href="https://hub.docker.com/_/node/">https://hub.docker.com/_/node/</a>.</p> 
         <p><span>You can check your&nbsp;bitbucket-pipelines.yml file&nbsp;with our&nbsp;</span><a rel="nofollow" class="external-link" href="https://bitbucket-pipelines.atlassian.io/validator">online validator</a><span>.</span></p> 
         <h2 id="Javascript(Node.js)withBitbucketPipelines-Installdependencies">Install dependencies</h2> 
         <h3 id="Javascript(Node.js)withBitbucketPipelines-Managedependencieswithnpm">Manage dependencies with npm</h3> 
         <p>The default Node.js image on Docker Hub comes with npm installed by default. You can use it directly in your script to install specific packages locally or globally.</p> 
         <p>If you are using a package.json file, you can simply run&nbsp;<code>npm install</code>&nbsp;at the beginning of your script to install all the required dependencies:</p> 
         <div class="code-block conf-macro output-block" data-hasbody="true" data-macro-name="sp-macrooverride-plaintextbody-block"> 
          <pre><code class="language-none">image: node:6.9.4
pipelines:
  default:
    - step:
        script:
          - npm install</code></pre> 
         </div> 
         <h3 id="Javascript(Node.js)withBitbucketPipelines-ManagedependencieswithYarn">Manage dependencies with Yarn</h3> 
         <p>You will have to install Yarn as part of your build script to be able to use it. You can simply copy and paste the instructions in this example:</p> 
         <div class="code-block conf-macro output-block" data-hasbody="true" data-macro-name="sp-macrooverride-plaintextbody-block"> 
          <pre><code class="language-none">image: node:6.9.4
pipelines:
  default:
    - step:
        script:
          - curl -o- -L https://yarnpkg.com/install.sh | bash -s -- --version 0.18.1
          - export PATH=$HOME/.yarn/bin:$PATH
          - yarn</code></pre> 
         </div> 
         <h3 id="Javascript(Node.js)withBitbucketPipelines-Installdependenciesglobally(Gulp,Grunt,AngularCLI...)">Install dependencies globally (Gulp, Grunt, Angular CLI...)</h3> 
         <p><span class="inline-comment-marker" data-ref="d9991e9c-86ab-405f-8d5f-c6e7e94ad76d">Node packages can also be installed globally. For instance, if your pipeline depends on a task runner</span> like Gulp or Grunt, you will need to install them globally. You can use the <code>npm install -g</code><em>&nbsp;</em>command if you're using npm as your package manager:</p> 
         <div class="code-block conf-macro output-block" data-hasbody="true" data-macro-name="sp-macrooverride-plaintextbody-block"> 
          <pre><code class="language-none">npm install -g grunt-cli
npm install
grunt</code></pre> 
         </div> 
         <p>or use&nbsp;<code>yarn global add</code>&nbsp;if you're using Yarn.</p> 
         <div class="code-block conf-macro output-block" data-hasbody="true" data-macro-name="sp-macrooverride-plaintextbody-block"> 
          <pre><code class="language-none">yarn global add grunt-cli
yarn
grunt</code></pre> 
         </div> 
         <h3 id="Javascript(Node.js)withBitbucketPipelines-Downloadandpublishmodulestoaprivateregistry">Download and publish modules to a private registry</h3> 
         <p>If you are using private modules from either the public npm registry or your own private registry you can simply check in a .npmrc file at the root of your repository with the using environment variables. To avoid conflict with the local ~/.npmrc file during development you can simply name it .npmrc_config&nbsp;and rename it as part of the pipeline script.</p> 
         <div class="code-block conf-macro output-block" data-hasbody="true" data-macro-name="sp-macrooverride-plaintextbody-block"> 
          <div class="code-block-title">
            .npmrc_config 
          </div> 
          <pre><code class="language-none">//registry.npmjs.org/:_authToken=${NPM_TOKEN}
registry=https://registry.npmjs.org</code></pre> 
         </div> 
         <p class="p1"><span class="s1">If you are using a private npm registry you just need to replace <code><span class="s2">registry.npmjs.org</span></code> with the address of your registry.</span></p> 
         <p class="auto-cursor-target">Then <a rel="nofollow" href="https://confluence.atlassian.com/bitbucket/environment-variables-in-bitbucket-pipelines-794502608.html">configure the following environment variable</a>&nbsp;in Bitbucket Pipelines:</p> 
         <ul> 
          <li>NPM_TOKEN: <span>This is the authentication token to your registry. You can find it in your local&nbsp;~/.npmrc, after you login to your registry.</span></li> 
         </ul> 
         <p>Your bitbucket-pipelines.yml script will then look like the following to install your private dependencies.</p> 
         <div class="code-block conf-macro output-block" data-hasbody="true" data-macro-name="sp-macrooverride-plaintextbody-block"> 
          <div class="code-block-title">
            bitbucket-pipelines.yml 
          </div> 
          <pre><code class="language-none">image: node:6.9.4
pipelines:
  default:
    - step:
        script:
          - mv .npmrc_config .npmrc
          - npm install</code></pre> 
         </div> 
         <h2 id="Javascript(Node.js)withBitbucketPipelines-Testing">Testing</h2> 
         <p>You simply need to add to your bitbucket-pipelines.yml file the same commands that you would run locally to test your application. For instance, if you are running <code>npm test</code>&nbsp;locally to run your test suite the following example should work for you.</p> 
         <div class="code-block conf-macro output-block" data-hasbody="true" data-macro-name="sp-macrooverride-plaintextbody-block"> 
          <pre><code class="language-none">image: node:6.9.4
pipelines:
  default:
    - step:
        script:
          - npm install
          - npm test</code></pre> 
         </div>
