         <p><a href="https://howlerjs.com"><img src="https://camo.githubusercontent.com/98b571ff7230cf5d95e1d41d5d8ea0620c20a79d/68747470733a2f2f73332e616d617a6f6e6177732e636f6d2f686f776c65722e6a732f686f776c65722d6c6f676f2e706e67" alt="howler.js" title="howler.js" data-canonical-src="https://s3.amazonaws.com/howler.js/howler-logo.png" style="max-width:100%;"></a></p> 
         <h1><a id="user-content-description" class="anchor" href="#description" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Description</h1> 
         <p><a href="https://howlerjs.com">howler.js</a> is an audio library for the modern web. It defaults to <a href="http://webaudio.github.io/web-audio-api/">Web Audio API</a> and falls back to <a href="https://html.spec.whatwg.org/multipage/embedded-content.html#the-audio-element">HTML5 Audio</a>. This makes working with audio in JavaScript easy and reliable across all platforms.</p> 
         <p>Additional information, live demos and a user showcase are available at <a href="https://howlerjs.com">howlerjs.com</a>.</p> 
         <h3><a id="user-content-features" class="anchor" href="#features" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Features</h3> 
         <ul> 
          <li>Single API for all audio needs</li> 
          <li>Defaults to Web Audio API and falls back to HTML5 Audio</li> 
          <li>Handles edge cases and bugs across environments</li> 
          <li>Supports all codecs for full cross-browser support</li> 
          <li>Automatic caching for improved performance</li> 
          <li>Control sounds individually, in groups or globally</li> 
          <li>Playback of multiple sounds at once</li> 
          <li>Easy sound sprite definition and playback</li> 
          <li>Full control for fading, rate, seek, volume, etc.</li> 
          <li>Easily add 3D spatial sound or stereo panning</li> 
          <li>Modular - use what you want and easy to extend</li> 
          <li>No outside dependencies, just pure JavaScript</li> 
          <li>As light as 7kb gzipped</li> 
         </ul> 
         <h3><a id="user-content-browser-compatibility" class="anchor" href="#browser-compatibility" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">

           </svg></a>Format Recommendations</h3> 
         <p>Howler.js supports a wide array of audio codecs that have varying browser support ("mp3", "opus", "ogg", "wav", "aac", "m4a", "mp4", "webm", ...), but if you want full browser coverage you still need to use at least two of them. If your goal is to have the best balance of small filesize and high quality, based on extensive production testing, your best bet is to default to <code>webm</code> and fallback to <code>mp3</code>. <code>webm</code> has nearly full browser coverage with a great combination of compression and quality. You'll need the <code>mp3</code> fallback for Internet Explorer.</p> 
         <p>It is important to remember that howler.js selects the first compatible sound from your array of sources. So if you want <code>webm</code> to be used before <code>mp3</code>, you need to put the sources in that order.</p> 
         <p>If you want your <code>webm</code> files to be seekable in Firefox, be sure to encode them with the cues element. One way to do this is by using the <code>dash</code> flag in <a href="https://www.ffmpeg.org/">ffmpeg</a>:</p> 
         <pre><code>ffmpeg -i sound1.wav -dash 1 sound1.webm
</code></pre>
