         <h1 class="content__title">Known Issues with the Command-Line Interface and the Package for Sublime Text</h1> 
        </div> 
       </div> 
       <div class="content__article"> 
        <article class="article article--code"> 
         <blockquote> 
          <p><strong>Product version:</strong> AppBuilder 3.7.5</p> 
          <p><strong>Released:</strong> August 23, 2017</p> 
         </blockquote> 
         <p>Check frequently for updates and additions to this list.</p> 
         <p>If you identify an issue that is not on the list, use our <a href="http://www.telerik.com/forums/appbuilder/report-a-bug" target="_blank">forums</a> to report it. You can also browse the community to check for other issues that might not have made it into this list.</p> 
         <p>The command-line interface and package for Sublime Text are known to have the following issues.</p> 
         <h3 id="appbuilder-37x-introduces-the-following-issue-with-the-command-line-interface"><a href="#appbuilder-37x-introduces-the-following-issue-with-the-command-line-interface">AppBuilder 3.7.x introduces the following issue with the command-line interface.</a></h3> 
         <ul> 
          <li><p>In the command-line interface, when you downgrade your app to target Apache Cordova 4.0.0, all core plugins are enabled.<br><strong>Workaround:</strong> Disable any unused core plugins.</p></li> 
          <li><p>In the command-line interface, the Telerik Platform Enterprise subscriptions are shown as Business subscriptions.<br><strong>Workaround:</strong> None.</p></li> 
         </ul> 
         <h3 id="the-command-line-interface-and-package-for-sublime-text-are-known-to-have-the-following-issues-from-earlier-releases"><a href="#the-command-line-interface-and-package-for-sublime-text-are-known-to-have-the-following-issues-from-earlier-releases">The command-line interface and package for Sublime Text are known to have the following issues from earlier releases.</a></h3> 
         <ul> 
          <li><p>The device simulator does not persist your configuration settings across sessions.<br><strong>Workaround:</strong> None.</p></li> 
          <li> <p>When you run <code>$ appbuilder fetch</code> or <code>$ appbuilder find</code>, you might receive the following error message: <code>CALL_AND_RETRY_LAST Allocation failed - process out of memory</code>.<br>When you are running a 32-bit Node.js, you cannot run <code>fetch</code> and <code>find</code> operations because of memory limitations.<br><strong>Workaround:</strong> Choose between the following workarounds:</p> 
           <ul> 
            <li>(Recommended) Install 64-bit Node.js.</li> 
            <li> <p>Run the <code>fetch</code> and <code>find</code> operations using the following alternative syntax:</p> <pre lang="Shell"><code>node --max-old-space-size=2000 &lt;Path to .../bin/appbuilder.js&gt; &lt;fetch OR find&gt;
</code></pre> </li> 
           </ul> </li> 
          <li><p>If you delete, rename or move the file for the view currently open in the device simulator, your change is not synchronized on save.<br><strong>Workaround:</strong> Reload the simulator.</p></li> 
          <li><p>When using the built-in AppBuilder debug tools or the debug tools in the device simulator, you cannot edit HTML attributes by double-clicking them.<br><strong>Workaround:</strong> None.</p></li> 
          <li><p>You cannot run NativeScript apps in the device simulator.<br><strong>Workaround:</strong> None. Run your app on device or in the native emulators.</p></li> 
          <li><p>If your app contains the Telerik Mobile Testing plugin, when you run it on an Android device in the simulator, the following error message appears in the console: <code>Failed to load resource: the server responded with a status of 404 (Not Found) -- local://simulator/plugins/com.telerik.mobiletesting/src/android/cordova/www/deviceready.js</code>.<br><strong>Workaround:</strong> Ignore the error and continue your work.</p></li> 
          <li><p>On macOS systems with Node.js 0.12.x, <code>appbuilder create &lt;Name&gt;</code> and <code>appbuilder create screenbuilder &lt;Name&gt;</code> operations might not release the console after the app is created.<br><strong>Workaround:</strong> Press <code>Ctrl+C</code> to release the console manually.</p></li> 
          <li><p>On macOS and Linux systems, you might not be able to install the command-line interface or run it using <code>sudo</code>, if you have installed Node.js from <a href="http://nodejs.org/">http://nodejs.org/</a>. You might experience <code>gyp WARN EACCES user&nbsp;"root"&nbsp;does not have permission to access the dev&nbsp;dir&nbsp;"/</code> errors.<br>The following <a href="https://docs.npmjs.com/misc/scripts#user">known behavior</a> of npm causes this issue.<br><strong>Workaround:</strong> Uninstall the command-line interface and install it again using <code>sudo npm install appbuilder -g --unsafe-perm</code>.</p></li> 
          <li><p>Sometimes when you debug your NativeScript app on an iOS device, the app might exit on device and the debug session might end.<br><strong>Workaround:</strong> Restart the app and try again.</p></li> 
          <li><p>You cannot debug your NativeScript apps on Android devices and emulators on macOS systems.<br><strong>Workaround:</strong> Debug your app in the classic Windows desktop client or the extension for Visual Studio.</p></li> 
          <li><p>You cannot debug your NativeScript app on multiple connected devices simultaneously.<br><strong>Workaround:</strong> Debug on one device at a time.</p></li> 
          <li><p>When you attempt to run your app on Windows Phone 8 device, the app becomes unresponsive and needs to close.<br>AppBuilder lets you build an app with the Windows Phone 8.1 SDK. If you deploy such an app on a Windows Phone 8 device, the app cannot run. It immediately becomes unresponsive and needs to close.<br><strong>Workaround:</strong> If you want to test your app on a Windows Phone 8 device, verify that the target SDK is Windows Phone 8.0. For more information, see <a href="/platform/appbuilder/cordova/configuring-your-app/configuring-your-app">Configure Your App</a> and <a href="/platform/appbuilder/cordova/configuring-your-app/wp-properties">Windows Phone Properties</a>.</p></li> 
          <li><p>In Sublime Text 2, you cannot work with apps whose names contain non-ASCII characters.<br><strong>Workaround:</strong> Rename your app to contain only ASCII characters or upgrade to Sublime Text 3. </p></li> 
          <li><p>On Windows systems, when you right-click on the device screen in the device simulator, AppBuilder fires the events attached to the UI element on which you clicked.<br><strong>Workaround:</strong> None.</p></li> 
          <li><p>When you run <code>appbuilder simulate</code>, the command-line interface does not compile your TypeScript automatically.<br><strong>Workaround:</strong> Before running your app in the device simulator, run <code>appbuilder build &lt;Platform&gt;</code> first to trigger the TypeScript compilation.</p></li> 
          <li><p>When you run <code>appbuilder simulate</code>, the command-line interface does not LiveSync changes to your TypeScript automatically.<br><strong>Workaround:</strong> Close the device simulator, run <code>appbuilder build &lt;Platform&gt;</code> to trigger the TypeScript compilation and <code>appbuilder simulate</code> to run the app in the simulator again.</p></li> 
          <li><p>When you debug on iOS 7.x devices, you cannot view and examine the network requests for your app. The <strong>Network</strong> pane remains blank.<br><strong>Workaround:</strong> None.</p></li> 
          <li> <p>The command-line interface does not automatically compile TypeScript to JavaScript when you run the <code>livesync</code> command with the <code>--watch</code> option.<br><strong>Workaround:</strong> You can choose between the following workarounds.</p> 
           <ul> 
            <li>Deploy your app via QR code and LiveSync your changes with the three-finger refresh gesture. Note that you need to run <code>appbuilder livesync cloud</code> before using the three-finger refresh gesture on device.</li> 
            <li>Re-build and re-deploy your app via cable connection to reflect changes to your TypeScript code.</li> 
           </ul> </li> 
          <li> <p>When you run two or more AppBuilder clients simultaneously, multiple issues might occur.</p> 
           <ul> 
            <li>You might lose changes to your app because of synchronization issues.</li> 
            <li>AppBuilder might not recognize connected devices or might not be able to communicate with connected devices properly. You might not be able to deploy your apps on connected device, debug your code or LiveSync changes via cable connection properly.</li> 
            <li>The app might become corrupted.</li> 
           </ul> <p><br><strong>Workaround:</strong> Avoid running multiple AppBuilder clients simultaneously. </p> </li> 
          <li><p>When you debug apps with the device simulator on Windows systems, the command-line interface does not show the contents of <code>index.html</code> in the <strong>Sources</strong> tab.<br><strong>Workaround</strong>: In the main menu bar of the simulator, click <strong>Refresh</strong>.</p></li> 
          <li><p>If you remove the script reference to <code>cordova.js</code> from your home page, the device simulator always returns iPhone as the user agent.<br><strong>Workaround:</strong> Verify that <code>&lt;script src="cordova.js"&gt;&lt;/script&gt;</code> is present in your home page.</p></li> 
          <li><p>The fonts in the device simulator appear pixelated if the DPI size for your display exceeds 100%.<br>When you increase the DPI size for your display, fonts in the device simulator do not adjust to the change and appear pixelated.<br><strong>Workaround</strong>: Decrease the DPI size to 100%. For more information about how to change this setting in Windows, see <a href="http://windows.microsoft.com/en-us/windows/make-screen-text-larger-smaller#make-screen-text-larger-smaller=windows-7" target="_blank">Make the text on your screen larger or smaller</a>.</p></li> 
          <li><p>The device simulator might not show fonts in your application accurately.<br><strong>Workaround:</strong> Run your app on a physical device.</p></li> 
          <li><p>You cannot play audio or video in the simulator using the &lt;audio&gt; or &lt;video&gt; tags.<br><strong>Workaround:</strong> Run your app on a physical device.</p></li> 
         </ul>
