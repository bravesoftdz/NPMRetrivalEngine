         <p>There are already thousands of packages for the Meteor JavaScript framework, so no matter what feature you want to add to your web application, someone’s probably gone ahead and made a package that slickly implements that precise ideas. Convenient, right?</p> 
         <p>But of course, with more choice comes the paradox of choice. It can be overwhelming to figure out which packages might come in handy.</p> 
         <p>Here’s a run-down of what I consider to be the few most helpful:</p> 
         <h2 id="1-iron-router">1. <a href="https://atmospherejs.com/iron/router">Iron Router</a></h2> 
         <p>A question I’ve received a few times from beginning developers is, “How do I create a multi-page application with Meteor?”</p> 
         <p>Because, when someone’s never built a web application before, it’s not immediately intuitive that creating a new page isn’t as simple as creating a new file. What I introduce them to, then, is the wide world of <em>routing</em>.</p> 
         <p>Routing allows developers to make muti-page applications with less code and a flexible project structure. They have many other advantages, most of which become apparent as you build bigger and more complicated applications, but for the moment, there’s two main points to understand:</p> 
         <ol> 
          <li>The vast majority of Meteor applications will use routing in some way.</li> 
          <li>To handle this routing, the Iron Router package is the best option.</li> 
         </ol> 
         <p>Iron Router is everything —&nbsp;friendly, deep, and well-supported —&nbsp;and you can add it to a project with the following command:</p> 
         <pre><code class="language-">meteor add iron:router</code></pre> 
         <p>Once installed, create a route inside a JavaScript file:</p> 
         <pre><code class="language-javascript">this.route('about');</code></pre> 
         <p>Then create a template of the same name:</p> 
         <pre><code class="language-markup">&lt;template name="about"&gt;
    &lt;h1&gt;About&lt;/h1&gt;
&lt;/template&gt;</code></pre> 
         <p>You’ll now be able to visit the <a href="http://localhost:3000/about">http://localhost:3000/about</a> path and see the “about” template.</p> 
         <p>This, however, is a very simple example of routing. For a deeper introduction, <a href="https://learnable.com/hub/play/82">watch this video I made for Learnable</a>.</p> 
         <h2 id="2-collection2">2. <a href="https://atmospherejs.com/aldeed/collection2">Collection2</a></h2> 
         <p>Most Meteor applications will interact with a database in some way. By default though, you’ll have to manually validate the data that users are inserting, editing, and removing from the database.</p> 
         <p>Collection2 helps with this process by extending Meteor’s functionality, allowing it to “provide support for specifying a schema and then validating against that schema when inserting and updating.” For example, you can make it so a “Books” collection has a <code>title</code> field that must be a string, and a <code>lastCheckedOut</code> field that must be a date.</p> 
         <p>Here’s an example schema:</p> 
         <pre><code class="language-javascript">var Schemas = {};

Schemas.Book = new SimpleSchema({
    title: {
        type: String,
        label: "Title",
        max: 200
    },
    author: {
        type: String,
        label: "Author"
    },
    copies: {
        type: Number,
        label: "Number of copies",
        min: 0
    },
    lastCheckedOut: {
        type: Date,
        label: "Last date this book was checked out",
        optional: true
    },
    summary: {
        type: String,
        label: "Brief summary",
        optional: true,
        max: 1000
    }
});</code></pre> 
         <p>To add this package to a project, write the following command:</p> 
         <pre><code class="language-">meteor add aldeed:collection2</code></pre> 
         <p>But to see the full scope of Collection2’s power, make sure to read <a href="https://github.com/aldeed/meteor-collection2/#attaching-a-schema-to-a-collection">the official documentation</a>.</p> 
         <h2 id="3-kadira">3. Kadira</h2> 
         <p>Performance isn’t a sexy part of building web apps, but it’s nevertheless important to think about. You can have the most incredible set of features in the world but, if users are forced to sit and wait around at every turn, you’re really going to test their patience.</p> 
         <p>To deal with this problem, <a href="https://kadira.io">Kadira</a>. It’s a performance monitoring tool, similar to New Relic. You register for an online account, add the package to a Meteor project, and then, from a web-based interface, you’ll gain insight into the various performance bottlenecks and other things to consider.</p> 
         <p>You can add Kadira to your project with the following command:</p> 
         <pre><code class="language-">meteor add meteorhacks:kadira</code></pre> 
         <p>If you have no idea where to start when it comes to performance montioring, I’d suggest checking out <a href="http://bulletproofmeteor.com">Bulletproof Meteor</a> —&nbsp;an interactive course, designed by the creators of Kadira, that dives deep into this precise topic.</p> 
         <h2 id="4-spinner">4. <a href="https://atmospherejs.com/sacha/spin">Spinner</a></h2> 
         <p>Even with all of the performance tweaks in the world, some things will simply take some time to load, and to make this loading process seem less broken, we can use the Spinner package. This package can quickly add one of those classic, spinning loading symbols to an application.</p> 
         <p>To add Spinner to your project, use this command:</p> 
         <pre><code class="language-">meteor add sacha:spin</code></pre> 
         <p>Then use a “spinner” template whenever it needs to be used:</p> 
         <pre><code class="language-markup">{{&gt; spinner}}</code></pre> 
         <p>You can also configure it with a number of options:</p> 
         <pre><code class="language-javascript">Meteor.Spinner.options = {
    lines: 13, // The number of lines to draw
    length: 10, // The length of each line
    width: 5, // The line thickness
    radius: 15, // The radius of the inner circle
    corners: 0.7, // Corner roundness (0..1)
    rotate: 0, // The rotation offset
    direction: 1, // 1: clockwise, -1: counterclockwise
    color: '#fff', // #rgb or #rrggbb
    speed: 1, // Rounds per second
    trail: 60, // Afterglow percentage
    shadow: true, // Whether to render a shadow
    hwaccel: false, // Whether to use hardware acceleration
    className: 'spinner', // The CSS class to assign to the spinner
    zIndex: 2e9, // The z-index (defaults to 2000000000)
    top: 'auto', // Top position relative to parent in px
    left: 'auto' // Left position relative to parent in px
};</code></pre> 
         <p>The easiest way to use the Spinner package though is to <a href="http://stackoverflow.com/a/22778504">integrate it with the Iron Router package</a>.</p> 
         <h2 id="5-moment">5. <a href="https://atmospherejs.com/mrt/moment">Moment</a></h2> 
         <p><a href="http://momentjs.com">Moment.js</a> makes it easy to parse, validate, manipulate, and display dates in JavaScript, and installing the Moment package for Meteor is the quickest way to get started.</p> 
         <p>To install the package, use the following command:</p> 
         <pre><code class="language-">meteor add mrt:moment</code></pre> 
         <p>How you use Moment, of course, will depend on what you’re working on, and what makes it so incredible is how flexible it is. Most applications will make use of time and dates in some fashion and, whenever that’s the case, Moment will feel like a heavenly blessing. I’d suggest checking out <a href="http://momentjs.com/docs/#/parsing/">the documentation</a> for the full scope of what it can do.</p> 
         <h2 id="6-npm">6. <a href="https://atmospherejs.com/meteorhacks/npm">NPM</a></h2> 
         <p>Meteor is built on top of Node, and Node itself has countless packages available through NPM. To use these packages, the simplest method is to install the NPM package:</p> 
         <pre><code class="language-">meteor add meteorhacks:npm</code></pre> 
         <p>A <code>packages.json</code> file will be created within your project’s directory, and this is where you can define what packages you’d like to use in your project:</p> 
         <pre><code class="language-javascript">{
  "redis": "0.8.2",
  "github": "0.1.8"
}</code></pre> 
         <p>Then a <code>npmRequire</code> function can be used to harness the functionality of that package:</p> 
         <pre><code class="language-javascript">var Github = Meteor.npmRequire('github');</code></pre> 
         <p>You’ll need to use this code on the server only (it won’t work on the client), and if you come across a problem, it’s probably something to do with <a href="https://github.com/meteorhacks/npm/#async-utilities">the asyncronous nature of most NPM packages</a>, but this is nevertheless a quick and easy way to get even more “free” functionality for your creations.</p> 
         <h2 id="conclusion">Conclusion</h2> 
         <p>Admittedly, it’s a little disingenuous to say that a small handful of Meteor packages are the “best” since, in reality, the value of a package is determined by its appropriateness in a certain situation. Even so, these options are some of the most widely appropriate packages available, and if you haven’t checked any of them out, then I’d suggest you do exactly that.</p>
