      <p>#AIOM+ (All In One Minify)#</p>
      <h2><a class="anchor" href="#simple-minify-and-parsing-for-css-less-js-and-html"></a>####Simple minify and parsing for CSS, LESS, JS and HTML####</h2>
      <p>AIOM+ (All In One Minify) is a ProcessWire module to easily improve the performance of your website. By a simple function call Stylesheets, LESS and Javascript files can be parsed, minimized and combined into one single file. This reduces the server requests, loading time and minimizes the traffic. In addition, the generated HTML source code can be minimized and all generated files can be loaded over a cookieless domain (domain sharding).</p>
      <hr>
      <p>####Information####</p>
      <ul>
       <li>All paths are relative to the template folder. URLs in css files will be automatically corrected. Nothing needs to be changed.</li>
       <li>If you make changes to the source stylesheet, LESS or javascript files, a new parsed and combined version is created automatically.</li>
       <li>All parameters can be adjusted via the backend.</li>
       <li>During development, you can enable developer mode. Files are parsed and combined, but not minimized and browser caching is prevented.</li>
       <li>You can use the short syntax <code>AIOM</code> or use the full class name <code>AllInOneMinify</code> in your templates.</li>
       <li>The generated files can be delivered via a subdomain (Domain sharding / Cookieless domain)</li>
       <li>LESS files can directly server side generated on the fly, without plugins. AIOM+ has a complete, high-performance PHP ported LESS library of the official LESS processor included!</li>
       <li><strong>NOTE</strong>: There are a few unsupported LESS features: 
        <ul>
         <li>Evaluation of JavaScript expressions within back-ticks (for obvious reasons)</li>
         <li>Definition of custom functions</li>
        </ul></li>
       <li>Conditional loading of files based on a API selector</li>
      </ul>

      <p>##Installation##</p>
      <ol>
       <li>Copy the files for this module to /site/modules/AllInOneMinify/</li>
       <li>In admin: Modules &gt; Check for new modules.</li>
       <li>Install Module "AIOM+ (All In One Minify) for CSS, LESS, JS and HTML".</li>
      </ol>
      <p><strong>Alternative in ProcessWire 2.4</strong></p>
      <ol>
       <li>Login to PW backend and go to Modules</li>
       <li>Click tab "new" and enter Module Class Name: "AllInOneMinify"</li>
       <li>Click "Download and Install"</li>
      </ol>
      <p>##Minimize Stylesheets and parse LESS files##</p>
      <p>Minimization of a single file.</p>
      <div class="github-highlight highlight-text-html-php">
       <pre>&lt;!-- CSS Stylesheet --&gt;
&lt;link rel="stylesheet" type="text/css" href="&lt;?php echo AllInOneMinify::CSS('css/stylesheet.css'); ?&gt;"&gt;
&lt;!-- LESS file --&gt;
&lt;link rel="stylesheet" type="text/css" href="&lt;?php echo AllInOneMinify::CSS('css/stylesheet.less'); ?&gt;"&gt;</pre>
      </div>
      <p>Minimize multiple files into one file. You can even mix stylesheet and LESS files in parsing and combining process!</p>
      <div class="github-highlight highlight-text-html-php">
       <pre>&lt;link rel="stylesheet" href="&lt;?php echo AllInOneMinify::CSS(array('css/file-1.css', 'css/file-2.less', 'css/file-3.css', 'css/file-4.less')); ?&gt;"&gt;</pre>
      </div>
      <p><strong>Tip:</strong> You can also use the short syntax <strong>"AIOM"</strong>. For example, <code>AIOM::CSS()</code>.</p>
      <p>##LESS variables access in multiple files##</p>
      <p>You have a LESS file in which you are defining, for example, all colors and another LESS file that defines the actual layout? Now you need in the layout LESS file access to the variables of the color LESS file? It's easier than you think. Through a simple referencing of source LESS file. For example:</p>
      <div class="github-highlight highlight-text-html-php">
       <pre>&lt;link rel="stylesheet" type="text/css" href="&lt;?php echo AllInOneMinify::CSS('css/color.less'); ?&gt;"&gt;
...
&lt;link rel="stylesheet" type="text/css" href="&lt;?php echo AllInOneMinify::CSS('css/layout.less'); ?&gt;"&gt;</pre>
      </div>
      <p>Example content of <code>color.less</code></p>
      <div class="github-highlight highlight-source-css">
       <pre>@my-color: #ff0000;</pre>
      </div>
      <p>Example content of <code>layout.less</code></p>
      <div class="github-highlight highlight-source-css">
       <pre>@import (reference) "css/color.less";

body {
    background-color: @my-color;
}</pre>
      </div>
      <p>That's all. Pretty, hu? The full documentation of LESS you can find at: <a rel="nofollow" href="http://www.lesscss.org">www.lesscss.org</a></p>
      <p>##Minimize Javascripts##</p>
      <p>Minimization of a single file.</p>
      <div class="github-highlight highlight-text-html-php">
       <pre>&lt;script src="&lt;?php echo AllInOneMinify::JS('js/javascript.js'); ?&gt;"&gt;&lt;/script&gt;</pre>
      </div>
      <p>Minimize multiple files into one file.</p>
      <div class="github-highlight highlight-text-html-php">
       <pre>&lt;script src="&lt;?php echo AllInOneMinify::JS(array('js/file-1.js', 'js/file-2.js', 'js/file-3.js', 'js/file-4.js')); ?&gt;"&gt;&lt;/script&gt;</pre>
      </div>
      <p><strong>Tip:</strong> You can also use the short syntax <strong>"AIOM"</strong>. For example, <code>AIOM::JS()</code>.</p>
      <p>##Conditional loading##</p>
      <p>Since AIOM+ version 3.1.1 javascripts, stylesheets and LESS files can be loaded based on a API selector. Here is an example of conditional loading:</p>
      <div class="github-highlight highlight-text-html-php">
       <pre>&lt;?php $stylesheets  = array('css/reset.css',
                            'css/main.less',
					        array('loadOn'  =&gt; 'id|template=1002|1004|sitemap', 
						          'files'   =&gt; array('css/special.css', 'css/special-theme.less'))); ?&gt;

&lt;link rel="stylesheet" type="text/css" href="&lt;?php echo AllInOneMinify::CSS($stylesheets); ?&gt;" /&gt;</pre>
      </div>
      <p>The same you can do with <code>AIOM::JS()</code>. <code>loadOn</code> must be an <a rel="nofollow" href="http://processwire.com/api/selectors/">ProcessWire API selector</a>.</p>
      <p>##Directory Traversal Filter##</p>
      <p>By default, only files can be included, which are in ProcessWire template folder. If you wish to add files outside that folder, you have to activate the backend "Allow Directory Traversal" option. Then you can jump back in the path. For example:</p>
      <div class="github-highlight highlight-text-html-php">
       <pre>AIOM::CSS('../third-party-packages/package/css/example.css');</pre>
      </div>
      <p><strong>All paths are still automatically corrected!</strong></p>
      <p>##Already minimized files no longer minimized##</p>
      <p>To further enhance the performance and to give you maximum flexibility in the combining process, you now have the option to exclude certain files from the minimization (since version 2.2). All files that have the abbreviation ".min" or "-min" at the end of the file name and before the file extension, are no longer minimized. For example: <code>file-1.js</code> is minimized. <code>file-1-min.js</code> or <code>file-1.min.js</code> is not minimized. The same for CSS. <code>file-1.css</code> is minimized. <code>file-1-min.css</code> or <code>file-1.min.css</code> is not minimized.</p>
      <p>##Exemplary template structure##</p>
      <pre><code>site/
    templates/
        css/
        js/</code></pre>
      <p>##Minimize HTML##</p>
      <p>The generated HTML source code is automatically minimized when rendering. This requires no change to the templates. Conditional Comments, textareas, code tags, etc. are excluded from the minimization.</p>
      <p><strong>NOTE</strong>: AIOM+ removes all whitespaces between two tags. If you explicitly need a whitespace, change the whitespace into an HTML entity: <code>&amp;nbsp;</code>. See (<a rel="nofollow" href="https://github.com/conclurer/ProcessWire-AIOM-All-In-One-Minify/issues/6">#6</a>)</p>
      <p>##Development mode##</p>
      <p>If you are currently in development of the site, caching can be a problem. For this, you can enable the development mode since version 1.1.0 in the Backend (Module &gt; AIOM &gt; Config). The files will be combined, but not minimized and re-generated at each call. In addition, a no-cache GET parameter is appended with a timestamp to prevent the browser caching. For example: <code>css_031ea978b0e6486c828ba444c6297ca5_dev.css?no-cache=1335939007</code></p>
