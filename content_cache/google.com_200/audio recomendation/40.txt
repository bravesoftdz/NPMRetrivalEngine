     <h1 id="html5-audio" sourcefile="dev-guide/html5/audio.md" sourcestartlinenumber="13" sourceendlinenumber="13">HTML5 Audio</h1>

      <div class="content"> 
       <p>You can use the <code>audio</code> element to embed simple sound effects, background audio, or more sophisticated audio engies to your webpage without requiring script or add-on controls.</p> 
       <p>To do more complex processing and syntehsizing of audio in web applications, check out the <a href="../multimedia/web-audio" data-linktype="relative-path">Web Audio API</a>. </p> 
       <h2 id="adding-an-audio-element">Adding an audio element</h2> 
       <p>To add audio to your webpage, you can declare an <a href="https://msdn.microsoft.com/library/hh772923(v=vs.85).aspx" data-linktype="external"><code>audio</code></a> element in your HTML, instantiate a new <code>audio</code> element in JavaScript, or embed an audio stream into your page. </p> 
       <p>The <code>audio</code> element is added directly to your HTML code, using the <a href="https://msdn.microsoft.com/library/ff974762(v=vs.85).aspx" data-linktype="external"><code>src</code></a> property to specify the audio file to play and the <code>controls</code> attribute so that the built-in, player controls are used. If you use the browser's intrinsic player, no additional scripting is required.</p> 
       <pre><code class="lang-html">&lt;audio src="audio/sample.mp3" controls autoplay&gt;&lt;/audio&gt;
</code></pre>
       <p>Because not all browsers support the same audio formats, you can use the <a href="https://msdn.microsoft.com/library/ff975070(v=vs.85).aspx" data-linktype="external"><code>source</code></a> element to specify several file formats to try. The following example shows three formats:</p> 
       <pre><code class="lang-html">&lt;!-- The browser will automatically choose the format it supports. --&gt;
&lt;audio controls="true"&gt;
   &lt;source src="audio/sample.mp3" type="audio/mp3"&gt; 
   &lt;source src="audio/sample.ogg" type="audio/ogg"&gt; 
   &lt;source src="audio/sample.aac" type="audio/mp4"&gt; 
   &lt;!-- If no support at all. --&gt;
   HTML5 audio not supported 
&lt;/audio&gt;
</code></pre>
       <p>Instantiating a new <code>audio</code> element in JavaScript gives you more flexibility and better management of the network flow, as it defers the loading of the audio clip to a specific time during the application lifecycle.</p> 
       <pre><code class="lang-js">var audio = document.createElement("audio");
if (audio != null &amp;&amp; audio.canPlayType &amp;&amp; audio.canPlayType("audio/mpeg"))
{
   audio.src = "audio/sample.mp3";
   audio.play();
}
</code></pre>
       <p>Embedding an audio stream into your page (less recommended) consists of embedding the audio files as data-uri into the page, reducing the number of requests to the server. </p> 
       <pre><code class="lang-html">&lt;audio src="data:audio/mpeg,ID3%02%00%00%00%00%…" autoplay&gt;&lt;/audio&gt;
</code></pre>
       <h2 id="preloading-audio">Preloading audio</h2> 
       <p>Once you have your audio element, you can choose the best preloading strategy. The <a href="https://msdn.microsoft.com/library/hh772923(v=vs.85).aspx" data-linktype="external"><code>audio</code></a> element has a <a href="https://msdn.microsoft.com/library/ff974759(v=vs.85).aspx" data-linktype="external"><code>preload</code></a> property with three possible values:</p> 
       <ul> 
        <li><code>none</code>: The document author does not expect the user to need the media resource, or the server wants to minimize unnecessary traffic.</li> 
        <li><code>metadata</code>: The document author does not expect the user to need the resource. However, if the resource metadata (dimensions, first frame, track list, duration, and so on) is available, using a resource is preferred unless it impacts performance to do so.</li> 
        <li><code>auto</code>: The document author gives the user access to media content, which includes the ability to download the entire resource.</li> 
       </ul> 
       <div class="NOTE">
        <h5>Note</h5>
        <p>When you set the <code>src</code> property of the <code>audio</code> element, the <code>preload</code> property will be set to <code>auto</code>.</p> 
       </div> 
       <h2 id="looping-audio">Looping audio</h2> 
       <p>Using the <a href="https://msdn.microsoft.com/library/hh772923(v=vs.85).aspx" data-linktype="external"><code>audio</code></a> element, you can also loop a sound clip. You can do this using the <a href="https://msdn.microsoft.com/library/ff974753(v=vs.85).aspx" data-linktype="external"><code>loop</code></a> property. This setting will loop your clip forever, or until the user or the application activates the pause() audio control.</p> 
       <pre><code class="lang-html">&lt;audio src="sample.mp3" autoplay loop&gt;&lt;/audio&gt;
</code></pre>
       <p>You can also loop an audio file by calling the <a href="https://msdn.microsoft.com/library/ff975194(v=vs.85).aspx" data-linktype="external"><code>play()</code></a> method when the audio clip ends; doing so will allow you eventually to manage the delay between one loop and the other.</p> 
       <pre><code class="lang-js">var audio = document.createElement("audio");
audio.src = "sample.mp3";
audio.addEventListener('ended', function () {
   // Wait 500 milliseconds before next loop
   setTimeout(function () { audio.play(); }, 500);
}, false);

audio.play(); 
</code></pre>
       <h2 id="fast-forwarding-rewinding-and-restarting-audio">Fast-forwarding, rewinding, and restarting audio</h2> 
       <p>The <a href="https://msdn.microsoft.com/library/hh772923(v=vs.85).aspx" data-linktype="external"><code>audio</code></a> object in HTML5 provides methods, properties, and events that you can use to control playback from JavaScript. While an audio file is playing, the <a href="https://msdn.microsoft.com/library/ff974748(v=vs.85).aspx" data-linktype="external"><code>currentTime</code></a> property tracks where the playback is in the audio clip. By changing the value of <code>currentTime</code>, you can skip forward, backward, or restart the playback. </p> 
       <p>Using a progress bar created with the <a href="https://msdn.microsoft.com/library/ff975062(v=vs.85).aspx" data-linktype="external"><code>canvas</code></a> element, this <a href="https://codepen.io/MicrosoftEdgeDocumentation/pen/wGGGRp" data-linktype="external">example</a> uses three functions to increment the value of <code>currentTime</code> (fast forward), decrement the value of <code>currentTime</code> (rewind), and set the value of <code>currentTime</code> to zero (restart). </p> 
       <p></p>

        <p data-height="300" data-theme-id="23761" data-slug-hash="wGGGRp" data-default-tab="result" data-user="MicrosoftEdgeDocumentation" data-embed-version="2" data-editable="true" class="codepen">See this example by <a href="https://codepen.io/MicrosoftEdgeDocumentation" data-linktype="external">Microsoft Edge Docs</a> on <a href="https://codepen.io/MicrosoftEdgeDocumentation/pen/wGGGRp" data-linktype="external">CodePen</a>.</p>

       <h2 id="supported-audio-file-formats">Supported audio file formats</h2> 
       <p>Below is a list of audio formats supported by Microsoft Edge.</p> 
       <table> 
        <thead> 
         <tr> 
          <th>Media file to serve</th> 
          <th>Extension setting</th> 
          <th>Mime type setting</th> 
         </tr> 
        </thead> 
        <tbody> 
         <tr> 
          <td>Audio mp3</td> 
          <td>mp3</td> 
          <td>audio/mpeg</td> 
         </tr> 
         <tr> 
          <td>Audio mp4</td> 
          <td>m4a</td> 
          <td>audio/mp4</td> 
         </tr> 
         <tr> 
          <td>Audio mp4</td> 
          <td>aac</td> 
          <td>audio/mp4</td> 
         </tr> 
         <tr> 
          <td>Audio WebM</td> 
          <td>webm</td> 
          <td>audio/webm</td> 
         </tr> 
         <tr> 
          <td>Audio WAV</td> 
          <td>wav</td> 
          <td>audio/wav, audio/wave, audio/x-wave, audio/vnd.wave</td> 
         </tr> 
        </tbody> 
       </table> 
       <blockquote>
        <p>WebM audio and video files can be supported by installing the WebM components from <a href="http://go.microsoft.com/fwlink/p/?LinkID=218894" data-linktype="external">The WebM project</a>. </p> 
        <p>Audio WAV support was introduced in Microsoft Edge.</p> 
       </blockquote> 
       <p>To test other browser support of audio file formats, use the <a href="https://msdn.microsoft.com/library/ff975191(v=vs.85).aspx" data-linktype="external"><code>canPlayType</code></a> method. This method returns <code>probably</code>, <code>maybe</code>, or an empty string depending on whether or not the client can play a given media resource type. The example below demonstrates how to use the <code>canPlayType</code> method.</p> 
       <pre><code class="lang-js">function checkAudioCompat() {
   var myAudio = document.createElement('audio');
   var msg = document.getElementById("display");

   msg.innerHTML = "";

   if (myAudio.canPlayType) {
       // CanPlayType returns maybe, probably, or an empty string.
       var playMsg = myAudio.canPlayType('audio/mpeg');
       if ( "" != playMsg) {
           msg.innerHTML += "mp3 is " + playMsg + " supported";
       }
   }
   else {
       // Type is not supported               
   }
}
</code></pre>
       <h2 id="dolby-audio">Dolby Audio</h2> 
       <p>Microsoft Edge includes support for the Dolby Audio format in HTML5 media, <a href="../multimedia/media-source-extensions" data-linktype="relative-path">Media Source Extensions (MSE)</a>, and <a href="../multimedia/encrypted-media-extensions" data-linktype="relative-path">Encrypted Media Extensions (EME)</a>, providing rich, multi-channel audio support that works well with AVC/H.264 video and HLS and DASH streaming. You can feature detect support for Dolby Audio formats using <a href="https://msdn.microsoft.com/en-us/library/ff975191(v=vs.85).aspx" data-linktype="external"><code>canPlayType</code></a> (HTML5 media) and <code>isTypeSupported</code> (MSE)/(EME) with the following strings:</p> 
       <p><strong>Dolby Digital Plus (Enhanced AC-3)</strong>: <code>'audio/mp4; codecs="ec-3"'</code> <strong>Digital Plus (AC-3)</strong>: <code>'audio/mp4; codecs="ac-3"'</code> </p> 
       <p>For more info and usage guidance, see <a href="http://go.microsoft.com/fwlink/p/?LinkID=615479" data-linktype="external">Announcing Dolby Audio for high performance audio in Microsoft Edge</a>.</p>
