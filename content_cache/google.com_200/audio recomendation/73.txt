   <h2 id="mega-boilerplate">Mega Boilerplate</h2> 
   <p>Mega Boilerplate is a starter project generator that focuses on simplicity and ease of use, while providing you with flexibility of choices. It was heavily inspired by the Hackathon Starter, but unlike it, you can customize any part of your application stack — from web framework and database to CSS preprocessor and client-side JavaScript framework. Currently, generators are primarily limited to Node.js web apps, but I am planning to expand support for other platforms and languages in the near future.</p>

   <h2 id="showtrackr">ShowTrackr</h2> 
   <p>This TV Show Tracker was built for my very first talk at the <a href="http://meetup.com/nodejs/">Node.js NYC Meetup</a>. This project uses <a href="http://expressjs.com">Express</a> REST API on the back-end and <a href="http://angularjs.org">Angular.js</a> on the front-end. Both user accounts and shows are stored in <a href="http://mongodb.org">MongoDB</a>, while authentication sessions are stored in <a href="http://redislabs.com">Redis</a>. The UI is created using <a href="http://getbootstrap.com/css/#sass">Bootstrap Sass</a>. The JavaScript and CSS concatenation &amp; minification is done via <a href="http://gulpjs.com">Gulp</a>. Authentication is handled by <a href="http://passportjs.org/">Passport.js</a>. Email notifications are sent out two hours before the show starts via <a href="http://sendgrid.com">SendGrid</a>.</p> 
   <p><i class="fa fa-github"></i><a href="https://github.com/sahat/tvshow-tracker/" class="btn">Source</a></p>

   <h2 id="cloudbucket-ccny-capstone-project">CloudBucket (CCNY Capstone Project)</h2> 
   <p>CloudBucket is the online cloud storage system with a semantic search capability. Whenever you upload a file to CloudBucket, it is tagged automatically based on its file contents. For instance, a music file will receive a covert art, lyrics, artist information, similar artists and last.fm tags that describe that particular song. You can then, for example, search for a keyword that is contained in the lyrics of that song and it will return this file in the search results. Typical search systems return results that match a keyword in the filename only, not based on its file contents.</p> 
   <p>It is built using the following technologies: Node.js, Python, MongoDB, AWS S3, Grunt.js, jQuery, Bower, Google OAuth 2.0, Google Books API, SkyBiometry API, Last.fm API, Musixmatch API and other libraries.</p> 
   <p><i class="fa fa-play-circle-o"></i><a href="http://cloudbucket-sahat.rhcloud.com" class="btn">Demo</a></p>

   <h2 id="new-eden-faces">New Eden Faces</h2> 
   <p>The New Eden Faces lets you vote between two randomly selected <a href="http://eveonline.com">EVE Online</a> characters of the same gender. The goal? To see, who are the best looking characters of EVE Online, of course! Other features include browsing all 7000+ characters using infinite pagination, browse Top 100 sorted by gender, race and bloodline. This was my first <a href="http://backbonejs.org">Backbone.js</a> project. Real-time <strong>online users counter</strong> was added later using <a href="http://socket.io">Socket.IO</a>. All that client-side code in a single <strong>app.js</strong> with over 1200 lines of code was later refactored to use <a href="http://requirejs.org/">RequireJS</a> module loader.</p>
