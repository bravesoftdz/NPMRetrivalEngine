         <h2><a href="#about" aria-hidden="true" class="anchor" id="user-content-about">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>About</h2> 
         <p>discord.js is a powerful <a href="https://nodejs.org">node.js</a> module that allows you to interact with the <a href="https://discordapp.com/developers/docs/intro">Discord API</a> very easily.</p> 
         <ul> 
          <li>Object-oriented</li> 
          <li>Predictable abstractions</li> 
          <li>Performant</li> 
          <li>100% coverage of the Discord API</li> 
         </ul> 
         <h2><a href="#installation" aria-hidden="true" class="anchor" id="user-content-installation">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Installation</h2> 
         <p><strong>Node.js 8.0.0 or newer is required.</strong><br> Ignore any warnings about unmet peer dependencies, as they're all optional.</p> 
         <p>Without voice support: <code>npm install discord.js --save</code><br> With voice support (<a href="https://www.npmjs.com/package/node-opus">node-opus</a>): <code>npm install discord.js node-opus --save</code><br> With voice support (<a href="https://www.npmjs.com/package/opusscript">opusscript</a>): <code>npm install discord.js opusscript --save</code></p> 
         <h3><a href="#audio-engines" aria-hidden="true" class="anchor" id="user-content-audio-engines">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Audio engines</h3> 
         <p>The preferred audio engine is node-opus, as it performs significantly better than opusscript. When both are available, discord.js will automatically choose node-opus. Using opusscript is only recommended for development environments where node-opus is tough to get working. For production bots, using node-opus should be considered a necessity, especially if they're going to be running on multiple servers.</p> 
         <h3><a href="#optional-packages" aria-hidden="true" class="anchor" id="user-content-optional-packages">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Optional packages</h3> 
         <ul> 
          <li><a href="https://www.npmjs.com/package/bufferutil">bufferutil</a> to greatly speed up the WebSocket when <em>not</em> using uws (<code>npm install bufferutil --save</code>)</li> 
          <li><a href="https://github.com/hammerandchisel/erlpack">erlpack</a> for significantly faster WebSocket data (de)serialisation (<code>npm install discordapp/erlpack --save</code>)</li> 
          <li>One of the following packages can be installed for faster voice packet encryption and decryption: 
           <ul> 
            <li><a href="https://www.npmjs.com/package/sodium">sodium</a> (<code>npm install sodium --save</code>)</li> 
            <li><a href="https://www.npmjs.com/package/libsodium-wrappers">libsodium.js</a> (<code>npm install libsodium-wrappers --save</code>)</li> 
           </ul> </li> 
          <li><a href="https://www.npmjs.com/package/uws">uws</a> for a much faster WebSocket connection (<code>npm install uws --save</code>)</li> 
         </ul>
