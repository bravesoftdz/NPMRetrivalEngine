      <h1>wger 1.8</h1> 
      <div id="download-button"> 
       <a class="button green" style="float:right;" href="#downloads">Downloads ?</a> 
      </div> 
      <p style="font-style: italic">FLOSS workout, fitness and weight manager/tracker written with Django</p> Thank you for downloading wger Workout Manager. wger (?v???) is a free, open source web
      <br>application that manages your exercises and personal workouts, weight and diet
      <br>plans. It can also be used as a simple gym management utility, providing different
      <br>administrative roles (trainer, manager, etc.). It offers a REST API as well, for
      <br>easy integration with other projects and tools.
      <br>
      <br>It is written with python/django and uses jQuery and some D3js for charts.
      <br>
      <br>For more details and a live system, refer to the project's site: https://wger.de/
      <br>
      <br>
      <br>Installation
      <br>============
      <br>
      <br>These are the basic steps to install and run the application locally on a linux
      <br>system. There are more detailed instructions, other deployment options as well
      <br>as an administration guide available at https://wger.readthedocs.io or locally
      <br>in your code repository in the docs folder (``make html`` to compile, then open
      <br>_build/index.html).
      <br>
      <br>Please consult the commands' help for further information and available
      <br>parameters.
      <br>
      <br>
      <br>Docker
      <br>------
      <br>
      <br>Useful to just try it out::
      <br>
      <br> docker run -ti --name wger.apache --publish 8000:80 wger/apache
      <br>
      <br>Then just open http://localhost:8000 and log in as: **admin**, password **admin**
      <br>
      <br>Alternatively, there are docker images for development as well, ``wger/devel``
      <br>and ``wger/devel-fedora``. Both images contain an instance of the application
      <br>running with django's development server using a sqlite database and can be
      <br>used to quickly setup a development instance (vim and tmux are already
      <br>installed). The only difference is that devel has an ubuntu base image while
      <br>devel-fedora uses fedora.
      <br>
      <br>::
      <br>
      <br> $ docker run -ti --name wger.devel --publish 8000:8000 wger/devel
      <br>
      <br>Then, *within the docker image*, activate the virtualenv
      <br>
      <br>::
      <br>
      <br> $ source ~/venv/bin/activate
      <br>
      <br>and start the development server
      <br>
      <br>::
      <br>
      <br> $ python manage.py runserver 0.0.0.0:8000
      <br>
      <br>Then just open http://localhost:8000 and log in as: **admin**, password **admin**
      <br>
      <br>
      <br>Development version (from git)
      <br>------------------------------
      <br>
      <br>**Note:** You can safely install from master, it is almost always in a usable
      <br>and stable state.
      <br>
      <br>
      <br>1) Install the necessary packages
      <br>
      <br>::
      <br>
      <br> $ sudo apt-get install python3-dev python-virtualenv nodejs nodejs-legacy npm libjpeg8-dev zlib1g-dev git
      <br>
      <br>
      <br>Then install the python packages from pypi in the virtualenv::
      <br>
      <br> $ virtualenv --python python3 venv-django
      <br> $ source venv-django/bin/activate
      <br>
      <br>
      <br>2) Start the application. This will download the required JS and CSS libraries
      <br> and create a SQlite database and populate it with data on the first run.
      <br>
      <br>::
      <br>
      <br> $ git clone https://github.com/wger-project/wger.git
      <br> $ cd wger
      <br> $ pip install -r requirements.txt # or requirements_devel.txt to develop
      <br> $ python setup.py develop
      <br> $ wger create_settings \
      <br> --settings-path /path/to/current/folder/settings.py \
      <br> --database-path /path/to/current/folder/database.sqlite
      <br> $ wger bootstrap \
      <br> --settings-path /path/to/current/folder/settings.py \
      <br> --no-start-server
      <br> $ python manage.py runserver
      <br>
      <br>3) Log in as: **admin**, password **admin**
      <br>
      <br>After the first run you can just use django's development server::
      <br>
      <br> $ python manage.py runserver
      <br>
      <br>
      <br>Stable version (from PyPI)
      <br>--------------------------
      <br>
      <br>1) Install the necessary packages and their dependencies in a virtualenv
      <br>
      <br>::
      <br>
      <br> $ sudo apt-get install python3-dev python-virtualenv nodejs nodejs-legacy npm libjpeg8-dev zlib1g-dev
      <br> $ virtualenv venv-django
      <br> $ source venv-django/bin/activate
      <br> $ pip install wger
      <br>
      <br>
      <br>2) Start the application. This will download the required JS and CSS libraries
      <br> and create a SQlite database and populate it with data on the first run.
      <br> Then, log in as: **admin**, password **admin**
      <br>::
      <br>
      <br> $ wger bootstrap
      <br>
      <br>
      <br>3) To start the installation again, just call wger start::
      <br>
      <br> $ wger start
      <br>
      <br>
      <br>Command line options
      <br>--------------------
      <br>You can get a list of all available commands by calling ``wger`` without any
      <br>arguments::
      <br>
      <br> Available tasks:
      <br>
      <br> bootstrap Performs all steps necessary to bootstrap the application
      <br> config_location Returns the default location for the settings file and the data folder
      <br> create_or_reset_admin Creates an admin user or resets the password for an existing one
      <br> create_settings Creates a local settings file
      <br> load_fixtures Loads all fixtures
      <br> migrate_db Run all database migrations
      <br> start Start the application using django's built in webserver
      <br>
      <br>You can also get help on a specific command with ``wger --help &lt;command&gt;``.
      <br>

            var outage = 0;
            var maintenance = 0;
            for (i in data.incidents) {
              var incident = data.incidents[i];
              var message, status = incident.status;
              if (status === 'scheduled') {
                message = '<li><a href="' + incident.shortlink + '">' + incident.name + ' scheduled.</a></li>\n';
                reports += message;
                maintenance += 1;
              } else if (status === 'in_progress') {
                message = '<li><a href="' + incident.shortlink + '">' + incident.name + ' is currently in progress.' + '</a></li>\n';
                reports += message;
                maintenance += 1;
              } else if (status !== 'resolved' && status !== 'postmortem' && status !== 'completed') {
                message = '<li><a href="' + incident.shortlink + '">' + incident.name + ': ' + incident.status + '</a></li>\n';
                reports += message;
                outage += 1;
              }
            }
            for (i in data.scheduled_maintenances) {
              var incident = data.scheduled_maintenances[i];
              var message, status = incident.status;
              if (status === 'scheduled') {
                message = '<li><a href="' + incident.shortlink + '">' + incident.name + ' scheduled.</a></li>\n';
                reports += message;
                maintenance += 1;
              } else if (status === 'in_progress') {
                message = '<li><a href="' + incident.shortlink + '">' + incident.name + ' is currently in progress.' + '</a></li>\n';
                reports += message;
                maintenance += 1;
              } else if (status !== 'resolved' && status !== 'postmortem' && status !== 'completed') {
                message = '<li><a href="' + incident.shortlink + '">' + incident.name + ': ' + incident.status + '</a></li>\n';
                reports += message;
                outage += 1;
              }
            }
            if (outage + maintenance === 0) {
              reports += "<li><a href='http://status.python.org'>Nothing to report</a></li>";
            }
            div.innerHTML=reports;
            if (outage > 0) {
              var statusbox = document.getElementById("statusbox");
              statusbox.style.background = '#FC234A';
            }
          }
