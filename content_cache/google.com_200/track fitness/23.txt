        <article class="markdown-body entry-content" itemprop="text">
         <p>Thank you for downloading wger Workout Manager. wger (?v???) is a free, open source web application that manages your exercises and personal workouts, weight and diet plans. It can also be used as a simple gym management utility, providing different administrative roles (trainer, manager, etc.). It offers a REST API as well, for easy integration with other projects and tools.</p> 
         <p>It is written with python/django and uses jQuery and some D3js for charts.</p> 
         <p>For more details and a live system, refer to the project's site: <a href="https://wger.de/">https://wger.de/</a></p>

            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Installation</h2> 
         <p>These are the basic steps to install and run the application locally on a linux system. There are more detailed instructions, other deployment options as well as an administration guide available at <a href="https://wger.readthedocs.io">https://wger.readthedocs.io</a> or locally in your code repository in the docs folder (<code>make html</code> to compile, then open _build/index.html).</p> 
         <p>Please consult the commands' help for further information and available parameters.</p> 
         <a name="user-content-docker"></a> 
         <h3><a id="user-content-docker" class="anchor" href="#docker" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Docker</h3> 
         <p>Useful to just try it out:</p> 
         <pre>docker run -ti --name wger.apache --publish 8000:80 wger/apache
</pre> 
         <p>Then just open <a href="http://localhost:8000">http://localhost:8000</a> and log in as: <strong>admin</strong>, password <strong>admin</strong></p> 
         <p>Alternatively, there are docker images for development as well, <code>wger/devel</code> and <code>wger/devel-fedora</code>. Both images contain an instance of the application running with django's development server using a sqlite database and can be used to quickly setup a development instance (vim and tmux are already installed). The only difference is that devel has an ubuntu base image while devel-fedora uses fedora.</p> 
         <pre>$ docker run -ti --name wger.devel --publish 8000:8000 wger/devel
</pre> 
         <p>Then, <em>within the docker image</em>, activate the virtualenv</p> 
         <pre>$ source ~/venv/bin/activate
</pre> 
         <p>and start the development server</p> 
         <pre>$ python manage.py runserver 0.0.0.0:8000
</pre> 
         <p>Then just open <a href="http://localhost:8000">http://localhost:8000</a> and log in as: <strong>admin</strong>, password <strong>admin</strong></p> 
         <a name="user-content-development-version-from-git"></a> 
         <h3><a id="user-content-development-version-from-git" class="anchor" href="#development-version-from-git" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Development version (from git)</h3> 
         <p><strong>Note:</strong> You can safely install from master, it is almost always in a usable and stable state.</p> 
         <ol> 
          <li>Install the necessary packages</li> 
         </ol> 
         <pre>$ sudo apt-get install python3-dev python-virtualenv nodejs nodejs-legacy npm libjpeg8-dev zlib1g-dev git
</pre> 
         <p>Then install the python packages from pypi in the virtualenv:</p> 
         <pre>$ virtualenv --python python3 venv-django
$ source venv-django/bin/activate
</pre> 
         <ol start="2"> 
          <li>Start the application. This will download the required JS and CSS libraries and create a SQlite database and populate it with data on the first run.</li> 
         </ol> 
         <pre>$ git clone https://github.com/wger-project/wger.git
$ cd wger
$ pip install -r requirements.txt  # or requirements_devel.txt to develop
$ python setup.py develop
$ wger create_settings \
       --settings-path /path/to/current/folder/settings.py \
       --database-path /path/to/current/folder/database.sqlite
$ wger bootstrap \
       --settings-path /path/to/current/folder/settings.py \
       --no-start-server
$ python manage.py runserver
</pre> 
         <ol start="3"> 
          <li>Log in as: <strong>admin</strong>, password <strong>admin</strong></li> 
         </ol> 
         <p>After the first run you can just use django's development server:</p> 
         <pre>$ python manage.py runserver
</pre> 
         <a name="user-content-stable-version-from-pypi"></a> 
         <h3><a id="user-content-stable-version-from-pypi" class="anchor" href="#stable-version-from-pypi" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Stable version (from PyPI)</h3> 
         <ol> 
          <li>Install the necessary packages and their dependencies in a virtualenv</li> 
         </ol> 
         <pre>$ sudo apt-get install python3-dev python-virtualenv nodejs nodejs-legacy npm libjpeg8-dev zlib1g-dev
$ virtualenv venv-django
$ source venv-django/bin/activate
$ pip install wger
</pre> 
         <ol start="2"> 
          <li>Start the application. This will download the required JS and CSS libraries and create a SQlite database and populate it with data on the first run. Then, log in as: <strong>admin</strong>, password <strong>admin</strong></li> 
         </ol> 
         <pre>$ wger bootstrap
</pre> 
         <ol start="3"> 
          <li>To start the installation again, just call wger start:</li> 
         </ol> 
         <blockquote>
           $ wger start
         </blockquote>

         <h3><a id="user-content-command-line-options" class="anchor" href="#command-line-options" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Command line options</h3> 
         <p>You can get a list of all available commands by calling <code>wger</code> without any arguments:</p> 
         <pre>Available tasks:

bootstrap               Performs all steps necessary to bootstrap the application
config_location         Returns the default location for the settings file and the data folder
create_or_reset_admin   Creates an admin user or resets the password for an existing one
create_settings         Creates a local settings file
load_fixtures           Loads all fixtures
migrate_db              Run all database migrations
start                   Start the application using django's built in webserver
</pre> 
         <p>You can also get help on a specific command with <code>wger --help &lt;command&gt;</code>.</p>
