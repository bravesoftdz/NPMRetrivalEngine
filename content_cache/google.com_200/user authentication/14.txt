                    <h2>What is Passport?</h2> 
                    <p>Passport is Express-consistent authentication middleware for Node.js. Passport can be convenient and simple. It will use be any Express based web application.</p> 
                    <p>Passport supports many set of strategies like,<br> Local Strategy :- Username (req.body.username) and<br> Password (req.body.password),</p> 
                    <p>Single sign-on using OAuth like,<br> Google<br> e.g passport.authenticate(google);<br> Facebook,<br> Twitter,<br> GitHub,<br> LinkedIn<br> Instagram and more</p> 
                    <p>To get started we have to install passport into our node application, we will do via <strong>npm (Node Package Manager)</strong></p> 
                    <p>Open the command prompt in Windows, Linux, or Mac Terminal. I assume that your present working directory is application in which you are going to implement the passport.</p>

                    <h4>Sessions :-</h4> 
                    <p>Passport will maintain constant login sessions.<br> In order to have continual work with session , serialized user must be use to session for authentication, and deserialized when consecutive requests are made.</p> 
                    <p>Passport does not contain any condition on how your users records are stored. In place of that, you provide functions to Passport which implements needful serialize and deserialize logic. In a typical application, this will be done as easily as serializing the User ID and finding the user by ID when deserialize.</p>
