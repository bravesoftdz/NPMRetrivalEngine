            <h1 class="package-description-redundant">Amazon Cognito Identity SDK for JavaScript</h1> 
            <p>You can now use Amazon Cognito to easily add user sign-up and sign-in to your mobile and web apps. Your User Pool in Amazon Cognito is a fully managed user directory that can scale to hundreds of millions of users, so you don't have to worry about building, securing, and scaling a solution to handle user management and authentication.</p> 
            <p>We welcome developer feedback on this project. You can reach us by creating an issue on the GitHub repository or posting to the Amazon Cognito Identity forums and the below blog post:</p> 
            <ul> 
             <li>https://github.com/aws/amazon-cognito-identity-js</li> 
             <li>https://forums.aws.amazon.com/forum.jspa?forumID=173</li> 
             <li>https://aws.amazon.com/blogs/mobile/accessing-your-user-pools-using-the-amazon-cognito-identity-sdk-for-javascript/</li> 
            </ul> 
            <p>For an overview of the Cognito authentication flow, refer to the following blog post:</p> 
            <ul> 
             <li>https://aws.amazon.com/blogs/mobile/customizing-your-user-pool-authentication-flow/</li> 
            </ul> 
            <h1>Introduction</h1> 
            <p>The Amazon Cognito Identity SDK for JavaScript allows JavaScript enabled applications to sign-up users, authenticate users, view, delete, and update user attributes within the Amazon Cognito Identity service. Other functionality includes password changes for authenticated users and initiating and completing forgot password flows for unauthenticated users.</p> 
            <p>Your users will benefit from a number of security features including SMS-based Multi-Factor Authentication (MFA) and account verification via phone or email. The password features use the Secure Remote Password (SRP) protocol to avoid sending cleartext passwords over the wire.</p> 
            <h1>Setup</h1> 
            <p>The Amazon Cognito Identity SDK for JavaScript depends on:</p> 
            <ol> 
             <li>The <code>CognitoIdentityServiceProvider</code> service from the <a href="https://github.com/aws/aws-sdk-js">AWS SDK for JavaScript</a></li> 
            </ol> 
            <p>There are two ways to install the Amazon Cognito Identity SDK for JavaScript and its dependencies, depending on your project setup and experience with modern JavaScript build tools:</p> 
            <ul> 
             <li> <p>Download each JavaScript library and include them in your HTML, or</p> </li> 
             <li> <p>Install the dependencies with npm and use a bundler like webpack.</p> </li> 
            </ul> 
            <h2>Install using separate JavaScript files</h2> 
            <p>This method is simpler and does not require additional tools, but may have worse performance due to the browser having to download multiple files.</p> 
            <p>Download each of the following JavaScript files for the required libraries and place them in your project:</p> 
            <ol> 
             <li> <p>The Amazon Cognito AWS SDK for JavaScript, from <a href="https://raw.githubusercontent.com/aws/amazon-cognito-identity-js/master/dist/aws-cognito-sdk.min.js">/dist/aws-cognito-sdk.min.js</a></p> <p>Note that the Amazon Cognito AWS SDK for JavaScript is just a slimmed down version of the AWS Javascript SDK namespaced as <code>AWSCognito</code> instead of <code>AWS</code>. It references only the Amazon Cognito Identity service.</p> </li> 
             <li> <p>The Amazon Cognito Identity SDK for JavaScript, from <a href="https://raw.githubusercontent.com/aws/amazon-cognito-identity-js/master/dist/amazon-cognito-identity.min.js">/dist/amazon-cognito-identity.min.js</a></p> </li> 
            </ol> 
            <p>Optionally, to use other AWS services, include a build of the <a href="http://aws.amazon.com/sdk-for-browser/">AWS SDK for JavaScript</a>.</p> 
            <p>Include all of the files in your HTML page before calling any Amazon Cognito Identity SDK APIs:</p> 
            <pre><code class="language-html">    &lt;script src="/path/to/aws-cognito-sdk.min.js"&gt;&lt;/script&gt;
    &lt;script src="/path/to/amazon-cognito-identity.min.js"&gt;&lt;/script&gt;
    &lt;!-- optional: only if you use other AWS services --&gt;
    &lt;script src="/path/to/aws-sdk-2.6.10.js"&gt;&lt;/script&gt;
</code></pre> 
            <h2>Using NPM and Webpack</h2> 
            <p>Webpack is a popular JavaScript bundling and optimization tool, it has many configuration features that can build your source JavaScript into one or more files for distribution. The following is a quick setup guide with specific notes for using the Amazon Cognito Identity SDK for JavaScript with it, but there are many more ways it can be used, see <a href="https://webpack.github.io/">the Webpack site</a>, and in particular the <a href="http://webpack.github.io/docs/configuration.html">configuration documentation</a></p> 
            <p>Note that webpack expects your source files to be structured as <a href="https://webpack.github.io/docs/commonjs.html">CommonJS (Node.js-style) modules</a> (or <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/import">ECMAScript 2015 modules</a> if you are using a transpiler such as <a href="https://babeljs.io/">Babel</a>.) If your project is not already using modules you may wish to use <a href="http://webpack.github.io/docs/shimming-modules.html">Webpack's module shimming features</a> to ease migration.</p> 
            <ul> 
             <li> <p>Install <a href="https://nodejs.org">Node.js</a> on your development machine (this will not be needed on your server.)</p> </li> 
             <li> <p>In your project add a <code>package.json</code>, either use <code>npm init</code> or the minimal:</p> <pre><code class="language-json">{
  "private": true
}
</code></pre> </li> 
             <li> <p>Install the Amazon Cognito Identity SDK for JavaScript and the Webpack tool into your project with <code>npm</code> (the Node Package Manager, which is installed with Node.js):</p> <pre><code>&gt; npm install --save-dev webpack json-loader
&gt; npm install --save amazon-cognito-identity-js
</code></pre> <p>These will add a <code>node_modules</code> directory containing these tools and dependencies into your project, you will probably want to exclude this directory from source control. Adding the <code>--save</code> parameters will update the <code>package.json</code> file with instructions on what should be installed, so you can simply call <code>npm install</code> without any parameters to recreate this folder later.</p> </li> 
             <li> <p>Create the configuration file for <code>webpack</code>, named <code>webpack.config.js</code>:</p> <pre><code class="language-js">module.exports = {
  // Example setup for your project:
  // The entry module that requires or imports the rest of your project.
  // Must start with `./`!
  entry: './src/entry',
  // Place output files in `./dist/my-app.js`
  output: {
    path: 'dist',
    filename: 'my-app.js'
  },
  module: {
    loaders: [
      {
        test: /\.json$/,
        loader: 'json'
      }
    ]
  }
};
</code></pre> </li> 
             <li> <p>Add the following into your <code>package.json</code></p> <pre><code class="language-json">{
  "scripts": {
    "build": "webpack"
  }
}
</code></pre> </li> 
             <li> <p>Build your application bundle with <code>npm run build</code></p> </li> 
            </ul> 
            <h2>Configuration</h2> 
            <p>The Amazon Cognito Identity SDK for JavaScript requires two configuration values from your AWS Account in order to access your Cognito User Pool:</p> 
            <ul> 
             <li>The User Pool Id, e.g. <code>us-east-1_aB12cDe34</code></li> 
             <li>A User Pool App Client Id, e.g. <code>7ghr5379orhbo88d52vphda6s9</code> 
              <ul> 
               <li>When creating the App, the generate client secret box must be <strong>unchecked</strong> because the JavaScript SDK doesn't support apps that have a client secret.</li> 
              </ul> </li> 
            </ul> 
            <p>The <a href="https://console.aws.amazon.com/cognito/users/">AWS Console for Cognito User Pools</a> can be used to get or create these values.</p> 
            <p>If you will be using Cognito Federated Identity to provide access to your AWS resources or Cognito Sync you will also need the Id of a Cognito Identity Pool that will accept logins from the above Cognito User Pool and App, i.e. <code>us-east-1:85156295-afa8-482c-8933-1371f8b3b145</code>.</p> 
            <p>Note that the various errors returned by the service are valid JSON so one can access the different exception types (err.code) and status codes (err.statusCode).</p> 
            <h2>Relevant examples</h2> 
            <p>For an example using babel-webpack of a React setup, see <a href="https://github.com/aws/amazon-cognito-identity-js/tree/master/examples/babel-webpack">babel-webpack example</a>.</p> 
            <p>For a working example using angular, see <a href="https://github.com/awslabs/aws-cognito-angular2-quickstart">cognito-angular2-quickstart</a>.</p> 
            <p>If you are having issues when using Aurelia, please see the following <a href="http://stackoverflow.com/questions/39714424/how-can-i-get-the-amazon-cognito-identity-sdk-working-in-aurelia">Stack Overflow post</a>.</p> 
            <h2>Usage</h2> 
            <p>The usage examples below use the unqualified names for types in the Amazon Cognito Identity SDK for JavaScript. Remember to import or qualify access to any of these types:</p> 
            <pre><code class="language-javascript">    // When using loose Javascript files:
    var CognitoUserPool = AmazonCognitoIdentity.CognitoUserPool;

    // Under the original name:
    var CognitoUserPool = AWSCognito.CognitoIndentityServiceProvider.CognitoUserPool;

    // Modules, e.g. Webpack:
    var AmazonCognitoIdentity = require('amazon-cognito-identity-js');
    var CognitoUserPool = AmazonCognitoIdentity.CognitoUserPool;

    // ES Modules, e.g. transpiling with Babel
    import { CognitoUserPool, CognitoUserAttribute, CognitoUser } from 'amazon-cognito-identity-js';
</code></pre> 
            <p><strong>Use case 1.</strong> Registering a user with the application. One needs to create a CognitoUserPool object by providing a UserPoolId and a ClientId and signing up by using a username, password, attribute list, and validation data.</p> 
            <pre><code class="language-javascript">
    var poolData = {
        UserPoolId : '...', // Your user pool id here
        ClientId : '...' // Your client id here
    };
    var userPool = new AWSCognito.CognitoIdentityServiceProvider.CognitoUserPool(poolData);

    var attributeList = [];

    var dataEmail = {
        Name : 'email',
        Value : 'email@mydomain.com'
    };

    var dataPhoneNumber = {
        Name : 'phone_number',
        Value : '+15555555555'
    };
    var attributeEmail = new AWSCognito.CognitoIdentityServiceProvider.CognitoUserAttribute(dataEmail);
    var attributePhoneNumber = new AWSCognito.CognitoIdentityServiceProvider.CognitoUserAttribute(dataPhoneNumber);

    attributeList.push(attributeEmail);
    attributeList.push(attributePhoneNumber);

    userPool.signUp('username', 'password', attributeList, null, function(err, result){
        if (err) {
            alert(err);
            return;
        }
        cognitoUser = result.user;
        console.log('user name is ' + cognitoUser.getUsername());
    });
</code></pre> 
            <p><strong>Use case 2.</strong> Confirming a registered, unauthenticated user using a confirmation code received via SMS.</p> 
            <pre><code class="language-javascript">    var poolData = {
        UserPoolId : '...', // Your user pool id here
        ClientId : '...' // Your client id here
    };

    var userPool = new AWSCognito.CognitoIdentityServiceProvider.CognitoUserPool(poolData);
    var userData = {
        Username : 'username',
        Pool : userPool
    };

    var cognitoUser = new AWSCognito.CognitoIdentityServiceProvider.CognitoUser(userData);
    cognitoUser.confirmRegistration('123456', true, function(err, result) {
        if (err) {
            alert(err);
            return;
        }
        console.log('call result: ' + result);
    });
</code></pre> 
            <p><strong>Use case 3.</strong> Resending a confirmation code via SMS for confirming registration for a unauthenticated user.</p> 
            <pre><code class="language-javascript">    cognitoUser.resendConfirmationCode(function(err, result) {
        if (err) {
            alert(err);
            return;
        }
        console.log('call result: ' + result);
    });
</code></pre> 
            <p><strong>Use case 4.</strong> Authenticating a user and establishing a user session with the Amazon Cognito Identity service.</p> 
            <pre><code class="language-javascript">    var authenticationData = {
        Username : 'username',
        Password : 'password',
    };
    var authenticationDetails = new AWSCognito.CognitoIdentityServiceProvider.AuthenticationDetails(authenticationData);
    var poolData = {
        UserPoolId : '...', // Your user pool id here
        ClientId : '...' // Your client id here
    };
    var userPool = new AWSCognito.CognitoIdentityServiceProvider.CognitoUserPool(poolData);
    var userData = {
        Username : 'username',
        Pool : userPool
    };
    var cognitoUser = new AWSCognito.CognitoIdentityServiceProvider.CognitoUser(userData);
    cognitoUser.authenticateUser(authenticationDetails, {
        onSuccess: function (result) {
            console.log('access token + ' + result.getAccessToken().getJwtToken());

            //POTENTIAL: Region needs to be set if not already set previously elsewhere.
            AWS.config.region = '&lt;region&gt;';

            AWS.config.credentials = new AWS.CognitoIdentityCredentials({
                IdentityPoolId : '...', // your identity pool id here
                Logins : {
                    // Change the key below according to the specific region your user pool is in.
                    'cognito-idp.&lt;region&gt;.amazonaws.com/&lt;YOUR_USER_POOL_ID&gt;' : result.getIdToken().getJwtToken()
                }
            });

            // Instantiate aws sdk service objects now that the credentials have been updated.
            // example: var s3 = new AWS.S3();

        },

        onFailure: function(err) {
            alert(err);
        },

    });
</code></pre> 
            <p>Note that if device tracking is enabled for the user pool with a setting that user opt-in is required, you need to implement an onSuccess(result, userConfirmationNecessary) callback, collect user input and call either setDeviceStatusRemembered to remember the device or setDeviceStatusNotRemembered to not remember the device.</p> 
            <p>Note also that if CognitoUser.authenticateUser throws ReferenceError: navigator is not defined when running on Node.js, follow the instructions on the following <a href="http://stackoverflow.com/questions/40219518/aws-cognito-unauthenticated-login-error-window-is-not-defined-js">Stack Overflow post</a>.</p> 
            <p><strong>Use case 5.</strong> Retrieve user attributes for an authenticated user.</p> 
            <pre><code class="language-javascript">    cognitoUser.getUserAttributes(function(err, result) {
        if (err) {
            alert(err);
            return;
        }
        for (i = 0; i &lt; result.length; i++) {
            console.log('attribute ' + result[i].getName() + ' has value ' + result[i].getValue());
        }
    });
</code></pre> 
            <p><strong>Use case 6.</strong> Verify user attribute for an authenticated user.</p> 
            <p>Note that the inputVerificationCode method needs to be defined but does not need to actually do anything. If you would like the user to input the verification code on another page, you can set inputVerificationCode to null. If inputVerificationCode is null, onSuccess will be called immediately (assuming there is no error).</p> 
            <pre><code class="language-javascript">    cognitoUser.getAttributeVerificationCode('email', {
        onSuccess: function (result) {
            console.log('call result: ' + result);
        },
        onFailure: function(err) {
            alert(err);
        },
        inputVerificationCode: function() {
            var verificationCode = prompt('Please input verification code: ' ,'');
            cognitoUser.verifyAttribute('email', verificationCode, this);
        }
    });
</code></pre> 
            <p><strong>Use case 7.</strong> Delete user attribute for an authenticated user.</p> 
            <pre><code class="language-javascript">    var attributeList = [];
    attributeList.push('nickname');

    cognitoUser.deleteAttributes(attributeList, function(err, result) {
        if (err) {
            alert(err);
            return;
        }
        console.log('call result: ' + result);
    });
</code></pre> 
            <p><strong>Use case 8.</strong> Update user attributes for an authenticated user.</p> 
            <pre><code class="language-javascript">    var attributeList = [];
    var attribute = {
        Name : 'nickname',
        Value : 'joe'
    };
    var attribute = new AWSCognito.CognitoIdentityServiceProvider.CognitoUserAttribute(attribute);
    attributeList.push(attribute);

            <h2>Network Configuration</h2> 
            <p>The Amazon Cognito Identity JavaScript SDK will make requests to the following endpoints</p> 
            <ul> 
             <li>For Amazon Cognito Identity request handling: "https://cognito-idp.us-east-1.amazonaws.com" 
              <ul> 
               <li>This endpoint may change based on which region your Identity Pool was created in.</li> 
              </ul> </li> 
            </ul> 
            <p>For most frameworks you can whitelist the domain by whitelisting all AWS endpoints with "*.amazonaws.com".</p> 
            <h2>Random numbers</h2> 
            <p>In order to authenticate with the Amazon Cognito Identity Service, the client needs to generate a random number as part of the SRP protocol. The AWS SDK is only compatible with modern browsers, and these include support for cryptographically strong random values. If you do need to support older browsers then you should be aware that this is less secure, and if possible include a strong polyfill for <code>window.crypto.getRandomValues()</code> before including this library.</p>
