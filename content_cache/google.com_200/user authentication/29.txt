               <p><b>Authentication</b> is a big part of any application. The way authentication is handled traditionally is by creating a user session on the server where we usually store the users information in memory or on disk. This is also known as server/session based authentication.</p> 
               <p>Web applications have come a long way in a past few year, we have the stateless web, hand-held devices came along, we have load balancer then came along the micro-services architecture. Storing session data on the server made our app less scalable and then sharing the session between multiple services or servers is also a problem. That is why <strong>Token Based Authentication</strong> ?</p> 
               <p><b>In this tutorial, we will learn to implement token based authentication in our node.js applications.</b></p> 
               <h3>What is token based authentication?</h3> 
               <p>Token-based authentication is state-less and session less, meaning when we authenticate the user we do not store any user information on the server. Instead, we generate a token signed by a private key and send it to the client. The way it works is as follows.</p> 
               <ul> 
                <li>User makes a request to the server with username/password</li> 
                <li>The server verifies the user.</li> 
                <li>The server generates a signed token and provides it to the client.</li> 
                <li>The token may contain the user data.</li> 
                <li>The client stores the token and sends it along with every request.</li> 
                <li>The server verifies the token and processes the request.</li> 
               </ul> 
               <p>Tokens can be sent to server in any way but the best practice tells us to send it in an <b>HTTP header</b>.</p> 
               <p style="text-align: center;"> <a data-fasc-style="background-color:#1fa67a;color:#ffffff;" href="https://github.com/rahil471/jwt-in-nodejs" target="_blank" class="fasc-button fasc-size-large fasc-type-popout fasc-ico-before dashicons-download">DOWNLOAD</a></p> 
               <h3><a id="Our_Application_14"></a>Our Application</h3> 
               <p>In this tutorial, we will see how we can easily add token based authentication using JSON web Tokens in Node.js.<br> We will build a few APIs using NodeJS and ExpressJS and see how we can protect/authenticate them using JWTs<br> We will be using.</p> 
               <ul> 
                <li>MongoDB as our DB</li> 
                <li>ExpressJS for routes</li> 
                <li><kbd>jsonwebtoken</kbd> an npm module for managing tokens.</li> 
               </ul>

                  <span class="br0">(</span>
                  <span class="st0">'/protected'</span>
                  <span class="sy0">,</span> verifyToken
                  <span class="sy0">,</span> require
                  <span class="br0">(</span>
                  <span class="st0">'./protected'</span>
                  <span class="br0">)</span>
                  <span class="br0">)</span>
                  <span class="sy0">;</span>
                  <br> 
                  <br> module.
                  <span class="me1">exports</span> 
                  <span class="sy0">=</span> router
                  <span class="sy0">;</span>
                 </div>
                </div>

</pre> 
               <p>Note that we are using the <kbd>verifyToken</kbd> middleware to protect our APIs. Once again restart your server and test the API using postman. Remember if we hit the API without the token it will result in a <b>403 forbidden error</b>.</p>

               <h3>Conclusion</h3> 
               <p>We have seen how we can add token-based authentication to our <b>node.js</b> application using <b>jsonwebtoken</b>. This was just a simple use-case to help get an understanding on how token based authentication works. Token based authentication scales well and makes it easier to manage cross devices authentication.</p>
