           <h3 name="7d7d" id="7d7d" class="graf graf--h3 graf--leading">Authentication?</h3>
           <p name="f29f" id="f29f" class="graf graf--p graf-after--h3">Authentication is for identifying users and provide different access rights and content depending on their id. In most cases the application provides a login form with certain credentials to verify a user.</p>
           <p name="53b8" id="53b8" class="graf graf--p graf-after--p">It’s necessary to understand:</p>
           <ul class="postList">
            <li name="3c36" id="3c36" class="graf graf--li graf-after--p">authentication as</li>
            <li name="03ea" id="03ea" class="graf graf--li graf-after--li">authorization as</li>
            <li name="c73f" id="c73f" class="graf graf--li graf-after--li">a session</li>
            <li name="d528" id="d528" class="graf graf--li graf-after--li">cookies</li>
           </ul>
           <h3 name="1c9a" id="1c9a" class="graf graf--h3 graf-after--li">What I will use for this introduction</h3>
           <h4 name="ddce" id="ddce" class="graf graf--h4 graf-after--h3">Development Environment</h4>
           <p name="aca5" id="aca5" class="graf graf--p graf-after--h4">In this example here I will use</p>
           <ul class="postList">
            <li name="72aa" id="72aa" class="graf graf--li graf-after--p">Plain JavaScript</li>
            <li name="3ce5" id="3ce5" class="graf graf--li graf-after--li">Node.js</li>
            <li name="2224" id="2224" class="graf graf--li graf-after--li">Express (JS framework)</li>
            <li name="1945" id="1945" class="graf graf--li graf-after--li">MongoDB (Database)</li>
            <li name="d34b" id="d34b" class="graf graf--li graf-after--li">Yarn (package management)</li>
            <li name="f8d0" id="f8d0" class="graf graf--li graf-after--li">Visual Studio Code as editor</li>
           </ul>
           <p name="4ad1" id="4ad1" class="graf graf--p graf-after--li">for writing the authentication.</p>
           <p name="7af0" id="7af0" class="graf graf--p graf-after--p">For the login mask I will use the awesome <a href="https://w3layouts.com/register-login-widget-flat-responsive-widget-template/" data-href="https://w3layouts.com/register-login-widget-flat-responsive-widget-template/" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">template from w3layouts</a>.</p>

           <h3 name="50cb" id="50cb" class="graf graf--h3 graf-after--li">User registration</h3>
           <p name="2e79" id="2e79" class="graf graf--p graf-after--h3">I’ll start with a basic express starter setup, which simply creates a webserver and serves the static files from the template on the home route. <a href="https://github.com/DDCSLearning/authenticationIntro/commit/97da94b5b6b52c4f2452d92eb0e92a110a97a1f4" data-href="https://github.com/DDCSLearning/authenticationIntro/commit/97da94b5b6b52c4f2452d92eb0e92a110a97a1f4" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">(see on Github commit)</a></p>
           <h4 name="ad27" id="ad27" class="graf graf--h4 graf-after--p">Connect to&nbsp;MongoDB</h4>
           <ul class="postList">
            <li name="2765" id="2765" class="graf graf--li graf-after--h4">install <a href="http://mongoosejs.com/docs/" data-href="http://mongoosejs.com/docs/" class="markup--anchor markup--li-anchor" rel="nofollow noopener" target="_blank">Mongoose</a></li>
            <li name="8e06" id="8e06" class="graf graf--li graf-after--li">install mondodb</li>
            <li name="07aa" id="07aa" class="graf graf--li graf-after--li">setup up mongod if you haven’t <a href="https://treehouse.github.io/installation-guides/mac/mongo-mac.html" data-href="https://treehouse.github.io/installation-guides/mac/mongo-mac.html" class="markup--anchor markup--li-anchor" rel="nofollow noopener" target="_blank">(tutorial)</a></li>

           <h4 name="554d" id="554d" class="graf graf--h4 graf-after--li">Create a&nbsp;schema</h4>
           <p name="88d5" id="88d5" class="graf graf--p graf-after--h4">MongoDB is a document database, which stores JSON like objects. The model/schema describes what this objects should contain.</p>
           <ul class="postList">
            <li name="ab04" id="ab04" class="graf graf--li graf-after--p">create a schema according to the <a href="http://mongoosejs.com/docs/guide.html" data-href="http://mongoosejs.com/docs/guide.html" class="markup--anchor markup--li-anchor" rel="nofollow noopener" target="_blank">docs</a> and store it in an own folder</li>
            <li name="b3e9" id="b3e9" class="graf graf--li graf-after--li">the schema should describe the fields we have in our form and specify the data it can expect</li>
           </ul>
           <p name="d02c" id="d02c" class="graf graf--p graf-after--li">It should look something like this:</p>
           <pre name="f66e" id="f66e" class="graf graf--pre graf-after--p">var mongoose = require('mongoose');<br>var UserSchema = new mongoose.Schema({<br>  email: {<br>    type: String,<br>    unique: true,<br>    required: true,<br>    trim: true<br>  },<br>  username: {<br>    type: String,<br>    unique: true,<br>    required: true,<br>    trim: true<br>  },<br>  password: {<br>    type: String,<br>    required: true,<br>  },<br>  passwordConf: {<br>    type: String,<br>    required: true,<br>  }<br>});<br>var User = mongoose.model('User', UserSchema);<br>module.exports = User;</pre>
           <h4 name="4870" id="4870" class="graf graf--h4 graf-after--pre">Insert data into&nbsp;MongoDB</h4>
           <ul class="postList">
            <li name="1981" id="1981" class="graf graf--li graf-after--h4">add <a href="https://github.com/expressjs/body-parser" data-href="https://github.com/expressjs/body-parser" class="markup--anchor markup--li-anchor" rel="nofollow noopener" target="_blank">body-parser</a> for parsing incoming request bodies in a middleware</li>
            <li name="5592" id="5592" class="graf graf--li graf-after--li">create a POST route for sending the data to the server</li>
            <li name="d022" id="d022" class="graf graf--li graf-after--li">store the values of the filled out form and store the data in the db with the schema</li>
            <li name="aa15" id="aa15" class="graf graf--li graf-after--li">it should look like this:</li>
           </ul>
           <pre name="c845" id="c845" class="graf graf--pre graf-after--li">if (req.body.email &amp;&amp;<br>  req.body.username &amp;&amp;<br>  req.body.password &amp;&amp;<br>  req.body.passwordConf) {</pre>
           <pre name="9008" id="9008" class="graf graf--pre graf-after--pre">  var userData = {<br>    email: req.body.email,<br>    username: req.body.username,<br>    password: req.body.password,<br>    passwordConf: req.body.passwordConf,<br>  }</pre>
           <pre name="f055" id="f055" class="graf graf--pre graf-after--pre">  //use schema.create to insert data into the db<br>  User.create(userData, function (err, user) {<br>    if (err) {<br>      return next(err)<br>    } else {<br>      return res.redirect('/profile');<br>    }<br>  });<br>}</pre>
           <ul class="postList">
            <li name="650c" id="650c" class="graf graf--li graf-after--pre">use the <a href="https://docs.mongodb.com/manual/reference/mongo-shell/" data-href="https://docs.mongodb.com/manual/reference/mongo-shell/" class="markup--anchor markup--li-anchor" rel="nofollow noopener" target="_blank">mongo shell</a> to see if your data has been saved to the database (it should show a document in <code class="markup--code markup--li-code">db.users.find()</code> )</li>
           </ul>
           <h4 name="8fde" id="8fde" class="graf graf--h4 graf-after--li">Hashing and&nbsp;salting</h4>
           <p name="b04a" id="b04a" class="graf graf--p graf-after--h4">Cryptographic hash functions take a piece of information and return a string, representing this information. Hash values cannot easily be “unhashed” or decrypted and that’s why they are a perfect fit for passwords.</p>
           <p name="c499" id="c499" class="graf graf--p graf-after--p">Salt values are random data that is included with the input for the hash function.</p>
           <p name="8b17" id="8b17" class="graf graf--p graf-after--p">In this tutorial we are using <a href="https://www.npmjs.com/package/bcrypt" data-href="https://www.npmjs.com/package/bcrypt" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">bcrypt</a>.</p>
           <p name="ac6e" id="ac6e" class="graf graf--p graf-after--p">So next:</p>
           <ul class="postList">
            <li name="3674" id="3674" class="graf graf--li graf-after--p">install the <a href="https://www.npmjs.com/package/bcrypt" data-href="https://www.npmjs.com/package/bcrypt" class="markup--anchor markup--li-anchor" rel="nofollow noopener" target="_blank">bcrypt</a> package</li>
            <li name="4949" id="4949" class="graf graf--li graf-after--li">add a prehook to your mongoose schema. should look like this:</li>
           </ul>
           <pre name="6d9c" id="6d9c" class="graf graf--pre graf-after--li">//hashing a password before saving it to the database<br>UserSchema.pre('save', function (next) {<br>  var user = this;<br>  bcrypt.hash(user.password, 10, function (err, hash){<br>    if (err) {<br>      return next(err);<br>    }<br>    user.password = hash;<br>    next();<br>  })<br>});</pre>
           <ul class="postList">
            <li name="2d36" id="2d36" class="graf graf--li graf-after--pre">test with mongod if the new data is inserted with a hashed password (it should work)</li>
           </ul>
           <p name="e9e3" id="e9e3" class="graf graf--p graf-after--li graf--trailing">Compare with my working <a href="https://github.com/DDCSLearning/authenticationIntro/commit/33ac4662c38f7c3115615983cf60effe2ebbd7ed" data-href="https://github.com/DDCSLearning/authenticationIntro/commit/33ac4662c38f7c3115615983cf60effe2ebbd7ed" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">commit</a> if needed.</p>

           <h3 name="08a3" id="08a3" class="graf graf--h3 graf--leading">Sessions and&nbsp;Cookies</h3>
           <p name="6156" id="6156" class="graf graf--p graf-after--h3">HTTP is a stateless protocol, which means that web servers don’t keep track of who is visiting a page. Displaying specific content to logged-in users require this tracking. Therefore sessions with a session ID are created. Cookies are key/value pairs managed by browsers. Those correspond with the sessions of the server.</p>
           <h4 name="baa7" id="baa7" class="graf graf--h4 graf-after--p">Set up&nbsp;Sessions</h4>
           <ul class="postList">
            <li name="83cc" id="83cc" class="graf graf--li graf-after--h4">add the <a href="https://www.npmjs.com/package/express-session" data-href="https://www.npmjs.com/package/express-session" class="markup--anchor markup--li-anchor" rel="nofollow noopener" target="_blank">express session</a> package</li>
            <li name="e149" id="e149" class="graf graf--li graf-after--li">add the session middleware in your app. A simple one looks like that:</li>
           </ul>
           <pre name="e0c8" id="e0c8" class="graf graf--pre graf-after--li">//use sessions for tracking logins<br>app.use(session({<br>  secret: 'work hard',<br>  resave: true,<br>  saveUninitialized: false<br>}));</pre>
           <ul class="postList">
            <li name="80c3" id="80c3" class="graf graf--li graf-after--pre">store the MongoDB userId (<code class="markup--code markup--li-code">_id</code>) in the <code class="markup--code markup--li-code">req.session.userId</code></li>
            <li name="b5fc" id="b5fc" class="graf graf--li graf-after--li">setup the login route the same way you set up the register route (in the login you only have the username and password)</li>
            <li name="273c" id="273c" class="graf graf--li graf-after--li">authenticate the input against the data in the database in the user schema. It should look like this:</li>
           </ul>
           <pre name="4aa8" id="4aa8" class="graf graf--pre graf-after--li graf--trailing">//authenticate input against database<br>UserSchema.statics.authenticate = function (email, password, callback) {<br>  User.findOne({ email: email })<br>    .exec(function (err, user) {<br>      if (err) {<br>        return callback(err)<br>      } else if (!user) {<br>        var err = new Error('User not found.');<br>        err.status = 401;<br>        return callback(err);<br>      }<br>      bcrypt.compare(password, user.password, function (err, result) {<br>        if (result === true) {<br>          return callback(null, user);<br>        } else {<br>          return callback();<br>        }<br>      })<br>    });<br>}</pre>
          </div>

           <h4 name="e760" id="e760" class="graf graf--h4 graf-after--figure">Refining the&nbsp;app</h4>
           <ul class="postList">
            <li name="0ccc" id="0ccc" class="graf graf--li graf-after--h4">make sure to adapt your layout accordingly to the sessions (hiding register fields and providing logout buttons)</li>
            <li name="95e0" id="95e0" class="graf graf--li graf-after--li">create a middleware to make user IDs available in HTML</li>
            <li name="bffd" id="bffd" class="graf graf--li graf-after--li">create a logout route that destroys the session id and redirects back to the home route. It can look like this:</li>
           </ul>
           <pre name="33f9" id="33f9" class="graf graf--pre graf-after--li">// GET /logout<br>router.get('/logout', function(req, res, next) {<br>  if (req.session) {<br>    // delete session object<br>    req.session.destroy(function(err) {<br>      if(err) {<br>        return next(err);<br>      } else {<br>        return res.redirect('/');<br>      }<br>    });<br>  }<br>});</pre>
           <p name="8e02" id="8e02" class="graf graf--p graf-after--pre">There is much more to add but logging out and destroying the session is important for each authentication system! That’s why I’ve included it here as well.</p>
           <h3 name="90ab" id="90ab" class="graf graf--h3 graf-after--p">Creating custom middleware</h3>
           <p name="37c9" id="37c9" class="graf graf--p graf-after--h3">Middleware runs after a request is received, but before a response is sent back. In this example the body-parser package is used as middleware. It converts incoming requests into a format that is easy to use for a JS program.</p>
           <p name="5e81" id="5e81" class="graf graf--p graf-after--p">Middleware functions can be chained after each other and fit into the request/response cycle of the application. When writing custom middleware, <code class="markup--code markup--p-code">next()</code> always has to be called at the end of that middleware to move to the next one in the cycle.</p>
           <p name="f605" id="f605" class="graf graf--p graf-after--p">Middleware can be used in many cases in this example, however, for simplicity reasons, I just reference an example to give an idea.</p>
           <p name="e7cc" id="e7cc" class="graf graf--p graf-after--p">Example: Creating middleware that requires a login for certain pages.</p>
           <pre name="895c" id="895c" class="graf graf--pre graf-after--p">function requiresLogin(req, res, next) {<br>  if (req.session &amp;&amp; req.session.userId) {<br>    return next();<br>  } else {<br>    var err = new Error('You must be logged in to view this page.');<br>    err.status = 401;<br>    return next(err);<br>  }<br>}</pre>
           <pre name="1782" id="1782" class="graf graf--pre graf-after--pre">router.get('/profile', mid.requiresLogin, function(req, res, next) {<br>  //...<br>});</pre>
           <p name="c757" id="c757" class="graf graf--p graf-after--pre">Writing your own middleware gives you the freedom for ultimate flexibility when refining authentication routes.</p>
           <h3 name="3769" id="3769" class="graf graf--h3 graf-after--p">A note on scalability with&nbsp;sessions</h3>
           <p name="3a03" id="3a03" class="graf graf--p graf-after--h3">Currently sessions are stored in RAM. To store have more size we can connect the session store to MongoDB. I’ll use the <a href="https://www.npmjs.com/package/connect-mongo" data-href="https://www.npmjs.com/package/connect-mongo" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">connect-mongo</a> package for that.</p>
           <ul class="postList">
            <li name="60e5" id="60e5" class="graf graf--li graf-after--p">simply add the store like instructed on their docs. It should now look like this:</li>
           </ul>
           <pre name="dbf1" id="dbf1" class="graf graf--pre graf-after--li">//use sessions for tracking logins<br>app.use(session({<br>  secret: 'work hard',<br>  resave: true,<br>  saveUninitialized: false,<br>  store: new MongoStore({<br>    mongooseConnection: db<br>  })<br>}));</pre>
           <ul class="postList">
            <li name="5e96" id="5e96" class="graf graf--li graf-after--pre">when checking with the mongo shell you should see how the new collection “sessions” is created. When logging in or out the data in that collection changes accordingly.</li>
           </ul>
           <h3 name="3509" id="3509" class="graf graf--h3 graf-after--li">Wrap up</h3>
           <ul class="postList">
            <li name="3ca4" id="3ca4" class="graf graf--li graf-after--h3">always make sure to transfer credentials in an encrypted way from the browser to the server and backwards</li>
            <li name="e52a" id="e52a" class="graf graf--li graf-after--li">be sure to add a security certificate to your HTTP (HTTPS)</li>
            <li name="4e41" id="4e41" class="graf graf--li graf-after--li">keep in mind that this way (with sessions and cookies) was only one way to authenticate, others are for example</li>
            <li name="e73a" id="e73a" class="graf graf--li graf-after--li">token based authentication with OAuth or JSON Web Tokens</li>
            <li name="803b" id="803b" class="graf graf--li graf-after--li">or with the <a href="http://passportjs.org/" data-href="http://passportjs.org/" class="markup--anchor markup--li-anchor" rel="nofollow noopener" target="_blank">passport middleware</a></li>
           </ul>
