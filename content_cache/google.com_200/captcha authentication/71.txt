          <div class="article__body"> 
           <p>Laravel is one of the most popular PHP frameworks for developing web applications. It offers a number of great features such as simple and fast routing, different ways for accessing relational databases, powerful dependency injection and much more.</p> 
           <p>In this article we are going to share with you 15 excellent open-source PHP libraries for extending Laravel. You can easily include them in any Laravel project to add various utilities and improve your workflow.</p> 
           <hr> 
           <h2><a href="https://github.com/barryvdh/laravel-debugbar" target="_blank" rel="noreferrer noopener">Laravel Debugbar</a></h2> 
           <p>A package for Laravel 5 which adds a developer toolbar for debugging the PHP and Laravel code of your app. There are lots of options which allow you to show all queries, get information about the current Route, show the currently loaded Views, and much more. </p> 
           <pre data-type="php">
// All arguments will be dumped as a debug message
debug($var1, $someString, $intValue, $object);

// Measure render time or other events.
start_measure('render','Time for rendering');
stop_measure('render');
add_measure('now', LARAVEL_START, microtime(true));
measure('My long operation', function() {
    // Do something
});
</pre> 
           <hr> 
           <h2><a href="https://github.com/Zizaco/entrust" target="_blank" rel="noreferrer noopener">Entrust</a></h2> 
           <p>Entrust is a Laravel 5 package which gives you a flexible way to add role-based permissions to your project. The library creates four new tables: roles, permissions, role_user and permission_role, which you can use to set up roles with different levels of access.</p> 
           <pre data-type="php">// Creating role and permissions
$admin = new Role();
$admin-&gt;name = 'admin';
$admin-&gt;display_name = 'User Administrator'; // optional
$admin-&gt;description  = 'User is allowed to manage and edit other users'; // optional
$admin-&gt;save();
</pre> 
           <hr> 
           <h2><a href="https://github.com/laravel/socialite" target="_blank" rel="noreferrer noopener">Socialite</a></h2> 
           <p>Socialite offers a simple and easy way to handle OAuth authentication. It makes it possible for your users to log-in via some of the most popular social networks and services including Facebook, Twitter, Google, GitHub and BitBucket. </p> 
           <pre data-type="php">$user = Socialite::driver('github')-&gt;user();

// OAuth Two Providers
$token = $user-&gt;token;
$refreshToken = $user-&gt;refreshToken; // not always provided
$expiresIn = $user-&gt;expiresIn;

// All Providers
$user-&gt;getId();
$user-&gt;getName();
$user-&gt;getEmail();
$user-&gt;getAvatar();
</pre> 
           <hr> 
           <h2><a href="https://github.com/jrean/laravel-user-verification" target="_blank" rel="noreferrer noopener">User Verification</a></h2> 
           <p>A package that allows you to verify users and validate emails. It generates and stores a verification token for the registered user, sends an email with the verification token link, handles the token verification, and sets the user as verified.</p> 
           <pre data-type="php">protected function validator(array $data)
{
    return Validator::make($data, [
        'name' =&gt; 'required|max:255',
        'email' =&gt; 'required|email|max:255|unique:users',
        'password' =&gt; 'required|min:6|confirmed',
    ]);
}
</pre> 
           <hr> 
           <h2><a href="https://github.com/laravel/tinker" target="_blank" rel="noreferrer noopener">Tinker</a></h2> 
           <p>Tinker allows you to interact with your entire Laravel application from the command line and access all the Eloquent jobs, events, and objects. It used to be part of Laravel, but after version 5.4 it is in a optional add-on that needs to be installed separately. </p> 
           <hr> 
           <h2><a href="https://github.com/davejamesmiller/laravel-breadcrumbs" target="_blank" rel="noreferrer noopener">Breadcrumbs</a></h2> 
           <p>With this package you can create breadcrumb page controls in a simple and easy way. It supports some of the most popular front-end frameworks such as Bootstrap, Bulma, Foundation and Materialize.</p> 
           <pre data-type="php">// Home &gt; Photos
Breadcrumbs::register('photo.index', function ($breadcrumbs) {
    $breadcrumbs-&gt;parent('Home');
    $breadcrumbs-&gt;push('Photos', route('photo.index'));
});

//  Home &gt; Photos &gt; Upload Photo
Breadcrumbs::register('photo.create', function ($breadcrumbs) {
    $breadcrumbs-&gt;parent('photo.index');
    $breadcrumbs-&gt;push('Upload Photo', route('photo.create'));
});
</pre> 
           <hr> 
           <h2><a href="https://github.com/cviebrock/eloquent-sluggable" target="_blank" rel="noreferrer noopener">Eloquent-Sluggable</a></h2> 
           <p>Slugging is creating a simplified, URL-friendly version of a string by converting it to one case and removing spaces, accented letters, ampersands, etc. With Eloquent-Sluggable you can easily create slugs for all the Eloquent models in your project. </p> 
           <pre data-type="php">class Post extends Eloquent
{
    use Sluggable;
    protected $fillable = ['title'];
    public function sluggable() {
        return [
            'slug' =&gt; [
                'source' =&gt; ['title']
            ]
        ];
    }
}

$post = new Post([
    'title' =&gt; 'My Awesome Blog Post',
]);
// $post-&gt;slug is "my-awesome-blog-post"
</pre> 
           <hr> 
           <h2><a href="https://github.com/Xethron/migrations-generator" target="_blank" rel="noreferrer noopener">Migrations Generator</a></h2> 
           <p>A Laravel package which can be used to generate migrations from an existing database, including indexes and foreign keys. When you run the following command you can create migrations for all the tables in your database.</p> 
           <pre data-type="php"> php artisan migrate:generate</pre> 
           <p>You can also choose only certain tables that you want to use:</p> 
           <pre data-type="php"> php artisan migrate:generate table1,table2 </pre> 
           <hr> 
           <h2><a href="https://github.com/anhskohbo/no-captcha" target="_blank" rel="noreferrer noopener">NoCaptcha</a></h2> 
           <p>Laravel 5 package for implementing Google's <a href="https://www.google.com/recaptcha/intro/android.html#" target="_blank" rel="noreferrer noopener">reCAPTCHA</a> "I'm not a robot" validation and protecting your forms from spam. To use the service you will need to obtain a free API key.</p> 
           <pre data-type="php">// prevent validation error on captcha
NoCaptcha::shouldReceive('verifyResponse')
    -&gt;once()
    -&gt;andReturn(true);
// provide hidden input for your 'required' validation
NoCaptcha::shouldReceive('display')
    -&gt;zeroOrMoreTimes()
    -&gt;andReturn('<input type="hidden" name="g-recaptcha-response">');
    </pre> 
           <hr> 
           <h2><a href="https://github.com/svenluijten/artisan-view" target="_blank" rel="noreferrer noopener">Artisan View</a></h2> 
           <p>A command line utility that adds a number of Artisan commands for working with the views in your app. It allows you to automatically generate view templates without having to manually create new blade files.</p> 
           <pre data-type="php"># Create a view 'index.blade.php' in the default directory
$ php artisan make:view index

# Create a view 'index.blade.php' in a subdirectory ('pages')
$ php artisan make:view pages.index

# Add a section to the view
$ php artisan make:view index --section=content
</pre> 
           <hr> 
           <h2><a href="https://github.com/spatie/laravel-backup" target="_blank" rel="noreferrer noopener">Laravel Backup</a></h2> 
           <p>With this Laravel package you can back up all of the files in your project. All you need to do is run this command:</p> 
           <pre data-type="php">php artisan backup:run
</pre> 
           <p>It creates a zipfile with all the files in the directory and a dump of your database. Can be stored on any file system. </p> 
           <hr> 
           <h2><a href="https://github.com/barryvdh/laravel-cors" target="_blank" rel="noreferrer noopener">CORS Middleware</a></h2> 
           <p>Setting up CORS (Cross-Origin Resource Sharing Headers) on your website can be a lot of work. With this Laravel library the configuration process is very simplified. It handles CORS pre-flight options requests and adds CORS headers to your responses</p> 
           <pre data-type="php">return [
    'supportsCredentials' =&gt; false,
    'allowedOrigins' =&gt; ['*'],
    'allowedHeaders' =&gt; ['Content-Type', 'X-Requested-With'],
    'allowedMethods' =&gt; ['*'], // ex: ['GET', 'POST', 'PUT',  'DELETE']
    'exposedHeaders' =&gt; [],
    'maxAge' =&gt; 0,
]
</pre> 
           <hr> 
           <h2><a href="https://github.com/Folkloreatelier/laravel-graphql" target="_blank" rel="noreferrer noopener">Laravel GraphQL</a></h2> 
           <p>GraphQL is a data query language that provides an alternative to traditional REST architectures. Developers define the structure of the data required, and get exactly the same structure from the server. This package will help you set up and use GraphQL in your Laravel apps.</p> 
           <hr> 
           <h2><a href="https://github.com/JeffreyWay/laravel-mix" target="_blank" rel="noreferrer noopener">Laravel Mix</a></h2> 
           <p>Laravel Mix provides a rich API for defining Webpack build steps for your project. It uses a few common CSS and JavaScript pre-processors that can be chained together to transform and format your assets. </p> 
           <pre data-type="php">mix.js('resources/assets/js/app.js', 'public/js')
   .sass('resources/assets/sass/app.scss', 'public/css');
</pre> 
           <hr> 
           <h2><a href="https://github.com/laracasts/Laravel-5-Generators-Extended" target="_blank" rel="noreferrer noopener">Laravel Extended Generators</a></h2> 
           <p>A library by the team from <a href="https://laracasts.com/" target="_blank" rel="noreferrer noopener">Laracasts</a> that offers a number of generators that can save you a lot of time when developing your project. It allows you to quickly setup new models, views, controllers, migrations, seeds, and more.</p>
