        <p class="post-date"> <a href="http://blog.spoonx.nl/updates-changes-and-new-aurelia-plugins/" title="11:51">October 5, 2016</a> </p> 
        <h2 class="post-title">Updates, changes and new aurelia plugins</h2> 
       </div> 
       <!-- /post-header --> 
       <div class="post-content"> 
        <p>Hello coders,</p> 
        <p>Today we’re releasing a stable version for almost all of our <a href="https://github.com/SpoonX?utf8=%E2%9C%93&amp;query=aurelia-" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/SpoonX?utf8=%E2%9C%93&amp;query=aurelia-', 'aurelia plugins']);">aurelia plugins</a>, a couple new plugins and a preview-version of our new Node.js ORM <a href="https://github.com/SpoonX/wetland" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/SpoonX/wetland', 'wetland']);">wetland</a>.</p> 
        <h2>General changes</h2> 
        <p>First we’ll be talking about general changes in our community. It has been growing, so we had to make a couple of changes.</p> 
        <h3>Organisational changes</h3> 
        <p>We’ve hired <a href="https://github.com/doktordirk" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/doktordirk', '@doktordirk']);">@doktordirk</a> as our community manager. He’s responsible for answering issues, reviewing pull requests, answering questions on <a href="https://gitter.im/SpoonX/Dev" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://gitter.im/SpoonX/Dev', 'Gitter']);">Gitter</a>. A big, and finally public welcome to Dirk!</p> 
        <p>We’ve also assigned plugin owners to make sure that plugins keep moving forward. This has proven itself very useful and the plugins now have their own roadmaps (more on that later). <a href="https://github.com/RWOverdijk" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/RWOverdijk', '@RWOverdijk']);">@RWOverdijk</a> as the lead developer is now focussed on the architecture and development of consistency.</p> 
        <h3>Release schedule</h3> 
        <p>We’ll no longer work with release candidates for as long as we have. The next versions will follow semver, and due to our new testing environment be rolled out a lot more frequent.</p> 
        <h2>Big updates</h2> 
        <p>The following is a summary of changes made to existing libraries.</p> 
        <h3><a href="https://aurelia-api.spoonx.org" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://aurelia-api.spoonx.org', 'api']);">api</a></h3> 
        <ul> 
         <li>Migration to npm and name changed spoonx/aurelia-api -&gt; aurelia-api</li> 
         <li>Allow multipart/form-data and x-ww-form-urlencoded</li> 
         <li>Add findOne, updateOne,.. methods to allow for path/id?filter=some</li> 
         <li>Package for esNext/typescript , jspm/webpack/aurelia-cli</li> 
        </ul> 
        <h3><a href="https://aurelia-authentication.spoonx.org" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://aurelia-authentication.spoonx.org', 'authentication']);">authentication</a></h3> 
        <ul> 
         <li>Migration to npm and name changed spoonx/aurelia-authentication-&gt; aurelia-authentication</li> 
         <li>Package for esNext/typescript , jspm/webpack/aurelia-cli</li> 
         <li>Completely refactored for consistency. many parameters renamed. getter/setter for the old names ensure BC</li> 
         <li>AuthService.authenticated status automatically updated according to token ttl. including across tabs and optional auto logout page reload/redirect</li> 
         <li>Binding signal ‘authentication-change’</li> 
         <li>EventAggregator publishes ‘authentication-change’ with authenticated status data</li> 
         <li>Optional function can be set to customize data extraction from non-jwt</li> 
         <li>Refresh_token with optional auto-refresh</li> 
         <li><a href="https://auth0.com" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://auth0.com', 'Auth0']);">Auth0</a> support using auth0-lock</li> 
         <li>OIDC logout support</li> 
         <li>Updated provider defaults</li> 
        </ul> 
        <h3><a href="https://aurelia-orm.spoonx.org" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://aurelia-orm.spoonx.org', 'orm']);">orm</a></h3> 
        <ul> 
         <li>Migration to npm and name changed spoonx/aurelia-orm-&gt; aurelia-orm</li> 
         <li>Package for esNext/typescript , jspm/webpack/aurelia-cli</li> 
         <li>Association-select component much more customizable (translation, placeholder …)</li> 
         <li>Paged component added </li> 
         <li>Add generic metadata with the data decorator</li> 
         <li>Set an entities id property name with the idProperty decorator</li> 
         <li>Entity: Save new children and mark as dirty when children added or removed. (https://github.com/SpoonX/aurelia-orm/commit/37f6b515)</li> 
         <li>Added entity.reset(shallow)</li> 
         <li>Entity: setData with optional markClean (https://github.com/SpoonX/aurelia-orm/commit/ee01480)</li> 
         <li><strong>Validation:</strong> upgrade to aurelia-validation 0.12+</li> 
        </ul> 
        <h3><a href="https://github.com/SpoonX/aurelia-notification" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/SpoonX/aurelia-notification', 'notification']);">notification</a></h3> 
        <ul> 
         <li>Migration to npm and name changed spoonx/aurelia-notification-&gt; aurelia-notification</li> 
         <li>Updated to new aurelia-i18n. BREAKING CHANGE: Cannot be used without aurelia-i18n anymore</li> 
         <li>Package for esNext/typescript , jspm/webpack/aurelia-cli</li> 
        </ul> 
        <h3>all plugins</h3> 
        <ul> 
         <li>Added gitbooks (https://plugin_name.spoonx.org/ e.g. https://aurelia-orm.spoonx.org)</li> 
         <li>All plugins are published on npm</li> 
         <li>Adding basic d.ts files for typescript users (typescript 2 install ready)</li> 
         <li>Made the plugins @easyWebpack ready</li> 
         <li>Added bundle instructions of jspm and aurelia-cli</li> 
        </ul> 
        <h2>New stuff</h2> 
        <p>We’ve also been working on some new stuff! The author of each plugin wrote a little piece on it.</p> 
        <h3>swan-example</h3> 
        <p><strong><em>By <a href="https://github.com/doktordirk" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/doktordirk', '@doktordirk']);">@doktordirk</a> and <a href="https://github.com/RWOverdijk" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/RWOverdijk', '@RWOverdijk']);">@RWOverdijk</a>.</em></strong></p> 
        <p>We needed a place to demo our plugins, and so we created swan-example.</p> 
        <p><a href="https://github.com/SpoonX/swan-cli" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/SpoonX/swan-cli', 'SWAN']);">SWAN</a> is a nifty little stack, which stands for Sails Waterline Aurelia and Node. It comes with a <a href="https://github.com/SpoonX/swan-cli" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/SpoonX/swan-cli', 'cli tool']);">cli tool</a>, which isn’t much yet, but does help you define a project structure and get started quickly using our opinionated <a href="https://github.com/SpoonX/aurelia-skeleton" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/SpoonX/aurelia-skeleton', 'aurelia-skeleton']);">aurelia-skeleton</a> and auth-ready node server. We’ve used this tool to build the swan-example, which you can see <a href="http://swan-example.spoonx.org" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'http://swan-example.spoonx.org', 'by clicking here']);">by clicking here</a>.</p> 
        <ul> 
         <li><strong>Username</strong>: example </li> 
         <li><strong>Password</strong>: example</li> 
        </ul>

        <h3><a href="https://aurelia-view-manager.spoonx.org/" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://aurelia-view-manager.spoonx.org/', 'aurelia-view-manager']);">aurelia-view-manager</a></h3>

    ], appConfigOverwrites);

        <p></p> 
        <p>That’s the main feature of aurelia-config, but there’s more to it.</p> 
        <ul> 
         <li>initialize and configure you plugins in one place with a single namespaced configuration object.</li> 
         <li>automatically load and merge defaults which can be directly loaded from plugins by aurelia-config itself.</li> 
         <li>access the merged config by using our resolver or injecting aurelia-config everywhere.</li> 
         <li>since the merged setting are an instance of homefront, all it’s method are available, too.</li> 
        </ul> 
        <h3><a href="https://github.com/SpoonX/homefront" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/SpoonX/homefront', 'Homefront']);">Homefront</a></h3> 
        <p><strong><em>By <a href="https://github.com/Rawphs" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/Rawphs', '@Rawphs']);">@Rawphs</a>.</em></strong></p> 
        <p>Homefront is a module that allows you to merge, flatten, expand, search in, fetch from, remove from, put in and work with objects easily. It was started as an educational project but has made its way into wetland, aurelia-config and <a href="https://github.com/SpoonX/json-statham" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/SpoonX/json-statham', 'json-statham']);">json-statham</a>. It is fully tested, and makes working with objects a lot more fun.</p> 
        <p><em><a href="https://github.com/SpoonX/homefront" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/SpoonX/homefront', 'Github']);">Github</a></em></p> 
        <h3><a href="https://github.com/SpoonX/aurelia-charts" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/SpoonX/aurelia-charts', 'aurelia-charts']);">aurelia-charts</a></h3> 
        <p><strong><em>By <a href="https://github.com/bas080" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/bas080', '@bas080']);">@bas080</a>.</em></strong></p> 
        <p>Aurelia-charts is a plugin that allows you to use the chart library you like most. It has a simple api and standardizes the way charts are used in your projects. It does so by letting others define sane defaults.</p> 
        <p>It allows for you to write your own plugins, or use an existing one (for example <a href="https://github.com/SpoonX/aurelia-charts-c3" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/SpoonX/aurelia-charts-c3', 'aurelia-charts-c3']);">aurelia-charts-c3</a>).</p> 
        <p>This project is being used by us, <strong>but isn’t labeled “stable”</strong> yet, as is visible by the lack of documentation.</p> 
        <p><em><a href="https://github.com/SpoonX/aurelia-charts" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/SpoonX/aurelia-charts', 'Github']);">Github</a></em></p> 
        <h3>Bonus: <a href="https://github.com/spoonx/wetland" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/spoonx/wetland', 'Wetland']);">Wetland</a></h3> 
        <p><strong><em>By <a href="https://github.com/RWOverdijk" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/RWOverdijk', '@RWOverdijk']);">@RWOverdijk</a>.</em></strong></p> 
        <p>For the past two months we’ve been working on a brand new nodeJS orm. For now it lacks documentation, but has neat features. It’s based on the <a href="https://en.wikibooks.org/wiki/Java_Persistence/What_is_JPA%3F" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://en.wikibooks.org/wiki/Java_Persistence/What_is_JPA%3F', 'JPA spec']);">JPA spec</a> (which took <a href="https://jcp.org/en/jsr/detail?id=338" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://jcp.org/en/jsr/detail?id=338', 'some reading']);">some reading</a>) and took inspiration from the way <a href="http://www.doctrine-project.org/projects/orm.html" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'http://www.doctrine-project.org/projects/orm.html', 'doctrine']);">doctrine</a> implemented it. It obviously required some changes to fit the node.js world, but those were kept to a minimum.</p> 
        <p>A glossary of its features to date:</p> 
        <ul> 
         <li>Abstracted principles (repositories, entities, hydrators)</li> 
         <li>Unit of work using proxies</li> 
         <li>Nested joins</li> 
         <li>Nested persist</li> 
         <li>Mapping</li> 
         <li>Transactions</li> 
         <li>Migrations</li> 
         <li>Great performance (smart hydrator and state management)</li> 
         <li>Strict typing (written in typescript)</li> 
        </ul> 
        <p>Feel free to <a href="https://github.com/spoonx/wetland" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/spoonx/wetland', 'browse the code']);">browse the code</a>, and if you’re interested leave a star so you can track the progress.</p> 
        <p><em><a href="https://github.com/spoonx/wetland" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://github.com/spoonx/wetland', 'Github']);">Github</a></em></p> 
        <h2>Conclusion</h2> 
        <p>We’re very happy with the work we’ve done. The plugins are stable and the documentation is clear.</p> 
        <ul> 
         <li>If you have questions, suggestions or a need for banter, feel free to join us on <a href="https://gitter.im/SpoonX/Dev" onclick="_gaq.push(['_trackEvent', 'outbound-article', 'https://gitter.im/SpoonX/Dev', 'gitter']);">gitter</a>.</li> 
         <li>If you like our work, help us out and star our repositories.</li> 
        </ul>
