      <p>Es común ver en formularios de una aplicación web un campo captcha, es decir, un campo donde el usuario debe introducir los caracteres presentes en una imagen distorsionada; esto con el fin de comprobar que quien esté usando la aplicación sea un humano. Podemos usar la solución que nos ofrece Google con reCAPTCHA, donde el usuario solo debe hacer clic en un checkbox y luego seleccionar un conjunto de imágenes según las condiciones que le solicita. &nbsp;En este tutorial vamos a implementar reCAPTCHA para el login y registro de una aplicación de Laravel 5.2.</p> 
      <p><span id="more-10048"></span></p> 
      <h3>Configuración</h3> 
      <p>Usaremos el paquete <a href="https://github.com/greggilbert/recaptcha" target="_blank">https://github.com/greggilbert/recaptcha</a>&nbsp;el cual&nbsp;funciona para todas las versiones de Laravel 5 (5.0, 5.1 y 5.2) pero para este tutorial sólo vamos a implementarlo en un proyecto de Laravel 5.2 (para las demás versiones se siguen pasos similares).&nbsp;Para ello debemos tener un <a href="https://styde.net/instalacion-y-configuracion-de-laravel-5-2/">proyecto de Laravel 5.2 instalado</a> con el <a href="https://styde.net/registro-login-y-recuperacion-de-clave-con-el-comando-makeauth-en-laravel-5-2/">registro y login que viene incluido en esta versión</a>.</p>
