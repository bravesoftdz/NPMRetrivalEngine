         <h1 class="entry-title entry-title-single">How to configure Visual Studio 2015 with the latest version of Node.js and NPM</h1> 
         <div class="entry-meta"> 
          <span class="entry-author author vcard"><a href="https://blog.josequinto.com/author/jquinto/" title="by José Quinto Zamora" rel="author">José Quinto Zamora</a></span>
          <span class="entry-meta-sep"> ? </span>
          <span class="comments-link"><a href="https://blog.josequinto.com/2016/05/04/how-to-configure-visual-studio-2015-integration-with-latest-version-of-node-js-and-npm/#comments">1 Comment</a></span> 
         </div>

       <div class="entry-content-wrapper-bottom clearfix"> 
        <div class="entry-content clearfix"> 
         <p>Hi,</p> 
         <p>I recently installed Visual Studio 2015 Update 2 in order to work in a project, in which, we would like to use a combination of&nbsp; different technologies like SharePoint stuff, React, Node.js, LESS and Gulp.</p> 
         <p><a href="https://i2.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/image.png?ssl=1"><img style="background-image: none; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-width: 0px;" title="image" src="https://i1.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/image_thumb.png?resize=241%2C153&amp;ssl=1" alt="image" border="0" data-recalc-dims="1"></a></p> 
         <p>In my opinion, is good to to use new technologies at the right time, in terms of, learning curve, adoption and maturity. And now, I think is the right time for SharePoint developers to introduce little more technologies like Node.js and Gulp in our projects, which will help us on saving lot of development time.</p> 
         <p>&nbsp;</p> 
         <h2>Node.js</h2> 
         <p>I was quite surprised when I first saw<strong> Node.js</strong> because even if it is only a <strong>JavaScript runtime</strong> implemented to run in a server. It has huge success in the development world because JavaScript is the common and shared language between all web developers. Which means that there is a lot of people involved on developing custom libraries and modules. Due to its success, there is a <strong>great package ecosystem called NPM</strong> (<strong>Node Package Manager</strong>) to help contributors to publish libraries and developers to use them. See more info here: <a href="https://nodejs.org">https://nodejs.org</a>. I recommend to read this book as well:<br> <a href="https://www.amazon.com/Beginning-Node-js-Basarat-Ali-Syed-ebook/dp/B00MBL25GI/ref=as_li_ss_il?_encoding=UTF8&amp;me=&amp;linkCode=li2&amp;tag=joseq21-20&amp;linkId=8ccb5c3a233ab57189e1d5e3641a945d" target="_blank"><img border="0" src="//ws-na.amazon-adsystem.com/widgets/q?_encoding=UTF8&amp;ASIN=B00MBL25GI&amp;Format=_SL160_&amp;ID=AsinImage&amp;MarketPlace=US&amp;ServiceVersion=20070822&amp;WS=1&amp;tag=joseq21-20"></a><img src="https://ir-na.amazon-adsystem.com/e/ir?t=joseq21-20&amp;l=li2&amp;o=1&amp;a=B00MBL25GI" width="1" height="1" border="0" alt="" style="border:none !important; margin:0px !important;"><br> &nbsp;</p> 
         <h2>NPM</h2> 
         <p>It is the <strong>largest ecosystem of open libraries</strong> in the world, doesn’t matter if you are Java, PHP, C#, VB or ASP .Net Developer! That means that there is a los of modules and open libraries already implemented just ready to use them.</p> 
         <p>As a SharePoint and Web Developer one example that will make you to love NPM free libraries is <a href="http://gulpjs.com/" target="_blank">Gulp</a>, that is&nbsp; a tool that helps you out with several tasks when it comes to web development. It’s often used to do front end tasks like:</p> 
         <ul> 
          <li><strong>Spinning up a web server</strong> (not needed IIS, …)</li> 
          <li><strong>Reloading the browser</strong> automatically whenever a file is saved</li> 
          <li>Using CSS pre-processors like <strong>SASS</strong> or <strong>LESS</strong> (<a title="https://www.npmjs.com/package/gulp-less" href="https://www.npmjs.com/package/gulp-less">https://www.npmjs.com/package/gulp-less</a>)</li> 
          <li><strong>Optimizing assets like CSS</strong>, JavaScript, and images (<a title="https://www.npmjs.com/package/gulp-csso" href="https://www.npmjs.com/package/gulp-csso">https://www.npmjs.com/package/gulp-csso</a>)</li> 
          <li><strong>Upload files to SharePoint</strong> libraries automatically (<a title="https://www.npmjs.com/package/gulp-spsave" href="https://www.npmjs.com/package/gulp-spsave">https://www.npmjs.com/package/gulp-spsave</a>) 
           <ul> 
            <li>You can see a good article of my colleague Vardhaman explaining how to use <a href="http://www.vrdmn.com/2016/05/simple-bundle-minify-and-upload-js-to.html" target="_blank">here</a>.</li> 
           </ul> </li> 
         </ul> 
         <p><strong>YES</strong>!! We can have a good way to automatize all the web compilers, bundlers, optimizers and uploaders when we work in SharePoint projects, which is <strong>really good in terms of productivity</strong>.</p> 
         <p>&nbsp;</p> 
         <p>&nbsp;</p> 
         <h2>Visual Studio 2015 Integration with NPM</h2> 
         <p>By default Visual Studio 2015 knows what NPM is, I mean, <strong>Visual Studio 2015</strong> ships with it’s own version of Node.js and NPM. The OOTB version of NPM is located here: <strong>C:\Program Files (x86)\Microsoft Visual Studio 14.0\Common7\IDE\Extensions\Microsoft\Web Tools\External\npm\node_modules\npm\bin</strong></p> 
         <p><a href="https://i2.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/image-1.png?ssl=1"><img style="background-image: none; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-width: 0px;" title="image" src="https://i0.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/image_thumb-1.png?resize=580%2C237&amp;ssl=1" alt="image" border="0" data-recalc-dims="1"></a></p> 
         <p>The bad thing here, is that VS uses an OLD version of NPM, and I <strong>HIGHLY recommend</strong> to use latest version of Node.js and NPM. For me the primary motivator was the path length limitations in Windows. Nested<strong> node_modules</strong> folders buried 19 levels deep is no fun when you hit the max path length. The latest major version of<strong> NPM– version 3.0.x and above – creates a flat store of packages</strong>.</p> 
         <p>Using native VS 2015 Update 2 NPM version</p> 
         <p><a href="https://i2.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/clip_image001.png?ssl=1"><img style="background-image: none; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-width: 0px;" title="clip_image001" src="https://i0.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/clip_image001_thumb.png?resize=345%2C196&amp;ssl=1" alt="clip_image001" border="0" data-recalc-dims="1"></a></p> 
         <p>Using <strong>NPM 3.8.6</strong> downloaded from nodejs.org (installed via Node.js executable)</p> 
         <p><a href="https://i1.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/clip_image002.png?ssl=1"><img style="background-image: none; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-width: 0px;" title="clip_image002" src="https://i0.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/clip_image002_thumb.png?resize=345%2C189&amp;ssl=1" alt="clip_image002" border="0" data-recalc-dims="1"></a></p> 
         <p>Thanks <a href="https://twitter.com/spcfran" target="_blank">Fran</a> for helping me on this!</p> 
         <p>&nbsp;</p> 
         <h2>Upgrade NPM in Visual Studio 2015</h2> 
         <p>1. Install in your development environment the latest version of <strong>Node.js that includes NPM</strong>: <a href="https://nodejs.org">https://nodejs.org</a>.</p> 
         <p>In my case (May 2016) it is v<strong>6.0.0.</strong></p> 
         <p><a href="https://i2.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/image-2.png?ssl=1"><img style="background-image: none; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-width: 0px;" title="image" src="https://i1.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/image_thumb-2.png?resize=323%2C104&amp;ssl=1" alt="image" border="0" data-recalc-dims="1"></a></p> 
         <p>Check that NPM is installed as well:</p> 
         <p><a href="https://i0.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/image-3.png?ssl=1"><img style="background-image: none; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-width: 0px;" title="image" src="https://i1.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/image_thumb-3.png?resize=485%2C378&amp;ssl=1" alt="image" border="0" data-recalc-dims="1"></a></p> 
         <p>After install it, you can see the version for both tools, just typing:</p> 
         <p><a href="https://i2.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/image-4.png?ssl=1"><img style="background-image: none; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-width: 0px;" title="image" src="https://i0.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/image_thumb-4.png?resize=282%2C106&amp;ssl=1" alt="image" border="0" data-recalc-dims="1"></a></p> 
         <p>&nbsp;</p> 
         <p>2. Now change “<strong>Locations of External Tools</strong>” configurations in VS 2015 in order to tell VS to use the latest version of NPM:</p> 
         <p>Configure it in<strong> Options &gt; Project and Solutions &gt; External Web Tools</strong></p> 
         <p>Just add a new entry and move it to the top. NPM is located in the Node.js install directory:</p> 
         <ol> 
          <ol> 
           <li><strong>C:\Program Files (x86)\nodejs</strong> or</li> 
           <li><strong>C:\Program Files\nodejs</strong> (if 64 bits of Node.js installed)</li> 
          </ol> 
         </ol> 
         <p><a href="https://i2.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/clip_image0015.png?ssl=1"><img style="background-image: none; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-width: 0px;" title="clip_image001[5]" src="https://i2.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/clip_image0015_thumb.png?resize=580%2C340&amp;ssl=1" alt="clip_image001[5]" border="0" data-recalc-dims="1"></a></p> 
         <p>Note may need to <strong>restart Visual Studio</strong> for the tooling paths to be picked up.</p> 
         <p>&nbsp;</p> 
         <p>&nbsp;</p> 
         <h2>How can we check our VS 2015 is using version 3.8.6</h2> 
         <p><strong>VS 2015</strong> automatically runs NPM every time we save changes in the <strong>package.json</strong> file at the root folder of our project (SharePoint project as well). Try to crate <a href="https://docs.npmjs.com/files/package.json" target="_blank">package.json</a> in your project root directory, save it and you will see in the output called “Bower/npm”:</p> 
         <p><a href="https://i1.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/image-5.png?ssl=1"><img style="background-image: none; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-width: 0px;" title="image" src="https://i2.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/image_thumb-5.png?resize=580%2C464&amp;ssl=1" alt="image" border="0" data-recalc-dims="1"></a></p> 
         <p>Here an example of package.json file:</p> 
         <p><script src="https://gist.github.com/jquintozamora/5f1dd8c277feacace8040c4b7855ad3f.js"></script></p> 
         <p>You also can see how “<strong>node_modules</strong>” folder has been created in your project folder:</p> 
         <p><a href="https://i0.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/clip_image0017.png?ssl=1"><img style="background-image: none; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-width: 0px;" title="clip_image001[7]" src="https://i2.wp.com/blog.josequinto.com/wp-content/uploads/2016/05/clip_image0017_thumb.png?resize=580%2C235&amp;ssl=1" alt="clip_image001[7]" border="0" data-recalc-dims="1"></a></p> 
         <p>&nbsp;</p> 
         <h2>Recap</h2> 
         <p>– If you are using<strong> NPM version 3.X.X</strong> then you should have a flat store of packages in <strong>node_modules</strong>, which means no more node_modules subdirectories. You should appreciate a improvement on performance,</p> 
         <p>– You don’t need extra configuration to run NPM, the only thing to do is <strong>create package.json</strong> file in you root directory and when you save it an event is triggered to evaluate using the pre-configured NPM version.</p> 
         <p>– NPM is really good accepted in the development community and you have a lot of information around Internet, just google it!</p> 
         <p>– There are some good articles and courses from&nbsp; John Papa: <a title="http://www.johnpapa.net/get-up-and-running-with-node-and-visual-studio/" href="http://www.johnpapa.net/get-up-and-running-with-node-and-visual-studio/">http://www.johnpapa.net/get-up-and-running-with-node-and-visual-studio/</a></p> 
         <p>&nbsp;</p> 
         <p>&nbsp;</p> 
         <p>Regards!</p>
