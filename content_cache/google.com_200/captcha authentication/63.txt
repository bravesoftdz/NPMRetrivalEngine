        <h1 class="entry-title">OIM 11g R2 Self Registration with CAPTCHA</h1> 
        <div class="post-info">
         <span class="date published time" title="2013-05-17T09:58:40+00:00">May 17, 2013</span> by 
         <span class="author vcard"><span class="fn"><a href="http://www.ateam-oracle.com/author/daniel-gralewski/" rel="author">Daniel Gralewski</a></span></span> 
         <span class="post-comments"><a href="http://www.ateam-oracle.com/oim-11g-r2-self-registration-with-captcha/#comments">2 Comments</a></span> 
        </div>
        <div class="entry-content">
         <p>This post walks you through the fun of customizing OIM and adding a CAPTCHA solution to the self-registration page. Captcha solutions are largely used in web sites to try to prevent automated robots from registering, filling forms, sending messages and many other things.</p> 
         <p>The captcha solution used is Simple Captcha and it is <a href="http://simplecaptcha.sourceforge.net/" target="_blank">available here</a>. It is easy to use and easy to hook into applications.</p> 
         <p>This is another post of the Oracle Identity Manager Academy. To check other tricks, tips and examples you can <a title="Oracle Identity Manager Academy" href="http://www.ateam-oracle.com/oracle-identity-manager-academy-from-the-fusion-security-blog/" target="_blank">find the academy post here</a>.<span id="more-13424"></span></p> 
         <h3>Deploying the captcha solution</h3> 
         <p>Unfortunately, a usually non-recommended step is required to hook the captcha solution into OIM: the capctha servlet must be deployed to the OIM self-service console web application. To do so, you have to:</p> 
         <ul> 
          <li>Make a backup of $OIM_ORACLE_HOME/server/apps/oracle.iam.console.identity.self-service.ear</li> 
          <li>Copy the file <i>oracle.iam.console.identity.self-service.ear</i> to a temporary location</li> 
          <li>Using a ZIP tool like 7zip, open the <i>oracle.iam.console.identity.self-service.ear</i></li> 
          <li>Then open <i>oracle.iam.console.identity.self-service.war</i></li> 
          <li>Then navigate to <i>WEB-INF</i> folder and add the following to the <i>web.xml</i> file:</li> 
         </ul> 
         <pre class="brush: xml; gutter: true">    &lt;servlet-name&gt;CaptchaServlet&lt;/servlet-name&gt;
    &lt;servlet-class&gt;nl.captcha.servlet.SimpleCaptchaServlet&lt;/servlet-class&gt;
    &lt;init-param&gt;
            &lt;param-name&gt;width&lt;/param-name&gt;
            &lt;param-value&gt;250&lt;/param-value&gt;
    &lt;/init-param&gt;
    &lt;init-param&gt;
       &lt;param-name&gt;height&lt;/param-name&gt;
       &lt;param-value&gt;75&lt;/param-value&gt;
    &lt;/init-param&gt;
  &lt;/servlet&gt;</pre> 
         <p>You also need to add the captcha jar file (simplecaptcha-1.2.1.jar) into the <i>oracle.iam.console.identity.self-service.war</i> <i>lib</i> folder (if the folder does not exist, just create it).</p> 
         <p>Then copy the file <i>oracle.iam.console.identity.self-service.ear</i> over the original one and restart WebLogic (you may need to re-deploy the oracle.iam.ui.self-service application in the WebLobig console).</p> 
         <h3>Customizing OIM self registration page</h3> 
         <p>Before starting this section, it is important to mention that the post assumes you have some knowledge of OIM UI customization features. If you dont, you may want to spend some time in the following posts:</p> 
         <ul> 
          <li><a style="-webkit-text-size-adjust: auto; -webkit-text-stroke-width: 0px; background-color: white; color: #336699; font-family: Verdana, Arial, sans-serif; font-size: 13px; font-style: normal; font-variant: normal; font-weight: normal; letter-spacing: normal; line-height: 16.890625px; orphans: auto; text-align: left; text-indent: 0px; text-transform: none; white-space: normal; widows: auto; word-spacing: 0px;" href="http://fusionsecurity.blogspot.com/2012/09/oim-11g-r2-ui-customization.html" target="_blank">OIM 11g R2 UI Customization</a></li> 
          <li><a style="-webkit-text-size-adjust: auto; -webkit-text-stroke-width: 0px; background-color: white; color: #336699; font-family: Verdana, Arial, sans-serif; font-size: 13px; font-style: normal; font-variant: normal; font-weight: normal; letter-spacing: normal; line-height: 16.890625px; orphans: auto; text-align: left; text-indent: 0px; text-transform: none; white-space: normal; widows: auto; word-spacing: 0px;" href="http://fusionsecurity.blogspot.com/2012/09/oim-11g-r2-catalog-customization-example.html" target="_blank">OIM 11g R2 Catalog Customization Example</a></li> 
         </ul> 
         <p>There are a couple of tricks to use to customize the self registration page:</p> 
         <ul> 
          <li>To direct access the self registration page you can type in the browser url: <i>http://&lt;OIM_HOST&gt;:&lt;OIM_PORT&gt;:/identity/faces/register</i>. This URL works with and without authentication, but OIM will prevent you from submitting a request for self registration if you are authenticated.</li> 
          <li>When on the page, make sure you provide values to the mandatory fields before trying to customize it.</li> 
         </ul> 
         <h4>1. The managed bean</h4> 
         <p>There is a need to create and deploy a managed bean. This managed bean must be deployed into the oracle.iam.ui.custom-dev-starter-pack.war placeholder library. In this example, the managed bean is&nbsp; called captchaBean. The managed bean code:</p> 
         <pre class="brush: java; gutter: true">package com.oracle.demo.iam.captcha.view;

import javax.faces.application.FacesMessage;
import javax.faces.component.UIComponent;
import javax.faces.context.ExternalContext;
import javax.faces.context.FacesContext;
import javax.faces.event.ActionEvent;
import javax.faces.validator.ValidatorException;

import nl.captcha.Captcha;

import oracle.adf.view.rich.component.rich.output.RichImage;

public class CaptchaBean {

    private UIComponent rootPanel;
    private UIComponent captchaImage;

    public CaptchaBean() {
        super();
    }

    public void refreshCaptchaActionListener(ActionEvent e) {

        RichImage image = (RichImage)captchaImage;
        String randomString= String.valueOf(Math.random());
        image.setSource("/captcha?rand="+randomString);           

        FacesUtils.partialRender(rootPanel);
    }

    public void verifyCaptchaValue(FacesContext facesContext, UIComponent uiComponent, Object compValue) {

        FacesContext fctx = FacesContext.getCurrentInstance();

        ExternalContext ectx = fctx.getExternalContext();

        Captcha captcha = (Captcha)ectx.getSessionMap().get(Captcha.NAME);
        String answer = (String)compValue;

        if (!captcha.isCorrect(answer)) {
            throw new ValidatorException(this.getFacesMessage("Please provide a correct answer"));
        }
    }

    private FacesMessage getFacesMessage(String msg) {

        FacesMessage message = new FacesMessage();
        message.setDetail(msg);
        message.setSummary(msg);
        message.setSeverity(FacesMessage.SEVERITY_ERROR);

        return message;
    }

    public void setRootPanel(UIComponent rootPanel) {
        this.rootPanel = rootPanel;
    }

    public UIComponent getRootPanel() {
        return rootPanel;
    }

    public void setCaptchaImage(UIComponent captchaImage) {
        this.captchaImage = captchaImage;
    }

    public UIComponent getCaptchaImage() {
        return captchaImage;
    }
}</pre> 
         <p>A few comments about the code:</p> 
         <ul> 
          <li>The method <i>verifyCaptchaValue</i> will be invoked by ADF when the self-registration form is submitted. The property <i>validator</i> in an input text filed indicates which method ADF will invoked. If the method raises the <i>ValidatorException</i>, ADF will catch it and show the <i>Please provide a correct answer</i> message on the page.</li> 
          <li>The method <i>refreshCaptchaActionListener</i> will be invoked when the user clicks on a button to refresh the captcha image. It actually changes the image URL by adding a random string to the end, this is required to make sure that the image gets refreshed.</li> 
          <li>This code uses the class <i>FacesUtils</i>, such class <a href="http://docs.oracle.com/cd/E27559_01/dev.1112/e27150/facesutils.htm#OMDEV5216" target="_blank">can be found here</a></li> 
         </ul> 
         <p>The managed bean declaration is done in the <i>adfc-config.xml</i> file:</p> 
         <pre class="brush: xml; gutter: true">&lt;?xml version="1.0" encoding="windows-1252" ?&gt;
&lt;adfc-config xmlns="http://xmlns.oracle.com/adf/controller" version="1.2"&gt;
  &lt;managed-bean id="__222"&gt;
    &lt;managed-bean-name id="__4"&gt;captchaBean&lt;/managed-bean-name&gt;
    &lt;managed-bean-class id="__1"&gt;com.oracle.demo.iam.captcha.view.CaptchaBean&lt;/managed-bean-class&gt;
    &lt;managed-bean-scope id="__3"&gt;backingBean&lt;/managed-bean-scope&gt;
  &lt;/managed-bean&gt;
&lt;/adfc-config&gt;</pre> 
         <p>The managed bean must be deployed before proceeding with the UI customizations because the methods will be referenced from the custom UI components.</p> 
         <h4>2. Adding the UI components</h4> 
         <h4></h4> 
         <p>There are two main UI components to be added through UI customization using the composer: a texfield where end users will enter the captch value, and a button to refresh the captcha image. There is also a need for a couple of layout components to organize the button and the textfield in the self registration page.</p> 
         <p>2.1 The first step is to create and activate a sandbox that will hold the UI customizations.</p> 
         <p>2.2 With the sandbox created and actived, hit the <i>customize</i> OIM link on the top right. Then on your browser enter the self registration URL mentioned above.</p> 
         <p>2.3 Customize the page by adding the following elements under the <i>panelHeader: User Registration</i> component</p> 
         <ul> 
          <li><i>Web Components</i>  <i>Show Detail Header</i></li> 
          <li><i>Web Components</i>  <i>Spacer</i></li> 
         </ul> 
         <p>2.4&nbsp; Edit the <i>Spacer</i> and set <i>Height</i> to 10 and clean up the <i>Width</i> value</p> 
         <p>2.5 Edit the <i>showDetailHeader:Header</i> component and set the <i>Text</i> property to <i>Are you Human?</i></p> 
         <p>2.6 Edit the <i>panelHeader: User Registration</i> and re-order the child components, the order should be as the one shown in the picture below:</p> 
         <p><a href="http://www.ateam-oracle.com/wp-content/uploads/2013/05/captcha3-ordering.png"><img class="alignnone size-full wp-image-13430" alt="captcha3-ordering" src="http://www.ateam-oracle.com/wp-content/uploads/2013/05/captcha3-ordering.png" width="1227" height="806" srcset="http://www.ateam-oracle.com/wp-content/uploads/2013/05/captcha3-ordering.png 1227w, http://www.ateam-oracle.com/wp-content/uploads/2013/05/captcha3-ordering-300x197.png 300w, http://www.ateam-oracle.com/wp-content/uploads/2013/05/captcha3-ordering-1024x672.png 1024w" sizes="(max-width: 1227px) 100vw, 1227px"></a></p> 
         <p>2.7 Click on <i>showDetailHeader:Header</i> and add a <i>Web Components  Panel Form Layout</i> to it</p> 
         <p>2.8 Edit the <i>panelFormLayout</i> and set the <i>Binding</i> property to <i>#{backingBeanScope.captchaBean.rootPanel}</i>.</p> 
         <p>2.9 Still with the <i>panelFormLayout </i>selected, add a <i>Web Component</i>s  <i>Input Text</i> and a <i>Web Component</i>s  <i>Command Toolbar Button</i></p> 
         <p>2.10 Edit the <i>inputText: Label</i>, set the <i>Label</i> to <i>Enter the code above</i>, and click on the <i>Required</i> checkbox. The <i>Required</i> flag tells ADF to validate the field before the form is submitted.</p> 
         <p>2.11 Edit the <i>commandToolbarButton</i>, set the <i>Text</i> to <i>Cannot see? Refresh Image</i> and click on the <i>Immediate</i> checkbox. The <i>Immediate</i> flag tells that a partial form submit is allowed.</p> 
         <p>At this point, the page should be like the one below:</p> 
         <p><a href="http://www.ateam-oracle.com/wp-content/uploads/2013/05/captcha4-half-way.png"><img class="alignnone size-full wp-image-13431" alt="captcha4-half-way" src="http://www.ateam-oracle.com/wp-content/uploads/2013/05/captcha4-half-way.png" width="1227" height="806" srcset="http://www.ateam-oracle.com/wp-content/uploads/2013/05/captcha4-half-way.png 1227w, http://www.ateam-oracle.com/wp-content/uploads/2013/05/captcha4-half-way-300x197.png 300w, http://www.ateam-oracle.com/wp-content/uploads/2013/05/captcha4-half-way-1024x672.png 1024w" sizes="(max-width: 1227px) 100vw, 1227px"></a></p> 
         <p>2.12 Close the customization by clicking on the link on the top right and navigate back to the identity self service console</p> 
         <h4>3. Manually customizing UI components</h4> 
         <p>OIM UI customization features does not expose all ADF components attributes, so there is a need to do some manual intervention to get the components configured.</p> 
         <p>3.1 De-activate the sandbox and export it</p> 
         <p>3.2 Open the sandbox zip file and locate the following file: <i>oracle\iam\ui\unauthenticated\selfregistration\pages\mdssys\cust\site\site\user-self-registration.jsff.xml</i>. Open it in a text editor, it should show similar to the one below:</p> 
         <p><a href="http://www.ateam-oracle.com/wp-content/uploads/2013/05/sandbox-1-1.png"><img class="alignnone size-full wp-image-13432" alt="sandbox-1-1" src="http://www.ateam-oracle.com/wp-content/uploads/2013/05/sandbox-1-1.png" width="1186" height="709" srcset="http://www.ateam-oracle.com/wp-content/uploads/2013/05/sandbox-1-1.png 1186w, http://www.ateam-oracle.com/wp-content/uploads/2013/05/sandbox-1-1-300x179.png 300w, http://www.ateam-oracle.com/wp-content/uploads/2013/05/sandbox-1-1-1024x612.png 1024w" sizes="(max-width: 1186px) 100vw, 1186px"></a></p> 
         <p>3.3 The first thing to be added to the file above, is the actual captcha image. You can do this by adding the following under the <i>af:panelFormLayout</i> component:</p> 
         <pre class="brush: xml; gutter: true">&lt;af:image xmlns:af="http://xmlns.oracle.com/adf/faces/rich" 
             source="/captcha" 
             id="icaptcha1" 
             inlineStyle="width:250px; height:75.0px;"
             binding="#{backingBeanScope.captchaBean.captchaImage}"/&gt;</pre> 
         <p>3.4 Add the action listener property to the <i>af:commandToolbarButton</i>: <i>actionListener=#{backingBeanScope.captchaBean.refreshCaptchaActionListener}</i></p> 
         <p>3.5 Add the <i>validator</i> property to the <i>af:inputText</i>:<i> validator=#{backingBeanScope.captchaBean.verifyCaptchaValue}</i></p> 
         <p>3.6 Add the <i>disclosed</i> property to <i>af:showDetailHeader</i>: disclosed=true</p> 
         <p>At this point the file should be similar to the image below:</p> 
         <p><a href="http://www.ateam-oracle.com/wp-content/uploads/2013/05/sandbox-2-2.png"><img class="alignnone size-full wp-image-13433" alt="sandbox-2-2" src="http://www.ateam-oracle.com/wp-content/uploads/2013/05/sandbox-2-2.png" width="1189" height="694" srcset="http://www.ateam-oracle.com/wp-content/uploads/2013/05/sandbox-2-2.png 1189w, http://www.ateam-oracle.com/wp-content/uploads/2013/05/sandbox-2-2-300x175.png 300w, http://www.ateam-oracle.com/wp-content/uploads/2013/05/sandbox-2-2-1024x597.png 1024w" sizes="(max-width: 1189px) 100vw, 1189px"></a></p>

         <h4>4. Testing</h4>
