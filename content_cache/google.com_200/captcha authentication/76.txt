        <h1 id="accounts.registerJS-Description">Description</h1>
        <p>This method registers a new user at your site, in accordance with&nbsp;the predefined site&nbsp;<a href="/display/GD/Policies">Policies&nbsp;</a>and the&nbsp;<a href="/display/GD/accounts.setSchema+REST">Schema</a>&nbsp;of the&nbsp;<a href="/display/GD/Accounts+API#AccountsAPI-AccountsStorage">Accounts Storage</a>. On-s<span style="font-size: 14.0px;line-height: 18.0px;">ite registration requires three API calls: 1.&nbsp;<a href="/display/GD/accounts.initRegistration+JS">accounts.initRegistration</a>, 2. accoun</span><span style="font-size: 14.0px;line-height: 18.0px;">ts.register</span><span style="font-size: 14.0px;line-height: 18.0px;">&nbsp;and 3.&nbsp;</span><a href="/display/GD/accounts.finalizeRegistration+JS">accounts.finalizeRegistration</a><span style="font-size: 14.0px;line-height: 18.0px;">.&nbsp;</span><span style="font-size: 14.0px;">If the&nbsp;</span><em>finalizeRegistration</em><span style="font-size: 14.0px;"><em>&nbsp;</em>parameter of accounts.register is set to 'true' then there is no need to call accounts.</span>finalizeRegistration.&nbsp;</p>
        <p><span style="font-size: 14.0px;line-height: 18.0px;">A</span><span style="font-size: 14.0px;line-height: 1.4;">&nbsp;</span><strong style="font-size: 14.0px;line-height: 1.4;"><em>regToken</em></strong><span style="font-size: 14.0px;line-height: 1.4;">&nbsp;(registration token), generated by accounts.initRegistration, is required when using this method</span><span style="font-size: 14.0px;line-height: 1.4;">. </span></p>
        <p><span style="font-size: 14.0px;">For registration through a social network, see&nbsp;</span><a href="/display/GD/accounts.socialLogin+JS">accounts.socialLogin</a><span style="font-size: 14.0px;">.</span></p>

        <h1 id="accounts.registerJS-Errors">Errors</h1>
        <p>Gigya defines specific error codes and messages that are used with the Accounts API. These errors are returned with the APIs, indicating that some information is incorrect or missing.&nbsp;</p>
        <p>This section describes the errors that are related to this API, the reasons for each error, and the expected next step.</p>
        <ul>
         <li><strong>Account pending registration&nbsp;</strong>(error code&nbsp;206001) - returned when you call a method that performs social login, such as this method, and&nbsp;the registration process has not been finalized, or the schema defines fields as required and one or more of these fields are missing from the user&nbsp;<a href="/display/GD/Profile+REST">Profile</a>&nbsp;or data. The expected next step is: if the schema defines fields that are required and one or more of these fields are missing&nbsp;from the user profile or data, call&nbsp;<a href="/display/GD/accounts.setAccountInfo+JS">accounts.setAccountInfo</a>.&nbsp;If the registration process has not been finalized, call&nbsp;<a href="/display/GD/accounts.finalizeRegistration+JS">accounts.finalizeRegistration</a>.</li>
         <li><strong>CAPTCHA verification failed&nbsp;</strong>(error code 400021) - returned&nbsp;when the registration policy states that&nbsp;<em><strong>requireCaptcha</strong></em>&nbsp;is "true", and the CAPTCHA verification fails when a user tries to register.</li>
         <li><strong>Login identifier exists&nbsp;</strong>(error code 403043) - returned when email is defined as the&nbsp;<em><strong>loginIdentifier&nbsp;</strong></em>in the&nbsp;<em><strong>accountOptions&nbsp;</strong></em>policy, and the email address received from the provider exists in the system&nbsp;but is associated with a different user. The expected next step: call&nbsp;<a href="/display/GD/accounts.linkAccounts+REST">accounts.linkAccounts</a>&nbsp;to&nbsp;merge&nbsp;between the account identified by the&nbsp;provided&nbsp;<em>UID</em>&nbsp;and the account identified by the provided&nbsp;login&nbsp;credentials (<em>loginID</em>&nbsp;+&nbsp;<em>password</em>).</li>
         <li><strong>Validation errors </strong>-&nbsp;the validationErrors object is an array of validation errors; each validation error is made up of an&nbsp;<em><strong>errorCode</strong></em>, a&nbsp;<em><strong>message</strong></em>, and a&nbsp;<em><strong>fieldName</strong></em>. The errorCode and message specify what error occured and the fieldName specifies which field had a validation error. When an invalid field type is used, i.e. a string instead of an integer, or if a wrong format is used, i.e. an email address that is not in a correct format, an "Invalid parameter value" error is returned (error code 400006).&nbsp;A validation error is returned whenever there is a data validation error regarding one of the following&nbsp;required fields:&nbsp;<em>username, password, securityQuestion, secretAnswer, email</em>.&nbsp;The expected next step is: call the API method again with the missing info.</li>
        </ul>
