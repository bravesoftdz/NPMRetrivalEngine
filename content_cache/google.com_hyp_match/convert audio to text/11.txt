             <div class="post-text" itemprop="text"> 
              <p>I'm developing a website, and I would like to help blind people to use it by the voice, so I will use:</p> 
              <ul> 
               <li>Text-to-speech, to give some posibilities to the user</li> 
               <li>Speech-to-text, to allow user to use her voice to select one</li> 
              </ul> 
              <p>I already have some text-to-speech JavaScript libraries (like <a href="http://syntensity.com/static/espeak.html" rel="noreferrer">speak.js</a>), but now I need a good speech-to-text one. There are some solutions for this purpose (like <a href="http://www.speechapi.com/" rel="noreferrer">speechapi</a>), but they use Java Applets or Flash, and I want to depend only on JavaScript, to avoid plugins.</p> 
              <p>I'm trying HTML5's speech input with x-webkit-speech and Google Chrome, and it is good, but you need to click over an icon (and blind people can't use a mouse well). Is it posible to use x-webkit-speech pressing a key? Do you know any alternative API (JavaScript)?</p> 
              <p>Thank you!</p>

             <div class="post-text" itemprop="text"> 
              <p><em>Is it posible to use x-webkit-speech pressing a key?</em></p> 
              <p>According to the this <a href="https://stackoverflow.com/questions/9817373/x-webkit-speech-how-to-auto-click-the-mic-icon-to-record-many-words-automatical">post</a> and this <a href="https://stackoverflow.com/questions/9360822/webkit-speech-jquery-trigger-mic-listen">post</a> you cannot override the start of speech by clicking the microphone.</p> 
              <p>What the x-webkit-speech is doing is using the audio capture capabilities of HTML5 and sending the audio to Google's servers for processing, returning the results in JSON. This <a href="http://mikepultz.com/2011/03/accessing-google-speech-api-chrome-11/" rel="nofollow noreferrer">blogger has reversed engineered it</a>. You could develop a JavaScript library that looks for a key press to start capturing audio on HTML5 enabled browsers and send it to Google's service or to one you have created. The downside to using Google's service is that it is an unsupported API and subject to change at any time. The downside to developing your own service is that it can be expensive to develop and maintain.</p> 
              <p><em>Do you know any alternative API (JavaScript)?</em></p> 
              <p>This <a href="https://stackoverflow.com/questions/2647406/speech-recognition-web-services">post</a> and this <a href="https://stackoverflow.com/questions/4141893/speech-recognition-api">post</a> lists some services available for speech recognition. I did not see Nuance listed. You may be able to use the <a href="http://www.nuance.com/for-developers/dragon-mobile-sdk/index.htm" rel="nofollow noreferrer">Dragon Mobile SDK</a> for this. And you may want to check into <a href="https://www.ispeech.org/developers" rel="nofollow noreferrer">ISpeech</a>.</p>
