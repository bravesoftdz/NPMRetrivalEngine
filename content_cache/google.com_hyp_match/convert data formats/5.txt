     <p>This package assumes that the <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Intl"><code>Intl</code></a> global object exists in the runtime. <code>Intl</code> is present in all modern browsers <a href="http://caniuse.com/#search=intl"><em>except</em> Internet Explorer 10 and Safari 9</a> (which can be solved with the Intl polyfill).</p> 
     <p>Node.js starting from <code>0.12</code> has the <code>Intl</code> APIs built-in, but only includes the English locale data by default. If your app needs to support more locales than English, you'll need to <a href="https://github.com/nodejs/node/wiki/Intl">get Node to load the extra locale data</a>, or (a much simpler approach) just install the Intl polyfill.</p>

     <p>One can pass <code>options</code> as a second parameter to the <code>.format(date, options)</code> function. It's called a <code>style</code> (see <code>twitter</code> style for example). The <code>options</code> object can specify:</p> 
     <ul> 
      <li><code>flavour</code> – preferred labels style (e.g. <code>tiny</code>, <code>short</code>, <code>long</code>)</li> 
      <li><code>units</code> – a list of time interval measurement units which can be used in the formatted output (e.g. <code>['second', 'minute', 'hour']</code>)</li> 
      <li><code>gradation</code> – custom time interval measurement units scale</li> 
      <li><code>override</code> – is a function of <code>{ elapsed, time, date, now }</code>. If the <code>override</code> function returns a value, then the <code>.format()</code> call will return that value. Otherwise it has no effect.</li> 
     </ul>
