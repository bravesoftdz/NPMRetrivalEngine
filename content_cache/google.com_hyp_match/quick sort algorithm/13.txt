           <h1 name="a866" id="a866" class="graf graf--h3 graf-after--figure graf--title">Quicksort Algorithm</h1>
           <p name="2197" id="2197" class="graf graf--p graf--startsWithDoubleQuote graf-after--h3"><em class="markup--em markup--p-em">“What is </em><a href="https://en.wikipedia.org/wiki/Quicksort" data-href="https://en.wikipedia.org/wiki/Quicksort" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank"><em class="markup--em markup--p-em">Quicksort</em></a><em class="markup--em markup--p-em">?”</em> This is another question you’ll face in almost every tech job interview. Quicksort it is probably the most used sorting algorithm because it’s easy to implement on average, gives high efficiency for large-data applications, it can be optimized for different needs and it’s relatively flexible on it’s input data.</p>
           <p name="89c0" id="89c0" class="graf graf--p graf-after--p">Quicksort is a <strong class="markup--strong markup--p-strong">logarithmic-time algorithm</strong>, in other words, it has a <a href="https://en.wikipedia.org/wiki/Big_O_notation" data-href="https://en.wikipedia.org/wiki/Big_O_notation" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Big O notation</a> of O(log n)-<em class="markup--em markup--p-em">(</em><a href="https://medium.com/cesars-tech-insights/big-o-notation-javascript-25c79f50b19b" data-href="https://medium.com/cesars-tech-insights/big-o-notation-javascript-25c79f50b19b" class="markup--anchor markup--p-anchor" target="_blank"><em class="markup--em markup--p-em">more about Big O Notation</em></a><em class="markup--em markup--p-em">)- </em>and depending on the way you implement it, it can be up to 2x or even 3x faster than <a href="https://en.wikipedia.org/wiki/Merge_sort" data-href="https://en.wikipedia.org/wiki/Merge_sort" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Merge Sort</a> or <a href="https://en.wikipedia.org/wiki/Heapsort" data-href="https://en.wikipedia.org/wiki/Heapsort" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Heap Sort</a>. Do note that the O(log n) speed is a <strong class="markup--strong markup--p-strong">best-case/average time</strong>, in worst case scenarios it can be O(<em class="markup--em markup--p-em">n</em>2) depending on the implementation.</p>
           <p name="1d53" id="1d53" class="graf graf--p graf-after--p">Let’s explore one implementation of Quicksort using <a href="https://medium.com/tag/javascript" data-href="https://medium.com/tag/javascript" class="markup--anchor markup--p-anchor" target="_blank">JavaScript</a> (<a href="http://es6-features.org/#Constants" data-href="http://es6-features.org/#Constants" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">ECMA6</a>):</p>
           <pre name="0026" id="0026" class="graf graf--pre graf-after--p">/**<br> *  <a href="http://twitter.com/method" data-href="http://twitter.com/method" class="markup--anchor markup--pre-anchor" title="Twitter profile for @method" rel="nofollow noopener" target="_blank">@method</a>      quickSort<br> *  <a href="http://twitter.com/param" data-href="http://twitter.com/param" class="markup--anchor markup--pre-anchor" title="Twitter profile for @param" rel="nofollow noopener" target="_blank">@param</a>       list {json} ['','']<br> *  <a href="http://twitter.com/description" data-href="http://twitter.com/description" class="markup--anchor markup--pre-anchor" title="Twitter profile for @description" rel="nofollow noopener" target="_blank">@description</a> orders an array using quicksort<br> */ <br> const quickSort = ( list ) =&gt; {<br>    if ( list.length &lt; 2) return list;<br>    let pivot   = list[0];<br>    let left    = []; <br>    let right   = [];<br>    for ( let i=1, total=list.length; i&lt;total; i++ ){<br>       switch ( true ){<br>          case ( list[i] &lt; pivot ):<br>             left.push( list[i] );<br>             break;<br>          case ( list[i] &gt;= pivot ):<br>             if( list[i] )<br>             right.push( list[i] );<br>             break;<br>       }<br>    }<br>    return [].concat( quickSort( left ), pivot, quickSort( right ));<br>  };</pre>
           <p name="5742" id="5742" class="graf graf--p graf-after--pre">Quicksort uses <a href="https://en.wikipedia.org/wiki/Recursion_%28computer_science%29" data-href="https://en.wikipedia.org/wiki/Recursion_%28computer_science%29" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">recursion</a>, <a href="https://en.wikipedia.org/wiki/Divide_and_conquer_algorithms" data-href="https://en.wikipedia.org/wiki/Divide_and_conquer_algorithms" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">divide-and-conquer</a> and <a href="https://en.wikipedia.org/wiki/Comparison_sort" data-href="https://en.wikipedia.org/wiki/Comparison_sort" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">comparison-sort</a>. It works by partitioning an array into two sub-arrays and then recursively sorting those arrays independently. To make it clear, let’s put this in 3 main steps:</p>
           <ul class="postList">
            <li name="1fd3" id="1fd3" class="graf graf--li graf-after--p"><strong class="markup--strong markup--li-strong">Choose </strong>the pivot (reference value).</li>
            <li name="c1b6" id="c1b6" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">Divide </strong>the rest of the array in two, placing everything <strong class="markup--strong markup--li-strong">smaller than</strong> the pivot on the <strong class="markup--strong markup--li-strong">left </strong>and <strong class="markup--strong markup--li-strong">greater than</strong> the pivot on the <strong class="markup--strong markup--li-strong">right</strong>.</li>
            <li name="ca2f" id="ca2f" class="graf graf--li graf-after--li"><strong class="markup--strong markup--li-strong">Recursively apply</strong> the previous steps to the sub-arrays if they have more than 1 element.</li>
           </ul>
           <p name="f5d2" id="f5d2" class="graf graf--p graf-after--li">In the following chart you can see our example algorithm in action; pivots are green, left arrays are red, and right ones are blue.</p>

           <p name="c3ce" id="c3ce" class="graf graf--p graf-after--figure">Another great advantage of Quicksort is that it can be implemented using <a href="https://en.wikipedia.org/wiki/Task_parallelism" data-href="https://en.wikipedia.org/wiki/Task_parallelism" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank"><strong class="markup--strong markup--p-strong">task </strong></a><a href="https://www.google.com/search?q=parallelism&amp;spell=1&amp;sa=X&amp;ved=0ahUKEwj47cerrI_QAhXpx4MKHaHmAsUQvwUIGigA" data-href="https://www.google.com/search?q=parallelism&amp;spell=1&amp;sa=X&amp;ved=0ahUKEwj47cerrI_QAhXpx4MKHaHmAsUQvwUIGigA" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank"><strong class="markup--strong markup--p-strong">parallelism</strong></a>, so, with some tweaking of the code it can be used to get a great performance boost in multi-core processing or even better, GPU processing for extreme performance.</p>
           <p name="05d7" id="05d7" class="graf graf--p graf-after--p">Although JavaScript is a single-threaded language, if your application requires a lot of power, you can enable <strong class="markup--strong markup--p-strong">parallel high-performance GPU processing</strong> via WebCL or use <a href="https://nodejs.org/en/" data-href="https://nodejs.org/en/" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">NodeJS </a>and a library called <a href="https://github.com/kashif/node-cuda" data-href="https://github.com/kashif/node-cuda" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank"><strong class="markup--strong markup--p-strong">node-cuda</strong></a>, the latter allows you to use NVIDIA CUDA™ -although you need an NVIDIA CUDA-enabled GPU-, but with hundreds of compatible models, you probably can get some science out of your current gaming computer!</p>
