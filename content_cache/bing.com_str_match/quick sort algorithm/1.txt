private static void sort1(byte x[], int off, int len) {
// Insertion sort on smallest arrays
if (len &lt; 7) {
    for (int i=off; i&lt;len+off; i++)
    for (int j=i; j&gt;off &amp;&amp; x[j-1]&gt;x[j]; j--)

    return;
}

              <p>The quoted code <em>is</em> an insertion sort. A bubble sort repeatedly passes through the entire array, whereas an insertion sort sorts the first element, then the first two elements, then the first three elements, etc. You can tell because the code has two indexed loops, whereas the outer loop on a bubble sort just checks whether the whole array is in order or not.</p>

             <div class="post-text" itemprop="text"> 
              <p>This whole sorting algorithm is an optimized quick sort that use median of 3 indexed elements to get pivot element, and the code that you showed, is an optimization when the input array (or from the the recursion) is small.</p> 
              <p>Although, the quoted part is an insertion sort, no doubt.</p> 
              <p>But it is wrong just look this part of algorithm, so, <a href="http://www.google.com/codesearch/p?hl=en#a9vGG9ncPig/src/share/classes/java/util/Arrays.java&amp;q=Arrays.java&amp;l=569" rel="nofollow">using this link</a>:</p> 
              <ul> 
               <li>Lines 573-577 make an insertion sort, for small input arrays.</li> 
               <li>Lines 581-593 choice the pivot element, using median of 3.</li> 
               <li>Lines 596-611 does the sorting using the pivot element.</li> 
               <li>Lines 614-616 puts the partition element elements back to the middle (quick sort stuff).</li> 
               <li>Lines 619-622 recursion of the two halves of the input array.</li> 
              </ul> 
              <p>A good explanation about quick sort could be find at <a href="http://en.wikipedia.org/wiki/Quicksort" rel="nofollow">http://en.wikipedia.org/wiki/Quicksort</a>.</p>
