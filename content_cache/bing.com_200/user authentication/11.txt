           <p><img class="alignnone size-full wp-image-4953" src="https://stormpath.com/wp-content/uploads/2016/06/10-minutes-to-user-authentication-in-ASP.NET-1.jpg" alt="Ten minutes to user authentication in ASP.NET" width="960" height="440" srcset="https://stormpath.com/wp-content/uploads/2016/06/10-minutes-to-user-authentication-in-ASP.NET-1.jpg 960w, https://stormpath.com/wp-content/uploads/2016/06/10-minutes-to-user-authentication-in-ASP.NET-1-300x138.jpg 300w, https://stormpath.com/wp-content/uploads/2016/06/10-minutes-to-user-authentication-in-ASP.NET-1-768x352.jpg 768w" sizes="(max-width: 960px) 100vw, 960px"></p> 
           <p><strong>Were excited to announce</strong> that our <a href="https://github.com/stormpath/stormpath-aspnet">open-source ASP.NET authentication library</a> is now out of beta! This means you can add the power of Stormpath to your ASP.NET projects for easy user management and authentication, absolutely no database required.</p> 
           <p>The ASP.NET platform has been getting a lot of attention lately, with the pending release of <a href="https://stormpath.com/blog/asp-net-core-authentication">ASP.NET Core</a>. Core will bring ASP.NET to a whole new set of environments  hello, Linux!  and has a lot of promise as the future direction of the platform.</p> 
           <p>The current version (can we call it classic ASP.NET yet?) running on Windows and .NET framework 4.5/4.6 isnt dead yet, though! Its a stable, mature platform that is used for mission-critical applications across the globe. (After all, what could possibly be more mission-critical than <a href="http://nickcraver.com/blog/2016/02/17/stack-overflow-the-architecture-2016-edition/">StackOverflow</a>?)</p> 
           <p>If youre working on a new or existing project in <a href="https://stormpath.com/blog/token-authentication-asp-net-core" target="_blank">ASP.NET, authentication</a> is one of the biggest (and sometimes, trickiest!) pieces of any web application. Its a concern that cuts right across your database, business logic, and presentation layers and has implications for security and usability.</p> 
           <p>Thanks to this new release, you can use Stormpath to significantly simplify the task of securing and managing users in an ASP.NET application! To demonstrate how easy it is to set up, Ill show you how to set up a complete MVC application with user management in ten minutes or less, using Stormpath and ASP.NET.</p> 
           <p><span id="more-4931"></span></p> 
           <h2 id="what-is-stormpath">What is Stormpath?<a href="#what-is-stormpath" class="heading-link"><i class="stp-icons stp-icon-link2"></i></a></h2> 
           <p>Stormpath is a hosted user management API that lets you add authentication and authorization into your applications quickly, with strong security built by experts. Think of Stormpath as an alternative to wrangling and managing <a href="http://www.asp.net/identity">ASP.NET Identity</a> plus a SQL Server database.</p> 
           <p>On top of authentication and authorization, Stormpath enables a lot of other great functionality in your application, like self-service password reset, social/external login, support for multi-tenancy, and more.</p> 
           <p>Lets get started!</p> 
           <p>First, register for a <a href="https://api.stormpath.com/register">free Stormpath account</a>. Youll need to grab some API credentials so your application can communicate with the Stormpath API.</p> 
           <h2 id="get-your-api-credentials">Get Your API Credentials<a href="#get-your-api-credentials" class="heading-link"><i class="stp-icons stp-icon-link2"></i></a></h2> 
           <p>Your ASP.NET application will need an API Key and Secret in order to communicate with Stormpath. The best way to provide these is through environment variables. You can also hardcode the values into your application code, but we recommend environment variables as a best practice for security.</p> 
           <p>To save your Stormpath API credentials as environment variables, follow these steps:</p> 
           <ol> 
            <li>If you havent registered for Stormpath already, <a href="https://api.stormpath.com/register">create a free developer account</a>.</li> 
            <li>Log in to the <a href="https://api.stormpath.com/">Admin Console</a> and click the <strong>Create API Key</strong> button on the right side of the page to create and download an API key file.</li> 
            <li>Open the <code>apiKey.properties</code> file up in Notepad or your favorite text editor. Using the command line (or PowerShell), execute these commands:</li> 
           </ol>

           <p></p> 
           <h2 id="create-a-project">Create a Project<a href="#create-a-project" class="heading-link"><i class="stp-icons stp-icon-link2"></i></a></h2> 
           <p><em>Note: If youre developing for ASP.NET Core, head over to the <a href="https://stormpath.com/blog/asp-net-core-authentication">Core tutorial</a> instead.</em></p> 
           <ol> 
            <li>Using Visual Studio, create a new ASP.NET application by choosing <strong>File  New Project</strong>. Select the <strong>ASP.NET Web Application</strong> template.</li> 
            <li>In the New ASP.NET Application dialog box, pick the MVC template.</li> 
            <li>Click <strong>Change Authentication</strong> and pick <strong>No Authentication</strong>. (Youll be adding your own authentication in a minute!)</li> 
           </ol> 
           <h2 id="install-the-middleware">Install the Middleware<a href="#install-the-middleware" class="heading-link"><i class="stp-icons stp-icon-link2"></i></a></h2> 
           <p>The <code>Stormpath.AspNet</code> NuGet package comes with everything you need to plug Stormpath into an ASP.NET project.</p>
