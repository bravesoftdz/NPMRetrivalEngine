         <h2>How Does React Work?</h2> 
         <p>Now that we have the skeleton for our app done, we can focus on building our React app. But before we write any code, let’s take a look at what React is and what it does for us.</p> 
         <h3>Components</h3> 
         <p>In React, everything is built upon components. You can think of a component as something that renders a DOM node. A simple React component looks like this:</p> 
         <pre><code>class HelloWorld extends React.Component {
  render() {
    return &lt;span&gt;Hello World!&lt;/span&gt;;
  }
}
</code></pre> 
         <p>That was simple. Now, if you wanted to render this component to a page, then all you’d have to do is import React and then call:</p> 
         <pre><code>ReactDOM.render(
  &lt;HelloWorld /&gt;,
  document.getElementById('hello-world-element')
);
</code></pre> 
         <p>And React would render the component to that element.</p> 
         <p>There are, of course, more things to a React component, such as state. Below is an example of a counter component that starts counting when added to the DOM and stops when removed.</p> 
         <pre><code>class Counter extends React.Component {
  state = {
    current: 0
  }

  constructor() {
    super(arguments...);
    this.intervalId = null;
  }

  updateCounter() {
    this.setState({ counter: this.state.current + 1 });
  }

  componentWillMount() {
    this.setState({ counter: this.props.from || 0 });
    this.intervalId = setInterval(this.updateCounter.bind(this), 1000);
  }

  componentWillUnmount() {
    clearInterval(this.intervalId);
  }

  render() {
    return &lt;span&gt;{ this.state.current }&lt;/span&gt;;
  }
}
</code></pre> 
         <p>Notice the methods <code>componentWillMount()</code> and <code>componentWillUnmount()</code>. These are component life-cycle methods that will be executed at various points of a component’s life-cycle (in this case, mount and unmount). These methods are usually used for setting up and tearing down a component and is necessary to use because React will error if you try to set the state of a component when it hasn’t been mounted yet.</p> 
         <p>Also notice <code>this.props.from</code>. The member <code>this.props</code> is a collection of all the properties (inputs) passed to a component. Properties of a component can be set as shown below:</p> 
         <pre><code>&lt;Counter from="50" /&gt;
&lt;Counter from={ myVariable } /&gt;
</code></pre> 
         <h3>JSX Variables</h3> 
         <p>Variables can easily be interpolated into your JSX DOM using <code>{ nameOfVariable }</code>, e.g. as shown below:</p> 
         <pre><code>render() {
  var myVariable = 123;
  return &lt;span&gt;{ myVariable }&lt;/span&gt;;
}
</code></pre> 
         <h3>JSX and Reserved JavaScript Identifiers</h3> 
         <p>Since JSX is JavaScript, there are some caveats that you need to know when working with React. I.e. when setting properties of a React DOM component you cannot use neither <code>for</code> or <code>class</code> since those are considered reserved JavaScript identifiers. To get around this problem, React has come up with <code>htmlFor</code> and <code>className</code> that you should use instead.</p> 
         <p>To illustrate the issue, this won’t work:</p> 
         <pre><code language="markup">&lt;label for="my-input" class="my-label"&gt;My Input&lt;/label&gt;
</code></pre> 
         <p>But this will:</p> 
         <pre><code language="markup">&lt;label htmlFor="my-input" className="my-label"&gt;My Input&lt;/label&gt;
</code></pre> 
         <h3>Virtual DOM</h3> 
         <p>Instead of working directly against the DOM, in React all components are kept in their own virtual DOM. You can think of the virtual DOM as a DOM implementation in JavaScript (because it actually is). This virtual DOM is then mapped to a real DOM element. So when you render your React component, React will look at the DOM output from the component, compare it to its representation in the virtual DOM, and then generate a patch for the real DOM.</p> 
         <p>What this means is that you never have to think of manually manipulating DOM elements again. All you have to do is tell React how you want your component to look like, and it will take care of transforming the DOM the ways necessary (with minimal effort).</p> 
         <h3>Installing React Dependencies</h3> 
         <p>Now when we are acquainted with React, we’ll kick things off by installing some React dependencies:</p> 
         <pre><code language="bash">$ npm install --save react react-dom react-router react-stormpath react-document-title history
</code></pre> 
         <p>Before we start coding, we need a place to put our React files, so create a new directory named <code>src</code>, and then use that as your working directory.</p> 
         <pre><code language="bash">$ mkdir src
$ cd src
</code></pre> 
         <p>Now, let’s start with the entry point of our app. This will be the place where we will set up our React application and its routing. So create a new file named <code>app.js</code> and enter this code:</p> 
         <pre><code>import React from 'react';

         <h2>Index Page</h2> 
         <p>In our <code>MasterPage</code> notice the property <code>this.props.children</code>. This will contain the components of the child routes that our router match. So if we had a route that looked like:</p> 
         <pre><code>&lt;Route path='/' component={MasterPage}&gt;
  &lt;Route path='/hello' component={HelloPage} /&gt;
&lt;/Route&gt;
</code></pre> 
         <p>And we tried to access <code>/hello</code>. The <code>this.props.children</code> array would be populated with a <code>HelloPage</code> component and for that reason that component would be rendered in our master page.</p> 
         <p>Now imagine the scenario where you try to access <code>/</code>. Without any <code>this.props.children</code>, this would only render your master page but with empty content. This is where <a href="https://github.com/rackt/react-router/blob/master/docs/guides/basics/IndexRoutes.md"><code>IndexRoute</code></a> comes into play. With an <a href="https://github.com/rackt/react-router/blob/master/docs/guides/basics/IndexRoutes.md"><code>IndexRoute</code></a> you can specify the component that should be rendered when you hit the path of the master page route (in our case `/`).</p> 
         <p>But before we add our <a href="https://github.com/rackt/react-router/blob/master/docs/guides/basics/IndexRoutes.md"><code>IndexRoute</code></a> to our router, let’s create a new file in our <code>pages</code> directory named <code>IndexPage.js</code> and add the following to it:</p> 
         <pre><code>import { Link } from 'react-router';

        &lt;/div&gt;
      &lt;/div&gt;

</code></pre> 
         <p>Now let’s add our <a href="https://github.com/rackt/react-router/blob/master/docs/guides/basics/IndexRoutes.md"><code>IndexRoute</code></a>. Open up <code>app.js</code> and inside the tag <code>&lt;Route path='/' component={MasterPage}&gt;</code> add your <a href="https://github.com/rackt/react-router/blob/master/docs/guides/basics/IndexRoutes.md"><code>IndexRoute</code></a> so that it looks like the following:</p>

</code></pre> 
         <p>Notice the <a href="https://github.com/stormpath/stormpath-sdk-react/blob/master/docs/api.md#loginform"><code>LoginForm</code></a> component. This is all we have to add in order for us to have a fully working form in which people can sign up from.</p> 
         <p>But before we can use it, we need to open up <code>app.js</code> and add a route for the page in our router. So inside the tag <code>&lt;Route path='/' component={MasterPage}&gt;</code> add the following:</p> 
         <pre><code>&lt;LoginRoute path='/login' component={LoginPage} /&gt;
</code></pre> 
         <p>In order to be able to access the login page, we need to add this to our menu. So go ahead and open up <code>Header.js</code> and inside the element <code>&lt;ul className="nav navbar-nav navbar-right"&gt;</code> add the following:</p> 
         <pre><code>&lt;NotAuthenticated&gt;
  &lt;li&gt;
    &lt;LoginLink /&gt;
  &lt;/li&gt;
&lt;/NotAuthenticated&gt;
</code></pre> 
         <p>As you can see we’re using the <a href="https://github.com/stormpath/stormpath-sdk-react/blob/master/docs/api.md#notauthenticated"><code>NotAuthenticated</code></a> component. With this we’ll only show a <a href="https://github.com/stormpath/stormpath-sdk-react/blob/master/docs/api.md#loginlink"><code>LoginLink</code></a> when the user isn’t logged in yet.</p> 
         <h2>Registration Page</h2> 
         <p>Now, let’s add a page where people can sign up. We’ll call it <code>RegistrationPage</code>. So create a new file named <code>RegistrationPage.js</code> and put the following content in it:</p> 
         <pre><code>import React from 'react';
import DocumentTitle from 'react-document-title';
import { RegistrationForm } from 'react-stormpath';

export default class RegistrationPage extends React.Component {
  render() {
    return (
      &lt;DocumentTitle title={`Registration`}&gt;
        &lt;div className="container"&gt;
          &lt;div className="row"&gt;
            &lt;div className="col-xs-12"&gt;
              &lt;h3&gt;Registration&lt;/h3&gt;
              &lt;hr /&gt;
            &lt;/div&gt;
          &lt;/div&gt;
          &lt;RegistrationForm /&gt;
        &lt;/div&gt;
      &lt;/DocumentTitle&gt;
    );
  }
}
</code></pre> 
         <p>Notice that we used the <a href="https://github.com/stormpath/stormpath-sdk-react/blob/master/docs/api.md#registrationform"><code>RegistrationForm</code></a> component. As you might have guessed, this will render a Stormpath registration form. And once you’ve signed up it will point users to the login page where they’ll be able to login.</p> 
         <p>In order to access this page. We need to add a route. So go ahead and open up <code>app.js</code> and inside the tag <code>&lt;Route path='/' component={MasterPage}&gt;</code> add:</p> 
         <pre><code>&lt;Route path='/register' component={RegistrationPage} /&gt;
</code></pre> 
         <p>We now have a route, but people won’t be able to find the page unless we link to it, so open up <code>Header.js</code> and add the following right before the closing tag (<code>&lt;/ul&gt;</code>) of <code>&lt;ul className="nav navbar-nav navbar-right"&gt;</code>:</p> 
         <pre><code>&lt;NotAuthenticated&gt;
  &lt;li&gt;
    &lt;Link to="/register"&gt;Create Account&lt;/Link&gt;
  &lt;/li&gt;
&lt;/NotAuthenticated&gt;
</code></pre> 
         <p>Notice the use of the <a href="https://github.com/stormpath/stormpath-sdk-react/blob/master/docs/api.md#notauthenticated"><code>NotAuthenticated</code></a> component. With this we’ll only show the <code>/register</code> link when the user isn’t logged in.</p> 
         <h2>Profile Page</h2> 
         <p>Once a user is logged in, we want to be able to show them some personalized content (their user data). So create a new file named <code>ProfilePage.js</code> and put the following code in it:</p> 
         <pre><code>import React from 'react';
import DocumentTitle from 'react-document-title';
import { UserProfileForm } from 'react-stormpath';

export default class ProfilePage extends React.Component {
  render() {
    return (
      &lt;DocumentTitle title={`My Profile`}&gt;
      &lt;div className="container"&gt;
          &lt;div className="row"&gt;
            &lt;div className="col-xs-12"&gt;
              &lt;h3&gt;My Profile&lt;/h3&gt;
              &lt;hr /&gt;
            &lt;/div&gt;
          &lt;/div&gt;
          &lt;div className="row"&gt;
            &lt;div className="col-xs-12"&gt;
              &lt;UserProfileForm /&gt;
            &lt;/div&gt;
          &lt;/div&gt;
        &lt;/div&gt;
      &lt;/DocumentTitle&gt;
    );
  }
}
</code></pre> 
         <p>Notice that we use the <a href="https://github.com/stormpath/stormpath-sdk-react/blob/master/docs/api.md#userprofileform"><code>UserProfileForm</code></a>. This is a simple helper form that allows you to edit the most basic user fields.</p> 
         <p>Though, in order to actually modify the user profile, we need to change a few things in our server. So open up <code>server.js</code> and add the following route underneath <code>app.use(stormpath.init(app, ...));</code>:</p> 
         <pre><code>app.post('/me', bodyParser.json(), stormpath.loginRequired, function (req, res) {
  function writeError(message) {
    res.status(400);
    res.json({ message: message, status: 400 });
    res.end();
  }

  function saveAccount () {
    req.user.givenName = req.body.givenName;
    req.user.surname = req.body.surname;
    req.user.email = req.body.email;

    req.user.save(function (err) {
      if (err) {
        return writeError(err.userMessage || err.message);
      }
      res.end();
    });
  }

  if (req.body.password) {
    var application = req.app.get('stormpathApplication');

    application.authenticateAccount({
      username: req.user.username,
      password: req.body.existingPassword
    }, function (err) {
      if (err) {
        return writeError('The existing password that you entered was incorrect.');
      }

      req.user.password = req.body.password;

      saveAccount();
    });
  } else {
    saveAccount();
  }
});
</code></pre> 
         <p>This will allow the form to change both the given name, surname, email and password of user.</p> 
         <p>If you have additional fields that you wish to edit, then simply customize the <a href="https://github.com/stormpath/stormpath-sdk-react/blob/master/docs/api.md#userprofileform"><code>UserProfileForm</code></a> form and add the fields that you wish to edit in the route above.</p> 
         <p>Now, in order for us to access this page from the menu, open up <code>Header.js</code> and right below <code>&lt;li&gt;&lt;Link to="/"&gt;Home&lt;/Link&gt;&lt;/li&gt;</code> add:</p> 
         <pre><code>&lt;Authenticated&gt;
  &lt;li&gt;
    &lt;Link to="/profile"&gt;Profile&lt;/Link&gt;
  &lt;/li&gt;
&lt;/Authenticated&gt;
</code></pre> 
         <p>With this, using the <a href="https://github.com/stormpath/stormpath-sdk-react/blob/master/docs/api.md#authenticated"><code>Authenticated</code></a> component, when we have a user session we’ll render a link to the <code>/profile</code> page and allow our users to view their user profile.</p> 
         <p>In order for us to be able to access the page, we must as with the other pages add it to the router. Open up <code>app.js</code> and inside the tag <code>&lt;Route path='/' component={MasterPage}&gt;</code> add:</p> 
         <pre><code>&lt;AuthenticatedRoute path='/profile' component={ProfilePage} /&gt;
</code></pre> 
         <p>Notice that we’re using <a href="https://github.com/stormpath/stormpath-sdk-react/blob/master/docs/api.md#authenticatedroute"><code>AuthenticatedRoute</code></a>. This is a route that can only be accessed if there is an authenticated user session. If there’s no session, then the user will automatically be redirected to the path of the <a href="https://github.com/stormpath/stormpath-sdk-react/blob/master/docs/api.md#loginlink"><code>LoginLink</code></a>.</p> 
         <h2>Home Route</h2> 
         <p>Now when we’ve setup most of our routing. Let’s look at a special route called the <a href="https://github.com/stormpath/stormpath-sdk-react/blob/master/docs/api.md#homeroute"><code>HomeRoute</code></a>. This route itself doesn’t do anything. But acts as a “marker”, to indicate where to redirect to when logging in and logging out.</p> 
         <p>So in order to specify where we want to end up when we log out, open up <code>app.js</code> and change the:</p> 
         <pre><code>&lt;Route path='/' component={MasterPage}&gt;
  ...
&lt;/Route&gt;
</code></pre> 
         <p>Into:</p> 
         <pre><code>
&lt;HomeRoute path='/' component={MasterPage}&gt;
  ...
&lt;/HomeRoute&gt;
</code></pre> 
         <p>Now when logging out, the Stormpath SDK will know that it should redirect to the ‘/’ path. Now, to specify where to redirect when logging out, change the <a href="https://github.com/stormpath/stormpath-sdk-react/blob/master/docs/api.md#authenticatedroute"><code>AuthenticatedRoute</code></a> that we created in the previous step:</p> 
         <pre><code>&lt;AuthenticatedRoute path='/profile' component={ProfilePage} /&gt;
</code></pre> 
         <p>So that it looks like:</p> 
         <pre><code>&lt;AuthenticatedRoute&gt;
  &lt;HomeRoute path='/profile' component={ProfilePage} /&gt;
&lt;/AuthenticatedRoute&gt;
</code></pre> 
         <p>Notice how the <a href="https://github.com/stormpath/stormpath-sdk-react/blob/master/docs/api.md#authenticatedroute"><code>AuthenticatedRoute</code></a> wraps the <a href="https://github.com/stormpath/stormpath-sdk-react/blob/master/docs/api.md#homeroute"><code>HomeRoute</code></a>. This is used to indicate the authenticated route that we want to redirect to after login.</p> 
         <h2>Logout</h2> 
         <p>Finally, once our users have signed up and logged in. We want to give them the option to logout. Fortunately, adding this is really simple.</p> 
         <p>So open up <code>Header.js</code> and inside <code>&lt;ul className="nav navbar-nav navbar-right"&gt;</code> add this code to the end:</p> 
         <pre><code>&lt;Authenticated&gt;
  &lt;li&gt;
    &lt;LogoutLink /&gt;
  &lt;/li&gt;
&lt;/Authenticated&gt;
</code></pre> 
         <p>Notice the <a href="https://github.com/stormpath/stormpath-sdk-react/blob/master/docs/api.md#logoutlink"><code>LogoutLink</code></a> component. Once this is clicked, the user session will be automatically destroyed and the user will be redirected to the unauthenticated <a href="https://github.com/stormpath/stormpath-sdk-react/blob/master/docs/api.md#homeroute"><code>HomeRoute</code></a>.</p> 
         <h2>User State in Components</h2> 
         <p>Access user state in your components by requesting the <a href="https://github.com/stormpath/stormpath-sdk-react/blob/master/docs/api.md#authenticated-bool">authenticated</a> and <a href="https://github.com/stormpath/stormpath-sdk-react/blob/master/docs/api.md#user-object">user</a> context types:</p> 
         <pre><code>class ContextExample extends React.Component {
  static contextTypes = {
    authenticated: React.PropTypes.bool,
    user: React.PropTypes.object
  };

  render() {
    if (!this.context.authenticated) {
      return (
        &lt;div&gt;
          You need to &lt;LoginLink /&gt;.
        &lt;/div&gt;
      );
    }

    return (
      &lt;div&gt;
        Welcome {this.context.user.username}!
      &lt;/div&gt;
    );
  }
}
</code></pre> 
         <h2>Importing Components</h2> 
         <p>To be able to reference our pages we need to import them. And in order to make importing easy, we’ll put them all together in an <code>index.js</code> file so we only have to import it once. So let’s create a new file named <code>index.js</code> in our <code>pages</code> directory and export all of our pages from it, as shown below:</p> 
         <pre><code>export MasterPage from './MasterPage'
export IndexPage from './IndexPage'
export LoginPage from './LoginPage'
export RegistrationPage from './RegistrationPage'
export ProfilePage from './ProfilePage'
</code></pre> 
         <p>With this, we’ll only have to do one import in order to have access to all of our pages.</p> 
         <p>So let’s do that. Open up <code>app.js</code> file and at the top of the file, add the following import statement:</p> 
         <pre><code>import { MasterPage, IndexPage, LoginPage, RegistrationPage, ProfilePage } from './pages';
</code></pre> 
         <h2>Running The Project</h2> 
         <p>Now we have an application where our users can sign up, login, and show their user data. So let’s try it out!</p> 
         <p>As before, start our server by running the following:</p> 
         <pre><code language="bash">$ node server.js
</code></pre> 
         <p>And if everything is running successfully you should be able to see this message:</p> 
         <pre><code language="bash">Listening at http://localhost:3000
</code></pre> 
         <p>So, open up <a href="http://localhost:3000">http://localhost:3000</a> in your browser and try it out!</p> 
         <h2>Summary</h2> 
         <p>As you have seen in this article, React is a really powerful tool and when used together with ES6, JSX, and Stormpath, building apps suddenly becomes fun again.</p> 
         <p>If there are any parts that you feel unsure of, feel free to check out the <a href="https://github.com/stormpath/stormpath-sdk-react">example project</a> and use that as a reference implementation. I also enjoyed <a href="http://jlongster.com/Removing-User-Interface-Complexity,-or-Why-React-is-Awesome">this post on React’s design</a> – it explains in beautiful detail why React is awesome.</p> 
         <p>And if you have questions regarding the <a href="https://github.com/stormpath/stormpath-sdk-react">Stormpath React SDK</a>, be sure to check out its <a href="https://github.com/stormpath/stormpath-sdk-react/blob/master/docs/api.md">API documentation</a>.</p>
