       <div class="entry-content single"> 
        <p>In this post i am trying to look into steps to follow while running SSIS package using SQL Authentication and DontSaveSensitive as ProtectionLevel.</p> 
        <p>I created a simple SSIS package using a connection manager going to SQL Server using SQL authentication, I have my ProtectionLevel as <b>EncryptSensitiveWithUserKey</b></p> 
        <p><a href="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image001_2.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image001_2.jpg"><img style="border-bottom: 0px; border-left: 0px; display: inline; border-top: 0px; border-right: 0px" title="clip_image001" border="0" alt="clip_image001" src="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image001_thumb.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image001_thumb.jpg" width="366" height="379"></a></p> 
        <p><a href="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image003_2.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image003_2.jpg"><img style="border-bottom: 0px; border-left: 0px; display: inline; border-top: 0px; border-right: 0px" title="clip_image003" border="0" alt="clip_image003" src="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image003_thumb.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image003_thumb.jpg" width="367" height="151"></a></p> 
        <p>My package has an OLEDB Source (using connection manager above) which queries a table in database and dumps few rows in flat file.</p> 
        <p><a href="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image005_2.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image005_2.jpg"><img style="border-bottom: 0px; border-left: 0px; display: inline; border-top: 0px; border-right: 0px" title="clip_image005" border="0" alt="clip_image005" src="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image005_thumb.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image005_thumb.jpg" width="244" height="218"></a></p> 
        <p><a href="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image006_2.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image006_2.jpg"><img style="border-bottom: 0px; border-left: 0px; display: inline; border-top: 0px; border-right: 0px" title="clip_image006" border="0" alt="clip_image006" src="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image006_thumb.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image006_thumb.jpg" width="382" height="388"></a></p> 
        <p>With above settings ,when I execute my package, Its works like a charm.</p> 
        <p><a href="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image007_2.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image007_2.jpg"><img style="border-bottom: 0px; border-left: 0px; display: inline; border-top: 0px; border-right: 0px" title="clip_image007" border="0" alt="clip_image007" src="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image007_thumb.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image007_thumb.jpg" width="232" height="182"></a></p> 
        <p>Now, I changed the ProtectionLevel&nbsp; to <b>DontSaveSensitive</b></p> 
        <p><a href="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image008_2.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image008_2.jpg"><img style="border-bottom: 0px; border-left: 0px; display: inline; border-top: 0px; border-right: 0px" title="clip_image008" border="0" alt="clip_image008" src="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image008_thumb.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image008_thumb.jpg" width="399" height="312"></a></p> 
        <p>And package failed</p> 
        <p><a href="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image009_2.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image009_2.jpg"><img style="border-bottom: 0px; border-left: 0px; display: inline; border-top: 0px; border-right: 0px" title="clip_image009" border="0" alt="clip_image009" src="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image009_thumb.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image009_thumb.jpg" width="182" height="153"></a></p> 
        <p>With following error</p> 
        <p><font color="#ff0000">SSIS package "Package.dtsx" starting.</font></p> 
        <p><font color="#ff0000">Information: 0x4004300A at Data Flow Task, DTS.Pipeline: Validation phase is beginning.</font></p> 
        <p><font color="#ff0000">Error: 0xC0202009 at Package, Connection manager "runeet2k8.sa": An OLE DB error has occurred. Error code: 0x80040E4D.</font></p> 
        <p><font color="#ff0000">An OLE DB record is available.&nbsp; Source: "Microsoft SQL Native Client"&nbsp; Hresult: 0x80040E4D&nbsp; Description: "Login failed for user 'sa'.".</font></p> 
        <p><font color="#ff0000">Error: 0xC020801C at Data Flow Task, OLE DB Source [1]: The AcquireConnection method call to the connection manager "runeet2k8.sa" failed with error code 0xC0202009.</font></p> 
        <p><font color="#ff0000">Error: 0xC0047017 at Data Flow Task, DTS.Pipeline: component "OLE DB Source" (1) failed validation and returned error code 0xC020801C.</font></p> 
        <p><font color="#ff0000">Error: 0xC004700C at Data Flow Task, DTS.Pipeline: One or more component failed validation.</font></p> 
        <p><font color="#ff0000">Error: 0xC0024107 at Data Flow Task: There were errors during task validation.</font></p> 
        <p><font color="#ff0000">SSIS package "Package.dtsx" finished: Failure.</font></p> 
        <p>What we see above is expected as we have sensitive information in our package (password for SQL account) and have chosen DontSaveSensitive as protection level. The affect is Password being cleared from our connection manager hence login failure.</p> 
        <p>For this situation, We would have to save this password somewhere outside the package. I chose to save it in configuration file as shown below. I used XML Configuration file</p> 
        <p><a href="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image011_2.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image011_2.jpg"><img style="border-bottom: 0px; border-left: 0px; display: inline; border-top: 0px; border-right: 0px" title="clip_image011" border="0" alt="clip_image011" src="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image011_thumb.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image011_thumb.jpg" width="244" height="210"></a></p> 
        <p><a href="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image012_2.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image012_2.jpg"><img style="border-bottom: 0px; border-left: 0px; display: inline; border-top: 0px; border-right: 0px" title="clip_image012" border="0" alt="clip_image012" src="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image012_thumb.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image012_thumb.jpg" width="500" height="423"></a></p> 
        <p>I had to chose correct connection manager, Its called runeet2k8.sa in my case</p> 
        <p><a href="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image013_2.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image013_2.jpg"><img style="border-bottom: 0px; border-left: 0px; display: inline; border-top: 0px; border-right: 0px" title="clip_image013" border="0" alt="clip_image013" src="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image013_thumb.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image013_thumb.jpg" width="503" height="422"></a></p> 
        <p>This is the XML configuration file that BIDS wrote for me</p> 
        <p><font color="#808080">&lt;?xml version="1.0"?&gt;</font></p> 
        <p><font color="#808080">&lt;DTSConfiguration&gt;</font></p> 
        <p><font color="#808080">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;DTSConfigurationHeading&gt;</font></p> 
        <p><font color="#808080">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;DTSConfigurationFileInfo GeneratedBy="Domain\UserName" GeneratedFromPackageName="Package" GeneratedFromPackageID="{77FB98FB-E1AF-48D9-8A43-9FD6B1790837}" GeneratedDate="22-12-2009 16:12:59"/&gt;</font></p> 
        <p><font color="#808080">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;/DTSConfigurationHeading&gt;</font></p> 
        <p><font color="#808080">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;Configuration ConfiguredType="Property" Path="\Package.Connections[runeet2k8.sa].Properties[Password]" ValueType="String"&gt;</font></p> 
        <p><font color="#808080">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;ConfiguredValue&gt;&lt;/ConfiguredValue&gt;</font></p> 
        <p><font color="#808080">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;/Configuration&gt;</font></p> 
        <p><font color="#808080">&lt;/DTSConfiguration&gt;</font></p> 
        <p>&nbsp;</p> 
        <p>I have to go in and type password value.</p> 
        <p>&nbsp;</p> 
        <p><font color="#808080">&lt;?xml version="1.0"?&gt;</font></p> 
        <p><font color="#808080">&lt;DTSConfiguration&gt;</font></p> 
        <p><font color="#808080">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;DTSConfigurationHeading&gt;</font></p> 
        <p><font color="#808080">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;DTSConfigurationFileInfo GeneratedBy="FAREAST\runeetv" GeneratedFromPackageName="Package" GeneratedFromPackageID="{77FB98FB-E1AF-48D9-8A43-9FD6B1790837}" GeneratedDate="22-12-2009 16:12:59"/&gt;</font></p> 
        <p><font color="#808080">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;/DTSConfigurationHeading&gt;</font></p> 
        <p><font color="#808080">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;Configuration ConfiguredType="Property" Path="\Package.Connections[runeet2k8.sa].Properties[Password]" ValueType="String"&gt;</font></p> 
        <p>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; <strong>&nbsp; <font color="#ff0000"> &lt;ConfiguredValue&gt;Password&lt;/ConfiguredValue&gt;</font></strong></p> 
        <p>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; <font color="#808080">&lt;/Configuration&gt;</font></p> 
        <p><font color="#808080">&lt;/DTSConfiguration&gt;</font></p> 
        <p>Now i have a package with ProtectionLevel as DontSaveSensitive and password value for connection manager saved in a configuration file outside the package. Package executed absolutely fine after this.</p> 
        <p><a href="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image014_2.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image014_2.jpg"><img style="border-bottom: 0px; border-left: 0px; display: inline; border-top: 0px; border-right: 0px" title="clip_image014" border="0" alt="clip_image014" src="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image014_thumb.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image014_thumb.jpg" width="183" height="161"></a></p> 
        <p>I went ahead and set this package as Integration Services Job under SQL Agent.</p> 
        <p><a href="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image015_2.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image015_2.jpg"><img style="border-bottom: 0px; border-left: 0px; display: inline; border-top: 0px; border-right: 0px" title="clip_image015" border="0" alt="clip_image015" src="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image015_thumb.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image015_thumb.jpg" width="452" height="405"></a></p> 
        <p>Executed the job and it ran successfully.</p> 
        <p><a href="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image016_2.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image016_2.jpg"><img style="border-bottom: 0px; border-left: 0px; display: inline; border-top: 0px; border-right: 0px" title="clip_image016" border="0" alt="clip_image016" src="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image016_thumb.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image016_thumb.jpg" width="244" height="153"></a></p> 
        <p></p> 
        <p>One important thing to note is that if you are moving your package from your development machine to say test server for execution. You would have to move your configuration file as well and put it at same path on target machine where its placed on your development box.</p> 
        <p>For example, on my box, the config file PackageConfig.dtsconfig&nbsp; is placed at <b>D:\Cases\Integration Services Project1\PackageConfig.dtsconfig</b> and if I am moving my package from my development machine to somewhere else, I would have to move my config file as well and place it at location <b>D:\Cases\Integration Services Project1\ </b>on that machine.</p> 
        <p>If you don’t have dtsconfig at expected location, Job would fail. </p> 
        <p>If you done want to create same complex folder path for dtsconfig on target machine as its on your dev box (eg: <b>D:\Cases\Integration Services Project1\</b>) You could place this dtsconfig file anywhere you want and give the pointer of this path from Configuration tab&nbsp; under you SSIS job step.</p> 
        <p><a href="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image001%5B4%5D.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image001%5B4%5D.jpg"><img style="border-bottom: 0px; border-left: 0px; display: inline; border-top: 0px; border-right: 0px" title="clip_image001[4]" border="0" alt="clip_image001[4]" src="https://msdnshared.blob.core.windows.net/media/TNBlogsFS/BlogFileStorage/blogs_msdn/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image001%5B4%5D_thumb.jpg" original-url="http://blogs.msdn.com/blogfiles/runeetv/WindowsLiveWriter/SSISpackageusingSQLAuthenticationandDont_3197/clip_image001%5B4%5D_thumb.jpg" width="490" height="442"></a></p>
