                     <div class="body">
                      <p>I have been having this issue of accounts constantly getting locked out once their password changes. They are just normal Active Directory accounts that are used to login to the domain and to get onto the Exchange server. The error i am seeing in our domain controller is:</p> 
                      <p>Authentication Package: MICROSOFT_AUTHENTICATION_PACKAGE_v1_0</p> 
                      <p>Error Code 0xc000006a &nbsp;and &nbsp;Error Code 0xc0000234</p> 
                      <p>This particular user had not changed his passwords in a few years so i checked all of his mobile devices and any computer that he would have logged into and still nothing. The account gets locked out seconds after i unlock the account. I have the source workstation and have checked all of the services on it, and even deleted the Exchange profile and re-created it.&nbsp;</p>

                      <div class="body">
                       <p>Id start by using the account lock out tools to try and find what computer is causing the lockout.&nbsp;</p> 
                       <p>http://www.microsoft.com/en-gb/download/details.aspx?id=18465</p> 
                       <p>Use event comb to grab the event logs from all your DC's, then open the event log in excel and filter by computers - you should hopefully see a pattern showing the lockouts source.&nbsp;</p> 
                       <p>Then on that computer make sure that you do not have anything on that computer / device which is running under the users account - scheduled tasks etc.&nbsp;</p> 
                       <p>Run a full virus scan and malware scan to make sure that the device is clean.&nbsp;</p>

                      <div class="body">
                       <p>Id start by using the account lock out tools to try and find what computer is causing the lockout.&nbsp;</p> 
                       <p>http://www.microsoft.com/en-gb/download/details.aspx?id=18465</p> 
                       <p>Use event comb to grab the event logs from all your DC's, then open the event log in excel and filter by computers - you should hopefully see a pattern showing the lockouts source.&nbsp;</p> 
                       <p>Then on that computer make sure that you do not have anything on that computer / device which is running under the users account - scheduled tasks etc.&nbsp;</p> 
                       <p>Run a full virus scan and malware scan to make sure that the device is clean.&nbsp;</p>
