           <p>In this tutorial, we will build a simple Android app that stores user generated notes remotely, with <a href="https://stormpath.com/product/authentication/">user authentication</a> managed by Stormpath and our new <a href="https://github.com/stormpath/stormpath-sdk-android">Android SDK</a>. This tutorial will take about 15 minutes, and will demonstrate how easy and seamless it is to integrate <a href="https://stormpath.com">Stormpath</a> into new and existing Android applications, as well as cover some security and performance issues in Android development.</p> 
           <p><img src="/images/blog/2016-03-22-build-user-authentication-for-android-app/demo.gif" alt="Android Demo"></p> 
           <h2 id="why-stormpath-for-android-applications">Why Stormpath for Android Applications?<a href="#why-stormpath-for-android-applications" class="heading-link"><i class="stp-icons stp-icon-link2"></i></a></h2> 
           <p>Stormpath is an authentication service that securely stores your user accounts and private user information. Plus, its free to use for small projects!</p> 
           <p>Having an authentication service like Stormpath allows you to build secure mobile applications that can register and login users in a variety of ways, without compromising your device security.</p> 
           <p>Using Stormpath, users are logged in securely using the <a href="https://stormpath.com/blog/what-the-heck-is-oauth/">OAuth2</a> protocol with signed <a href="https://stormpath.com/blog/build-secure-user-interfaces-using-jwts/">JSON Web Tokens</a>.<span id="more-2009"></span></p> 
           <p></p> 
           <p><img src="/images/blog/2016-03-22-build-user-authentication-for-android-app/network_service_diagram.png" alt="Network Service Diagram"></p> 
           <p>With Stormpath, you dont need to be an expert in cryptography to create a trusted front door to your application. In the context of this tutorial, the User is on the client side mobile app, and the Application is the server hosted on <a href="https://www.heroku.com/">Heroku</a>.</p> 
           <p>Using the Stormpath Android library makes it simple to implement user authentication correctly, as opposed to spending lots of time and effort building secure authentication methods into your next mobile app!</p> 
           <p>The Stormpath Android library can also be used to help build more complex authentication scenarios, such as <a href="https://stormpath.com/blog/multi-tenancy-for-saas-applications/">multi-tenancy</a> and <a href="https://stormpath.com/product/sso/">single sign on</a>.</p> 
           <p>In network-driven mobile apps, implementing authentication incorrectly can be an enormous risk to your users, and cause lots of wasted time and development effort fixing bugs that should never have been introduced.</p> 
           <p>Getting a proper authentication library implemented into your mobile app codebase early on is a great way to speed up development and reduce frustration later on.</p> 
           <h2 id="ok-lets-build-something">OK! Lets Build Something!<a href="#ok-lets-build-something" class="heading-link"><i class="stp-icons stp-icon-link2"></i></a></h2> 
           <p>Lets build a simple note-taking application in Android. By the end of this tutorial you will have an app that allows you to register and login as a user, save a note, and retrieve that note from your server. Literally, synchronize your note data with a cloud backend!</p> 
           <p>This tutorial is for Android apps written in Java and the <a href="https://github.com/stormpath/stormpath-sdk-android">Android SDK</a>. If you arent hip enough to write code that works on 2,300 different devices, there is an <a href="https://stormpath.com/blog/build-note-taking-app-swift-ios">iOS tutorial</a> written in Swift as well &gt;:)</p> 
           <h2 id="setting-up-our-android-project">Setting up Our Android Project<a href="#setting-up-our-android-project" class="heading-link"><i class="stp-icons stp-icon-link2"></i></a></h2> 
           <p>In this tutorial, weve done all of the backend work for you, and the API server is hosted at https://stormpathnotes.herokuapp.com</p> 
           <p>A typical Stormpath integration primarily involves your API server architecture communicating with Stormpaths backend service. This allows you to keep your Stormpath API keys on your server instead of hard-coded in your mobile app.</p> 
           <p>To integrate with your existing backend (<em>instead of our example one</em>), take some time to look at our language specific <a href="https://stormpath.com/blog/heroku-stormpath-and-expressjs/">server side tutorials</a>. Or  just shoot that link to your backend team =)</p> 
           <p>The backend API service were using exposes two protected endpoints for your Android application to use:</p> 
           <ul> 
            <li><code>GET /notes</code>  Returns the notes for the authenticated user in the form of a JSON object.</li> 
            <li><code>POST /notes</code>  Takes a JSON object with the notes and saves it for the authenticated user.</li> 
           </ul>

           <p></p> 
           <p>In case youre curious, we used the following tools to build the backend for Stormpath Notes:</p> 
           <ul> 
            <li><a href="http://expressjs.com/">Express</a>  A Node.js framework that makes it easy to build API services.</li> 
            <li><a href="https://github.com/stormpath/express-stormpath">Express-Stormpath</a>  Exposes a configurable REST API for our mobile clients within Express.</li> 
            <li><a href="https://stormpath.com/">Stormpath</a>  Allows us to store and authenticate users without having to create our own backend for it.</li> 
            <li><a href="https://www.heroku.com/">Heroku</a>  Hosts the code for Stormpath Notes online.</li> 
           </ul>

           <p></p> 
           <p>Look at the private methods <code>getNotes()</code> and <code>saveNote()</code> for examples of OkHttp3 method preparation.</p> 
           <p>The way both of these methods work is that they form the request, add headers, make the network call, and broadcast to another part of the app on a successful response from the network.</p> 
           <p></p> 
           <div id="crayon-59b4c365e14ee496031184" class="crayon-syntax crayon-theme-stormpath crayon-font-monaco crayon-os-pc print-yes notranslate" data-settings=" minimize scroll-mouseover" style=" margin-top: 12px; margin-bottom: 12px; font-size: 12px !important; line-height: 15px !important;"> 
            <div class="crayon-plain-wrap">
             <textarea wrap="soft" class="crayon-plain print-no" data-settings="dblclick" readonly style="-moz-tab-size:4; -o-tab-size:4; -webkit-tab-size:4; tab-size:4; font-size: 12px !important; line-height: 15px !important;">
private void saveNote() {
    RequestBody requestBody = new FormBody.Builder()
            .add("notes", mNote.getText().toString())
            .build();

    Request request = new Request.Builder()
            .url(NotesApp.baseUrl + "notes")
            .headers(buildStandardHeaders((Stormpath.accessToken())))
            .post(requestBody)
            .build();

    okHttpClient.newCall(request).enqueue(new Callback() {
        @Override public
        void onFailure(Call call, IOException e) {
        }

        @Override public void onResponse(Call call, Response response)
                throws IOException {
            Intent intent = new Intent(ACTION_POST_NOTES);
            LocalBroadcastManager.getInstance(context).sendBroadcast(intent);
        }
    });
}

private void getNotes() {
    Request request = new Request.Builder()
            .url(NotesApp.baseUrl + "notes")
            .headers(buildStandardHeaders(Stormpath.accessToken()))
            .get()
            .build();

    okHttpClient.newCall(request).enqueue(new Callback() {
        @Override public
        void onFailure(Call call, IOException e) {
        }

        @Override public void onResponse(Call call, Response response)
                throws IOException {
            JSONObject mNotes;

            try {
                mNotes = new JSONObject(response.body().string());
                String noteCloud = mNotes.getString("notes");

                // You can also include some extra data.
                Intent intent = new Intent(ACTION_GET_NOTES);
                intent.putExtra("notes", noteCloud);

                LocalBroadcastManager.getInstance(context).sendBroadcast(intent);
            } catch (JSONException e) {
            }
        }
    });
}
