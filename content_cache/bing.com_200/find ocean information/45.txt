              <p>You can list the installed files with</p> 
              <pre><code>rpm -ql packagename
</code></pre> 
              <p>You will see somewhere a <code>bin</code> directory with <code>java executable</code></p> 
              <p>But if the JDK RPM was correctly installed you should already find <code>java</code> in you path.</p> 
              <p>Try</p> 
              <pre><code>javac MyFirstJavaClass.java
</code></pre> 
              <p>and if everything compiles</p> 
              <pre><code>java MyFirstClass
</code></pre> 
              <p>(If you didn't change anything the current directory <code>.</code> should already be in your class path)</p>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>Since this question is RPM specific, <code>rpm</code> is the way to get started (as <a href="https://unix.stackexchange.com/a/21691/63264">answered</a> by <a href="https://unix.stackexchange.com/users/10616/matteo">@Matteo</a>).</p> 
              <p><strong>rpm flags</strong></p> 
              <pre><code>-q is short for --query
-l is short for --list
</code></pre> 
              <p><strong>Example</strong></p> 
              <pre><code>rpm -ql jdk1.8.0_20 | grep "jdk1.8.0_20/bin$"
</code></pre> 
              <blockquote> 
               <pre><code>/usr/java/jdk1.8.0_20/bin
</code></pre> 
              </blockquote> 
              <p>Knowing this may be desirable for setting a user or application's <code>$JAVA_HOME</code> variable. This is often needed when a system has multiple versions of java installed, or multiple distributions of java installed, such as <a href="http://openjdk.java.net/" rel="nofollow noreferrer">OpenJDK</a> and <a href="https://www.oracle.com/java/index.html" rel="nofollow noreferrer">Oracle/Sun</a>. </p> 
              <p><strong><code>$JAVA_HOME</code> Example</strong></p> 
              <p>In the <code>~/.bash_profile</code>, or related file (<code>.bashrc</code>, <code>.zshrc</code>, <code>.cshrc</code>, <code>setenv.sh</code>), something similar to the below may be used. </p> 
              <pre><code>JAVA_HOME='/usr/java/jdk1.8.0_20'
export JAVA_HOME
PATH="$JAVA_HOME/bin:$PATH"
export PATH
</code></pre> 
              <hr> 
              <p>If you would like more control over where Java gets installed, such as in <code>/opt</code>, then the tarball can be used instead of the RPM file. </p> 
              <hr> 
              <p><em>Other similar questions, are asking about how to find any binary or file, in the general case.</em></p>
