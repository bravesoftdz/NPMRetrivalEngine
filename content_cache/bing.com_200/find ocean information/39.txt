   <h1 class="SECT1"><a name="INFO">3. Getting information about packages</a></h1>
   <div class="SECT2">
    <h2 class="SECT2"><a name="FINDING">3.1. Finding packages -- <span class="emphasis"><i class="EMPHASIS">apt-cache search</i></span></a></h2>
    <p><a name="FINDING"> Whether you're online or not--</a></p>
    <p><a name="FINDING"> How do you find the package that's got the feature you're looking for? First, do </a></p>
    <table border="0" bgcolor="#BBDDFF" width="100%" cellpadding="0" cellspacing="0">
     <tbody>
      <tr>
       <td><pre class="SCREEN"># apt-get update
		</pre></td>
      </tr>
     </tbody>
    </table>
    <a name="FINDING"> so your package list is up-to-date, and then try something like 
     <table border="0" bgcolor="#BBDDFF" width="100%" cellpadding="0" cellspacing="0">
      <tbody>
       <tr>
        <td><pre class="SCREEN">                                                     
% apt-cache search tunnel
% apt-cache search 'php.*sql'
% apt-cache search apache.\*perl
% apt-cache search elvis\|vim
		</pre></td>
       </tr>
      </tbody>
     </table> That is how you tell <b class="KEYCAP">apt</b> to search the packages you've downloaded, using REGEX (regular expression, a pattern-matching 'language') -- if your pattern uses any keystrokes that mean something to your command shell (e.g. <b class="COMMAND">[|?*]</b> ) you'll need to quote them so that <b class="COMMAND">apt-cache</b> will be able to see them, instead of having the shell expand the term to a list of file names that mean something else entirely. <p></p></a>
    <p><a name="FINDING"> <span class="QUOTE">" NOTE -- <b class="COMMAND">apt-cache</b> only knows about the package descriptions you've already downloaded. To search among ALL known Debian packages just browse to <b class="COMMAND">http://packages.debian.org/PACKAGESUBSTRING</b> to see what's available. For example: </span></a><a href="http://packages.debian.org/vnc" target="_top"> http://packages.debian.org/vnc</a> That would get you a listing of packages that contain the term "vnc" somewhere in the title." </p>
   </div>
   <div class="SECT2">
    <h2 class="SECT2"><a name="DESCRIBING">3.2. Describing packages -- <span class="emphasis"><i class="EMPHASIS">apt-cache show</i></span></a></h2>
    <p><a name="DESCRIBING"> The simplest way to see the description of a package: </a></p>
    <table border="0" bgcolor="#BBDDFF" width="100%" cellpadding="0" cellspacing="0">
     <tbody>
      <tr>
       <td><pre class="SCREEN">% apt-cache show postgresql
% apt-cache show iproute
		</pre></td>
      </tr>
     </tbody>
    </table>
    <a name="DESCRIBING"> That displays what the package is designed to do, version info and so forth. (Pretty simple, huh? Not everything worthwhile needs to be complicated...) <p></p></a>
   </div>
   <div class="SECT2">
    <h2 class="SECT2"><a name="LISTING">3.3. Listing packages -- <span class="emphasis"><i class="EMPHASIS">dpkg -l</i></span></a></h2>
    <p><a name="LISTING"> Which packages are installed? Do any packages need configuring? </a></p>
    <table border="0" bgcolor="#BBDDFF" width="100%" cellpadding="0" cellspacing="0">
     <tbody>
      <tr>
       <td><pre class="SCREEN">% dpkg -l	
		</pre></td>
      </tr>
     </tbody>
    </table>
    <a name="LISTING"> (that's a lower-case EL, not a one.) That lists all INSTALLED packages. 
     <table border="0" bgcolor="#BBDDFF" width="100%" cellpadding="0" cellspacing="0">
      <tbody>
       <tr>
        <td><pre class="SCREEN">% dpkg -l \*
% dpkg -l '*'
		</pre></td>
       </tr>
      </tbody>
     </table> These list all packages. (Without the star glob, <b class="COMMAND">dpkg </b> only lists installed packages.) 
     <table border="0" bgcolor="#BBDDFF" width="100%" cellpadding="0" cellspacing="0">
      <tbody>
       <tr>
        <td><pre class="SCREEN">% dpkg -l \*postgres\*
% dpkg -l '*postgres*'
		</pre></td>
       </tr>
      </tbody>
     </table> That shows the status of packages matching that GLOB (it's not a regular expression [REGEX]: a regex interprets <b class="KEYCAP">.*</b> to mean '"." = any character, "*" = zero or more times'; a glob interprets <b class="KEYCAP">.*</b> to mean '"." = a dot, "*" = followed by zero or more characters'). <p></p><p> NOTE: If your pattern uses fancy characters that have a special meaning to your command shell (<b class="KEYCAP">* ? |</b> etc.) you'll need to QUOTE them so that your command shell doesn't interpret them -- you want <b class="COMMAND">dpkg</b> to see the pattern, instead. Quote such characters by either enclosing them in quotes: </p>
     <table border="0" bgcolor="#BBDDFF" width="100%" cellpadding="0" cellspacing="0">
      <tbody>
       <tr>
        <td><pre class="SCREEN">'*like|this?or that'
"even[this|is]acceptable"
		</pre></td>
       </tr>
      </tbody>
     </table> or precede them with the BACKSLASH: 
     <table border="0" bgcolor="#BBDDFF" width="100%" cellpadding="0" cellspacing="0">
      <tbody>
       <tr>
        <td><pre class="SCREEN">this\ acts\ like\ one\ word\ cuz\ spaces\ are\ quoted
\[one\|two\?three\*four\]
		</pre></td>
       </tr>
      </tbody>
     </table> And note that the /SLASH/ (forward slash) leans right, in the same direction as the text you're reading, whereas the \BACKSLASH\ leans to the left. SLASH is used to delineate components (folders and subfolders, if you like) along a directory path (<tt class="FILENAME">/home/will/public_html/index.html</tt> for example); BACKSLASH is used to alter the meaning of the keystroke that follows. Very important distinction! (And curses to the weenie who first thought QDOS -- which became MS-DOS -- should use backslashes as path delimiters! Bad dog!) <p></p></a>
   </div>
   <div class="SECT2">
    <h2 class="SECT2"><a name="GREP-SEARCH">3.4. Combine <span class="emphasis"><i class="EMPHASIS">dpkg</i></span> with <span class="emphasis"><i class="EMPHASIS">grep</i></span> for some powerful searches</a></h2>
    <p><a name="GREP-SEARCH"> </a></p>
    <table border="0" bgcolor="#BBDDFF" width="100%" cellpadding="0" cellspacing="0">
     <tbody>
      <tr>
       <td><pre class="SCREEN">% dpkg -l \* | grep ^pi
		</pre></td>
      </tr>
     </tbody>
    </table>
    <a name="GREP-SEARCH"> finds installed packages marked to be purged. 
     <table border="0" bgcolor="#BBDDFF" width="100%" cellpadding="0" cellspacing="0">
      <tbody>
       <tr>
        <td><pre class="SCREEN">% dpkg -l '*' | grep "^i[^i]"
		</pre></td>
       </tr>
      </tbody>
     </table> lists packages marked for installation, that aren't installed yet. 
     <table border="0" bgcolor="#BBDDFF" width="100%" cellpadding="0" cellspacing="0">
      <tbody>
       <tr>
        <td><pre class="SCREEN">% dpkg -l \* | grep '^[^i]i'
		</pre></td>
       </tr>
      </tbody>
     </table> shows installed packages that are marked for anything but installation (i.e.uninstallation or purgery). See <b class="COMMAND">man grep </b> for more info on <b class="COMMAND">grep</b> and <b class="COMMAND">man dpkg</b> for more on the listing format. <p></p><p> On my system, to find out which <b class="KEYCAP">apt</b> packages I have installed, I can do this: </p>
     <table border="0" bgcolor="#BBDDFF" width="100%" cellpadding="0" cellspacing="0">
      <tbody>
       <tr>
        <td><pre class="SCREEN">%  dpkg -l \*apt\* 
Desired=Unknown/Install/Remove/Purge/Hold
| Status=Not/Installed/Config-files/Unpacked/Failed-config/Half-installed
|/ Err?=(none)/Hold/Reinst-required/X=both-problems (Status,Err: uppercase=bad)
||/ Name              Version           Description
+++-=================-=================-=============================================================
ii  <b class="KEYCAP">apt</b>               0.3.19            Advanced front-end for dpkg
ii  <b class="KEYCAP">apt</b>-move          3.0-13            Move cache of Debian packages into a mirror hierarchy
ii  <b class="KEYCAP">apt</b>itude          0.0.4a-4.1        Console based apt frontend
un  lib<b class="KEYCAP">apt</b>-pkg-dev    &lt;none&gt;            (no description available)
un  lib<b class="KEYCAP">apt</b>-pkg-doc    &lt;none&gt;            (no description available)
un  lib<b class="KEYCAP">apt</b>-pkg2.7     &lt;none&gt;            (no description available)
pn  task-l<b class="KEYCAP">apt</b>op       &lt;none&gt;            (no description available)
		</pre></td>
       </tr>
      </tbody>
     </table> <p></p><p> Here, <b class="COMMAND">dpkg</b> displays its headers, and then shows all packages that have "apt" anywhere in the title. The three <b class="KEYCAP">libapt-*</b> packages are not installed (I've <b class="KEYCAP">hilighted</b> the "apt" strings that belong to non-<b class="KEYCAP">apt</b> packages, so you can see why they showed up. Interesting, no?) </p><p> For a slightly different approach, how about this: </p>
     <table border="0" bgcolor="#BBDDFF" width="100%" cellpadding="0" cellspacing="0">
      <tbody>
       <tr>
        <td><pre class="SCREEN">%  dpkg -l \* | grep apt 
ii  <b class="KEYCAP">apt</b>               0.3.19            Advanced front-end for dpkg
ii  <b class="KEYCAP">apt</b>-move          3.0-13            Move cache of Debian packages into a mirror hierarchy
ii  <b class="KEYCAP">apt</b>itude          0.0.4a-4.1        Console based <b class="KEYCAP">apt</b> frontend
un  lib<b class="KEYCAP">apt</b>-pkg-dev    &lt;none&gt;            (no description available)
un  lib<b class="KEYCAP">apt</b>-pkg-doc    &lt;none&gt;            (no description available)
un  lib<b class="KEYCAP">apt</b>-pkg2.7     &lt;none&gt;            (no description available)
ii  libpcap0          0.4a6-3           System interface for user-level packet c<b class="KEYCAP">apt</b>ure.
pn  task-l<b class="KEYCAP">apt</b>op       &lt;none&gt;            (no description available)
%
		</pre></td>
       </tr>
      </tbody>
     </table> Here I asked to see ALL packages (<b class="COMMAND">dpkg -l \*</b>) and thenI used <b class="COMMAND">grep</b> to display only those lines from the output that contained "apt". The headers disappeared, since none of them contain "apt" -- so <b class="COMMAND">grep</b> discards them. <p></p></a>
   </div>
   <div class="SECT2">
    <h2 class="SECT2"><a name="CONTENTS">3.5. Package contents -- <span class="emphasis"><i class="EMPHASIS">dpkg -L</i></span></a></h2>
    <p><a name="CONTENTS"> </a></p>
    <table border="0" bgcolor="#BBDDFF" width="100%" cellpadding="0" cellspacing="0">
     <tbody>
      <tr>
       <td><pre class="SCREEN">% dpkg -L mysql
% dpkg -L apt
		</pre></td>
      </tr>
     </tbody>
    </table>
    <a name="CONTENTS"> (with an upper-case EL) shows all the files -- with full path names -- that are provided by the package. <p></p></a>
   </div>
   <div class="SECT2">
    <h2 class="SECT2"><a name="CONTAINS">3.6. Finding which package contains... -- <span class="emphasis"><i class="EMPHASIS">dpkg -S</i></span></a></h2>
    <p><a name="CONTAINS"> How can you determine here did file XYZ come from? Here's how to find which package contains/supplies a certain file: </a></p>
    <table border="0" bgcolor="#BBDDFF" width="100%" cellpadding="0" cellspacing="0">
     <tbody>
      <tr>
       <td><pre class="SCREEN">% dpkg -S postmaster
% dpkg -S 'doc/*sql' | cut -f1 -d: | sort -u
		</pre></td>
      </tr>
     </tbody>
    </table>
    <a name="CONTAINS"> That searches for packages that supply files whose paths contain the requested GLOB. See <b class="COMMAND">man cut</b> and <b class="COMMAND">man sort</b> for info on how to use these tools in your day-to-day fiddling. <p></p><p> If you're using <b class="KEYCAP">unstable</b> then you might be able to tinker with <b class="COMMAND">dlocate</b> as well. I hear it's highly thought-of by all who've used it. (If you know about it and want to include some info here, please let me know!) </p><p> By the way -- here's a quickie command to show what packages I've got installed that provided all of the <b class="COMMAND">apt-*</b> files you saw above when I tried <b class="KEYCAP"><tt class="LITERAL">apt&lt;^D&gt;</tt> </b>: </p>
     <table border="0" bgcolor="#BBDDFF" width="100%" cellpadding="0" cellspacing="0">
      <tbody>
       <tr>
        <td><pre class="SCREEN">%  dpkg -S apt- | sort 
apt-move: /etc/apt-move.conf
apt-move: /etc/cron.weekly/apt-move
apt-move: /usr/bin/apt-move
apt-move: /usr/share/doc/apt-move
apt-move: /usr/share/doc/apt-move/README.Debian
apt-move: /usr/share/doc/apt-move/README.gz
apt-move: /usr/share/doc/apt-move/TODO
apt-move: /usr/share/doc/apt-move/TODO.Debian
apt-move: /usr/share/doc/apt-move/changelog.Debian.gz
apt-move: /usr/share/doc/apt-move/changelog.gz
apt-move: /usr/share/doc/apt-move/copyright
apt-move: /usr/share/doc/apt-move/examples
apt-move: /usr/share/doc/apt-move/examples/SAMPLE.exclude
apt-move: /usr/share/man/man8/apt-move.8.gz
apt: /usr/bin/apt-cache
apt: /usr/bin/apt-cdrom
apt: /usr/bin/apt-config
apt: /usr/bin/apt-get
apt: /usr/lib/libapt-pkg.so.2.7
apt: /usr/lib/libapt-pkg.so.2.7.1
apt: /usr/share/man/man8/apt-cache.8.gz
apt: /usr/share/man/man8/apt-cdrom.8.gz
apt: /usr/share/man/man8/apt-config.8.gz
apt: /usr/share/man/man8/apt-get.8.gz
base-config: /usr/sbin/apt-setup
base-config: /usr/share/debconf/templates/apt-setup.templates
base-config: /usr/share/man/man8/apt-setup.8.gz
		</pre></td>
       </tr>
      </tbody>
     </table> That shows three of the apt packages I've got installed: <b class="KEYCAP">apt, apt-move,</b> and <b class="KEYCAP">base-config.</b> You'll notice that <b class="KEYCAP">aptitude</b> isn't listed -- which is understandable, since it doesn't contain the "apt DASH" string. Also, <b class="KEYCAP">base-config</b> didn't show up above when I searched via <b class="COMMAND">dpkg -l \* | grep apt</b> since the string "apt" doesn't appear within <b class="KEYCAP">base-config</b>. See? <p></p><p> As is the case with many packages, most of the files are documentation: either <tt class="FILENAME">/usr/share/doc/*</tt> or manpages. </p><p> NOTE that if I do <b class="COMMAND">dpkg -S apt</b> without the dash after "apt-" i get more irrelevant stuff, because many packages contain files that have the string "apt" in them, including: </p><p></p>
     <ul>
      <li><p><tt class="FILENAME">/usr/bin/<b class="KEYCAP">apt</b>itude</tt> it'll be a nice APT front-end when it's done</p></li>
      <li><p><tt class="FILENAME"> /usr/share/doc/gpm/README.synaptics.gz</tt></p></li>
      <li><p><tt class="FILENAME"> /usr/share/doc/postgresql-doc/postgres/libpq-chapter18164.htm</tt></p></li>
      <li><p><tt class="FILENAME"> /usr/share/texmf/tex/latex/caption</tt> </p></li>
     </ul> ...so you must learn to be careful what you search for... you'll find it! <p></p></a>
