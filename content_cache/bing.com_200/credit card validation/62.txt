       <h1><a href="/blog/jquery-payment">jQuery.payment<br></a></h1> 
       <p> <a class="author" href="/about#alexmaccaw">Alex MacCaw</a> <span> on February 7, 2013</span> in <a href="/blog/engineering">Engineering</a> </p> 
      </header> 
      <div class="content"> 
       <script src="/assets/jquery-63d95594326d783037113b616637e19e.js"></script> 
       <script src="/assets/blog/jquery.payment-cc6912ed8e82841bc7bbb6acad4178c5.js"></script> 
       <script type="text/javascript" charset="utf-8">
  jQuery(function($){
    $('input.cc-num').formatCardNumber();
    $('input.cc-exp').formatCardExpiry();
  });
</script> 
       <p>A rising tide lifts all boats, and we’d like to help improve payment experiences for consumers everywhere, whether or not they use Stripe. Today, we’re releasing <a href="https://github.com/stripe/jquery.payment">jQuery.payment</a>, a general purpose library for building credit card forms, validating input, and formatting numbers. This library is behind a lot of the functionality in <a href="/blog/stripe-checkout">Checkout</a>.</p> 
       <!-- more --> 
       <p>Some sites require a bit more flexibility than our Checkout provides. This is where jQuery.payment shines. You can have some of the same formatting and validation as in the Checkout along with as much flexibility as you need.</p> 
       <h2>Features</h2> 
       <p>For example, you can ensure that a text input is formatted as a credit card number, with digits in groups of four and limited to 16 characters.</p> 
       <div class="example"> 
        <pre>$('input.cc-num').payment('formatCardNumber');</pre> 
        <input type="text" class="cc-num" placeholder="1234 5678 9123 4567" pattern="\d*"> 
       </div> 
       <p>Or you can ensure input is formatted as a MM/YYYY card expiry:</p> 
       <div class="example"> 
        <pre>$('input.cc-exp').payment('formatCardExpiry');</pre> 
        <input type="text" class="cc-exp" placeholder="MM / YY" pattern="\d*"> 
       </div> 
       <p>The library includes a bunch of utility and validation methods, for example:</p> 
       <pre>
$.payment.validateCardNumber('4242 4242 4242 4242'); //=&gt; true
$.payment.validateCardCVC('123', 'amex'); //=&gt; false
$.payment.validateCardExpiry('05', '20'); //=&gt; true

$.payment.cardType('4242 4242 4242 4242'); //=&gt; 'visa'
</pre> 
       <h2>Robust and tested</h2> 
       <p>It turns out that rolling your own code that restricts and formats input is particularly tricky in JavaScript. You have to cater for lots of edge cases such as users pasting text, selecting and replacing numbers, as well as the different ways credit card numbers are formatted. </p> 
       <p>We’ve spent a lot of time tuning our formatting and validation logic as well as testing and ensuring cross browser compatibility, so you don't have to reinvent the wheel. We look forward to seeing what you build! You can find a <a href="http://stripe.github.com/jquery.payment/example/">live demo of the library</a>, as well as the <a href="https://github.com/stripe/jquery.payment">source on GitHub</a>.</p>
