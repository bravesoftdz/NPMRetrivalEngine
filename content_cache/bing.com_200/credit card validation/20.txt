         <h1 class="entry-title" itemprop="headline">Java Regex to Validate Credit Card Numbers</h1>
         <p class="entry-meta"><time class="entry-time" itemprop="datePublished" datetime="2014-11-18T07:41:07+00:00">November 18, 2014</time> by <span class="entry-author" itemprop="author" itemscope itemtype="https://schema.org/Person"><span class="entry-author-name" itemprop="name">Lokesh Gupta</span></span></p>
        </header>
        <div class="entry-content" itemprop="text">
         <div style="width: 336px; height: 280px; margin: 6px 20px 20px 0px; float: left;">
          <script async src="//pagead2.googlesyndication.com/pagead/js/adsbygoogle.js"></script>
          <ins class="adsbygoogle" style="display:inline-block;width:336px;height:280px" data-ad-client="ca-pub-2827350136257294" data-ad-slot="6068795363"></ins>
          <script>(adsbygoogle = window.adsbygoogle || []).push({});</script>
         </div>
         <p>In this java regex tutorial, we will Learn to use regular expressions to validate Credit Card Numbers. In this tutorial, we will learn about format and validations of Credit Card Numbers from multiple providers such as VISA, Mastercard, Amex and Diners etc.</p>
         <h2>Valid Credit Card Numbers Format</h2>
         <p>On an actual credit card, the digits of the embossed card number are usually placed into groups of four. That makes the card number easier for humans to read. Each of the credit card companies uses this number format. Well exploit that difference of formats between each company to allow users to enter a number without specifying a company. The company can be determined from the number. The format for each company is:</p>
         <ul>
          <li><strong>Visa</strong> : 13 or 16 digits, starting with 4.</li>
          <li><strong>MasterCard</strong> : 16 digits, starting with 51 through 55.</li>
          <li><strong>Discover</strong> : 16 digits, starting with 6011 or 65.</li>
          <li><strong>American Express</strong> : 15 digits, starting with 34 or 37.</li>
          <li><strong>Diners Club</strong> : 14 digits, starting with 300 through 305, 36, or 38.</li>
          <li><strong>JCB</strong> : 15 digits, starting with 2131 or 1800, or 16 digits starting with 35.</li>
         </ul>
         <div class="noticeboxesde bluenoticebox">
          Below given regex assumes that before performing the check for a valid number, you will search-and-replace all spaces and hyphens explicitly.
         </div>
         <p>With spaces and hyphens stripped from the input, the next regular expression checks if the credit card number uses the format of any of the six major credit card companies. It uses named capture to detect which brand of credit card the customer has. If you dont need to determine which type the card is, you can remove the six capturing groups that surround the pattern for each card type, as they dont serve any other purpose.</p>
         <p>If you accept only certain brands of credit cards, you can delete the cards that you dont accept from the regular expression. For example, when deleting JCB, make sure to delete the last remaining | in the regular expression as well. If you end up with | in your regular expression, it will accept the empty string as a valid card number as well.</p>
         <pre>Regex : ^(?:(?&lt;visa&gt;4[0-9]{12}(?:[0-9]{3})?)|
		(?&lt;mastercard&gt;5[1-5][0-9]{14})|
		(?&lt;discover&gt;6(?:011|5[0-9]{2})[0-9]{12})|
		(?&lt;amex&gt;3[47][0-9]{13})|
		(?&lt;diners&gt;3(?:0[0-5]|[68][0-9])?[0-9]{11})|
		(?&lt;jcb&gt;(?:2131|1800|35[0-9]{3})[0-9]{11}))$</pre>
         <p>Read more about credit card number formats in this <strong><a href="https://en.wikipedia.org/wiki/Bank_card_number" title="credit card formats" target="_blank">wiki page</a></strong>.</p>
         <h2>Example Code to Validate Credit Card Numbers</h2>
         <pre class="brush: java; title: ; notranslate" title="">
public static void main(String[] args)
{
List&lt;String&gt; cards = new ArrayList&lt;String&gt;();

//Valid Credit Cards
cards.add("xxxx-xxxx-xxxx-xxxx");  //Masked to avoid any inconvenience unknowingly

//Invalid Credit Card
cards.add("xxxx-xxxx-xxxx-xxxx"); //Masked to avoid any inconvenience unknowingly

String regex = "^(?:(?&lt;visa&gt;4[0-9]{12}(?:[0-9]{3})?)|" +
		"(?&lt;mastercard&gt;5[1-5][0-9]{14})|" +
		"(?&lt;discover&gt;6(?:011|5[0-9]{2})[0-9]{12})|" +
		"(?&lt;amex&gt;3[47][0-9]{13})|" +
		"(?&lt;diners&gt;3(?:0[0-5]|[68][0-9])?[0-9]{11})|" +
		"(?&lt;jcb&gt;(?:2131|1800|35[0-9]{3})[0-9]{11}))$";

Pattern pattern = Pattern.compile(regex);

for (String card : cards)
{
	//Strip all hyphens
	card = card.replaceAll("-", "");

	//Match the card
	Matcher matcher = pattern.matcher(card);

	System.out.println(matcher.matches());

	if(matcher.matches()) {
		//If card is valid then verify which group it belong 
		System.out.println(matcher.group("mastercard"));
	}
}
</pre>
         <h2>Checksum Validation with the Luhn Algorithm</h2>
         <p>There is an extra validation check that you can do on the credit card number before processing the order. The last digit in the credit card number is a checksum calculated according to the <strong><a href="https://en.wikipedia.org/wiki/Luhn_algorithm" title="Luhn algorithm" target="_blank">Luhn algorithm</a></strong>. Since this algorithm requires basic arithmetic, you cannot implement it with a regular expression.</p>
         <p>Below is the method which you can use to run checksum validation using Luhn Algorithm. This function takes a string with the credit card number as a parameter. The card number should consist only of digits.</p>
         <p>The actual algorithm runs on the array of digits, calculating a checksum. If the sum modulus 10 is zero, then the card number is valid. If not, the number is invalid.</p>
         <p>I have taken the reference implementation of Luhn Algo from <strong><a href="https://code.google.com/p/gnuc-credit-card-checker/source/browse/trunk/CCCheckerPro/src/com/gnuc/java/ccc/Luhn.java" title="luhn algo" target="_blank">Google Code</a></strong>.</p>
         <pre class="brush: java; title: ; notranslate" title="">
public class Luhn
{
	public static boolean Check(String ccNumber)
	{
		int sum = 0;
		boolean alternate = false;
		for (int i = ccNumber.length() - 1; i &gt;= 0; i--)
		{
			int n = Integer.parseInt(ccNumber.substring(i, i + 1));
			if (alternate)
			{
				n *= 2;
				if (n &gt; 9)
				{
					n = (n % 10) + 1;
				}
			}
			sum += n;
			alternate = !alternate;
		}
		return (sum % 10 == 0);
	}
}
</pre>
         <p>Feel free to modify above code samples to match other validation rules in above regex, if you need it.</p>
