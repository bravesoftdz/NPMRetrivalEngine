             <h2>Introduction</h2> 
             <p>A while ago I started working on converting an eCommerce payment gateway's (<a href="http://www.datacash.com/">DataCash</a>) COM server to a native .NET assembly using their XML API. Once I had got a basic version working I decided to produce a simple web form to test it out, and so opened it up for all comers (and received some very generous donations from CP members -- thanks guys :). As part of this web form I wanted to include support to check that users had entered a card number, expiration date etc., and then wanted to extend it further to include support for checking that the card number was valid before issuing a request to the payment gateway's server. This is the result, a drop-in replacement for any of the other validation controls.</p> 
             <p>Incidentally, you can see a demo of the validator in use (as well as the card payment gateway assembly) at the following address: <a href="https://ssl500.securepod.com/oobaloo/DataCash/">https://ssl500.securepod.com/oobaloo/DataCash/</a>, besides this you may also be interested in the <a href="http://euro.ecom.cmu.edu/resources/elibrary/everycc.htm">everything you ever wanted to know about CC's</a> guide.</p> 
             <p>Before getting into any of the implementation details here is a simple UML class diagram to show the rough layout of the Control.</p> 
             <p align="left"><img height="484" width="521" src="/KB/E-commerce/creditcardvalidator/umldiagram_v3.gif"></p> 
             <p>The diagram is missing information about parameter types since its not essential to understanding the model. For those who are not familiar with UML, it shows a specialisation relationship between the <code>BaseValidator</code> and <code>CreditCardValidator</code> classes - an is a relationship - demonstrating inheritance from <code>BaseValidator</code> to the more specialised <code>CreditCardValidator</code> class. New with the third incarnation of the control is the <code>AcceptedCardTypes</code> property which is used to specify what types of card should pass the validation using the <code>CardType</code> enumeration.</p> 
             <p>The control includes support for validating card numbers in two ways. Firstly, through checking the card number using Luhn's formula, the details of which are included in the next part of the article. Secondly, the card type itself is examined, and the length is checked. The card type can be determined through a prefix and each type has a specified length, by examining these an additional level of control can be added - the types of card to accept. The method that implements this is <code>IsValidCardType</code>, and whether this is used during the validation is set by the <code>ValidateCardType</code> property. </p> 
             <p>The main way the card number is going to be validated is through Luhn's formula, so firstly a little bit of background information and a demo of how the validation is performed.</p> 
             <h3>Luhn's Formula</h3> 
             <p>The <code>CreditCardValidator</code> control will perform a check on the contents of the textbox using Luhn's formula which is used to validate card numbers. It can be used to check against a number of cards, including the following:</p> 
             <ul> 
              <li>MasterCard</li> 
              <li>VISA</li> 
              <li>American Express</li> 
              <li>Diners Club/Carte Blanche</li> 
              <li>enRoute</li> 
              <li>Discover</li> 
              <li>JCB</li> 
              <li>Solo*</li> 
              <li>Switch*</li> 
             </ul> 
             <p>* These are UK only cards from my recollection, but have been tested by myself and work.</p> 
             <p>Information on the <a href="http://www.webopedia.com/TERM/L/Luhn_formula.html">history of the formula can be found on WebOpedia</a> but so you don't have to read that here is a summary of <a href="http://perl.about.com/library/weekly/aa073000a.htm">how it is performed</a>:</p> 
             <ol> 
              <li> <strong>Double the value of alternating digits</strong><br> The first step is to double each of the alternating digits in the number. But the trick is to start with the second digit from the right and work backwards. Say we have a credit card number 1234 5678 1234 5670. We'll start with the rightmost number 7, double it, and then do the same for every other digit. <p>1234 5678 1234 5670<br> <br> This will give us the following values.<br> <br> 7 x 2 = 14<br> 5 x 2 = 10<br> 3 x 2 = 6<br> .<br> . <br> etc. </p> </li> 
              <li> <p> <strong>Add the separate digits of all the products<br> </strong>Now we'll the separate digits of all the products, and come up with a final sum. </p><p>(1 + 4) + (1 + 0) + 6 + 2 + (1 + 4) + (1 + 0) + 6 + 2 = 28</p> <p> Be sure to add the digits, not just the number.</p> </li> 
              <li><strong>Add the unaffected digits<br> </strong>Now we'll go back to the original number and add all the digits that we didn't double. We'll still start from the right, but this time we'll start from the rightmost number. <p>1234 5678 1234 5670<br> 0 + 6 + 4 + 2 + 8 + 6 + 4 + 2 = 32</p> </li> 
              <li> <p><strong>Add the results and divide by 10 <br> </strong>Finally, we'll add both the results and divide the answer by 10. </p> <p>28 + 32 = 60<br> <br> 60 is evenly divided by 10, so the credit card number is well formed and ready for further processing.</p> </li> 
             </ol> 
             <p>This will be converted into a method which will perform all of the steps listed above on the contents of the specified textbox. By deriving the new validator control from <code>BaseValidator</code> it's possible to produce a control which behaves exactly as any other validator for the easiest deployment.</p> 
             <h2>Luhn's implementation</h2> 
             <p>The code for Luhn's formula is in the <code>ValidateCardNumber</code> method which is implemented as follows:</p> 
             <pre><span class="code-keyword">private</span> <span class="code-keyword">static</span> <span class="code-keyword">bool</span> ValidateCardNumber( <span class="code-keyword">string</span> cardNumber )
{
    <span class="code-keyword">try</span> 
    {
        <span class="code-comment">//</span><span class="code-comment"> Array to contain individual numbers</span>
        System.Collections.ArrayList CheckNumbers = <span class="code-keyword">new</span> ArrayList();
        <span class="code-comment">//</span><span class="code-comment"> So, get length of card</span>
        <span class="code-keyword">int</span> CardLength = cardNumber.Length;
    
        <span class="code-comment">//</span><span class="code-comment"> Double the value of alternate digits, starting with the second digit</span>
        <span class="code-comment">//</span><span class="code-comment"> from the right, i.e. back to front.</span>
        <span class="code-comment">//</span><span class="code-comment"> Loop through starting at the end</span>
        <span class="code-keyword">for</span> (<span class="code-keyword">int</span> i = CardLength-2; i <span class="code-keyword">&gt;</span>= <span class="code-digit">0</span>; i = i - <span class="code-digit">2</span>)
        {
            <span class="code-comment">//</span><span class="code-comment"> Now read the contents at each index, this</span>
            <span class="code-comment">//</span><span class="code-comment"> can then be stored as an array of integers</span>

            <span class="code-comment">//</span><span class="code-comment"> Double the number returned</span>
            CheckNumbers.Add( <span class="code-sdkkeyword">Int32</span>.Parse(cardNumber[i].ToString())*2 );
        }

        <span class="code-keyword">int</span> CheckSum = <span class="code-digit">0</span>;    <span class="code-comment">//</span><span class="code-comment"> Will hold the total sum of all checksum digits</span>
            
        <span class="code-comment">//</span><span class="code-comment"> Second stage, add separate digits of all products</span>
        <span class="code-keyword">for</span> (<span class="code-keyword">int</span> iCount = <span class="code-digit">0</span>; iCount <span class="code-keyword">&lt;</span>= CheckNumbers.Count-1; iCount++)
        {
            <span class="code-keyword">int</span> _count = <span class="code-digit">0</span>;    <span class="code-comment">//</span><span class="code-comment"> will hold the sum of the digits</span>

            <span class="code-comment">//</span><span class="code-comment"> determine if current number has more than one digit</span>
            <span class="code-keyword">if</span> ((<span class="code-keyword">int</span>)CheckNumbers[iCount] <span class="code-keyword">&gt;</span> <span class="code-digit">9</span>)
            {
                <span class="code-keyword">int</span> _numLength = ((<span class="code-keyword">int</span>)CheckNumbers[iCount]).ToString().Length;
                <span class="code-comment">//</span><span class="code-comment"> add count to each digit</span>
                <span class="code-keyword">for</span> (<span class="code-keyword">int</span> x = <span class="code-digit">0</span>; x <span class="code-keyword">&lt;</span> _numLength; x++)
                {
                    _count = _count + <span class="code-sdkkeyword">Int32</span>.Parse( 
                          ((<span class="code-keyword">int</span>)CheckNumbers[iCount]).ToString()[x].ToString() );
                }
            }
            <span class="code-keyword">else</span>
            {
                <span class="code-comment">//</span><span class="code-comment"> single digit, just add it by itself</span>
                _count = (<span class="code-keyword">int</span>)CheckNumbers[iCount];   
            }
            CheckSum = CheckSum + _count;    <span class="code-comment">//</span><span class="code-comment"> add sum to the total sum</span>
        }
        <span class="code-comment">//</span><span class="code-comment"> Stage 3, add the unaffected digits</span>
        <span class="code-comment">//</span><span class="code-comment"> Add all the digits that we didn't double still starting from the</span>
        <span class="code-comment">//</span><span class="code-comment"> right but this time we'll start from the rightmost number with </span>
        <span class="code-comment">//</span><span class="code-comment"> alternating digits</span>
        <span class="code-keyword">int</span> OriginalSum = <span class="code-digit">0</span>;
        <span class="code-keyword">for</span> (<span class="code-keyword">int</span> y = CardLength-1; y <span class="code-keyword">&gt;</span>= <span class="code-digit">0</span>; y = y - <span class="code-digit">2</span>)
        {
            OriginalSum = OriginalSum + <span class="code-sdkkeyword">Int32</span>.Parse(cardNumber[y].ToString());
        }

        <span class="code-comment">//</span><span class="code-comment"> Perform the final calculation, if the sum Mod 10 results in 0 then</span>
        <span class="code-comment">//</span><span class="code-comment"> it's valid, otherwise its false.</span>
        <span class="code-keyword">return</span> (((OriginalSum+CheckSum)%10)==<span class="code-digit">0</span>);
    }
    <span class="code-keyword">catch</span>
    {
        <span class="code-keyword">return</span> <span class="code-keyword">false</span>;
    }
}</pre> 
             <p>The code includes comments that explains how it works, however, here is a summary.</p> 
             <ol> 
              <li>Build an <code>ArrayList</code> which will contain the alternating digits taken in step 1. This is so that the original values can be used again in step 2 but without looping back through the number. This was primarily done for readability.<br> </li> 
              <li>Once the list has been created, a sum is created of the individual digits if the number is greater than 9 (i.e. has more than one digit).<br> </li> 
              <li>The original digits that were untouched are added together, these create the <code>OriginalSum</code> variable. This is then added to the number created as a result of steps 1 and 2, and the value is divided by 10 and the result tested against 0 which provides the return value for the function.</li> 
             </ol> 
             <p>If an exception is thrown throughout the code, then false is returned.</p>

             <p>The <code>CardType</code> (which is an instance of an Int32-based enumerated type) will be used as a set of bit flags - each bit reflects a single card type. So, 0...0001 is MasterCard, 0...0010 is VISA. By using a set of bit flags it will be possible to set a variable to more than one card type, and be able to determine which ones are to be supported.</p> 
             <p>This card type check is to be performed alongside the length check (ensuring that the card number matches the card type's expected length) and for this check we will use a Regular Expression using the .NET Framework's Regex class. Regular Expressions let you perform pattern matches, and can be extremely powerful. For more details on Regular Expressions take a look at <a href="http://msdn.microsoft.com/library/default.asp?url=/library/en-us/cpguide/html/cpconcomregularexpressions.asp">.NET Framework Regular Expressions</a> on MSDN, and if you only want to include this kind of validation you can use the <a href="http://msdn.microsoft.com/library/default.asp?url=/library/en-us/vbcon/html/vbtskValidatingAgainstPatterns.asp">Regular Expression validation control</a>. </p> 
             <p>The card type check also includes support for the end user to specify which card types should pass the validation, this is set through the <code>AcceptedCardTypes</code> property (and then stored in the <code>_cardTypes</code> member variable). The code looks like this:</p> 
             <pre><span class="code-keyword">public</span> <span class="code-keyword">bool</span> IsValidCardType( <span class="code-keyword">string</span> cardNumber )
    {    
    <span class="code-comment">//</span><span class="code-comment"> AMEX -- 34 or 37 -- 15 length</span>
    <span class="code-keyword">if</span> ( (Regex.IsMatch(cardNumber,<span class="code-string">"</span><span class="code-string">^(34|37)"</span>)) 
         &amp;&amp; ((_cardTypes &amp; CardType.Amex)!=<span class="code-digit">0</span>) )
        <span class="code-keyword">return</span> (<span class="code-digit">15</span>==cardNumber.Length);
            
    <span class="code-comment">//</span><span class="code-comment"> MasterCard -- 51 through 55 -- 16 length</span>
    <span class="code-keyword">else</span> <span class="code-keyword">if</span> ( (Regex.IsMatch(cardNumber,<span class="code-string">"</span><span class="code-string">^(51|52|53|54|55)"</span>)) &amp;&amp; 
              ((_cardTypes &amp; CardType.MasterCard)!=<span class="code-digit">0</span>) )
        <span class="code-keyword">return</span> (<span class="code-digit">16</span>==cardNumber.Length);

    <span class="code-comment">//</span><span class="code-comment"> VISA -- 4 -- 13 and 16 length</span>
    <span class="code-keyword">else</span> <span class="code-keyword">if</span> ( (Regex.IsMatch(cardNumber,<span class="code-string">"</span><span class="code-string">^(4)"</span>)) &amp;&amp; 
              ((_cardTypes &amp; CardType.VISA)!=<span class="code-digit">0</span>) )
        <span class="code-keyword">return</span> (<span class="code-digit">13</span>==cardNumber.Length||16==cardNumber.Length);

    <span class="code-comment">//</span><span class="code-comment"> Diners Club -- 300-305, 36 or 38 -- 14 length</span>
    <span class="code-keyword">else</span> <span class="code-keyword">if</span> ( (Regex.IsMatch(cardNumber,<span class="code-string">"</span><span class="code-string">^(300|301|302|303|304|305|36|38)"</span>)) &amp;&amp; 
              ((_cardTypes &amp; CardType.DinersClub)!=<span class="code-digit">0</span>) )
        <span class="code-keyword">return</span> (<span class="code-digit">14</span>==cardNumber.Length);

    <span class="code-comment">//</span><span class="code-comment"> enRoute -- 2014,2149 -- 15 length</span>
    <span class="code-keyword">else</span> <span class="code-keyword">if</span> ( (Regex.IsMatch(cardNumber,<span class="code-string">"</span><span class="code-string">^(2014|2149)"</span>)) &amp;&amp; 
              ((_cardTypes &amp; CardType.DinersClub)!=<span class="code-digit">0</span>) )
        <span class="code-keyword">return</span> (<span class="code-digit">15</span>==cardNumber.Length);

    <span class="code-comment">//</span><span class="code-comment"> Discover -- 6011 -- 16 length</span>
    <span class="code-keyword">else</span> <span class="code-keyword">if</span> ( (Regex.IsMatch(cardNumber,<span class="code-string">"</span><span class="code-string">^(6011)"</span>)) &amp;&amp;
             ((_cardTypes &amp; CardType.Discover)!=<span class="code-digit">0</span>) )
        <span class="code-keyword">return</span> (<span class="code-digit">16</span>==cardNumber.Length);

    <span class="code-comment">//</span><span class="code-comment"> JCB -- 3 -- 16 length</span>
    <span class="code-keyword">else</span> <span class="code-keyword">if</span> ( (Regex.IsMatch(cardNumber,<span class="code-string">"</span><span class="code-string">^(3)"</span>)) &amp;&amp; 
             ((_cardTypes &amp; CardType.JCB)!=<span class="code-digit">0</span>) )
        <span class="code-keyword">return</span> (<span class="code-digit">16</span>==cardNumber.Length);

    <span class="code-comment">//</span><span class="code-comment"> JCB -- 2131, 1800 -- 15 length</span>
    <span class="code-keyword">else</span> <span class="code-keyword">if</span> ( (Regex.IsMatch(cardNumber,<span class="code-string">"</span><span class="code-string">^(2131|1800)"</span>)) &amp;&amp; 
               ((_cardTypes &amp; CardType.JCB)!=<span class="code-digit">0</span>) )
        <span class="code-keyword">return</span> (<span class="code-digit">15</span>==cardNumber.Length);
    <span class="code-keyword">else</span>
    {
        <span class="code-comment">//</span><span class="code-comment"> Card type wasn't recognised, provided Unknown is in the </span>
        <span class="code-comment">//</span><span class="code-comment"> CardTypes property, then return true, otherwise return false.</span>
        <span class="code-keyword">if</span> ( (_cardTypes &amp; CardType.Unknown)!=<span class="code-digit">0</span> )
            <span class="code-keyword">return</span> <span class="code-keyword">true</span>;
        <span class="code-keyword">else</span>
            <span class="code-keyword">return</span> <span class="code-keyword">false</span>;
    }
}</pre> 
             <p>It's not the prettiest of code, but it effectively performs a RegEx comparison on the <code>cardNumber</code> for each possible card type. The regular expression is very simple, and searches for any of the numbers separated by the pipe character. So, for the AMEX type it searches for either 34 or 37. Because the string is prefixed by the epsilon (^) character, the search is performed at the start of the <code>cardNumber</code>. This search is performed through the <code>IsMatch</code> static method of the <code>Regex</code> class. </p> 
             <p>A further test is also done at the same time to determine whether or not the card's type exists in the <code>AcceptedCardTypes</code> property:</p> 
             <pre>&amp;&amp; ((_cardTypes &amp; CardType.Amex)!=<span class="code-digit">0</span>)</pre> 
             <p>Provided both the tests return true (i.e. the prefix is matched, and the card type exists in the <code>_cardTypes</code> member variable) a length test is performed and provided the card number is a valid length then the <code>IsValidCardType</code> method will return true.</p> 
             <p>If the card type is not recognised then as long as <code>Unknown</code> has been set in <code>_cardTypes</code> then <code>IsValidCardType</code> will return true - since the user will have specified that an <code>Unknown</code> card type is an accepted card type. Otherwise, it will return false and will fail.</p> 
             <p>The <code>_cardTypes</code> variable is set using a property accessor which is implemented as follows:</p> 
             <pre><span class="code-keyword">public</span> <span class="code-keyword">string</span> AcceptedCardTypes
{
    <span class="code-keyword">get</span>
    {
        <span class="code-keyword">return</span> _cardTypes.ToString();
    }
    <span class="code-keyword">set</span>
    {
        _cardTypes = (Etier.CardType)
                  Enum.Parse(<span class="code-keyword">typeof</span>(Etier.CardType), <span class="code-sdkkeyword">value</span>, <span class="code-keyword">false</span> );
    }
}</pre> 
             <p>This enables users to specify the card types using a string (e.g. "Amex, VISA, Unknown"), as opposed to having to programmatically set it via. the OnLoad event (e.g. <code>AcceptedCardTypes = CardType.VISA | CardType.Amex</code> etc.)</p> 
             <p>That's the second validation method implemented, all that's left to do is to produce the <code>BaseValidator</code> derived class that uses the above methods to validate the text in the associated control.</p> 
             <h2>The Validation Control's implementation</h2> 
             <p>That is the majority of the code written, all that's left to do is produce a class dervied from <code>System.Web.UI.WebControls.BaseValidator</code> and override the necessary functions. Incidentally, I used <a href="http://www.codeproject.com/aspnet/listcontrol_validator.asp">Cenk Civici's article for a "ListControl SelectedItem Validator"</a> as the main source for this.</p> 
             <p>The <code>BaseValidator</code> requires that we override the <code>EvaluateIsValid</code> method, which suprisingly enough, is the function that determines whether or not the associated control has valid content -- in our case whether or not the associated text box has a valid credit card number entered. Following Cenk Cevi's article I also included an implementation of the <code>ControlPropertiesValid</code> helper function that determines whether the control specified by the <code>ControlToValidate</code> property is a valid control - thus ensuring that it is a textbox we're checking. Since most controls have a text property it's probably not a major issue but it would be strange to be validating the text property of a button etc., so as an extra precaution I included it.</p> 
             <h3>Firstly, ControlPropertiesValid</h3> 
             <pre><span class="code-keyword">protected</span> <span class="code-keyword">override</span> <span class="code-keyword">bool</span> ControlPropertiesValid()
{
    <span class="code-comment">//</span><span class="code-comment"> Should have a text box control to check</span>
    Control ctrl = FindControl(ControlToValidate);

    <span class="code-keyword">if</span> ( <span class="code-keyword">null</span> != ctrl )
    {
        <span class="code-keyword">if</span> (ctrl <span class="code-keyword">is</span> System.Web.UI.WebControls.TextBox)
        {
            _creditCardTextBox = (System.Web.UI.WebControls.TextBox) ctrl;
            <span class="code-keyword">return</span> ( <span class="code-keyword">null</span> != _creditCardTextBox );
        } <span class="code-keyword">else</span>
            <span class="code-keyword">return</span> <span class="code-keyword">false</span>;
    }
    <span class="code-keyword">else</span>
        <span class="code-keyword">return</span> <span class="code-keyword">false</span>;
}
</pre> 
             <p>The code first finds the <code>ControlToValidate</code> and checks that it does indeed point to something, and then checks whether it is a <code>TextBox</code>. If so, it sets the member variable <code>_creditCardTextBox</code> to the <code>TextBox</code> on the web form. If anything bad happens it returns <code>false</code>.</p> 
             <h3>Finally, EvaluateIsValid</h3> 
             <p>This method is declared as abstract in the <code>BaseValidator</code> class, and so has to be implemented by our derived class. It is also the method that is called to check that the contents of the associated control is valid.</p> 
             <p>The <code>CreditCardValidator</code> control includes two additional properties, one of which is <code>ValidateCardType</code> which can be used to set whether or not the card type should also be checked. If it is to be checked, then the length is checked before the number is evaluated against Luhn's formula. However, if the <code>ValidateCardType</code> property is set to false then the card number is validated against Luhn's formula directly.</p> 
             <pre><span class="code-keyword">protected</span> <span class="code-keyword">override</span> <span class="code-keyword">bool</span> EvaluateIsValid()
{
    <span class="code-keyword">if</span> (_validateCardType)    <span class="code-comment">//</span><span class="code-comment"> should the length be validated also?</span>
    {
        <span class="code-comment">//</span><span class="code-comment"> Check the length, if the length is fine then validate the</span>
        <span class="code-comment">//</span><span class="code-comment"> card number</span>
        <span class="code-keyword">if</span> (IsValidCardType(_creditCardTextBox.Text))
            <span class="code-keyword">return</span> ValidateCardNumber( _creditCardTextBox.Text );
        <span class="code-keyword">else</span>
            <span class="code-keyword">return</span> <span class="code-keyword">false</span>;        <span class="code-comment">//</span><span class="code-comment"> Invalid length</span>
    }
    <span class="code-keyword">else</span>
        <span class="code-comment">//</span><span class="code-comment"> Check that the text box contains a valid number using </span>
        <span class="code-comment">//</span><span class="code-comment"> the ValidateCardNumber method</span>
        <span class="code-keyword">return</span> ValidateCardNumber( _creditCardTextBox.Text );
}</pre> 
             <p>Provided the <code>ValidateCardNumber</code> method succeeds, the validation is considered a success. </p>
             <h2>Using the Credit Card Validator Control</h2> 
             <p>That's all that's necessary to have the <code>CreditCardValidator</code> control finished. Now for a quick example of how it can be used in a real web form (the full code for this is included as a download at the top of the page).</p> 
             <p>The first that needs to be done is to include the declaration at the top of the aspx page that imports the assembly and maps the namespace to a prefix. This also requires that the assembly's DLL file is copied to the bin directory for the application. The location of this directory depends upon the setup of your application, but assuming there's a virtual directory off the root called /CreditCard/ then your bin directory would be /CreditCard/bin/.</p> 
             <pre><span class="code-pagedirective">&lt;%@</span><span class="code-attribute"> </span><span class="code-leadattribute">Register</span><span class="code-attribute"> </span><span class="code-attribute">TagPrefix</span><span class="code-keyword">="</span><span class="code-keyword">etier"</span><span class="code-attribute"> </span><span class="code-attribute">Namespace</span><span class="code-keyword">="</span><span class="code-keyword">Etier"</span><span class="code-attribute"> </span><span class="code-attribute">Assembly</span><span class="code-keyword">="</span><span class="code-keyword">CreditCardValidator"</span><span class="code-attribute"> </span><span class="code-pagedirective">%&gt;</span></pre>

             <pre>&lt;etier:CreditCardValidator

  AcceptedCardTypes=<span class="code-string">"</span><span class="code-string">Amex, VISA, MasterCard"</span>
/&gt;</pre> 
             <p>The <code>CreditCardValidator</code> offers a few properties that are not taken from the <code>BaseValidator</code> base type, these are <code>ValidateCardType</code> and <code>AcceptedCardTypes</code>. The <code>ValidateCardType</code> property sets whether or not the card type should be checked. This will mean that a length test is performed, as well as specifying which card types should be accepted. The AcceptedCardTypes property can be set using the CardType enumeration. In the above code the accepted types are Amex, VISA, and MasterCard. If unrecognised card types are also to be accepted then you can include "Unknown" in the list. If you want all the recognised types accepted then you can use "All". Thus, to accept anything you should use "Unknown, All". </p>
             <p>Since I chose to have the error message displayed in a <code>ValidationSummary</code> control, I set the <code>Display</code> property to <code>none</code> to ensure that it wasn't displayed in-line. Otherwise, the error message would be put at the location of this control. </p>
             <h2>Conclusion</h2> 
             <p>ASP.NET provides a huge amount of functionality built in, and I can't remember how many times I've produced custom code to do the very thing that Validation controls provide. This, coupled with the Object-Oriented nature of .NET, means that you can extend the existing .NET offerings into something you can use time and again. Now that xcopy deployment is a real option with web applications, dropping in support for your own custom validation controls can be done in a fraction of the time than with traditional ASP development.</p> 
             <p>This concludes my second article, and I hope its of use to people. It doesn't really show anything new, but does roll up some important functionality into a re-usable control that can be deployed and used extremely easily.</p> 
             <h2>Things that are missing/Ideas for improvement</h2> 
             <p>At the moment, all validation is taken care of on the server-side. This is fine since the aim was to prevent requests being issued to the payment gateway with incorrect details - especially since there's likely to be a delay of a few seconds. However, since the validation algorithm is not a secret and is relatively simple to implement, a client side version could be implemented to check the value before the data is posted back to the form and thus saving round-trips.</p> 
             <p>If anyone decides to extend this to include client side scripting, or has any comments or questions then it'd be great to hear from you. </p>
