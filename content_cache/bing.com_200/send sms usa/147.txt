      <h1>SMS Packet Format</h1> 
      <!-- <link rel="File-List" href="sms_messaging_files/filelist.xml"> --> 
      <h2>1. Introduction</h2> 
      <p> To use the SMS you have to declare the number of the Short Message Service Centre (SMSC<a href="#_ftn1" name="a_ftnref1" title>[1]</a>) in the Mobile Station (MS), provided that the MS supports Short Message Service-Mobile Orginated (SMS-MO). </p> 
      <p>The <b>M20 Terminal</b> supports SMS-MO.</p> 
      <table> 
       <thead> 
        <tr> 
         <th width="142">Network</th> 
         <th width="170">SMSC-number (Australia)</th> 
        </tr> 
       </thead> 
       <tbody> 
        <tr> 
         <td width="142">Telstra</td> 
         <td width="170">61418706700 </td> 
        </tr> 
        <tr> 
         <td width="142">Optus Vodafone</td> 
         <td width="170">61411990000 / 61415011501</td> 
        </tr> 
       </tbody> 
      </table> 
      <p>At the M20 Terminal you enter the SMSC-number with the AT+Celular command:</p> 
      <pre class="ax_code">at+csca = ?&lt;SMSC-number&gt;?</pre> 
      <p>If the receiver of the SMS possesses a Telstra SIM card, the AT command has to be entered in the following way:</p> 
      <pre class="ax_code">at+csca = "+61418706700"</pre> 
      <p>With the command</p> 
      <pre class="ax_code">at+csca? </pre> 
      <p>you can question the current SMSC-number.</p> 
      <p><b>Ask your network operator for the right SMSC-number !!</b></p> 
      <p> <b> ! Notice: In addition to the AT+CSCA command it is possible to enter the SMSC-number in front of the Protocol Data Unit (PDU). Refer to section 3.1 for details! </b> </p>

      <p>Mobile Station</p> 
      <p>Short Message Entity</p> 
      <p>Short Message Service Centre</p> 
      <p>Man Machine Interface</p> 
      <p>Protocol Data Units</p> 
      <p>Short Message Aplication Layer</p> 
      <p>Short Message Transport Layer</p> 
      <p>Short Message Relay Layer</p> 
      <p>Short Message Link Layer</p> 
      <p> The MMI is based on the command set of AT+Cellular, and could be realized by means of a terminal (for example Win-Terminal, HyperTerminal, etc) or the display of a handy. </p> 
      <p> The SM-TL provides a service to the Short Message Application Layer. This service enables the SM-AL to transfer short messages to its peer entity, receive short messages from its peer entity and receive reports about earlier requests for short messages to be transferred. The SM-TL communicates with its peer entity with six several PDUs (Protocol Data Units): </p> 
      <p>SMS-DELIVER, conveying a short message from the SMSC to the MS</p> 
      <p>SMS-DELIVER-REPORT, conveying a failure cause (if necessary)</p> 
      <p>SMS-SUBMIT, conveying a short message from the MS to the SMSC</p> 
      <p>SMS-SUBMIT-REPORT, conveying a failure cause (if necessary)</p> 
      <p>SMS-STATUS-REPORT, conveying a status report from the SMSC to the MS</p> 
      <p>SMS-COMMAND, conveying a command from the MS to the SMSC</p> 
      <p> The M20 Terminal supports the SMS-DELIVER and SMS-SUBMIT PDUs as described in the following sections. </p>

      <h3>3.1 Service Centre address information element (SCA info element)</h3> 
      <p><img width="511" height="182" src="sms_messaging_files/image008.gif"></p> 
      <p>The octet ?len? contains the number of octets required for the number of the Service Centre plus the 1 byte ?type of number?.</p> 
      <p>type of number:</p> 
      <p>81H:&nbsp; the following number is national</p> 
      <p>91H:&nbsp; the following number international </p> 
      <p>(For further information see GSM 04.08 chapter 10.5.4.6)</p> 
      <p>octet:</p> 
      <p>One octet includes two BCD-digit Fields. If the called party BCD number contains an odd number of digits, the last digit shall be filled with an end mark coded as ?FH?.</p> 
      <p>Example:</p> 
      <p>if you have the SC-number&nbsp; +61418706700 you have to type:</p> 
      <p>07911614786007F0</p> 
      <p> Notice: If the ?len? field is set to Zero the M20 Terminal takes the default value of the Service Centre address set by the AT+CSCA command </p>

      <h3>3.2 Protocol Data Unit Type (PDU Type)</h3> 
      <p class="ax_MsoFooter">&nbsp;</p> 
      <p>SMS-SUBMIT: SMS-DELIVER:</p> 
      <img width="247" height="82" src="sms_messaging_files/image010.gif"> 
      <img width="237" height="83" src="sms_messaging_files/image012.gif"> 
      <p>Notice: you have to write the PDU-type in Hex-Format, a possible example is ?11H?!</p> 
      <p>RP: 0 Reply Path parameter is not set in this PDU</p> 
      <p>1 Reply Path parameter is set in this PDU</p> 
      <p>UDHI:&nbsp; <b>0</b>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;<b>The UD field contains only the short message</b></p> 
      <p><b>1 The beginning of the UD field contains a header in addition of the short message </b></p> 
      <p>SRI: (is only set by the SMSC)</p> 
      <p>0 A status report will not be returned to the SME</p> 
      <p>1 A status report will be returned to the SME</p> 
      <p>SRR:&nbsp;&nbsp;&nbsp; <b>0 A status report is not requested</b></p> 
      <p><b>1 A status report is requested</b></p> 
      <p>VPF: bit4&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; bit3</p> 
      <p><b>0 VP field is not present</b></p> 
      <p>0 1 Reserved</p> 
      <p><b>1 &nbsp; 0&nbsp; VP field present an integer represented (relative)</b></p> 
      <p><b>1 &nbsp; 1&nbsp; VP field present an semi-octet represented (absolute)</b></p>

      <p>MMS: (is only set by the SMSC)</p>

      <p>1&nbsp; No more messages are waiting for the MS in the SMSC</p> 
      <p>RD:&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; <b> 0&nbsp; Instruct the SMSC to accept an SMS-SUBMIT for an short message still held in the SMSC which has the same MR and DA as a previously submitted short message from the same OA.</b> </p> 
      <p><b>&nbsp; 1&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Instruct the SMSC to reject an SMS-SUBMIT for a short message still </b></p>

      <p>submitted short message from the same OA.</p>

      <p>0&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; 1&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; SMS-SUBMIT REPORT (SMSC ==&gt; MS)</p>

      <h3>3.5 Protocol Identifier (PID)</h3> 
      <img width="265" height="74" src="sms_messaging_files/image018.gif"> 
      <p> The PID is the information element by which the Transport Layer either refers to the higher layer protocol being used, or indicates interworking with a certain type of telematic device. Here are some examples of PID codings: </p> 
      <p><b>&nbsp;&nbsp; 00H: The PDU has to be treat as a short message</b></p> 
      <p>&nbsp;&nbsp; 01H: The PDU has to be treat as a telex</p> 
      <p>&nbsp;&nbsp; 02H: The PDU has to be treat as group3 telefax</p> 
      <p>&nbsp;&nbsp; 03H: The PDU has to be treat as group4 telefax</p> 
      <p>(For further information see GSM 03.40 chapter 9.2.3.9)</p> 
      <p>Notice: it is not guaranteed that the SMSC supports every PID codings</p>

      <h3>3.7 Service Centre Time Stamp (SCTS)</h3> 
      <p> The SCTS is the information element by which the SMSC informs the recipient MS about the time of arrival of the short message at the Transport Layer entity of the SMSC. The time value is included in every SMS-DELIVER being delivered to the SMSC, and represents the local time in the following way: </p> 
      <img width="554" height="214" src="sms_messaging_files/image022.gif"> 
      <p> The Time Zone indicates the difference, expressed in quarters of an hour, between the local time and GMT (Greenwich Main Time). </p> 
      <h3><a name="a_Toc457134854">3.8 Validity Period (VP</a>)</h3> 
      <p> The Validity-Period is the information element which gives an MS submitting an SMS-SUBMIT to the SMSC the possibility to include a specific time period value in the short message. The Validity Period parameter value indicates the time period for which the short message is valid, i.e. for how long the SMSC shall guarantee its existence in the SMSC memory before delivery to the recipient has been carried out. </p> 
      <img width="538" height="227" src="sms_messaging_files/image024.gif"> 
      <p> The VP field is given in either integer or semi-octet representation. In the first case, the VP comprises 1 octet, giving the length of the validity period, counted from when the SMS-SUBMIT is received by the SMSC. In the second case, the VP comprises 7 octets, giving the absolute time of the vality period termination. In the first case, the representation of time is as follows: </p> 
      <table> 
       <tbody>
        <tr> 
         <td width="137">VP Value</td> 
         <td width="472">Validity period value</td> 
        </tr> 
        <tr> 
         <td width="137">0-143</td> 
         <td width="472">(VP + 1) x 5 minutes (i.e 5 minutes intervals up to 12 hours)</td> 
        </tr> 
        <tr> 
         <td width="137">144-167</td> 
         <td width="472">12 hours + ((VP-143) x 30 minutes)</td> 
        </tr> 
        <tr> 
         <td width="137">168-196</td> 
         <td width="472">(VP-166) x 1 day</td> 
        </tr> 
        <tr> 
         <td width="137">197-255</td> 
         <td width="472">(VP - 192) x 1 week</td> 
        </tr> 
       </tbody>
      </table> 
      <p> In the second case, the representation of time is identical to the representation or the SCTS (Service Centre Time Stamp). The case of representation is set in the VPF (Validity Period Format) in the PDU-type. </p>

      <h2>Appendix B- SMS set up for the M20 Terminal</h2> 
      <p> This document describes the process of sending SMS messages between a mobile phone and the M20 Terminal. The mobile phone referred to in this document is the Ericsson GH688. However, SMS message can be sent out or received in a similar fashion using other mobile phones. </p> 
      <h3>Hardware Requirements</h3> 
      <p>The following items are needed.</p> 
      <p>1. A mobile phone that is capable of sending and receiving SMS messages</p> 
      <p>2. A M20 Terminal</p> 
      <p>3. Two SIM cards (one for the mobile phone and the other for the M20 Terminal) </p> 
      <p>4. A GSM antenna</p> 
      <p>5. A power cable for the M20 Terminal</p> 
      <p>6. A RS-232 cable </p> 
      <p>7. A PC running on Windows Terminal, HyperTerminal or equivalent.</p> 
      <h3>Hardware Set Up</h3> 
      <p>1. Mobile Phone Set Up</p> 
      <p> Insert a SIM card into the mobile phone and turn the phone on. The phone is now ready for sending and receiving SMS. Note that you need the phone number for SMS messages. </p> 
      <p>2. M20 Terminal Set Up</p> 
      <p> Connect the M20 Terminal to a PC as shown in Figure 1

      <p>1. Turn on the PC and run Windows Terminal, or HyperTerminal.</p> 
      <p>2. Connect the M20 Terminal to COM1 or COM2 of the PC.</p>

      <h3>Sending a SMS Message</h3> 
      <p><b>1) Phone initiated SMS Message</b></p> 
      <p> A mobile phone that is capable of sending and receiving SMS messages can be used to send a SMS message to the M20 Terminal. Note that the SIM card for the mobile phone must be on the same network as the SIM card in the M20 Terminal for SMS messages. eg. both SIM cards must be Telstra, or Optus. </p> 
      <p> To send a SMS message to the M20 Terminal, select [Send Messages] from the [Mail] menu and then select [New]. Enter your text and when you have finished press the ?YES? button. You will then need to enter the destination number for the SMS message. This is the phone number on the SIM card used by the M20 Terminal<a href="#_ftn4" name="a_ftnref4" title>
        <!--[if !supportFootnotes]-->[4]
        <!--[endif]--></a>. </p> 
      <p><b>2) M20 Terminal initiated SMS Message</b></p> 
      <p><i>a) Send a PDU SMS</i></p> 
      <p> In PDU mode, to send a message like the word ?hello?, initially, you have to convert it to a PDU format message. Refer to section 3 (Parameter description) for details on how to construct the PDU message. Note that the actual length of the PDU string (without the Service Centre Address) must be specified for all SMS. Follow the steps below for sending the SMS message. </p> 
      <p> Step 1. Enter the actual length of the SMS message in octets<a href="#_ftn5" name="a_ftnref5" title>
        <!--[if !supportFootnotes]-->[5]
        <!--[endif]--></a> 
       <!--[if !supportLineBreakNewLine]-->&nbsp;
       <!--[endif]--> </p> 
      <p>AT+CMGS=18<a href="#_ftn6" name="a_ftnref6" title>
        <!--[if !supportFootnotes]-->[6]
        <!--[endif]--></a></p> 
      <p>Step 2. Enter the SMS message in PDU format and terminate it with ?CTRL Z?</p> 
      <p>&gt;0011000A81409178699100000105E8329BFD06[<b>CTRL Z</b>]</p> 
      <p>The M20 Terminal should return</p> 
      <p>+CMGS: 12</p> 
      <p>where 12 is the message reference MR, which is different for every SMS message sent.</p> 
      <p><b>b) Send a text SMS</b></p> 
      <p>In text mode, to send a message, eg. ?hello?, follow the three steps below.</p> 
      <p>Step 1: Set the M20 Terminal for text SMS using</p> 
      <p>AT+CMGF=1[ENTER]</p> 
      <p>Step 2: Enter the destination phone number in international format<a href="#_ftn7" name="a_ftnref7" title>
        <!--[if !supportFootnotes]-->[7]
        <!--[endif]--></a></p> 
      <pre class="ax_code">
   AT+CMGS=?+61419879619?[ENTER]
   </pre> 
      <p>Step 3: Enter the text message and terminate it with ?CTRL Z?</p> 
      <pre class="ax_code">
   &gt;hello[<b>CTRL Z</b>]
   </pre> 
      <p>The M20 Terminal should return</p> 
      <pre class="ax_code">
  +CMGS: 1
  OK
  </pre> 
      <p>where 1 is the message reference MR, which is different for every SMS message sent.</p> 
      <h3>Receiving/Reading/Deleting a SMS Message</h3> 
      <p>1) Mobile Phone</p> 
      <p>When there is a new SMS message arrived, the phone will beep and the SMS message indicator will appear on the phone screen.</p> 
      <p>To read the SMS message, select [Read Messages] from the [Mail] menu using the left or right arrow button and the new message is usually shown first. Press the ?YES? button</p> 
      <p>to read the message. The message can be deleted using the ?CLR? button.</p> 
      <p>2) M20 Terminal</p> 
      <p><b>a)Read a PDU SMS</b></p> 
      <p> In PDU mode, when the M20 Terminal receives a SMS message, the following message will appear on the PC screen.</p> 
      <pre class="ax_code">
    &nbsp; +CMTI: ?SM?, 1
    </pre> 
      <p>where 1 is the memory location in which the message can be read from.</p> 
      <p>To read a SMS message from a particular location in memory (eg. location 1) use the AT+CGMR command as follow.</p> 
      <pre class="ax_code">
    AT+CMGR=1[ENTER]
    </pre> 
      <p>The M20 Terminal should return the PDU message as follow.</p> 
      <pre class="ax_code">
    +CMGR: 
    0,,24
    07911614786007F0040B911604994743F400009930139100406B05E8329BFD06
    &nbsp;&nbsp; OK
    </pre> 
      <p>where 0 is the status code indicating a received and already read message, 24 is the hexadecimal number indicating the length of the message.</p> 
      <p><b>b) Read a text SMS</b></p> 
      <p>Like PDU mode, when the M20 Terminal receives a SMS message, the following message will appear on the PC screen.</p> 
      <pre class="ax_code">
    +CMTI: ?SM?, 1
    </pre> 
      <p>where 1 is the memory location in which the message can be read from.</p> 
      <p>To read the SMS message use the AT+CGMR command as follow.</p> 
      <pre class="ax_code">
    AT+CMGR=1[ENTER]
    </pre> 
      <p>The M20 Terminal should return the text message as follow.</p> 
      <pre class="ax_code">
    +CMGR: ?REC READ?,?+61407809050?,?98/12/01,20:16:11+44?
    hello
    OK
    </pre> 
      <p><b>c) Delete a SMS message</b></p> 
      <p> The SMS message can be deleted from memory (eg. location 1) using the AT+CMGD command as follow. Note that there is no AT command to delete all the SMS messages at once. </p> 
      <pre class="ax_code">
    AT+CMGD=1[ENTER]
    </pre> 
      <p>The M20 Terminal should return OK.</p>
