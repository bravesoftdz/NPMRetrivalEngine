      <div class="topic"> 
       <h1 class="title">How to Create Packages and Programs in Configuration Manager</h1> 
       <div id="mainSection"> 
        <div id="mainBody">
         &nbsp;
         <p>Updated: May 14, 2015</p>
         <p>Applies To: System Center 2012 Configuration Manager, System Center 2012 Configuration Manager SP1, System Center 2012 Configuration Manager SP2, System Center 2012 R2 Configuration Manager, System Center 2012 R2 Configuration Manager SP1</p>
         <div class="introduction">
          <p>You can create or import a Microsoft&nbsp;System&nbsp;Center&nbsp;2012 Configuration&nbsp;Manager package and program by using one of the following procedures in this topic:</p>
          <ul class="unordered">
           <li><p><a href="#BKMK_Create_Package">How to Create a Package and Program by using the Create Package and Program Wizard</a></p></li>
           <li><p><a href="#BKMK_Pacakage_Def">How to Create a Package and Program from a Package Definition File</a></p></li>
           <li><p><a href="#BKMK_Import">How to Import a Package and Program</a></p></li>
          </ul>
         </div>
         <a id="BKMK_Create_Package"></a>
         <div>
          <h2 class="LW_CollapsibleArea_TitleDiv">
           <div>
            <a class="LW_CollapsibleArea_TitleAhref" title="Expand" role="heading"><span class="cl_CollapsibleArea_collapsing LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">How to Create a Package and Program by using the Create Package and Program Wizard</span></a>
            <div id="Anchor_0" class="LW_CollapsibleArea_Anchor_Div">
             <a href="/en-us/library/gg682112.aspx#Anchor_0" class="LW_CollapsibleArea_Anchor_Img" title="Right-click to copy and share the link for this section"></a>
            </div>
            <div class="LW_CollapsibleArea_HrDiv">
             <hr class="LW_CollapsibleArea_Hr">
            </div>
           </div></h2>
          <div class="sectionnone">
           <div class="section">
            <p>You can create a new package and program by using the Create Package and Program Wizard. To do so, use the following procedure.</p>
            <div>
             <h4 class="LW_CollapsibleArea_TitleDiv">
              <div>
               <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">To create a package and program</span></a>
               <div class="LW_CollapsibleArea_HrDiv">
                <hr class="LW_CollapsibleArea_Hr">
               </div>
              </div></h4>
             <div class="sectionblock">
              <ol class="steps">
               <li class="step">
                <div class="section">
                 <p>In the Configuration Manager console, click <strong>Software Library</strong>.</p>
                </div></li>
               <li class="step">
                <div class="section">
                 <p>In the <strong>Software Library</strong> workspace, expand <strong>Application Management</strong>, and then click <strong>Packages</strong>.</p>
                </div></li>
               <li class="step">
                <div class="section">
                 <p>In the <strong>Home</strong> tab, in the <strong>Create</strong> group, click <strong>Create Package</strong>.</p>
                </div></li>
               <li class="step">
                <div class="section">
                 <p>On the <strong>Package</strong> page of the <strong>Create Package and Program Wizard</strong>, specify the following information:</p>
                 <ul class="unordered">
                  <li><p><strong>Name:</strong> Specify a name for the package with a maximum of 50 characters.</p></li>
                  <li><p><strong>Description:</strong> Optionally specify a description for this package with a maximum of 128 characters.</p></li>
                  <li><p><strong>Manufacturer:</strong> Optionally specify a manufacturer name to help you identify the package in the Configuration Manager console. This name can be a maximum of 32 characters.</p></li>
                  <li><p><strong>Language:</strong> Optionally specify the language version of the package with a maximum of 32 characters.</p></li>
                  <li><p><strong>Version:</strong> Optionally specify a version number for the package with a maximum of 32 characters.</p></li>
                  <li><p><strong>This package contains source files</strong> - This setting indicates whether the package requires source files to be present on client devices. By default, this check box is cleared and Configuration Manager does not use distribution points for the package. When this check box is selected, distribution points are used.</p></li>
                  <li><p><strong>Source folder:</strong> If the package contains source files, click <strong>Browse</strong> to open the <strong>Set Source Folder</strong> dialog box and specify the location of the source files for the package.</p>
                   <div class="alert">
                    <table summary="table">
                     <tbody>
                      <tr>
                       <th align="left" scope="col"><span><img id="s-e6f6a65cf14f462597b64ac058dbe1d0-system-media-system-caps-note" alt="System_CAPS_note" src="https://i-technet.sec.s-msft.com/dynimg/IC101471.jpeg" title="System_CAPS_note" xmlns=""></span><span class="alertTitle">Note </span></th>
                      </tr>
                      <tr>
                       <td><p>The computer account of the site server must have read access permissions to the source folder that you specify.</p></td>
                      </tr>
                     </tbody>
                    </table>
                   </div></li>
                 </ul>
                </div></li>
               <li class="step">
                <div class="section">
                 <p>On the <strong>Program Type</strong> page of the <strong>Create Package and Program Wizard</strong>, select the type of program to create, and then click <strong>Next</strong>. You can create a program for a computer or device, or you can skip this step and create a program later.</p>
                 <div class="alert">
                  <table summary="table">
                   <tbody>
                    <tr>
                     <th align="left" scope="col"><span><img id="s-e6f6a65cf14f462597b64ac058dbe1d0-system-media-system-caps-important" alt="System_CAPS_important" src="https://i-technet.sec.s-msft.com/dynimg/IC160177.jpeg" title="System_CAPS_important" xmlns=""></span><span class="alertTitle">Important </span></th>
                    </tr>
                    <tr>
                     <td><p>You can only create packages and programs for devices running Windows CE.</p></td>
                    </tr>
                   </tbody>
                  </table>
                 </div>
                 <div class="alert">
                  <table summary="table">
                   <tbody>
                    <tr>
                     <th align="left" scope="col"><span><img id="s-e6f6a65cf14f462597b64ac058dbe1d0-system-media-system-caps-note" alt="System_CAPS_note" src="https://i-technet.sec.s-msft.com/dynimg/IC101471.jpeg" title="System_CAPS_note" xmlns=""></span><span class="alertTitle">Note </span></th>
                    </tr>
                    <tr>
                     <td><p>To create a new program for an existing package, select the package, and then, in the <strong>Home</strong> tab, in the <strong>Package</strong> group, click <strong>Create Program</strong> to open the <strong>Create Program Wizard</strong>.</p></td>
                    </tr>
                   </tbody>
                  </table>
                 </div>
                </div></li>
               <li class="step">
                <div class="section">
                 <p>Use one of the following procedures to create a standard program or a device program.</p>
                </div></li>
              </ol>
             </div>
            </div>
            <div class="section" bookmarktitle="To create a standard program">
             <h3 class="subHeading">To create a standard program</h3>
             <ol class="steps">
              <li class="step">
               <div class="section">
                <p>On the <strong>Program Type</strong> page of the <strong>Create Package and Program Wizard</strong>, select <strong>Standard Program</strong>, and then click <strong>Next</strong>.</p>
               </div></li>
              <li class="step">
               <div class="section">
                <p>On the <strong>Standard Program</strong> page of the Wizard, specify the following information:</p>
                <ul class="unordered">
                 <li><p><strong>Name:</strong> Specify a name for the program with a maximum of 50 characters.</p>
                  <div class="alert">
                   <table summary="table">
                    <tbody>
                     <tr>
                      <th align="left" scope="col"><span><img id="s-e6f6a65cf14f462597b64ac058dbe1d0-system-media-system-caps-note" alt="System_CAPS_note" src="https://i-technet.sec.s-msft.com/dynimg/IC101471.jpeg" title="System_CAPS_note" xmlns=""></span><span class="alertTitle">Note </span></th>
                     </tr>
                     <tr>
                      <td><p>The program name must be unique within a package. After you create a program, you cannot modify its name.</p></td>
                     </tr>
                    </tbody>
                   </table>
                  </div></li>
                 <li><p><strong>Command Line:</strong> Enter the command line to be used to start this program, or click <strong>Browse</strong> to browse to the file location.</p><p>If a specified file name does not have an extension specified, Configuration Manager attempts to use .com, .exe, and .bat as possible extensions.</p><p>When the program is run on a client, Configuration Manager first searches for the command-line file name within the package, searches next in the local Windows folder, and then searches in local %path%. If the file cannot be found, the program fails.</p></li>
                 <li><p><strong>Startup folder:</strong> Optionally use this field to specify the folder from which the program runs, up to 127 characters. This folder can be an absolute path on the client or a path relative to the distribution point folder that contains the package.</p></li>
                 <li><p><strong>Run:</strong> Specifies the mode in which the program will run on client computers. Select one of the following:</p>
                  <ul class="unordered">
                   <li><p><strong>Normal</strong> - The program runs in the normal mode based on system and program defaults. This is the default mode.</p></li>
                   <li><p><strong>Minimized</strong> – The program runs minimized on client devices. Users might see installation activity in the notification area or taskbar.</p></li>
                   <li><p><strong>Maximized</strong> – The program runs maximized on client devices. Users will see all installation activity.</p></li>
                   <li><p><strong>Hidden</strong> – The program runs hidden on client devices. Users will not see any installation activity.</p></li>
                  </ul></li>
                 <li><p><strong>Program can run:</strong> Specify whether the program can run only when a user is logged on, run only when no user is logged on, or run regardless of whether a user is logged on to the client computer.</p></li>
                 <li><p><strong>Run mode:</strong> Specify whether the program will run with administrative permissions or with the permissions of the currently logged on user.</p></li>
                 <li><p><strong>Allow users to view and interact with the program installation</strong> - Use this setting, if available, to specify whether to allow users to interact with the program installation. This check box is available only when <strong>Only when no user is logged on</strong> or <strong>Whether or not a user is logged on</strong> is selected for <strong>Program can run</strong> and <strong>Run with administrative rights</strong> is selected for <strong>Run mode</strong>.</p></li>
                 <li><p><strong>Drive mode:</strong> Specify information about how this program will runs on the network. Choose one of the following:</p>
                  <ul class="unordered">
                   <li><p><strong>Runs with UNC name</strong> - Indicates that the program runs with a Universal Naming Convention (UNC) name. This is the default setting.</p></li>
                   <li><p><strong>Requires drive letter</strong> - Indicates that the program requires a drive letter to fully qualify its location. For this setting, Configuration Manager can use any available drive letter on the client.</p></li>
                   <li><p><strong>Requires specific drive letter (example: Z:)</strong> - Indicates that the program requires a specific drive letter that you specify to fully qualify its location. If the specified drive letter is already used on a client, the program does not run.</p></li>
                  </ul></li>
                 <li><p><strong>Reconnect to distribution point at log on</strong> - Use this check box to indicate whether the client computer reconnects to the distribution point when the user logs on. By default, this check box is cleared.</p></li>
                </ul>
               </div></li>
              <li class="step">
               <div class="section">
                <p>On the <strong>Requirements</strong> page of the Create Package and Program Wizard, specify the following information:</p>
                <ul class="unordered">
                 <li><p><strong>Run another program first</strong> – You can use this setting to identify a package and program that will be run before this package and program will be run.</p></li>
                 <li><p><strong>Platform requirements</strong> – Select <strong>This program can run on any platform</strong> or select <strong>This program can run only on specified platforms</strong> and then choose the operating systems that clients must be running to be able to install the package and program.</p></li>
                 <li><p><strong>Estimated disk space:</strong> Specify the amount of disk space that the software program requires to be able to run on the computer. This can be specified as <strong>Unknown</strong> (the default setting) or as a whole number greater than or equal to zero. If a value is specified, units for the value must also be specified.</p></li>
                 <li><p><strong>Maximum allowed run time (minutes):</strong> Specify the maximum time that the program is expected to run on the client computer. This can be specified as <strong>Unknown</strong> (the default setting) or as a whole number greater than zero.</p><p>By default, this value is set to 120 minutes.</p>
                  <div class="alert">
                   <table summary="table">
                    <tbody>
                     <tr>
                      <th align="left" scope="col"><span><img id="s-e6f6a65cf14f462597b64ac058dbe1d0-system-media-system-caps-important" alt="System_CAPS_important" src="https://i-technet.sec.s-msft.com/dynimg/IC160177.jpeg" title="System_CAPS_important" xmlns=""></span><span class="alertTitle">Important </span></th>
                     </tr>
                     <tr>
                      <td><p>If you are using maintenance windows for the collection on which this program is run, a conflict may occur if the <strong>Maximum allowed run time</strong> is longer than the scheduled maintenance window. However, if the maximum run time is set to <strong>Unknown</strong>, the program will start to run during the maintenance window and will continue to run as needed after the maintenance window is closed. If the user sets the maximum run time to a specific period that exceeds the length of any available maintenance window, then the program will not be run.</p></td>
                     </tr>
                    </tbody>
                   </table>
                  </div><p>If the value is set as <strong>Unknown</strong>, Configuration Manager sets the maximum allowed run time as 12 hours (720 minutes).</p>
                  <div class="alert">
                   <table summary="table">
                    <tbody>
                     <tr>
                      <th align="left" scope="col"><span><img id="s-e6f6a65cf14f462597b64ac058dbe1d0-system-media-system-caps-note" alt="System_CAPS_note" src="https://i-technet.sec.s-msft.com/dynimg/IC101471.jpeg" title="System_CAPS_note" xmlns=""></span><span class="alertTitle">Note </span></th>
                     </tr>
                     <tr>
                      <td><p>If the maximum run time (whether set by the user or as the default value) is exceeded, Configuration Manager will stop the program if <strong>run with administrative rights</strong> is selected and <strong>Allow users to view and interact with the program installation</strong> is not selected.</p></td>
                     </tr>
                    </tbody>
                   </table>
                  </div></li>
                </ul>
               </div></li>
              <li class="step">
               <div class="section">
                <p>Click <strong>Next</strong> and continue to <a href="#BKMK_Summ">To complete the Create Package and Program Wizard</a>.</p>
               </div></li>
             </ol>
            </div>
            <div class="section" bookmarktitle="To create a device program">
             <h3 class="subHeading">To create a device program</h3>
             <ol class="steps">
              <li class="step">
               <div class="section">
                <p>On the <strong>Program Type</strong> page of the <strong>Create Package and Program Wizard</strong>, select <strong>Program for device</strong>, and then click <strong>Next</strong>.</p>
               </div></li>
              <li class="step">
               <div class="section">
                <p>On the <strong>Program for Device</strong> page of the Wizard, specify the following information:</p>
                <ul class="unordered">
                 <li><p><strong>Name:</strong> Specify a name for the program with a maximum of 50 characters.</p>
                  <div class="alert">
                   <table summary="table">
                    <tbody>
                     <tr>
                      <th align="left" scope="col"><span><img id="s-e6f6a65cf14f462597b64ac058dbe1d0-system-media-system-caps-note" alt="System_CAPS_note" src="https://i-technet.sec.s-msft.com/dynimg/IC101471.jpeg" title="System_CAPS_note" xmlns=""></span><span class="alertTitle">Note </span></th>
                     </tr>
                     <tr>
                      <td><p>The program name must be unique within a package. After you create a program, you cannot modify its name.</p></td>
                     </tr>
                    </tbody>
                   </table>
                  </div></li>
                 <li><p><strong>Comment:</strong> Optionally, specify a comment for this device program with a maximum of 127 characters.</p></li>
                 <li><p><strong>Download folder:</strong> Specify the name of the folder on the Windows CE device in which the package source files will be stored. The default value is <strong>\Temp\</strong>.</p></li>
                 <li><p><strong>Command Line:</strong> Enter the command line to use to start this program, or click <strong>Browse</strong> to browse to the file location.</p></li>
                 <li><p><strong>Run command line in download folder</strong> – Select this option to run the program from the previously specified download folder.</p></li>
                 <li><p><strong>Run command line from this folder</strong> – Select this option to specify a different folder from which to run the program.</p></li>
                </ul>
               </div></li>
              <li class="step">
               <div class="section">
                <p>On the <strong>Requirements</strong> page of the Wizard, specify the following information:</p>
                <ul class="unordered">
                 <li><p><strong>Estimated disk space:</strong> Specify the amount of disk space required for the software. This will be displayed to users of mobile devices before they install the program.</p></li>
                 <li><p><strong>Download program:</strong> Specify information regarding when this program can be downloaded to mobile devices. You can specify <strong>As soon as possible</strong>, <strong>Only over a fast network</strong>, or <strong>Only when the device is docked</strong>.</p></li>
                 <li><p><strong>Additional requirements:</strong> Specify any additional requirements for this program. These will be displayed to users before they install the software. For example, you could notify users that they need to close all other applications before running the program.</p></li>
                </ul>
               </div></li>
              <li class="step">
               <div class="section">
                <p>Click <strong>Next</strong>.</p>
               </div></li>
             </ol>
            </div>
            <div class="section" id="BKMK_Summ" bookmarktitle="To complete the Create Package and Program Wizard">
             <h3 class="subHeading">To complete the Create Package and Program Wizard</h3>
             <ol class="steps">
              <li class="step">
               <div class="section">
                <p>On the <strong>Summary</strong> page of the Wizard, review the actions that will be taken, then complete the Wizard. </p>
               </div></li>
              <li class="step">
               <div class="section">
                <p>Optionally, verify that the new package and program is displayed in the <strong>Packages</strong> node of the <strong>Software Library</strong> workspace.</p>
               </div></li>
             </ol>
            </div>

         <a id="BKMK_Pacakage_Def"></a>
         <div>
          <h2 class="LW_CollapsibleArea_TitleDiv">
           <div>
            <a class="LW_CollapsibleArea_TitleAhref" title="Expand" role="heading"><span class="cl_CollapsibleArea_collapsing LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">How to Create a Package and Program from a Package Definition File</span></a>
            <div id="Anchor_1" class="LW_CollapsibleArea_Anchor_Div">
             <a href="/en-us/library/gg682112.aspx#Anchor_1" class="LW_CollapsibleArea_Anchor_Img" title="Right-click to copy and share the link for this section"></a>
            </div>
            <div class="LW_CollapsibleArea_HrDiv">
             <hr class="LW_CollapsibleArea_Hr">
            </div>
           </div></h2>
          <div class="sectionnone">
           <div class="section">
            <p>Use the following procedure to create a package and program from a package definition file. For more information about package definition files, see <a href="#BKMK_PDF">About the Package Definition File Format</a> in this topic.</p>
            <div>
             <h4 class="LW_CollapsibleArea_TitleDiv">
              <div>
               <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">To import a package and program from a definition file</span></a>
               <div class="LW_CollapsibleArea_HrDiv">
                <hr class="LW_CollapsibleArea_Hr">
               </div>
              </div></h4>
             <div class="sectionblock">
              <ol class="steps">
               <li class="step">
                <div class="section">
                 <p>In the Configuration Manager console, click <strong>Software Library</strong>.</p>
                </div></li>
               <li class="step">
                <div class="section">
                 <p>In the <strong>Software Library</strong> workspace, expand <strong>Application Management</strong>, and then click <strong>Packages</strong>.</p>
                </div></li>
               <li class="step">
                <div class="section">
                 <p>In the <strong>Home</strong> tab, in the <strong>Create</strong> group, click <strong>Create Package from Definition</strong>.</p>
                </div></li>
               <li class="step">
                <div class="section">
                 <p>On the <strong>Package Definition</strong> page of the <strong>Create Package from Definition Wizard</strong>, choose an existing package definition file, or click <strong>Browse</strong> to open a new package definition file. After you have specified a new package definition file, select it from the <strong>Package definition</strong> list, and then click <strong>Next</strong>.</p>
                </div></li>
               <li class="step">
                <div class="section">
                 <p>On the <strong>Source Files</strong> page of the Wizard, specify information about any required source files for the package and program, and then click <strong>Next</strong>.</p>
                </div></li>
               <li class="step">
                <div class="section">
                 <p>If the package requires source files, on the <strong>Source Folder</strong> page of the Wizard, specify the location from which the source files are to be obtained, and then click <strong>Next</strong>.</p>
                </div></li>
               <li class="step">
                <div class="section">
                 <p>On the <strong>Summary</strong> page of the Wizard, review the actions that will be taken and then complete the Wizard. The new package and program is displayed in the <strong>Packages</strong> node of the <strong>Software Library</strong> workspace.</p>
                </div></li>
              </ol>
             </div>
            </div>
           </div>
           <a id="BKMK_PDF"></a>
           <div>
            <h3 class="LW_CollapsibleArea_TitleDiv">
             <div>
              <a class="LW_CollapsibleArea_TitleAhref" title="Expand" role="heading"><span class="cl_CollapsibleArea_collapsing LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">About the Package Definition File Format</span></a>
              <div class="LW_CollapsibleArea_HrDiv">
               <hr class="LW_CollapsibleArea_Hr">
              </div>
             </div></h3>
            <div class="sectionnone">
             <div class="section">
              <p>Package definition files are scripts that you can use to help automate package and program creation with Configuration Manager. They provide all of the information that Configuration Manager needs in order to create a package and program, except for the location of package source files. Each package definition file is an ASCII or UTF-8 text file following the .ini file format and containing the following described sections:</p>
              <ul class="unordered">
               <li><p><a href="#BKMK_Address">[PDF]</a></p></li>
               <li><p><a href="#BKMK_Pack">[Package Definition]</a></p></li>
               <li><p><a href="#BKMK_Prog">[Program]</a></p></li>
              </ul>
             </div>
             <a id="BKMK_Address"></a>
             <div>
              <h4 class="LW_CollapsibleArea_TitleDiv">
               <div>
                <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">[PDF]</span></a>
                <div class="LW_CollapsibleArea_HrDiv">
                 <hr class="LW_CollapsibleArea_Hr">
                </div>
               </div></h4>
              <div class="sectionblock">
               <div class="section">
                <p>This section identifies the file as a package definition file. It contains the following information:</p>
                <ul class="unordered">
                 <li><p><span class="label">Version</span>: This specifies the version of the package definition file format that is used by the file. This corresponds to the version of System Management Server (SMS) or Configuration Manager for which it was written. This entry is required.</p></li>
                </ul>
               </div>
              </div>
             </div>
             <a id="BKMK_Pack"></a>
             <div>
              <h4 class="LW_CollapsibleArea_TitleDiv">
               <div>
                <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">[Package Definition]</span></a>
                <div class="LW_CollapsibleArea_HrDiv">
                 <hr class="LW_CollapsibleArea_Hr">
                </div>
               </div></h4>
              <div class="sectionblock">
               <div class="section">
                <p>This section of the package definition file specifies the properties of the package and program. It provides the following information:</p>
                <ul class="unordered">
                 <li><p><span class="label">Name</span>: The name of the package, up to&nbsp;50 characters. This entry is required.</p></li>
                 <li><p><span class="label">Version</span>: The version of the package, up to&nbsp;32 characters. This entry is optional.</p></li>
                 <li><p><span class="label">Icon</span>: Optionally, the file containing the icon to use for this package. If specified, this icon will replace the default package icon in the Configuration Manager console.</p></li>
                 <li><p><span class="label">Publisher</span>: The publisher of the package, up to&nbsp;32 characters. This entry is required.</p></li>
                 <li><p><span class="label">Language</span>: The language version of the package, up to&nbsp;32 characters. This entry is required.</p></li>
                 <li><p><span class="label">Comment</span>: An optional comment about the package, up to&nbsp;127 characters.</p></li>
                 <li><p><span class="label">ContainsNoFiles</span>: This entry indicates whether or not a source is associated with the package.</p></li>
                 <li><p><span class="label">Programs</span>: The programs defined for this package. Each program name corresponds to a <span class="label">[Program]</span> section in this package definition file. This entry is required. </p><p>Example:</p><p><span class="code">Programs=Typical, Custom, Uninstall</span></p></li>
                 <li><p><span class="label">MIFFileName</span>: The name of the Management Information Format (MIF) file that contains the package status, up to&nbsp;50 characters.</p></li>
                 <li><p><span class="label">MIFName</span>: The name of the package (for MIF matching), up to&nbsp;50 characters.</p></li>
                 <li><p><span class="label">MIFVersion</span>: The version number of the package (for MIF matching), up to&nbsp;32 characters.</p></li>
                 <li><p><span class="label">MIFPublisher</span>: The software publisher of the package (for MIF matching), up to&nbsp;32 characters.</p></li>
                </ul>
               </div>
              </div>
             </div>
             <a id="BKMK_Prog"></a>
             <div>
              <h4 class="LW_CollapsibleArea_TitleDiv">
               <div>
                <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">[Program]</span></a>
                <div class="LW_CollapsibleArea_HrDiv">
                 <hr class="LW_CollapsibleArea_Hr">
                </div>
               </div></h4>
              <div class="sectionblock">
               <div class="section">
                <p>For each program specified in the <span class="label">Programs</span> entry in the <span class="label">[Package Definition]</span> section, the package definition file must include a [Program] section that defines that program. Each Program section provides the following information:</p>
                <ul class="unordered">
                 <li><p><span class="label">Name</span>: The name of the program, up to&nbsp;50 characters. This entry must be unique within a package. This name is used when defining advertisements. On client computers, the name of the program is shown in <strong>Run Advertised Programs </strong>in Control Panel. This entry is required.</p></li>
                 <li><p><span class="label">Icon</span>: Optionally specifies the file containing the icon to use for this program. If specified, this icon will replace the default program icon in the Configuration Manager console and will be displayed on client computers when the program is advertised.</p></li>
                 <li><p><span class="label">Comment</span>: An optional comment about the program, up to&nbsp;127 characters. </p></li>
                 <li><p><span class="label">CommandLine</span>: Specifies the command line for the program, up to&nbsp;127 characters. The command is relative to the package source folder. This entry is required.</p></li>
                 <li><p><span class="label">StartIn</span>: Specifies the working folder for the program, up to&nbsp;127 characters. This entry can be an absolute path on the client computer or a path relative to the package source folder. This entry is required.</p></li>
                 <li><p><span class="label">Run</span>: Specifies the program mode in which the program will run. You can specify <span class="label">Minimized</span>, <span class="label">Maximized</span>, or <span class="label">Hidden</span>. If this entry is not included, the program will run in normal mode.</p></li>
                 <li><p><span class="label">AfterRunning</span>: Specifies any special action that occurs after the program is successfully completed. Options available are <span class="label">SMSRestart</span>, <span class="label">ProgramRestart</span>, or <span class="label">SMSLogoff</span>. If this entry is not included, the program will not run a special action.</p></li>
                 <li><p><span class="label">EstimatedDiskSpace</span>: Specifies the amount of disk space that the software program requires to be able run on the computer. This can be specified as <span class="label">Unknown</span> (the default setting) or as a whole number greater than or equal to zero. If a value is specified, the units for the value must also be specified.</p><p>Example:</p><p><span class="code">EstimatedDiskSpace=38MB</span></p></li>
                 <li><p><span class="label">EstimatedRunTime</span>: Specifies the estimated duration (in minutes) that the program is expected to run on the client computer. This can be specified as <span class="label">Unknown</span> (the default setting) or as a whole number greater than zero. </p><p>Example:</p><p><span class="code">EstimatedRunTime=25</span></p></li>
                 <li><p><span class="label">SupportedClients</span>: Specifies the processors and operating systems on which this program will run. The specified platforms must be separated by commas. If this entry is not included, supported platform checking will be disabled for this program.</p></li>
                 <li><p><span class="label">SupportedClientMinVersionX</span>, <span class="label">SupportedClientMaxVersionX</span>: Specifies the beginning-to-ending range for version numbers for the operating systems specified in the <span class="label">SupportedClients</span> entry. </p><p>Example:</p>

                  </div> </li>
                 <li><p><span class="label">AdditionalProgramRequirements</span>: Optionally provide any other information or requirements for client computers, up to&nbsp;127 characters.</p></li>
                 <li><p><span class="label">CanRunWhen</span>: Specifies the user status that the program requires to be able run on the client computer. Available values are <span class="label">UserLoggedOn</span>, <span class="label">NoUserLoggedOn</span>, or <span class="label">AnyUserStatus</span>. The default value is <span class="label">UserLoggedOn</span>.</p></li>
                 <li><p><span class="label">UserInputRequired</span>: Specifies whether the program requires interaction with the user. Available values are <span class="label">True</span> or <span class="label">False</span>. The default value is <span class="label">True</span>. This entry is set to <span class="label">False</span> if <span class="label">CanRunWhen</span> is not set to <span class="label">UserLoggedOn</span>.</p></li>
                 <li><p><span class="label">AdminRightsRequired</span>: Specifies whether the program requires administrative credentials on the computer to be able to run. Available values are <span class="label">True</span> or <span class="label">False</span>. The default value is <span class="label">False</span>. This entry is set to <span class="label">True</span> if <span class="label">CanRunWhen</span> is not set to <span class="label">UserLoggedOn</span>.</p></li>
                 <li><p><span class="label">UseInstallAccount</span>: Specifies whether the program uses the Client Software Installation Account when it runs on client computers. By default, this value is <span class="label">False</span>. This value is also <span class="label">False</span> if <span class="label">CanRunWhen</span> is set to <span class="label">UserLoggedOn</span>.</p></li>
                 <li><p><span class="label">DriveLetterConnection</span>: Specifies whether the program requires a drive letter connection to the package files that are located on the distribution point. You can specify <span class="label">True</span> or <span class="label">False</span>. The default value is <span class="label">False</span>, which allows the program to use a Universal Naming Convention (UNC) connection. When this value is set to <span class="label">True</span>, the next available drive letter will be used (starting with Z: and proceeding backward).</p></li>
                 <li><p><span class="label">SpecifyDrive</span>: Optionally, specifies a drive letter that the program requires to connect to the package files on the distribution point. This specification forces the use of the specified drive letter for client connections to distribution points. </p></li>
                 <li><p><span class="label">ReconnectDriveAtLogon</span>: Specifies whether the computer reconnects to the distribution point when the user logs on. Available values are <span class="label">True</span> or <span class="label">False</span>. The default value is <span class="label">False</span>.</p></li>
                 <li><p><span class="label">DependentProgram</span>: Specifies a program in this package that must run before the current program. This entry uses the format <span class="label">DependentProgram</span>=&lt;<span class="label">ProgramName&gt;</span>, where <span class="label">&lt;ProgramName&gt;</span> is the <strong>Name</strong> entry for that program in the package definition file. If there are no dependent programs, leave this entry empty.</p><p>Example:</p><p>DependentProgram=Admin<br>DependentProgram=</p></li>
                 <li><p><span class="label">Assignment</span>: Specifies how the program is assigned to users. This value can be: <span class="label">FirstUser,</span> only the first user who logs on runs the program; or <span class="label">EveryUser,</span> every user who logs on to the client runs the program. When <span class="label">CanRunWhen</span> is not set to <span class="label">UserLoggedOn</span>, this entry is set to <span class="label">FirstUser</span>.</p></li>
                 <li><p><span class="label">Disabled</span>: Specifies whether this program can be advertised to clients. Available values are <span class="label">True</span> or <span class="label">False</span>. The default value is <span class="label">False</span>.</p></li>
                </ul>
               </div>
              </div>

         <a id="BKMK_Import"></a>
         <div>
          <h2 class="LW_CollapsibleArea_TitleDiv">
           <div>
            <a class="LW_CollapsibleArea_TitleAhref" title="Expand" role="heading"><span class="cl_CollapsibleArea_collapsing LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">How to Import a Package and Program</span></a>
            <div id="Anchor_2" class="LW_CollapsibleArea_Anchor_Div">
             <a href="/en-us/library/gg682112.aspx#Anchor_2" class="LW_CollapsibleArea_Anchor_Img" title="Right-click to copy and share the link for this section"></a>
            </div>
            <div class="LW_CollapsibleArea_HrDiv">
             <hr class="LW_CollapsibleArea_Hr">
            </div>
           </div></h2>
          <div class="sectionnone">
           <div class="section">
            <p></p>
            <div>
             <h4 class="LW_CollapsibleArea_TitleDiv">
              <div>
               <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">To import a package and program</span></a>
               <div class="LW_CollapsibleArea_HrDiv">
                <hr class="LW_CollapsibleArea_Hr">
               </div>
              </div></h4>
             <div class="sectionblock">
              <ol class="steps">
               <li class="step">
                <div class="section">
                 <p>In the Configuration Manager console, click <strong>Software Library</strong>.</p>
                </div></li>
               <li class="step">
                <div class="section">
                 <p>In the <strong>Software Library</strong> workspace, expand <strong>Application Management</strong>, and then click <strong>Packages</strong>.</p>
                </div></li>
               <li class="step">
                <div class="section">
                 <p>In the <strong>Home</strong> tab, in the <strong>Create</strong> group, Click <strong>Import</strong>.</p>
                </div></li>
               <li class="step">
                <div class="section">
                 <p>On the <strong>General</strong> page of the <strong>Import Package Wizard</strong>, specify or browse to the compressed (.zip) file that contains the package and program to import, and then click <strong>Next</strong>.</p>
                </div></li>
               <li class="step">
                <div class="section">
                 <p>On the <strong>File Content</strong> page of the Wizard, review the items that will be imported, and then click <strong>Next</strong>. You can click <strong>View Failure</strong> to examine the details of items that cannot be imported. If the package you are trying to import already exists, you can choose to either ignore the duplicate package or overwrite the original package.</p>
                </div></li>
               <li class="step">
                <div class="section">
                 <p>On the <strong>Summary</strong> page of the Wizard, review the actions that will be taken and then complete the Wizard. The new package and program is displayed in the <strong>Packages</strong> node of the <strong>Software Library</strong> workspace.</p>
                </div></li>
              </ol>
             </div>
            </div>
