      <h2>Send SMS using AT commands</h2> 
      <p> Some advanced GSM modems like WaveCom and Multitech, support the SMS text mode. This mode allows you to send SMS messages using AT commands, without the need to encode the binairy PDU field of the SMS first. This is done by the GSM modem </p> 
      <h2>Check if your GSM phone or modem supports SMS text mode</h2> 
      <p> To check if your modem supports this text mode, you can try the following command: </p> 
      <b>AT+CMGF=1 &lt;ENTER&gt;</b> 
      <p> If the modem reponds with "OK" this mode is supported. Please note that using this mode it is onluy possible to send simple text messages. It is not possible to send multipart, Unicode, data and other types of messages. </p> 
      <h2>Setting up the modem</h2> 
      <p> If the modem contains a SIM card with is secured with a PIN code, we have to enter this pin code first: </p> 
      <b>AT+CPIN="0000" &lt;ENTER&gt;</b>&nbsp; (replace 0000 with your PIN code). 
      <p> Please not that in most cases you have only 3 attemps to set the correct PIN code. After setting the PIN code, wait some seconds before issueing the next command to give the modem some time to register with the GSM network. </p> 
      <p> In order to send a SMS, the modem has to be put in SMS text mode first using the following command: </p> 
      <b>AT+CMGF=1 &lt;ENTER&gt;</b> 
      <p> In text mode there are some additional parameters that can be set. Using the following command we can read the current values: </p> 
      <b>AT+CSMP? &lt;ENTER&gt;</b> 
      <p> The modem will reponse with a string like this: </p> 
      <b>+CSMP: 1,169,0,0OK</b> 
      <p> The first value is a combination of some option bits: </p> 
      <table> 
       <tbody>
        <tr> 
         <td width="100">bit 7</td> 
         <td width="100">RP</td> 
         <td>Reply path, not used in text mode</td> 
        </tr> 
        <tr> 
         <td>bit 6</td> 
         <td>UDHI</td> 
         <td>User Data Header Information</td> 
        </tr> 
        <tr> 
         <td>bit 5</td> 
         <td>SRR</td> 
         <td>Set this bit to request a delivery report</td> 
        </tr> 
        <tr> 
         <td>bit 3,4</td> 
         <td>VPF</td> 
         <td>Validity Period, set b4=1 if a VP value is present</td> 
        </tr> 
        <tr> 
         <td>bit 2</td> 
         <td>RD</td> 
         <td>Reject Duplicates, do not return a message ID when a message with the same destination and ID is still pending</td> 
        </tr> 
        <tr> 
         <td valign="top">bit 0,1</td> 
         <td valign="top">MTI</td> 
         <td valign="top">Message Type Indicatorb1=0 &amp; b0=0 -&gt; SMS-DELIVERb1=0 &amp; b0=1 -&gt; SMS-SUBMIT</td> 
        </tr> 
       </tbody>
      </table> 
      <p> Bit 0 of the message is always set when sending messages (SMS-SUBMIT). So the first value should be 1 or higher. The second parameter sets the Validity Period of the message. This value is encoded as follows: </p> 
      <table> 
       <tbody>
        <tr> 
         <td width="100">0 - 143</td> 
         <td>(VP + 1) x 5 minutes</td> 
        </tr> 
        <tr> 
         <td>144 - 167</td> 
         <td>12 Hours + ((VP-143) x 30 minutes)</td> 
        </tr> 
        <tr> 
         <td>168 - 196</td> 
         <td>(VP-166) x 1 day</td> 
        </tr> 
        <tr> 
         <td>197 - 255</td> 
         <td>(VP-192) x 1 week</td> 
        </tr> 
       </tbody>
      </table> 
      <p> The third parameter contains the PID (Protocol Identifier). This parameter is only used for advanced messaging. The fourth parameter contains the DCS (Data Coding Scheme). This parameter is used to select the characterset/messagetype. When setting the DCS parameter to '0' standard 7 bit text is send. When setting this parameter to '16' the message is sent as a flash message. </p> 
      <p> To send a message with a validity period of 1 day, the parameters have to be set like this: </p> 
      <p> Bit 0 and 4 of the first field has to be set, so the first value will become 1 + 16 = 17. </p> 
      <p> Send the following command to the modem to set this parameters: </p> 
      <b>AT+CSMP=17,167,0,16 &lt;ENTER&gt;</b> 
      <p> If the modem responds with "OK" ,the modem is ready to send (flash) text messages with a validity period of 1 day. </p> 
      <h2>Sending the message</h2> 
      <p> To send the SMS message, type the following command: </p> 
      <b>AT+CMGS="+31638740161" &lt;ENTER&gt;</b> 
      <p> Replace the above phone number with your own cell phone number. The modem will respond with: </p> 
      <b>&gt;</b> 
      <p> You can now type the message text and send the message using the &lt;CTRL&gt;-&lt;Z&gt; key combination: </p> 
      <b>Hello World !&nbsp;&lt;CTRL-Z&gt;</b> 
      <p> After some seconds the modem will respond with the message ID of the message, indicating that the message was sent correctly: </p> 
      <b>+CMGS: 62</b> 
      <p> The message will arrive on the mobile phone shortly. </p> 
      <h2>Sending an Unicode SMS message</h2> 
      <p> Some modems also have the capability to send Unicode or UCS2 messages without encoding a PDU. You can send Unicode messages by only converting the Unicode data to a HEX string and send this string to the modem. </p> 
      <p>To check whether your modem supports this mode, just type the following command:</p> 
      <b>AT+CSCS=?</b> 
      <p>This commands displays the codepages supported by the modem. The modem will respond like this:</p> 
      <b>+CSCS: ("GSM","PCCP437","CUSTOM","HEX")</b> 
      <p> If this string contains "HEX" or "UCS2", Unicode seems to be supported. To specify that you will use an HEX string to send the message, set the codepage to "HEX" or "UCS2" depending on the modem response. In our example we will set the modem to "HEX" : </p> 
      <b>AT+CSCS="HEX" &lt;ENTER&gt;</b> 
      <p> Next, we have to specify the correct DCS (Data Coding Scheme) for Unicode messages, which is 0x08. We can set this value by changing the fourth parameter of the AT+CSMP command to '8': </p> 
      <b>AT+CSMP=1,167,0,8 &lt;ENTER&gt;</b> 
      <p>The modem is now ready to send messages as Unicode. Now is the time to send the actual message:</p> 
      <b>AT+CMGS="+31638740161" &lt;ENTER&gt;</b> 
      <p>Replace the above phone number with your own cell phone number. The modem will respond with:</p> 
      <b>&gt;</b> 
      <p> The only thing you have to program by yourself, is a simple routine which converts the Unicode string to an hexidecimal string like this: </p> ????? 
      <p>Which is 'Hello' in arabic will be converted like this:</p> "06450631062D06280627" 
      <p>You can send this hexidecimal string to the modem:</p> 
      <b>06450631062D06280627&nbsp;&lt;CTRL-Z&gt;</b> 
      <p>After some seconds the modem will respond with the message ID of the message, indicating that the message was sent correctly:</p> 
      <b>+CMGS: 63</b> 
      <p>The message will arrive on the mobile phone shortly.</p>
