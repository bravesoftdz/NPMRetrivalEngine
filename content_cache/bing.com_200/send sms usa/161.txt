       <p>This guide demonstrates how to perform common programming tasks with the Twilio API service on Azure. The scenarios covered include making a phone call and sending a Short Message Service (SMS) message. For more information on Twilio and using voice and SMS in your applications, see the <a href="#NextSteps" data-linktype="self-bookmark">Next steps</a> section.</p> 
       <h2 id="WhatIs">What is Twilio?</h2> 
       <p>Twilio is powering the future of business communications, enabling developers to embed voice, VoIP, and messaging into applications. They virtualize all infrastructure needed in a cloud-based, global environment, exposing it through the Twilio communications API platform. Applications are simple to build and scalable. Enjoy flexibility with pay-as-you-go pricing, and benefit from cloud reliability.</p> 
       <p><strong>Twilio Voice</strong> allows your applications to make and receive phone calls. <strong>Twilio SMS</strong> enables your applications to send and receive SMS messages. <strong>Twilio Client</strong> allows you to make VoIP calls from any phone, tablet, or browser and supports WebRTC.</p> 
       <h2 id="Pricing">Twilio Pricing and Special Offers</h2> 
       <p>Azure customers receive a <a href="http://www.twilio.com/azure" data-linktype="external">special offer</a>: complimentary $10 of Twilio Credit when you upgrade your Twilio Account. This Twilio Credit can be applied to any Twilio usage ($10 credit equivalent to sending as many as 1,000 SMS messages or receiving up to 1000 inbound Voice minutes, depending on the location of your phone number and message or call destination). Redeem this Twilio credit and get started at <a href="http://ahoy.twilio.com/azure" data-linktype="external">ahoy.twilio.com/azure</a>.</p> 
       <p>Twilio is a pay-as-you-go service. There are no set-up fees, and you can close your account at any time. You can find more details at <a href="http://www.twilio.com/voice/pricing" data-linktype="external">Twilio Pricing</a>.</p> 
       <h2 id="Concepts">Concepts</h2> 
       <p>The Twilio API is a RESTful API that provides voice and SMS functionality for applications. Client libraries are available in multiple languages; for a list, see <a href="https://www.twilio.com/docs/libraries" data-linktype="external">Twilio API Libraries</a>.</p> 
       <p>Key aspects of the Twilio API are Twilio verbs and Twilio Markup Language (TwiML).</p> 
       <h3 id="Verbs">Twilio verbs</h3> 
       <p>The API makes use of Twilio verbs; for example, the <strong>&lt;Say&gt;</strong> verb instructs Twilio to audibly deliver a message on a call.</p> 
       <p>The following is a list of Twilio verbs. Learn about the other verbs and capabilities via <a href="http://www.twilio.com/docs/api/twiml" data-linktype="external">Twilio Markup Language documentation</a>.</p> 
       <ul> 
        <li><strong>&lt;Dial&gt;</strong>: Connects the caller to another phone.</li> 
        <li><strong>&lt;Gather&gt;</strong>: Collects numeric digits entered on the telephone keypad.</li> 
        <li><strong>&lt;Hangup&gt;</strong>: Ends a call.</li> 
        <li><strong>&lt;Play&gt;</strong>: Plays an audio file.</li> 
        <li><strong>&lt;Pause&gt;</strong>: Waits silently for a specified number of seconds.</li> 
        <li><strong>&lt;Record&gt;</strong>: Records the caller's voice and returns an URL of a file that contains the recording.</li> 
        <li><strong>&lt;Redirect&gt;</strong>: Transfers control of a call or SMS to the TwiML at a different URL.</li> 
        <li><strong>&lt;Reject&gt;</strong>: Rejects an incoming call to your Twilio number without billing you</li> 
        <li><strong>&lt;Say&gt;</strong>: Converts text to speech that is made on a call.</li> 
        <li><strong>&lt;Sms&gt;</strong>: Sends an SMS message.</li> 
       </ul> 
       <h3 id="TwiML">TwiML</h3> 
       <p>TwiML is a set of XML-based instructions based on the Twilio verbs that inform Twilio of how to process a call or SMS.</p> 
       <p>As an example, the following TwiML would convert the text <strong>Hello World</strong> to speech.</p> 
       <pre><code>&lt;?xml version="1.0" encoding="UTF-8" ?&gt;
&lt;Response&gt;
  &lt;Say&gt;Hello World&lt;/Say&gt;
&lt;/Response&gt;
</code></pre>
       <p>When your application calls the Twilio API, one of the API parameters is the URL that returns the TwiML response. For development purposes, you can use Twilio-provided URLs to provide the TwiML responses used by your applications. You could also host your own URLs to produce the TwiML responses, and another option is to use the <strong>TwiMLResponse</strong> object.</p> 
       <p>For more information about Twilio verbs, their attributes, and TwiML, see <a href="http://www.twilio.com/docs/api/twiml" data-linktype="external">TwiML</a>. For additional information about the Twilio API, see <a href="http://www.twilio.com/api" data-linktype="external">Twilio API</a>.</p> 
       <h2 id="CreateAccount">Create a Twilio Account</h2> 
       <p>When you're ready to get a Twilio account, sign up at <a href="https://www.twilio.com/try-twilio" data-linktype="external">Try Twilio</a>. You can start with a free account, and upgrade your account later.</p> 
       <p>When you sign up for a Twilio account, you'll receive an account ID and an authentication token. Both will be needed to make Twilio API calls. To prevent unauthorized access to your account, keep your authentication token secure. Your account ID and authentication token are viewable at the <a href="https://www.twilio.com/console" data-linktype="external">Twilio account page</a>, in the fields labeled <strong>ACCOUNT SID</strong> and <strong>AUTH TOKEN</strong>, respectively.</p> 
       <h2 id="create_app">Create an Azure Application</h2> 
       <p>An Azure application that hosts a Twilio enabled application is no different from any other Azure application. You add the Twilio .NET library and configure the role to use the Twilio .NET libraries. For information on creating an initial Azure project, see <a href="http://msdn.microsoft.com/library/windowsazure/ee405487.aspx" data-linktype="external">Creating an Azure project with Visual Studio</a>.</p> 
       <h2 id="configure_app">Configure Your Application to use Twilio Libraries</h2> 
       <p>Twilio provides a set of .NET helper libraries that wrap various aspects of Twilio to provide simple and easy ways to interact with the Twilio REST API and Twilio Client to generate TwiML responses.</p> 
       <p>Twilio provides five libraries for .NET developers:</p> 
       <table> 
        <thead> 
         <tr> 
          <th>Library</th> 
          <th>Description</th> 
         </tr> 
        </thead> 
        <tbody> 
         <tr> 
          <td>Twilio.API</td> 
          <td>The core Twilio library that wraps the Twilio REST API in a friendly .NET library. This library is available for .NET, Silverlight, and Windows Phone 7.</td> 
         </tr> 
         <tr> 
          <td>Twilio.TwiML</td> 
          <td>Provides a .NET friendly way to generate TwiML markup.</td> 
         </tr> 
         <tr> 
          <td>Twilio.MVC</td> 
          <td>For developers using ASP.NET MVC, this library includes a TwilioController, TwiML ActionResult, and request validation attribute.</td> 
         </tr> 
         <tr> 
          <td>Twilio.WebMatrix</td> 
          <td>For developers using Microsoft's free WebMatrix development tool, this library contains Razor syntax helpers for various Twilio actions.</td> 
         </tr> 
         <tr> 
          <td>Twilio.Client.Capability</td> 
          <td>Contains the Capability token generator for use with the Twilio Client JavaScript SDK.</td> 
         </tr> 
        </tbody> 
       </table> 
       <p>Note that all libraries require .NET 3.5, Silverlight 4, or Windows Phone 7 or later.</p> 
       <p>The samples provided in this guide use the Twilio.API library.</p> 
       <p>The libraries can be <a href="http://www.twilio.com/docs/csharp/install" data-linktype="external">installed using the NuGet package manager extension</a> available for Visual Studio 2010 up to 2015. The source code is hosted on <a href="https://github.com/twilio/twilio-csharp" data-linktype="external">GitHub</a>, which includes a Wiki that contains complete documentation for using the libraries.</p> 
       <p>By default, Microsoft Visual Studio 2010 installs version 1.2 of NuGet. Installing the Twilio libraries requires version 1.6 of NuGet or higher. For information on installing or updating NuGet, see <a href="http://nuget.org/" data-linktype="external">http://nuget.org/</a>.</p> 
       <div class="NOTE">
        <h5>Note</h5>
        <p>To install the latest version of NuGet, you must first uninstall the loaded version using the Visual Studio Extension Manager. To do so, you must run Visual Studio as administrator. Otherwise, the Uninstall button is disabled.</p> 
       </div> 
       <h3 id="use_nuget">To add the Twilio libraries to your Visual Studio project:</h3> 
       <ol> 
        <li>Open your solution in Visual Studio.</li> 
        <li>Right-click <strong>References</strong>.</li> 
        <li>Click <strong>Manage NuGet Packages...</strong></li> 
        <li>Click <strong>Online</strong>.</li> 
        <li>In the search online box, type <em>twilio</em>.</li> 
        <li>Click <strong>Install</strong> on the Twilio package.</li> 
       </ol> 
       <h2 id="howto_make_call">How to: Make an outgoing call</h2> 
       <p>The following shows how to make an outgoing call using the <strong>CallResource</strong> class. This code also uses a Twilio-provided site to return the Twilio Markup Language (TwiML) response. Substitute your values for the <strong>to</strong> and <strong>from</strong> phone numbers, and ensure that you verify the <strong>from</strong> phone number for your Twilio account before running the code.</p> 
       <pre><code>// Use your account SID and authentication token instead
// of the placeholders shown here.
const string accountSID = "your_twilio_account";
const string authToken = "your_twilio_authentication_token";

// Initialize the TwilioClient.
TwilioClient.Init(accountSID, authToken);

// Use the Twilio-provided site for the TwiML response.
var url = "http://twimlets.com/message";
url = $"{url}?Message%5B0%5D=Hello%20World";

// Set the call From, To, and URL values to use for the call.
// This sample uses the sandbox number provided by
// Twilio to make the call.
var call = CallResource.Create(
    to: new PhoneNumber("+NNNNNNNNNN"),
    from: new PhoneNumber("NNNNNNNNNN"),
    url: new Uri(url));
    }
</code></pre>
       <p>For more information about the parameters passed in to the <strong>CallResource.Create</strong> method, see <a href="http://www.twilio.com/docs/api/rest/making-calls" data-linktype="external">http://www.twilio.com/docs/api/rest/making-calls</a>.</p> 
       <p>As mentioned, this code uses a Twilio-provided site to return the TwiML response. You could instead use your own site to provide the TwiML response. For more information, see <a href="#howto_provide_twiml_responses" data-linktype="self-bookmark">How to: Provide TwiML responses from your own website</a>.</p> 
       <h2 id="howto_send_sms">How to: Send an SMS message</h2> 
       <p>The following screenshot shows how to send an SMS message using the <strong>MessageResource</strong> class. The <strong>from</strong> number is provided by Twilio for trial accounts to send SMS messages. The <strong>to</strong> number must be verified for your Twilio account before you run the code.</p> 
       <pre><code>// Use your account SID and authentication token instead
// of the placeholders shown here.
const string accountSID = "your_twilio_account";
const string authToken = "your_twilio_authentication_token";

// Initialize the TwilioClient.
TwilioClient.Init(accountSID, authToken);

try
{
    // Send an SMS message.
    var message = MessageResource.Create(
        to: new PhoneNumber("+12069419717"),
        from: new PhoneNumber("+14155992671"),
        body: "This is my SMS message.");
}
catch (TwilioException ex)
{
    // An exception occurred making the REST call
    Console.WriteLine(ex.Message);
}
</code></pre>
       <h2 id="howto_provide_twiml_responses">How to: Provide TwiML Responses from your own website</h2> 
       <p>When your application initiates a call to the Twilio API - for example, via the <strong>CallResource.Create</strong> method - Twilio sends your request to an URL that is expected to return a TwiML response. The example in <a href="#howto_make_call" data-linktype="self-bookmark">How to: Make an outgoing call</a> uses the Twilio-provided URL <a href="http://twimlets.com/message" data-linktype="external">http://twimlets.com/message</a> to return the response.</p> 
       <div class="NOTE">
        <h5>Note</h5>
        <p>While TwiML is designed for use by web services, you can view the TwiML in your browser. For example, click <a href="http://twimlets.com/message" data-linktype="external">http://twimlets.com/message</a> to see an empty &lt;Response&gt; element; as another example, click <a href="http://twimlets.com/message?Message%5B0%5D=Hello%20World" data-linktype="external">http://twimlets.com/message?Message%5B0%5D=Hello%20World</a> to see a &lt;Response&gt; element that contains a &lt;Say&gt; element.</p> 
       </div> 
       <p>Instead of relying on the Twilio-provided URL, you can create your own URL site that returns HTTP responses. You can create the site in any language that returns HTTP responses. This topic assumes you'll be hosting the URL from an ASP.NET generic handler.</p> 
       <p>The following ASP.NET Handler crafts a TwiML response that says <strong>Hello World</strong> on the call.</p> 
       <pre><code>using System.Text;
using System.Web;

namespace WebRole1
{
    /// &lt;summary&gt;
    /// Summary description for Handler1
    /// &lt;/summary&gt;
    public class Handler1 : IHttpHandler
    {
        public void ProcessRequest(HttpContext context)
        {
            const string twiMLResponse =
                "&lt;Response&gt;&lt;Say&gt;Hello World.&lt;/Say&gt;&lt;/Response&gt;";

            context.Response.Clear();
            context.Response.ContentType = "text/xml";
            context.Response.ContentEncoding = Encoding.UTF8;
            context.Response.Write(twiMLResponse);
            context.Response.End();
        }

        public bool IsReusable
        {
            get
            {
                return false;
            }
        }
    }
}
</code></pre>
       <p>As you can see from the example above, the TwiML response is simply an XML document. The Twilio.TwiML library contains classes that will generate TwiML for you. The example below produces the equivalent response as shown above, but uses the <strong>VoiceResponse</strong> class.</p> 
       <pre><code>using System.Web;
using Twilio.TwiML;

namespace WebRole1
{
    /// &lt;summary&gt;
    /// Summary description for Handler1
    /// &lt;/summary&gt;
    public class Handler1 : IHttpHandler
    {

        public void ProcessRequest(HttpContext context)
        {
            var twiml = new VoiceResponse();
            twiml.Say("Hello World.");

            context.Response.Clear();
            context.Response.ContentType = "text/xml";
            context.Response.Write(twiml.ToString());
            context.Response.End();
        }

        public bool IsReusable
        {
            get
            {
                return false;
            }
        }
    }
}
</code></pre>
       <p>For more information about TwiML, see <a href="https://www.twilio.com/docs/api/twiml" data-linktype="external">https://www.twilio.com/docs/api/twiml</a>.</p> 
       <p>Once you have set up a way to provide TwiML responses, you can pass that URL to the <strong>CallResource.Create</strong> method. For example, if you have a web application named MyTwiML deployed to an Azure cloud service, and the name of your ASP.NET Handler is mytwiml.ashx, the URL can be passed to <strong>CallResource.Create</strong> as shown in the following code sample:</p> 
       <pre><code>// This sample uses the sandbox number provided by Twilio to make the call.
// Place the call.
var call = CallResource.Create(
    to: new PhoneNumber("+NNNNNNNNNN"),
    from: new PhoneNumber("NNNNNNNNNN"),
    url: new Uri("http://&lt;your_hosted_service&gt;.cloudapp.net/MyTwiML/mytwiml.ashx"));
    }
</code></pre>
       <p>For additional information about using Twilio on Azure with ASP.NET, see <a href="partner-twilio-cloud-services-dotnet-phone-call-web-role" data-linktype="relative-path">How to make a phone call using Twilio in a web role on Azure</a>.</p>
