          <h3> <span class="mw-headline">Pre-requisites</span></h3> 
          <hr> 
          <ul>
           <li> <a href="http://java.sun.com/products/javamail/index.jsp" class="external text" title="http://java.sun.com/products/javamail/index.jsp" rel="nofollow">JavaMail</a> </li>
           <li> <a href="http://java.sun.com/javase/technologies/desktop/javabeans/jaf/downloads/index.html" class="external text" title="http://java.sun.com/javase/technologies/desktop/javabeans/jaf/downloads/index.html" rel="nofollow">JavaBeans Activation Framework</a> </li>

          <p><br> </p> 
          <a name="Why_so_many_downloads.3F" id="Why_so_many_downloads.3F"></a>
          <h3> <span class="mw-headline">Why so many downloads?</span></h3> 
          <hr> 
          <p><b>JavaMail</b> consists of various jar files such as <b>mail.jar, mailapi.jar, smtp.jar, pop3.jar</b> whereas <b>JAF</b> mainly consists of <b>activation.jar</b> file. You need to download them and include these jar files in your <i>CLASSPATH</i> so that your <b>JVM</b>(Java Virtual Machine) can find the <b>JAF</b> classes. </p>
          <p><br> </p> 
          <a name="Getting_Started" id="Getting_Started"></a>
          <h3> <span class="mw-headline">Getting Started</span></h3> 
          <hr> 
          <ul>
           <li> Open the NetBeans 6.1 IDE. </li>
          </ul> 
          <p><a href="/File:Nbstartup_SendMailUsingJavaFromNetBeans.jpg" class="image" title="File:nbstartup_SendMailUsingJavaFromNetBeans.jpg"><img alt="File:nbstartup_SendMailUsingJavaFromNetBeans.jpg" src="/wiki/images/3/39/Nbstartup_SendMailUsingJavaFromNetBeans.jpg" width="475" height="300" border="0"></a> </p> 
          <ul>
           <li> Pretty fast than previous releases. Took about only 14.35 seconds on my machine. </li>
          </ul> 
          <p><br> </p> 
          <a name="Creating_Project" id="Creating_Project"></a>
          <h3> <span class="mw-headline">Creating Project</span></h3> 
          <hr> 
          <ul>
           <li> Go to <b>File &gt; New Project</b> </li>
           <li> Under <i>Categories</i>, choose <b>Web</b> and then select to create a simple <b>Web Application</b>. </li>
          </ul> 
          <p><a href="/File:Blog1_SendMailUsingJavaFromNetBeans.jpg" class="image" title="File:blog1_SendMailUsingJavaFromNetBeans.jpg"><img alt="File:blog1_SendMailUsingJavaFromNetBeans.jpg" src="/wiki/images/8/80/Blog1_SendMailUsingJavaFromNetBeans.jpg" width="740" height="508" border="0"></a> </p> 
          <ul>
           <li> Click <b>Next</b> and <b>Type</b> in <i>Project Name</i> as <b>JavaMail</b>. Be sure to select <b>Glassfish</b> as your server. </li>
          </ul> 
          <p><a href="/File:Blog2_SendMailUsingJavaFromNetBeans.jpg" class="image" title="File:blog2_SendMailUsingJavaFromNetBeans.jpg"><img alt="File:blog2_SendMailUsingJavaFromNetBeans.jpg" src="/wiki/images/f/f9/Blog2_SendMailUsingJavaFromNetBeans.jpg" width="739" height="507" border="0"></a> </p> 
          <ul>
           <li> Click <b>Finish</b> and allow NetBeans to create files for your project. </li>
          </ul>

          </ul>

          <ul>
           <li> Save it and it looks like- </li>
          </ul> 
          <p><a href="/File:Blog3_SendMailUsingJavaFromNetBeans.jpg" class="image" title="File:blog3_SendMailUsingJavaFromNetBeans.jpg"><img alt="File:blog3_SendMailUsingJavaFromNetBeans.jpg" src="/wiki/images/e/e5/Blog3_SendMailUsingJavaFromNetBeans.jpg" width="742" height="458" border="0"></a> </p>

           <li> Now, we create a <b>bean</b> in which we store all the mail related information&nbsp;: sender's <b>email id</b> and <b>password</b> and receiver's email id, <b>subject</b> and <b>message body</b> of the email and the name of the <b>smtp server</b> that we use to send email. </li>

           <li> Add the following <pre>instance variables</pre> to the bean- </li>
          </ul> 
          <pre>
String to;
String from;
String message;
String subject;
String smtpServ;
</pre> 
          <ul>
           <li> <b>Generate</b> the <b>getter, setter</b> methods for them by <i>right-clicking</i> in the editor and select <b>Refactor &gt; Encapsulate Fields</b>. Then <b>check</b> all the setter and getter methods and <i>click</i> <b>Refactor</b> so that the IDE generates code for them. </li>

          <p><a href="/File:Blog-refactor_SendMailUsingJavaFromNetBeans.jpg" class="image" title="File:blog-refactor_SendMailUsingJavaFromNetBeans.jpg"><img alt="File:blog-refactor_SendMailUsingJavaFromNetBeans.jpg" src="/wiki/images/7/76/Blog-refactor_SendMailUsingJavaFromNetBeans.jpg" width="531" height="409" border="0"></a> </p> 
          <a name="Using_JavaMail_API" id="Using_JavaMail_API"></a>
          <h4> <span class="mw-headline">Using JavaMail API</span></h4> 
          <ul>
           <li> Now, we make use of the <b>JavaMail API</b> and <i>send mail</i> using the following code. But before doing that, import the required packages. </li>
          </ul> 
          <pre>
import javax.mail.*;
import javax.mail.internet.*;
import java.util.*;
</pre> 
          <ul>
           <li> Now, as you has already noticed, NetBeans is showing errors showing that it doesn't know of <i>javax.mail</i> package even if you have correctly downloaded and installed the <i>JAF</i> and <i>JavaMail API</i>. To make NetBeans aware of the existence of these packages, perform the following steps: </li>
           <li> <i>Right click</i> the project node (JavaMail), <i>click</i> <b>Properties</b> to change properties of the project. </li>
           <li> Now go to <b>Libraries</b> Tab (since we need to add jar files to our project). </li>
          </ul> 
          <p><a href="/File:Blog5_SendMailUsingJavaFromNetBeans.jpg" class="image" title="File:blog5_SendMailUsingJavaFromNetBeans.jpg"><img alt="File:blog5_SendMailUsingJavaFromNetBeans.jpg" src="/wiki/images/e/ef/Blog5_SendMailUsingJavaFromNetBeans.jpg" width="768" height="639" border="0"></a> </p> 
          <ul>
           <li> Click on Add JAR/Folder Button. A window opens up. </li>
           <li> Browse to the location where you have installed/unzipped your JavaMail API and then include following jar's; </li>
          </ul> 
          <dl>
           <dd>
            <ul>
             <li> activation.jar </li>
             <li> pop3.jar </li>
             <li> mail.jar </li>
             <li> mailapi.jar </li>
             <li> smtp.jar </li>
            </ul> 
           </dd>
          </dl> 
          <ul>
           <li> If you do not perform this step, then NetBeans will not recognize JAF and JavaMail classes and hence will not compile the program given below which makes use of classes such as Message, Session, Transport etc. </li>
           <li> Compile your program to check whether you have been able to successfully include these jar files or not. </li>
          </ul> 
          <a name="Implementing_sendMail.28.29" id="Implementing_sendMail.28.29"></a>
          <h4> <span class="mw-headline">Implementing sendMail()</span></h4> 
          <ul>
           <li> Now, the main function <pre>sendMail( )</pre>, which is actually responsible for sending mail: </li>
          </ul> 
          <pre>
public int sendMail(){
        try
        {
            Properties props = System.getProperties();
              // -- Attaching to default Session, or we could start a new one --
              props.put("mail.transport.protocol", "smtp" );
              props.put("mail.smtp.starttls.enable","true" );
              props.put("mail.smtp.host",smtpServ);
              props.put("mail.smtp.auth", "true" );
              Authenticator auth = new SMTPAuthenticator();
              Session session = Session.getInstance(props, auth);
              // -- Create a new message --
              Message msg = new MimeMessage(session);
              // -- Set the FROM and TO fields --
              msg.setFrom(new InternetAddress(from));
              msg.setRecipients(Message.RecipientType.TO, InternetAddress.parse(to, false));
              msg.setSubject(subject);
              msg.setText(message);
              // -- Set some other header information --
              msg.setHeader("MyMail", "Mr. XYZ" );
              msg.setSentDate(new Date());
              // -- Send the message --
              Transport.send(msg);
              System.out.println("Message sent to"+to+" OK." );
              return 0;
        }
        catch (Exception ex)
        {
          ex.printStackTrace();
          System.out.println("Exception "+ex);
          return -1;
        }
  }

// Also include an inner class that is used for authentication purposes

private class SMTPAuthenticator extends javax.mail.Authenticator {
        @Override
        public PasswordAuthentication getPasswordAuthentication() {
            String username =  "Java.Mail.CA@gmail.com";           // specify your email id here (sender's email id)
            String password = "javamail";                                      // specify your password here
            return new PasswordAuthentication(username, password);
        }
  }
</pre> 
          <ul>
           <li><b>Save</b> the file and compile it. </li>
           <li>Now add another JSP, name it <b>sendMail.jsp</b> </li>
           <li>Add the following code to it that instantiates <pre>JavaBean Object</pre> and set properties of <tt>{JavaBean</tt>} using the parameters passed to it using <b>index.jsp</b> </li>
          </ul> 
          <pre>
&lt;jsp:useBean id="mail" scope="session" class="jMail.Mail" /&gt;
&lt;jsp:setProperty name="mail" property="to" param="to" /&gt;
&lt;jsp:setProperty name="mail" property="from" value="Java.Mail.CA@gmail.com" /&gt;
&lt;jsp:setProperty name="mail" property="smtpServ" value="smtp.gmail.com" /&gt;
&lt;jsp:setProperty name="mail" property="subject" param="subject" /&gt;
&lt;jsp:setProperty name="mail" property="message" param="message" /&gt;
</pre> 
          <ul>
           <li> The main point to note here is the value of <pre>smtpServ</pre> that we have passed. Here, we are using a gmail id, so its smtp server name should be used, which is "smtp.gmail.com". </li>
           <li> If you want to use your own email id, then you have to make few changes as specified above and also change the value of <pre>smtpServ</pre> property to your smtp server name. </li>
          </ul> 
          <a name="Invoking_sendMail.28.29" id="Invoking_sendMail.28.29"></a>
          <h4> <span class="mw-headline">Invoking sendMail()</span></h4> 
          <ul>
           <li> Add the following code to <b>sendMail.jsp</b> that calls the <pre>sendMail()</pre> method of the bean. Yes, a bean can have other methods except getter and setter methods. Its perfectly legal.&nbsp;:-) </li>
          </ul> 
          <pre>
 &lt;%
String to = mail.getTo();
int result;
result = mail.sendMail();
if(result == 0){
    out.println(" Mail Successfully Sent to "+to);
}
else{
    out.println(" Mail NOT Sent to "+to);
}  
%&gt;
  </pre> 
          <ul>
           <li> <b>Save </b>your project and <b>Run</b> it to see that it actually works </li>
           <li> The whole project is available here for <a href="http://blogs.sun.com/agraj/resource/JavaMail/JavaMail.rar" class="external text" title="http://blogs.sun.com/agraj/resource/JavaMail/JavaMail.rar" rel="nofollow">download</a>. </li>
          </ul> 
          <a name="Notes" id="Notes"></a>
          <h3> <span class="mw-headline">Notes</span></h3> 
          <hr> 
          <ul>
           <li> There is much to JavaMail than what is mentioned here. For more read on <a href="https://glassfish.dev.java.net/javaee5/docs/DG/beaow.html" class="external text" title="https://glassfish.dev.java.net/javaee5/docs/DG/beaow.html" rel="nofollow">JavaMail &amp; Glassfish</a> </li>
           <li> Read the following links to understand what's actually happening in the code given above <a href="http://www.javaworld.com/jw-10-2001/jw-1026-javamail.html" class="external text" title="http://www.javaworld.com/jw-10-2001/jw-1026-javamail.html" rel="nofollow">JavaMail Quick Start jGuru&nbsp;: Fundamentals of JavaMail API</a> </li>
           <li> Related &amp; Advanced- <a href="http://blogs.sun.com/geertjan/entry/javamail_client_on_the_netbeans" class="external text" title="http://blogs.sun.com/geertjan/entry/javamail_client_on_the_netbeans" rel="nofollow">Creating JavaMail Client On NB</a> </li>
           <li> Please <b>do not modify </b>the <b>password</b> for the account <a href="mailto:Java.Mail.CA@gmail.com" class="external text" title="mailto:Java.Mail.CA@gmail.com" rel="nofollow">Java.Mail.CA@gmail.com</a> </li>
           <li> I created it only for demo purposes and have given out the account details as on all tutorials/demos about <b>JavaMail</b> that i have seen online leave these fields empty and also leave smtp server name fields empty, which only adds to confusion for novices. </li>

          </ul>
