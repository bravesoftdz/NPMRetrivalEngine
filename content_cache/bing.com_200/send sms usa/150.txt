   <a name="intro"> 
    <div class="Header">
     Introduction
    </div> </a> 
   <p> This API provides a way for Java applications to connect and use the Yahoo Instant Messenger protocol, version 10 (YMSG-10) in a reasonably coder friendly and abstract way. Although methods the API exposes are clearly matched to Yahoo IM features, the API attempts to hide as much of the nitty-gritty as possible, taking responsibility upon itself as far as it possibly can. </p> 
   <p> Written using J2SE 1.3/1.4 - but will probably work under JDK/JRE 1.1 </p> 
   <p> This is not an application, it is an API which may be employed by an application to create an instant messenger using the Yahoo chat/pager system. As such it is targeted at experienced programmers. </p> 
   <p> The project has matured quite a bit since it was first released, with a companion support package now providing advanced features like Swing models, spam detection, a chat emote manager, and customisable styled text decoding. I've kept this project described as being in an 'alpha' stage - even though the code has been downloaded by countless Java programmers and has held up quite well - as there is still a lot of work to be done catching up with all the features Yahoo have added over the years. </p> 
   <!--<p><font color="Red">
The latest patch which updates the jYMSG for protocol changes since the
release of v0.6 is <a href="./patch_0_63.zip">v0.63</a>.
</font></p>--> 
   <p> For more information on jYMSG, <a href="faq.html">consult the FAQ</a>. Beginners note: Kulvir Singh Bhogal has written a rather nice <a href="http://www.devx.com/Java/Article/22546/0/page/1">tutorial</a> to jYMSG, explaining how to get started. </p> 
   <p> This project is released under the GPL. </p>

   <a name="supported"> 
    <div class="Header">
     Features
    </div> </a> 
   <p> </p>
   <div class="List"> 
    <div class="ListSubHeader">
     Connection
    </div> 
    <img src="./bubble.gif" align="top">
    <b>Login/Logout</b> - wouldn't be much good without it.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Authentification</b> - protocol version 9, 10 and 11 authentification.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Fallback ports</b> - optionally attempt to connect via different routes.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Login timeouts</b> - configurable login and chat login timeouts.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>HTML/SOCKS proxy</b> - connect via SOCKS and HTTP proxies.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>HTTP authentication</b> - use proxy username/passwords with HTTP proxy.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Configurable</b> - host/port names accessible via properties.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Foreign chat</b> - use non-English chatroom hierarchies.
    <br> 
    <br> 
    <div class="ListSubHeader">
     Messaging
    </div> 
    <img src="./bubble.gif" align="top">
    <b>Instant messages</b> - send messages to other Yahoo users.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Conferences</b> - create, join, and use multi-user private chat.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Chatrooms</b> - find and connect to chatroom, including adult rooms.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>File transfer</b> - with easy-to-use file save methods.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Identities</b> - send messages using aliases.
    <br> 
    <br> 
    <div class="ListSubHeader">
     Data and Status
    </div> 
    <img src="./bubble.gif" align="top">
    <b>Status</b> - change status, or set custom status messages.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Groups and Friends list</b> - with events when they change.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Adding and removing friends</b> - including automatic group handling.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Identities</b> - all available aliases for current user.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Ignore</b> - for all those users who bore you.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>New mail update</b> - be informed of new Yahoo mail.
    <br> 
    <br> 
    <div class="ListSubHeader">
     Mechanics
    </div> 
    <img src="./bubble.gif" align="top">
    <b>Typing notification</b> - automatic, if you provide an AWT event source.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Connection pinging</b> - automatic keep alive pinging on open connections.
    <br> 
    <br> 
    <div class="ListSubHeader">
     Support Package
    </div> Separate support package - with the following features...
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Decoding messages</b> - to plain text, HTML or Swing components.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Encoding messages</b> - build styled text messages and output to Yahoo format.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Colour fade and alt text</b> - fancy text colouring supported.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Emoticons (smileys)</b> - decoded for Swing Document based text components.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Anti-Spam</b> - support class to help block annoying chat behaviour.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Overriding styles</b> - restrict message font sizes, colours, etc.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Swing models</b> - automatically updates user list/tree models.
    <br> 
    <img src="./bubble.gif" align="top">
    <b>Emote manager</b> - loads and manages chat Emotes from config files.
    <br>

   <a name="credits"> 
    <div class="Header">
     Credits and Thanks
    </div> </a> 
   <p> This project could not have existed without pioneering work done by other coders elsewhere. Although jYMSG is not a port of any other project, I did rely on other Open Source implementations to help fine-tune this Java version, and answer the recuring question <i>"what the hell does this tag do?"</i> </p> 
   <p> In particular I am indebted to the libyahoo2 project, who's documentation and source code was most useful in getting me started, as well as decyphering the more obscure parts of the protocol. I'd like to think by adding to the pool of documentation myself (as well as publishing details on the workings of YMSG's HTTP proxy scheme) I am in some small way repaying that debt. </p> 
   <p> A number of other Java coders, too many to list, also contributed comments and suggestions - many of which made it into the package. Special mentions go to J. Morris, who submitted some example code for the support library; Phinaes Gage, who submitted a working SHA-1 implementation after the Java API version could not be used; and Kulvir Singh Bhogal, who wrote a beginner's tutorial for jYMSG. </p> 
   <p> Naturally thanks must also go to SourceForge, who host this project for free, along with countless other GPL and Open Source efforts. </p>
   <p> jYMSG was created entirely <b>without</b> reverse engineering of any Yahoo! software code. </p>
