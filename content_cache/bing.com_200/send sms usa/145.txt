  <a name="at"></a> 
  <h2>Using AT commands to send SMS messages</h2> 
  <p>Using AT commands you're not restricted to a specific phone model, as long as the phone supports the commands you are using.</p> 
  <p>This program uses the text mode for sending the message, which is good for starting, but it may not be supported by every phone. If you want to use the PDU mode instead of the text mode - which is supported by all mobiles but is more complicated - see the <a href="#pdu">PDU encoding sample</a> for creating the required data.</p> 
  <p>Sample source code is available for VB6.</p> 
  <p>Details about the VB6 sample:</p> 
  <ul> 
   <li>Shows how to use the MSComm ActiveX control </li>
   <li>Shows how to use the SMS AT commands </li>
   <li>Includes a trace output to see what's going on </li>
   <li>Includes error handling for common errors </li>
   <li>Is not optimized for performance </li>
  </ul>

     <td colspan="2" class="dlbottom">&nbsp;</td>

  <p>Note: This program does only the coding, it does not actually send the message. For testing you can use a terminal program like HyperTerminal (included in Windows) to transfer the encoded message, as given in the following sample: </p>
  <p> </p>
  <table border="0"> 
   <tbody>
    <tr>
     <td bgcolor="#FFEEDD"> <pre>
AT+CMGF=0&lt;CR&gt;
AT+CMGS=&lt;actual PDU length&gt;&lt;CR&gt;
&lt;encoded message&gt;&lt;EOF&gt;</pre> </td>
    </tr>
   </tbody>
  </table> 
  <p> &lt;CR&gt; = ASCII 13 = ENTER<br> &lt;EOF&gt; = ASCII 26 = CTRL+Z<br> </p> 
  <p>You can also modify the <a href="#at">AT command sample</a> to use the PDU mode.</p>

  <h2>Using AT commands to read SMS messages</h2>

  <p>This program uses the text mode for reading the messages, which is good for learning how it works, but it may not be supported by every phone. You may have to change character sets depending on the characters used in the messages to get the original text back.</p>
