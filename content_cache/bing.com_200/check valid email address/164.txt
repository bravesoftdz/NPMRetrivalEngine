       <h1 class="topictitle" id="verify-email-addresses">Verifying Email Addresses in Amazon SES </h1> 
       <p>Amazon SES requires that you verify your email address or domain, to confirm that you own it and to prevent others from using it. This section discusses verifying individual email addresses. For information about domain verification, see <a href="verify-domains.html">Verifying Domains in Amazon SES</a>. </p> 
       <h2 id="verify-email-addresses-overview">Overview</h2> 
       <p>With the exception of addresses containing labels (see below), you must verify each email address (or the domain of the email address) that you will use as a "From", "Source", "Sender", or "Return-Path" address for your messages. Until your account is out of the Amazon SES sandbox, you must also verify the email address of every recipient except for the recipients provided by the Amazon SES mailbox simulator. For more information about the mailbox simulator, see <a href="mailbox-simulator.html">Testing Amazon SES Email Sending</a>. For more information about moving out of the sandbox, see <a href="request-production-access.html">Moving Out of the Amazon SES Sandbox</a>. </p> 
       <p>Important notes about email address verification are as follows:</p> 
       <div class="itemizedlist"> 
        <ul class="itemizedlist" type="disc"> 
         <li class="listitem"> <p>The entire email address is case-sensitive. For example, if you verify <em>sender@example.com</em>, you cannot send emails from <em>sender@EXAMPLE.com</em> unless you verify <em>sender@EXAMPLE.com</em> also. (Domain verification, however, is case-insensitive. For more information, see <a href="verify-domains.html">Verifying Domains in Amazon SES</a>.) </p> </li> 
         <li class="listitem"> <p>If you individually verify an email address and you also verify the domain of that address, the verified identity settings (such as DKIM and feedback notifications) of the email address override the domain-level settings. For example, if you verify <em>example.com</em> and <em>sender@example.com</em>, and you have DKIM enabled for <em>example.com</em> but not enabled for <em>sender@example.com</em>, then emails you send from <em>sender@example.com</em> will not be DKIM-signed. </p> </li> 
         <li class="listitem"> <p>Amazon SES has endpoints in multiple AWS regions, and email address verification status is separate for each AWS region. You must complete the email address verification process for each sender in the AWS region(s) you want to use. For information about using Amazon SES in multiple AWS regions, see <a href="regions.html">Regions and Amazon SES</a>. </p> </li> 
         <li class="listitem"> <p>You can verify as many as 10,000 identities (domains and email addresses, in any combination) per AWS account. </p> </li> 
         <li class="listitem"> <p>Verifying an email address also allows you to set the "From" and "Return-Path" address to any address formed by adding a label to the verified address. Addresses that contain labels are of the form <em>name+label@example.com</em>, with user-specified text between the plus sign (+) and the at sign (@). </p> <p>For example, if you verify <em>user@example.com</em>, you can also send email from <em>user+recipient1@example.com</em>, <em>user+recipient2@example.com</em>, and so on. This makes it possible to support Variable Envelope Return Path (VERP)  the use of a different return path for each recipient. For more information about VERP, see <a href="http://en.wikipedia.org/wiki/Variable_envelope_return_path" target="_blank"> http://en.wikipedia.org/wiki/Variable_envelope_return_path</a>. </p> <p>When you verify an unlabeled address, then you are essentially verifying all addresses that are formed by adding a label to the verified address. The opposite, however is not true. Verifying an email address that already contains a label does not allow you to send from other addresses. For example, verifying <em>andrew+recipient1@example.com</em> does not allow you to send from <em>andrew@example.com</em>, <em>andrew+recipient2@example.com</em>, or <em>andrew+recipient1+recipient2@example.com</em>. </p> </li> 
         <li class="listitem"> <p>If you want to use the <code class="code">SendRawEmail</code> API action to send a message that contains a "Sender" header, then you must first verify the email address or domain in that header. For more information, see <a href="send-email-raw.html#send-email-raw-headers">About Email Header Fields</a>. </p> </li> 
        </ul> 
       </div> 
       <h2 id="verify-email-addresses-procedure">Verification Procedures</h2> 
       <p>The following procedures show how to use the Amazon SES console to verify and view email addresses. </p> 
       <p class="title"><b>To verify an email address</b></p> 
       <ol> 
        <li> <p>Go to your <a href="https://console.aws.amazon.com/ses/home?#verified-senders-email:" target="_blank"> email address list</a> in the Amazon SES console, or follow these instructions to navigate to it: </p> 
         <ol> 
          <li> <p>Sign in to the AWS Management Console and open the Amazon SES console at <a href="https://console.aws.amazon.com/ses/" target="_blank">https://console.aws.amazon.com/ses/</a>. </p> </li> 
          <li> <p>In the navigation pane, under <b>Identity Management</b>, choose <b>Email Addresses</b>. </p> </li> 
         </ol> </li> 
        <li> <p>Choose <b>Verify a New Email Address</b>. </p> </li> 
        <li> <p>In the <b>Verify a New Email Address</b> dialog box, type your email address in the indicated field, and then choose <b>Verify This Email Address</b>. </p> 
         <div class="mediaobject"> 
          <img src="images/verify_a_new_email_address.png" alt="
                                Verify a New Email Address
                            " width="100%"> 
         </div> </li> 
        <li> <p>In your email client, open the message from Amazon SES asking you to confirm that you are the owner of this email address. </p> </li> 
        <li> <p>Click the link in the message.</p> 
         <div class="aws-note"> 
          <p class="aws-note">Note</p> 
          <p>The link in the verification message expires 24 hours after your original verification request. </p> 
         </div> </li> 
        <li> <p>The status of the email address in the Amazon SES console will change from "pending verification" to "verified". </p> </li> 
        <li> <p>You can now use Amazon SES to send email from this address. To send a test email, check the box next to the verified email address, and then choose <b>Send a Test Email</b>. </p> </li> 
       </ol> 
       <p class="title"><b>To view your verified email addresses</b></p> 
       <ol> 
        <li> <p>Go to your <a href="https://console.aws.amazon.com/ses/home?#verified-senders-email:" target="_blank"> email address list</a> in the Amazon SES console, or follow these instructions to navigate to it: </p> 
         <ol> 
          <li> <p>Sign in to the AWS Management Console and open the Amazon SES console at <a href="https://console.aws.amazon.com/ses/" target="_blank">https://console.aws.amazon.com/ses/</a>. </p> </li> 
          <li> <p>In the navigation pane, under <b>Identity Management</b>, choose <b>Email Addresses</b>. </p> </li> 
         </ol> </li> 
        <li> <p>In the list of verified email addresses, you can expand one or more email addresses to view the details. </p> </li> 
       </ol> 
       <p class="title"><b>To remove verified email addresses</b></p> 
       <ol> 
        <li> <p>Go to your <a href="https://console.aws.amazon.com/ses/home?#verified-senders-email:" target="_blank"> email address list</a> in the Amazon SES console, or follow these instructions to navigate to it: </p> 
         <ol> 
          <li> <p>Sign in to the AWS Management Console and open the Amazon SES console at <a href="https://console.aws.amazon.com/ses/" target="_blank">https://console.aws.amazon.com/ses/</a>. </p> </li> 
          <li> <p>In the navigation pane, under <b>Identity Management</b>, choose <b>Email Addresses</b>. </p> </li> 
         </ol> </li> 
        <li> <p>Check the box beside each email address that you want to remove, and then choose <b>Remove</b>. </p> </li> 
       </ol> 
       <p><b>Using the Amazon SES API</b></p> 
       <p>You can also manage verified email addresses with the Amazon SES API. The following actions are available: </p> 
       <div class="itemizedlist"> 
        <ul class="itemizedlist" type="disc"> 
         <li class="listitem"> <p><code class="code">VerifyEmailIdentity</code></p> </li> 
         <li class="listitem"> <p><code class="code">ListIdentities</code></p> </li> 
         <li class="listitem"> <p><code class="code">DeleteIdentity</code></p> </li> 
         <li class="listitem"> <p><code class="code">GetIdentityVerificationAttributes</code></p> </li> 
        </ul> 
       </div> 
       <div class="aws-note"> 
        <p class="aws-note">Note</p> 
        <p>The API actions above are preferable to the following older API actions, which are deprecated as of the May 15, 2012 release of Domain Verification. </p> 
        <div class="itemizedlist"> 
         <ul class="itemizedlist" type="disc"> 
          <li class="listitem"> <p><code class="code">VerifyEmailAddress</code></p> </li> 
          <li class="listitem"> <p><code class="code">ListVerifiedEmailAddresses</code></p> </li> 
          <li class="listitem"> <p><code class="code">DeleteVerifiedEmailAddress</code></p> </li> 
         </ul> 
        </div> 
       </div> 
       <p>You can use these API actions to write a customized front-end application for email address verification. For a complete description of the API actions related to email verification, go to the <a href="http://docs.aws.amazon.com/ses/latest/APIReference/" target="_blank">Amazon Simple Email Service API Reference</a>. </p>
