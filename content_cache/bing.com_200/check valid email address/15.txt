         <h1 class="entry-title" itemprop="headline">Java Email Address Validation using Regex</h1>
         <p class="entry-meta"><time class="entry-time" itemprop="datePublished" datetime="2014-11-11T14:39:54+00:00">November 11, 2014</time> by <span class="entry-author" itemprop="author" itemscope itemtype="https://schema.org/Person"><span class="entry-author-name" itemprop="name">Lokesh Gupta</span></span></p>
        </header>
        <div class="entry-content" itemprop="text">
         <div style="width: 336px; height: 280px; margin: 6px 20px 20px 0px; float: left;">
          <script async src="//pagead2.googlesyndication.com/pagead/js/adsbygoogle.js"></script>
          <ins class="adsbygoogle" style="display:inline-block;width:336px;height:280px" data-ad-client="ca-pub-2827350136257294" data-ad-slot="6068795363"></ins>
          <script>(adsbygoogle = window.adsbygoogle || []).push({});</script>
         </div>
         <p>Email validation using regular expressions is common task which may be required in any application which seek email address as required information in registration step. There may be more usecase but that’s not point of discussion here. Let’s directly jump into main discussion i.e. validating email address in java using regular expressions.</p>
         <h2>Simplest regex to validate emails</h2>
         <blockquote>
          <p><strong>Regex : ^(.+)@(.+)$</strong></p>
         </blockquote>
         <p>This one is simplest and only cares about ‘@’ symbol. Before and after ‘@’ symbol, there can be any number of characters. Let’s see a quick example to see what I mean.</p>
         <pre class="brush: java; title: ; notranslate" title="">
List emails = new ArrayList();
emails.add("user@domain.com");
emails.add("user@domain.co.in");
emails.add("user1@domain.com");
emails.add("user.name@domain.com");
emails.add("user#@domain.co.in");
emails.add("user@domaincom");

//Invalid emails
emails.add("user#domain.com");
emails.add("@yahoo.com");

String regex = "^(.+)@(.+)$";

Pattern pattern = Pattern.compile(regex);

for(String email : emails){
	Matcher matcher = pattern.matcher(email);
	System.out.println(email +" : "+ matcher.matches());
}

Output:

user@domain.com : 		true
user@domain.co.in : 	true
user1@domain.com : 		true
user.name@domain.com : 	true
user#@domain.co.in : 	true
user@domaincom : 		true

user#domain.com : 		false
@yahoo.com : 			false
</pre>
         <p>Note: This pattern is available in Common lang’s <a title="EmailValidator" href="https://commons.apache.org/proper/commons-validator/javadocs/api-1.4.0/org/apache/commons/validator/EmailValidator.html" target="_blank"><strong>EmailValidator</strong></a> class. So if it fit’s your need, you can directly use this class.</p>
         <h2>Adding Restrictions on User Name part</h2>
         <blockquote>
          <p><strong>Regex : ^[A-Za-z0-9+_.-]+@(.+)$</strong></p>
         </blockquote>
         <p>In this regex, we have added some restriction osn username part of email address. Restrictions in above regex are:</p>
         <p>1) A-Z characters allowed<br> 2) a-z characters allowed<br> 3) 0-9 numbers allowed<br> 4) Additionally email may contain only dot(.), dash(-) and underscore(_)<br> 5) Rest all characters are not allowed</p>
         <p>Let’s test some email addresses against above regex.</p>
         <pre class="brush: java; title: ; notranslate" title="">
List emails = new ArrayList();
emails.add("user@domain.com");
emails.add("user@domain.co.in");
emails.add("user1@domain.com");
emails.add("user.name@domain.com");
emails.add("user_name@domain.co.in");
emails.add("user-name@domain.co.in");
emails.add("user@domaincom");

//Invalid emails
emails.add("@yahoo.com");

String regex = "^[A-Za-z0-9+_.-]+@(.+)$";

Pattern pattern = Pattern.compile(regex);

for(String email : emails){
	Matcher matcher = pattern.matcher(email);
	System.out.println(email +" : "+ matcher.matches());
}

Output:

user@domain.com : 			true
user@domain.co.in : 		true
user1@domain.com : 			true
user.name@domain.com : 		true
user_name@domain.co.in : 	true
user-name@domain.co.in : 	true
user@domaincom : 			true

@yahoo.com : 				false
</pre>
         <p>Please note that similar restriction you can apply to domain name part as well. Then regular expression will become like this :<br> ^[A-Z0-9+_.-]+@[A-Z0-9.-]+$</p>
         <h2>Regex allowing email addresses permitted by RFC 5322</h2>
         <blockquote>
          <p><strong>Regex : ^[a-zA-Z0-9_!#$%&amp;’*+/=?`{|}~^.-]+@[a-zA-Z0-9.-]+$</strong></p>
         </blockquote>
         <p>This regex example uses all the characters permitted by RFC 5322, which governs the email message format. Among the permitted characters are some that present a security risk if passed directly from user input to an SQL statement, such as the single quote (‘) and the pipe character (|). You should be sure to escape sensitive characters when inserting the email address into a string passed to another program, in order to prevent security holes such as SQL injection attacks.</p>
         <pre class="brush: java; title: ; notranslate" title="">
List emails = new ArrayList();
emails.add("user@domain.com");
emails.add("user@domain.co.in");
emails.add("user.name@domain.com");
emails.add("user?name@domain.co.in");
emails.add("user'name@domain.co.in");

//Invalid emails
emails.add("@yahoo.com");

String regex = "^[a-zA-Z0-9_!#$%&amp;'*+/=?`{|}~^.-]+@[a-zA-Z0-9.-]+$";

Pattern pattern = Pattern.compile(regex);

for(String email : emails){
	Matcher matcher = pattern.matcher(email);
	System.out.println(email +" : "+ matcher.matches());
}

Output:

user@domain.com : 			true
user@domain.co.in : 		true
user.name@domain.com : 		true
user?name@domain.co.in : 	true
user'name@domain.co.in : 	true
@yahoo.com : 				false
</pre>
         <h2>Regex to restrict leading, trailing, or consecutive dots in emails</h2>
         <blockquote>
          <p><strong>Regex : ^[a-zA-Z0-9_!#$%&amp;’*+/=?`{|}~^-]+(?:\\.[a-zA-Z0-9_!#$%&amp;’*+/=?`{|}~^-]+)*@[a-zA-Z0-9-]+(?:\\.[a-zA-Z0-9-]+)*$</strong></p>
         </blockquote>
         <p>Both the local part and the domain name can contain one or more dots, but no two dots can appear right next to each other. Furthermore, the first and last characters in the local part and in the domain name must not be dots:</p>
         <pre class="brush: java; title: ; notranslate" title="">
List emails = new ArrayList();
emails.add("user@domain.com");
emails.add("user@domain.co.in");
emails.add("user.name@domain.com");
emails.add("user'name@domain.co.in");

//Invalid emails
emails.add(".username@yahoo.com");
emails.add("username@yahoo.com.");
emails.add("username@yahoo..com");

String regex = "^[a-zA-Z0-9_!#$%&amp;'*+/=?`{|}~^-]+(?:\\.[a-zA-Z0-9_!#$%&amp;'*+/=?`{|}~^-]+)*@[a-zA-Z0-9-]+(?:\\.[a-zA-Z0-9-]+)*$";

Pattern pattern = Pattern.compile(regex);

for(String email : emails){
	Matcher matcher = pattern.matcher(email);
	System.out.println(email +" : "+ matcher.matches());
}

Output:

user@domain.com : 			true
user@domain.co.in : 		true
user.name@domain.com : 		true
user'name@domain.co.in : 	true

.username@yahoo.com : 		false
username@yahoo.com. : 		false
username@yahoo..com : 		false

</pre>
         <h2>Recommended : Regex to restrict no. of characters in top level domain</h2>
         <p>Now lets modify the regex such that domain name must include at least one dot, and that the part of the domain name after the last dot can only consist of letters.</p>
         <p>Let’s say domain names are like secondlevel.com or thirdlevel.secondlevel.com. The top-level domain (.com in these examples) must consist of two to six letters only.</p>
         <blockquote>
          <p><strong>Regex : ^[\\w!#$%&amp;’*+/=?`{|}~^-]+(?:\\.[\\w!#$%&amp;’*+/=?`{|}~^-]+)*@(?:[a-zA-Z0-9-]+\\.)+[a-zA-Z]{2,6}$</strong></p>
         </blockquote>
         <pre class="brush: java; title: ; notranslate" title="">
List emails = new ArrayList();
emails.add("user@domain.com");
emails.add("user@domain.co.in");
emails.add("user.name@domain.com");
emails.add("user_name@domain.com");
emails.add("username@yahoo.corporate.in");

//Invalid emails
emails.add(".username@yahoo.com");
emails.add("username@yahoo.com.");
emails.add("username@yahoo..com");
emails.add("username@yahoo.c");
emails.add("username@yahoo.corporate");

String regex = "^[\\w!#$%&amp;'*+/=?`{|}~^-]+(?:\\.[\\w!#$%&amp;'*+/=?`{|}~^-]+)*@(?:[a-zA-Z0-9-]+\\.)+[a-zA-Z]{2,6}$";

Pattern pattern = Pattern.compile(regex);

for(String email : emails){
	Matcher matcher = pattern.matcher(email);
	System.out.println(email +" : "+ matcher.matches());
}

Output:

user@domain.com : 				true
user@domain.co.in : 			true
user.name@domain.com : 			true
user_name@domain.com : 			true
username@yahoo.corporate.in : 	true

.username@yahoo.com : 			false
username@yahoo.com. : 			false
username@yahoo..com : 			false
username@yahoo.c : 				false
username@yahoo.corporate : 		false

</pre>
         <p>This last regex is my recommendation to validate emails in your project. Please feel free to use this regex as well as edit it as per your application’s additional needs.</p>
         <p><strong>Happy Learning !!</strong></p>
         <p><strong>Reference:</strong> <a href="https://www.rfc-editor.org/rfc/rfc5322.txt" target="_blank">http://www.rfc-editor.org/rfc/rfc5322.txt</a></p>
