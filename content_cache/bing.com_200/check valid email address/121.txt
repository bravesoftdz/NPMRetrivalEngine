            <h1 itemprop="name">Implementing Email Validation Using PHP</h1> 
            <i class="fa fa-calendar-o"></i> 
            <time data-pubdate="" itemprop="datePublished" content="2015-01-24 00:00:00Z" datetime="2015-01-24 00:00:00Z">Saturday, January 24, 2015</time> 
            <br>
            <br> 
           </header> 
           <p class="emh-text-lead" itemprop="headline"> Here's how to implement email address validation on your site using a very simple <abbr title="Personal Home Page">PHP</abbr> script. </p> 
           <br> 
           <img class="center-block" src="/s/php-1-250x249.png" alt="PHP email validation" itemprop="image"> 
           <br> 
           <p> <abbr title="Personal Home Page">PHP</abbr> is the most widely used server side scripting language for web sites, applications and content management systems. As such, I've created a fully working example email validation script for <abbr title="Personal Home Page">PHP</abbr>. It's free to use, modify as you see fit. </p> 
           <h2>Warnings, Disclaimers and Caveats</h2> 
           <p>The script provided here is for education purposes only and is provided "as-is".</p> 
           <p>It has many known short-comings in non-functional and functional aspects.</p> 
           <ul> 
            <li> <strong>It doesn't perform or scale well</strong> - don't expect to be doing millions of email verifications with this script </li> 
            <li><strong>Limited coverage</strong> - does not work very well with some mail services (e.g. Hotmail, Yahoo)</li> 
            <li><strong>Intermittent connectivity</strong> - your servers IP address can be temporarily or permanently banned with excessive use of this script. When IP blocks happen, the script stops working</li> 
            <li><strong>Incorrect results</strong> - results can come back as "Ok" when infact the actual answer is "Bad" (confirmed by sending email to address). This is because script has no way of identifying "catch all" domains. Yahoo is a "catch all" domain and answers "Ok" to all queries</li> 
           </ul> 
           <p> Please don't expect industrial grade results from this script. It is simple, excludes a lot of the fine tuning code (omitted for clarity) needed to deliver robust, accurate email verification systems. </p> 
           <h2>Email Verification - The Process Defined</h2> 
           <p> Before deep diving into the code itself, let's go through the sequence of processing (with some code snippets) used in typical server side email verification. </p> 
           <h3>Step 1 - Syntax Validation</h3> 
           <p> Check the email for valid syntax. For example, <em>me.here.com</em> is clearly not an email address whereas <em><a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="c7aaa287afa2b5a2e9a4a8aa">[email&nbsp;protected]</a></em> could be. </p> 
           <p> We can check for basic syntax compliance by doing pattern recognition checks (perhaps using Regular Expressions (regex)) with the example below: </p> 
           <h4>Example of Syntax Validation of Email Address Using Regular Expressions</h4> 
           <pre><code title="Syntax validation of email address using Regular Expressions">
        if (!eregi("^[_a-z0-9-]+(\.[_a-z0-9-]+)*@[a-z0-9-]+(\.[a-z0-9-]+)*(\.[a-z]{2,4})$", "<a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="761b13361e1304135815191b">[email&nbsp;protected]</a>"))
        {
            echo "bad syntax";
        }
    </code></pre> 
           <br> 
           <h3>Step 2 - <abbr title="Domain Name System">DNS</abbr> Checks</h3> 
           <p> Extract the "domain" from the email address. As in, the domain for <em><a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="402d2500282532256e232f2d">[email&nbsp;protected]</a></em> is <em>here.com</em>. </p> 
           <h4>Example Of How To Extract Domain From Email Address</h4> 
           <pre><code title="Example of How To Extract Domain From Email Address">
        // load the user and domain name into a local list from email address using string split function.
        list ( $Username, $Domain ) = split ("@","<a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="9bf6fedbf3fee9feb5f8f4f6">[email&nbsp;protected]</a>");
    </code></pre> 
           <p> There are a couple of records in <abbr title="Domain Name System">DNS</abbr> that we are interested in for our purposes: </p> 
           <ol> 
            <li>The A record.</li> 
            <li>The <abbr title="Mail Exchanger">MX</abbr> record.</li> 
           </ol> 
           <p> Technically, in most (but not all) situations a valid email address will have both A record(s) and <abbr title="Mail Exchanger">MX</abbr> record(s). </p> 
           <p> For simplicity, the demonstration script just looks for at least one valid <abbr title="Mail Exchanger">MX</abbr> record. </p> 
           <h4>Example Of How To Get MX Record Using PHP</h4> 
           <pre><code title="Example of How To Get MX Record Using PHP">
        // check if domain has MX record(s)
        if ( checkdnsrr ( "here.com", "MX" ) )
        {
            //MX record(s) present for domain, do more processing
            echo "Yay! Possibly a valid email address. Let's do more to find out.";
        }
        else
        {
            /*No MX record(s) present for domain. This means no mail servers defined for domain. 
            Unlikely, therefore, that email address is valid.*/
            echo "Boo! Bailing out to report invalid email address.";
        }
    </code></pre> 
           <br> 
           <h3>Step 3 - Mailbox Validation</h3> 
           <p> Once <abbr title="Domain Name System">DNS</abbr> checks are complete, we pick a <abbr title="Mail Exchanger">MX</abbr> record for the domain and establish a TCP connection on port 25 (default port for <abbr title="Simple Mail Transfer Protocol">SMTP</abbr>). </p> 
           <h4>Example Of How To Connect To a Mail Server Using PHP</h4> 
           <pre><code title="Example of How To Get MX Record Using PHP">
        // Get DNS MX records from domain
        getmxrr ("here.com", $MXHost));
        // Get the first MX record IP address
        $ConnectAddress = $MXHost[0];
        // Open TCP connection to IP address on port 25 (default <abbr title="Simple Mail Transfer Protocol">SMTP</abbr> port)
        $Connect = fsockopen ( $ConnectAddress, 25 );
    </code></pre> 
           <p> Once connected to the remote mail server, the next bit of the process consists of our script having a "chat" with the remote mail server according to a pre-defined application level chat protocol known as <abbr title="Simple Mail Transfer Protocol">SMTP</abbr>. </p> 
           <p> <abbr title="Simple Mail Transfer Protocol">SMTP</abbr> is well documented elsewhere so I'm not going to repeat the entire conversation here. For mailbox verification we go as far as the <em>RCPT TO:&lt;<a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="a2cfc7e2cac7d0c78cc1cdcf">[email&nbsp;protected]</a>&gt;</em> step in the protocol. The response code (a number) returned from RCPT TO: is usually enough to give a good enough indication as to whether a mail address is valid or not. </p> 
           <h4>Example Of RCPT TO Email Validation</h4> 
           <pre><code title="Example of RCPT TO Email Validation">
        //... stuff that came before
        // Next, do RCPT TO:
        fputs ( $Connect, "RCPT TO: &lt;me@here.com&gt;\r\n" );
        $to_reply = fgets ( $Connect, 1024 );
        // Quit the SMTP conversation and disconnect from remote mail server.
        fputs ( $Connect, "QUIT\r\n");
    </code></pre> 
           <h2>Complete Server Side PHP For Email Validation</h2>
