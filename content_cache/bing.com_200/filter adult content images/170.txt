           <h1 class="post-title">Java Servlet Filter tutorial example using Eclipse &amp; Tomcat</h1> 
           <p class="post-byline">by <a href="http://viralpatel.net/blogs/author/viral/" title="Posts by Viral Patel" rel="author">Viral Patel</a> · January 8, 2009</p> 
           <div class="clear"></div> 
           <div class="entry"> 
            <div class="entry-inner"> 
             <p>[ad name=AD_INBETWEEN_POST]Servlet Filters are the latest components that are added in Servlet 2.3 specifications. These filters are used basically for intercepting and modifying requests and response from server.&nbsp; Consider a scenario where you want to check session from the every users request and if it is valid then only you want to let the user access the page. You can acheive this by checking sessions on all the servlet pages (or JSP pages) which users queries or you can do this by using Filter.</p> 
             <p>Let us create a Servlet Filter that just prints the clients ip address and date time. This is just to log users who are accessing the application.</p> 
             <!-- Quick Adsense WordPress Plugin: http://quicksense.net/ --> 
             <div style="float:none;margin:10px 0 10px 0;text-align:center;"> 
              <script async src="//pagead2.googlesyndication.com/pagead/js/adsbygoogle.js"></script> 
              <!-- BLOGS_INBETWEEN_POST --> 
              <ins class="adsbygoogle" style="display:inline-block;width:300px;height:250px" data-ad-client="ca-pub-2652062086099003" data-ad-slot="1602773796"></ins>

             </div> 
             <p>We we use Eclipse for developing our application and Apache Tomcat for deploying and running our application.</p> 
             <h2>Step 1: Create dynamic web project in Eclipse.</h2> 
             <p><img class="aligncenter size-medium wp-image-607" title="servlet-filter-create-new-project-eclipse" src="//img.viralpatel.net/servlet-filter-create-new-project-eclipse-300x292.png" alt="" width="300" height="292" srcset="http://img.viralpatel.net/servlet-filter-create-new-project-eclipse-300x292.png 300w, http://img.viralpatel.net/servlet-filter-create-new-project-eclipse.png 363w" sizes="(max-width: 300px) 100vw, 300px"></p> 
             <p>Starts eclipse and create a new dynamic web project with name ServletFilterProject. Select Target runtime environment. I have selected Apache Tomcat v6.0, you can select any Tomcat version that you have installed. Click on Finish.</p> 
             <h2>Step 2: Create package &amp; Servlet Filter class.</h2> 
             <p><img class="aligncenter size-full wp-image-609" title="servlet-filter-package-creation" src="//img.viralpatel.net/servlet-filter-package-creation.png" alt="" width="213" height="123"></p> 
             <p>Create a package for Servlet Filters in your source folder of Project. I have created a package <strong>net.viralpatel.servlet.filters</strong>. Inside the package, create a Java class file called LogFilter.java.</p> 
             <h2>Step 3: Writting Servlet Filter class code.</h2> 
             <p>Modify the LogFilter class and add following code into it.</p> 
             <pre><code class="language-java">package net.viralpatel.servlet.filters;

import java.io.IOException;
import java.util.Date;

import javax.servlet.Filter;
import javax.servlet.FilterChain;
import javax.servlet.FilterConfig;
import javax.servlet.ServletException;
import javax.servlet.ServletRequest;
import javax.servlet.ServletResponse;
import javax.servlet.http.HttpServletRequest;

public class LogFilter implements Filter {

	public void doFilter(ServletRequest req, ServletResponse res,
			FilterChain chain) throws IOException, ServletException {

		HttpServletRequest request = (HttpServletRequest) req;
		
		//Get the IP address of client machine.
		String ipAddress = request.getRemoteAddr();
		
		//Log the IP address and current timestamp.
		System.out.println("IP "+ipAddress + ", Time " 
							+ new Date().toString());
		
		chain.doFilter(req, res);
	}
	public void init(FilterConfig config) throws ServletException {
		
		//Get init parameter
		String testParam = config.getInitParameter("test-param");
		
		//Print the init parameter
		System.out.println("Test Param: " + testParam);
	}
	public void destroy() {
		//add code to release any resource
	}
}
</code></pre> 
             <p>In this filter example, we have implemented an interface javax.servlet.Filter and override its methods init, doFilter and destroy.</p> 
             <p>The <strong>init()</strong> method is used to initialize any code that is used by Filter. Also note that, init() method will get an object of FilterConfig which contains different Filter level information as well as init parameters which is passed from Web.xml (Deployment descriptor).</p> 
             <p>The <strong>doFilter()</strong> method will do the actual logging of information. You can modify this method and add your code which can modify request/session/response, add any attribute in request etc.</p> 
             <p>The <strong>destroy()</strong> method is called by the container when it wants to garbage collect the filter. This is usually done when filter is not used for long time and server wants to allocate memory for other applications.</p> 
             <h2>Step 4: Create Servlet Filter Mapping in Web.xml</h2> 
             <p>Open <strong>web.xml</strong> file from WEB-INF directory of your Project and add following entry for filter tag.</p> 
             <pre><code class="language-markup">&lt;filter&gt;
	&lt;filter-name&gt;LogFilter&lt;/filter-name&gt;
	&lt;filter-class&gt;
		net.viralpatel.servlet.filters.LogFilter
	&lt;/filter-class&gt;
	&lt;init-param&gt;
		&lt;param-name&gt;test-param&lt;/param-name&gt;
		&lt;param-value&gt;This parameter is for testing.&lt;/param-value&gt;
	&lt;/init-param&gt;
&lt;/filter&gt;
&lt;filter-mapping&gt;
	&lt;filter-name&gt;LogFilter&lt;/filter-name&gt;
	&lt;url-pattern&gt;/*&lt;/url-pattern&gt;
&lt;/filter-mapping&gt;
</code></pre> 
             <p>In this entry, we have added LogFilter class in Web xml and mapped it with URL /*. Hence any request from client will generated a call to this filter. Also we have passed a parameter <strong>test-param</strong>. This is just to show how to pass and retrieve a parameter in servlet filter.</p> 
             <h2>Step 5: Execute the web application</h2> 
             <p>We are done with the coding part of Servlet Filter example. Now execute the project by Run -&gt; Run As -&gt; Run on server (shortcut Alt+Shift+X, R).<br> Check the console you will see the output that we print using System.out.</p> 
             <p>Change the code in the way you want your filter to work.</p>
