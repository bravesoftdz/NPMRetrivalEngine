     <h1>Eclipse for Java</h1> 
     <h2>How To Install Eclipse and Get Started with Java Programming<br> (on Windows, Mac OS and Ubuntu)</h2> 
    </div> 
    <div id="content-main"> 
     <p>Eclipse (@ <a href="http://www.eclipse.org/">www.eclipse.org</a>) is an <em>open-source</em> Integrated Development Environment (IDE) supported by IBM. Eclipse is popular for Java application development (Java SE and Java EE) and Android apps. It also supports C/C++, PHP, Python, Perl, and other web project developments via extensible plug-ins. Eclipse is cross-platform and runs under Windows, Linux and Mac OS.</p>

     <h3 id="eclipse-java-install">How to Install Eclipse 4.6 (Neon) for Java Developers</h3> 
     <h4>How to Install Eclipse on Windows</h4> 
     <h5>Step 0: Install JDK</h5> 
     <p>To use Eclipse for Java programming, you need to first install Java Development Kit (JDK). Read "<a href="JDK_HowTo.html#jdk-install">How to Install JDK for Windows</a>".</p> 
     <h5>Step 1: Download</h5> 
     <p>Download Eclipse from <a href="https://www.eclipse.org/downloads/">https://www.eclipse.org/downloads</a>. Under "Get Eclipse Neon" ? Click "Download Packages". For beginners, choose the 3rd entry "<strong>Eclipse IDE for Java Developers</strong>" (32-bit or 64-bit) (e.g., "<code>eclipse-java-neon-2-win32-x86_64.zip</code>" 161MB) ? Download.</p> 
     <h5>Step 2: Unzip</h5> 
     <p>To install Eclipse, simply unzip the downloaded file into a directory of your choice (e.g., "<code>d:\myproject</code>").</p> 
     <p>There is no need to run any installer. Moreover, you can simply delete the entire Eclipse directory when it is no longer needed (without running any un-installer). You are free to move or rename the directory. You can install (unzip) multiple copies of Eclipse in the same machine.</p> 
     <h4 id="install_mac">How to Install Eclipse on Mac OS</h4> 
     <p>To use Eclipse for Java programming, you need to first install JDK. Read "<a href="JDK_HowTo.html#jdk_mac">How to install JDK for Mac OS</a>".</p> 
     <p>To install Eclipse:</p> 
     <ol> 
      <li>Goto <a href="http://www.eclipse.org/downloads/">http://www.eclipse.org/downloads/</a>. Under "Get Eclipse Neon" ? Click "Download Packages". Choose "Mac OS X (Cocoa)". For beginners, select the 2nd entry "Eclipse IDE for Java Developers". You will receive a TAR file (e.g., "<code>eclipse-java-neon-2-macosx-cocoa-x86_64.tar.gz</code>") in "<code>~/Downloads</code>" folder.</li> 
      <li>Double-click the downloaded TAR file to expand it into an folder called "eclipse".</li> 
      <li>Drag the expanded "eclipse" folder to the "<code>Applications</code>" folder.</li> 
     </ol> 
     <p>To start Eclipse, open the "eclipse" folder and click on the "Eclipse" icon. You may drag the "Eclipse" icon to the dock.</p> 
     <p><span class="line-heading">Notes:</span> To uninstall Eclipse, drag the "<code>/Applications/eclipse</code>" folder to trash.</p> 
     <h4 id="install_ubuntu">How to Install Eclipse on Ubuntu Linux</h4> 
     <p>Eclipse comes with many flavors (See "Eclipse Packages" @ <a href="https://www.eclipse.org/downloads/compare.php?release=luna">https://www.eclipse.org/downloads/compare.php?release=neon</a>):</p> 
     <ul> 
      <li>To use Eclipse for Java programming, choose "Eclipse IDE for Java Developers" or "Eclipse IDE for Java EE Developers". You need to first install JDK. Read "<a href="#jdk">How to install JDK on Ubuntu</a>".</li> 
      <li>To use Eclipse for PHP programming, choose "Eclipse IDE for PHP Developers".</li> 
      <li>To use Eclipse for C/C++ programming, choose "Eclipse IDE for C/C++ Developers".</li> 
     </ul> 
     <p>Nonetheless, you can install any package, and then add more features when needed.</p> 
     <p>To install Eclipse (e.g, for Java Programming):</p> 
     <ol>

"Eclipse IDE for Java Developers" (or "Eclipse Standard|Classic") for Java SE program development; or "Eclipse IDE for Java EE Developers" for developing webapps. You will receive a TAR file (e.g., "<code>eclipse-jee-neon-2-linux-gtk-x86_64.tar.gz</code>") in the "<code>~/Downloads</code>" folder.</li> 
      <li>We shall install Eclipse under <code>/usr/local</code>. <pre class="color-command">

     <h3 id="eclipse-java-first">Writing your First Java Program in Eclipse</h3> 
     <h5>Step 0: Launch Eclipse</h5> 
     <ol> 
      <li>Launch Eclipse by running "<code>eclipse.exe</code>" from the Eclipse installed directory.</li> 
      <li>Choose an appropriate directory for your <em>workspace</em>, i.e., where you would like to save your files (e.g., <code>d:\myproject\eclipse</code> for Windows)</li> 
      <li>If the "Welcome" screen shows up, close it by clicking the "cross" button next to the "Welcome" title.</li> 
     </ol> 
     <h5>Step 1: Create a new Java Project</h5> For each Java application, you need to create a 
     <em>project</em> to keep all the source files, classes and relevant resources. 
     <p>To create a new Java project:</p> 
     <ol> 
      <li>Choose "File" menu ? "New" ? "Java project".</li> 
      <li>The "New Java Project" dialog pops up. 
       <ol> 
        <li>In "Project name", enter "<code>FirstProject</code>".</li> 
        <li>Check "Use default location".</li> 
        <li>In "JRE", select "Use default JRE (currently 'JDK1.8.xx')". But make sure that your JDK is 1.5 and above.</li> 
        <li>In "Project Layout", check "Use project folder as root for sources and class files"</li> 
       </ol> Push "Finish" button.</li> 
     </ol> 
     <h5>Step 2: Write a Hello-world Java Program</h5> 
     <ol> 
      <li>In the "Package Explorer" (left pane) ? Right-click on "<code>FirstProject</code>" (or use the "File" menu) ? New ? Class.</li> 
      <li>The "New Java Class" dialog pops up. 
       <ol> 
        <li>In "Source folder", keep the "FirstProject"</li> 
        <li>In "Package", delete the content if it is not empty</li> 
        <li>In "Name", enter "<code>Hello</code>"</li> 
        <li>Check "<code>public static void main(String[] args)</code>"</li> 
        <li>Don't change the rest.</li> 
       </ol> Push "Finish" button.</li> 
      <li>The source file "<code>Hello.java</code>" opens on the editor panel (the center pane). Enter the following codes: <pre class="color-example">
public class Hello {
   public static void main(String[] args) {
      <strong>System.out.println("Hello, world!");</strong>
   }
}</pre> </li>
     </ol> 
     <h5>Step 3: Compile &amp; Execute the Java Program</h5> 
     <ol> 
      <li>There is no need to <em>compile</em> the Java source file in Eclipse explicitly. It is because Eclipse performs the so-called <em>incremental compilation</em>, i.e., the Java statement is compiled as and when it is entered.</li> 
      <li>To run the program, right-click anywhere on the source file "<code>Hello.java</code>" (or choose "Run" menu) ? Run As ? Java Application.</li> 
      <li>The output "Hello, world!" appears on the Console panel (the bottom pane).</li> 
     </ol> 
     <p>NOTES:</p> 
     <ul> 
      <li>You should create a NEW Java project for EACH of your Java application.</li> 
      <li>Nonetheless, Eclipse allows you to keep more than one programs in a project, which is handy for writing toy programs (such as your tutorial exercises). To run a particular program, open and right-click on the source file ? Run As ? Java Application. </li> 
      <li>Clicking the "Run" button (with a "Play" icon) runs the recently-run program (based on the previous configuration). Try clicking on the "down-arrow" besides the "Run" button.</li> 
     </ul> 
     <h3>Read the Eclipse Documentation</h3> 
     <p>At a minimum, you SHOULD browse through Eclipse's "<strong>Workbench User Guide</strong>" and "<strong>Java Development User Guide</strong>" - accessible via the Eclipse's "Welcome" page or "Help" menu. This will save you many agonizing hours trying to figure out how to do somethings later.</p>

     <p>Let's use the graphic debugger to debug the program.</p> 
     <h5>Step 1: Set an Initial Breakpoint</h5> 
     <img class="image-float-right" src="images/EclipseJavaDebugSetBreakpoint.png" alt="EclipseJavaDebugSetBreakpoint.png"> 
     <p>A <em>breakpoint</em> suspends program execution for you to examine the internal states (e.g., value of variables) of the program. Before starting the debugger, you need to set at least one breakpoint to suspend the execution inside the program. Set a breakpoint at <code>main()</code> method by double-clicking on the <em>left-margin</em> of the line containing <code>main()</code>. A <em>blue circle</em> appears in the left-margin indicating a breakpoint is set at that line. </p> 
     <h5 class="float-clear">Step 2: Start Debugger</h5> 
     <img class="image-float-right" src="images/EclipseJavaDebugStart.png" alt="EclipseJavaDebugStart.png"> 
     <p>Right click anywhere on the source code (or from the "Run" menu) ? "Debug As" ? "Java Application" ? choose "Yes" to switch into "Debug" perspective (A <em>perspective</em> is a particular arrangement of panels to suits a certain development task such as editing or debugging). The program begins execution but suspends its operation at the breakpoint, i.e., the <code>main()</code> method.</p> 
     <p>As illustrated in the following diagram, the highlighted line (also pointed to by a blue arrow) indicates the statement to be executed in the <em>next</em> step.</p> 
     <h5 class="float-clear">Step 3: Step-Over and Watch the Variables and Outputs</h5> 
     <img class="image-float-right" src="images/EclipseJavaDebugVariables.png" alt="EclipseJavaDebugVariables.png"> 
     <img class="image-float-right" src="images/EclipseJavaDebugToolbar.png" alt="EclipseJavaDebugToolbar.png"> 
     <p>Click the "Step Over" button (or select "Step Over" from "Run" menu) to <em>single-step</em> thru your program. At each of the step, examine the value of the variables (in the "Variable" panel) and the outputs produced by your program (in the "Console" Panel), if any. You can also place your cursor at any variable to inspect the content of the variable.</p> 
     <p>Single-stepping thru the program and watching the values of internal variables and the outputs produced is the <em>ultimate</em> mean in debugging programs - because it is exactly how the computer runs your program!</p> 
     <h5>Step 4: Breakpoint, Run-To-Line, Resume and Terminate</h5> 
     <p>As mentioned, a breakpoint <em>suspends</em> program execution and let you examine the internal states of the program. To set a breakpoint on a particular statement, double-click the left-margin of that line (or select "Toggle Breakpoint" from "Run" menu).</p> 
     <p>"Resume" continues the program execution, up to the next breakpoint, or till the end of the program.</p> 
     <p>"Single-step" thru a loop with a large count is time-consuming. You could set a breakpoint at the statement immediately outside the loop (e.g., Line 11 of the above program), and issue "Resume" to complete the loop.</p> 
     <p>Alternatively, you can place the cursor on a particular statement, and issue "Run-To-Line" from the "Run" menu to continue execution up to the line.</p> 
     <p> "Terminate" ends the debugging session. Always terminate your current debugging session using "Terminate" or "Resume" till the end of the program.</p> 
     <h5>Step 5: Switching Back to Java perspective</h5> 
     <p>Click the "Java" perspective icon on the upper-right corner to switch back to the "Java" perspective for further programming (or "Window" menu ? Open Perspective ? Java).</p> 
     <p><strong>Important:</strong> I can's stress more that mastering the use of debugger is crucial in programming. Explore the features provided by the debuggers.</p> 
     <h5>Other Debugger's Features</h5> 
     <p><strong>Modify the Value of a Variable:</strong> You can modify the value of a variable by entering a new value in the "Variable" panel. This is handy for temporarily modifying the behavior of a program, without changing the source code.</p> 
     <p><strong>Step-Into and Step-Return:</strong> To debug a <em>method</em>, you need to use "Step-Into" to step into the <em>first</em> statement of the method. ("Step-Over" runs the function in a single step without stepping through the statements within the function.) You could use "Step-Return" to return back to the caller, anywhere within the method. Alternatively, you could set a breakpoint inside a method.</p> 
     <h3 id="tips">Tips &amp; Tricks</h3> 
     <h4>General Usages (for all Programming Tasks)</h4> 
     <p>These are the features that I find to be most useful in Eclipse:</p> 
     <ol> 
      <li><span class="line-heading">Maximizing Window (Double-Clicking):</span> You can double-click on the "header" of any panel to <em>maximize</em> that particular panel, and double-click again to <em>restore</em> it back. This feature is particularly useful for writing source code in full panel.</li> 
      <li><span class="line-heading">Shorthand Templates (sysout, for,...):</span> You can type "<code>sysout</code>" followed by a ctrl+space (or alt-/) as a shorthand for typing "<code>System.out.println()</code>".<br> The default shortcut key (ctrl-space or alt-/) depends on the system. Check your system's shortcut key setting in "Edit" ? "Content Assist" ? "Default". Take note that many of you use ctrl+space to switch between input languages. You need to reconfigure either your language switching hot-key or Eclipse.<br> Similarly, you can type "for" followed by ctrl-space (or alt-/) to get a for-loop.<br> You can create your own shorthand in "Window" menu ? "Preferences" ? "Java" ? "Editor" ? "Templates". (Alternatively, in "Window" ? "Preferences" ? type "template" as filter text and choose "Java" ? "Editor" ? "Templates".)<br> You can change your key settings in "Window" menu ? "Preferences" ? "General" ? "Key" ? choose "Command", "Content Assist". (Alternatively, in "Window" ? "Preferences" ? type "key" as filter text and choose "General" ? "Key".)</li> 
      <li><span class="line-heading">Intelli-Sense (ctrl-space):</span> You can use ctrl-space to activate the "intelli-sense" (or content assist). That is, Eclipse will offer you the choices, while you are typing.</li> 
      <li><span class="line-heading">Source Formatting (ctrl-shift-f):</span> Right-click on the source. Choose "Source" ? "Format" to let Eclipse to layout your source codes with the proper indentation.</li> 
      <li><span class="line-heading">Source Toggle Comment (ctrl-/):</span> To comment/uncomment a block of codes, choose "Source" ? "Toggle Comment".</li> 
      <li><span class="line-heading">Hints for Correcting Syntax Error:</span> If there is a syntax error on a statement, a red mark will show up on the left-margin on that statement. You could click on the "light bulb" to display the error message, and also select from the available hints for correcting that syntax error.</li> 
      <li><span class="line-heading">Refactor (or Rename) (alt-shift-r):</span> You can rename a variable, method, class, package or even the project easily in Eclipse. Select and right-click on the entity to be renamed ? "Refactor" ? "Rename". Eclipse can rename all the occurrences of the entity.</li> 
      <li><span class="line-heading">Line Numbers:</span> To show the line numbers, choose "Window" menu ? "Preferences" ? "General" ? "Editors" ? "Text Editors" ? Check the "Show Line Numbers" Box. You can also configure many editor options, such as the number of spaces for tab. Alternatively, you can right-click on the left-margin, and check "Show Line Numbers".</li> 
      <li><span class="line-heading">Error Message Hyperlink:</span> Click on an error message will hyperlink to the corresponding source statement.</li> 
      <li><span class="line-heading">Changing Font Type and Size:</span> From "Window" menu ? "Preferences" ? "General" ? "Appearance" ? "Colors and Fonts" ? expand "Java" ? "Java Editor Text Font" ? "Edit". (Alternatively, in "Window" ? "Preferences" ? type "font" as filter text and choose the appropriate entry.)</li> 
      <li><span class="line-heading">Unicode Support:</span> To enable Unicode support, select "Window" menu ? Preferences ? General ? Workspace ? Text file encoding ? UTF-8. This sets the default character set used for file encoding, similar to VM's command-line option <code>-Dfile.encoding=UTF-8</code>. Commonly used charsets for Unicode are UTF-8, UTF-16 (with BOM), UTF-16BE, UTF-16LE. Other charsets are US-ASCII, ISO-8859-1.</li> 
      <li><span class="line-heading">Mouse Hover-over:</span> In debug mode, you could configure to show the variable's value when the mouse hovers over the variable. Select "Window" menu ? "Preferences" ? "Java" ? "Editor" ? "Hover".</li> 
      <li><span class="line-heading">Comparing Two Files:</span> In "Package Explorer", select two files (hold the control key) ? Right-click ? Compare with.</li> 
      <li><span class="line-heading">Useful Eclipse Shortcut Keys:</span> 
       <ul> 
        <li>F3: Goto the declaration of the highlighted variable/method.</li> 
        <li>Ctrl-Shift-G: Search for ALL references of the highlighted variable/method in workspace.</li> 
        <li>Ctrl-G: Search for the Declaration of a variable/method in workspace.<br> Don't use Find (Ctrl-F), but use the above context-sensitive search.</li> 
        <li>Ctrl-Shift-F: Format the source code.</li> 
        <li>Ctrl-Shift-O: Organize imports.</li> 
        <li>Alt-Shift-R: Rename. (Don't use Find/Replace.)</li> 
        <li>Ctrl-Space: auto-complete.</li> 
       </ul></li> 
      <li><span class="line-heading">Package Explorer vs. Navigator:</span> We usually use "Package Explorer" in programming, but it will not show you all the folders and files under the project. On the other hand, "Navigator" is a file manager that shows the exact file structure of the project (similar to Windows Explorer). You can enable the Navigator by "Window" ? Show view ? Navigator.</li> 
      <li><span class="line-heading">Spell Check:</span> To enable spell check, select Window ? Preferences ? type "spell" in the filter ? General ? Editors ? Text Editors ? Spelling ? Check "Enable spell checking". Also provide a "User defined dictionary" (with an initially empty text file).<br> To correct mis-spell words, right-click and press ctrl-1 (or Edit menu ? Quick Fix).</li> 
      <li><span class="line-heading">Eclipse's Log File:</span> Goto Help ? about Eclipse ? Installation details ? Configuration ? View Error Log.</li> 
      <li><span class="line-heading">Viewing two files in split screen:</span> Simply click and hold on the title of one file and drag it to the lower side of the screen. [To view the same file on split screen, create a new editor window by selecting Window ? New Editor; and drag one window to the lower side of the screen.]</li> 
      <li><span class="line-heading">Block Select (Column Select):</span> Push Alt-Shift-A to toggle between block-select mode and normal mode.</li> 
      <li><span class="line-heading">Snippets:</span> 
       <ul> 
        <li>To view the snippet window: choose "Window" ? Show View ? Snippets.</li> 
        <li>To create a new snippet category: Right-click ? Customize ? New.</li> 
        <li>To create a new snippet item: Copy the desired text ? Select the snippet category ? paste as snippet.</li> 
        <li>To insert a snippet: place the cursor on the desired location at the editor panel ? click the snippet item.</li> 
       </ul> </li> 
      <li><span class="line-heading">Word Wrap (Line Wrap):</span> Word-wrap (or line-wrap) is essential for editing long HTML documents without the horizontal scroll bar. However, the Eclipse's HTML Editor and Text Editor do not support word-wrap.<br> You could install a plug-in called "Word Wrap" from http://ahtik.com/eclipse-update/.<br> Choose "Help" ? Install New Software ? in "Work with" Enter "http://ahtik.com/eclipse-update/".<br> To activate word wrap, right-click on the editor panel ? select "Word Wrap".</li> 
      <li><span class="line-heading">Creating "link folder" in project</span>: You do not have to place all the folders under the project base directory, instead, you can use so-called "link folders" to link to folder outside the project base directory.<br> To create a link folder in a project, right-click on the project ? File ? New ? Folder ? Advanced ? Check Link to alternate Location (Linked Folder).</li>

     <h3 id="io"> File I/O in Eclipse</h3> 
     <p>Suppose that your want to write a Java program, which inputs from a text file called "<code>xxxx.in</code>" and outputs to a text file called "<code>xxxx.out</code>". This is a little tricky under Eclipse due to:</p> 
     <ol> 
      <li>When you create a text file in Windows' Notepad and saved it as "<code>xxxx.in</code>", Notepad will append the "<code>.txt</code>" to your file and it becomes "<code>xxxx.in.txt</code>". Worse still, the Windows' Explorer, by default, will not show the "<code>.txt</code>" extension. (The first thing I always do to an alien computer is to change this setting. From "Tools" menu ? Folder Options... ? View ? Uncheck "Hide extensions for known file types".) You need to put a pair of double quotes around <code>xxxx.in</code> to override the default "<code>.txt</code>" extension. This is one good reason not to use Notepad for programming at all. You should use Eclipse to create the text file instead.</li> 
      <li>Which directory to keep the input file "<code>xxxx.in</code>" in Eclipse? 
       <ul> 
        <li>If you did not separate the sources and class files into two separate directories, then the answer is straight forward, because there is only one directory to place your input file.</li> 
        <li>If you choose to keep your sources and class files in two separate directories, eclipse will create two sub-directories "src" and "bin" under the <em>base</em> directory. BUT you need to put your input file "<code>xxxx.in</code>" in the <em>base</em> directory of your project, instead of the "src" or "bin"..</li> 
       </ul> </li> 
     </ol> For writing simple programs:
     <br> 
     <ul> 
      <li>Put the sources, class files, and the input/output files in the same directory. (When you create a new project, select "Use project folder as root for sources and class files" in "Project Layout".) (But put your sources and class files in separate directories for big project.)</li> 
      <li>You can create you input file from eclipse directly via "File" menu ? "New" ? "File".</li> 
      <li>Remember to add a newline to the end of your input file.</li> 
      <li>You may need to right-click the project and select "Refresh" to see the output file "<code>xxxx.out</code>" created in the package explorer.</li> 
      <li>To open the "<code>xxxx.in</code>" and "<code>xxxx.out</code>": right-click ? Open With ? Text Editor.</li> 
     </ul> 
     <p>This is a sample JDK 1.5 program for file input/output:</p> 
     <pre class="color-example">
import java.util.Scanner;
import java.util.Formatter;
import java.io.File;
import java.io.IOException;
   
public class <strong>FileIOTest</strong> {  <span class="color-comment">// saved as "FileIOTest.java"</span>
   public static void main (String [] args) throws IOException {
      Scanner in = new Scanner(new File("FileIOTest.in"));        <span class="color-comment">// file input</span>
      Formatter out = new Formatter(new File("FileIOTest.out"));  <span class="color-comment">// file output</span>
   
      int a = in.nextInt();
      int b = in.nextInt();
      out.format("%d\n",a+b);  <span class="color-comment">// format() has the same syntax as printf()</span>
   
      out.close();    <span class="color-comment">// flush the output and close the output file</span>
   }
}</pre>
