       <h2 id="installing-iis-on-nano-server">Installing IIS on Nano Server</h2> 
       <p>You can install this server role either offline (with the Nano Server off) or online (with the Nano Server running); offline installation is the recommended option. </p> 
       <p>For offline installation, add the package with the -Packages parameter of New-NanoServerImage, as in this example: </p> 
       <p><code>New-NanoServerImage -Edition Standard -DeploymentType Guest -MediaPath f:\ -BasePath .\Base -TargetPath .\Nano1.vhd -ComputerName Nano1 -Package Microsoft-NanoServer-IIS-Package</code> </p> 
       <p>If you have an existing VHD file, you can install IIS offline with DISM.exe by mounting the VHD, and then using the <strong>Add-Package</strong> option.<br>The following example steps assume that you are running from the directory specified by BasePath option, which was created after running New-NanoServerImage. </p> 
       <ol> 
        <li>mkdir mountdir</li> 
        <li>.\Tools\dism.exe /Mount-Image /ImageFile:.\NanoServer.vhd /Index:1 /MountDir:.\mountdir</li> 
        <li>.\Tools\dism.exe /Add-Package /PackagePath:.\packages\Microsoft-NanoServer-IIS-Package.cab /Image:.\mountdir</li> 
        <li>.\Tools\dism.exe /Add-Package /PackagePath:.\packages\en-us\Microsoft-NanoServer-IIS-Package_en-us.cab /Image:.\mountdir</li> 
        <li>.\Tools\dism.exe /Unmount-Image /MountDir:.\MountDir /Commit</li> 
       </ol> 
       <div class="NOTE">
        <h5>Note</h5>
        <p>Note that Step 4 adds the language pack--this example installs EN-US. </p> 
       </div> 
       <p>At this point you can start Nano Server with IIS. </p> 
       <h3 id="installing-iis-on-nano-server-online">Installing IIS on Nano Server online</h3> 
       <p>Though offline installation of the server role is recommended, you might need to install it online (with the Nano Server running) in container scenarios. To do this, follow these steps: </p> 
       <ol> 
        <li><p>Copy the Packages folder from the installation media locally to the running Nano Server (for example, to C:\packages). </p> </li> 
        <li><p>Create a new Unattend.xml file on another computer and then copy it to the Nano Server. You can copy and paste this XML content into the XML file you created: </p> </li> 
       </ol> 
       <pre><code>
    &lt;unattend xmlns="urn:schemas-microsoft-com:unattend"&gt;  
    &lt;servicing&gt;  
        &lt;package action="install"&gt;  
            &lt;assemblyIdentity name="Microsoft-NanoServer-IIS-Package" version="10.0.14393.0" processorArchitecture="amd64" publicKeyToken="31bf3856ad364e35" language="neutral" /&gt;  
            &lt;source location="c:\packages\Microsoft-NanoServer-IIS-Package.cab" /&gt;  
        &lt;/package&gt;  
        &lt;package action="install"&gt;  
            &lt;assemblyIdentity name="Microsoft-NanoServer-IIS-Package" version="10.0.14393.0" processorArchitecture="amd64" publicKeyToken="31bf3856ad364e35" language="en-US" /&gt;  
            &lt;source location="c:\packages\en-us\Microsoft-NanoServer-IIS-Package_en-us.cab" /&gt;  
        &lt;/package&gt;  
    &lt;/servicing&gt;  
    &lt;cpi:offlineImage cpi:source="" xmlns:cpi="urn:schemas-microsoft-com:cpi" /&gt;  
&lt;/unattend&gt;  
</code></pre>
       <ol> 
        <li><p>In the new XML file you created (or copied), edit C:\packages to the directory you copied the content of Packages to. </p> </li> 
        <li><p>Switch to the directory with the newly created XML file and run </p> <p><strong>dism /online /apply-unattend:.\unattend.xml</strong> </p> </li> 
        <li><p>Confirm that the IIS package and its associated language pack is installed correctly by running: </p> <p><strong>dism /online /get-packages</strong> </p> <p>You should see "Package Identity : Microsoft-NanoServer-IIS-Package~31bf3856ad364e35~amd64~~10.0.14393.1000" listed twice, once for Release Type : Language Pack and once for Release Type : Feature Pack. </p> </li> 
        <li><p>Start the W3SVC service either with <strong>net start w3svc</strong> or by restarting the Nano Server. </p> </li> 
       </ol> 
       <h2 id="starting-iis">Starting IIS</h2> 
       <p>Once IIS is installed and running, it is ready to serve web requests. Verify that IIS is running by browsing the default IIS web page at http://&lt;IP address of Nano Server&gt;. On a physical computer, you can determine the IP address by using the Recovery Console. On a virtual machine, you can get the IP address by using a Windows PowerShell prompt and running: </p> 
       <p><code>Get-VM -name &lt;VM name&gt; | Select -ExpandProperty networkadapters | select IPAddresses</code> </p> 
       <p>If you are not able to access the default IIS web page, double-check the IIS installation by looking for the <strong>c:\inetpub</strong> directory on the Nano Server. </p> 
       <h2 id="enabling-and-disabling-iis-features">Enabling and disabling IIS features</h2> 
       <p>A number of IIS features are enabled by default when you install the IIS role (see the table in the "Overview of IIS on Nano Server" section of this topic). You can enable (or disable) additional features using DISM.exe</p> 
       <p>Each feature of IIS exists as a set of configuration elements. For example, the Windows authentication feature comprises these elements: </p> 
       <table> 
        <thead> 
         <tr> 
          <th>Section</th> 
          <th>Configuration elements</th> 
         </tr> 
        </thead> 
        <tbody> 
         <tr> 
          <td><code>&lt;globalModules&gt;</code></td> 
          <td><code>&lt;add name="WindowsAuthenticationModule" image="%windir%\System32\inetsrv\authsspi.dll</code></td> 
         </tr> 
         <tr> 
          <td><code>&lt;modules&gt;</code></td> 
          <td><code>&lt;add name="WindowsAuthenticationModule" lockItem="true" \/&gt;</code></td> 
         </tr> 
         <tr> 
          <td><code>&lt;windowsAuthentication&gt;</code></td> 
          <td><code>&lt;windowsAuthentication enabled="false" authPersistNonNTLM\="true"&gt;&lt;providers&gt;&lt;add value="Negotiate" /&gt;&lt;add value="NTLM" /&gt;&lt;br /&gt;&lt;/providers&gt;&lt;br /&gt;&lt;/windowsAuthentication&gt;</code></td> 
         </tr> 
        </tbody> 
       </table> 
       <p>The full set of IIS sub-features are included in Appendix 1 of this topic and their corresponding configuration elements is included in Appendix 2 of this topic. </p> 
       <h3 id="example-installing-windows-authentication">Example: installing Windows authentication</h3> 
       <ol> 
        <li><p>Open a Windows PowerShell remote session console on the Nano Server. </p> </li> 
        <li><p>Use <code>DISM.exe</code> to install the Windows authentication module:</p> <pre><code>dism /Enable-Feature /online /featurename:IIS-WindowsAuthentication /all
</code></pre><p>The <code>/all</code> switch will install any feature that the chosen feature depends on.</p> </li> 
       </ol> 
       <h3 id="example-uninstalling-windows-authentication">Example: uninstalling Windows authentication</h3> 
       <ol> 
        <li><p>Open a Windows PowerShell remote session console on the Nano Server. </p> </li> 
        <li><p>Use <code>DISM.exe</code> to uninstall the Windows authentication module:</p> <pre><code>dism /Disable-Feature /online /featurename:IIS-WindowsAuthentication
</code></pre></li> 
       </ol> 
       <h2 id="other-common-iis-configuration-tasks">Other common IIS configuration tasks</h2> 
       <p><strong>Creating websites</strong> </p> 
       <p>Use this cmdlet: </p> 
       <p><code>PS D:\&gt; New-IISSite -Name TestSite -BindingInformation "*:80:TestSite" -PhysicalPath c:\test</code> </p> 
       <p>You can then run <code>Get-IISSite</code> to verify the state of the site (returns the web site name, ID, state, physical path, and bindings). </p> 
       <p><strong>Deleting web sites</strong> </p> 
       <p>Run <code>Remove-IISSite -Name TestSite -Confirm:$false</code>. </p> 
       <p><strong>Creating virtual directories</strong> </p> 
       <p>You can create virtual directories by using the IISServerManager object returned by Get-IISServerManager, which exposes the .NET Microsoft.Web.Administration.ServerManager API. In this example, these commands access the "Default Web Site" element of the Sites collection and the root application element ("/") of the Applications section. They then call the Add() method of the VirtualDirectories collection for that application element to create the new directory: </p> 
       <pre><code>PS C:\&gt; $sm = Get-IISServerManager  
PS C:\&gt; $sm.Sites["Default Web Site"].Applications["/"].VirtualDirectories.Add("/DemoVirtualDir1", "c:\test\virtualDirectory1")  
PS C:\&gt; $sm.Sites["Default Web Site"].Applications["/"].VirtualDirectories.Add("/DemoVirtualDir2", "c:\test\virtualDirectory2")  
PS C:\&gt; $sm.CommitChanges()  
</code></pre>
       <p><strong>Creating application pools</strong> </p> 
       <p>Similarly you can use Get-IISServerManager to create application pools: </p> 
       <pre><code>PS C:\&gt; $sm = Get-IISServerManager  
PS C:\&gt; $sm.ApplicationPools.Add("DemoAppPool")  
</code></pre>
       <p><strong>Configuring HTTPS and certificates</strong> </p> 
       <p>Use the Certoc.exe utility to import certificates, as in this example, which shows configuring HTTPS for a website on a Nano Server: </p> 
       <ol> 
        <li><p>On another computer that is not running Nano Server, create a certificate (using your own certificate name and password), and then export it to c:\temp\test.pfx. </p> <p><code>$newCert = New-SelfSignedCertificate -DnsName "www.foo.bar.com" -CertStoreLocation cert:\LocalMachine\my</code> </p> <p><code>$mypwd = ConvertTo-SecureString -String "YOUR_PFX_PASSWD" -Force -AsPlainText</code> </p> <p><code>Export-PfxCertificate -FilePath c:\temp\test.pfx -Cert $newCert -Password $mypwd</code> </p> </li> 
        <li><p>Copy the test.pfx file to the Nano Server computer. </p> </li> 
        <li><p>On the Nano Server, import the certificate to the "My" store with this command: </p> <p><strong>certoc.exe -ImportPFX -p YOUR_PFX_PASSWD My c:\temp\test.pfx</strong> </p> </li> 
        <li><p>Retrieve the thumbprint of this new certificate (in this example, 61E71251294B2A7BB8259C2AC5CF7BA622777E73) with <code>Get-ChildItem Cert:\LocalMachine\my</code>. </p> </li> 
        <li><p>Add the HTTPS binding to the Default Web Site (or whatever website you want to add the binding to) by using these Windows PowerShell commands: </p> <pre><code>$certificate = get-item Cert:\LocalMachine\my\61E71251294B2A7BB8259C2AC5CF7BA622777E73  
# Use your actual thumbprint instead of this example  
$hash = $certificate.GetCertHash()  

Import-Module IISAdministration  
$sm = Get-IISServerManager  
$sm.Sites["Default Web Site"].Bindings.Add("*:443:", $hash, "My", "0")    # My is the certificate store name  
$sm.CommitChanges()  
</code></pre><p>You could also use Server Name Indication (SNI) with a specific host name with this syntax: <code>$sm.Sites["Default Web Site"].Bindings.Add("*:443:www.foo.bar.com", $hash, "My", "Sni".</code> </p> </li> 
       </ol>
