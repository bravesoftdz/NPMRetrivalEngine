       <h1 class="title">Packages and Programs in Configuration Manager</h1> 
       <div id="mainSection"> 
        <div id="mainBody">
         &nbsp;
         <p>Updated: May 14, 2015</p>
         <p>Applies To: System Center 2012 Configuration Manager, System Center 2012 Configuration Manager SP1, System Center 2012 Configuration Manager SP2, System Center 2012 R2 Configuration Manager, System Center 2012 R2 Configuration Manager SP1</p>
         <div class="introduction">
          <p>Microsoft&nbsp;System&nbsp;Center&nbsp;2012 Configuration&nbsp;Manager continues to support packages and programs that were used in Configuration Manager 2007. A deployment that uses packages and programs might be more suitable than a deployment that uses an application when you deploy any of the following:</p>
          <ul class="unordered">
           <li><p>Scripts that do not install an application on a computer, such as a script to defragment the computer disk drive.</p></li>
           <li><p>One-off scripts that do not need to be continually monitored.</p></li>
           <li><p>Scripts that run on a recurring schedule and cannot use global evaluation.</p></li>
          </ul>
          <p>When you migrate a Configuration Manager 2007 site to a Configuration Manager hierarchy, you can migrate existing packages and deploy them in your Configuration Manager hierarchy. After migration is complete, your Configuration Manager 2007 packages appear in the <strong>Packages</strong> node in the <strong>Software Library</strong> workspace. You can modify and deploy these packages in the same way as you did by using Configuration Manager 2007 software distribution. The Import Package from Definition Wizard remains in Configuration Manager to import legacy packages. Advertisements are converted to deployments when they are migrated from Configuration Manager 2007 to a Configuration Manager hierarchy.</p>
          <div class="alert">
           <table summary="table">
            <tbody>
             <tr>
              <th align="left" scope="col"><span><img id="s-e6f6a65cf14f462597b64ac058dbe1d0-system-media-system-caps-note" alt="System_CAPS_note" src="https://i-technet.sec.s-msft.com/dynimg/IC101471.jpeg" title="System_CAPS_note" xmlns=""></span><span class="alertTitle">Note </span></th>
             </tr>
             <tr>
              <td><p>You can use Microsoft System Center Configuration Manager Package Conversion Manager to convert packages and programs into Configuration Manager applications. </p><p>Download Package Conversion Manager from the <a href="http://go.microsoft.com/fwlink/p/?LinkId=212950">Microsoft Download Center</a>.</p><p>For more information, see <a href="https://technet.microsoft.com/en-us/library/hh531519.aspx">Configuration Manager Package Conversion Manager</a>.</p></td>
             </tr>
            </tbody>
           </table>
          </div>
          <p>Packages can use some new features of Configuration Manager, including distribution point groups and the new monitoring functionality. Microsoft Application Virtualization applications can no longer be distributed by using packages and programs in Configuration Manager. To distribute virtual applications, you must create these as Configuration Manager applications.</p>
          <div class="alert">
           <table summary="table">
            <tbody>
             <tr>
              <th align="left" scope="col"><span><img id="s-e6f6a65cf14f462597b64ac058dbe1d0-system-media-system-caps-note" alt="System_CAPS_note" src="https://i-technet.sec.s-msft.com/dynimg/IC101471.jpeg" title="System_CAPS_note" xmlns=""></span><span class="alertTitle">Note </span></th>
             </tr>
             <tr>
              <td><p>To successfully create virtual applications in System&nbsp;Center&nbsp;2012 Configuration&nbsp;Manager, 64-bit client computers must have the App-V 4.6 or later client installed before the Configuration Manager client is upgraded.</p></td>
             </tr>
            </tbody>
           </table>
          </div>

           <table summary="table">

             </tr>
             <tr>
              <td><p>For System&nbsp;Center&nbsp;2012 Configuration&nbsp;Manager&nbsp;SP1 and later:</p><p>For information about how to deploy packages and programs to clients that run Linux and UNIX, see <a href="https://technet.microsoft.com/en-us/library/jj573943.aspx">Deploying Software to Linux and UNIX Servers in Configuration Manager</a>.</p></td>
