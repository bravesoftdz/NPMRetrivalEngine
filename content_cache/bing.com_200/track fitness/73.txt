   <h1 id="firstHeading" class="firstHeading" lang="es">Paquete Java</h1>

    <div id="mw-content-text" lang="es" dir="ltr" class="mw-content-ltr">
     <div class="mw-parser-output">
      <p>Un <b>Paquete</b> en Java es un contenedor de <a href="/wiki/Clase_(inform%C3%A1tica)" title="Clase (informática)">clases</a> que permite agrupar las distintas partes de un programa y que por lo general tiene una funcionalidad y elementos comunes, definiendo la ubicación de dichas <a href="/wiki/Clase_(inform%C3%A1tica)" title="Clase (informática)">clases</a> en un directorio de estructura jerárquica.</p> 
      <p></p>

      <p></p> 
      <h2><span class="mw-headline" id="Ventajas">Ventajas</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Paquete_Java&amp;action=edit&amp;section=1" title="Editar sección: Ventajas">editar</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>El uso de paquetes proporciona las siguientes ventajas:</p> 
      <ul> 
       <li>Agrupamiento de clases con características comunes.</li> 
       <li>Reutilización de código al promover principios de <a href="/wiki/Programaci%C3%B3n_orientada_a_objetos" title="Programación orientada a objetos">programación orientada a objetos</a> como la <a href="/wiki/Encapsulaci%C3%B3n_(programaci%C3%B3n_orientada_a_objetos)" class="mw-redirect" title="Encapsulación (programación orientada a objetos)">encapsulación</a> y <a href="/wiki/Modularidad_(inform%C3%A1tica)" title="Modularidad (informática)">modularidad</a>.</li> 
       <li>Mayor seguridad al existir niveles de acceso.</li> 
       <li>Evita la colisión de clases que tengan el mismo nombre. Pueden existir clases con el mismo nombre siempre y cuando su <b>fully qualified class name</b> sean únicos.</li> 
       <li>Mantenibilidad de código. Si un paquete se enfoca en la agrupación de clases con características comunes, el cambio en la funcionalidad se limita a las clases contenidas en dicho paquete, además, si es un paquete grande soporta la reusabilidad, si por el contrario es pequeño soporta su mantenibilidad.</li> 
      </ul> 
      <h2><span class="mw-headline" id="Contenido_de_un_paquete">Contenido de un paquete</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Paquete_Java&amp;action=edit&amp;section=2" title="Editar sección: Contenido de un paquete">editar</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Un paquete puede contener:</p> 
      <ul> 
       <li><a href="/wiki/Clase_(inform%C3%A1tica)" title="Clase (informática)">Clases</a></li> 
       <li><a href="/wiki/Interfaz_(Java)" title="Interfaz (Java)">Interfaces</a></li> 
       <li>Tipos Enumerados</li> 
       <li><a href="/wiki/Anotaci%C3%B3n_Java" title="Anotación Java">Anotaciones</a></li> 
      </ul> 
      <h2><span class="mw-headline" id="Atributos_de_un_paquete_Java">Atributos de un paquete Java</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Paquete_Java&amp;action=edit&amp;section=3" title="Editar sección: Atributos de un paquete Java">editar</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Un paquete Java tiene los siguientes atributos:</p> 
      <ul> 
       <li>Son opcionales. Si no se define un paquete para un fichero de código Java se definirá un paquete llamado "<i>default" automáticamente</i>.</li> 
       <li>Es definido un paquete por cada ficheros de código Java.</li> 
       <li>Los nombres de paquete que comienzan con el nombre <code>java.*</code> y <code>javax.*</code> son reservados.</li> 
       <li>El nombre del paquete equivale a una estructura de ficheros. El nombre de paquete <code>com.dominio_empresa.utilidades</code> debería ser igual al directorio <code>[ruta_directorio]\com\dominio_empresa\utilidades</code>. Si una clase incluida en un paquete no se relaciona con su respectiva estructura de directorio, la clase no podrá usarse.</li> 
      </ul> 
      <h2><span id="Convenciones_en_la_definición_de_paquetes_Java"></span><span class="mw-headline" id="Convenciones_en_la_definici.C3.B3n_de_paquetes_Java">Convenciones en la definición de paquetes Java</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Paquete_Java&amp;action=edit&amp;section=4" title="Editar sección: Convenciones en la definición de paquetes Java">editar</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Los siguientes son convenciones o estándares acordados en la definición de paquetes en Java:</p> 
      <ul> 
       <li>El nombre del paquete se define de manera inversa al dominio de la organización o grupo. Por ejemplo, <code>dominioempresa.com</code> puede ser usado como nombre de paquete así: <code>com.dominio_empresa.utilidades</code>.</li> 
       <li>El nombre del paquete debería definirse en minúscula. Si existen varias palabras en el nombre, se pueden separar con guión bajo (_).</li> 
      </ul> 
      <h2><span class="mw-headline" id="Uso_de_paquetes">Uso de paquetes</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Paquete_Java&amp;action=edit&amp;section=5" title="Editar sección: Uso de paquetes">editar</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>En los ficheros de código Java se usa la palabra reservada <code>package</code> para especificar a qué paquete pertenecen. Suele indicarse como primera sentencia:</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
<span class="kn">package</span> <span class="nn">java.awt.event</span><span class="o">;</span>
</pre>
      </div> 
      <p>Para usar un paquete dentro del código se usa la declaración <code>import</code>. Si sólo se indica el nombre del paquete, se importarán todas las clases que contiene:</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
<span class="kn">import</span> <span class="nn">java.awt.event.*</span><span class="o">;</span>
</pre>
      </div> 
      <p>Si además del nombre del paquete se especifica una clase, sólo se importa esa clase:</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
<span class="kn">import</span> <span class="nn">java.awt.event.ActionEvent</span><span class="o">;</span>
</pre>
      </div> 
      <p>Después de añadir alguna de las sentencias anteriores, se puede hacer referencia a la clase <code>ActionEvent</code> usando su nombre:</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
<span class="n">ActionEvent</span> <span class="n">myEvent</span> <span class="o">=</span> <span class="k">new</span> <span class="n">ActionEvent</span><span class="o">();</span>
</pre>
      </div> 
      <p>Si no se hubiera importado la clase o el paquete, cada vez que tuviéramos que usarla habría que especificarla por su <b>fully qualified class name,</b> que no es más que el nombre del paquete seguido por el nombre de la clase:</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
<span class="n">java</span><span class="o">.</span><span class="na">awt</span><span class="o">.</span><span class="na">event</span><span class="o">.</span><span class="na">ActionEvent</span> <span class="n">myEvent</span> <span class="o">=</span> <span class="k">new</span> <span class="n">java</span><span class="o">.</span><span class="na">awt</span><span class="o">.</span><span class="na">event</span><span class="o">.</span><span class="na">ActionEvent</span><span class="o">();</span>
</pre>
      </div> 
      <p>Si lo que se desea es importar todos los miembros estáticos de una clase, note la sentencia <code>static</code> despúes de <code>import</code>. (A partir de J2SE 5.0 en adelante)</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
<span class="kn">import static</span> <span class="nn">java.awt.Color.*</span><span class="o">;</span>
</pre>
      </div> 
      <p>Si lo que se desea es importar un sólo miembro estático de una clase. (A partir de J2SE 5.0 en adelante)</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
<span class="kn">import static</span> <span class="nn">java.awt.color.ColorSpace.CS_GRAY*</span><span class="o">;</span>
</pre>
      </div> 
      <h2><span class="mw-headline" id="Paquetes_importantes_de_Java">Paquetes importantes de Java</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Paquete_Java&amp;action=edit&amp;section=6" title="Editar sección: Paquetes importantes de Java">editar</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Estos son los paquetes más importantes de la API de Java:</p> 
      <table class="wikitable"> 
       <tbody>
        <tr> 
         <th>Paquete</th> 
         <th>Descripción</th> 
        </tr> 
        <tr> 
         <td><code>java.applet</code></td> 
         <td>Contiene clases para la creación de <a href="/wiki/Applet" title="Applet">applets</a>.</td> 
        </tr> 
        <tr> 
         <td><code>java.awt</code></td> 
         <td>Contiene clases para crear interfaces de usuario con ventanas.</td> 
        </tr> 
        <tr> 
         <td><code>java.io</code></td> 
         <td>Contiene clases para manejar la entrada/salida.</td> 
        </tr> 
        <tr> 
         <td><code>java.lang</code></td> 
         <td>Contiene clases variadas pero imprescindibles para el lenguaje, como <i>Object</i>, <i>Thread</i>, <i>Math</i>... <p>El paquete <code>java.lang</code> es importado por defecto en el ficheros de código Java.</p> </td> 
        </tr> 
        <tr> 
         <td><code>java.net</code></td> 
         <td>Contiene clases para soportar aplicaciones que acceden a redes <a href="/wiki/TCP/IP" class="mw-redirect" title="TCP/IP">TCP/IP</a>.</td> 
        </tr> 
        <tr> 
         <td><code>java.util</code></td> 
         <td>Contiene clases que permiten el acceso a recursos del sistema, etc.</td> 
        </tr> 
        <tr> 
         <td><code>javax.swing</code></td> 
         <td>Contiene clases para crear interfaces de usuario mejorando la <a href="/wiki/AWT" class="mw-redirect" title="AWT">AWT</a>.</td> 
        </tr> 
       </tbody>
      </table>
