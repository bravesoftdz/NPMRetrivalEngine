      <section id="api-documentation" class="api__section"> 
       <h1 class="api__title">Tracking API</h1> 
       <h2 class="anchored">Introduction</h2> 
       <p>The tracking API provides the opportunity to track shipments by reference, package or shipment number.<br> It is an easy way to display details and the status of shipments.<br> The information available in this API is the same information available from the <a href="http://tracking.bring.com/">Tracking web site</a>.<br> Response is available as either XML or JSON.</p> 
       <p><em>Oh by the way</em>, we also have a <a href="https://github.com/bring/bring-api-java">Java client library</a> for the Tracking API.</p> 
       <h2 class="anchored">Versioning</h2> 
       <p>We encourage you to use the latest version of our API all time. Of course, we keep the previous version for some time so that you will get enough time to convert your application. This API supports versioning by two means.</p> 
       <ul> 
        <li> <p><strong>Setting requst header with version number.</strong><br> Then you use the normal URL but add Request Header.</p> 
         <table> 
          <thead> 
           <tr> 
            <th style="text-align: left">Header</th> 
            <th style="text-align: left">Example</th> 
           </tr> 
          </thead> 
          <tbody> 
           <tr> 
            <td style="text-align: left"><code class="highlighter-rouge">api-version</code></td> 
            <td style="text-align: left"><code class="highlighter-rouge">2</code></td> 
           </tr> 
          </tbody> 
         </table> </li> 
        <li> <p><strong>Setting version number in the url.</strong><br> Then you have to update the url by adding <em>v2</em><br> example for open tracking:<br> https://tracking.bring.com/api/v2/tracking.json?q=TESTPACKAGE-AT-PICKUPPOINT <br> example for logged in tracking:<br> https://mybring.com/tracking/api/v2/tracking.json?q=TESTPACKAGE-AT-PICKUPPOINT</p> <p>You can verify that you are doing it correctly by checking that the response should contain the element <em>“apiVersion”: “2”</em>.</p> </li> 
       </ul> 
       <h2 class="anchored">Authentication</h2> 
       <p>If you are a Mybring user, you can authenticate requests to the Tracking API. Authenticated requests have the following benefits:</p> 
       <ul> 
        <li>You get more information about your parcels, for example prices, name, address and signatures for proof of develiery</li> 
        <li>Less strict rate limiting</li> 
       </ul> 
       <p>See the <a href="/api/#authentication">getting started guide on authentication</a> if you’re not sure what this means.</p>

       <h1 class="large anchored" id="tracking-endpoint">Tracking endpoint</h1> 
       <p>For testing you can use “TESTPACKAGE-AT-PICKUPPOINT” as the tracking number to generate test results. For real tracking this must be a valid number.</p> 
       <p>You can find an XSD for the XML response here: <a href="https://tracking.bring.com/api/schema/sporing-v2.xsd">tracking-v2.xsd</a>.</p> 
       <p>A parcel has a number of events connected to it during its travel. Here are a list of current event statuses that can be returned.</p> 
       <ul> 
        <li>ARRIVED_DELIVERY<br> Arrived Delivery point - This is when a container arrives at recipients address, and can start unloading.?</li> 
        <li>CUSTOMS?<br> Package is in customs clearance.</li> 
        <li>COLLECTED?<br> The parcel has been collected at pickup address.</li> 
        <li>DELIVERED?<br> Package has been delivered.</li> 
        <li>DELIVERED_SENDER<br> Package has been returned to the sender</li> 
        <li>DELIVERY_CANCELLED<br> ?Home delivery has been canceled by the customer.</li> 
        <li>DELIVERY_CHANGED?<br> Date for Home delivery has been changed by customer.</li> 
        <li>DELIVERY_ORDERED?<br> Home delivery has been ordered.</li> 
        <li>DEVIATION?<br> Deviation in production. Something wrong has happened and there is a probability for delay.</li> 
        <li>HANDED_IN?<br> Package has been handed in to Bring.</li> 
        <li>INTERNATIONAL?<br> Package has been sent from origin country or arrived at destination country.?</li> 
        <li>IN_TRANSIT?<br> Package is in transit.</li> 
        <li>NOTIFICATION_SENT?<br> Notification for this package has been sent by sms, push and/or mail. This can be informational notifications and action notification like pickup notice.</li> 
        <li>PRE_NOTIFIED?<br> EDI message for the package has been received by Bring.</li> 
        <li>READY_FOR_PICKUP?<br> Package has arrived at pickup point.</li> 
        <li>RETURN?<br> The package has been returned to sender.?</li> 
        <li>TRANSPORT_TO_RECIPIENT?<br> Package has been loaded for delivery to the recipient.</li> 
       </ul>
