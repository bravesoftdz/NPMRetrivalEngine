      <h2 class="page-header">Overview</h2> 
      <p class="lead"><strong>Jenetics</strong> is designed with a clear separation of the several concepts of the algorithm, e.g. <em>Gene</em>, <em>Chromosome</em>, <em>Genotype</em>, <em>Phenotype</em>, <em>Population</em> and fitness <em>Function</em>. Jenetics allows you to minimize and maximize the given fitness function without tweaking it. In contrast to other GA implementations, the library uses the concept of an evolution stream (<a href="javadoc/jenetics/3.9/org/jenetics/engine/EvolutionStream.html"><code>EvolutionStream</code></a>) for executing the evolution steps. Since the <a href="javadoc/jenetics/3.9/org/jenetics/engine/EvolutionStream.html"><code>EvolutionStream</code></a> implements the Java <a href="http://docs.oracle.com/javase/8/docs/api/java/util/stream/Stream.html"><code>Stream</code></a> interface, it works smoothly with the rest of the Java Stream API. </p> 
      <h3>Features</h3> 
      <div class="row"> 
       <div class="item col-md-4 col-sm-6 col-xs-12"> 
        <div class="content"> 
         <h4><i class="fa fa-check-circle"></i> Genetic programming (new)</h4> 
         <p>Since version <a href="https://github.com/jenetics/jenetics/releases/tag/v3.9.0"><i>3.9.0</i></a> the <a href="https://github.com/jenetics/jenetics/tree/master/jenetics.prog"><i>jenetics.prog</i></a> module allows to do <a href="https://en.wikipedia.org/wiki/Genetic_programming"><i>Genetic programming</i></a> with the Jenetics library.</p> 
        </div> 
       </div> 
       <div class="item col-md-4 col-sm-6 col-xs-12"> 
        <div class="content"> 
         <h4><i class="fa fa-check-circle"></i> Frictionless minimization</h4> 
         <p>No need to tweak the fitness function for minimization problems. Just change the configuration of the evolution <a href="javadoc/jenetics/3.9/org/jenetics/engine/Engine.html"><code>Engine</code></a>.</p> 
        </div> 
       </div> 
       <div class="item col-md-4 col-sm-6 col-xs-12"> 
        <div class="content"> 
         <h4><i class="fa fa-check-circle"></i> Plugable PRNG</h4> 
         <p>Easy changeable PRNG (<a href="javadoc/jenetics/3.9/org/jenetics/util/RandomRegistry.html"><code>RandomRegistry</code></a>). No <i>special</i> PRNG interface; the standard Java <a href="https://docs.oracle.com/javase/8/docs/api/java/util/Random.html"><code>Random</code></a> engine allows the use of existing random generators.</p> 
        </div> 
       </div> 
       <div class="item col-md-4 col-sm-6 col-xs-12"> 
        <div class="content"> 
         <h4><i class="fa fa-check-circle"></i> Parallel processing support</h4> 
         <p>Processing of the evolutionary steps can be executed in parallel!</p> 
        </div> 
       </div> 
      </div> 
      <div class="row"> 
       <div class="item col-md-4 col-sm-6 col-xs-12"> 
        <div class="content"> 
         <h4><i class="fa fa-check-circle"></i> Dependency free</h4> 
         <p>No runtime dependencies to Third Party libraries! No mismatch and class loading problems with other libraries.</p> 
        </div> 
       </div> 
       <div class="item col-md-4 col-sm-6 col-xs-12"> 
        <div class="content"> 
         <h4><i class="fa fa-check-circle"></i> Java 8 ready</h4> 
         <p>Fully support of Java 8 Streams and Lambdas!</p> 
        </div> 
       </div> 
       <div class="item col-md-4 col-sm-6 col-xs-12"> 
        <div class="content"> 
         <h4><i class="fa fa-check-circle"></i> Fully documented</h4> 
         <p>Complete <a href="javadoc/jenetics/3.9/index.html">Documentation</a> and <a href="manual/manual-3.9.0.pdf">User guide</a> available, including <a href="manual/html/internals.html">documentation</a> of selected implementation details.</p> 
        </div> 
       </div>

      <h2>Examples</h2> 
      <h3>Hello World (Ones counting)</h3> 
      <p> The minimum evolution Engine setup needs a genotype factory, <code>Factory&lt;Genotype&lt;?&gt;&gt;</code>, and a fitness <em>Function</em>. The <code>Genotype</code> implements the <code>Factory</code> interface and can therefore be used as prototype for creating the initial <em>Population</em> and for creating new random <em>Genotypes</em>. </p> 
      <pre class="prettyprint linenums">
import org.jenetics.BitChromosome;
import org.jenetics.BitGene;
import org.jenetics.Genotype;
import org.jenetics.engine.Engine;
import org.jenetics.engine.EvolutionResult;
import org.jenetics.util.Factory;

public class HelloWorld {
    // 2.) Definition of the fitness function.
    private static int eval(Genotype&lt;BitGene&gt; gt) {
        return gt.getChromosome()
            .as(BitChromosome.class)
            .bitCount();
    }

    public static void main(String[] args) {
        // 1.) Define the genotype (factory) suitable
        //     for the problem.
        Factory&lt;Genotype&lt;BitGene&gt;&gt; gtf =
            Genotype.of(BitChromosome.of(10, 0.5));

        // 3.) Create the execution environment.
        Engine&lt;BitGene, Integer&gt; engine = Engine
            .builder(HelloWorld::eval, gtf)
            .build();

        // 4.) Start the execution (evolution) and
        //     collect the result.
        Genotype&lt;BitGene&gt; result = engine.stream()
            .limit(100)
            .collect(EvolutionResult.toBestGenotype());

        System.out.println("Hello World:\n" + result);
    }
}</pre> In contrast to other GA implementations, the library uses the concept of an evolution stream (
      <code>EvolutionStream</code>) for executing the evolution steps. Since the 
      <code>EvolutionStream</code> implements the Java Stream interface, it works smoothly with the rest of the Java streaming API. Now let's have a closer look at listing above and discuss this simple program step by step: 
      <ol> 
       <li>The probably most challenging part, when setting up a new evolution <code>Engine</code>, is to transform the problem domain into a appropriate <code>Genotype</code> (factory) representation. In our example we want to count the number of ones of a <code>BitChromosome</code>. Since we are counting only the ones of one chromosome, we are adding only one <code>BitChromosome</code> to our <code>Genotype</code>. In general, the <code>Genotype</code> can be created with 1 to n chromosomes.</li> 
       <li> Once this is done, the fitness function which should be maximized, can be defined. Utilizing the new language features introduced in Java 8, we simply write a private static method, which takes the genotype we defined and calculate it's fitness value. If we want to use the optimized bit-counting method, <code>bitCount()</code>, we have to cast the <code>Chromosome&lt;BitGene&gt;</code> class to the actual used <code>BitChromosome</code> class. Since we know for sure that we created the Genotype with a <code>BitChromosome</code>, this can be done safely. A reference to the eval method is then used as fitness function and passed to the <code>Engine.build</code> method.</li> 
       <li> In the third step we are creating the evolution <code>Engine</code>, which is responsible for changing, respectively evolving, a given population. The <code>Engine</code> is highly configurable and takes parameters for controlling the evolutionary and the computational environment. For changing the evolutionary behavior, you can set different alterers and selectors. By changing the used <code>Executor</code> service, you control the number of threads, the Engine is allowed to use. An new <code>Engine</code> instance can only be created via its builder, which is created by calling the <code>Engine.builder</code> method.</li> 
       <li>In the last step, we can create a new <code>EvolutionStream</code> from our <code>Engine</code>. The <code>EvolutionStream</code> is the model or view of the evolutionary process. It serves as a »process handle« and also allows you, among other things, to control the termination of the evolution. In our example, we simply truncate the stream after 100 generations. If you don't limit the stream, the <code>EvolutionStream</code> will not terminate and run forever. Since the <code>EvolutionStream</code> extends the <code>java.util.stream.Stream</code> interface, it integrates smoothly with the rest of the Java Stream API. The final result, the best <code>Genotype</code> in our example, is then collected with one of the predefined collectors of the <code>EvolutionResult</code> class.</li> 
      </ol> 
      <h3>Evolving images</h3> 
      <p>This example tries to approximate a given image by semitransparent polygons. It comes with an Swing UI, where you can immediately start your own experiments.</p>

      <p>For a detailed description on how to execute this example, have a look at the <a href="https://github.com/jenetics/jenetics#evolving-images">Github</a> project page.</p> 
      <h3>Symbolic regression (GP)</h3> 
      <p><em>Symbolic regression involves finding a mathematical expression, in symbolic form, that provides a good, best, or perfect fit between a given finite sampling of values of the independent variables and the associated values of the dependent variables. --- John R. Koza</em></p> 
      <p>Symbolic regression is a classical example in genetic programming and tries to find a mathematical expression for a given set of values. The <a href="https://github.com/jenetics/jenetics/blob/master/jenetics.prog/README.adoc">example</a> shows how to solve the GP problem with Jenetics. We are trying to find the polynomial, 4x<sup>3</sup> - 3x<sup>2</sup> + x, which fits a given data set. The sample data where created with the polynomial we are searching for. This makes it easy to check the quality of the approximation found by the GP.</p> 
      <h2>Extension modules</h2> 
      <ul> 
       <li><a href="https://github.com/jenetics/jenetics/tree/master/jenetics.ext"><b>jenetics.ext</b> (</a><a href="javadoc/jenetics.ext/3.9/index.html">javadoc</a>): Additional, non-standard, genetic operators and chromosomes. Containing classes for the <a href="https://en.wikipedia.org/wiki/Weasel_program">Weasel program</a> and tree data structures.</li> 
       <li><a href="https://github.com/jenetics/jenetics/tree/master/jenetics.prog"><b>jenetics.prog</b> (</a><a href="javadoc/jenetics.prog/3.9/index.html">javadoc</a>):The modules contains classes which allows to do genetic programming (GP). It seamlessly works with the existing <code>EvolutionStream</code> and evolution <code>Engine</code>.</li> 
       <li><a href="https://github.com/jenetics/jenetics/tree/master/jenetics.xml"><b>jenetics.xml</b> (</a><a href="javadoc/jenetics.xml/3.9/index.html">javadoc</a>): XML marshalling module for the Jenetics base data structures.</li> 
      </ul> 
      <h2>Additional libraries</h2> 
      <p>This libraries doesn't have any dependency to Jenetics and can be used as they are.</p> 
      <ul> 
       <li><a href="https://github.com/jenetics/prngine"><b>PRNGine</b></a>: A pseudo-random number generator library for sequential and parallel <a href="https://de.wikipedia.org/wiki/Monte-Carlo-Simulation">Monte Carlo simulations</a>. It has been designed to work smoothly with the <em>Jenetics</em> library, but it has no dependency to it. All PRNG implementations of this library extends the Java <a href="http://docs.oracle.com/javase/8/docs/api/java/util/Random.html">Random</a> class, which makes it easily usable in other projects.</li> 
       <li><a href="https://github.com/jenetics/jpx"><b>JPX</b>:</a> A Java library for creating, reading and writing <a href="https://en.wikipedia.org/wiki/Global_Positioning_System">GPS</a> data in <a href="https://en.wikipedia.org/wiki/GPS_Exchange_Format">GPX</a> format. It is a <b>full</b> implementation of version <a href="http://www.topografix.com/GPX/1/1/">1.1</a> of the GPX format. The data classes are completely immutable and allows a functional programming style. They are working also nicely with the Java 8 <a href="http://docs.oracle.com/javase/8/docs/api/java/util/stream/Stream.html">Stream</a> API. <em>Since it is also possible to calculate the distance between way-points, it is a good fit for the TSP.</em></li> 
      </ul>
