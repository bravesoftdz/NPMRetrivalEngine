   <h2>FitNesse is <i>the</i> acceptance test wiki.</h2> 
   <h3>FitNesse is an easy to use wiki web server</h3> The wiki syntax in FitNesse is minimalistic and helps you concentrate on the content. It has a rich-text editor for those not really into the wiki markup. The rich-text editor has extra features to support table creation and modification, something you'll love when creating test tables in FitNesse.
   <br>
   <br>
   <h3>Easy to set up: just download the application (a Java jar file) and start it.</h3> FitNesse comes as an all-on-one package. Starting FitNesse is as simple as starting the Java jar file. Wiki pages are stored as files, so there is no hassle with databases. FitNesse can be run on your machine or on a server. The 
   <a href="http://fitnesse.org">http://fitnesse.org</a> website is running FitNesse.
   <br>
   <br>
   <h3>Specifications/requirements can be used as test input</h3> Since FitNesse is a wiki, it's not in the way when you reason about requirements for your (software) project. FitNesse allows you to validate those requirements with the actual software implementation. If your project is written in Java, it's a piece of cake. However FitNesse plays well with&nbsp;
   <a href="PlugIns">every language and environment</a>.
   <br>
   <br>
   <h3>The major languages are supported</h3> Yes! Whether your project is written in Java, Python or C#, FitNesse can be hooked in and you can automate the test process. See the 
   <a href="PlugIns">PlugIns</a> page for impementations.
