           <h1></h1>
           <p><em>by César Hernández</em></p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p><strong> Learn how to take advantage of Maven to automate the development lifecycle and the management of Java dependencies. </strong></p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>Maven is a powerful build automation tool mature enough to be nowadays the de facto tool for dependency management in Java projects. It is also a crucial part of the principle of continuous integration in the Java ecosystem because of its automation, build, package, and distribution mechanisms and capabilities.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>One of the key components of the Maven tools is repositories—places where you can obtain and share your dependencies (artifacts). These repositories allow to developers to access thousands of tools, libraries, frameworks, and so on from the open source community and from leaders of the IT industry. In early 2015, Oracle launched the Oracle Maven Repository to facilitate development work inside and outside the Oracle Fusion Middleware ecosystem.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>This article aims to help you build a solid foundation about how Maven works and how you can take advantage of this powerful tool to manage your Java dependencies in an automated manner.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <h2>Dependency Management</h2>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>As you develop applications that require Java APIs that extend the base functionality of the language, you might have the need to import libraries (commonly, as files that have the <code>.jar</code> extension) developed by third parties. It is at this point that your applications start to have dependencies.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>The process of adding, updating, deleting, and distribute such dependencies among the members of a development team is complex and highly error-prone. A first approach that might be intended to reduce complexity for such dependencies is using versioning in a repository such as Apache Subversion or GIT. Although this approach is practical for the source code, it is not practical for dependencies, because it requires additional manipulations, such as the resolution of transitive dependencies that are produced when a dependency you import into your project needs one or more other libraries.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>Through standardization and automation, and by following the principle of convention over configuration, Maven provides mechanisms that help resolve conflicts that arise when there are transitive dependencies that have different version numbers.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <h2>Standardizing Maven-Based Projects</h2>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>Maven is based on the concept of a project object model (POM), which allows the objects (dependencies, project metadata, and plugins) to be explicitly declared to standardize the construction and packaging of Java projects that together form an application or system. The Maven configuration file used for this purpose is named <code>pom.xml</code>.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>For practical purposes, the basic structure of a <code>pom.xml</code> file can divided into three sections:</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>1. General information about the project. This section contains information that is unique to the artifact that the project generates. The following are the mandatory fields:</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <ul>
            <li><code>groupId</code>: Identifies the group to which the project belongs</li>
            <li><code>id</code>: Specifies the project name</li>
            <li><code>packaging</code>: Specifies the package type to be produced by Maven (JAR or WAR)</li>
           </ul>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>2. Dependencies section. This section lists the dependencies that the project uses. At this point, you can even indicate the scope of each dependency. For example, you can indicate that the JUnit dependency is going to be taken into count only for unit testing purposes but not will be included as part of the project's final packaged artifact.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>3. Plugins section. This section contains the declaration of plugins that extend the basic functionality of Maven. This feature has become very popular because Maven officially provides support to over 50 plugins, and the list of plugins developed by third parties is long. There are plugins for packaging, reporting, static code analysis, integration testing, and so on.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <h2>Installing Maven</h2>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>The binary distribution is available from the <a class="jive-link-external-small" href="http://maven.apache.org/download.cgi" rel="nofollow">official site</a>. After it is downloaded, it must be decompressed. Then you must add an environment variable in your operating system point to the <code>bin</code> folder inside the folder structure created by the decompression process.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>To check the installation, run the following command at the command line:</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <pre>mvn -version</pre>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>Figure 1 shows the message that is displayed to indicate a successful installation:</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p><span style="color: red;"><a href="https://community.oracle.com/servlet/JiveServlet/showImage/102-995861-3-170733/f1.png"><img alt="f1.png" class="image-1 jive-image" height="109" src="https://community.oracle.com/servlet/JiveServlet/downloadImage/102-995861-3-170733/f1.png" style="height: 109px; width: 620px;" width="622"></a></span></p>
           <p><strong>Figure 1. Output indicating a successful installation</strong></p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <h2>Maven Command Syntax</h2>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>Here is the syntax:</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p><code>mvn [<em>options</em>] [<em>goal(s)</em>] [<em>phase(s</em>)]</code></p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>where:</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <ul>
            <li><code>mvn</code>: This is the command you use to run Maven.</li>
            <li><code>[<em>goal(s)</em>]</code>: A goal represents a Maven plugin that you can add to a stage to specify or extend the behavior of that particular phase.</li>
            <li><code>[<em>phase(s)</em>]</code>: A phase indicates a specific phase executed for the construction of the project.</li>
           </ul>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>To see the entire list of goals and phases, see the <a class="jive-link-external-small" href="https://maven.apache.org/guides/introduction/introduction-to-the-lifecycle.html" rel="nofollow">official documentation</a>.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <h2>Creating a Java Project with Maven</h2>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>To create a Java project with Maven, you must indicate the type of action you want to execute (<code><em>goal</em></code>), identify unequivocally your project (<code>groupId</code> and <code>artifacId</code>), and select the initial template from which you want your project be created (<code>archetypeArtifacId</code>).</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>In the following example, for practical purposes, Maven's console interaction mode (<code>interactiveMode</code>), which can be used to provide additional information, is disabled:</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <pre>mvn archetype:generate -DgroupId=com.yourorganization -DartifactId=my-application1 -DarchetypeArtifacId=maven-archetype-quickstart -DinteractiveMode=false</pre>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>Figure 2 shows the standard Maven structure provided by the <code>maven-archetype-quickstart</code> archetype:</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p><span style="color: red;"><a href="https://community.oracle.com/servlet/JiveServlet/showImage/102-995861-3-170734/f2.png"><img alt="f2.png" class="image-2 jive-image" height="236" src="https://community.oracle.com/servlet/JiveServlet/downloadImage/102-995861-3-170734/f2.png" style="height: auto; width: auto;" width="288"></a></span></p>
           <p><strong>Figure 2. Standard Maven structure</strong></p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>Within the obtained directory structure, notice that Maven also creates classes and Java packages as well as the <code>pom.xml</code> file (see Figure 3).</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p><span style="color: red;"><a href="https://community.oracle.com/servlet/JiveServlet/showImage/102-995861-3-170750/f3.png"><img alt="f3.png" class="image-3 jive-image" height="279" src="https://community.oracle.com/servlet/JiveServlet/downloadImage/102-995861-3-170750/f3.png" style="height: 236px; width: 620px;" width="734"></a></span></p>
           <p><strong>Figure 3. The <code>pom.xml</code> file</strong></p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>Inside the <code>pom.xml</code> file, the version of your project is automatically filled with <code>1.0-SNAPSHOT</code>. The snapshot version is used to denote that your project is under development and subject to constant changes. As good practice, after the initial development phase ends, you should update the version of your project to a specific version that does not include the word <em>SNAPSHOT</em>. This change is done simply by editing the <code>pom.xm</code>l file. It is also worthwhile to indicate that the packaging of the project will be a JAR artifact.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <h2>Automating the Development Phases of a Java Project Based in Maven</h2>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>Maven uses a construction lifecycle to automate various stages of your project: compilation, unit testing, packaging, integration testing, additional verification, and distribution of the resulting package to a local Maven repository and/or a centralized repository. When you install Maven, it automatically creates a local repository for you (usually in a folder named <code>.m2</code>), which serves as a point of cache between the Maven internet servers and your workstation.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>Inside the directory <code>my-application1</code>, you can run multiple stages of your project in an independent manner by using the following commands.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <h3><code>mvn compile</code></h3>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>This command compiles the project and creates a new directory called <code>target</code> where your compiled classes are stored. It is important to note that at this stage, Maven has not yet carried out the packaging stage (the JAR file generation).</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>Figure 4 shows the generated structure:</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p><span style="color: red;"><a href="https://community.oracle.com/servlet/JiveServlet/showImage/102-995861-3-170751/f4.png"><img alt="f4.png" class="image-4 jive-image" height="265" src="https://community.oracle.com/servlet/JiveServlet/downloadImage/102-995861-3-170751/f4.png" style="height: auto; width: auto;" width="264"></a></span></p>
           <p><strong>Figure 4. Structure created during the compilation phase</strong></p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <h3><code>mvn test</code></h3>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>This command compiles the project and then runs the unit test phase of the project (see Figure 5).</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p><span style="color: red;"><a href="https://community.oracle.com/servlet/JiveServlet/showImage/102-995861-3-170752/f5.png"><img alt="f5.png" class="image-5 jive-image" height="141" src="https://community.oracle.com/servlet/JiveServlet/downloadImage/102-995861-3-170752/f5.png" style="height: auto; width: auto;" width="521"></a></span></p>
           <p><strong>Figure 5. Unit test phase</strong></p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <h3><code>mvn package</code></h3>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>This command compiles the project, runs the unit testing phase, and finally packages the project into a JAR file. The generated JAR file is automatically stored within the <code>target</code> folder (see Figure 6).</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p><span style="color: red;"><a href="https://community.oracle.com/servlet/JiveServlet/showImage/102-995861-3-170753/f6.png"><img alt="f6.png" class="image-6 jive-image" height="122" src="https://community.oracle.com/servlet/JiveServlet/downloadImage/102-995861-3-170753/f6.png" style="height: 92px; width: 620px;" width="818"></a></span></p>
           <p><strong>Figure 6. Package phase</strong></p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <h3><code>mvn install</code></h3>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>This command compiles the project, runs the unit testing phase, packages the project into a JAR file, and finally installs a copy of the JAR file into the local Maven repository. Remember that the local repository is the <code>.m2</code> folder located in your <code>home</code> directory on your workstation (see Figure 7).</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p><span style="color: red;"><a href="https://community.oracle.com/servlet/JiveServlet/showImage/102-995861-3-170754/f7.png"><img alt="f7.png" class="image-7 jive-image" height="170" src="https://community.oracle.com/servlet/JiveServlet/downloadImage/102-995861-3-170754/f7.png" style="height: 131px; width: 620px;" width="804"></a></span></p>
           <p><strong>Figure 7. Installation phase</strong></p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>As shown in Figure 7, the folder called <code>.m2</code> has been modified to add a copy of the artifact (the JAR file) that was generated.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <h2>Integrating Development Environments with Maven</h2>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>Currently Maven has extensive integration with popular integrated development environments (IDEs) for Java development. This, along with continuous integration and continuous delivery, offers a great synergy to speed up your software application lifecycle.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <h2>Maven Repositories</h2>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>The default Maven repository on the internet is Maven Central. Many other repositories are available, depending on an organization's needs and software licenses policies.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>The release of the <a class="jive-link-external-small" href="http://docs.oracle.com/middleware/1213/core/MAVEN/config_maven_repo.htm" rel="nofollow">Oracle Maven Repository</a> offers developers a number of dependencies and plugins that will further facilitate the development of Java projects in different areas (Java ME, Java SE, Java EE, and JavaFX) inside and outside the Oracle Fusion Middleware ecosystem.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>Some of the advantages can be easily seen. For example, in the <a class="jive-link-external-small" href="https://docs.oracle.com/middleware/1212/wls/WLPRG/maven.htm#WLPRG585" rel="nofollow">WebLogic Development Maven Plugin</a> allows you to manage Oracle WebLogic Server by doing tasks such as installation, creation of Oracle WebLogic Server domains, and starting, stopping, and deploying applications and modules.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <h2>Maven Plugins</h2>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>One key factor that had allowed Maven stay up to date with emerging technologies is its plugin capability. These capabilities range from old-school duties, such as adding a software license to every Java class of a project, to the creation and <a class="jive-link-external-small" href="https://github.com/rhuss/docker-maven-plugin" rel="nofollow">management of a Docker container lifecycle</a>. The Maven site provides the official list of well-known <a class="jive-link-external-small" href="https://maven.apache.org/plugins/" rel="nofollow">Maven plugins</a>.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <h2>The Power of Maven Artifacts</h2>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>In simple words, <a class="jive-link-external-small" href="https://maven.apache.org/guides/introduction/introduction-to-archetypes.html" rel="nofollow">Maven artifacts</a> are project templates that can include a predefined folder structure, configuration files, Java classes, and a <code>pom.xml</code> file with dependencies, plugins, and configurations.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>In one simple command you can obtain a ready-to-deploy JAX-RS web service or an Enterprise JavaBeans (EJB) module. The <a class="jive-link-external-small" href="https://docs.oracle.com/middleware/1213/core/MAVEN/weblogic_maven.htm" rel="nofollow">Oracle WebLogic Server Maven archetype</a> covers creating a wide range of projects, such as a Basic WebApp project, a WebApp with EJB project, a Basic MDB project, and a Basic WebServices project.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <h2>Conclusion</h2>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
           <p>Maven allows you to optimize the development time for systems composed of Java projects by providing a standardized way to work and automate various stages during the construction, validation, packaging, and distribution of such projects. The next time you use Maven, remember the basic concepts covered in this article to smooth your user experience.</p>
           <p style="min-height: 8pt; padding: 0px;">&nbsp;</p>
