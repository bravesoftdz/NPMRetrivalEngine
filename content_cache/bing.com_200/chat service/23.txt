            <h2>Creating different Websocket Chat Clients in Java</h2> 
            <small>November 9th, 2014 by <a href="/about/" rel="author">Micha Kops</a></small> 
            <div class="entry"> 
             <p>Having written two articles about different websocket based chat server implementations in Java, I was recently asked how an implementation of the client side would look like in Java.</p> 
             <p>That’s why I added this article to demonstrate how to create a websocket chat client applications within a few steps with the Java API for Websocket.</p> 
             <p>In the following tutorial, we’re going to write a text-based chat client for the console first and afterwards we’re going to program a chat client with a graphical user interface, implemented in JavaFX.</p>

             <span id="Chat_Server"><h3>Chat Server</h3></span> 
             <p>To keep it easy we’re using a pre-built chat-server from one of my articles – so on the one hand, there is a solution using vert.x from my article “<a href="/2013/11/creating-a-websocket-chat-application-with-vert-x-and-java/">Creating a Websocket Chat Application with Vert.x and Java</a>” or on the other hand a solution based on Java EE 7 with an embedded GlassFish server from my tutorial “<a href="/2013/08/creating-a-chat-application-using-java-ee-7-websockets-and-glassfish-4/">Creating a Chat Application using Java EE 7, Websockets and GlassFish 4</a>“. *Update*: In between I have also written an implementation in Go: “<a href="/2016/10/writing-a-websocket-chat-in-go/">Writing a Websocket Chat in Go</a>“.</p> 
             <p>Which one we chose does not matter as both variants allow us to start a full-blown websocket chat server with only Java and Maven as prerequisites.</p> 
             <p>You may download the corresponding vertx-fat-jar or the war file from my repositories on Bitbucket &nbsp;<a onclick="javascript:pageTracker._trackPageview('/outgoing/bitbucket.org/hascode/vertx-websocket-chat/downloads');" href="https://bitbucket.org/hascode/vertx-websocket-chat/downloads" target="_blank">here</a> and <a onclick="javascript:pageTracker._trackPageview('/outgoing/bitbucket.org/hascode/javaee7-websocket-chat/downloads');" href="https://bitbucket.org/hascode/javaee7-websocket-chat/downloads" target="_blank">there</a>.</p> 
             <span id="Java_API_for_WebSockets_JSR_356"><h3>Java API for WebSockets JSR 356</h3></span> 
             <p>The Java API for Websockets specifies not only the server- but also the client side API to handle a websocket connection.</p> 
             <p>I found a nice tutorial, written by Jiji Sasidharan that explains the client implementation in detail <a onclick="javascript:pageTracker._trackPageview('/outgoing/www.programmingforliving.com/2013/08/jsr-356-java-api-for-websocket-client-api.html');" href="http://www.programmingforliving.com/2013/08/jsr-356-java-api-for-websocket-client-api.html" target="_blank">here</a>.</p> 
             <span id="Dependencies"><h4>Dependencies</h4></span> 
             <p>The following dependencies are needed for the following examples. The important ones are&nbsp; <em>javax.websocket-client-api</em> for the API and the <em>tyrus</em> dependencies for the implementation.</p> 
             <p>The two last dependencies are needed because the server implementation of our chat demands a specific JSON structure and we’re using the Java API for JSON Processing aka JSR 353 here to handle this.</p>

             <span id="Client_Endpoint"><h4>Client Endpoint</h4></span> 
             <p>This is our client endpoint, marked as an endpoint by the <em>javax.websocket.ClientEndpoint</em> annotation.</p> 
             <p>There are several lifecycle annotations that allow us to listen for specific states of our designated connection.</p> 
             <p>Our implementation accepts a message handler for incoming messages.</p>

             <span id="Troubleshooting"><h3>Troubleshooting</h3></span>
