       <p><a href="app-service-web-overview" data-linktype="relative-path">Azure Web Apps</a> provides a highly scalable, self-patching web hosting service. This quickstart shows how to deploy a Node.js app to Azure Web Apps. You create the web app using the <a href="https://docs.microsoft.com/cli/azure/get-started-with-azure-cli" data-linktype="external">Azure CLI</a>, and you use Git to deploy sample Node.js code to the web app.</p> 
       <p><img src="media/app-service-web-get-started-nodejs-poc/hello-world-in-browser.png" alt="Sample app running in Azure" data-linktype="relative-path"></p> 
       <p>You can follow the steps below using a Mac, Windows, or Linux machine. Once the prerequisites are installed, it takes about five minutes to complete the steps. </p> 
       <div class="embeddedvideo">
        <iframe src="https://channel9.msdn.com/Shows/Azure-for-Node-Developers/Create-a-Nodejs-app-in-Azure-Quickstart/player" frameborder="0" allowfullscreen="true"></iframe>
       </div> 
       <h2 id="prerequisites">Prerequisites</h2> 
       <p>To complete this quickstart:</p> 
       <ul> 
        <li><a href="https://git-scm.com/" data-linktype="external">Install Git</a></li> 
        <li><a href="https://nodejs.org/" data-linktype="external">Install Node.js and NPM</a></li> 
       </ul> 
       <p>If you don't have an Azure subscription, create a <a href="https://azure.microsoft.com/free/?WT.mc_id=A261C142F" data-linktype="external">free account</a> before you begin.</p> 
       <h2 id="download-the-sample">Download the sample</h2> 
       <p>In a terminal window, run the following command to clone the sample app repository to your local machine.</p> 
       <pre><code class="lang-bash">git clone https://github.com/Azure-Samples/nodejs-docs-hello-world
</code></pre>
       <p>You use this terminal window to run all the commands in this quickstart.</p> 
       <p>Change to the directory that contains the sample code.</p> 
       <pre><code class="lang-bash">cd nodejs-docs-hello-world
</code></pre>
       <h2 id="run-the-app-locally">Run the app locally</h2> 
       <p>Run the application locally by opening a terminal window and using the <code>npm start</code> script to launch the built in Node.js HTTP server.</p> 
       <pre><code class="lang-bash">npm start
</code></pre>
       <p>Open a web browser, and navigate to the sample app at <code>http://localhost:1337</code>.</p> 
       <p>You see the <strong>Hello World</strong> message from the sample app displayed in the page.</p> 
       <p><img src="media/app-service-web-get-started-nodejs-poc/localhost-hello-world-in-browser.png" alt="Sample app running locally" data-linktype="relative-path"></p> 
       <p>In your terminal window, press <strong>Ctrl+C</strong> to exit the web server.</p> 
       <h2 id="launch-azure-cloud-shell">Launch Azure Cloud Shell</h2> 
       <p>The Azure Cloud Shell is a free Bash shell that you can run directly within the Azure portal. It has the Azure CLI preinstalled and configured to use with your account. Click the <strong>Cloud Shell</strong> button on the menu in the upper-right of the <a href="https://portal.azure.com" data-linktype="external">Azure portal</a>.</p> 
       <p><a href="https://portal.azure.com" data-linktype="external"><img src="../includes/media/cloud-shell-try-it/cloud-shell-menu.png" alt="Cloud Shell" data-linktype="relative-path"></a></p> 
       <p>The button launches an interactive shell that you can use to run the steps in this topic:</p> 
       <p><a href="https://portal.azure.com" data-linktype="external"><img src="../includes/media/cloud-shell-try-it/cloud-shell-safari.png" alt="Screenshot showing the Cloud Shell window in the portal" data-linktype="relative-path"></a></p> 
       <h2 id="create-a-deployment-user">Create a deployment user</h2> 
       <p>In the Cloud Shell, create deployment credentials with the <a href="/en-us/cli/azure/webapp/deployment/user#set" data-linktype="absolute-path">az webapp deployment user set</a> command.</p> 
       <p>A deployment user is required for FTP and local Git deployment to a web app. The user name and password are account level. <em>They are different from your Azure subscription credentials.</em></p> 
       <p>In the following command, replace <em>&lt;username&gt;</em> and <em>&lt;password&gt;</em> with a new user name and password. The user name must be unique. The password must be at least eight characters long, with two of the following three elements: letters, numbers, symbols. </p> 
       <pre data-interactive="azurecli"><code class="lang-azurecli">az webapp deployment user set --user-name &lt;username&gt; --password &lt;password&gt;
</code></pre>
       <p>If you get a <code>'Conflict'. Details: 409</code> error, change the username. If you get a <code>'Bad Request'. Details: 400</code> error, use a stronger password.</p> 
       <p>You create this deployment user only once; you can use it for all your Azure deployments.</p> 
       <div class="NOTE">
        <h5>Note</h5>
        <p>Record the user name and password. You need them to deploy the web app later.</p> 
       </div> 
       <h2 id="create-a-resource-group">Create a resource group</h2> 
       <p>In the Cloud Shell, create a resource group with the <a href="/en-us/cli/azure/group#create" data-linktype="absolute-path">az group create</a> command.</p> 
       <p>A <a href="../azure-resource-manager/resource-group-overview#terminology" data-linktype="relative-path">resource group</a> is a logical container into which Azure resources like web apps, databases, and storage accounts are deployed and managed.</p> 
       <p>The following example creates a resource group named <em>myResourceGroup</em> in the <em>West Europe</em> location.</p> 
       <pre data-interactive="azurecli"><code class="lang-azurecli">az group create --name myResourceGroup --location "West Europe"
</code></pre>
       <p>You generally create your resource group and the resources in a region near you. To see all supported locations for Azure Web Apps, run the <code>az appservice list-locations</code> command. </p> 
       <h2 id="create-an-azure-app-service-plan">Create an Azure App Service plan</h2> 
       <p>In the Cloud Shell, create an App Service plan with the <a href="/en-us/cli/azure/appservice/plan#create" data-linktype="absolute-path">az appservice plan create</a> command.</p> 
       <p>An <a href="azure-web-sites-web-hosting-plans-in-depth-overview" data-linktype="relative-path">App Service plan</a> specifies the location, size, and features of the web server farm that hosts your app. You can save money when hosting multiple apps by configuring the web apps to share a single App Service plan.</p> 
       <p>App Service plans define:</p> 
       <ul> 
        <li>Region (for example: North Europe, East US, or Southeast Asia)</li> 
        <li>Instance size (small, medium, or large)</li> 
        <li>Scale count (1 to 20 instances)</li> 
        <li>SKU (Free, Shared, Basic, Standard, or Premium)</li> 
       </ul> 
       <p>The following example creates an App Service plan named <code>myAppServicePlan</code> in the <strong>Free</strong> pricing tier:</p> 
       <pre data-interactive="azurecli"><code class="lang-azurecli">az appservice plan create --name myAppServicePlan --resource-group myResourceGroup --sku FREE
</code></pre>
       <p>When the App Service plan has been created, the Azure CLI shows information similar to the following example:</p> 
       <pre><code class="lang-json">{ 
  "adminSiteName": null,
  "appServicePlanName": "myAppServicePlan",
  "geoRegion": "West Europe",
  "hostingEnvironmentProfile": null,
  "id": "/subscriptions/0000-0000/resourceGroups/myResourceGroup/providers/Microsoft.Web/serverfarms/myAppServicePlan",
  "kind": "app",
  "location": "West Europe",
  "maximumNumberOfWorkers": 1,
  "name": "myAppServicePlan",
  &lt; JSON data removed for brevity. &gt;
  "targetWorkerSizeId": 0,
  "type": "Microsoft.Web/serverfarms",
  "workerTierName": null
} 
</code></pre>
       <h2 id="create-a-web-app">Create a web app</h2> 
       <p>In the Cloud Shell, create a <a href="app-service-web-overview" data-linktype="relative-path">web app</a> in the <code>myAppServicePlan</code> App Service plan with the <a href="/en-us/cli/azure/webapp#create" data-linktype="absolute-path">az webapp create</a> command. </p> 
       <p>The web app provides a hosting space for your code and provides a URL to view the deployed app.</p> 
       <p>In the following command, replace <em>&lt;app_name&gt;</em> with a unique name (valid characters are <code>a-z</code>, <code>0-9</code>, and <code>-</code>). If <code>&lt;app_name&gt;</code> is not unique, you get the error message "Website with given name 
        <app_name>
          already exists." The default URL of the web app is 
         <code>https://&lt;app_name&gt;.azurewebsites.net</code>. 
        </app_name></p>
       <p> </p>
       <pre data-interactive="azurecli"><code class="lang-azurecli">az webapp create --name &lt;app_name&gt; --resource-group myResourceGroup --plan myAppServicePlan --deployment-local-git
</code></pre>
       <p>When the web app has been created, the Azure CLI shows information similar to the following example:</p> 
       <pre><code class="lang-json">{
  "availabilityState": "Normal",
  "clientAffinityEnabled": true,
  "clientCertEnabled": false,
  "cloningInfo": null,
  "containerSize": 0,
  "dailyMemoryTimeQuota": 0,
  "defaultHostName": "&lt;app_name&gt;.azurewebsites.net",
  "enabled": true,
  "enabledHostNames": [
    "&lt;app_name&gt;.azurewebsites.net",
    "&lt;app_name&gt;.scm.azurewebsites.net"
  ],
  "gatewaySiteName": null,
  "hostNameSslStates": [
    {
      "hostType": "Standard",
      "name": "&lt;app_name&gt;.azurewebsites.net",
      "sslState": "Disabled",
      "thumbprint": null,
      "toUpdate": null,
      "virtualIp": null
    }
    &lt; JSON data removed for brevity. &gt;
}
</code></pre>
       <p>Browse to the site to see your newly created web app.</p> 
       <pre><code class="lang-bash">http://&lt;app_name&gt;.azurewebsites.net
</code></pre>
       <p><img src="media/app-service-web-get-started-php/app-service-web-service-created.png" alt="Empty web app page" data-linktype="relative-path"></p> 
       <p>You’ve created an empty new web app in Azure.</p> 
       <h2 id="configure-local-git-deployment">Configure local Git deployment</h2> 
       <p>Configure local Git deployment to the web app with the <a href="/en-us/cli/azure/webapp/deployment/source#config-local-git" data-linktype="absolute-path">az webapp deployment source config-local-git</a> command.</p> 
       <p>App Service supports several ways to deploy content to a web app, such as FTP, local Git, GitHub, Visual Studio Team Services, and Bitbucket. For this quickstart, you deploy by using local Git. That means you deploy by using a Git command to push from a local repository to a repository in Azure. </p> 
       <p>In the following command, replace <em>&lt;app_name&gt;</em> with your web app's name.</p> 
       <pre data-interactive="azurecli"><code class="lang-azurecli">az webapp deployment source config-local-git --name &lt;app_name&gt; --resource-group myResourceGroup --query url --output tsv
</code></pre>
       <p>The output has the following format:</p> 
       <pre><code class="lang-bash">https://&lt;username&gt;@&lt;app_name&gt;.scm.azurewebsites.net:443/&lt;app_name&gt;.git
</code></pre>
       <p>The <code>&lt;username&gt;</code> is the <a href="#configure-a-deployment-user" data-linktype="self-bookmark">deployment user</a> that you created in a previous step.</p> 
       <p>Copy the URI shown; you'll use it in the next step.</p> 
       <h2 id="push-to-azure-from-git">Push to Azure from Git</h2> 
       <p>In the local terminal window, add an Azure remote to your local Git repository.</p> 
       <pre><code class="lang-bash">git remote add azure &lt;URI from previous step&gt;
</code></pre>
       <p>Push to the Azure remote to deploy your app with the following command. When prompted for a password, make sure that you enter the password you created in <a href="#configure-a-deployment-user" data-linktype="self-bookmark">Configure a deployment user</a>, not the password you use to log in to the Azure portal.</p> 
       <pre><code class="lang-bash">git push azure master
</code></pre>

       <h2 id="browse-to-the-app">Browse to the app</h2> 
       <p>Browse to the deployed application using your web browser.</p> 
       <pre><code class="lang-bash">http://&lt;app_name&gt;.azurewebsites.net
</code></pre>
       <p>The Node.js sample code is running in an Azure App Service web app.</p> 
       <p><img src="media/app-service-web-get-started-nodejs-poc/hello-world-in-browser.png" alt="Sample app running in Azure" data-linktype="relative-path"></p> 
       <p><strong>Congratulations!</strong> You've deployed your first Node.js app to App Service.</p> 
       <h2 id="update-and-redeploy-the-code">Update and redeploy the code</h2> 
       <p>Using a text editor, open the <code>index.js</code> file in the Node.js app, and make a small change to the text in the call to <code>response.end</code>:</p> 
       <pre><code class="lang-nodejs">response.end("Hello Azure!");
</code></pre>
       <p>Commit your changes in Git, and then push the code changes to Azure.</p> 
       <pre><code class="lang-bash">git commit -am "updated output"
git push azure master
</code></pre>
       <p>Once deployment has completed, switch back to the browser window that opened in the <strong>Browse to the app</strong> step, and hit refresh.</p> 
       <p><img src="media/app-service-web-get-started-nodejs-poc/hello-azure-in-browser.png" alt="Updated sample app running in Azure" data-linktype="relative-path"></p> 
       <h2 id="manage-your-new-azure-web-app">Manage your new Azure web app</h2> 
       <p>Go to the <a href="https://portal.azure.com" target="_blank" data-linktype="external">Azure portal</a> to manage the web app you created.</p> 
       <p>From the left menu, click <strong>App Services</strong>, and then click the name of your Azure web app.</p> 
       <p><img src="media/app-service-web-get-started-nodejs-poc/nodejs-docs-hello-world-app-service-list.png" alt="Portal navigation to Azure web app" data-linktype="relative-path"></p> 
       <p>You see your web app's Overview page. Here, you can perform basic management tasks like browse, stop, start, restart, and delete. </p> 
       <p><img src="media/app-service-web-get-started-nodejs-poc/nodejs-docs-hello-world-app-service-detail.png" alt="App Service blade in Azure portal" data-linktype="relative-path"></p> 
       <p>The left menu provides different pages for configuring your app. </p> 
       <h2 id="clean-up-resources">Clean up resources</h2> 
       <p>To clean up your resources, run the following command:</p> 
       <pre data-interactive="azurecli"><code class="lang-azurecli">az group delete --name myResourceGroup
</code></pre>
