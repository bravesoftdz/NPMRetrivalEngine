             <h3>1. Create Linux Commandline Chat Server</h3> 
             <p>We all have been using chat service since a long time. We are familiar with Google chat, Hangout, Facebook chat, Whatsapp, Hike and several other application and integrated chat services. Do you know Linux <strong>nc</strong> command can make your Linux box a chat server with just one line of command.</p> 
             <h6>What is nc command in Linux and what it does?</h6> 
             <p><strong>nc</strong> is the depreciation of Linux <strong>netcat</strong> command. The <strong>nc</strong> utility is often referred as Swiss army knife based upon the number of its built-in capabilities. It is used as debugging tool, investigation tool, reading and writing to network connection using TCP/UDP, DNS forward/reverse checking.</p> 
             <center style="padding-bottom: 20px;padding-top: 10px;">
              <ins class="adsbygoogle" style="display:block" data-ad-client="ca-pub-2601749019656699" data-ad-slot="5590002574" data-ad-format="auto"></ins>
              <script type="application/ld+json">(adsbygoogle = window.adsbygoogle || []).push({});</script>
             </center>
             <p>It is prominently used for port scanning, file transferring, backdoor and port listening. nc has the ability to use any local unused port and any local network source address.</p> 
             <p>Use <strong>nc</strong> command (On Server with IP address: <strong>192.168.0.7</strong>) to create a command line messaging server instantly.</p> 
             <pre>$ nc -l -vv -p 11119
</pre> 
             <p>Explanation of the above command switches.</p> 
             <ol> 
              <li><strong>-v</strong> : means Verbose</li> 
              <li><strong>-vv</strong> : more verbose</li> 
              <li><strong>-p</strong> : The local port Number</li> 
             </ol> 
             <p>You may replace <strong>11119</strong> with any other local port number.</p> 
             <p>Next on the client machine (IP address: 192.168.0.15) run the following command to initialize chat session to machine (where messaging server is running).</p> 
             <pre>$ nc 192.168.0.7 11119
</pre> 
             <p><a href="https://www.tecmint.com/wp-content/uploads/2015/04/Chat-on-Linux-Commandline.gif"><img src="https://www.tecmint.com/wp-content/plugins/lazy-load/images/1x1.trans.gif" data-lazy-src="https://www.tecmint.com/wp-content/uploads/2015/04/Chat-on-Linux-Commandline.gif" class="size-medium wp-image-13146" alt="Linux Commandline Chat with nc Command">

               </noscript></a></p> 
             <p><strong style="color: red;">Note:</strong> You can terminate chat session by hitting <strong>ctrl+c</strong> key and also nc chat is one-to-one service.</p> 
             <h3>2. How to Sum Values in a Column in Linux</h3> 
             <p>How to sum the numerical values of a column, generated as an output of a command, on the fly in the terminal.</p> 
             <p>The output of the ‘<strong>ls -l</strong>‘ command.</p> 
             <pre>$ ls -l
</pre> 
             <p><a href="https://www.tecmint.com/wp-content/uploads/2015/04/Sum-Values.gif"><img src="https://www.tecmint.com/wp-content/plugins/lazy-load/images/1x1.trans.gif" data-lazy-src="https://www.tecmint.com/wp-content/uploads/2015/04/Sum-Values.gif" class="size-medium wp-image-13150" alt="Sum Numerical Values">
               <noscript>
                <img class="size-medium wp-image-13150" src="https://www.tecmint.com/wp-content/uploads/2015/04/Sum-Values.gif" alt="Sum Numerical Values">
               </noscript></a></p> 
             <p>Notice that the second column is numerical which represents number of symbolic links and the <strong>5th</strong> column is numerical which represents the size of he file. Say we need to sum the values of fifth column on the fly.</p> 
             <p>List the content of <strong>5th</strong> column without printing anything else. We will be using ‘<strong>awk</strong>‘ command to do this. ‘<strong>$5</strong>‘ represents <strong>5th</strong> column.</p> 
             <pre>$ ls -l | awk '{print $5}'
</pre> 
             <p><a href="https://www.tecmint.com/wp-content/uploads/2015/04/List-Content-Column.gif"><img src="https://www.tecmint.com/wp-content/plugins/lazy-load/images/1x1.trans.gif" data-lazy-src="https://www.tecmint.com/wp-content/uploads/2015/04/List-Content-Column.gif" class="size-medium wp-image-13151" alt="List Content Column">
               <noscript>
                <img class="size-medium wp-image-13151" src="https://www.tecmint.com/wp-content/uploads/2015/04/List-Content-Column.gif" alt="List Content Column">
               </noscript></a></p> 
             <p>Now use <strong>awk</strong> to print the sum of the output of <strong>5th</strong> column by pipelining it.</p> 
             <pre>$ ls -l | awk '{print $5}' | awk '{total = total + $1}END{print total}'
</pre> 
             <p><a href="https://www.tecmint.com/wp-content/uploads/2015/04/Sum-Columns.gif"><img src="https://www.tecmint.com/wp-content/plugins/lazy-load/images/1x1.trans.gif" data-lazy-src="https://www.tecmint.com/wp-content/uploads/2015/04/Sum-Columns.gif" class="size-medium wp-image-13152" alt="Sum and Print Columns">

             <h3>How to Remove Orphan Packages in Linux?</h3> 
             <p><strong>Orphan</strong> packages are those packages that are installed as a dependency of another package and no longer required when the original package is removed.</p> 
             <p>Say we installed a package <strong>gtprogram</strong> which was dependent of <strong>gtdependency</strong>. We can’t install <strong>gtprogram</strong> unless <strong>gtdependency</strong> is installed.</p> 
             <p>When we remove <strong>gtprogram</strong> it won’t remove <strong>gtdependency</strong> by default. And if we don’t remove <strong>gtdependency</strong>, it will remain as Orpahn Package with no connection to any other package.</p> 
             <pre># yum autoremove                [On RedHat Systems]
</pre> 
             <p><a href="https://www.tecmint.com/wp-content/uploads/2015/04/Remove-Orphan-Packages-in-CentOS1.gif"><img src="https://www.tecmint.com/wp-content/plugins/lazy-load/images/1x1.trans.gif" data-lazy-src="https://www.tecmint.com/wp-content/uploads/2015/04/Remove-Orphan-Packages-in-CentOS1.gif" class="size-medium wp-image-13159" alt="Remove Orphan Packages in CentOS">

               </noscript></a></p> 
             <pre># apt-get autoremove                [On Debian Systems]
</pre> 
             <p><a href="https://www.tecmint.com/wp-content/uploads/2015/04/Remove-Orphan-Packages-in-Debian.gif"><img src="https://www.tecmint.com/wp-content/plugins/lazy-load/images/1x1.trans.gif" data-lazy-src="https://www.tecmint.com/wp-content/uploads/2015/04/Remove-Orphan-Packages-in-Debian.gif" class="size-medium wp-image-13155" alt="Remove Orphan Packages in Debian">

             <p>You should always remove Orphan Packages to keep the Linux box loaded with just necessary stuff and nothing else.</p> 
             <h3>4. How to Get Local and Public IP Address of Linux Server</h3> 
             <p>To get you local IP address run the below one liner script.</p> 
             <pre>$ ifconfig | grep "inet addr:" | awk '{print $2}' | grep -v '127.0.0.1' | cut -f2 -d:
</pre> 
             <p>You must have installed <strong>ifconfig</strong>, if not, <strong>apt</strong> or <strong>yum</strong> the required packages. Here we will be pipelining the output of <strong>ifconfig</strong> with <strong>grep</strong> command to find the string “<strong>intel addr:</strong>”.</p> 
             <p>We know <strong>ifconfig</strong> command is sufficient to output local IP Address. But <strong>ifconfig</strong> generate lots of other outputs and our concern here is to generate only local IP address and nothing else.</p> 
             <pre># ifconfig | grep "inet addr:"
</pre> 
             <p><a href="https://www.tecmint.com/wp-content/uploads/2015/04/Check-Local-IP-Address.gif"><img src="https://www.tecmint.com/wp-content/plugins/lazy-load/images/1x1.trans.gif" data-lazy-src="https://www.tecmint.com/wp-content/uploads/2015/04/Check-Local-IP-Address.gif" class="size-medium wp-image-13160" alt="Check Local IP Address">
               <noscript>
                <img class="size-medium wp-image-13160" src="https://www.tecmint.com/wp-content/uploads/2015/04/Check-Local-IP-Address.gif" alt="Check Local IP Address">
               </noscript></a></p> 
             <p>Although the output is more custom now, but we need to filter our local IP address only and nothing else. For this we will use awk to print the second column only by pipelining it with the above script.</p> 
             <pre># ifconfig | grep “inet addr:” | awk '{print $2}'
</pre> 
             <p><a href="https://www.tecmint.com/wp-content/uploads/2015/04/Filter-IP-Address.gif"><img src="https://www.tecmint.com/wp-content/plugins/lazy-load/images/1x1.trans.gif" data-lazy-src="https://www.tecmint.com/wp-content/uploads/2015/04/Filter-IP-Address.gif" class="size-medium wp-image-13161" alt="Filter Only IP Address">
               <noscript>
                <img class="size-medium wp-image-13161" src="https://www.tecmint.com/wp-content/uploads/2015/04/Filter-IP-Address.gif" alt="Filter Only IP Address">
               </noscript></a></p> 
             <p>Clear from the above image that we have customised the output very much but still not what we want. The loopback address <strong>127.0.0.1</strong> is still there in the result.</p> 
             <p>We use use <strong>-v</strong> flag with <strong>grep</strong> that will print only those lines that don’t match the one provided in argument. Every machine have the same <strong>loopback address 127.0.0.1</strong>, so use <strong>grep -v</strong> to print those lines that don’t have this string, by pipelining it with above output.</p> 
             <pre># ifconfig | grep "inet addr" | awk '{print $2}' | grep -v '127.0.0.1'
</pre> 
             <p><a href="https://www.tecmint.com/wp-content/uploads/2015/04/Print-IP-Address.gif"><img src="https://www.tecmint.com/wp-content/plugins/lazy-load/images/1x1.trans.gif" data-lazy-src="https://www.tecmint.com/wp-content/uploads/2015/04/Print-IP-Address.gif" class="size-medium wp-image-13162" alt="Print IP Address">
               <noscript>
                <img class="size-medium wp-image-13162" src="https://www.tecmint.com/wp-content/uploads/2015/04/Print-IP-Address.gif" alt="Print IP Address">
               </noscript></a></p> 
             <p>We have almost generated desired output, just replace the string <code>(addr:)</code> from the beginning. We will use <strong>cut</strong> command to print only column two. The <strong>column 1</strong> and <strong>column 2</strong> are not separated by tab but by <code>(:)</code>, so we need to use delimiter <code>(-d)</code> by pipelining the above output.</p> 
             <pre># ifconfig | grep "inet addr:" | awk '{print $2}' | grep -v '127.0.0.1' | cut -f2 -d:
</pre> 
             <p><a href="https://www.tecmint.com/wp-content/uploads/2015/04/Custome-IP-Address.gif"><img src="https://www.tecmint.com/wp-content/plugins/lazy-load/images/1x1.trans.gif" data-lazy-src="https://www.tecmint.com/wp-content/uploads/2015/04/Custome-IP-Address.gif" class="size-medium wp-image-13163" alt="Customized IP Address">
               <noscript>
                <img class="size-medium wp-image-13163" src="https://www.tecmint.com/wp-content/uploads/2015/04/Custome-IP-Address.gif" alt="Customized IP Address">
               </noscript></a></p> 
             <p>Finally! The desired result has been generated.</p> 
             <h3>5. How to Color Linux Terminal</h3> 
             <p>You might have seen colored output in terminal. Also you would be knowing to enable/disable colored output in terminal. If not you may follow the below steps.</p> 
             <p>In Linux every user has <code>'.bashrc'</code> file, this file is used to handle your terminal output. Open and edit this file with your choice of editor. Note that, this file is hidden (dot beginning of file means hidden).</p> 
             <pre>$ vi /home/$USER/.bashrc
</pre> 
             <p>Make sure that the following lines below are uncommented. ie., it don’t start with a <strong>#</strong>.</p> 
             <pre>if [ -x /usr/bin/dircolors ]; then
test -r ~/.dircolors &amp;&amp; eval "$(dircolors -b ~/.dircolors)" || eval "$(dirc$
alias ls='ls --color=auto'
#alias dir='dir --color=auto'
#alias vdir='vdir --color=auto'
alias grep='grep --color=auto'
alias fgrep='fgrep --color=auto'
alias egrep='egrep --color=auto'
fi
</pre> 
             <p><a href="https://www.tecmint.com/wp-content/uploads/2015/04/bashrc-file.gif"><img src="https://www.tecmint.com/wp-content/plugins/lazy-load/images/1x1.trans.gif" data-lazy-src="https://www.tecmint.com/wp-content/uploads/2015/04/bashrc-file.gif" class="size-medium wp-image-13164" alt="User .bashrc File">

                <img class="size-medium wp-image-13164" src="https://www.tecmint.com/wp-content/uploads/2015/04/bashrc-file.gif" alt="User .bashrc File">
               </noscript></a></p> 
             <p>Once done! Save and exit. To make the changes taken into effect logout and again login.</p> 
             <p>Now you will see files and folders are listed in various colors based upon type of file. To decode the color code run the below command.</p> 
             <pre>$ dircolors -p
</pre> 
             <p>Since the output is too long, lets pipeline the output with less command so that we get output one screen at a time.</p> 
             <pre>$ dircolors -p | less
</pre>

             <h3>6. How to Hash Tag Linux Commands and Scripts</h3> 
             <p>We are using hash tags on <strong>Twitter</strong>, <strong>Facebook</strong> and <strong>Google Plus</strong> (may be some other places, I have not noticed). These hash tags make it easier for others to search for a hash tag. Very few know that we can use hash tag in Linux command Line.</p> 
             <p>We already know that <code>#</code> in configuration files and most of the programming languages is treated as comment line and is excluded from execution.</p> 
             <p>Run a command and then create a hash tag of the command so that we can find it later. Say we have a long script that was executed in <strong>point 4</strong> above. Now create a hash tag for this. We know <strong>ifconfig</strong> can be run by <strong>sudo</strong> or <strong>root</strong> user hence acting as root.</p> 
             <pre># ifconfig | grep "inet addr:" | awk '{print $2}' | grep -v '127.0.0.1' | cut -f2 -d: #myip
</pre> 
             <p>The script above has been hash tagged with ‘<strong>myip</strong>‘. Now search for the hash tag in <strong>reverse-i-serach</strong> (<strong>press ctrl+r</strong>), in the terminal and type ‘<strong>myip</strong>‘. You may execute it from there, as well.</p> 
             <p><a href="https://www.tecmint.com/wp-content/uploads/2015/04/Create-Command-Hash-Tags.gif"><img src="https://www.tecmint.com/wp-content/plugins/lazy-load/images/1x1.trans.gif" data-lazy-src="https://www.tecmint.com/wp-content/uploads/2015/04/Create-Command-Hash-Tags.gif" class="size-medium wp-image-13166" alt="Create Command Hash Tags">
               <noscript>
                <img class="size-medium wp-image-13166" src="https://www.tecmint.com/wp-content/uploads/2015/04/Create-Command-Hash-Tags.gif" alt="Create Command Hash Tags">
               </noscript></a></p> 
             <p>You may create as many hash tags for every command and find it later using <strong>reverse-i-search</strong>.</p>
