function addTags(){
    var tags = $("#form_tags").val();
    var options = {
            url: "/presentation/2526619/addtags",
            request: {tags: tags},
            callback: function(xhr) {
                $("#addtags").fadeOut();
                $("#form_tags").val("");
                if (true === xhr.result) {
                    $.blink("Tag(s) added", 3);
                    var tags = xhr.tags;
                    var htmlTags = new Array();
                    $.each(tags, function(index, value){
                        var htmlValue = '<a href="/search/presentations/' + value + '">' + value + '</a>';
                        htmlTags.push(htmlValue);
                    });
                    $('#tags').html(htmlTags.join(' | '));
                }else{
                    if (xhr.hasOwnProperty("error_message")) {
                        $.blink(xhr.error_message, 5);
                    } else {
                        $.blink("Tag(s) not added", 3);
                    }
                }
            }
    };
    AppAjax.get("#tags", options);
}

            $("#view-bottom-ads-boxes").fadeOut(200);

            var height1 = $("#additional-content").height();

                    $('#aboutpowershowcontent').height(height3 + 'px');

                }
            }
        }
    }, timeout);
}

             <strong> 1</strong>
             <br>A CHAT CLIENT-SERVER MODULE IN JAVA
             <br>ISE 582 FALL 2003 PROJECT 
             <br>BY MAHTAB M HUSSAIN MAYANK MOHAN 
             <br>
             <strong> 2</strong>
             <br>NETWORKING BASICS
             <ul>
              <li> Transmission Control Protocol (TCP) </li>
              <li> </li>
              <li> This protocol ensures that the data sent over <br>two points in a Network </li>
              <li> is received in the same order as it is sent </li>
              <li> Ports </li>
              <li> Is an address which determines the data origin <br>and delivery points over </li>
              <li> the Network </li>
              <li> Sockets </li>
              <li> A socket is one end point of a two way <br>communication link between </li>
              <li> two programs running on the Network </li>
             </ul> 
             <br>
             <strong> 3</strong>
             <br>THE JAVA .net PACKAGE
             <ul>
              <li>The java .net package is used as the basis for <br>this project. It provides two classes </li>
              <li> Socket </li>
              <li> Server Socket </li>
              <li>The Socket class implements the client side of <br>the connection and the Server Socket Implements <br>the </li>
              <li>server side of the connection </li>
              <li> On the server side </li>
              <li>Normally a server runs on a specific computer and <br>has a socket bound to a specific port number. <br>The server just waits, listening to the socket <br>for a client to make a connection request </li>
              <li> On the client side </li>
              <li>The client knows the host name of the machine on <br>which the server is running and the port number <br>to which the server is connected. To make a <br>connection request, the client tries to <br>communicate with the server on the servers <br>machine and port. </li>
             </ul>
             <br>Port
             <br>Server
             <br>Port
             <br>connection
             <br>Port
             <br>Client 
             <br>
             <strong> 4</strong>
             <br>THE FINAL SOURCE CODE
             <ul>
              <li>Supporting Multiple Clients </li>
              <li>Client connection requests are queued at the <br>port, so the server must accept the </li>
              <li>Connections sequentially. However, the server can <br>service them simultaneously </li>
              <li>Through the use of threads One thread per <br>client connection </li>
              <li>Basic Flow of Logic in such a server is </li>
              <li> <br> while(true) </li>
              <li> <br> accept a connection </li>
              <li> <br> create a thread to deal with <br>the client </li>
              <li> <br> </li>
              <li> <br> end while </li>
              <li>A Brief Description of the various files in the <br>source code </li>
              <li>Server Section </li>
             </ul> 
             <br>
             <strong> 5</strong>
             <ul>
              <li>Server </li>
              <li>The Server Class creates a server object, <br>instantiates a new server listener, opens a </li>
              <li>new server socket, launches a new thread and <br>listens to it. Whenever a new client </li>
              <li>connects, the server object creates a new session <br>object for the client and goes back </li>
              <li>to listening. </li>
              <li>Client Section </li>
              <li>Client </li>
              <li>The Client object opens a socket to the server <br>and creates a user interface for the user </li>
              <li>Once everything has been initialized it launches <br>a ClientListener object (running on a </li>
              <li>Separate thread) which listens to the server and <br>informs the client when new text </li>
              <li>Arrives. The client object then waits for an <br>action event, and sends any text typed by </li>
              <li>The user back to the server. </li>
              <li>2. Message </li>
             </ul>
