            <h1 class="title"> Where are Oracle/Sun/IBM Java packages located? </h1> 
            <div class="header-meta"> 
             <span class="status verified" data-toggle="popover" data-trigger="hover" data-placement="bottom" data-content="This solution has been verified to work by Red Hat Customers and Support Engineers for the specified product version(s)." data-original-title="" title=""> <span aria-hidden="true" class="icon-solution"></span> Solution Verified </span> - Updated

           <section class="field_kcs_issue_txt"> 
            <h2>Issue</h2> 
            <ul> 
             <li>How to install Oracle Java SE packages ?</li> 
             <li>Where are Oracle Java SE packages located?</li> 
             <li>Need satellite certificate for Oracle Java channel.</li> 
             <li>Oracle has stated that Java 7 patch updates will no longer be available to the general public through Oracle.<br> My question is we use Satellite to download Java 6 &amp; 7 patches, will we continue to get Java updates through Red Hat after April and moving forward?</li> 
            </ul> 
           </section> 
           <section class="field_kcs_resolution_txt"> 
            <h2>Resolution</h2> 
            <p>Users are required to enable a new repository on their systems to access Oracle Java software. The steps to configure the new repository are as follows (root privilege required):</p> 
            <p><strong>Note:</strong> The channel names in the examples below are for <em>rhel-x-server</em>. If the system uses a <em>workstation</em> entitlement, then the word 'server' should be replaced with 'workstation'.</p> 
            <p><strong><em>The following content is provided as a convenience. Red Hat will assist in setting up access to the repository. Red Hat does not have permission to alter the source code for this proprietary content and relies on the vendor to provide timely resolution to bug fixes with updated packages.</em></strong></p> 
            <h4>Red Hat Enterprise Linux 5</h4> 
            <p><em>RHN Classic:</em></p> 
            <pre><code>rhn-channel --add --channel rhel-x86_64-server-5-thirdparty-oracle-java
</code></pre>
            <p><em>Red Hat Subscription Management:</em></p> 
            <pre><code>subscription-manager repos --enable rhel-5-server-thirdparty-oracle-java-rpms
</code></pre>
            <h4>Red Hat Enterprise Linux 6</h4> 
            <p><em>RHN Classic:</em></p> 
            <pre><code>rhn-channel --add --channel rhel-x86_64-server-6-thirdparty-oracle-java
</code></pre>
            <p><em>Red Hat Subscription Management:</em></p> 
            <pre><code>subscription-manager repos --enable rhel-6-server-thirdparty-oracle-java-rpms
</code></pre>
            <h4>Red Hat Enterprise Linux 7</h4> 
            <p><em>RHN Classic (Red Hat Satellite Only):</em></p> 
            <pre><code>rhn-channel --add --channel rhel-x86_64-server-7-thirdparty-oracle-java
</code></pre>
            <p><em>Red Hat Subscription Management (All Others):</em></p> 
            <pre><code>subscription-manager repos --enable rhel-7-server-thirdparty-oracle-java-rpms (Oracle JDK)
subscription-manager repos --enable rhel-7-server-supplementary-rpms (IBM JDK)
</code></pre>
            <h4>Red Hat Satellite Notes</h4> 
            <p>Satellite customers will need a new certificate in order to get to the new channels. See <a href="https://access.redhat.com/site/tools/satcert/">https://access.redhat.com/site/tools/satcert/</a> for instructions on how to do obtain updated certificates to gain access to the new channels.</p> 
            <p>Once the certificate is activated successfully, use below commands to find the <code>oracle</code> channel and sync it on satellite server.</p> 
            <p>to find the channel:</p> 
            <pre><code># satellite-sync --list-channels | grep -i oracle
</code></pre>
            <p>to sync channel:</p> 
            <pre><code># satellite-sync -c rhel-x86_64-server-6-thirdparty-oracle-java
</code></pre>
            <p>The system should now be registered with Red Hat Network and subscribed to the Oracle Java channels for access to Oracle JDKs and JREs.</p> 
            <h3>OpenJDK</h3> 
            <p>Red Hat Enterprise Linux includes OpenJDK as the default Java development and runtime environment. Java development and runtime is also available from IBM via the Supplementary media and RHN channels. Access to OpenJDK and IBM JDK are not affected by this change.</p> 
            <h3>Additional Information</h3> 
            <ul> 
             <li><a href="https://access.redhat.com/site/solutions/314713">Oracle Java SE 6 - Notification of End of Public Updates and suggested alternatives</a> (Oracle Java SE 6 life cycle information as it pertains to supportability and distribution by Red Hat)</li> 
             <li><a href="https://access.redhat.com/articles/1312723">Continuing Updates of Oracle Java SE 7 with Red Hat JBoss Middleware Products</a></li> 
             <li><a href="https://access.redhat.com/articles/1299013">OpenJDK Life Cycle and Support Policy</a></li> 
            </ul>
