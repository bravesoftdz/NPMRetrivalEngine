     <h2>Introduction</h2> 
     <p>This article will explore the main PHP libraries used to create real time, bi-directional applications between clients and servers over WebSockets.</p> 
     <p>&nbsp;</p> 
     <p>WebSocket is full-duplex client/server communication over TCP. It is also a new feature available in browsers as a part of the HTML5 specs that allows JavaScript clients to open bi-directional socket connections to a server. The WebSocket protocol is currently supported in several browsers, including Chrome, Internet Explorer, Firefox, Safari and Opera.</p> 
     <p> </p>
     <div id="olal-container"> 
      <script language="javascript" src="http://www.phpbuilder.com/sl/assetlisting/?wsn=www.phpbuilder.com&amp;wpu=//articles/application-architecture/optimization/creating-real-time-applications-with-php-and-websockets.html&amp;lurl=http://o1.qnsr.com/cgi/r?WT.qs_dlk=WdPVLwrIZ70AABya8tYAAAAH;;n=203;c=1395317;s=9524;x=7936;f=201308291506150;u=j;z=TIMESTAMP;k=http://assetform.phpbuilder.com/controller&amp;wpos=onelineassetlisting&amp;lyt=l6&amp;css=http://www.phpbuilder.com/css/onePaidAsset.css&amp;wi=19903110">
</script> 
     </div> 
     <div style="clear:both;"></div> &nbsp;
     <p></p> 
     <p>The main characteristic of WebSockets is the ability to perform bi-directional communication without breaking the connection. This is why WebSockets makes it possible to have more interaction between a browser and a web site, facilitating live content and the creation of real-time games.</p> 
     <p>&nbsp;</p> 
     <p>WebSocket connections are basically TCP socket connections that following the WebSocket rules to communicate. The WebSocket Protocol is an independent TCP-based protocol.</p> 
     <p>&nbsp;</p> 
     <h2>WebSocket URL</h2> 
     <p>A WebSocket URL looks like this:</p> 
     <pre><code>ws://host:port/path (ex. ws://localhost:8000/echo) </code></pre> 
     <p>The main parts of the URL are the host and port. The connection is made to the host on the given port number.</p> 
     <p>&nbsp;</p> 
     <h2>Using WebSockets</h2> 
     <p>To use WebSockets, you need to have a <em>browser</em> and a <em>server</em>that both support the WebSocket protocol. You will also need a traditional web server such as Apache or IIS to serve your website's static content. Once you have downloaded the WebSocket server (this is not the same as Apache), you need to start it and make sure its running in the background. You'll need to make sure not to close the command prompt window!</p> 
     <p>&nbsp;</p> 
     <p>Once the server is running, your client can connect to it to push messages. The client establishes a WebSocket connection through a process known as the WebSocket handshake. This process starts with the client sending a regular HTTP request to the server. An Upgrade header is included in this request that informs the server that the client wishes to establish a WebSocket connection.</p> 
     <p>&nbsp;</p> 
     <h2>PHPWebSockets</h2> 
     <p>PHPWebSockets is a WebSockets server written in PHP (<a href="https://github.com/ghedipunk/PHP-WebSockets" target="new">https://github.com/ghedipunk/PHP-WebSockets</a>).</p> 
     <p>&nbsp;</p> 
     <p>The WebSocket server itself is a class that can be extended to write the rest of the application. The WebSockets.php is the base class and we need to extend that class to write our own simple WebSocket server application. The WebSockets.php base class does the socket management and WebSocket handshake. In the next figure you can see the PHPWebSocket file structure:</p> 
     <p>&nbsp;</p> 
     <p><img src="/imagesvr_ce/3638/WebSocketsfig1.png" style="border: 0px none;" width="129"></p> 
     <p>&nbsp;</p> 
     <p>Below is a simple chat sample application that uses the WebSockets.php class:</p> 
     <p>&nbsp;</p> 
     <p><strong>Listing testwebsock.php</strong></p> 
     <pre><code>#!/usr/bin/env php
&lt;?php

require_once('./WebSockets.php');

class echoServer extends WebSocketServer {
    
  protected function process ($user, $message) {
    $this-&gt;send($user,$message);
  }
  
  protected function connected ($user) {
    // Do nothing: This is just an echo server, there's no need to track the user.
    // However, if we did care about the users, we would probably have a cookie to
    // parse at this step, would be looking them up in permanent storage, etc.
  }
  
  protected function closed ($user) {
    // Do nothing: This is where cleanup would go, in case the user had any sort of
    // open files or other objects associated with them.  This runs after the socket 
    // has been closed, so there is no need to clean up the socket itself here.
  }
}

$echo = new echoServer("0.0.0.0","9000");

try {
  $echo-&gt;run();
}
catch (Exception $e) {
  $echo-&gt;stdout($e-&gt;getMessage());
}</code></pre>

     <h2>Ratchet</h2> 
     <p>Ratchet (<a href="http://socketo.me/" target="new">http://socketo.me/</a>) is one of the main PHP libraries used to create real time, bi-directional applications between clients and servers over WebSockets and also provides you many components to easily add functionality in your application, such as I/O (socket transport), HTTP Protocol Handler, WebSocket Protocol Handler, Session Provider and WAMP Protocol Handler. To install Ratchet you need to add the code below in your composer.json file and run the: &gt;php ~/composer.phar install in command prompt.</p> 
     <p>&nbsp;</p> 
     <pre><code>{
    "require": {
        "cboden/ratchet": "0.3.*"
    }
}</code></pre> 
     <p>&nbsp;</p> 
     <p>An simple chat example that uses Ratchet implements the <a href="http://socketo.me/api/class-Ratchet.MessageComponentInterface.html" target="new">MessageComponentInterface</a>, which implements the decorator pattern to build an application stack, and that inherits from the ComponentInterface and from the MessageInterface the following methods, described next:</p> 
     <p>&nbsp;</p> 
     <p>This basic application will listen for 4 events:</p> 
     <p><strong>onOpen</strong> - Called when a new client has Connected</p> 
     <p><strong>onMessage</strong> - Called when a message is received by a <em>Connection</em></p> 
     <p><strong>onClose</strong> - Called when a <em>Connection</em> is closed</p> 
     <p><strong>onError</strong> - Called when an error occurs on a <em>Connection</em></p> 
     <p>&nbsp;</p> 
     <p>Next class is the chat "application."</p> 
     <p>&nbsp;</p> 
     <p><strong>Listing chat.php</strong></p> 
     <pre><code>&lt;?php
use Ratchet\MessageComponentInterface;
use Ratchet\ConnectionInterface;

    // Make sure composer dependencies have been installed
    require __DIR__ . '/vendor/autoload.php';

class Chat implements MessageComponentInterface {
    protected $clients;

    public function __construct() {
        $this-&gt;clients = new \SplObjectStorage;
    }

    public function onOpen(ConnectionInterface $conn) {
        $this-&gt;clients-&gt;attach($conn);
    }

    public function onMessage(ConnectionInterface $from, $msg) {
        foreach ($this-&gt;clients as $client) {
            if ($from != $client) {
                $client-&gt;send($msg);
            }
        }
    }

    public function onClose(ConnectionInterface $conn) {
        $this-&gt;clients-&gt;detach($conn);
    }

    public function onError(ConnectionInterface $conn, \Exception $e) {
        $conn-&gt;close();
    }
}</code></pre> 
     <p>&nbsp;</p> 
     <p>Run this script using the &gt;php chat.php in command prompt.</p> 
     <pre><code>// Run the server application through the WebSocket protocol on port 8080
    $app = new Ratchet\App('localhost', 8080);
    $app-&gt;route('/chat', new Chat);
    $app-&gt;route('/echo', new Ratchet\Server\EchoServer, array('*'));
    $app-&gt;run();</code></pre> 
     <p>&nbsp;</p> 
     <p>And then open a page with the following JavaScript:</p> 
     <pre><code>var conn = new WebSocket('ws://localhost:8080/echo');
    conn.onmessage = function(e) { console.log(e.data); };
    conn.send('Hello!'); </code></pre>

     <p>You can find some complex working Ratchet examples <a href="https://github.com/cboden/Ratchet-examples" target="new">here</a> and you can also learn about the composer dependencies that Ratchet example use, such as AutobahnJS (the JavaScript library to be used in conjunction with the WAMPServerComponent. It provides an Pub/Sub and RPC sub-protocol on top of WebSockets.) When (JavaScript library required by AutobahnJS for deferred calls.), Guzzle (a PHP library meant for abstracting cURL calls and working with RESTful APIs. Ratchet requires Guzzle to act as its HTTP server).</p>

     <h2>Elephant.io</h2> 
     <p><a href="https://github.com/Wisembly/elephant.io" target="new">Elephant.io</a> is a library that provides a socket.io client fully written in PHP and its goal is to ease communications between your PHP application and a socket.io server. This library has some dependencies in order to function correctly, such as Socket.IO 0.8+, PHP 5.3+ and NodeJS 0.6.5+ and you also need openssl in your PHP config.</p> 
     <p>&nbsp;</p> 
     <p><a href="http://socket.io/" target="new">Socket.IO</a> is a JavaScript library that enables realtime applications through bi-directional communication between web clients and server. It has two parts: a client-side library that runs in the browser, and a server-side library for node.js. To be installed, you need first to install Node.js (<a href="http://nodejs.org/M" target="new">http://nodejs.org/M</a>) - software platform for scalable server-side and networking applications. Node.js applications are written in JavaScript. After you install Node.js, open it and type npm install socket.io installation command.</p> 
     <p>&nbsp;</p> 
     <p>Below are listed the PHP server side and the Socket.io side code of an example using the Elephant.io library.</p> 
     <p>&nbsp;</p> 
     <p>The PHP server side allows you to connect to your socket.io instance and then gives something to it. At the end of the script it will be disconnected. You may notice in the code below the init and send available methods, described here:</p> 
     <p>&nbsp;</p> 
     <pre><code>init($keepalive = false) - Init will initialize the connection to socket.io
send($type, $id = null, $endpoint = null, $message = null) - Send allow you to send message to socket.io. </code></pre> 
     <p>&nbsp;</p> 
     <pre><code>use ElephantIO\Client as Elephant;

$elephant = new Elephant('http://localhost:8000', 'socket.io', 1, false, true, true);

$elephant-&gt;init();
$elephant-&gt;send(
    ElephantIOClient::TYPE_EVENT,
    null,
    null,
    json_encode(array('name' =&gt; 'foo', 'args' =&gt; 'bar'))
);
$elephant-&gt;close();

echo 'tryin to send `bar` to the event `foo`'; </code></pre> 
     <p>&nbsp;</p> 
     <p>Socket.io server side looks like this:</p> 
     <p>&nbsp;</p> 
     <pre><code>var io = require('socket.io').listen(8000);

io.sockets.on('connection', function (socket) {
  console.log('user connected!');

  socket.on('foo', function (data) {
    console.log('here we are in action event and data is: ' + data);
  });
});</code></pre>
