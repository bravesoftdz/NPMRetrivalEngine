        <div class="post-body__content">
         <p><a href="http://nodejs.org/">Node.js</a> and Websockets are the perfect combination to write very fast, lag free applications which can send data to a huge number of clients. So why don't we start learning about these two topics by building a chat service! We will see how to install Node.js packages, serve a static page to the client with a basic web-server, and configure <a href="http://socket.io/">Socket.io</a> to communicate with the client.</p> 
         <p>
          <!--more--></p> 
         <hr> 
         <h2>Why Choose Node.js and Socket.io?</h2> 
         <figure class="tutorial_image"> 
          <img src="https://cdn.tutsplus.com/net/uploads/2013/09/1.jpg" data-original-url="http://cdn.tutsplus.com/net.tutsplus.com/uploads/2013/09/1.jpg" alt="1" width="600" height="191" class="alignnone size-full wp-image-34483">
          <br> 
         </figure> 
         <p>So why use this combo?</p> 
         <p>There are a lot of platforms which can run a chat application, but by choosing Node.js we don't have to learn a completely different language, it's just JavaScript, but server-side.</p> 
         <p> Node.js is a platform built on Chromes JavaScript runtime to make building applications in JavaScript that run on the server, easy. Node.js uses an event-driven, non-blocking I/O model, which makes it perfect for building real time apps.</p> 
         <p>More and more Node.js applications are being written with real-time communication in mind. A famous example is <a href="http://browserquest.mozilla.org/">BrowserQuest</a> from Mozilla, an MMORPG written entirely in Node.js whose source code has been <a href="https://github.com/mozilla/BrowserQuest">released on Github</a>.</p> 
         <p>Node.js comes with a built-in package manager : <a href="http://npmjs.org/">npm</a>. We will use it to install packages that will help speed up our application development process.</p> 
         <p>We'll be using three packages for this tutorial: Jade, Express, and Socket.io.</p> 
         <h3>Socket.io: the Node.js Websockets Plugin</h3> 
         <p>The main feature of our application is the real-time communication between the client and the server.</p> 
         <p>HTML5 introduces Websockets, but it is far away from being supported by all users, so we need a backup solution.</p> 
         <p>Socket.io is our backup solution : it will test Websocket compatibility and if it's not supported it will use Adobe Flash, AJAX, or an iFrame.</p> 
         <p>Finally, it supports a very large set of browsers:</p> 
         <ul> 
          <li>Internet Explorer 5.5+</li> 
          <li>Safari 3+</li> 
          <li>Google Chrome 4+</li> 
          <li>Firefox 3+</li> 
          <li>Opera 10.61+</li> 
          <li>iPhone Safari</li> 
          <li>iPad Safari</li> 
          <li>Android WebKit</li> 
          <li>WebOs WebKit</li> 
         </ul> 
         <p>It also offers very easy functions to communicate between the server and the client, on both sides.</p> 
         <p>Let's start by installing the three packages we will need.</p> 
         <hr> 
         <h2>Installing Our Dependencies</h2> 
         <p>Npm allows us to install packages very fast, using one line, so first go to your directory and have npm download the needed packages:</p> 
         <pre class="brush: bash noskimlinks noskimwords">npm install express jade socket.io</pre> 
         <figure class="tutorial_image"> 
          <img src="https://cdn.tutsplus.com/net/uploads/2013/09/2.jpg" data-original-url="http://cdn.tutsplus.com/net.tutsplus.com/uploads/2013/09/2.jpg" alt="2" width="600" height="309" class="alignnone size-full wp-image-34484">
          <br> 
         </figure> 
         <p>Now we can start building our server-side controller to serve the main page.</p> 
         <p>We are going to save all the server-side code into a <code>"server.js"</code> file which will be executed by Node.js.</p> 
         <hr> 
         <h2>Serving a Single Static Page</h2> 
         <figure class="tutorial_image"> 
          <img src="https://cdn.tutsplus.com/net/uploads/2013/09/7.jpg" data-original-url="http://cdn.tutsplus.com/net.tutsplus.com/uploads/2013/09/7.jpg" alt="7" width="600" height="82" class="alignnone size-full wp-image-34489">
          <br> 
         </figure> 
         <p>To serve our static page we will use <a href="http://expressjs.com/">Express</a>, a package which simplifies the whole server-side page send process.</p> 
         <p>So let's include this package into our project and start the server:</p> 
         <pre class="brush: js noskimlinks noskimwords">
var express = require('express'), app = express.createServer();</pre> 
         <p>Next, we need to configure Express to serve the page from the repertory views with the Jade templating engine that we installed earlier.</p> 
         <p>Express uses a layout file by default, but we don't need it because we will only serve one page, so instead, we will disable it.</p> 
         <p>Express can also serve a static repertory to the client like a classic web server, so we will send a <code>"public"</code> folder which will contain all our JavaScript, CSS and image files.</p> 
         <pre class="brush: js noskimlinks noskimwords">
app.set('views', __dirname + '/views');
app.set('view engine', 'jade');
app.set("view options", { layout: false });
app.configure(function() {
	app.use(express.static(__dirname + '/public'));
});</pre> 
         <p>Next, let's create two folders inside our project folder named <code>"public"</code> and <code>"views"</code>.</p> 
         <p>Now we just need to configure Express to serve a <code>"home.jade"</code> file, which we will create in a moment, and then set Express to listen for a specific port.</p> 
         <p>I will use port 3000 but you can use whatever you'd prefer.</p> 
         <pre class="brush: js noskimlinks noskimwords">
app.get('/', function(req, res){
  res.render('home.jade');
});
app.listen(3000);</pre> 
         <hr> 
         <h2>Creating the Jade Template Page</h2> 
         <figure class="tutorial_image"> 
          <img src="https://cdn.tutsplus.com/net/uploads/2013/09/3.jpg" data-original-url="http://cdn.tutsplus.com/net.tutsplus.com/uploads/2013/09/3.jpg" alt="3" width="600" height="98" class="alignnone size-full wp-image-34485">
          <br> 
         </figure> 
         <p>Node.js uses templating engines to serve webpages. It's useful to send dynamic pages and to build them faster.</p> 
         <p>In this tutorial, we will use <a href="http://jade-lang.com/">Jade</a>. Its syntax is very clear and it supports everything we need.</p> 
         <blockquote> 
          <p> "Jade is a high performance templating engine heavily influenced by Haml and implemented with JavaScript for Node."</p> 
         </blockquote> 
         <p>Now, I'm not going to go over Jade in detail, if you need more help, you can find very well written documentation on its <a href="https://github.com/visionmedia/jade#readme">Github repo</a>.</p> 
         <h3>Jade Configuration</h3> 
         <p>We installed Jade earlier, but we need to include it in our <code>server.js</code> file like we did for Express.</p> 
         <p>By convention, we include our libraries at the top of our file to use them later, without having to check if they are already included. So place the following code at the top of your <code>"server.js"</code> file :</p> 
         <pre class="brush: js noskimlinks noskimwords">
var jade = require('jade');</pre> 
         <p>And that completes our Jade configuration. Express is already setup to use Jade with our view files, to send an HTML response, we just need to create that file.</p> 
         <h3>Creating Our Home Page</h3> 
         <p>If we start our server now it will crash because we're requesting our app to send a page which doesn't exist yet.</p> 
         <p>We're not going to create a full featured page, just something basic that has a title, a container for the messages, a text area, a send button, and a user counter.</p> 
         <p>Go ahead and create a <code>"home.jade"</code> page inside the <code>"views"</code> folder with the following code:</p> 
         <pre class="brush: plain noskimlinks noskimwords">
doctype 5
html
	head
		title Chat
		script(src='https://ajax.googleapis.com/ajax/libs/jquery/1.7.2/jquery.min.js')
		script(src="/socket.io/socket.io.js")
		script(src="script.js")
	body
		div.container
			header
				h1 A Chat application with Node.js and Socket.io
			input(type='text')#pseudoInput
			button#pseudoSet Set Pseudo
			div#chatEntries
			div#chatControls
				input(type='text')#messageInput
				button#submit Send</pre> 
         <blockquote class="pullquote pqRight"> 
          <p> "Jade is all about indentation"</p> 
         </blockquote> 
         <p>The Jade language is all about indentation. As you can see, we don't need to close our containers, just indenting the children of the parent container is enough.</p> 
         <p>We also use a period <code>"."</code> and a pound sign <code>"#"</code> to indicate the class or ID of the element, just like in a CSS file.</p> 
         <p>We also link in three scripts at the top of the file. The first is jQuery from Google CDN, next we have the Socket.io script which is served automatically by the package, and finally a <code>"script.js"</code> file which will keep all of our custom JS functions.</p> 
         <hr> 
         <h2>The Socket.io Server-Side Configuration</h2> 
         <figure class="tutorial_image"> 
          <img src="https://cdn.tutsplus.com/net/uploads/2013/09/6.jpg" data-original-url="http://cdn.tutsplus.com/net.tutsplus.com/uploads/2013/09/6.jpg" alt="6" width="600" height="168" class="alignnone size-full wp-image-34488">
          <br> 
         </figure> 
         <p> Socket.io is event based, just like Node. It aims to make real-time apps possible in every browser and mobile device, blurring the lines between these different transport mechanisms. It's care-free, real-time, and 100% JavaScript.</p> 
         <p>Like the other modules, we need to include it in our <code>server.js</code> file. We will also chain on our express server to listen for connections from the same address and port.</p> 
         <pre class="brush: js noskimlinks noskimwords">
var io = require('socket.io').listen(app);</pre> 
         <p>The first event we will use is the connection event. It is fired when a client tries to connect to the server; Socket.io creates a new socket that we will use to receive or send messages to the client.</p> 
         <p>Let's start by initializing the connection:</p> 
         <pre class="brush: js noskimlinks noskimwords">
io.sockets.on('connection', function (socket) {
	//our other events...
});</pre> 
         <p>This function takes two arguments, the first one is the event and the second is the callback function, with the socket object.</p> 
         <p>Using code like this, we can create new events on the client and on the server with Socket.io. We will set the <code>"pseudo"</code> event and the <code>"message"</code> event next.</p> 
         <p>To do this, it's really simple, we just use the same syntax, but this time with our <code>socket</code> object and not with the <code>"io.sockets"</code> (with an "s") object. This allows us to communicate specifically with one client.</p> 
         <p>So inside our connection function, let's add in the <code>"pseudo"</code> event code.</p> 
         <pre class="brush: js noskimlinks noskimwords">
socket.on('setPseudo', function (data) {
	socket.set('pseudo', data);
});</pre> 
         <p>The callback function takes one argument, this is the data from the client and in our case it contains the <code>pseudo</code>. With the <code>"set"</code> function, we assign a variable to the socket. The first argument is the name of this variable and the second is the value.</p> 
         <p>Next, we need to add in the code for the <code>"message"</code> event. It will get the user's pseudo, broadcast an array to all clients which contains the message we received as well as the user's pseudo and log it into our console.</p> 
         <pre class="brush: js noskimlinks noskimwords">
socket.on('message', function (message) {
	socket.get('pseudo', function (error, name) {
		var data = { 'message' : message, pseudo : name };
		socket.broadcast.emit('message', data);
		console.log("user " + name + " send this : " + message);
	})
});</pre> 
         <p>This completes our server-side configuration. If you'd like, you can go ahead and use other events to add new features to the chat.</p> 
         <p>The nice thing about Socket.io is that we don't have to worry about handling client disconnections. When it disconnects, Socket.io will no longer receive responses to "heartbeat" messages and will deactivate the session associated with the client. If it was just a temporary disconnection, the client will reconnect and continue with the session.</p> 
         <hr> 
         <h2>The Socket.io Client-Side Configuration</h2> 
         <p>Now that our server is configured to manage messages, we need a client to send them.</p> 
         <p>The client-side of Socket.io is almost the same as the server-side. It also works with custom events and we will create the same ones as on the server.</p> 
         <p>So first, create a <code>"script.js"</code> file inside the <code>public</code> folder. We will store all of our functions inside of it.</p> 
         <p>We first need to start the socket.io connection between the client and the server. It will be stored in a variable, which we will use later to send or receive data. When the connection is not passed any arguments, it will automatically connect to the server which will serve the page.</p> 
         <pre class="brush: js noskimlinks noskimwords">
var socket = io.connect();</pre> 
         <p>Next, let's create some helper functions that we will need later. The first is a simple function to add a message to the screen with the user's pseudo.</p> 
         <pre class="brush: js noskimlinks noskimwords">
function addMessage(msg, pseudo) {
	$("#chatEntries").append('&lt;div class="message"&gt;&lt;p&gt;' + pseudo + ' : ' + msg + '&lt;/p&gt;&lt;/div&gt;');
}</pre> 
         <p>This helper uses the <a href="http://api.jquery.com/append/">append</a> function from jQuery to add a <code>div</code> at the end of the <code>#chatEntries div</code>.</p> 
         <p>Now we are going to write a function that we can call when we want to send a new message.</p> 
         <pre class="brush: js noskimlinks noskimwords">
function sentMessage() {
	if ($('#messageInput').val() != "") 
	{
		socket.emit('message', $('#messageInput').val());
		addMessage($('#messageInput').val(), "Me", new Date().toISOString(), true);
		$('#messageInput').val('');
	}
}</pre> 
         <p>First, we verify that our textarea is not empty, then we send a packet named <code>"message"</code> to the server which contains the message text, we print it on the screen with our <code>"addMessage"</code> function, and finally we remove all the text from the textarea.</p> 
         <p>Now, when the client opens the page, we need to set the user's pseudo first. This function will send the pseudo to the server and <a href="http://api.jquery.com/show/">show</a> the textarea and the submit button.</p> 
         <pre class="brush: js noskimlinks noskimwords">
function setPseudo() {
	if ($("#pseudoInput").val() != "")
	{
		socket.emit('setPseudo', $("#pseudoInput").val());
		$('#chatControls').show();
		$('#pseudoInput').hide();
		$('#pseudoSet').hide();
	}
}</pre> 
         <p>Additionally, we <a href="http://api.jquery.com/hide/">hide</a> the pseudo setting controls when it's sent to the server.</p> 
         <p>Now just like we did on the server-side, we need to make sure we can receive incoming messages and this time we'll print them on the screen. We'll use the same syntax but this time we call the <code>"addMessage"</code> function.</p> 
         <pre class="brush: js noskimlinks noskimwords">
socket.on('message', function(data) {
	addMessage(data['message'], data['pseudo']);
});</pre> 
         <p>Just like with our server configuration, the packet that is sent to the client is an array containing the message and the pseudo. So we just call our <code>"addMessage"</code> function passing in the message and the pseudo, which we extract from the received data packet.</p> 
         <p>Now we just need to add the initialization function which is fired once the page is fully loaded.</p> 
         <pre class="brush: js noskimlinks noskimwords">
$(function() {
	$("#chatControls").hide();
	$("#pseudoSet").click(function() {setPseudo()});
	$("#submit").click(function() {sentMessage();});
});</pre> 
         <p>First, we hide the chat controls before the pseudo is set and then we set two click listeners which listen for clicks on our two submit buttons. The first is for the pseudo and the second is for the messages.</p> 
         <p>And that wraps up our client-side script.</p> 
         <hr> 
         <h2>Conclusion</h2> 
         <p>We now have a working chat service. To start it, just run the following command :</p> 
         <pre class="brush: bash noskimlinks noskimwords">
node server.js</pre> 
         <p>In your terminal you should get a message from Socket.io saying that the server is started. To see your page go to <code>127.0.0.1:3000</code> (or whichever port you chose previously).</p> 
         <figure class="tutorial_image"> 
          <img src="https://cdn.tutsplus.com/net/uploads/2013/09/4.png" data-original-url="http://cdn.tutsplus.com/net.tutsplus.com/uploads/2013/09/4.png" alt="4" width="600" height="270" class="alignnone size-full wp-image-34486">
          <br> 
         </figure> 
         <p>The design is very basic, but you could easily add in a stylesheet with CSS3 transitions for incoming messages, HTML5 sounds, or <a href="http://twitter.github.com/bootstrap/">Bootstrap</a> from Twitter.</p> 
         <p>As you can see, the server and client scripts are fairly similar: this is the power of Node.js. You can build an application without having to write the code twice.</p> 
         <p>Finally, you may have noticed that it only took 25 lines of code inside our <code>server.js</code> file to create a functional chat app, with amazing performance. It is very short, but it also works very well.</p> 
         <p>Now if you're interested, I have created a better chat service application, with a good looking design, along with some additional features. It is hosted on <a href="http://geek-chat.nodester.com/">Nodester</a> and the source code is on <a href="https://github.com/geekuillaume/Node.js-Chat">Github</a>.</p> 
         <p>Here is a preview of it.</p>
