          Task
         </dt> 
         <dd></dd>
        </dl> 
        <p>Write a server for a minimal text based chat. </p>
        <p>People should be able to connect via ‘telnet’, sign on with a nickname, and type messages which will then be seen by all other connected users. Arrivals and departures of chat members should generate appropriate notification messages. <br><br> </p>

        <h2><span class="mw-headline" id="C"><a href="/wiki/Category:C" title="Category:C">C</a></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/mw/index.php?title=Chat_server&amp;action=edit&amp;section=2" title="Edit section: C">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
        <p>C has no built-in networking functions, but the POSIX library does provide some low-level networking functions. The functions of interest relating to sockets include <i>bind</i>, <i>listen</i>, <i>select</i>, <i>accept</i>, <i>read</i>, <i>write</i> and <i>close</i>. </p>
        <p>The example below was compiled on Cygwin, and accepts PuTTY connections under the RAW protocol. </p>
        <p>A glitch occurs if a connection is made using the Telnet protocol - user names are preceded by garbled text. </p>

        <h2><span class="mw-headline" id="Go"><a href="/wiki/Category:Go" title="Category:Go">Go</a></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/mw/index.php?title=Chat_server&amp;action=edit&amp;section=6" title="Edit section: Go">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
        <p>This example uses the Go idiom of <a rel="nofollow" class="external text" href="http://blog.golang.org/share-memory-by-communicating"><i>Do not communicate by sharing memory; instead, share memory by communicating</i></a>; there are no explicit locks used, instead Go channels are used to safely synchronize where required. </p>
        <p>A similar exercise of a chat roulette (different in that messages only have to be written to a single partner rather than broadcast, this simplifies the code greatly) was the topic of a <a rel="nofollow" class="external text" href="http://talks.golang.org/2012/chat.slide#1">2012 Go Talk</a>. </p>
        <p>This example handles the case of one specific client "falling behind" by relying on the underlying TCP stack to do a reasonable job of buffering. Once that buffer fills, a write to the that client's connection will time out and the connection will dropped. Other minor improvements would include enabling TCP keep alives, handling temporary errors from accept, and better logging. Not ideal, but it should be good enough for this example. </p>

        <p>Broadcasting of messages is done by the thread that received the message, so a bad client could potentially disrupt the server. The output buffer is set to 16K in an attempt to alleviate possible symptoms, but I'm not sure if it's effective. Server does not allow duplicate client names, and lists all users online after a successful connection. Client can type "/quit" to quit. </p>

        <p>Notes: </p> 
        <ul>
         <li> It operates asynchronously (using <tt>IO::Socket::Async</tt>), so a slow connection to one client won't affect other clients.</li> 
         <li> It accepts messages encoded in UTF-8.</li> 
         <li> It tokenizes the message streams at newline boundaries (using the <tt>Supply.lines</tt> method), which I think makes the most sense for a chat application.</li>
        </ul>

        <p>On my Linux box, telnet seems to only want to connect to port 23. </p> 
        <pre class="text highlighted_source">const PORT=23;<br>&nbsp;<br>var users=Dictionary(); // ( handle:socket, ...)<br>pipe:=Thread.Pipe(); // how server tells thread to connect to user<br>&nbsp;<br>fcn accept(pipe){ // a thread waiting for the server to send a socket<br>   while(socket:=pipe.read()){<br>      println("Somebody is connecting ...");<br>      socket.read();  // telnet stuff<br>      while(True){    // get credentials<br>	 reg name;<br>	 socket.write("Your handle: ");  // bottle neck<br>	 try{ name = socket.read().text.strip() } catch(IOError){ continue }<br>	 if(users.holds(name)) socket.write("Handle is already in use.\n");<br>	 else if(name){<br>	    users[name] = socket;<br>	    chat.launch(name,socket);	// thread<br>	    broadcast(name, "+++&nbsp;%s arrived +++".fmt(name));<br>	    break;	// wait for next connection<br>	 }<br>      }//while<br>   }//while<br>}.launch(pipe); // thread<br>&nbsp;<br>fcn chat(name,socket){		// a thread, one per user<br>   try{<br>      socket.write("^D to disconnect\n");<br>      while(True){<br>	 message:=socket.read().text.strip();<br>	 if(message=="\xff\xec") break; // ^D to disconnect.<br>	 broadcast(name, "%s&gt;&nbsp;%s".fmt(name,message));<br>      }<br>   }catch{}  // eg socket pukes<br>   users.del(name); socket.close(); <br>   broadcast(name, "---&nbsp;%s leaves ---".fmt(name));<br>}<br>&nbsp;<br>    // Send a message to all users from the given name.<br>fcn broadcast(name, message){ // called from user thread<br>   println(message);  // log message to server console<br>   users.pump(Void,'wrap([(toName,socket)]){<br>      if(toName&nbsp;!= name) try{ socket.write(message + "\n") } catch(IOError){}<br>   });<br>}<br>&nbsp;<br>   // Set up the server socket.<br>server:=Network.TCPServerSocket.open(PORT);<br>println("Listening on&nbsp;%s:%s".fmt(server.hostname,server.port));<br>server.listen(pipe);  // Main event loop </pre>
