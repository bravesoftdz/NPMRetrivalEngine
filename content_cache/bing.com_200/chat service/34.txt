               <div align="center"></div> <p><b><u><font color="#000099" size="+1" face="Arial"><br> <a name="install"></a>How to install jzChat</font></u><font color="#000099" size="+1" face="Arial"> ?</font></b></p> <p> </p>
               <center> 
                <p><b><font face="Arial">Download the package <font color="#FF0000">v1.12</font> <a href="jzChat1.12.zip" target="_self">here</a> (39.9 KB - zip file).</font></b><b><font face="Arial"><br> <br> </font></b></p> 
                <p align="left"><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>To install jzChat, your <font color="#FF0000">WebServer or ISP has to support JAVA SERVLETS</font></b><font color="#FF0000"> </font>(note that servlets support is not CGI support). Click <a href="http://www.servlets.com/isps/" target="_blank">here</a> to get a list. Some are free.</font></p> 
                <p align="left"><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"> The package is built as follows : <br> </font></p> 
                <div align="left"></div> 
                <table width="100%" border="0" cellspacing="0" cellpadding="0"> 
                 <tbody>
                  <tr> 
                   <td width="50%" valign="top"> <p><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>servlet/</b> <br> This directory contains all class files needed to run jzChat :</font></p> 
                    <ul> 
                     <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1">jzchat.class</font><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"> (the servlet)</font></li> 
                     <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1">jzchat.initArgs</font><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"> (servlet init parameter file, Jserv/JavaWebServer only)</font></li> 
                     <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1">jzChat/*.class (core)</font></li> 
                     <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1">jzChat/Util/*.class (core) </font> </li> 
                    </ul> </td> 
                   <td width="50%" valign="top"> <p><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>web/</b> <br> This directory contains all files needed for jzChat configuration and customization :</font></p> 
                    <ul> 
                     <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1" color="#CC0000">jzchat.conf</font><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"> (chat configuration file)</font></li> 
                     <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1">adminlogin.jsp (root login web page)</font></li> 
                     <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1">openchatroom.jsp (root chatroom opening web page)</font></li> 
                     <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1">viewchatroom.jsp (root chatroom management web page)</font></li> 
                     <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1">chatlogin.jsp (guest login web page)</font></li> 
                     <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1">chatroomjs.jsp (chat web page)</font></li> 
                    </ul> </td>

                <div align="left">
                 <br> 
                 <font face="Verdana, Arial, Helvetica, sans-serif" size="-1">The <b>servlet/</b> directory should be placed under the servlet directory of the web server (servlets/ or servlet/ as usually). The <b>web/</b> directory could be (and should be) placed outside the web root. <font color="#FF0000">Then you have to setup <b>config</b> parameter through init parameters of your servlet engine</font>. Fill in it with jzchat.conf file (absolute path needed). If you're using JSERV servlet engine then edit jzchat.initArgs to do so.</font>
                 <br> 
                 <br> 
                </div> 
                <p align="left"><b><u><font color="#000099" size="+1" face="Arial"><a name="setup"></a>How to setup parameters</font></u><font color="#000099" size="+1" face="Arial"> ?</font></b><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"> <br> </font></p> 
                <p align="left"><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"> Edit <font color="#CC0000">jzchat.conf</font> from <b>web/</b> directory change parameters if needed. Pay attention to parameters in red :</font></p> 
               </center> 
               <ul> 
                <li> 
                 <div align="left">
                  <font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b><font color="#CC0000">STOREPATH</font></b><font color="#CC0000"> : </font><br> This parameter defines the absolute path for stored files (log files, backup file and transcript file). Here are some examples : <br> STOREPATH = /usr/local/apache/wwwroot/logs (unix os)<br> STOREPATH = c:\Apache\JServ\web (win32 os)</font>
                 </div> </li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>STOREFILE </b>: <br> Use this parameter to setup chatroom backup filename. It's a binary file saved at regular intervals. It's also used to restore chatroom after a servlet engine halt or crash.</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>TRANSCRIPTFILE</b> : <br> Use this parameter to setup chatroom transcript. It's a text file with the whole chat.</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b><font color="#CC0000">SERVLET</font></b><font color="#CC0000"> : </font><br> URI to the jzchat servlet (/servlet/jzchat or /servets/jzchat as usually).</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b><font color="#CC0000">HOSTURL</font></b><font color="#CC0000"> &lt;== deprecated since v1.10: </font><br> http://www.YourDomainName.com. HOSTURL parameter followed by SERVLET parameter must be the absolute URL of the jzChat. (e.g. HOSTURL=http://127.0.01:8080)</font></li> 
                <li><font size="-1" face="Verdana, Arial, Helvetica, sans-serif"><b><font color="#CC0000">JSPPATH</font></b><font color="#CC0000"> :</font><br> This parameter defines the absolute path for JSP files (i.e. web/).</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b><a name="modes"></a>BUFFERING</b> :<br> This parameter is linked with the two parameters below. It allows to define whether or not you use buffered-frame mode. Values could be YES or NO.<br> <font color="#000099">jzChat provides three displaying mode : <br> - Non-Framed mode (no frames)<br> - Framed mode (two frames for a more convenient usage)<br> - Buffered-Framed mode (three frames to avoid flickering effect)</font><br> Default mode is buffered-framed one.</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>CHAT</b> : <br> Chat web page (dynamic-content). This page is sent back to end user with the chat content and users listing. <u>Use buffering.jsp for buffered-framed mode, chatroom.jsp for framed mode or chatroomjs.jsp for non-framed mode</u>.</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>ROOM</b> : <br> FrameSet web page. <u>Use roombf.jsp for buffered-framed mode, room.jsp for framed mode or chatroomjs.jsp for non-framed mode </u>.</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>TEXT</b> : <br> Message frame. Not used in non-framed mode.</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>CHATBF</b> :<br> Customizable chat web page for buffered-framed mode only.<br> </font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>GUESTLOGINPAGE</b> : <br> Guest login web page (dynamic-content). This page is sent back to guest user when he asks for the servlet without parameters.</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>ADMINLOGINPAGE</b> :</font><br> <font face="Verdana, Arial, Helvetica, sans-serif" size="-1">Root login web page. This page is sent back when servlet is called with todo=adminlogin parameter then root user can login to administrate the chatroom.</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>OPENCHAT</b> : <br> Root chatroom opening web page. Once root user has been logged, he could open a chatroom through this web page.</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>VIEWCHAT</b> :<br> Root chatroom administration web page. Once root user has been logged and chatroom has been opened, root user can administrate the chatroom (enter, users listing, kick off user,close ...).</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1" color="#CC0000"><b>ADMINLOGIN</b> : </font><br> <font face="Verdana, Arial, Helvetica, sans-serif" size="-1">This parameter defines the root user login. Change it !!<br> Note that root user will be seen as @rootuserlogin in chatroom users list.<br> Guest users could not use nickname starting with a @.<br> </font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1" color="#CC0000"><b>ADMINPASSWORD</b> :</font><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><br> This parameter defines the root password. <font color="#FF0000">Change it !!</font></font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>ADMINLOGINCOMMAND</b> :</font><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"></font><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><br> This parameter defines the command (default=adminlogin) to access admin panel. You can change it to improve the security of your chatroom.</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>KEEPOPEN</b> :<br> This parameter allows the chatroom to be re-opened automatically after a servlet engine halt or crash. This feature is available only if the backup file hasn't been removed.</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b><font color="#0000FF">PRIVATEMSG</font></b> : <br> This parameter enables private messages feature for the chatroom. Values could be YES or NO. Default is YES.</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>HISTORY</b> :<br> This parameters defines history's length of the chat (max. displayed lines on user's browser).</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>AUTOREFRESHLIMIT</b> :<br> Choose the autorefresh time-limit (seconds) before user's browser asks for refresh. Value between 12 and 30 seems the best. The refresh policy depends on user's activity. The more he chats the more refresh-time is low. If he stops chatting then refresh-time converges to AUTOREFRESHLIMIT value.</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>MAXINACTIVE</b> : <br> User's session time-out (seconds). When session expires user is logged out.</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>MAXNICKNAMELENGTH</b> : <br> Admin user can choose the maximum nickname length for guest users. Default is 10.</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>MINUTESTOWAIT</b> :<br> Time interval (minutes) to wait between each backup of the chatroom.</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>DATEPATTERN</b> :<br> Date format for classic look &amp; feel.</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>MSGTIMEPATTERN</b> :<br> Time format for classic look &amp; feel.</font></li> 
                <li><font face="Verdana, Arial, Helvetica, sans-serif" size="-1"><b>CHARSET</b> :<br> Admin user can include an HTML line below &lt;head&gt; tag in all jzChat JSP files. This should be used to set up charset through <a href="#charset">Content-Type meta tag</a>. Default value is blank.<br> </font></li> 
               </ul> <p><i>JzChat also provides MultiLanguage support. You can change a few messages (login, logout, kickoff ...) from jzchat.conf file and customize/translate *.jsp web pages.</i></p> <p></p> <p> </p><p><b><u><font color="#000099" size="+1" face="Arial"><a name="customize"></a>How to customize jzChat</font></u><font color="#000099" size="+1" face="Arial"> ?</font></b></p> <p><font face="Verdana, Arial, Helvetica, sans-serif" size="-1">You can customize *.jsp files under <b>web/ </b>but you should pay attention to <font color="#FF0000"><b>&lt;%= variablename %&gt;</b></font>. <b>Do not remove them and do not remove JavaScripts !</b>.<b> </b>Dreamweaver seems to be one of the best tool to do such customization.<br> In addition to HTML customization, you can change "internal chat parameters" like fontsize, fontcolor and fontname for both mIRC and Classic look an feel. To do so have a look at the end of jzchat.conf.</font></p> <p> </p><p><b><u><font color="#000099" size="+1" face="Arial"><a name="faq"></a>F.A.Q</font></u><font color="#000099" size="+1" face="Arial"> </font></b></p> <p><font face="Arial"><b><font size="-1">How to access to the admin web panel ?</font></b><br> <font size="-1"> Try http://www.mydomain.com/servlet/jzchat?todo=adminlogin (</font><font face="Arial"><font size="-1">if your servlet is under /servlet/jzchat)</font></font><font size="-1">. You should get the root login web page.</font></font></p> <p><font face="Arial"><b><font size="-1">How to access to the guest web panel ?</font></b><br> <font size="-1"> Try http://www.mydomain.com/servlet/jzchat (</font><font face="Arial"><font size="-1">if your servlet is under /servlet/jzchat)</font></font><font size="-1">. You should get the guest login web page.</font></font></p> <p><font face="Arial"><b><font size="-1">Which servlet engine does jzChat support ?</font></b><br> <font size="-1"> jzChat V1.x is Servlet API 2.0 compliant so it should run under any servlet engine. Until now we've already passed tests it under Jwsdk, JServ, Tomcat, Jrun, JavaWebServer and Weblogic.</font></font></p> <p><font face="Arial"><b><font size="-1">How to configure jzChat.initArgs file ?</font></b><br> <font size="-1"> The jzChat.initArgs file is only needed for JSERV or JavaWebServer servlet engines.<br> You have to fill in config parameter with the absolute path to jzchat.conf e.g. : <br> Unix system : config=/usr/local/apache/web/jzchat.conf<br> Win32 system : config=c:\\Apache\\JServ\\web\\jzchat.conf</font></font></p> <p><font face="Arial"><b><font size="-1">What are init parameters of a servlet engine ?</font></b><br> <font size="-1"> Init parameters allow to pass arguments to a servlet at the first time. jzChat needs a <b>config</b> init parameter. This config's value must be </font><font face="Arial"><font size="-1"> the absolute path to jzchat.conf </font></font><font size="-1">(c.f. above).</font></font></p> <p><font face="Arial"><font face="Arial"><b><font size="-1">How to configure web.xml under Tomcat ?</font></b><br> <font size="-1"> Add the following lines under &lt;web-app&gt; tag : <br> <font size="-2" face="Verdana, Arial, Helvetica, sans-serif">&lt;servlet&gt;<br> &nbsp;&nbsp;&lt;servlet-name&gt;<br> &nbsp;&nbsp;&nbsp;&nbsp;jzchat<br> &nbsp;&nbsp;&lt;/servlet-name&gt;<br> &nbsp;&nbsp;&lt;servlet-class&gt;<br> &nbsp;&nbsp;&nbsp;jzchat<br> &nbsp;&nbsp;&lt;/servlet-class&gt;<br> &nbsp;&nbsp;&lt;init-param&gt;<br> &nbsp;&nbsp;&nbsp;&nbsp;&lt;param-name&gt;config&lt;/param-name&gt;<br> &nbsp;&nbsp;&nbsp;&nbsp;&lt;param-value&gt;d:\\anAbsoluteWin32Path\\WEB-INF\\web\\jzchat.conf&lt;/param-value&gt;<br> &nbsp;&nbsp;&nbsp;&lt;/init-param&gt;<br> &lt;/servlet&gt;</font><br> </font></font></font> </p> <p><font face="Arial"><b><font size="-1">Which Servlet API jzChat supports ?</font></b><br> <font size="-1"> jzChat V1.0x runs under Servlets API V2.0. So it should work under any servlet engine.</font></font></p> <p><font face="Arial"><b><font size="-1">Does jzChat work under application servers like weblogic, websphere, silverstream ?</font></b><br> <font size="-1"> Yes, it works. You have to pay attention of the <u>init parameter</u>. Each application server (and servlet engine) have different way to configure init parameters. So don't forget you have to fill in the <u>config</u> init parameter with the absolute path (including filename) to jzchat.conf. </font></font></p> <p><font face="Arial"><b><font size="-1">I'm running under APACHE web server, how can I add servlet support ?</font></b><br> <font size="-1"> You should install <a href="http://java.apache.org/jserv/" target="_blank">JSERV</a>. It's a free servlet engine quite powerful. You can also use <a href="http://jakarta.apache.org" target="_blank">Tomcat</a>.</font></font></p> <p><font face="Arial"><b><font size="-1">jzChat doesn't work under www.mycgiserver.com ISP ?</font></b><br> <font size="-1"> mycgiserver.com hosts servlets for free but all servlets must be included in a package named as the account. It's a build operation. mycgiserver.com doesn't support servlets that you can't rebuild. Source code of jzChat is not available so you can't rebuild it for mycgiserver.com. We don't provide support for mycgiserver.com anymore (too many requests).</font></font></p> <p><font face="Arial"><b><font size="-1"><a name="charset"></a>How to change charset ?</font></b><br> <font size="-1"> charset could be modified for each web page through CHARSET parameter. You should use <a href="#modes">FRAMED mode</a> (or NON FRAMED mode) for charset different from iso-8859-1. For instance, to have a BIG5 charset then add the following CHARSET line in jzchat.conf :<br> <font color="#000066">CHARSET = &lt;meta http-equiv="Content-Type" content="text/html; charset=Big5"&gt;</font></font></font></p></td>
