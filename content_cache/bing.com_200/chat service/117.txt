      <p><b>Internet Relay Chat services</b> (usually called <b>IRC services</b>) is a name for a set of features implemented on many modern <a href="/wiki/Internet_Relay_Chat" title="Internet Relay Chat">Internet Relay Chat</a> networks. Services are automated <a href="/wiki/Internet_bot" title="Internet bot">bots</a> with special status which are generally used to provide users with access with certain privileges and protection. They usually implement some sort of login system so that only people on the <a href="/wiki/Access_control_list" title="Access control list">access control list</a> can obtain these services.</p>

      <h2><span class="mw-headline" id="Components">Components</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IRC_services&amp;action=edit&amp;section=1" title="Edit section: Components">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>The most popular style of services uses a number of fake clients whose names end in "Serv". The four below are nearly always seen in services of this style.</p> 
      <ul> 
       <li><i>ChanServ</i>, a channel service bot, is used to protect <a href="/wiki/IRC_channel_operator" class="mw-redirect" title="IRC channel operator">channel operators</a> and help run the channel. This is convenient for the operators because they do not need to have an operator on the channel all of the time to keep their status.</li> 
       <li><i>NickServ</i>, a nickname service bot, is used to register and protect user nicknames.</li> 
       <li><i>MemoServ</i>, a memo service bot, is used to record and deliver messages to users who are currently offline.</li> 
       <li><i>OperServ</i> is used by <a href="/wiki/IRC_operator" title="IRC operator">IRC operators</a> to perform administrative functions. Also known as AdminServ, RootServ, or OpServ (<a href="/w/index.php?title=Srvx&amp;action=edit&amp;redlink=1" class="new" title="Srvx (page does not exist)">srvx</a>).</li> 
      </ul> 
      <p>Some others are also seen following this naming convention in some services packages including:</p> 
      <ul> 
       <li><i>BotServ</i>, a bot which allows channel operators to assign bots to their channels. These bots are mostly a ChanServ representative in the channel.</li> 
       <li><i>HelpServ</i>, the help service bot, is used to provide help on a variety of IRC topics.</li> 
       <li><i>HostServ</i>, a host service bot, is used to allow general users to register a <a href="/wiki/Vhost_(IRC)" class="mw-redirect" title="Vhost (IRC)">vhost</a> so that their real <a href="/wiki/IP_address" title="IP address">IP address</a> on IRC is hidden.</li> 
       <li><i>RootServ</i>, used on specific networks, is utilized by Services Root Administrators to perform Root Administrative functions for the IRC Network and Services Program.</li> 
       <li><i>SpamServ</i>, used to protect channels against spam.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup></li> 
       <li><i>StatServ</i>, a statistic services bot, is used to perform various statistical analysis on the IRC Network.</li> 
      </ul> 
      <p>Other commonly used services bots:</p> 
      <ul> 
       <li><i>Global</i>, a message agent, is a specific service that is used to send global messages via services. This agent is usually given a global IRC Operator status as to be able to perform the function. This service must be on a <a href="/wiki/U-line" class="mw-redirect" title="U-line">U-lined</a><sup class="noprint Inline-Template" style="margin-left:0.1em; white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Please_clarify" title="Wikipedia:Please clarify"><span title="The text near this tag may need clarification or removal of jargon. (February 2012)">clarification needed</span></a></i>]</sup> server so it has the ability to spoof nicknames. This service is often renamed to the name of the network (e.g. <i><a href="/wiki/DalNet" class="mw-redirect" title="DalNet">DalNet</a></i> or <i><a href="/w/index.php?title=Xevium&amp;action=edit&amp;redlink=1" class="new" title="Xevium (page does not exist)">Xevium</a></i>.).</li> 
      </ul> 
      <h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IRC_services&amp;action=edit&amp;section=2" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>The original <b>NickServ</b> was written by <a href="/w/index.php?title=Armin_Gruner&amp;action=edit&amp;redlink=1" class="new" title="Armin Gruner (page does not exist)">Armin Gruner</a> at the <a href="/wiki/Munich_University_of_Technology" class="mw-redirect" title="Munich University of Technology">Munich University of Technology</a> in 1990.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> The original intention was to give out friendly warnings that somebody else already likes to use a specific <a href="/wiki/Nickname" title="Nickname">nickname</a>, since the IRC protocol has no built-in provision to protect electronic identities.</p> 
      <p><i>NoteServ</i> and <i>MsgServ</i> followed up soon after the introduction of NickServ, and had a similar job to today's <i>MemoServ</i>.</p> 
      <p>It was considered acceptable behavior simply to <i>/ignore</i> NickServ's notices, but an operator decided to <i>/kill NickServ</i> and use the nickname <i>NickServ</i> himself, subsequently collecting all <i>identify</i> passwords from users and being amused by that. This led to a public debate over the usefulness and safety of services, the discontinuation of NickServ on EFnet and <a href="/wiki/EFnet" title="EFnet">EFnet</a>'s current <i>Nicknames are not owned</i> policy arose.</p> 
      <p>In 1995 <a href="/wiki/DALnet" title="DALnet">DALnet</a> invented <b>ChanServ</b>, then re-implemented <i>NickServ</i> and <i>MemoServ</i>. Today, EFnet and <a href="/wiki/IRCnet" title="IRCnet">IRCnet</a> are the last large IRC networks which stick to their policy of living without identity protection. Some debate still remains over the value and benefits of Services on IRC, although they are generally considered essential to providing a pleasant chatting experience. This is proven in the fact that most service-less networks have died out.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2010)">citation needed</span></a></i>]</sup></p> 
      <p>Many variants of services have been implemented over time, so each IRC network speaks a different service dialect, making it close to impossible for client developers to implement unified graphical interfaces to the various registration procedures. Still, most of them are inspired by DALnet's and Armin's original works, only <a href="/wiki/Undernet" title="Undernet">Undernet</a> and <a href="/wiki/QuakeNet" title="QuakeNet">QuakeNet</a> stick out with a completely different approach.</p> 
      <h2><span class="mw-headline" id="ChanServ">ChanServ <span id="ChanServ"></span></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IRC_services&amp;action=edit&amp;section=3" title="Edit section: ChanServ">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p><b>ChanServ</b>, on many <a href="/wiki/IRC" class="mw-redirect" title="IRC">IRC</a> networks, is an IRC service which maintains <a href="/wiki/Internet_Relay_Chat#Channels_and_modes" title="Internet Relay Chat">channel</a> registration and access information.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup> If a channel is registered with ChanServ, its owners (and those they have designated) can use ChanServ to obtain control over the channel, gaining <a href="/wiki/IRC_channel_operator" class="mw-redirect" title="IRC channel operator">channel operator</a> privileges. Most channel service bots also allow other sorts of channel management options, such as topic locks (in addition to +t), mode locks, and providing users automatically with operator status, half-operator status, voice, or other access, as designated by the channel owners. They may also provide some form of automated ban-list management.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup></p> 
      <p>There are several implementations of IRC services which have a ChanServ bot; some other services use other names, such as <b>CS</b>. The exact commands and features available depend on the service implementation and the network in question.</p>

      <h2><span class="mw-headline" id="OperServ">OperServ <span id="OperServ"></span><span id="AKill"></span></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IRC_services&amp;action=edit&amp;section=5" title="Edit section: OperServ">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p><b>OperServ</b> is a component of most IRC services packages used to help administer an <a href="/wiki/Internet_Relay_Chat" title="Internet Relay Chat">Internet Relay Chat</a> network. It provides <a href="/wiki/IRC_operator" title="IRC operator">IRC operators</a> with functions to help monitor and maintain the network, manage the other services, and deal with problematic users.</p> 
      <p>Some of the more infamous functions of OperServ are the "lines", such as the <a href="/wiki/K-line_(IRC)" class="mw-redirect" title="K-line (IRC)">K-line</a> and <a href="/wiki/G-line" class="mw-redirect" title="G-line">G-line</a>. These are used to discipline troublesome network users, however, other network lines are available for use. The exact features available depend upon a particular operator's permissions, and the specific implementation in use.</p> 
      <p>On <a href="/wiki/DALnet" title="DALnet">DALnet</a> a similar concept known as an "AKill" was used instead of a G-line. The term AKill comes from an earlier implementation in which the IRC services would automatically "kill" (disconnect) the user remotely upon login, rather than the individual servers simply denying the connection.</p> 
      <p>On <a href="/wiki/OpenNap" title="OpenNap">OpenNap</a> networks, OperServ is a pseudo-user which allows users above a specific permission level to execute server administration commands. Although primarily intended to supplement clients without built-in support for such actions, there are no restrictions on its use.</p> 
      <h2><span class="mw-headline" id="IRC_services_software_packages">IRC services software packages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=IRC_services&amp;action=edit&amp;section=6" title="Edit section: IRC services software packages">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>There are several <a href="/wiki/Computer_software" class="mw-redirect" title="Computer software">software</a> packages that implement services. Some of the popular ones include <b>srvx</b>, <b>Atheme</b>, <b>ChatServices</b>, Hybserv (developed for use with the <b>Hybrid IRCd</b> software), <b>Epona</b>, <b>Anope</b>, and <b>IRC Services 5</b> (supporting a wide range of IRC daemons, considered the "original" IRC Services). While some of these packages provide excellent features in the eyes of many users, some network operators decide to write their own services, with the most important reasons being flexibility, customization, and often, mostly on smaller networks, fun also plays an important role. Not all packages implement the above service bots. Also, sometimes the bots have different names, and may or may not implement these functions in the form listed above. Even so, the above form is the most common.</p>

      <p>The IRC+ initiative was working on producing recommendations for services to use common numerics for essential functions, so that clients could have unified and simplified user interfaces for dealing with services for the most essential functions (like, registering an identity on an IRC network). However, due to lack of support, the IRC+ initiative have ceased their work.</p>
