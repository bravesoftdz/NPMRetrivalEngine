        <article class="markdown-body entry-content" itemprop="text">
         <h1><a id="user-content-node-speaker" class="anchor" href="#node-speaker" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>node-speaker</h1> 
         <h2><a id="user-content-output-pcm-audio-data-to-the-speakers" class="anchor" href="#output-pcm-audio-data-to-the-speakers" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Output <a href="http://en.wikipedia.org/wiki/Pulse-code_modulation">PCM audio</a> data to the speakers</h2> 
         <p><a href="https://travis-ci.org/TooTallNate/node-speaker"><img src="https://camo.githubusercontent.com/2f74ebc3999b2436ff777386d07669bdc134b0ef/68747470733a2f2f7365637572652e7472617669732d63692e6f72672f546f6f54616c6c4e6174652f6e6f64652d737065616b65722e737667" alt="Build Status" data-canonical-src="https://secure.travis-ci.org/TooTallNate/node-speaker.svg" style="max-width:100%;"></a> <a href="https://ci.appveyor.com/project/TooTallNate/node-speaker"><img src="https://camo.githubusercontent.com/042c0d9ee8c0f8b3a726bebb75433036ec06b6c2/68747470733a2f2f63692e6170707665796f722e636f6d2f6170692f70726f6a656374732f7374617475732f77697837776d6c337635353637306b773f7376673d74727565" alt="Build Status" data-canonical-src="https://ci.appveyor.com/api/projects/status/wix7wml3v55670kw?svg=true" style="max-width:100%;"></a></p> 
         <p>A Writable stream instance that accepts <a href="http://en.wikipedia.org/wiki/Pulse-code_modulation">PCM audio</a> data and outputs it to the speakers. The output is backed by <code>mpg123</code>'s audio output modules, which in turn use any number of audio backends commonly found on Operating Systems these days.</p> 
         <h2><a id="user-content-installation" class="anchor" href="#installation" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Installation</h2> 
         <p>Simply compile and install <code>node-speaker</code> using <code>npm</code>:</p> 
         <div class="highlight highlight-source-shell">
          <pre>npm install speaker</pre>
         </div> 
         <p>On Debian/Ubuntu, the <a href="http://www.alsa-project.org/">ALSA</a> backend is selected by default, so be sure to have the <code>alsa.h</code> header file in place:</p> 
         <div class="highlight highlight-source-shell">
          <pre>sudo apt-get install libasound2-dev</pre>
         </div> 
         <h2><a id="user-content-example" class="anchor" href="#example" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Example</h2> 
         <p>Here's an example of piping <code>stdin</code> to the speaker, which should be 2 channel, 16-bit audio at 44,100 samples per second (a.k.a CD quality audio).</p> 
         <div class="highlight highlight-source-js">
          <pre><span class="pl-k">const</span> <span class="pl-c1">Speaker</span> <span class="pl-k">=</span> <span class="pl-c1">require</span>(<span class="pl-s"><span class="pl-pds">'</span>speaker<span class="pl-pds">'</span></span>);

<span class="pl-c"><span class="pl-c">//</span> Create the Speaker instance</span>
<span class="pl-k">const</span> <span class="pl-c1">speaker</span> <span class="pl-k">=</span> <span class="pl-k">new</span> <span class="pl-en">Speaker</span>({
  channels<span class="pl-k">:</span> <span class="pl-c1">2</span>,          <span class="pl-c"><span class="pl-c">//</span> 2 channels</span>
  bitDepth<span class="pl-k">:</span> <span class="pl-c1">16</span>,         <span class="pl-c"><span class="pl-c">//</span> 16-bit samples</span>
  sampleRate<span class="pl-k">:</span> <span class="pl-c1">44100</span>     <span class="pl-c"><span class="pl-c">//</span> 44,100 Hz sample rate</span>
});

<span class="pl-c"><span class="pl-c">//</span> PCM data from stdin gets piped into the speaker</span>
<span class="pl-c1">process</span>.<span class="pl-smi">stdin</span>.<span class="pl-en">pipe</span>(speaker);</pre>
         </div> 
         <h2><a id="user-content-api" class="anchor" href="#api" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>API</h2> 
         <p><code>require('speaker')</code> directly returns the <code>Speaker</code> constructor. It is the only interface exported by <code>node-speaker</code>.</p> 
         <h3><a id="user-content-new-speaker-format----speaker-instance" class="anchor" href="#new-speaker-format----speaker-instance" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>new Speaker([ format ]) -&gt; Speaker instance</h3> 
         <p>Creates a new <code>Speaker</code> instance, which is a writable stream that you can pipe PCM audio data to. The optional <code>format</code> object may contain any of the <code>Writable</code> base class options, as well as any of these PCM formatting options:</p> 
         <ul> 
          <li><code>channels</code> - The number of audio channels. PCM data must be interleaved. Defaults to <code>2</code>.</li> 
          <li><code>bitDepth</code> - The number of bits per sample. Defaults to <code>16</code> (16-bit).</li> 
          <li><code>sampleRate</code> - The number of samples per second per channel. Defaults to <code>44100</code>.</li> 
          <li><code>signed</code> - Boolean specifying if the samples are signed or unsigned. Defaults to <code>true</code> when bit depth is 8-bit, <code>false</code> otherwise.</li> 
          <li><code>float</code> - Boolean specifying if the samples are floating-point values. Defaults to <code>false</code>.</li> 
          <li><code>samplesPerFrame</code> - The number of samples to send to the audio backend at a time. You likely don't need to mess with this value. Defaults to <code>1024</code>.</li> 
         </ul> 
         <h4><a id="user-content-open-event" class="anchor" href="#open-event" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>"open" event</h4> 
         <p>Fired when the backend <code>open()</code> call has completed. This happens once the first <code>write()</code> call happens on the speaker instance.</p> 
         <h4><a id="user-content-flush-event" class="anchor" href="#flush-event" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>"flush" event</h4> 
         <p>Fired after the speaker instance has had <code>end()</code> called, and after the audio data has been flushed to the speakers.</p> 
         <h4><a id="user-content-close-event" class="anchor" href="#close-event" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>"close" event</h4> 
         <p>Fired after the "flush" event, after the backend <code>close()</code> call has completed. This speaker instance is essentially finished after this point.</p> 
         <h2><a id="user-content-audio-backend-selection" class="anchor" href="#audio-backend-selection" aria-hidden="true">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Audio Backend Selection</h2> 
         <p><code>node-speaker</code> is backed by <code>mpg123</code>'s "output modules", which in turn use one of many popular audio backends like ALSA, OSS, SDL, and lots more. The default backends for each operating system are described in the table below:</p> 
         <table> 
          <thead> 
           <tr> 
            <th align="left"><strong>Operating System</strong></th> 
            <th align="left"><strong>Audio Backend</strong></th> 
            <th align="left"><strong>Description</strong></th> 
           </tr> 
          </thead> 
          <tbody> 
           <tr> 
            <td align="left">Linux</td> 
            <td align="left"><code>alsa</code></td> 
            <td align="left">Output audio using <a href="http://www.alsa-project.org/">Advanced Linux Sound Architecture (ALSA)</a>.</td> 
           </tr> 
           <tr> 
            <td align="left">Mac OS X</td> 
            <td align="left"><code>coreaudio</code></td> 
            <td align="left">Output audio using Mac OS X's CoreAudio.</td> 
           </tr> 
           <tr> 
            <td align="left">Windows</td> 
            <td align="left"><code>win32</code></td> 
            <td align="left">Audio output for Windows (winmm).</td> 
           </tr> 
           <tr> 
            <td align="left">Solaris</td> 
            <td align="left"><code>sun</code></td> 
            <td align="left">Audio output for Sun Audio.</td> 
           </tr>
          </tbody>
         </table> 
         <p>To manually override the default backend, pass the <code>--mpg123-backend</code> switch to <code>npm</code>/<code>node-gyp</code>:</p> 
         <div class="highlight highlight-source-shell">
          <pre>npm install speaker --mpg123-backend=openal</pre>
         </div>
