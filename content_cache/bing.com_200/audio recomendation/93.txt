          <div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr">
           <p>Installing Eclipse is relatively easy, but does involve a few steps and software from at least two different sources. <b>Eclipse is a Java-based application</b> and, as such,&nbsp;<i><b>requires a Java runtime environment (JRE)</b></i> in order to run. </p>
           <p>Note that on recent versions of Mac, <b>a full JDK needs to be installed</b>, not just a JRE; see <a href="#Install_a_JVM">instructions</a> below. </p>
           <p><br> </p>

           <h2><span class="mw-headline" id="Eclipse_4.7_.28Oxygen.29">Eclipse 4.7 (Oxygen)</span></h2> 
           <p>Eclipse 4.7 (<a href="/Oxygen" title="Oxygen" class="mw-redirect">Oxygen</a>) was released on June 28, 2017. It is the supported release. See <a href="/Oxygen/Simultaneous_Release_Plan#Schedule" title="Oxygen/Simultaneous Release Plan">Oxygen schedule</a>. </p>
           <p>Consider using the Installer. Please see <a rel="nofollow" class="external text" href="https://www.eclipse.org/downloads/eclipse-packages/?show_instructions=TRUE">5 Steps to Install Eclipse</a>. </p>
           <p>A <a href="#Install_a_JVM"><b>Java 8</b> JRE/JDK is required</a> to run all Oxygen packages based on Eclipse 4.7, including running the Installer. The reasoning behind requiring Java 8 are discussed <a href="/Eclipse/Installation/Java8Required" title="Eclipse/Installation/Java8Required">here</a>. </p> 
           <h2><span class="mw-headline" id="Eclipse_4.6_.28Neon.29">Eclipse 4.6 (Neon)</span></h2> 
           <p>Eclipse 4.6 (<a href="/Neon" title="Neon" class="mw-redirect">Neon</a>) was released on June 22, 2016. See <a href="/Neon/Simultaneous_Release_Plan#Schedule" title="Neon/Simultaneous Release Plan">Neon schedule</a>. </p>
           <p>Consider using the Installer. Please see <a rel="nofollow" class="external text" href="https://www.eclipse.org/downloads/eclipse-packages/?show_instructions=TRUE">5 Steps to Install Eclipse</a>. </p>
           <p>A <a href="#Install_a_JVM"><b>Java 8</b> JRE/JDK is required</a> to run all Neon packages based on Eclipse 4.6, including the Installer. The reasoning behind requiring Java 8 are discussed <a href="/Eclipse/Installation/Java8Required" title="Eclipse/Installation/Java8Required">here</a>. </p> 
           <h2><span class="mw-headline" id="Eclipse_4.5_.28Mars.29">Eclipse 4.5 (Mars)</span></h2> 
           <p>Eclipse 4.5 (Mars) was released on June 24, 2015. </p>
           <p>Consider using the Installer! Please see <a rel="nofollow" class="external text" href="https://www.eclipse.org/downloads/eclipse-packages/?show_instructions=TRUE">5 Steps to Install Eclipse</a>. </p>
           <p>A <a href="#Install_a_JVM"><b>Java 7</b> JRE/JDK is <b>required</b></a> for all Mars package downloads based on Eclipse 4.5, including the Installer. Information concerning tested configurations for Eclipse 4.5 is provided <a rel="nofollow" class="external text" href="http://www.eclipse.org/eclipse/development/readme_eclipse_4.5.html#TargetOperatingEnvironments">here</a>. </p> 
           <h2><span class="mw-headline" id="Eclipse_4.4_.28Luna.29">Eclipse 4.4 (Luna)</span></h2> 
           <p>Eclipse 4.4 (Luna) was released on June 25, 2014. </p>
           <p>A <a href="#Install_a_JVM"><b>Java 7</b> JRE/JDK is <b>required</b></a> for most of the Luna package downloads based on Eclipse 4.4. Information concerning tested configurations for Eclipse 4.4 is provided <a rel="nofollow" class="external text" href="http://www.eclipse.org/eclipse/development/readme_eclipse_4.4.html#TargetOperatingEnvironments">here</a>. </p> 
           <h2><span class="mw-headline" id="Eclipse_4.3_.28Kepler.29">Eclipse 4.3 (Kepler)</span></h2> 
           <p>Eclipse 4.3 (Kepler) was released in June 2013. </p>
           <p>A <a href="#Install_a_JVM">Java 6 JRE/JDK is recommended</a> for Eclipse 4.3. More information concerning tested configurations for Eclipse 4.3 is provided <a rel="nofollow" class="external text" href="http://www.eclipse.org/eclipse/development/readme_eclipse_4.3.html#TargetOperatingEnvironments">here</a>. </p> 
           <h2><span class="mw-headline" id="Install_a_JVM">Install a JVM</span></h2> 
           <div class="messagebox" style="background-color: #f9f6b7; border: 1px solid #c4c295; color: black; padding: 5px; margin: 1ex 0; min-height: 35px; padding-left: 45px;"> 
            <div style="float: left; margin-left: -40px;">
             <a href="/File:Warning2.png" class="image"><img alt="Warning2.png" src="/images/3/37/Warning2.png" width="35" height="35"></a>
            </div> 
            <div>
             <b>If you are using Eclipse to do Java development, install a JDK<br></b> A Java Development Kit (JDK) includes many useful extras for Java developers including the source code for the standard Java libraries.
             <b><br></b>
            </div> 
           </div> 
           <p><br> Regardless of your operating system, you will need to install some Java virtual machine (JVM). You may either install a Java Runtime Environment (JRE), or a Java Development Kit (JDK), depending on what you want to do with Eclipse. <i>If you intend to use Eclipse for Java development, then you should install a JDK</i>. If you aren't planning to use Eclipse for Java development and want to save some disk space, install a JRE. </p> 
           <ul>
            <li>If you're using Windows, you may already have a JRE installed, but upgrading won't hurt. </li> 
            <li> If you're using Mac, and you don't have a JDK installed, you may get a bogus message from the OS stating that you should "install the legacy Java SE 6 runtime". Installing that will not solve the problem, because recent versions of Eclipse require a higher version. If you install just a JRE, and not a full JDK, that error message will persist. You must install a full JDK.</li> 
            <li>If using Linux, <a href="/IRC_FAQ#I_just_installed_Eclipse_on_Linux.2C_but_it_does_not_start._What_is_the_problem.3F" title="IRC FAQ">read this</a> 
             <ul>
              <li>GCJ will <b>NOT</b> work. </li> 
              <li>Also see: <a rel="nofollow" class="external text" href="http://www.64bitjungle.com/ubuntu/install-java-jre-160-update-x-on-hardy-as-the-default-java-runtime">Installing JRE 1.6.0 (Update x) as the Default Runtime in Linux</a>.</li>
             </ul></li>
           </ul> 
           <h3><span class="mw-headline" id="JRE.2FJDK_Sources">JRE/JDK Sources</span></h3> 
           <div class="messagebox" style="background-color: #f9f6b7; border: 1px solid #c4c295; color: black; padding: 5px; margin: 1ex 0; min-height: 35px; padding-left: 45px;"> 
            <div style="float: left; margin-left: -40px;">
             <a href="/File:Warning2.png" class="image"><img alt="Warning2.png" src="/images/3/37/Warning2.png" width="35" height="35"></a>
            </div> 
            <div>
             <b>Be sure to install a JVM with the same bit level as Eclipse<br></b> i.e. install a 32-bit JRE to run 32-bit Eclipse; install a 64-bit JRE to run 64-bit Eclipse
             <b><br></b>
            </div> 
           </div> 
           <p>There are several sources for a JRE/JDK. Here are some of the more common/popular ones (listed alphabetically): </p> 
           <ul>
            <li><a rel="nofollow" class="external text" href="http://www.ibm.com/developerworks/java/jdk/index.html">IBM JDK</a> </li> 
            <li><a rel="nofollow" class="external text" href="http://openjdk.java.net/">OpenJDK</a> </li> 
            <li><a rel="nofollow" class="external text" href="http://www.oracle.com/technetwork/java/javase/downloads/index.html">Oracle JDK</a></li>
           </ul> 
           <h2><span class="mw-headline" id="Download_Eclipse">Download Eclipse</span></h2> 
           <p>Download Eclipse from the <a rel="nofollow" class="external text" href="http://www.eclipse.org/downloads">Eclipse Downloads Page</a>. </p>
           <p>There are several package choices. Note that you can install the features from any package into any other package. If you are, for example, planning to do mostly Java development and some C/C++ development, you should download the <i>Eclipse IDE for Java Developers</i> and then add the C/C++ development tools via the "Help &gt; Install New Software..." menu option. </p>
           <p>The download will be delivered as a compressed (i.e. a ".zip", or ".tar.gz") file. Decompress this file into the directory of your choice (e.g. "c:\eclipse" on Windows) and ensure you have full Read and Execute permissions. You can optionally create a shortcut of the executable file ("eclipse.exe" on Windows, or "eclipse" on Linux). </p>
           <p>Note that there is a <b>known problem</b> with the built-in decompression utility on all current versions of <b>Windows</b>. We recommend that you use a more robust decompression utility such as the open source <a rel="nofollow" class="external text" href="http://www.7-zip.org/">7zip</a> when decompressing an Eclipse download. Some people report success when initially decompressing Eclipse into a root directory (e.g. c:\) and then moving it to a more appropriate home (e.g. c:\Program Files\Eclipse) </p> 
           <h2><span class="mw-headline" id="Configure_Eclipse_to_use_the_JVM">Configure Eclipse to use the JVM</span></h2> 
           <p>It is strongly recommended to configure Eclipse with the specific JVM that you want. See the instructions at <a href="/Eclipse.ini" title="Eclipse.ini">Eclipse.ini</a> This is a very important step to be sure that Eclipse is using the JVM you intend and can't be influenced by any other software that might alter your system. The JVM used to launch Eclipse has no affect on whether it can compile Java sources for other Java language versions. </p> 
           <h2><span class="mw-headline" id="Extending_Eclipse">Extending Eclipse</span></h2> 
           <p>Use the <i>Help &gt; Install new software...</i> menu option to add Kepler features to your Eclipse installation (you can, for example, use this option to add C/C++ development support). Additionally, you can tap into a vast collection of extensions provided by the Eclipse community and ecosystem via the <b>Eclipse Marketplace Client</b> (<i>Help &gt; Eclipse Marketplace</i>). Note that not all Eclipse packages contain the Eclipse Marketplace Client. </p> 
           <h2><span class="mw-headline" id="Troubleshooting">Troubleshooting</span></h2> 
           <p><b>Java was started but returned exit code = 13</b> </p>
           <p>If you've "installed" Eclipse but are having trouble getting it to run, the most likely cause is that you have not correctly specified the JVM for it to run under. You may need to edit the <a href="/Eclipse.ini" title="Eclipse.ini">eclipse.ini file</a>. </p>
           <p>Another common mistake on Microsoft Windows is a mismatch between the "bittedness" of Eclipse and the JVM/JDK. This is the most frequent cause of an Error 13. 64-bit Eclipse requires a 64-bit JVM, and 32-bit Eclipse requires 32-bit JVM--you can not mix-and-match between 32-bit and 64-bit, so make sure the version of Eclipse you installed matches the JVM/JDK that you're using to run it (and make sure you're using eclipse.ini to specify the exact JVM used to run Eclipse, described above). </p>
           <p>As a simple test, open a Command Prompt window, move to the directory that is pointed to by the -vm argument in your eclipse.ini, and run the intended java.exe with the -d32 switch to test if it supports 32-bit, or -d64 to test for 64-bit support. It's often simplest to download a version of Eclipse that will work with whatever Java you already have installed. </p>
           <p><b>To open "Eclipse" you need to install the legacy Java SE 6 runtime</b> </p>
           <p>On more recent versions of the Mac, if you don't have a full JDK of an appropriately high version installed, the OS produces this bogus message. Installing any JRE will not eliminate this problem. A full JDK needs to be installed on the Mac. </p>
           <p><b>Extraction requires a password or otherwise fails on Windows.</b> </p>
           <p>Eclipse downloads <i>are not</i> password protected. This is a known problem with the built-in decompression utility on all current versions of Windows. We recommend that you either download the installer or use a more robust decompression utility such as the open source 7zip when decompressing an Eclipse download. Some people report success when initially decompressing Eclipse into a root directory (e.g. c:\) and then moving it to a more appropriate home (e.g. c:\Program Files\Eclipse) </p>
