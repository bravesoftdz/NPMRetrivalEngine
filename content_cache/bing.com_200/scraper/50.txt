      <p><i><b>Web scraping</b></i> es una técnica utilizada mediante <a href="/wiki/Programa_inform%C3%A1tico" title="Programa informático">programas de software</a> para extraer información de <a href="/wiki/Sitio_web" title="Sitio web">sitios web</a>. Usualmente, estos programas simulan la navegación de un humano en la <a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a> ya sea utilizando el <a href="/wiki/Hypertext_Transfer_Protocol" class="mw-redirect" title="Hypertext Transfer Protocol">protocolo HTTP</a> manualmente, o incrustando un <a href="/wiki/Navegador_web" title="Navegador web">navegador</a> en una <a href="/wiki/Aplicaci%C3%B3n_inform%C3%A1tica" title="Aplicación informática">aplicación</a>.</p> 
      <p>El <i>web scraping</i> está muy relacionado con la indexación de la web, la cual indexa la información de la web utilizando un <a href="/wiki/Bot" title="Bot">robot</a> y es una técnica universal adoptada por la mayoría de los <a href="/wiki/Motor_de_b%C3%BAsqueda" title="Motor de búsqueda">motores de búsqueda</a>. Sin embargo, el <i>web scraping</i> se enfoca más en la transformación de datos sin estructura en la web (como el formato <a href="/wiki/HTML" title="HTML">HTML</a>) en datos estructurados que pueden ser almacenados y analizados en una base de datos central, en una hoja de cálculo o en alguna otra fuente de almacenamiento. Alguno de los usos del <i>web scraping</i> son la comparación de precios en tiendas, la monitorización de datos relacionados con el clima de cierta región, la detección de cambios en sitios webs y la integración de datos en sitios webs. También es utilizado para obtener información relevante de un sitio a través de los <i><a href="/wiki/Rich_snippets" title="Rich snippets">rich snippets</a></i>.</p> 
      <p>En los últimos años el web scraping se ha convertido en una técnica muy utilizada dentro del sector del <a href="/wiki/Posicionamiento_web" class="mw-redirect" title="Posicionamiento web">posicionamiento web</a> gracias a su capacidad de generar grandes cantidades de datos para crear contenidos de calidad.<sup id="cite_ref-1" class="reference separada"><a href="#cite_note-1"><span class="corchete-llamada">[</span>1<span class="corchete-llamada">]</span></a></sup>?</p> 
      <p></p>

      <h2><span id="Técnicas"></span><span class="mw-headline" id="T.C3.A9cnicas">Técnicas</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_scraping&amp;action=edit&amp;section=1" title="Editar sección: Técnicas">editar</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p><i>Web scraping</i> es el proceso de recopilar información de forma automática de la Web. Es un campo con desarrollos activos, compartiendo un propósito en común con la visión de la <a href="/wiki/Web_sem%C3%A1ntica" title="Web semántica">Web semántica</a>. Utiliza soluciones prácticas basadas en tecnologías existentes que son comúnmente <i>ad hoc</i>. Existen distintos niveles de automatización que las existentes tecnologías de Web Scraping pueden brindar:</p> 
      <ul> 
       <li>«Copiar y pegar» humano: algunas veces incluso las mejores técnicas de <i>web scraping</i> no pueden reemplazar la examinación manual de un humano, y a veces esta puede ser la única vía de solución cuando el sitio que tenemos en mente pone ciertas barreras para prevenir que se creen softwares para realizar tareas automáticas en este.</li> 
       <li>Uso de <a href="/wiki/Expresi%C3%B3n_regular" title="Expresión regular">expresiones regulares</a>: una posible vía para extraer información de páginas webs pueden ser las expresiones regulares, aunque comúnmente no se recomienda utilizarlas para parsear el formato <a href="/wiki/HTML" title="HTML">HTML</a>.</li> 
       <li><a href="/wiki/Hypertext_Transfer_Protocol" class="mw-redirect" title="Hypertext Transfer Protocol">Protocolo HTTP</a>: páginas webs estáticas y dinámicas pueden ser obtenidas haciendo peticiones HTTP al servidor remoto utilizando sockets, etc.</li> 
       <li>Algoritmos de <a href="/wiki/Miner%C3%ADa_de_datos" title="Minería de datos">minería de datos</a>: muchos sitios webs tienen grandes colecciones de páginas generadas dinámicamente a partir de una base de datos. Datos de la misma categoría aparecen usualmente en páginas similares mediante un <i>script</i> o una plantilla. En la minería de datos, un programa detecta estas plantillas en un contexto específico y extrae su contenido.</li> 
       <li>Parsers de HTML: Algunos lenguajes, como <a href="/wiki/XQuery" title="XQuery">XQuery</a> y <a href="/w/index.php?title=HTQL&amp;action=edit&amp;redlink=1" class="new" title="HTQL (aún no redactado)">HTQL</a> pueden ser utilizados para parsear documentos, recuperar y transformar el contenido de documentos HTML.</li> 
       <li>Aplicaciones para <i>web scraping</i>: existen muchas aplicaciones disponibles que pueden ser utilizadas para personalizar soluciones de Web Scraping. Estas aplicaciones pudieran reconocer automáticamente la estructura de cierta página o brindar una interfaz al usuario donde este pudiera seleccionar los campos que son de interés dentro del documento. De esta forma no es necesario escribir manualmente código para realizar estas tareas.</li> 
       <li>Reconocimiento de información semántica: las páginas que son analizadas podrían incluir metadatos o cierta información semántica como anotaciones o comentarios, los cuales pueden ser usados comúnmente. Si estas anotaciones están en las mismas páginas, como sucede con los <a href="/wiki/Microformato" title="Microformato">microformatos</a>, estas podrían ser de utilidad cuando parseamos el <a href="/wiki/Document_Object_Model" title="Document Object Model">DOM</a> del documento. En otro caso, las anotaciones, organizadas en una capa semántica, son almacenadas y manejadas de forma separada desde otras páginas, por lo que los scrapers pueden recuperar estos esquemas y las instrucciones desde esta capa antes de analizar los documentos.</li> 
      </ul> 
      <h2><span class="mw-headline" id="Cuestiones_legales">Cuestiones legales</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_scraping&amp;action=edit&amp;section=2" title="Editar sección: Cuestiones legales">editar</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>El <i>web scraping</i> pudiera ir en contra de los términos de uso de algunos sitios webs. El cumplimiento de estos términos no está totalmente claro. Mientras que la duplicación de expresiones originales puede ser en muchos casos ilegal, en <a href="/wiki/Estados_Unidos" title="Estados Unidos">Estados Unidos</a> la corte dictó en el caso Feist Publications v. Rural Telephone Service que la duplicación de hechos es permitida. Las cortes de Estados Unidos en ciertas ocasiones han reconocido que ciertos usos de los <i>scrapers</i> no deberían estar permitidos. Podría considerarse una computadora como una propiedad personal, y de esta forma el <i>scraper</i> estaría entrando sin autorización en esta propiedad. En el caso más conocido, <a href="/w/index.php?title=EBay_vs_Bidder%27s_Edge&amp;action=edit&amp;redlink=1" class="new" title="EBay vs Bidder's Edge (aún no redactado)">eBay vs Bidder's Edge</a>, la segunda empresa tuvo que parar de realizar peticiones automáticas al sitio de eBay. En este caso, Bidder's Edge pujaba automáticamente por ciertos productos en este sitio.</p> 
      <p>Uno de las principales pruebas de scraping involucró a <a href="/wiki/American_Airlines" title="American Airlines">American Airlines</a> y a una empresa llamada FareChase. American Airlines ganó esta batalla, haciendo que FareChase parara de vender un software que le permitía a los usuarios comparar tarifas en línea si el sitio de American Airlines era incluido. La aerolínea dijo que las búsquedas de FareChase entraban sin autorización en los servidores cuando recopilaban la información públicamente disponible.</p> 
      <p><a href="/wiki/Southwest_Airlines" title="Southwest Airlines">Southwest Airlines</a> también ha sido víctima de prácticas de <i>web scraping</i>.</p> 
      <p>Aunque las decisiones actualmente tomadas no son uniformes, es difícil ignorar que un patrón está emergiendo, en el cual podemos ver que las cortes están preparándose para proteger el contenido propietario en sitios webs comerciales, previendo de esta forma que este sea utilizado sin el consentimiento de los propietarios de los sitios. Sin embargo, el grado de protección de estos contenidos aún no está establecido, y dependerá de el tipo de acceso realizado por los scrapers, de la cantidad de información recopilada y de el grado en el que afecten estos factores al propietario del sitio web.</p> 
      <h2><span class="mw-headline" id="Medidas_para_detener_a_los_scrapers">Medidas para detener a los <i>scrapers</i></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_scraping&amp;action=edit&amp;section=3" title="Editar sección: Medidas para detener a los scrapers">editar</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>El administrador de un sitio web puede utilizar varias técnicas para detener o disminuir los pedidos de los scrapers. Algunas técnicas incluyen:</p> 
      <ul> 
       <li>Añadir entradas al fichero <a href="/wiki/Est%C3%A1ndar_de_exclusi%C3%B3n_de_robots" title="Estándar de exclusión de robots">robots.txt</a>. <a href="/wiki/Google" title="Google">Google</a> y otros bots pueden ser detenidos de esta forma.</li> 
       <li>Bloquear la <a href="/wiki/Direcci%C3%B3n_IP" title="Dirección IP">dirección IP</a>. Esto también bloqueará todos los accesos desde esa misma IP, por lo que los usuarios no podrán navegar por el sitio web si acceden desde ésta.</li> 
       <li>Deshabilitar cualquier <a href="/wiki/Interfaz_de_programaci%C3%B3n_de_aplicaciones" title="Interfaz de programación de aplicaciones">interfaz de programación de aplicaciones</a> que el sitio web pudiera estar brindando.</li> 
       <li>Los bots o scrapers algunas veces declaran quienes son, y gracias a esto pueden ser bloqueados. «<a href="/wiki/Googlebot" title="Googlebot">googlebot</a>» es un ejemplo. Algunos scrapers no hacen distinción entre ellos y un navegador común.</li> 
       <li>Monitorear el exceso de tráfico proveniente de cierta IP.</li> 
       <li>Añadir un <i><a href="/wiki/Captcha" title="Captcha">captcha</a></i> u otro sistema de verificación manual al sitio web. No se garantiza el completo bloqueo de los scrapers, pero mediante esta técnica se dificulta el acceso de los mismos a los sitios webs.</li> 
       <li>Servicios comerciales antibots: algunas empresas ofrecen servicios antibots y <i>antiscraping</i>.</li> 
       <li>Incrementar el uso de <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> y <a href="/wiki/AJAX" title="AJAX">AJAX</a>. De esta forma es más difícil para los <i>scrapers</i> simular las peticiones como si fueran un navegador común.</li> 
      </ul>
