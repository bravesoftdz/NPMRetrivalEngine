    <h1 class="article_title">How do I import my own java packages and classes for use in JSP pages? In what directory should they reside? How do I put them on the CLASSPATH?</h1> 
    <div class="article_info" style="border-bottom: 1px dotted #666666;"> 
     <p>Created May 7, 2012</p> 
     <div class="socialmedia"> 
      <ul> 
       <li> 
        <!-- Twitter Share Button --> <a href="https://twitter.com/share" class="twitter-share-button" data-count="vertical">Tweet</a> <script>!function(d,s,id){var js,fjs=d.getElementsByTagName(s)[0];if(!d.getElementById(id)){js=d.createElement(s);js.id=id;js.src="//platform.twitter.com/widgets.js";fjs.parentNode.insertBefore(js,fjs);}}(document,"script","twitter-wjs");

    <div id="article_body"> 
     <strong>Byron Tymvios</strong> 
     <p></p>
     <p></p>
     <p> If your classes are packaged in a jar file you can put the jar file into the WEB-INF/lib directory of your web application. If you have unpackaged classes you can put them in the WEB-INF/classes directory, however the directory tree under WEB-INF/classes needs to look the same as the class's package. So if your classes package name is com.xyz then the directory structure should look like this web-inf/classes/com/xyz with your classes in the xyz directory.<br><br> Once the jar and/or classes are in these directories they will automatically be on the classpath, the container/application server should take care of that.<br><br> To import these classes for use in a JSP page you need to use the JSP page directive with the import attribute, and the value of the import attribute should be set to the class or classes you would like to import.<br><br> Example:<br> // To import one class<br> &lt;%@ page import="com.xyz.MyClass" %&gt;<br> OR<br> // To import multiple classes from the com.xyz package<br> &lt;%@ page import="com.xyz.*" %&gt; </p>
