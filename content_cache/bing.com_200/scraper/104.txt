   <h1 id="firstHeading" class="firstHeading" lang="de">Screen Scraping</h1> 
   <div id="bodyContent" class="mw-body-content"> 
    <div id="siteSub" class="noprint">

    <div id="mw-content-text" lang="de" dir="ltr" class="mw-content-ltr">
     <div class="mw-parser-output">
      <p>Der Begriff <b>Screen Scraping</b> (engl., etwa: „am Bildschirm schürfen“) umfasst generell alle Verfahren zum Auslesen von Texten aus Computerbildschirmen. Gegenwärtig wird der Ausdruck jedoch beinahe ausschließlich in Bezug auf <a href="/wiki/Webseite" title="Webseite">Webseiten</a> verwendet (daher auch <b>Web Scraping</b>). In diesem Fall bezeichnet Screen Scraping speziell die <a href="/wiki/Technologie" title="Technologie">Technologien</a>, die der Gewinnung von <a href="/wiki/Information" title="Information">Informationen</a> durch gezieltes Extrahieren der benötigten <a href="/wiki/Daten" title="Daten">Daten</a> dienen.</p>

      <h2><span class="mw-headline" id="Einsatzgebiete">Einsatzgebiete</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=1" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Einsatzgebiete">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=1" title="Abschnitt bearbeiten: Einsatzgebiete">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <h3><span class="mw-headline" id="Suchmaschinen_und_Web_Mining">Suchmaschinen und Web Mining</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=2" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Suchmaschinen und Web Mining">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=2" title="Abschnitt bearbeiten: Suchmaschinen und Web Mining">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Suchmaschinen verwenden sogenannte <a href="/wiki/Webcrawler" title="Webcrawler">Crawler</a> zum Durchsuchen des <a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a>, zur Analyse von Webseiten und Sammeln von Daten, wie <a href="/wiki/Web-Feed" title="Web-Feed">Web-Feeds</a> oder <a href="/wiki/E-Mail-Adresse" title="E-Mail-Adresse">E-Mail-Adressen</a>. Screen-Scraping-Verfahren werden auch beim <a href="/wiki/Web_Mining" title="Web Mining">Web Mining</a> angewandt.</p> 
      <h3><span class="mw-headline" id="Ersatz_von_Web_Services">Ersatz von Web Services</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=3" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Ersatz von Web Services">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=3" title="Abschnitt bearbeiten: Ersatz von Web Services">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Um den Abruf und die Weiterverarbeitung von Informationen aus Webseiten für den <a href="/wiki/Kunde" title="Kunde">Kunden</a> deutlich zu erleichtern, hat der Anbieter des Seiteninhalts (auch Content-Anbieter) die Möglichkeit, die Daten nicht nur in Form einer (menschenlesbaren) Webseite darzustellen, sondern sie zusätzlich in einem maschinenlesbaren <a href="/wiki/Dateiformat" title="Dateiformat">Format</a> (etwa <a href="/wiki/Extensible_Markup_Language" title="Extensible Markup Language">XML</a>) aufzubereiten. Gezielt abgefragte Daten könnten dem Kunden dadurch als <a href="/wiki/Webservice" title="Webservice">Webservice</a> zur <a href="/wiki/Automat" title="Automat">automatisierten</a> Weiterverarbeitung zur Verfügung gestellt werden.</p> 
      <p>Häufig hat der Content-Anbieter jedoch kein Interesse an dem mechanisierten Abruf seiner Daten bzw. der automatisierten Nutzung seines Dienstes (insbesondere bezüglich spezieller Funktionen, die ausschließlich realen Nutzern vorbehalten sein sollten), oder die Errichtung eines Web Service wäre mit zu hohen <a href="/wiki/Kosten" title="Kosten">Kosten</a> verbunden und daher unwirtschaftlich. In solchen Fällen kommt häufig das Screen Scraping zum Einsatz, um die gewünschten Daten dennoch aus der Webseite zu filtern.</p> 
      <h3><span class="mw-headline" id="Erweitertes_Browsen">Erweitertes Browsen</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=4" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Erweitertes Browsen">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=4" title="Abschnitt bearbeiten: Erweitertes Browsen">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Screen Scraping kann zum Einsatz kommen, um den Browser mit weiteren Funktionen auszustatten oder bisher umständliche Vorgänge zu vereinfachen. So können Anmeldevorgänge bei Foren automatisiert oder Dienste einer Webseite abgerufen werden, ohne dass der Nutzer die Webseite besuchen muss, sondern etwa über eine Browser-Symbolleiste.</p> 
      <p>Eine einfache Form von derartigen Screen Scrapern stellen <a href="/wiki/Bookmarklet" title="Bookmarklet">Bookmarklets</a> dar.</p> 
      <h3><span class="mw-headline" id="Remixing">Remixing</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=5" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Remixing">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=5" title="Abschnitt bearbeiten: Remixing">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Remixing ist eine Technik, bei der Webinhalte verschiedener Dienste zu einem neuen Dienst verbunden werden (<i>siehe auch</i> <a href="/wiki/Mashup_(Internet)" title="Mashup (Internet)">Mashup</a>). Wenn keine offenen Programmierschnittstellen zur Verfügung stehen, muss hier ebenfalls auf Screen-Scraping-Mechanismen zurückgegriffen werden.</p> 
      <h3><span class="mw-headline" id="Missbrauch">Missbrauch</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=6" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Missbrauch">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=6" title="Abschnitt bearbeiten: Missbrauch">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Screen Scraping Techniken können jedoch auch missbraucht werden, indem Inhalte fremder Webseiten gegen den Willen des Anbieters kopiert und auf einem eigenen Server angeboten werden.</p> 
      <h2><span class="mw-headline" id="Funktionsweise">Funktionsweise</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=7" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Funktionsweise">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=7" title="Abschnitt bearbeiten: Funktionsweise">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Screen Scraping besteht im Wesentlichen aus zwei Schritten:</p> 
      <ul> 
       <li>Abrufen von Webseiten</li> 
       <li>Extraktion der relevanten Daten</li> 
      </ul> 
      <h3><span class="mw-headline" id="Abrufen_von_Webseiten">Abrufen von Webseiten</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=8" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Abrufen von Webseiten">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=8" title="Abschnitt bearbeiten: Abrufen von Webseiten">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <h4><span class="mw-headline" id="Statische_Webseiten">Statische Webseiten</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=9" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Statische Webseiten">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=9" title="Abschnitt bearbeiten: Statische Webseiten">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h4> 
      <p>Idealerweise befinden sich die interessanten Daten auf einer Webseite, die über eine <a href="/wiki/Uniform_Resource_Locator" title="Uniform Resource Locator">URL</a> abgerufen werden kann. Alle für den Abruf der Informationen benötigten Parameter werden über URL-Parameter (Query-String, siehe <a href="/wiki/HTTP#HTTP_GET" class="mw-redirect" title="HTTP">GET-Request</a>) übergeben. In diesem einfachen Fall wird einfach die Webseite heruntergeladen und die Daten werden mit einem geeigneten Mechanismus extrahiert.</p> 
      <h4><span class="mw-headline" id="Formulare">Formulare</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=10" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Formulare">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=10" title="Abschnitt bearbeiten: Formulare">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h4> 
      <p>In vielen Fällen werden die Parameter durch Ausfüllen eines <a href="/wiki/Webformular" title="Webformular">Webformulars</a> abgefragt. Dabei werden die Parameter oft nicht in der URL übergeben, sondern im Nachrichtenkörper (<a href="/wiki/HTTP#HTTP_POST" class="mw-redirect" title="HTTP">POST-Request</a>).</p> 
      <h4><span class="mw-headline" id="Personalisierte_Webseiten">Personalisierte Webseiten</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=11" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Personalisierte Webseiten">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=11" title="Abschnitt bearbeiten: Personalisierte Webseiten">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h4> 
      <p>Viele Webseiten enthalten personalisierte Informationen. Das <a href="/wiki/Hypertext_Transfer_Protocol" title="Hypertext Transfer Protocol">Hypertext Transfer Protocol</a> (HTTP) bietet jedoch keine native Möglichkeit, Anfragen einer bestimmten Person zuzuordnen. Um eine bestimmte Person wiederzuerkennen, muss die Serveranwendung auf HTTP aufgesetzte <a href="/wiki/Sitzung_(Informatik)" title="Sitzung (Informatik)">Sitzungskonzepte</a> verwenden. Eine häufig genutzte Möglichkeit ist die Übertragung von <a href="/wiki/Session-ID" class="mw-redirect" title="Session-ID">Session-IDs</a> durch die URL oder durch <a href="/wiki/HTTP-Cookie" title="HTTP-Cookie">Cookies</a>. Diese Sitzungskonzepte müssen von einer Screen-Scraping-Anwendung unterstützt werden.</p> 
      <h3><span class="mw-headline" id="Datenextraktion">Datenextraktion</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=12" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Datenextraktion">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=12" title="Abschnitt bearbeiten: Datenextraktion">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Ein Programm zur Extraktion von Daten aus Webseiten wird auch <a href="/wiki/Wrapper_(Informationsextraktion)" title="Wrapper (Informationsextraktion)">Wrapper</a> genannt.</p> 
      <p>Nachdem die Webseite heruntergeladen wurde, ist es für die Extraktion der Daten zunächst wichtig, ob der genaue Ort der Daten auf der Webseite bekannt ist (etwa <i>zweite Tabelle, dritte Spalte</i>).</p> 
      <p>Wenn dies der Fall ist, stehen für die Extraktion der Daten verschiedene Möglichkeiten zur Verfügung. Man kann zum einen die heruntergeladenen Webseiten als Zeichenketten interpretieren und etwa mit regulären Ausdrücken die gewünschten Daten extrahieren.</p> 
      <p>Wenn die Webseite <a href="/wiki/Extensible_Hypertext_Markup_Language" title="Extensible Hypertext Markup Language">XHTML</a>-konform ist, bietet sich die Nutzung eines XML-<a href="/wiki/Parser" title="Parser">Parsers</a> an. Für den Zugriff auf XML gibt es zahlreiche unterstützende Technologien (<a href="/wiki/Simple_API_for_XML" title="Simple API for XML">SAX</a>, <a href="/wiki/Document_Object_Model" title="Document Object Model">DOM</a>, <a href="/wiki/XPath" title="XPath">XPath</a>, <a href="/wiki/XQuery" title="XQuery">XQuery</a>). Oft werden die Webseiten jedoch lediglich im (möglicherweise sogar fehlerhaften) <a href="/wiki/Hypertext_Markup_Language" title="Hypertext Markup Language">HTML</a>-Format ausgeliefert, welches nicht dem XML-Standard entspricht. Mit einem geeigneten Parser lässt sich unter Umständen dennoch ein XML-konformes Dokument herstellen. Alternativ kann das HTML vor dem Parsen mit <a href="/wiki/HTML_Tidy" title="HTML Tidy">HTML Tidy</a> bereinigt werden. Manche Screen Scraper verwenden eine eigens für HTML entwickelte Anfragesprache.</p> 
      <p>Ein Kriterium für die Güte der Extraktionsmechanismen ist die Robustheit gegenüber Änderungen an der Struktur der Webseite. Hierfür sind fehlertolerante Extraktionsalgorithmen erforderlich.</p> 
      <p>In vielen Fällen ist die Struktur der Webseite jedoch unbekannt (etwa beim Einsatz von Crawlern). Datenstrukturen wie etwa Kaufpreisangaben oder Zeitangaben müssen dann auch ohne feste Vorgaben erkannt und interpretiert werden.</p> 
      <h2><span class="mw-headline" id="Architektur">Architektur</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=13" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Architektur">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=13" title="Abschnitt bearbeiten: Architektur">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <h3><span class="mw-headline" id="Zentralisierte_Architektur">Zentralisierte Architektur</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=14" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Zentralisierte Architektur">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=14" title="Abschnitt bearbeiten: Zentralisierte Architektur">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Ein Screen Scraper kann auf einem speziellen <a href="/wiki/Webserver" title="Webserver">Webserver</a> installiert sein, der in regelmäßigen Abständen oder auf Anfrage die geforderten Daten abruft und seinerseits in aufbereiteter Form anbietet. Dieses serverseitige Vorgehen kann jedoch unter Umständen rechtliche Probleme mit sich ziehen und vom Content-Anbieter auch leicht durch Blockieren der Server-<a href="/wiki/IP-Adresse" title="IP-Adresse">IP</a> verhindert werden.</p> 
      <h3><span class="mw-headline" id="Verteilte_Architektur">Verteilte Architektur</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=15" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Verteilte Architektur">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=15" title="Abschnitt bearbeiten: Verteilte Architektur">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Beim verteilten Vorgehen werden die Informationen direkt vom Client abgerufen. Je nach Anwendung werden die Informationen in einer Datenbank gespeichert, an andere Anwendungen weitergegeben oder aufbereitet im Browser angezeigt. Die verteilte Architektur kann nicht nur schwieriger blockiert werden, sondern skaliert auch besser.</p> 
      <h2><span id="Anbieterseitige_Abwehrmaßnahmen"></span><span class="mw-headline" id="Anbieterseitige_Abwehrma.C3.9Fnahmen">Anbieterseitige Abwehrmaßnahmen</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=16" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Anbieterseitige Abwehrmaßnahmen">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=16" title="Abschnitt bearbeiten: Anbieterseitige Abwehrmaßnahmen">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Viele Content-Anbieter haben kein Interesse an einem isolierten Abrufen bestimmter Informationen. Grund dafür kann sein, dass sich der Anbieter durch Werbeeinblendungen finanziert, die durch Screen Scraping leicht gefiltert werden können. Zudem könnte der Content-Anbieter ein Interesse daran haben, den Benutzer zu einer bestimmten Navigationsreihenfolge zu zwingen. Um diese Interessen zu gewährleisten, gibt es verschiedene Strategien.</p> 
      <h3><span class="mw-headline" id="Kontrolle_des_Benutzerverhaltens">Kontrolle des Benutzerverhaltens</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=17" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Kontrolle des Benutzerverhaltens">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=17" title="Abschnitt bearbeiten: Kontrolle des Benutzerverhaltens">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Der Server zwingt den Benutzer durch Verwenden von Session-IDs zu einer bestimmten Navigationsreihenfolge. Beim Aufruf der Verkehrslenkungsseite des Webangebotes wird eine temporär gültige Session-ID erzeugt. Diese wird über die URL, versteckte Formularfelder oder durch Cookies übertragen. Wenn ein Nutzer oder ein Bot durch einen <a href="/wiki/Deep_Link" class="mw-redirect" title="Deep Link">Deep Link</a> auf die Seite stößt, kann er keine gültige Session-ID vorweisen. Der Server leitet ihn dann auf die Verkehrslenkungsseite um. Diese Strategie verwendet beispielsweise <a href="/wiki/EBay" title="EBay">eBay</a>, um Deep Links auf Auktionslisten zu verhindern. Ein speziell programmierter Screen Scraper kann sich jedoch zunächst eine gültige Session-ID holen und dann die gewünschten Daten herunterladen.</p> 
      <p>Das folgende Beispiel zeigt einen <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>-basierten Screen Scraper, der die von eBay benutzte Strategie umging. Es lud sich zunächst die Hauptseite herunter, extrahierte mit einem <a href="/wiki/Regul%C3%A4rer_Ausdruck" title="Regulärer Ausdruck">Regulären Ausdruck</a> eine gültige URL (in diesem Fall die Liste der Auktionen, bei denen Disketten ersteigert werden) und öffnete diese im Browser.</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
 <span class="kd">function</span> <span class="nx">EbayScraper</span><span class="p">()</span> <span class="p">{</span>
    <span class="nx">req</span> <span class="o">=</span> <span class="k">new</span> <span class="nx">XMLHttpRequest</span><span class="p">();</span>
    <span class="nx">req</span><span class="p">.</span><span class="nx">open</span><span class="p">(</span><span class="s1">'GET'</span><span class="p">,</span> <span class="s1">'http://computer.ebay.de'</span><span class="p">,</span> <span class="kc">false</span><span class="p">);</span>
    <span class="nx">req</span><span class="p">.</span><span class="nx">send</span><span class="p">(</span><span class="kc">null</span><span class="p">);</span>
    <span class="kd">var</span> <span class="nx">regex</span> <span class="o">=</span> <span class="k">new</span> <span class="nb">RegExp</span><span class="p">(</span><span class="s1">'http:\/\/computer\.listings\.ebay\.de\/Floppy-Zip-Streamer_Disketten_[a-zA-Z0-9]*'</span><span class="p">);</span>
    <span class="nb">window</span><span class="p">.</span><span class="nx">location</span> <span class="o">=</span> <span class="nx">req</span><span class="p">.</span><span class="nx">responseText</span><span class="p">.</span><span class="nx">match</span><span class="p">(</span><span class="nx">regex</span><span class="p">);</span>
 <span class="p">}</span>
</pre>
      </div> 
      <p>Neben der Zweckentfremdung von Session-IDs gibt es weitere Möglichkeiten, das Benutzerverhalten zu überprüfen:</p> 
      <ul> 
       <li>Kontrolle des <a href="/wiki/Referrer" title="Referrer">Referrers</a> zur Abwehr von Deep Links</li> 
       <li>Kontrolle, ob in die Seite eingebettete Elemente (Grafiken etc.) zeitnah heruntergeladen werden</li> 
       <li>Kontrolle, ob JavaScript-Elemente ausgeführt werden</li> 
      </ul> 
      <p>Alle diese Methoden beinhalten jedoch gewisse Problematiken, etwa weil Referrer-Angaben nicht zwingend sind, weil eingebettete Elemente möglicherweise von einem <a href="/wiki/Proxy_(Rechnernetz)" title="Proxy (Rechnernetz)">Proxy</a> oder aus dem <a href="/wiki/Cache" title="Cache">Cache</a> geliefert werden oder weil der Anwender schlichtweg die Anzeige von Grafiken oder das Ausführen von JavaScript deaktiviert hat.</p> 
      <h3><span class="mw-headline" id="Unterscheiden_zwischen_Mensch_und_Bot">Unterscheiden zwischen Mensch und Bot</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=18" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Unterscheiden zwischen Mensch und Bot">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=18" title="Abschnitt bearbeiten: Unterscheiden zwischen Mensch und Bot">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Der Server versucht vor dem Ausliefern der Daten zu erkennen, ob es sich beim <a href="/wiki/Client" title="Client">Client</a> um einen Menschen oder um einen <a href="/wiki/Bot" title="Bot">Bot</a> handelt. Eine häufig eingesetzte Methode dafür ist die Verwendung von <a href="/wiki/Captcha" title="Captcha">Captchas</a>. Dabei wird dem Client eine Aufgabe gestellt, welche für Menschen möglichst einfach, für eine Maschine jedoch sehr schwer lösbar ist. Dies kann eine Rechenaufgabe oder das Abtippen von Buchstaben sein, wobei oft die Schwierigkeit für die Maschine im Erkennen der Aufgabe liegt. Dies kann z.&nbsp;B. erreicht werden, indem die Rechenaufgabe nicht als Text, sondern als Bild übermittelt wird.</p> 
      <p>Captchas werden für bestimmte Online-Dienste wie Foren, Wikis, Downloadseiten oder Online-Netzwerke eingesetzt etwa gegen automatisches Registrieren, automatisches Ausspähen von Profilen anderer Nutzer sowie automatische Downloads durch Bots. Mitunter muss ein Client erst nach einer bestimmten Anzahl von Aktionen ein Captcha lösen.</p> 
      <p>Theoretisch lassen sich für alle Captchas auch Bots entwickeln, die diese Aufgaben auf Basis von <a href="/wiki/Optical_Character_Recognition" class="mw-redirect" title="Optical Character Recognition">Optical Character Recognition</a> (Extraktion der Aufgabe aus einem Bild) lösen können, so dass dieser Schutz umgangen werden kann. Des Weiteren besteht die Möglichkeit, die Teilaufgabe an einen Menschen weiterzugeben, so dass dieser das Captcha für die Maschine löst. Beides bedeutet jedoch einen erheblichen Mehraufwand für den Botbetreiber.</p> 
      <h3><span class="mw-headline" id="Verschleierung">Verschleierung</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=19" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Verschleierung">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=19" title="Abschnitt bearbeiten: Verschleierung">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Die Informationen werden in für Maschinen nicht oder nur schwer lesbarer Form angeboten. Etwa als Grafik, in <a href="/wiki/Adobe_Flash" title="Adobe Flash">Flash-Animationen</a> oder <a href="/wiki/Java-Applet" title="Java-Applet">Java-Applets</a>. Allerdings leidet hierunter häufig die <a href="/wiki/Gebrauchstauglichkeit_(Produkt)" title="Gebrauchstauglichkeit (Produkt)">Gebrauchstauglichkeit</a>.</p> 
      <p>Zur Verschleierung der Daten kann auch <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> zum Einsatz kommen. Diese Methode wird vor allem auch gegen <a href="/wiki/E-Mail-Harvester" title="E-Mail-Harvester">E-Mail-Harvester</a> eingesetzt, die E-Mail-Adressen zur Versendung von Spam sammeln. Die eigentlichen Daten werden nicht im HTML-Code übertragen, sondern werden erst durch JavaScript in die Webseite geschrieben. Die Daten können zusätzlich verschlüsselt übertragen und erst beim Anzeigen der Seite entschlüsselt werden. Mit Hilfe eines <a href="/wiki/Obfuscator" class="mw-redirect" title="Obfuscator">Obfuscators</a> kann der Programmcode verschleiert werden, um die Entwicklung eines Screen Scrapers zu erschweren.</p> 
      <p>Einfaches Beispiel zur Verschleierung einer E-Mail-Adresse mit JavaScript (ohne Verschlüsselung):</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
  <span class="kd">function</span> <span class="nx">mail</span><span class="p">()</span> <span class="p">{</span>
     <span class="kd">var</span> <span class="nx">name</span> <span class="o">=</span> <span class="s2">"info"</span><span class="p">;</span>
     <span class="kd">var</span> <span class="nx">domain</span> <span class="o">=</span> <span class="s2">"example.com"</span><span class="p">;</span>
     <span class="kd">var</span> <span class="nx">mailto</span> <span class="o">=</span> <span class="s1">'mailto:'</span> <span class="o">+</span> <span class="nx">name</span> <span class="o">+</span> <span class="s1">'@'</span> <span class="o">+</span> <span class="nx">domain</span><span class="p">;</span>
     <span class="nb">document</span><span class="p">.</span><span class="nx">write</span><span class="p">(</span><span class="nx">mailto</span><span class="p">);</span>
  <span class="p">}</span>
</pre>
      </div> 
      <h2><span class="mw-headline" id="Erstellung_von_Screen_Scrapern">Erstellung von Screen Scrapern</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=20" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Erstellung von Screen Scrapern">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=20" title="Abschnitt bearbeiten: Erstellung von Screen Scrapern">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Je nach Komplexität der Aufgabe muss ein Screen Scraper neu programmiert werden. Mithilfe von Toolkits lassen sich Screen Scraper jedoch auch ohne Programmierkenntnisse erstellen. Für die Implementierungsform gibt es verschiedene Möglichkeiten, etwa als <a href="/wiki/Programmbibliothek" title="Programmbibliothek">Bibliothek</a>, als <a href="/wiki/Proxy-Server" class="mw-redirect" title="Proxy-Server">Proxy-Server</a> oder als eigenständiges Programm.</p> 
      <h2><span class="mw-headline" id="Anwendungen">Anwendungen</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=21" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Anwendungen">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=21" title="Abschnitt bearbeiten: Anwendungen">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p><i>Piggy Bank</i> war eine vom Simile-Projekt am <a href="/wiki/Massachusetts_Institute_of_Technology" title="Massachusetts Institute of Technology">MIT</a> entwickelte Erweiterung für <a href="/wiki/Mozilla_Firefox" title="Mozilla Firefox">Firefox</a>. Mit ihr ließen sich Verknüpfungen von Diensten verschiedener Anbieter realisieren. Es erkannte automatisch auf einer Webseite angebotene <a href="/wiki/Resource_Description_Framework" title="Resource Description Framework">RDF</a>-Ressourcen. Diese konnten gespeichert, verwaltet und mit anderen Diensten (etwa geographische Informationen mit <a href="/wiki/Google_Maps" title="Google Maps">Google Maps</a>) kombiniert werden. Piggy Bank wird nicht mehr angeboten. Als Ersatz bietet sich Selenium<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> an, womit man einen Web-Browser wie Firefox programmatisch steuern kann.</p> 
      <p>Eine weitere bekannte Firefox-Erweiterung ist <i><a href="/wiki/Greasemonkey" title="Greasemonkey">Greasemonkey</a></i>. Sie erlaubt es dem Nutzer eigene JavaScript-Dateien im Browser auszuführen, die das Erscheinungsbild und Verhalten der angezeigten Webseite individualisieren können, ohne einen Zugriff auf die eigentliche Webseite zu benötigen. Dadurch ist es beispielsweise möglich, Webseiten um Funktionen zu erweitern, Fehler in der Darstellung zu beheben, Inhalte von anderen Webseiten einzubinden und wiederkehrende Aufgaben automatisch zu erledigen.</p> 
      <p><i><a href="/wiki/A9.com" title="A9.com">A9</a></i> von <a href="/wiki/Amazon.com" title="Amazon.com">Amazon</a> ist ein Beispiel für eine zentralisierte Remix-Architektur. A9 kann Suchergebnisse aus verschiedenen Webdiensten wie <a href="/wiki/Windows_Live" title="Windows Live">Windows Live</a>, <a href="/wiki/Wikipedia" title="Wikipedia">Wikipedia</a>, answers.com und vielen anderen in einem Fenster anzeigen.</p> 
      <h2><span class="mw-headline" id="Programmierbibliotheken">Programmierbibliotheken</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=22" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Programmierbibliotheken">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=22" title="Abschnitt bearbeiten: Programmierbibliotheken">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Programmierkundige nutzen oft <a href="/wiki/Skriptsprache" title="Skriptsprache">Skriptsprachen</a> für maßgeschneiderte Screenscraping-Projekte. Für <a href="/wiki/Python_(Programmiersprache)" title="Python (Programmiersprache)">Python</a> etwa gibt es die Programmbibliothek <a href="/wiki/Beautiful_Soup" title="Beautiful Soup">Beautiful Soup</a>,<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> die den Umgang mit real existierendem <a href="/wiki/Hypertext_Markup_Language" title="Hypertext Markup Language">HTML</a> erleichtert. Ebenfalls auf Python basiert die <a href="/wiki/Dom%C3%A4nenspezifische_Sprache" title="Domänenspezifische Sprache">domänenspezifische Sprache</a> <a href="/w/index.php?title=Redex&amp;action=edit&amp;redlink=1" class="new" title="Redex (Seite nicht vorhanden)">redex</a> (Regular Document Expressions)<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup> von Marcin Wojnarski, die speziell für das Webscraping geschaffen wurde und die Lücke zwischen den praktischen, aber kleinteiligen <a href="/wiki/Regul%C3%A4rer_Ausdruck" title="Regulärer Ausdruck">regulären Ausdrücken</a> und der mächtigen, aber sehr rigiden <a href="/wiki/XPath" title="XPath">XPath</a>-Syntax schließen soll.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup></p> 
      <h2><span class="mw-headline" id="Rechtliche_Probleme">Rechtliche Probleme</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_Scraping&amp;veaction=edit&amp;section=23" class="mw-editsection-visualeditor" title="Abschnitt bearbeiten: Rechtliche Probleme">Bearbeiten</a><span class="mw-editsection-divider"> | </span><a href="/w/index.php?title=Screen_Scraping&amp;action=edit&amp;section=23" title="Abschnitt bearbeiten: Rechtliche Probleme">Quelltext bearbeiten</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Beim Scraping von Webseiten fremder Anbieter muss auf die Einhaltung der <a href="/wiki/Urheberrecht" title="Urheberrecht">Urheberrechte</a> geachtet werden, vor allem wenn die Inhalte über ein eigenes Angebot eingebunden werden. Eine rechtliche Grauzone ist dagegen das Anbieten von Programmen, die ein clientseitiges Screen Scraping ermöglichen. Einige Anbieter verbieten das automatische Auslesen von Daten auch explizit in den Nutzungsbedingungen.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[5]</a></sup></p> 
      <p>Ein weiteres Problem stellt unter Umständen das Ausblenden von Informationen dar, etwa von Werbung oder rechtlich relevanten Informationen wie <a href="/wiki/Disclaimer" title="Disclaimer">Disclaimer</a>, Warnungen oder gar die automatische Bestätigung der <a href="/wiki/Allgemeine_Gesch%C3%A4ftsbedingungen" title="Allgemeine Geschäftsbedingungen">AGB</a> durch den Screen Scraper, ohne dass der Nutzer diese zu Gesicht bekommt.</p>
