       <h2 class="title">Simple Captcha in JSF with and without Acegi&nbsp;Security</h2> 
       <div class="post-info"> 
        <div> 
         <span class="comments-meta"><a href="https://mahmudfahmi.wordpress.com/2012/07/01/simple-captcha-in-jsf-with-and-without-acegi-security/#respond">1 Comment</a></span> Posted by 
         <span class="author"><a href="https://mahmudfahmi.wordpress.com/author/imfahmi/" title="Posts by Fahmi" rel="author">Fahmi</a></span> on 
         <span class="time">July 1, 2012</span> 
        </div> 
       </div> 
       <div class="content"> 
        <p>At first, goto <a href="http://simplecaptcha.sourceforge.net/" target="_blank">SimpleCaptcha</a>&nbsp;and read all that they are saying.</p> 
        <p>Right now, I’m using captcha in JSF. I have an jsf application, I built that through Appfuse.</p> 
        <p>It has <strong>Acegi security 1.0.5</strong>&nbsp;&nbsp;and <strong>spring 2.5, </strong>now i’m trying to integrate<strong>&nbsp;<strong>Simple captcha 1.2.1 </strong></strong>in it<strong><strong>.</strong></strong></p> 
        <p><span id="more-321"></span></p> 
        <p>Acegi uses a java filter class to&nbsp;authenticate user in the system. It is</p> 
        <blockquote>
         <p>org.acegisecurity.ui.webapp.AuthenticationProcessingFilter</p>
        </blockquote> 
        <p>It is mentioned one of the configuration xml files. In Appfuse, it mentioned on&nbsp;<em>security.xml</em></p> 
        <p>Look for the tag</p> 
        <blockquote>
         <p>&lt;bean id=”authenticationProcessingFilter” class=”org.acegisecurity.ui.webapp.AuthenticationProcessingFilter”&gt;<br> &lt;property name=”authenticationManager” ref=”authenticationManager”/&gt;<br> &lt;property name=”authenticationFailureUrl” value=”/login.jsp?error=true”/&gt;<br> &lt;property name=”defaultTargetUrl” value=”/”/&gt;<br> &lt;property name=”filterProcessesUrl” value=”/j_security_check”/&gt;<br> &lt;property name=”rememberMeServices” ref=”rememberMeServices”/&gt;<br> &lt;/bean&gt;</p>
        </blockquote> 
        <p>Now, basic idea is, we are going to replace the filter class, with our own class. Our class will extend&nbsp;<em>AuthenticationProcessingFilter</em>, override the&nbsp;<em>attemptAuthentication</em> method, do some own work and give back the control to <em>super’s&nbsp;attemptAuthentication</em>. At first I changed the&nbsp;<em>authenticationProcessingFilter</em>&nbsp;bean tag.</p> 
        <blockquote>
         <p>&lt;bean id=”authenticationProcessingFilter” class=”com.webapp.filter.CaptchaAuthenticationProcessingFilter”&gt;<br> &lt;property name=”authenticationManager” ref=”authenticationManager”/&gt;<br> &lt;property name=”authenticationFailureUrl” value=”/login.jsp?error=true”/&gt;<br> &lt;property name=”defaultTargetUrl” value=”/”/&gt;<br> &lt;property name=”filterProcessesUrl” value=”/j_security_check”/&gt;<br> &lt;property name=”rememberMeServices” ref=”rememberMeServices”/&gt;<br> &lt;/bean&gt;</p>
        </blockquote> 
        <p>My Filter class</p> 
        <blockquote>
         <p>package com.webapp.filter;</p> 
         <p>import javax.servlet.http.HttpServletRequest;</p> 
         <p>import org.acegisecurity.Authentication;<br> import org.acegisecurity.AuthenticationException;<br> import org.acegisecurity.ui.webapp.AuthenticationProcessingFilter;</p> 
         <p>import com.seblit.webapp.filter.exception.CaptchaAuthenticationException;</p> 
         <p>import nl.captcha.Captcha;</p> 
         <p>public class CaptchaAuthenticationProcessingFilter extends<br> AuthenticationProcessingFilter<br> {</p> 
         <p>public static final String CAPTCHA_KEY = “j_captcha”;</p> 
         <p>/**<br> * Perform captcha authentication using {@link SimpleCaptcha}.<br> *<br> */<br> @Override<br> public Authentication attemptAuthentication(HttpServletRequest request)<br> throws AuthenticationException<br> {<br> Captcha captcha = (Captcha) request.getSession().getAttribute(<br> Captcha.NAME);<br> String answer = request.getParameter(CAPTCHA_KEY);<br> if (!captcha.isCorrect(answer))<br> {<br> throw new CaptchaAuthenticationException(<br> “captcha authentication error”);<br> }</p> 
         <p>return super.attemptAuthentication(request);<br> }</p> 
         <p>}</p>
        </blockquote> 
        <p>Adjacent exception class</p> 
        <blockquote>
         <p>package com.webapp.filter.exception;</p> 
         <p>import org.acegisecurity.AuthenticationException;</p> 
         <p>public class CaptchaAuthenticationException extends AuthenticationException<br> {</p> 
         <p>public CaptchaAuthenticationException(String msg)<br> {<br> super(msg);<br> }</p> 
         <p>/**<br> *<br> */<br> private static final long serialVersionUID = 1L;</p> 
         <p>}</p>
        </blockquote> 
        <p>Now, I had to add captcha field to my login page</p> 
        <blockquote>
         <p>&lt;img src=”Captcha.jpg”&nbsp;style=”margin-top: 10px; margin-bottom: 10px;” /&gt;</p> 
         <p>&lt;input type=”text” name=”j_captcha”&nbsp;id=”j_captcha” /&gt;</p>
        </blockquote> 
        <p>after that my login page looked like this.</p> 
        <p><a href="https://mahmudfahmi.files.wordpress.com/2012/07/login.png"><img data-attachment-id="322" data-permalink="https://mahmudfahmi.wordpress.com/2012/07/01/simple-captcha-in-jsf-with-and-without-acegi-security/login/" data-orig-file="https://mahmudfahmi.files.wordpress.com/2012/07/login.png" data-orig-size="431,442" data-comments-opened="1" data-image-meta="{&quot;aperture&quot;:&quot;0&quot;,&quot;credit&quot;:&quot;&quot;,&quot;camera&quot;:&quot;&quot;,&quot;caption&quot;:&quot;&quot;,&quot;created_timestamp&quot;:&quot;0&quot;,&quot;copyright&quot;:&quot;&quot;,&quot;focal_length&quot;:&quot;0&quot;,&quot;iso&quot;:&quot;0&quot;,&quot;shutter_speed&quot;:&quot;0&quot;,&quot;title&quot;:&quot;&quot;}" data-image-title="login" data-image-description="" data-medium-file="https://mahmudfahmi.files.wordpress.com/2012/07/login.png?w=293" data-large-file="https://mahmudfahmi.files.wordpress.com/2012/07/login.png?w=431" class="aligncenter size-medium wp-image-322" title="login" src="https://mahmudfahmi.files.wordpress.com/2012/07/login.png?w=292&amp;h=300" alt="" width="292" height="300" srcset="https://mahmudfahmi.files.wordpress.com/2012/07/login.png?w=292&amp;h=300 292w, https://mahmudfahmi.files.wordpress.com/2012/07/login.png?w=146&amp;h=150 146w, https://mahmudfahmi.files.wordpress.com/2012/07/login.png 431w" sizes="(max-width: 292px) 100vw, 292px"></a></p> 
        <p>Now, I need to add captcha in my signup page. There is no filter, so i have to use captcha in managed bean.</p> 
        <p>Firstly, I added the captcha fields in the <em>signup html page</em></p> 
        <blockquote>
         <p>&lt;h:outputLabel styleClass=”desc” value=”” /&gt;<br> &lt;img src=”Captcha.jpg”&nbsp;style=”margin-top: 10px; margin-bottom: 10px;” /&gt;</p> 
         <p>&lt;h:outputLabel for=”captchaText” styleClass=”desc”&nbsp;value=”captcha” /&gt;<br> &lt;t:message for=”captchaText” styleClass=”fieldError” /&gt;<br> &lt;h:inputText value=”#{signupForm.captchaText}” id=”captchaText”&nbsp;required=”true” styleClass=”text large”&gt;<br> &lt;/h:inputText&gt;</p>
        </blockquote> 
        <p>Now, captcha text is bind with managed bean property&nbsp;<em>captchaText</em>.</p> 
        <p>On action method i have written following code.</p> 
        <blockquote>
         <p>HttpServletRequest request = (HttpServletRequest)</p> 
         <p>FacesContext.getCurrentInstance().getExternalContext().getRequest();<br> Captcha captcha = (Captcha) request.getSession().getAttribute(Captcha.NAME);</p> 
         <p>if (!captcha.isCorrect(captchaText))<br> {<br> captchaText = “”;<br> return “”;<br> }</p>
        </blockquote> 
        <p>ha.. it was fun.</p>
