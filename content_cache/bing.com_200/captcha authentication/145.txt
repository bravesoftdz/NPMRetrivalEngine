         <h1>JIRA REST API Example - OAuth authentication</h1> 
         <div class="conf-macro output-block" data-hasbody="true" data-macro-name="sp-pagelayout"> 
          <div class="aui-group"> 
           <div class="aui-item" style=""> 
            <p>This page shows you how to allow REST clients to authenticate themselves using <a rel="nofollow" class="external-link" href="https://en.wikipedia.org/wiki/OAuth">OAuth</a>. This is one of three methods that you can use for authentication against the JIRA REST API; the other two being <strong>basic authentication</strong> and&nbsp;<strong>cookie-based&nbsp;authentication</strong><strong>&nbsp;</strong>(see&nbsp;<a data-macro-name="sp-link" href="#JIRARESTAPIExample-OAuthauthentication-relatedtopics" class="conf-macro output-inline" data-hasbody="true">related information</a>).</p>

            <h2 id="JIRARESTAPIExample-OAuthauthentication-Overview">Overview</h2> 
            <p>The instructions below describe how to use a Java client to provide OAuth authentication when making requests to JIRA's REST endpoints. It assumes you are familiar with the OAuth terminology (e.g. Consumer, Service Provider, request token, access token, etc.). For more information about OAuth refer to the OAuth specification.</p> 
            <div class="panel panel-default conf-macro output-block" data-hasbody="true" data-macro-name="sp-macrooverride-block"> 
             <div class="panel-wrapper"> 
              <div class="panel-heading"> 
               <b class="panel-title">Looking for a Provider in a Language other than Java?</b> 
              </div> 
              <div class="panel-body"> 
               <p>Atlassian provides samples of OAuth providers in a number of other languages. <a rel="nofollow" class="external-link" href="https://bitbucket.org/atlassian_tutorial/atlassian-oauth-examples">Visit the sample repo on Bitbucket</a> to download and work with these samples.</p> 
              </div> 
             </div> 
            </div> 
            <h2 id="JIRARESTAPIExample-OAuthauthentication-step1Step1:ConfiguringJIRA"><span class="confluence-anchor-link conf-macro output-inline" id="JIRARESTAPIExample-OAuthauthentication-step1" data-hasbody="false" data-macro-name="anchor"> </span>Step 1: Configuring JIRA</h2> 
            <p>The first step is to register a new consumer in JIRA. This is done through the Application Links administration screens in JIRA. Create a new Application Link. <br> When creating the Application Link use a placeholder URL or the correct URL to your client, if your client can be reached via HTTP and choose the Generic Application type. After this Application Link has been created, edit the configuration and go to the incoming authentication configuration screen and select OAuth. Enter in this the public key and the consumer key which your client will use when making requests to JIRA.<br> After you have entered all the information click OK and ensure OAuth authentication is enabled.</p> 
            <h2 id="JIRARESTAPIExample-OAuthauthentication-Step2:Configuringtheclient">Step 2: Configuring the client</h2> 
            <p>Your client will require the following information to be able to make authentication requests to JIRA.</p>

            <h2 id="JIRARESTAPIExample-OAuthauthentication-ExampleJavaOAuthclient">Example Java OAuth client</h2> 
            <p>This example java code demonstrates how to write a client to make requests to JIRA's rest endpoints using OAuth authentication.<br> To be able to use OAuth authentication the client application has to do the "OAuth dance" with JIRA. This dance consists of three parts.</p> 
            <ol> 
             <li>Obtain a request token</li> 
             <li>Ask the user to authorize this request token</li> 
             <li>Swap the request token for an access token</li> 
            </ol> 
            <p>After the client application has a valid access token, this can be used to make authenticated requests to JIRA.</p>

            <h4 id="JIRARESTAPIExample-OAuthauthentication-1.ObtainarequesttokenfromJIRA">1. Obtain a request token from JIRA</h4> 
            <p>Execute this command:</p> 
            <div class="code panel pdl conf-macro output-block" style="border-width: 1px;" data-hasbody="true" data-macro-name="code"> 
             <div class="codeContent panelContent pdl"> 
              <pre class="syntaxhighlighter-pre" data-syntaxhighlighter-params="brush: jscript; gutter: false; theme: Confluence" data-theme="Confluence">java -jar rest-oauth-client-1.0.one-jar.jar requestToken JIRA_BASE_URL
</pre> 
             </div> 
            </div> 
            <p>Replace&nbsp;JIRA_BASE_URL with the URL to your JIRA instance.<br> After executing this command you should see a response like</p> 
            <div class="code panel pdl conf-macro output-block" style="border-width: 1px;" data-hasbody="true" data-macro-name="code">

Retrieved request token. go to http://localhost:8090/jira/plugins/servlet/oauth/authorize?oauth_token=W9QuE8hba6laXm2RcPGANVusKHnXUJcx
</pre> 
             </div> 
            </div> 
            <h4 id="JIRARESTAPIExample-OAuthauthentication-2.Authorizethistoken">2. Authorize this token</h4> 
            <p>Go to the URL in system out and login into JIRA and approve the access. Afterwards JIRA will say that you have successfully authorised the access. It mentions a verification code which we need for the next step.</p> 
            <h4 id="JIRARESTAPIExample-OAuthauthentication-3.Swaptherequesttokenwithanaccesstoken">3. Swap the request token with an access token</h4> 
            <p>Execute the following command</p> 
            <div class="code panel pdl conf-macro output-block" style="border-width: 1px;" data-hasbody="true" data-macro-name="code"> 
             <div class="codeContent panelContent pdl"> 
              <pre class="syntaxhighlighter-pre" data-syntaxhighlighter-params="brush: jscript; gutter: false; theme: Confluence" data-theme="Confluence">java -jar rest-oauth-client-1.0.one-jar.jar accessToken JIRA_BASE_URL REQUEST_TOKEN TOKEN_SECRET VERIFIER
</pre> 
             </div> 
            </div> 
            <p>Replace JIRA_BASE_URL, REQUEST_TOKEN, TOKEN_SECRET and VERIFIER with the correct values.</p> 
            <p>In the response you should see</p> 
            <div class="code panel pdl conf-macro output-block" style="border-width: 1px;" data-hasbody="true" data-macro-name="code"> 
             <div class="codeContent panelContent pdl"> 
              <pre class="syntaxhighlighter-pre" data-syntaxhighlighter-params="brush: jscript; gutter: false; theme: Confluence" data-theme="Confluence">Access token is : QddAGsDSS0FkXCb1zRCCzzeShZRnUXYH
</pre> 
             </div> 
            </div> 
            <p>This access token will allow you to make authenticated requests to JIRA.</p> 
            <h4 id="JIRARESTAPIExample-OAuthauthentication-4.Makeanauthenticationrequesttoarest-endpoint">4. Make an authentication request to a rest-end point</h4> 
            <p>To make an authenticated request to a rest resource in JIRA execute this command:</p> 
            <div class="code panel pdl conf-macro output-block" style="border-width: 1px;" data-hasbody="true" data-macro-name="code"> 
             <div class="codeContent panelContent pdl"> 
              <pre class="syntaxhighlighter-pre" data-syntaxhighlighter-params="brush: jscript; gutter: false; theme: Confluence" data-theme="Confluence">java -jar rest-oauth-client-1.0.one-jar.jar request ACCESS_TOKEN JIRA_REST_URL
</pre> 
             </div> 
            </div> 
            <p>Replace ACCESS_TOKEN, JIRA_REST_URL and ISSUE_KEY with the correct values.<br> JIRA_REST_URL, e.g. <a rel="nofollow" class="external-link" href="http://localhost:8090/jira/rest/api/2/issue/HSP-1">http://localhost:8090/jira/rest/api/2/issue/HSP-1</a><br> This will return the issue JSON object for the issue with the key "HSP-1"</p>

            <h4 id="JIRARESTAPIExample-OAuthauthentication-CAPTCHA">CAPTCHA</h4> 
            <p>CAPTCHA is 'triggered' after several consecutive failed log in attempts, after which the user is required to interpret a distorted picture of a word and type that word into a text field with each subsequent log in attempt. If CAPTCHA has been triggered, you cannot use&nbsp;JIRA's REST API to authenticate with the JIRA site.</p> 
            <p>You can check this in the error response from JIRA &nbsp;If there is an&nbsp;<strong><code>X-Seraph-LoginReason</code></strong>&nbsp;header with a a value of&nbsp;<strong><code>AUTHENTICATION_DENIED </code></strong><code>or</code><strong><code>&nbsp;<strong>AUTHENTICATED_FAILED</strong></code></strong>, this means the application rejected the login without even checking the password. This is the most common indication that JIRA's CAPTCHA feature has been triggered.</p>
