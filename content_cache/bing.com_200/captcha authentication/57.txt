     <h1><img src="images/lock.png" alt="totp-me"> totp-me - TOTP for Java ME</h1> 
     <p>Java ME TOTP (<a class="externalLink" href="http://tools.ietf.org/html/rfc6238">RFC 6238</a>) authenticator. It can be used as a token generator for</p> 
     <ul> 
      <li>Google’s two phase authentication</li> 
      <li>LinOTP authentication</li> 
      <li>other authentication servers which support TOTP</li> 
     </ul> 
     <div class="section"> 
      <h2>Download<a name="Download"></a></h2> 
      <p>Download the <a class="externalLink" href="https://sourceforge.net/projects/totpme/files/latest/download">latest binaries</a> from the <a class="externalLink" href="https://sourceforge.net/projects/totpme/">SourceForge project pages</a>.</p> 
      <p>If you want to use direct installation to your device, use either <a class="externalLink" href="http://totpme.sourceforge.net/totp-me.jad">JAD</a> or <a class="externalLink" href="http://totpme.sourceforge.net/totp-me.jar">JAR</a>.</p>
     </div> 
     <div class="section"> 
      <h2>Features<a name="Features"></a></h2> 
      <p>Key (and only) features:</p> 
      <ul> 
       <li>multiple profiles/accounts supported</li> 
       <li>configurable parameters</li> 
       <li>secret key</li> 
       <li>profile name</li> 
       <li>digest algorithm: SHA-1 (default), SHA-256, SHA-512</li> 
       <li>number of token digits (default is 6)</li> 
       <li>time step (default is 30)</li> 
       <li>time correction (default is 0) - advanced feature, the value (may be negative) is added to the device’s system time during computing a token value</li> 
       <li>input validation</li> 
       <li>key generator with Base32 and HEX output</li> 
      </ul>

      <h2>How to install it<a name="How_to_install_it"></a></h2> 
      <p>Unzip files from the distribution package and copy <tt>totp-me.jar</tt> to your device which supports Java ME. Some devices may also need the description file <tt>totp-me.jad</tt> to be copied together with the JAR.</p>
     </div> 
     <div class="section"> 
      <h2>How to use it<a name="How_to_use_it"></a></h2> 
      <p>You either have already a secret key, then fill it as Base32 encoded String after the start (Options form is displayed if no key is set already). Or you can generate the secret key directly by the application and then fill it as a shared secret in the authentication server.</p> 
      <div class="section"> 
       <h3>Generate new secret key<a name="Generate_new_secret_key"></a></h3> 
       <ul> 
        <li>choose your preferred digest algorithm in the <tt>Options</tt> screen (the default is <tt>SHA-1</tt>)</li> 
        <li>choose <tt>Key generator</tt> from the menu - it will switch you to screen for generating the new key</li> 
        <li>use <tt>New key</tt> command to generate a new key, you can use it more times if you are not satisfied with the generated value</li> 
        <li>fill the <tt>HEX</tt> value in you authentication server configuration</li> 
        <li>press <tt>OK</tt> command and you will be switched back to the <tt>Options</tt> screen; confirm your configuration and press <tt>OK</tt> command again</li> 
        <li>if no problem occurs, you are switched to the main application screen, where the <tt>Token</tt> value is already present</li> 
        <li><i>you can synchronize the authentication server with your token now</i></li> 
       </ul>
      </div>
