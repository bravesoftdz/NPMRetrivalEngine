        <span id="Overview"><h2>Overview</h2></span> 
        <span id="Http_Authentication"><h3>Http Authentication</h3></span> 
        <p>* Java SE supports:<br> - HTTP Basic authentication<br> - HTTP Digest authentication<br> - NTLM<br> - Http SPNEGO Negotiate with Kerberos and NTLM<br> * Uses java.net.Authenticator to<br> - enable authentication<br> - provide access to a store of usernames and passwords<br> * Work with both proxies and servers</p> 
        <span id="java.net.Authenticator"><h3>java.net.Authenticator</h3></span> 
        <p>* Is an abstract class.<br> * Represents an object that knows how to obtain authentication for a network connection, usually by prompting user for information.</p> 
        <span id=""><h6><a href="http://jianmingli.com/wp/wp-content/uploads/2012/11/JavaNetAuthenticator_classDiagram.jpg"><img src="http://jianmingli.com/wp/wp-content/uploads/2012/11/JavaNetAuthenticator_classDiagram-300x225.jpg" alt="" title="JavaNetAuthenticator_classDiagram" width="300" height="225" class="aligncenter size-medium wp-image-6726" srcset="http://jianmingli.com/wp/wp-content/uploads/2012/11/JavaNetAuthenticator_classDiagram-300x225.jpg 300w, http://jianmingli.com/wp/wp-content/uploads/2012/11/JavaNetAuthenticator_classDiagram.jpg 841w" sizes="(max-width: 300px) 100vw, 300px"></a></h6></span> 
        <p>* Subclasses override<br> - <em>getPasswordAuthentication()</em> to acquire username and password.<br> - <em>setDefault(Authenticator)</em> to register with the system and <strong>enable</strong> autehntication.<br> * Example:</p>

        <span id="Authentication_Schemes"><h2>Authentication Schemes</h2></span> 
        <p>* Scheme preference: <em>GSS/SPNEGO -&gt; Digest -&gt; NTLM -&gt; Basic</em><br> * Specify no fallback with system proprety: <em>http.auth.preference</em></p> 
        <span id="Http_Basic"><h3>Http Basic</h3></span> 
        <p>* Not secure<br> * Defined in RFC 2317<br> * Username and password are encoded in base64<br> * <em>getRequestingPrompt()</em> returns the basic authentication realm as provided by the server</p> 
        <span id="Http_Digest"><h3>Http Digest</h3></span> 
        <p>* More secure than Basic.<br> * Username and password are hashed with MD5<br> * Turn on mutual authentication:</p>

        <span id="NTLM"><h3>NTLM</h3></span> 
        <p>* More secure than Basic, but less secure than Digest.<br> * Can be used with proxies or servers, but <strong>not</strong> both at the same time.<br> * Specify domain name:<br> - Prefix username with domain name followed by a backslash: <em>domainName\username</em><br> - Use system property: <em>http.auth.ntlm.domain</em></p> 
        <span id="SPNEGO"><h3>SPNEGO</h3></span> 
        <p>* Negotiate scheme<br> * Only supports<br> - NTLM<br> - Kerberos</p> 
        <span id="Kerberos_5_Configuration"><h4>Kerberos 5 Configuration</h4></span>
