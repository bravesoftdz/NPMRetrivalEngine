       <p> Java NIO (New IO) is an alternative IO API for Java (from Java 1.4), meaning alternative to the standard <a href="/java-io/index.html">Java IO</a> and <a href="/java-networking/index.html">Java Networking</a> API's. Java NIO offers a different way of working with IO than the standard IO API's. </p> 
       <h2>Java NIO: Channels and Buffers</h2> 
       <p> In the standard IO API you work with byte streams and character streams. In NIO you work with channels and buffers. Data is always read from a channel into a buffer, or written from a buffer to a channel. </p> 
       <h2>Java NIO: Non-blocking IO</h2> 
       <p> Java NIO enables you to do non-blocking IO. For instance, a thread can ask a channel to read data into a buffer. While the channel reads data into the buffer, the thread can do something else. Once data is read into the buffer, the thread can then continue processing it. The same is true for writing data to channels. </p> 
       <h2>Java NIO: Selectors</h2> 
       <p> Java NIO contains the concept of "selectors". A selector is an object that can monitor multiple channels for events (like: connection opened, data arrived etc.). Thus, a single thread can monitor multiple channels for data. </p> 
       <p> How all this works is explained in more detail in the next text in this series - the Java NIO overview. </p>
