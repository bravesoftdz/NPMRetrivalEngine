             <div class="ibm-container ibm-alternate"> 
              <h2 class=" ibm-h4 ibm-bold">Technote (troubleshooting)</h2> 
              <div class="ibm-container-body"> 
               <br> 
               <h2 class=" ibm-h4 ibm-bold">Problem(Abstract)</h2> 
               <p> <br> Some types of Java Objects ( e.g., Java Swing related ) are recognized according to an index value rather than by their Object name. This can be a problem if the index values change with recompilations. <br> <br> Example: A Java Swing application consists of some push buttons ( JButton) and a <br> JTextField (editbox). When you select the pushbutton Robot object recognition is based on the object name. However, when you select the JTextField - the Robot object recognition is based on the Index value.<br> <br> This is a problem for the customer because these indexes keep changing making it hard for them to update the scripts when ever the screen gets changes. The object names are unique. So we would want the robot to record the object names. <br> </p> 
               <h2 class=" ibm-h4 ibm-bold">Cause</h2> 
               <div class="ibm-domino-rtf"> 
                <p>Robot by default uses "accessible name" rather than the "name" set on the particular object ( Object.setName() ).</p> 
                <h2 class=" ibm-h4 ibm-bold">Resolving the problem</h2> 
                <div class="ibm-domino-rtf"> 
                 <p>Forcing object recognition by Object Name is accomplished by the following technique: </p> 
                 <p>Go into the Windows Registry. Navigate to the key directory named: HKEY_CURRENT_USER\Software\Rational Software\Rational Test\8\Robot.<br> <br> Now add a new string value called: "JavaUseAnyProgrammaticName".<br> Set its value to the ( string ) value of 1. Now start robot and record.<br> This will remedy the problem for all new recordings (with no Java application changes).<br> <br> In addition, or if preferred, another technique will also work that requires a modification to the Java application itself. For the JTextField object, the following line of could be added to make sure that the robot succeeds in using name based Object recognition.<br> <br> JTextField jt = new JTextField();<br> jt.getAccessibleContext().setAccessibleName("Test"); </p> 
                 <p> </p> 
                 <p>Either of these approaches can be used to force name based Object Recognition.<br> <br> <br> <br> </p> 
                </div>
