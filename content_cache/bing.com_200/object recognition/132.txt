         <h1 class="page-title"><span class="field field--name-title field--type-string field--label-hidden">How to put common code in Scala package objects</span> </h1> 
        </div> 
        <div id="block-aadotcom-content" class="block block-system block-system-main-block"> 
         <article data-history-node-id="6684" role="article" about="/scala/scala-package-objects-how-to-name-location-cookbook-recipe" class="node node--type-blog node--view-mode-full"> 
          <div class="submitted_by">
           By Alvin Alexander. Last updated: August 30 2017
          </div> 
          <div class="node__content"> 
           <div class="clearfix text-formatted field field--name-body field--type-text-with-summary field--label-hidden field__item">
            <p>This is an excerpt from the <a href="http://amzn.to/2c0XcgG">Scala Cookbook</a> (partially modified for the internet). This is Recipe 6.7, “How to put common code in Scala <em>package objects</em>.”</p> 
            <h2>Problem</h2> 
            <p>You want to make <a href="http://scala-lang.org/">Scala</a> functions, fields, and other code available at a package level, without requiring a class or object.</p> 
            <h2>Solution</h2> 
            <p>Put the code you want to make available to all classes within a package in a package object.</p> 
            <p>By convention, put your code in a file named <em>package.scala</em> in the directory where you want your code to be available. For instance, if you want your code to be available to all classes in the <em>com.alvinalexander.myapp.model</em> package, create a file named <em>package.scala</em> in the <em>com/alvinalexander/myapp/model</em> directory of your project. In the <em>package.scala</em> source code, remove the word <code>model</code> from the end of the package statement, and use that name to declare the name of the package object. Including a blank line, the first three lines of your file will look like this:</p> 
            <pre>
package com.alvinalexander.myapp

package object model {</pre> 
            <p>Now write the rest of your code as you normally would. The following example shows how to create a field, method, enumeration, and type definition in your package object:</p> 
            <pre>
package com.alvinalexander.myapp

package object model {

    // field
    val MAGIC_NUM = 42

    // method
    def echo(a: Any) { println(a) }

    // enumeration
    object Margin extends Enumeration {
        type Margin = Value
        val TOP, BOTTOM, LEFT, RIGHT = Value
    }

    // type definition
    type MutableMap[K, V] = scala.collection.mutable.Map[K, V]
    val MutableMap = scala.collection.mutable.Map

}</pre> 
            <p>You can now access this code directly from within other classes, traits, and objects in the package <em>com.alvinalexander.myapp.model</em> as shown here:</p> 
            <pre>
package com.alvinalexander.myapp.model

object MainDriver extends App {

    // access our method, constant, and enumeration
    echo("Hello, world")
    echo(MAGIC_NUM)
    echo(Margin.LEFT)

    // use our MutableMap type (scala.collection.mutable.Map)
    val mm = MutableMap("name" -&gt; "Al")
    mm += ("password" -&gt; "123")
    for ((k,v) &lt;- mm) printf("key: %s, value: %s\n", k, v)

}</pre> 
            <h2>Discussion</h2> 
            <p>The most confusing part about package objects is where to put them, along with what their package and object names should be.</p> 
            <p>Where to put them isn’t too hard; by convention, create a file named <em>package.scala</em> in the directory where you want your code to be available. In the example shown, I want the package code to be available in the <em>com.alvinalexander.myapp.model</em> package, so I put the file <em>package.scala</em> in the <em>com/alvinalexander/myapp/model</em> source code directory:</p> 
            <pre>
+-- com
    +-- alvinalexander
        +-- myapp
            +-- model
                +-- package.scala</pre> 
            <p>In regards to the first few lines of the <em>package.scala</em> source code, simply start with the usual name of the package:</p> 
            <pre>
package com.alvinalexander.myapp.model</pre> 
            <p>Then take the name of the last package level (model) off that statement, leaving you with this:</p> 
            <pre>
package com.alvinalexander.myapp</pre> 
            <p>Then use that name (<code>model</code>) as the name of your package object:</p> 
            <pre>
package object model {</pre> 
            <p>As shown earlier, the first several lines of your <em>package.scala</em> file will look like this:</p> 
            <pre>
package com.alvinalexander.myapp
package object model {</pre> 
            <p>The Scala package object documentation states, “Any kind of definition that you can put inside a class, you can also put at the top level of a package.” In my experience, package objects are a great place to put methods and functions that are common to the package, as well as constants, enumerations, and implicit conversions.</p> 
            <p>As described in the second page of the Scala package object documentation, “The standard Scala package also has its package object. Because <code>scala._</code> is automatically imported into every Scala file, the definitions of this object are available without prefix.” If you create something like a <code>StringBuilder</code> or <code>Range</code>, you’re using this code.</p>

            <h2>See Also</h2> 
            <p>Scala’s root package object is full of type aliases, like these:</p> 
            <pre>
type Throwable = java.lang.Throwable
type Exception = java.lang.Exception
type Error     = java.lang.Error

type RuntimeException     = java.lang.RuntimeException
type NullPointerException = java.lang.NullPointerException
type ClassCastException   = java.lang.ClassCastException</pre> 
            <p>Like the <code>Predef</code> object, its source code is worth looking at if you want to know more about how Scala works. You can find its source by following the “source” link on its Scaladoc page.</p>
