     <h1 id="introduction">Introduction</h1> 
     <div class="annotation-block"> 
      <div class="annotation-block--left"> 
       <p> <strong>Similarity Search and collections closed</strong>: <em>As of September 8, 2017, the beta period for Similarity Search is closed. For more information, see <a target="_blank" href="https://www.ibm.com/blogs/bluemix/2017/08/visual-recognition-api-similarity-search-update/">Visual Recognition API – Similarity Search Update</a>.</em> </p> 
       <p> The IBM Watson™ Visual Recognition service identifies scenes, objects, and celebrity faces in images you upload to the service. You can create and train a custom classifier to identify subjects that suit your needs. </p> 
       <p class="method-node"> Descriptions of Node classes referred to in this reference are available in the Node documentation for the <a target="_blank" href="https://github.com/watson-developer-cloud/node-sdk">Watson Developer Cloud Node.js SDK</a>. </p>

     <h2 id="classify_an_image">Classify an image</h2> 
     <p> Classify a URL, or upload a single image or a .zip file of multiple images with the built-in classes. To identify custom classifiers, include the <span class="method-curl method-node method-python"><strong>classifier_ids</strong> or <strong>owners</strong> parameters.</span><span class="method-java"><strong>classifierIDs</strong> parameter.</span> </p> 
     <pre class="method-curl"><code class="nohighlight">GET /v3/classify
POST /v3/classify</code></pre> 
     <pre class="method-java"><code class="lang-java">ServiceCall&lt;VisualClassification&gt; classify(ClassifyImagesOptions options)</code></pre> 
     <!-- GET/POST classify curl annotation block --> 
     <div class="annotation-block"> 
      <!-- GET classify classify parameters --> 
      <div class="annotation-block method-curl"> 
       <div class="annotation-block--left"> 
        <h3 id="classify_an_image-get">GET Request</h3> 
        <p> Use the <tt>GET</tt> method to classify URLs against classifiers as a query parameter. </p> 
        <table class="align-right-1 unwrap-1"> 
         <thead> 
          <tr> 
           <th>Name</th> 
           <th>Description</th> 
          </tr> 
         </thead> 
         <tbody>
          <tr> 
           <td> Accept-Language <span class="type">header</span> <span class="type">string</span> </td> 
           <td> <p> The 2-letter primary language code as assigned in ISO standard 639. Supported languages are <tt>en</tt> (English), <tt>ar</tt> (Arabic), <tt>de</tt> (German), <tt>es</tt> (Spanish), <tt>it</tt> (Italian), <tt>ja</tt> (Japanese), and <tt>ko</tt> (Korean). </p> <p> The response might not be in the specified language in these conditions: </p> 
            <ul> 
             <li>English is returned when the requested language is not supported (see the parameter description for details).</li> 
             <li>Classes are not returned when there is no translation for them.</li> 
             <li>Custom classifiers returned with this method return tags in the language of the custom classifier.</li> 
            </ul> </td> 
          </tr> 
          <tr> 
           <td> api_key <span class="type">query</span> <span class="type">string</span> <span class="req"></span> </td> 
           <td> Your API key. </td> 
          </tr> 
          <tr> 
           <td> version <span class="type">query</span> <span class="type">string</span> <span class="req"></span> </td> 
           <td> The release date of the version of the API you want to use. Specify dates in YYYY-MM-DD format. The current version is <tt>2016-05-20</tt>. </td> 
          </tr> 
          <tr> 
           <td> url <span class="type">query</span> <span class="type">string</span> <span class="req"></span> </td> 
           <td> <p> The URL of an image (.jpg, .png). The minimum recommended pixel density is 32X32 pixels per inch, and the maximum image size is 2 MB. </p> <p> Redirects are followed, so you can use a shortened URL. The resolved URL is returned in the response. </p> </td> 
          </tr> 
          <tr> 
           <td> owners <span class="type">query</span> <span class="type">string[, string...]</span> </td> 
           <td> A comma-separated list with the values <tt>IBM</tt>, <tt>me</tt>, or both to specify which classifiers to run. </td> 
          </tr> 
          <tr> 
           <td> classifier_ids <span class="type">query</span> <span class="type">string[, string...]</span> </td> 
           <td> A comma-separated list of the classifier IDs used to classify the images. "default" is the classifier_id of the built-in classifier. A specialty "food" classifier is also available, to enhance specificity and accuracy for images of food items. </td> 
          </tr> 
          <tr> 
           <td> threshold <span class="type">query</span> <span class="type">number</span> </td> 
           <td> A floating value that specifies the minimum score a class must have to be displayed in the response. The default threshold for returning scores from a classifier is <tt>0.5</tt>. Set the threshold to <tt>0.0</tt> to ignore the classification score and return all values. </td> 
          </tr> 
         </tbody>
        </table>

       <div class="annotation-block--right">

This feature uses a built-in classifier, so you do not train it on custom classifiers. The Detect faces method does not support general biometric facial recognition. </p>
