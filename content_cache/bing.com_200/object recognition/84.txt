       <h1>Object Recognition Kitchen<a class="headerlink" href="#object-recognition-kitchen" title="Permalink to this headline">¶</a></h1> 
       <p>The Object Recognition Kitchen (<code class="docutils literal"><span class="pre">ORK</span></code>) is a project started at Willow Garage for object recognition.</p> 
       <p>There is currently no unique method to perform object recognition. Objects can be textured, non textured, transparent, articulated, etc. For this reason, the Object Recognition Kitchen was designed to easily develop and run simultaneously several object recognition techniques. In short, ORK takes care of all the non-vision aspects of the problem for you (database management, inputs/outputs handling, robot/ROS integration ...) and eases reuse of code.</p> 
       <p><code class="docutils literal"><span class="pre">ORK</span></code> is built on top of <a class="reference external" href="http://plasmodic.github.com/ecto">ecto</a> which is a lightweight hybrid C++/Python framework for organizing computations as directed acyclic graphs.</p> 
       <p class="rubric">Install</p> 
       <p>Well, all good things must start so check out the <a class="reference internal" href="install.html#install"><span>Install</span></a>.</p> 
       <p class="rubric">Quickguide</p> 
       <p>We know you can’t wait; if you don’t care about the intricacies and want to have a quick overview, follow the <a class="reference internal" href="quickguide.html#quickguide"><span>Quick Guide</span></a></p> 
       <p class="rubric">Tutorials</p> 
       <p>Ok, now that you know a little, you can follow the <a class="reference external" href="http://wg-perception.github.com/ork_tutorials/index.html#object-recognition-tutorials" title="(in object_recognition_tutorials v)"><span class="xref std std-ref">Tutorials</span></a>.</p> 
       <p class="rubric">General Usage</p> 
       <p>Now that you have a bit more time, we suggest you read about the <a class="reference internal" href="infrastructure/infrastructure.html#infrastructure"><span>Infrastructure</span></a> to understand how to interact with <code class="docutils literal"><span class="pre">ORK</span></code>. You can then go through the different steps of object recognition:</p> 
       <blockquote> 
        <div>
         <ul class="simple"> 
          <li><a class="reference external" href="http://wg-perception.github.com/capture/index.html#ork-capture" title="(in object_recognition_capture v0.1.0)"><span class="xref std std-ref">Data Capture</span></a> ...</li> 
          <li>... from which you can perform <a class="reference internal" href="training/training.html#training"><span>Training</span></a> of object models ...</li> 
          <li>... that you can then use for <a class="reference internal" href="detection/detection.html#detection"><span>Detection</span></a></li> 
         </ul> 
        </div>
       </blockquote> 
       <p class="rubric">ROS integration</p> 
       <p>The recognition kitchen was built in a ROS agnostic way, but ROS components were also developed for integration with the ROS ecosystem (e.g. publishers, subscribers, actionlib server, RViz plugin ...). For more info, check out the <a class="reference external" href="http://wg-perception.github.com/object_recognition_ros/index.html#ros" title="(in object_recognition_ros v)"><span class="xref std std-ref">ROS Integration</span></a>.</p>
