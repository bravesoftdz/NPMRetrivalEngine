      <h1> <a href="#es6-javascript"></a>es6-javascript</h1> 
      <p>A collection of commands and ES6 focused snippets for optimizing modern Javascript development productivity. It aims to be compliant with <a href="https://github.com/airbnb/javascript">AirBnB's mostly reasonable approach to Javascript</a>.</p> 
      <p><em>Note: this is a fork of <a href="https://github.com/hharnisc/atom-es6-javascript/blob/v1.0.0/extrabacon/atom-turbo-javascript">turbo-javascript</a> that uses arrow functions by default and adds a few more snippets for chai and classes for convenience.</em></p> 
      <h2> <a href="#changelog"></a>Changelog</h2> 
      <p><strong>Version 1.0.0</strong></p> 
      <ul> 
       <li>Uses single quotes to comply with AirBnB guidelines</li> 
      </ul> 
      <p>If you need double quotes please downgrade <code>apm install es6-javascript@0.7.0</code></p> 
      <h2> <a href="#commands"></a>Commands</h2> 
      <p>Use the following keymaps to speed up your development. You can quickly terminate lines with semicolons or manipulate blocks of code with ease.</p> 
      <h4> <a href="#end-line-ctrl-"></a>End Line <code>CTRL-;</code> </h4> 
      <p>Terminates the current line with a semicolon.</p> 
      <h4> <a href="#end-line-with-a-comma-ctrl-"></a>End Line with a comma <code>CTRL-,</code> </h4> 
      <p>Terminates the current line with a comma (great for object literals).</p> 
      <h4> <a href="#end-new-line-ctrl-enter"></a>End New Line <code>CTRL-ENTER</code> </h4> 
      <p>Terminates the current line with a colon or semicolon, followed with a new line. A comma is inserted when the cursor is inside an object literal, otherwise a semicolon is inserted.</p> 
      <h4> <a href="#easy-blocks-ctrl-b"></a>Easy Blocks <code>CTRL-B</code> </h4> 
      <p>Creates a statement block <code>{ ... }</code> with the selected text placed inside and properly indented. If the selection is already wrapped with a block, the block is removed and its content is unindented.</p> 
      <h2> <a href="#snippets"></a>Snippets</h2> 
      <p>Snippets are optimized to be short and easy to remember. Some snippets are "chainable" and render differently when preceded by a ".".</p> 
      <p>For example, <code>.fe</code> renders a chain-friendly version of the forEach snippet, while <code>fe</code> renders a full code block.</p>
