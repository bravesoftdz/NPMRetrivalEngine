      <dl class="docutils" id="install"> 
       <dt>
        To use ORK, you will need a compiled ORK and couchDB (for object management). Following instructions in this page will help you to:
       </dt> 
       <dd>
        <ul class="first last simple"> 
         <li>Install ORK either from precompiled ROS packages or from source code</li> 
         <li>Install couchDB and setup the database for ORK</li> 
         <li>Compile the documentation so that you can consult it locally (useful when no internet connection is around)</li> 
        </ul> 
       </dd> 
      </dl> 
      <div class="section" id="installation"> 
       <h1>Installation<a class="headerlink" href="#installation" title="Permalink to this headline">¶</a></h1> 
       <div class="section" id="from-the-ros-repositories"> 
        <h2>From the ROS repositories<a class="headerlink" href="#from-the-ros-repositories" title="Permalink to this headline">¶</a></h2> 
        <p>Install the <code class="docutils literal"><span class="pre">ros-*-object-recognition-core</span></code> package from the official ROS repositories and the different pipelines (packages starting with <code class="docutils literal"><span class="pre">ros-*-object-recognition-</span></code>).</p> 
       </div>

         <div class="toggleable_div label_Non-ROS">
          <p>If you want to install from source without ROS, you need to have common dependencies (OpenCV, PCL) on your path. You also need to execute the following:</p> 
          <div class="highlight-bash">
           <div class="highlight">
            <pre>git clone http://github.com/ros/catkin.git
ln -s catkin/cmake/toplevel.cmake CMakeLists.txt
<span class="nb">cd</span> ../ <span class="o">&amp;&amp;</span> git clone http://github.com/ros-infrastructure/catkin_pkg.git
<span class="nb">export</span> <span class="nv">PYTHONPATH</span><span class="o">=</span><span class="sb">`</span><span class="nb">pwd</span><span class="sb">`</span>/catkin_pkg/src:$PYTHONPATH
<span class="nb">cd</span> src
</pre>
           </div> 
          </div> 
          <p><code class="docutils literal"><span class="pre">catkin</span></code> is a set of CMake macros that simplify build and maintenance.</p> 
          <p>Then install the <code class="docutils literal"><span class="pre">ecto</span></code> modules:</p> 
          <div class="highlight-sh">
           <div class="highlight">
            <pre>git clone http://github.com/plasmodic/ecto
git clone http://github.com/plasmodic/ecto_image_pipeline
git clone http://github.com/plasmodic/ecto_openni
git clone http://github.com/plasmodic/ecto_opencv
git clone http://github.com/plasmodic/ecto_pcl
git clone http://github.com/plasmodic/ecto_ros
git clone http://github.com/wg-perception/opencv_candidate
</pre>
           </div> 
          </div> 
         </div>
        </div> 
        <div class="container"> 
         <div class="toggleable_div label_ROS">
          <p>First source your ROS setup file (adapt <code class="docutils literal"><span class="pre">DISTRO</span></code> to the ROS distribution you are using):</p> 
          <div class="highlight-sh">
           <div class="highlight">
            <pre><span class="nb">export</span> <span class="nv">DISTRO</span><span class="o">=</span>indigo
sudo apt-get install libopenni-dev ros-<span class="si">${</span><span class="nv">DISTRO</span><span class="si">}</span>-catkin ros-<span class="si">${</span><span class="nv">DISTRO</span><span class="si">}</span>-ecto* ros-<span class="si">${</span><span class="nv">DISTRO</span><span class="si">}</span>-opencv-candidate ros-<span class="si">${</span><span class="nv">DISTRO</span><span class="si">}</span>-moveit-msgs
<span class="nb">source</span> /opt/ros/<span class="si">${</span><span class="nv">DISTRO</span><span class="si">}</span>/setup.sh
</pre>
           </div> 
          </div> 
         </div>
        </div> 
        <p>Then install any pipeline you need:</p> 
        <div class="highlight-sh">
         <div class="highlight">
          <pre>git clone http://github.com/wg-perception/object_recognition_core
git clone http://github.com/wg-perception/capture
git clone http://github.com/wg-perception/reconstruction
git clone http://github.com/wg-perception/linemod
git clone http://github.com/wg-perception/ork_renderer
git clone http://github.com/wg-perception/tabletop
git clone http://github.com/wg-perception/tod
git clone http://github.com/wg-perception/transparent_objects
</pre>
         </div>

       <div class="section" id="rosinstall-file"> 
        <h2>rosinstall file<a class="headerlink" href="#rosinstall-file" title="Permalink to this headline">¶</a></h2> 
        <p>Under ROS, you can alternatively use that <a class="reference external" href="http://www.ros.org/wiki/rosinstall">rosinstall</a> file <a class="reference download internal" href="_downloads/ork.rosinstall.indigo.jade" download=""><code class="xref download docutils literal"><span class="pre">ork.rosinstall.indigo.jade</span></code></a> on Indigo/Jade or <a class="reference download internal" href="_downloads/ork.rosinstall.kinetic.plus" download=""><code class="xref download docutils literal"><span class="pre">ork.rosinstall.kinetic.plus</span></code></a> on Kinetic and above.</p> 
        <p>To use on Kinetic, do the following:</p> 
        <div class="highlight-bash">
         <div class="highlight">
          <pre>mkdir ws <span class="o">&amp;&amp;</span> <span class="nb">cd</span> ws
wstool init src https://raw.github.com/wg-perception/object_recognition_core/master/doc/source/ork.rosinstall.kinetic.plus
<span class="nb">cd</span> src <span class="o">&amp;&amp;</span> wstool update -j8
<span class="nb">cd</span> .. <span class="o">&amp;&amp;</span> rosdep install --from-paths src -i -y
catkin_make
<span class="nb">source</span> devel/setup.bash
</pre>
         </div>

      <div class="section" id="building-the-documentation"> 
       <h1>Building the documentation<a class="headerlink" href="#building-the-documentation" title="Permalink to this headline">¶</a></h1> 
       <p>Before you can build the documentation (which you are reading right now), you need to have followed the installation instructions and have already successfully called <code class="docutils literal"><span class="pre">make</span></code> in the <code class="docutils literal"><span class="pre">build</span></code> folder.</p> 
       <p>You then need some up to date packages:</p> 
       <div class="highlight-sh">
        <div class="highlight">
         <pre>sudo pip install -U breathe catkin-sphinx sphinxcontrib-programoutput
</pre>
        </div> 
       </div> 
       <p>From root, just type:</p> 
       <div class="highlight-sh">
        <div class="highlight">
         <pre><span class="nb">cd</span> build
make doxygen
make sphinx-doc
</pre>
        </div> 
       </div> 
       <p>You will find the generated documentation under <code class="docutils literal"><span class="pre">build/doc/html</span></code>.</p> 
       <p>Once the documentation is built, you can simply copy it (except for the <code class="docutils literal"><span class="pre">.doctree</span></code> folder) to the <code class="docutils literal"><span class="pre">gh-pages</span></code> branch on GitHub.</p>
