    <h1>Object Detection with HTML5 getUserMedia</h1>
    <div id="meta">
     <span>Last updated on <strong>March 31, 2015</strong> in </span>
     <a href="http://techslides.com/category/development" title="View all posts in Development" rel="category tag">Development</a>
    </div>
    <p><img src="http://i.imgur.com/H6yLk.jpg" alt="Face Detection with JavaScript"></p> 
    <p>Recently, I have seen a couple different JavaScript <a href="https://github.com/neave/face-detection">face detection</a> <a href="http://wesbos.com/html5-video-face-detection-canvas-javascript/">demos</a> based on the HTML5 getUserMedia API. I even made <a href="http://techslides.com/demos/video-face/">my own face detection demo based on the HTML5 video tag</a>. Beyond human faces, we also have <a href="http://harthur.github.com/kittydar/">face detection for cats in JavaScript</a>, <a href="http://www.patrick-wied.at/blog/nude-js-update-1">nudity detection</a>, and much more with <a href="https://github.com/mtschirs/js-objectdetect">js-objectdetect</a>, a JavaScript library for real-time object detection.<br> <span id="more-630"></span><br> JS-objectdetect let’s you detect eyes, hands, and faces in JavaScript and all of these JavaScript object detection scripts are based on <a href="http://opencv.org/">OpenCV</a>, the defacto computer vision library. But OpenCV only has C++, C, Python and Java interfaces so how are these demos working with just JavaScript? JS-objectdetect, similarly to <a href="https://github.com/foo123/HAAR.js">HAAR.js</a>, are using feature detection based on Haar Cascades in JavaScript (Viola-Jones Algorithm). I believe that this has the most reliable detection based on a partial OpenCV port.</p> 
    <p><a href="https://github.com/liuliu/ccv">CCV</a> was the first port of OpenCV into JavaScript and it powers most of the demos I have found online, but its classifier seems to be <a href="http://liuliu.me/eyes/javascript-face-detection-explained/">less reliable</a> than the one provided by OpenCV (HAAR classifier). Finally, <a href="https://github.com/auduno/headtrackr">headtrackr</a>, a JavaScript library for real-time face and head tracking with a webcam (webRTC/getUserMedia), implements the camshift algorithm also found in OpenCV. If you want to learn more about how OpenCV and object detection works, you should check out these papers on <a href="http://pixel.otago.ac.nz/ipapers/60.pdf">Fast Face Recognition</a> and <a href="http://www.cs.cmu.edu/~efros/courses/LBMV07/Papers/viola-cvpr-01.pdf">Rapid Object Detection using a Boosted Cascade of Simple Features</a>.</p> 
    <p>Beyond detecting certain objects from pixel data coming from images, videos, and webcams, I have also seen many demos that manipulate the getUserMedia feed in various ways. Here are some cool getUserMedia demos:</p> 
    <p><a href="http://webcamtoy.com/">WebCamToy</a> – a toy camera using getUserMedia</p> 
    <p><a href="http://www.shinydemos.com/photo-booth/">Photo Booth</a> – one of many cool examples from shinydemos</p> 
    <p><a href="http://comorichweb.nko3.jitsu.com/">ASCIIGRAM</a> – create ASCII images and videos via getUserMedia.</p> 
    <p><a href="http://pierreloicdoulcet.fr/pixelchat/">PixelRoulette</a> – grabs your picture via getUserMedia and randomly matches you with people on the internet</p> 
    <p><a href="http://www.easymustache.com/">Easy Mustache</a> – put a mustache on your face.</p> 
    <p><a href="http://www.airtightinteractive.com/category/3d/">WebCamMesh</a> – a HTML5 demo that projects webcam video onto a WebGL 3D Mesh.</p> 
    <p><a href="http://demo.simplecv.org/">SimpleCV JS</a> – computer-vision library that brings functionality from the SimpleCV python library to the browser. It uses getUserMedia and then allows you to apply some filters on the images.</p> 
    <p><a href="http://www.soundstep.com/blog/2012/03/22/javascript-motion-detection/">JS Motion Detection</a> – playing xylophone via motion detection in getUserMedia.</p> 
    <p><a href="https://plus.google.com/113862800338869870683/posts/WAEDGeeNvra">Displace</a> – a really cool getUserMedia() and WebGL demo with awesome effect</p> 
    <p><a href="http://p5lyon.tumblr.com/ProcessingJSGetUserMedia">ProcessingJS and GetUserMedia</a> – awesome demo of using the getUserMedia API with ProcessingJS</p> 
    <p><a href="http://html5-demos.appspot.com/static/getusermedia/photobooth.html">Photobooth</a> – capture your webcam picture and apply CSS filters.</p> 
    <p><a href="http://iambrandonn.github.com/WebcamSwiper/">WebcamSwiper</a> lets you turn pages of a book by swiping your hand in front of your webcam.</p> 
    <p><a href="http://auduno.tumblr.com/post/25125149521/head-tracking-with-webrtc">Head tracking with WebRTC</a> – a game where movement is controlled by tracking your head with webRTC and getUserMedia. </p> 
    <p><a href="https://hacks.mozilla.org/2013/02/cross-browser-camera-capture-with-getusermediawebrtc/">Cross-browser camera capture with getUserMedia/WebRTC</a></p> 
    <p>The following demos from Opera are also interesting:</p> 
    <p><a href="http://people.opera.com/shwetankd/external/demos/warholiser/warholiser.htm">Warholiser</a><br> <a href="http://people.opera.com/danield/webapps/instant-camera/">Instant Camera</a><br> <a href="http://people.opera.com/danield/html5/explode/">Explode</a><br> <a href="http://people.opera.com/shwetankd/external/demos/rlcp/rlcolorpicker.htm">RLColorPicker</a><br> <a href="http://people.opera.com/brucel/articles/magic-html5-moustache.html">Magic HTML5 Moustache</a></p> 
    <h3>Final Thoughts</h3> 
    <p>This stuff is really cool and I expect that we will see many more cool websites, libraries, and demos in the future around getUserMedia API and object detection. If you are looking for some good tutorials, I have found the following two articles very informative:</p> 
    <p><a href="http://40win.com/blog/2012/11/14/web-gestures-with-getusermedia-part1/">Web Gestures with getUserMedia</a></p> 
    <p><a href="http://www.smartjava.org/content/face-detection-using-html5-javascript-webrtc-websockets-jetty-and-javacvopencv">Face detection using HTML5, javascript, webrtc, websockets, Jetty and OpenCV</a></p> 
    <p>I should also mention that there is a <a href="http://facedetection.jaysalvat.com/">face detection jQuery plugin</a> for all the jQuery lovers (here is an <a href="http://davidwalsh.name/face-detection-jquery">example</a> of using that plugin) and <a href="https://github.com/peterbraden/node-opencv">node-opencv</a> contains OpenCV bindings for NodeJS if you want to play around with OpenCV in Node.</p> 
    <p><a href="http://webdesign.maratz.com/lab/responsivetypography/realtime/">Realtime Responsive Typography</a> is a project by Marko Dugonji? which uses the headtrackr library mentioned above to adjust the font on the browser screen to the distance that your face is from the monitor (or webcam).</p> 
    <p>Check out this <a href="http://www.storminthecastle.com/2013/05/07/how-you-can-build-an-html5-photobooth-app/">article and demo</a> on quickly building an HTML photo booth app with filters.</p> 
    <h3>External:</h3> 
    <p><a href="http://www.raymondcamden.com/index.cfm/2013/5/20/Capturing-camerapicture-data-without-PhoneGap">Capture mobile camera picture data with pure JavaScript and HTML</a><br> <a href="http://www.shinydemos.com/facekat/">Opera Demo of steering a game with your webcam</a><br> <a href="http://viget.com/extend/masking-video-tags-using-html5-canvas">Masking Video with HTML5 Canvas</a></p>
