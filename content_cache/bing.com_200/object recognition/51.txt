       <h2><b>Java - Package</b></h2>
       <p> A Java <strong>package</strong> is a mechanism for organizing Java classes. Package are used in Java, in-order to avoid name conflicts and to control access of class, <a href="http://www.visionsdeveloper.com/tutorial/java/java-interfaces.jsp">interface</a> and enumeration etc. A package can be defined as a group of similar types of classes, interface, enumeration and <strong>sub-packages</strong>. We can define package using package keyword.</p> 
       <p> &nbsp;</p> 
       <p> <strong>Built-in Package :-&nbsp;</strong></p> 
       <p> Existing built-in java packages like java.lang, java.util, java.sql etc.</p> 
       <p> &nbsp;</p> 
       <p> <strong>User-define package :-</strong></p> 
       <p> Java package created by user to categorized classes, interface and enumeration.</p> 
       <p> &nbsp;</p> 
       <p> <strong>Creating a Package :-</strong></p> 
       <p> &nbsp;</p> 
       <p> Creating a package in java is quite easy. Simply <strong>include a package</strong> command follow by name of package as a first statement of java source file. There can be only one package statement in each source file.</p> 
       <p> &nbsp;</p> 
       <p> If package statement not used in source file than it takes default package for all classes, interfaces and enumeration.</p> 
       <p> &nbsp;</p> 
       <p> <u>Example:</u></p> 
       <pre>
<code class="language-java">
package firstpack;
public class Test{
    public static void main(String args[]){
      System.out.println(&#x201c;Testing package&#x201d;);
    }
}
</code>
</pre> 
       <p>&nbsp;</p> 
       <u>To run this program:</u>
       <p></p> 
       <p> &nbsp;</p> 
       <p> - Create firstpack folder in your currently development working directory.</p> 
       <p> - Compile the source file.</p> 
       <p> - Put the class file into the directory you have created.</p> 
       <p> - Execute the program from development directory.</p> 
       <p> &nbsp;</p> 
       <p> &nbsp;</p> 
       <p> <strong>Package is a way to organize files in Java</strong>, it is used when project consist multiple modules. Package&#x2019;s access level also allows you to protect data from being used by the non-authorized classes.</p> 
       <p> &nbsp;</p> 
       <p> &nbsp;</p> 
       <p> <strong>import keyword:</strong></p> 
       <p> &nbsp;</p> 
       <p> import keyword is used to import built-in and user defined packages into your java source files. &nbsp;If a class wants to use another class in the same package, the package name does not need to be used. Classes in to the same package use without any import statement.</p> 
       <p> &nbsp;</p> 
       <p> Import statement is kept after package statement.</p> 
       <p> package firstpackage;</p> 
       <p> import &nbsp;java.util.Date;</p> 
       <p> &nbsp;</p> 
       <p> There are three ways to refer that is present in different package.</p> 
       <p> <strong>1) Using fully qualified name</strong></p> 
       <p> <u>Example :</u></p> 
       <pre>
<code class="language-java">
class Test extends java.util.Date{

 &lt;&lt;statements&gt;&gt;
}
</code>
</pre> 
       <p> &nbsp;</p> 
       <p> <strong>2) import the class you want to use :</strong></p> 
       <p> <u>Example:</u></p> 
       <pre>
<code class="language-java">
import java.util.Date;

class Test extends Date{
  &lt;&lt;statements&gt;&gt;
}
</code>
</pre> 
       <p> &nbsp;</p> 
       <p> <strong>3) import all the classes from the particular package:</strong></p> 
       <p> <u>Example:</u></p> 
       <pre>
<code class="language-java">
import java.util.*;

class Test extends Date{
  &lt;&lt;statements&gt;&gt;
}
</code>
</pre> 
       <p> &nbsp;</p> 
       <p> &nbsp;</p> 
       <p> <strong>Static import:</strong></p> 
       <p> Static Import is a new feature added in <strong>Java 5 specification</strong>.&nbsp;</p> 
       <p> In order to access static members, it is necessary to qualify references with the class they came from. For example, one must say:</p> 
       <p> &nbsp;</p> 
       <p> double r = Math.cos(Math.PI * theta);</p> 
       <p> or</p> 
       <p> System.out.println("Testing");</p> 
       <p> &nbsp;</p> 
       <p> You may want to avoid unnecessary use of static class members like Math. and System. For this use static import. For example above code when changed using static import is changed to:</p> 
       <p> &nbsp;</p> 
       <pre>
<code class="language-java">
import static java.lang.System.out;
import static java.lang.Math.PI;
import static java.lang.Math.cos;
class Test{
public static void main(String args[]){
  double r = cos(PI * theta);
  out.println("Testing");

}
}
</code>
</pre> 
       <p> &nbsp;</p> 
       <p> So whats the advantage of using above technique? Only advantage that I see is readability of the code. Instead of writing name of static class, one can directly write the method or member variable name.</p> 
       <p> &nbsp;</p> 
       <p> Also keep one thing in mind here. Ambiguous static import is not allowed. i.e. if you have imported java.lang.Math.PI and you want to import mypackage.Someclass.PI, the compiler will throw an error. Thus you can import only one member PI.</p>
