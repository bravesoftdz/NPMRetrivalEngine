              <p>A <strong>package </strong>is a collection of related classes. It helps <strong>Organize </strong>your classes into a folder structure and make it easy to locate and use them. More importantly, it helps improve <strong>re-usability.</strong> </p>
              <p>Each package in Java has its unique name and organizes its classes and interfaces into a separate namespace, or name group. </p>
              <p>Although interfaces and classes with the same name cannot appear in the same package, they can appear in different packages. This is possible by assigning a separate namespace to each package. </p>
              <p><strong>Syntax:-</strong> </p>
              <pre>package;</pre>

              <h3>Importing packages </h3>
              <p>To create an object of a class (bundled in a package), in your code, you have to use its fully qualified name. </p>
              <p><strong>Ex.</strong> </p>
              <pre>java.awt.event.actionListner object = new java.awt.event.actionListner();
</pre>
              <p>But , it could become tedious to type in the long dot-separated package path name for every class you want to use. Instead it is recommended you use the import statement. </p>
              <p><strong>Syntax</strong> </p>
              <pre>import;			</pre>
              <p>Once imported , you can use the class <strong>without </strong>mentioning its fully qualified name. </p>
              <pre>import java.awt.event.*; // * signifies all classes in this package
import javax.swing.JFrame // here only the JFrame class is imported
//Usage
JFrame f = new JFrame; // without fully qualified name.

</pre>
              <p><strong>Assignment 3</strong>: </p>
              <p>To import package </p>
              <p>Step 1) Copy the code into an editor. </p>
              <pre>// Using packages created in earlier assignment
package p3;
import p1.*; //imports classes only in package p1 and NOT  in the sub-package p2
class c3{
  public   void m3(){
     System.out.println("Method m3 of Class c3");
  }
  public static void main(String args[]){
    c1 obj1 = new c1();
    obj1.m1();
    p1.p2.c2 obj2 = new p1.p2.c2();
    obj2.m2();
  }
}

</pre>

              <h2>Packages - points to note:</h2>
              <ul>
               <li>To avoid naming conflicts packages are given names of the domain name of the company in reverse Ex : com.guru99. com.microsoft, com.infosys etc.</li>
               <li>When a package name is not specified , a class is into the default package (the current working directory) and the package itself is given no name. Hence you were able to execute assignments earlier.</li>
               <li>While creating a package, care should be taken that the statement for creating package must be written before any other import statements</li>
              </ul>
              <pre>// not allowed
import package p1.*;
package p3;
</pre> 
              <pre>//correct syntax
package p3;
import package p1.*;
</pre>
              <p>the java.lang package is imported by default for any class that you create in Java. You can study it more in detail <a href="http://docs.oracle.com/javase/7/docs/api/index.html">here.</a></p>

More often than not , you will be using API files in your code. You can see the API documentation <a href="http://java.sun.com/javase/6/docs/">here.</a> </p>
