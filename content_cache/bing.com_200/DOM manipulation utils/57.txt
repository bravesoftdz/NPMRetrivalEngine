            <h3 class="LW_CollapsibleArea_TitleDiv">

             <div class="section">
              <p>When you customize the Microsoft Dynamics 365 command bar, you can configure commands for controls that you add. These commands contain rules that control whether the control is enabled and what action is performed when the control is used. For more information, see <a href="https://msdn.microsoft.com/en-us/library/gg309639.aspx">Customize commands and the ribbon</a>.</p>

           <a id="bkmk_webResources"></a>
           <div>
            <h3 class="LW_CollapsibleArea_TitleDiv">
             <div>
              <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Web resources</span></a>
              <div class="LW_CollapsibleArea_HrDiv">
               <hr class="LW_CollapsibleArea_Hr">
              </div>
             </div></h3>
            <div class="sectionblock">
             <div class="section">
              <p>Microsoft Dynamics 365 provides an organization-owned entity that stores a binary representation of a file that can be accessed by using a URL. This file is called a web resource. There are several types of web resources. A web resource that represents a JavaScript library is called a JavaScript web resource. You can use a webpage (HTML) web resource to provide a user interface with JavaScript libraries included just as you would for files on a web server. Because these files are part of Microsoft Dynamics 365, users who access them are already authenticated. Therefore, you can use Microsoft Dynamics 365 web services without having to write code to authenticate the user. For more information, see <a href="https://msdn.microsoft.com/en-us/library/gg309473.aspx">Web resources for Microsoft Dynamics 365</a> and <a href="https://msdn.microsoft.com/en-us/library/gg490659.aspx">Work with Dynamics 365 data using web resources</a>.</p>

         <a id="BKMK_UsingjQuery"></a>
         <div>
          <h2 class="LW_CollapsibleArea_TitleDiv">
           <div>
            <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Use of jQuery</span></a>
            <div id="Anchor_2" class="LW_CollapsibleArea_Anchor_Div">
             <a href="/en-us/library/hh771584.aspx#Anchor_2" class="LW_CollapsibleArea_Anchor_Img" title="Right-click to copy and share the link for this section"></a>
            </div>
            <div class="LW_CollapsibleArea_HrDiv">
             <hr class="LW_CollapsibleArea_Hr">
            </div>
           </div></h2>
          <div class="sectionblock">
           <div class="section">
            <dl class="definitionTable">
             <dt>
              <span class="label">Use jQuery with HTML web resources</span>
             </dt>
             <dd>
              <p>We recommend that you use jQuery together with HTML web resources to provide user interfaces because it is an excellent cross-browser library.</p>
              <p>With HTML web resources, you control the libraries that are present and there is no restriction against manipulating the DOM. Feel free to use jQuery within your HTML Web resources.</p>
             </dd>
             <dt>
              <span class="label">Avoid using jQuery with form scripts or ribbon commands</span>
             </dt>
             <dd>
              <p>We do not recommend using jQuery in form scripts and ribbon commands.</p>
              <p>Most of the benefit provided by jQuery is that it allows for easy cross-browser manipulation of the DOM. This is explicitly unsupported within form scripts and ribbon commands. Restrict your scripts to use the <strong>Xrm.Page</strong> and <strong>Xrm.Utility</strong> libraries available in form scripts and ribbon commands. If you decide to use the remaining capabilities of jQuery that are useful with Microsoft Dynamics 365 and include the ability to use <strong>$.ajax</strong>, consider the following:</p>
              <ul class="unordered">
               <li><p>For best performance, don’t load jQuery in the page if you do not need it </p></li>
               <li><p>Using <strong>$.ajax</strong> to perform requests against the Microsoft Dynamics 365 web services is supported, but there are alternatives. The alternative to using <strong>$.ajax</strong> is to use the browsers XMLHttpRequest object directly. The jQuery <strong>$.ajax</strong> method is just a wrapper for this object. If you use the native XMLHttpRequest object directly, you do not need to load jQuery.</p></li>
               <li><p>Each version of jQuery that is loaded in a page can be a different version. Different versions of jQuery have different behaviors and these can cause problems when multiple versions of jQuery are loaded on the same page. There is a technique to mitigate this, but it depends on editing the jQuery library and any other libraries that depend on jQuery. More information: <a href="https://community.dynamics.com/crm/b/develop1/archive/2013/08/08/jquery-and-jquery-ui-with-dynamics-crm-2011-amp-2013.aspx">jQuery and jQuery UI with Dynamics CRM 2011 &amp; 2013</a>, <a href="http://api.jquery.com/jQuery.noConflict/">jQuery.noConflict()</a></p>
                <div class="alert">
                 <table summary="table">
                  <tbody>
                   <tr>
                    <th align="left" scope="col"><span><img id="s-63ed43fe79224917a9711fb438a97d86-system-media-system-caps-note" alt="System_CAPS_note" src="https://i-msdn.sec.s-msft.com/dynimg/IC101471.jpeg" title="System_CAPS_note" xmlns=""></span><span class="alertTitle">Note </span></th>
                   </tr>
                   <tr>
                    <td><p>Following Microsoft Dynamics CRM Online 2015 Update 1, form scripts run in a different scope than the jQuery instance used by the application. This means that there may not be an instance of jQuery available when your code attempts to use jQuery.noConflict. If you must use jQuery, you should first detect whether an instance of jQuery exists before you attempt to use jQuery.noConflict.</p></td>
                   </tr>
                  </tbody>
                 </table>
                </div></li>
              </ul>
             </dd>
            </dl>
           </div>

         <a id="BKMK_WriteJavaScriptForMutlipleBrowsers"></a>
         <div>
          <h2 class="LW_CollapsibleArea_TitleDiv">
           <div>
            <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Write JavaScript for multiple browsers</span></a>
            <div id="Anchor_3" class="LW_CollapsibleArea_Anchor_Div">
             <a href="/en-us/library/hh771584.aspx#Anchor_3" class="LW_CollapsibleArea_Anchor_Img" title="Right-click to copy and share the link for this section"></a>
            </div>
            <div class="LW_CollapsibleArea_HrDiv">
             <hr class="LW_CollapsibleArea_Hr">
            </div>
           </div></h2>
          <div class="sectionblock">
           <div class="section">
            <p>Because you don’t know what browser will be in use, you should make sure that any scripts that you use will work with all supported browsers. Most of the significant differences between Internet Explorer and other browser have to do with HTML and XML DOM manipulation. Because HTML DOM manipulation is not supported, if script logic is only performing supported actions and using the <strong>Xrm.Page</strong> API the changes required to support other browsers could be small. Use the <a href="http://go.microsoft.com/fwlink/p/?LinkId=518207">Custom Code Validation Tool</a> to identify code that only works for Internet Explorer.</p>
            <p>A cross-browser library like jQuery is a good solution for developing web resources but should not be necessary for form scripts or ribbon commands. More information: <a href="#BKMK_UsingjQuery">Use of jQuery</a></p>
           </div>
           <a id="BKMK_SupportedBrowsers"></a>
           <div>
            <h3 class="LW_CollapsibleArea_TitleDiv">
             <div>
              <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Support for other browsers</span></a>
              <div class="LW_CollapsibleArea_HrDiv">
               <hr class="LW_CollapsibleArea_Hr">
              </div>
             </div></h3>
            <div class="sectionblock">
             <div class="section">
              <p>Because Microsoft Dynamics 365 (online &amp; on-premises) supports W3C standards, the application will be accessible using any modern browser and from any platform that supports those standards. However Microsoft Dynamics 365 will only be tested using a specific set of browsers and platforms. For the list of supported platforms and browsers see <a href="https://technet.microsoft.com/library/dn531055.aspx">TechNet: Supported web browsers</a>.</p>
              <p>If you use a browser, or browser version, that is not supported and you type just the server name or server name with organization as the URL, you will be redirected to the Microsoft Dynamics 365 for phones page. The Dynamics 365 for phones pages can be expected to work properly on most browsers, such as browsers used in mobile devices, with limited functionality.</p>

          <div class="sectionblock">
           <div class="section">
            <p>The following sections describe best practices when you use JavaScript with Microsoft Dynamics 365.</p>
           </div>
           <a id="bkmk_avoidUnsupportedMethods"></a>
           <div>
            <h3 class="LW_CollapsibleArea_TitleDiv">
             <div>
              <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Avoid using unsupported methods</span></a>
              <div class="LW_CollapsibleArea_HrDiv">
               <hr class="LW_CollapsibleArea_Hr">
              </div>
             </div></h3>
            <div class="sectionblock">
             <div class="section">
              <p>On the Internet, you can find many examples or suggestions that describe using unsupported methods. These may include leveraging undocumented internal function for page controls. These methods may work but because they are not supported you can’t expect that they will continue to work in future versions of Microsoft Dynamics 365. </p>
              <p>Use the <a href="http://go.microsoft.com/fwlink/p/?LinkId=518207">Custom Code Validation Tool</a> to identify code that is using unsupported methods.</p>
             </div>
            </div>
           </div>
           <a id="bkmk_useJavaScriptFramework"></a>
           <div>
            <h3 class="LW_CollapsibleArea_TitleDiv">
             <div>
              <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Use a cross-browser JavaScript library for HTML web resource user interfaces</span></a>
              <div class="LW_CollapsibleArea_HrDiv">
               <hr class="LW_CollapsibleArea_Hr">
              </div>
             </div></h3>
            <div class="sectionblock">
             <div class="section">
              <p>A cross-browser JavaScript library, such as <a href="http://jquery.com/">jQuery</a>, provides many advantages when developing HTML web resources that must support multiple browsers. JavaScript libraries like jQuery provide a unified development experience for all browsers supported by Microsoft Dynamics 365. These capabilities are appropriate when you are using HTML web resources to provide user interfaces. JavaScript libraries like jQuery provide consistent ways to interact with the Document Object Model (DOM). </p>
             </div>
            </div>
           </div>
           <a id="bkmk_nojQuery"></a>
           <div>
            <h3 class="LW_CollapsibleArea_TitleDiv">
             <div>
              <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Do not use jQuery for form script or commands</span></a>
              <div class="LW_CollapsibleArea_HrDiv">
               <hr class="LW_CollapsibleArea_Hr">
              </div>
             </div></h3>
            <div class="sectionblock">
             <div class="section">
              <p>We do not recommend or support using jQuery for any pages within the application. This includes form scripts and ribbon commands. More information: <a href="#BKMK_UsingjQuery">Use of jQuery</a>.</p>
             </div>
            </div>

           <a id="bkmk_CDNLimitations"></a>
           <div>
            <h3 class="LW_CollapsibleArea_TitleDiv">
             <div>
              <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Recognize limitations for content delivery network (CDN) libraries</span></a>
              <div class="LW_CollapsibleArea_HrDiv">
               <hr class="LW_CollapsibleArea_Hr">
              </div>
             </div></h3>
            <div class="sectionblock">
             <div class="section">
              <p>Content delivery network (CDN) JavaScript libraries provide many advantages for public websites. Because these libraries are hosted on the Internet, you do not need to create web resources that contain the content of the libraries. For Microsoft Dynamics 365 you should consider the following issues before you use a CDN JavaScript library.</p>
              <ul class="unordered">
               <li><p>Users of the Microsoft Dynamics&nbsp;365 for Microsoft Office Outlook with Offline Access client have the capability to work with no Internet connection while working offline. If you are depending on an Internet connection for your JavaScript libraries, your code will fail.</p></li>
               <li><p>Some organizations will restrict Internet access for employees. Unless they configure the network to allow access to the CDN library sites, your code may fail for those organizations.</p></li>
              </ul>
              <p>The alternative to using CDN libraries is to create a script (JavaScript) web resource with the contents of the library. Because web resources are organization-owned entities they will be synchronized when a Microsoft Dynamics 365 for Outlook with Offline Access user goes offline. Because these web resources now become part of the application they will not be blocked if an organization restricts access to the Internet.</p>
             </div>
            </div>
           </div>
           <a id="bkmk_useFeatureDetection"></a>
           <div>
            <h3 class="LW_CollapsibleArea_TitleDiv">
             <div>
              <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Use feature detection when writing functions for multiple browsers</span></a>
              <div class="LW_CollapsibleArea_HrDiv">
               <hr class="LW_CollapsibleArea_Hr">
              </div>
             </div></h3>
            <div class="sectionblock">
             <div class="section">
              <p>Even when you use a cross-browser library like jQuery, you need to be very aware of differences between browsers. You can generally detect which browser is being used by querying the <strong>navigator.useragent</strong> property. This is called browser detection. Browser detection is not a good strategy for most cases because it can’t take into account what features newer versions of a browser have. Also, some browsers provide the capability to modify the <strong>navigation.useragent</strong> property so that they appear to be a different browser. </p>
              <p>Feature detection is the recommended approach. By detecting what features are available, you can create code paths for the browsers you support without knowing exactly which browser is being used. For more information about feature detection, see <a href="https://msdn.microsoft.com/library/hh273397(vs.85).aspx">How to Detect Features Instead of Browsers</a>.</p>
             </div>
            </div>
           </div>
           <a id="bkmk_DoNotAccessDOM"></a>
           <div>
            <h3 class="LW_CollapsibleArea_TitleDiv">
             <div>
              <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Do not access the DOM</span></a>
              <div class="LW_CollapsibleArea_HrDiv">
               <hr class="LW_CollapsibleArea_Hr">
              </div>
             </div></h3>
            <div class="sectionblock">
             <div class="section">
              <p>JavaScript developers are used to interacting with Document Object Model (DOM) elements in code. You might use the <strong>window.getElementById</strong> method or the jQuery library. You are free to use these techniques in your HTML web resources, but they are not supported to access elements in Microsoft Dynamics 365 application pages or entity forms. Instead, access to entity form elements are exposed through the <strong>Xrm.Page</strong> object model. The Microsoft Dynamics 365 development team reserves the right to change how pages are composed, including the <strong>ID</strong> values for elements, so using the <strong>Xrm.Page</strong> object model protects your code from changes in how pages are implemented. For more information, see <a href="https://msdn.microsoft.com/en-us/library/gg328474.aspx">Use the Xrm.Page object model</a>.</p>
             </div>
            </div>
           </div>
           <a id="bkmk_DefineUniqueNames"></a>
           <div>
            <h3 class="LW_CollapsibleArea_TitleDiv">
             <div>
              <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Define unique names for your JavaScript functions</span></a>
              <div class="LW_CollapsibleArea_HrDiv">
               <hr class="LW_CollapsibleArea_Hr">
              </div>
             </div></h3>
            <div class="sectionblock">
             <div class="section">
              <p>When you are the only developer for an HTML page you can easily manage the names of the JavaScript functions you use. In Microsoft Dynamics 365, other solutions may add JavaScript functions to the page where your function is used. </p>
              <p>If two JavaScript functions on a page have the same name, the first function defined is overwritten by the second. For this reason, make sure that you define unique names for your JavaScript functions. For more information, see <a href="https://msdn.microsoft.com/en-us/library/gg309562.aspx#BKMK_CreatingScriptLibraries">Creating script libraries</a>.</p>
             </div>
            </div>
           </div>
           <a id="bkmk_useasynchronous"></a>
           <div>
            <h3 class="LW_CollapsibleArea_TitleDiv">
             <div>
              <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Use asynchronous data access methods</span></a>
              <div class="LW_CollapsibleArea_HrDiv">
               <hr class="LW_CollapsibleArea_Hr">
              </div>
             </div></h3>
            <div class="sectionblock">
             <div class="section">
              <p>When you access data by using the Microsoft Dynamics 365 web services, always use an <a href="https://msdn.microsoft.com/library/ms535874(vs.85).aspx">XMLHttpRequest</a> that is configured to execute asynchronously. The reason is that the browser operates on a single thread. If that thread is being used to execute a long-running process synchronously the browser will stop responding.</p>
              <div class="alert">
               <table summary="table">
                <tbody>
                 <tr>
                  <th align="left" scope="col"><span><img id="s-63ed43fe79224917a9711fb438a97d86-system-media-system-caps-note" alt="System_CAPS_note" src="https://i-msdn.sec.s-msft.com/dynimg/IC101471.jpeg" title="System_CAPS_note" xmlns=""></span><span class="alertTitle">Note </span></th>
                 </tr>
                 <tr>
                  <td><p>Synchronous XMLHttpRequests are deprecated on the main thread of the browser because of the detrimental effects to the end user’s experience. Some browsers now provide a warning when this is detected. If browsers implement the specification at some time in the future an InvalidAccessError exception will be thrown. More information: <a href="http://www.w3.org/TR/XMLHttpRequest/">http://www.w3.org/TR/XMLHttpRequest/#synchronous-flag</a> and <a href="https://xhr.spec.whatwg.org/">https://xhr.spec.whatwg.org/#the-open()-method</a></p></td>
                 </tr>

         <a id="BKMK_DebuggingJavaScript"></a>
         <div>
          <h2 class="LW_CollapsibleArea_TitleDiv">
           <div>
            <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Debugging JavaScript in Microsoft Dynamics 365</span></a>
            <div id="Anchor_5" class="LW_CollapsibleArea_Anchor_Div">
             <a href="/en-us/library/hh771584.aspx#Anchor_5" class="LW_CollapsibleArea_Anchor_Img" title="Right-click to copy and share the link for this section"></a>
            </div>
            <div class="LW_CollapsibleArea_HrDiv">
             <hr class="LW_CollapsibleArea_Hr">
            </div>
           </div></h2>
          <div class="sectionblock">
           <div class="section">
            <p>Each browser provides some kind of debugging extension. Internet Explorer provides developer tools you can use to debug scripts in Microsoft Dynamics 365. The Internet Explorer developer tools can be opened by pressing F12 when viewing a page using Internet Explorer. For more information, see <a href="https://msdn.microsoft.com/library/ie/bg182326">Using the F12 developer tools</a>. </p>
            <p>For Google Chrome, press F12 to open developer tools. <a href="http://getfirebug.com/">Firebug</a> is a popular browser extension for web development using Mozilla Firefox. For Apple Safari, you must first select the <strong>Show Develop menu in menu bar</strong> in <strong>Advanced Preferences</strong>. Then you can select <strong>Show Web Inspector</strong> from the <strong>Develop</strong> menu.</p>
            <p>You can also use Microsoft Visual Studio. For more information, see <a href="http://social.technet.microsoft.com/wiki/contents/articles/how-to-debug-jscript-in-microsoft-dynamics-crm-2011.aspx">How to Debug JScript in Microsoft Dynamics CRM 2011</a>. </p>
            <p>When you use JavaScript libraries in Microsoft Dynamics 365, your libraries are loaded with the web page. It can sometimes be difficult to isolate your specific library in the debugging environment. When using debugging tools in Microsoft Edge, on the <strong>Debugger</strong> tab, click on the folder icon at the top-left corner, and expand the available scripts and find the one with the name that corresponds to the name of your JavaScript web resource, such as the new_myCustomJavaScript.js web resource shown below. You can also search for your JavaScript library by typing the file name in the search box.</p>
            <div class="inset">
             <span><img id="a03496f3-cc31-4a1f-942d-b4be8b14e794" alt="Debugging JavaScript" src="https://i-msdn.sec.s-msft.com/dynimg/IC838701.jpeg" title="Debugging JavaScript" xmlns=""></span>
            </div>
            <p></p>
            <p>Debugging tools for different browsers have similar capabilities. Once you have found your library, you can set a break point and recreate the event that should cause your code to run.</p>
            <p>In the new <a href="http://go.microsoft.com/fwlink/p/?LinkId=626611">interactive service hub client</a> (introduced in Dynamics 365), Dynamics 365 dynamically injects the contents of your JavaScript library instead of loading it with the page. This implies that you cannot set break points in your code because every time the page loads, your code is injected into one of the dynamics libraries that will prevent the break point to be hit. In this case, you will have to use some other techniques to debug your code. More information: <a href="http://go.microsoft.com/fwlink/p/?LinkId=715699">Blog: Debugging JavaScript code in CRM using browser developer tools</a></p>
           </div>
           <a id="bkmk_writeToConsole"></a>
           <div>
            <h3 class="LW_CollapsibleArea_TitleDiv">
             <div>
              <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Write messages to the console</span></a>
              <div class="LW_CollapsibleArea_HrDiv">
               <hr class="LW_CollapsibleArea_Hr">
              </div>
             </div></h3>
            <div class="sectionblock">
             <div class="section">
              <p>Using the <a href="https://msdn.microsoft.com/library/ie/ms535933(v=vs.85).aspx">window.alert method</a> when debugging JavaScript is still a common way to troubleshoot code in the application. But now that all modern browsers provide easy access to debugging tools, it is not a best practice, especially when others might be using the application you are debugging.</p>
              <p>Consider writing your messages to the console instead. The following is a small function you can add to your libraries that you can use to send any messages you want to view to the console when it is open.</p> 
              <div id="code-snippet-1" class="codeSnippetContainer" xmlns=""> 
               <div class="codeSnippetContainerTabs">

                 <div style="color:Black;">
                  <pre>
<span style="color:Blue;">function</span> writeToConsole(message)
{
 <span style="color:Blue;">if</span> (<span style="color:Blue;">typeof</span> console != <span style="color:#A31515;">'undefined'</span>) {
  console.log(message);
 }
}

              <p>Unlike using the alert method, if you forget to remove any code that uses this function, people using the application will not see your messages.</p>
