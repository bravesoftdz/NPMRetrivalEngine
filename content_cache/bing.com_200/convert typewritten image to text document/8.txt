      <p>You can import external graphics using package <code>graphicx</code>. The most important command is <code>\includegraphics</code>. LaTeX itself treats the image like normal text, <i>i.e.</i> as a box of certain height and width.</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
<span class="k">\documentclass</span><span class="nb">{</span>article<span class="nb">}</span>
<span class="k">\usepackage</span><span class="nb">{</span>graphicx<span class="nb">}</span>
<span class="k">\begin</span><span class="nb">{</span>document<span class="nb">}</span>
The following image does not show any wombats
<span class="k">\includegraphics</span><span class="na">[height=\baselineskip]</span><span class="nb">{</span>example-image<span class="nb">}</span>.

<span class="k">\includegraphics</span><span class="na">[height=3cm]</span><span class="nb">{</span>example-image-a<span class="nb">}</span><span class="k">\includegraphics</span><span class="na">[width=5cm]</span><span class="nb">{</span>example-image-b<span class="nb">}</span>

<span class="k">\includegraphics</span><span class="na">[height=3cm]</span><span class="nb">{</span>example-image-a<span class="nb">}</span> <span class="k">\includegraphics</span><span class="na">[width=5cm]</span><span class="nb">{</span>example-image-b<span class="nb">}</span>
<span class="k">\end</span><span class="nb">{</span>document<span class="nb">}</span>
</pre>
      </div> 
      <p>The package documentation list the options <code>width</code> and <code>height</code>, as well as others.</p> 
      <p>Using <code>pdflatex</code> several graphics formats are supported: <code>pdf</code>, <code>png</code> and <code>jpg</code>. Modern installations of LaTeX can use <code>eps</code> files as well, but indirectly.</p> 
      <p>LaTeX in <code>dvi</code>-mode supports only <code>eps</code>-files.</p> 
      <p><br></p> 
      <p><a href="/wiki/File:UsualGraphics.png" class="image" title="width=200px"><img alt="width=200px" src="//upload.wikimedia.org/wikipedia/commons/7/79/UsualGraphics.png" width="745" height="581" data-file-width="745" data-file-height="581"></a></p>

      <h2><span class="mw-headline" id="Converting_graphics">Converting graphics</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Importing_Graphics&amp;action=edit&amp;section=2" title="Edit section: Converting graphics">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <dl> 
       <dt>
        Note
       </dt> 
      </dl> 
      <p>You should also take a look at <a href="/wiki/LaTeX/Export_To_Other_Formats" title="LaTeX/Export To Other Formats">Export To Other Formats</a> for other possibilities.</p> 
      <dl> 
       <dt>
        epstopdf
       </dt> 
      </dl> 
      <p>You can convert EPS to PDF with the <a rel="nofollow" class="external text" href="http://www.ctan.org/tex-archive/support/epstopdf/">epstopdf utility</a>, included in package of the same name. This tool is actually called by <tt>pdflatex</tt> to convert EPS files to PDF in the background when the <span style="font-family: monospace; color: #2020C0; font-weight: normal;">graphicx</span> package is loaded. This process is completely invisible to the user.</p> 
      <p>You can batch convert files using the command-line. In Bourne Shell (Unix) this can be done by:</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
$ <span class="k">for</span> i in *.eps<span class="p">;</span> <span class="k">do</span> epstopdf <span class="s2">"</span><span class="nv">$i</span><span class="s2">"</span><span class="p">;</span> <span class="k">done</span>
</pre>
      </div> 
      <p>In Windows, multiple files can be converted by placing the following line in a <a href="https://en.wikipedia.org/wiki/Batch_file" class="extiw" title="w:Batch file">batch file</a> (a text file with a <tt>.bat</tt> extension) in the same directory as the images:</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
<span class="k">for</span> %%f in <span class="o">(</span>*.eps<span class="o">)</span> <span class="k">do</span> epstopdf %%f
</pre>
      </div> 
      <p>which can then be run from the command line.</p> 
      <p>If <span style="font-family: monospace; color: #2020C0; font-weight: normal;">epstopdf</span> produces whole page with your small graphics somewhere on it, use</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
$ epstopdf --gsopt<span class="o">=</span>-dEPSCrop foo.eps
</pre>
      </div> 
      <p>or try using <tt>ps2pdf</tt> utility which should be installed with Ghostscript (required for any TeX distribution).</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
$ ps2pdf -dEPSCrop foo.eps
</pre>
      </div> 
      <p>to crop final PDF.</p> 
      <dl> 
       <dt>
        eps2eps
       </dt> 
      </dl> 
      <p>When all of the above fails, one can simplify the EPS file before attempting other conversions, by using the <a rel="nofollow" class="external text" href="http://linuxcommand.org/man_pages/eps2eps1.html">eps2eps</a> tool (also see next section):</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
$ eps2eps input.eps input-e2.eps
</pre>
      </div> 
      <p>This will convert all the fonts to pre-drawn images, which is sometimes desirable when submitting manuscripts for publication. However, on the downside, the fonts are NOT converted to lines, but instead to bitmaps, which reduces the quality of the fonts.</p> 
      <dl> 
       <dt>
        imgtops
       </dt> 
      </dl> 
      <p><a rel="nofollow" class="external text" href="http://imgtops.sourceforge.net/">imgtops</a> is a lightweight graphics utility for conversions between raster graphics (JPG, PNG, ...) and EPS/PS files.</p> 
      <dl> 
       <dt>
        Inkscape
       </dt> 
      </dl> 
      <p>Inkscape can also convert files from and to several formats, either from the GUI or from the command-line. For instance, to obtain a PDF from a SVG image you can do:</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
$ inkscape -z -D --file<span class="o">=</span>input.svg --export-pdf<span class="o">=</span>output.pdf
</pre>
      </div> 
      <p>It is possible to run this from within a LaTeX file, the <a href="/w/index.php?title=Template:LaTeX/package&amp;action=edit&amp;redlink=1" class="new" title="Template:LaTeX/package (does not exist)">Template:LaTeX/package</a> package (when running (pdf)latex with the --shell-escape option) can do this using Inkscape's pdf+tex export option, or a simple macro can be used. See <a rel="nofollow" class="external text" href="http://tex.stackexchange.com/q/2099/28808">How to include SVG diagrams in LaTeX? -- Stackexchange</a> See <a href="/wiki/LaTeX/Export_To_Other_Formats" title="LaTeX/Export To Other Formats">Export To Other Formats</a> for more details.</p> 
      <dl> 
       <dt>
        pstoedit
       </dt> 
      </dl> 
      <p>To properly edit an EPS file, you can convert it to an <i>editable</i> format using <a rel="nofollow" class="external text" href="http://www.pstoedit.net/">pstoedit</a>. For instance, to get an Xfig-editable file, do:</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
$ pstoedit -f fig input.eps output.fig
</pre>
      </div> 
      <p>And to get an SVG file (editable with any vector graphics tool like Inkscape) you can do:</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
$ pstoedit -f plot-svg input.eps output.svg
</pre>
      </div> 
      <p>Sometimes pstoedit fails to create the target format (for example when the EPS file contains clipping information).</p> 
      <dl> 
       <dt>
        PDFCreator
       </dt> 
      </dl> 
      <p>Under Windows, <a rel="nofollow" class="external text" href="http://sourceforge.net/projects/pdfcreator/">PDFCreator</a> is an open source software that can create PDF as well as EPS files. It installs a virtual printer that can be accessed from other software having a "print..." entry in their menu (virtually any program).</p> 
      <dl> 
       <dt>
        Raster graphics converters
       </dt> 
      </dl> 
      <ul> 
       <li><a rel="nofollow" class="external text" href="http://pts.szit.bme.hu/sam2p/">Sam2p</a> (<tt>convert</tt>) or</li> 
       <li><a rel="nofollow" class="external text" href="http://www.imagemagick.org/">ImageMagick</a> (<tt>convert</tt>) or</li> 
       <li><a rel="nofollow" class="external text" href="http://www.graphicsmagick.org/">GraphicsMagick</a> (<tt>gm convert</tt>).</li> 
      </ul> 
      <p>These three programs operate much the same way, and can convert between most graphics formats. Sam2p however is the most recent of the three and seems to offer both the best quality and to result in the smallest files.</p> 
      <h3><span class="mw-headline" id="PNG_alpha_channel">PNG alpha channel</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Importing_Graphics&amp;action=edit&amp;section=3" title="Edit section: PNG alpha channel">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Acrobat Reader sometimes has problems with displaying colors correctly if you include graphics in PNG format with alpha channel. You can solve this problem by dropping the alpha channel. On Linux it can be achieved with <tt>convert</tt> from the <a href="https://en.wikipedia.org/wiki/ImageMagick" class="extiw" title="w:ImageMagick">ImageMagick</a> program:</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
convert -alpha off input.png output.png
</pre>
      </div> 
      <h3><span class="mw-headline" id="Converting_a_color_EPS_to_grayscale">Converting a color EPS to grayscale</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Importing_Graphics&amp;action=edit&amp;section=4" title="Edit section: Converting a color EPS to grayscale">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>Sometimes color EPS figures need to be converted to black-and-white or grayscale to meet publication requirements. This can be achieved with the <a rel="nofollow" class="external text" href="http://linuxcommand.org/man_pages/eps2eps1.html">eps2eps</a> of the <a rel="nofollow" class="external text" href="http://ghostscript.com/">Ghostscript</a> package and <a rel="nofollow" class="external autonumber" href="http://www.pa.op.dlr.de/~PatrickJoeckel/pscol/index.html">[1]</a> programs:</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
$ eps2eps input.eps input-e2.eps
$ pscol -0gray input-e2.eps input-gray.eps
</pre>
      </div> 
      <h2><span class="mw-headline" id="Third-party_graphics_tools">Third-party graphics tools</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Importing_Graphics&amp;action=edit&amp;section=5" title="Edit section: Third-party graphics tools">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>We will not tackle the topic of procedural graphics created from within LaTeX code here (TikZ, PSTricks, MetaPost and friends). See <a href="/wiki/LaTeX/Introducing_Procedural_Graphics" title="LaTeX/Introducing Procedural Graphics">Introducing Procedural Graphics</a> for that.</p> 
      <p>You should prefer vector graphics over raster graphics for their quality. Raster graphics should only be used in case of photos. Diagrams of any sort should be vectors.</p> 
      <p>As we have seen before, LaTeX handles</p> 
      <ul> 
       <li>EPS and PDF for vector graphics;</li> 
       <li>PNG and JPG for raster graphics.</li> 
      </ul> 
      <p>If some tools cannot save in those formats, you may want to <a href="#Converting_graphics">convert</a> them before importing them.</p> 
      <h3><span class="mw-headline" id="Vector_graphics">Vector graphics</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Importing_Graphics&amp;action=edit&amp;section=6" title="Edit section: Vector graphics">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <dl> 
       <dt>
        Dia
       </dt> 
      </dl> 
      <p><a href="https://en.wikipedia.org/wiki/Dia_(software)" class="extiw" title="w:Dia (software)">Dia</a> is a cross platform diagramming utility which can export eps images, or generate tex drawn using the <span style="font-family: monospace; color: #2020C0; font-weight: normal;">tikz</span> package.</p> 
      <dl> 
       <dt>
        Inkscape
       </dt> 
      </dl> 
      <p>Another program for creating vector graphics is <a href="https://en.wikipedia.org/wiki/Inkscape" class="extiw" title="w:Inkscape">Inkscape</a>. It can run natively under Windows, Linux or Mac OS X (with X11). It works with <a href="https://en.wikipedia.org/wiki/Scalable_Vector_Graphics" class="extiw" title="w:Scalable Vector Graphics">Scalable Vector Graphics (SVG)</a> files, although it can export to many formats that can be included in <a href="/wiki/LaTeX" title="LaTeX">LaTeX</a> files, such as EPS and PDF. From version 0.48, there is a combined PDF/EPS/PS+LaTeX output option, similar to that offered by <a href="https://en.wikipedia.org/wiki/Xfig" class="extiw" title="w:Xfig">Xfig</a>. <a rel="nofollow" class="external text" href="http://mirrors.ctan.org/info/svg-inkscape/InkscapePDFLaTeX.pdf">There are instructions</a> on how to save your vector images in a PDF format understood by LaTeX and have LaTeX manage the text styles and sizes in the image automatically.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup>. Today there is the <span style="font-family: monospace; color: #2020C0; font-weight: normal;">svg package</span><sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> which provides an <code class="mw-highlight" dir="ltr"><span class="k">\includesvg</span></code> command to convert and include svg-graphics directly in your LaTeX document using Inkscape. You may have a look at this <a rel="nofollow" class="external text" href="http://laclaro.wordpress.com/2013/09/09/updated-includesvg-example/">extended example</a> too.</p> 
      <p>An extremely useful plug-in is <a rel="nofollow" class="external text" href="http://pav.iki.fi/software/textext/">textext</a>, which can import LaTeX objects. This can be used for inserting mathematical notation or LaTeX fonts into graphics (which may then be imported into LaTeX documents).</p> 
      <dl> 
       <dt>
        Ipe
       </dt> 
      </dl> 
      <p>The <a href="https://en.wikipedia.org/wiki/Ipe_(program)" class="extiw" title="w:Ipe (program)">Ipe</a> extensible drawing editor is a free vector graphics editor for creating figures in PDF or EPS format. Unlike Xfig, Ipe represents <a href="/wiki/LaTeX" title="LaTeX">LaTeX</a> fonts in their correct size on the screen which makes it easier to place text labels at the right spot. Ipe also has various snapping modes (for example, snapping to points, lines, or intersections) that can be used for geometric constructions.</p> 
      <dl> 
       <dt>
        lpic
       </dt> 
      </dl> 
      <p>Yet another solution is provided by the <span style="font-family: monospace; color: #2020C0; font-weight: normal;">lpic</span> packages <a rel="nofollow" class="external autonumber" href="http://www.math.uni-leipzig.de/~matveyev/lpic/">[2]</a>, which allows TeX annotations to imported graphics. See <a href="/wiki/LaTeX/Floats,_Figures_and_Captions#Labels_in_the_figures" title="LaTeX/Floats, Figures and Captions">Labels in the figures</a>.</p> 
      <dl> 
       <dt>
        LibreOffice
       </dt> 
      </dl> 
      <p>It is also possible to export vector graphics to EPS format using <a href="https://en.wikipedia.org/wiki/LibreOffice" class="extiw" title="w:LibreOffice">LibreOffice</a> Draw, which is an open source office suite available for Windows, Linux and Mac.</p> 
      <dl> 
       <dt>
        TpX
       </dt> 
      </dl> 
      <p>Vector editor <a rel="nofollow" class="external text" href="http://tpx.sourceforge.net/">TpX</a> separates geometric objects from text objects. Geometric objects are saved into .PDF file, the rest is saved in .TpX file to be processed by LaTeX. User just create the graphics in TpX editor and calls the .TpX file from latex file by command \input{...TpX}.</p> 
      <dl> 
       <dt>
        Xfig
       </dt> 
      </dl> 
      <p><a href="https://en.wikipedia.org/wiki/Xfig" class="extiw" title="w:Xfig">Xfig</a> is a basic program that can produce vector graphics, which can be exported to LaTeX. It can be installed on Unix platforms.</p> 
      <p>On Microsoft Windows systems, Xfig can only be installed using <a rel="nofollow" class="external text" href="http://www.cygwin.com/">Cygwin-X</a>; however, this will require a fast internet connection and about 2 gigabytes of space on your computer. With Cygwin, to run Xfig, you need to first start the "Start X - Server", then launch "xterm" to bring up a terminal. In this terminal type "xfig" (without the quotation marks) and press return.</p> 
      <p>Alternatively, <a rel="nofollow" class="external text" href="http://www.schmidt-web-berlin.de/winfig/index.shtml">WinFIG</a> is an attempt to achieve the functionality of xfig on Windows computers.</p> 
      <p>There are many ways to use xfig to create graphics for LaTeX documents. One method is to export the drawing as a LaTeX document. This method, however, suffers from various drawbacks: lines can be drawn only at angles that are multiples of 30 and 45 degrees, lines with arrows can only be drawn at angles that are multiples of 45 degrees, several curves are not supported, etc.</p> 
      <p>Exporting a file as PDF/LaTeX or PS/LaTeX, on the other hand, offers a good deal more flexibility in drawing. Here's how it's done:</p> 
      <ol> 
       <li>Create the drawing in xfig. Wherever you need LaTeX text, such as a mathematical formula, enter a LaTeX string in a textbox.</li> 
       <li>Use the Edit tool to open the properties of each of those textboxes, and change the option on the "Special Flag" field to Special. This tells LaTeX to interpret these textboxes when it opens the figure.</li> 
       <li>Go to File -&gt; Export and export the file as PDF/LaTeX (both parts) or PS/LaTeX (both parts), depending on whether you are using pdflatex or pslatex to compile your file.</li> 
       <li>In your LaTeX document, where the picture should be, use the following, where "test" is replaced by the name of the image: 
        <table style="margin-top: 1.5em; margin-bottom: 1.5em"> 
         <tbody>
          <tr> 
           <td style="padding-left:3em;"> 
            <div class="mw-highlight mw-content-ltr" dir="ltr"> 
             <pre>
<span class="k">\begin</span><span class="nb">{</span>figure<span class="nb">}</span>
 <span class="k">\centering</span>
 <span class="k">\input</span><span class="nb">{</span>test.pdf<span class="nb">_</span>t<span class="nb">}</span>
 <span class="k">\caption</span><span class="nb">{</span>Your figure<span class="nb">}</span>
 <span class="k">\label</span><span class="nb">{</span>figure:example<span class="nb">}</span>
<span class="k">\end</span><span class="nb">{</span>figure<span class="nb">}</span>
</pre>
            </div> </td> 
          </tr> 
         </tbody>
        </table> <p>Observe that this is just like including a picture, except that rather than using <tt>\includegraphics</tt>, we use <tt>\input</tt>. If the export was into PS/LaTeX, the file extension to include would be .pstex_t instead of .pdf_t.</p> </li> 
       <li>Make sure to include packages <span style="font-family: monospace; color: #2020C0; font-weight: normal;">graphicx</span> and <span style="font-family: monospace; color: #2020C0; font-weight: normal;">color</span> in the file, with the <code class="mw-highlight" dir="ltr"><span class="k">\usepackage</span></code> command right below the <code class="mw-highlight" dir="ltr"><span class="k">\documentclass</span></code> command, like this: 
        <table style="margin-top: 1.5em; margin-bottom: 1.5em"> 
         <tbody>
          <tr> 
           <td style="padding-left:3em;"> 
            <div class="mw-highlight mw-content-ltr" dir="ltr"> 
             <pre>
<span class="k">\usepackage</span><span class="nb">{</span>graphicx<span class="nb">}</span>
<span class="k">\usepackage</span><span class="nb">{</span>color<span class="nb">}</span>
</pre>
            </div> </td> 
          </tr> 
         </tbody>
        </table> </li> 
      </ol> 
      <p>And you're done!</p> 
      <p>For more details on using xfig with LaTeX, <a rel="nofollow" class="external text" href="http://www-epb.lbl.gov/xfig/latex_and_xfig.html">this chapter</a> of the <a rel="nofollow" class="external text" href="http://www-epb.lbl.gov/xfig/contents.html">xfig User Manual</a> may prove helpful.</p> 
      <dl> 
       <dt>
        Other tools
       </dt> 
      </dl> 
      <p>Commercial vector graphics software, such as Adobe Illustrator, CorelDRAW, and FreeHand are commonly used and can <i>read</i> and <i>write</i> EPS figures. However, these products are limited to Windows and Mac OS X platforms.</p> 
      <h3><span class="mw-headline" id="Raster_graphics">Raster graphics</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Importing_Graphics&amp;action=edit&amp;section=7" title="Edit section: Raster graphics">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <dl> 
       <dt>
        Adobe Photoshop
       </dt> 
      </dl> 
      <p>Photoshop can save to EPS.</p> 
      <dl> 
       <dt>
        GIMP
       </dt> 
      </dl> 
      <p><a rel="nofollow" class="external text" href="http://www.gimp.org">GIMP</a>, has a graphical user interface, and it is multi-platform. It can save to EPS and PDF.</p> 
      <h3><span class="mw-headline" id="Plots_and_Charts">Plots and Charts</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Importing_Graphics&amp;action=edit&amp;section=8" title="Edit section: Plots and Charts">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <dl> 
       <dt>
        Generic Mapping Tools (GMT)
       </dt> 
      </dl> 
      <p><a rel="nofollow" class="external text" href="http://gmt.soest.hawaii.edu/">Generic Mapping Tools (GMT)</a>, maps and a wide range of highly customisable plots.</p> 
      <dl> 
       <dt>
        Gnumeric
       </dt> 
      </dl> 
      <p><a rel="nofollow" class="external text" href="http://projects.gnome.org/gnumeric/">Gnumeric</a>, spreadsheets has SVG, EPS, PDF export</p> 
      <dl> 
       <dt>
        Gnuplot
       </dt> 
      </dl> 
      <p><a rel="nofollow" class="external text" href="http://www.gnuplot.info">Gnuplot</a>, producing scientific graphics since 1986. If you want to make mathematical plots, then <a rel="nofollow" class="external text" href="http://www.gnuplot.info">Gnuplot</a> can save in any format. You can get best results when used along <a href="/wiki/LaTeX/PGF/TikZ" title="LaTeX/PGF/TikZ">PGF/TikZ</a>.</p> 
      <dl> 
       <dt>
        matplotlib
       </dt> 
      </dl> 
      <p><a rel="nofollow" class="external text" href="http://matplotlib.sourceforge.net/">matplotlib</a>, plotting library written in python, with PDF and EPS export. On the other hand there is a PGF export also. There are some tricks to be able to import formats other than EPS into your DVI document, but they're very complicated. On the other hand, converting any image to EPS is very simple, so it's not worth considering them.</p> 
      <dl> 
       <dt>
        R
       </dt> 
      </dl> 
      <p><a rel="nofollow" class="external text" href="http://www.r-project.org/">R</a>, statistical and scientific figures.</p> 
      <h3><span class="mw-headline" id="Editing_EPS_graphics">Editing EPS graphics</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=LaTeX/Importing_Graphics&amp;action=edit&amp;section=9" title="Edit section: Editing EPS graphics">edit</a><span class="mw-editsection-bracket">]</span></span></h3> 
      <p>As described above, graphics content can be imported into <a href="/wiki/LaTeX" title="LaTeX">LaTeX</a> from outside programs as EPS files. But sometimes you want to edit or retouch these graphics files. An EPS file can be edited with any text editor since it is formatted as ASCII. In a text editor, you can achieve simple operations like replacing strings, changing the bounding box, or moving items slightly, but anything further becomes cumbersome. Vector graphics editors, like Inkscape, may also be able to import EPS files for subsequent editing. This approach also for easier editing. However, the importing process may occasionally modify the original EPS image.</p>
