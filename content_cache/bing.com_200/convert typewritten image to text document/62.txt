     <h1>What is jOpenDocument?</h1> 
     <p> jOpenDocument is a <strong>free</strong> library for developers looking to use Open Document files without OpenOffice.org.</p> 
     <p>jOpenDocument is Open Source (under GPL or commercial license).</p> 
     <p>You can automate the document creation and manipulation process. You can use jOpenDocument to:</p> 
     <ul> 
      <li>- <strong>Generate</strong> dynamic documents from Java, XML or databases</li> 
      <li>- Display and print files <strong>with built-in viewers</strong></li> 
      <li>- Split, concatenate, and manipulate pages</li> 
      <li>- <strong>Automate</strong> filling out of template (created by OpenOffice <strong>or not</strong>)</li> 
      <li>- Use your <strong>preferred langage</strong> via standard ScriptEngine interface</li> 
      <li>- And much more...</li> 
     </ul>

     <h1>News</h1> 
     <p><strong>September 22, 2014 : jOpenDocument 1.4rc2</strong></p> 
     <p> <strong>Added</strong><br> - Span class<br> - ODNodeDesc class<br> - Style.resolveReference()<br> <br> <strong>Improved :</strong><br> - ODPackage.validateSubDocuments() now handles extended documents (OpenDocument v1.2 §2.2.2)<br> - TextNode no longer restricted to TextDocument<br> - Support for multiple lines in cells<br> <br> <strong>Fixed :</strong><br> - Remove hard limit for ODXMLDocument.findUnusedName(), now print a warning<br> - Handles Vertical Tab, see TextNode.VERTICAL_TAB_CHAR</p> 
     <p><strong>April 11, 2014 : jOpenDocument 1.4rc1</strong></p> 
     <p> <strong>Added:</strong><br> - Cell.getFormulaAndNamespace() and MutableCell.setFormula().<br> - Class Length to efficiently and accurately handle computations and conversion between different units.<br> <br> <strong>Improved :</strong><br> - Google Drive compatibility : sheets now always have display=true ; use centimetres by default for lengths.<br> - LibreOffice compatibility : by default no longer include years, months or days for durations. See MutableCell.getTimeValueMode().<br> - Framework base classes (XMLVersion, ODValueType, Style, etc) are now thread safe. This allows 2 threads to use 2 documents concurrently, but one document still cannot be used by 2 threads. <br> - ODPackage.validateSubDocuments() now also validates the manifest.<br> <br> <strong>Fixed :</strong><br> - Handle missing version in documents as it's optional in OpenDocument 1.0 and 1.1 (e.g. Microsoft Office 2007 and 2010 support v1.1 and omit it)<br> - No longer write time zone part, allow to configure the framework to behave like LO&lt;4.1 or LO&gt;=4.1 with regards to parsing dates with time zone (see ODValueType.isTimeZoneIgnored()).<br> See also FixTimeZone for existing documents.<br> - Column.setWidth() now updates the width of its table.<br> - Handle Range with only one cell.<br> <br> API changed (incompatible with previous revision):<br> - Cell.getFormula() now throws an exception if the name space isn't OpenFormula. The old behaviour is available as getRawFormula()<br> - A lot of style classes now return Length instead of Float or BigDecimal. </p> 
     <p><strong>May 31, 2013 : jOpenDocument 1.3</strong></p> 
     <p><a href="http://jopendocument.org/downloads.html">Here</a> is the stable version of jOpenDocument 1.3.<br> The next step is the creation of a 1.4 branch.</p> 
     <p>jOpenDocument 1.3 is a major version of jOpenDocument. If you are using a previous version, we strongly encourage you to update.</p> 
     <p><strong>Changes since 1.3rc2:</strong><br> Added:<br> - Table.removeRows()<br> - getRangesNames() and getRange() in SpreadSheet and Table<br> Fixed:<br> - Handle times with no seconds in cells with no format<br> - Remove stale document statistics in meta.xml<br> </p> 
     <p><strong>March 11, 2013</strong> : jOpenDocument 1.3 rc2</p> 
     <p>jOpenDocument 1.3 rc2 is now available for <a href="downloads.html">download</a>. </p> 
     <p><strong>Added:</strong><br> - Table.createColumnStyle(width)<br> - TextDocument.getCharacterContent()</p> 
     <p><strong>Improved:</strong><br> - LengthUnit.format() now takes an arbitrary Number instead of BigDecimal (allow to avoid turning 2.357f into 2.3570001125335693)<br> - Table.setColumnCount() can now take a ColumnStyle (allow to create a column of arbitrary width)<br> - SpreadSheet.create() adds an automatic table style with display=true (LibreOffice always generates one, and Google Docs expects it)</p> 
     <p><strong>Fixed:</strong><br> - ODSingleXMLDocument.add() correctly orders body children </p> 
     <p><br> </p> 
     <p><strong>February 15, 2013</strong> : jOpenDocument 1.3 rc1</p> 
     <p>jOpenDocument 1.3 rc1 is now available for <a href="downloads.html">download</a>. </p> 
     <p><strong>Added:</strong><br> - Support for opening and saving flat XML with binary data (e.g. pictures)<br> - SpreadSheet.create(int sheetCount, int colCount, int rowCount)<br> - StyleStyle.getParentStyle()<br> - StyleDesc.createCommonStyle() to add styles<br> - TextDocument.getParagraphCount()<br> - Scripts (macros) and event listeners are now supported : added ODPackage.readBasicLibraries(), addBasicLibraries(), removeBasicLibraries() and readEventListeners()</p> 
     <p><strong>Improved:</strong><br> - OOConnexion handles UNC paths.<br> - OOInstallation handles LibreOffice 3.5 and 3.6.<br> - Optimized MutableCell.setValue().<br> - Various getTextValue() methods (using TextNode.getCharacterContent()) now correctly ignore graphical elements anchored to the paragraph.</p> 
     <p><strong>Fixed:</strong><br> - Cells with conditional styles having values that cannot be evaluated by conditions (e.g. a cell with &gt;=5 condition and 'foo' value).<br> - Searching for the value of a formatting property (see §16.2 of OpenDocument v1.2, we now look in ancestor styles and in enclosing elements).</p> 
     <p><strong>API changed</strong> (incompatible with previous revision):<br> - ODSingleXMLDocument.saveAs() now saves to a flat XML, use saveToPackageAs() to continue to save to a package.<br> - All XML entries in ODPackage are now parsed to Document (e.g. Basic/script-lc.xml).<br> Previously only sub-documents (content.xml, styles.xml...) were parsed, all other entries were byte[].</p> 
     <p><br> </p> 
     <p><strong>March 22, 2012</strong> : jOpenDocument 1.3 beta1</p> 
     <p>jOpenDocument 1.3 beta 1 is now available for <a href="downloads.html">download</a>. <br> It contains a lot of improvements and requested features.<br><br> Because we are focusing on our open source ERP named <a href="http://www.openconcerto.org">OpenConcerto</a>, we had to postpone the release of the 1.3 branch.<br> Our ERP is powered by the jOpenDocument 1.3 branch for document generation, PDF conversion and display.<br> OpenConcerto being downloaded 300 times per day, we are pleased to announce that our library is now the most used one!<br> <br> New and Noteworthy:<br> <strong>Added</strong>: <br> - ODPackage.createFromStream()/createFromFile() handling both package and flat XML files<br> - Validator to unify JAXP and DTD validation<br> - XMLFormatVersion to handle differences in office versions<br> - Instructions in the README on how to validate XML<br> - Support for row and column groups<br> - Table.getUsedRange() to find out the range that covers all used cells<br> - Table.getCurrentRegion() to find out the range containing the passed cell and completely surrounded by empty rows and columns<br> - Cell.getError() to find out if a computation resulted in an error<br> - Support for default-style<br> - Support for data styles, MutableCell now correctly formats its value (including parsing of conditions)<br> <strong>Improved</strong>: <br> - added POINT and PICA in LengthUnit<br> - handle arbitrary table name (i.e. quotes)<br> - Table.duplicateRows() can now update shapes coordinates and handle merged cells<br> - performance: no longer expand repeated rows and lots of other small improvements<br> - ODDocument is now a superclass of SpreadSheet &amp; TextDocument<br> <strong>Fixed</strong>: <br> - ODPackage now saves valid OpenDocument package files (i.e. split flat XML before zipping)<br> - index bug in Sheet.move()<br> - handle white space encoding and decoding in cells according to office version<br> </p>
