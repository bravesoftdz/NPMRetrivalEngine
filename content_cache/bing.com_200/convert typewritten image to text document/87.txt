                       One of the most overwhelming topics for any Java newbie is the File I/O and a 
                      <b>simple reading of a file</b> which seems so easy in C becomes cryptic with so many classes involved. I plan to document all those different ways to Read A Complete File into a String.
                      <br> 
                      <br> 
                      <h2> Java Code(s) for Reading An Entire Text File into a String</h2> 
                      <br> 
                      <ol style="text-align: left;"> 
                       <li><h3> Using the <span style="font-family: 'Courier New', Courier, monospace;"><b>Scanner</b></span>&nbsp;class.</h3> <br> <pre class="brush:java">public class ReadEntireFileIntoAString {
 public static void main(String[] args) throws FileNotFoundException {

  String entireFileText = new Scanner(new File("readme.txt"))
    .useDelimiter("\\A").next();

  System.out.println(entireFileText);
 }
}
</pre> </li> 
                       <li><h3> Using the <b><span style="font-family: 'Courier New', Courier, monospace;">BufferedReader</span></b> to <b>read line by line</b>.</h3> But remember, we need to add a "\n" at the end, since readline strips the new line at the end<br> <pre class="brush:java">public class ReadFileIntoAStringLineByLine {

 public static void main(String[] args) throws IOException {

  BufferedReader bufferedReader = new BufferedReader(new FileReader("readme.txt"));

  StringBuffer stringBuffer = new StringBuffer();
  String line = null;

  while((line =bufferedReader.readLine())!=null){

   stringBuffer.append(line).append("\n");
  }
  
  System.out.println(stringBuffer);
 }
}
</pre> </li> 
                       <li><h3> Using the barebones&nbsp; <b style="background-color: white; text-align: -webkit-auto;"><span style="font-family: 'Courier New', Courier, monospace;">FileReader&nbsp;</span></b>to<b> read character by character</b> using the <a href="http://docs.oracle.com/javase/1.4.2/docs/api/java/io/InputStreamReader.html#read()" rel="nofollow" target="_blank">read()</a> method.</h3> We are avoiding an extra wrapper, the<span style="font-family: 'Courier New', Courier, monospace;"><b> BufferedReader</b></span> that offers us more tools, one of which we saw above. If you are using any other character set other than the default, use an&nbsp;<b><span style="font-family: 'Courier New', Courier, monospace;">InputSreamReader</span></b> or a <span style="font-family: 'Courier New', Courier, monospace;"><b>FileInputStream&nbsp;</b></span> to set the encoding.<br> <pre class="brush:java">public class ReadEntireFileIntoAStringCharByChar {

 public static void main(String[] args) throws IOException {
  FileReader fileReader = new FileReader("readme.txt");
   
  String fileContents = "";

  int i ;

  while((i =  fileReader.read())!=-1){
   char ch = (char)i;

   fileContents = fileContents + ch; 
  }

  System.out.println(fileContents);
 }
}</pre> </li> 
                       <li><h3> The most <i>Beautiful</i> of them all! The <a href="http://commons.apache.org/proper/commons-io/" rel="nofollow" target="_blank">Apache CommonsIO</a> library's <a href="http://commons.apache.org/proper/commons-io/javadocs/api-2.4/org/apache/commons/io/FileUtils.html" rel="nofollow" target="_blank">FileUtils </a>class.</h3> However, the tradeoff is that you need to <a href="http://commons.apache.org/proper/commons-io/download_io.cgi" rel="nofollow" target="_blank">download</a> the library and add it to your classpath. It also has overloaded versions of the <a href="http://commons.apache.org/proper/commons-io/javadocs/api-2.4/org/apache/commons/io/FileUtils.html#readFileToString%28java.io.File,%20java.nio.charset.Charset%29" rel="nofollow" target="_blank">readFileToString</a> method with the <a href="http://docs.oracle.com/javase/6/docs/api/java/nio/charset/Charset.html" rel="nofollow" target="_blank">charset</a> encoding. Do check it out! <pre class="brush:java"> import org.apache.commons.io.FileUtils;
 try {
    String str = FileUtils.readFileToString(new File("test.txt"));
    System.out.println(str);
  } catch (IOException e) {
     e.printStackTrace();
 }
</pre> </li> 
                      </ol>
