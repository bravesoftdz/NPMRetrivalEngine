    unoconv: Convert between any document format supported by OpenOffice
   </div> 
   <div class="text"> 
    <!-- body start --> 
    <div class="intro">
      unoconv converts between any document format that OpenOffice understands. It uses OpenOffice's UNO bindings for non-interactive conversion of documents. 
     <br>
     <br> Supported document formats include Open Document Format (.odt), MS Word (.doc), MS Office Open/MS OOXML (.xml), Portable Document Format (.pdf), HTML, XHTML, RTF, Docbook (.xml), and more. 
    </div> 
    <div class="title">
     <a name="features">Features</a>
    </div> 
    <ul> 
     <li> Converts between different document formats that OpenOffice understands </li>
     <li> OpenOffice can export to about 100 different document formats </li>
     <li> Can be used for batch processing </li>
     <li> Combines with asciidoc and docbook2odf/xhtml2odt to create PDF or Word (.doc) files </li>
     <li> Can apply custom style templates during conversion (to enforce corporate identity) </li>
     <li> Autostarts OpenOffice for processing if necessary </li>
     <li> Can be used in a client/server environment to process documents centrally </li>
     <li> Can influence OpenOffice filters during import and export </li>
     <li> Supports OpenOffice on Linux, Windows and MacOSX </li>
    </ul> 
    <div class="title">
     <a name="output-formats">Output formats</a>
    </div> OpenOffice (and thus unoconv) support various import and export formats. Not all formats that can be imported can be exported and vice versa. The below list of formats only lists the export formats.
    <br>
    <br> If you need a list of formats that unoconv can import, please look at the 
    <a href="http://wiki.services.openoffice.org/wiki/Framework/Article/Filter/FilterList_OOo_3_0">filter list</a> from OpenOffice.

    <div class="title">
     <a name="screenshot">Screenshot</a>
    </div> This may not be your typical set of screenshots, but here is a workflow I use for converting an AsciiDoc (text) formatted résumé into professional ODT, PDF, HTML and DOC versions: 
    <pre>
  [dag@moria cv]$ make odt pdf html doc
  rm -f *.{odt,pdf,html,doc}
  asciidoc -b docbook -d article -o resume.xml resume.txt
  docbook2odf -f --params generate.meta=0 -o resume.tmp.odt resume.xml
  Saved resume.tmp.odt
  unoconv -f odt -t template.ott -o resume.odt resume.tmp.odt
  unoconv -f pdf -t template.ott -o resume.pdf resume.odt
  unoconv -f html -t template.ott -o resume.html resume.odt
  unoconv -f doc -t template.ott -o resume.doc resume.odt
</pre> The original files are: 
    <ul> 
     <li> A recipe: <a href="/cv/Makefile">Makefile</a> </li>
     <li> An AsciiDoc [.txt] file: <a href="/cv/curriculum-vitae-dag-wieers.asciidoc">resume.txt</a> </li>
     <li> An OpenOffice template [.ott] file: <a href="/cv/curriculum-vitae-docbook.ott">template.ott</a></li>
    </ul> Converted into the following files: 
    <ul> 
     <li> Open Document [.odt]: <a href="/cv/curriculum-vitae-dag-wieers.odt">resume.odt</a> <small><i>- converted from TXT using asciidoc and stylesheet</i></small> </li>
     <li> PDF [.pdf]: <a href="/cv/curriculum-vitae-dag-wieers.pdf">resume.pdf</a> <small><i>- converted from ODF using unoconv</i></small> </li>
     <li> HTML [.html]: <a href="/cv/curriculum-vitae-dag-wieers.html">resume.html</a> <small><i>- converted from ODF using unoconv</i></small> </li>
     <li> Word [.doc]: <a href="/cv/curriculum-vitae-dag-wieers.doc">resume.doc</a> <small><i>- converted from ODF using unoconv</i></small> </li>
    </ul>
