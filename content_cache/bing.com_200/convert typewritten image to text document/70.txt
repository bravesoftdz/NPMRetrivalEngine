      <h1>OpenDocument Format for developers</h1> 
      <p>Open Document Format (ODF) is an international family of standards that is standardised at <a href="http://www.oasis-open.org/committees/tc_home.php?wg_abbrev=office">OASIS</a> and was first adopted by ISO/IEC JTC1 SC34 in 2005. </p> 
      <p>Writing and reading ODF documents from applications is easy, whether you are talking about text documents, spreadsheet or presentatations. Check for instance these simple freely reusable code examples where you <a href="http://recipes.opendocsociety.org/recipes/get_text_from_odt">extract all text</a> from a document or <a href="http://recipes.opendocsociety.org/recipes/copy-style-from-another-document">appy the look from another document</a>, generate a <a href="http://recipes.opendocsociety.org/recipes/make-a-basic-spreadsheet">basic spreadsheet</a>, create a presentation from <a href="http://recipes.opendocsociety.org/recipes/make-a-presentation-from-images-with-parameters-cli-options">a folder containing images</a> or <a href="http://recipes.opendocsociety.org/recipes/make-a-presentation-from-pictures-included-in-a-text-document">pictures in another document</a>. </p> 
      <p>Or do more advanced stuff like <a href="http://recipes.opendocsociety.org/recipes/swap-rows-columns-of-a-spreadsheet">swap and row/columns of a table</a>, <a href="http://recipes.opendocsociety.org/recipes/extract-sub-table-from-a-big-table">extract a subtable from a bigger table</a>, automate things you used to have to do by hand (like <a href="http://recipes.opendocsociety.org/recipes/transform-all-links-into-footnotes">transforming all links into footnotes</a>, <a href="http://recipes.opendocsociety.org/recipes/remove-all-hypertext-links-from-a-document">remove all hypertext links from a document</a>, <a href="http://recipes.opendocsociety.org/recipes/swapping-old-new-image-from-documents-cli">replace images </a> (like old logos) or improve <a href="http://recipes.opendocsociety.org/recipes/recipe-for-simple-accessibility-tag-insertion-cli">accessibility</a> for people with visual impairments. </p> 
      <p>There really are a myriad of new possibilities you have for working with office documents and optimising your workflow. On this page you will find an overview of dozens of software libraries that make it easy to access content stored in OpenDocument Format. If your product or project is not listed, <a href="/contact">contact us</a>.</p>

       <h2 class="yellow">The technology behind ODF</h2> 
       <p>ODF is based on XML, which makes it easy to develop your own software if none of these existing solutions fit your need. There is an extensive repository of <a href="http://recipes.opendocsociety.org">available example code ready for use</a>. </p> 
       <p>The technical specifications of ODF are available online for free.</p> 
       <p style="text-align: right;"><a href="http://docs.oasis-open.org/office/v1.2/cs01/OpenDocument-v1.2-cs01.html">Read the latest version of the specification</a></p>

      <q class="foxscripty">Real world interoperability</q> 
      <h2>ODF plugfests</h2> 
      <p>The ODF plugfests are an ongoing series of vendor-neutral events, bringing together implementers and stakeholders of the standard. The goal is to achieve maximum interoperability by running scenario-based tests in a hands-on manner and discuss new and proposed features of the ODF specification. And of course they are a great place to meet other developers working with OpenDocument Format.</p> 
      <img src="/img/10thplugfestlogo.png" alt="ODF Plugfest"> 
      <p>So far nine plugfests have taken place, hosted by governments, academic institutions and companies in a number of countries. The next plugfest will be held in <a href="http://plugfest.opendocumentformat.org/2014-london">London</a>, hosted by the <em>UK Cabinet Office</em>.</p> 
      <p>Visit: <a href="http://plugfest.opendocumentformat.org">http://plugfest.opendocumentformat.org</a></p>
