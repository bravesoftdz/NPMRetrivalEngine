      <div class="topic" xmlns="http://www.w3.org/1999/xhtml" xmlns:mtps="http://msdn2.microsoft.com/mtps" xmlns:msxsl="urn:schemas-microsoft-com:xslt" xmlns:cs="http://msdn.microsoft.com/en-us/"> 
       <h1 class="title">Overview of the XML file formats in Office 2010</h1>
       <div class="lw_vs">
        <div id="curversion">
         <strong> Office 2010 </strong>
        </div>
       </div>
       <div style="clear:both;"></div> 
       <div id="mainSection"> 
        <div id="mainBody"> 
         <span id="changeHistory">&nbsp;</span> 
         <p><em><strong>Applies to: </strong>Office 2010</em></p> 
         <p><em><strong>Topic Last Modified: </strong>2011-08-05</em></p> 
         <p>The Open XML file formats simplify the exchange of data between Office applications and enterprise business systems. Based on open standards, these XML file formats enable the rapid creation of documents from different data sources and speed up document assembly, data mining, and content reuse.</p> 
         <p>The 2007 Office system supports the ECMA-376 Office Open XML Formats standard, which was later submitted to ISO/IEC and was published in late 2008 as the ISO/IEC 29500 Office Open XML Formats standard. Office 2010 provides read support for ECMA-376, read/write support for ISO/IEC 29500 Transitional, and read support for ISO/IEC 29500 Strict.</p> 
         <p>Documentation for the ISO/IEC 29500 Office Open XML Formats is available from ISO/IEC, and documentation for ECMA-376 is available from Ecma International. For detailed information about how these formats are supported in Office 2010 and the 2007 Office system, see <a href="http://go.microsoft.com/fwlink/p/?LinkId=191143">Microsoft Office File Format Documents</a> (http://go.microsoft.com/fwlink/p/?LinkId=191143) on MSDN.</p> 
         <p>In this article:</p> 
         <ul>
          <li> <p> <a href="#section1">Benefits of the Open XML Formats</a> </p> </li>
          <li> <p> <a href="#section2">Structure of the Open XML Formats</a> </p> </li>
         </ul> 
         <a id="section1"></a>
         <div>
          <h2 class="LW_CollapsibleArea_TitleDiv">
           <div>
            <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Benefits of the Open XML Formats</span></a>
            <div id="Anchor_0" class="LW_CollapsibleArea_Anchor_Div">
             <a href="/en-us/library/cc179190(v=office.14).aspx#Anchor_0" class="LW_CollapsibleArea_Anchor_Img" title="Right-click to copy and share the link for this section"></a>
            </div>
            <div class="LW_CollapsibleArea_HrDiv">
             <hr class="LW_CollapsibleArea_Hr">
            </div>
           </div></h2>
          <div class="sectionblock"> 
           <p>The Open XML Formats provide several benefits for developers, IT professionals, and users. These benefits include the following:</p> 
           <ul>
            <li> <p> <strong>Compact file format</strong>&nbsp;&nbsp;&nbsp;Documents are automatically compressed, up to 75 percent smaller.</p> </li>
            <li> <p> <strong>Improved damaged file recovery</strong>&nbsp;&nbsp;&nbsp;Modular data storage enables files to open even if a component within the file, such as a chart or table, is damaged.</p> </li>
            <li> <p> <strong>Safer documents</strong>&nbsp;&nbsp;&nbsp;Embedded code, such as OLE objects or Microsoft Visual Basic for Applications (VBA) code, is stored in a separate section within the file so that it can easily be identified for special processing. IT administrators can block the documents that contain unwanted macros or controls. This helps make documents safer for users when they are opened.</p> </li>
            <li> <p> <strong>Easier integration</strong>&nbsp;&nbsp;&nbsp;Developers have direct access to specific contents within the file, such as charts, comments, and document metadata.</p> </li>
            <li> <p> <strong>Transparency and improved information security</strong>&nbsp;&nbsp;&nbsp;Documents can be shared confidentially because personally identifiable information and business-sensitive information, such as user names, comments, tracked changes, and file paths, can easily be identified and removed.</p> </li>
            <li> <p> <strong>Compatibility</strong>&nbsp;&nbsp;&nbsp;By installing the Microsoft Office Compatibility Pack, users of Microsoft Office 2000, Microsoft Office XP, and Microsoft Office 2003 editions can open, edit, and save documents in one of the new XML formats.</p> </li>
           </ul> 
          </div>
         </div>
         <a id="section2"></a>
         <div>
          <h2 class="LW_CollapsibleArea_TitleDiv">
           <div>
            <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Structure of the Open XML Formats</span></a>
            <div id="Anchor_1" class="LW_CollapsibleArea_Anchor_Div">
             <a href="/en-us/library/cc179190(v=office.14).aspx#Anchor_1" class="LW_CollapsibleArea_Anchor_Img" title="Right-click to copy and share the link for this section"></a>
            </div>
            <div class="LW_CollapsibleArea_HrDiv">
             <hr class="LW_CollapsibleArea_Hr">
            </div>
           </div></h2>
          <div class="sectionblock"> 
           <p>The basic structure of the Open XML Formats consists of the following five elements, which are described in more detail in the sections that follow.</p> 
           <ul>
            <li> <p> <strong>Start part</strong>&nbsp;&nbsp;&nbsp;The highest order part in the hierarchy.</p> </li>
            <li> <p> <strong>XML parts</strong>&nbsp;&nbsp;&nbsp;Files or folders consisting of XML that comprise the content of the file.</p> </li>
            <li> <p> <strong>Non-XML parts</strong>&nbsp;&nbsp;&nbsp;Parts that are not XML and generally are either images or OLE objects.</p> </li>
            <li> <p> <strong>Relationship part</strong>&nbsp;&nbsp;&nbsp;A type of part that generally points to other parts to define the relational hierarchy of the part structure.</p> </li>
            <li> <p> <strong>ZIP package</strong>&nbsp;&nbsp;&nbsp;Bundles parts into a single file.</p> </li>
           </ul> 
           <div>
            <h3 class="LW_CollapsibleArea_TitleDiv">
             <div>
              <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Start part</span></a>
              <div class="LW_CollapsibleArea_HrDiv">
               <hr class="LW_CollapsibleArea_Hr">
              </div>
             </div></h3>
            <div class="sectionblock"> 
             <p>The start part, an XML part that is a relationship part and could be considered the highest order part, determines the file type. For example, if the name of the core container is WordDoc, the file name extension is .docx.</p> 
            </div>
           </div> 
           <div>
            <h3 class="LW_CollapsibleArea_TitleDiv">
             <div>
              <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">XML parts</span></a>
              <div class="LW_CollapsibleArea_HrDiv">
               <hr class="LW_CollapsibleArea_Hr">
              </div>
             </div></h3>
            <div class="sectionblock"> 
             <p>When an Office XML formatted file is saved in Office 2010 or the 2007 Office system, the file is divided into a set of logical parts that describes the entire file. For Microsoft Word, dividing the file into these parts enables the file to be easily queried or modified outside of the original Office application.</p> 
             <p>For example, it is easier for a developer to remove document properties from a file because the properties are placed in a single part, and the part can be deleted from within the document container by deleting the part. With WordprocessingML (provided as an optional XML file format in Office 2003), removing comments involved parsing the entire file to find and remove the XML representing the contents of the comment. With the new file format, feature-related data is divided into parts. Comments, links, headers, footers, and other data are in separate parts that can be removed. You do not need to parse the entire Word document.</p> 
            </div>
           </div> 
           <div>
            <h3 class="LW_CollapsibleArea_TitleDiv">
             <div>
              <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Non-XML parts</span></a>
              <div class="LW_CollapsibleArea_HrDiv">
               <hr class="LW_CollapsibleArea_Hr">
              </div>
             </div></h3>
            <div class="sectionblock"> 
             <p>Non-XML parts are generally images and OLE objects. Any file type that uses binary content or does not use XML is identified as non-XML. A non-XML part is usually a file attached to or embedded within a document. The Word XML format schema documentation explains the literal relationship and schema hierarchy used by Word for files of this type.</p> 
            </div>
           </div> 
           <div>
            <h3 class="LW_CollapsibleArea_TitleDiv">
             <div>
              <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">Relationship part</span></a>
              <div class="LW_CollapsibleArea_HrDiv">
               <hr class="LW_CollapsibleArea_Hr">
              </div>
             </div></h3>
            <div class="sectionblock"> 
             <p>A relationship part is an XML part that points to other parts and defines the relational hierarchy of the parts. Most high-level XML parts are relationship parts. XML parts that contain data and do not point to other parts are also known as primitives, and usually have a content type of application/xml.</p> 
            </div>
           </div> 
           <div>
            <h3 class="LW_CollapsibleArea_TitleDiv">
             <div>
              <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"><span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span><span class="LW_CollapsibleArea_Title">ZIP package</span></a>
              <div class="LW_CollapsibleArea_HrDiv">
               <hr class="LW_CollapsibleArea_Hr">
              </div>
             </div></h3>
            <div class="sectionblock"> 
             <p>Using a ZIP package provides the following benefits in all applications:</p> 
             <ul>
              <li> <p> <strong>Open standard</strong>&nbsp;&nbsp;&nbsp;The ZIP compression algorithm is a well-defined open standard.</p> </li>
              <li> <p> <strong>Reduced file size</strong>&nbsp;&nbsp;&nbsp;Files are generally smaller than an equivalent binary file. On average, Word Open XML files are 75&nbsp;percent smaller than their binary counterparts, depending on the number of images.</p> </li>
              <li> <p> <strong>Increased robustness</strong>&nbsp;&nbsp;&nbsp;Files are more robust and less sensitive to potential errors in the file. Previous files required the file to be completely intact to function correctly.</p> </li>
             </ul> 
             <p>Although use of a ZIP package means the file is binary, the WinFX application programming interface (API) set provides native support for the package format in the System.IO.Packaging name space. This enables developers to create tools that process the format and work directly against the logical model (the parts) without having to consider expansion or compression of the package.</p> 
            </div>
           </div> 
          </div>
         </div>
