             <h2>5. Connecting to Oracle using Java JDBC</h2> 
             <pre><code class="language-java">public class OracleJdbcTest
{
	String driverClass = "oracle.jdbc.driver.OracleDriver";

	Connection con;
	
	public void init(FileInputStream fs) throws ClassNotFoundException, SQLException, FileNotFoundException, IOException
	{
		Properties props = new Properties();
		props.load(fs);
		String url = props.getProperty("db.url");
		String userName = props.getProperty("db.user");
		String password = props.getProperty("db.password");

		con=DriverManager.getConnection(url, userName, password);
	}

              <p>The singleton example is bad!</p> 
              <p>As written, it wont even compile, since singleInstance isnt static.</p> 
              <p>Secondly, unless you absolutely need lazy allocation and you can live with the singleton not being thread safe, you should change it to the much simpler and more robust:</p> 
              <pre><code class="language-java">public class SimpleSingleton {  
    private static SimpleSingleton singleInstance = 
                                    new SimpleSingleton();  
  
    private SimpleSingleton() {  
    }  
  
    public static SimpleSingleton getInstance() {  
        return singleInstance;  
    }  
}
</code></pre>

              <pre><code class="language-java">public enum SimpleSingleton {
  INSTANCE;
  public void doSomething() {  ....  }
}
</code></pre> 
              <p>call the method from the singleton:</p> 
              <pre><code class="language-markup">SimpleSingleton.INSTANCE.doSomething();</code></pre>
