          <h1>Quicksort</h1> 
          <p>Quicksort is a fast sorting algorithm, which is used not only for educational purposes, but widely applied in practice. On the average, it has O(n log n) complexity, making quicksort suitable for sorting big data volumes. The idea of the algorithm is quite simple and once you realize it, you can write quicksort as fast as <a href="Bubble_sort">bubble sort</a>.</p> 
          <h2>Algorithm</h2> The divide-and-conquer strategy is used in quicksort. Below the recursion step is described: 
          <ol> 
           <li><strong>Choose a pivot value. </strong>We take the value of the middle element as pivot value, but it can be any value, which is in range of sorted values, even if it doesn't present in the array.</li> 
           <li><strong>Partition. </strong>Rearrange elements in such a way, that all elements which are lesser than the pivot go to the left part of the array and all elements greater than the pivot, go to the right part of the array. Values equal to the pivot can stay in any part of the array. Notice, that array may be divided in non-equal parts.</li> 
           <li><strong>Sort both parts. </strong>Apply quicksort algorithm recursively to the left and the right parts.</li> 
          </ol> 
          <h3>Partition algorithm in detail</h3> 
          <p>There are two indices <strong>i</strong> and <strong>j</strong> and at the very beginning of the <span style="font-family:'Times New Roman'; font-size:12.0pt; ">partition </span> algorithm <strong>i</strong> points to the first element in the array and <strong>j</strong> points to the last one. Then algorithm moves <strong>i</strong> forward, until an element with value greater or equal to the pivot is found. Index <strong>j</strong> is moved backward, until an element with value lesser or equal to the pivot is found. If <strong>i ? j </strong>then they are swapped and i steps to the next position (<strong>i + 1</strong>), j steps to the previous one <strong>(j - 1)</strong>. Algorithm stops, when<strong> i</strong> becomes greater than <strong>j</strong>.</p> 
          <p>After partition, all values before <strong>i-th</strong> element are less or equal than the pivot and all values after <strong>j-th</strong> element are greater or equal to the pivot.</p> 
          <em>Example. </em>Sort {1, 12, 5, 26, 7, 14, 3, 7, 2} using quicksort. 
          <p><img src="/img/sorts/quick-sort.png" alt="Quicksort example"></p> 
          <p> Notice, that we show here only the first recursion step, in order not to make example too long. But, in fact, {1, 2, 5, 7, 3} and {14, 7, 26, 12} are sorted then recursively. </p> 
          <h2>Why does it work?</h2> On the partition step algorithm divides the array into two parts and every element 
          <strong>a</strong> from the left part is less or equal than every element 
          <strong>b</strong> from the right part. Also 
          <strong>a</strong> and 
          <strong>b</strong> satisfy 
          <strong>a ? pivot ? b</strong> inequality. After completion of the recursion calls both of the parts become sorted and, taking into account arguments stated above,
          <span style="color:rgb(128, 128, 128); "> </span> the whole array is sorted.
          <br> 
          <h2>Complexity analysis</h2> 
          <p>On the average quicksort has O(n log n) complexity, but strong proof of this fact is not trivial and not presented here. Still, you can find the proof in <a href="#cormen_book" style="text-decoration:none">[1]</a>. In worst case, quicksort runs O(n<sup>2</sup>) time, but on the most "practical" data it works just fine and outperforms other O(n log n) sorting algorithms.</p>
