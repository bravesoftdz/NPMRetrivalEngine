         <p><strong>Quick Sort</strong> also uses divide and conquer technique like <strong><a href="/algorithms/merge-sort-algorithm-explanation-implementation-and-complexity/" target="_blank">merge sort</a></strong>, but does not require additional storage space. It is one of the most famous comparison based sorting <strong><a href="/category/algorithms/" target="_blank">algorithm</a> </strong>which is also called as partition exchange sort. Like merge sort, it also uses recursive call for sorting elements.</p>
         <p>In Quick Sort pivot element is chosen and partition the array such that all elements smaller than pivot are arranged to left of pivot and element bigger than pivot are arranged to its right.<br> <strong>There are various ways to choose pivot element:</strong></p>
         <ol>
          <li>Chose pivot as first element.</li>
          <li>Chose pivot as end element.</li>
          <li>Chose pivot as median element.</li>
          <li>Chose pivot as random element.</li>
         </ol>

         <h3><span class="ez-toc-section" id="here_are_some_key_points_of_quick_sort_algorithm"><strong>Here are some key points of quick sort algorithm –</strong></span></h3>
         <ul>
          <li>Quick Sort is also a good example of a recursive algorithm.</li>
          <li>We can express time complexity of quick sort by this recurrence relation:<br> T(n) = T(k) + T(n-k-1)+ ?(n).<br> T(k) -&gt; recursion relation for elements left of pivot. k is a number of element smaller than the&nbsp;pivot.<br> T(k) -&gt; recursion relation for elements right of pivot.<br> ?(n) -&gt; It is for partition process.</li>
          <li>Time complexity of Quick Sort is O(n*logn) in best and average case and O(n*n) in the worst case.<br> Worst case is one when all elements of given array are smaller than pivot or larger than the pivot.</li>
          <li>Worst case can be easily eliminated by choosing random element as a pivot or best way is to choose median element as a pivot.</li>
          <li>It is an in-place sorting algorithm as it requires small additional storage space.</li>
          <li>Quick Sort is not a stable sort, which means the “equal” elements may not appear in the same order in the sorted array as they were in the unsorted array.</li>
         </ul>
         <hr>
         <h3><span class="ez-toc-section" id="lets_understand_it_with_an_example_in_which_our_last_element_is_pivot">Let’s understand it with an example in which our last element is pivot-</span></h3>
         <p>Since a picture speaks a thousand words. Just carefully read the steps in the image below and you will be able to visualize the concept of this algorithm.<br> The following diagram is from hackerrank which shows partitioning of array choosing the last element of array as a pivot.<br> <img class="aligncenter" src="http://www.codingeek.com/wp-content/uploads/2016/06/word-image-2.png" alt="C:\Users\admin\Downloads\partition.png" width="496" height="549"><strong>Working of partition function:</strong><br> Here the last element is chosen as a pivot. We initialize our partition index as the first element of the array. Then we compare each element with pivot from beginning, if the element is less than the pivot, so we will swap the element with the element at the partition index and increment the partition index . At the end, we will get our partitioned array in which all elements smaller than pivot will appear to the left of pivot and elements greater than pivot to its right in the array.</p>

         <h3><span class="ez-toc-section" id="pseudocode_of_quick_sort">Pseudocode of Quick Sort</span></h3>
         <pre class="brush: cpp; title: ; notranslate" title="">
QUICK-SORT(A,start,end)
    if start &lt; end
        pIndex = PARTITION(A,start,end)
        QUICK-SORT(A,start,pIndex-1)
        QUICK-SORT(A,pIndex+1,end)
end func
/*To sort an entire array A,
the initial call is QUICK-SORT(A,0,A.length-1)*/

//partitioning the array with respect to pivot.
PARTITION(A,start,end)
    pivot = A[end]
    pIndex = start
        for i = start to end-1
           if A[i] &lt; pivot
               swap A[i] with A[pIndex]
                pIndex=pIndex+1
    swap A[pIndex] with A[end]
   return pIndex
end func
</pre>

         <h3><span class="ez-toc-section" id="asymptotic_analysis_of_quick_sort">Asymptotic Analysis of Quick Sort</span></h3>
         <ul>
          <li><strong>General relation for quick sort: T(n) = T(k) + T(n-k-1)+</strong> ?(n).</li>
          <li><strong>Best Case:</strong> T(n)= 2T(n/2)+ ?(n). <strong>Time complexity</strong>: O(n*logn)<br> Best case occurs when a middle element is chosen as a pivot. It means a number of elements smaller than pivot are equal to the number of elements greater than the pivot.</li>
          <li><strong>Average Case</strong>: T(n)= T(n/3)+T(2n/3)+ ?(n). <strong>Time complexity</strong>: O(n*logn)<br> Average case can be considered when partition puts O(n/3) elements in one set and O(2n/13) elements in other set.<br> <strong>Worst Case</strong>: T(n)= T(n-1)+ ?(n). <strong>Time complexity</strong>: O(n*n)</li>
          <li>Worst case occurs when all elements of given array are smaller than pivot or larger than pivot. Ex- Suppose array is already sorted and we choose the last element as a pivot.</li>
          <li><strong>Algorithmic Paradigm</strong>: Divide and Conquer</li>
          <li><strong>Space Complexity</strong>: O(n*logn ) in an&nbsp;average case.</li>
          <li><strong>Sorting In Place</strong>: yes</li>
          <li><strong>Stable</strong>: no</li>
         </ul>

         <h3><span class="ez-toc-section" id="implementation_of_quick_sort_in_c_programming_language">Implementation of Quick Sort in C programming language</span></h3>
         <pre class="brush: cpp; title: ; notranslate" title="">
/* C implementation QuickSort */
#include&lt;stdio.h&gt;
#include&lt;stdlib.h&gt;
#include&lt;math.h&gt;
// Swapping numbers using call by reference
void swap(int* x, int* y)
{
    int temp = *x;
    *x = *y;
    *y = temp;
}

/* In this function last element is chosen as pivot,
   then elements are arranged such that,all elements
   smaller than pivot are arranged to left of pivot
   and all greater elements to right of pivot */
int partition(int A[], int start, int end)
{
    int pivot = A[end];    // choosing pivot element
    int pIndex = start;  // Index of first element
    int i;
    for (i=start; i&lt;=end-1; i++)
    {
        /* If current element is smaller than or
         equal to pivot then exchange it with element
         at pIndex and increment the pIndex*/
        if (A[i]&lt;= pivot)
        {
            swap(&amp;A[pIndex], &amp;A[i]);
            pIndex=pIndex+1;
        }
    }
    /*exchange pivot with pIndex at the completion
    of loop*/
    swap(&amp;A[pIndex], &amp;A[end]);
    return pIndex;
}

 /* The main function that implements QuickSort
    A[] --&gt; array to be sorted,
    start  --&gt; Starting index,
    end  --&gt; Ending index */
void quick_sort(int A[], int start, int end)
{   int diff=(end-start);
    if (start &lt; end)
    {
        /* p is pivot index after partitioning*/
        int p = partition(A, start, end);
        // Recursively sort elements left of pivot
        // and elements right of pivot
        quick_sort(A, start, p-1);
        quick_sort(A, p+1, end);
    }
}

/*  function to print an array */
void print_array(int A[], int size)
{
    int i;
    for (i=0; i &lt; size; i++)
        printf("%d ", A[i]);
    printf("\n");
}

// Driver program to test above functions
int main()
{
    int A[] = {1, 3, 9, 8, 2, 7, 5};
    int n = sizeof(A)/sizeof(A[0]);

    printf("Unsorted array: ");
    print_array(A, n);
    printf("\n");

    quick_sort(A, 0, n-1);

    printf("Sorted array: ");
    print_array(A, n);
    return 0;
}

         <h4><span class="ez-toc-section" id="partition_function_for_random_element_as_pivot">Partition function for random element as pivot</span></h4>
         <p>In the above code where we choose the&nbsp;last element as a pivot, it may lead to the worst case of quick sort. So, we can eliminate this case by choosing random element as a pivot. Here is a partition function for a&nbsp;random element as a pivot.</p>
         <pre class="brush: cpp; title: ; notranslate" title="">
//C code for random pivot partition function.
/*Here we just found random index and swap that number with last element
  and then just done partitioning in the same way as we have done for 
  last element as pivot.*/

int random_partition(int A[], int start, int end)
{
    int diff=(end-start);
    //rand() function to get random index between our start and end index
    int randNumb=(int)(((double)(diff+1)/RAND_MAX) * rand() + start);

   //replace random index number with last element.
    swap(&amp;A[end], &amp;A[randNumb]);

    // Then following same logic for partition
    int pivot = A[end];    // choosing pivot element
    int pIndex = start;  // Index of first element
    int i;
    for (i=start; i&lt;=end-1; i++)
    {
        /* If current element is smaller than or
         equal to pivot then exchange it with element
         at pIndex and increment the pIndex*/
        if (A[i]&lt;= pivot)
        {
            swap(&amp;A[pIndex], &amp;A[i]);
            pIndex=pIndex+1;
        }
    }
    /*exchange pivot with pIndex at the completion
    of loop*/
    swap(&amp;A[pIndex], &amp;A[end]);
    return pIndex;
}

         <h3><span class="ez-toc-section" id="implementation_of_quick_sort_in_java_programming_language">Implementation of Quick Sort in Java programming language</span></h3>
         <pre class="brush: java; title: ; notranslate" title="">
package com.codingeek;

import java.util.Arrays;

/* Java implementation QuickSort */
public class QuickSort {
	int A[];
	
	/* In this function last element is chosen as pivot,
	   then elements are arranged such that,all elements
	   smaller than pivot are arranged to left of pivot
	   and all greater elements to right of pivot */
	
 int partition(int start, int end)
 {
     int pivot = A[end]; // choosing pivot element
     int pIndex = start; // Index of first element
    
     for (int i=start; i&lt;=end-1; i++)
     {
    	 /* If current element is smaller than or
         equal to pivot then exchange it with element
         at pIndex and increment the pIndex*/
         if (A[i] &lt;= pivot)
         {
        	 // swap A[pIndex] and A[i]
        	 swap(pIndex,i); 
             pIndex=pIndex+1;
         }
     }
     // swap A[pIndex] and A[end] (or pivot)
     swap(pIndex,end);
     return pIndex;
 }

 /* The main function that implements QuickSort
 arr[] --&gt; array to be sorted,
 start  --&gt; Starting index,
 end  --&gt; Ending index */
 void sort(int arr[], int start, int end)
 {
    this.A=arr;
	 if (start &lt; end)
     {
         /* pi is partitioning index, A[pi] is 
           now at right place */
         int p = partition(start, end);

         // Recursively sort elements left of pivot
         // and elements right of pivot
         sort(A, start, p-1);
         sort(A, p+1, end);
     }
 }
 //function two swap two numbers.
 //we will pass index of array to swap
 private void swap(int i, int j) {
     int temp = A[i];
     A[i] = A[j];
     A[j] = temp;
 }

//Driver program to test above functions
 public static void main(String args[])
 {
     int arr[] = {1, 3, 9, 8, 2, 7, 5};
     int n = arr.length;
   //print unsorted array using Arrays.toString()
     System.out.print("Unsorted array: ");
     System.out.println(Arrays.toString(arr));
     
     QuickSort ob = new QuickSort();
     ob.sort(arr, 0, n-1);

     System.out.print("Sorted array: ");
   //print sorted array
     System.out.println(Arrays.toString(arr)); 
 }
}

</pre>
         <pre class="output"><span style="text-decoration: underline;"><strong>Output:-</strong></span>
Unsorted array:  1 3 9 8 2 7 5 
Sorted array:    1 2 3 5 7 8 9</pre>
         <hr>
         <p>Quick sort is one of the fast and important sorting algorithms, which is widely used for commercial applications. Never use quick sort for applications which requires guaranteed response time<b>. </b>As a good programmer, you should be aware of this algorithm and it is fast sorting algorithm with time complexity of O(n log n) in an average case. Always use quick sort or merge sort for faster and efficient programming. The only disadvantage of quick sort is that its worst time complexity is O(n*logn). But, it has an advantage over merge sort as it is in-place sorting algorithm.</p>
