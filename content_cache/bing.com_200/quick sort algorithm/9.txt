         <h2><span style="text-decoration: underline">Quicksort algorithm in java</span></h2> 
         <p>The basic idea behind quick-sort is this: Specify one element in the list as a pivot point. Then, go through all of the elements in the list, swapping items that are on the wrong side of the pivot. In other words, swap items that are smaller than the pivot but on the right side of the pivot with items that are larger than the pivot but on the left side of the pivot. Once youve done all possible swaps, move the pivot to wherever it belongs in the list. Now we can ignore the pivot, since its in position, and repeat the process for the two halves of the list (on each side of the pivot). We repeat this until all of the items in the list have been sorted. Quick-sort is an example of a divide and conquer algorithm. Quick sort sorts a list effectively by dividing the list into smaller and smaller lists, and sorting the smaller lists in turn.</p> 
         <p>&nbsp;</p> 
         <p>&nbsp;</p> 
         <p><strong><span style="text-decoration: underline">Example:</span></strong></p> 
         <p>&nbsp;</p> 
         <p><a href="https://i1.wp.com/j2eereference.com/wp-content/uploads/2015/06/Picture11.png"><img class="alignnone  wp-image-8886" src="https://i2.wp.com/j2eereference.com/wp-content/uploads/2015/06/Picture11-300x221.png?resize=654%2C482" alt="Picture11" srcset="https://i1.wp.com/j2eereference.com/wp-content/uploads/2015/06/Picture11.png?resize=300%2C221&amp;ssl=1 300w, https://i1.wp.com/j2eereference.com/wp-content/uploads/2015/06/Picture11.png?resize=1024%2C755&amp;ssl=1 1024w, https://i1.wp.com/j2eereference.com/wp-content/uploads/2015/06/Picture11.png?w=1395&amp;ssl=1 1395w" sizes="(max-width: 654px) 100vw, 654px" data-recalc-dims="1"></a></p> 
         <p>&nbsp;</p> 
         <p>&nbsp;</p> 
         <p>As you can see in the above example: after pass 1 all the elements which are less than or equal to 65 are before 65(pivot) and all the other elements which are greater than 65 are at the right side.</p> 
         <p>Now we need to repeat same process for left half and right half of the array. then we will get sorted array.</p> 
         <p>&nbsp;</p> 
         <p><strong><u>Java implementation for Quick&nbsp;sort:</u></strong></p> 
         <p>The Java implementation of quicksort is a bit more complex than our previous sort algorithms.&nbsp;In fact, we will use two different methods in order to implement quicksort. The first method is the primary method: it sets up the algorithm and then calls the other method, which does the actual sorting.</p>

         <p>Before we get to the partition method, lets take a close look at the quicksort() method. quicksort() actually calls itself, twice! A method that calls itself is called a recursive method. Recursive methods are useful when we need to repeat &nbsp;some series of tasks number of times with a parameter that varies each time the calculation is done.</p> 
         <p>Recursive methods have three defining features:</p> 
         <p> A base case or stop case, that indicates when the method should stop calling itself</p> 
         <p> A test to determine if recursion should continue</p> 
         <p> One or more calls to itself</p> 
         <p>In other words, each time we call the method, we change the parameters that we send to the method, test to see if we should call the method again, and provide some way to ultimately return from the method.</p> 
         <p>In the quicksort() method, the conditional statement serves as the test. The method is called twice, once for the lower half of the list and once for the upper half of the list. The stop case occurs when low is greater than or equal to high: in this case, nothing is done because the condition in the conditional statement is false. The quicksort() method calls another method that selects the pivot and moves the elements in the list around as in the example. It returns the index of the pivot point in the list, which is the item in the list that is now fixed in place.</p>

public int partition(int[] list, int low, int high) 
{
	// The pivot point is the first item in the subarray
	int pivot = list[low];
	// Loop through the array. Move items up or down the array so that they
	// are in the proper spot with regards to the pivot point.
	do 
	{
		// can we find a number smaller than the pivot point? keep
		// moving the high marker down the array until we find
		// this, or until high=low
		while (low &lt; high &amp;&amp; list[high] &gt;= pivot) 
		{
			high--;
		}
		if (low &lt; high) 
		{
			// found a smaller number; swap it into position
			list[low] = list[high];
			// now look for a number larger than the pivot point
			while (low &lt; high &amp;&amp; list[low] &lt;= pivot) 
			{
				low++;
			}
			if (low &lt; high) 
			{
				// found one! move it into position
				list[high] = list[low];
			}
		}
	} while (low &lt; high);
	// move the pivot back into the array and return its index
	list[low] = pivot;
	return low;
}</textarea>

         <p><strong><u>Performance of Quick&nbsp;sort algorithm:</u></strong></p> 
         <ol> 
          <li>&nbsp;The best case of quicksort occurs, obviously, when the list is already sorted. For this algorithm, the best case resembles the average case in terms of performance.</li> 
          <li>The average case occurs when the pivot splits the list in half or nearly in half on each pass. Each pass through the algorithm requires N comparisons. But how many passes does quicksort take? Recall that every time we divide something in half repeatedly, as in binary search, the number of operations is approximately log2 N. So, the number of passes through the algorithm is approximately log2N, and thus the number of comparisons in the average and best cases is O(NlogN). The number of swaps differs in the best and average cases: for the best case, we have no swaps, but for the average case, by the same reasoning, we have O(NlogN) swaps.</li> 
          <li>The worst case for quicksort occurs when the pivot is always the largest or smallest item on each pass through the list. In this instance, we do not split the list in half or nearly in half, so we do N comparisons over roughly N passes, which means that in the worst case quicksort is closer to O(N2 ) in performance. For the same reason, the number of swaps in the worst case can be as high as O(N2 ).</li> 
         </ol> 
         <p>Lets see whether quick&nbsp;sort is stable and in-place or not:</p> 
         <p><span style="text-decoration: underline"><strong>Stability:</strong></span></p> 
         <p>Quick sort is not stable, since it exchanges non adjacent elements.</p> 
         <p>Suppose the array is:5 2 3 5 1<br> Lets distinguish the two 5s as 5(a) and 5(b) .<br> So our array is:<br> 5(a) &nbsp;2 3 &nbsp;5(b) &nbsp;1</p> 
         <p>Suppose we choose <code>3</code> as the pivot. The two <span style="font-family: monospace">5</span>&nbsp;elements will end up after pivot&nbsp;and the <code>1</code> and the <code>2&nbsp;</code>before pivot.</p> 
         <p>After iteration 1: &nbsp;1&nbsp; 2 3 &nbsp;5(b) &nbsp;5(a)</p> 
         <p>after last iteration our array becomes:<br> 1&nbsp; 2 3 &nbsp;5(b) &nbsp;5(a)<br> Since now our array is in sorted order and we clearly see that 5(a) comes before 5(b) in initial array but not&nbsp;in&nbsp;the sorted array.<br> So we can clearly see that quick&nbsp;sort is not stable.</p> 
         <p><span style="text-decoration: underline"><strong>In place:</strong></span></p> 
         <p>An <b>in-place algorithm&nbsp;</b>is an algorithm&nbsp;which transforms input using a data structure&nbsp;with a small, constant amount of extra storage space. The input is usually overwritten by the output as the algorithm executes</p> 
         <p>you can have in-place &nbsp;version of quick sort and not in-place version of quicksort also.</p> 
         <p>Quicksort with in-place and unstable partitioning uses only constant additional space before making any recursive call. Quicksort must store a constant amount of information for each nested recursive call.&nbsp;QuickSort as <em>in-place</em>  it sorts the elements within the array with <em>at most</em> a constant amount of them outside the array at any given time.</p> 
         <p>Another, not-in-place, version of quicksort uses <span class="texhtml"><i>O</i>(<i>n</i>)</span> space for working storage and can implement a stable sort. The working storage allows the input array to be easily partitioned in a stable manner and then copied back to the input array for successive recursive calls.</p>
