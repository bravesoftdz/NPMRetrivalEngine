     <h1>Lesson 8: Package your JavaScript Code with Functions</h1> 
     <p><em>By <a href="http://www.mariaantoniettaperna.com">Maria Antonietta Perna</a></em></p> 
     <p>You can already add quite a bit of JavaScript functionality to web pages, and that's great. However, what if you decide to implement the same kind of functionality on more than one page, or in different places on the same page?</p> 
     <p>This is where <strong>JavaScript functions</strong> come into play.</p> 
     <p>In this lesson, you will learn:</p> 
     <ul> 
      <li>what <strong>functions</strong> are and how they are used in JavaScript;</li> 
      <li>how to <strong>retrieve information from functions</strong>;</li> 
      <li>how to <strong>give information to functions</strong>;</li> 
      <li>what <strong>variable scope</strong> is all about.</li> 
     </ul> 
     <p>In the process you will also learn how to:</p> 
     <ul> 
      <li>get today's date dynamically;</li> 
      <li>change the value of an HTML button element dynamically.</li> 
     </ul> 
     <h2>Functions: what they are and what they are for</h2> 
     <p>In previous lessons you used JavaScript functions to perform some actions quickly and easily like generating random numbers, writing something on a web page, etc. In fact, JavaScript has a great many built-in functions like <strong>write(), alert(), getElementById(), random(), floor()</strong>, and several others. Wherever you spot a pair of round brackets there's likely a function in action.</p> 
     <p>JavaScript also gives you the option to craft your own functions.</p> 
     <p><strong>A function is a way of packaging your JavaScript commands so you can easily reuse them every time you need the same piece of functionality implemented in your website</strong>.</p> 
     <p>This suggests that there are 2 distinct phases to a function:</p> 
     <ol> 
      <li> <h3>The phase where the <strong>function is declared (created)</strong>;</h3> </li> 
      <li> <h3>The phase where the <strong>function is called (used)</strong>.</h3> </li> 
     </ol> 
     <p>Let's go through each phase in turn.</p> 
     <h2>Declare a function</h2> 
     <p>The basic structure of a JavaScript function looks like this:</p> 
     <div class="codebox"> 
      <pre>
	
	  <span class="code-comments">//keyword <em>function</em> followed by your</span>
	
	  <span class="code-comments">//chosen name for the function</span>
	
	  <span class="code-comments">//<strong>Don't forget the round brackets!</strong></span>
	
	  <span class="code-javascript">function <em>functionName()</em></span>
	
	  <span class="code-comments">//the code block is <strong>inside curly braces</strong></span>
	
	  <span class="code-javascript">{</span>
	
	  <span class="code-comments">//code you want to run goes here</span>
	
	  <span class="code-javascript">}</span>
	
	  </pre> 
     </div> 
     <h2>Call a function</h2> 
     <p>Once you put a block of code into a function, you can use it anywhere you need that functionality in your website. What you need to do is just <strong>call the function</strong>.</p> 
     <p>You do this by typing the function name followed by brackets and the function will do its job - just like you did with alert(). Here's how it's done:</p> 
     <div class="codebox"> 
      <pre>
	
	  <span class="code-comments">//Type the function name (without the keyword <em>function</em>)</span>
	
	  <span class="code-comments">//<strong>Don't forget the round brackets!</strong></span>
	
	  <span class="code-javascript"><em>functionName()</em>;</span>
	
	  </pre> 
     </div> 
     <p>That's all you need: whatever code you stuffed into your function gets executed at this point.</p> 
     <p>Let's see how this works in practice. Fire off your text editor: it's time to get coding!</p> 
     <h2>Try out: declare your function</h2> 
     <p>Prepare a simple HTML page like the one below. The program we're going to build has the following goals:</p> 
     <ul> 
      <li>to get today's date dynamically when the user clicks a button;</li> 
      <li>to display today's date on the web page;</li> 
      <li>to change the value attribute of the button after it's clicked: to have the button still displaying <em>Get Date</em> after the date has been displayed looks a bit confusing to the user.</li> 
     </ul> 
     <div class="codebox"> 
      <pre>
	
	  &lt;!DOCTYPE html&gt;
	  &lt;html&gt;
	  &lt;head&gt;
	  &lt;title&gt;Lesson 8: Declare a function&lt;/title&gt;
	  &lt;/head&gt;
	  &lt;body&gt;
	  &lt;h1&gt;Lesson 8: Declare a function&lt;/h1&gt;
	
	  &lt;div&gt;
	
	    &lt;h2&gt;Today's date is:&lt;/h2&gt;
	  
	    &lt;span id="calendar"&gt;&lt;/span&gt;
	  
	    &lt;input type="button" id="myButton" value="Get Date" /&gt;
	
	
	  &lt;/div&gt;
	  
	  &lt;script type="text/javascript"&gt;
	
	  <span class="code-comments">//Declare your function here</span>
	
	  <span class="code-javascript">function showDate()</span>
	
	  <span class="code-javascript">{</span>
	
	  <span class="code-comments">//the block of code starts here:</span>
	
	  <span class="code-comments">//First get all your vars ready</span>
	
	  <span class="code-comments">//This is how JavaScript retrieves today's date</span>
	
	  <span class="code-javascript">var today = new Date();</span>
	
	  <span class="code-comments">//get hold of the calendar span element</span>
	
	  <span class="code-comments">//where today's date will be inserted</span>
	
	  <span class="code-javascript">var myCalendar = document.getElementById("calendar");</span>
	
	  <span class="code-comments">//get hold of the button:you need this when it comes</span>
	
	  <span class="code-comments">//to change its value attribute</span>
	
	  <span class="code-javascript">var myButton = document.getElementById("myButton");</span>
	
	  <span class="code-comments">//insert the date in the span element.</span>
	
	  <span class="code-comments">//toDateString() changes the date just retrieved</span>
	
	  <span class="code-comments">//into a user-friendly format for display</span>
	
	  <span class="code-javascript">myCalendar.innerHTML = today.toDateString();</span>
	
	  <span class="code-comments">//change the value attribute of the button</span>
	
	  <span class="code-comments">//to say something more appropriate once the date is displayed</span>
	
	  <span class="code-javascript">myButton.value = "Well done!";</span>
	
	  <span class="code-javascript">}</span>
	
	  &lt;/script&gt;
		
	  &lt;/body&gt;
	  &lt;/html&gt;
	
	  </pre> 
     </div> 
     <h2>Try out: call your function</h2> 
     <p>If you review your program's goals as set out at the beginning, you'll see that <strong>all the action takes place after the user clicks the button</strong> on the page. This tells us that we need to <strong>handle the button's onclick event</strong>.</p> 
     <p>If you go back to lesson 3 for a moment, you remember that one way in which this can easily be done is to put some JavaScript code as the value of the <strong>HTML element onclick attribute</strong>.</p> 
     <p>Just add an <em>onclick</em> attribute to the button element and plug your showDate() function right in there, like so:</p> 
     <div class="codebox"> 
      <pre>
	
	  &lt;input type="button" id="myButton" value="Get Date" <strong>onclick="showDate();"</strong> /&gt;
	
	  </pre> 
     </div> 
     <div class="examplelink">
      <img src="/site/graphics/icon.example.gif" alt=" - "> 
      <a href="lesson8_ex1.html" title="View the above code as an example">Show example</a>
     </div> 
     <p>Save your work and run the page in the browser. Click the button and you should see today's date displayed on the page and the button value attribute giving you an appropriate message.</p> 
     <h2>Questions, questions, questions</h2> 
     <p>I know, there's some new stuff in the code samples above. You might be wondering: 1) what's this <em>new Date()</em> business? And 2) What does toDateString() do to the date? Let's tackle each question in turn.</p> 
     <ol> 
      <li> <h3>What's new Date() all about?</h3> <p>The <em>new</em> keyword creates a <strong>new instance of a JavaScript object</strong>, in this case a Date object based on the user's computer clock. Objects are the topic of the next lesson, and the Date object is the topic of lesson 11. Therefore, I keep things really short at this point.</p> <p>The important thing you need to know now is that once you've created an instance of the Date object, you've got all sorts of useful functions (called <em>the object's methods</em>) at your fingertips to manipulate date and time.</p> </li> 
      <li> <h3>Why did I use <em>toDateString()</em> with the date?</h3> <p>Following on from the previous point, <strong>toDateString()</strong> is only one of the numerous methods you can use with the Date object. The JavaScript interpreter reads it as follows:</p> 
       <blockquote>
        "Hey browser, take the date you've been attached to and make it a bit more human-friendly!"
       </blockquote> <p>This is what I mean. Type the following inside enclosing &lt;script&gt; tags of an HTML page: <em>var myDate = new Date(); document.write(myDate);</em></p> <p>Save the page and run it in the browser: your date should be displayed in a format like the one below (obviously I expect the date to be different):</p> <p><strong>Sun Nov 06 2011 14:45:30 GMT+0000 (GMT Standard Time)</strong></p> <p>Ugly! Luckily, the Date object has its own beauty remedy. Rewrite the previous code snippet as follows: <em>var myDate = new Date(); document.write(myDate.toDateString());</em></p> <p>And here's what the date above looks like after its beauty treatment with <em>toDateString()</em>:</p> <p><strong>Sun Nov 06 2011</strong>.</p> <p>Much better, less scary, and far more readable for us humans.</p> </li> 
     </ol> 
     <h2>Feed information to and retrieve information from a function</h2> 
     <p>Functions manipulate data: the alert() function has messages to display as its data, the write() function has text to write on the web page as its data, etc.</p> 
     <p>You can also <strong>feed data to a function</strong> for manipulation. The way you do this is through <strong>arguments (or parameters)</strong>. An <strong>argument is placed inside the function's brackets when the function is declared</strong>. You can place one or more arguments separated by commas( , ) inside a function.</p> 
     <p>Here's what the basic structure looks like:</p> 
     <div class="codebox"> 
      <pre>
	
	  <span class="code-javascript">function <em>functionName(arg1, arg2, arg3)</em></span>
	
	  <span class="code-javascript">{</span>
	
	  <span class="code-comments">//body of the function goes here</span>
	
	  <span class="code-javascript">}</span>
	
	  </pre> 
     </div> 
     <p>The function argument is like a <strong>placeholder for the value that gets fed when the function is called</strong>. This will appear clearer in the example below.</p> 
     <p>Finally, one useful thing functions can do with data, once it's been manipulated, is to <strong>return it</strong>.</p> 
     <p>For example, take the <strong>floor(number) function</strong> that you already know from the previous lesson. This function <strong>takes in a number argument</strong> and <strong>returns an integer number</strong>.</p> 
     <p>If your script does something with the <strong>returned value then it needs to assign the function to a variable</strong> when calling the function.</p> 
     <p>Let's put this knowledge into practice right away. Create a function and then call it to use its <strong>return value</strong> in your document.</p> 
     <h3>Declare a function with a parameter</h3> 
     <p>Prepare a simple HTML page and type the following JavaScript code within <em>&lt;script&gt; ... &lt;/script&gt;</em> tags:</p> 
     <div class="codebox"> 
      <pre>
	
	  &lt;!DOCTYPE html&gt;
	  &lt;html&gt;
	  &lt;head&gt;
	  &lt;title&gt;Lesson 8: Function Arguments and Return Values&lt;/title&gt;
	  &lt;/head&gt;
	  &lt;body&gt;
	  &lt;h1&gt;Lesson 8: Function Arguments and Return Values&lt;/h1&gt;
	  
	  &lt;script type="text/javascript"&gt;
	
	  <span class="code-comments">//declare a function with two arguments of type number</span>
	
	  <span class="code-javascript">function addNumbers(num1, num2)</span>
	
	  <span class="code-javascript">{</span>
	
	  <span class="code-comments">//this is a simple function:</span>
	
	  <span class="code-comments">// just return the sum of the two number arguments</span>
	
	  <span class="code-javascript">return num1 + num2;</span>
	
	  <span class="code-comments">//your function ends here</span>
	
	  <span class="code-javascript">}</span>
	
	  &lt;/script&gt;
		
	  &lt;/body&gt;
	  &lt;/html&gt;
	
	  </pre> 
     </div> 
     <h3>Call your function and use its return value</h3> 
     <p>Continue on from the previous code. Outside your function, just <strong>after the } closing curly brace</strong>, type the following:</p> 
     <div class="codebox"> 
      <pre>
	
	  <span class="code-comments">//Call your function: create a var</span>
	
	  <span class="code-comments">//and assign the value returned by your function to it</span>
	
	  <span class="code-comments">//Also, give a value to the number arguments</span>
	
	  <span class="code-comments">//by typing 2 comma-separated numbers within brackets</span>
	
	  <span class="code-javascript"><strong>var total = addNumbers(3, 5);</strong></span>
	
	  <span class="code-comments">//Display the returned data on the page</span>
	
	  <span class="code-javascript">document.write(total);</span>
	
	  </pre> 
     </div> 
     <p>Save your work and run the page in the browser. If all goes well, the sum of the numbers you inserted as arguments of your addNumbers() function should be displayed on the web page.</p> 
     <p>Function arguments are not limited to a specific data type. You can also use strings and booleans, and even other functions. However, we don't want to be too involved at this stage.</p> 
     <h2>Variables inside and outside functions: scope</h2> 
     <p>A variable can have <strong>local or global scope on the basis of whether it's declared inside or outside a function block.</strong>. But, what does this mean exactly?</p> 
     <p>Simply put, a variable having <strong>local scope means that it's visible, or accessible, only within the function in which it lives</strong>. No other portion of your code can see a local variable.</p> 
     <p>On the other hand, a variable having <strong>global scope means that it's accessible, that is, it can be used, anywhere in your script.</strong>. Let's see this in practice. Get your text editor ready and type the following snippet between opening and closing &lt;script&gt; tags in an HTML page:</p> 
     <div class="codebox"> 
      <pre>
	
	  <span class="code-comments">//create 2 global variables</span>
	
	  <span class="code-javascript">var message = "outside global message";</span>
	
	  <span class="code-javascript">var otherGlobalVariable = "other global variable";</span>
		
	  <span class="code-comments">//create a function with a local variable</span>
	
	  <span class="code-comments">//having the same name as the first global variable</span>
	
	  <span class="code-javascript">function getMessage()</span>
	
	  <span class="code-javascript">{</span>
	
	  <span class="code-javascript">var message = "inside local variable";</span>
	
	  <span class="code-comments">//the function alerts the message variable</span>
	
	  <span class="code-javascript">alert(message);</span>
	
	  <span class="code-comments">//and it also alerts the second global variable</span>
	
	  <span class="code-javascript">alert(otherGlobalVariable);</span>
	
	  <span class="code-comments">//the function ends here</span>
	
	  <span class="code-javascript">}</span>
	
	  <span class="code-comments">//call the function</span>
	
	  <span class="code-javascript">getMessage();</span>
		
	  <span class="code-comments">//alert the message variable</span>
	
	  <span class="code-javascript">//(which one will it be, local or global?)</span>
	
	  <span class="code-javascript">alert(message);</span>
	
	  </pre> 
     </div> 
     <p>Save your work and preview the page in your browser. As you can see, the <strong>alert() function outside the variable doesn't have access to the message variable inside the function</strong>. On the other hand, the <strong>function can have access both to its local variable and to any variable in the global space</strong>.</p> 
     <p>This might seem a bit confusing at first, but it's all a matter of practicing your coding skills as often as you can. The important thing to remember is that, if at times the variables seem not to have the values you expect, it might be a scope-related bug.</p> 
     <h2>Summary</h2> 
     <p>That's all for this lesson. You can now package your JavaScript code into reusable and manageable functions, input and output data via a function, use the Date object to retrieve and display the current date, change the value of an HTML button element on-the-fly, and distinguish between local and global variables. You've accomplished a great deal, congratulations!</p> 
     <p>Take a break and get ready for the next big topic, a core feature of contemporary programming languages, <strong>objects</strong>.</p>
