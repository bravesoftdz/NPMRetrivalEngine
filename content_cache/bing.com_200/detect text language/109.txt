        <h1 class="entry-title">Server Side Language Detection with Google Language API</h1> 
        <div class="entry-meta"> 
         <span class="entry-date"><a href="http://blog.lodeblomme.be/2008/03/21/server-side-language-detection-with-google-language-api/" rel="bookmark"><time class="entry-date" datetime="2008-03-21T18:35:06+00:00">March 21, 2008</time></a></span> 
         <span class="byline"><span class="author vcard"><a class="url fn n" href="http://blog.lodeblomme.be/author/lode/" rel="author">Lode</a></span></span> 
         <span class="comments-link"><a href="http://blog.lodeblomme.be/2008/03/21/server-side-language-detection-with-google-language-api/#comments">9 Comments</a></span>

       </header>
       <!-- .entry-header --> 
       <div class="entry-content"> 
        <p>Yesterday Google <a href="http://googleblog.blogspot.com/2008/03/new-google-ajax-language-api-tools-for.html" title="New Google AJAX Language API - Tools for translation and language detection" target="_blank">announced</a> a new <a href="http://code.google.com/apis/ajaxlanguage/" target="_blank" title="Google AJAX Language API">AJAX API for translation and language detection</a>. It’s a Javascript API to translate and detect the language of blocks of text within a webpage. But I need server side language detection of a text using PHP and the Google AJAX Language API.<span id="more-6"></span></p> 
        <h1>Step 1 : Find Out Internal Working</h1> 
        <p>You can test the language detection in <a href="http://googleblog.blogspot.com/2008/03/new-google-ajax-language-api-tools-for.html" title="The Official Google Blog" target="_blank">the announcement post on The Official Google Blog</a>. So I did, and I took my friend <a href="http://www.getfirebug.com/" title="Monitor network activity with FireBug" target="_blank">FireBug</a> with me. This revealed the URL from which the Javascript code requests the magic :</p>

        <p>Which is without any doubt <a href="http://ajaxian.com/archives/jsonp-json-with-padding" title="JSONP: JSON With Padding" target="_blank">JSONP</a>. It invokes a Javascript method “google.language.callbacks.id102”, passing 5 parameters to it.</p> 
        <h1>Step 2 : Analyze</h1> 
        <h2>Request</h2> 
        <p>Lets analyze all parameters in the request URL :</p> 
        <ul> 
         <li><em>callback</em> : This is the name of the function that is called when the response is ready. It can be left-off.</li> 
         <li><em>context</em> : No idea. It can be left-off.</li> 
         <li><em>q</em> : The text you want to detect the language for. Is required (DUH!).</li> 
         <li><em>key</em> : An API key. It can be left-off as it seems.</li> 
         <li><em>v</em> : The API version. Is required.</li> 
        </ul> 
        <h2> Response</h2> 
        <p>Like I said, the reponse is a call to the Javascript method “google.language.callbacks.id102”. It takes 5 parameters :</p> 
        <ul> 
         <li><em>22</em> : Same value as the context parameter in the request URL.</li> 
         <li>A Javascript object with 3 properties : 
          <ul> 
           <li><em>language</em> : The ISO2 language code of the language detected by Google.</li> 
           <li><em>isReliable</em> : Is the quess by Google reliable?</li> 
           <li><em>confidence</em> : How confident is Google about the guess? I think it takes values from 0 to 1, 0 being least reliable and 1 being most reliable.</li> 
          </ul> </li> 
         <li><em>200</em> : No idea.</li> 
         <li><em>null</em> : No idea.</li> 
         <li><em>200</em> : Again, no idea.</li> 
        </ul> 
        <h1>Step 3 : Bend and Break</h1> 
        <p>Now lets see what we need to do to make this work in PHP.</p>

        <h2>Dump Unknown Request Parameters</h2> 
        <p>Don’t use magic you don’t understand, so I dump the request parameters “context” and “key”. The result doesn’t seem to change when leaving these off.</p> 
        <h2>No JSONP</h2> 
        <p>Since I’m going to use PHP to request the language detection, I won’t be needing the callback parameter. So I dump that one too. The result is nice :</p>

Just like the <a href="http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.2.1" title="HTTP Status Code Definitions" target="_blank">HTTP Code 200 OK</a>.</p>
