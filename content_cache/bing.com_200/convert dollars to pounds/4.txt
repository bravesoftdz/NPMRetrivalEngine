      <h2 class="entry-title"> <a href="http://www.cssnewbie.com/javascript-currency-conversion-script/" title="Permalink to A JavaScript Currency Conversion Script" rel="bookmark">A JavaScript Currency Conversion Script</a> </h2> 
      <div class="meta"> 
       <p>Published November 28, 2011 by CSS Newbies.</p> 
      </div> 
      <div class="entry-content"> 
       <p><img src="http://www.cssnewbie.com/wp-content/uploads/2011/11/currency-convert-example.png" alt="" width="500" height="100" class="alignnone size-full wp-image-641"></p> 
       <p>It turns out that converting random numbers into formatted currency (with dollar signs, commas, and periods) is more difficult in JavaScript than I would have expected. There’s no built-in function for it, and it’s something I run into a lot. The function I wrote has worked well for me on a couple of projects now, so I thought I’d share it with you-all.</p> 
       <p>The little JavaScript function below does exactly one thing: it converts numbers (integers, floats, strings, whatever) into formatted currency. My example is in US dollars, but it’d be a quick change to make it work for other currencies as well.</p> 
       <h3>The Problem This Solves</h3> 
       <p>I wrote this script because I had to display dollar amounts in a “friendly” format in several places on a website, but I wasn’t being handed friendly numbers by the server. The numbers never included commas, of course. But there’s also the problem of whole-dollar and multiple-of-ten-cent amounts: I would have “500” when the user would expect “500.00”, or “250.2” when the user would expect “250.20”.<span id="more-635"></span></p> 
       <h3>The JavaScript</h3> 
       <p>First the script, then the explanation.</p> 
       <pre lang="javascript" escaped="true">function toUSD(number) {
    var number = number.toString(), 
    dollars = number.split('.')[0], 
    cents = (number.split('.')[1] || '') +'00';
    dollars = dollars.split('').reverse().join('')
        .replace(/(\d{3}(?!$))/g, '$1,')
        .split('').reverse().join('');
    return '$' + dollars + '.' + cents.slice(0, 2);
}</pre> 
       <h3>The Explanation</h3> 
       <p>The function expects to be passed one variable: the number you wanted converted. I first convert it to a string for easier manipulation. From that, I create a few more variables. The “dollars” variable is everything before the decimal point. The “cents” variable is everything after the decimal point, plus a couple of extra zeroes just to be safe (we’ll lop off extra zeroes later). If there is no decimal point, cents becomes “00”.</p> 
       <p>Next, I do a whole heck of a lot of manipulation to the dollars string. This is all to add the commas in the correct places. I’ll walk through each of the manipulations one by one to show you what we have after each one. We’ll assume we have an initial dollar string of “123456”.</p> 
       <p><strong>dollars.split(”)</strong> converts our string into an array of individual digits:</p> 
       <pre>["1", "2" ,"3" , "4", "5", "6"]</pre> 
       <p><strong>.reverse()</strong> reverses our array:</p> 
       <pre>["6", "5", "4", "3", "2", "1"]</pre> 
       <p><strong>.join(”)</strong> turns our array back into a single string again:</p> 
       <pre>"654321"</pre> 
       <p><strong>.replace(/(\d{3}(?!$))/g, ‘$1,’)</strong> is a complicated (for me) regular expression that basically says “add a comma to the end of every group of three numbers, unless it is the last group of three numbers (to avoid trailing commas)”:</p> 
       <pre>"654,321"</pre> 
       <p><strong>.split(”)</strong> converts our newly comma’d string to an array again:</p> 
       <pre>["6", "5", "4", ",", "3", "2", "1"]</pre> 
       <p><strong>.reverse()</strong> puts the numbers back in the proper order:</p> 
       <pre>["1", "2", "3", ",", "4", "5", "6"]</pre> 
       <p><strong>.join(”)</strong> makes them a string again:</p> 
       <pre>"123,456"</pre> 
       <p>After that, all we do is return our manipulated number with a dollar sign out front and a decimal point between dollars and cents (and I’m using slice to only return the first two digits in the cents string).</p> 
       <h3>How To Internationalize</h3> 
       <ul> 
        <li>If your currency of choice uses periods instead of commas as a thousands-place divider, change ‘$1,’ in the replace function to be ‘$1.’ Instead.</li> 
        <li>If you use a comma instead of a period between dollars and cents, change “dollars + ‘.'” in the return line to read “dollars + ‘,'” instead. If you are starting with a number that uses a comma to denote a decimal, you’d also need to change the dollars and cents variables to have “.split(‘,’)” instead of “.split(‘.’)”. </li> 
        <li>To use something other than a dollar sign in the currency, simply replace the ‘$’ in the return line with your currency sign.</li> 
       </ul>

       <p>If you’d like to see it in action, <a href=" http://www.cssnewbie.com/example/currency-conversion/">here’s a demo.</a></p>
