        <article id="ocpArticleContent" class="ocpArticleContent"> 
         <a name="__top" aria-hidden="true"> </a> 
         <section class="ocpIntroduction"> 
          <p>This article describes the formula syntax and usage of the <b class="ocpLegacyBold">DOLLAR</b>&nbsp;function in Microsoft Excel.</p> 
         </section> 
         <section class="ocpSection"> 
          <h2>Description</h2> 
          <p>The function described in this Help topic converts a number to text format and applies a currency symbol. The name of the function (and the symbol that it applies) depends upon your language settings.</p> 
          <p>This function converts a number to text using currency format, with the decimals rounded to the specified place. The format used is $#,##0.00_);($#,##0.00).</p> 
         </section> 
         <section class="ocpSection"> 
          <h2>Syntax</h2> 
          <p>DOLLAR(number, [decimals])</p> 
          <p>The DOLLAR function syntax has the following arguments:</p> 
          <ul> 
           <li> <p> <b class="ocpRunInHead">Number</b>&nbsp;&nbsp;&nbsp;&nbsp;Required. A number, a reference to a cell containing a number, or a formula that evaluates to a number.</p> </li> 
           <li> <p> <b class="ocpRunInHead">Decimals</b>&nbsp;&nbsp;&nbsp;&nbsp;Optional. The number of digits to the right of the decimal point. If decimals is negative, number is rounded to the left of the decimal point. If you omit decimals, it is assumed to be 2.</p> </li> 
          </ul> 
         </section> 
         <section class="ocpSection"> 
          <h2>Remark</h2> 
          <p>The difference between formatting a cell with a ribbon command and using the DOLLAR function is that DOLLAR converts its result to text. A number formatted with the <b class="ocpUI">Format Cells</b> dialog box is still a number. You can continue to use the results generated by DOLLAR in other formulas, because Excel converts numbers entered as text to numbers when it calculates.</p> 
         </section>
