is the exchange rate between the euro currency and the currency at the date specified in the &lt;ALLCURRENCIES&gt;&lt;DATE&gt; Tag<br> &lt;LIBEN&gt; is the name in English of the country (or organization) issuing the currency, for example "United States" for USD)<br> &lt;LIBDEVEN&gt; is the currency label in English (for example "US dollar" for USD)<br> &lt;LIBFR&gt; is the name in French of the country (or organization) issuing the currency, for example "Etats-Unis" for USD)<br> &lt;LIBDEVFR&gt; is the currency label in French (for example "dollar américain" for USD)<br> &lt;LIBIT&gt;, &lt;LIBES&gt;, &lt;LIBDE&gt;, &lt;LIBDK&gt;, &lt;LIBSE&gt;, &lt;LIBNO&gt;, &lt;LIBPT&gt;, &lt;LIBFI&gt;, &lt;LIBNL&gt; : label of the country issuing currency respectively in Italian, Spanish, German, Danish, Swedish, Norwegian, Portuguese, Finnish, Flemmish<br> &lt;LIBDEVIT&gt;, &lt;LIBDEVES&gt;, &lt;LIBDEVDE&gt; : label of the currency respectively in Italian, Spanish, German<br> &lt;ISOCODE&gt; is the three letter code for the currency<br> &lt;SHLIB&gt; is the short label, for example "$" for US Dollar, "F" for French Franc<br> &lt;NBDEC&gt; is the number of digits after the decimal point, for example 2 for US Dollar because people use cents for Dollars but 0 for Italian Lira because they have no cents.<br> &lt;MODEC&gt; is a number corresponding to the importance or the category of the currency (for example 3 for a currency belonging to Euroland) )<br> Example :<br> &lt;ALLCURRENCIES&gt; &lt;DATE&gt;08/09/2000&lt;/DATE&gt; &lt;PREVIOUSDATE&gt;07/09/2000&lt;/PREVIOUSDATE&gt; &lt;CURRENCY&gt; &lt;ISOCODE&gt;ADF&lt;/ISOCODE&gt; &lt;LIBEN&gt;Andorra (Franc)&lt;/LIBEN&gt; &lt;DAYPRICE&gt;5.7250&lt;/DAYPRICE&gt; &lt;/CURRENCY&gt; <br> Means that 1 euro (EUR) = 5.7250 Andorra Franc (ADF) on 08/09/2000 <br> To get the exchange rate between the US Dollar and the British pound you have to do a division : 1 USD = EURGBP/EURUSD GBP <br> with EURGBP=0.61240 (1 EUR =0.61240 GBP) and EURUSD= 0.8735 (1 EUR= 0.8735 USD)<br> so you get :<br> 1 USD = 0.61240 / 0.8735 GBP = 0.70108 GBP <p></p> </td>

             <td align="left"> We allow third parties to call some of our functions to convert amounts from one currency to another, retrieve exchange rates (today or in the past), retrieve currency description, list of currencies. <br> Some services are commercial services so you need subscription, please <a href="contactus.php">contact us</a> to get a quote. <br> <h3>Standard web service using SOAP protocol and WSDL file</h3> SOAP is the standard way to use web service, it is based on a Web Service Definition Language (WSDL) file (special file written in XML way). It can be used by any application, XML files are exchanged with that protocol. <br> We provide such web service based on SOAP protocol, we based it on http protocol which is more convenient (http port is not blocked by firewalls). We use standard Unicode (UTF-8) charset for encoding strings. <br> Our <a href="/dev/service.wsdl">Web Service Definition Language (WSDL) file is here</a> <br> This file describes 3 services and associated parameters : <br> <strong>DescCurrency</strong> : return description of a currency : 3 char Iso Code , language (2 chars long : FR, EN, DE, IT, ES, FI, DK, NL, PT SE NO, JP), label of country of the currency in this language, label of currency (the label of country exists in 12 languages but currency label is given only in French (FR), English (EN), German (DE), Italian (IT) and spanish (ES) so far) used charset is standard Unicode UTF-8. <br> <strong>ListCurrencies</strong> : return list of currencies dealt by our site (over 200 currencies) : it is a sequence with same content as DescCurrency service. <br> <strong>Convert</strong>: converts amount from one currency to another at a date (today's date if no date is given). For this service, you need to provide User/Login parameters, if you don't provide them or if they are incorrect, conversions will be done with exchange rates in year 1999, whatever date is passed ! Format date is DD/MM/YYYY. <br> <p></p> You can find a sample of execution of these three services on <a href="/dev/clientwsdl.php">a PHP page on http://fxtop.com/dev/clientwsdl.php</a>, this page is not necessarily hosted on our server to work.             <tr>
             <td align="left"> SOAP stands for Simple Object Access Protocol, is a protocol specification for exchanging structured information in the implementation of Web Services in computer networks. It relies on Extensible Markup Language (XML) for its message format, and usually relies on other Application Layer protocols, most notably Hypertext Transfer Protocol (HTTP), for message negotiation and transmission. <p></p> The Web Services Description Language (WSDL) is an XML-based language that is used for describing the functionality offered by a Web service. <p></p> Representational state transfer (REST) is a style of software architecture for distributed systems such as the World Wide Web. REST has emerged over the past few years as a predominant Web service design model. REST has increasingly displaced other design models such as SOAP and WSDL due to its simpler style. <br> REST uses HTTP protocol to exchange messages and facilitates the transaction between web servers by allowing loose coupling between different services. REST is less strongly typed than its counterpart, SOAP. The REST language is based off the use of nouns and verbs, and has an emphasis on readability. Unlike SOAP, REST does not require XML parsing and doesn't require a message header to and from a service provider. This ultimately uses less bandwidth. <p></p> Please note that Ajax means "Asynchronous JavaScript and XML" is a set of technologies to perform Web 2.0 dynamic and interactive content (part of the page is refreshed without having to reload all page). <br>. Ajax is based on following technologies : <br> - JavaScript (client side programming language), so it means that some processing are done on browser. <br> - XMLHttpRequest : a JavaScript object to deal with asynchronous queries<br> - Document Object Model (DOM)<br> - XML (eXtended Markup Language) used to format messages that are exchanged between client and server <br> - XHTML content (language used to describe the content of web page)<br> - Cascade Style Sheet (CSS) <p></p> JQuery is a cross-browser JavaScript library designed to simplify the client-side scripting of HTML. <p></p> SSL is Secured Socket Layer, it is a protocol to securely exchange data over the Internet (data is encrypted between client and server for privacy reason), it is used when address starts with "http<strong>s</strong>://". <p></p> GET method is a request method supported by HTTP protocol. With this method, parameters are passed in the address (URI /URL) you can see them in address bar like C1=EUR&amp;A=110 means that C1 (first currency) is EUR and Amount is 110. <p></p> POST method is a request method supported by HTTP protocol, parameters are not sent in address bar like with GET method but are grouped in body of the request message (there is no body part of message with GET method). <p></p>Part of content of this terminology section comes from related articles of Wikipedia, grants to the authors of these articles. <p></p> </td>
