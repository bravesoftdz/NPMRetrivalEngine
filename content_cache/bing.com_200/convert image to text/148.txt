      <h2>Smaller, faster, better: The importance of choosing correctly </h2> 
      <div class="page-content__copy" itemprop="articleBody">  
       <p>We're asked some questions more than others at BFO, and one of the most common concerns conversion of a PDF to a bitmap image format - typically TIFF, but sometimes JPEG or another format. This process is called "rasterization" and while it's very easy to do with the "extended plus viewer" version of our <a href="/products/pdf/">PDF Library</a>, it's worth going over in more detail. </p> 
       <h3>Summary for the impatient</h3> 
       <p style="border: 1px dotted gray; padding: 1em; font-style: italic;"> How fast your rasterization runs depends on the resolution of the output bitmap, the compression used and the complexity of the source document, in decreasing order of significance. For text, typically a 200dpi 1-bit TIFF image is the way to go. JPEG is the wrong answer. </p> 
       <p>In order to get small files and fast, acceptable results you need to decide three things: bit depth, format and resolution. File size is dependent on these factors, and speed is dependent on these factors and the contents of the page you're rasterizing: a page with large images in unusual ColorSpaces will slow things down, even if you're rendering to a thumbnail size output. </p> 
       <h3>1. Bit Depth</h3> 
       <p>This is the number of bits required to represent each pixel. Values typically range from 1 up to 8, then 24 and 32. 1bit gives you black and white only, with values up to 8bit giving up to 256 unique colors. 24bit is RGB and 32bit is CMYK (and you can add an optional alpha (transparency) channel to both of these for 32bit or 40bit images, although we won't cover that here). In Java, the bit-depth is determined by the <a href="http://docs.oracle.com/javase/8/docs/api/java/awt/image/ColorModel.html"> java.awt.image.ColorModel</a>. </p> 
       <h3>2. Format</h3> 
       <p>The format affects the compression used. Broadly your choices are: </p> 
       <ul> 
        <li style="margin: 1em 0em;"> <strong>JPEG</strong>- a lossy format, designed for compression of photographs, it's typically 24bit RGB (8bit and 32bit are possible but poorly supported by Java). JPEG is bad choice for anything other than photographs as it adds noise to the image: Here's a JPEG conversion of a PDF containing text, and a blown-up version of the same image (with the contrast increased a bit to better illustrate the point):

             </div> </td> 
            <td> <img alt="" src="HelloUnicodeWorld-zoom.png"> <br> 
             <div class="caption">
              Fig 2. Zoom of JPEG
             </div> </td> 
           </tr> 
          </tbody> 
         </table> </li> 
        <li style="margin: 2em 0pt;"> <strong>PNG</strong>- for text and lineart destined for the web, PNG is an excellent choice. Images can be saved as 24bit RGB or as indexed images of 8bit or less. 
         <table> 
          <tbody>

           </tr> 
           <tr> 
            <td> <img alt="" src="HelloUnicodeWorld-8.png"> 
             <div class="caption">
              Fig 4. 8bit PNG (4.5KB)
             </div> </td> 
            <td>8bit PNG can reproduce 256 colors and is very similar to GIF but with better compression. For grayscale images it's lossless. </td> 
           </tr> 
           <tr> 
            <td> <img alt="" src="HelloUnicodeWorld-4.png"> 
             <div class="caption">
              Fig 5. 4bit PNG (2.5KB)
             </div> </td> 
            <td>4bit PNG can reproduce 16 colors, which is still plenty for a Grayscale image as you can see here. Other bit depths are possible too. </td> 
           </tr> 
          </tbody> 
         </table> </li> 
        <li style="margin: 2em 0pt;"> <strong>TIFF</strong>- almost always the best choice unless you're viewing your images on the web, where it's poorly supported. TIFF is a container format and can contain multiple images with different compression types. The BFO PDF Library can create 1bit Images compressed with CCITT Group 4 compression, 24bit RGB with LZW compression (similar to PNG) or 32bit CMYK with LZW compression. These are the best options available in baseline TIFF for those bit-depths </li> 
       </ul> 
       <h3>3. Resolution</h3> 
       <p>The number of dots-per-inch of the output image is arguably the <strong>single most important factor</strong>that determines the size of the bitmap and the time required to create it. For 1bit images, 200dpi (fax resolution) is typical, and 300dpi gives good results when printed and is usually enough. Lower resolutions may be appropriate for grayscale or color documents, but while they may look OK on screen they'll look blocky when printed. </p> 
       <h3>How to convert a PDF to a TIFF image</h3> 
       <p>If you chose TIFF for your image format, creating the TIFF file is easy: </p> 
       <pre class="brush:java">
PDFParser parser = new PDFParser(pdf);
ColorModel cm = PDFParser.BLACKANDWHITE;
int dpi = 200;
OutputStream out = new FileOutputStream("out.tif");
parser.writeAsTIFF(out, cm, dpi);
out.close();
</pre> 
       <div class="caption">
        Example 1. Creating a multi-page TIFF image 
       </div> 
       <p>This will create a 200dpi 1bit CCITT TIFF image, which is pretty typical for black and white documents. A multi-page TIFF will be created if the PDF has multiple pages, but if you'd prefer to create a single TIFF image for each page: </p> 
       <pre class="brush:java">
PDFParser parser = new PDFParser(pdf);
ColorModel cm = PDFParser.BLACKANDWHITE;
int dpi = 200;
List copy = new ArrayList(pdf.getPages());
for (int i=0;i&lt;pages.size();i++) {
    pdf.getPages().clear();
    pdf.getPages().add(copy.get(i));
    OutputStream out = new FileOutputStream("page"+i+".tif");
    parser.writeAsTIFF(out, cm, dpi);
    out.close();
}
</pre> 
       <div class="caption">
        Example 2. Creating single-page TIFF images 
       </div> 
       <p>The ColorModel determines the type of image created - the <a href="/products/pdf/docs/api/org/faceless/pdf2/PDFParser.html"> PDFParser</a>class has models for RGB, CMYK and two ways of creating a 1bit image: BLACKANDWHITE or getBlackAndWhiteColorModel(). Which of these last two is fastest is <em>theoretically</em>dependent on the JVM and operating system: </p>

       <p>I say "theoretically" because the above table looks pretty clear cut! I'm convinced we got different results last time we benchmarked this. Do check which is fastest on your system yourself. </p> 
       <h3>How to create a PNG or JPEG image?</h3>If you want to create a bitmap other than a TIFF, you need to use the 
       <a href="http://docs.oracle.com/javase/8/docs/api/javax/imageio/package-summary.html"> javax.imageio</a>package added in Java 1.4 (there other approaches, but this is the standard one). Broadly it goes as follows: 
       <pre class="brush:java">
PDFParser parser = new PDFParser(pdf);
ColorModel cm = PDFParser.RGB;
int dpi = 200;
for (int i=0;i&lt;pdf.getNumberOfPages();i++) {
    PagePainter painter = parser.getPagePainter(i);
    BufferedImage image = painter.getImage(dpi, cm);
    ImageIO.write(image, "PNG", new File("page"+i+".png"));
}
</pre> 
       <div class="caption">
        Example 3. Creating PNG images for each page 
       </div> 
       <p>Choosing the colormodel will determine what sort of output you get. The above example will create a 24bit PNG image, but if you wanted to create the image as an 8bit or 4bit grayscale PNG: </p> 
       <pre class="brush:java">
// 8-bit grayscale index
byte[] v = new byte[256];
for (int i=0;i&lt;256;i++) v[i] = (byte)i;
ColorModel cm = new IndexColorModel(8, 256, v, v, v);

// 4-bit grayscale index
byte[] v = new byte[16];
for (int i=0;i&lt;16;i++) v[i] = (byte)Math.max(255, i&lt;&lt;4);
ColorModel cm = new IndexColorModel(4, 16, v, v, v);
</pre> 
       <div class="caption">
        Example 4. ColorModels to create index PNG images 
       </div> 
       <p>You can use the same approach as above but specify "JPEG" as the format to <code>ImageIO.write</code>to create JPEG images, and other formats are possible too. If you want to use an indexed PNG image that's not grayscale you can either choose the colors yourself in advance, or you can "quantize" the image down from 24bit to 8bit or less. This requires an external package such as the one supplied by <a href="http://www.gif4j.com/java-color-quantizer.htm">GIF4J</a>or the public domain <a href="http://rsbweb.nih.gov/ij/">ImageJ</a>projects (here's the <a href="http://rsbweb.nih.gov/ij/source/ij/process/MedianCut.java"> source code</a>for their quanitizer). </p>
