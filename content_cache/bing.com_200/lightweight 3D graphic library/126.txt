         <h1 class="entry-title main_title">Download OGRE</h1>
         <div class="entry-content">
          <p><strong>Note:</strong> Checkout <strong><a href="http://www.ogre3d.org/about/what-version-to-choose">What version to choose?</a></strong> if youre confused which version of Ogre to download. We have several branches concurrently in active development, although our long term future is towards version 2.1.</p>
          <h2>Source</h2>
          <p>Pull the latest version from <a href="https://bitbucket.org/sinbad/ogre">source on BitBucket</a>&nbsp;or <a href="https://github.com/OGRECave/ogre">Github</a>.</p>
          <p><em>default</em>/ <em>master</em> branch contains the latest Ogre 1.10 version. The branch <em>v2-1</em> contains our latest Ogre 2.1 version.</p>
          <ul>
           <li>Checkout our <a href="http://www.ogre3d.org/tikiwiki/CMake+Quick+Start+Guide">Quick Start Guide</a> when compiling from source.</li>
           <li>Note for 2.1 users: <a href="http://ogre3d.org/forums/viewtopic.php?f=25&amp;t=84348">Pass by our forum</a> for additional information on compiling the 2.1 samples.</li>
          </ul>
          <h2>Pre-built SDK</h2>
          <p>Here you can download prebuilt SDKs for a range of environments. These are completely self-contained development kits so you can start using OGRE very quickly. Please make sure you download the correct SDK for your platform &amp; compiler since SDKs are not transferable between them.</p>
          <p>Once you have downloaded, if you are new to using OGRE you can find information in the wiki about <a href="../../tikiwiki/tiki-index.php?page=Installing+the+Ogre+SDK">installing and using SDKs</a>, and a <a href="https://ogrecave.github.io/ogre/api/1.10/tutorials.html">series of tutorials</a>. Remember, these include already built libraries and samples. No need to try to build again.</p>
          <p><strong>Note</strong> Some SDKs do not include debug libraries due to their size. If you need them you will have to download and build the source code.</p>
          <p>&nbsp;</p>
          <h3>Current Stable Release (Ogre 1.10)</h3>
          <p><em>Will be added as they become packed.</em> <em>In the meantime please grab the source directly from our <a href="https://github.com/OGRECave/ogre/releases">Ogre 1.10 repository</a>.</em><strong><span style="text-decoration: underline;"><br> </span></strong></p>
          <p>&nbsp;</p>
          <h3>Unofficial pre-built SDK</h3>
          <p>Our community member &lt;<em><a href="http://www.ogre3d.org/forums/memberlist.php?mode=viewprofile&amp;u=39857">Transporter</a></em>&gt; keeps a handy list of up-to-date SDK builds of the latest 1.9, 1.10, 2.0 and 2.1 builds (as well as builds of various Ogre related tools).<br> Visit <a href="http://ogre3d.org/forums/viewtopic.php?t=69274">his forum post</a> to get them.</p>

          <h2>Language bindings</h2>
          <p>You might also like to use <a href="http://www.ogre3d.org/tikiwiki/tiki-index.php?page=Alternative+Languages" target="_blank">other languages</a> to access Ogre, some of the most popular ones are:</p>
          <ul>
           <li><a href="http://www.ogre3d.org/tikiwiki/tiki-index.php?page=Python-Ogre" target="_blank">Python-Ogre</a>  Python wrapper</li>
           <li><a href="http://www.ogre3d.org/tikiwiki/tiki-index.php?page=MOGRE" target="_blank">MOGRE</a>  .Net wrapper</li>
           <li><a href="http://ogre4j.sourceforge.net/" target="_blank">Ogre4J</a>  Java wrapper</li>
          </ul>
          <p>These are unofficial community projects but regularly update their wrappers so are worth a look if you use those environments.</p>
