              <p>If you were using JGraph, you should give a try to <a href="http://jgrapht.org/" rel="noreferrer">JGraphT</a> which is designed for algorithms. One of its features is visualization using the JGraph library. It's still developed, but pretty stable. I analyzed the complexity of JGraphT algorithms some time ago. Some of them aren't the quickest, but if you're going to implement them on your own and need to display your graph, then it might be the best choice. I really liked using its API, when I quickly had to write an app that was working on graph and displaying it later.</p>

              <pre><code>UndirectedGraph&lt;String, DefaultEdge&gt; g =
        new SimpleGraph&lt;String, DefaultEdge&gt;(DefaultEdge.class);

    String v1 = "v1";
    String v2 = "v2";
    String v3 = "v3";
    String v4 = "v4";

    // add the vertices
    g.addVertex(v1);
    g.addVertex(v2);
    g.addVertex(v3);
    g.addVertex(v4);

    // add edges to create a circuit
    g.addEdge(v1, v2);
    g.addEdge(v2, v3);
    g.addEdge(v3, v4);
    g.addEdge(v4, v1);
</code></pre>

              <p>Summary:</p> 
              <ul> 
               <li><a href="https://github.com/jgrapht/jgrapht/" rel="nofollow noreferrer">JGraphT</a> if you are more interested in data structures and algorithms.</li> 
               <li><a href="https://www.jgraph.com/jgraph.html" rel="nofollow noreferrer">JGraph</a> if your primary focus is visualization.</li> 
               <li><a href="http://jung.sourceforge.net/" rel="nofollow noreferrer">Jung</a>, <a href="http://www.yworks.com" rel="nofollow noreferrer">yWorks</a>, and <a href="http://bfo.com/products/graph/" rel="nofollow noreferrer">BFG</a> are other things people tried using.</li> 
               <li><a href="https://github.com/prefuse/Prefuse" rel="nofollow noreferrer">Prefuse</a> is a no no since one has to rewrite most of it.</li> 
               <li><a href="https://github.com/google/guava/wiki/GraphsExplained" rel="nofollow noreferrer">Google Guava</a> if you need good datastructures only.</li> 
               <li><a href="http://commons.apache.org/sandbox/commons-graph/" rel="nofollow noreferrer">Apache Commons Graph</a>. Currently dormant, but provides implementations for many algorithms. See <a href="https://issues.apache.org/jira/browse/SANDBOX-458" rel="nofollow noreferrer">https://issues.apache.org/jira/browse/SANDBOX-458</a> for a list of implemented algorithms, also compared with Jung, GraphT, Prefuse, <a href="https://github.com/jbpt/codebase" rel="nofollow noreferrer">jBPT</a></li> 
              </ul>

             <div class="post-text" itemprop="text"> 
              <p>Apache Commons offers <a href="http://commons.apache.org/sandbox/commons-graph/" rel="nofollow noreferrer">commons-graph</a>. Under <a href="http://svn.apache.org/viewvc/commons/sandbox/graph/trunk/" rel="nofollow noreferrer">http://svn.apache.org/viewvc/commons/sandbox/graph/trunk/</a> one can inspect the source. Sample API usage is <a href="http://svn.apache.org/viewvc/commons/sandbox/graph/trunk/src/test/java/org/apache/commons/graph/model/BaseMutableGraphTestCase.java?view=markup" rel="nofollow noreferrer">in the SVN</a>, too. See <a href="https://issues.apache.org/jira/browse/SANDBOX-458" rel="nofollow noreferrer">https://issues.apache.org/jira/browse/SANDBOX-458</a> for a list of implemented algorithms, also compared with Jung, GraphT, Prefuse, <a href="https://github.com/jbpt/codebase" rel="nofollow noreferrer">jBPT</a></p> 
              <p><a href="https://github.com/google/guava/wiki/GraphsExplained" rel="nofollow noreferrer">Google Guava</a> if you need good datastructures only.</p> 
              <p><a href="https://github.com/jgrapht/jgrapht" rel="nofollow noreferrer">JGraphT</a> is a graph library with many Algorithms implemented and having (in my oppinion) a good graph model. <a href="https://github.com/jgrapht/jgrapht/wiki/HelloWorld" rel="nofollow noreferrer">Helloworld Example</a>. License: LGPL+EPL.</p> 
              <p><a href="http://jung.sourceforge.net" rel="nofollow noreferrer">JUNG2</a> is also a BSD-licensed library with the data structure similar to JGraphT. It offers layouting algorithms, which are currently missing in JGraphT. The most recent commit is from 2010 and packages <code>hep.aida.*</code> are LGPL (via the <a href="http://acs.lbl.gov/software/colt/license.html" rel="nofollow noreferrer">colt library</a>, which <a href="http://jung.sourceforge.net/download.html" rel="nofollow noreferrer">is imported by JUNG</a>). This prevents JUNG from being used in projects under the umbrella of ASF and ESF. Maybe one should use the <a href="http://github.com/rortian/jung2" rel="nofollow noreferrer">github fork</a> and remove that dependency. <a href="https://github.com/rortian/jung2/commit/f4ca0cdcd3312589cbb48de7350b84cbff6067b9" rel="nofollow noreferrer">Commit f4ca0cd</a> is mirroring the last CVS commit. The current commits seem to remove visualization functionality. <a href="https://github.com/rortian/jung2/commit/d0fb491c0b5d2156e12df3a344521cfe7581c3d8" rel="nofollow noreferrer">Commit d0fb491c</a> adds a <code>.gitignore</code>.</p> 
              <p><a href="https://github.com/prefuse/Prefuse" rel="nofollow noreferrer">Prefuse</a> stores the graphs using a matrix structure, which is not memory efficient for sparse graphs. License: BSD</p> 
              <p><a href="http://www.eclipse.org/gef/zest/" rel="nofollow noreferrer">Eclipse Zest</a> has built in graph layout algorithms, which can be used independently of SWT. See <a href="http://git.eclipse.org/c/gef/org.eclipse.zest.git/tree/org.eclipse.zest.layouts/src/org/eclipse/zest/layouts/algorithms" rel="nofollow noreferrer">org.eclipse.zest.layouts.algorithms</a>. The graph structure used is the one of <a href="http://www.eclipse.org/gef/draw2d/" rel="nofollow noreferrer">Eclipse Draw2d</a>, where <a href="http://help.eclipse.org/galileo/index.jsp?topic=/org.eclipse.draw2d.doc.isv/reference/api/org/eclipse/draw2d/graph/DirectedGraph.html" rel="nofollow noreferrer">Nodes are explicit objects</a> and not injected via Generics (as it happens in Apache Commons Graph, JGraphT, and JUNG2).</p>

             <div class="post-text" itemprop="text"> 
              <p>It's also good to be convinced that a Graph can be represented as simply as : </p> 
              <pre><code>class Node {
   int value;
   List&lt;Node&gt; adj;
}
</code></pre> 
              <p>and implement most the algorithms you find interesting by yourself. If you fall on this question in the middle of some practice/learning session on graphs, that's the best lib to consider. ;)</p> 
              <p>You can also prefer adjacency matrix for most common algorithms : </p> 
              <pre><code>class SparseGraph {
  int[] nodeValues;
  List&lt;Integer&gt;[] edges;     
}
</code></pre> 
              <p>or a matrix for some operations : </p> 
              <pre><code>class DenseGraph {
  int[] nodeValues;
  int[][] edges;     
}
</code></pre>
