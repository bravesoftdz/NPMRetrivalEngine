      <div class="content"> 
       <p><a name="introduction"></a> The 3-D functionality in Windows Presentation Foundation (WPF) enables developers to draw, transform, and animate 3-D graphics in both markup and procedural code. Developers can combine 2-D and 3-D graphics to create rich controls, provide complex illustrations of data, or enhance the user experience of an application's interface. 3-D support in WPF is not designed to provide a full-featured game-development platform. This topic provides an overview of 3-D functionality in the WPF graphics system. </p> 
       <p><a name="threed_in_2d"></a> </p> 
       <h2 id="3-d-in-a-2-d-container">3-D in a 2-D Container</h2> 
       <p> 3-D graphics content in WPF is encapsulated in an element, <a class="xref" href="../../../api/system.windows.controls.viewport3d" data-linktype="relative-path">Viewport3D</a>, that can participate in the two-dimensional element structure. The graphics system treats <a class="xref" href="../../../api/system.windows.controls.viewport3d" data-linktype="relative-path">Viewport3D</a> as a two-dimensional visual element like many others in WPF. <a class="xref" href="../../../api/system.windows.controls.viewport3d" data-linktype="relative-path">Viewport3D</a> functions as a window—a viewport—into a three-dimensional scene. More accurately, it is a surface on which a 3-D scene is projected. </p> 
       <p> In a conventional 2-D application, use <a class="xref" href="../../../api/system.windows.controls.viewport3d" data-linktype="relative-path">Viewport3D</a> as you would another container element like Grid or Canvas. Although you can use <a class="xref" href="../../../api/system.windows.controls.viewport3d" data-linktype="relative-path">Viewport3D</a> with other 2-D drawing objects in the same scene graph, you cannot interpenetrate 2-D and 3-D objects within a <a class="xref" href="../../../api/system.windows.controls.viewport3d" data-linktype="relative-path">Viewport3D</a>. This topic will focus on how to draw 3-D graphics inside the <a class="xref" href="../../../api/system.windows.controls.viewport3d" data-linktype="relative-path">Viewport3D</a>. </p> 
       <p><a name="coord_space"></a> </p> 
       <h2 id="3-d-coordinate-space">3-D Coordinate Space</h2> 
       <p> The WPF coordinate system for 2-D graphics locates the origin in the upper left of the rendering area (typically the screen). In the 2-D system, positive x-axis values proceed to the right and positive y-axis values proceed downward. In the 3-D coordinate system, however, the origin is located in the center of the rendering area, with positive x-axis values proceeding to the right but positive y-axis values proceeding upward instead, and positive z-axis values proceeding outward from the origin, toward the viewer. </p> 
       <p> <img src="media/coordsystem-1.png" alt="Coordinate systems" title="CoordSystem-1" data-linktype="relative-path"><br>Conventional 2-D and 3-D coordinate system representations </p> 
       <p> The space defined by these axes is the stationary frame of reference for 3-D objects in WPF. As you build models in this space and create lights and cameras to view them, it's helpful to distinguish this stationary frame of reference, or "world space," from the local frame of reference you create for each model when you apply transformations to it. Remember also that objects in world space might look entirely different, or not be visible at all, depending on light and camera settings, but the position of the camera does not change the location of objects in world space. </p> 
       <p><a name="cameras"></a> </p> 
       <h2 id="cameras-and-projections">Cameras and Projections</h2> 
       <p> Developers who work in 2-D are accustomed to positioning drawing primitives on a two-dimensional screen. When you create a 3-D scene, it's important to remember that you are really creating a 2-D representation of 3-D objects. Because a 3-D scene looks different depending on the onlooker's point of view, you must specify that point of view. The <a class="xref" href="../../../api/system.windows.media.media3d.camera" data-linktype="relative-path">Camera</a> class allows you to specify this point of view for a 3-D scene. </p> 
       <p> Another way to understand how a 3-D scene is represented on a 2-D surface is by describing the scene as a projection onto the viewing surface. The <a class="xref" href="../../../api/system.windows.media.media3d.projectioncamera" data-linktype="relative-path">ProjectionCamera</a> allows you to specify different projections and their properties to change how the onlooker sees 3-D models. A <a class="xref" href="../../../api/system.windows.media.media3d.perspectivecamera" data-linktype="relative-path">PerspectiveCamera</a> specifies a projection that foreshortens the scene. In other words, the <a class="xref" href="../../../api/system.windows.media.media3d.perspectivecamera" data-linktype="relative-path">PerspectiveCamera</a> provides vanishing-point perspective. You can specify the position of the camera in the coordinate space of the scene, the direction and field of view for the camera, and a vector that defines the direction of "up" in the scene. The following diagram illustrates the <a class="xref" href="../../../api/system.windows.media.media3d.perspectivecamera" data-linktype="relative-path">PerspectiveCamera</a>'s projection. </p> 
       <p> The <a class="xref" href="../../../api/system.windows.media.media3d.projectioncamera.nearplanedistance" data-linktype="relative-path">NearPlaneDistance</a> and <a class="xref" href="../../../api/system.windows.media.media3d.projectioncamera.farplanedistance" data-linktype="relative-path">FarPlaneDistance</a> properties of <a class="xref" href="../../../api/system.windows.media.media3d.projectioncamera" data-linktype="relative-path">ProjectionCamera</a> limit the range of the camera's projection. Because cameras can be located anywhere in the scene, it's possible for the camera to be actually positioned inside a model or very near a model, making it hard to distinguish objects properly. <a class="xref" href="../../../api/system.windows.media.media3d.projectioncamera.nearplanedistance" data-linktype="relative-path">NearPlaneDistance</a> allows you to specify a minimum distance from the camera beyond which objects will not be drawn. Conversely, <a class="xref" href="../../../api/system.windows.media.media3d.projectioncamera.farplanedistance" data-linktype="relative-path">FarPlaneDistance</a> lets you specify a distance from the camera beyond which objects will not be drawn, which ensures that objects too far away to be recognizable won't be included in the scene. </p> 
       <p> <img src="media/coordsystem-6.png" alt="Camera setup" title="CoordSystem-6" data-linktype="relative-path"><br>Camera position </p> 
       <p> <a class="xref" href="../../../api/system.windows.media.media3d.orthographiccamera" data-linktype="relative-path">OrthographicCamera</a> specifies an orthogonal projection of a 3-D model to a 2-D visual surface. Like other cameras, it specifies a position, viewing direction, and "upward" direction. Unlike <a class="xref" href="../../../api/system.windows.media.media3d.perspectivecamera" data-linktype="relative-path">PerspectiveCamera</a>, however, <a class="xref" href="../../../api/system.windows.media.media3d.orthographiccamera" data-linktype="relative-path">OrthographicCamera</a> describes a projection that does not include perspective foreshortening. In other words, <a class="xref" href="../../../api/system.windows.media.media3d.orthographiccamera" data-linktype="relative-path">OrthographicCamera</a> describes a viewing box whose sides are parallel, instead of one whose sides meet in a point at the camera. The following image shows the same model as viewed using <a class="xref" href="../../../api/system.windows.media.media3d.perspectivecamera" data-linktype="relative-path">PerspectiveCamera</a> and <a class="xref" href="../../../api/system.windows.media.media3d.orthographiccamera" data-linktype="relative-path">OrthographicCamera</a>. </p> 
       <p> <img src="media/camera-projections4.png" alt="Orthographic and perspective projection" title="Camera_projections4" data-linktype="relative-path"><br>Perspective and Orthographic Projections </p> 
       <p> The following code shows some typical camera settings. </p>

       <p><a name="models_meshes"></a> </p> 
       <h2 id="model-and-mesh-primitives">Model and Mesh Primitives</h2> 
       <p> <a class="xref" href="../../../api/system.windows.media.media3d.model3d" data-linktype="relative-path">Model3D</a> is the abstract base class that represents a generic 3-D object. To build a 3-D scene, you need some objects to view, and the objects that make up the scene graph derive from <a class="xref" href="../../../api/system.windows.media.media3d.model3d" data-linktype="relative-path">Model3D</a>. Currently, the WPF supports modeling geometries with <a class="xref" href="../../../api/system.windows.media.media3d.geometrymodel3d" data-linktype="relative-path">GeometryModel3D</a>. The <a class="xref" href="../../../api/system.windows.media.media3d.geometrymodel3d.geometry" data-linktype="relative-path">Geometry</a> property of this model takes a mesh primitive. </p> 
       <p> To build a model, begin by building a primitive, or mesh. A 3-D primitive is a collection of vertices that form a single 3-D entity. Most 3-D systems provide primitives modeled on the simplest closed figure: a triangle defined by three vertices. Because the three points of a triangle are coplanar, you can continue adding triangles in order to model more complex shapes, called meshes. </p> 
       <p> The WPF 3-D system currently provides the <a class="xref" href="../../../api/system.windows.media.media3d.meshgeometry3d" data-linktype="relative-path">MeshGeometry3D</a> class, which allows you to specify any geometry; it does not currently support predefined 3-D primitives like spheres and cubic forms. Begin creating a <a class="xref" href="../../../api/system.windows.media.media3d.meshgeometry3d" data-linktype="relative-path">MeshGeometry3D</a> by specifying a list of triangle vertices as its <a class="xref" href="../../../api/system.windows.media.media3d.meshgeometry3d.positions" data-linktype="relative-path">Positions</a> property. Each vertex is specified as a <a class="xref" href="../../../api/system.windows.media.media3d.point3d" data-linktype="relative-path">Point3D</a>. (In Extensible Application Markup Language (XAML), specify this property as a list of numbers grouped in threes that represent the coordinates of each vertex.) Depending on its geometry, your mesh might be composed of many triangles, some of which share the same corners (vertices). To draw the mesh correctly, the WPF needs information about which vertices are shared by which triangles. You provide this information by specifying a list of triangle indices with the <a class="xref" href="../../../api/system.windows.media.media3d.meshgeometry3d.triangleindices" data-linktype="relative-path">TriangleIndices</a> property. This list specifies the order in which the points specified in the <a class="xref" href="../../../api/system.windows.media.media3d.meshgeometry3d.positions" data-linktype="relative-path">Positions</a> list will determine a triangle. </p> 
       <pre><code class="lang-xaml" data-src="https://github.com/dotnet/docs/blob/live/samples/snippets/xaml/VS_Snippets_Wpf/Basic3D/XAML/Window1.xaml" name="basic3d#Basic3DXAML3DN3">&lt;GeometryModel3D&gt;

       <p> In the preceding example, the <a class="xref" href="../../../api/system.windows.media.media3d.meshgeometry3d.positions" data-linktype="relative-path">Positions</a> list specifies eight vertices to define a cube-shaped mesh. The <a class="xref" href="../../../api/system.windows.media.media3d.meshgeometry3d.triangleindices" data-linktype="relative-path">TriangleIndices</a> property specifies a list of twelve groups of three indices. Each number in the list refers to an offset into the <a class="xref" href="../../../api/system.windows.media.media3d.meshgeometry3d.positions" data-linktype="relative-path">Positions</a> list. For example, the first three vertices specified by the <a class="xref" href="../../../api/system.windows.media.media3d.meshgeometry3d.positions" data-linktype="relative-path">Positions</a> list are (1,1,0), (0,1,0), and (0,0,0). The first three indices specified by the <a class="xref" href="../../../api/system.windows.media.media3d.meshgeometry3d.triangleindices" data-linktype="relative-path">TriangleIndices</a> list are 0, 2, and 1, which correspond to the first, third, and second points in the <a class="xref" href="../../../api/system.windows.media.media3d.meshgeometry3d.positions" data-linktype="relative-path">Positions</a> list. As a result, the first triangle that makes up the cube model will be composed from (1,1,0) to (0,1,0) to (0,0,0), and the remaining eleven triangles will be determined similarly. </p> 
       <p> You can continue defining the model by specifying values for the <a class="xref" href="../../../api/system.windows.media.media3d.meshgeometry3d.normals" data-linktype="relative-path">Normals</a> and <a class="xref" href="../../../api/system.windows.media.media3d.meshgeometry3d.texturecoordinates" data-linktype="relative-path">TextureCoordinates</a> properties. To render the surface of the model, the graphics system needs information about which direction the surface is facing at any given triangle. It uses this information to make lighting calculations for the model: surfaces that face directly towards a light source appear brighter than those angled away from the light. Though the WPF can determine default normal vectors by using the position coordinates, you can also specify different normal vectors to approximate the appearance of curved surfaces. </p> 
       <p> The <a class="xref" href="../../../api/system.windows.media.media3d.meshgeometry3d.texturecoordinates" data-linktype="relative-path">TextureCoordinates</a> property specifies a collection of <a class="xref" href="../../../api/system.windows.point" data-linktype="relative-path">Point</a>s that tell the graphics system how to map the coordinates that determine how a texture is drawn to the vertices of the mesh. <a class="xref" href="../../../api/system.windows.media.media3d.meshgeometry3d.texturecoordinates" data-linktype="relative-path">TextureCoordinates</a> are specified as a value between zero and 1, inclusive. As with the <a class="xref" href="../../../api/system.windows.media.media3d.meshgeometry3d.normals" data-linktype="relative-path">Normals</a> property, the graphics system can calculate default texture coordinates, but you might choose to set different texture coordinates to control the mapping of a texture that includes part of a repeating pattern, for example. More information about texture coordinates can be found in subsequent topics or in the Managed Direct3D SDK. </p>

       <p><a name="materials"></a> </p> 
       <h2 id="applying-materials-to-the-model">Applying Materials to the Model</h2> 
       <p> For a mesh to look like a three-dimensional object, it must have an applied texture to cover the surface defined by its vertices and triangles so it can be lit and projected by the camera. In 2-D, you use the <a class="xref" href="../../../api/system.windows.media.brush" data-linktype="relative-path">Brush</a> class to apply colors, patterns, gradients, or other visual content to areas of the screen. The appearance of 3-D objects, however, is a function of the lighting model, not just of the color or pattern applied to them. Real-world objects reflect light differently depending on the quality of their surfaces: glossy and shiny surfaces don't look the same as rough or matte surfaces, and some objects seem to absorb light while others glow. You can apply all the same brushes to 3-D objects that you can apply to 2-D objects, but you can't apply them directly. </p> 
       <p> To define the characteristics of a model's surface, WPF uses the <a class="xref" href="../../../api/system.windows.media.media3d.material" data-linktype="relative-path">Material</a> abstract class. The concrete subclasses of Material determine some of the appearance characteristics of the model's surface, and each also provides a Brush property to which you can pass a SolidColorBrush, TileBrush, or VisualBrush. </p> 
       <ul> 
        <li><p><a class="xref" href="../../../api/system.windows.media.media3d.diffusematerial" data-linktype="relative-path">DiffuseMaterial</a> specifies that the brush will be applied to the model as though that model were lit diffusely. Using DiffuseMaterial most resembles using brushes directly on 2-D models; model surfaces do not reflect light as though shiny. </p> </li> 
        <li><p><a class="xref" href="../../../api/system.windows.media.media3d.specularmaterial" data-linktype="relative-path">SpecularMaterial</a> specifies that the brush will be applied to the model as though the model's surface were hard or shiny, capable of reflecting highlights. You can set the degree to which the texture will suggest this reflective quality, or "shine," by specifying a value for the <a class="xref" href="../../../api/system.windows.media.media3d.specularmaterial.specularpower" data-linktype="relative-path">SpecularPower</a> property. </p> </li> 
        <li><p><a class="xref" href="../../../api/system.windows.media.media3d.emissivematerial" data-linktype="relative-path">EmissiveMaterial</a> allows you to specify that the texture will be applied as though the model were emitting light equal to the color of the brush. This does not make the model a light; however, it will participate differently in shadowing than it would if textured with DiffuseMaterial or SpecularMaterial. </p> </li> 
       </ul> 
       <p> For better performance, the backfaces of a <a class="xref" href="../../../api/system.windows.media.media3d.geometrymodel3d" data-linktype="relative-path">GeometryModel3D</a> (those faces that are out of view because they are on the opposite side of the model from the camera) are culled from the scene. To specify a <a class="xref" href="../../../api/system.windows.media.media3d.material" data-linktype="relative-path">Material</a> to apply to the backface of a model like a plane, set the model's <a class="xref" href="../../../api/system.windows.media.media3d.geometrymodel3d.backmaterial" data-linktype="relative-path">BackMaterial</a> property. </p> 
       <p> To achieve some surface qualities, like glowing or reflective effects, you might want to apply several different brushes to a model in succession. You can apply and reuse multiple Materials by using the <a class="xref" href="../../../api/system.windows.media.media3d.materialgroup" data-linktype="relative-path">MaterialGroup</a> class. The children of the MaterialGroup are applied first to last in multiple rendering passes. </p>

       <p><a name="lights"></a> </p> 
       <h2 id="illuminating-the-scene">Illuminating the Scene</h2> 
       <p> Lights in 3-D graphics do what lights do in the real world: they make surfaces visible. More to the point, lights determine what part of a scene will be included in the projection. Light objects in WPF create a variety of light and shadow effects and are modeled after the behavior of various real-world lights. You must include at least one light in your scene, or no models will be visible. </p> 
       <p> The following lights derive from the base class <a class="xref" href="../../../api/system.windows.media.media3d.light" data-linktype="relative-path">Light</a>: </p> 
       <ul> 
        <li><p><a class="xref" href="../../../api/system.windows.media.media3d.ambientlight" data-linktype="relative-path">AmbientLight</a>: Provides ambient lighting that illuminates all objects uniformly regardless of their location or orientation. </p> </li> 
        <li><p><a class="xref" href="../../../api/system.windows.media.media3d.directionallight" data-linktype="relative-path">DirectionalLight</a>: Illuminates like a distant light source. Directional lights have a <a class="xref" href="../../../api/system.windows.media.media3d.directionallight.direction" data-linktype="relative-path">Direction</a> specified as a Vector3D, but no specified location. </p> </li> 
        <li><p><a class="xref" href="../../../api/system.windows.media.media3d.pointlight" data-linktype="relative-path">PointLight</a>: Illuminates like a nearby light source. PointLights have a position and cast light from that position. Objects in the scene are illuminated depending on their position and distance with respect to the light. <a class="xref" href="../../../api/system.windows.media.media3d.pointlightbase" data-linktype="relative-path">PointLightBase</a> exposes a <a class="xref" href="../../../api/system.windows.media.media3d.pointlightbase.range" data-linktype="relative-path">Range</a> property, which determines a distance beyond which models will not be illuminated by the light. PointLight also exposes attenuation properties which determine how the light's intensity diminishes over distance. You can specify constant, linear, or quadratic interpolations for the light's attenuation. </p> </li> 
        <li><p><a class="xref" href="../../../api/system.windows.media.media3d.spotlight" data-linktype="relative-path">SpotLight</a>: Inherits from <a class="xref" href="../../../api/system.windows.media.media3d.pointlight" data-linktype="relative-path">PointLight</a>. Spotlights illuminate like PointLight and have both position and direction. They project light in a cone-shaped area set by <a class="xref" href="../../../api/system.windows.media.media3d.spotlight.innerconeangle" data-linktype="relative-path">InnerConeAngle</a> and <a class="xref" href="../../../api/system.windows.media.media3d.spotlight.outerconeangle" data-linktype="relative-path">OuterConeAngle</a> properties, specified in degrees. </p> </li> 
       </ul> 
       <p> Lights are <a class="xref" href="../../../api/system.windows.media.media3d.model3d" data-linktype="relative-path">Model3D</a> objects, so you can transform and animate light properties, including position, color, direction, and range. </p>

       <p><a name="transforms"></a> </p> 
       <h2 id="transforming-models">Transforming Models</h2> 
       <p> When you create models, they have a particular location in the scene. To move those models around in the scene, to rotate them, or to change their size, it's not practical to change the vertices that define the models themselves. Instead, just as in 2-D, you apply transformations to models. </p> 
       <p> Each model object has a <a class="xref" href="../../../api/system.windows.media.media3d.model3d.transform" data-linktype="relative-path">Transform</a> property with which you can move, re-orient, or resize the model. When you apply a transform, you effectively offset all the points of the model by whatever vector or value specified by the transform. In other words, you've transformed the coordinate space in which the model is defined ("model space"), but you haven't changed the values that make up the model's geometry in the coordinate system of the entire scene ("world space"). </p> 
       <p> For more information about transforming models, see <a href="3-d-transformations-overview" data-linktype="relative-path">3-D Transformations Overview</a>. </p> 
       <p><a name="animations"></a> </p> 
       <h2 id="animating-models">Animating Models</h2> 
       <p> The WPF 3-D implementation participates in the same timing and animation system as 2-D graphics. In other words, to animate a 3-D scene, animate the properties of its models. It's possible to animate properties of primitives directly, but it's typically easier to animate transformations that change the position or appearance of models. Because transformations can be applied to <a class="xref" href="../../../api/system.windows.media.media3d.model3dgroup" data-linktype="relative-path">Model3DGroup</a> objects as well as individual models, it's possible to apply one set of animations to a child of a Model3DGroup and another set of animations to a group of child objects. You can also achieve a variety of visual effects by animating the properties of your scene's lighting. Finally, you might choose to animate the projection itself by animating the camera position or field of view. For background information on the WPF timing and animation system, see the <a href="animation-overview" data-linktype="relative-path">Animation Overview</a>, <a href="storyboards-overview" data-linktype="relative-path">Storyboards Overview</a>, and <a href="../advanced/freezable-objects-overview" data-linktype="relative-path">Freezable Objects Overview</a> topics. </p> 
       <p> To animate an object in WPF, you create a timeline, define an animation (which is really a change in some property value over time), and specify the property to which to apply the animation. Because all the objects in a 3-D scene are children of <a class="xref" href="../../../api/system.windows.controls.viewport3d" data-linktype="relative-path">Viewport3D</a>, the properties targeted by any animation you want to apply to the scene are properties of properties of Viewport3D. </p>
