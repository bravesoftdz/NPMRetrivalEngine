       <h2 class="entry-title"><a href="https://anujarosha.wordpress.com/2011/07/20/how-to-send-and-receive-sms-in-j2me/" rel="bookmark">How to send and receive SMS in&nbsp;J2ME</a></h2> 
       <div class="entry-meta"> 
        <span class="meta-prep meta-prep-author">Posted on</span> 
        <a href="https://anujarosha.wordpress.com/2011/07/20/how-to-send-and-receive-sms-in-j2me/" title="8:36 pm" rel="bookmark"><span class="entry-date">July 20, 2011</span></a> 
        <span class="by-author"><span class="sep">by</span> <span class="author vcard"><a class="url fn n" href="https://anujarosha.wordpress.com/author/anujarosha/" title="View all posts by AnujAroshA" rel="author">AnujAroshA</a></span> </span> 
       </div>
       <!-- .entry-meta --> 
       <div class="entry-content"> 
        <p style="text-align:justify;">Hi friends…! In my <a title="Sending SMS app in&nbsp;J2ME" href="https://anujarosha.wordpress.com/2011/07/13/sending-sms-app-in-j2me/" target="_blank">previous post</a> I have shown you sending SMS in J2ME. In this post also I am using that sending SMS part again and addition to that I have included receiving SMS part also. I have used&nbsp;separate&nbsp;class and a separate <span style="color:#000000;"><em>Thread</em></span> to implement sending SMS functionality. Receiving SMS function I included with in the same class (which extends MIDlet class) as a <span style="color:#000000;"><em>Regular Inner Class</em></span> but also in a separate <em>Thread</em>.</p> 
        <p style="text-align:justify;"><span id="more-241"></span></p> 
        <p style="text-align:justify;">As usual let’s take a look what are the new classes and interfaces that I am going to show you today. New classes are,</p> 
        <ul style="text-align:justify;"> 
         <li>ConnectionNotFoundException</li> 
         <li>PushRegistry</li> 
        </ul> 
        <p style="text-align:justify;">New Interfaces are,</p> 
        <ul style="text-align:justify;"> 
         <li>MessageListener</li> 
        </ul> 
        <p style="text-align:justify;">There are several things you should know before you run this code in your <span style="color:#000000;"><em>NetBeans IDE</em></span>. With your common sense you may already figure out that you need to devices. One mobile for send the SMS and other for receive the SMS. There are two ways that you can complete this task. First I will go with the default requirements that I am using. I have mentioned those configuration that I am using in my <a title="Let’s learn J2ME with mobile&nbsp;applications" href="https://anujarosha.wordpress.com/2011/05/03/lets-learn-j2me-with-mobile-applications/" target="_blank">very first blog in J2ME category</a>.</p> 
        <p style="text-align:justify;">If you try to run two instances of same emulator, you will get an error as your output. So what you have to do is first run just one emulator and then change the device in to a different one. You can do that by,</p> 
        <ul style="text-align:justify;"> 
         <li>Right click on the project</li> 
         <li>Select <span style="color:#000000;"><em>Properties</em></span> from the popup menu</li> 
         <li>Select <span style="color:#000000;"><em>Platform</em></span> in <span style="color:#000000;"><em>Category</em></span> pane</li> 
         <li>Choose a different device from the <span style="color:#000000;"><em>Device</em></span> drop down</li> 
         <li>Press <span style="color:#000000;"><em>OK</em></span></li> 
        </ul> 
        <p style="text-align:justify;">Then run the project again and you can see two different devices as in the <em>Figure 1</em>.</p> 
        <div class="mceTemp mceIEcenter" style="text-align:justify;">

        <p style="text-align:justify;">As I have mentioned you earlier I am using two classes. First take a look at the class we design our GUI as well as the class which we have extended MIDlet class.</p> 
        <pre>package anuja;

import java.io.IOException;
import javax.microedition.io.ConnectionNotFoundException;
import javax.microedition.io.Connector;
import javax.microedition.io.PushRegistry;
import javax.microedition.lcdui.Alert;
import javax.microedition.lcdui.AlertType;
import javax.microedition.lcdui.Command;
import javax.microedition.lcdui.CommandListener;
import javax.microedition.lcdui.Display;
import javax.microedition.lcdui.Displayable;
import javax.microedition.lcdui.Form;
import javax.microedition.lcdui.Image;
import javax.microedition.lcdui.List;
import javax.microedition.lcdui.TextBox;
import javax.microedition.lcdui.TextField;
import javax.microedition.midlet.MIDlet;
import javax.wireless.messaging.MessageConnection;
import javax.wireless.messaging.MessageListener;
import javax.wireless.messaging.TextMessage;

/**
 * @author Anuja
 */
public class Messaging extends MIDlet implements CommandListener, MessageListener {
    private List smsList;
    private Display display;
    private Image newMessageImg;
    private Image inboxImg;
    private Command exitCmd;
    private Command openCmd;
    private int getSelInd;
    private Form newMsgForm;
    private TextField phnNoTxtField;
    private TextField msgTxtField;
    private Command sendCmd;
    private Command backCmd;
    private Alert infoAlert;
    private Image alertImg;

    protected String phnNoStr;
    protected String msgStr;

    MessageConnection receiveCon;

    public void startApp() {
        display = Display.getDisplay(this);

        //-------------Select either New message or Inbox-----------------------
        smsList = new List("Messaging", List.IMPLICIT);

        try {
            newMessageImg = Image.createImage("newMessage.png");
            inboxImg = Image.createImage("inbox.png");
            alertImg = Image.createImage("alert.png");
        } catch (IOException ex) {
            ex.printStackTrace();
        }

        smsList.append("New message", newMessageImg);
        smsList.append("Inbox", inboxImg);

        exitCmd = new Command("Exit", Command.EXIT, 7);
        smsList.addCommand(exitCmd);
        openCmd = new Command("Open", Command.OK, 4);
        smsList.addCommand(openCmd);

        smsList.setCommandListener(this);
        display.setCurrent(smsList);

        //----------------create New message interface--------------------------
        newMsgForm = new Form("Text message");
        phnNoTxtField = new TextField("To: ", "", 12, TextField.PHONENUMBER);
        newMsgForm.append(phnNoTxtField);
        msgTxtField = new TextField("Message: ", "", 160, TextField.ANY);
        newMsgForm.append(msgTxtField);

        sendCmd = new Command("Send", Command.OK, 4);
        newMsgForm.addCommand(sendCmd);
        backCmd = new Command("Back", Command.BACK, 2);
        newMsgForm.addCommand(backCmd);
        newMsgForm.addCommand(exitCmd);
        newMsgForm.setCommandListener(this);

        //-------------------------Receive SMS----------------------------------

        long time = System.currentTimeMillis();
        //System.out.println(time);
        //System.out.println(time + 1000 * 60 * 60);

        try {
            // The PushRegistry maintains a list of inbound connections.
            // registerAlarm() register a time to launch the specified application.
            PushRegistry.registerAlarm("anuja.Messaging", time + 1000 * 60 * 60);
        } catch (ClassNotFoundException ex) {
            ex.printStackTrace();
        } catch (ConnectionNotFoundException ex) {
            ex.printStackTrace();
        }

        try {
            // "sms://:50000" is the Connection String that we have to give in the Push Registry
            receiveCon = (MessageConnection) Connector.open("sms://:50000");
        } catch (IOException ex) {
            ex.printStackTrace();
        }

        try {
            receiveCon.setMessageListener(this);
        } catch (IOException ex) {
            ex.printStackTrace();
        }

    }

    public void pauseApp() {
    }

    public void destroyApp(boolean unconditional) {
    }

    public void commandAction(Command c, Displayable d) {
        if(c == exitCmd){
            // Used by an MIDlet to notify the application management software that it has entered into the Destroyed state
            notifyDestroyed();
        }else if(c == openCmd){
            getSelInd = smsList.getSelectedIndex();

            if(getSelInd == 0){
                System.out.println("New Message");
                display.setCurrent(newMsgForm);
            }else{
                System.out.println("Inbox");
            }
        }else if(c == sendCmd){

            phnNoStr = getPhnNoTxtField().getString();
            msgStr = getMsgTxtField().getString();

            new SmsSenderThread(phnNoStr, msgStr).start();

            infoAlert = new Alert("Success...!", "Message sent successfully.", alertImg, AlertType.INFO);
            display.setCurrent(infoAlert, smsList);

        }else if(c == backCmd){
            display.setCurrent(smsList);
        }
    }

    public TextField getPhnNoTxtField() {
        return phnNoTxtField;
    }

    public TextField getMsgTxtField() {
        return msgTxtField;
    }

    public Display getDisplay() {
        return display;
    }

    public void notifyIncomingMessage(MessageConnection mc) {
        new SmsReceiverThread().start();

        System.out.println("Message received");
    }

    // Regular inner class
    private class SmsReceiverThread extends Thread {

        public void run() {
            try {
                //  Receive Message
                // Eventhough receiveCon is an outer class member, we can access that because this is a regular inner class
                TextMessage txtMsg = (TextMessage) receiveCon.receive();

                // Get the receiving SMS phone number
                String senderTpNo = txtMsg.getAddress();

                // Get the receiving SMS message
                String senderMsg = txtMsg.getPayloadText();

                // Create a TextBox and direct the incomming message to that
                TextBox txtBox = new TextBox(senderTpNo, senderMsg, 160, TextField.ANY);

                getDisplay().setCurrent(txtBox);

            } catch (IOException ex) {
                ex.printStackTrace();
            }
        }
    }
}</pre> 
        <p style="text-align:justify;">In the above source code you cannot see the sending SMS part because I have used a separate class for that. Below is the source code of that class.</p> 
        <pre>package anuja;

import java.io.IOException;
import javax.microedition.io.Connector;
import javax.wireless.messaging.MessageConnection;
import javax.wireless.messaging.TextMessage;

/**
 *
 * @author Anuja
 */
public class SmsSenderThread extends Thread {

    private String thisPhnNo;
    private String thisMsg;

    // Get the values from the Messaging midlet class
    public SmsSenderThread(String passPhnNo, String passMsg) {
        this.thisPhnNo = passPhnNo;
        this.thisMsg = passMsg;
    }

    public void run() {
        System.out.println("Running");
        System.out.println(thisPhnNo);
        System.out.println(thisMsg);

        // Creating the connection
        MessageConnection msgCon = null;

        try {
            // Open the connection with a port
            msgCon = (MessageConnection) Connector.open("sms://" + thisPhnNo + ":50000");
        } catch (IOException ex) {
            ex.printStackTrace();
        }

        // Create platform tp put a new text message
        TextMessage txtMsg = (TextMessage) msgCon.newMessage(MessageConnection.TEXT_MESSAGE);

        // Set the aircraft (here the SMS) for which payment is received
        txtMsg.setPayloadText(thisMsg);

        try {
            // Send the SMS
            msgCon.send(txtMsg);
        } catch (IOException ex) {
            ex.printStackTrace();
        } finally{
            // If the connection is open...
            if(msgCon != null){
                try {
                    // Close the connection
                    msgCon.close();
                } catch (IOException ex) {
                    ex.printStackTrace();
                }
            }
        }
    }
}</pre> 
        <p style="text-align:justify;">Yes you have completed the application. But wait…!</p> 
        <p style="text-align:justify;">I have used comments to describe the code. Therefore I am not going to repeat it here. There is one thing that you should know when you are using <span style="color:#000000;">PushRegistry</span> class. We can use <em>PushRegistry</em> in two ways. One is called <span style="color:#000000;"><em>Static</em></span> and other one is called <span style="color:#000000;"><em>Dynamic</em></span>. If we use that only in program code (simply in the source code) we call it <em>Dynamic</em>. To put the <em>PushRegistry</em>, <span style="color:#000000;"><em>Static</em></span> go through the following steps.</p> 
        <ul style="text-align:justify;"> 
         <li>Right click on the project and go to the <span style="color:#000000;"><em>Properties</em></span></li> 
         <li>Select <span style="color:#000000;"><em>Application Descriptor</em></span> from the <span style="color:#000000;"><em>Category</em></span> pane</li> 
         <li>Select <span style="color:#000000;"><em>Push Registry</em></span> tab</li> 
         <li>Click on the <em>Add</em> button</li> 
         <li>Give “sms://:50000” as <span style="color:#000000;"><em>Connection String</em></span></li> 
         <li>Press <span style="color:#000000;"><em>OK</em></span></li> 
        </ul> 
        <p style="text-align:justify;">The number <span style="color:#000000;"><em>50000</em></span> is the port that we are providing to receive SMS. So it can be change. Now comes the biggest problem. If you use <span style="color:#000000;"><em>Static</em></span> way of <span style="color:#000000;"><em>Push Registry</em></span>, your mobile phone will not allow you to install this application. ? So better go with <span style="color:#000000;"><em>Dynamic</em></span> way if you need to install this application in your mobile device.</p>
