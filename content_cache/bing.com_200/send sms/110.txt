       <p> <b>User manual</b> </p> 
       <p> The Ozeki Phone System XE makes it possible to connect the customers and the appropriate extensions together through a fully customizable WebPhone. It can be done by using your PBX and the configurable dial plan rules. For this purpose you only need to install a Webphone outside line in the Ozeki Phone Sytem XE and add the required routing rule. If the default dial plan rule has been added, no need to specify the called number before making a call. In addition, it is also possible to dial a previously installed extension or an outside line number directly by entering its telephone number. <a href="voip-how-to-add-a-webphone-to-your-website-with-ozeki-xe-pbx-724.html">How to setup a Webphone outside line</a> </p> 
       <p> For using WebPhoneAPI a <b>webserver is essentially needed</b> on which your written code will be executable. Any webserver can be used, just select one and download it. In this example <a href="http://www.wampserver.com/en/#download-wrapper">Wampserver</a> and Apache was used. After installing WampServer, you need to modify the 'Listen 80' value in the httpd.conf file belonging to Apache in order to avoid port collision. Rewrite it to 'Listen 8080' (or to any other number that is not used by other applications). Do the modification then restart the service. After that open the folder where you have installed Wamp then select the 'www' folder. Here create a 'WebPhoneAPIExample.html' file with the following content: </p> 
       <div class="ShowHideDiv"> 
        <pre class="syntax brush-html">
&lt;!DOCTYPE html&gt;
&lt;!-- Important: This file is have to be hosted on a web server (e.g. Apache,IIS etc.)--&gt;
&lt;html&gt;
&lt;head&gt;
	&lt;script type="text/javascript"&gt;

	&lt;/script&gt;
&lt;/head&gt;
&lt;body&gt;
&lt;/body&gt;
&lt;/html&gt;
</pre> 
       </div> 
       <span class="CodeExample">Code example 1</span> 
       <span class="CodeExampleName"> - XML codes to using Wampserver</span> 
       <p> The JavaScript code which you will write needs to be placed between &lt;script&gt; &lt;/script&gt; tags. </p> 
       <p> <b>Implementation</b> </p> 
       <p> Before developing JavaScript, reference the WebphoneAPI JavaScript file into your source. Place the following row directly after the opening &lt;head&gt; tag: </p> 
       <div class="ShowHideDiv"> 
        <pre class="syntax brush-html">
&lt;script type="text/javascript" <br>
src="http://ozekixepbx.ip:PBX_ServerPort/WebphoneAPI"&gt;&lt;/script&gt;
</pre> 
       </div> 
       <span class="CodeExample">Code example 2</span> 
       <span class="CodeExampleName"> - XML codes to complete &lt;head&gt; tag</span> 
       <p> In this script the 'ozekixepbx.ip' and the 'PBX_ServerPort' needs to be the server address and port number of the Ozeki Phone System XE configured previously. The default port number is 7777. </p> 
       <p> First check if an online extension is available on the other side of the line. For this purpose, call the following function ensured by OzWebClient. </p> 
       <div class="ShowHideDiv"> 
        <pre class="syntax brush-js">
OzWebClient.checkWebphoneState(serverAddress, webphoneId, webphoneStateReceived);
</pre> 
       </div> 
       <span class="CodeExample">Code example 3</span> 
       <span class="CodeExampleName"> - Function to check extensions</span> 
       <p> Here you need to specify the IP address of the Ozeki Phone System XE and the ID of a Webphone Outside Line installed previously. The third parameter of the function is the name of a subscriber function that looks like this: </p> 
       <div class="ShowHideDiv"> 
        <pre class="syntax brush-js">
function webphoneStateReceived(info) {
   if (!info.isAvailable)
		return;

   OzWebClient.onConnectionStateChanged(connectionStateChanged);
   OzWebClient.connect(serverAddress, webphoneId);
}
</pre> 
       </div> 
       <span class="CodeExample">Code example 4</span> 
       <span class="CodeExampleName"> - Javascript codes to Function which subscribe to check the state</span> 
       <p> The function above shows if an extension is available. Before connecting, you need to subscribe to the events of onConnectionStateChanged then call the OzWebClient.connect method. The subscriber connectionStateChanged function can be seen below: </p> 
       <div class="ShowHideDiv"> 
        <pre class="syntax brush-js">
function connectionStateChanged(info) {
   if (info.State == ConnectionState.ACCESS_GRANTED)
       isConnected = true;
   else
       isConnected = false;
}
</pre> 
       </div> 
       <span class="CodeExample">Code example 5</span> 
       <span class="CodeExampleName"> - Function which subscribe to the connection state change event <p> After you have done the preparations, see how to receive an incoming call and make an outgoing call. </p> <h3>3. Make outgoing voice calls</h3> <p> After connecting to the server, call the pickup() function in order to initiate a call. </p> 
        <div class="ShowHideDiv"> 
         <pre class="syntax brush-js">
function pickUp() {
	if (isConnected) {
		call = OzWebClient.createCall(calledNumber);
		call.onCallStateChanged(callStateChanged);
		call.start();
	}
}	
</pre> 
        </div> <span class="CodeExample">Code example 6</span> <span class="CodeExampleName"> - Function to pick up calls</span> <p> Define the already existing call object with the new object which was given by OzWebClient.createCall(calledNumber). After dialling, the called number or the number which was associated to the webphone (if you used default routing rule) will ring. <br> You can get information about your call in callStateChanged. </p> 
        <div class="ShowHideDiv"> 
         <pre class="syntax brush-js">
function callStateChanged(state){
	console.log("Actual call state is:" + state);
}
</pre> 
        </div> <span class="CodeExample">Code example 7</span> <span class="CodeExampleName"> - Function to get information about actual state of the call</span>
