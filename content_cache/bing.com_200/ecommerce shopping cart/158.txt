        <h1 class="entry-title" itemprop="headline">How To Code an HTML5/CSS3 Shopping Cart Webpage Layout</h1> 
        <p class="entry-meta"><time class="entry-time" itemprop="datePublished" datetime="2013-12-19T07:47:23+00:00">December 19, 2013</time> by <span class="entry-author" itemprop="author" itemscope itemtype="https://schema.org/Person"><a href="http://spyrestudios.com/author/jakerocheleau/" class="entry-author-link" itemprop="url" rel="author"><span class="entry-author-name" itemprop="name">Jake Rocheleau</span></a></span> </p>

        <p class="bigger">I’ve wanted to put together a frontend web tutorial for a few months now. These types of articles provide an excellent point of reference for new designers who are growing accustomed to popular UI/UX trends. Specifically for designers who aren’t sure where to go on their next project, and may need a small boost of inspiration to get there.</p> 
        <p>For this tutorial I put together a small agile eCommerce checkout page. The structure itself is centered and running a fixed width to ensure an even layout regardless of the web browser. There are no jQuery animations or dynamic effects, so the layout itself is meant to be a single page running on CSS2/CSS3. Take a peek at my live demo to get an idea of the final product.</p> 
        <p class="nobrdr"><a href="http://codepen.io/jakestuts/full/qKvew"><img src="http://spyrestudios.com/wp-content/uploads/spyre-shopping-cart-ui-preview.png" class="frame" alt="html5 css3 frontend webdev tutorial preview screenshot"></a></p> 
        <p><strong><a href="http://codepen.io/jakestuts/full/qKvew">Live Demo</a></strong> – <strong><a href="http://cl.ly/2R3M3f1W220v">Download Source Code</a></strong></p> 
        <p><span id="more-16779"></span></p> 
        <h2>Getting Started</h2> 
        <p>I have only included a single external file named <strong>styles.css</strong> in the document header. This will include a list of typical page resets along with common elements for structuring the cart itself. I chose to write this using an HTML table with the top heading and inner rows for shop products.</p> 
        <pre>  &lt;div id="w"&gt;
    &lt;header id="title"&gt;
      &lt;h1&gt;HTML5/CSS3 Shopping Cart UI&lt;/h1&gt;
    &lt;/header&gt;
    &lt;div id="page"&gt;
      &lt;table id="cart"&gt;
        &lt;thead&gt;
          &lt;tr&gt;
            &lt;th class="first"&gt;Photo&lt;/th&gt;
            &lt;th class="second"&gt;Qty&lt;/th&gt;
            &lt;th class="third"&gt;Product&lt;/th&gt;
            &lt;th class="fourth"&gt;Line Total&lt;/th&gt;
            &lt;th class="fifth"&gt;&amp;nbsp;&lt;/th&gt;
          &lt;/tr&gt;
        &lt;/thead&gt;
</pre> 
        <p>Naturally any table you write in HTML5 should utilize a <strong>thead</strong> followed by a table row with individual <strong>th</strong> elements. Every other row in the table should be using <strong>td</strong> cells, but the heading is a special circumstance. Also these headings will provide the spacing outline for how the table is structured – I’ve created a max width of 600px with 5 columns in total.</p> 
        <p>These columns break down into a product photo, the order quantity, the product name, the total cost for that item, and a <a href="http://findicons.com/icon/261449/trash_can?id=397422">trash can icon</a> to remove any items from the cart with a single click. Each row is set to a fixed width using pixels for the width/height properties. Since the layout isn’t responsive I figured this makes the table sit nicer without any excess spacing.</p> 
        <pre>&lt;tbody&gt;
  &lt;!-- shopping cart contents --&gt;
  &lt;tr class="productitm"&gt;
    &lt;!-- http://www.inkydeals.com/deal/ginormous-bundle/ --&gt;
    &lt;td&gt;&lt;img src="images/design-bundle-pack.png" class="thumb"&gt;&lt;/td&gt;
    &lt;td&gt;&lt;input type="number" value="1" min="0" max="99" class="qtyinput"&gt;&lt;/td&gt;
    &lt;td&gt;Design Bundle Package&lt;/td&gt;
    &lt;td&gt;$79.00&lt;/td&gt;
    &lt;td&gt;&lt;span class="remove"&gt;&lt;img src="images/trash.png" alt="X"&gt;&lt;/span&gt;&lt;/td&gt;
  &lt;/tr&gt;
</pre> 
        <p>I also created a number of items in the list, but the code above is just one example of the first product row. Inside the <strong>tbody</strong> element each row represents a product item when using the class <strong>.productitm</strong>. These rows have extra spacing and a border to separate them from each other. Also notice the quantity input uses the numeric type, as opposed to a text field type.</p> 
        <p>In Google Chrome this gives the user small up/down arrows to quickly update values. Other browsers don’t seem to have this native support just yet. But I’m almost positive there are jQuery plugins out there which can replicate this effect if you need full browser support.</p> 
        <h2>Bottom Row Structure</h2> 
        <p>Looking down towards the very bottom of the table you’ll find the last 3 lines are somewhat different. They contain the shipping+tax costs, then finally the grand total along with a checkout button. This button is styled in a way using <a href="http://codepen.io/guvootes/pen/eyDAb">open source CSS3 gradients</a> from a CodePen entry. It’s also using a fixed width and height, but can be manually resized or updated to increase the font size or something similar.</p> 
        <pre style="overflow:scroll;">  &lt;!-- tax + subtotal --&gt;
  &lt;tr class="extracosts"&gt;

    &lt;td&gt;&amp;nbsp;&lt;/td&gt;

&lt;/tbody&gt;
</pre> 
        <p>Since this isn’t a real form I used a <strong>&lt;button&gt;</strong> element in place of a typical submit input. Since these rows do not follow the same structure as the headings I’m including lots of <strong>colspan</strong> attributes on some of the table cells. This will span over a multitude of cells to better organize the prices so that they’re easier to read.</p> 
        <p>I find this to be an elegant solution because everything is fairly clear right off the bat. If you needed to add some extra features it wouldn’t be difficult to widen the layout and create more room for the content… or even create more table columns altogether!</p>
