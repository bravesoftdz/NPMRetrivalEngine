Then this <b>free</b> JavaScript shopping cart is exactly what you need. This script also features the ability to have your products on many pages -- you do not have to have all of your products on one page, like so many of the other solutions. This complete, simple to use shopping cart is easy to implement on almost any site. All that is required is pasting in the source code on every page that you can purchase products from, and on the final checkout page. </p><p> Who uses this shopping cart?        </ul> <h3>Ready to use the script?</h3> Most of us learn by example, so I recommend <a href="samplesite.zip">downloading</a> the fully working <a href="http://www.shop.nopdesign.com" target="_external">sample site</a>, and reading the posts in the <a href="http://www.nopdesign.com/forum" target="_external2">Support Forums</a>. <p> For those looking for a more complete description, there are a few easy steps to follow to run the shopping cart: </p></font>
      <ol>
       <font face="ARIAL,HELVETICA" size="2"> <li> Download the cart here: 
         <ul> 
          <li> <a href="nopcart.zip">Shopping cart script, version 4.4</a> Distributed as a ZIP archive. </li>
         </ul> <p></p></li><li> Uncompress the shopping cart with a zip file decompressor. If you do not have one, head over to <a href="http://www.winzip.com" target="_external">www.winzip.com</a> to get one. <p></p></li><li> Configure the script options, if you wish. To configure the options, open up the file nopcart.js in your favorite text editor. I recommend notepad under Windows, or kwrite under Linux. The following options are configurable: <br> <b>Options For Everyone:</b><br> 
         <ul> 
          <li> <u>Display Notice:</u>Controls whether the user is provided with a popup letting them know their product is added to the cart. </li>
          <li> <u>Display Shipping Column:</u>Controls whether the managecart and checkout pages display individual shipping cost columns for each product. Set this to 'true' if each product you sell has it's own shipping cost associated with it. </li>
          <li> <u>Display Shipping Row:</u> Controls whether the managecart and checkout pages display a row at the bottom of the table for shipping cost totals. </li>
          <li> <u>Display Tax Row:</u> Controls whether the managecart and checkout pages display tax cost total row at the bottom of the table. </li>
          <li> <u>Tax Rate:</u> Your area's current tax rate, ie: if your tax rate was 7.5%, you would set Tax Rate = 0.075. You can set this to 0.0 if you do not wish tax to be calculated. </li>
          <li> <u>Tax By Region:</u> Turning this option on makes your shopping cart prompt users for information to determine if they should be charged tax. For example, if your store is in Arizona, USA you only need to charge tax to Arizona residents, so you can set this option to allow you to collect tax only from Arizona residents. </li>
          <li> <u>Tax Prompts:</u> The messages that users are prompted with are configurable. </li>
          <li> <u>MinimumOrder:</u> number, the minium dollar amount that must be purchased before a user is allowed to checkout. Set to 0.00 to disable. </li>
          <li> <u>MinimumOrderPrompt:</u> string, Message to prompt users with when they have not met the minimum order amount. </li>
          <li> <u>Monetary Symbol:</u> The symbol which represents dollars/euro, in your locale. </li>
         </ul> </li><li> <u>PaymentProcessor:</u> The two digit processor code. Support payment processor codes are:
         <ul> 
          <li> Authorize.net (an) </li>
          <li> Worldpay (wp) </li>
          <li> LinkPoint (lp) </li>
         </ul>  <b>Options For Programmers:</b> <i>(Most people won't use these)</i><br> 
         <ul> 
          <li> <u>OutputItem&lt;...&gt;:</u> The name of the pair value passed at checkout time. Change these only if you are connecting to a CGI script and need other field names, or are using a secure service that requires specific field names. </li>
          <li> <u>AppendItemNumToOutput:</u> true/false, if set to true, the number of each ordered item will be appended to the output string. For example if OutputItemId is 'ID_' and this is set to true, the output field name will be 'ID_1', 'ID_2' ... for each item. </li>
          <li> <u>HiddenFieldsToCheckout:</u> true/false, if set to true, hidden fields for the cart items will be passed TO the checkout page, from the ManageCart page. This is set to true for CGI/PHP/Script based checkout pages, but should be left false if you are using an HTML/Javascript Checkout Page. Hidden fields will ALWAYS be passed FROM the checkout page to the Checkout CGI/PHP/ASP/Script </li>
         </ul> You will see these options near the top of the file. Each option is either true/false, a string, or a number. For true/false fields, you must enter either true or false, replacing the existing option. For string options, enter in any text you wish between the single quotes. This can include HTML tags, or escape characters. For number fields, simply enter the number you wish. <p></p></li><li> On each page you wish to sell products from, or display the cart, include the lines: <br> <font color="RED"><pre>
&lt;SCRIPT SRC="language-en.js"&gt;&lt;/SCRIPT&gt;
&lt;SCRIPT SRC="nopcart.js"&gt;
  //=====================================================================||
  //               NOP Design JavaScript Shopping Cart                   ||
  //---------------------------------------------------------------------||
  //       Visit NOP Design at http://www.nopdesign.com/freecart         ||
  //=====================================================================||
&lt;/SCRIPT&gt;
</pre> </font> You can change the language that NOPCart uses by including different language packs in the first line. The available language packs are: 
         <ul> 
          <li> English: language-en.js </li>
          <li> Brazilian: language-br.js </li>
          <li> Chinese [Traditional Big5 Encoded]: language-cb5.js </li>
          <li> Czech: language-cz.js </li>
          <li> Danish: language-dk.js </li>
          <li> Dutch: language-nl.js </li>
          <li> Finnish: language-fi.js </li>
          <li> French: language-fr.js </li>
          <li> German: language-ge.js </li>
          <li> Hebrew: language-il.js </li>
          <li> Hungarian: language-hu.js </li>
          <li> Italian: language-it.js </li>
          <li> Malay: language-my.js </li>
          <li> Norwegian: language-no.js </li>
          <li> Polish: language-pl.js </li>
          <li> Romanian: language-ro.js </li>
          <li> Slovak: language-sk.js </li>
          <li> Spanish: language-sp.js </li>
          <li> Swedish: language-se.js </li>
         </ul> <p> Don't worry -- none of this text will be visible to users on your site. The only way you'll see this is if you choose 'view source' on a page. It's important you include this line, since it instructs the browser to load the shopping cart file. <br> <br><i>Don't forget to include the proper relative path to the nopcart.js file if your files are not all in the same directory!</i> </p><p></p></li><li> For each product you wish to sell, create a small FORM tag with:
         <ul> 
          <li> Price </li>
          <li> Name or Short Description </li>
          <li> Product ID or Part Number </li>
          <li> Shipping Cost </li>
          <li> Additional Information (Up to 4) </li>
         </ul> <br> This form will look something like:  <pre>
    <font color="GREEN">&lt;!--Shopping Cart Product Begin--&gt;
    &lt;P&gt;&lt;FORM NAME=order&gt;
    Quantity: 
    &lt;input type=text size=2 maxlength=3 name=QUANTITY onChange='this.value=CKquantity(this.value)' value="1"&gt;    
    &lt;input type=hidden name="PRICE" value="19.95"&gt;
    &lt;input type=hidden name="NAME" value="My Sample Product"&gt;
    &lt;input type=hidden name="ID_NUM" value="Sample ID 001"&gt;
    &lt;input type=hidden name="SHIPPING" value="5.95"&gt;
    Select a color: 
    &lt;select name=ADDITIONALINFO&gt;
        &lt;option value="Red"&gt;  Red
        &lt;option value="Blue"&gt; Blue
        &lt;option value="White"&gt;White
    &lt;/select&gt;
    Select a size: 
    &lt;select name=ADDITIONALINFO2&gt;
        &lt;option value="Small"&gt;  Small
        &lt;option value="Medium"&gt; Medium
        &lt;option value="Large"&gt;Large
    &lt;/select&gt;
    &lt;input type=button value='  Add to Cart  ' onClick='AddToCart(this.form)'&gt;
    &lt;a href="managecart.html"&gt;Check out now&lt;/a&gt;
    &lt;/FORM&gt;
    &lt;!--Shopping Cart Product End  --&gt;</font>
	</pre> <font face="ARIAL,HELVETICA" size="2"> Of course you will replace the 'value=' section with the information for your product. If you do not have any additional information to prompt for (ie. Size, Color, Style, etc.), or do not wish to charge for shipping simply leave those fields off the form. <p> If you wish to have the cart go directly to a manage cart page when a product is added, just replace the FORM tag like this: <font color="GREEN">&lt;FORM NAME=order ACTION="managecart.html"&gt;</font> -- of course replacing managecart.html with the name of your managecart page. </p><p></p></font></li><li><font face="ARIAL,HELVETICA" size="2"> Setup a 'manage cart' page. All you have to do is add the following line into your HTML page to have the shopping cart displayed, and the form elements written into the page: </font> <pre>
    <font color="GREEN">
    &lt;!--Shopping Cart ManageCart Begin--&gt;
    The items listed below are currently in your shopping cart:
    &lt;p&gt;
    &lt;FORM ACTION="checkout.html" METHOD="GET" onsubmit="return ValidateCart(this)"&gt;	
    &lt;SCRIPT&gt;
        ManageCart();
    &lt;/SCRIPT&gt;
    
    &lt;INPUT type=SUBMIT value="   Check Out   "&gt;
    &lt;/FORM&gt;	
    &lt;!--Shopping Cart ManageCart End  --&gt;
    </font></pre> <font face="ARIAL,HELVETICA" size="2"> <p> Make sure to specify the correct location of your checkout page as the FORM ACTION. For example, if your checkout page was on a secure server (www.secure.com), named checkout.cgi, and could accept POST mode, your FORM tag would be: <font color="GREEN">&lt;FORM ACTION="https://www.secure.com/checkout.cgi" METHOD="POST" onsubmit="return ValidateCart(this)"&gt;</font> Most user's will not use CGI based checkout pages, but if you do, make sure to set the HiddenFieldsToCheckout option to true. </p><p></p></font></li><li><font face="ARIAL,HELVETICA" size="2"> Setup a checkout page. If you have a secure server, and CGI access, the form action on this page should point at it. If you are accepting credit card numbers, a secure server really should be used. If you have CGI access, place the file <font color="GREEN"><a href="checkout.zip">checkout.pl</a></font> into your cgi-bin directory, and point the FORM ACTION from this checkout page to it. If you don't have CGI, but you can access scripts, place the file <font color="GREEN"><a href="checkout.zip">checkout.php</a></font> on your server, and point the FORM ACTION from the checkout page to it. This checkout script (either Perl or PHP) will write all of your orders to a CSV file which can be viewed by Microsofy Excel, or Sun Star Office -- as well as send both a confirmation email to you and your customer(s). To set the location of this file, and your E-Mail address, simply open the checkout.pl or checkout.php file in a text editor, and read the options at the top. <p> Your checkout page is going to look something like: </p><p> </p><pre>
    <font color="GREEN">
    &lt;!--Shopping Cart Checkout Begin--&gt;
    The items listed below are currently in your shopping cart:
    &lt;p&gt;
    &lt;FORM ACTION="/cgi-bin/checkout.pl" METHOD="POST"&gt;	
    &lt;SCRIPT&gt;
        CheckoutCart();
    &lt;/SCRIPT&gt;
    
    Name:    &lt;input type="text" name="b_first"&gt; &lt;input type="text" name="b_last"&gt;
    Email:   &lt;input type="text" name="b_email"&gt;
      ... Your other form data here...

    &lt;INPUT type=SUBMIT value="   Submit Order   "&gt;
    &lt;/FORM&gt;	
    &lt;!--Shopping Cart Checkout End  --&gt;
    </font></pre> <p> If you do not have your own CGI or PHP capability, then use your ISP's 'formmail' or 'mailto' scripts. Most EVERY ISP has one available for you to use. Your checkout page will look exactly the same, except for the FORM ACTION, and any 'hidden' fields your ISP will tell you about. In most cases, your form action will look something like: <font color="GREEN">&lt;FORM ACTION="/cgi-bin/formmail" METHOD="POST"&gt;</font> <font face="ARIAL,HELVETICA" size="2"> <br> At this point,most of you are done! </font></p><p><font face="ARIAL,HELVETICA" size="2"> If you are <b><i>absolutely sure</i></b> you have no mail script available, you can use the FORM line <font color="GREEN">&lt;FORM ACTION="mailto:spam@nopdesign.com" ENCTYPE="text/plain"&gt;</font> where the email address is your own. This syntax will not work with some browsers/aol. Do not write me about this not working, or opening up a blank email.... </font></p><p><font face="ARIAL,HELVETICA" size="2"> <b>Do you use a payment processor/gateway?</b> If so -- this section is just for you. There are more options that the documentation your payment processor provided will explain. </font></p>
          <ul>
