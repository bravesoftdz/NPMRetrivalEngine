       <h1 itemprop="name" class="devsite-page-title"> Ecommerce Tracking </h1> 
       <nav class="devsite-page-nav-embedded devsite-nav"></nav> 
       <div class="devsite-article-body clearfix
            " itemprop="articleBody"> 
        <p>This guide describes how to collect ecommerce data using analytics.js.</p> 
        <h2 id="overview">Overview</h2> 
        <p>Ecommerce tracking allows you to measure the number of transactions and revenue that your website generates. On a typical ecommerce site, once a user clicks the "purchase" button in the browser, the user's purchase information is sent to the web server, which carries out the transaction. If successful, the server redirects the user to a "Thank You" or receipt page with transaction details and a receipt of the purchase. You can use the analytics.js library to send the ecommerce data from the "Thank You" page to Google Analytics.</p> 
        <aside class="caution">
         <strong>Important</strong>: The Ecommerce plug-in should not be used alongside the 
         <a href="https://developers.google.com/analytics/devguides/collection/analyticsjs/enhanced-ecommerce"> Enhanced Ecommerce (ec.js)</a> plug-in.
        </aside> 
        <p>There are two types of ecommerce data you can send using analytics.js: <strong>transaction</strong> and <strong>item</strong> data.</p> 
        <h3 id="transaction">Transaction Data</h3> 
        <p>A transaction represents the entire transaction that occurs on your site, and contains the following values:</p> 
        <table> 
         <tbody>
          <tr> 
           <th>Key</th> 
           <th>Value Type</th> 
           <th>Required</th> 
           <th>Description</th> 
          </tr> 
          <tr> 
           <td><code>id</code></td> 
           <td>text</td> 
           <td><strong>Yes</strong></td> 
           <td>The transaction ID. (e.g. 1234)</td> 
          </tr> 
          <tr> 
           <td><code>affiliation</code></td> 
           <td>text</td> 
           <td>No</td> 
           <td>The store or affiliation from which this transaction occurred (e.g. Acme Clothing).</td> 
          </tr> 
          <tr> 
           <td><code>revenue</code></td> 
           <td>currency</td> 
           <td>No</td> 
           <td>Specifies the total revenue or grand total associated with the transaction (e.g. 11.99). This value may include shipping, tax costs, or other adjustments to total revenue that you want to include as part of your revenue calculations.</td> 
          </tr> 
          <tr> 
           <td><code>shipping</code></td> 
           <td>currency</td> 
           <td>No</td> 
           <td>Specifies the total shipping cost of the transaction. (e.g. 5)</td> 
          </tr> 
          <tr> 
           <td><code>tax</code></td> 
           <td>currency</td> 
           <td>No</td> 
           <td>Specifies the total tax of the transaction. (e.g. 1.29)</td> 
          </tr> 
         </tbody>
        </table> 
        <h3 id="item">Item Data</h3> 
        <p>An item represents the individual products that were in the shopping cart, and contains the following values:</p> 
        <table> 
         <tbody>
          <tr> 
           <th>Key</th> 
           <th>Value Type</th> 
           <th>Required</th> 
           <th>Description</th> 
          </tr> 
          <tr> 
           <td><code>id</code></td> 
           <td>text</td> 
           <td><strong>Yes</strong></td> 
           <td>The transaction ID. This ID is what links items to the transactions to which they belong. (e.g. 1234)</td> 
          </tr> 
          <tr> 
           <td><code>name</code></td> 
           <td>text</td> 
           <td><strong>Yes</strong></td> 
           <td>The item name. (e.g. Fluffy Pink Bunnies)</td> 
          </tr> 
          <tr> 
           <td><code>sku</code></td> 
           <td>text</td> 
           <td>No</td> 
           <td>Specifies the SKU or item code. (e.g. SKU47)</td> 
          </tr> 
          <tr> 
           <td><code>category</code></td> 
           <td>text</td> 
           <td>No</td> 
           <td>The category to which the item belongs (e.g. Party Toys)</td> 
          </tr> 
          <tr> 
           <td><code>price</code></td> 
           <td>currency</td> 
           <td>No</td> 
           <td>The individual, unit, price for each item. (e.g. 11.99)</td> 
          </tr> 
          <tr> 
           <td><code>quantity</code></td> 
           <td>integer</td> 
           <td>No</td> 
           <td>The number of units purchased in the transaction. If a non-integer value is passed into this field (e.g. 1.5), it will be rounded to the closest integer value.</td> 
          </tr> 
         </tbody>
        </table> 
        <h2 id="implementation">Implementation</h2> 
        <p>You typically implement ecommerce tracking once the user has completed the checkout process. This generally occurs on the "Thank You" page. Once you have, and are ready to send ecommerce data to Google Analytics, there are a couple of steps you need to go through:</p> 
        <ul> 
         <li><a href="#loadit">Load the ecommerce plugin</a></li> 
         <li><a href="#addTrans">Adding a Transaction</a></li> 
         <li><a href="#addItem">Adding Items</a></li> 
         <li><a href="#sendingData">Sending Data</a></li> 
         <li><a href="#clearingData">Clearing Data</a></li> 
        </ul> 
        <h3 id="loadit">Load the Ecommerce Plugin</h3> 
        <p>To reduce the size of the analytics.js library, ecommerce tracking is not provided in the default library. Instead it is provided as a plugin module that must be loaded before being used.</p> 
        <p>To load the ecommerce plugin, use the following command:</p>
        <p> </p>
        <pre class="prettyprint">
ga('require', 'ecommerce');
</pre> 
        <p>This command must occur <strong>after</strong> you create your tracker object and <strong>before</strong> you use any of the ecommerce specific functionality.</p> 
        <p>Once loaded, a couple of new commands specific to ecommerce tracking will be added to the default tracker.</p> 
        <h3 id="addTrans">Adding a Transaction</h3> 
        <p>Once the plugin has been loaded, it creates a transparent shopping cart object. You can add transaction and item data to the shopping cart, and once fully configured, you send all the data at once.</p> 
        <p>You add transaction data to the shopping cart using the <code>ecommerce:addTransaction</code> command:</p> 
        <pre class="prettyprint">
ga('ecommerce:addTransaction', {
  'id': '1234',                     // Transaction ID. Required.
  'affiliation': 'Acme Clothing',   // Affiliation or store name.
  'revenue': '11.99',               // Grand Total.
  'shipping': '5',                  // Shipping.
  'tax': '1.29'                     // Tax.
});
</pre> 
        <h3 id="addItem">Adding Items</h3> 
        <p>Next, to add items to the shopping cart, you use the <code>ecommerce:addItem</code> command:</p> 
        <pre class="prettyprint">
ga('ecommerce:addItem', {
  'id': '1234',                     // Transaction ID. Required.
  'name': 'Fluffy Pink Bunnies',    // Product name. Required.
  'sku': 'DD23444',                 // SKU/code.
  'category': 'Party Toys',         // Category or variation.
  'price': '11.99',                 // Unit price.
  'quantity': '1'                   // Quantity.
});
</pre> 
        <h3 id="sendingData">Sending Data</h3> 
        <p>Finally, once you have configured all your ecommerce data in the shopping cart, you send the data to Google Analytics using the <code>ecommerce:send</code> command:</p> 
        <pre class="prettyprint">
ga('ecommerce:send');
</pre> 
        <p>This command will go through each transaction and item in the shopping cart and send the respective data to Google Analytics. Once complete, the shopping cart is cleared and ready to send data for a new transaction. If a previous <code>ecommerce:send</code> command was issued, only new transaction and item data will be sent.</p> 
        <aside class="note">
         <strong>Note</strong>: While most implementations will send both transaction and item data, you can send transactions without items, and items without transactions. If you send an item hit without a transaction, a transaction hit with only the ID will be sent automatically.
        </aside> 
        <h3 id="clearingData">Clearing Data</h3> 
        <p>If you need to manually clear the shopping cart of all transactions and items, you use the following command:</p> 
        <pre class="prettyprint">
ga('ecommerce:clear');
</pre> 
        <h3 id="multicurrency">Specifying Local Currencies</h3> 
        <p>By default, you can configure a common, global, currency for all transactions and items through the Google Analytics management web interface. By default, the global currency is used for all items and transactions. For websites that conduct transactions in multiple currencies, the ecommerce plugin allows you to specify the local currency of the transaction as well as individual products.</p> 
        <!-- TODO(awales): Remove comment when currency field is enabled for 'set'
<p>For example, to specify euros (<code>EUR</code>) as the local currency
   for all items and transactions, you use the <code>set</code> command
   with the <code>currency/code> field and the value of the local
   currency:</p>

<pre class="prettyprint">
ga('set', 'currency', 'EUR');
</pre> --> 
        <p>The local currency must be specified in the <code>ISO 4217</code> standard. Read the <a href="https://developers.google.com/analytics/devguides/platform/features/currencies">Currency Codes Reference</a> document for a complete list of supported conversion currencies.</p> 
        <p>To set the local currency of a particular transaction and all its items, you only need to specify the currency for the transaction:</p> 
        <pre class="prettyprint">
ga('ecommerce:addTransaction', {
  'id': '1234',
  'affiliation': 'Acme Clothing',
  'revenue': '11.99',
  'shipping': '5',
  'tax': '1.29',
  'currency': 'EUR'  // local currency code.
});
</pre> 
        <p>Finally, you can also specify the currency per item:</p>
        <p> </p>
        <pre class="prettyprint">
  ga('ecommerce:addItem', {
    'id': '1234',
    'name': 'Fluffy Pink Bunnies',
    'sku': 'DD23444',
    'category': 'Party Toys',
    'price': '11.99',
    'quantity': '1',
    'currency': 'GBP' // local currency code.
  });
</pre> 
        <aside class="special">
         To learn more about how currency conversion works in Google Analytics, read the 
         <a href="https://developers.google.com/analytics/devguides/platform/features/ecommerce#specifying-currencies">Multiple Currencies</a> section in the ecommerce feature reference 
        </aside> 
        <h3 id="multitracker">Multiple Tracker Support</h3> 
        <p>You can also use the ecommerce plugin if you have implemented multiple (named) trackers on your page. The plugin works exactly the same as the default tracker, except that the format is: <code>trackerName.pluginName:method</code>. For example if you created a tracker named <code>myTracker</code>:</p> 
        <pre class="prettyprint">
ga('create', 'UA-XXXXX-Y', 'auto', {'name': 'myTracker'});
</pre> 
        <p>You would then load the ecommerce plugin for that named tracker using:</p> 
        <pre class="prettyprint">
ga('myTracker.require', 'ecommerce');
</pre> 
        <p>The to send a transaction, you can create a transaction object and pass it to the named tracker as follows:</p> 
        <pre class="prettyprint">
var transaction = {
  'id': '1234',                    // Transaction ID.
  'affiliation': 'Acme Clothing',  // Affiliation or store name.
  'revenue': '11.99',              // Grand Total.
  'shipping': '5' ,                // Shipping.
  'tax': '1.29'                    // Tax.
};

ga('myTracker.ecommerce:addTransaction', transaction);
</pre> 
        <p>Using this syntax, the transaction object can be used on multiple trackers.</p> 
        <p>Finally you would send the transaction data as follows:</p> 
        <pre class="prettyprint">
ga('myTracker.ecommerce:send');
</pre> 
        <h2 id="booya">Example</h2> 
        <p>Most ecommerce sites perform transactions on the server, while the analytics.js library sends data to Google Analytics from the browser. So a little bit of coordination is required between server and client to properly send ecommerce data to Google Analytics.</p> 
        <p>Most ecommerce sites render their "Thank You" page using a server-side templating engine. In this case, you would add the ecommerce tracking code to the server-side template, and use server logic to dynamically write the ecommerce data values to the final page. Here is an example of how this would look in PHP.</p> 
        <p>In PHP, you typically would have some representation of the ecommerce data. In this example, the data is stored in an associative array:</p> 
        <pre class="prettyprint">
&lt;?php
// Transaction Data
$trans = array('id'=&gt;'1234', 'affiliation'=&gt;'Acme Clothing',
               'revenue'=&gt;'11.99', 'shipping'=&gt;'5', 'tax'=&gt;'1.29');

// List of Items Purchased.
$items = array(
  array('sku'=&gt;'SDFSDF', 'name'=&gt;'Shoes', 'category'=&gt;'Footwear', 'price'=&gt;'100', 'quantity'=&gt;'1'),
  array('sku'=&gt;'123DSW', 'name'=&gt;'Sandals', 'category'=&gt;'Footwear', 'price'=&gt;'87', 'quantity'=&gt;'1'),
  array('sku'=&gt;'UHDF93', 'name'=&gt;'Socks', 'category'=&gt;'Footwear', 'price'=&gt;'5.99', 'quantity'=&gt;'2')
);
?&gt;
</pre> 
        <p>The first step is to write some logic to transform the ecommerce data into the JavaScript string required by analytics.js:</p> 
        <pre class="prettyprint">
&lt;?php
// Function to return the JavaScript representation of a TransactionData object.
function getTransactionJs(&amp;$trans) {
  return &lt;&lt;&lt;HTML
ga('ecommerce:addTransaction', {
  'id': '{$trans['id']}',
  'affiliation': '{$trans['affiliation']}',
  'revenue': '{$trans['revenue']}',
  'shipping': '{$trans['shipping']}',
  'tax': '{$trans['tax']}'
});
HTML;
}

// Function to return the JavaScript representation of an ItemData object.
function getItemJs(&amp;$transId, &amp;$item) {
  return &lt;&lt;&lt;HTML
ga('ecommerce:addItem', {
  'id': '$transId',
  'name': '{$item['name']}',
  'sku': '{$item['sku']}',
  'category': '{$item['category']}',
  'price': '{$item['price']}',
  'quantity': '{$item['quantity']}'
});
HTML;
}
?&gt;
</pre> 
        <p>Then in the <code>&lt;script&gt;</code> tag you would add the additional PHP logic to dynamically output the transaction and item data:</p> 
        <pre class="prettyprint">
&lt;!-- Begin HTML --&gt;
&lt;script&gt;
ga('require', 'ecommerce');

&lt;?php
echo getTransactionJs($trans);

foreach ($items as &amp;$item) {
  echo getItemJs($trans['id'], $item);
}
?&gt;

ga('ecommerce:send');
&lt;/script&gt;
</pre> 
        <p>Once the PHP script is done executing, the transaction and item data required by analytics.js will be printed to the page. Once the JavaScript on the page is rendered in the browser, all the ecommerce data will be sent to Google Analytics.</p> 
        <h2 id="currency">Currency Types</h2> 
        <p>The default currency type can be configured through the Management Interface. When you send currency values using analytics.js, the value represents the total value of a currency.</p> 
        <p>A decimal point can be used as a delimiter between the whole and fractional portion of the currency. The precision is up to 6 decimal places. The following is valid for a currency field:</p> 
        <pre>
1000.000001
</pre> 
        <p>Once the value is sent to Google Analytics, all text is removed up until the first digit, the <code>-</code> character or the <code>.</code> (decimal) character. So:</p> 
        <pre>
$-55.00
</pre> 
        <p>will become:</p> 
        <pre>
-55.00
</pre>
