                              <td width="49%" class="page-content"><p align="left"><span class="page-content-heading1 style18"><span class="style10">Developing a Shopping Cart Application</span><br> </span><br> We are providing relevant details to develop an ecommerce<br> application (a shopping cart) with Struts, Hibernate and Spring.<br> <br> <span class="style10"><strong> Features provided in the shopping cart</strong></span><br> <br> <strong>Customers Registration:</strong> Existing user can login to the shopping cart by signing in using their username and password. Moreover they can update their profile such as address, contact number, Email-id etc.<br> <br> <strong>Product catalog:</strong> Product catalog displays the list of items. It helps the customers to quickly browse the details of the products available to buy. Adding items to the cart: This feature provides a mechanism to add multiple products to a cart. Moreover options are provided to choose quantity of a particular item.<br> <br> <strong>Updating Quantity:</strong> This feature enables to update<br> the quantity of a product..<br> <br> <strong>Remove Items/Product:</strong> Remove feature enables to remove a selected item from the shopping cart. <br> <br> <strong> Checkout /Place order Form:</strong> After selecting the multiple items, customers can place an order by filling the Proceed to Checkout form. In this form customer has to fill personal as well as his credit card information. <br> <span class="content-heading1 style20"><br> Administrative Features</span><br> <strong><br> Catalog management:</strong> This feature enables to add and edit categories and subcategories. Here, the administrator can add the number of products to the Catalog Listing by roviding the product details like Category ID, Category Name and Parent ID etc. Following options can be opted.<br> <br> <strong>Add/Edit Categories or Subcategories</strong><br> <br> <strong>Add/Edit Product </strong><br> <br> <strong>Check orders:</strong> Here, the administrator check the orders placed by the customers with the details such as Order Id, Customer, Email, Address, Phone number, Ordered products, Order total, Order time and Quantity. <br> <br> <strong> Configuration:</strong> Here, the administrator can update the following relevant information.</p> </td> 
                              <td width="2%">&nbsp;</td> 
                              <td width="49%" class="page-content"><p>&#x2022; Store Name<br> &#x2022; Store URL<br> &#x2022; Notification mail-id<br> &#x2022; Currency Symbol &amp; Code<br> &#x2022; About Us content<br> &#x2022; Shipping and Delivery content<br> &#x2022; Changing password<br> <br> <span class="page-content-heading1 style20">Technical Features</span><br> <br> &#x2022; Free Java source code. Java Source code is provided free with the shopping cart application.<br> <br> &#x2022; Choice of web server and a database: This ecommerce<br> application has been tested on Apache Tomcat 5.0 and 5.5, and is conformant with other JEE Web containers like Resin and JRun. Databases like MySQL, PostgreSQL, MS SQL Server, and Oracle are all supported out-of-the-box.<br> <br> &#x2022; Integration of Struts, Spring and Hibernate: Struts is configured with Hibernate persistence framework that handles all of the application&#x2019;s database access. Spring Framework is used to manage java beans for example within the application, ShoppingCartDAO object is managed through spring framework. It is also acting as transaction manager for Hibernate SessionFactory to manage the database transactions.<br> <br> <span class="page-content-heading1 style20"> Application Architecture</span><br> <br> <strong>I </strong>Shopping Cart follows a strict Model, Controller, View (MVC) architecture to organize its classes and files.<br> <br> <strong>II</strong> Controller employs two packages to control the model and the view sections. One package contains classes performing different actions and other one contains lasses<br> acting as various action forms.<br> <br> <strong>III</strong> The Model is organized into different packages where classes are devoted to interact with the database and to process business logic.<br> <br> <span class="page-content-heading1 style20">CONTROLLER LAYER</span><br> <br> The Web Controller layer provides the mechanisms<br> that control the routing of each web request through the<br> system (the &#x201c;C&#x201d; in MVC).
