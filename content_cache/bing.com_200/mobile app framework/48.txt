      <div class="entry-content"> 
       <p>Visual Studio Code is a great tool for building traditional web applications using JavaScript, TypeScript, HTML/CSS, Node, as well as many of today’s modern frameworks, such as AngularJS. Because of this, it is also a great tool for building HTML5 and JavaScript based mobile hybrid applications that run on platforms such as Apache Cordova.</p> 
       <p>Ionic is a very popular (and cool!) UI framework for building these types of applications. It relies heavily on the also very popular AngularJS framework, delivering very rich UI components and behaviors through its CSS, JavaScript, and graphics libraries. If you are not familiar with Angular, I recommend starting with the Angular tutorial as described in <a href="http://blogs.msdn.com/b/vscode/archive/2015/05/22/getting-started-with-angular-and-visual-studio-code.aspx">Getting started with Angular and VS Code</a>.</p> 
       <p>The <a href="http://ionicframework.com/docs/guide/">Ionic Book</a> does a great job walking you through the process of building your first Ionic based application. I thought it would be interesting to see how Visual Studio Code can make a great companion for writing Ionic based applications.</p> 
       <p>I’ll assume you’ve followed the tutorial and have the canonical "ToDo" application running on your Mac. If not, you can clone a version of the application that has all of the VS Code tweaks that we will discuss in this blog from here: <a href="https://github.com/chrisdias/ionic-todo-vscode">https://github.com/chrisdias/ionic-todo-vscode</a>.</p> 
       <p><a href="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/3247.clip_image002_54FC4A84.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/3247.clip_5F00_image002_5F00_54FC4A84.png"><img title="clip_image002" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="clip_image002" src="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/8484.clip_image002_thumb_36456099.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/8484.clip_5F00_image002_5F00_thumb_5F00_36456099.png" width="640" height="462"></a></p> 
       <h2>Rich Code Authoring and Navigation</h2> 
       <p>Out of the box, VS Code has great code editing and navigation experiences for JavaScript, HTML, CSS, SASS, JSON, and more. For example, Code provides IntelliSense and validation for well-known JSON files. Using Quick Open (Cmd+E or Cmd+P), let’s open bower.json.</p> 
       <h3>JSON</h3> 
       <p>Let’s navigate to bower.json using Quick Open, either by clicking on it in the Explorer or through Quick Open (Cmd+E or Cmd+P) and then typing in the first few characters of the file name. Press Enter to open the file.</p> 
       <p><a href="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/3124.clip_image004_1AA36554.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/3124.clip_5F00_image004_5F00_1AA36554.png"><img title="clip_image004" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="clip_image004" src="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/8764.clip_image004_thumb_558A981D.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/8764.clip_5F00_image004_5F00_thumb_5F00_558A981D.png" width="640" height="348"></a></p> 
       <p>The first thing to notice is the warning on line 3.</p> 
       <p><a href="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/5224.clip_image006_3A54CFCD.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/5224.clip_5F00_image006_5F00_3A54CFCD.png"><img title="clip_image006" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="clip_image006" src="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/3007.clip_image006_thumb_15570F54.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/3007.clip_5F00_image006_5F00_thumb_5F00_15570F54.png" width="640" height="324"></a></p> 
       <p>If you hover over the warning or press F8, you will see that the expected type is boolean and not a string.</p> 
       <p><a href="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/4405.clip_image008_09551F20.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/4405.clip_5F00_image008_5F00_09551F20.png"><img title="clip_image008" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="clip_image008" src="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/2474.clip_image008_thumb_273633D2.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/2474.clip_5F00_image008_5F00_thumb_5F00_273633D2.png" width="640" height="359"></a></p> 
       <p>Tip! The status bar will show the total number of errors and warnings it currently knows about. Here we can see that I have zero errors and one warning.</p> 
       <p><a href="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/0211.image_5EA05418.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/0211.image_5F00_5EA05418.png"><img title="image" style="border-top: 0px; border-right: 0px; background-image: none; border-bottom: 0px; padding-top: 0px; padding-left: 0px; border-left: 0px; display: inline; padding-right: 0px" border="0" alt="image" src="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/0083.image_thumb_0CB25226.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/0083.image_5F00_thumb_5F00_0CB25226.png" width="413" height="112"></a></p> 
       <p>Clicking on the status will bring up the command palette with a full list of those errors and warnings, along with the offending file, line, and column, which you can select and quickly navigate to.</p> 
       <p><a href="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/2766.clip_image012_2AD6DF60.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/2766.clip_5F00_image012_5F00_2AD6DF60.png"><img title="clip_image012" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="clip_image012" src="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/6888.clip_image012_thumb_65BE1229.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/6888.clip_5F00_image012_5F00_thumb_5F00_65BE1229.png" width="640" height="246"></a></p> 
       <p>Not only can we validate the JSON, we can provide IntelliSense with rich descriptions for all of the properties and values that are valid based on the schema for this type of JSON file. Simply press Ctrl+Space to bring up the list.</p>

       <h2>Task Running</h2> 
       <p>Visual Studio Code provides a lightweight service that makes it easy to integrate your existing task running framework (gulp for example) into the tool. Code does not try to replace your task runner. Instead, it strives to provide keyboard access to your build and test tasks and to surface errors and warnings in the editor, where they originated.</p> 
       <p>If a <a href="http://gulpjs.com/">gulp</a> file is detected in the root of the workspace, VS Code will automatically surface the tasks found within the file to the Command Palette. In our project we have a gulpfile.js that was created when we initialized SASS. By default, the Ionic CLI creates a handful of tasks for watching files and compiling SASS to CSS. I’ve added a few more tasks to call the Ionic CLI directly to launch the application in the browser (serve), the emulator (run), and as well as a task to build (build) the application.</p> 
       <p><a href="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/2313.clip_image042_6BCC0396.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/2313.clip_5F00_image042_5F00_6BCC0396.png"><img title="clip_image042" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="clip_image042" src="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/3581.clip_image042_thumb_0DB7661B.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/3581.clip_5F00_image042_5F00_thumb_5F00_0DB7661B.png" width="626" height="480"></a></p> 
       <p>Press Shift+Cmd+P to bring up the Command Palette and search for and select “Tasks: Run Task”. All of the tasks from the gulp file are now available to you from directly within VS Code.</p> 
       <p><a href="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/3010.clip_image044_68B9A5A1.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/3010.clip_5F00_image044_5F00_68B9A5A1.png"><img title="clip_image044" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="clip_image044" src="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/2055.clip_image044_thumb_1CEDCEE8.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/2055.clip_5F00_image044_5F00_thumb_5F00_1CEDCEE8.png" width="640" height="226"></a></p> 
       <p>I can attach key bindings for Build and Test by configuring the task runner. Open the Command Palette (Shift+Cmd+P) and search for “Tasks: Configure Task Runner” which will open ./.settings/tasks.json with the following content.</p> 
       <p><a href="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/7848.image_0188C7DC.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/7848.image_5F00_0188C7DC.png"><img title="image" style="border-top: 0px; border-right: 0px; background-image: none; border-bottom: 0px; padding-top: 0px; padding-left: 0px; border-left: 0px; display: inline; padding-right: 0px" border="0" alt="image" src="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/0508.image_thumb_4C7C38AB.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/0508.image_5F00_thumb_5F00_4C7C38AB.png" width="551" height="484"></a></p> 
       <p>This tells Code that our task runner is gulp. I’ve “overridden” the “build ios” task to say that it is our default Build command, meaning it is attached to the key combination Shift+Cmd+B. Because “showOutput” is “true, <a>when</a> I execute the command the Output pane opens with the Tasks channel selected. File references are automatically <a>detected</a><a href="https://microsoft.sharepoint.com/teams/DD_OTP/Documents/Ticino/Blogs/#_msocom_4" name="_msoanchor_4"></a> and shown as links, so I can quickly navigate to that file directly from the editor.</p> 
       <p><a href="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/0272.clip_image047_6A205924.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/0272.clip_5F00_image047_5F00_6A205924.png"><img title="clip_image047" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="clip_image047" src="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/2781.clip_image047_thumb_79C2F4E6.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/2781.clip_5F00_image047_5F00_thumb_5F00_79C2F4E6.png" width="640" height="384"></a></p> 
       <h2>AutoSave</h2> 
       <p>Visual Studio Code has its <a href="http://blogs.msdn.com/b/monaco/">roots in browser based tools</a> where AutoSave is common, as you are never sure when the user will navigate away from the tool. When developing web applications, having a fast “edit-refresh” cycle is very productive, and AutoSave enables that when running locally. Make changes and reload the browser, or better yet use <a href="http://www.browsersync.io/">browsersync</a> to reload the browser automatically.</p> 
       <p>You can enable AutoSave in VS Code on the File Menu.</p> 
       <p><a href="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/6431.clip_image048_504EB3A6.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/6431.clip_5F00_image048_5F00_504EB3A6.png"><img title="clip_image048" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="clip_image048" src="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/3618.clip_image048_thumb_202768E3.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/3618.clip_5F00_image048_5F00_thumb_5F00_202768E3.png" width="632" height="480"></a></p> 
       <p>By combining AutoSave with Ionic’s “live reload”, which is on by default when you run the “serve” or “run” tasks, whenever you make changes those changes will automatically be reflected in the running application. Let’s run the “Run” task and make some changes to /www/index.html. We will change the style of the bar from “bar-dark” to “bar-royal” and after a moment, we will see our app updated right in the emulator.</p> 
       <p><a href="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/6708.clip_image050_2D2148F4.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/6708.clip_5F00_image050_5F00_2D2148F4.png"><img title="clip_image050" style="border-left-width: 0px; border-right-width: 0px; background-image: none; border-bottom-width: 0px; padding-top: 0px; padding-left: 0px; display: inline; padding-right: 0px; border-top-width: 0px" border="0" alt="clip_image050" src="https://msdnshared.blob.core.windows.net/media/MSDNBlogsFS/prod.evol.blogs.msdn.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/01/70/67/metablogapi/6646.clip_image050_thumb_117F4DAF.png" original-url="http://blogs.msdn.com/cfs-file.ashx/__key/communityserver-blogs-components-weblogfiles/00-00-01-70-67-metablogapi/6646.clip_5F00_image050_5F00_thumb_5F00_117F4DAF.png" width="640" height="370"></a></p> 
       <p>In the status bar, between the git status and the errors and warnings status, you will see a spinning line. This indicates that a task is running. To stop the “Run” task, simply press Shift+Cmd+X.</p> 
       <h2>Conclusion</h2> 
       <p>In this post, we’ve seen all of the ways that you can use Visual Studio Code to create mobile hybrid applications using the Ionic Framework and Cordova. You can find more documentation on VS Code at <a href="http://code.visualstudio.com">http://code.visualstudio.com</a>. Let us know what you think!</p>
