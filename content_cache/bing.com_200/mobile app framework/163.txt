         <tbody>
          <tr>
           <th>Scenario</th>
           <th>Launch parameter</th>
           <th>Example</th>
          </tr> 
          <tr>
           <td>You just want to launch the app with the default page.</td>
           <td>Specify an empty string.</td>

          <tr>

           <td><code>pkg.Launch("?content=1234&amp;param2=test");</code></td>
          </tr> 
          <tr>
           <td>You want to launch to a particular XAML page and pass in parameters.</td>
           <td>Use a combination of the previous two examples.</td>

          <span class="lang-javascript"><a class="xref" href="windows.foundation.iasyncoperation_tresult_" data-linktype="relative-path">IAsyncOperation&lt;bool&gt;</a></span>

         <p><strong>True</strong> indicates that the operation to set whether the package is in use or not was successful; <strong>false</strong> otherwise.</p>

        <h4 class="accented">Remarks</h4> 
        <p>This API specifies whether the app is using an optional package. With this information the system can determine whether servicing the optional package will require shutting down the app. This provides a better user experience because the app can manage when it is shut down for optional package updates or removals.</p> 
        <p>By default, all optional packages are in use. Servicing an optional package causes the app to shut down. To avoid shutting down the app when an optional package that is not being used is updated or removed, call this API with <strong>inUse</strong> set to <strong>false</strong>.</p>
