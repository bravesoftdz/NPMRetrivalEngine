           <div class="title">
            How do I run another application from Java?
           </div> 
           <div>
            Author: Deron Eriksson
           </div> 
           <div class="smaller">
            Description: This Java tutorial describes using Runtime's exec method to run another application from Java.
           </div> 
           <div class="smaller">
             Tutorial created using: Windows XP || JDK 1.5.0_09 || Eclipse Web Tools Platform 1.5.1
           </div> 
          </div> 
          <hr> <br> 
          <div style="font-weight: bold">
           Page: &nbsp;&nbsp; 1 
           <a href="/tutorials/lessons/how-do-i-run-another-application-from-java.html?page=2" title="Go to page 2">2</a>&nbsp;
           <a href="/tutorials/lessons/how-do-i-run-another-application-from-java.html?page=2" title="Go to page 2">&gt;</a>
          </div><br> <p>In Java<a class="extlink2" href="http://java.sun.com/" title="Go to Site for 'Sun Java Developers Site'">S</a><a class="extlink" href="http://en.wikipedia.org/wiki/Java_(programming_language)" title="Go to 'Java (programming language)' in Wikipedia">W</a>, you can run another application by calling the exec method on the Runtime object, which may be obtained via a call to Runtime.getRuntime(). This tutorial will feature two examples, the first of which will open the Notepad application and close it after 5 seconds. The second example will run another Java process and read the output and error streams from that process. The project structure for this tutorial is shown below. </p> <img alt="'testing' project" title="'testing' project" src="../general-java/how-do-i-run-another-application-from-java/how-do-i-run-another-application-from-java-01.gif"> <p>The first example features the RuntimeExecTest1 class. The Runtime object is obtained by calling Runtime.getRuntime(). Following this, the exec method is called on the runTime object, with "notepad" passed as a parameter. This call works because notepad is an executable that is present in the system path. If the directory containing notepad was not in the system path, this call (without the fully qualified path to notepad) would not work. </p> <p>The call to exec opens a notepad window, and the 'process' object is obtained from the exec call. Following this, the call to Thread.sleep(5000) causes a pause in execution of RuntimeExecTest1 for 5 seconds. The destroy method is then called on 'process' which closes the notepad application. </p> <h3><a href="../general-java/how-do-i-run-another-application-from-java/RuntimeExecTest1.java">RuntimeExecTest1.java</a></h3> <pre><span class="javaKeyword">package</span> test;

<span class="javaKeyword">import</span> java.io.IOException;

<span class="javaKeyword">public</span> <span class="javaKeyword">class</span> RuntimeExecTest1 {

	<span class="javaKeyword">public</span> <span class="javaKeyword">static</span> <span class="javaKeyword">void</span> main(String[] args) {
		<span class="javaKeyword">try</span> {
			System.out.println(<span class="javaString">"Opening notepad"</span>);
			Runtime runTime = Runtime.getRuntime();
			Process process = runTime.exec(<span class="javaString">"notepad"</span>);
			<span class="javaKeyword">try</span> {
				Thread.sleep(5000);
			} <span class="javaKeyword">catch</span> (InterruptedException e) {
				e.printStackTrace();
			}
			System.out.println(<span class="javaString">"Closing notepad"</span>);
			process.destroy();
		} <span class="javaKeyword">catch</span> (IOException e) {
			e.printStackTrace();
		}
	}
}
</pre> <p>The execution of RuntimeExecTest1 is shown below. The screencapture is taken during the 5 second interval that the Notepad window was open. After 5 seconds, the process.destroy() call closes the notepad window. </p> <img alt="Execution of RuntimeExecTest1" title="Execution of RuntimeExecTest1" src="../general-java/how-do-i-run-another-application-from-java/how-do-i-run-another-application-from-java-02.gif"> <p class="continue">(Continued on page 2)</p>
