     <h3><b>How do I download and install 32-bit Java for Linux? </b></h3> 
     <hr> This article applies to: 
     <ul> 
      <li><b>Platform(s): </b>Oracle Enterprise Linux, Oracle Linux, Red Hat Linux, SLES, SUSE Linux </li> 
      <li><b>Java version(s): </b>7.0, 8.0 </li> 
     </ul> 
     <hr> 
     <span class="bodytext"> 
      <!-- BEGIN: jWidget_C/Raw-HTML/Display --> <h5 class="sub">Linux System Requirements</h5> <p> See supported <a href="/en/download/help/sysreq.xml">System Configurations</a> for information about supported platforms, operating systems, desktop managers, and browsers.</p> <strong>Note</strong>: For downloading Java other flavors of Linux see <a href="https://help.ubuntu.com/community/Java">Java for Ubuntu</a>, <a href="http://openjdk.java.net/install/">Java for Fedora</a> <br> <br> 
      <!-- END: jWidget_C/Raw-HTML/Display --> </span> 
     <!-- no bodytext -->
     <hr> 
     <span class="bodytext"> Follow these steps to download and install 32- bit Java for Linux. 
      <ol> 
       <li><a href="#download"><b>Download</b></a></li> 
       <li><a href="#install"><b>Install</b></a></li> 
       <li><a href="#enable"><b>Enable and Configure</b></a></li> 
      </ol> </span> 
     <span class="bodytext"> <h4><a name="download">Download</a></h4> 
      <ol> 
       <li>Go to <a href="/">http://java.com </a> and click on the <a href="/en/download/linux_manual.jsp"><b>Download</b></a> button</li> 
       <li>There are two types of installation packages.<br> 
        <ul> 
         <li> <strong>Java on Linux Platforms</strong><br> This installs the Java Runtime Environment (JRE) for 32-bit Linux, using an archive binary file (<code>.tar.gz</code>) that can be installed by anyone (not only the root users), in any location that you can write to. However, only the root user can install Java into the system location. <br></li> 
         <li><strong>Java on RPM-based Linux Platforms</strong><br> This installs the Java Runtime Environment (JRE) for 32-bit RPM-based Linux platforms, such as Red Hat and SuSE, using an RPM binary file (<code>.rpm</code>) in the system location. You must be root to perform this installation. </li> 
        </ul> <br> Download the package that best suits your needs. You can download the file to any of the directories on your system.</li> 
       <li>Before the file can be downloaded, you must accept the license agreement.</li> 
       <li> <strong>Download and check the download file size</strong> to ensure that you have downloaded the full, uncorrupted software bundle. Before you download the file, notice its byte size provided on the download page on the web site. Once the download has completed, compare that file size to the size of the downloaded file to make sure they are equal.</li> 
      </ol> </span> 
     <!-- no bodytext -->
     <hr> 
     <span class="bodytext"> <h4><a name="install">Install</a></h4> 
      <ul>
       <li><a href="#Java for Linux Platforms"><b>Java for Linux Platforms</b></a></li> 
       <li><a href="#rpm"><b>Java for RPM based Linux Platforms</b></a></li>
      </ul> </span> 
     <span class="bodytext"> 
      <!-- BEGIN: jWidget_C/Raw-HTML/Display --> <img class="infoic" src="/ga/images/ic-info16.png">The instructions below are for installing version Java 8 Update 73 (8u73). If you are installing another version, make sure you change the version number appropriately when you type the commands at the terminal. <b>Example</b>: For Java 8u79 replace <b>8u73</b> with <b>8u79</b>. Note that, as in the preceding example, the version number is sometimes preceded with the letter <code>u</code>, and sometimes it is preceded with an underbar, for example, <code>jre1.8.0_73</code>. <br><br> 
      <!-- END: jWidget_C/Raw-HTML/Display --> </span> 
     <span class="bodytext"> <p><b>Note about root access:</b> <em>To install Java in a system-wide location such as /usr/local, you must login as the root user to gain the necessary permissions. If you do not have root access, install the Java in your home directory or a sub directory for which you have write permissions</em> </p> </span> 
     <span class="bodytext"> <h5 class="sub"><a name="Java for Linux Platforms"> Java for Linux Platforms</a></h5> 
      <ol> 
       <li><strong>Change to the directory in which you want to install.</strong> Type:<br> <code>cd </code> <em>directory_path_name</em><br> For example, to install the software in the /usr/java/ directory, Type:<br> <code>cd /usr/java/</code><br> <br> </li> 
       <li>Move the .tar.gz archive binary to the current directory.</li> 
       <li><strong>Unpack the tarball and install Java</strong> <br> <code>tar zxvf jre-8u73-linux-i586.tar.gz</code><br><br> The Java files are installed in a directory called <code>jre1.8.0_73</code> in the current directory. <br> In this example, it is installed in the <code>/usr/java/jre1.8.0_73</code> directory. </li> 
       <li><strong>Delete the <tt>.tar.gz</tt> file</strong> if you want to save disk space. </li> 
      </ol> <br> </span> 
     <span class="bodytext"> <h5 class="sub"><a name="rpm">Java for RPM based Linux Platforms </a></h5> 
      <ol> 
       <li><strong>Become root</strong> by running <code>su</code> and entering the super-user password.</li> 
       <li><strong>Uninstall any earlier installations of the Java packages.</strong><br> <code>rpm -e </code><em>package_name</em> </li> 
       <li><strong>Change to the directory in which you want to install.</strong> Type:<br> <code>cd </code><em>directory_path_name</em><br> For example, to install the software in the /usr/java/ directory, Type:<br> <code>cd /usr/java</code><br> <br> </li> 
       <li><strong>Install the package</strong>. <br> <code>rpm -ivh jre-8u73-linux-i586.rpm</code><br> <br> To upgrade a package, <br> <code>rpm -Uvh jre-8u73-linux-i586.rpm</code><br> <br> </li> 
       <li><strong>Exit the root shell.</strong> No need to reboot.</li> 
       <li><strong>Delete the <tt>.rpm</tt> file</strong> if you want to save disk space.</li> 
      </ol> <p>The installation is now complete. Go to the <a href="#enable"><b>Enable and Configure</b></a> section.</p> </span> 
     <!-- no bodytext -->
     <hr> 
     <span class="bodytext"> <h4><a name="enable">Enable and Configure</a></h4> <h5 class="sub">Firefox</h5> <p> When you install the Java platform, the Java plugin file is included as part of that install. If you want to use Java within Firefox, you need to manually create a symbolic link from the plugin file in the release to one of the locations that Firefox expects. For Firefox version 21 and higher, you must create the symbolic link in your home directory, <code>~/.mozilla/plugins</code>. Beginning with Firefox version 21, creating the symbolic link in the <code>plugins</code> subdirectory of the Firefox application's directory is not supported. </p> </span> 
     <span class="bodytext"> To configure the Java Plugin follow these steps: 
      <ol> 
       <li><strong>Exit Firefox browser if it is already running.</strong></li> 
       <li><strong>Uninstall any previous installations of Java Plugin.</strong><br> Only one Java Plugin can be used at a time. When you want to use a different plugin, or version of a plugin, remove the symbolic links to any other versions and create a fresh symbolic link to the new one. <br><br> Remove the symbolic links (or move them to another directory) to <code>javaplugin-oji.so</code> and <code>libnpjp2.so</code> from the Firefox <code>plugins</code> directory. <br><br> </li> 
       <li><strong>Create a symbolic link to the Java Plugin</strong> in the Firefox <code>plugins</code> directory 
        <ul> 
         <li>Go to the Firefox <code>plugins</code> directory<br> <code>cd ~/.mozilla/plugins</code><br> Create the plugins directory if it does not exist.</li> 
         <li>Create the symbolic link <br> 32-bit plugin:<br> <code>ln -s </code><em>Java installation directory</em><code>/lib/i386/libnpjp2.so .</code> <br> 64-bit plugin:<br> <code>ln -s </code><em>Java installation directory</em><code>/lib/amd64/libnpjp2.so .</code> </li> 
        </ul> <br> <strong>Example</strong> 
        <ul> 
         <li> If the Java is installed at this directory:<br> <code>/usr/java/</code><em>Java installation directory</em></li> 
         <li>Then type in the terminal window to go to the browser plug-in directory:<br> <code>cd ~/.mozilla/plugins/</code></li> 
         <li>Enter the following command to create a symbolic link to the Java Plug-in for Firefox:<br> <code>ln -s /usr/java/</code><em>Java installation directory</em><code>/lib/i386/libnpjp2.so .</code> </li> 
        </ul><br> </li> 
       <li><strong>Start the Firefox browser</strong>, or restart it if it is already up.<br><br> In Firefox, type <b>about:plugins</b> in the Location bar to confirm that the Java Plugin is loaded. You can also click the Tools menu to confirm that Java Console is there.</li>
      </ol> </span>
