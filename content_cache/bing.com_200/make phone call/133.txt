         <p>Google's common Java, C++ and JavaScript library for parsing, formatting, and validating international phone numbers. The Java version is optimized for running on smartphones, and is used by the Android framework since 4.0 (Ice Cream Sandwich).</p>

           </svg></a>Highlights of functionality</h1> 
         <ul> 
          <li>Parsing, formatting, and validating phone numbers for all countries/regions of the world.</li> 
          <li><code>getNumberType</code> - gets the type of the number based on the number itself; able to distinguish Fixed-line, Mobile, Toll-free, Premium Rate, Shared Cost, VoIP, Personal Numbers, UAN, Pager, and Voicemail (whenever feasible).</li> 
          <li><code>isNumberMatch</code> - gets a confidence level on whether two numbers could be the same.</li> 
          <li><code>getExampleNumber</code> and <code>getExampleNumberForType</code> - provide valid example numbers for all countries/regions, with the option of specifying which type of example phone number is needed.</li> 
          <li><code>isPossibleNumber</code> - quickly guesses whether a number is a possible phone number by using only the length information, much faster than a full validation.</li> 
          <li><code>isValidNumber</code> - full validation of a phone number for a region using length and prefix information.</li> 
          <li><code>AsYouTypeFormatter</code> - formats phone numbers on-the-fly when users enter each digit.</li> 
          <li><code>findNumbers</code> - finds numbers in text.</li> 
          <li><code>PhoneNumberOfflineGeocoder</code> - provides geographical information related to a phone number.</li> 
          <li><code>PhoneNumberToCarrierMapper</code> - provides carrier information related to a phone number.</li> 
          <li><code>PhoneNumberToTimeZonesMapper</code> - provides timezone information related to a phone number.</li> 
         </ul>

         <p>We generally choose the release number following these guidelines.</p> 
         <p>If any of the changes pushed to master since the last release are incompatible with the intent / specification of an existing libphonenumber API or may cause libphonenumber (Java, C++, or JS) clients to have to change their code to keep building, we publish a major release. For example, if the last release were 7.7.3, the new one would be 8.0.0.</p> 
         <p>If any of those changes <em>enable</em> clients to update their code to take advantage of new functionality, and if clients would have to roll-back these changes in the event that the release was marked as "bad", we publish a minor release. For example, we'd go from 7.7.3 to 7.8.0.</p> 
         <p>Otherwise, including when a release contains only <a href="/googlei18n/libphonenumber/blob/master/FAQ.md#metadata_definition">metadata</a> changes, we publish a sub-minor release, e.g. 7.7.3 to 7.7.4.</p> 
         <p>Sometimes we make internal changes to the code or metadata that, while not affecting compatibility for clients, could affect compatibility for <strong>porters</strong> of the library. For such changes we make announcements to <a href="http://groups.google.com/forum/#!forum/libphonenumber-discuss">libphonenumber-discuss</a>. Such changes are not reflected in the version number, and we would publish a sub-minor release if there were no other changes.</p> 
         <p>Want to get notified of new releases? During most of the year, excepting holidays and extenuating circumstances, we release fortnightly. We update <a href="http://github.com/googlei18n/libphonenumber/releases">release tags</a> and document detailed <a href="http://github.com/googlei18n/libphonenumber/blob/master/release_notes.txt">release notes</a>. We also send an announcement to <a href="http://groups.google.com/forum/#!forum/libphonenumber-discuss">libphonenumber-discuss</a> for every release.</p> 
         <h1><a href="#quick-examples" aria-hidden="true" class="anchor" id="user-content-quick-examples">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Quick Examples</h1> 
         <p>Let's say you have a string representing a phone number from Switzerland. This is how you parse/normalize it into a <code>PhoneNumber</code> object:</p> 
         <div class="highlight highlight-source-java">
          <pre><span class="pl-smi">String</span> swissNumberStr <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">"</span>044 668 18 00<span class="pl-pds">"</span></span>;
<span class="pl-smi">PhoneNumberUtil</span> phoneUtil <span class="pl-k">=</span> <span class="pl-smi">PhoneNumberUtil</span><span class="pl-k">.</span>getInstance();
<span class="pl-k">try</span> {
  <span class="pl-smi">PhoneNumber</span> swissNumberProto <span class="pl-k">=</span> phoneUtil<span class="pl-k">.</span>parse(swissNumberStr, <span class="pl-s"><span class="pl-pds">"</span>CH<span class="pl-pds">"</span></span>);
} <span class="pl-k">catch</span> (<span class="pl-smi">NumberParseException</span> e) {
  <span class="pl-smi">System</span><span class="pl-k">.</span>err<span class="pl-k">.</span>println(<span class="pl-s"><span class="pl-pds">"</span>NumberParseException was thrown: <span class="pl-pds">"</span></span> <span class="pl-k">+</span> e<span class="pl-k">.</span>toString());
}</pre>
         </div> 
         <p>At this point, <code>swissNumberProto</code> contains:</p> 
         <div class="highlight highlight-source-json">
          <pre>{
  <span class="pl-s"><span class="pl-pds">"</span>country_code<span class="pl-pds">"</span></span>: <span class="pl-c1">41</span>,
  <span class="pl-s"><span class="pl-pds">"</span>national_number<span class="pl-pds">"</span></span>: <span class="pl-c1">446681800</span>
}</pre>
         </div> 
         <p><code>PhoneNumber</code> is a class that was originally auto-generated from <code>phonenumber.proto</code> with necessary modifications for efficiency. For details on the meaning of each field, refer to <code>resources/phonenumber.proto</code>.</p> 
         <p>Now let us validate whether the number is valid:</p> 
         <div class="highlight highlight-source-java">
          <pre><span class="pl-k">boolean</span> isValid <span class="pl-k">=</span> phoneUtil<span class="pl-k">.</span>isValidNumber(swissNumberProto); <span class="pl-c"><span class="pl-c">//</span> returns true</span></pre>
         </div> 
         <p>There are a few formats supported by the formatting method, as illustrated below:</p> 
         <div class="highlight highlight-source-java">
          <pre><span class="pl-c"><span class="pl-c">//</span> Produces "+41 44 668 18 00"</span>
<span class="pl-smi">System</span><span class="pl-k">.</span>out<span class="pl-k">.</span>println(phoneUtil<span class="pl-k">.</span>format(swissNumberProto, <span class="pl-smi">PhoneNumberFormat</span><span class="pl-c1"><span class="pl-k">.</span>INTERNATIONAL</span>));
<span class="pl-c"><span class="pl-c">//</span> Produces "044 668 18 00"</span>
<span class="pl-smi">System</span><span class="pl-k">.</span>out<span class="pl-k">.</span>println(phoneUtil<span class="pl-k">.</span>format(swissNumberProto, <span class="pl-smi">PhoneNumberFormat</span><span class="pl-c1"><span class="pl-k">.</span>NATIONAL</span>));
<span class="pl-c"><span class="pl-c">//</span> Produces "+41446681800"</span>
<span class="pl-smi">System</span><span class="pl-k">.</span>out<span class="pl-k">.</span>println(phoneUtil<span class="pl-k">.</span>format(swissNumberProto, <span class="pl-smi">PhoneNumberFormat</span><span class="pl-c1"><span class="pl-k">.</span>E164</span>));</pre>
         </div> 
         <p>You could also choose to format the number in the way it is dialed from another country:</p> 
         <div class="highlight highlight-source-java">
          <pre><span class="pl-c"><span class="pl-c">//</span> Produces "011 41 44 668 1800", the number when it is dialed in the United States.</span>
<span class="pl-smi">System</span><span class="pl-k">.</span>out<span class="pl-k">.</span>println(phoneUtil<span class="pl-k">.</span>formatOutOfCountryCallingNumber(swissNumberProto, <span class="pl-s"><span class="pl-pds">"</span>US<span class="pl-pds">"</span></span>));</pre>
         </div> 
         <h2><a href="#formatting-phone-numbers-as-you-type" aria-hidden="true" class="anchor" id="user-content-formatting-phone-numbers-as-you-type">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>Formatting Phone Numbers 'as you type'</h2> 
         <div class="highlight highlight-source-java">
          <pre><span class="pl-smi">PhoneNumberUtil</span> phoneUtil <span class="pl-k">=</span> <span class="pl-smi">PhoneNumberUtil</span><span class="pl-k">.</span>getInstance();
<span class="pl-smi">AsYouTypeFormatter</span> formatter <span class="pl-k">=</span> phoneUtil<span class="pl-k">.</span>getAsYouTypeFormatter(<span class="pl-s"><span class="pl-pds">"</span>US<span class="pl-pds">"</span></span>);
<span class="pl-smi">System</span><span class="pl-k">.</span>out<span class="pl-k">.</span>println(formatter<span class="pl-k">.</span>inputDigit(<span class="pl-s"><span class="pl-pds">'</span>6<span class="pl-pds">'</span></span>));  <span class="pl-c"><span class="pl-c">//</span> Outputs "6"</span>
<span class="pl-c1">...</span>  <span class="pl-c"><span class="pl-c">//</span> Input more digits</span>

         </div>

            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>

          <pre><span class="pl-smi">PhoneNumberOfflineGeocoder</span> geocoder <span class="pl-k">=</span> <span class="pl-smi">PhoneNumberOfflineGeocoder</span><span class="pl-k">.</span>getInstance();

         <p>Caveat: We do not provide data about the current carrier of a phone number, only the original carrier who is assigned the corresponding range. Read about <a href="/googlei18n/libphonenumber/blob/master/FAQ.md#what-is-mobile-number-portability">number portability</a>.</p>

          <pre><span class="pl-smi">PhoneNumber</span> swissMobileNumber <span class="pl-k">=</span>
