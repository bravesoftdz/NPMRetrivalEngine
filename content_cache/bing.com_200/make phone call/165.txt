Especially, if we look at cellular phones, they use a different standard called "Wireless Markup Language " (WML) for consuming web pages. By using ASP.NET Mobile Software Development kit (Mobile SDK), a developer can create web pages with the same norms and constructs that are required for building any traditional page. </p> <p>This article is aimed to explore some dimensions of ASP.NET Mobile SDK.         </ol> <h2>Variance across different mobile devices</h2> <p>There is inherent difficulty while we deal with mobile devices. They have limitations and constraints. Some of them can use JavaScript and many of them do not. Some of them can display HTML pages but most cell phones can not. They have to rely on WML standard for accessing web. So, making a website that supports all of these devices is difficult. </p> <p>When you study their limitations in details, you would come to know that designing PDA accessible websites is relatively easy. Because: </p> <p>1. PDA's can render HTML pages and display many existing websites without problems.<br>2. Many of them support JavaScript.<br>3. Many of them support cookies and Image files.<br>4. However, their main limitation is smaller screen than normal PC's have. On the contrast, dealing with cellular phones is not very easy. Reasons being: <br>5. Many cell phones can't render HTML pages. <br>6. There are some high-end cellular phones which can render HTML but they can support only a limited subset of HTML standard. <br>7. Many of them do not support JavaScript, Cookies or Image files. </p> <h2>What is WML &amp; XML?</h2> <p>Generally, cell phones can consume WML files only. WML is Wireless Markup Language which is based on XML (eXtensible Markup Language). XML in contrast to HTML supports user defined tags. It supports a natural way of representing data. XML files have schema associated with them which tells the structure in which these files are made up of. Do not worry if you don't know XML in detail. However, you are encouraged to take a look at it ( <a href="http://www.w3schools.com/" target="_blank">www.w3schools.com </a>). </p> <p>The code below shows a sample WML page. </p> <p> </p>

You should use emulators to test the web pages you design for mobile devices. </p> <p>Two emulators have been developed by Microsoft. One is for Cellular phones and the other one is for PDA's.
