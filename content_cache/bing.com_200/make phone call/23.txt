       <h2>Example: A Simple Java Program</h2> 
       <p>The code shown in Listing 1 is a simple Java program called <tt>WelcomeMsg.java</tt> that asks the user to enter a name. After entering the name, when the user presses Enter, a welcome message appears, including the name the user typed. For example, if the user enters the name "Kelly," the message, "Welcome Kelly!" appears on the screen.</p> 
       <h4>Listing 1 Code in <tt>WelcomeMsg.java</tt>.</h4> 
       <pre>import java.io.* ;                                                        #1

class WelcomeMsg {
     public static void main(String args[])
     {
          InputStreamReader istream = new InputStreamReader(System.in) ;  #2
          BufferedReader bufRead = new BufferedReader(istream) ;          #3
          try {                                                           #4
               System.out.println("Enter your name: ");                   #5
               String name = bufRead.readLine();                          #6
               System.out.println("Welcome "+ name + " !");               #7
          }
          catch (IOException err) {                                       #8
              System.out.println("Error reading line");
          }
     }
}</pre> 
       <p> In <tt>WelcomeMsg.java</tt>, statement #1 imports the packages required for input/output operations. Statement #2 creates a reader called <tt>istream</tt>. The reader will read from the standard input stream (<tt>System.in</tt>); that is, from the keyboard. The entered text is stored in the form of Unicode characters. Statement #3 uses the <tt>istream</tt> reader to convert the data into string form. Statement #4 defines a <tt>try..catch</tt> block to check for any errors that might have occurred while the reader entered the text. Statements #5 and #6 ask the user to enter a name that is then assigned to the string <tt>name</tt>. The entered name is then displayed on the screen by statement #8.</p> 
       <p> When the program is run, it asks the user to enter a name. After the user types a name and presses Enter, the welcome message is displayed with the entered name, as shown in Figure 1.</p>

       <h2>Creating an Android Project Similar to the Java Program</h2> 
       <p>Now let's create the same application in Android. Assuming that the Java Development Kit (JDK) is installed on your machine, download <a href="http://developer.android.com/sdk/installing/studio.html">Android Studio</a>. Then follow these steps:</p> 
       <ol> 
        <li>Double-click the downloaded executable file to install Android Studio.</li> 
        <li>After the installation completes successfully, launch Android Studio. The first screen that opens is a welcome screen, displaying icons such as New Project, Import Project, Open Project, and so on. Click the New Project icon to create a new Android project.</li> 
        <li>In the New Project dialog, enter information for the new project, as shown in Figure 2.</li> 
        <ul> 
         <li>In the Application Name box, enter the name of the Android project. Let's name the application <strong><tt>WelcomeMsg</tt></strong>. The Module Name entry is assigned automatically; by default, it's the same as the application name. To make it a unique identifier, the Package Name assigned is <tt>com.<em>yourname</em>.welcomemsg</tt>. The location to store the Android project is specified in Project Location box.</li> 
         <li>Select API 8: Android 2.2 (Froyo) from the Minimum Required SDK drop-down list, to indicate that the application requires at least API level 8 to run. Select API 17: Android 4.2 (Jelly Bean) as the Target Platform, as we expect this to be the version commonly used by your target audience. Select API 17: Android 4.2 (Jelly Bean) as the platform to compile the project. The default Theme assigned to the project is Holo Light with Dark Action Bar.</li> 
         <li>The Create Custom Launcher Icon checkbox is checked by default. This feature allows you to configure the launcher icon for the application. Because we want to create a blank activity, click the Create Activity checkbox and then click the Next button.</li> 
        </ul> 
        <div class="figure"> 
         <a href="javascript:popUp('/content/images/art_harwani_convertjavaappandroid/elementLinks/harwani1_fig02.jpg')"> <img src="/content/images/art_harwani_convertjavaappandroid/elementLinks/harwani1_fig02.jpg" width="400"></a> 
         <p><a href="javascript:popUp('/content/images/art_harwani_convertjavaappandroid/elementLinks/harwani1_fig02.jpg')">Figure 2</a> Creating a new Android project</p>

        <li>The next dialog is Configure Launcher Icon, which is used for configuring the icon for the application. Because we want to use the default icon for our application, keep the default options selected in the dialog box, and click Next.</li> 
        <li>The next dialog prompts us to create an activity. Because we want to create a blank activity, select the BlankActivity option from the list and then click Next.</li> 
        <li>The next dialog asks us to enter information about the newly created activity. Name the activity <strong><tt>WelcomeMsgActivity</tt></strong>. The layout file is assigned the default name <tt>activity_main.xml</tt>. Keeping that default name for the layout file, click Finish to create the Android project.</li> 
       </ol> 
       <p> Android Studio will automatically create several files and folders for our new Android project (see Figure 3). To customize our new project, we just need to work with two files:</p> 
       <ul> 
        <li><strong><tt>activity_main.xml</tt>.</strong> This file is shown at right in Figure 3, in the node <tt>WelcomeMsg &gt; src &gt; main &gt; res &gt; layout</tt>. It's an XML file in which we'll define the controls (<tt>TextView</tt>, <tt>EditText</tt>, <tt>Button</tt>, etc.) for our Android app's graphical user interface. Through this file, the user interacts with the Android project.</li> 
        <li><strong><tt>WelcomeMsgActivity.java</tt>.</strong> This file is in the node <tt>WelcomeMsg &gt; src &gt; main &gt; java &gt; com.<em>yourname</em>.welcomemsg</tt>. This Java file loads the controls defined in the layout file <tt>activity_main.xml</tt>, listens for various events, and runs the required code if any of those events occur.</li> 
       </ul>

       <h2>Organizing Controls in the Layout Container</h2> 
       <p>The most popular approach for creating an Android user interface is via an XML file. The controls or views defined in the XML file for use in the Android app are laid out or organized in a <em>layout container</em>. Android provides several layout containers: <tt>RelativeLayout</tt>, <tt>LinearLayout</tt>, <tt>GridLayout</tt>, <tt>FrameLayout</tt>, and so on. The default arrangement is <tt>RelativeLayout</tt>. The simplest is <tt>LinearLayout</tt>, which displays the controls linearly, one below the other. (The default orientation of <tt>LinearLayout</tt> is vertical.)</p> 
       <p> To define the controls for your new Android project, open the <tt>activity_main.xml</tt> file (also known as the <em>activity layout file</em>) by double-clicking it in the Project window; then replace the default content with the code shown in Listing 2.</p> 
       <h4>Listing 2 Code for <tt>activity_main.xml</tt> file.</h4> 
       <pre>&lt;LinearLayout
        xmlns:android="http://schemas.android.com/apk/res/android"
        android:orientation="vertical"
        android:layout_width="match_parent"
        android:layout_height="match_parent"&gt;
    &lt;TextView
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:text="Enter your name:"
            android:textSize="24dp"/&gt;
    &lt;EditText
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:id="@+id/user_name"
            android:textSize="24dp" /&gt;
    &lt;Button
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:id="@+id/click_btn"
            android:text="Click Me"/&gt;
    &lt;TextView
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:id="@+id/response"
            android:textSize="24dp" /&gt;
&lt;/LinearLayout&gt;</pre> 
       <p> The layout file in Listing 2 uses <tt>TextView</tt>, <tt>EditText</tt>, and <tt>Button</tt> controls:</p> 
       <ul> 
        <li>The <tt>TextView</tt> control displays a text message on the screen in the specified font, size, and color.</li> 
        <li>The <tt>EditText</tt> control displays a blank textbox to accept data from the user.</li> 
        <li>The <tt>Button</tt> control in most applications displays a button which, when clicked, initiates the desired action. The caption displayed on this <tt>Button</tt> control is "Click Me."</li> 
       </ul> 
       <p> Because controls are created by hand in XML, there's no default caption for a <tt>Button</tt>. If you don't assign a caption, the button will appear with no text on it.</p> 
       <p> In Listing 2, the <tt>EditText</tt>, <tt>Button</tt>, and the second <tt>TextView</tt> controls are assigned the IDs <tt>user_name</tt>, <tt>click_btn</tt>, and <tt>response</tt>, respectively. These IDs will be used to identify and access the corresponding controls in the Java code file. The last <tt>TextView</tt> control in Listing 2 displays a welcome message to the user.</p> 
       <p> When laid out in a <tt>LinearLayout</tt> container, these controls would appear as shown in Figure 4.</p> 
       <div class="figure"> 
        <a href="javascript:popUp('/content/images/art_harwani_convertjavaappandroid/elementLinks/harwani1_fig04.jpg')"> <img src="/content/images/art_harwani_convertjavaappandroid/elementLinks/harwani1_fig04.jpg " width="300"></a> 
        <p><a href="javascript:popUp('/content/images/art_harwani_convertjavaappandroid/elementLinks/harwani1_fig04.jpg')">Figure 4</a> <tt>TextView</tt>, <tt>EditText</tt>, and <tt>Button</tt> controls arranged in a <tt>LinearLayout</tt> container</p> 
       </div> 
       <p> The title <tt>5554:PhoneAVD</tt> in Figure 4 refers to the Android Virtual Device (AVD) in which we'll run our Android app. Android Virtual Devices are explained later in this article.</p> 
       <h2>Writing Java Code to Load the Layout File</h2> 
       <p>The controls defined in the activity layout file need to be loaded to display them on the screen. In addition to loading the controls, we want to perform certain tasks such as associating events to controls, listening for events, and taking action when events occur. To do all these tasks, write the Java code shown in Listing 3 in the Java activity file <tt>WelcomeMsgActivity.java</tt>.</p> 
       <h4>Listing 3 Code in <tt>WelcomeMsgActivity.java</tt> file.</h4> 
       <pre>package com.bmharwani.welcomemsg;

import android.os.Bundle;
import android.app.Activity;
import android.view.Menu;
import android.widget.TextView;
import android.widget.EditText;
import android.widget.Button;
import android.view.View;

public class WelcomeMsgActivity extends Activity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);                             #1
        Button b = (Button)this.findViewById(R.id.click_btn);               #2
        b.setOnClickListener(new Button.OnClickListener(){                  #3
            public void onClick(View v)  {
                TextView resp = (TextView) findViewById(R.id.response);     #4
                EditText name = (EditText) findViewById(R.id.user_name);    #5
                String str = "Welcome " + name.getText().toString() + " !"; #6
                resp.setText(str);                                          #7
            }
        });

    }
}</pre> 
       <p> In this Java activity file, the following tasks are performed:</p> 
       <ul> 
        <li>Loading the controls that are laid in the activity layout file <tt>activity_main.xml</tt> through statement #1.</li> 
        <li>Accessing the <tt>Button</tt> control from the activity layout file using its ID, (<tt>click_btn</tt>) and mapping it to the <tt>Button</tt> object <tt>b</tt> (statement #2).</li> 
        <li>Associating a <tt>click</tt> event with the <tt>Button</tt> control and listening for the occurrence of the <tt>click</tt> event (statement #3).</li> 
        <li>Accessing the <tt>TextView</tt> control from the activity layout file using its ID, <tt>response</tt>, and mapping it to the <tt>TextView</tt> object <tt>resp</tt> (statement #4).</li> 
        <li>Accessing the <tt>EditText</tt> control from the activity layout file using its ID, <tt>user_name</tt>, and mapping it to the <tt>EditText</tt> object <tt>name</tt> (statement #5).</li> 
        <li>Creating a <tt>String</tt>, <tt>str</tt>, that contains a welcome message along with the user name entered in the <tt>EditText</tt> control (statement #6).</li> 
        <li>Displaying the string in <tt>str</tt> through the <tt>TextView</tt> object <tt>resp</tt> (statement #7).</li> 
       </ul> 
       <p> We can run the Android project on a physical device as well on a virtual device. For this example, we'll run the Android project by creating an Android Virtual Device (AVD) as described in the next section.</p> 
       <h2>Creating an Android Virtual Device (AVD)</h2> 
       <p>There are many Android devices, each with its own configuration. To test whether your Android application is compatible with a set of Android devices, we'll create an Android Virtual Device to represent that device configuration.</p> 
       <ol> 
        <li>To create an AVD, select <tt>Tools &gt; Android &gt; AVD Manager</tt>. An Android Virtual Device Manager dialog opens, displaying a list of existing AVDs. You can use this dialog to work with an existing AVD or create a new AVD.</li> 
        <li>Click the New button to create a new AVD. The Create New Android Virtual Device (AVD) dialog box appears.</li> 
        <li>Fill in the dialog box as shown in Figure 5.</li> 
        <ul> 
         <li><strong>AVD Name.</strong> Specify the name of the AVD. For this example, use the name <strong>PhoneAVD</strong>.</li> 
         <li><strong>Device.</strong> Specify the device for which the application has to be tested. For this example, select the device <strong>5.1” WVGA (480 × 800: mdpi)</strong>.</li> 
         <li><strong>Target.</strong> Specify the target API level. Our application will be tested against the specified API, so let's set the target to the latest API, <strong>Android 4.2.2 - API Level 17</strong>.</li> 
         <li><strong>CPU/ABI.</strong> Indicates the processor that we want to emulate on our device. Select the <strong>ARM (armeabi-v7a)</strong> option.</li> 
         <li><strong>Keyboard.</strong> To use the physical keyboard on the computer along with the one displayed on the emulator screen, check the Hardware Keyboard Present checkbox.</li> 
         <li><strong>Skin.</strong> Select the Display a Skin with Hardware Controls checkbox. The emulator will be displayed along with physical device buttons on the right side. The controls include basics such as speaker and on/off buttons, Home, Menu, Back, and Search buttons.</li> 
         <li><strong>Front Camera/Back Camera.</strong> If a webcam is attached to your computer and you want to use it in your application, select <strong>webcam0</strong> from the drop-down menu. Choose an emulated option if you don't have a webcam. Leave the default None if the application doesn't require a camera.</li> 
         <li><strong>Memory Options.</strong> Define the device RAM and VM Heap settings. Leave the default values here.</li> 
         <li><strong>Internal Storage.</strong> Define the internal storage of the device. Leave the default value (200 MiB).</li> 
         <li><strong>SD Card.</strong> Extends the storage capacity of the device. Large data files, such as audio and video (for which the built-in flash memory is insufficient), are stored on the SD Card. Let's set the size of the SD card to <strong>128 MiB</strong>. The larger the allocated SD card space, the longer the AVD will take to create. Unless it's really required, keep the SD card space as low as possible.</li> 
         <li><strong>Snapshot.</strong> Enable this option to avoid booting the emulator and start from the last saved snapshot. This option is used to start the Android emulator quickly.</li> 
         <li><strong>Use Host GPU.</strong> This option enables GPU emulation, which improves the emulator's performance.</li> 
        </ul> 
        <div class="figure"> 
         <a href="javascript:popUp('/content/images/art_harwani_convertjavaappandroid/elementLinks/harwani1_fig05.jpg')"> <img src="/content/images/art_harwani_convertjavaappandroid/elementLinks/harwani1_fig05.jpg" width="400"></a> 
         <p><a href="javascript:popUp('/content/images/art_harwani_convertjavaappandroid/elementLinks/harwani1_fig05.jpg')">Figure 5</a> Creating a new Android Virtual Device (left). The newly created AVD is displayed in the Android Virtual Device Manager (right)</p> 
        </div> 
        <li>After entering the data, click OK. The newly created AVD, <tt>PhoneAVD</tt>, is created and displayed in the list of existing AVDs, as shown at right in Figure 5. Click the Refresh button if the new AVD doesn't appear in the list immediately.</li> 
       </ol> 
       <p>Our new application is ready to run! Click the Run icon on the toolbar, press Shift-F10, or select Run &gt; Run. You'll be prompted to select the device or AVD on which to show the output. Select the PhoneAVD from the Android virtual device drop-down list, and then click OK. The Android project will run, asking you to enter a name. After typing a name, click the Click Me button, and the new welcome message will be displayed (see Figure 6).</p> 
       <div class="figure"> 
        <a href="javascript:popUp('/content/images/art_harwani_convertjavaappandroid/elementLinks/harwani1_fig06.jpg')"> <img src="/content/images/art_harwani_convertjavaappandroid/elementLinks/harwani1_fig06.jpg" width="400"></a> 
        <p><a href="javascript:popUp('/content/images/art_harwani_convertjavaappandroid/elementLinks/harwani1_fig06.jpg')">Figure 6</a> Select the device or AVD to show the project output (left) and enter a name in response to the prompt. Your new Android project shows the welcome message (right)</p> 
       </div>

       <p>You've learned how easily you can convert any Java application into an Android application. Just define the user interface in XML; then access and process the data entered in the user interface controls through the Java code written in the Activity file.</p>
