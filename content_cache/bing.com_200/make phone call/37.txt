     <div class="doc-content"> 
      <p class="small feature">This recipe shows how to launch the phone dialer application.</p> 
      <p><a name="Recipe" class="injected"></a></p> 
      <p>Each screen in an application is represented by an activity. Using an asynchronous message called an <em>intent</em> causes the system to load another activity (and another screen) to handle that type of intent. An intent that is created from a <code>Uri</code> causes the system to load an activity that can handle the <code>Uri</code> scheme. In this recipe, a <code>Uri</code> beginning with <code>tel:</code> loads an activity in the phone dialer application, and the number following the <code>tel:</code> prefix is automatically loaded into the dialer.</p> 
      <h1>Recipe</h1> 
      <p><a href="Images/01-main-activity.png" class=" fancybox"> <img src="Images/01-main-activity.png" alt=""></a> <a href="Images/02-dialer-activity.png" class=" fancybox"> <img src="Images/02-dialer-activity.png" alt=""></a></p> 
      <ol> 
       <li><p>Create a new Xamarin.Android application. The project template will create a single activity named <em>Activity1</em> (<strong>MainActivity.cs</strong>), which contains a button.</p></li> 
       <li><p>From the <code>button.Click</code> handler in <strong>MainActivity.cs</strong>, create a <code>Uri</code> and a dialer <code>Intent</code>.</p></li> 
       <li><p>Call <code>StartActivity</code> to launch the intent.</p></li> 
      </ol> 
      <p>The following code sample illustrates the updated <code>button.Click</code> handler:</p> 
      <pre><code syntax="C#" class=" syntax brush-C#">intent.button.Click += delegate {
   var uri = Android.Net.Uri.Parse ("tel:1112223333");
   var intent = new Intent (Intent.ActionDial, uri);
   StartActivity (intent);
};</code></pre> 
      <p>In this code example, calling <code>StartActivity</code> and passing it the <code>ActionDial</code> intent launches the phone dialer app. The dialer activity in the phone dialer app is automatically loaded with the number <code>1112223333</code> when it starts - the user can view this number before pressing the <em>Call</em> button. The <code>ActionCall</code> intent is not used here because <code>ActionCall</code> would initiate the call directly; instead, we use the <code>ActionDial</code> intent so that the user can control the call from the dialer. Note that you do not have to set the <code>CALL_PHONE</code> permission in order to use the <code>ActionDial</code> intent.</p>
