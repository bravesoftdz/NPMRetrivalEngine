    <h1 itemprop="name">Building Web Apps in WebView</h1>

      <h2>Quickview</h2> 
      <ul> 
       <li>Use <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> to display web pages in your Android application layout</li>

      <h2>Key classes</h2> 
      <ol> 
       <li><code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code></li> 
       <li><code><a href="https://developer.android.com/reference/android/webkit/WebSettings.html">WebSettings</a></code></li> 
       <li><code><a href="https://developer.android.com/reference/android/webkit/WebViewClient.html">WebViewClient</a></code></li> 
      </ol> 
     </div> 
    </div> 
    <p>If you want to deliver a web application (or just a web page) as a part of a client application, you can do it using <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code>. The <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> class is an extension of Android's <code><a href="https://developer.android.com/reference/android/view/View.html">View</a></code> class that allows you to display web pages as a part of your activity layout. It does <em>not</em> include any features of a fully developed web browser, such as navigation controls or an address bar. All that <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> does, by default, is show a web page.</p> 
    <p>A common scenario in which using <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> is helpful is when you want to provide information in your application that you might need to update, such as an end-user agreement or a user guide. Within your Android application, you can create an <code><a href="https://developer.android.com/reference/android/app/Activity.html">Activity</a></code> that contains a <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code>, then use that to display your document that's hosted online.</p> 
    <p>Another scenario in which <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> can help is if your application provides data to the user that always requires an Internet connection to retrieve data, such as email. In this case, you might find that it's easier to build a <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> in your Android application that shows a web page with all the user data, rather than performing a network request, then parsing the data and rendering it in an Android layout. Instead, you can design a web page that's tailored for Android devices and then implement a <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> in your Android application that loads the web page.</p> 
    <p>This document shows you how to get started with <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> and how to do some additional things, such as handle page navigation and bind JavaScript from your web page to client-side code in your Android application.</p> 
    <h2 id="AddingWebView">Adding a WebView to Your Application</h2> 
    <p>To add a <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> to your Application, simply include the <code>&lt;WebView&gt;</code> element in your activity layout. For example, here's a layout file in which the <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> fills the screen:</p> 
    <pre>
&lt;?xml version="1.0" encoding="utf-8"?&gt;
&lt;WebView  xmlns:android="http://schemas.android.com/apk/res/android"
    android:id="@+id/webview"
    android:layout_width="fill_parent"
    android:layout_height="fill_parent"
/&gt;
</pre> 
    <p>To load a web page in the <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code>, use <code><a href="https://developer.android.com/reference/android/webkit/WebView.html#loadUrl(java.lang.String)">loadUrl()</a></code>. For example:</p> 
    <pre>
WebView myWebView = (WebView) findViewById(R.id.webview);
myWebView.loadUrl("http://www.example.com");
</pre> 
    <p>Before this will work, however, your application must have access to the Internet. To get Internet access, request the <code><a href="https://developer.android.com/reference/android/Manifest.permission.html#INTERNET">INTERNET</a></code> permission in your manifest file. For example:</p> 
    <pre>
&lt;manifest ... &gt;
    &lt;uses-permission android:name="android.permission.INTERNET" /&gt;
    ...
&lt;/manifest&gt;
</pre> 
    <p>That's all you need for a basic <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> that displays a web page.</p> 
    <h2 id="UsingJavaScript">Using JavaScript in WebView</h2> 
    <p>If the web page you plan to load in your <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> use JavaScript, you must enable JavaScript for your <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code>. Once JavaScript is enabled, you can also create interfaces between your application code and your JavaScript code.</p> 
    <h3 id="EnablingJavaScript">Enabling JavaScript</h3> 
    <p>JavaScript is disabled in a <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> by default. You can enable it through the <code><a href="https://developer.android.com/reference/android/webkit/WebSettings.html">WebSettings</a></code> attached to your <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code>. You can retrieve <code><a href="https://developer.android.com/reference/android/webkit/WebSettings.html">WebSettings</a></code> with <code><a href="https://developer.android.com/reference/android/webkit/WebView.html#getSettings()">getSettings()</a></code>, then enable JavaScript with <code><a href="https://developer.android.com/reference/android/webkit/WebSettings.html#setJavaScriptEnabled(boolean)">setJavaScriptEnabled()</a></code>.</p> 
    <p>For example:</p> 
    <pre>
WebView myWebView = (WebView) findViewById(R.id.webview);
WebSettings webSettings = myWebView.getSettings();
webSettings.setJavaScriptEnabled(true);
</pre> 
    <p><code><a href="https://developer.android.com/reference/android/webkit/WebSettings.html">WebSettings</a></code> provides access to a variety of other settings that you might find useful. For example, if you're developing a web application that's designed specifically for the <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> in your Android application, then you can define a custom user agent string with <code><a href="https://developer.android.com/reference/android/webkit/WebSettings.html#setUserAgentString(java.lang.String)">setUserAgentString()</a></code>, then query the custom user agent in your web page to verify that the client requesting your web page is actually your Android application.</p> 
    <h3 id="BindingJavaScript">Binding JavaScript code to Android code</h3> 
    <p>When developing a web application that's designed specifically for the <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> in your Android application, you can create interfaces between your JavaScript code and client-side Android code. For example, your JavaScript code can call a method in your Android code to display a <code><a href="https://developer.android.com/reference/android/app/Dialog.html">Dialog</a></code>, instead of using JavaScript's <code>alert()</code> function.</p> 
    <p>To bind a new interface between your JavaScript and Android code, call <code><a href="https://developer.android.com/reference/android/webkit/WebView.html#addJavascriptInterface(java.lang.Object, java.lang.String)">addJavascriptInterface()</a></code>, passing it a class instance to bind to your JavaScript and an interface name that your JavaScript can call to access the class.</p> 
    <p>For example, you can include the following class in your Android application:</p> 
    <pre>
public class WebAppInterface {
    Context mContext;

    /** Instantiate the interface and set the context */
    WebAppInterface(Context c) {
        mContext = c;
    }

    /** Show a toast from the web page */
    @JavascriptInterface
    public void showToast(String toast) {
        Toast.makeText(mContext, toast, Toast.LENGTH_SHORT).show();
    }
}
</pre> 
    <p class="caution"><strong>Caution:</strong> If you've set your <a href="https://developer.android.com/guide/topics/manifest/uses-sdk-element.html#target"><code>targetSdkVersion</code></a> to 17 or higher, <strong>you must add the <code>@JavascriptInterface</code> annotation</strong> to any method that you want available to your JavaScript (the method must also be public). If you do not provide the annotation, the method is not accessible by your web page when running on Android 4.2 or higher.</p> 
    <p>In this example, the <code>WebAppInterface</code> class allows the web page to create a <code><a href="https://developer.android.com/reference/android/widget/Toast.html">Toast</a></code> message, using the <code>showToast()</code> method.</p> 
    <p>You can bind this class to the JavaScript that runs in your <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> with <code><a href="https://developer.android.com/reference/android/webkit/WebView.html#addJavascriptInterface(java.lang.Object, java.lang.String)">addJavascriptInterface()</a></code> and name the interface <code>Android</code>. For example:</p> 
    <pre>
WebView webView = (WebView) findViewById(R.id.webview);
webView.addJavascriptInterface(new WebAppInterface(this), "Android");
</pre> 
    <p>This creates an interface called <code>Android</code> for JavaScript running in the <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code>. At this point, your web application has access to the <code>WebAppInterface</code> class. For example, here's some HTML and JavaScript that creates a toast message using the new interface when the user clicks a button:</p> 
    <pre>
&lt;input type="button" value="Say hello" onClick="showAndroidToast('Hello Android!')" /&gt;

&lt;script type="text/javascript"&gt;
    function showAndroidToast(toast) {
        Android.showToast(toast);
    }
&lt;/script&gt;
</pre> 
    <p>There's no need to initialize the <code>Android</code> interface from JavaScript. The <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> automatically makes it available to your web page. So, at the click of the button, the <code>showAndroidToast()</code> function uses the <code>Android</code> interface to call the <code>WebAppInterface.showToast()</code> method.</p> 
    <p class="note"><strong>Note:</strong> The object that is bound to your JavaScript runs in another thread and not in the thread in which it was constructed.</p> 
    <p class="caution"><strong>Caution:</strong> Using <code><a href="https://developer.android.com/reference/android/webkit/WebView.html#addJavascriptInterface(java.lang.Object, java.lang.String)">addJavascriptInterface()</a></code> allows JavaScript to control your Android application. This can be a very useful feature or a dangerous security issue. When the HTML in the <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> is untrustworthy (for example, part or all of the HTML is provided by an unknown person or process), then an attacker can include HTML that executes your client-side code and possibly any code of the attacker's choosing. As such, you should not use <code><a href="https://developer.android.com/reference/android/webkit/WebView.html#addJavascriptInterface(java.lang.Object, java.lang.String)">addJavascriptInterface()</a></code> unless you wrote all of the HTML and JavaScript that appears in your <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code>. You should also not allow the user to navigate to other web pages that are not your own, within your <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> (instead, allow the user's default browser application to open foreign links—by default, the user's web browser opens all URL links, so be careful only if you handle page navigation as described in the following section).</p> 
    <h2 id="HandlingNavigation">Handling Page Navigation</h2> 
    <p>When the user clicks a link from a web page in your <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code>, the default behavior is for Android to launch an application that handles URLs. Usually, the default web browser opens and loads the destination URL. However, you can override this behavior for your <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code>, so links open within your <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code>. You can then allow the user to navigate backward and forward through their web page history that's maintained by your <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code>.</p> 
    <p>To open links clicked by the user, simply provide a <code><a href="https://developer.android.com/reference/android/webkit/WebViewClient.html">WebViewClient</a></code> for your <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code>, using <code><a href="https://developer.android.com/reference/android/webkit/WebView.html#setWebViewClient(android.webkit.WebViewClient)">setWebViewClient()</a></code>. For example:</p> 
    <pre>
WebView myWebView = (WebView) findViewById(R.id.webview);
myWebView.<code><a href="https://developer.android.com/reference/android/webkit/WebView.html#setWebViewClient(android.webkit.WebViewClient)">setWebViewClient</a></code>(new WebViewClient());
</pre> 
    <p>That's it. Now all links the user clicks load in your <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code>.</p> 
    <p>If you want more control over where a clicked link load, create your own <code><a href="https://developer.android.com/reference/android/webkit/WebViewClient.html">WebViewClient</a></code> that overrides the <code><a href="https://developer.android.com/reference/android/webkit/WebViewClient.html#shouldOverrideUrlLoading(android.webkit.WebView, java.lang.String)">shouldOverrideUrlLoading()</a></code> method. For example:</p> 
    <pre>
private class MyWebViewClient extends WebViewClient {
    @Override
    public boolean <code><a href="https://developer.android.com/reference/android/webkit/WebViewClient.html#shouldOverrideUrlLoading(android.webkit.WebView, java.lang.String)">shouldOverrideUrlLoading</a></code>(WebView view, String url) {
        if (Uri.parse(url).getHost().equals("www.example.com")) {
            // This is my web site, so do not override; let my WebView load the page
            return false;
        }
        // Otherwise, the link is not for a page on my site, so launch another Activity that handles URLs
        Intent intent = new Intent(Intent.ACTION_VIEW, Uri.parse(url));
        startActivity(intent);
        return true;
    }
}
</pre> 
    <p>Then create an instance of this new <code><a href="https://developer.android.com/reference/android/webkit/WebViewClient.html">WebViewClient</a></code> for the <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code>:</p> 
    <pre>
WebView myWebView = (WebView) findViewById(R.id.webview);
myWebView.<code><a href="https://developer.android.com/reference/android/webkit/WebView.html#setWebViewClient(android.webkit.WebViewClient)">setWebViewClient</a></code>(new MyWebViewClient());
</pre> 
    <p>Now when the user clicks a link, the system calls <code><a href="https://developer.android.com/reference/android/webkit/WebViewClient.html#shouldOverrideUrlLoading(android.webkit.WebView, java.lang.String)">shouldOverrideUrlLoading()</a></code>, which checks whether the URL host matches a specific domain (as defined above). If it does match, then the method returns false in order to <em>not</em> override the URL loading (it allows the <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> to load the URL as usual). If the URL host does not match, then an <code><a href="https://developer.android.com/reference/android/content/Intent.html">Intent</a></code> is created to launch the default Activity for handling URLs (which resolves to the user's default web browser).</p> 
    <h3 id="NavigatingHistory">Navigating web page history</h3> 
    <p>When your <code><a href="https://developer.android.com/reference/android/webkit/WebView.html">WebView</a></code> overrides URL loading, it automatically accumulates a history of visited web pages. You can navigate backward and forward through the history with <code><a href="https://developer.android.com/reference/android/webkit/WebView.html#goBack()">goBack()</a></code> and <code><a href="https://developer.android.com/reference/android/webkit/WebView.html#goForward()">goForward()</a></code>.</p> 
    <p>For example, here's how your <code><a href="https://developer.android.com/reference/android/app/Activity.html">Activity</a></code> can use the device <em>Back</em> button to navigate backward:</p> 
    <pre>
@Override
public boolean <code><a href="https://developer.android.com/reference/android/app/Activity.html#onKeyDown(int, android.view.KeyEvent)">onKeyDown</a></code>(int keyCode, KeyEvent event) {
    // Check if the key event was the Back button and if there's history
    if ((keyCode == KeyEvent.KEYCODE_BACK) &amp;&amp; myWebView.<code><a href="https://developer.android.com/reference/android/webkit/WebView.html#canGoBack()">canGoBack</a></code>()) {
        myWebView.<code><a href="https://developer.android.com/reference/android/webkit/WebView.html#goBack()">goBack</a></code>();
        return true;
    }
    // If it wasn't the Back key or there's no web page history, bubble up to the default
    // system behavior (probably exit the activity)
    return super.onKeyDown(keyCode, event);
}
</pre> 
    <p>The <code><a href="https://developer.android.com/reference/android/webkit/WebView.html#canGoBack()">canGoBack()</a></code> method returns true if there is actually web page history for the user to visit. Likewise, you can use <code><a href="https://developer.android.com/reference/android/webkit/WebView.html#canGoForward()">canGoForward()</a></code> to check whether there is a forward history. If you don't perform this check, then once the user reaches the end of the history, <code><a href="https://developer.android.com/reference/android/webkit/WebView.html#goBack()">goBack()</a></code> or <code><a href="https://developer.android.com/reference/android/webkit/WebView.html#goForward()">goForward()</a></code> does nothing.</p>
