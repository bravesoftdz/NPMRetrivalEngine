      <h1 class="post-title"><a href="https://blogs.windows.com/buildingapps/2014/10/23/apache-cordova-and-winjs/">Apache Cordova and WinJS</a></h1> 
      <div class="author">
        By 
       <a href="https://blogs.windows.com/buildingapps/author/windowsappsteam/" title="Posts by Windows Apps Team" class="author url fn" rel="author">Windows Apps Team</a> 
      </div> 
      <div id="addthis-8173" class="addthis_toolbox addthis_default_style expanded" addthis:url="https://blogs.windows.com/buildingapps/2014/10/23/apache-cordova-and-winjs/" addthis:title="Apache Cordova and WinJS"> 
       <a class="addthis_button_facebook"><i class="icon icon-facebook"></i> <span>Share</span></a> 
       <a class="addthis_button_twitter"><i class="icon icon-twitter"></i> <span>Tweet</span></a> 
       <a class="addthis_button_reddit"><i class="icon icon-reddit"></i> <span>Share</span></a> 
       <a class="addthis_button_linkedin"><i class="icon icon-linkedin"></i> <span>Share</span></a> 
       <a class="addthis_button_skype" href="#"><i class="icon icon-skype"></i> <span>Skype</span></a> 
      </div> 
     </header> 
     <p><em>This post was written by Josh Rennert, Program Manager, Windows Ecosystem &amp; Frameworks team</em></p> 
     <p><a href="http://cordova.apache.org/">Apache Cordova</a> enables developers to build native applications by using the most common web programming languages: HTML, CSS, and JavaScript. Tapping into its set of APIs—colloquially known as <a href="http://plugins.cordova.io/">plugins</a>—permits access to the native device functionality within iOS, Android, Windows, and Windows Phone. With the <a href="http://cordova.apache.org/announcements/2014/09/22/cordova-361.html">release</a> of Apache Cordova 3.6, <a href="http://msopentech.com/">Microsoft Open Technologies</a> (MS Open Tech) has <a href="http://msopentech.com/blog/2014/09/25/apache-cordova-gains-windows-8-1-and-windows-phone-8-1-support-2-2/">increased support</a> for Microsoft platforms to include not only Windows 8 and Windows Phone 8, but also their <a href="http://cordova.apache.org/docs/en/3.6.0/guide_platforms_win8_index.md.html#Windows%20Platform%20Guide">8.1 counterparts</a>, through the advent of <a href="http://msdn.microsoft.com/en-us/library/windows/apps/dn609832.aspx">universal Windows apps</a>. Moreover, a developer can reach a wide array of devices, and their capabilities, simply by using one set of shared app code.</p> 
     <p>We recently <a href="http://blogs.windows.com/buildingapps/2014/09/17/winjs-everywhere/">announced</a> the general availability of <a href="http://try.buildwinjs.com/#get">WinJS 3.0</a>, featuring cross-browser and cross-platform support, improved universal control designs, and JavaScript modularization. WinJS is now a prime candidate for building quality mobile and desktop apps, consistent across the breadth of device platforms, including the web.</p> 
     <p>Combining WinJS and Apache Cordova offers you a set of UI controls with high polish and performance, in addition to a cohesive set of components and utilities, which spans not only Windows but other leading platforms as well. To see an example of how the two can interact together, be sure to check out the WinJS/Cordova <a href="https://github.com/banguero/winjs-cordova-stocks-sample">Stocks sample app</a>.</p> 
     <p><iframe style="width: 640px; height: 360px;" src="https://channel9.msdn.com/Shows/Web+Camps+TV/A-Quick-Look-at-WinJS/player?h=360&amp;w=640" width="300" height="150" frameborder="0" scrolling="no" allowfullscreen></iframe></p> 
     <p>Creating a Cordova project today is fairly <a href="http://cordova.apache.org/docs/en/3.6.0/guide_overview_index.md.html#Overview">straightforward</a>. There are two workflows you can take. The first provides cross-platform compatibility. By taking advantage of your shared web assets, using the <a href="http://cordova.apache.org/docs/en/3.6.0/guide_overview_index.md.html#The%0ACommand-Line%20Interface">Command-Line Interface</a> (CLI) lets you target multiple platforms, abstracting away from the lower-level project implementations. The second focuses more on platform-specific concentration. Targeting a <a href="http://cordova.apache.org/docs/en/3.6.0/guide_platforms_index.md.html#Platform%20Guides">single platform</a> allows for additional customization and specialization you wouldn’t normally achieve with the cross-platform workflow.</p> 
     <p>Regardless of which path you choose, developing a Cordova app normally requires using a combination of the CLI and your favorite integrated development environment (IDE). But if you’re developing on a Windows device, there’s an easier way.</p> 
     <p>Released as a <a href="http://blogs.msdn.com/b/somasegar/archive/2014/05/12/mobile-first-cloud-first-development-visual-studio-apache-cordova-tooling-and-cloud-optimized-net-futures.aspx">preview</a> alongside the Visual Studio 2013 Update 2 earlier this year, the <a href="http://www.visualstudio.com/en-US/explore/cordova-vs">Multi-Device Hybrid Apps</a> extension (MDD) provides tooling support for Apache Cordova from within Visual Studio. It offers the same great end-to-end development experience—complete with IntelliSense, cross-platform debugging and emulation—while abstracting project configuration away from the command line. The MDD minimizes the barrier to entry, eliminating many of the initial steps necessary to create your first Cordova project.</p> 
     <p>The Apache Cordova community supplies a set of <a href="http://plugins.cordova.io/#/search?search=org.apache.cordova">core plugins</a>, with (currently) eleven of the most popular device and platform capabilities that are ordinarily unavailable to web-based apps. There are actually over four hundred plugins listed in the Cordova Plugin Registry. Now you can take this concept of added functionality one step further by utilizing the <a href="http://msdn.microsoft.com/en-us/library/windows/apps/br211377.aspx">Windows Runtime</a>. Indeed, many of the core plugin implementations on Windows simply call the appropriate JavaScript API.</p> 
     <p><a href="https://winblogs.azureedge.net/win/2014/10/WinJS.jpg"><img class="alignnone  wp-image-8193" src="https://winblogs.azureedge.net/win/2014/10/WinJS.jpg" alt="WinJS" width="416" height="241"></a></p> 
     <p>Using WinJS in a Cordova project is no different than a Windows Store app or web site implementation. You just reference the JavaScript file and appropriate stylesheet as you normally would, in addition to the Cordova script. We <a href="http://try.buildwinjs.com/#get">recommend</a> accessing the library through a package manager, such as <a href="http://www.nuget.org/">NuGet</a>, in order to ensure you are using the latest version.</p> 
     <p>Even though you’re using WinJS in your application, this doesn’t restrict you from taking advantage of other JavaScript libraries as well. In fact, there are shims available which facilitate the usage of WinJS controls with other libraries, such as <a href="https://github.com/codemonkeychris/angular-winjs">AngularJS</a> and <a href="https://github.com/spankyj/knockout-winjs">Knockout</a>.</p> 
     <p>Note that when using third-party JavaScript libraries in Visual Studio, you may encounter errors about dynamic content. As a result, MS Open Tech released the <a href="https://github.com/msopentech/winstore-jscompat/">JavaScript Dynamic Content shim</a> for Windows Store apps in order to unblock such setbacks. More detailed information can be found in the readme file in their GitHub <a href="https://github.com/msopentech/winstore-jscompat/">repository</a>.</p> 
     <p>With Apache Cordova and WinJS you can develop a truly cross-platform application using one consistent set of programming languages, leveraging a plethora of native features which light up on any device. Visitthe following links to learn more about using Apache Cordova and WinJS today:</p> 
     <ul> 
      <li><a href="http://msopentech.com/blog/2014/09/25/apache-cordova-gains-windows-8-1-and-windows-phone-8-1-support-2-2/">MS Open Tech blog: Apache Cordova gains Windows 8.1 and Windows Phone 8.1 support</a></li> 
      <li><a href="http://cordova.apache.org/docs/en/3.6.0/guide_platforms_win8_index.md.html#Windows%20Platform%20Guide">Apache Cordova: Windows Platform Guide</a></li> 
      <li><a href="https://github.com/banguero/winjs-cordova-stocks-sample">WinJS/Cordova Stocks sample app</a></li> 
      <li><a href="http://try.buildwinjs.com/">Try WinJS</a></li> 
      <li><a href="https://github.com/winjs/winjs">WinJS GitHub repository</a></li> 
      <li><a href="https://github.com/msopentech/winstore-jscompat">JavaScript Dynamic Content shim for Windows Store apps</a></li> 
      <li><a href="http://www.visualstudio.com/en-US/explore/cordova-vs">Multi-Device Hybrid Apps extension for Visual Studio</a></li> 
     </ul>
