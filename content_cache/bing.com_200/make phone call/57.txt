       <h1 class="page-title "> The Twilio Java Helper Library </h1> 
       <div class="block-markdown "> 
        <div> 
         <div style="border:1px solid #faebcc;background-color:#fcf8e3;color:#8a6d3b;padding:10px;"> 
          <b>Deprecation notice:</b> New functionality will only be added to the new libraries (Java 7.x). The old libraries (Java 6.x) are deprecated and Twilio will no longer provide bug fixes. Support might ask you to upgrade before debugging issues. 
         </div> 
        </div> 
       </div> 
       <div class="block-rich_text "> 
        <div class="rich-text">
         <p>The Twilio Java SDK makes it easy to interact with the Twilio API from your Java application.&nbsp;<a href="http://mvnrepository.com/artifact/com.twilio.sdk/twilio">The most recent version of the Twilio Java SDK can be found on Maven Repository</a>. The Twilio SDK requires Java SE 7 or higher. If you are interested in migrating to the newer 7.x version of the Twilio Java SDK from the 6.x version,&nbsp;<a href="/docs/libraries/java/migration-guide">check out this guide</a>.</p> 
         <h2 id="using-with-a-build-automation-tool"><a class="toclink" href="#using-with-a-build-automation-tool">Using with&nbsp;a&nbsp;build automation tool</a></h2> 
         <p>The recommended method for installing the SDK&nbsp;is with a build automation tool, like Maven or Gradle. You can add the <strong>Twilio</strong>&nbsp;dependency to your existing project, specifying the latest version.</p> 
         <p>In <strong>Maven</strong>, place the following within your <strong>&lt;dependencies&gt;</strong> tag&nbsp;in your <strong>pom.xml</strong> file:</p> 
         <div class="codehilite">
          <pre><span class="code-line"><span class="nt">&lt;dependency&gt;</span>
</span><span class="code-line">  <span class="nt">&lt;groupId&gt;</span>com.twilio.sdk<span class="nt">&lt;/groupId&gt;</span>
</span><span class="code-line">  <span class="nt">&lt;artifactId&gt;</span>twilio<span class="nt">&lt;/artifactId&gt;</span>
</span><span class="code-line">  <span class="nt">&lt;version&gt;</span>(7.11.0,7.12.0]<span class="nt">&lt;/version&gt;</span>
</span><span class="code-line"><span class="nt">&lt;/dependency&gt;</span>
</span></pre>
         </div> 
         <p>In&nbsp;<strong>Gradle</strong>, paste the next line inside the&nbsp;<strong>dependencies</strong> block of your&nbsp;<strong>build.gradle</strong> file:</p> 
         <div class="codehilite">
          <pre><span class="code-line"><span class="n">compile</span> <span class="ss">group</span><span class="p">:</span> <span class="s2">"com.twilio.sdk"</span><span class="p">,</span> <span class="nb">name</span><span class="p">:</span> <span class="s2">"twilio"</span><span class="p">,</span> <span class="ss">version</span><span class="p">:</span> <span class="s2">"7.11.+"</span>
</span></pre>
         </div> 
         <p>Regardless of the package manager you are using, remember to specify the latest version of the&nbsp;<strong>Twilio Java SDK</strong>.</p> 
         <h3 id="install-the-sdk-locally"><a class="toclink" href="#install-the-sdk-locally">Install the SDK locally</a></h3> 
         <p>You can also&nbsp;<a href="https://github.com/twilio/twilio-java">clone the source code</a> for the SDK, and&nbsp;install the library from there.</p> 
         <div class="codehilite">
          <pre><span class="code-line">git clone https://github.com/twilio/twilio-java
</span><span class="code-line">cd twilio-java
</span><span class="code-line">mvn install       # Requires maven, download from http://maven.apache.org/download.html
</span></pre>
         </div> 
         <p>You may need to run the above command with <strong>sudo</strong>&nbsp;(e.g. <strong>sudo mvn install)</strong>&nbsp;if&nbsp;you get "Permission Denied" errors while trying to install the library.</p> 
         <p>If you are using <strong>Gradle</strong> in your project, and you want to install the&nbsp;<strong>SDK</strong>&nbsp;locally as we just described, you may need to add the next line inside the&nbsp;<strong>repositories</strong> block of your&nbsp;<strong>build.gradle</strong> file:</p> 
         <div class="codehilite">
          <pre><span class="code-line"><span class="n">mavenLocal</span><span class="p">()</span>
</span></pre>
         </div> 
         <p>This line will instruct Gradle to look for packages on the local Maven repository, where we installed previously the&nbsp;<strong>Twilio Java SDK</strong>.</p> 
         <h2 id="using-without-a-build-automation-tool"><a class="toclink" href="#using-without-a-build-automation-tool">Using without a build automation tool</a></h2> 
         <p>While we&nbsp;recommend using a package manager to track the dependencies&nbsp;in your application, it is possible to download and use the Java SDK manually by <a href="http://search.maven.org/#search%7Cga%7C1%7Cg%3A%22com.twilio.sdk%22%20AND%20a%3A%22twilio%22">downloading a pre-built jar file</a>.&nbsp;Select the directory for&nbsp;the latest version and download one of these jar files:</p> 
         <ul> 
          <li>twilio-7.x.x-with-dependencies.jar</li> 
          <li>twilio-7.x.x.jar -- use this if you have issues with conflicting jars in your project. You'll need to include versions of the dependencies yourself. See the <a href="https://github.com/twilio/twilio-java/blob/master/pom.xml">pom.xml</a> file in <a href="https://github.com/twilio/twilio-java">the source code</a> for the list of libraries.</li> 
         </ul>

         <h2 id="accessing-the-6x-version-of-the-sdk"><a class="toclink" href="#accessing-the-6x-version-of-the-sdk">Accessing the 6.x Version of the SDK</a></h2> 
         <p>The most recent version of the Java SDK is not API compatible with the previous 6.x version of the Java SDK you may have used in previous Twilio applications. The older version will continue to work, and you will continue to find sample code for this version throughout our documentation. Should you need to install this version with Maven, you can do so with the following dependency in your <strong>pom.xml</strong>&nbsp;file:</p> 
         <div class="codehilite">
          <pre><span class="code-line"><span class="nt">&lt;dependency&gt;</span>
</span><span class="code-line">  <span class="nt">&lt;groupId&gt;</span>com.twilio.sdk<span class="nt">&lt;/groupId&gt;</span>

</span><span class="code-line"><span class="nt">&lt;/dependency&gt;</span>
</span></pre>
         </div> 
         <p>Alternately, you can <a href="http://search.maven.org/#search%7Cga%7C1%7Cg%3A%22com.twilio.sdk%22%20AND%20a%3A%22twilio-java-sdk%22%20AND%20v%3A%226.3.0%22">find a pre-built jar</a> and download it directly from there.&nbsp;Select the directory for&nbsp;the latest version and download one of these jar files:</p> 
         <ul>

          <li>twilio-6.x.x.jar -- use this if you have issues with conflicting jars in your project. You'll need to include versions of the dependencies yourself.
