            <h1 class="entry-title ">How to create click-to-call links for mobile browsers</h1> 
            <div class="entry-meta"> 
             <span class="posted-on"><i class="fa fa-calendar"></i> <a href="http://www.mobilexweb.com/blog/click-to-call-links-mobile-browsers" rel="bookmark"><time class="entry-date published" datetime="2010-11-18T18:29:47+00:00">November 18, 2010</time><time class="updated" datetime="2010-11-18T18:34:52+00:00">November 18, 2010</time></a></span>

            <p>Remember: <strong>most mobile devices are also phones!</strong> So, why not create link-to-call actions? If you’re creating a business guide, or even for your own unique phonebook, most people will prefer to call a person instead of filling in a form on the device.</p> 
            <p>In this article, we analyze the new URI schemes, some meta tags and also how to call Skype or Facetime applications from a website.<span id="more-35"></span></p> 
            <p>This content is partially extracted from my book <a href="book">“Programming the Mobile Web”</a>, published by O’Reilly.</p> 
            <h3>The tel: scheme</h3> 
            <p><img title="webOS and Android Browser showing a tel: URI" src="http://answers.oreilly.com/uploads/monthly_10_2010/post-142-128682384614.png" alt="" width="500" height="358"></p> 
            <p>The first de facto standard (copied from the Japanese i-Mode standards) is to use the <strong>tel: </strong>scheme. It was proposed as a standard in the <a href="http://tools.ietf.org/html/rfc5341" target="_blank">RFC 5341</a>, but be careful because most of the parameters proposed there do not work on any device.</p> 
            <p>Today, we can find support for tel: URI-scheme in almost every mobile device, including: Safari on iOS, Android Browser (image above), webOS Browser (image above), Symbian browser, Internet Explorer, Opera Mini and&nbsp;low-end devices browsers.</p> 
            <p>The very basic and simple syntax is:</p> 
            <p><code>&lt;a href="tel:+1800229933"&gt;Call us free!&lt;/a&gt;</code></p> 
            <p>If the user activates a call link she will receive a confirmation alert asking whether to place the call, showing the full number. This is to avoid frauds tricking the user into calling another country or a premium number.</p> 
            <p>I recommend inserting the phone number in the international format: the plus sign (+), the country code, the local area code, and the local number. We do not really know where our visitors will be located. If they are in the same country, or even in the same local area, the international format will still work.</p> 
            <h3>What happens when the user clicks</h3> 
            <p>While many browsers, like iPhone, Android, Nokia and BlackBerry offer a confirmation alert for the call action, Sony Ericsson’s NetFront browser presents the user with a menu proposing different actions to take.</p> 
            <p><img class="alignnone" title="Low and mid-end devices with a tel: URI-scheme" src="http://answers.oreilly.com/uploads/monthly_10_2010/post-142-128682385355.png" alt="" width="500" height="187"></p> 
            <p>Some non-phone mobile device (like iPod Touch, iPad), don’t allow voice calls. Instead, they show a prompt to add the phone number to the phonebook (see the image below).</p> 
            <p><a href="http://www.mobilexweb.com/wp-content/uploads/2010/11/call2.png"><img class="size-full wp-image-36 alignnone" title="Safari on iPad on a tel link" src="http://www.mobilexweb.com/wp-content/uploads/2010/11/call2.png" alt="" width="475" height="285" srcset="http://www.mobilexweb.com/wp-content/uploads/2010/11/call2.png 475w, http://www.mobilexweb.com/wp-content/uploads/2010/11/call2-300x180.png 300w" sizes="(max-width: 475px) 100vw, 475px"></a></p> 
            <h3>Sending DMTF Tones</h3> 
            <p>Some devices also allow sending DMTF tones after the call has been answered by the destination. This is useful for accessing tone-controlled services, helpdesk systems, or voicemail; you can say to the link, “call this phone number and, when the call is answered, press 2, wait 2 seconds, and then press 913#”. You do this using the <strong>postd</strong> parameter after the number: the syntax is <strong>;postd=</strong>. You can use numbers, *, # (using the URL-encoded %23 value), as well as <strong>p</strong> for a one-second pause and <strong>w</strong> for a wait-for-tone pause.</p> 
            <p>This function doesn’t work on all mobile devices, but on devices that don’t understand it, the primary telephone number should at least be called. The compatibility list for this feature is complex, and I don’t recommend relying on it.</p> 
            <h3>Autodetecting phone numbers</h3> 
            <p>The BlackBerry browser and Safari for iOS (iPhone/iPod/iPad) automatically detect phone numbers and email addresses and convert them to links. If you don’t want this feature, you should use the some meta tags.</p> 
            <p>For Safari</p> 
            <p><code>&lt;meta name="format-detection" content="telephone=no"&gt;</code></p> 
            <p>For BlackBerry</p> 
            <p><code>&lt;meta http-equiv="x-rim-auto-match" content="none"&gt;</code></p> 
            <h3>Opening native applications</h3> 
            <p>There are some native applications that can be opened from a link, mostly from iOS. One sample is the <a href="http://www.apple.com/iphone/features/facetime.html" target="_blank">Facetime</a> videochat application from iPhone 4 and iPod Touch. From iOS 4.1 we can use the facetime: URI to open the Facetime application and call somebody, for example:</p> 
            <p><code>&lt;a href="facetime://5555555555"&gt;Call us free using Facetime!&lt;/a&gt;</code></p> 
            <p><a href="http://www.skype.com">Skype for iOS</a> is another application having its own URI-scheme. We need the Skype username. Optionally, we can add a ?call parameter to initiate immediately a call. Without it, we will see the user’s profile instead.</p> 
            <p><code>&lt;a href="skype:skype_user?call"&gt;Call us using Skype!&lt;/a&gt;</code></p> 
            <p>Remember that there is no way to detect on-the-fly if the user has the native application installed. You can find on <a href="book">my book</a> a little hack to solve this problem.</p> 
            <p>There are no URI-schemes for Google Voice for iOS published until now.</p> 
            <h3>WTAI: scheme</h3> 
            <p>Although the table below shows that it is not as well supported as tel:, the other way to originate a call is using the WTAI standard, via the wp public library and the mc (make call) function. As I mentioned in my book, the WTAI is an old standard coming from the WAP 1.0 era and available from WML. . This URI-scheme was created for contacting public libraries on the device and it is not working on Safari on iOS or webOS Browser.<br> <code>&lt;a href="wtai://wp/mc;+1800229933"&gt;Call us free!&lt;/a&gt;</code></p> 
            <p>WTAI also accepts a link to be used while the call is in progress, but this is useful only if the user is in hands-free mode or using a headset. This link can include tones to be sent to the destination as if the user had pressed them on the keypad, specified using the wp library’s sd (send DTMF tones) function.</p> 
            <h3>Making radio calls</h3> 
            <p>iDEN networks (like Nextel) use radio packets to make internal calls inside the network. If you are working with customers of such a network—for example, for an intranet—you can allow users to launch internal calls to other members of the team (or external calls) using the Direct Connect URL scheme ([inlinecode]dc:[/inlinecode]). This is also compatible with BlackBerry iDEN devices:</p> 
            <p><code>&lt;a href="dc:5040*0077"&gt;Ping John&lt;/a&gt;</code></p> 
            <h3>Invoking video-calls</h3> 
            <p>Some models present users with a submenu when they click a tel: link so they can choose whether to place a voice-only or a video call (available in 3G systems). Some Japanese phones also allow you to specify that a link should initiate a video call, using the protocol <strong>tel-av:</strong>.</p> 
            <h3>Other techniques</h3> 
            <p>Do you know any other technique? Feel free to add it on the comments sections below.</p>
