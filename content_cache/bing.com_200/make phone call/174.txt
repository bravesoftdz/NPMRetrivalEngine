           <h2><span class="icon-overview"></span>Overview</h2>
           <div data-scroll-index="11"> 
            <h3>Description</h3> 
            <p><strong>Linphone</strong> is an open source SIP Phone, available on mobile and desktop environments (iOS, Android, GNU/Linux, MAC OSX,&nbsp;Windows Desktop, Windows 10 UWP).</p> 
            <p><strong>Linphone</strong> has inside a separation between the user interfaces and the core engine, allowing to create various kind of user interface on top of the same functionalities.</p> 
            <ul> 
             <li><strong>The user interface frontends</strong>: 
              <ul> 
               <li>Qt/QML interface for Windows, Mac and&nbsp;GNU/Linux</li> 
               <li>The console interface (linphonec, linphonecsh)</li> 
               <li>The iPhone application built in objective C</li> 
               <li>The Android application running in java</li> 
               <li>The Windows 10 application (UWP : mobile and desktop) written in C#</li> 
              </ul> </li> 
             <li><strong>Liblinphone</strong>, the core engine: this is the library that implements all the functionalities of <strong>Linphone</strong>. <br><strong>Liblinphone</strong> is a powerful SIP VoIP video SDK that anyone can use to add audio or video call capabilities to an application. It provides a high level api to initiate, receive, terminate audio &amp; video calls.</li> 
             <li><strong>Liblinphone</strong> relies on the following software components: 
              <ul> 
               <li><strong>Mediastreamer2</strong>, a powerful multimedia SDK to make audio/video streaming and processing.</li> 
               <li><strong>oRTP</strong>, a simple RTP library.</li> 
               <li><strong>belle-sip,</strong> the SIP library.</li> 
              </ul> <strong>Liblinphone</strong> and all its dependencies are written in pure C.</li> 
            </ul> 
           </div> 
           <div data-scroll-index="12"> 
            <h3>Architecture</h3> 
            <p><img class="schema" src="uploads/images/schema-linphone-2017/Schema-Linphone.gif" alt="" width="676" height="763"></p> 
           </div> 
           <div data-scroll-index="13"> 
            <h3>Console tools</h3> 
            <p><strong>Linphonec</strong> is a command line interface that can use readline on GNU/Linux&nbsp;to get completion, history, such as bash.<br>Linphonecsh is another command line tool to remotely control a linphonec daemon. Unlike linphonec, linphonecsh immediately exits once the command is executed.<br><em>See the documentation tab for more information.</em></p> 
            <p><strong>Linphone-daemon</strong> is a new alternative to linphonec which is more easily usable from scripting languages because the outputs of the program are consistent and always in response to a command. Commands can be passed via standard input or a unix socket. To learn more about Linphone-daemon's capabilities, use the help command.</p> 
            <p>&nbsp;</p>

           <h2><span class="icon-overview"></span>Features</h2>
           <div data-scroll-index="21"> 
            <h3>Common Features (Mobile and Desktop)</h3> 
            <h4>Overall features</h4> 
            <ul> 
             <li>Audio &amp; HD video calls</li> 
             <li>Multiple calls management (pause &amp; resume)</li> 
             <li>Call transfer</li> 
             <li>Audio conferencing (merge calls into a conference)</li> 
             <li>Instant Messaging with message delivery status (IMDN)</li> 
             <li>Pictures and files sharing</li> 
             <li>Contact list</li> 
             <li>Call History</li> 
             <li>Display of advanced call statistics</li> 
             <li>Echo Cancellation</li> 
             <li>Call quality indicator</li> 
             <li>Secure communications: zRTP, TLS, SRTP</li> 
             <li>Supported languages: English, French</li> 
             <li>Account creation assistant</li> 
            </ul> 
            <h4>Advanced features</h4> 
            <ul> 
             <li>Audio codecs: &nbsp;OPUS, SILK, SPEEX, G722, AMR-WB (G722.2), AMR-NB,&nbsp;GSM 6.10, ILBC, G729, ISAC, BV16, G711, Codec2</li> 
             <li>Video codecs: VP8, H264, MPEG4</li> 
             <li>HD video support</li> 
             <li>Integration with push notification (requires compatible SIP server, <a href="https://www.linphone.org/free-sip-service.html?page=free-sip-service.html">linphone.org sip service</a> has push enabled)</li> 
             <li>ICE support (RFC5246) to allow peer to peer audio &amp; video connections without media relay server</li> 
             <li>Low bandwidth mode for audio calls over 2G networks.</li> 
             <li>Call handover accross network access type change (start a call in wifi and continue in 3G)</li> 
             <li>Ability to configure multiple proxy accounts with different transports (UDP, TCP, TLS)</li> 
             <li>IPv6 (dual stack and v6-only support)</li> 
             <li>DTMF with RFC4733</li> 
            </ul> 
           </div> 
           <div data-scroll-index="22"> 
            <h3>Mobile specific features</h3> 
            <ul> 
             <li>Smart contact list (shows people using the service)</li> 
             <li>Bluetooth headset support</li> 
             <li>Dedicated tablet user interface</li> 
             <li>Languages : contributions for Japanese and Arabic have been integrated</li> 
            </ul> 
            <p>Linphone is available on following mobile platforms :</p> 
            <ul> 
             <li>Apple iOS 8 to 10 (ARM v7, ARM 64)</li> 
             <li>Google Android 4.1 to 7 (ARM v5 to v7, x86)</li> 
             <li>BlackBerry OS10 (ARM v7)</li> 
             <li>Windows 10 UWP : mobile and desktop (ARM v7)</li> 
            </ul> 
            <h4>iOS additional features</h4> 
            <ul> 
             <li>Blind accessibility</li> 
             <li>AAC-ELD support</li> 
            </ul> 
            <h4>Android additional features</h4> 
            <ul> 
             <li>Video overlay</li> 
            </ul> 
            <h4>Windows 10 uwp</h4> 
            <ul> 
             <li>Windows 10 support</li> 
             <li>Multiple calls management and audio conferencing are not supported.</li> 
            </ul> 
            <h4>Blackberry 10</h4> 
            <ul> 
             <li>Blackberry Hub integration</li> 
             <li>Video calls with VP8 only</li> 
            </ul> 
           </div> 
           <div data-scroll-index="23"> 
            <h3 dir="ltr">Desktop specific features</h3> 
            <p>The Qt 5.9 (QML) graphical interface is running on GNU/Linux, Windows and Mac OS X.</p> 
            <h4>Specific features&nbsp;</h4> 
            <ul> 
             <li>HD video calls in fullscreen mode</li> 
             <li>Call recording (audio &amp; video)</li> 
             <li>Presence status</li> 
             <li>Smart search bar</li> 
             <li>Chat access during calls</li> 
             <li>Unified history</li> 
             <li>Quick access through my recent events</li> 
             <li>Ability to create audio conference calls instantaneously</li> 
            </ul> 
            <h4>Portability</h4> 
            <ul> 
             <li><strong>GNU/Linux: x86, x86-64, ARM v5, v7, arm64 ;&nbsp;Debian 7/8, Centos 6/7</strong></li> 
             <li><strong>Windows Desktop: x86 (works also on x86_64) &nbsp; Vista and later</strong></li> 
             <li><strong>Mac OS X: x86_64 ;&nbsp;10.7 and later</strong>.</li> 
             <li><strong>GNU/Linux embedded:</strong>&nbsp;Linphonec or&nbsp;<a href="http://www.linphone.org/technical-corner/liblinphone.html">liblinphone</a> are good candidates to provide the software stack of an hardware phone or hardware communication system.</li> 
            </ul> 
            <p>&nbsp;</p>

           <div data-scroll-index="42"> 
            <h3>Linphone command line tools</h3> 
            <h4>Linphonec</h4> 
            <p>Linphonec is a terminal based version of linphone that reads commands from the user prompt.</p> 
            <p><strong>Example</strong>:</p> 
            <div class="code">
             Start linphonec in a terminal
             <br>$ linphonec
             <br>Ready
             <br>Warning: video is disabled in linphonec, use -V or -C or -D to enable.
             <br>Place an outgoing call
             <br>linphonec&gt; call sip:594305005@someproxy.net
             <br>Alternatively, if someproxy.net is your default proxy configuration (entered via 'proxy add' command), you call simply like this
             <br>linphonec&gt; call 594305005
             <br>Terminate a call
             <br>linphonec&gt; terminate
             <br>At any time you can list available commands using 'help':
             <br>linphonec&gt; help
            </div> 
            <p><strong>Commands are:</strong></p> 
            <div class="code">
             help Print commands help
             <br>call Call a SIP uri
             <br>chat Chat with a SIP uri
             <br>terminate Terminate the current call
             <br>answer Answer a call
             <br>autoanswer Show/set auto-answer mode
             <br>proxy Manage proxies
             <br>soundcard Manage soundcards
             <br>webcam Manage webcams
             <br>staticpic Manage static pictures when nowebcam
             <br>ipv6 Use IPV6
             <br>refer Refer the current call to the specified destination.
             <br>nat Set nat address
             <br>stun Set stun server address
             <br>firewall Set firewall policy
             <br>call-logs Calls history
             <br>friend Manage friends
             <br>play play from a wav file
             <br>record record to a wav file
             <br>quit Exit linphonec
             <br>register Register in one line to a proxy
             <br>unregister Unregister from default proxy
             <br>duration Print duration in seconds of the last call.
             <br>status Print various status information
             <br>ports Network ports configuration
             <br>speak Speak a sentence using espeak TTS engine
             <br>codec Audio codec configuration
             <br>vcodec Video codec configuration
             <br>ec Echo cancellation
             <br>mute Mute microphone and suspend voice transmission.
             <br>unmute Unmute microphone and resume voice transmission.
             <br>nortp-on-a Set the rtp_no_xmit_on_audio_mute configuration parameter
             <br>
             <br>Type 'help &lt;ltcommand&gt;' for more details.
            </div> 
            <h4>Linphonecsh - control a linphonec daemon</h4> 
            <p>Linphonecsh is a console utility to send non-blocking commands to an instance of linphonec runnning in the background.<br>Here is an example to run within your favourite shell and terminal:</p> 
            <div class="code">
             #spawn a linphonec daemon:
             <br>$ linphonecsh init
             <br>#register to a proxy
             <br>$ linphonecsh register --host myproxy.net --username bill --password thisisasecret
             <br>#dial out
             <br>$ linphonecsh dial "sip:alice@myproxy.net"
             <br>#terminate the call
             <br>$ linphonecsh hangup
             <br>#linphonecsh can transmit any command understood by linphonec using the 'generic' keyword
             <br>$ linphonecsh generic "proxy list"
             <br>#exit the daemon
             <br>$ linphonecsh exit
            </div> 
            <p>This feature is still being developed, stay connected to svn if you use it !<br>What could be the applications of this tool ? For example:</p> 
            <ul> 
             <li>run VoIP calls from scripts</li> 
             <li>from web cgi pages</li> 
             <li>from javascript in a browser window...</li> 
            </ul> 
           </div>

            <h3>Configuration file documentation</h3> 
            <p>Linphone has a linphonerc configuration file in which all SIP, RTP, sound, video settings are read and stored. By default no configuration file is required and linphone always generates a default one at first startup.<br>Most of the parameters are used internally by accessors of liblinphone (ex: linphone_core_set/get_audio_port()) and thus do not need to be accessed directly by editing the config file.<br>However some parameters are related to fine grained settings: noise gate, echo canceler parameters, echo limiter etc.<br>Follows here a description of sound related tunables and a documented linphonerc config file with all settings.</p> 
            <h4>Echo canceller</h4> 
            <p>Linphone uses an echo canceller from libspeexdsp. Filter tail length, echo delay and frame size can be changed.</p> 
            <div class="code">
             [sound]
             <br>#turn on/off echo cancellation
             <br>echocancellation=1
             <br>
             <br>#Expected delay of echo in milliseconds
             <br>
             <br>#Use this when you have a fixed latency in the sound hardware.
             <br>#This allows to reduce the tail length of the echo canceller, which speeds up convergence
             <br>#and reduces complexity of computations.
             <br>ec_delay=0
             <br>
             <br>#Tail length of echo canceller in milliseconds.
             <br>#Ideally it should be no more than the expected duration of the echo.
             <br>ec_tail_len=60
             <br>
             <br>#Frame size for AU-MDF echo canceller algorithm
             <br>#This is a parameter internal to the echo canceller, recommended is too keep to its default value.
             <br>ec_frame_size=128
            </div> 
            <h4>Echo limiter</h4> 
            <p>The echo limiter is an algorithm that consists in lowering the gain of the mic input when the speaker is talking. Combined with the noise gate (see next section) it gives good results when the echo canceller no more works, because of non linear distorsion (saturation) of the echo path.<br>Its drawback is that it turns the call in a kind of automatic half-duplex mode, which makes impossible to interrupt the person who is talking.</p> 
            <div class="code">
             [sound]
             <br>#Enables echo limiter
             <br>#this basically consists in lowering the mic input (in software)
             <br>#when the speaker level is above a certain threshold
             <br>#the attenuation is made proportionnal to the speaker detected level
             <br>echolimiter=0
             <br>
             <br>#el_speed parameter: gain changes are smoothed with a coefficent
             <br>#el_speed is this coefficient. It's a value between 0 and 1
             <br>#0.1 is already very fast, 0.001 is very low
             <br>#default value is 0.03
             <br>#recommendation is to keep it unchanged
             <br>el_speed=0.03
             <br>
             <br>#el_thres parameter
             <br>#Threshold above which the system becomes active.
             <br>#It is a normalized power, between 0 and 1.
             <br>#Default value is 0.1
             <br># A smaller value can be better.
             <br>el_thres=0.1
             <br>
             <br>#el_force parameter
             <br>#The proportional coefficient controlling the mic attenuation.
             <br>#Default value is 10
             <br>el_force=10
             <br>
             <br>#el_sustain parameter
             <br>#Time in milliseconds for which the attenuation is kept unchanged after&nbsp;
             <br>#resuming from speech to silence on the network-&gt;speaker channel.
             <br>#This is a very important parameter that needs to be adjusted
             <br>#to take in account the latency of the sound card/driver.
             <br>#Indeed when the echo limiter sees there is no more energy going to the&nbsp;
             <br>#speaker, there can be still some audio buffers pending to be played
             <br>#in the audio driver. These buffers are out of control of the application
             <br>#and will generate echo as they are non-silence.
             <br>#The purpose of the parameter is to keep the mic attenuated
             <br>#for some time until the echo of these buffer is finished.
             <br>#100 ms is a reasonable value to start, can be higher depending
             <br>#on the hardware.
             <br>el_sustain=100
            </div> 
            <h4>Noise gate</h4> 
            <p>The goal of the noise gate is to remove (or attenuate a lot) the background noise heared by the microphone. Noise and speech are distinguished using an energy threshold. The use of the noise gate can prevent feedback to produce between two devices running linphone.</p> 
            <div class="code">
             [sound]
             <br>#The noise gate is located just after mic input
             <br>#Tells whether noise gate is active:
             <br>noisegate=0
             <br>
             <br>#Noise gate threshold in linear power between 0 and 1:
             <br>#Above this threshold the noise gate becomes bypass.
             <br>ng_thres=0.05
             <br>
             <br>#Noise gate's floorgain: gain applied to the signal when its energy is below the threshold.
             <br>#It is expect to be low so that noise is attenuated.
             <br>ng_floorgain=0.0005
            </div> 
            <h4>Equalizer</h4> 
            <p>A parametric equalizer can be used to recover from speakers with bad spectral response. The frequency response of the equalizer is entirely configurable.</p> 
            <div class="code">
             [sound]
             <br>#equalizer: it is located before speaker output
             <br>#Tells whether equalizer is active.
             <br>eq_active=0
             <br>
             <br>#equalizer gains.
             <br>#It is a list of triplets ::
             <br>#for example:
             <br>#eq_gains=300:0.1:100 700:0.2:250
            </div> 
            <h4>Miscellaneous audio parameters</h4> 
            <div class="code">
             [sound]
             <br>#Automatic gain control (of mic input) - turns on or off:
             <br>agc=0
             <br>
             <br>#static software gain (linear scale) to be applied to microphone signal
             <br>
             <br>mic_gain=1.0
             <br>
             <br>#static software gain (log scale) to be applied to signal sent to speaker
             <br>playback_gain_db=0.0
             <br>
             <br>#Enable or disable DC removal of mic input:
             <br>dc_removal=0
            </div>
