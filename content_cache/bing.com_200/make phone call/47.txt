   <h1>1.1 &nbsp; Your First Java Program: &nbsp; Hello World</h1> 
   <p><br> </p>
   <p> In this section, our plan is to lead you into the world of Java programming by taking you through the three basic steps required to get a simple program running.

   <h2>Programming in Java.</h2> We break the process of programming in Java into three steps: 
   <ol> 
    <li> <em>Create</em> the program by typing it into a text editor and saving it to a file named, say, <tt>MyProgram.java</tt>. </li> 
    <li> <em>Compile</em> it by typing "<tt>javac MyProgram.java</tt>" in the terminal window. </li> 
    <li> <em>Execute</em> (or <em>run</em>) it by typing "<tt>java MyProgram</tt>" in the terminal window. </li> 
   </ol> 
   <p> The first step creates the program; the second translates it into a language more suitable for machine execution (and puts the result in a file named <tt>MyProgram.class</tt>); the third actually runs the program. </p>
   <p> </p>
   <ul> 
    <a name="1.1.1"></a> 
    <li><em>Creating a Java program.</em> A program is nothing more than a sequence of characters, like a sentence, a paragraph, or a poem. To create one, we need only define that sequence characters using a text editor in the same way as we do for email. <a href="HelloWorld.java.html">HelloWorld.java</a> is an example program. Type these character into your text editor and save it into a file named <tt>HelloWorld.java</tt>.

    <li><em>Creating your own Java program.</em> For the time being, all of our programs will be just like <tt>HelloWorld.java</tt>, except with a different sequence of statements in <tt>main()</tt>. The easiest way to write such a program is to: 
     <ul> 
      <li>Copy <tt>HelloWorld.java</tt> into a new file whose name is the program name followed by <tt>.java</tt>. </li> 
      <li>Replace <tt>HelloWorld</tt> with the program name everywhere. </li> 
      <li>Replace the print statement by a sequence of statements. </li>

   <p> </p>
   <h2>Errors.</h2> Most errors are easily fixed by carefully examining the program as we create it, in just the same way as we fix spelling and grammatical errors when we type an e-mail message. 
   <ul> 
    <li><em>Compile-time errors.</em> These errors are caught by the system when we compile the program, because they prevent the compiler from doing the translation (so it issues an error message that tries to explain why). </li> 
    <li><em>Run-time errors.</em> These errors are caught by the system when we execute the program, because the program tries to perform an invalid operation (e.g., division by zero). </li> 
    <li><em>Logical errors.</em> These errors are (hopefully) caught by the programmer when we execute the program and it produces the wrong answer. Bugs are the bane of a programmer's existence. They can be subtle and very hard to find. </li> 
   </ul> One of the very first skills that you will learn is to identify errors; one of the next will be to be sufficiently careful when coding to avoid many of them.

<p>

<H4>Q + A</H4>


<p><b>Q.</b> 
Why Java? 
<p><b>A.</b> 
The programs that we are writing are very similar to their counterparts 
in several other languages, so our choice of language is not crucial. We use 
Java because it is widely available, embraces a full set of modern abstractions,
and has a variety of automatic checks for mistakes in programs, so it 
is suitable for learning to program. There is no perfect language, and you 
certainly will be programming in other languages in the future. 


<p><b>Q.</b> 
Do I really have to type in the programs in the book to try them out? 
<p><b>A.</b> 
Everyone should type in <tt>HelloWorld.java</tt>, but you can find all of the 
code in this book (and much more) on this booksite.


<p><b>Q.</b> 
What are Java's rules regarding tabs, spaces and newline characters? 
<p><b>A.</b> 
There are not many. Java compilers treat them all to be equivalent. For 
example, we could also write HelloWorld as follows: 

<blockquote>
<table>
<TR><TD><pre>
public class HelloWorld { public static void main ( 
String [] args) { System.out.println("Hello World") ; } }
</pre></td></tr>
</table>
</blockquote>


But we do normally adhere to spacing and indenting conventions when 
we write Java programs, just as we always indent paragraphs and lines 
consistently when we write prose or poetry.

<p><b>Q.</b>
What are the rules regarding quotation marks? 
<p><b>A.</b>
Material inside quotation marks is an exception to the rule of the
previous question: things within quotes are taken literally so that you can 
precisely specify what gets printed. If you put any number of successive 
spaces within the quotes, you get that number of spaces in the output. If 
you accidentally omit a quotation mark, the compiler may get very confused,
because it needs that mark to distinguish between characters in the 
string and other parts of the program. To print a quotation mark, a
newline, or a tab, use <tt>\"</tt>, <tt>\n</tt>, or <tt>\t</tt>,
respectively, within the quotation marks. 

<p><b>Q.</b> What is the meaning of the words <tt>public</tt>, <tt>static</tt>
and <Tt>void</tt>? 
<p><b>A.</b> These keywords specify certain properties of <tt>main()</tt> that you will learn 
about later in the book. For the moment, we just include these keywords in 
the code (because they are required) but do not refer to them in the text.

<p><b>Q.</b>
 What happens when you omit a brace or misspell one of the words, 
like <tt>public</tt> or <tt>static</tt> or <tt>main</tt>? 
A. It depends upon precisely what you do. Such errors are called <em>syntax 
errors</em>. Try it out and see.

<p><b>Q.</b>
Can a program use more than one command-line argument? 
<p><b>A.</b>
 Yes, you can put several, though we normally use only a few. You refer 
to the second one as <tt>args[1]</tt>, the third one as <tt>args[2]</tt>, and so forth. 
Note that we start counting from 0 in Java.

<p><b>Q.</b>
What Java systems libraries and methods are available for me to use? 
<p><b>A.</b>
There are thousands of them, but we introduce them to you in 
a deliberate fashion in this book to avoid overwhelming you with choices. 

<p><b>Q.</b> 
How should I format my code? How should I comment my code?

<p><b>A.</b> 
Programmers use coding guidelines to make programs easier to read, understand,
and maintain. As you gain experience, you will develop a coding style,
just as you develop style when writing prose.
<a href = "../11style">Appendix B</a> provides some guidelines
for formatting and commenting your code. We recommend returning to this
appendix after you've written a few programs.


<p><b>Q.</b>
What exactly is a <tt>.class</tt> file?
<p><b>A.</b>
It's a binary file (sequence of 0s and 1s).
If you are using Unix or OS X, you can
examine its contents by typing <tt>od -x HelloWorld.class</tt>
at the command prompt.
This displays <a href = "HelloWorld.class.txt">the results</a>
in hexadecimal (base 16).
In deference to Java's name, the first word of every <tt>.class</tt>
file is <tt>cafe</tt>.
