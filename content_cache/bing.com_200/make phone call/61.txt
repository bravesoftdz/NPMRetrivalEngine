     <h2 class="accented">Remarks</h2> 
     <p>This namespace is divided into three primary categories of classes and enumerations. These categories are the lock screen, the phone call history, and the telephony options and information.</p> 
     <p>The lock-screen-call contract lets an app, with user consent, perform these tasks:</p> 
     <ul> 
      <li>Raise a notification toast with up to three additional buttons: Accept with Video, Accept with Audio, Ignore.</li> 
      <li>If the notification toast appears on the lock screen and the user chooses to accept the call, the app displays its call UI directly on the lock screen without requiring the user to unlock.</li> 
     </ul> 
     <p>The telephony options and information classes use the <strong>CallsPhoneContract</strong>. In order to use these classes, you will need to declare the <code>phoneCall</code> capability in your manifest.</p> 
     <div class="IMPORTANT">
      <h5>Important</h5>
      <p>If your application uses the <strong>phoneCall</strong> capability, you must request a waiver if you want to publish your application in China. Contact your premier support representative if you need the waiver.</p> 
     </div> 
     <p>An example that demonstrates how to use much of the functionality of the <a class="xref" href="windows.applicationmodel.calls" data-linktype="relative-path">Windows.ApplicationModel.Calls</a> API can be found <a href="http://go.microsoft.com/fwlink/p/?linkid=620586" data-linktype="external">here</a>.</p>
