       <a href="http://jpadilla.com/post/73791304724/auth-with-json-web-tokens" title="Auth with JSON Web Tokens" class="title"><h2>Auth with JSON Web Tokens</h2></a>
       <a href="#" title="Notes" class="like-post" data-reblog="https://www.tumblr.com/reblog/73791304724/YT4VPPpq" data-postid="73791304724"><span class="count">15</span><span class="trigger">?</span></a>

      <section class="content">
       <p><img alt="image" src="http://i.imgur.com/VBN5fJw.jpg"></p>
       <p>When it comes to implementing authentication on web apps, one solution you’ll definitely hear about first are cookies. Cookie-based authentication uses a server side cookies to authenticate the user on every request. This means that you’ll need to keep a session store, whether it’s on a database or on something like Redis. A solution you’ll probably not hear as often is token-based authentication which relies on a signed token that is sent to the server on each request.</p>
       <p><a href="https://twitter.com/thepose" target="_blank">Alberto Pose</a> over at Auth0 wrote a great <a href="http://t.umblr.com/redirect?z=http%3A%2F%2Fblog.auth0.com%2F2014%2F01%2F07%2Fangularjs-authentication-with-cookies-vs-token%2F&amp;t=YTAyOGFlNWIyNWNjYTg4MmJkMmFmZTA3MjRjMmEzNmNiYWNiZTVlYyxZVDRWUFBwcQ%3D%3D&amp;b=t%3AQpz2MwMZivrwC6CE-z3q-w&amp;p=http%3A%2F%2Fjpadilla.com%2Fpost%2F73791304724%2Fauth-with-json-web-tokens&amp;m=1" target="_blank">blog post</a> about Cookie vs Tokens, diagrams how both of these method works, and goes over the benefits of using a token-based approach for authentication.</p>
       <p>There’s a relatively new standard called JSON Web Token that happens to be backed by companies like Firebase, Google, Microsoft, and Zendesk. There are already many libraries to handle encoding and decoding of this tokens in the backend.</p>
       <p>
        <!-- more --></p>
       <p>After a couple of days of research I was convinced that this was something we could definitely use for <a href="http://t.umblr.com/redirect?z=http%3A%2F%2Fblog.getblimp.com%2Fcategory%2Fodp%2F&amp;t=MWUxZTI4MDIxZTE2ZDgxOGQ4NDdlYzEwZWE1MmQ4Mjk0YzQ5ZTAxOSxZVDRWUFBwcQ%3D%3D&amp;b=t%3AQpz2MwMZivrwC6CE-z3q-w&amp;p=http%3A%2F%2Fjpadilla.com%2Fpost%2F73791304724%2Fauth-with-json-web-tokens&amp;m=1" target="_blank">Blimp 2</a>. This new version will be an ambitious web application written on JavaScript. This means that there will be no HTML rendering from the server. The backend application will only serve a RESTful API that will be used by our own application as well as by developers working on third party applications.</p>
       <p>The only difference when accessing the API will be the authentication method used. We’ll use JWT but third party application will use OAuth2.</p>
       <p><strong>The JWT Spec</strong></p>
       <p>JWT(pronounced “jot”) or JSON Web Token is a compact URL-safe representation format intented for space constrained environments such as HTTP Authorization headers and URL query paramaters. JWTs represent a set of claims as a JSON object that is encoded in a JSON Web Signature or JSON Web Encryption structure.</p>
       <p>A JWT is represented as a sequence of base64url encoded values(with all trailing ‘=’ characters omitted.) that are separated by period characters.</p>
       <p>The following is an example of a JWT Header</p>
       <script src="https://gist.github.com/jpadilla/8496617.js?file=headers.json" type="text/javascript"></script>
       <p><span>Base64url encoding of the JWT Header yields this encoded JWT Header value:</span></p>
       <script src="https://gist.github.com/jpadilla/8496617.js?file=header_encoded" type="text/javascript"></script>
       <p>The following is an example of a JWT Claims Set:</p>
       <script src="https://gist.github.com/jpadilla/8496617.js?file=payload.json" type="text/javascript"></script>
       <p>Base64url encoding of the JWS Payload yields this encoded JWS Payload:</p>
       <script src="https://gist.github.com/jpadilla/8496617.js?file=payload_encoded" type="text/javascript"></script>
       <p>Computing the MAC of the encoded JWS Header and encoded JWS Payload with the HMAC SHA-256 algorithm(using the key ‘my_secret_key’) and base64url encoding the HMAC value, yields this encoded JWS Signature:</p>
       <script src="https://gist.github.com/jpadilla/8496617.js?file=signature" type="text/javascript"></script>
       <p>Concatenating these encoded parts in this order with period characters between the parts, yields this complete JWT.</p>
       <script src="https://gist.github.com/jpadilla/8496617.js?file=jwt" type="text/javascript"></script>
       <p>You can read more about JWT in the submitted&nbsp;<a href="http://t.umblr.com/redirect?z=http%3A%2F%2Ftools.ietf.org%2Fhtml%2Fdraft-ietf-oauth-json-web-token&amp;t=Y2NkYTljMWM4YWEyZTZlYTc0ZDQwYzQyNmZkYjcxMGE2MmJlMmRjZixZVDRWUFBwcQ%3D%3D&amp;b=t%3AQpz2MwMZivrwC6CE-z3q-w&amp;p=http%3A%2F%2Fjpadilla.com%2Fpost%2F73791304724%2Fauth-with-json-web-tokens&amp;m=1" target="_blank">Internet-Draft</a>.</p>
       <p><strong>An Example Scenario</strong></p>
       <p>First enters <a href="http://t.umblr.com/redirect?z=https%3A%2F%2Fgithub.com%2FGetBlimp%2Fdjango-rest-framework-jwt&amp;t=OThkZjRmOGU5ODkwODJlYmYzNDUxNjE1NzVlNTA5YTQyMGVjYzdhZSxZVDRWUFBwcQ%3D%3D&amp;b=t%3AQpz2MwMZivrwC6CE-z3q-w&amp;p=http%3A%2F%2Fjpadilla.com%2Fpost%2F73791304724%2Fauth-with-json-web-tokens&amp;m=1" target="_blank">djangorestframework-jwt</a>,&nbsp;a package I just released that provides&nbsp;JWT Authentication&nbsp;support for&nbsp;<a href="http://t.umblr.com/redirect?z=http%3A%2F%2Fdjango-rest-framework.org%2F&amp;t=NDg4NjBhNDllYmVjZjBjMWQ1ZjkwZWUwNjRhN2ZhZjRmMmE0OGY1ZCxZVDRWUFBwcQ%3D%3D&amp;b=t%3AQpz2MwMZivrwC6CE-z3q-w&amp;p=http%3A%2F%2Fjpadilla.com%2Fpost%2F73791304724%2Fauth-with-json-web-tokens&amp;m=1" target="_blank">Django REST framework</a>. It also provides an endpoint that can be used to obtain a JWT for a given username and password.</p>
       <p>Once you’ve obtained a JWT, you can access resources that required authentication by passing the HTTP Authorization header.</p>
       <p>A simple example of how this would be handled with jQuery and a Django REST Framework API endpoint with JWT Authentication.</p>

       <p><strong>And that’s a wrap!</strong></p>
       <p>And that’s basically it! I’d love to know what you think about JWT. Would you consider using token-based authentication? Any questions or feedback? Feel free to leave a comment.</p>
