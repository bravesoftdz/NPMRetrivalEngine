Start leveraging container management using Platform9's&nbsp;<a href="https://get.platform9.com/jzlp-kubernetes-deployment-models-the-ultimate-guide/" target="_blank" rel="nofollow" style="box-sizing: border-box; color: rgb(2, 136, 209); text-decoration: none; font-family: &amp;quot;Helvetica Neue&amp;quot;, Helvetica, Arial, sans-serif; font-size: 15px; font-style: normal; font-variant-ligatures: normal; font-variant-caps: normal; font-weight: normal; letter-spacing: normal; line-height: 21.4286px; orphans: 2; text-align: left; text-indent: 0px; text-transform: none; white-space: normal; widows: 2; word-spacing: 0px; -webkit-text-stroke-width: 0px; background: rgb(255, 255, 255);">ultimate guide to Kubernetes deployment</a>.</p>

         <div ng-if="!edition || (articles[0].rawType != 'refcard' &amp;&amp; articles[0].rawType != 'guide')">

           <p>I have seen this question asked many times now: “How do I install third party software on Windows Azure?” This is a reasonably important question to address as Windows Azure applications often need to use third party software components.</p> 
           <p>In some cases, using a software component can be as simple as <a href="http://msdn.microsoft.com/en-us/library/wkze6zky.aspx" rel="nofollow">adding a reference</a> to it. You can also set the <strong>Copy Local</strong> property to <strong>True</strong> to bring the component along with your service package to the cloud. However, in some cases a proper installation is required. This is because the installation does other things than just copying the component to the system (such as: modifying registry, register the components to GAC, etc.) One example would be when installing <a href="http://www.microsoft.com/en-us/download/details.aspx?id=6442" rel="nofollow">Report Viewer</a> on the Web Role to display reports.</p> 
           <p>This article will explain three techniques you can use to install third party software on Windows Azure. We will cover why and how to install third party software, and the catches that come with each technique.</p> 
           <p>Before diving into the specific techniques, let’s refresh the concept behind the current version of Windows Azure PAAS as it relates to what we’ll be discussing.</p> 
           <h1>Design for Scale: Windows Azure Stateless VM</h1> 
           <p>Windows Azure emphasizes the application philosophy of scaling-out (horizontally) instead of scaling-up (vertically). To achieve this, Windows Azure introduces the stateless virtual machine (VM). This means a VM’s local disks will not be used for storage since they are considered stateless or non-persistent. Any changes made after the VM is provisioned will be gone if the VM is re-imaged. This can happen if a hardware failure occurs on the machine where the VM is hosted.</p> 
           <p align="center"><span style="font-size: xx-small"><strong><img src="http://acloudyplace.com/wp-content/uploads/2012/05/persistentstorage.png" alt="Windows Azure persistent storage" title="Persistent Storage" width="276" height="248"></strong></span></p> 
           <p align="center"><span style="font-size: xx-small"><strong>Figure 1 – Windows Azure Stateless VM and Persistent Storage</strong></span></p> 
           <p>Instead, the recommended approach is to store data to dedicated persistent storage such as SQL Azure or Windows Azure Storage.</p> 
           <p>Now, let’s discuss each technique to install software on Windows Azure in more detail.</p> 
           <h2>Technique 1: Manual Installation through RDP</h2> 
           <p>The first technique we discuss here is the fastest and easiest, but unfortunately also the most fragile. The idea is to <a href="http://wely-lau.net/tag/remote-desktop/" rel="nofollow">perform a remote desktop (RDP) to a specific instance</a> and perform manual installation. This might sound silly to some of you as we just discussed the stateless VM above. Nonetheless, this technique is pretty useful in staging or testing environments, when we need to quickly assess if a specific software can run in a Windows Azure environment.</p> 
           <h3>The Catch</h3> 
           <p>The software installed will not be persistent.</p> 
           <p>NOTE: <strong>Do not use this technique in production</strong>.</p> 
           <h2>Technique 2: Start-up Task</h2> 
           <p>The second technique we cover here is a Start-up Task. In my opinion, this will probably be the best solution depending on your circumstances. The idea of a Start-up Task is to execute a script (in form of a batch file) prior to the role initialization. As it will be always executed prior role initialization, even if the instance is re-imaged it will still be executed.</p> 
           <h3>How to?</h3> 
           <p><strong>1. Preparing your startup script</strong></p> 
           <p>Create a file name <strong>startup.cmd</strong> using Notepad or other ASCII editor. Copy the following example and save it.</p> 
           <pre>powershell -c “(new-object
system.net.webclient).downloadfile(”http://download.microsoft.com/download/E/A/1/EA1BF9E8-D164-4354-8959-F96843DD8F46/ReportViewer.exe”, ” ReportViewer.exe”)
ReportViewer.exe /passive</pre> 
           <ul> 
            <li>The first line is to download a file from the given URL to local storage. </li> 
            <li>The second line is to run the installer “ReportViewer.exe” using passive mode. We should install using passive or silent mode so there aren’t any dialog pop-up screens. Please also note that each installer may have different silent or passive mode installation parameter.</li> 
           </ul> 
           <p><strong>2. Including startup.cmd to Visual Studio</strong></p> 
           <p>The next step is to include your <strong>startup.cmd</strong> script to Visual Studio. To do that, simply right click on the project name and choose “<strong>Add Existing Item”</strong>. Browse the <strong>startup.cmd</strong> file. Next, set “<strong>Copy to Output Directory</strong>” to “<strong>Copy always</strong>”, to ensure that the script will be included inside your package when it is built.</p> 
           <p align="center"><a href="http://acloudyplace.com/wp-content/uploads/2012/05/Including-startup.png" rel="nofollow"><strong><span style="font-size: xx-small"><img src="http://acloudyplace.com/wp-content/uploads/2012/05/Including-startup.png" alt="Including Startup.cmd in the Service" title="Including startup.cmd in the service" width="330" height="184"></span></strong></a></p> 
           <p align="center"><strong><span style="font-size: xx-small">Fiure 2 – Incuding startup.cmd in the Service</span></strong></p> 
           <p><strong>3. Adding Startup Task on your ServiceDefinition.csdef file</strong></p> 
           <p>The final step is to add a startup section in ServiceDefinition.csdef file, specifically below the intended Role tag as illustrated in below figure.</p> 
           <p align="center"><a href="http://acloudyplace.com/wp-content/uploads/2012/05/adding-startup-task.png" rel="nofollow"><span style="font-size: xx-small"><strong><img src="http://acloudyplace.com/wp-content/uploads/2012/05/adding-startup-task.png" alt="Adding Startup Task in ServiceDefinition.csdef" title="adding startup task" width="550" height="119"></strong></span></a></p> 
           <p align="center"><span style="font-size: xx-small"><strong>Figure 3 – Adding Startup Task in ServiceDefinition.csdef</strong></span></p> 
           <ul> 
            <li>The <strong>commandLine</strong> attribute requires the path of our startup script </li> 
            <li>The <strong>executionContext </strong>attribute requires us to choose either: 
             <ul> 
              <li>elevated (which will run as admin-role) or </li> 
              <li>limited (non admin-role)</li> 
             </ul> </li> 
            <li>The <strong>taskType</strong>has following options: 
             <ul> 
              <li>Simple [Default] – System waits for the task to exit before any other tasks are launched </li> 
              <li>Background – System does not wait for the task to exit </li> 
              <li>Foreground – Similar to background, except role is not restarted until all foreground tasks exit</li> 
             </ul> </li> 
           </ul> 
           <h3>The Catches</h3> 
           <p>Here are some situations where a startup task cannot be used:</p> 
           <p>1. Installation that cannot be scripted out</p> 
           <p>2. Installation that requires many user involvement</p> 
           <p>3. Installation that takes a very long time to complete</p> 
           <h2>Technique 3: VM Role</h2> 
           <p>The final technique we are looking at is VM Role. In fact, one of the reasons why Microsoft introduced VM Role is to address the issues that couldn’t be done by Startup Task.</p> 
           <p>In reality, VM Role is another option amongst Windows Azure Compute Roles. However, unlike Web and Worker Roles, you will have more responsibility when using VM Role. People often make the mistake of treating VM Role as <a href="http://wely-lau.net/2012/04/16/comparing-iaas-and-paas-a-developers-perspective/" rel="nofollow">IAAS</a>. This is not appropriate as VM Role still inherits behaviors from Web and Worker Roles. VM Role still can be easily scaled out just like Web and Worker Roles. Similarly, storing data in VM Role’s local disk is considered non-persistent.</p> 
           <p>The following figure illustrates the lifecycle of VM Role.</p> 
           <p align="center"><strong><span style="font-size: xx-small"><img src="http://acloudyplace.com/wp-content/uploads/2012/05/VM-role-lifecycle.png" alt="" title="VM role lifecycle" width="553" height="294"></span></strong></p> 
           <p align="center"><strong><span style="font-size: xx-small">Figure 4 – VM Role Lifecycle from the Windows Azure Platform Training Kit. Find the whole PowerPoint presentation here: http://acloudyplace.com/wp-content/uploads/2012/05/MovingApplicationsToTheCloudWithVMRole.pptx</span></strong></p> 
           <p>Let’s drill down to the first step “Build VM Image” in more detail. There are several tasks that should be done here. First of all is to <a href="http://msdn.microsoft.com/en-us/library/windowsazure/gg465391.aspx" rel="nofollow">create the VHD</a> that contains the operating system. The next step is to <a href="http://msdn.microsoft.com/en-us/library/gg465409.aspx" rel="nofollow">install Windows Azure Integration Component</a> onto the image. Subsequently, you can install and configure the third party software. Finally, you do a <a href="http://technet.microsoft.com/en-us/library/cc783215%28v=ws.10%29.aspx" rel="nofollow">SysPrep</a> to generalize the VM image.</p> 
           <h3>The Catches</h3> 
           <p>There are several catches when using VM Role:</p> 
           <p>1. You will have more responsibility when using VM Role, including: building, customizing, installing, uploading, and eventually maintaining the VM image.</p> 
           <p>2. Up to now, the only supported OS for VM Role is Windows Server 2008 R2.</p> 
           <p>3. At the time of writing this article, VM Role is still at beta. As we know, significant changes may happen to the beta product.</p> 
           <h1>Conclusion</h1> 
           <p>We have covered three techniques to install software in Windows Azure so far. Although, Startup task remains the recommended option in most cases, it may not be the most suitable all the time. RDP and VM Role can sometimes be advantageous depending on the scenario.</p>

    $scope.toggleLimitComments = function (article) {
        if (!article.isLimited) {
            TH$Service.action('articles.limitNode', {type: 'node', id: article.id}).then(function (result) {
                if (result) {
                    article.isLimited = true;
                    TH$Dialog.success('You have limited comments for this Article. Now all comments will go through moderation.');
                } else {
                    TH$Dialog.error('error', 'Your requested was denied')
                }
            });
        } else {
            TH$Service.action('articles.unlimitNode', {type: 'node', id: article.id}).then(function (result) {
                if (result) {
                    article.isLimited = false;
                    TH$Dialog.success('You removed the limits for comments on this Article');
                } else {
                    TH$Dialog.error('error', 'Your requested was denied')
                }
            });
        }
    };

        }).then(function (result) {
            $scope.link.title = result.title;

    $scope.deleteLink = function(article) {
        var title = article.title;
        var type = 'link';
        if(article.header){
            title = (article.header.type == 'article') ? article.header.title : article.title;
            type = (article.header.type == 'article') ? 'article' : 'link';
        }
        TH$Dialog.confirm('Do you want to delete "' + title + '"?').then(function() {
            return TH$Service.action('delete', {type: type, id: article.id});
        }).then(function() {
            article.deleted = true;
        });
    };
