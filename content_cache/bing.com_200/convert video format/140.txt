        <h2 id="d8e3922.html">Using eLML lessons with a Learning Management System (LMS)</h2> 
        <p class="paragraph" id="d8e3923">A very convenient way of using eLML lessons is to integrate them into an open source or commercial <a class="term" href="website_glossary.html#d8e5493" onmouseover="Tip('A Learning Management System (or LMS) is a software package, usually on a large scale (that scale is decreasing rapidly), that enables the management and delivery of learning content and resources to students. Most LMS systems are web-based to facilitate &quot;anytime, anywhere&quot; access to learning content and administration. Some famous open source LMS are OLAT and Moodle, famous commercial LMS are WebCT and Blackboard. ', TITLE, 'LMS')" onmouseout="UnTip()"> <img src="../../../_templates/elml/icons/term.gif" width="12" height="10" alt="term" class="term_icon"> <dfn>Learning Management System</dfn> </a> <script src="wz_tooltip.js" type="text/javascript"><!----></script> like <a class="term" href="website_glossary.html#d8e5536" onmouseover="Tip('The development of the open source LMS OLAT (Online Learning And Training) started at the University of Zurich in 1999 and won the Medida Prix for best eLearning software in 2000. Today OLAT is already in its fifth version and is the strategic platform of the University of Zurich. Besides Zurich other universities like Bern, Sachsen (Germany) etc. are using OLAT as their main LMS. More information and download of the software can be found on the OLAT website.', TITLE, 'OLAT')" onmouseout="UnTip()"> <img src="../../../_templates/elml/icons/term.gif" width="12" height="10" alt="term" class="term_icon"> <dfn>OLAT</dfn> </a> <script src="wz_tooltip.js" type="text/javascript"><!----></script> or WebCT. This is done using the <a class="term" href="website_glossary.html#d8e5483" onmouseover="Tip('The IMS Global Learning Consortium (usually known as IMS) is a non-profit standards organization concerned with establishing interoperability for learning systems and learning content and the enterprise integration of these capabilities. Their mission is to &quot;support the adoption and use of learning technology worldwide&quot;. Some famous IMS standards are the CP (Content Package) standard used to import/export of content, the &quot;Learning Resource Meta-data Specification&quot; (LOM) or the QTI standard for question and test interoperability.', TITLE, 'IMS')" onmouseout="UnTip()"> <img src="../../../_templates/elml/icons/term.gif" width="12" height="10" alt="term" class="term_icon"> <dfn>IMS Content Package</dfn> </a> <script src="wz_tooltip.js" type="text/javascript"><!----></script> or <a class="term" href="website_glossary.html#d8e5488" onmouseover="Tip('The Shareable Content Object Reference Model (SCORM) is a standard for web-based eLearning. It defines how the individual instruction elements are combined at a technical level and sets conditions for the software needed for using the content. SCORM is distributed by the Advanced Distributed Learning (ADL) Initiative, a US organization under the Department of Defense (DoD).', TITLE, 'SCORM')" onmouseout="UnTip()"> <img src="../../../_templates/elml/icons/term.gif" width="12" height="10" alt="term" class="term_icon"> <dfn>SCORM</dfn> </a> <script src="wz_tooltip.js" type="text/javascript"><!----></script> standard. A content package is basically a <a class="term" href="website_glossary.html#d8e5570" onmouseover="Tip('ZIP is a compression algorithm widely used on most of todays operating systems. A ZIP archive can contain multiple files, folders and subfolders. Both SCORM and IMS content packages are actually ZIP files that must contain a valid imsmanifest.xml file on their root level. With MacOS X you can right-mouse-click files or folders in the finder and choose &quot;Create archive of ...&quot; from the context menu. On Windows you have to install a tool like WinZIP to create ZIP archives.', TITLE, 'ZIP')" onmouseout="UnTip()"> <img src="../../../_templates/elml/icons/term.gif" width="12" height="10" alt="term" class="term_icon"> <dfn>ZIP</dfn> </a> <script src="wz_tooltip.js" type="text/javascript"><!----></script> archive with all your XHTML files, images, flashes etc. and with a XML file called imsmanifest.xml at the root level. The imsmanifest.xml contains both the metadata about the lesson (according to the IMS Metadata standard) and references to the different files. </p>

        <h2 id="d8e3988.html">How to create a SCORM or IMS CP in eLML?</h2> 
        <p class="paragraph" id="d8e3989">To generate a content package of your lessons please follow these steps: </p> 
        <ol class="list" id="d8e3991"> 
         <li class="itemAlt">Be sure that your lesson is valid and that the transformation of standalone XHTML pages works.</li> 
         <li class="item">Set the "pagebreak_level" parameter to either 'unit' or 'lo' to generate multiple output pages.</li> 
         <li class="itemAlt">Set the "use_navigation" parameter to 'no' since you want the LMS to generate your navigation and not have it included into the XHTML files. Read the <a class="link" href="output_configXML.html" title="configuration chapter">configuration chapter</a> for more information about the transformation parameters you can set.</li> 
         <li class="item">Set the "manifest_type" parameter to either 'ims' or 'scorm' depending on the type of content package you want to create. Please note that if you use 'both' the imsmanifest.xml files are stored into two separate folders and you will manually have to move one of the to the root folder. There is no way eLML can create for you automatically both the IMS CP and the SCORM package.</li> 
         <li class="itemAlt">Use a simple layout like the "plain" layout since the LMS will be responsible for the layout. You can also use the <a class="link" href="tools_template_builder.html" title="Template Builder">Template Builder</a> to create a simple layout that look good within your LMS. If you transform your lesson using the elml.xsl file directly, the plain layout is used by default.</li> 
         <li class="item">Go into the eLML project folder and use the command: "<code>zip -r lessonname.zip .</code>"<br>(This step can be done on OS X by using the context menu and choose "Create archive of ...". On Windows there are similar shortcuts to create ZIP archives.) </li> 
         <li class="itemAlt">Upload this ZIP file into your LMS and it will be recognized as a content package.</li> 
        </ol> 
        <p class="paragraph" id="d8e4017">Please note that in point 6 it is important that you do the zipping within your project folder and not outside of it. This way it is guaranteed that the imsmanifest.xml file is at the root level of the ZIP archive. Else it would be under yourpoject/imsmanifest.xml in the ZIP archive and would not be recognized by the LMS.</p>

        <h2 id="d8e4037.html">SCORM and user tracking</h2> 
        <p class="paragraph" id="d8e4038">eLML supports basic user tracking functionality of the SCORM standard such as setting the completed status and tracking the session time. Tracking the session time requires the call of a javascript function on every body onunload event. If you are using your own online transformation stylesheet instead of the default online tranformation stylesheet provided in the elml core folder, you may change the body tag in your online transformation stylesheet to: </p> 
        <p class="paragraph" id="d8e4040"> <code>&lt;body&gt;</code> <br> <code>&lt;xsl:if test="$manifest_type='scorm'"&gt;</code> <br> <code>&lt;xsl:attribute name="onunload"&gt; &lt;xsl:value-of&gt;finish()&lt;/xsl:value-of&gt; &lt;/xsl:attribute&gt;</code> <br> <code>&lt;/xsl:if&gt;</code> <br> <code>... </code> </p>
