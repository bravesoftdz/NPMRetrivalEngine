        <p>The Scratch <a href="/wiki/Project" title="Project">project</a> format (.sb or .sb2) can only be read by the Scratch editor. Compiling a scratch project can convert it to another format that can be opened without scratch. Compiling was a popular suggestion for <a href="/wiki/Scratch_2.0" title="Scratch 2.0">Scratch 2.0</a>,<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> However, it wasn't added in the normal online or offline editors. To get around this, several Scratchers have made ways to turn them into more widely known formats such as exe and app. </p>
        <p>Porting is useful for those who want to distribute the project to various computers offline, especially if it's a game or not appropriate for Scratch. </p>
        <p>This tutorial will cover how to turn a Scratch project into various executable file formats for Windows, Macintosh and Linux. </p>

        <h2><span class="mw-headline" id="Scratch_online_to_EXE">Scratch online to EXE</span></h2> 
        <h3><span class="mw-headline" id="Online_Converter">Online Converter</span></h3> 
        <ol> 
         <li> Open the online converter <a rel="nofollow" class="external text" href="http://edxtech.uk/data/scratchtoexe">here</a>. </li> 
         <li> On the converter page, enter your project ID in the box. This is the long number in all scratch project URL's. For example, if your project URL is <a rel="nofollow" class="external free" href="https://scratch.mit.edu/projects/128316889/">https://scratch.mit.edu/projects/128316889/</a>, then the ID would be 128316889. </li> 
         <li> Click on next step and enter details about the project, then click next and change the options as you please. </li> 
         <li> When you click "Build and Download", your application will be built and the ZIP file shall start downloading. </li> 
         <li> Finally, extract the files from the downloaded ZIP archive and open your app. A small disclaimer box should appear, read this carefully and when you're done, click the Accept button. You can now share the app without this screen showing. </li> 
        </ol> 
        <h3><span class="mw-headline" id="Notes">Notes</span></h3> 
        <ul> 
         <li>The executable file must be in the same folder as CORE.dll. In order to include all files in one downloadable file, use a ZIP archive. </li> 
        </ul> 
        <h2><span class="mw-headline" id="SB2_.28Scratch_2.0.29_to_SB_.28Scratch_1.4.29">SB2 (Scratch 2.0) to SB (Scratch 1.4)</span></h2> 
        <p>It is possible for one to convert an SB2 file from Scratch 2 to a Scratch 1.4 SB file using blob8108's <a rel="nofollow" class="external text" href="http://kurt.herokuapp.com/20to14">Retro Converter</a>. Then, if desired, other tools can be used to port it to various formats. </p> 
        <table cellspacing="0" cellpadding="0" border="0" style="background: transparent;">
         <tbody>
          <tr>
           <td nowrap valign="top"><span style="position: relative; top: 4px;"><img alt="Warning" src="/w/images/Information.png" width="18" height="18"></span> <b>Note:</b> </td>
           <td valign="top" style="padding-left: 0.5em; position: relative; top: 6px; padding-bottom: 6px;">The converter can only convert 1.4 blocks. It cannot convert 2.0 This means converted projects cannot use <a href="/wiki/Clone" title="Clone" class="mw-redirect">clones</a>, cloud, custom blocks, or other new Scratch 2.0 features.</td>
          </tr>
         </tbody>
        </table> 
        <p><b>SB2 to SB</b> </p> 
        <ol> 
         <li>Download your 2.0 project from the <a href="/wiki/Scratch_Website" title="Scratch Website">Scratch Website</a> or the <a href="/wiki/Scratch_2.0_Offline_Editor" title="Scratch 2.0 Offline Editor">2.0 Offline Editor</a>. </li> 
         <li>Go to the <a rel="nofollow" class="external text" href="http://kurt.herokuapp.com/20to14">Retro Converter</a> and follow the instructions there to convert to SB. </li> 
         <li>Once converted, a pop-up should appear and you will be able to download a 1.4 version (SB) of your 2.0 (SB2) Scratch project. </li> 
        </ol> 
        <p><b>SB to Other Formats</b> </p> 
        <ol> 
         <li>Now, follow the other Scratch 1.4 tutorials on this page to convert to various other formats. </li> 
        </ol> 
        <h2><span class="mw-headline" id="SB_to_EXE.2FAPP">SB to EXE/APP</span></h2> 
        <table cellspacing="0" cellpadding="0" border="0" style="background: transparent;">
         <tbody>
          <tr>
           <td nowrap valign="top"><span style="position: relative; top: 4px;"><img alt="Warning" src="/w/images/Information.png" width="18" height="18"></span> <b>Note:</b> </td>
           <td valign="top" style="padding-left: 0.5em; position: relative; top: 6px; padding-bottom: 6px;">This converts to an EXE on Windows, and an APP on Mac OS X.</td>
          </tr>
         </tbody>
        </table> 
        <table cellspacing="0" cellpadding="0" border="0" style="background: transparent;">
         <tbody>

           <td valign="top" style="padding-left: 0.5em; position: relative; top: 6px; padding-bottom: 6px;">This only works for Scratch 1.4 and earlier.</td>
          </tr>

        <ol> 
         <li>Install <a href="/wiki/Build_Your_Own_Blocks_(BYOB)_(Scratch_Modification)" title="Build Your Own Blocks (BYOB) (Scratch Modification)" class="mw-redirect">BYOB</a> (see resources and links for a link to the website). </li> 
         <li>Open your SB file. </li> 
         <li>Click "Compile this project" under the Share menu. </li> 
        </ol>

        <h2><span class="mw-headline" id="SB_to_JAR_.28Java_executable.29">SB to JAR (Java executable)</span></h2> 
        <table cellspacing="0" cellpadding="0" border="0" style="background: transparent;">
         <tbody>
          <tr>
           <td nowrap valign="top"><span style="position: relative; top: 4px;"><img alt="Warning" src="/w/images/Information.png" width="18" height="18"></span> <b>Note:</b> </td>
           <td valign="top" style="padding-left: 0.5em; position: relative; top: 6px; padding-bottom: 6px;">This only works for Scratch 1.4 and earlier.</td>
          </tr>
         </tbody>
        </table> 
        <p>This method works on all operating systems with a Java runtime installed, including Mac, Windows, and Linux. </p> 
        <ol> 
         <li>Download ToJAR (see <a href="#Resources_and_Links">resources and links</a>)<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> </li> 
         <li>Unpack/extract the .zip file </li> 
         <li>In the ToJAR folder, double click on the program folder, then on the dist folder and then double click on the ToJAR.jar file. </li> 
         <li>This will launch the ToJAR application which looks like this:<br><a href="/wiki/File:Tojar.png" class="image" title="inline"><img alt="inline" src="/w/images/Tojar.png" width="500" height="176"></a> </li> 
         <li>Click the first select button and choose the input (SB) file. </li> 
         <li>Choose your output name, and be sure to append the <code>.jar</code> file extension, for example <code>Example.jar</code> </li> 
         <li>Choose your output location, being sure to put the name of your JAR file at the end. For example, <code>C:/Documents and Settings/Documents/Example.jar</code> </li> 
         <li>Click "Compile" </li> 
        </ol> 
        <h2><span class="mw-headline" id="SB_to_APP_via_Java">SB to APP via Java</span></h2> 
        <table cellspacing="0" cellpadding="0" border="0" style="background: transparent;">

        <p><a href="/wiki/File:Jarb.png" class="image"><img alt="Jarb.png" src="/w/images/Jarb.png" width="500" height="465"></a> </p> 
        <ol> 
         <li>From the tabs at the top, the first tab says build information. You can set a custom icon in this tab if you wish. The icon must be in .icns format (use Icon Composer (comes with Mac) to convert a PNG file into .icns format). If no icon is provided, the icon shown will be used). </li> 
         <li>Select the JAR file you created in toJAR as the Main Class. </li> 
         <li>Click the button at the bottom that says Create Application... </li> 
         <li>Choose an output folder to save your project in </li> 
        </ol>
