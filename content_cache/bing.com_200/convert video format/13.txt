       <h1> <a href="https://thethemefoundry.com/blog/convert-mp4-to-webm/" rel="bookmark" title="Permanent Link to How to convert MP4 to WebM">How to convert MP4 to WebM</a> </h1> 
       <p> By Drew Strojny on August 30, 2013 </p> 
      </header> 
      <p>Here at The Theme Foundry we sell <a href="https://thethemefoundry.com">WordPress templates</a> and host quite a few tutorial videos over in the <a href="/support/">Help Center</a>. We use the excellent <a href="http://sublimevideo.net/">SublimeVideo player</a> to power the videos. Unlike YouTube or Vimeo, SublimeVideo lets us host and serve our own video files and keep them white-label. You add the player via one line of JavaScript and you can write native <a href="http://en.wikipedia.org/wiki/HTML5_video#.3Cvideo.3E_element_examples">HTML5 video syntax</a>. SublimeVideo provides a clean and flexible player that automatically picks up the video and also handles playing it via Flash when a browser doesn’t support HTML5.</p> 
      <p>HTML5 video is great, but to support all the modern browsers (Chrome, Safari, Firefox, Opera, IE9/10), you need to serve two versions of your video files. For Chrome, IE9/10, and Safari, you need <a href="http://en.wikipedia.org/wiki/MPEG-4_Part_14">MP4</a> video files, and for Firefox and Opera, you need <a href="http://en.wikipedia.org/wiki/WebM">WebM</a> files.</p> 
      <p>Most video applications can export videos in the MP4 format, but they usually can’t export in the WebM format. You need a tool to convert those MP4 files to WebM.</p> 
      <p><span id="more-27931"></span></p> 
      <h2>Quick and easy options</h2> 
      <p>There are two good applications that can do these conversions quickly for you: the <a href="http://firefogg.org/">Firefogg Firefox extension</a> and the <a href="http://www.mirovideoconverter.com/">Miro Video Converter</a>. However, they both have some limitations: Firefogg works well, but it doesn’t have batch converting. Miro does have batch converting, but it doesn’t offer total control over the conversion settings and it can be slow. Both are good options if you just want something quick and easy.</p> 
      <h2>Power option: FFmpeg</h2> 
      <p>Let’s learn how to convert MP4 to WebM using <a href="http://www.ffmpeg.org/">FFmpeg</a> for Mac or Linux. FFmpeg is an open source command line utility that actually serves as the engine for the applications noted above. FFmpeg is fast, powerful, and extremely flexible, but it can be difficult to understand and use.</p> 
      <h2>Install FFmpeg</h2> 
      <p>The easiest way to install FFmpeg on a Mac is to use <a href="http://brew.sh/">Homebrew</a>. Once you’ve got Homebrew, install FFmpeg with the following command from the Terminal on your Mac:</p> 
      <pre><code>brew install ffmpeg --with-libvpx --with-libvorbis --with-fdk-aacc
</code></pre> 
      <p>This will install and build FFmpeg with support for the libraries required to convert videos to the WebM format. If you’re on Linux, find the appropriate FFmpeg package for your distribution and install it with support for the libraries mentioned above.</p>
      <p class="in-post-make"><em>Did you know that Surf Office, Postmatic, Yeah Dave, and over 700,000 small businesses run their websites with Make, our free WordPress page builder. <a href="/wordpress-themes/make/?utm_source=blog&amp;utm_campaign=in-post">Discover the Make page builder now.</a></em></p> 
      <h2>Converting a single file</h2> 
      <p>Convert a single video file by opening up a Terminal window and navigating to the location of your file. If the file is on your Desktop:</p> 
      <pre><code>cd ~/Desktop
</code></pre> 
      <p>Next, run the FFmpeg command on the file:</p> 
      <pre><code>ffmpeg -i input-file.mp4 -c:v libvpx -crf 10 -b:v 1M -c:a libvorbis output-file.webm
</code></pre> 
      <p>When the command is finished you’ll have a shiny new WebM file. Keep in mind, this is a generic way to convert your MP4 videos to WebM. For the absolute best quality, it’s recommended you tweak the settings to work well for the size and quality of your source file(s). Read the <a href="http://ffmpeg.org/ffmpeg.html">FFmpeg documentation</a> and the <a href="http://trac.ffmpeg.org/wiki/vpxEncodingGuide">WebM encoding guide</a> for more details.</p> 
      <p>Remember, <a href="http://ffmpeg.org/ffmpeg.html#Description">the basic format for the FFmpeg command</a> is:</p> 
      <pre><code>ffmpeg [global options] [input file options] -i input-file-name [output options] output-file-name
</code></pre> 
      <h2>Batch converting files</h2> 
      <p>The easiest way to batch convert a bunch of MP4 files to WebM is to use FFmpeg with a <a href="http://en.wikipedia.org/wiki/Bash_(Unix_shell)">Bash</a> script. Let’s whip one up.</p> 
      <ol> 
       <li>Create a new file on your desktop called <code>convert-mp4-webm.sh</code></li> 
       <li>Set executable permissions by right clicking on the file, choosing <strong>Show info</strong>, and setting the <strong>Permissions</strong> value to 755 (lower right hand corner of the section).</li> 
       <li>Paste the code snippet below in the file itself and replace the “video-folder” part with the name of the folder on your Desktop containing the MP4 video files. Remember, spaces in folder names look different in a Bash script. Try to name your folder with a hyphen or an underscore to avoid any problems.</li> 
       <li> <p>Save the file and exit.</p> <pre>#!/bin/bash
VIDEOS=~/Desktop/video-folder/
find "$VIDEOS" -name '*.mp4' -exec sh -c 'ffmpeg -i "$0" -c:v libvpx -crf 10 -b:v 1M -c:a libvorbis "${0%%.mp4}.webm"' {} \;
exit;</pre> </li> 
      </ol> 
      <p>Let’s break down this script into English:</p> 
      <ol> 
       <li>The first line tells the system this is a Bash script.</li> 
       <li>The second line assigns the videos folder to the <code>$VIDEOS</code> variable.</li> 
       <li>The first part of the third line <code>find "$VIDEOS" -name '*.mp4' -exec sh -c</code> says: Find any files in the <code>$VIDEOS</code> folder with an <code>.mp4</code> extension and run a command on each file one by one.</li> 
       <li>The second part of the third line is the actual command: <code>ffmpeg -i "$0" -c:v libvpx -crf 10 -b:v 1M -c:a libvorbis "${0%%.mp4}.webm"</code>. This is the same basic FFmpeg conversion command we used in the single file example above. The $0 and 0%% tells the script to use the existing filename for the input and output variables.</li> 
      </ol> 
      <p>Now we need to actually run the script and start the conversion. Open up a new Terminal and navigate to your Desktop:</p> 
      <pre><code>cd ~/Desktop
</code></pre> 
      <p>Now run the script:</p> 
      <pre><code>./convert-mp4-webm.sh
</code></pre> 
      <p>You should see the first file start converting. Go grab some coffee and relax, it might take awhile.</p> 
      <h2>Get converting!</h2> 
      <p>Hopefully this tutorial helps speed up you conversion workflow for creating HTML5 compatible videos. Let us know if you have any other better approaches in the comments.</p>
