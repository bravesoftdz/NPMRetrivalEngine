      <p>Data can be stored in a large variety of formats. Each statistical package has its own format for data (xls for Microsoft Excel, dta for Stata, sas7bdat for SAS, ...). R can read almost all file formats. We present a method for each kind of file. If none of the following methods work, you can use a specific software for data conversion such as the free software <a href="https://en.wikipedia.org/wiki/Google_Refine" class="extiw" title="w:Google Refine">OpenRefine</a> or the commercial software Stat Transfer<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup>. In any case, most statistical software can export data in a CSV (comma separated values) format and all of them can read CSV data. This is often the best solution to make data available to everyone.</p>

      <h2><span id="CSV_(csv,txt,dat)"></span><span class="mw-headline" id="CSV_.28csv.2Ctxt.2Cdat.29">CSV (csv,txt,dat)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=R_Programming/Importing_and_exporting_data&amp;action=edit&amp;section=2" title="Edit section: CSV (csv,txt,dat)">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>You can import data from a text file (often CSV) using <code>read.table()</code>, <code>read.csv()</code> or <code>read.csv2()</code>. The option <code>header = TRUE</code> indicates that the first line of the CSV file should be interpreted as variables names and the option <code>sep =</code> gives the separator (generally "," or ";").</p> 
      <p><code>csv.get()</code> (<b>Hmisc</b>) is another possibility.</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
mydata <span class="o">&lt;-</span> read.table<span class="p">(</span><span class="s">"data.txt"</span><span class="p">,</span>header<span class="o">=</span><span class="kc">TRUE</span><span class="p">)</span>
mydata <span class="o">&lt;-</span> read.table<span class="p">(</span><span class="s">"data.csv"</span><span class="p">,</span> header <span class="o">=</span> <span class="kc">TRUE</span><span class="p">,</span> sep<span class="o">=</span><span class="s">","</span><span class="p">)</span>  <span class="c1"># import from a CSV</span>
mydata <span class="o">&lt;-</span> read.csv<span class="p">(</span><span class="s">"data.csv"</span><span class="p">,</span> header<span class="o">=</span><span class="bp">T</span><span class="p">)</span>
mydata <span class="o">&lt;-</span> read.table<span class="p">(</span><span class="s">"data.csv"</span><span class="p">,</span> header <span class="o">=</span> <span class="kc">TRUE</span><span class="p">,</span> sep<span class="o">=</span><span class="s">";"</span><span class="p">)</span> 
mydata <span class="o">&lt;-</span> read.csv2<span class="p">(</span><span class="s">"data.csv"</span><span class="p">,</span> header<span class="o">=</span><span class="bp">T</span><span class="p">)</span>
</pre>
      </div> 
      <p>Note that there is no problem if your data are stored on the internet.</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
df <span class="o">&lt;-</span> read.table<span class="p">(</span><span class="s">"http://www.mywebsite.com/.../data.csv"</span><span class="p">,</span> header <span class="o">=</span> <span class="kc">TRUE</span><span class="p">,</span> sep <span class="o">=</span> <span class="s">","</span><span class="p">)</span>

      <p>By default, strings are converted to factors. If you want to avoid this conversion, you can specify the option <code>stringsAsFactors = FALSE</code>.</p> 
      <p>You can export data to a text file using <code>write.table()</code>.</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
write.table<span class="p">(</span>mydat<span class="p">,</span>file<span class="o">=</span><span class="s">"mydat.csv"</span><span class="p">,</span>quote<span class="o">=</span><span class="bp">T</span><span class="p">,</span>append<span class="o">=</span><span class="bp">F</span><span class="p">,</span>sep<span class="o">=</span><span class="s">","</span><span class="p">,</span>eol <span class="o">=</span> <span class="s">"\n"</span><span class="p">,</span> na <span class="o">=</span> <span class="s">"NA"</span><span class="p">,</span> dec <span class="o">=</span> <span class="s">"."</span><span class="p">,</span> row.names <span class="o">=</span> <span class="bp">T</span><span class="p">,</span>col.names <span class="o">=</span> <span class="bp">T</span><span class="p">)</span>

      <h2><span id="Excel_(xls,xlsx)"></span><span class="mw-headline" id="Excel_.28xls.2Cxlsx.29">Excel (xls,xlsx)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=R_Programming/Importing_and_exporting_data&amp;action=edit&amp;section=10" title="Edit section: Excel (xls,xlsx)">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Importing data from Excel is not easy. The solution depends on your operating system. If none of the methods below works, you can always export each Excel spreadsheets to CSV format and read the CSV in R. This is often the simplest and quickest solution.</p> 
      <p><b>XLConnect</b> supports reading and writing both xls and xlsx file formats. Since it is based on Apache POI it only requires a Java installation and as such works on many platforms including Windows, UNIX/Linux and Mac. Besides reading &amp; writing data it provides a number of additional features such as adding plots, cell styling &amp; style actions and many more.</p> 
      <div class="mw-highlight mw-content-ltr" dir="ltr">

<span class="c1"># defined names, hidden sheets, active sheet name, ...)</span>
<span class="kp">summary</span><span class="p">(</span>wb<span class="p">)</span>
<span class="c1"># Read data from a worksheet interpreting the first row as column names</span>
df1 <span class="o">&lt;-</span> readWorksheet<span class="p">(</span>wb<span class="p">,</span> sheet <span class="o">=</span> <span class="s">"mysheet"</span><span class="p">)</span>
<span class="c1"># Read data from a named region/range interpreting the first row as column</span>
<span class="c1"># names</span>
df2 <span class="o">&lt;-</span> readNamedRegion<span class="p">(</span>wb<span class="p">,</span> name <span class="o">=</span> <span class="s">"myname"</span><span class="p">,</span> header <span class="o">=</span> <span class="kc">TRUE</span><span class="p">)</span>
</pre>

      <p><a href="https://en.wikipedia.org/wiki/JSON" class="extiw" title="w:JSON">JSON</a> (JavaScript Object Notation) is a very common format on the internet.       <div class="mw-highlight mw-content-ltr" dir="ltr"> 
       <pre>
<span class="c1"># df : a data frame</span>
<span class="kn">library</span><span class="p">(</span><span class="s">"rjson"</span><span class="p">)</span>

Then import the resulting data as per the <a class="external text" href="http://en.wikibooks.org/w/index.php?title=R_Programming/Importing_and_exporting_data&amp;stable=0#CSV_.28csv.2Ctxt.2Cdat.29">CSV instructions above</a>.</p>
