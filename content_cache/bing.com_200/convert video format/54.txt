     <h2>Release Notes</h2> 
     <p>For Mono / .NET Core / custom ffmpeg build use: <br><a href="https://www.nuget.org/packages/NReco.VideoConverter.LT/">https://www.nuget.org/packages/NReco.VideoConverter.LT/</a> <br> <br>More info (online demo, usage examples): <a href="https://www.nrecosite.com/video_converter_net.aspx">https://www.nrecosite.com/video_converter_net.aspx</a> <br> <br>v.1.1.2 changes: <br>- FFMpeg updated to v.3.2.2 <br>- ConvertLiveMediaTask.Stop uses only 'q' command if possible (CTRL+C method not used because of instability) <br>- Fixed IOException issue thrown by ConvertLiveMedia <br>- added FFMpegConverter.LogLevel property <br>- added FFMpegConverter.ExtractFFmpeg() method (forces extraction of ffmpeg binaries) <br> <br>v.1.1.1 changes: <br>- FFMpegConverter.Stop switched from sending CTRL+C to ffmpeg console to 'q' command written to stdin: this approach is more stable and compatible with .NET Core <br>- Fixed issue with NullReferenceException from Process.WaitForExit (thrown when Stop method is called from another thread) <br>- added NReco.VideoConverter.LT build for NET Standards 1.5 (can be used from .NET Core apps) <br> <br>v.1.1.0 changes: <br>- FFMpeg updated to v.3.0.1 <br>- added FFMpegConverter.ConvertMedia overload that accepts several FFMpeg inputs -- this is useful for appending audio to video, applying watermark to video, video concatenation (with concat filter) <br>- now LogReceived event works for ConvertLiveMedia (useful for debugging purposes) <br>- now ConvertProgress event works for ConvertLiveMedia </p>
