     <h2>Conforming and converting video in to Digital Cinema Package&nbsp;format.</h2> 
     <small class="single-by"></small> 
     <div class="entry"> 
      <p>This might be quite a long read, make yourself comfy.</p> 
      <p>When I started using an action camera to record my biking, someone asked me if it was possible to play the video back at the local cinema. I will call this guy Dave, because that’s his name but it also has a degree of anonymity because it’s a popular name. At time I was working at a local cinema, which had recently changed all the film projectors for digital projectors. I doubt Dave was being completely serious about the idea, but it set me thinking about the possibilities.</p> 
      <p>I present a work-flow to convert the output of my camera (CamSports HD-S 720p). Reasonably you could use any digital video camera if you make the appropriate adjustments to the conforming instructions.</p> 
      <p>I use Ubuntu as my “mostly-used” operating system, this is tested on version 12.10. It should work on most, if not all, Debian-type distributions. An advantage of using linux for this task is that you don’t need to be worried about file-size or filename limitations, because you can save directly on to an EXT3 formatted disc, with an MBR type partition table.<br> You need lots of disk space, as you will be converting your video to a sequence of still frames, and doing conversions on that. The sound will grow in size too, because D-Cinema does not understand compressed sound (AC3, DTS and SDDS are so ‘old hat’). Consider having your OS installed on an SSD, with your workspace on a big “traditional” hard disk.</p> 
      <h3>Installing the software, and general preparation.</h3> 
      <p>Visit <a href="http://www.opendcp.org/">http://www.opendcp.org/</a> and download the corresponding package for your system. Install it manually:</p> 
      <pre>sudo dpkg -i opendcp_0.27.1_xubuntu1210_i386.deb</pre> 
      <p>If you get errors about other packages not being installed, they can usually be fixed with this command:</p> 
      <pre>apt-get -f install</pre> 
      <p>There are some other packages you will need:</p> 
      <pre>sudo apt-get install openjpeg-tools
sudo apt-get install imagemagick
sudo apt-get install sox
sudo apt-get install ffmpeg</pre> 
      <p>I am not sure if OpenDCP needs <em>openjpeg-tools</em>, but it’s useful to be able to sample check your JPEG2000 encoded frames.</p> 
      <p>Some other preparation, make yourself a work folder and some sub-folders within:</p> 
      <pre>mkdir DCP_Project
cd DCP_Project
mkdir audio
mkdir tiff
mkdir j2k
mkdir mxf
mkdir dcp</pre> 
      <p>Copy your video file in to the DCP_Project folder.</p> 
      <h3>Conforming your source file.</h3> 
      <p>This is where you will adjust your video and audio, to make it fit within the rigid formats D-Cinema uses. In this example I am using the file “20120624.mp4”. The first step is to use ffmpeg to convert the video in to a series of individual frames. Notice that I sneak in a frame-rate conversion here too:</p> 
      <pre>ffmpeg -y -i 20120624.mp4 -an -r 24 -pix_fmt rgb24 -vcodec tiff tiff/%08d.tiff</pre> 
      <p>These frames are all the wrong size, and need tiny black bars left and right. The target is a vanilla 2k, JPEG2000, 2D, 24fps DCP.</p> 
      <pre>cd tiff
mogrify -resize 1920x1080 *.tiff
mogrify -border 39x0 -bordercolor Black *.tiff
cd ..</pre> 
      <p>The frames should all be 1998×1080 pixels which corresponds to the “flat” DCI format.</p> 
      <p>The next job is the audio. My camera records one sound channel only, and it does that badly. For simplicity’s sake I will make that channel appear on the centre (screen) channel in the cinema, and supply silence for the other 5 channels:</p> 
      <pre>ffmpeg -y -i 20120624.mp4 -acodec pcm_s24le -ar 48000 audio/audio.wav
cd audio
sox audio.wav left.wav remix 0
sox audio.wav right.wav remix 0
sox audio.wav center.wav remix 1
sox audio.wav sub.wav remix 0
sox audio.wav surrleft.wav remix 0
sox audio.wav surrright.wav remix 0
cd ..</pre> 
      <h3>Converting and creating the DCP.</h3> 
      <p>Start the OpenDCP application and select the JPEG2000 tab. The default options should be correct (OpenJPEG, Cinema 2K, 24 frame rate, 125Mb/s bandwidth, sRGB source, X’Y’Z’ Transform, DPX Logarithmic does not matter for tiff input, DCI Resize: None).<br> Set the input directory to your “tiff” folder, and the output folder to your “j2k” folder. Click on Convert, and wait.</p> 
      <p><em>Optional: Check one or two sample images from the “j2k” folder, by opening them in GIMP.</em></p> 
      <p>In OpenDCP select the MXF tab next. Convert the sequence of JPEG2000 images (“j2k” folder) in to an MXF file. The options should be JPEG2000, SMPTE, frame rate 24. Leave Stereoscopic and Slideshow unselected. Set the picture input to your “j2k” folder, and the output should have filename “video.mxf” in the “mxf” folder.</p> 
      <p>Staying with the MXF tab, change the type option to “WAV” and select input type “Mono” and output type “5.1”. There are now six input files required (in order):<br> Left<br> Right<br> Center<br> Sub<br> Left Surround<br> Right Surround<br> Match those up with the six files created with <em>sox</em> earlier. Set the output file to be “audio.mxf” to be saved in the “mxf” folder, and create the MXF file.</p> 
      <p><em>Optional: Check listen to the audio.mxf using VLC.</em></p> 
      <p>This project does not have subtitles, so skip over that step and go to the DCP tab.</p> 
      <p>Use the Title Generator, as the DCI specification has a tightly controlled title naming system. This project is a “short” so that would be a good type or kind for setting up the Composition Parameters.<br> Adding a XML Digital Signature is a good idea, it detects if your DCP has been corrupted in transit. You might find the process of creating that signature seems to hang at 99%. This is a <a href="http://code.google.com/p/opendcp/issues/detail?id=160">known issue</a> with OpenDCP right now.<br> As this project only has one “reel” just fill in the Picture source (“video.mxf”) and Sound source (“audio.mxf”). Leave the Subtitle source blank. You might notice that frame counts are not the same, reduce the largest to make it the same as the smallest. This depends on your source material, and may also depend on the phase of the moon.<br> Create your DCP, and you’re nearly finished.</p> 
      <p>As you are using linux, you can use gparted to create your distribution disc (MBR partition table, EXT3 format) and copy over your completed Digital Cinema Package. A good thing is to create a folder with the same name as the package, and put the six files in that folder. In my example I created a DCP called “20120624_SHR_F_EN-XX_UK-U_51_2K_20130324_OPE_2D_OV” so I put all the files in a folder of that name.</p> 
      <p>Unmount the drive, and send it to your cinema.</p> 
      <h3>Optional verification step.</h3> 
      <p>You could follow the instructions at<br> <a href="https://github.com/wolfgangw/digital_cinema_tools_distribution/wiki/Setup">https://github.com/wolfgangw/digital_cinema_tools_distribution/wiki/Setup</a><br> to check what you have created.</p> 
      <p>The general form of the command is to point it at the folder containing your DCP. Using my example:</p> 
      <pre>dcp_inspect dcp</pre>
