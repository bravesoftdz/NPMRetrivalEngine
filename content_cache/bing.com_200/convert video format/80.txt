          <h2 class="LW_CollapsibleArea_TitleDiv"> 
           <div> 
            <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"> <span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span> <span class="LW_CollapsibleArea_Title">Overview</span> </a> 
            <div id="Anchor_0" class="LW_CollapsibleArea_Anchor_Div"> 
             <a href="/en-us/library/office/jj649005(v=office.14).aspx#Anchor_0" class="LW_CollapsibleArea_Anchor_Img" title="Right-click to copy and share the link for this section"></a> 
            </div> 
            <div class="LW_CollapsibleArea_HrDiv"> 
             <hr class="LW_CollapsibleArea_Hr"> 
            </div> 
           </div> </h2> 
          <div class="sectionblock"> 
           <a id="sectionToggle0"> </a> 
           <p>Converting Microsoft Office documents from binary formats such as .doc, .ppt, and .xls to Open XML formats can improve compatibility with newer versions of Office and Office-compatible software. Converting can also improve compliance with official standards and simplify interoperability with internal applications.</p> 
           <p>You can convert individual files by opening them in a current Microsoft Office application and saving them in the new format but this may be cumbersome for large file sets. To convert groups of files, you can use existing tools or create your own tools. This article describes both options. </p> 
          </div> 
         </div> 
         <a id="OpenXMLtoBinary_MethodsConv"> </a> 
         <div> 
          <h2 class="LW_CollapsibleArea_TitleDiv"> 
           <div> 
            <a class="LW_CollapsibleArea_TitleAhref" title="Collapse" role="heading"> <span class="cl_CollapsibleArea_expanding LW_CollapsibleArea_Img"></span> <span class="LW_CollapsibleArea_Title">Methods of Conversion</span> </a> 
            <div id="Anchor_1" class="LW_CollapsibleArea_Anchor_Div"> 
             <a href="/en-us/library/office/jj649005(v=office.14).aspx#Anchor_1" class="LW_CollapsibleArea_Anchor_Img" title="Right-click to copy and share the link for this section"></a> 
            </div> 
            <div class="LW_CollapsibleArea_HrDiv"> 
             <hr class="LW_CollapsibleArea_Hr"> 
            </div> 
           </div> </h2> 
          <div class="sectionblock"> 
           <a id="sectionToggle1"> </a> 
           <p>Most binary-to-XML conversions of Microsoft Office files can be done with existing tools. This is by far the easiest approach. If you need to convert a group of documents quickly, you should use a tool that has a command-line interface. </p> 
          </div> 
         </div> 
         <a id="OpenXMLtoBinary_ConversionTools"> </a> 
         <div> 
          <h2 class="LW_CollapsibleArea_TitleDiv"> 
           <div>

            <div id="Anchor_2" class="LW_CollapsibleArea_Anchor_Div"> 
             <a href="/en-us/library/office/jj649005(v=office.14).aspx#Anchor_2" class="LW_CollapsibleArea_Anchor_Img" title="Right-click to copy and share the link for this section"></a>

            <div class="LW_CollapsibleArea_HrDiv"> 
             <hr class="LW_CollapsibleArea_Hr">

           </div> </h2> 
          <div class="sectionblock"> 
           <a id="sectionToggle2"> </a> 
           <p> </p> 
           <a id="OpenXMLtoBinary_OFC"> </a> 
           <h3 class="subHeading">Office File Converter</h3> 
           <div class="subsection"> 
            <p>You can use the Office File Converter (OFC), which is part of the <a href="https://www.microsoft.com/download/en/details.aspx?&amp;id=21888" target="_blank">2007 Microsoft Office System Migration Guidance: Microsoft Office Migration Planning Manager</a>, to translate groups of documents from binary to Open XML formats. To run Office File Converter, customize the ofc.ini file to point to the directory with the files to convert, then run ofc.exe. If you use an alternate .ini file, include the path to the file as a command parameter. For more information, see the documentation that comes with the Office File Converter download, or read the blog post <a href="http://blogs.msdn.com/b/dmahugh/archive/2007/02/09/converting-office-documents-to-open-xml.aspx" target="_blank">Converting Office documents to Open XML</a>.</p> 
           </div> 
           <a id="OpenXMLtoBinary_OCP"> </a> 
           <h3 class="subHeading">Office Compatibility Pack</h3> 
           <div class="subsection"> 
            <p>You can use the <a href="https://www.microsoft.com/download/en/details.aspx?&amp;id=3" target="_blank">Microsoft Office Compatibility Pack for Word, Excel, and PowerPoint File Formats</a> if you are using Microsoft Office 2000, Microsoft Office XP, or Microsoft Office 2003 to convert your own individual files to Open XML formats. The Office Compatibility Pack does not support bulk conversions.</p> 
            <p>You can also use the Office Compatibility Pack to open, edit, and save files from other users that are already in Open XML formats.</p> 
           </div> 
          </div> 
         </div>

           <a id="OpenXMLtoBinary_OfficeAutomation"> </a> 
           <h3 class="subHeading">Office Automation</h3> 
           <div class="subsection"> 
            <p>Office Automation is a framework and object model available through Visual Studio Tools for Office (VSTO) that enables developers to programmatically run Office client applications and execute commands within those applications. It is easy to use and well-suited to batch conversions. It requires loading files in the client application, which may limit performance over large document stores.</p> 
            <p>The following code, run from within a VSTO application, opens a binary Word document and saves it in Open XML format with a new name.</p> 
            <div id="code-snippet-1" class="codeSnippetContainer" xmlns=""> 
             <div class="codeSnippetContainerTabs"> 
              <div class="codeSnippetContainerTabSingle" dir="ltr">
               <a>C#</a>
              </div> 
             </div> 
             <div class="codeSnippetContainerCodeContainer"> 
              <div class="codeSnippetToolBar"> 
               <div class="codeSnippetToolBarText"> 
                <a name="CodeSnippetCopyLink" style="display: none;" title="Copy to clipboard." href="javascript:if (window.epx.codeSnippet)window.epx.codeSnippet.copyCode('CodeSnippetContainerCode_f667a0d4-071c-4fa3-bead-39c74aab1ad1');" ms.cmptyp="CodeSnippet">Copy</a> 
               </div> 
              </div> 
              <div id="CodeSnippetContainerCode_f667a0d4-071c-4fa3-bead-39c74aab1ad1" class="codeSnippetContainerCode" dir="ltr"> 
               <div style="color:Black;">
                <pre>
<span style="color:Blue;">this</span>.Application.Documents.Open(<span style="color:#A31515;">@"C:\BinaryDocument.doc"</span>);
<span style="color:Blue;">this</span>.SaveAs(<span style="color:#A31515;">@"C:\OpenXmlDocument.docx"</span>, WdSaveFormat.wdFormatXMLDocument);
</pre>
               </div> 
              </div>

           <a id="OpenXMLtoBinary_WordAS"> </a> 
           <h3 class="subHeading">Word Automation Services</h3> 
           <div class="subsection"> 
            <p>Microsoft Word Automation Services is a feature of Microsoft SharePoint 2010 that emulates some aspects of Word Automation on the server. You can use Word Automation Services to convert files between supported formats across a SharePoint farm. Supported formats include .pdf, .rtf, .doc, and .docx.</p> 
            <p>The following example loads a binary .doc file and creates an Open XML .docx file with the same content. It does not delete or overwrite the original.</p> 
            <div id="code-snippet-2" class="codeSnippetContainer" xmlns=""> 
             <div class="codeSnippetContainerTabs"> 
              <div class="codeSnippetContainerTabSingle" dir="ltr">
               <a>C#</a>
              </div> 
             </div> 
             <div class="codeSnippetContainerCodeContainer"> 
              <div class="codeSnippetToolBar"> 
               <div class="codeSnippetToolBarText"> 
                <a name="CodeSnippetCopyLink" style="display: none;" title="Copy to clipboard." href="javascript:if (window.epx.codeSnippet)window.epx.codeSnippet.copyCode('CodeSnippetContainerCode_729a7acc-a466-4431-8f93-b3ef53f478dc');" ms.cmptyp="CodeSnippet">Copy</a> 
               </div> 
              </div> 
              <div id="CodeSnippetContainerCode_729a7acc-a466-4431-8f93-b3ef53f478dc" class="codeSnippetContainerCode" dir="ltr"> 
               <div style="color:Black;">
                <pre>
ConversionJob job = <span style="color:Blue;">new</span> ConversionJob(<span style="color:#A31515;">"Word Automation Services"</span>); 
job.Name = <span style="color:#A31515;">"Binary to OpenXML"</span>; 
job.UserToken = myWebsite.CurrentUser.UserToken; 
job.AddFile(fileName, fileName.Replace(<span style="color:#A31515;">".doc"</span>, <span style="color:#A31515;">".docx"</span>)); 
job.Start();

</pre>
               </div> 
              </div> 
             </div> 
            </div>

           <a id="OpenXMLtoBinary_BinaryRead"> </a> 
           <h3 class="subHeading">Converting by Reading the Binary Data</h3> 
           <div class="subsection"> 
            <p>In the rare case where none of the existing tools or development platforms meet your needs, you can work directly from the binary data as specified in the <span><a href="https://msdn.microsoft.com/en-us/library/office/cc313105(v=office.14).aspx">Microsoft Office File Format Documents</a></span>. First, create a model of the relevant Open XML schema in memory, then read the binary file stream and extract the data to fill in the schema piece by piece. </p> 
            <p> </p> 
           </div>
