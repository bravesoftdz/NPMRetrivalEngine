           <h3>Why Use Video to Picture Image Converter to Capture Picture from Video Instead of key "Print Screen"?</h3> 
          </div> <p><img src="images/print-screen.jpg" width="700" height="179" align="right" alt="Print Screen">You know when pressing key "Print Screen" (often abbreviated <i>Prt Scr</i>, refer to the right figure) Windows takes a snapshot or picture of your computer screen and copies it to the clipboard. You will get the picture when you paste on Paint. It's an easy and effective way to get screen image. However, when the image screenshot from clipboard is pasted into an image editor such as Paint (Start -&gt; All Programs -&gt; Accessories -&gt; Paint), the capture is a <b>black blank screen</b> instead of the actual video.</p> <p>Why is that? When the video plays, it is actually displayed on a different surface/layer called overlay that is produced by hardware acceleration. When you take a normal screen capture, you're taking it of the normal surface where the video isn't displayed. That's why it comes out black as it is invisible to the screen capture software.</p> <p>The Video to Picture Image Converter is a <b>professional video to picture conversion software</b> that captures every frame of video and then save to still picture/image file such as JPG, BMP, PNG, TIFF, etc. After conversion, you will get an image sequence, and then you could easily choose the best picture from the sequence.</p> <br>

           <h3>About Picture/Image Formats</h3> 
          </div> 
          <ul> 
           <li><b>BMP</b> (<i>lossless, uncompressed, big file</i>): <b>b</b>it<b>m</b>a<b>p</b> image file or Device Independent Bitmap (DIB) file format, is a raster graphics image file format used to store bitmap digital images, independently of the display device.</li> 
           <li><b>JPG/JPEG</b> (<i>lossy, compressed, small file</i>): <b>J</b>oint <b>P</b>hotographic <b>E</b>xperts <b>G</b>roup, a commonly used method of lossy compression for digital photography (image); JPEG typically achieves 10:1 compression with little perceptible loss in image quality.</li> 
           <li><b>PCX</b> (<i>lossless, compressed, bigger than PNG</i>): <b>P</b>ersonal <b>C</b>omputer e<b>X</b>change, the native file format for PC Paintbrush and became one of the first widely accepted DOS imaging standards.</li> 
           <li><b>PGM</b> (<i>lossy, uncompressed, big file</i>): <b>P</b>ortable <b>G</b>ray<b>M</b>ap Format, a lowest common denominator grayscale file format. It is designed to be extremely easy to learn and write programs for.</li> 
           <li><b>PIX</b> (lossless, compressed): Alias PIX (PIX) is a raster image file format. It uses RLE compression, and supports truecolor and grayscale images.</li> 
           <li><b>PNG</b> (<i>lossless, compressed, small file, bigger than JPG sometimes</i>): <b>P</b>ortable <b>N</b>etwork <b>G</b>raphics, a bitmapped image format that employs lossless data compression. It was designed to replace GIF and TIFF formats. PNG does not require a patent license.</li> 
           <li><b>PPM</b> (<i>lossless, uncompressed, big file</i>): <b>P</b>ortable <b>P</b>ix<b>M</b>ap Format, supports full-color images. PPM is a convenient and simple method of saving image data. It is equally easy to read in ones own applications.</li> 
           <li><b>RAS</b> (lossless, compressed or uncompressed, big file): <b>Ras</b>ter image format used by Sun Microsystems computers, typically created on a Unix workstation; supports 1, 8, 24, and 32 bits per pixel; can be uncompressed or compressed using RLE compression; recognized by most image viewing programs.</li> 
           <li><b>SGI</b> (<i>lossless, compressed, bigger than PNG</i>): <b>S</b>ilicon <b>G</b>raphics <b>I</b>mage, the native raster graphics file format for Silicon Graphics workstations.</li> 
           <li><b>TGA</b> (<i>lossless, compressed, bigger than PNG</i>): <b>T</b>ruevision <b>G</b>raphics <b>A</b>dapter, a raster graphics file format created by Truevision Inc. It was the native format of TARGA and VISTA boards, which were the first graphic cards for IBM-compatible PCs to support Highcolor/truecolor display.</li> 
           <li><b>TIFF</b> (<i>lossless, compressed, middle file, bigger than PNG sometimes</i>): <b>T</b>agged <b>I</b>mage <b>F</b>ile <b>F</b>ormat, a variable-resolution bitmapped image format. TIFF is very common for transporting color or gray-scale images into page layout applications.</li> 
           <li><b>WebP</b> ((<i>lossy, compressed, smaller file</i>): WebP is an image format employing both lossy and lossless compression. It is currently developed by Google, based on technology acquired with the purchase of On2 Technologies. WebP was first announced in 2010 as a new open standard for lossily compressed true-color graphics on the web, producing smaller files of comparable image quality to the older JPEG scheme.</li> 
           <li><b>XBM</b> (uncompressed): XBM is a monochrome bitmap format in which data is stored as a C language data array. Primarily used for the storage of cursor and icon bitmaps for use in the X graphical user interface.</li> 
           <li><b>GIF Animation</b>: Graphics Interchange Format is a bitmap image format that was introduced by CompuServe in 1987 and has since come into widespread usage on the World Wide Web due to its wide support and portability. The format supports up to 8 bits per pixel for each image, allowing a single image to reference its own palette of up to 256 different colors chosen from the 24-bit RGB color space. It also supports animations and allows a separate palette of up to 256 colors for each frame.</li> 
          </ul> <br>
