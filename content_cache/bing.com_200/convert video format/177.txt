             <div class="post-text" itemprop="text"> 
              <p>You can make use of <code>ffmpeg</code> to convert h265 format to h264 format.</p> 
              <ol> 
               <li>Install <code>ffmpeg</code> using command <code>sudo apt-get install ffmpeg</code></li> 
               <li>Download h265 codecs. <code>wget http://ftp.videolan.org/pub/videolan/x265/x265_1.7.tar.gz</code></li> 
               <li>Install dependencies for compilation. <code>sudo apt-get install cmake cmake-curses-gui build-essential yasm</code></li> 
               <li>untar it by <code>tar -xvzf x265_1.7.tar.gz</code></li> 
               <li><code>cd x265_1.7/build/linux</code></li> 
               <li><code>./make-Makefiles.bash</code> It will ask you for cmake option. Press <code>c</code> to configure and <code>g</code> to generate and exit</li> 
               <li><code>make</code> to compile them</li> 
               <li><code>sudo make install</code> to install libx265 to default <code>lib</code> path</li> 
              </ol> 
              <p>Now that we have <code>ffmpeg</code> and <code>libx265</code> installed. To convert just video, <code>ffmpeg -i &lt;input-file&gt; -an -x265-params crf=25 &lt;output-file.mp4&gt;</code> To convert with audio, <code>ffmpeg -i &lt;input-file&gt; -c:a copy -x265-params crf=25 &lt;output-file.mp4&gt;</code></p> 
              <p>Please note that, it takes a decent amount of time to convert on x86_64 architecture. Converting it natively on Rpi will take about 4 - 5 x amount of time.</p> 
              <p>For more details : Read <a href="https://superuser.com/questions/785528/how-to-generate-an-mp4-with-h-265-codec-using-ffmpeg">this</a></p> 
              <p>Hope it helps.</p>
