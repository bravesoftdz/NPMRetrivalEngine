       <h1 class="post-title">String Similarity: Hackerrank</h1> 
       <div class="entry-meta"> 
        <span class="cat"><a href="https://algoexplode.wordpress.com/category/hacker-rank/algorithm-strings/" rel="category tag">Algorithm-Strings</a>, <a href="https://algoexplode.wordpress.com/category/hacker-rank/" rel="category tag">Hacker Rank</a></span> 
        <span class="date">August 1, 2013</span> 
        <span class="comments"><a href="https://algoexplode.wordpress.com/2013/08/01/string-similarity-hackerrank/#comments">Comments: 6</a></span>

       <div class="entry-content"> 
        <p><strong>Problem link:</strong> <a href="https://www.hackerrank.com/challenges/string-similarity" target="_blank">https://www.hackerrank.com/challenges/string-similarity</a></p> 
        <p><strong>Problem Statement:</strong><br> For two strings A and B, we define the similarity of the strings to be the length of the longest prefix common to both strings. For example, the similarity of strings “abc” and “abd” is 2, while the similarity of strings “aaa” and “aaab” is 3.</p> 
        <p>Calculate the sum of similarities of a string S with each of it’s suffixes.<br> <span id="more-108"></span><br> <strong>Input:</strong><br> The first line contains the number of test cases T. Each of the next T lines contains a string each.</p> 
        <p><strong>Output:</strong><br> Output T lines containing the answer for the corresponding test case.</p> 
        <p><strong>Constraints:</strong><br> 1. <img src="https://s0.wp.com/latex.php?latex=1%5Cleq+T%5Cleq+10&amp;bg=ffffff&amp;fg=000000&amp;s=0" alt="1\leq T\leq 10" title="1\leq T\leq 10" class="latex"><br> 2. The length of each string is at most 100000 and contains only lower case characters.</p> 
        <p><strong>Solution Idea:</strong><br> The obvious solution is straight forward. You write a function that calculates the similarity (length of longest prefix) between 2 strings. Now you call that function with the string and each of its suffix (by using the substring method). </p> 
        <p><strong>Edit:</strong><br> The solution mentioned above is not good enough for an acceptance in the HackerRank Website as pointed out by Shiv. The solution is commented out in the code. Now, how can we improve the code. First of all, calculating the substring is an expensive operation and we can achieve what we want by keeping track of the start index. Secondly, String.charAt() seems to be slower than accessing an array with an index. So, I first convert the string to an array and then use index. With this 2 improvement, I was able to get it accepted in the HackerRank Website, Yay!</p> 
        <p><strong>Important:</strong><br> Although, I got the code accepted in the website, I still believe the original intention of the author of the problem is to make us find a linear time algorithm. There is one using Suffix array. A suffix array is an array of integers giving the starting positions of all suffixes of a string in lexicographical order. A suffix array can be constructed in linear time (Look at this research: <a href="http://www.cs.sysu.edu.cn/nong/index.files/Two%20Efficient%20Algorithms%20for%20Linear%20Suffix%20Array%20Construction.pdf" target="_blank">http://www.cs.sysu.edu.cn/nong/index.files/Two%20Efficient%20Algorithms%20for%20Linear%20Suffix%20Array%20Construction.pdf</a> ). When the suffix array is constructed then this problem can be solved in linear time (Look at this stackoverflow: <a href="http://stackoverflow.com/questions/11069467/longest-common-prefixes" target="_blank">http://stackoverflow.com/questions/11069467/longest-common-prefixes</a> ) </p>
