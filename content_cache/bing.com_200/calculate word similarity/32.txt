      <h1>Distance 0.1.3</h1> 
      <div id="download-button"> 
       <a class="button green" style="float:right;" href="https://pypi.python.org/packages/5c/1a/883e47df323437aefa0d0a92ccfb38895d9416bd0b56262c2e46a47767b8/Distance-0.1.3.tar.gz">Download<br> <span style="font-size: 75%">Distance-0.1.3.tar.gz</span> </a> 
      </div> 
      <p style="font-style: italic">Utilities for comparing sequences</p> 
      <p>This package provides helpers for computing similarities between arbitrary sequences. Included metrics are Levenshtein, Hamming, Jaccard, and Sorensen distance, plus some bonuses. All distance computations are implemented in pure Python, and most of them are also implemented in C.</p> 
      <div id="installation"> 
       <h2>Installation</h2> 
       <p>If you don’t want or need to use the C extension, just unpack the archive and run, as root:</p> 
       <blockquote>
         # python setup.py install
       </blockquote> 
       <p>For the C extension to work, you need the Python source files, and a C compiler (typically Microsoft Visual C++ 2010 on Windows, and GCC on Mac and Linux). On a Debian-like system, you can get all of these with:</p> 
       <blockquote>
         # apt-get install gcc pythonX.X-dev
       </blockquote> 
       <p>where X.X is the number of your Python version.</p> 
       <p>Then you should type:</p> 
       <blockquote>
         # python setup.py install –with-c
       </blockquote> 
       <p>Note the use of the <cite>–with-c</cite> switch.</p> 
      </div> 
      <div id="usage"> 
       <h2>Usage</h2> 
       <p>A common use case for this module is to compare single words for similarity:</p> 
       <blockquote> 
        <pre>&gt;&gt;&gt; distance.levenshtein("lenvestein", "levenshtein")
3
&gt;&gt;&gt; distance.hamming("hamming", "hamning")
1
</pre> 
       </blockquote> 
       <p>If there is not a one-to-one mapping between sounds and glyphs in your language, or if you want to compare not glyphs, but syllables or phonems, you can pass in tuples of characters:</p> 
       <blockquote> 
        <pre>&gt;&gt;&gt; t1 = ("de", "ci", "si", "ve")
&gt;&gt;&gt; t2 = ("de", "ri", "si", "ve")
&gt;&gt;&gt; distance.levenshtein(t1, t2)
1
</pre> 
       </blockquote> 
       <p>Comparing lists of strings can also be useful for computing similarities between sentences, paragraphs, etc.:</p> 
       <blockquote> 
        <pre>&gt;&gt;&gt; sent1 = ['the', 'quick', 'brown', 'fox', 'jumps', 'over', 'the', 'lazy', 'dog']
&gt;&gt;&gt; sent2 = ['the', 'lazy', 'fox', 'jumps', 'over', 'the', 'crazy', 'dog']
&gt;&gt;&gt; distance.levenshtein(sent1, sent2)
3
</pre> 
       </blockquote> 
       <p>Hamming and Levenshtein distance can be normalized, so that the results of several distance measures can be meaningfully compared. Two strategies are available for Levenshtein: either the length of the shortest alignment between the sequences is taken as factor, or the length of the longer one. Example uses:</p> 
       <blockquote> 
        <pre>&gt;&gt;&gt; distance.hamming("fat", "cat", normalized=True)
0.3333333333333333
&gt;&gt;&gt; distance.nlevenshtein("abc", "acd", method=1)  # shortest alignment
0.6666666666666666
&gt;&gt;&gt; distance.nlevenshtein("abc", "acd", method=2)  # longest alignment
0.5
</pre> 
       </blockquote> 
       <p><cite>jaccard</cite> and <cite>sorensen</cite> return a normalized value per default:</p> 
       <blockquote> 
        <pre>&gt;&gt;&gt; distance.sorensen("decide", "resize")
0.5555555555555556
&gt;&gt;&gt; distance.jaccard("decide", "resize")
0.7142857142857143
</pre> 
       </blockquote> 
       <p>As for the bonuses, there is a <cite>fast_comp</cite> function, which computes the distance between two strings up to a value of 2 included. If the distance between the strings is higher than that, -1 is returned. This function is of limited use, but on the other hand it is quite faster than <cite>levenshtein</cite>. There is also a <cite>lcsubstrings</cite> function which can be used to find the longest common substrings in two sequences.</p> 
       <p>Finally, two convenience iterators <cite>ilevenshtein</cite> and <cite>ifast_comp</cite> are provided, which are intended to be used for filtering from a long list of sequences the ones that are close to a reference one. They both return a series of tuples (distance, sequence). Example:</p> 
       <blockquote> 
        <pre>&gt;&gt;&gt; tokens = ["fo", "bar", "foob", "foo", "fooba", "foobar"]
&gt;&gt;&gt; sorted(distance.ifast_comp("foo", tokens))
[(0, 'foo'), (1, 'fo'), (1, 'foob'), (2, 'fooba')]
&gt;&gt;&gt; sorted(distance.ilevenshtein("foo", tokens, max_dist=1))
[(0, 'foo'), (1, 'fo'), (1, 'foob')]
</pre> 
       </blockquote> 
       <p><cite>ifast_comp</cite> is particularly efficient, and can handle 1 million tokens without a problem.</p> 
       <p>For more informations, see the functions documentation (<cite>help(funcname)</cite>).</p> 
       <p>Have fun!</p> 
      </div> 
      <div id="changelog"> 
       <h2>Changelog</h2> 
       <p>20/11/13: * Switched back to using the to-be-deprecated Python unicode api. Good news is that this makes the C extension compatible with Python 2.7+, and that distance computations on unicode strings is now much faster. * Added a C version of <cite>lcsubstrings</cite>. * Added a new method for computing normalized Levenshtein distance. * Added some tests.</p> 
       <p>12/11/13: Expanded <cite>fast_comp</cite> (formerly <cite>quick_levenshtein</cite>) so that it can handle transpositions. Fixed variable interversions in (C) <cite>levenshtein</cite> which produced sometimes strange results.</p> 
       <p>10/11/13: Added <cite>quick_levenshtein</cite> and <cite>iquick_levenshtein</cite>.</p> 
       <p>05/11/13: Added Sorensen and Jaccard metrics, fixed memory issue in Levenshtein.</p>
