           <p>Only a few years ago, developing mobile apps via JavaScript was nothing more then a quirky experiment. The idea that you could build iOS and Android apps, without having to operate with Java and Objective C, seemed appealing to many web devs. Now we have a plethora of frameworks bringing us closer than ever to creating a native mobile experience using only web technologies.</p> 
           <h2>What are Hybrid Mobile Apps?</h2> 
           <p>A hybrid app is just a regular mobile optimized website, written in CSS, HTML and JavaScript, that is displayed in a webview (this is a basically a stripped down web browser). The advantage is that you only need to write a single application, which in most cases runs unmodified on Android, iOS and Windows Phone. Most of the frameworks listed in this article do this using <a href="http://cordova.apache.org/" target="_blank" rel="noreferrer noopener">Cordova</a> or <a href="http://phonegap.com/" target="_blank" rel="noreferrer noopener">PhoneGap</a>, which give you a bridge to the device APIs in JavaScript.</p> 
           <p>In this article we're going to compare the most popular JavaScript frameworks for building hybrid and native mobile apps. <strong>Scroll to the bottom of the article to see the scores.</strong></p> 
           <h2>1. <a href="http://ionicframework.com/" target="_blank" rel="noreferrer noopener">Ionic</a></h2> 
           <figure>
            <a href="http://ionicframework.com/" target="_blank" rel="noreferrer noopener"><img src="https://tutorialzine.com/media/2015/10/ionic.jpg" width="1024" height="596" alt="ionic.jpg"></a>
            <figcaption>
              Ionic
            </figcaption>
           </figure> 
           <p>Ionic is the most popular framework on our list and probably the first choice of many developers. You can use the CSS portion of the framework to create native looking designs, but to harness the full potential of Ionic, it's best to pair it with AngularJS. A big bonus you get with Ionic is the command-line interface which is full of awesome features including integrated emulators and a Cordova based app packager.</p> 
           <p><strong>Pros:</strong></p> 
           <ul>
            <li>Works with predefined components</li> 
            <li>Great community</li> 
            <li>Command Line Interface with lots of useful features</li> 
           </ul> 
           <p><strong>Cons:</strong></p> 
           <ul>
            <li>You need to know AngularJS to do anything complex</li> 
           </ul> 
           <h2>2. <a href="https://onsen.io/" target="_blank" rel="noreferrer noopener">Onsen UI</a></h2> 
           <figure>
            <a href="https://onsen.io/" target="_blank" rel="noreferrer noopener"><img src="https://tutorialzine.com/media/2015/10/onsen_ui.jpg" width="1024" height="704" alt="onsen_ui.jpg"></a>
            <figcaption>
              Onsen UI
            </figcaption>
           </figure> 
           <p>This is an open source framework that allows developers to build apps by combining native-looking components. It's fairly simple to use, can work with or without AngularJS, and has great documentation that includes lots of examples and layouts for the most common app structures. A disadvantage of Onsen UI is that it currently offers only an iOS theme, although the next version promises Material Design support.</p> 
           <p><strong>Pros:</strong></p> 
           <ul>
            <li>Works with predefined components</li> 
            <li>Excellent documentation with examples</li> 
           </ul> 
           <p><strong>Cons:</strong></p> 
           <ul>
            <li>PhoneGap/Cordova builder not included, but supported</li> 
            <li>No support for Material Design (yet)</li> 
           </ul> 
           <h2>3. <a href="http://www.idangero.us/framework7" target="_blank" rel="noreferrer noopener">Framework 7</a></h2> 
           <figure>
            <a href="http://www.idangero.us/framework7" target="_blank" rel="noreferrer noopener"><img src="https://tutorialzine.com/media/2015/10/framework_7.jpg" width="1024" height="521" alt="framework_7.jpg"></a>
            <figcaption>
              Framework 7
            </figcaption>
           </figure> 
           <p>The cool thing about Framework 7 is that it is completely framework agnostic (doesn't have external dependencies like Angular or React) and still manages to make apps look and feel native, with properly styled components and animations. Anyone who understands HTML, CSS and JavaScript can create an app without making the code convoluted. Framework 7 doesn't include any tools for emulation or app packaging so you will need to combine it with Cordova or PhoneGap.</p> 
           <p><strong>Pros:</strong></p> 
           <ul>
            <li>Simple to use, relies only on HTML, CSS and JavaScript</li> 
            <li>Good performance</li> 
            <li>Can be combined with any JavaScript framework of choice</li> 
           </ul> 
           <p><strong>Cons:</strong></p> 
           <ul>
            <li>PhoneGap/Cordova builder not included, but supported</li> 
           </ul> 
           <h2>4. <a href="http://www.reactnative.com/" target="_blank" rel="noreferrer noopener">React Native</a></h2> 
           <figure>
            <a href="http://www.reactnative.com/" target="_blank" rel="noreferrer noopener"><img src="https://tutorialzine.com/media/2015/10/react_native.jpg" width="1024" height="572" alt="react_native.jpg"></a>
            <figcaption>
              React Native
            </figcaption>
           </figure> 
           <p>As the name implies, React Native's purpose is to build proper native apps, instead of creating hybrid ones that run in a Webview. Development, however, is still done completely via JavaScript and React. This frameworks isn't tailored towards beginners in web development, but on the plus side, there is a huge community behind it that will help you in every part of the way. Recently the framework rolled support for Android, so you can have real cross-platform apps.</p> 
           <p><strong>Pros:</strong></p> 
           <ul>
            <li>Native-like performance</li> 
            <li>Huge community</li> 
           </ul> 
           <p><strong>Cons:</strong></p> 
           <ul>
            <li>Steep learning curve</li> 
            <li>The development tools only work on OS X at the moment</li> 
           </ul> 
           <h2>5. <a href="http://jquerymobile.com/" target="_blank" rel="noreferrer noopener">jQuery Mobile</a></h2> 
           <figure>
            <a href="http://jquerymobile.com/" target="_blank" rel="noreferrer noopener"><img src="https://tutorialzine.com/media/2015/10/jquery_mobile.jpg" width="1024" height="613" alt="jquery_mobile.jpg"></a>
            <figcaption>
              jQuery Mobile
            </figcaption>
           </figure> 
           <p>The grandpa of all mobile frameworks, jQuery Mobile doesn't try to make apps that look like Android or iOS. Instead, it's purpose is to help develop web apps that will work equally well on all mobile browsers (including oldies such as Windows Phone, Blackberry, and Symbian). As such, it is very lightweight, depends only on jQuery and is quite easy to learn, while still offering good touch recognition and PhoneGap/Cordova support.</p> 
           <p><strong>Pros:</strong></p> 
           <ul>
            <li>Support for wide range of mobile browsers</li> 
            <li>Simple to use</li> 
           </ul> 
           <p><strong>Cons:</strong></p> 
           <ul>
            <li>Dated styles that don't resemble either iOS or Android</li> 
            <li>PhoneGap/Cordova builder not included, but supported</li> 
           </ul> 
           <h2>6. <a href="https://www.nativescript.org/" target="_blank" rel="noreferrer noopener">Native Script</a></h2> 
           <figure>
            <a href="https://www.nativescript.org/" target="_blank" rel="noreferrer noopener"><img src="https://tutorialzine.com/media/2015/10/native_script.jpg" width="1024" height="574" alt="native_script.jpg"></a>
            <figcaption>
              NativeScript
            </figcaption>
           </figure> 
           <p>Native script's biggest feature is that it allows you to write the functionality of your app in JavaScript once, which then will be transformed accordingly to Android, iOS and Windows Phone. Packaged, the compiled applications start in a native fashion, without opening and running in a browser. This frameworks does require some coding skills, but compensates for it with an extensive, in-depth documentation.</p> 
           <p><strong>Pros:</strong></p> 
           <ul>
            <li><em>Write once, use everywhere</em> approach.</li> 
            <li>Great documentation</li> 
           </ul> 
           <p><strong>Cons:</strong></p> 
           <ul>
            <li>Steep learning curve</li> 
            <li>Small community</li> 
           </ul> 
           <h2>7. <a href="http://famous.org/" target="_blank" rel="noreferrer noopener">Famous</a></h2> 
           <figure>
            <a href="http://famous.org/" target="_blank" rel="noreferrer noopener"><img src="https://tutorialzine.com/media/2015/10/famous.jpg" width="1024" height="544" alt="famous.jpg"></a>
            <figcaption>
              Famous
            </figcaption>
           </figure> 
           <p>Famous has a unique approach to web and mobile development. It combines the DOM tree (your HTML) with WebGL, displaying everything in a canvas, similar to what HTML game engines do. This novel technique allows the framework to run your apps in 60 fps, which is as smooth as most native apps. Sadly, this project is no longer being actively developed and doesn't have good docs for reference.</p> 
           <p><strong>Pros:</strong></p> 
           <ul>
            <li>Native-like performance</li> 
           </ul> 
           <p><strong>Cons:</strong></p> 
           <ul>
            <li>No longer actively developed</li> 
            <li>Documentation not full</li> 
            <li>Small community</li> 
           </ul> 
           <h2>The Rundown</h2> 
           <div>
            <iframe src="https://demo.tutorialzine.com/2015/10/comparing-the-top-frameworks-for-hybrid-apps/comparison5.html" width="100%" height="410" scrolling="yes" frameborder="0"></iframe>
           </div> 
           <h2>Conclusion</h2> 
           <p>There is no best framework - all of them have their pros and cons and it's up to you to decide depending on what you plan on using them for.</p> 
           <p>We hope this quick comparison has been of great use to you! If you have experience with any of the listed frameworks or similar ones, please leave a comment and share your opinion on hybrid app development!</p>
