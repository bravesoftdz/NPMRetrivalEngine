         <p><img alt="HTML5 Mobile Ui Frameworks" height="305" width="610" src="https://software.intel.com/sites/default/files/managed/19/f7/html5-mobile-ui-frameworks.png"></p>
         <p><strong>By <a href="/en-us/html5/hub/developers/andrew-smith" rel="nofollow">Andrew Smith</a></strong></p>
         <p>In response to the growing market for mobile apps several HTML5 Mobile UI Frameworks are now available to help developers create attractive and responsive apps. &nbsp;These libraries enable Web developers to create great user experiences for their apps using technology and skills that they are familiar with. &nbsp;The best of these UI/UX frameworks include App Framework, Twitter Bootstrap 3, jQuery Mobile, Sencha Touch, Kendo UI, and TopCoat. &nbsp;Each one provides a unique set of features to help Web developers make great apps.</p>
         <h2>What are UI Frameworks?</h2>
         <p>HTML5 Mobile User Interface Frameworks are libraries that will enable developers to easily create a professional grade user experience for their HTML5-powered app. &nbsp;These libraries are included as one or more files written using JavaScript and CSS. &nbsp;They are meant to be used by including them in an HTML file. &nbsp;They are appropriate for a variety of uses. &nbsp;All these platforms are appropriate for mobile Web apps as well as for native hybrid applications where a native wrapper allows the entire user experience be driven by HTML5. &nbsp;These hybrid applications are built using a wrapping technology like Cordova to give HTML5 applications access to native features of the device such as the camera through a JavaScript bridge API.</p>
         <h2>Why would I want to use one for mobile?</h2>
         <p>There are a lot of great reasons why Web developers would choose to use an HTML5 Mobile UI Framework for their next app. &nbsp;Developers who learn how to use frameworks save a lot of time and effort styling and animating their app experiences. &nbsp;All the frameworks listed in this article provide the tools to create a great app for both tablet and phone form-factors, but some even adapt to add support for desktop browsers. &nbsp;HTML5 Mobile UI frameworks can give developers apps a unified look and feel to make the software much more professional than a home-grown solution can. &nbsp;Some HTML5 Mobile UI frameworks even adapt depending on the platform they are running on to present users with an experience customized for the operating system they are running the software on.</p>
         <h2>How are they all alike?</h2>
         <p>These frameworks have a lot in common to help developers make great mobile apps.</p>
         <p>All these frameworks are also designed to work on a variety of platforms. &nbsp;There is a panoply of mobile devices in the market, and these frameworks aim to support as many of them as possible. &nbsp;They do that by using the technique of Responsive Design or a combination of Responsive Design and Reactive Design. &nbsp;For those keeping score at home, Responsive Design is a process of laying out Web UI elements so that the UI can be resized at will with a minimum of redrawing on all form factors while Reactive Design is a process where UI elements are turned on or off depending on the platform the application is running on.</p>
         <p>Most of these frameworks are based on jQuery or a jQuery style query selector JavaScript engine. &nbsp;A query selector is a JavaScript library that allows the framework to select elements make changes to the Document Object Model or DOM. Without a query selector those frameworks that require a query selector wont work appropriately or not work at all.</p>
         <p>With a few exceptions, these frameworks provide JavaScript support for scrolling through HTML elements as well as animated transitions between pages. &nbsp;Additionally, each of these frameworks gives developers a way to display basic navigational elements and controls to create an application using Web development skills like HTML5, CSS3, and JavaScript.</p>
         <p>These frameworks all include methods for loading data from the Web dynamically using asynchronous techniques to grab data in XML or JSON format. &nbsp;By providing access to resources on the Internet, these frameworks accelerate development and extend the capability of what is possible with HTML5 Mobile Apps.</p>
         <p>There are a variety of HTML5 user interface elements that all these libraries provide. &nbsp;Buttons, menus, forms, lists, toolbars, tabs, modal overlays, and a variety of other controls are available from these frameworks. &nbsp;Most of these UI frameworks use font files to deliver icon images to the screen. &nbsp;Font files are a superior solution to flat files for icons because they consume less bandwidth and are naturally resizable.</p>
         <p>All these frameworks take advantage of touch events as well as click events, allowing them to be used by mobile devices as well as desktop mouse-driven devices. &nbsp;Many frameworks make sure HTML elements are sized appropriately for touch activation as well.</p>
         <p>Many of these frameworks include a method of adding functionality through plugins, or additional CSS or JavaScript library files. &nbsp;Developers only include what is necessary and nothing else. This progressive enhancement conserves as much speed and responsiveness, while providing options to add extra functionality as necessary.</p>
         <h2>How are they different?</h2>
         <p>Although the examples of HTML5 Mobile UI Frameworks mentioned earlier are very similar in a lot of ways, there are still significant differences as well. &nbsp;Here is a brief comparison of the strengths of App Framework, Twitter Bootstrap 3, jQuery Mobile, Sencha Touch, Kendo UI, and TopCoat.</p>
         <h3>App Framework</h3>
         <p>Intels App Framework is a reliable and responsive HTML5 Mobile UI Framework that is ideal for creating hybrid mobile apps. &nbsp;It is optimized for mobile, and so it lacks a lot of the moving parts that would allow the framework to adapt to the desktop. &nbsp;However, that simplicity can be a boon for newer developers. &nbsp;An app can be written with simple HTML5 markup alone, so even basic users mostly unfamiliar with JavaScript can use the framework. &nbsp;It gives developers the ability to write a single codebase, and have their application adapt a native-style look and feel depending on the device the app runs on. &nbsp;For example, the Mobile HTML5 app would look like an iOS app on an iPhone, a native Android app on a Galaxy, and so on. &nbsp;App Framework is used extensively in the Intel XDK which includes a great app prototyping tool named App Starter.</p>
         <h3>Twitter Bootstrap 3</h3>
         <p>Twitter Bootstrap 3 is the latest in a series of HTML5 UI Frameworks for desktop browsers that has added support for mobile as well. &nbsp;It is based on manipulating HTML5 elements by giving them specific CSS styles to compose an application. &nbsp;Customized style and JavaScript components may be added from the Twitter Bootstrap Website to extend the functionality. &nbsp;Twitter Bootstrap has a distinctive Web 2.0 look which is very crisp and modern, but its approach to mobile is an afterthought since it doesnt support JavaScript scrolling or animated transitions. &nbsp;It is a great option for an application that is meant for the desktop as well as a mobile platform, but that reusability makes it a poor choice for a mobile-only experience.</p>
         <h3>jQuery Mobile</h3>
         <p>The first and perhaps most commonly used HTML5 Mobile UI Framework is jQuery Mobile. &nbsp;Although jQuery UI was originally not created for the mobile space, jQuery Mobile has done an admirable job at providing a mobile user interface that reflects the same aesthetic and usability of its desktop product. &nbsp;The jQuery Mobile HTML5 Mobile UI Framework is very easy to use, and is programmed using HTML elements like the previous two frameworks. &nbsp;It has good documentation for a free product, and has a large installed base that can provide help and sample code for new projects.</p>
         <h3>Sencha Touch</h3>
         <p>Sencha Touch offers a unique take on HTML5 Mobile UI Frameworks because it requires that all UI code is written in JavaScript. &nbsp;That makes for a more precise experience, but it requires developers to have considerable JavaScript chops to get things done. &nbsp;It is a commercial venture, so it has some great documentation, tools, and support. &nbsp;However, it is free as long as it isnt part of a product that ships more than 5,000 units. &nbsp;Sencha Touch is able to create some great graphical charts, it supports the Yahoo Query Language, and it has themes to support every popular mobile platform.</p>
         <h3>Kendo UI</h3>
         <p>Kendo UI is a commercial venture based on jQuery, which helps support the outstanding documentation and tools that are available for developers who use the tool. &nbsp;For example, Kendo UIs theme builder can help developers cook up a custom color scheme for their UI quickly. &nbsp;Kendo UIs Mobile HTML5 library uses HTML with class tags to run widgets that provide a set block of functionality. &nbsp;It is able to change its UI based on the platform to emulate the native aesthetic of an iOS, Android, Blackberry, or Windows Phone 8 device depending on which platform it runs on. &nbsp;Kendo UI us used by the Hybrid App builder Icenium.</p>
         <h3>TopCoat</h3>
         <p>TopCoat is a free HTML5 Mobile UI framework build by Adobe. &nbsp;It is unique from the others in this list for several reasons. &nbsp;First, it relies on CSS libraries over JavaScript to provide everything developers need. &nbsp;That makes it easy to build using HTML5 elements only and gives the framework an edge in speed. &nbsp;Secondly, TopCoats look and feel sets it apart from the others. &nbsp;It is very much a stylish product, and if you are a fan of Adobes aesthetics you wont be disappointed by TopCoat. &nbsp;Finally, it is designed primarily for desktop browsers but includes a mobile stylesheet that features larger touch targets for controls.</p>

         <p>If all this technology sounds great, developers can try them out for themselves by following these links.</p>
