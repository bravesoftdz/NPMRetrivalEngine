            " itemprop="articleBody"> 
        <p class="warning">Cloud Endpoints Frameworks v1 has been deprecated and will be shut down on August 2, 2018. We recommend that you use the latest version of this feature, which is renamed to <a href="https://cloud.google.com/endpoints/docs/frameworks/java">Cloud Endpoints Frameworks for App Engine</a>. This new version supports App Engine standard environment, provides lower latency, and has better integration with App Engine. For more details, see <a href="https://cloud.google.com/endpoints/docs/frameworks/legacy/v1/java/migrating">Migrating to 2.0</a>.</p> 
        <p>Google Cloud Endpoints Frameworks consist of tools, libraries and capabilities that allow you to generate APIs and client libraries from an App Engine application, referred to as an <em>API backend</em>, to simplify client access to data from other applications. Endpoints Frameworks make it easier to create a web backend on App Engine for web clients and mobile clients such as Android or Apple's iOS.</p> 
        <p>For mobile developers, Endpoints Frameworks provide a simple way to develop a shared web backend and also provides critical infrastructures, such as OAuth 2.0 authentication, eliminating a great deal of work that would otherwise be needed. Furthermore, because the API backend is an App Engine app, the mobile developer can use all of the services and features available in App Engine, such as <a href="https://cloud.google.com/appengine/docs/standard/java/datastore">Datastore</a>, <a href="https://cloud.google.com/storage/docs/overview">Google Cloud Storage</a>, <a href="https://cloud.google.com/appengine/docs/standard/java/mail">Mail</a>, <a href="https://cloud.google.com/appengine/docs/standard/java/issue-requests">Url Fetch</a>, <a href="https://cloud.google.com/appengine/docs/standard/java/taskqueue">Task Queues</a>, and so forth. And finally, by using App Engine for the backend, developers are freed from system admin work, load balancing, scaling, and server maintenance.</p> 
        <p>It is possible to create mobile clients for App Engine backends without Endpoints Frameworks. However, using Endpoints Frameworks makes this process easier because it frees you from having to write wrappers to handle communication with App Engine. The client libraries generated by Endpoints allow you to simply make direct API calls.</p> 
        <aside class="special">
         <strong>Important:</strong>
         <span> Google Cloud Endpoints requires SSL. If you need to access your backend API in a system not supporting SSL, you'll need to either update the system to support SSL or use a proxy.</span>
        </aside> 
        <aside class="note">
         <strong>Note:</strong>
         <span> version 1 of the Google Cloud Endpoints Frameworks did not support custom domains but version 2 does.</span>
        </aside> 
        <h2 id="basic_endpoints_frameworks_architecture">Basic Endpoints Frameworks architecture</h2> 
        <p>Here's what using an Endpoints API allows you to do:</p> 
        <p><img alt="" src="https://cloud.google.com/endpoints/docs/images/endpoints.svg"></p> 
        <p>As shown above, the API backend is an App Engine app that performs business logic and other functions for Android and iOS clients, as well as JavaScript web clients. The functionality of the backend is made available to clients through Endpoints Frameworks, which exposes an API that clients can call.</p> 
        <aside class="note">
         <strong>Note:</strong>
         <span> You cannot call an API written with Google Cloud Endpoint Framework version 1 directly from a <a href="https://cloud.google.com/appengine/docs/standard/java/taskqueue/push/">push task queue</a> or a <a href="https://cloud.google.com/appengine/docs/standard/java/config/cron#calling_google_cloud_endpoints">cron job</a>.</span>
        </aside> 
        <h2 id="endpoints_libraries_tools_and_samples">Endpoints libraries, tools, and samples</h2> 
        <p>Google Cloud Endpoints Frameworks provide the following libraries and tools: </p>
        <ul> 
         <li>The <a href="https://cloud.google.com/endpoints/docs/frameworks/legacy/v1/java/javadoc">Endpoints</a> library in the SDK.</li> 
         <li>Maven artifacts for <a href="https://cloud.google.com/endpoints/docs/frameworks/legacy/v1/java/helloendpoints-java-maven">creating new backend API projects</a>, <a href="https://cloud.google.com/endpoints/docs/frameworks/legacy/v1/java/generate-client-libraries-android">generating client libraries</a> and <a href="https://cloud.google.com/endpoints/docs/frameworks/legacy/v1/java/generate-discovery-doc-ios">generating discovery docs</a>.</li> 
         <li>Alternatively to Maven, the <a href="https://cloud.google.com/endpoints/docs/frameworks/legacy/v1/java/endpoints_tool"><code>endpoints.sh</code></a> command line tool (for Linux), or <code>endpoints.cmd</code> (for Windows) that you can use to generate client libraries and discovery documents. </li> 
        </ul> 
        <p>Endpoints Frameworks provide the <a href="https://github.com/GoogleCloudPlatform/appengine-endpoints-tictactoe-java">Tic Tac Toe</a> sample that demonstrates a backend API. Companion client samples for that backend are available:</p> 
        <ul> 
         <li><a href="https://github.com/GoogleCloudPlatform/appengine-endpoints-tictactoe-android">Android</a></li> 
         <li><a href="https://github.com/GoogleCloudPlatform/appengine-endpoints-tictactoe-ios">iOS</a> </li> 
         <li><a href="https://github.com/GoogleCloudPlatform/appengine-endpoints-tictactoe-java">Java</a> </li> 
        </ul> 
        <p>For more samples, refer to the <a href="https://cloud.google.com/endpoints/docs/frameworks/legacy/v1/java/">Getting Started</a> tutorials.</p>  
        <h2 id="requirements">Requirements</h2> 
        <p>To use Endpoints Frameworks, be sure you're using the latest version of the <a href="https://developers.google.com/appengine/downloads#Google_App_Engine_SDK_for_Java">Google App Engine Java SDK</a>.</p> 
        <p>You'll also need to be familiar with development using <a href="https://cloud.google.com/appengine/docs/java">Google App Engine</a> and the Java runtime. And finally, you'll need to know how to develop the client of your choice, such as JavaScript web clients, or mobile clients of your choice, such as Android or iOS.</p> 
        <h2 id="development_process">Development process</h2> 
        <p>The general workflow for developing an app using Endpoints Frameworks is:</p> 
        <ol> 
         <li>Create your backend API project (using <a href="https://cloud.google.com/endpoints/docs/frameworks/legacy/v1/java/helloendpoints-java-maven">Maven</a> to do this is the easiest method), then write your API backend code.</li> 
         <li><a href="https://cloud.google.com/endpoints/docs/frameworks/legacy/v1/java/annotations">Annotate</a> your API backend code, so classes and client libraries can be generated from it.</li> 
         <li><a href="https://cloud.google.com/endpoints/docs/frameworks/legacy/v1/java/generate-client-libraries-android">Generate the client library</a> using Maven, or alternatively, the <code>endpoints.sh</code> command line tool.</li> 
         <li>Write your client app, using the client library when making calls to the API backend.</li> 
        </ol> 
        <h2 id="getting_started">Getting Started</h2> 
        <p>To get started, follow the tutorial for <a href="https://cloud.google.com/endpoints/docs/frameworks/legacy/v1/java/helloworld-java-maven">writing a backend API</a>. </p>
