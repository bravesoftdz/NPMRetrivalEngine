                    <span class="rendered_qtext">From my experience, I would be very careful with HTML Mobile Framework. They tend to be heavy and require lot of CSS/HTML elements. <br><br>My recommendation is to use jQuery and target webkit (iOS and Android) and try to optimize your DOM and UI component as much as possible. Then, you can degrade for other browsers. <br><br>If you are building a offline application, you can use a MVC. I built one (Brite <span class="qlink_container"><a href="https://github.com/BriteSnow/brite" rel="noopener nofollow" target="_blank" onclick="return Q.openUrl(this, 235829);" class="external_link">https://github.com/BriteSnow/brite</a></span>) which is a MVC/MVP framework on top of jQuery. <br><br>Anyway, the rule is that the less framework you use the better you are. So, pick carefully, and make sure they are not kitchen sink. <br><br>Often a custom component is much better than a big generic one that is hard to customize and take too much resources.<br><br>Frameworks to avoid IMO are: jQueryMobile, jQueryUI, SenchaTouch, Sencha...<br><br>Keep it simple and lightweight, and avoid any frameworks that requires double passes (meaning, add to the DOM, and travers the DOM to activate the components).<br><br>Also look at an HTML5 Architecture doc I wrote: <span class="qlink_container"><a href="http://britesnow.com/html5/html5-application-architecture" rel="noopener nofollow" target="_blank" onclick="return Q.openUrl(this, 235829);" class="external_link" data-qt-tooltip="britesnow.com">HTML5 Enterprise Application Architecture</a></span></span>

                    <div id="VxEwWk"></div>
                    <span class="rendered_qtext">It really depends on which way you want to go:<br><br>a. native App Store app or<br>b. a web app through Safari.<br><br>For the former, PhoneGap (<span class="qlink_container"><a href="http://www.phonegap.com/" rel="noopener" target="_blank" onclick="return Q.openUrl(this, 119387);" class="external_link">http://www.phonegap.com/</a></span>) is a very good library that lets you code the entire app in HTML, CSS, Javascript and wrap that into an objective C app that can be submitted (and approved) to the App Store.<br><br>For the latter, it might be useful to try out XUI (<span class="qlink_container"><a href="http://xuijs.com/" rel="noopener nofollow" target="_blank" onclick="return Q.openUrl(this, 119387);" class="external_link">http://xuijs.com/</a></span>) and jQTouch (<span class="qlink_container"><a href="http://www.jqtouch.com/" rel="noopener nofollow" target="_blank" onclick="return Q.openUrl(this, 119387);" class="external_link">http://www.jqtouch.com/</a></span>). Both are based on jQuery but jQTouch comes with the UI components (the look and feel of a native app) whereas XUI concentrates more on the Javascript. In my experience, XUI has been easier to get up and running very quickly (and has a small size footprint).<br><br>A word of advice, if you are thinking of the first option, seriously consider building it in native Objective C if you need to build an app any more complicated than a simple information viewer. The drawbacks of using APIs through shims (which is basically what PhoneGap is) and not getting access to the native UI controls, coupled with the performance issues results in a very overall low quality experience for the user.</span>
