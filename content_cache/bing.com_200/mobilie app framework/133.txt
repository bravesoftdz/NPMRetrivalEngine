        <div class="mf-section-0" id="mf-section-0">
         <div role="note" class="hatnote navigation-not-searchable">
          Not to be confused with 
          <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java (programming language)</a>, 
          <a href="/wiki/Java_(software_platform)" title="Java (software platform)">Java (software platform)</a>, or 
          <a href="/wiki/Javanese_script" title="Javanese script">Javanese script</a>.
         </div> 
         <p><b>JavaScript</b> (<span class="nowrap"><span class="IPA nopopups noexcerpt"><a href="/wiki/Help:IPA/English" title="Help:IPA/English">/<span style="border-bottom:1px dotted"><span title="/?/: primary stress follows">?</span><span title="/d?/: 'j' in 'jam'">d?</span><span title="/??/: 'a' in 'father'">??</span><span title="'v' in 'vie'">v</span><span title="/?/: 'a' in 'about'">?</span><span title="/?/: secondary stress follows">?</span><span title="'s' in 'sigh'">s</span><span title="'k' in 'kind'">k</span><span title="'r' in 'rye'">r</span><span title="/?/: 'i' in 'kit'">?</span><span title="'p' in 'pie'">p</span><span title="'t' in 'tie'">t</span></span>/</a></span></span><sup id="cite_ref-6" class="reference"><a href="#cite_note-6">[6]</a></sup>), often abbreviated as <b>JS</b>, is a <a href="/wiki/High-level_programming_language" title="High-level programming language">high-level</a>, <a href="/wiki/Dynamic_programming_language" title="Dynamic programming language">dynamic</a>, <a href="/wiki/Weak_typing" class="mw-redirect" title="Weak typing">weakly typed</a>, <a href="/wiki/Object-based_language" title="Object-based language">object-based</a>, <a href="/wiki/Multi-paradigm_programming_language" class="mw-redirect" title="Multi-paradigm programming language">multi-paradigm</a>, and <a href="/wiki/Interpreted_language" title="Interpreted language">interpreted</a> <a href="/wiki/Programming_language" title="Programming language">programming language</a>. Alongside <a href="/wiki/HTML" title="HTML">HTML</a> and <a href="/wiki/CSS" class="mw-redirect" title="CSS">CSS</a>, JavaScript is one of the three core technologies of <a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a> <a href="/wiki/Content_engineering" class="mw-redirect" title="Content engineering">content production</a>. It is used to make webpages interactive and provide online programs, including video games. The majority of <a href="/wiki/Website" title="Website">websites</a> employ it, and all modern <a href="/wiki/Web_browser" title="Web browser">web browsers</a> support it without the need for <a href="/wiki/Browser_extension" title="Browser extension">plug-ins</a> by means of a built-in <a href="/wiki/JavaScript_engine" title="JavaScript engine">JavaScript engine</a>. Each of the many JavaScript engines represent a different implementation of JavaScript, all based on the <a href="/wiki/ECMAScript" title="ECMAScript">ECMAScript</a> specification, with some engines not supporting the spectrum fully, and with many engines supporting additional features beyond ECMA.</p>

         <p>As a multi-paradigm language, JavaScript supports <a href="/wiki/Event-driven_programming" title="Event-driven programming">event-driven</a>, <a href="/wiki/Functional_programming" title="Functional programming">functional</a>, and <a href="/wiki/Imperative_programming" title="Imperative programming">imperative</a> (including <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented</a> and <a href="/wiki/Prototype-based_programming" title="Prototype-based programming">prototype-based</a>) <a href="/wiki/Programming_paradigm" title="Programming paradigm">programming styles</a>. It has an <a href="/wiki/Application_programming_interface" title="Application programming interface">API</a> for working with text, <a href="/wiki/Array_data_type" title="Array data type">arrays</a>, dates, <a href="/wiki/Regular_expression" title="Regular expression">regular expressions</a>, and basic manipulation of the <a href="/wiki/Document_Object_Model" title="Document Object Model">DOM</a>, but does not include any <a href="/wiki/Input/output" title="Input/output">I/O</a>, such as networking, storage, or graphics facilities, relying for these upon the host environment in which it is embedded.</p> 
         <p>Initially only implemented <a href="/wiki/Client-side" title="Client-side">client-side</a> in web browsers, JavaScript engines are now embedded in many other types of host software, including <a href="/wiki/Server-side" title="Server-side">server-side</a> in web servers and databases, and in non-web programs such as word processors and <a href="/wiki/Portable_Document_Format" title="Portable Document Format">PDF</a> software, and in runtime environments that make JavaScript available for writing mobile and desktop applications, including desktop widgets.</p> 
         <p>Although there are strong outward similarities between JavaScript and Java, including language name, <a href="/wiki/Syntax_(programming_languages)" title="Syntax (programming languages)">syntax</a>, and respective <a href="/wiki/Standard_library" title="Standard library">standard libraries</a>, the two languages are distinct and differ greatly in design; JavaScript was influenced by programming languages such as <a href="/wiki/Self_(programming_language)" title="Self (programming language)">Self</a> and <a href="/wiki/Scheme_(programming_language)" title="Scheme (programming language)">Scheme</a>.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup></p>

        <div class="mf-section-1 collapsible-block" id="mf-section-1"> 
         <h3 class="in-block"><span class="mw-headline" id="Beginnings_at_Netscape">Beginnings at Netscape</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=2" title="Edit section: Beginnings at Netscape" data-section="2" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p>In 1993, the <a href="/wiki/National_Center_for_Supercomputing_Applications" title="National Center for Supercomputing Applications">National Center for Supercomputing Applications</a> (NCSA), a unit of the <a href="/wiki/University_of_Illinois_at_Urbana%E2%80%93Champaign" title="University of Illinois at Urbana–Champaign">University of Illinois at Urbana-Champaign</a>, released <a href="/wiki/Mosaic_(web_browser)" title="Mosaic (web browser)">NCSA Mosaic</a>, the first popular graphical <a href="/wiki/Web_browser" title="Web browser">Web browser</a>, which played an important part in expanding the growth of the nascent <a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a>. In 1994, a company called <a href="/wiki/Netscape" title="Netscape">Mosaic Communications</a> was founded in <a href="/wiki/Mountain_View,_California" title="Mountain View, California">Mountain View, California</a> and employed many of the original NCSA Mosaic authors to create <a href="/wiki/Netscape_Navigator" title="Netscape Navigator">Mosaic Netscape</a>. However, it intentionally shared no code with NCSA Mosaic. The internal codename for the company's browser was Mozilla, which stood for "Mosaic killer", as the company's goal was to displace NCSA Mosaic as the world's number one web browser. The first version of the Web browser, Mosaic Netscape 0.9, was released in late 1994. Within four months it had already taken three-quarters of the browser market and became the main browser for the Internet in the 1990s. To avoid trademark ownership problems with the NCSA, the browser was subsequently renamed Netscape Navigator in the same year, and the company took the name Netscape Communications.</p> 
         <p>Netscape Communications realized that the Web needed to become more dynamic. <a href="/wiki/Marc_Andreessen" title="Marc Andreessen">Marc Andreessen</a>, the founder of the company believed that <a href="/wiki/HTML" title="HTML">HTML</a> needed a "glue language" that was easy to use by Web designers and part-time programmers to assemble components such as images and plugins, where the code could be written directly in the Web page markup. In 1995, the company recruited <a href="/wiki/Brendan_Eich" title="Brendan Eich">Brendan Eich</a> with the goal of embedding the <a href="/wiki/Scheme_(programming_language)" title="Scheme (programming language)">Scheme</a> programming language into its Netscape Navigator. Before he could get started, Netscape Communications collaborated with <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun Microsystems</a> to include in Netscape Navigator Sun's more static programming language <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, in fries to compete with <a href="/wiki/Microsoft_Outlook" title="Microsoft Outlook">Microsoft</a> for user adoption of Web technologies and platforms.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[8]</a></sup> Netscape Communications then decided that the scripting language they wanted to create would complement Java and should have a similar syntax, which excluded adopting other languages such as <a href="/wiki/Perl" title="Perl">Perl</a>, <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, <a href="/wiki/Tcl" title="Tcl">TCL</a>, or Scheme. To defend the idea of JavaScript against competing proposals, the company needed a prototype. Eich wrote one in 10 days, in May 1995.</p> 
         <p>Although it was developed under the name <b>Mocha</b>, the language was officially called <b>LiveScript</b> when it first shipped in beta releases of Netscape Navigator 2.0 in September 1995, but it was renamed <b>JavaScript</b><sup id="cite_ref-press_release_2-1" class="reference"><a href="#cite_note-press_release-2">[2]</a></sup> when it was deployed in the Netscape Navigator 2.0 beta 3 in December.<sup id="cite_ref-techvision_9-0" class="reference"><a href="#cite_note-techvision-9">[9]</a></sup> The final choice of name caused confusion, giving the impression that the language was a spin-off of the Java programming language, and the choice has been characterized as a marketing ploy by Netscape to give JavaScript the cachet of what was then the hot new Web programming language.</p> 
         <p>There is a common misconception that JavaScript was influenced by an earlier Web page scripting language developed by <a href="/wiki/Openwave" title="Openwave">Nombas</a> named C-- (not to be confused with the later <a href="/wiki/C--" title="C--">C--</a> created in 1997).<sup id="cite_ref-10" class="reference"><a href="#cite_note-10">[10]</a></sup><sup id="cite_ref-11" class="reference"><a href="#cite_note-11">[11]</a></sup> Brendan Eich, however, had never heard of C-- before he created LiveScript.<sup id="cite_ref-12" class="reference"><a href="#cite_note-12">[12]</a></sup> Nombas did pitch their embedded Web page scripting to Netscape, though Web page scripting was not a new concept, as shown by the <a href="/wiki/ViolaWWW" title="ViolaWWW">ViolaWWW</a> Web browser.<sup id="cite_ref-13" class="reference"><a href="#cite_note-13">[13]</a></sup> Nombas later switched to offering JavaScript instead of C-- in their ScriptEase product and was part of the TC39 group that standardized ECMAScript.<sup id="cite_ref-14" class="reference"><a href="#cite_note-14">[14]</a></sup></p> 
         <h3 class="in-block"><span class="mw-headline" id="Server-side_JavaScript">Server-side JavaScript</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=3" title="Edit section: Server-side JavaScript" data-section="3" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p>In December 1995, soon after releasing JavaScript for browsers, Netscape introduced an implementation of the language for <a href="/wiki/Server-side_scripting" title="Server-side scripting">server-side scripting</a> with <a href="/wiki/Netscape_Enterprise_Server" class="mw-redirect" title="Netscape Enterprise Server">Netscape Enterprise Server</a>.<sup id="cite_ref-15" class="reference"><a href="#cite_note-15">[15]</a></sup></p> 
         <p>Since the mid-2000s, additional <a href="/wiki/Server-side_JavaScript_implementations" title="Server-side JavaScript implementations">server-side JavaScript implementations</a> have been introduced, such as <a href="/wiki/Node.js" title="Node.js">Node.js</a> in 2009.<sup id="cite_ref-16" class="reference"><a href="#cite_note-16">[16]</a></sup></p> 
         <h3 class="in-block"><span class="mw-headline" id="Adoption_by_Microsoft">Adoption by Microsoft</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=4" title="Edit section: Adoption by Microsoft" data-section="4" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p>Microsoft script technologies including <a href="/wiki/VBScript" title="VBScript">VBScript</a> and <a href="/wiki/JScript" title="JScript">JScript</a> were released in 1996. JScript, a reverse-engineered implementation of Netscape's JavaScript, was part of <a href="/wiki/Internet_Explorer_3" title="Internet Explorer 3">Internet Explorer 3</a>. JScript was also available for server-side scripting in <a href="/wiki/Internet_Information_Server" class="mw-redirect" title="Internet Information Server">Internet Information Server</a>. Internet Explorer 3 also included Microsoft's first support for CSS and various extensions to HTML, but in each case the implementation was noticeably different to that found in Netscape Navigator at the time.<sup id="cite_ref-O.27Reilly-2001_17-0" class="reference"><a href="#cite_note-O.27Reilly-2001-17">[17]</a></sup><sup id="cite_ref-18" class="reference"><a href="#cite_note-18">[18]</a></sup> These differences made it difficult for designers and programmers to make a single website work well in both browsers, leading to the use of "best viewed in Netscape" and "best viewed in Internet Explorer" logos that characterized these early years of the <a href="/wiki/Browser_wars" title="Browser wars">browser wars</a>.<sup id="cite_ref-19" class="reference"><a href="#cite_note-19">[19]</a></sup> JavaScript began to acquire a reputation for being one of the roadblocks to a cross-platform and standards-driven Web. Some developers took on the difficult task of trying to make their sites work in both major browsers, but many could not afford the time.<sup id="cite_ref-O.27Reilly-2001_17-1" class="reference"><a href="#cite_note-O.27Reilly-2001-17">[17]</a></sup> With the release of <a href="/wiki/Internet_Explorer_4" title="Internet Explorer 4">Internet Explorer 4</a>, Microsoft introduced the concept of <a href="/wiki/Dynamic_HTML" title="Dynamic HTML">Dynamic HTML</a>, but the differences in language implementations and the different and proprietary <a href="/wiki/Document_Object_Model" title="Document Object Model">Document Object Models</a> remained and were obstacles to widespread take-up of JavaScript on the Web.<sup id="cite_ref-O.27Reilly-2001_17-2" class="reference"><a href="#cite_note-O.27Reilly-2001-17">[17]</a></sup></p> 
         <h3 class="in-block"><span class="mw-headline" id="Standardization">Standardization</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=5" title="Edit section: Standardization" data-section="5" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p>In November 1996, Netscape submitted JavaScript to <a href="/wiki/Ecma_International" title="Ecma International">Ecma International</a> to carve out a standard specification, which other browser vendors could then implement based on the work done at Netscape. This led to the official release of the language specification <a href="/wiki/ECMAScript" title="ECMAScript">ECMAScript</a> published in the first edition of the ECMA-262 standard in June 1997, with JavaScript being the most well known of the implementations. <a href="/wiki/ActionScript" title="ActionScript">ActionScript</a> and <a href="/wiki/JScript" title="JScript">JScript</a> are other well-known implementations of ECMAScript, with extensions.</p> 
         <p>The standards process continued in cycles, with the release of ECMAScript 2 in June 1998, which brings some modifications to conform to the ISO/IEC 16262 international standard. The release of ECMAScript 3 followed in December 1999, which is the baseline for modern day JavaScript. The original ECMAScript 4 work led by Waldemar Horwat (then at Netscape, now at <a href="/wiki/Google" title="Google">Google</a>) started in 2000 and at first, <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> seemed to participate and even implemented some of the proposals in their <a href="/wiki/JScript_.NET" title="JScript .NET">JScript .NET</a> language.</p> 
         <p>Over time it was clear though that Microsoft had no intention of cooperating or implementing proper JavaScript in Internet Explorer, even though they had no competing proposal and they had a partial (and diverged at this point) implementation on the <a href="/wiki/.NET" class="mw-redirect" title=".NET">.NET</a> server side. So by 2003, the original ECMAScript 4 work was mothballed.</p> 
         <p>The next major event was in 2005, with two major happenings in JavaScript's history. First, Brendan Eich and Mozilla rejoined Ecma International as a not-for-profit member and work started on <a href="/wiki/ECMAScript_for_XML" title="ECMAScript for XML">ECMAScript for XML</a> (E4X), the ECMA-357 standard, which came from ex-Microsoft employees at <a href="/wiki/BEA_Systems" title="BEA Systems">BEA Systems</a> (originally acquired as Crossgain). This led to working jointly with <a href="/wiki/Macromedia" title="Macromedia">Macromedia</a> (later acquired by <a href="/wiki/Adobe_Systems" title="Adobe Systems">Adobe Systems</a>), who were implementing E4X in ActionScript 3 (ActionScript 3 was a fork of original ECMAScript 4).</p> 
         <p>So, along with Macromedia, work restarted on ECMAScript 4 with the goal of standardizing what was in ActionScript 3. To this end, Adobe Systems released the ActionScript Virtual Machine 2, code named <a href="/wiki/Tamarin_(software)" title="Tamarin (software)">Tamarin</a>, as an <a href="/wiki/Open_source" class="mw-redirect" title="Open source">open source</a> project. But Tamarin and ActionScript 3 were too different from web JavaScript to converge, as was realized by the parties in 2007 and 2008.</p> 
         <p>Alas, there was still turmoil between the various players; <a href="/wiki/Douglas_Crockford" title="Douglas Crockford">Douglas Crockford</a>—then at <a href="/wiki/Yahoo!" title="Yahoo!">Yahoo!</a>—joined forces with Microsoft in 2007 to oppose ECMAScript 4, which led to the ECMAScript 3.1 effort. The development of ECMAScript 4 was never completed, but that work influenced subsequent versions.<sup id="cite_ref-20" class="reference"><a href="#cite_note-20">[20]</a></sup></p> 
         <p>While all of this was happening, the open source and developer communities set to work to revolutionize what could be done with JavaScript. This community effort was sparked in 2005 when <a href="/wiki/Jesse_James_Garrett" title="Jesse James Garrett">Jesse James Garrett</a> released a white paper in which he coined the term <a href="/wiki/Ajax_(programming)" title="Ajax (programming)">Ajax</a>, and described a set of technologies, of which JavaScript was the backbone, used to create web applications where data can be loaded in the background, avoiding the need for full page reloads and leading to more dynamic applications. This resulted in a renaissance period of JavaScript usage spearheaded by open source libraries and the communities that formed around them, with libraries such as <a href="/wiki/Prototype_JavaScript_Framework" title="Prototype JavaScript Framework">Prototype</a>, <a href="/wiki/JQuery" title="JQuery">jQuery</a>, <a href="/wiki/Dojo_Toolkit" title="Dojo Toolkit">Dojo Toolkit</a>, <a href="/wiki/MooTools" title="MooTools">MooTools</a>, and others being released.</p> 
         <p>In July 2008, the disparate parties on either side came together in Oslo. This led to the eventual agreement in early 2009 to rename ECMAScript 3.1 to ECMAScript 5 and drive the language forward using an agenda that is known as Harmony. ECMAScript 5 was finally released in December 2009.</p> 
         <p>In June 2011, ECMAScript 5.1 was released to fully align with the third edition of the ISO/IEC 16262 international standard. ECMAScript 2015 was released in June 2015. ECMAScript 2016 was released in June 2016. The current version is ECMAScript 2017, released in June 2017.<sup id="cite_ref-21" class="reference"><a href="#cite_note-21">[21]</a></sup></p> 
         <h3 class="in-block"><span class="mw-headline" id="Later_developments">Later developments</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=6" title="Edit section: Later developments" data-section="6" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p>JavaScript has become one of the most popular programming languages on the Web. Initially, however, many professional programmers denigrated the language because, among other reasons, its target audience consisted of Web authors and other such "amateurs".<sup id="cite_ref-22" class="reference"><a href="#cite_note-22">[22]</a></sup> The advent of <a href="/wiki/Ajax_(programming)" title="Ajax (programming)">Ajax</a> returned JavaScript to the spotlight and brought more professional programming attention. The result was a proliferation of comprehensive <a href="/wiki/List_of_JavaScript_libraries" title="List of JavaScript libraries">frameworks and libraries</a>, improved JavaScript programming practices, and increased usage of JavaScript outside Web browsers, as seen by the proliferation of <a href="/wiki/Server-side_JavaScript" class="mw-redirect" title="Server-side JavaScript">server-side JavaScript</a> platforms.</p> 
         <p>In January 2009, the <a href="/wiki/CommonJS" title="CommonJS">CommonJS</a> project was founded with the goal of specifying a common standard library mainly for JavaScript development outside the browser.<sup id="cite_ref-23" class="reference"><a href="#cite_note-23">[23]</a></sup></p> 
         <p>With the rise of <a href="/wiki/Single-page_application" title="Single-page application">single-page applications</a> and JavaScript-heavy sites, it is increasingly being used as a compile target for <a href="/wiki/Source-to-source_compiler" title="Source-to-source compiler">source-to-source compilers</a> from both <a href="/wiki/Dynamic_typing" class="mw-redirect" title="Dynamic typing">dynamic languages</a> and <a href="/wiki/Static_typing" class="mw-redirect" title="Static typing">static languages</a>.</p>

Scripts written in Vanilla JS are plain JavaScript code.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources.         <h2 class="in-block section-heading" onclick="javascript:mfTempOpenSection(4)">
         <div class="mw-ui-icon mw-ui-icon-element indicator"></div><span class="mw-headline" id="Features">Features</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=9" title="Edit section: Features" data-section="9" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h2>
        <div class="mf-section-4 collapsible-block" id="mf-section-4"> 
         <p>The following features are common to all conforming ECMAScript implementations, unless explicitly specified otherwise.</p> 
         <h3 class="in-block"><span class="mw-headline" id="Universal_support">Universal support</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=10" title="Edit section: Universal support" data-section="10" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p>All modern Web browsers support JavaScript with built-in interpreters.</p> 
         <h3 class="in-block"><span class="mw-headline" id="Imperative_and_structured">Imperative and structured</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=11" title="Edit section: Imperative and structured" data-section="11" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p>JavaScript supports much of the <a href="/wiki/Structured_programming" title="Structured programming">structured programming</a> syntax from <a href="/wiki/C_(computer_language)" class="mw-redirect" title="C (computer language)">C</a> (e.g., <tt>if</tt> statements, <tt>while</tt> loops, <tt>switch</tt> statements, <tt>do while</tt> loops, etc.). One partial exception is <a href="/wiki/Scope_(computer_science)" title="Scope (computer science)">scoping</a>: JavaScript originally had only <a href="/wiki/Function_scoping" class="mw-redirect" title="Function scoping">function scoping</a> with <tt>var</tt>. ECMAScript 2015 added keywords <tt>let</tt> and <tt>const</tt> for block scoping, meaning JavaScript now has both function and block scoping. Like C, JavaScript makes a distinction between <a href="/wiki/Expression_(computer_science)" title="Expression (computer science)">expressions</a> and <a href="/wiki/Statement_(computer_science)" title="Statement (computer science)">statements</a>. One syntactic difference from C is <a href="/wiki/Defensive_semicolon" class="mw-redirect" title="Defensive semicolon">automatic semicolon insertion</a>, which allows the semicolons that would normally terminate statements to be omitted.<sup id="cite_ref-FOOTNOTEFlanagan200616_26-0" class="reference"><a href="#cite_note-FOOTNOTEFlanagan200616-26">[26]</a></sup></p> 
         <h3 class="in-block"><span class="mw-headline" id="Dynamic">Dynamic</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=12" title="Edit section: Dynamic" data-section="12" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <dl>
          <dt>
           Typing
          </dt> 
          <dd>
           As with most 
           <a href="/wiki/Scripting_language" title="Scripting language">scripting languages</a>, JavaScript is 
           <a href="/wiki/Dynamic_typing" class="mw-redirect" title="Dynamic typing">dynamically typed</a>; a 
           <a href="/wiki/Type_system" title="Type system">type</a> is associated with each 
           <a href="/wiki/Value_(computer_science)" title="Value (computer science)">value</a>, rather than just with each expression. For example, a 
           <a href="/wiki/Variable_(programming)" class="mw-redirect" title="Variable (programming)">variable</a> that is at one time bound to a number may later be re-bound to a 
           <a href="/wiki/String_(computer_science)" title="String (computer science)">string</a>.
           <sup id="cite_ref-27" class="reference"><a href="#cite_note-27">[27]</a></sup> JavaScript supports various ways to test the type of an object, including 
           <a href="/wiki/Duck_typing" title="Duck typing">duck typing</a>.
           <sup id="cite_ref-FOOTNOTEFlanagan2006176.E2.80.93178_28-0" class="reference"><a href="#cite_note-FOOTNOTEFlanagan2006176.E2.80.93178-28">[28]</a></sup>
          </dd> 
          <dt>
           Run-time evaluation
          </dt> 
          <dd>
           JavaScript includes an 
           <tt><a href="/wiki/Eval" title="Eval">eval</a></tt> function that can execute statements provided as strings at run-time.
          </dd> 
         </dl>
         <h3 class="in-block"><span id="Prototype-based_(object-oriented)"></span><span class="mw-headline" id="Prototype-based_.28object-oriented.29">Prototype-based (object-oriented)</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=13" title="Edit section: Prototype-based (object-oriented)" data-section="13" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p>JavaScript is almost entirely <a href="/wiki/Object-based" class="mw-redirect" title="Object-based">object-based</a>. In JavaScript, an <a href="/wiki/Object_(computer_science)" title="Object (computer science)">object</a> is an <a href="/wiki/Associative_array" title="Associative array">associative array</a>, augmented with a prototype (see below); each string key provides the name for an object property, and there are two syntactical ways to specify such a name: dot notation (<tt>obj.x&nbsp;=&nbsp;10</tt>) and bracket notation (<tt>obj['x']&nbsp;=&nbsp;10</tt>). A property may be added, rebound, or deleted at run-time. Most properties of an object (and any property that belongs to an object's prototype inheritance chain) can be enumerated using a <tt>for...in</tt> loop.</p> 
         <p>JavaScript has a small number of built-in objects, including <span style="font-family: monospace, monospace;">Function</span> and <span style="font-family: monospace, monospace;">Date</span>.</p> 
         <dl>
          <dt>
           Prototypes
          </dt> 
          <dd>
           JavaScript uses 
           <a href="/wiki/Prototype-based_programming" title="Prototype-based programming">prototypes</a> where many other object-oriented languages use 
           <a href="/wiki/Class_(computer_science)" class="mw-redirect" title="Class (computer science)">classes</a> for 
           <a href="/wiki/Inheritance_(computer_science)" class="mw-redirect" title="Inheritance (computer science)">inheritance</a>.
           <sup id="cite_ref-29" class="reference"><a href="#cite_note-29">[29]</a></sup> It is possible to simulate many class-based features with prototypes in JavaScript.
           <sup id="cite_ref-30" class="reference"><a href="#cite_note-30">[30]</a></sup>
          </dd> 
          <dt>
           Functions as object constructors
          </dt> 
          <dd>
           Functions double as object constructors, along with their typical role. Prefixing a function call with 
           <i>new</i> will create an instance of a prototype, inheriting properties and methods from the constructor (including properties from the 
           <tt>Object</tt> prototype).
           <sup id="cite_ref-31" class="reference"><a href="#cite_note-31">[31]</a></sup> ECMAScript 5 offers the 
           <tt>Object.create</tt> method, allowing explicit creation of an instance without automatically inheriting from the 
           <tt>Object</tt> prototype (older environments can assign the prototype to 
           <tt>null</tt>).
           <sup id="cite_ref-32" class="reference"><a href="#cite_note-32">[32]</a></sup> The constructor's 
           <tt>prototype</tt> property determines the object used for the new object's internal prototype. New methods can be added by modifying the prototype of the function used as a constructor. JavaScript's built-in constructors, such as 
           <tt>Array</tt> or 
           <tt>Object</tt>, also have prototypes that can be modified. While it is possible to modify the 
           <tt>Object</tt> prototype, it is generally considered bad practice because most objects in JavaScript will inherit methods and properties from the 
           <tt>Object</tt> prototype, and they may not expect the prototype to be modified.
           <sup id="cite_ref-33" class="reference"><a href="#cite_note-33">[33]</a></sup>
          </dd> 
          <dt>
           Functions as methods
          </dt> 
          <dd>
           Unlike many object-oriented languages, there is no distinction between a function definition and a 
           <a href="/wiki/Method_(computer_science)" class="mw-redirect" title="Method (computer science)">method</a> definition. Rather, the distinction occurs during function calling; when a function is called as a method of an object, the function's local 
           <i>this</i> keyword is bound to that object for that invocation.
          </dd> 
         </dl>
         <h3 class="in-block"><span class="mw-headline" id="Functional">Functional</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=14" title="Edit section: Functional" data-section="14" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p>A <a href="/wiki/Subroutine" title="Subroutine">function</a> is <a href="/wiki/First-class_function" title="First-class function">first-class</a>; a function is considered to be an object. As such, a function may have properties and methods, such as <samp>.call()</samp> and <tt>.bind()</tt>.<sup id="cite_ref-34" class="reference"><a href="#cite_note-34">[34]</a></sup> A <i>nested</i> function is a function defined within another function. It is created each time the outer function is invoked. In addition, each nested function forms a <a href="/wiki/Closure_(computer_programming)" title="Closure (computer programming)">lexical closure</a>: The <a href="/wiki/Scope_(programming)#Lexical_scoping_vs._dynamic_scoping" class="mw-redirect" title="Scope (programming)">lexical scope</a> of the outer function (including any constant, local variable, or argument value) becomes part of the internal state of each inner function object, even after execution of the outer function concludes.<sup id="cite_ref-FOOTNOTEFlanagan2006141_35-0" class="reference"><a href="#cite_note-FOOTNOTEFlanagan2006141-35">[35]</a></sup> JavaScript also supports <a href="/wiki/Anonymous_function" title="Anonymous function">anonymous functions</a>.</p> 
         <h3 class="in-block"><span class="mw-headline" id="Delegative">Delegative</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=15" title="Edit section: Delegative" data-section="15" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p>JavaScript supports implicit and explicit <a href="/wiki/Delegation_(object-oriented_programming)" title="Delegation (object-oriented programming)">delegation</a>.</p> 
         <dl>
          <dt>
           Functions as roles (Traits and Mixins)
          </dt> 
          <dd>
           JavaScript natively supports various function-based implementations of 
           <a href="/wiki/Role-oriented_programming" title="Role-oriented programming">Role</a>
           <sup id="cite_ref-36" class="reference"><a href="#cite_note-36">[36]</a></sup> patterns like 
           <a href="/wiki/Traits_(computer_science)" class="mw-redirect" title="Traits (computer science)">Traits</a>
           <sup id="cite_ref-37" class="reference"><a href="#cite_note-37">[37]</a></sup>
           <sup id="cite_ref-38" class="reference"><a href="#cite_note-38">[38]</a></sup> and 
           <a href="/wiki/Mixin" title="Mixin">Mixins</a>.
           <sup id="cite_ref-39" class="reference"><a href="#cite_note-39">[39]</a></sup> Such a function defines additional behavior by at least one method bound to the 
           <tt>this</tt> keyword within its 
           <tt>function</tt> body. A Role then has to be delegated explicitly via 
           <tt>call</tt> or 
           <tt>apply</tt> to objects that need to feature additional behavior that is not shared via the prototype chain.
          </dd> 
          <dt>
           Object composition and inheritance
          </dt> 
          <dd>
           Whereas explicit function-based delegation does cover 
           <a href="/wiki/Object_composition" title="Object composition">composition</a> in JavaScript, implicit delegation already happens every time the prototype chain is walked in order to, e.g., find a method that might be related to but is not directly owned by an object. Once the method is found it gets called within this object's context. Thus 
           <a href="/wiki/Inheritance_(computer_science)" class="mw-redirect" title="Inheritance (computer science)">inheritance</a> in JavaScript is covered by a delegation automatism that is bound to the prototype property of constructor functions.
          </dd> 
         </dl>
         <h3 class="in-block"><span class="mw-headline" id="Miscellaneous">Miscellaneous</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=16" title="Edit section: Miscellaneous" data-section="16" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <dl>
          <dt>
           Run-time environment
          </dt> 
          <dd>
           JavaScript typically relies on a run-time environment (e.g., a 
           <a href="/wiki/Web_browser" title="Web browser">Web browser</a>) to provide objects and methods by which scripts can interact with the environment (e.g., a webpage DOM). It also relies on the run-time environment to provide the ability to include/import scripts (e.g., 
           <a href="/wiki/HTML" title="HTML">HTML</a> 
           <tt>&lt;script&gt;</tt> elements). This is not a language feature per se, but it is common in most JavaScript implementations.
          </dd> 
         </dl>
         <dl>
          <dd>
           JavaScript processes 
           <a href="/wiki/Message_(computer_science)" class="mw-redirect" title="Message (computer science)">messages</a> from a 
           <a href="/wiki/Queue_(abstract_data_type)" title="Queue (abstract data type)">queue</a> one at a time. Upon loading a new message, JavaScript calls a 
           <a href="/wiki/Subroutine" title="Subroutine">function</a> associated with that message, which creates a 
           <a href="/wiki/Call_stack" title="Call stack">call stack</a> frame (the function's 
           <a href="/wiki/Parameter_(computer_programming)" title="Parameter (computer programming)">arguments</a> and 
           <a href="/wiki/Local_variable" title="Local variable">local variables</a>). The call stack shrinks and grows based on the function's needs. Upon function completion, when the stack is empty, JavaScript proceeds to the next message in the queue. This is called the 
           <a href="/wiki/Event_loop" title="Event loop">event loop</a>, described as "run to completion" because each message is fully processed before the next message is considered. However, the language's 
           <a href="/wiki/Concurrency_(computer_science)" title="Concurrency (computer science)">concurrency model</a> describes the event loop as 
           <a href="/wiki/Asynchronous_I/O" title="Asynchronous I/O">non-blocking</a>: program 
           <a href="/wiki/Input/output" title="Input/output">input/output</a> is performed using 
           <a href="/wiki/Event_(computing)" title="Event (computing)">events</a> and 
           <a href="/wiki/Callback_(computer_programming)" title="Callback (computer programming)">callback functions</a>. This means, for instance, that JavaScript can process a mouse click while waiting for a database query to return information.
           <sup id="cite_ref-40" class="reference"><a href="#cite_note-40">[40]</a></sup>
          </dd> 
         </dl>
         <dl>
          <dt>
           Variadic functions
          </dt> 
          <dd>
           An indefinite number of parameters can be passed to a function. The function can access them through 
           <a href="/wiki/Formal_parameter" class="mw-redirect" title="Formal parameter">formal parameters</a> and also through the local 
           <span style="font-family: monospace, monospace;">arguments</span> object. 
           <a href="/wiki/Variadic_functions" class="mw-redirect" title="Variadic functions">Variadic functions</a> can also be created by using the 
           <tt><a rel="nofollow" class="external text" href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Function/bind">bind</a></tt> method.
          </dd> 
         </dl>
         <dl>
          <dt>
           Array and object literals
          </dt> 
          <dd>
           Like many scripting languages, arrays and objects (
           <a href="/wiki/Associative_arrays" class="mw-redirect" title="Associative arrays">associative arrays</a> in other languages) can each be created with a succinct shortcut syntax. In fact, these 
           <a href="/wiki/Object_literal" class="mw-redirect" title="Object literal">literals</a> form the basis of the 
           <a href="/wiki/JSON" title="JSON">JSON</a> data format.
          </dd> 
         </dl>
         <dl>
          <dt>
           Regular expressions
          </dt> 
          <dd>
           JavaScript also supports 
           <a href="/wiki/Regular_expression" title="Regular expression">regular expressions</a> in a manner similar to 
           <a href="/wiki/Perl" title="Perl">Perl</a>, which provide a concise and powerful syntax for text manipulation that is more sophisticated than the built-in string functions.
           <sup id="cite_ref-41" class="reference"><a href="#cite_note-41">[41]</a></sup>
          </dd> 
         </dl>
         <h3 class="in-block"><span class="mw-headline" id="Vendor-specific_extensions">Vendor-specific extensions</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=17" title="Edit section: Vendor-specific extensions" data-section="17" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p>JavaScript is officially managed by <a href="/wiki/Mozilla_Foundation" title="Mozilla Foundation">Mozilla Foundation</a>, and new language features are added periodically. However, only some <a href="/wiki/JavaScript_engine" title="JavaScript engine">JavaScript engines</a> support these new features:</p> 
         <ul>
          <li>property getter and setter functions (supported by WebKit, Gecko, Opera,<sup id="cite_ref-42" class="reference"><a href="#cite_note-42">[42]</a></sup> ActionScript, and Rhino)<sup id="cite_ref-43" class="reference"><a href="#cite_note-43">[43]</a></sup></li> 
          <li>conditional <tt>catch</tt> clauses</li> 
          <li>iterator protocol (adopted from <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>)</li> 
          <li>shallow <a href="/wiki/Generator_(computer_programming)" title="Generator (computer programming)">generators</a>-<a href="/wiki/Coroutine" title="Coroutine">coroutines</a> (adopted from Python)</li> 
          <li><a href="/wiki/List_comprehension" title="List comprehension">array comprehensions</a> and generator expressions (adopted from Python)</li> 
          <li>proper block scope via the <tt>let</tt> keyword</li> 
          <li>array and object destructuring (limited form of <a href="/wiki/Pattern_matching" title="Pattern matching">pattern matching</a>)</li> 
          <li>concise function expressions (<tt>function(args) expr</tt>)</li> 
          <li><a href="/wiki/ECMAScript_for_XML" title="ECMAScript for XML">ECMAScript for XML</a> (E4X), an extension that adds native XML support to ECMAScript (unsupported in Firefox since version 21<sup id="cite_ref-44" class="reference"><a href="#cite_note-44">[44]</a></sup>)</li> 
         </ul>
        </div>

         <h3 class="in-block"><span class="mw-headline" id="Simple_examples">Simple examples</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=19" title="Edit section: Simple examples" data-section="19" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p><a href="/wiki/Variable_(computer_science)" title="Variable (computer science)">Variables</a> in JavaScript can be defined using the <tt>var</tt> keyword:<sup id="cite_ref-45" class="reference"><a href="#cite_note-45">[45]</a></sup></p> 
         <div class="mw-highlight mw-content-ltr" dir="ltr"> 
          <pre>
<span class="kd">var</span> <span class="nx">x</span><span class="p">;</span> <span class="c1">// defines the variable x and assigns to it the special value "undefined" (not to be confused with an undefined value)</span>
<span class="kd">var</span> <span class="nx">y</span> <span class="o">=</span> <span class="mi">2</span><span class="p">;</span> <span class="c1">// defines the variable y and assigns to it the value 2</span>
<span class="kd">var</span> <span class="nx">z</span> <span class="o">=</span> <span class="s2">"Hello, World!"</span><span class="p">;</span> <span class="c1">// defines the variable z and assigns to it a string entitled "Hello, World!"</span>
</pre>
         </div> 
         <p>Note the <a href="/wiki/Comment_(computer_programming)" title="Comment (computer programming)">comments</a> in the example above, both of which were preceded with two <a href="/wiki/Slash_(punctuation)" title="Slash (punctuation)">forward slashes</a>.</p> 
         <p>There is no built-in <a href="/wiki/Input/output" title="Input/output">I/O</a> functionality in JavaScript; the run-time environment provides that. The ECMAScript specification in edition 5.1 mentions:<sup id="cite_ref-46" class="reference"><a href="#cite_note-46">[46]</a></sup></p> 
         <blockquote> 
          <p>… indeed, there are no provisions in this specification for input of external data or output of computed results.</p> 
         </blockquote> 
         <p>However, most runtime environments have a <tt>console</tt> object<sup id="cite_ref-47" class="reference"><a href="#cite_note-47">[47]</a></sup> that can be used to print output. Here is a minimalist <a href="/wiki/Hello_World_program" class="mw-redirect" title="Hello World program">Hello World program</a> in <b>JavaScript</b>:</p> 
         <div class="mw-highlight mw-content-ltr" dir="ltr"> 
          <pre>
<span class="nx">console</span><span class="p">.</span><span class="nx">log</span><span class="p">(</span><span class="s2">"Hello World!"</span><span class="p">);</span>
</pre>
         </div> 
         <p>A simple <a href="/wiki/Recursion_(computer_science)" title="Recursion (computer science)">recursive</a> function:</p> 
         <div class="mw-highlight mw-content-ltr" dir="ltr"> 
          <pre>
<span class="kd">function</span> <span class="nx">factorial</span><span class="p">(</span><span class="nx">n</span><span class="p">)</span> <span class="p">{</span>
    <span class="k">if</span> <span class="p">(</span><span class="nx">n</span> <span class="o">===</span> <span class="mi">0</span> <span class="o">||</span> <span class="nx">n</span> <span class="o">===</span> <span class="mi">1</span><span class="p">)</span> <span class="p">{</span>
        <span class="k">return</span> <span class="mi">1</span><span class="p">;</span>  <span class="c1">// 0! = 1! = 1</span>
    <span class="p">}</span>
    <span class="k">return</span> <span class="nx">n</span> <span class="o">*</span> <span class="nx">factorial</span><span class="p">(</span><span class="nx">n</span> <span class="o">-</span> <span class="mi">1</span><span class="p">);</span>
<span class="p">}</span>

<span class="nx">factorial</span><span class="p">(</span><span class="mi">3</span><span class="p">);</span> <span class="c1">// returns 6</span>
</pre>
         </div> 
         <p>An <a href="/wiki/Anonymous_function" title="Anonymous function">anonymous function</a> (or lambda):</p> 
         <div class="mw-highlight mw-content-ltr" dir="ltr"> 
          <pre>
<span class="kd">function</span> <span class="nx">counter</span><span class="p">()</span> <span class="p">{</span>
    <span class="kd">var</span> <span class="nx">count</span> <span class="o">=</span> <span class="mi">0</span><span class="p">;</span>
    <span class="k">return</span> <span class="kd">function</span><span class="p">()</span> <span class="p">{</span>
        <span class="k">return</span> <span class="o">++</span><span class="nx">count</span><span class="p">;</span>
    <span class="p">};</span>
<span class="p">}</span>

<span class="kd">var</span> <span class="nx">closure</span> <span class="o">=</span> <span class="nx">counter</span><span class="p">();</span>
<span class="nx">closure</span><span class="p">();</span> <span class="c1">// returns 1</span>
<span class="nx">closure</span><span class="p">();</span> <span class="c1">// returns 2</span>
<span class="nx">closure</span><span class="p">();</span> <span class="c1">// returns 3</span>
</pre>
         </div> 
         <p>In JavaScript, <a href="/wiki/Object_(computer_science)" title="Object (computer science)">objects</a> are created in the same way as functions, this is known as a <a href="/wiki/Function_object" title="Function object">function object</a>.</p> 
         <p>Object example:</p> 
         <div class="mw-highlight mw-content-ltr" dir="ltr"> 
          <pre>
<span class="kd">function</span> <span class="nx">Ball</span><span class="p">(</span><span class="nx">r</span><span class="p">)</span> <span class="p">{</span>
    <span class="k">this</span><span class="p">.</span><span class="nx">radius</span> <span class="o">=</span> <span class="nx">r</span><span class="p">;</span> <span class="c1">//the radius variable is local to the ball object</span>
    <span class="k">this</span><span class="p">.</span><span class="nx">area</span> <span class="o">=</span> <span class="nx">pi</span><span class="o">*</span><span class="nx">r</span><span class="o">**</span><span class="mi">2</span><span class="p">;</span>
    <span class="k">this</span><span class="p">.</span><span class="nx">show</span> <span class="o">=</span> <span class="kd">function</span><span class="p">(){</span> <span class="c1">//objects can contain functions</span>
        <span class="nx">drawCircle</span><span class="p">(</span><span class="nx">r</span><span class="p">);</span> <span class="c1">//references a circle drawing function</span>
    <span class="p">}</span>
<span class="p">}</span>
<span class="nx">myBall</span> <span class="o">=</span> <span class="k">new</span> <span class="nx">Ball</span><span class="p">(</span><span class="mi">5</span><span class="p">);</span> <span class="c1">//creates a new instance of the ball object with radius 5</span>
<span class="nx">myBall</span><span class="p">.</span><span class="nx">show</span><span class="p">();</span> <span class="c1">//this instance of the ball object has the show function performed on it</span>
</pre>
         </div> 
         <p>This example shows that, in JavaScript, <a href="/wiki/Closure_(computer_programming)" title="Closure (computer programming)">function closures</a> capture their non-local variables <i>by reference</i>.</p> 
         <p><a href="/wiki/Variadic_function" title="Variadic function">Variadic function</a> demonstration (<tt>arguments</tt> is a special <a href="/wiki/Variable_(programming)" class="mw-redirect" title="Variable (programming)">variable</a>):<sup id="cite_ref-48" class="reference"><a href="#cite_note-48">[48]</a></sup></p> 
         <div class="mw-highlight mw-content-ltr" dir="ltr"> 
          <pre>
<span class="kd">function</span> <span class="nx">sum</span><span class="p">()</span> <span class="p">{</span>
    <span class="kd">var</span> <span class="nx">x</span> <span class="o">=</span> <span class="mi">0</span><span class="p">;</span>
    <span class="k">for</span> <span class="p">(</span><span class="kd">var</span> <span class="nx">i</span> <span class="o">=</span> <span class="mi">0</span><span class="p">;</span> <span class="nx">i</span> <span class="o">&lt;</span> <span class="nx">arguments</span><span class="p">.</span><span class="nx">length</span><span class="p">;</span> <span class="o">++</span><span class="nx">i</span><span class="p">)</span> <span class="p">{</span>
        <span class="nx">x</span> <span class="o">+=</span> <span class="nx">arguments</span><span class="p">[</span><span class="nx">i</span><span class="p">];</span>
    <span class="p">}</span>
    <span class="k">return</span> <span class="nx">x</span><span class="p">;</span>
<span class="p">}</span>
<span class="nx">sum</span><span class="p">(</span><span class="mi">1</span><span class="p">,</span> <span class="mi">2</span><span class="p">);</span> <span class="c1">// returns 3</span>
<span class="nx">sum</span><span class="p">(</span><span class="mi">1</span><span class="p">,</span> <span class="mi">2</span><span class="p">,</span> <span class="mi">3</span><span class="p">);</span> <span class="c1">// returns 6</span>
</pre>
         </div> 
         <p><a href="/wiki/Immediately-invoked_function_expression" title="Immediately-invoked function expression">Immediately-invoked function expressions</a> are often used to create modules, as before ECMAScript 2015 there was not built-in construct in the language. Modules allow gathering properties and methods in a namespace and making some of them private:</p> 
         <div class="mw-highlight mw-content-ltr" dir="ltr"> 
          <pre>
<span class="kd">var</span> <span class="nx">counter</span> <span class="o">=</span> <span class="p">(</span><span class="kd">function</span> <span class="p">()</span> <span class="p">{</span>
    <span class="kd">var</span> <span class="nx">i</span> <span class="o">=</span> <span class="mi">0</span><span class="p">;</span> <span class="c1">// private property</span>

    <span class="k">return</span> <span class="p">{</span>   <span class="c1">// public methods</span>
        <span class="nx">get</span><span class="o">:</span> <span class="kd">function</span> <span class="p">()</span> <span class="p">{</span>
            <span class="nx">alert</span><span class="p">(</span><span class="nx">i</span><span class="p">);</span>
        <span class="p">},</span>
        <span class="nx">set</span><span class="o">:</span> <span class="kd">function</span> <span class="p">(</span><span class="nx">value</span><span class="p">)</span> <span class="p">{</span>
            <span class="nx">i</span> <span class="o">=</span> <span class="nx">value</span><span class="p">;</span>
        <span class="p">},</span>
        <span class="nx">increment</span><span class="o">:</span> <span class="kd">function</span> <span class="p">()</span> <span class="p">{</span>
            <span class="nx">alert</span><span class="p">(</span><span class="o">++</span><span class="nx">i</span><span class="p">);</span>
        <span class="p">}</span>
    <span class="p">};</span>
<span class="p">})();</span> <span class="c1">// module</span>

<span class="nx">counter</span><span class="p">.</span><span class="nx">get</span><span class="p">();</span>       <span class="c1">// shows 0</span>
<span class="nx">counter</span><span class="p">.</span><span class="nx">set</span><span class="p">(</span><span class="mi">6</span><span class="p">);</span>
<span class="nx">counter</span><span class="p">.</span><span class="nx">increment</span><span class="p">();</span> <span class="c1">// shows 7</span>
<span class="nx">counter</span><span class="p">.</span><span class="nx">increment</span><span class="p">();</span> <span class="c1">// shows 8</span>
</pre>
         </div> 
         <h3 class="in-block"><span class="mw-headline" id="More_advanced_example">More advanced example</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=20" title="Edit section: More advanced example" data-section="20" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p>This sample code displays various JavaScript features.</p> 
         <div class="mw-highlight mw-content-ltr" dir="ltr"> 
          <pre>
<span class="cm">/* Finds the lowest common multiple (LCM) of two numbers */</span>
<span class="kd">function</span> <span class="nx">LCMCalculator</span><span class="p">(</span><span class="nx">x</span><span class="p">,</span> <span class="nx">y</span><span class="p">)</span> <span class="p">{</span> <span class="c1">// constructor function</span>
  <span class="kd">var</span> <span class="nx">checkInt</span> <span class="o">=</span> <span class="kd">function</span><span class="p">(</span><span class="nx">x</span><span class="p">)</span> <span class="p">{</span> <span class="c1">// inner function</span>
    <span class="k">if</span> <span class="p">(</span><span class="nx">x</span> <span class="o">%</span> <span class="mi">1</span> <span class="o">!==</span> <span class="mi">0</span><span class="p">)</span> <span class="p">{</span>
      <span class="k">throw</span> <span class="k">new</span> <span class="nx">TypeError</span><span class="p">(</span><span class="nx">x</span> <span class="o">+</span> <span class="s2">"is not an integer"</span><span class="p">);</span> <span class="c1">// var a =  mouseX</span>
    <span class="p">};</span>

    <span class="k">return</span> <span class="nx">x</span>
  <span class="p">}</span>

         <div class="mw-ui-icon mw-ui-icon-element indicator"></div><span class="mw-headline" id="Use_in_Web_pages">Use in Web pages</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=21" title="Edit section: Use in Web pages" data-section="21" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h2>
        <div class="mf-section-6 collapsible-block" id="mf-section-6"> 
         <div role="note" class="hatnote navigation-not-searchable">
          See also: 
          <a href="/wiki/Dynamic_HTML" title="Dynamic HTML">Dynamic HTML</a> and 
          <a href="/wiki/Ajax_(programming)" title="Ajax (programming)">Ajax (programming)</a>
         </div> 
         <p>As of May 2017 94.5% of 10 million most popular web pages used JavaScript.<sup id="cite_ref-49" class="reference"><a href="#cite_note-49">[49]</a></sup> The most common use of JavaScript is to add client-side behavior to <a href="/wiki/HTML" title="HTML">HTML</a> pages, also known as <a href="/wiki/Dynamic_HTML" title="Dynamic HTML">Dynamic HTML</a> (DHTML). Scripts are embedded in or included from <a href="/wiki/HTML" title="HTML">HTML</a> pages and interact with the <a href="/wiki/Document_Object_Model" title="Document Object Model">Document Object Model</a> (DOM) of the page. Some simple examples of this usage are:</p> 
         <ul>
          <li>Loading new page content or submitting data to the server via <a href="/wiki/Ajax_(programming)" title="Ajax (programming)">Ajax</a> without reloading the page (for example, a social network might allow the user to post status updates without leaving the page).</li> 
          <li>Animation of page elements, fading them in and out, resizing them, moving them, etc.</li> 
          <li>Interactive content, for example games, and playing audio and video.</li> 
          <li><a href="/wiki/Data_validation" title="Data validation">Validating</a> input values of a <a href="/wiki/Form_(HTML)" title="Form (HTML)">Web form</a> to make sure that they are acceptable before being submitted to the server.</li> 
          <li>Transmitting information about the user's reading habits and browsing activities to various websites. Web pages frequently do this for <a href="/wiki/Web_analytics" title="Web analytics">Web analytics</a>, <a href="/wiki/Ad_tracking" title="Ad tracking">ad tracking</a>, <a href="/wiki/Personalization" title="Personalization">personalization</a> or other purposes.<sup id="cite_ref-50" class="reference"><a href="#cite_note-50">[50]</a></sup></li> 
         </ul>
         <p>Because JavaScript code can run locally in a user's browser (rather than on a remote server), the browser can respond to user actions quickly, making an application more responsive. Furthermore, JavaScript code can detect user actions that HTML alone cannot, such as individual keystrokes. Applications such as <a href="/wiki/Gmail" title="Gmail">Gmail</a> take advantage of this: much of the user-interface logic is written in JavaScript, and JavaScript dispatches requests for information (such as the content of an e-mail message) to the server. The wider trend of Ajax programming similarly exploits this strength.</p> 
         <p>A <a href="/wiki/JavaScript_engine" title="JavaScript engine">JavaScript engine</a> (also known as JavaScript interpreter or JavaScript implementation) is an <a href="/wiki/Interpreter_(computing)" title="Interpreter (computing)">interpreter</a> that interprets JavaScript <a href="/wiki/Source_code" title="Source code">source code</a> and executes the <a href="/wiki/Computer_program" title="Computer program">script</a> accordingly. The first JavaScript engine was created by Brendan Eich at Netscape, for the Netscape Navigator Web browser. The engine, code-named <a href="/wiki/SpiderMonkey_(JavaScript_engine)" class="mw-redirect" title="SpiderMonkey (JavaScript engine)">SpiderMonkey</a>, is implemented in <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>. It has since been updated (in JavaScript 1.5) to conform to ECMAScript 3. The <a href="/wiki/Rhino_(JavaScript_engine)" title="Rhino (JavaScript engine)">Rhino</a> engine, created primarily by Norris Boyd (formerly at Netscape, now at Google) is a JavaScript implementation in <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>. Rhino, like SpiderMonkey, is ECMAScript 3 compliant.</p> 
         <p>A Web browser is by far the most common host environment for JavaScript. Web browsers typically create "host objects" to represent the DOM in JavaScript. The <a href="/wiki/Web_server" title="Web server">Web server</a> is another common host environment. A <a href="/wiki/Server-side_JavaScript" class="mw-redirect" title="Server-side JavaScript">JavaScript Web server</a> would typically expose host objects representing <a href="/wiki/HTTP" class="mw-redirect" title="HTTP">HTTP</a> request and response objects, which a JavaScript program could then interrogate and manipulate to dynamically generate Web pages.</p> 
         <p>Because JavaScript is the only language that the most popular browsers share support for, it has become a <a href="/wiki/Target_language_(computing)" class="mw-redirect" title="Target language (computing)">target language</a> for many frameworks in other languages, even though JavaScript was never intended to be such a language.<sup id="cite_ref-computerworld_51-0" class="reference"><a href="#cite_note-computerworld-51">[51]</a></sup> Despite the performance limitations inherent to its dynamic nature, the increasing speed of JavaScript engines has made the language a surprisingly feasible compilation target.</p>

         <h3 class="in-block"><span class="mw-headline" id="Compatibility_considerations">Compatibility considerations</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=22" title="Edit section: Compatibility considerations" data-section="22" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <div role="note" class="hatnote navigation-not-searchable">
          Main article: 
          <a href="/wiki/Web_interoperability" title="Web interoperability">Web interoperability</a>
         </div> 
         <p>Because JavaScript runs in widely varying environments, an important part of testing and debugging is to test and verify that the JavaScript works across multiple browsers.</p> 
         <p>The DOM interfaces for manipulating Web pages are not part of the ECMAScript standard, or of JavaScript itself. Officially, the DOM interfaces are defined by a separate standardization effort by the <a href="/wiki/World_Wide_Web_Consortium" title="World Wide Web Consortium">W3C</a>; in practice, browser implementations differ from the standards and from each other, and not all browsers execute JavaScript.</p> 
         <p>To deal with these differences, JavaScript authors can attempt to write standards-compliant code that will also be executed correctly by most browsers; failing that, they can write code that checks for the presence of certain browser features and behaves differently if they are not available.<sup id="cite_ref-52" class="reference"><a href="#cite_note-52">[52]</a></sup> In some cases, two browsers may both implement a feature but with different behavior, and authors may find it practical to detect what browser is running and change their script's behavior to match.<sup id="cite_ref-53" class="reference"><a href="#cite_note-53">[53]</a></sup><sup id="cite_ref-54" class="reference"><a href="#cite_note-54">[54]</a></sup> Programmers may also use libraries or toolkits that take browser differences into account.</p> 
         <p>Furthermore, scripts may not work for some users. For example, a user may:</p> 
         <ul>
          <li>use an old or rare browser with incomplete or unusual DOM support;</li> 
          <li>use a <a href="/wiki/Personal_digital_assistant" title="Personal digital assistant">PDA</a> or <a href="/wiki/Mobile_phone" title="Mobile phone">mobile phone</a> browser that cannot execute JavaScript;</li> 
          <li>have JavaScript execution disabled as a security precaution;</li> 
          <li>use a speech browser due to, for example, a visual disability.</li> 
         </ul>
         <p>To support these users, Web authors can try to create pages that <a href="/wiki/Fault-tolerant_system" class="mw-redirect" title="Fault-tolerant system">degrade gracefully</a> on user agents (browsers) that do not support the page's JavaScript. In particular, the page should remain usable albeit without the extra features that the JavaScript would have added. Some sites use the HTML &lt;noscript&gt; tag, which contains alt content if JS is disabled. An alternative approach that many find preferable is to first author content using basic technologies that work in all browsers, then enhance the content for users that have JavaScript enabled. This is known as <a href="/wiki/Progressive_enhancement" title="Progressive enhancement">progressive enhancement</a>.</p>

        <h2 class="in-block section-heading" onclick="javascript:mfTempOpenSection(7)">
         <div class="mw-ui-icon mw-ui-icon-element indicator"></div><span class="mw-headline" id="Security">Security</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=23" title="Edit section: Security" data-section="23" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h2>
        <div class="mf-section-7 collapsible-block" id="mf-section-7"> 
         <div role="note" class="hatnote navigation-not-searchable">
          See also: 
          <a href="/wiki/Browser_security" title="Browser security">Browser security</a>
         </div> 
         <p>JavaScript and the DOM provide the potential for malicious authors to deliver scripts to run on a client computer via the Web. Browser authors contain this risk using two restrictions. First, scripts run in a <a href="/wiki/Sandbox_(computer_security)" title="Sandbox (computer security)">sandbox</a> in which they can only perform Web-related actions, not general-purpose programming tasks like creating files. Second, scripts are constrained by the <a href="/wiki/Same-origin_policy" title="Same-origin policy">same-origin policy</a>: scripts from one Web site do not have access to information such as usernames, passwords, or cookies sent to another site. Most JavaScript-related security bugs are breaches of either the same origin policy or the sandbox.</p> 
         <p>There are subsets of general JavaScript—ADsafe, Secure ECMAScript (SES)—that provide greater levels of security, especially on code created by third parties (such as advertisements).<sup id="cite_ref-55" class="reference"><a href="#cite_note-55">[55]</a></sup><sup id="cite_ref-56" class="reference"><a href="#cite_note-56">[56]</a></sup> <a href="/wiki/Caja_project" title="Caja project">Caja</a> is another project for safe embedding and isolation of third-party JavaScript and HTML.</p> 
         <p><a href="/wiki/Content_Security_Policy" title="Content Security Policy">Content Security Policy</a> is the main intended method of ensuring that only trusted code is executed on a Web page.</p> 
         <div role="note" class="hatnote navigation-not-searchable">
          See also: 
          <a href="/wiki/Content_Security_Policy" title="Content Security Policy">Content Security Policy</a>
         </div> 
         <h3 class="in-block"><span class="mw-headline" id="Cross-site_vulnerabilities">Cross-site vulnerabilities</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=24" title="Edit section: Cross-site vulnerabilities" data-section="24" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <div role="note" class="hatnote navigation-not-searchable">
          Main articles: 
          <a href="/wiki/Cross-site_scripting" title="Cross-site scripting">Cross-site scripting</a> and 
          <a href="/wiki/Cross-site_request_forgery" title="Cross-site request forgery">Cross-site request forgery</a>
         </div> 
         <p>A common JavaScript-related security problem is <a href="/wiki/Cross-site_scripting" title="Cross-site scripting">cross-site scripting</a> (XSS), a violation of the <a href="/wiki/Same-origin_policy" title="Same-origin policy">same-origin policy</a>. XSS vulnerabilities occur when an attacker is able to cause a target Web site, such as an online banking website, to include a malicious script in the webpage presented to a victim. The script in this example can then access the banking application with the privileges of the victim, potentially disclosing secret information or transferring money without the victim's authorization. A solution to XSS vulnerabilities is to use <i>HTML escaping</i> whenever displaying untrusted data.</p> 
         <p>Some browsers include partial protection against <i>reflected</i> XSS attacks, in which the attacker provides a URL including malicious script. However, even users of those browsers are vulnerable to other XSS attacks, such as those where the malicious code is stored in a database. Only correct design of Web applications on the server side can fully prevent XSS.</p> 
         <p>XSS vulnerabilities can also occur because of implementation mistakes by browser authors.<sup id="cite_ref-57" class="reference"><a href="#cite_note-57">[57]</a></sup></p> 
         <p>Another cross-site vulnerability is <a href="/wiki/Cross-site_request_forgery" title="Cross-site request forgery">cross-site request forgery</a> (CSRF). In CSRF, code on an attacker's site tricks the victim's browser into taking actions the user didn't intend at a target site (like transferring money at a bank). It works because, if the target site relies only on cookies to authenticate requests, then requests initiated by code on the attacker's site will carry the same legitimate login credentials as requests initiated by the user. In general, the solution to CSRF is to require an authentication value in a hidden form field, and not only in the cookies, to authenticate any request that might have lasting effects. Checking the HTTP Referrer header can also help.</p> 
         <p>"JavaScript hijacking" is a type of CSRF attack in which a <tt>&lt;script&gt;</tt> tag on an attacker's site exploits a page on the victim's site that returns private information such as <a href="/wiki/JSON" title="JSON">JSON</a> or JavaScript. Possible solutions include:</p> 
         <ul>
          <li>requiring an authentication token in the <a href="/wiki/POST_(HTTP)" title="POST (HTTP)">POST</a> and <a href="/wiki/GET_(HTTP)" class="mw-redirect" title="GET (HTTP)">GET</a> parameters for any response that returns private information.</li> 
         </ul>
         <h3 class="in-block"><span class="mw-headline" id="Misplaced_trust_in_the_client">Misplaced trust in the client</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=25" title="Edit section: Misplaced trust in the client" data-section="25" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p>Developers of client-server applications must recognize that untrusted clients may be under the control of attackers. The application author cannot assume that his JavaScript code will run as intended (or at all) because any secret embedded in the code could be extracted by a determined adversary. Some implications are:</p> 
         <ul>
          <li>Web site authors cannot perfectly conceal how their JavaScript operates because the raw source code must be sent to the client. The code can be <a href="/wiki/Obfuscated_code" class="mw-redirect" title="Obfuscated code">obfuscated</a>, but obfuscation can be reverse-engineered.</li> 
          <li>JavaScript form validation only provides convenience for users, not security. If a site verifies that the user agreed to its terms of service, or filters invalid characters out of fields that should only contain numbers, it must do so on the server, not only the client.</li> 
          <li>Scripts can be selectively disabled, so JavaScript can't be relied on to prevent operations such as right-clicking on an image to save it.<sup id="cite_ref-58" class="reference"><a href="#cite_note-58">[58]</a></sup></li> 
          <li>It is extremely bad practice to embed sensitive information such as passwords in JavaScript because it can be extracted by an attacker.</li> 
         </ul>
         <h3 class="in-block"><span class="mw-headline" id="Misplaced_trust_in_developers">Misplaced trust in developers</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=26" title="Edit section: Misplaced trust in developers" data-section="26" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p>Package management systems such as <a href="/wiki/Npm_(software)" title="Npm (software)">NPM</a> and <a href="/w/index.php?title=Bower_(software)&amp;action=edit&amp;redlink=1" class="new" title="Bower (software) (page does not exist)">Bower</a> are popular with JavaScript developers. Such systems allow a developer to easily manage their program's dependencies upon other developer's program libraries. Developers trust that the maintainers of the libraries will keep it secure and up to date, but that is not always the case. A vulnerability has emerged because of this blind trust. Relied upon libraries can release an update that causes bugs or vulnerabilities to appear in all programs that rely upon the library. Inversely, a library can go unpatched with known vulnerabilities out in the wild. In a study done looking over a sample of 133k websites, researchers found 37% of the websites included a library with at-least one known vulnerability.<sup id="cite_ref-jslibs_59-0" class="reference"><a href="#cite_note-jslibs-59">[59]</a></sup> "The median lag between the oldest library version used on each website and the newest available version of that library is 1,177 days in ALEXA, and development of some libraries still in active use ceased years ago."<sup id="cite_ref-jslibs_59-1" class="reference"><a href="#cite_note-jslibs-59">[59]</a></sup> Another possibility is that the maintainer of a library may remove the library entirely. This occurred in March 2016 when Azer Koçulu removed his repository from <a href="/wiki/Npm_(software)" title="Npm (software)">NPM</a>. This caused all tens of thousands of programs and websites depending upon his libraries to break.<sup id="cite_ref-60" class="reference"><a href="#cite_note-60">[60]</a></sup><sup id="cite_ref-61" class="reference"><a href="#cite_note-61">[61]</a></sup></p> 
         <h3 class="in-block"><span class="mw-headline" id="Browser_and_plugin_coding_errors">Browser and plugin coding errors</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=27" title="Edit section: Browser and plugin coding errors" data-section="27" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p>JavaScript provides an interface to a wide range of browser capabilities, some of which may have flaws such as <a href="/wiki/Buffer_overflow" title="Buffer overflow">buffer overflows</a>. These flaws can allow attackers to write scripts that would run any code they wish on the user's system. This code is not by any means limited to another JavaScript application. For example, a buffer overrun exploit can allow an attacker to gain access to the operating system's <a href="/wiki/API" class="mw-redirect" title="API">API</a> with superuser privileges.</p> 
         <p>These flaws have affected major browsers including Firefox,<sup id="cite_ref-62" class="reference"><a href="#cite_note-62">[62]</a></sup> Internet Explorer,<sup id="cite_ref-63" class="reference"><a href="#cite_note-63">[63]</a></sup> and Safari.<sup id="cite_ref-64" class="reference"><a href="#cite_note-64">[64]</a></sup></p> 
         <p>Plugins, such as video players, <a href="/wiki/Adobe_Flash#Flash_client_security" title="Adobe Flash">Adobe Flash</a>, and the wide range of <a href="/wiki/ActiveX" title="ActiveX">ActiveX</a> controls enabled by default in Microsoft Internet Explorer, may also have flaws exploitable via JavaScript (such flaws have been exploited in the past).<sup id="cite_ref-65" class="reference"><a href="#cite_note-65">[65]</a></sup><sup id="cite_ref-66" class="reference"><a href="#cite_note-66">[66]</a></sup></p> 
         <p>In Windows Vista, Microsoft has attempted to contain the risks of bugs such as buffer overflows by running the Internet Explorer process with limited privileges.<sup id="cite_ref-67" class="reference"><a href="#cite_note-67">[67]</a></sup> <a href="/wiki/Google_Chrome" title="Google Chrome">Google Chrome</a> similarly confines its page renderers to their own "sandbox".</p> 
         <h3 class="in-block"><span class="mw-headline" id="Sandbox_implementation_errors">Sandbox implementation errors</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=28" title="Edit section: Sandbox implementation errors" data-section="28" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p>Web browsers are capable of running JavaScript outside the sandbox, with the privileges necessary to, for example, create or delete files. Of course, such privileges aren't meant to be granted to code from the Web.</p> 
         <p>Incorrectly granting privileges to JavaScript from the Web has played a role in vulnerabilities in both Internet Explorer<sup id="cite_ref-68" class="reference"><a href="#cite_note-68">[68]</a></sup> and Firefox.<sup id="cite_ref-69" class="reference"><a href="#cite_note-69">[69]</a></sup> In Windows XP Service Pack 2, Microsoft demoted JScript's privileges in Internet Explorer.<sup id="cite_ref-70" class="reference"><a href="#cite_note-70">[70]</a></sup></p> 
         <p><a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> allows JavaScript source files on a computer's hard drive to be launched as general-purpose, non-sandboxed programs (see: <a href="/wiki/Windows_Script_Host" title="Windows Script Host">Windows Script Host</a>). This makes JavaScript (like <a href="/wiki/VBScript" title="VBScript">VBScript</a>) a theoretically viable vector for a <a href="/wiki/Trojan_horse_(computing)" title="Trojan horse (computing)">Trojan horse</a>, although JavaScript Trojan horses are uncommon in practice.<sup id="cite_ref-71" class="reference"><a href="#cite_note-71">[71]</a></sup><sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Verifiability" title="Wikipedia:Verifiability"><span title="The material near this tag failed verification of its source citation(s). (March 2017)">not in citation given</span></a></i>]</sup></p> 
         <h3 class="in-block"><span class="mw-headline" id="Hardware_vulnerabilities">Hardware vulnerabilities</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=29" title="Edit section: Hardware vulnerabilities" data-section="29" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p>In 2017 a JavaScript based attack via browser was demonstrated that could bypass <a href="/wiki/Address_space_layout_randomization" title="Address space layout randomization">ASLR</a>. It's called "ASLR?Cache" or AnC.<sup id="cite_ref-72" class="reference"><a href="#cite_note-72">[72]</a></sup><sup id="cite_ref-73" class="reference"><a href="#cite_note-73">[73]</a></sup></p> 
        </div>

        <h2 class="in-block section-heading" onclick="javascript:mfTempOpenSection(9)">
         <div class="mw-ui-icon mw-ui-icon-element indicator"></div><span class="mw-headline" id="Development_tools">Development tools</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=34" title="Edit section: Development tools" data-section="34" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h2>
        <div class="mf-section-9 collapsible-block" id="mf-section-9"> 
         <p>Within JavaScript, access to a <a href="/wiki/Debugger" title="Debugger">debugger</a> becomes invaluable when developing large, non-trivial programs. Because there can be implementation differences between the various browsers (particularly within the DOM), it is useful to have access to a debugger for each of the browsers that a Web application targets.<sup id="cite_ref-93" class="reference"><a href="#cite_note-93">[93]</a></sup></p> 
         <p>Script debuggers are integrated within <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a>, <a href="/wiki/Firefox" title="Firefox">Firefox</a>, <a href="/wiki/Safari_(Web_browser)" class="mw-redirect" title="Safari (Web browser)">Safari</a>, <a href="/wiki/Google_Chrome" title="Google Chrome">Google Chrome</a>, <a href="/wiki/Opera_(Web_browser)" class="mw-redirect" title="Opera (Web browser)">Opera</a> and <a href="/wiki/Node.js" title="Node.js">Node.js</a>.<sup id="cite_ref-94" class="reference"><a href="#cite_note-94">[94]</a></sup><sup id="cite_ref-95" class="reference"><a href="#cite_note-95">[95]</a></sup><sup id="cite_ref-96" class="reference"><a href="#cite_note-96">[96]</a></sup></p> 
         <p>In addition to the native <a href="/wiki/Internet_Explorer_Developer_Tools" title="Internet Explorer Developer Tools">Internet Explorer Developer Tools</a>, three debuggers are available for Internet Explorer: <a href="/wiki/Microsoft_Visual_Studio" title="Microsoft Visual Studio">Microsoft Visual Studio</a> is the richest of the three, closely followed by <a href="/wiki/Microsoft_Script_Editor" title="Microsoft Script Editor">Microsoft Script Editor</a> (a component of <a href="/wiki/Microsoft_Office" title="Microsoft Office">Microsoft Office</a>),<sup id="cite_ref-97" class="reference"><a href="#cite_note-97">[97]</a></sup> and finally the free <a href="/wiki/Microsoft_Script_Debugger" title="Microsoft Script Debugger">Microsoft Script Debugger</a> that is far more basic than the other two. The free <a href="/wiki/Microsoft_Visual_Web_Developer_Express" class="mw-redirect" title="Microsoft Visual Web Developer Express">Microsoft Visual Web Developer Express</a> provides a limited version of the JavaScript debugging functionality in Microsoft Visual Studio. Internet Explorer has included developer tools since version 8.</p> 
         <p>In comparison to Internet Explorer, Firefox has a more comprehensive set of developer tools, which include a debugger as well. Old versions of Firefox without these tools used a <a href="/wiki/Firefox_addon" class="mw-redirect" title="Firefox addon">Firefox addon</a> called <a href="/wiki/Firebug_(Firefox_extension)" class="mw-redirect" title="Firebug (Firefox extension)">Firebug</a>, or the older <a href="/wiki/Venkman" title="Venkman">Venkman</a> debugger. Also, <a href="/wiki/WebKit" title="WebKit">WebKit</a>'s Web Inspector includes a JavaScript debugger,<sup id="cite_ref-98" class="reference"><a href="#cite_note-98">[98]</a></sup> which is used in <a href="/wiki/Safari_(Web_browser)" class="mw-redirect" title="Safari (Web browser)">Safari</a>. A modified version called Blink DevTools is used in <a href="/wiki/Google_Chrome" title="Google Chrome">Google Chrome</a>. <a href="/wiki/Node.js" title="Node.js">Node.js</a> has Node Inspector, an interactive debugger that integrates with the Blink DevTools, available in <a href="/wiki/Google_Chrome" title="Google Chrome">Google Chrome</a>. <a href="/wiki/Opera_(Web_browser)" class="mw-redirect" title="Opera (Web browser)">Opera</a> includes a set of tools called <a href="/wiki/Opera_Dragonfly" title="Opera Dragonfly">Dragonfly</a>.<sup id="cite_ref-99" class="reference"><a href="#cite_note-99">[99]</a></sup></p> 
         <p><span id="Online_dev_tools"></span>In addition to the native computer software, there are online JavaScript IDEs, debugging aids that are themselves written in JavaScript and built to run on the Web. An example is the program <a href="/wiki/JSLint" title="JSLint">JSLint</a>, developed by <a href="/wiki/Douglas_Crockford" title="Douglas Crockford">Douglas Crockford</a> who has written extensively on the language. JSLint scans JavaScript code for conformance to a set of standards and guidelines. Many libraries for JavaScript, such as <a href="/wiki/Three.js" title="Three.js">three.js</a>, provide links to demonstration code that can be edited by users. They are also used as a pedagogical tool by institutions such as <a href="/wiki/Khan_Academy" title="Khan Academy">Khan Academy</a><sup id="cite_ref-100" class="reference"><a href="#cite_note-100">[100]</a></sup> to allow students to experience writing code in an environment where they can see the output of their programs, without needing any setup beyond a Web browser.</p>

         <div class="mw-ui-icon mw-ui-icon-element indicator"></div><span class="mw-headline" id="Benchmark_tools_for_developers">Benchmark tools for developers</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=35" title="Edit section: Benchmark tools for developers" data-section="35" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h2>
        <div class="mf-section-10 collapsible-block" id="mf-section-10"> 
         <p>Since JavaScript is getting more important for web development (frontend overtakes many aspects which were done in backend before), there is also more consideration done about performance. Especially mobile devices could have problems with rendering and processing unoptimized complex logic.</p> 
         <p>A library for doing benchmarks is <a rel="nofollow" class="external text" href="https://benchmarkjs.com/">benchmark.js</a>. A benchmarking library that supports high-resolution timers and returns statistically significant results<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="Need citation on statistically significant results (November 2016)">citation needed</span></a></i>]</sup>.</p> 
         <p>Another tool is <a rel="nofollow" class="external text" href="http://jsben.ch">jsben.ch</a>. An online JavaScript benchmarking tool, where code snippets can be tested against each other.</p>

In 1997 the first standardized version of the language was released under the name <a href="/wiki/ECMAScript" title="ECMAScript">ECMAScript</a> in the first edition of the ECMA-252 standard. The explicit versioning and opt-in of language features was Mozilla-specific and has been removed. Firefox 4 was the last version which referred to a JavaScript version (1.8.5). With new editions of the ECMA-262 standard, JavaScript language features are now often mentioned with their initial definition in the ECMA-262 editions.</p>

        <h2 class="in-block section-heading" onclick="javascript:mfTempOpenSection(12)">
         <div class="mw-ui-icon mw-ui-icon-element indicator"></div><span class="mw-headline" id="Related_languages_and_features">Related languages and features</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=37" title="Edit section: Related languages and features" data-section="37" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h2>
        <div class="mf-section-12 collapsible-block" id="mf-section-12"> 
         <p><a href="/wiki/JSON" title="JSON">JSON</a>, or JavaScript Object Notation, is a general-purpose data interchange format that is defined as a subset of JavaScript's object literal syntax. Like much of JavaScript (regexps and anonymous functions as 1st class elements, closures, flexible classes, 'use strict'), <a href="/wiki/JSON" title="JSON">JSON</a>, except for replacing <a href="/wiki/Perl" title="Perl">Perl</a>'s key-value operator '=&gt;' by an <a href="/wiki/RFC_822" class="mw-redirect" title="RFC 822">RFC 822</a><sup id="cite_ref-105" class="reference"><a href="#cite_note-105">[105]</a></sup> inspired ':', is syntactically pure Perl.</p> 
         <p><a href="/wiki/JQuery" title="JQuery">jQuery</a> is a popular JavaScript library designed to simplify <a href="/wiki/Document_Object_Model" title="Document Object Model">DOM</a>-oriented client-side HTML scripting along with offering cross-browser compatibility because various browsers respond differently to certain vanilla JavaScript code.</p> 
         <p><a href="/wiki/Underscore.js" title="Underscore.js">Underscore.js</a> is a utility JavaScript library for data manipulation that is used in both client-side and server-side network applications.</p> 
         <p><a href="/wiki/Angular_(application_platform)" title="Angular (application platform)">Angular</a> and <a href="/wiki/AngularJS" title="AngularJS">AngularJS</a> are <a href="/wiki/Web_application_framework" class="mw-redirect" title="Web application framework">web application frameworks</a> to use for developing <a href="/wiki/Single-page_application" title="Single-page application">single-page applications</a> and also cross-platform mobile apps.</p> 
         <p><a href="/wiki/React_(JavaScript_library)" title="React (JavaScript library)">React (JavaScript library)</a> is an <a href="/wiki/Open-source" class="mw-redirect" title="Open-source">open-source</a> JavaScript library providing a views that is rendered using components specified as custom HTML tags.</p> 
         <p>Mozilla browsers currently support <a href="/wiki/LiveConnect" class="mw-redirect" title="LiveConnect">LiveConnect</a>, a feature that allows JavaScript and Java to intercommunicate on the Web. However, Mozilla-specific support for LiveConnect was scheduled to be phased out in the future in favor of passing on the LiveConnect handling via <a href="/wiki/NPAPI" title="NPAPI">NPAPI</a> to the Java 1.6+ plug-in (not yet supported on the Mac as of March 2010<sup class="plainlinks noprint asof-tag update" style="display:none;"><a class="external text" href="//en.wikipedia.org/w/index.php?title=JavaScript&amp;action=edit">[update]</a></sup>).<sup id="cite_ref-106" class="reference"><a href="#cite_note-106">[106]</a></sup> Most browser inspection tools, such as <a href="/wiki/Firebug_(software)" title="Firebug (software)">Firebug</a> in Firefox, include JavaScript interpreters that can act on the visible page's DOM.</p> 
         <p><a href="/wiki/Asm.js" title="Asm.js">asm.js</a> is a subset of JavaScript that can be run in any JavaScript engine or run faster in an <a href="/wiki/Ahead-of-time" class="mw-redirect" title="Ahead-of-time">ahead-of-time</a> (AOT) compiling engine.<sup id="cite_ref-107" class="reference"><a href="#cite_note-107">[107]</a></sup></p> 
         <p><a href="/wiki/JSFuck" title="JSFuck">JSFuck</a> is an <a href="/wiki/Esoteric_programming_language" title="Esoteric programming language">esoteric programming language</a>. Programs are written using only six different characters, but are still valid JavaScript code.</p> 
         <p><a href="/wiki/Processing_(programming_language)#p5.js" title="Processing (programming language)">p5.js</a><sup id="cite_ref-108" class="reference"><a href="#cite_note-108">[108]</a></sup> is an object oriented JavaScript library designed for artists and designers. It is based on the ideas of the <a href="/wiki/Processing_(programming_language)" title="Processing (programming language)">Processing</a> project but is for the web.</p> 
         <p><a rel="nofollow" class="external text" href="http://jsben.ch">jsben.ch</a> is an online JavaScript benchmarking tool, where different code snippets can be tested against each other.</p> 
         <p><a rel="nofollow" class="external text" href="http://dspace.thapar.edu:8080/jspui/handle/10266/4162">CRISP: A Strategy guiding Cloud Application Development for Beginners</a> is a strategy proposed by Ayush Sahu to develop optimized and secure JavaScript application to be used in mobiles, PC's and other devices. CRISP (Conversion, Reformat code, Isolate module, Sandbox, Partition) strategy has been proposed for refined conversion of native application to JavaScript for cloud application development. JavaScript is chosen as medium for writing application because it is mostly used language among developers and provides rich API (Application Programming Interface) for writing applications.<sup id="cite_ref-109" class="reference"><a href="#cite_note-109">[109]</a></sup></p> 
         <h3 class="in-block"><span class="mw-headline" id="Use_as_an_intermediate_language">Use as an intermediate language</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=38" title="Edit section: Use as an intermediate language" data-section="38" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p>As JavaScript is the most widely supported client-side language that can run within a Web browser, it has become an <a href="/wiki/Intermediate_language" class="mw-redirect" title="Intermediate language">intermediate language</a> for other languages to target. This has included both newly created languages and ports of existing languages. Some of these include:</p> 
         <ul>
          <li>OberonScript, a full implementation of the <a href="/wiki/Oberon_(programming_language)" title="Oberon (programming language)">Oberon</a> programming language that compiles to high-level JavaScript.<sup id="cite_ref-110" class="reference"><a href="#cite_note-110">[110]</a></sup></li> 
          <li><a href="/wiki/Objective-J" title="Objective-J">Objective-J</a>, a superset of JavaScript that compiles to standard JavaScript. It adds traditional inheritance and <a href="/wiki/Smalltalk" title="Smalltalk">Smalltalk</a>/<a href="/wiki/Objective-C" title="Objective-C">Objective-C</a> style dynamic dispatch and optional pseudo-static typing to JavaScript.</li> 
          <li><a href="/wiki/Processing.js" title="Processing.js">Processing.js</a>, a JavaScript port of the Processing programming language designed to write visualizations, images, and interactive content. It allows Web browsers to display animations, visual applications, games and other graphical rich content without the need for a Java applet or Flash plugin.</li> 
          <li><a href="/wiki/CoffeeScript" title="CoffeeScript">CoffeeScript</a>, an alternate syntax for JavaScript intended to be more concise and readable. It adds features like array comprehensions (also available in JavaScript since version 1.7)<sup id="cite_ref-111" class="reference"><a href="#cite_note-111">[111]</a></sup> and pattern matching. Like Objective-J, it compiles to JavaScript. Ruby and Python have been cited as influential on CoffeeScript syntax.</li> 
          <li><a href="/wiki/Google_Web_Toolkit" title="Google Web Toolkit">Google Web Toolkit</a> translates a subset of Java to JavaScript.</li> 
          <li><a href="/wiki/Scala_(programming_language)" title="Scala (programming language)">Scala</a>, an object-oriented and functional programming language, has a Scala-to-JavaScript compiler.<sup id="cite_ref-112" class="reference"><a href="#cite_note-112">[112]</a></sup></li> 
          <li><a href="/wiki/Pyjs" title="Pyjs">Pyjs</a>, a port of <a href="/wiki/Google_Web_Toolkit" title="Google Web Toolkit">Google Web Toolkit</a> to <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a> translates a subset of Python to JavaScript.</li> 
          <li><a href="/wiki/Dart_(programming_language)" title="Dart (programming language)">Google Dart</a>, an all-purpose, <a href="/wiki/Open-source_software" title="Open-source software">open source</a> language that compiles to JavaScript.</li> 
          <li>Whalesong,<sup id="cite_ref-113" class="reference"><a href="#cite_note-113">[113]</a></sup> a <a href="/wiki/Racket_(programming_language)" title="Racket (programming language)">Racket</a>-to-JavaScript compiler.</li> 
          <li><a href="/wiki/Emscripten" title="Emscripten">Emscripten</a>, a <a href="/wiki/LLVM" title="LLVM">LLVM</a>-backend for porting native libraries to JavaScript, known as <a href="/wiki/Asm.js" title="Asm.js">asm.js</a><sup id="cite_ref-114" class="reference"><a href="#cite_note-114">[114]</a></sup></li> 
          <li><a href="/wiki/Fantom_(programming_language)" title="Fantom (programming language)">Fantom</a> a programming language that runs on JVM, .NET and JavaScript.</li> 
          <li><a href="/wiki/TypeScript" title="TypeScript">TypeScript</a>, a free and open-source programming language developed by Microsoft. It is a superset of JavaScript, and essentially adds support for optional <a href="/wiki/Type_annotation" class="mw-redirect" title="Type annotation">type annotations</a> and some other language extensions such as classes, interfaces and modules. A TS-script compiles into plain JavaScript and can be executed in any JS host supporting <a href="/wiki/ECMAScript" title="ECMAScript">ECMAScript</a> 3 or higher. The compiler is itself written in TypeScript.</li> 
          <li><a href="/wiki/Haxe" title="Haxe">Haxe</a>, an open-source high-level multiplatform programming language and compiler that can produce applications and source code for many different platforms including JavaScript.</li> 
          <li>ClojureScript,<sup id="cite_ref-115" class="reference"><a href="#cite_note-115">[115]</a></sup> a compiler for <a href="/wiki/Clojure" title="Clojure">Clojure</a> that targets JavaScript. It is designed to emit JavaScript code that is compatible with the advanced compilation mode of the Google Closure optimizing compiler.</li> 
          <li><a href="/wiki/Kotlin_(programming_language)" title="Kotlin (programming language)">Kotlin</a>, a <a href="/wiki/Static_type-checking" class="mw-redirect" title="Static type-checking">statically-typed</a> language that also compiles to <a href="/wiki/Java_byte_code" class="mw-redirect" title="Java byte code">Java byte code</a>.</li> 
          <li><a rel="nofollow" class="external text" href="https://bertfreudenberg.github.io/SqueakJS/">SqueakJS</a>, a virtual machine and DOM environment for the open-source <a href="/wiki/Squeak" title="Squeak">Squeak</a> implementation of the <a href="/wiki/Smalltalk" title="Smalltalk">Smalltalk</a> programming language.</li> 
         </ul>
         <p>As JavaScript has unusual limitations – such as no explicit integer type, only double-precision binary floating point – languages that compile to JavaScript and do not take care to use the integer-converting shift and bitwise logical operators may have slightly different behavior than in other environments.</p> 
         <h3 class="in-block"><span class="mw-headline" id="JavaScript_and_Java">JavaScript and Java</span><span><a href="/w/index.php?title=JavaScript&amp;action=edit&amp;section=39" title="Edit section: JavaScript and Java" data-section="39" class="mw-ui-icon mw-ui-icon-element mw-ui-icon-edit-enabled edit-page">Edit</a></span></h3> 
         <p>A common misconception is that JavaScript is similar or closely related to <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>. It is true that both have a C-like syntax (the C language being their most immediate common ancestor language). They also are both typically <a href="/wiki/Sandbox_(computer_security)" title="Sandbox (computer security)">sandboxed</a> (when used inside a browser), and JavaScript was designed with Java's syntax and standard library in mind. In particular, all Java keywords were reserved in original JavaScript, JavaScript's standard library follows Java's naming conventions, and JavaScript's Math and Date objects are based on classes from Java 1.0,<sup id="cite_ref-popularity_116-0" class="reference"><a href="#cite_note-popularity-116">[116]</a></sup> but the similarities end there.</p> 
         <p><a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> and JavaScript both first appeared on 23 May 1995, but Java was developed by James Gosling of Sun Microsystems, and JavaScript by Brendan Eich of NetScape Communications.</p> 
         <p>The differences between the two languages are more prominent than their similarities. Java has <a href="/wiki/Static_typing" class="mw-redirect" title="Static typing">static typing</a>, while JavaScript's typing is <a href="/wiki/Dynamic_typing" class="mw-redirect" title="Dynamic typing">dynamic</a>. Java is loaded from compiled bytecode, while JavaScript is loaded as human-readable source code. Java's objects are <a href="/wiki/Class-based_programming" title="Class-based programming">class-based</a>, while JavaScript's are <a href="/wiki/Prototype-based_programming" title="Prototype-based programming">prototype-based</a>. Finally, Java did not support <a href="/wiki/Functional_programming" title="Functional programming">functional programming</a> until Java 8, while JavaScript has done so from the beginning, being influenced by <a href="/wiki/Scheme_(programming_language)" title="Scheme (programming language)">Scheme</a>.</p>
