     <h1>PhoneJS - HTML5 JavaScript Mobile Development Framework</h1> 
     <div class="article_meta"> 
      <div style="margin-bottom: 5px;"> 
       <span class="timestamp">??&nbsp;2013-07-15 15:30:00 </span> 
       <span class="from"> <i class="icon-globe"></i> <a class="cut cut28 from" href="/sites/2i6Bfa" target="_blank">Architects Zone </a> </span> 
      </div> 
      <div class="source"> 
       <i style="float:left;">??</i>&nbsp; 
       <a class="cut cut70" href="http://www.tuicool.com/articles/hit/eqYRzyi" style="display:inline-block;">http://architects.dzone.com/articles/phonejs-html5-javascript</a>

       <p>As you know, there are many frameworks for mobile app development and a growing number of them are based on HTML5. These next-generation tools help developers create mobile apps for phones and tablets without the steep learning curve associated with native SDKs and other programming languages like Objective-C or Java. For countless developers throughout the world, HTML5 represents the future for cross-platform mobile app development.</p> 
       <p>But the question is why? Why has HTML5 become so popular?</p> 
       <p>The widespread adoption of HTML5 involves the emergence of the bring your own device (BYOD) movement. BYOD means that developers can no longer limit application usage to a single platform because consumers want their apps to run on the devices they use every day. HTML5 allows developers to target multiple devices with a single codebase and deliver experiences that closely emulate native solutions, without writing the same application multiple times, using multiple languages or SDKs. The evolution of modern web browsers means that HTML5 can deliver cross-platform, multi-device solutions that mirror behaviors and experiences of “native” apps to the point that it’s often difficult to distinguish between an app written using native development tools and those using HTML.</p> 
       <p>Multiple platform support, time to market and lower maintenance costs are just a few of the advantages inherent in HTML/JavaScript. It’s advantages don’t stop there. HTML’s ability to mitigate long-term risks associated with emerging technologies such as WinRT, ChromeOS, FirefoxOS, and Tizen is unmatched. Simply said, the only code that will work on all these platforms is HTML/JavaScript.</p> 
       <p>Is there a price? Yes, certainly native app consumes less memory and will have faster or more responsive user experience. But in all cases where a web app would work, you can make a step further and create a mobile web app or even packaged application for the store, for multiple platforms, from single codebase. PhoneJS lets you get started fast.</p> 
       <h2>PhoneJS for Your Next Mobile Project</h2> 
       <p> <a href="http://phonejs.devexpress.com/" rel="nofollow,noindex" target="_blank">PhoneJS</a> is a cross-platform HTML5 mobile app development framework that was built to be versatile, flexible and efficient. PhoneJS is a single page application (SPA) framework, with view management and URL routing. Its layout engine allows you to abstract navigation away from views, so the same app can be rendered differently across different platforms or form factors. PhoneJS includes a rich collection of touch-optimized UI widgets with built-in styles for today’s most popular mobile platforms including iOS, Android, and Windows Phone 8. </p> 
       <center> 
        <img src="http://img1.tuicool.com/i6B7Znn.png!web" class="alignCenter"> 
       </center> 
       <p> <span>To better understand the principles of PhoneJS development and how you can create and publish applications in platform stores, let’s take a look at a simple &nbsp;demo app called</span> <a href="http://phonejs.devexpress.com/Demos/?url=TipCalculator&amp;sm=3" rel="nofollow,noindex" target="_blank">TipCalculator</a> <span>. This application calculates tip amounts due based on a restaurant bill. &nbsp;The complete source code for this app is</span> <a href="https://github.com/DevExpress/PhoneJS-TipCalculator" rel="nofollow,noindex" target="_blank">available here</a> <span>.</span> </p> 
       <p>The app can be found in the AppStore, Google Play, and Windows Store.</p> 
       <p> <img src="http://img0.tuicool.com/yQ7VraF.gif" class="breakLine"> <img src="http://img1.tuicool.com/qmqeyuj.png!web" class="breakLine"> <img src="http://img0.tuicool.com/UnuaAv.png!web" class="breakLine"> </p> 
       <h2>PhoneJS Application Layout and Routing</h2> 
       <p> <a href="http://phonejs.devexpress.com/Demos/?url=TipCalculator&amp;sm=3" rel="nofollow,noindex" target="_blank">TipCalculator</a> is a Single-Page Application (SPA) built with HTML5. The start page is <a href="https://github.com/DevExpress/PhoneJS-TipCalculator/blob/master/index.html" rel="nofollow,noindex" target="_blank">index.html</a> , with standard meta tags and links to CSS and JavaScript resources. It includes a script reference to the JavaScript file <a href="https://github.com/DevExpress/PhoneJS-TipCalculator/blob/master/index.js" rel="nofollow,noindex" target="_blank">index.js</a> , where you’ll find the code that configures PhoneJS app framework logic: </p> 
       <pre class="prettyprint">TipCalculator.app = new DevExpress.framework.html.HtmlApplication({
    namespace: TipCalculator,
    defaultLayout: "empty"
});</pre> 
       <p> Within this section, we must specify the default layout for the app. In this example, we’ll use the simplest option, an <b> <i>empty</i> </b> layout. More advanced layouts are available with full support for interactive navigation styles described in the following images: </p> 
       <center> 
        <img src="http://img0.tuicool.com/yIjumaU.png!web" class="alignCenter"> 
       </center> 
       <p> PhoneJS uses well-established layout methodologies supported by many server-side frameworks, including Ruby on Rails and ASP.NET MVC. Detailed information about Views and Layouts can be found <a href="http://phonejs.devexpress.com/Howto/Views_and_Layouts" rel="nofollow,noindex" target="_blank">in our online documentation</a> . </p> 
       <p>To configure view routing in our SPA, we must add an additional line of code in index.js:</p> 
       <pre class="prettyprint">TipCalculator.app.router.register(":view", { view: "home" });</pre> 
       <p> This registers a simple route that retrieves the view name from the URL (from the hash segment of the URL). The <b> <i>home</i> </b> view is used by default. Each view is defined in its own HTML file and is linked into the main application page index.html like this: </p> 
       <pre class="prettyprint">&lt;link rel="dx-template" type="text/html" href="views/home.html" /&gt;</pre> 
       <h2>PhoneJS ViewModel</h2> 
       <p> A viewmodel is a representation of data and operations used by the view. Each view has a function with the same base name as the view itself and returns the viewmodel for the view. For the home view, the <a href="https://github.com/DevExpress/PhoneJS-TipCalculator/blob/master/views/home.js" rel="nofollow,noindex" target="_blank">views/home.js</a> script defines the function <b> <i>home</i> </b> which creates the corresponding viewmodel. </p> 
       <pre>TipCalculator.home = function(params) {
    ...
};</pre> 
       <p>Three input parameters are used for the tip calculation algorithm: bill total, the number of people sharing the bill, and a tip percentage. These variables are defined as observables, which will be bound to corresponding UI widgets.</p> 
       <p> <b>Note</b> : Observables functionality is supplied by Knockout.js, an important foundation for viewmodels used in PhoneJS. You can learn more about Knockout.js <a href="http://knockoutjs.com/documentation/observables.html" rel="nofollow,noindex" target="_blank">here</a> . </p> 
       <p> This is the code used in the <b> <i>home</i> </b> function to initialize the variables: </p> 
       <pre class="prettyprint">var billTotal = ko.observable(),
    tipPercent = ko.observable(DEFAULT_TIP_PERCENT),
    splitNum = ko.observable(1);</pre> 
       <p> <span> The result of the tip calculation is represented by four values: <b> <i>totalToPay</i> </b> , <b> <i>totalPerPerson</i> </b> , <b> <i>totalTip</i> </b> , <b> <i>tipPerPerson</i> </b> . Each value is a </span> <a href="http://knockoutjs.com/documentation/computedObservables.html" rel="nofollow,noindex" target="_blank">dependent observable</a> <span>(a computed value), which is automatically recalculated when any of the observables used in its definition change. Again, this is standard Knockout.js functionality.</span> </p> 
       <pre class="prettyprint">var totalTip = ko.computed(...);
var tipPerPerson = ko.computed(...);
var totalPerPerson = ko.computed(...);
var totalToPay = ko.computed(...);</pre> 
       <p> For an example of business logic implementation in a viewmodel, let’s take a closer look at the observable <b> <i>totalToPay</i> </b> . </p> 
       <p> The total sum to pay is usually rounded. For this purpose, we have two functions <b> <i>roundUp</i> </b> and <b> <i>roundDown</i> </b> that change the value of <b> <i>roundMode</i> </b> (another observable). These changes cause recalculation of <b> <i>totalToPay</i> </b> , because <b> <i>roundMode</i> </b> is used in the code associated with the <b> <i>totalToPay</i> </b> observable. </p> 
       <pre class="prettyprint">var totalToPay = ko.computed(function() {
    var value = totalTip() + billTotalAsNumber();

    switch(roundMode()) {
        case ROUND_DOWN:
            if(Math.floor(value) &gt;= billTotalAsNumber())
                return Math.floor(value);
            return value;

        case ROUND_UP:
            return Math.ceil(value);
        default:
            return value;
    }
});</pre> 
       <p>When any input parameter in the view changes, rounding should be disabled to allow the user to view precise values. We subscribe to the changes of the UI-bound observables to achieve this:</p> 
       <pre class="prettyprint">billTotal.subscribe(function() {
    roundMode(ROUND_NONE);
});

tipPercent.subscribe(function() {
    roundMode(ROUND_NONE);
});

splitNum.subscribe(function() {
    roundMode(ROUND_NONE);
});</pre> 
       <p>The complete viewmodel can be found in home.js. It represents a simple example of a typical viewmodel.</p> 
       <p> <b>Note</b> : In a more complex app, it may be useful to implement a structure that modularizes your viewmodels separate from view implementation files. In other words, a file like home.js need not contain the code to implement the viewmodel and instead call a helper function elsewhere for this purpose. In this walkthrough we’re trying to keep things structurally simple. </p> 
       <h2>PhoneJS Views</h2> 
       <p> Let’s now turn to the markup of the view located in the <a href="https://github.com/DevExpress/PhoneJS-TipCalculator/blob/master/views/home.html" rel="nofollow,noindex" target="_blank">view/home.html</a> file. The root div element represents a view with the name ‘ <b> <i>home</i> </b> ’. Within it is a div containing markup for a placeholder called ‘ <b> <i>content</i> </b> ’. </p> 
       <pre class="prettyprint">&lt;div data-options="dxView : { name: 'home' }"&gt;
     &lt;div data-options="dxContent : { targetPlaceholder: 'content' }"&gt;
         ...
     &lt;/div&gt;
&lt;/div&gt;</pre> 
       <p>A toolbar is located at the top of the view:</p> 
       <pre class="prettyprint">&lt;div data-bind="dxToolbar: { items: [ { align: 'center', text: 'Tip Calculator' } ] }"&gt;&lt;/div&gt;</pre> 
       <p> <b> <i>dxToolbar</i> </b> is a PhoneJS UI widget. It’s defined in the markup using Knockout.js binding. </p> 
       <p> A fieldset appears below the toolbar. To display a fieldset, we use two special CSS classes understood by PhoneJS: <b> <i>dx-fieldset</i> </b> and <b> <i>dx-field</i> </b> . The fieldset contains a text field for the bill total and two sliders for the tip percentage and the number of diners. </p> 
       <pre class="prettyprint">&lt;div data-bind="dxNumberBox: { value: billTotal, placeholder: 'Type here...', valueUpdateEvent: 'keyup',min: 0 }"&gt;
&lt;/div&gt;

&lt;div data-bind="dxSlider: { min: 0, max: 25, step: 1, activeStateEnabled: true, value: tipPercent }"&gt;&lt;/div&gt;

&lt;div data-bind="dxSlider: { min: 1, step: 1, max: 10, activeStateEnabled: true, value: splitNum }"&gt;&lt;/div&gt;</pre> 
       <p> Two buttons ( <b> <i>dxButton</i> </b> ) are displayed below the editors, allowing the user to round the total sum to pay. The remaining view displays fieldsets used for calculated results. </p> 
       <pre class="prettyprint">&lt;div class="round-buttons"&gt;
    &lt;div data-bind="dxButton: { text: 'Round Down', clickAction: roundDown }"&gt;&lt;/div&gt;
    &lt;div data-bind="dxButton: { text: 'Round Up', clickAction: roundUp }"&gt;&lt;/div&gt;
&lt;/div&gt;

&lt;div id="results" class="dx-fieldset"&gt;
    &lt;div class="dx-field"&gt;
        &lt;span class="dx-field-label"&gt;Total to pay&lt;/span&gt;
        &lt;span class="dx-field-value" style="font-weight: bold" data-bind="text: Globalize.format(totalToPay(), 'c')"&gt;&lt;/span&gt;
    &lt;/div&gt;
    &lt;div class="dx-field"&gt;
        &lt;span class="dx-field-label"&gt;Total per person&lt;/span&gt;
        &lt;span class="dx-field-value" data-bind="text: Globalize.format(totalPerPerson(), 'c')"&gt;&lt;/span&gt;
    &lt;/div&gt;
    &lt;div class="dx-field"&gt;
        &lt;span class="dx-field-label"&gt;Total tip&lt;/span&gt;
        &lt;span class="dx-field-value" data-bind="text: Globalize.format(totalTip(), 'c')"&gt;&lt;/span&gt;
    &lt;/div&gt;
    &lt;div class="dx-field"&gt;
        &lt;span class="dx-field-label"&gt;Tip per person&lt;/span&gt;
        &lt;span class="dx-field-value" data-bind="text: Globalize.format(tipPerPerson(), 'c')"&gt;&lt;/span&gt;
    &lt;/div&gt;
&lt;/div&gt;</pre> 
       <p>This completes the description of the files required to create a simple app using PhoneJS. &nbsp;As &nbsp;you’ve seen, the process is simple, straightforward and intuitive.</p> 
       <h2>Start, Debug and Build for Stores</h2> 
       <p>Starting and debugging a PhoneJS app is just like any other HTML5 based app. You must deploy the folder containing HTML and JavaScript sources, along with any other required file to your web server. Because there is no server-side component to the architectural model, it doesn’t matter which web server you use as long as it can provide file access through HTTP. Once deployed, you can open the app on a device, in an emulator or a desktop browser by simply navigating app’s start page URL.</p> 
       <p>If you want to view the app as it will appear in a phone or tablet within a desktop browser, you will have to override the UserAgent in the browser. Fortunately, this is easy to do with the developer tools that ship as part of today’s modern browsers:</p> 
       <center> 
        <img src="http://img0.tuicool.com/2e2A7vR.png!web" class="alignCenter"> 
       </center> 
       <p> If you prefer not to modify UserAgent settings, you can use the <a href="http://emulate.phonegap.com/" rel="nofollow,noindex" target="_blank">Ripple Emulator</a> to emulate multiple device types. </p> 
       <center> 
        <img src="http://img1.tuicool.com/iYbiU3A.png!web" class="alignCenter"> 
       </center> 
       <p>At this point you have a web application that will work in the browser on the mobile device and look like native app. Modern mobile browsers provide access to local storage, location api, camera, so good chances are that your app already has anything it needs.</p> 
       <h2>Creating Store Ready Applications Using PhoneJS and PhoneGap</h2> 
       <p> But what if you need access to device features that browser does not provide? What if you want an app in the app store, not just a webpage. Then you’ll have to create a hybrid application and de-facto standard for such an app is <a href="http://cordova.apache.org/" rel="nofollow,noindex" target="_blank">Apache Cordova</a> aka <a href="http://phonegap.com/" rel="nofollow,noindex" target="_blank">PhoneGap</a> . </p> 
       <p>PhoneGap project for each platform is a native app project that contains WebView (browser control) and a “bridge” that lets your JavaScript code inside WebView access native functions provided by PhoneGap libraries and plugins.</p> 
       <p>To use it, you need to have SDK for each platform you are targeting, but you don’t need to know details of native development, you just need to put your HTML, CSS, JS files into right places and specify your app’s properties like name, version, icons, splashcreens and so on.</p> 
       <p>To be able to publish your app, you will need to register as developer in the respective development portal. This process is well documented for each store and beyond the scope of this article. After that you’ll be able to receive certificates to sign your app package.</p> 
       <p>The need to have SDK for each platform installed sounds challenging - especially after “write one, run everywhere” promise of HTML5/JS approach. This is a small price to pay for building hybrid application and have everything under control. But still there are several services and products that solves this problem for you.</p> 
       <p> One is Adobe’s online service - <a href="https://build.phonegap.com/" rel="nofollow,noindex" target="_blank">PhoneGap Build</a> which allows you to build one app for free (to build more, you’ll need a paid account). If you have all the required platform certificate files, the service can build your app for all supported platforms with a few mouse clicks. You only need to prepare app descriptions, promotional and informational content and icons in order to submit your app to an individual store. </p> 
       <p> For Visual Studio developers, DevExpress offers a product called <a href="http://www.devexpress.com/Products/HTML-JS/" rel="nofollow,noindex" target="_blank">DevExtreme</a> (it includes PhoneJS), which can build applications for iOS, Android and Windows Phone 8 directly within the Microsoft Visual Studio IDE. </p> 
       <center> 
        <img src="http://img2.tuicool.com/aq6nUv.jpg!web" class="alignCenter"> 
       </center> 
       <p>To summarize, if you need a web application that looks and feels like native on a mobile device, you need PhoneJS - it contains everything required to build touch-enabled, native-looking web application. If you want to go further and access device features, like the contact list or camera, from JavaScript code, you will need Cordova aka PhoneGap. PhoneGap also lets you compile your web app into a native app package. If you don’t want to install an SDK for each platform you are targeting, you can use the PhoneGapBuild service to build your package. Finally, if you have DevExtreme, you can build packages right inside Visual Studio.</p>
