         <article id="wikiArticle"> 
          <p><strong>Base64</strong> is a group of similar <a href="https://en.wikipedia.org/wiki/Binary-to-text_encoding" class="external">binary-to-text encoding</a> schemes that represent binary data in an ASCII string format by translating it into a radix-64 representation. The term&nbsp;<em>Base64</em> originates from a specific <a href="https://en.wikipedia.org/wiki/MIME#Content-Transfer-Encoding" class="external">MIME content transfer encoding</a>.</p> 
          <p>Base64 encoding schemes are commonly used when there is a need to encode binary data that needs to be stored and transferred over media that are designed to deal with textual data. This is to ensure that the data remain intact without modification during transport. Base64 is commonly used in a number of applications including email via <a href="https://en.wikipedia.org/wiki/MIME" class="external">MIME</a>, and storing complex data in <a href="/en-US/docs/XML">XML</a>.</p> 
          <p>In JavaScript there are two functions respectively for decoding and encoding <em>base64</em> strings:</p> 
          <ul> 
           <li><a title="The documentation about this has not yet been written; please consider contributing!" href="/en-US/docs/Web/API/WindowBase64/atob"><code>atob()</code></a></li> 
           <li><a title="The documentation about this has not yet been written; please consider contributing!" href="/en-US/docs/Web/API/WindowBase64/btoa"><code>btoa()</code></a></li> 
          </ul> 
          <p>The <code>atob()</code> function decodes a string of data which has been encoded using base-64 encoding. Conversely, the <code>btoa()</code> function creates a base-64 encoded ASCII string from a "string" of binary data.</p> 
          <p>Both <code>atob()</code> and <code>btoa()</code> work on strings. If you want to work on <a href="/en-US/docs/Web/API/ArrayBuffer"><code>ArrayBuffers</code></a>, please, read <a href="#Solution_2__rewrite_the_DOMs_atob()_and_btoa()_using_JavaScript's_TypedArrays_and_UTF-8">this paragraph</a>.</p>

              <dl> 
               <dt>

               </dt> 
               <dd>
                <small><code>data</code> URIs, defined by <a title="http://tools.ietf.org/html/rfc2397" href="http://tools.ietf.org/html/rfc2397" class="external">RFC 2397</a>, allow content creators to embed small files inline in documents.</small>
               </dd>

                <a title="https://en.wikipedia.org/wiki/Base64" href="https://en.wikipedia.org/wiki/Base64" class="external">Base64</a>
               </dt> 
               <dd>
                <small>Wikipedia article about Base64 encoding.</small>
               </dd> 
               <dt>
                <a title="The documentation about this has not yet been written; please consider contributing!" href="/en-US/docs/Web/API/WindowBase64/atob"><code>atob()</code></a>
               </dt> 
               <dd>
                <small>Decodes a string of data which has been encoded using base-64 encoding.</small>
               </dd> 
               <dt>
                <a title="The documentation about this has not yet been written; please consider contributing!" href="/en-US/docs/Web/API/WindowBase64/btoa"><code>btoa()</code></a>
               </dt> 
               <dd>
                <small>Creates a base-64 encoded ASCII string from a "string" of binary data.</small>
               </dd> 
               <dt>
                <a href="#The_Unicode_Problem">The "Unicode Problem"</a>
               </dt> 
               <dd>
                <small>In most browsers, calling <code>btoa()</code> on a Unicode string will cause a <code>Character Out Of Range</code> exception. This paragraph shows some solutions.</small>
               </dd> 
               <dt>
                <a title="/en-US/docs/URIScheme" href="/en-US/docs/URIScheme">URIScheme</a>
               </dt> 
               <dd>
                <small>List of Mozilla supported URI schemes</small>
               </dd> 
               <dt>
                <a title="/en-US/docs/Web/JavaScript/Typed_arrays/StringView" href="/en-US/docs/Web/JavaScript/Typed_arrays/StringView"><code>StringView</code></a>
               </dt> 
               <dd>
                In this article is published a library of ours whose aims are: 
                <ul> 
                 <li>creating a <a title="http://en.wikipedia.org/wiki/C_%28programming_language%29" href="http://en.wikipedia.org/wiki/C_%28programming_language%29" class="external">C</a>-like interface for strings (i.e. array of characters codes <a title="/en-US/docs/Web/JavaScript/Typed_arrays/ArrayBufferView" href="/en-US/docs/Web/JavaScript/Typed_arrays/ArrayBufferView"> <code>ArrayBufferView</code></a> in JavaScript) based upon the JavaScript <a title="/en-US/docs/Web/JavaScript/Typed_arrays/ArrayBuffer" href="/en-US/docs/Web/JavaScript/Typed_arrays/ArrayBuffer"><code>ArrayBuffer</code></a> interface,</li> 
                 <li>creating a collection of methods for such string-like objects (since now: <code>stringView</code>s) which work <strong>strictly on array of numbers</strong> rather than on immutable JavaScript strings,</li> 
                 <li>working with other Unicode encodings, different from default JavaScript's UTF-16 <a title="/en-US/docs/Web/API/DOMString" href="/en-US/docs/Web/API/DOMString"><code>DOMString</code></a>s,</li> 
                </ul> 
               </dd>

          <h2 id="The_Unicode_Problem">The "Unicode Problem"</h2> 
          <p>Since <a title="/en-US/docs/Web/API/DOMString" href="/en-US/docs/Web/API/DOMString"><code>DOMString</code></a>s are 16-bit-encoded strings, in most browsers calling <code>window.btoa</code> on a Unicode string will cause a <code>Character Out Of Range</code> exception if a character exceeds the range of a 8-bit byte (0x00~0xFF). There are two possible methods to solve this problem:</p> 
          <ul> 
           <li>the first one is to escape the whole string&nbsp;(with UTF-8, see <a title="The encodeURIComponent()&nbsp;function encodes a Uniform Resource Identifier (URI) component by replacing each instance of certain characters by one, two, three, or four escape sequences representing the UTF-8 encoding of the character (will only be four escape sequences for characters composed of two &quot;surrogate&quot; characters)." href="/en-US/docs/Web/JavaScript/Reference/Global_Objects/encodeURIComponent"><code>encodeURIComponent</code></a>) and then encode it;</li> 
           <li>the second one is to convert the UTF-16 <a title="/en-US/docs/Web/API/DOMString" href="/en-US/docs/Web/API/DOMString"><code>DOMString</code></a> to an UTF-8 array of characters and then encode it.</li> 
          </ul> 
          <p>Here are the two possible methods.</p> 
          <h3 id="Solution_1__escaping_the_string_before_encoding_it">Solution #1&nbsp; escaping the string before encoding it</h3> 
          <pre class="brush:js">function b64EncodeUnicode(str) {
&nbsp;   // first we use encodeURIComponent to get percent-encoded UTF-8,
&nbsp;   // then we convert the percent encodings into raw bytes which
&nbsp;   // can be fed into btoa.
    return btoa(encodeURIComponent(str).replace(/%([0-9A-F]{2})/g,
&nbsp;       function toSolidBytes(match, p1) {
            return String.fromCharCode('0x' + p1);
    }));
}

b64EncodeUnicode('? à la mode'); // "4pyTIMOgIGxhIG1vZGU="
b64EncodeUnicode('\n'); // "Cg=="
</pre> 
          <p>To decode the Base64-encoded value back into a String:</p> 
          <pre class="brush: js">function b64DecodeUnicode(str) {
&nbsp;   // Going backwards: from bytestream, to percent-encoding, to original string.
    return decodeURIComponent(atob(str).split('').map(function(c) {
        return '%' + ('00' + c.charCodeAt(0).toString(16)).slice(-2);
    }).join(''));
}

b64DecodeUnicode('4pyTIMOgIGxhIG1vZGU='); // "? à la mode"
b64DecodeUnicode('Cg=='); // "\n"
</pre> 
          <p><a href="https://github.com/coolaj86/unibabel-js" class="external">Unibabel</a>&nbsp;is a library which includes common conversions using this strategy.</p> 
          <h3 id="Solution_2__rewrite_the_DOMs_atob()_and_btoa()_using_JavaScript's_TypedArrays_and_UTF-8">Solution #2&nbsp; rewrite&nbsp;the DOMs&nbsp;<code>atob()</code> and <code>btoa()</code> using JavaScript's&nbsp;<code>TypedArray</code>s and UTF-8</h3> 
          <p>Use a TextEncoder polyfill such as <a href="https://github.com/inexorabletash/text-encoding" class="external">TextEncoding</a>&nbsp;(also includes legacy windows, mac, and ISO encodings),&nbsp;<a href="https://github.com/coolaj86/TextEncoderLite" class="external">TextEncoderLite</a>, or <a href="https://github.com/feross/buffer" class="external">Buffer</a>&nbsp;and a Base64 polyfill such as <a href="https://github.com/beatgammit/base64-js" class="external">base64-js</a>.</p> 
          <p>The&nbsp;simplest, most light-weight solution would be to use&nbsp;<a href="https://github.com/coolaj86/TextEncoderLite" class="external">TextEncoderLite</a>&nbsp;and&nbsp;<a href="https://github.com/beatgammit/base64-js" class="external">base64-js</a>.</p> 
          <p>This function assumes using base64-js imported as minified <code>&lt;script type="text/javascript" src="base64js.min.js"/&gt;</code></p> 
          <pre class="brush: js">function base64EncodingUTF8(str) {
    var encoded = new TextEncoderLite('utf-8').encode(str);        
    var b64Encoded = base64js.fromByteArray(encoded);
    return b64Encoded;
}
</pre>
