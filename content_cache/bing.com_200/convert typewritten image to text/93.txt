             <div class="post-text" itemprop="text"> 
              <p>Install pdftk</p> 
              <pre><code>sudo apt-get install pdftk
</code></pre> 
              <p><strong>Pdftk</strong></p> 
              <p>If PDF is electronic paper, then pdftk is an electronic staple-remover, hole-punch, binder, secret-decoder-ring, and X-Ray-glasses. Pdftk is a simple tool for doing everyday things with PDF documents.</p> 
              <p>You can create pdf files from text or images with Libre Office then to stitch these togeter with other pdf files</p> 
              <pre><code>pdftk 1.pdf 2.pdf 3.pdf cat output 123.pdf
</code></pre> 
              <p>It can also </p> 
              <ul> 
               <li><p>Split PDF Pages into a New Document</p></li> 
               <li><p>Rotate PDF Pages or Documents</p></li> 
              </ul> 
              <p>and a lot more besides</p> 
              <p>More details here: <a href="http://www.ubuntugeek.com/list-of-pdf-editing-tools-for-ubuntu.html">Ubuntu Geek: List of PDF Editing tools</a> </p>

               <p>PDF Chain is a graphical user interface for the PDF Toolkit (PDFtk). The GUI supports all common features of the command line tool in a comfortable way.</p>

</code></pre>

               <p>You can reorder, rotate, and remove pages, export images from a document, edit the title, subject, author, and keywords, and combine documents via drag and drop.</p> 
              </blockquote>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>For multiple files inside a directory and its subdirectories with different extensions I couldn't find a neat answer, so here it is</p> 
              <pre><code>convert -quality 85 `find -type f -name '*.png' -or -name '*.jpg' | sort -V` output.pdf
</code></pre> 
              <p>I used <a href="https://en.wikipedia.org/wiki/Command_substitution" rel="nofollow noreferrer">command substitution</a> to pass the selected items returned by <code>find</code> command as an argument to <code>convert</code> command. Unfortunately <code>sort -n</code> didn't sort my files correctly so I tried <code>-V</code> option and it did the trick. Also make sure the name of your files and directories are in <a href="https://en.wikipedia.org/wiki/Natural_sort_order" rel="nofollow noreferrer">natural sort order</a> in advance. For example <code>dir1, dir2, dir3</code> not <code>dir1, dir_2, dir3</code>.</p>

             <div class="post-text" itemprop="text"> 
              <p>Try LaTeX with <code>pdflatex</code>.</p> 
              <p><strong>I had never used it before but it took me about 10 minutes to start making .PDFs with it</strong> and about 40 minutes to get them customized exactly as I wanted. I included the best formatting guides I found, at the end.</p> 
              <p><code>sudo apt-get install pdflatex &amp;&amp; sudo apt-get install texlive</code></p> 
              <p>Basically you create one <code>.tex</code> file - for example <code>hello.tex</code> - with the LaTeX language, then run <code>pdflatex hello.tex</code> on that file and it will generate the PDF. The basics of the language can be found here: <a href="http://www.maths.tcd.ie/~dwilkins/LaTeXPrimer/" rel="nofollow">http://www.maths.tcd.ie/~dwilkins/LaTeXPrimer/</a></p> 
              <hr> 
              <p>Here is a barebones example <code>.tex</code> file:</p> 
              <pre><code> \documentclass[a4paper,10pt]{article}

 \begin{document}

 {\footnotesize
 YOUR TEXT HERE

 YOUR TEXT HERE
 }

 \end{document}
</code></pre> 
              <hr> 
              <p><strong>Optional extra formatting:</strong></p> 
              <p>To add images: <a href="https://www.sharelatex.com/learn/Inserting_Images" rel="nofollow">https://www.sharelatex.com/learn/Inserting_Images</a></p> 
              <p>For different font sizes: <a href="https://www.sharelatex.com/learn/Font_sizes,_families,_and_styles" rel="nofollow">https://www.sharelatex.com/learn/Font_sizes,_families,_and_styles</a></p> 
              <p>For different fonts: <a href="https://www.sharelatex.com/learn/Font_typefaces" rel="nofollow">https://www.sharelatex.com/learn/Font_typefaces</a></p> 
              <p>To change page size and margins when using <code>pdflatex</code>: <code>\usepackage[pass,paperwidth=148mm,paperheight=210mm,margin=5mm]{geometry}</code></p>

             <div class="post-text" itemprop="text"> 
              <p>This is the solution i used to convert multiple TIFFs to PDFs.</p> 
              <p>I had to create more than 6.000 PDFs starting from 30.000 tiffs. <code>convert</code> estimate time: 6 to 7 hours.<br> I used <code>tiffcp</code> and <code>tiff2pdf</code>, they took few seconds.</p> 
              <pre><code>$ tiffcp 1.tiff 2.tiff ... multi.tiff
$ tiff2pdf multi.tiff &gt; final.pdf
</code></pre> 
              <p>This way is really fast because images are not converted, just packed.</p> 
              <p>Maybe there are some tiff formats that doesn't work so easily, for me it worked perfectly.</p> 
              <p>Hope it helps.</p>
