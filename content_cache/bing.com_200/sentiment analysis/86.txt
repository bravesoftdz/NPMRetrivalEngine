           <h1 name="d34b" id="d34b" class="graf graf--h3 graf--leading graf--title">Sentiment Analysis of Twitter Hashtags With&nbsp;Spark</h1>
           <h2 name="c4b7" id="c4b7" class="graf graf--h4 graf-after--h3 graf--subtitle">Revisited with PixieDust &amp; Jupyter Notebooks</h2>
           <h3 name="2bc9" id="2bc9" class="graf graf--h3 graf-after--h4">Why do it&nbsp;again?</h3>
           <p name="daea" id="daea" class="graf graf--p graf-after--h3">This is <em class="markup--em markup--p-em">yet another blog post</em> where I discuss the application I built for running sentiment analysis of Twitter content using <a href="http://spark.apache.org/" data-href="http://spark.apache.org/" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Apache Spark™</a> and <a href="https://console.ng.bluemix.net/catalog/services/tone-analyzer" data-href="https://console.ng.bluemix.net/catalog/services/tone-analyzer" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">Watson Tone Analyzer</a>. Before you quit reading, let me assure you that there is a good reason to revisit this code, and that you will hopefully learn something new.</p>
           <p name="a362" id="a362" class="graf graf--p graf-after--p">Let’s first recap the first two installments:</p>
           <ol class="postList">
            <li name="049b" id="049b" class="graf graf--li graf-after--p">In <a href="https://developer.ibm.com/clouddataservices/2015/10/06/sentiment-analysis-of-twitter-hashtags/" data-href="https://developer.ibm.com/clouddataservices/2015/10/06/sentiment-analysis-of-twitter-hashtags/" class="markup--anchor markup--li-anchor" rel="nofollow noopener" target="_blank">Part 1</a>, I built a Spark Streaming application in Scala that I invoked in a Scala Notebook to fetch live Twitter content. I then used a Python Notebook to build analytics on the data.</li>
            <li name="83c7" id="83c7" class="graf graf--li graf-after--li">In <a href="https://developer.ibm.com/clouddataservices/2016/01/15/real-time-sentiment-analysis-of-twitter-hashtags-with-spark/" data-href="https://developer.ibm.com/clouddataservices/2016/01/15/real-time-sentiment-analysis-of-twitter-hashtags-with-spark/" class="markup--anchor markup--li-anchor" rel="nofollow noopener" target="_blank">Part 2</a>, I tried to improve the application to update in real time. To do that, I ported the analytics in the Scala Spark Streaming piece, sending the results to a dashboard via <a href="https://console.ng.bluemix.net/catalog/services/message-hub" data-href="https://console.ng.bluemix.net/catalog/services/message-hub" class="markup--anchor markup--li-anchor" rel="nofollow noopener" target="_blank">MessageHub</a> events (based on <a href="https://kafka.apache.org/" data-href="https://kafka.apache.org/" class="markup--anchor markup--li-anchor" rel="nofollow noopener" target="_blank">Apache Kafka™</a>).             <li name="42d1" id="42d1" class="graf graf--li graf-after--p">The application was accessible to developers, but other users found it hard to deploy.            <h3 name="d56e" id="d56e" class="graf graf--h3 graf-after--li">Can we do&nbsp;better?</h3>
           <p name="8e23" id="8e23" class="graf graf--p graf-after--h3">For the third version of this app, I had two basic goals:</p>
           <ol class="postList">
            <li name="2a51" id="2a51" class="graf graf--li graf-after--p">Use only one flavor of data science notebook. Asking people to use a Scala Notebook first <em class="markup--em markup--li-em">and then</em> a Python Notebook created a lot of friction in previous versions of the app.</li>
            <li name="d9e3" id="d9e3" class="graf graf--li graf-after--li">No deployment and configuration changes should be required for the front-end.</li>
           </ol>
           <p name="bda9" id="bda9" class="graf graf--p graf-after--li">The user story should be simple: a developer, data scientist, or line-of-business user should be able to run the application end-to-end from within a single Python Notebook. <strong class="markup--strong markup--p-strong">Difficult? Yes. Impossible? No.</strong></p>
           <p name="a655" id="a655" class="graf graf--p graf-after--p">Now that I’ve piqued your interest, read on!</p>
           <h3 name="a4c0" id="a4c0" class="graf graf--h3 graf-after--p">PixieDust</h3>
           <p name="4c0e" id="4c0e" class="graf graf--p graf-after--h3">To achieve my goals I needed three capabilities:</p>
           <ol class="postList">
            <li name="b725" id="b725" class="graf graf--li graf-after--p">The ability to run Scala code from within a Python Notebook (the Spark Streaming application from Part 1 requires Scala).</li>
            <li name="b715" id="b715" class="graf graf--li graf-after--li">The ability to install third-party Java packages into a Python Notebook.</li>
            <li name="e95e" id="e95e" class="graf graf--li graf-after--li">The ability to run a fully functional UI from within a Python Notebook.</li>
           </ol>
           <p name="8e4b" id="8e4b" class="graf graf--p graf-after--li">Thankfully, PixieDust provides these three capabilities and more. You can learn more about PixieDust on GitHub: <a href="https://github.com/ibm-cds-labs/pixiedust" data-href="https://github.com/ibm-cds-labs/pixiedust" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">https://github.com/ibm-cds-labs/pixiedust</a>. There’s also an intro post: <a href="https://developer.ibm.com/clouddataservices/2016/10/11/pixiedust-magic-for-python-notebook/" data-href="https://developer.ibm.com/clouddataservices/2016/10/11/pixiedust-magic-for-python-notebook/" class="markup--anchor markup--p-anchor" rel="nofollow noopener" target="_blank">PixieDust: Magic for Your Python Notebook</a>.</p>

            You can easily install the PixieDust Python module in your notebook by using the following command:&nbsp;
            <code class="markup--code markup--blockquote-code">!pip install --user —-upgrade pixiedust</code>

           </blockquote>
           <p name="ee77" id="ee77" class="graf graf--p graf-after--blockquote">Let’s look at a few key points: the doRender method is called by the framework to process and render the visualization. The class is passed the following objects:</p>
           <ol class="postList">
            <li name="7b52" id="7b52" class="graf graf--li graf-after--p">self.entity: the entity containing the data being displayed (not used above, but available for use)</li>
            <li name="5f49" id="5f49" class="graf graf--li graf-after--li">self.options: dictionary of state variables</li>
            <li name="22ff" id="22ff" class="graf graf--li graf-after--li">self._addHTMLTemplate: helper method that takes a path to a <a href="http://jinja.pocoo.org/docs/dev/templates/" data-href="http://jinja.pocoo.org/docs/dev/templates/" class="markup--anchor markup--li-anchor" rel="nofollow noopener" target="_blank">jinja2</a> template; the template is guaranteed to be passed a few variables like entity, prefix, etc.</li>
            <li name="ffe6" id="ffe6" class="graf graf--li graf-after--li">self._addScriptElement: helper method that lets you insert a JavaScript file into the browser client</li>
           </ol>
           <blockquote name="b85b" id="b85b" class="graf graf--blockquote graf-after--li">

           <p name="64ce" id="64ce" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Step 5</strong>: Use our new application in a notebook. First, set the credentials again, this time for the PixiedustStreamingTwitter class:</p>
           <pre name="8d10" id="8d10" class="graf graf--pre graf-after--p"><code class="markup--code markup--pre-code">%%scala<br>val demo = com.ibm.cds.spark.samples.PixiedustStreamingTwitter<br>demo.setConfig("twitter4j.oauth.consumerKey",twitterConsumerKey)<br>demo.setConfig("twitter4j.oauth.consumerSecret",twitterConsumerSecret)<br>demo.setConfig("twitter4j.oauth.accessToken",twitterAccessToken)<br>demo.setConfig("twitter4j.oauth.accessTokenSecret",twitterAccessTokenSecret)<br>demo.setConfig("watson.tone.url","https://gateway.watsonplatform.net/tone-analyzer/api")<br>demo.setConfig("watson.tone.password",toneAnalyzerPassword)<br>demo.setConfig("watson.tone.username",toneAnalyzerUserName)<br>demo.setConfig("checkpointDir", System.getProperty("user.home") + "/pixiedust/ssc")</code></pre>

           <p name="83dd" id="83dd" class="graf graf--p graf-after--pre">You should see a dialog that lets you specify word filters in the top left corner.
