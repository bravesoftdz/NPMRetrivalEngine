       <h1 class="entry-title">Day 20: Stanford CoreNLP — Performing Sentiment Analysis of Twitter using Java</h1> 
       <time class="published" datetime="2013-11-17T00:00:00+00:00">November 17, 2013</time> 
       <p class="byline author vcard">By <a href="https://blog.openshift.com/author/shekhargulati/" rel="author" class="fn">Shekhar Gulati</a></p> 
       <script>

      <div class="entry-content"> 
       <p>Today for <a href="/learning-30-technologies-in-30-days-a-developer-challenge">my 30 day challenge</a>, I decided to learn how to use the Stanford CoreNLP Java API to perform <a href="http://nlp.stanford.edu/sentiment/index.html">sentiment analysis</a>. A few days ago, I also wrote about how you can do <a href="/day-9-textblob-finding-sentiments-in-text">sentiment analysis in Python</a> using TextBlob API. I have developed an application which gives you sentiments in the tweets for a given set of keywords. Let’s look at the application to understand what it does.</p> 
       <h2>Application</h2> 
       <p>The demo application is running on OpenShift <a href="http://sentiments-t20.rhcloud.com/" title="http://sentiments-t20.rhcloud.com/">http://sentiments-t20.rhcloud.com/</a>. It has two functionalities:</p> 
       <ol> 
        <li>The first functionality is that if you give it a list of twitter search terms it will show you sentiments in latest 20 tweets for the given search term. You have to enable this by checking the checkbox as shown below. The positive tweets are shown in green and negative tweets in red.<br> <img src="/wp-content/uploads/imported/SentimentsApp_First_Usecase.png" height="450" width="600"><p></p> </li>

       <h2>What is Stanford CoreNLP?</h2> 
       <p><a href="http://nlp.stanford.edu/software/corenlp.shtml">Stanford CoreNLP</a> is a Java natural language analysis library. Stanford CoreNLP integrates all our NLP tools, including the part-of-speech (POS) tagger, the named entity recognizer (NER), the parser, the coreference resolution system, and the sentiment analysis tools, and provides model files for analysis of English.</p>

       <h3>Search twitter for keywords</h3> 
       <p>Next we created a new class TwitterSearch which uses Twitter4J API to search twitter for keywords. The API requires twitter application configuration parameters. Instead of hard coding the values, we are using environment variables to get the values.</p> 
       <div class="geshifilter"> 
        <pre class="text geshifilter-text">import java.util.Collections;
import java.util.List;
&nbsp;
import twitter4j.Query;
import twitter4j.QueryResult;
import twitter4j.Status;
import twitter4j.Twitter;
import twitter4j.TwitterException;
import twitter4j.TwitterFactory;
import twitter4j.conf.ConfigurationBuilder;
&nbsp;
public class TwitterSearch {
&nbsp;
    public List&lt;Status&gt; search(String keyword) {
        ConfigurationBuilder cb = new ConfigurationBuilder();
        cb.setDebugEnabled(true).setOAuthConsumerKey(System.getenv("TWITTER_OAUTH_CONSUMER_KEY"))
                .setOAuthConsumerSecret(System.getenv("TWITTER_OAUTH_CONSUMER_SECRET"))
                .setOAuthAccessToken(System.getenv("TWITTER_OAUTH_ACCESS_TOKEN"))
                .setOAuthAccessTokenSecret(System.getenv("TWITTER_OAUTH_ACCESS_TOKEN_SECRET"));
        TwitterFactory tf = new TwitterFactory(cb.build());
        Twitter twitter = tf.getInstance();
        Query query = new Query(keyword + " -filter:retweets -filter:links -filter:replies -filter:images");
        query.setCount(20);
        query.setLocale("en");
        query.setLang("en");;
        try {
            QueryResult queryResult = twitter.search(query);
            return queryResult.getTweets();
        } catch (TwitterException e) {
            // ignore
            e.printStackTrace();
        }
        return Collections.emptyList();
&nbsp;
    }
&nbsp;
&nbsp;
}</pre> 
       </div> 
       <p>In the code shown above, we filter the twitter search results to make sure no retweet, or tweet with links, or tweet with images are returned. The reason for this is to make sure that we get tweets which have text.</p> 
       <h2>SentimentAnalyzer</h2> 
       <p>Next we created a class called SentimentAnalyzer which run sentiment analysis on a single tweet.</p> 
       <div class="geshifilter"> 
        <pre class="text geshifilter-text">public class SentimentAnalyzer {
&nbsp;
    public TweetWithSentiment findSentiment(String line) {
&nbsp;
        Properties props = new Properties();
        props.setProperty("annotators", "tokenize, ssplit, parse, sentiment");
        StanfordCoreNLP pipeline = new StanfordCoreNLP(props);
        int mainSentiment = 0;
        if (line != null &amp;&amp; line.length() &gt; 0) {
            int longest = 0;
            Annotation annotation = pipeline.process(line);
            for (CoreMap sentence : annotation.get(CoreAnnotations.SentencesAnnotation.class)) {
                Tree tree = sentence.get(SentimentCoreAnnotations.AnnotatedTree.class);
                int sentiment = RNNCoreAnnotations.getPredictedClass(tree);
                String partText = sentence.toString();
                if (partText.length() &gt; longest) {
                    mainSentiment = sentiment;
                    longest = partText.length();
                }
&nbsp;
            }
        }
        if (mainSentiment == 2 || mainSentiment &gt; 4 || mainSentiment &lt; 0) {
            return null;
        }
        TweetWithSentiment tweetWithSentiment = new TweetWithSentiment(line, toCss(mainSentiment));
        return tweetWithSentiment;
&nbsp;
    }
}</pre> 
       </div> 
       <p>We copied the <em>englishPCFG.ser.gz</em> and <em>sentiment.ser.gz</em> models to <em>src/main/resources/edu/stanford/nlp/models/lexparser</em> and <em>src/main/resources/edu/stanford/nlp/models/sentiment</em> folders.</p> 
       <h3>Created SentimentsResource</h3> 
       <p>Finally, we created the JAX-RS resource class.</p> 
       <div class="geshifilter"> 
        <pre class="text geshifilter-text">public class SentimentsResource {
&nbsp;
    @Inject
    private SentimentAnalyzer sentimentAnalyzer;
&nbsp;
    @Inject
    private TwitterSearch twitterSearch;
&nbsp;
    @GET
    @Produces(value = MediaType.APPLICATION_JSON)
    public List&lt;Result&gt; sentiments(@QueryParam("searchKeywords") String searchKeywords) {
        List&lt;Result&gt; results = new ArrayList&lt;&gt;();
        if (searchKeywords == null || searchKeywords.length() == 0) {
            return results;
        }
&nbsp;
        Set&lt;String&gt; keywords = new HashSet&lt;&gt;();
        for (String keyword : searchKeywords.split(",")) {
            keywords.add(keyword.trim().toLowerCase());
        }
        if (keywords.size() &gt; 3) {
            keywords = new HashSet&lt;&gt;(new ArrayList&lt;&gt;(keywords).subList(0, 3));
        }
        for (String keyword : keywords) {
            List&lt;Status&gt; statuses = twitterSearch.search(keyword);
            System.out.println("Found statuses ... " + statuses.size());
            List&lt;TweetWithSentiment&gt; sentiments = new ArrayList&lt;&gt;();
            for (Status status : statuses) {
                TweetWithSentiment tweetWithSentiment = sentimentAnalyzer.findSentiment(status.getText());
                if (tweetWithSentiment != null) {
                    sentiments.add(tweetWithSentiment);
                }
            }
&nbsp;
            Result result = new Result(keyword, sentiments);
            results.add(result);
        }
        return results;
    }
}</pre> 
       </div> 
       <p>This code does the following:</p> 
       <ol> 
        <li>Checks if searchkeywords is not null and not empty. Then it split the searchKeywords into an array. It only considers three search terms.</li> 
        <li>For every search term it finds the tweets and performs sentiment analysis.</li> 
        <li>Finally it returns the results list to the user.</li> 
       </ol>
