            <h1 class="entry-title ">Twitter sentiment analysis with R</h1> 
            <div class="entry-meta"> 
             <span class="posted-on"><i class="fa fa-calendar"></i> <a href="http://analyzecore.com/2014/04/28/twitter-sentiment-analysis/" rel="bookmark"><time class="entry-date published" datetime="2014-04-28T09:50:33+00:00">28/04/2014</time><time class="updated" datetime="2015-09-01T12:25:27+00:00">01/09/2015</time></a></span>
             <span class="byline"> <i class="fa fa-user"></i> <span class="author vcard"><a class="url fn n" href="http://analyzecore.com/author/admin/">admin</a></span></span> 
             <span class="cat-links"><i class="fa fa-folder-open-o"></i> <a href="http://analyzecore.com/category/r-language/" rel="category tag">R language</a>, <a href="http://analyzecore.com/category/sentiment-analysis/" rel="category tag">Sentiment Analysis</a>, <a href="http://analyzecore.com/category/twitter-analysis/" rel="category tag">Twitter analysis</a>, <a href="http://analyzecore.com/category/visualization/" rel="category tag">Visualization</a> </span>

           <div class="entry-content"> 
            <div class="ttr_start"></div>
            <p style="text-align: justify;">Recently I’ve designed&nbsp;a relatively simple code in R for analyzing Twitter posts content via calculating the number of positive, negative and neutral words. The idea of processing tweets is based on the presentation&nbsp;<a href="http://www.slideshare.net/ajayohri/twitter-analysis-by-kaify-rais" target="_blank">http://www.slideshare.net/ajayohri/twitter-analysis-by-kaify-rais</a>. The words in the tweet&nbsp;correspond with the words in&nbsp;dictionaries that you can find on the internet or create by yourself. It is also possible to edit these dictionaries. Really great work, but I’ve discovered some issue.</p> 
            <p style="text-align: justify;">There are some limitations in the API of Twitter. It depends on the total number of tweets you access&nbsp;via API, but usually you can get tweets for the last 7-8 days (not longer, and it can be 1-2 days only).&nbsp;The 7 to 8 days time limit doesn’t allow us to analyze historical trends.</p> 
            <p style="text-align: justify;">My idea is to create a storage file in order to accumulate&nbsp;historical data and bypass API’s limitations. If you extract tweets regularly, you would analyze the dynamics of sentiments with the chart like this one:</p> 
            <p style="text-align: justify;"><a href="https://i0.wp.com/analyzecore.com/wp-content/uploads/2014/04/plot.jpg"><img class="aligncenter size-full wp-image-888" src="https://i0.wp.com/analyzecore.com/wp-content/uploads/2014/04/plot.jpg?resize=648%2C439" alt="plot" srcset="https://i0.wp.com/analyzecore.com/wp-content/uploads/2014/04/plot.jpg?w=3695 3695w, https://i0.wp.com/analyzecore.com/wp-content/uploads/2014/04/plot.jpg?resize=300%2C203 300w, https://i0.wp.com/analyzecore.com/wp-content/uploads/2014/04/plot.jpg?resize=1024%2C693 1024w, https://i0.wp.com/analyzecore.com/wp-content/uploads/2014/04/plot.jpg?w=1296 1296w, https://i0.wp.com/analyzecore.com/wp-content/uploads/2014/04/plot.jpg?w=1944 1944w" sizes="(max-width: 648px) 100vw, 648px" data-recalc-dims="1"></a></p> 
            <p style="text-align: justify;">Furthermore, this algorithm includes a&nbsp;function that allows you to extract quite a few keywords that you are interested in. The process can be repeated several times a day and data set for each keyword will be saved separatly. It can helpful, for example, for doing competitors analysis.</p> 
            <p style="text-align: justify;">Let’s start. We need to create Twitter Application&nbsp;(<a href="https://apps.twitter.com/" target="_blank">https://apps.twitter.com/</a>) in order to have an access to Twitter’s API. Then&nbsp;we will get Consumer Key and&nbsp;Consumer Secret.</p> 
            <pre><span style="color: #339966;">#connect all libraries</span>
 <span style="color: #000000;">library(twitteR)</span>
 <span style="color: #000000;">library(ROAuth)</span>
 <span style="color: #000000;">library(plyr)</span>
 <span style="color: #000000;">library(dplyr)</span>
 <span style="color: #000000;">library(stringr)</span>
 <span style="color: #000000;">library(ggplot2)</span></pre> 
            <pre><span style="color: #339966;">#connect to&nbsp;API</span>
 <span style="color: #000000;">download.file(url='http://curl.haxx.se/ca/cacert.pem', destfile='cacert.pem')</span>
 <span style="color: #000000;">reqURL &lt;- 'https://api.twitter.com/oauth/request_token'</span>
 <span style="color: #000000;">accessURL &lt;- 'https://api.twitter.com/oauth/access_token'</span>
 <span style="color: #000000;">authURL &lt;- 'https://api.twitter.com/oauth/authorize'</span>
 <span style="color: #000000;">consumerKey &lt;- '____________'</span><span style="color: #339966;">&nbsp;#put the Consumer Key from Twitter Application</span>
 <span style="color: #000000;">consumerSecret &lt;- '______________'&nbsp;</span>&nbsp;<span style="color: #339966;">#put the Consumer Secret&nbsp;from Twitter Application</span>
 <span style="color: #000000;">Cred &lt;- OAuthFactory$new(consumerKey=consumerKey,</span>
 <span style="color: #000000;">&nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; consumerSecret=consumerSecret,</span>
 <span style="color: #000000;">&nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; requestURL=reqURL,</span>
 <span style="color: #000000;">&nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; accessURL=accessURL,</span>
 <span style="color: #000000;">&nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; authURL=authURL)</span>
 <span style="color: #000000;">Cred$handshake(cainfo = system.file('CurlSSL', 'cacert.pem', package = 'RCurl'))</span>&nbsp;<span style="color: #339966;">#There is URL in Console. You need to go to, get code and enter it in Console</span></pre> 
            <pre><span style="color: #000000;">save(Cred, file='twitter authentication.Rdata')</span>
 <span style="color: #000000;">load('twitter authentication.Rdata')</span>&nbsp;<span style="color: #339966;">#Once you launched the code first time, you can start from this line in the future (libraries should be connected)</span>
 <span style="color: #000000;">registerTwitterOAuth(Cred)</span></pre> 
            <pre><span style="color: #339966;">#the function for extracting and analyzing tweets</span>
 <span style="color: #000000;">search &lt;- function(searchterm)</span>
 <span style="color: #000000;">{</span>
 <span style="color: #339966;">#extact tweets and create storage file</span></pre> 
            <pre><span style="color: #000000;">list &lt;- searchTwitter(searchterm, cainfo='cacert.pem', n=1500)</span>
 <span style="color: #000000;">df &lt;- twListToDF(list)</span>
 <span style="color: #000000;">df &lt;- df[, order(names(df))]</span>
 <span style="color: #000000;">df$created &lt;- strftime(df$created, '%Y-%m-%d')</span>
 <span style="color: #000000;">if (file.exists(paste(searchterm, '_stack.csv'))==FALSE) write.csv(df, file=paste(searchterm, '_stack.csv'), row.names=F)</span></pre> 
            <pre><span style="color: #339966;">#merge the last extraction with storage file and remove duplicates</span>
 <span style="color: #000000;">stack &lt;- read.csv(file=paste(searchterm, '_stack.csv'))</span>
 <span style="color: #000000;">stack &lt;- rbind(stack, df)</span>
 <span style="color: #000000;">stack &lt;- subset(stack, !duplicated(stack$text))</span>
 <span style="color: #000000;">write.csv(stack, file=paste(searchterm, '_stack.csv'), row.names=F)</span></pre> 
            <pre><span style="color: #339966;">#tweets evaluation function</span>
 <span style="color: #000000;">score.sentiment &lt;- function(sentences, pos.words, neg.words, .progress='none')</span>
 <span style="color: #000000;">{</span>
 <span style="color: #000000;">require(plyr)</span>
 <span style="color: #000000;">require(stringr)</span>
 <span style="color: #000000;">scores &lt;- laply(sentences, function(sentence, pos.words, neg.words){</span>
 <span style="color: #000000;">sentence &lt;- gsub('[[:punct:]]', "", sentence)</span>
 <span style="color: #000000;">sentence &lt;- gsub('[[:cntrl:]]', "", sentence)</span>
 <span style="color: #000000;">sentence &lt;- gsub('\\d+', "", sentence)</span>
 <span style="color: #000000;">sentence &lt;- tolower(sentence)</span>
 <span style="color: #000000;">word.list &lt;- str_split(sentence, '\\s+')</span>
 <span style="color: #000000;">words &lt;- unlist(word.list)</span>
 <span style="color: #000000;">pos.matches &lt;- match(words, pos.words)</span>
 <span style="color: #000000;">neg.matches &lt;- match(words, neg.words)</span>
 <span style="color: #000000;">pos.matches &lt;- !is.na(pos.matches)</span>
 <span style="color: #000000;">neg.matches &lt;- !is.na(neg.matches)</span>
 <span style="color: #000000;">score &lt;- sum(pos.matches) - sum(neg.matches)</span>
 <span style="color: #000000;">return(score)</span>
 <span style="color: #000000;">}, pos.words, neg.words, .progress=.progress)</span>
 <span style="color: #000000;">scores.df &lt;- data.frame(score=scores, text=sentences)</span>
 <span style="color: #000000;">return(scores.df)</span>
 <span style="color: #000000;">}</span></pre> 
            <pre><span style="color: #000000;">pos &lt;- scan('C:/___________/positive-words.txt', what='character', comment.char=';')</span>&nbsp;<span style="color: #339966;">#folder with positive dictionary</span>
 <span style="color: #000000;">neg &lt;- scan('C:/___________/negative-words.txt', what='character', comment.char=';')</span>&nbsp;<span style="color: #339966;">#folder with negative dictionary</span>
 <span style="color: #000000;">pos.words &lt;- c(pos, 'upgrade')</span>
 <span style="color: #000000;">neg.words &lt;- c(neg, 'wtf', 'wait', 'waiting', 'epicfail')</span></pre> 
            <pre><span style="color: #000000;">Dataset &lt;- stack</span>
 <span style="color: #000000;">Dataset$text &lt;- as.factor(Dataset$text)</span>
 <span style="color: #000000;">scores &lt;- score.sentiment(Dataset$text, pos.words, neg.words, .progress='text')</span>
 <span style="color: #000000;">write.csv(scores, file=paste(searchterm, '_scores.csv'), row.names=TRUE)</span>&nbsp;<span style="color: #339966;">#save evaluation results</span></pre> 
            <pre><span style="color: #339966;">#total score calculation: positive&nbsp;/ negative&nbsp;/&nbsp;neutral</span>
 <span style="color: #000000;">stat &lt;- scores</span>
 <span style="color: #000000;">stat$created &lt;- stack$created</span>
 <span style="color: #000000;">stat$created &lt;- as.Date(stat$created)</span>
 <span style="color: #000000;">stat &lt;- mutate(stat, tweet=ifelse(stat$score &gt; 0, 'positive', ifelse(stat$score &lt; 0, 'negative', 'neutral')))</span>
 <span style="color: #000000;">by.tweet &lt;- group_by(stat, tweet, created)</span>
 <span style="color: #000000;">by.tweet &lt;- summarise(by.tweet, number=n())</span>
 <span style="color: #000000;">write.csv(by.tweet, file=paste(searchterm, '_opin.csv'), row.names=TRUE)</span></pre> 
            <pre><span style="color: #339966;">#chart</span>
 <span style="color: #000000;">ggplot(by.tweet, aes(created, number)) + geom_line(aes(group=tweet, color=tweet), size=2) +</span>
 <span style="color: #000000;">geom_point(aes(group=tweet, color=tweet), size=4) +</span>
 <span style="color: #000000;">theme(text = element_text(size=18), axis.text.x = element_text(angle=90, vjust=1)) +</span>
 <span style="color: #000000;">#stat_summary(fun.y = 'sum', fun.ymin='sum', fun.ymax='sum', colour = 'yellow', size=2, geom = 'line') +</span>
 <span style="color: #000000;">ggtitle(searchterm)</span></pre> 
            <pre><span style="color: #000000;">ggsave(file=paste(searchterm, '_plot.jpeg'))</span></pre> 
            <pre><span style="color: #000000;">}</span></pre> 
            <pre><span style="color: #000000;">search("______")</span>&nbsp;<span style="color: #339966;">#enter keyword</span></pre> 
            <p>Finally we will get four files:</p> 
            <ul> 
             <li>storage file with initial data,</li> 
             <li>file with tweets rating ,</li> 
             <li>file with the number of tweets of each type (positive / negative / neutral) as of date,</li>
