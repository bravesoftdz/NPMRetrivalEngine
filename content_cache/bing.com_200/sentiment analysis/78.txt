        <h1 class="post_title">Sentiment analysis with machine learning in R</h1> 
        <div class="author-date-comment">
         <span class="author_name"><i class="fa fa-clock-o fa-fw" aria-hidden="true"></i> Published on January 10, 2016</span>
         <span class="comment_page"><i class="fa fa-commenting-o fa-fw" aria-hidden="true"></i> <a href="https://datascienceplus.com/sentiment-analysis-with-machine-learning-in-r/#comments"><span class="dsq-postid" data-dsqidentifier="2730 http://datascienceplus.com/?p=2730">16 Comments</span></a></span> 
         <div class="a2a_kit a2a_default_style"> 
          <a class="a2a_button_facebook"></a> 
          <a class="a2a_button_twitter"></a> 
          <a class="a2a_button_google_plus"></a> 
          <a class="a2a_button_email"></a> 
          <a class="a2a_dd" href="https://www.addtoany.com/share"></a> 
         </div> 
         <script async src="https://static.addtoany.com/menu/page.js"></script>
        </div> 
        <div class="entry-single"> 
         <p>Machine learning makes sentiment analysis more convenient. This post would introduce how to do sentiment analysis with machine learning using R. In the landscape of R, the <strong>sentiment</strong> R package and the more general text mining package have been well developed by <a rel="nofollow" href="https://github.com/timjurka" target="_blank">Timothy P. Jurka</a>. You can check out the <a rel="nofollow" href="https://github.com/timjurka/sentiment" target="_blank">sentiment package</a> and the fantastic RTextTools package. Actually, Timothy also writes an <a rel="nofollow" href="https://github.com/timjurka/maxent" target="_blank">maxent package</a> for low-memory multinomial logistic regression (also known as maximum entropy). </p> 
         <p>However, the naive bayes method is not included into RTextTools. The e1071 package did a good job of implementing the naive bayes method. e1071 is a course of the Department of Statistics (e1071), TU Wien. Its primary developer is <a rel="nofollow" href="http://www.technikum-wien.at/fh/institute/wirtschaftsinformatik/team/meyer/" target="_blank">David Meyer</a>.</p> 
         <p>It is still necessary to learn more about <strong>text analysis</strong>. Text analysis in R has been well recognized (see <a rel="nofollow" href="http://cran.r-project.org/web/views/NaturalLanguageProcessing.html" target="_blank">the R views on natural language processing</a>). Part of the success belongs to the <a rel="nofollow" href="http://cran.r-project.org/web/packages/tm/index.html" target="_blank">tm package</a>: A framework for text mining applications within R. It did a good job for text cleaning (stemming, delete the stopwords, etc) and transforming texts to document-term matrix (dtm). There is <a rel="nofollow" href="http://www.jstatsoft.org/v25/i05/paper" target="_blank">one paper</a> about it. As you know the most important part of text analysis is to get the feature vectors for each document. The word feature is the most important one. Of course, you can also extend the <strong>unigram</strong> word features to <strong>bigram and trigram</strong>, and so on to <strong>n-grams</strong>. However, here for our simple case, we stick to the unigram word features. </p> 
         <p>Note that it’s easy to use <code>ngrams</code> in R. In the past, the package of <code>Rweka</code> supplies functions to do it, check <a rel="nofollow" href="http://stackoverflow.com/questions/8161167/what-algorithm-i-need-to-find-n-grams">this example</a>. Now, you can set the <code>ngramLength</code> in the function of create_matrix using <code>RTextTools</code>.</p> 
         <p>The first step is to read data:</p> 
         <pre>library(RTextTools)
library(e1071)

pos_tweets =  rbind(
  c('I love this car', 'positive'),
  c('This view is amazing', 'positive'),
  c('I feel great this morning', 'positive'),
  c('I am so excited about the concert', 'positive'),
  c('He is my best friend', 'positive')
)

neg_tweets = rbind(
  c('I do not like this car', 'negative'),
  c('This view is horrible', 'negative'),
  c('I feel tired this morning', 'negative'),
  c('I am not looking forward to the concert', 'negative'),
  c('He is my enemy', 'negative')
)

test_tweets = rbind(
  c('feel happy this morning', 'positive'),
  c('larry friend', 'positive'),
  c('not like that man', 'negative'),
  c('house not great', 'negative'),
  c('your song annoying', 'negative')
)

tweets = rbind(pos_tweets, neg_tweets, test_tweets)
</pre> 
         <p>Then we can build the document-term matrix: </p> 
         <pre># build dtm
matrix= create_matrix(tweets[,1], language="english", 
                      removeStopwords=FALSE, removeNumbers=TRUE, 
                      stemWords=FALSE) </pre> 
         <p>Now, we can train the naive Bayes model with the training set. Note that, e1071 asks the response variable to be numeric or factor. Thus, we convert characters to factors here. This is a little trick.</p> 
         <pre># train the model
mat = as.matrix(matrix)
classifier = naiveBayes(mat[1:10,], as.factor(tweets[1:10,2]) )</pre> 
         <p>Now we can step further to test the accuracy.</p> 
         <pre># test the validity
predicted = predict(classifier, mat[11:15,]); predicted
table(tweets[11:15, 2], predicted)
recall_accuracy(tweets[11:15, 2], predicted)</pre> 
         <p>Apparently, the result is the same with Python (compare it with the results in <a rel="nofollow" href="http://chengjun.github.io/en/2012/03/sentiment-analysi-with-python/" target="_blank">an another post</a>). </p> 
         <h2>How about the other machine learning methods?</h2> 
         <p>As I mentioned, we can do it using <code>RTextTools</code>. Let’s rock!</p> 
         <p>First, to specify our data: </p> 
         <pre># build the data to specify response variable, training set, testing set.
container = create_container(matrix, as.numeric(as.factor(tweets[,2])),
                             trainSize=1:10, testSize=11:15,virgin=FALSE)</pre> 
         <p>Second, to train the model with multiple machine learning algorithms: </p> 
         <pre>models = train_models(container, algorithms=c("MAXENT" , "SVM", "RF", "BAGGING", "TREE"))</pre> 
         <p>Now, we can classify the testing set using the trained models. </p> 
         <pre>results = classify_models(container, models)</pre> 
         <p>How about the accuracy?</p> 
         <pre># accuracy table
table(as.numeric(as.factor(tweets[11:15, 2])), results[,"FORESTS_LABEL"])
table(as.numeric(as.factor(tweets[11:15, 2])), results[,"MAXENTROPY_LABEL"])

# recall accuracy
recall_accuracy(as.numeric(as.factor(tweets[11:15, 2])), results[,"FORESTS_LABEL"])
recall_accuracy(as.numeric(as.factor(tweets[11:15, 2])), results[,"MAXENTROPY_LABEL"])
recall_accuracy(as.numeric(as.factor(tweets[11:15, 2])), results[,"TREE_LABEL"])
recall_accuracy(as.numeric(as.factor(tweets[11:15, 2])), results[,"BAGGING_LABEL"])
recall_accuracy(as.numeric(as.factor(tweets[11:15, 2])), results[,"SVM_LABEL"])</pre> 
         <p>To summarize the results (especially the validity) in a formal way:</p> 
         <pre># model summary
analytics = create_analytics(container, results)
summary(analytics)
head(<a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="1879767974616c717b6b587c777b6d757d766c">[email&nbsp;protected]</a>_summary)
<a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="a2c3ccc3cedbd6cbc1d1e2c7ccd1c7cfc0cec7">[email&nbsp;protected]</a>_summar</pre> 
         <p>To cross validate the results:</p> 
         <pre>N=4
set.seed(2014)
cross_validate(container,N,"MAXENT")
cross_validate(container,N,"TREE")
cross_validate(container,N,"SVM")
cross_validate(container,N,"RF")</pre> 
         <p>The results can be found on <a rel="nofollow" href="http://rpubs.com/chengjun/sentiment" target="_blank">my Rpub page</a>. It seems that maxent reached the same recall accuracy as naive Bayes. The other methods even did a worse job. This is understandable, since we have only a very small data set. To enlarge the training set, we can get a much better results for sentiment analysis of tweets using more sophisticated methods. I will show the results with anther example. </p> 
         <h2>Sentiment analysis for tweets</h2> 
         <p>The data comes from <a rel="nofollow" href="https://github.com/victorneo/Twitter-Sentimental-Analysis" target="_blank">victorneo</a>. victorneo shows how to do sentiment analysis for tweets using Python. Here, I will demonstrate how to do it in R. </p> 
         <p>Read data:</p> 
         <pre>###################
"load data"
###################
setwd("D:/Twitter-Sentimental-Analysis-master/")
happy = readLines("./happy.txt")
sad = readLines("./sad.txt")
happy_test = readLines("./happy_test.txt")
sad_test = readLines("./sad_test.txt")

tweet = c(happy, sad)
tweet_test= c(happy_test, sad_test)
tweet_all = c(tweet, tweet_test)
sentiment = c(rep("happy", length(happy) ), 
              rep("sad", length(sad)))
sentiment_test = c(rep("happy", length(happy_test) ), 
                   rep("sad", length(sad_test)))
sentiment_all = as.factor(c(sentiment, sentiment_test))

library(RTextTools)</pre> 
         <p>First, try naive Bayes. </p> 
         <pre># naive bayes
mat= create_matrix(tweet_all, language="english", 
                   removeStopwords=FALSE, removeNumbers=TRUE, 
                   stemWords=FALSE, tm::weightTfIdf)

mat = as.matrix(mat)

classifier = naiveBayes(mat[1:160,], as.factor(sentiment_all[1:160]))
predicted = predict(classifier, mat[161:180,]); predicted

table(sentiment_test, predicted)
recall_accuracy(sentiment_test, predicted)</pre> 
         <p>Then, try the other methods:</p> 
         <pre># the other methods
mat= create_matrix(tweet_all, language="english", 
                   removeStopwords=FALSE, removeNumbers=TRUE, 
                   stemWords=FALSE, tm::weightTfIdf)

container = create_container(mat, as.numeric(sentiment_all),
                             trainSize=1:160, testSize=161:180,virgin=FALSE) #????removeSparseTerms

models = train_models(container, algorithms=c("MAXENT",
                                              "SVM",
                                              #"GLMNET", "BOOSTING", 
                                              "SLDA","BAGGING", 
                                              "RF", # "NNET", 
                                              "TREE" 
))

# test the model
results = classify_models(container, models)
table(as.numeric(as.numeric(sentiment_all[161:180])), results[,"FORESTS_LABEL"])
recall_accuracy(as.numeric(as.numeric(sentiment_all[161:180])), results[,"FORESTS_LABEL"])
</pre> 
         <p>Here we also want to get the formal test results, including:</p> 
         <ul>

          <li><strong><a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="b0d1ded1dcc9c4d9d3c3f0dcd1d2d5dc">[email&nbsp;protected]</a>_summary</strong>: Summary of label (e.g. Topic) accuracy</li> 
          <li><strong><a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="264748474a5f524f455566424945534b434852">[email&nbsp;protected]</a>_summary</strong>: Raw summary of all data and scoring</li> 
          <li><strong><a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="c6a7a8a7aabfb2afa5b586a3a8b5a3aba4aaa3">[email&nbsp;protected]</a>_summary</strong>: Summary of ensemble precision/coverage. Uses the n variable passed into create_analytics()</li> 
         </ul> 
         <p>Now let’s see the results:</p> 
         <pre># formal tests
analytics = create_analytics(container, results)
summary(analytics)

head(<a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="ef8e818e83969b868c9caf8e8388809d869b8782">[email&nbsp;protected]</a>_summary)
head(<a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="c0a1aea1acb9b4a9a3b380aca1a2a5ac">[email&nbsp;protected]</a>_summary)

N=3
