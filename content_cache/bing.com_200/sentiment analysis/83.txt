         <div class="date"> 
          <div class="bg">

          <br>
          <p>Its been long time, I wrote a post on <strong><a href="https://rhandbook.wordpress.com/category/sentiment-analysis/step-2-r-sentiment-analysis/">Sentiment Analysis</a></strong> without using Sentiment Package. In this post, I will use Sentiment Package developed by&nbsp;<strong>Timothy Jurka</strong>. You can download this package from <strong><a href="http://cran.r-project.org/src/contrib/Archive/sentiment/">here</a></strong>. Before installing the sentiment package, you need to install <strong>tm</strong> and <strong>Rstem</strong> from CRAN. Sentiment package has two functions that server out purpose.</p> 
          <h4><span style="text-decoration:underline;"><strong>classify_emotion</strong></span><br> This function helps us to analyze tweets / text and classify it in different types of emotion: anger, disgust, fear, joy, sadness, and surprise. The classification can be performed using two algorithms: one is a naive Bayes classi?er trained on Carlo Strapparava and Alessandro Valituttis emotions lexicon; the other one is just a simple voter procedure.</h4> 
          <h4><span style="text-decoration:underline;"><strong>classify_polarity</strong></span><br> In contrast to the classification of emotions, the classify_polarity function allows us to classify some text as positive or negative. In this case, the classification can be done by using a naive Bayes algorithm trained on Janyce Wiebes subjectivity lexicon; or by a simple voter algorithm.</h4> 
          <h4>Lets import the necessary packages for Sentiment Analysis.</h4> 
          <pre class="brush: r; title: ; notranslate" title="">
library(twitteR)
library(sentiment)
library (stringr)
library(ggplot2)
library(wordcloud)
library(RColorBrewer)

</pre> 
          <p>Lets do the analysis on <strong>ObamaInIndia</strong> as I did in my previous <a href="https://rhandbook.wordpress.com/category/sentiment-analysis/step-2-r-sentiment-analysis/">sentiment analysis</a> post.&nbsp;I am using that code&nbsp;to pull the tweets and data cleaning. Lets move forward from that and sentiment Analysis.</p> 
          <pre class="brush: r; title: ; notranslate" title="">

# classify emotion
class_emotion = classify_emotion(tweet_txt, algorithm="bayes", prior=1.0)
# get emotion best fit
emotion = class_emotion[,7]
# substitute NA's by "unknown"
emotion[is.na(emotion)] = "unknown"

# classify polarity
class_polarity= classify_polarity(tweet_txt, algorithm="bayes")
# get polarity best fit
polarity = class_polarity[,4]

</pre> 
          <p>We have now emotions and polarity based on our tweets. Lets create data frame&nbsp;from the tweets, emotions and polarity.</p> 
          <p># data frame with results<br> tweet_df = data.frame(text=tweet_txt, emotion=emotion,<br> polarity=polarity, stringsAsFactors=FALSE)</p> 
          <p># sort data frame<br> tweet_df = within(sent_df,<br> emotion &lt;- factor(emotion, levels=names(sort(table(emotion), decreasing=TRUE))))</p> 
          <p>Lets generate some plot based on above data set. Plot&nbsp;tweet distribution based on emotions.</p> 
          <pre class="brush: r; title: ; notranslate" title="">

ggplot(tweet_df, aes(x=emotion)) +
geom_bar(aes(y=..count.., fill=emotion))+xlab("Emotions Categories") + ylab("Tweet Count")+ggtitle("Sentiment Analysis of Tweets on Emotions")

</pre>

          <p>Separate the text by emotions and visualize the words with a comparison cloud.</p> 
          <pre class="brush: r; title: ; notranslate" title="">

emos = levels(factor(tweet_df$emotion))
nemo = length(emos)
emo.docs = rep("", nemo)
for (i in 1:nemo)
{
tmp = tweet_txt[emotion == emos[i]]
emo.docs[i] = paste(tmp, collapse=" ")
}

# remove stopwords
emo.docs = removeWords(emo.docs, stopwords("english"))
# create corpus
corpus = Corpus(VectorSource(emo.docs))
tdm = TermDocumentMatrix(corpus)
tdm = as.matrix(tdm)
colnames(tdm) = emos

# comparison word cloud
comparison.cloud(tdm, colors = brewer.pal(nemo, "Dark2"),
