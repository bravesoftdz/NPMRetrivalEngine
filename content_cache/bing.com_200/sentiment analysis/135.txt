         <h1>Twitter sentiment analysis in 30&nbsp;seconds.</h1> 
         <a href="https://dataissexy.wordpress.com/2012/02/01/twitter-sentiment-analysis-in-30-seconds/" class="post-permalink"> <p class="post-date"> <strong>01</strong> <em>Wednesday</em> <span>Feb 2012</span> </p> </a> 
         <div class="post-info clear-fix"> 
          <p> Posted <span class="by-author"> by <span class="author vcard"><a class="url fn n" href="https://dataissexy.wordpress.com/author/datasenitment/" title="View all posts by Jason Bell" rel="author">Jason Bell</a></span></span> in <a href="https://dataissexy.wordpress.com/category/uncategorized/" rel="category tag">Uncategorized</a> </p>

         <div class="post-entry"> 
          <p>The more I play with R the more I love it and what it can do. &nbsp;Better still is CRAN which is the repository of packages to add more functionality.</p> 
          <p>Okay we’re not going to create the new Radion6 or Repknight but boy it’s a lot of fun. Well, okay, *I* think it’s a lot of fun, I can’t speak for you wonderful lot. &nbsp;</p> 
          <p>For a much more detailed version of below you might want to look at <a href="http://www.inside-r.org/howto/mining-twitter-airline-consumer-sentiment">Jeffrey Breen’s post on Airline consumer sentiment</a>.</p> 
          <p><span style="font-size:large;"><strong>Getting the Basics down with R&nbsp;</strong></span></p> 
          <p>Load the twitteR package for reading tweets. &nbsp;If you don’t have the package installed you can install it through R with <strong>install.packages(‘twitteR’, dependencies=T)</strong></p> 
          <p><strong>&gt; library(twitteR)</strong></p> 
          <p>Load the plyr package.</p> 
          <p><strong>&gt; library(plyr)</strong></p> 
          <p>Pull some tweets in to R.</p> 
          <p><strong>&gt; retail.tweets = searchTwitter(“#retail”, n=1500)</strong></p> 
          <p>Load a positive word list.</p> 
          <p><strong>&gt; hu.iu.pos = scan(‘/work/repos/lexicon/positive-words.txt’, what=’character’, comment.char=’;’)</strong></p> 
          <p>Load a negative word list.</p> 
          <p><strong>&gt; hu.iu.neg = scan(‘/work/repos/lexicon/negative-words.txt’, what=’character’, comment.char=’;’)</strong></p> 
          <p>Load the script that does the magic, it’s pretty short all in all. And you can look <a href="http://dl.dropbox.com/u/2698672/sentiment.r">at it here</a>.</p> 
          <p><strong>&gt; source(‘sentiment.r’)</strong></p> 
          <p>Create a positive word list.</p> 
          <p><strong>&gt; pos.words = c(hu.iu.pos)</strong></p> 
          <p>Create a negative word list</p> 
          <p><strong>&gt; neg.words = c(hu.iu.neg)</strong></p> 
          <p>Pull all the text from the tweets.</p> 
          <p><strong>&gt; retail.text = laply(retail.tweets, function(t) t$getText())</strong></p> 
          <p>Work out a sentiment score for each tweet.</p> 
          <p><strong>&gt; retail.score = score.sentiment(retail.text, pos.words, neg.words)</strong></p> 
          <p>Display a graph of the sentiment scores.</p> 
          <p><strong>&gt; hist(retail.score$score)</strong></p> 
          <p><strong></strong></p>

           </div> </strong>
          <p><strong></strong></p> 
          <p><strong><span style="font-size:large;">Where to go from here?</span></strong></p> 
          <p><span style="font-size:large;"><span style="font-size:small;">From my point of view there’s an obvious web element that’s missing. &nbsp;So there needs to be some sort of bridge from the web to R running in the background. &nbsp;The rJava package acts as a handy gateway between the two.</span></span></p> 
          <p><span style="font-size:large;"><span style="font-size:small;">The plots could do with some tinkering to make them look colourful. There’s a ton of decent plot libraries out there for R that extend the basic range.</span></span></p>
