           <h1>Text Classification &amp; Sentiment Analysis tutorial / blog</h1> 
           <ul class="navigation byline"> 
            <li><a class="nolink">Posted by </a><a href="/profile/AhmetTaspinar">Ahmet Taspinar</a><a class="nolink"> on January 11, 2016 at 6:30am</a></li> 
            <li><a class="xg_sprite xg_sprite-view" href="http://www.datasciencecentral.com/profiles/blog/list?user=0cqadyjtpv6kx">View Blog</a></li> 
           </ul>

          <div class="postbody"> 
           <div class="xg_user_generated"> 
            <p><span>We all know that with Machine Learning you can automatically classify text documents or analyze its subjectivity. We've just released a guide that gives a brief introduction to Text Classification.&nbsp;</span></p> 
            <p><span>It cover the three most used classifiers; Naive Bayes, Maximum Entropy and Support Vector Machines and will give practical examples in the form of the sentiment analysis of book reviews.&nbsp;</span></p> 
            <p><span><a rel="nofollow" href="https://ataspinar.wordpress.com/2015/11/16/text-classification-and-sentiment-analysis/" target="_blank">Originally posted here</a>.&nbsp;</span></p> 
            <h1><strong>Introduction:</strong></h1> 
            <p>Natural Language Processing (NLP) is a vast area of Computer Science that is concerned with the interaction between Computers and Human Language<a rel="nofollow" href="https://ataspinar.wordpress.com/2015/11/16/text-classification-and-sentiment-analysis/#ML_literature">[1]</a>.<br> Within NLP many tasks are – or can be reformulated as – classification tasks. In classification tasks we are trying to produce a classification function which can give the correlation between a certain ‘feature’&nbsp;<img src="https://s0.wp.com/latex.php?latex=D&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="D" title="D" class="latex">and&nbsp;a class&nbsp;<img src="https://s0.wp.com/latex.php?latex=C&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="C" title="C" class="latex">.&nbsp;This Classifier first has to be trained with a training dataset, and then it can be used to actually classify documents. Training means that we have to determine &nbsp;its model parameters. If the set of training examples is chosen correctly, the Classifier should predict the class probabilities of the actual documents with a similar accuracy (as the training examples).</p> 
            <p>After construction, such a Classifier could for example tell us that document containing the words “Bose-Einstein condensate” should&nbsp;be categorized as a Physics article, while documents containing the words “Arbitrage” and “Hedging” should&nbsp;be categorized as a Finance article.<br> Another Classifier could tell us&nbsp;that mails starting with “Dear Customer/Guest/Sir” (instead of your name) and containing words like “Great opportunity” or “one-time offer” can be classified as spam.<br> Here we can already see two uses of classification models:&nbsp;<em>topic classification</em>&nbsp;and&nbsp;<em>spam filtering</em>. For these purposes a Classifiers work quiet well and perform better than most trained professionals.</p> 
            <p>A third usage of Classifiers&nbsp;is Sentiment Analysis. Here the purpose is to determine the subjective value of a text-document, i.e. how positive or negative is the content&nbsp;of a text document. Unfortunately, for this purpose these Classifiers&nbsp;fail to achieve the same accuracy.&nbsp;This is due to the subtleties of human language; sarcasm, irony, context interpretation, use of slang, cultural differences and the different ways in which opinion can be expressed (subjective vs comparative, explicit vs implicit).</p> 
            <p>In this blog I will discuss the theory behind three popular Classifiers (Naive Bayes, Maximum Entropy and Support Vector Machines) in the context of Sentiment Analysis<a rel="nofollow" href="https://ataspinar.wordpress.com/2015/11/16/text-classification-and-sentiment-analysis/#SA_literature">[2]</a>. In the next blog I will apply this gained knowledge to automatically deduce the sentiment&nbsp;of collected Amazon.com book reviews.</p> 
            <p>The contents of this blog-post is as follows:</p> 
            <ol> 
             <li><a rel="nofollow" href="https://ataspinar.wordpress.com/2015/11/16/text-classification-and-sentiment-analysis/#ch1">Basic concepts of text classification</a>:
              <ul> 
               <li>Tokenization</li> 
               <li>Word normalization</li> 
               <li>bag-of-words model</li> 
               <li>Classifier evaluation</li> 
              </ul> </li> 
             <li><a rel="nofollow" href="https://ataspinar.wordpress.com/2015/11/16/text-classification-and-sentiment-analysis/#ch2">Naieve&nbsp;Bayesian Classifier</a></li> 
             <li><a rel="nofollow" href="https://ataspinar.wordpress.com/2015/11/16/text-classification-and-sentiment-analysis/#ch3">Maximum Entropy Classifier</a></li> 
             <li><a rel="nofollow" href="https://ataspinar.wordpress.com/2015/11/16/text-classification-and-sentiment-analysis/#ch3">Support Vector Machines</a></li> 
             <li><a rel="nofollow" href="https://ataspinar.wordpress.com/2015/11/16/text-classification-and-sentiment-analysis/#ch4">What to Expect</a></li> 
            </ol> 
            <p><strong><span style="font-size: 2em;">1. Basic Concepts</span></strong></p> 
            <p><strong>Tokenization:<br></strong> <a rel="nofollow" href="https://en.wikipedia.org/wiki/Tokenization_(lexical_analysis)">Tokenization&nbsp;</a>is the name given to the process of chopping up sentences into smaller pieces (words or tokens). The segmentation into tokens can be done with decision trees, which contains information to correctly solve the issues you might encounter. Some of these issues you would have to consider are:<br> 1. The choice for the delimiter will for most cases be a whitespace (“We’re going to Barcelona” -&gt; [“We’re”, “going”, “to”, “Barcelona.”]), but what should you do when you come across words with a&nbsp;white space in them (“We’re going to The Hague.”-&gt;[“We’re”, “going”,”to”,”The”, “Hague”]).<br> 2. What should you do with punctuation marks? Although many tokenizers are geared towards throwing punctuation away, for Sentiment analysis a lot of valuable information could be deduced from them.&nbsp;<strong>!</strong>&nbsp;puts extra emphasis on the negative/positive sentiment of the sentence, while&nbsp;<strong>?</strong>&nbsp;can mean uncertainty (no sentiment).<br> 2. “, ‘ , [], () can mean that the words belong together and should be treated as a separate sentence. Same goes for words which are&nbsp;<strong>bold</strong>,<em>italic</em>,&nbsp;<u>underlined</u>, or&nbsp;<a rel="nofollow" href="http://thequietplaceproject.com/">inside a link</a>. If you also want to take these last elements into considerating, you should scrape the html code and not just the text.</p> 
            <p><strong>Word Normalization:<br></strong> <strong><a rel="nofollow" href="https://class.coursera.org/nlp/lecture/126">Word Normalization</a>&nbsp;</strong>is the reduction of each word to its base/stem form (by chopping of the affixes). While doing this, we should consider the following issues:<br> 1. Capital letters should be normalized to lowercase, unless it occurs in the middle of a sentence; this could indicate the name of a writer, place, brand etc.<br> 2.&nbsp;What should be done with the apostrophe (‘); “George’s phone” should obviously be tokenized as “George” and “phone”, but I’m, we’re, they’re should be translated as I am, we are and they are. To make it even more difficult; it can also be used as a quotation mark.<br> 3.&nbsp;Ambigious words like High-tech, The Hague, P.h.D., USA, U.S.A.,&nbsp;US and us.</p> 
            <p><strong>Bag-of-words</strong>:<br> After the text has been segmented into sentences, each sentence has been segmented into words, the words have been tokenized and normalized, we can make a simple&nbsp;<a rel="nofollow" href="https://en.wikipedia.org/wiki/Bag-of-words_model">bag-of-words</a>&nbsp;model of the text. In this bag-of-words representation you only take individual words into account and give each word a specific subjectivity score. This subjectivity score can be looked up in a sentiment lexicon<a rel="nofollow" href="https://ataspinar.wordpress.com/2015/11/16/text-classification-and-sentiment-analysis/#SL_literature">[7]</a>.&nbsp;If the total score is negative the text will be classified as negative and if its positive the text will be classified as positive.</p> 
            <p><strong>Classifier Evaluation:<br></strong> For determining the accuracy of a single Classifier, or comparing the results of different Classifier, the&nbsp;<a rel="nofollow" href="https://en.wikipedia.org/wiki/F1_score">F-score</a>&nbsp;is usually used. This F-score is given by</p> 
            <p><img src="https://s0.wp.com/latex.php?latex=+F+%3D+%5Cfrac%7B2pr%7D%7Bp%2Br%7D&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt=" F = \frac{2pr}{p+r}" title=" F = \frac{2pr}{p+r}" class="latex"></p> 
            <p>where&nbsp;<img src="https://s0.wp.com/latex.php?latex=p&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="p" title="p" class="latex">&nbsp;is the precision and&nbsp;<img src="https://s0.wp.com/latex.php?latex=r&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="r" title="r" class="latex">&nbsp;is the recall. The precision is the number of correctly classified examples divided by the total number of classified examples. The recall is the number of correctly classified examples divided by the actual number of examples in the training set.</p> 
            <h1><strong>2. Naive Bayes:</strong></h1> 
            <p>Naive Bayes&nbsp;<a rel="nofollow" href="https://ataspinar.wordpress.com/2015/11/16/text-classification-and-sentiment-analysis/#NB_literature">[3]</a>&nbsp;classifiers are studying the classification task from a Statistical point of view. The starting point is that the probability of a class&nbsp;<img src="https://s0.wp.com/latex.php?latex=C&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="C" title="C" class="latex">&nbsp;is given by the&nbsp;<a rel="nofollow" href="https://en.wikipedia.org/wiki/Posterior_probability">posterior probability</a>&nbsp;<img src="https://s0.wp.com/latex.php?latex=P%28C%7CD%29&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="P(C|D)" title="P(C|D)" class="latex">&nbsp;given a training document&nbsp;<img src="https://s0.wp.com/latex.php?latex=D&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="D" title="D" class="latex">. Here&nbsp;<img src="https://s0.wp.com/latex.php?latex=D&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="D" title="D" class="latex">&nbsp;refers to all of the text in the entire training set. It is given by&nbsp;<img src="https://s0.wp.com/latex.php?latex=D+%3D+%28+d_1%2C+d_2%2C+..%2C+d_n+%29+&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="D = ( d_1, d_2, .., d_n ) " title="D = ( d_1, d_2, .., d_n ) " class="latex">,&nbsp;where&nbsp;<img src="https://s0.wp.com/latex.php?latex=d_i&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="d_i" title="d_i" class="latex">&nbsp;is the&nbsp;<img src="https://s0.wp.com/latex.php?latex=i_%7Bth%7D&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="i_{th}" title="i_{th}" class="latex">&nbsp;attribute (word) of document&nbsp;<img src="https://s0.wp.com/latex.php?latex=D&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="D" title="D" class="latex">.</p> 
            <p>Using Bayes’ rule, this posterior probability can be rewritten as:</p> 
            <p align="center"><img src="https://s0.wp.com/latex.php?latex=P%28C%3Dc_i%7CD%29+%3D+%5Cfrac%7BP%28D%7CC%3Dc_i%29+%5Ccdot+P%28C%3Dc_i%29%7D%7BP%28D%29%7D+&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="P(C=c_i|D) = \frac{P(D|C=c_i) \cdot P(C=c_i)}{P(D)} " title="P(C=c_i|D) = \frac{P(D|C=c_i) \cdot P(C=c_i)}{P(D)} " class="latex"></p> 
            <p>Since the marginal probability&nbsp;<img src="https://s0.wp.com/latex.php?latex=P%28D%29&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="P(D)" title="P(D)" class="latex">&nbsp;is equal for all classes, it can be disregarded and the equation becomes:</p> 
            <p><img src="https://s0.wp.com/latex.php?latex=P%28C%3Dc_i%7CD%29+%3D+P%28D%7CC%3Dc_i%29+%5Ccdot+P%28C%3Dc_i%29+&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="P(C=c_i|D) = P(D|C=c_i) \cdot P(C=c_i) " title="P(C=c_i|D) = P(D|C=c_i) \cdot P(C=c_i) " class="latex"></p> 
            <p>The document&nbsp;<img src="https://s0.wp.com/latex.php?latex=D&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="D" title="D" class="latex">&nbsp;belongs to the class&nbsp;<img src="https://s0.wp.com/latex.php?latex=C&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="C" title="C" class="latex">&nbsp;which maximizes this probability, so:</p> 
            <p align="center"><img src="https://s0.wp.com/latex.php?latex=C_%7BNB%7D+%3D+argmax+P%28D%7CC%29+%5Ccdot+P%28C%29&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="C_{NB} = argmax P(D|C) \cdot P(C)" title="C_{NB} = argmax P(D|C) \cdot P(C)" class="latex"></p> 
            <p align="center"><img src="https://s0.wp.com/latex.php?latex=C_%7BNB%7D+%3D+argmax+P%28d_1%2C+d_2%2C+..%2C+d_n+%7C+C%29+%5Ccdot+P%28C%29&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="C_{NB} = argmax P(d_1, d_2, .., d_n | C) \cdot P(C)" title="C_{NB} = argmax P(d_1, d_2, .., d_n | C) \cdot P(C)" class="latex"></p> 
            <p>Assuming&nbsp;<a rel="nofollow" href="https://en.wikipedia.org/wiki/Conditional_independence">conditional independence</a>&nbsp;of the words&nbsp;<img src="https://s0.wp.com/latex.php?latex=d_i&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="d_i" title="d_i" class="latex">, this equation simplifies to:</p> 
            <p align="center"><img src="https://s0.wp.com/latex.php?latex=C_%7BNB%7D+%3D+argmax+P%28d_1%7CC%29+%5Ccdot+P%28d_2%7CC%29+%5Ccdot+%5Ccdot+%5Ccdot+P%28d_n%7CC%29+%5Ccdot+P%28C%29&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="C_{NB} = argmax P(d_1|C) \cdot P(d_2|C) \cdot \cdot \cdot P(d_n|C) \cdot P(C)" title="C_{NB} = argmax P(d_1|C) \cdot P(d_2|C) \cdot \cdot \cdot P(d_n|C) \cdot P(C)" class="latex"></p> 
            <p align="center"><img src="https://s0.wp.com/latex.php?latex=C_%7BNB%7D+%3D+argmax+P%28C%29+%5Ccdot+%5Cprod_i+P%28d_i%7CC%29&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="C_{NB} = argmax P(C) \cdot \prod_i P(d_i|C)" title="C_{NB} = argmax P(C) \cdot \prod_i P(d_i|C)" class="latex"></p> 
            <p>Here&nbsp;<img src="https://s0.wp.com/latex.php?latex=P%28d_i+%7C+C%29&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="P(d_i | C)" title="P(d_i | C)" class="latex">&nbsp;is the conditional probability that word i belongs to class&nbsp;<img src="https://s0.wp.com/latex.php?latex=C&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="C" title="C" class="latex">. For the purpose of text classification, this probability can simply be calculated by calculating the frequency of word i in class C relative to the total number of words in class C.</p> 
            <p align="center"><img src="https://s0.wp.com/latex.php?latex=P%28d_i+%7C+C%29+%3D+%5Cfrac%7Bcount%28d_i%2C+C%29%7D%7B%5Csum_i+count%28d_i%2C+C%29%7D&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="P(d_i | C) = \frac{count(d_i, C)}{\sum_i count(d_i, C)}" title="P(d_i | C) = \frac{count(d_i, C)}{\sum_i count(d_i, C)}" class="latex"></p> 
            <p>We have seen that we need to multiply the class probability with all of the prior-probabilities of the individual&nbsp;words belonging to that class. The question then is, how do we know what the prior-probabilities of the words are? &nbsp;Here we need to remember that this is a supervised machine learning algorithm: we can estimate the prior-probabilities with&nbsp;a training set with documents that are already labeled with their classes. With this training set we can train the model and obtain values for the prior probabilities. This trained model can then be used for classifying unlabeled documents.</p>

            <h1><strong>3. Maximum Entropy:</strong></h1> 
            <p>The principle behind Maximum Entropy&nbsp;<a rel="nofollow" href="https://ataspinar.wordpress.com/2015/11/16/text-classification-and-sentiment-analysis/#ME_literature">[4]</a>&nbsp;is that the correct distribution is the one that maximizes the Entropy / uncertainty and still meets the constraints which are set by the ‘evidence’.</p> 
            <p>Let me explain this a bit more. In Information Theory, the word<a rel="nofollow" href="https://www.quora.com/What-is-an-intuitive-explanation-of-the-concept-of-entropy-in-information-theory">Entropy</a>&nbsp;is used as a unit of measure for the unpredictability of the content of information. If you would throw a fair dice, each of the six outcomes have the same probability of occuring (1/6). Therefore you have maximum uncertainty; an entropy of 1. If the dice is weighted you already know one of the six outcomes has a higher probability of occuring and the uncertainty becomes less. If the dice is weighted so much that the outcome is always six, there is zero uncertainty in the outcome and hence the information entropy is also zero.<br> The same applies to letters in a word (or words in a sentence): if you assume that every letter has the same probability of occuring you have maximum uncertainty in predicting the next letter. But if you know that letters like E, A, O or I have a higher probability of occuring you have less uncertainty.</p> 
            <p>Knowing this, we can say that complex data has a high entropy, patterns and trends have lower entropy, information you know for a fact to be true has zero entropy (and therefore can be excluded).<br> The idea behind Maximum Entropy is that you want a model which is as unbiased as possible; events which are not excluded by known constraints should be assigned as much uncertainty as possible, meaning the probability distribution should be as uniform as possible. You are looking for the maximum value of the Entropy. If this is not entirely clear, I recommend you to read through&nbsp;<a rel="nofollow" href="http://www.cs.cmu.edu/afs/cs/user/aberger/www/html/tutorial/node2.html">this</a>&nbsp;example.</p> 
            <p>The mathematical formula for Entropy is given by&nbsp;<img src="https://s0.wp.com/latex.php?latex=H%28p%29+%3D+-+%5Csum+p%28a%2Cb%29+log+p%28a%2Cb%29&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="H(p) = - \sum p(a,b) log p(a,b)" title="H(p) = - \sum p(a,b) log p(a,b)" class="latex">, so the most likely probability distribution&nbsp;<img src="https://s0.wp.com/latex.php?latex=p&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="p" title="p" class="latex">is the one that maximizes this entropy:</p> 
            <p><img src="https://s0.wp.com/latex.php?latex=p+%3D+argmax+H%28p%29&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="p = argmax H(p)" title="p = argmax H(p)" class="latex"></p> 
            <p>It can be&nbsp;<a rel="nofollow" href="http://www.cs.cmu.edu/afs/cs/user/aberger/www/html/tutorial/node7.html">shown&nbsp;</a>that the probability distribution has an exponential form and hence is given by:</p> 
            <p><img src="https://s0.wp.com/latex.php?latex=P%28c%7Cd%29+%3D+%5Cfrac%7B1%7D%7BZ%28d%29%7D+exp+%28+%5Csum_i+%5Clambda_i+f_i%28d%2Cc%29%29&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="P(c|d) = \frac{1}{Z(d)} exp ( \sum_i \lambda_i f_i(d,c))" title="P(c|d) = \frac{1}{Z(d)} exp ( \sum_i \lambda_i f_i(d,c))" class="latex">,</p> 
            <p>where&nbsp;<img src="https://s0.wp.com/latex.php?latex=f_i%28d%2Cc%29&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="f_i(d,c)" title="f_i(d,c)" class="latex">&nbsp;is a feature function,&nbsp;<img src="https://s0.wp.com/latex.php?latex=%5Clambda_i&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="\lambda_i" title="\lambda_i" class="latex">&nbsp;is the weight parameter of the feature function and&nbsp;<img src="https://s0.wp.com/latex.php?latex=Z%28d%29&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="Z(d)" title="Z(d)" class="latex">&nbsp;is a normalization factor given by<br> <img src="https://s0.wp.com/latex.php?latex=Z%28d%29+%3D+%5Csum_c+exp+%28+%5Csum_i+%5Clambda_i+f_i%28d%2Cc%29+%29+&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="Z(d) = \sum_c exp ( \sum_i \lambda_i f_i(d,c) ) " title="Z(d) = \sum_c exp ( \sum_i \lambda_i f_i(d,c) ) " class="latex">.</p> 
            <p>This feature function is an&nbsp;<a rel="nofollow" href="https://en.wikipedia.org/wiki/Indicator_function">indicator function</a>, which is expresses the expected value of the chosen statistics (words) in&nbsp;the training set. These feature functions can then be taken as constraints for the classification of the actual dataset (by eliminating the probability distributions P(c|d) which do not fit with these constraints).</p> 
            <p>Usually, the weight parameters are&nbsp;automatically determined by the Improved Iterative Scaling algorithm. This is simply a&nbsp;<a rel="nofollow" href="https://en.wikipedia.org/wiki/Gradient_descent">gradient descent</a>function which can be iterated over until it converges&nbsp;to the global maximum. The pseudocode for the this algorithm is as follows:</p> 
            <ol> 
             <li>Initialize all weight parameters&nbsp;<img src="https://s0.wp.com/latex.php?latex=%5Clambda_i&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="\lambda_i" title="\lambda_i" class="latex">&nbsp;to zero.</li> 
             <li>Repeat until convergence:
              <ol> 
               <li>calculate the probability distribution&nbsp;<img src="https://s0.wp.com/latex.php?latex=P_%7B%5CLambda%7D%28c%7Cd%29&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="P_{\Lambda}(c|d)" title="P_{\Lambda}(c|d)" class="latex">&nbsp;with the weight parameters filled in.</li> 
               <li>for each parameter&nbsp;<img src="https://s0.wp.com/latex.php?latex=%5Clambda_i&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="\lambda_i" title="\lambda_i" class="latex">&nbsp;calculate&nbsp;<img src="https://s0.wp.com/latex.php?latex=%5CDelta+%5Clambda_i&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="\Delta \lambda_i" title="\Delta \lambda_i" class="latex">. This is the solution to:<br> <img src="https://s0.wp.com/latex.php?latex=%5Csum_c+P_%7B%5CLambda%7D%28c%2Cd%29+%5Ccdot+f_i%28d%2Cc%29+%5Ccdot+exp+%28+%5CDelta+%5Clambda_i+f%5E%7B%5C%23%7D+%28d%2Cc%29+%29+%3D+%5Csum_d+f_i%28d%2Cc%29&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="\sum_c P_{\Lambda}(c,d) \cdot f_i(d,c) \cdot exp ( \Delta \lambda_i f^{\#} (d,c) ) = \sum_d f_i(d,c)" title="\sum_c P_{\Lambda}(c,d) \cdot f_i(d,c) \cdot exp ( \Delta \lambda_i f^{\#} (d,c) ) = \sum_d f_i(d,c)" class="latex"></li> 
               <li>update the value for the weight parameter:<br> <img src="https://s0.wp.com/latex.php?latex=%5Clambda_i+%3A%3D+%5Clambda_i+%2B+%5CDelta+%5Clambda_i+&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="\lambda_i := \lambda_i + \Delta \lambda_i " title="\lambda_i := \lambda_i + \Delta \lambda_i " class="latex"></li> 
              </ol> </li> 
            </ol> 
            <p>In step 2b&nbsp;<img src="https://s0.wp.com/latex.php?latex=f%5E%7B%5C%23%7D%28d%2Cc%29&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="f^{\#}(d,c)" title="f^{\#}(d,c)" class="latex">&nbsp;is given by the sum of all features in the training dataset d:&nbsp;<img src="https://s0.wp.com/latex.php?latex=f%5E%7B%5C%23%7D+%28d%2Cc%29+%3D+%5Csum_i+f_i+%28d%2Cc%29+&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="f^{\#} (d,c) = \sum_i f_i (d,c) " title="f^{\#} (d,c) = \sum_i f_i (d,c) " class="latex"></p> 
            <p>Maximum Entropy is a general statistical classification algorithm and can be used to estimate any probability distribution. For the specific case of text classification, we can limit its form a bit more by&nbsp;using word counts as features:</p> 
            <p><img src="https://s0.wp.com/latex.php?latex=+f_%7Bw%2Cc%27%7D%28d%2Cc%29+%3D+0&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt=" f_{w,c'}(d,c) = 0" title=" f_{w,c'}(d,c) = 0" class="latex">&nbsp;if&nbsp;<img src="https://s0.wp.com/latex.php?latex=c+%5Cneq+c%27+&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="c \neq c' " title="c \neq c' " class="latex"><br> <img src="https://s0.wp.com/latex.php?latex=+f_%7Bw%2Cc%27%7D%28d%2Cc%29+%3D+%5Cfrac%7Bcount%28d%2Cw%29%7D%7Bcount%28d%29%7D&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt=" f_{w,c'}(d,c) = \frac{count(d,w)}{count(d)}" title=" f_{w,c'}(d,c) = \frac{count(d,w)}{count(d)}" class="latex">&nbsp;otherwise.</p> 
            <h1><strong>4. Support Vector Machines:</strong></h1> 
            <p>Although it is not immediatly obvious from the name, the SVM algorithm is a ‘simple’ linear classification/regression algorithm<a rel="nofollow" href="https://ataspinar.wordpress.com/2015/11/16/text-classification-and-sentiment-analysis/#SVM_literature">[6]</a>. It tries to find a<a rel="nofollow" href="https://en.wikipedia.org/wiki/Hyperplane">hyperplane&nbsp;</a>which seperates the&nbsp;data in two classes as optimally as possible.<br> Here as optimally as possible means&nbsp;that as much points as possible&nbsp;of label&nbsp;A should be seperated to one side of the hyperplane&nbsp;and as points of label&nbsp;B to the other side, while maximizing the distance of each point to this hyperplane.</p> 
            <p>&nbsp;</p> 
            <p><img class=" size-medium wp-image-1712 aligncenter" src="https://ataspinar.files.wordpress.com/2015/11/svm_max_sep_hyperplane_with_margin.png?w=278&amp;h=300" alt="svm_max_sep_hyperplane_with_margin" width="278" height="300"></p> 
            <p>In the image above we can see this illustrated for the example of&nbsp;points plotted in 2D-space. The set of points are labeled &nbsp;with two categories (illustrated here with black and white points) and SVM chooses the hypeplane that maximizes the margin between the two classes. This hyperplane is given by</p> 
            <p><img src="https://s0.wp.com/latex.php?latex=%3C+%5Cvec%7Bw%7D+%5Ccdot+%5Cvec%7Bx%7D+%3E+%2B+b+%3D+%5Csum_i+y_i+%5Calpha_i+%3C+%5Cvec%7Bx_i%7D+%5Ccdot+%5Cvec%7Bx%7D%3E+%2B+b+%3D+0+&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="< \vec{w} \cdot \vec{x} > + b = \sum_i y_i \alpha_i < \vec{x_i} \cdot \vec{x}> + b = 0 " title="< \vec{w} \cdot \vec{x} > + b = \sum_i y_i \alpha_i < \vec{x_i} \cdot \vec{x}> + b = 0 " class="latex"></p> 
            <p>where&nbsp;<img src="https://s0.wp.com/latex.php?latex=%5Cvec%7Bx_i%7D+%3D+%28x_%7Bi1%7D%2C+x_%7Bi2%7D%2C+..+%2C+x_%7Bin%7D+%29&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="\vec{x_i} = (x_{i1}, x_{i2}, .. , x_{in} )" title="\vec{x_i} = (x_{i1}, x_{i2}, .. , x_{in} )" class="latex">&nbsp;is a n-dimensional input vector,&nbsp;<img src="https://s0.wp.com/latex.php?latex=y_i&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="y_i" title="y_i" class="latex">&nbsp;is its output value,&nbsp;<img src="https://s0.wp.com/latex.php?latex=%5Cvec%7Bw%7D+%3D+%28w_1%2C+w_2%2C+..+%2C+w_n+%29+&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="\vec{w} = (w_1, w_2, .. , w_n ) " title="\vec{w} = (w_1, w_2, .. , w_n ) " class="latex">&nbsp;is the weight vector (the&nbsp;<a rel="nofollow" href="https://en.wikipedia.org/wiki/Normal_(geometry)">normal&nbsp;</a>vector) defining&nbsp;the hyperplane and the&nbsp;<img src="https://s0.wp.com/latex.php?latex=%5Calpha_i&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="\alpha_i" title="\alpha_i" class="latex">&nbsp;terms are the Lagrangian multipliers.</p> 
            <p>Once the hyperplane is constructed (the vector&nbsp;<img src="https://s0.wp.com/latex.php?latex=%5Cvec%7Bw%7D&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="\vec{w}" title="\vec{w}" class="latex">&nbsp;is defined) with a training set, the class of any other input vector&nbsp;<img src="https://s0.wp.com/latex.php?latex=%5Cvec%7Bx_i%7D&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="\vec{x_i}" title="\vec{x_i}" class="latex">&nbsp;can be determined: &nbsp;if<img src="https://s0.wp.com/latex.php?latex=%5Cvec%7Bw%7D+%5Ccdot+%5Cvec%7Bx_i%7D+%2B+b+%5Cge+0&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="\vec{w} \cdot \vec{x_i} + b \ge 0" title="\vec{w} \cdot \vec{x_i} + b \ge 0" class="latex">&nbsp;then it belongs to the positive class (the class we are interested in), otherwise it belongs to the negative class (all of the other classes).</p> 
            <p>We can already see this leads to two interesting questions:<br> 1. SVM only seems to work when the two classes are&nbsp;<a rel="nofollow" href="https://en.wikipedia.org/wiki/Linear_separability">linearly separable</a>. How can we deal with&nbsp;<a rel="nofollow" href="https://en.wikipedia.org/wiki/Reading_path">non-linear datasets</a>? Here I feel the urge to point out that the Naive Bayes and Maximum Entropy are linear classifiers as well and most text documents will be linear. Our training example of Amazon book reviews will be linear as well. But an explanation of the SVM system will not be complete without an explanation of Kernel functions.<br> 2. SVM only seems to be able to separate the dataset into two classes? How can we deal with datasets with more than two classes. For Sentiment Classification we have for example three classes (positive, neutral, negative) and for Topic Classification we can have even more than that.</p> 
            <p><strong>Kernel Functions:<br></strong> The classical SVM system requires that the dataset is linearly separable, i.e. there is a single hyperplane which can separate the two classes. For non-linear datasets a Kernel function is used to map the data to a higher dimensional space in which it is linearly separable.&nbsp;<a rel="nofollow" href="https://www.youtube.com/watch?v=3liCbRZPrZA">This</a>video gives a good illustation of such a mapping. In this higher dimensional feature space, the classical SVM system can then be used to construct a hyperplane.</p> 
            <p><strong>Multiclass classification:</strong><br> The classical SVM system is a binary classifier, meaning that it can only separate the dataset into &nbsp;two classes. To deal with datasets with more than two classes usually the dataset is reduced to a binary class dataset with which the SVM can work. There are two approaches for decomposing a multiclass classification problem&nbsp;to a binary classification problem: the one-vs-all and one-vs-one approach.<br> In the one-vs-all approach one SVM Classifier is build per class. This Classifier takes that one class as the positive class and the rest of the classes as the negative class. A datapoint is then only classified within a specific class if it is accepted by that Class’ Classifier and rejected by all other classifiers. Although this can lead to accurate results (if the dataset is clustered), a lot of datapoints can also be left unclassified (if the dataset is not clustered).<br> In the one-vs-one approach, you build one SVM Classifier per chosen pair of classes. Since there are&nbsp;<img src="https://s0.wp.com/latex.php?latex=0.5N%28N-1%29&amp;bg=ffffff&amp;fg=111111&amp;s=0" alt="0.5N(N-1)" title="0.5N(N-1)" class="latex">&nbsp;possible pair combinations for a set of N classes, this means you have to construct more Classifiers. Datapoints are then categorized in the class for which they have received the most points.</p> 
            <p>In our example, there are only three classes (positive, neutral, negative) so there is no real difference between these two approaches. In both approaches we have to construct two hyperplanes; positive vs the rest and negative vs the rest.&nbsp;</p> 
            <h1>What to expect:</h1>

I will use a part of these book reviews for training purposes and a part as the test dataset. In the next few blogs I will try to automatically classify the sentiment of these reviews with the four models described above.</p>
