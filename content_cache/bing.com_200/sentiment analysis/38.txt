        <h1 class="entry-title">HP IDOL OnDemand Sentiment Analysis Java&nbsp;Tutorial</h1> 
        <div class="entry-meta"> 
         <span class="sep">Posted on </span>
         <a href="https://albusshin.wordpress.com/2014/07/25/hp-idol-ondemand-sentiment-analysis-java-tutorial/" title="2:28 pm" rel="bookmark"><time class="entry-date" datetime="2014-07-25T14:28:36+00:00">July 25, 2014</time></a>
         <span class="by-author"> <span class="sep"> by </span> <span class="author vcard"><a class="url fn n" href="https://albusshin.wordpress.com/author/albusshin/" title="View all posts by albusshin" rel="author">albusshin</a></span></span> 
        </div>

       <div class="entry-content"> 
        <p>Hello there! Today we’re going to&nbsp;go through a tutorial of IDOL OnDemand Sentiment Analysis Java Tutorial. This tutorial will be based on the platform of Java EE and IDOLOnDemand, so get yourself prepared with <a href="http://eclipse.org/downloads/packages/eclipse-ide-java-ee-developers/lunar">Eclipse for JavaEE Developers</a>, <a href="http://tomcat.apache.org/">Tomcat</a>, and of course, <a href="http://java.com/">Java</a>.</p> 
        <p><span class="embed-youtube" style="text-align:center; display: block;"><iframe class="youtube-player" type="text/html" width="584" height="359" src="https://www.youtube.com/embed/ldqM31VcMGI?version=3&amp;rel=1&amp;fs=1&amp;autohide=2&amp;showsearch=0&amp;showinfo=1&amp;iv_load_policy=1&amp;wmode=transparent" allowfullscreen="true" style="border:0;"></iframe></span></p> 
        <p>&nbsp;</p> 
        <p>This tutorial is not supposed to be a step-by-step tutorial.&nbsp;It’s more of a crash course. I suppose you have knowledge in Java EE development, JavaScript and JQuery.</p> 
        <h1>1. Libraries and dependencies</h1> 
        <p>For this Java EE project, we need some jar files to be copied into the folder of</p> 
        <pre><code>WebContent/WEB-INF/lib</code></pre> 
        <p>The following two libraries should be imported: <a href="http://hc.apache.org/downloads.cgi">HttpComponent-4.3.4</a>, <a href="http://twitter4j.org/en/">twitter4j-4.0.2</a>. We’re going to&nbsp;use the HTTP Request function in HttpComponent, and we’re going to use the Java wrappers provided by twitter4j to get tweets from Twitter instead of making our own API calls.</p> 
        <p>If you’re developing this project as a newly created one, copy all the jar files into the folder above, and configure the build path in Eclipse. Add all the jar files into the build path as External&nbsp;JARs. Otherwise, if you’re importing a project folder into Eclipse, you don’t need to copy the jar files into the folder, but you need to configure your build path as well. The deployment guide in the zip file should give you enough information.</p> 
        <p>&nbsp;</p> 
        <h1>2. Make HTTP Request in Java</h1> 
        <p>As this is a tutorial of how to make API calls from Java to IDOLOnDemand, we’re certainly supposed to make HTTP requests in our Java source code. The following piece of code would handle that.</p> 
        <pre><code>
	private String sendGetRequest(String url) throws Exception {
		 
		HttpClient client = new DefaultHttpClient();
		HttpGet request = new HttpGet(url);
 
		HttpResponse response = client.execute(request);
 
		System.out.println("\nSending 'GET' request to URL : " + url);
		System.out.println("Response Code : " + 
                       response.getStatusLine().getStatusCode());
 
		BufferedReader reader = new BufferedReader(
                       new InputStreamReader(response.getEntity().getContent()));
 
		StringBuffer result = new StringBuffer();
		String line = "";
		while ((line = reader.readLine()) != null) {
			result.append(line);
		}
 
		return result.toString();
	}
</code></pre> 
        <p>Here, the&nbsp;HttpComponent library helped us a lot. The code basically explains itself. First we declare an HttpClient object, and then we use the object to make a request. Getting the response, we use a BufferedReader to read the response text, and then return the response text as a String.</p> 
        <h1>3. Declare our IDOLOnDemand API key and Twitter API keys</h1> 
        <p>We need API keys to access APIs from IDOLOnDemand and Twitter, for sure. Create a class called Constants, and put all your API keys there.</p> 
        <p>There should be five keys in total, one for IDOLOnDemand and four for Twitter.</p> 
        <p>The source file should be looking like this:</p> 
        <p>&nbsp;</p> 
        <pre><code>package sentimentanalysis;

public class Constants {
	public static final String apikey="********-****-****-****-************";
	
	public static final String TwitterOAuthConsumerKey = "***************";
	public static final String TwitterOAuthConsumerSecret = "*************************************";
	public static final String TwitterOAuthAccessToken = "********-**********************************";
	public static final String TwitterOAuthAccessTokenSecret = "*******************************";
}

</code></pre> 
        <h1>4. Getting Tweets</h1> 
        <p>In this tutorial, we’re going&nbsp;to get a load of tweets from Twitter from the #F1 Twitter feed (<a href="https://twitter.com/hashtag/f1" target="_blank">https://twitter.com/hashtag/f1</a>).</p> 
        <p>Twitter API needs to be called with OAuth data, and it’s going to be a lot of trouble if we make our own API calls against Twitter APIs. Fortunately we have a library for this. twitter4j&nbsp;for the win.</p> 
        <p>First we declare a TweetInfo model:</p> 
        <pre><code>
package sentimentanalysis;

public class TweetInfo {
	private String user;
	private String text;
	
	public String getText() {
		return text;
	}
	public void setText(String text) {
		this.text = text;
	}
	public String getUser() {
		return user;
	}
	public void setUser(String user) {
		this.user = user;
	}
}
</code></pre> 
        <p>We’ll use this model class to represent Tweet data in our jsp file.</p> 
        <p>The getTweetsTaggedF1() method would request 100 recent tweets that are tagged with the label “F1”, and return a List of TweetInfos. Before calling that method, we need to initialize a Twitter object instance using our Twitter API keys.</p> 
        <pre><code>
private Twitter twitter;

private void initializeTwitter(){
	ConfigurationBuilder cb = new ConfigurationBuilder();
	cb.setDebugEnabled(true)
	  .setOAuthConsumerKey(Constants.TwitterOAuthConsumerKey)
	  .setOAuthConsumerSecret(Constants.TwitterOAuthConsumerSecret)
	  .setOAuthAccessToken(Constants.TwitterOAuthAccessToken)
	  .setOAuthAccessTokenSecret(Constants.TwitterOAuthAccessTokenSecret);
	TwitterFactory tf = new TwitterFactory(cb.build());
	twitter = tf.getInstance();
}

public List getTweetsTaggedF1(){
	try {
            Query query = new Query("#F1");
            query.count(50);
            QueryResult result;
            result = twitter.search(query);
            List tweets = result.getTweets();
            List tweetsInfo= new ArrayList();
            for(Status tweet : tweets){
            	TweetInfo tweetInfo = new TweetInfo();
            	tweetInfo.setUser(tweet.getUser().getScreenName());
            	tweetInfo.setText(tweet.getText());
            	tweetsInfo.add(tweetInfo);
            }
            return tweetsInfo;
        } catch (TwitterException te) {
            te.printStackTrace();
            System.out.println("Failed to search tweets: " + te.getMessage());

}
</code></pre>

        <h1>5.How to make an IDOLOnDemand API call on Language identification, Sentiment Analysis and Highlight Text</h1> 
        <p>On the <a href="https://www.idolondemand.com/developer/apis/identifylanguage#request">Language Identification request details page</a>&nbsp;we find two major parts of the API call: the API key, and the input source. As we’ve solved the API key part, we’re going into input source here.</p> 
        <p>In this tutorial, we use the “text” parameter as the input source, as it is the simplest&nbsp;choice.</p> 
        <pre><code>
	public String getTweetLanguage(String tweetText) throws Exception {
		String languageIdentificationURL = "https://api.idolondemand.com/1/api/sync/identifylanguage/v1"
				+ "?"
				+ "apikey=" + Constants.apikey
				+ "&amp;"
				+ "text="
				+ URLEncoder.encode(tweetText, "UTF-8");
		String result = sendGetRequest(languageIdentificationURL);
		System.out.println(result);
		JSONObject languageResult = new JSONObject(result);
		return languageResult.getString("language_iso639_2b");
	}
</code></pre> 
        <p>The method above is the API call method of identifying the language of a single tweet. The reason we return&nbsp;languageResult.getString(“language_iso639_2b”) here is that this value is the value we’re going to pass to&nbsp;the Sentiment Analysis API call.</p> 
        <p>One thing that needs to be mentioned is that, the tweet text needs to be encoded before adding to the request URL. We use URLEncoder.encode() method for that.</p> 
        <pre><code>
	private String[] languages = {"eng", "fre", "spa", "ger", "ita", "chi"};

	private String getLanguageForSentimentAnalysis(String language){
		for (String lan : languages){
			if (lan.equals(language.toLowerCase())){
				return lan;
			}
		}
		return "eng";
	}

</code></pre> 
        <p>The code above is to make sure that we don’t pass an invalid language parameter into the sentiment analysis API.</p> 
        <pre><code>
	public JSONObject getTweetSentiment(String tweetText, String language) throws Exception{

	}
</code></pre> 
        <p>In the getTweetSentiment() method, we return the JSONObject for future use, because we need to display all kinds of data on the webpage.</p>

        <h1>7. Ajax call for language and sentiment</h1> 
        <p>We use ajax here, because we don’t want&nbsp;the API calls to be synchronized. If we make synchronized API calls against each of the tweets, we’ll wait for a very long time before we can get a visual response on our webpage.</p> 
        <p>We create a script.js file under WebContent folder.</p> 
        <p>In the script file, we’re going to write a function that executes itself when the document is ready.</p> 
        <pre><code>
$(function() { //start-up function, when document's ready, make language identification and sentiment analysis call against each tweet
	$.each($("tr.tweetinfo"), function(indexInArray, tr){
		$tr = $(tr);
		var tweetText = $(".tweettext", $tr).html();
		showTweetLanguageAndSentimentScore(tweetText, $tr);
	});
});
</code></pre> 
        <p>According to the piece of code above, as soon as the document is ready, we make the ajax call against our own server(GET the servlets) , which handles the API call against IDOLOnDemand. Each row of tweet would get a language identification result and an aggregate sentiment score here.</p> 
        <pre><code>
    var onLanguageCompleted = function($tr, language){ //language identification ajax call complete callback
		$(".language", $tr).html(language);
	},
	onSentimentCompleted = function($tr, aggregatescore){ //sentiment analysis ajax call complete callback
		var aggregateScoreJSON = $.parseJSON(aggregatescore);
		var score = aggregateScoreJSON["aggregate"]["score"]; //the aggregate score
		$(".aggregatescore", $tr).html(score);
		$(".scoredetail", $tr).html(aggregatescore);
		if (score == 0){ // if score == 0, we don't need to decorate the score color
			return;
		}
		else{
			if (score &gt; 0){ // Score is positive, colored green
				$(".aggregatescore", $tr).addClass("green");
			} else { // score &lt; 0, negative, colored red
				$(".aggregatescore", $tr).addClass("red");
			}
		}
	},
	showTweetLanguageAndSentimentScore = function(tweetText, $tr) { // show tweet language in the table, using ajax call
		$.ajax({
			url: "GetTweetLanguage?tweetText=" + getEncodedTweet(tweetText),
			complete : function(jqXHR, textStatus){
				onLanguageCompleted($tr, jqXHR.responseText);
				showTweetSentimentScore(getEncodedTweet(tweetText), jqXHR.responseText, $tr); //after getting language, automatically make sentiment score request
			}
		});
	},
	showTweetSentimentScore = function(tweetText, language, $tr) { // show sentiment score in the table
		$.ajax({
			url: "GetTweetSentiment?tweetText=" + tweetText + "&amp;language=" + language,
			complete : function(jqXHR, textStatus){
				onSentimentCompleted($tr, jqXHR.responseText);
			}
		});
	};
</code></pre> 
        <p>When one API call is finished ,the functions of onXXXCompleted() would be called, showing the result in the table, replacing the loading icon.</p>

        <h1>8. JQuery UI Dialog showing details of a tweet</h1> 
        <p>We use a jQuery UI Dialog to show the details of a tweet, because it’s simple to use. The dialog would be created when we click on a row of the table of tweets. The js code is below.</p> 
        <pre><code>
$("tr.tweetinfo").click(function(event){ //Clicking a table row, we process the detail of the tweet clicked, and show a dialog
	var $tr = $(this),
		htmlString = "",
		negativeTerms = [],
		positiveTerms = [],
		scoredetail = $(".scoredetail", $tr).html(),

		positives = scoreJSON["positive"],

		negativeTermsParameter = "",
		tweetText = $(".tweettext", $tr).html();
	

	
	htmlString += "&lt;div class='highlightedtweet'&gt;" + tweetText + "&lt;br&gt;&lt;br&gt; &lt;/div&gt;";

		}
		negativeTerms.push(negative["original_text"]);
	});
	$.each(positives, function(i, positive){

						url: "GetHighlightedTweet?tweetText=" + getEncodedTweet(highlightedTweetText) + "&amp;terms=" + negativeTermsParameter + "&amp;pn=negative",

	} else if (negativeTerms.length &gt; 0){ // if negative terms exist, we make ajax call against negative terms highlighting

				$("div.loadinghighlightedtweet").empty();
				$("div.highlightedtweet").html(jqXHR.responseText + "&lt;br&gt;&lt;br&gt;"); //show the highligted text

		});
	}
	
});

        <p>I know that’s a whole load of code to read, but we can break the function into several pieces. The major purpose of this function is threefold: to display the tweet detail in a dialog, to process the positive and negative terms into arrays so that we can use them, and to make highlight text API calls so that the tweet would be highlighted.</p> 
        <p>When a table row (tr) is clicked, the handler function would be called. In this function we get the sentiment analysis JSON we got from the server before, and analyze the terms that are driving&nbsp;the sentiment score. After getting all the terms, we build a parameter to be passed to&nbsp;Highlight Text API call, and then calls GET from our server to get the Highlighted Text, which would GET the highlighted text from IDOLOnDemand.<br> Similarly, when the ajax call is finished and returned, the complete callback function would replace the returned highlighted tweet text with the original tweet text, and we can see the highlighted result in the jQuery UI Dialog.</p>
