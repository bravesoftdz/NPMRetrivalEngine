              <p>All java files should be in "animal" directory:</p> 
              <pre><code>- animal
|- Eat.java
|- Animal.java
|- Dog.java
</code></pre> 
              <p>If you have all classes in a package like "com.some.package" the path should be</p> 
              <pre><code>- com
|- some
 |-package
  |- Eat.java
  |- Animal.java
  |- Dog.java
</code></pre>

              <p>Go to the directory <code>D:\javapractice\interface</code> and open the cmd.Keep all the three java files under directory <code>D:\javapractice\interface\animals</code>.</p> 
              <pre><code>D:\javapractice\interface
- animal
|- Eat.java
|- Animal.java
|- Dog.java
</code></pre> 
              <p>Now start compiling the java classes </p> 
              <p>Cmd : <code>D:\javapractice\interface&gt;</code></p> 
              <pre><code>javac animal\Eat.java
javac animal\Animal.java
javac animal\Dog.java
</code></pre> 
              <p>You will find three .class at the location D:\javapractice\interface\animal</p> 
              <p>Now time to run it :</p> 
              <pre><code>java animal.Dog
</code></pre>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>You want to create separate source and output directories where your classes will be compiled to. Unless you specify an output directory, your classes will be compiled into your source directories. That's a real recipe for pain when you need to delete your old class files (and you'll need to eventually).</p> 
              <p>Supposing you have this directory structure:</p> 
              <pre><code>.
??? classes
??? src
    ??? animal
        ??? Animal.java
        ??? Dog.java
        ??? Eat.java
</code></pre> 
              <p>Compile like this:</p> 
              <pre><code>javac -d classes -sourcepath src src/animal/Dog.java 
</code></pre> 
              <p>You'll end up with:</p> 
              <pre><code>.
??? classes
?&nbsp;&nbsp; ??? animal
?&nbsp;&nbsp;     ??? Animal.class
?&nbsp;&nbsp;     ??? Dog.class
?&nbsp;&nbsp;     ??? Eat.class
??? src
    ??? animal
        ??? Animal.java
        ??? Dog.java
        ??? Eat.java
</code></pre> 
              <p>One tip: If you compile the most dependent class (i.e. Dog.java), javac will compile its dependencies (Animal, Eat). That way you don't have to execute javac 3 times.</p>
