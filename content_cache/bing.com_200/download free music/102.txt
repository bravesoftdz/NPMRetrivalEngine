    <h2 id="starters">Play Starter Projects</h2> 
    <h3>New to Play? See the starter projects!</h3> 
   </article> 
   <article class="starter-projects"> 
    <p> If you've never used Play before, download a starter project. The starter projects have lots of comments explaining how everything works and have links to documentation that goes more in depth. </p> 
    <p> If you download and unzip the .zip file below, you'll see the <code>sbt</code> file -- this is a packaged version of <a href="http://scala-sbt.org">sbt</a>, the build tool that Play uses. </p> 
    <p>Type the following at the prompt and it will run the starter project: </p>
    <pre><code>./sbt run</code></pre> 
    <p> If you type <code>./sbt</code> by itself, then you can run it in <a href="http://www.scala-sbt.org/1.0/docs/Howto-Interactive-Mode.html">interactive mode</a> -- see <a href="https://www.playframework.com/documentation/latest/PlayConsole">Using the Play Console</a> for more detail of how to run Play and the <a href="http://www.scala-sbt.org/1.0/docs">sbt documentation</a> for more about SBT. </p> 
    <p> Please see the <a href="https://playframework.com/documentation/latest/Home">documentation</a>, chat with the <a href="https://playframework.com/get-involved">community</a>, and check out the <a href="https://playframework.com/documentation/latest/Tutorials">tutorials</a> for more examples and blog posts! </p>

    <h2 id="seeds">Play Seed Templates</h2> 
    <h3>Experienced With Play? Use a Seed Template!</h3> 
   </article> 
   <hr> 
   <article class="seed-projects"> 
    <p> If you have <a href="http://scala-sbt.org">sbt 0.13.13 or higher</a> installed, you can create your own Play project using <code>sbt new</code> using a minimal <a href="http://foundweekends.org/giter8">giter8</a> template (roughly like a maven archetype). </p> 
    <p> This is a good choice if you already know Play and want to create a new project immediately. </p> 
    <p> Note that the seed templates are already configured with CSRF and security headers filters, whereas the other projects are not specifically set up for security out of the box. </p> 
    <p> Use <code>sbt run</code> and then go to <a href="http://localhost:9000">http://localhost:9000</a> to see the running server. </p> 
    <p> Type <code>g8Scaffold form</code> from sbt to create the scaffold controller, template and tests needed to process a form. You can also create your own giter8 seeds and scaffolds based off this one by forking from the <a href="https://github.com/playframework/play-java-seed.g8">https://github.com/playframework/play-java-seed.g8</a> or <a href="https://github.com/playframework/play-scala-seed.g8">https://github.com/playframework/play-scala-seed.g8</a> github projects. </p> 
    <div class="seed-project"> 
     <h4>Play Scala Seed</h4> 
     <pre><code>sbt new playframework/play-scala-seed.g8</code></pre>

    <p> Note that these example projects are not configured for out of the box security, and are intended to showcase particular areas of Play functionality. </p>
