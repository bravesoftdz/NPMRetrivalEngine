          <ul> 
           <li>HTMLAsText automatically removes all tags and scripts from the document. </li>
           <li>The remained text is formatted according to the number of characters per line that you select. </li>
           <li>All HTML entities (e.g.: &amp;amp, &amp;lt) are converted into the corresponding ASCII characters. </li>
           <li>Unordered lists (&lt;ul&gt; tag) and ordered lists (&lt;ol&gt; tag) are formatted accordingly. The bullets beside the items of unordered lists are replaced by ASCII characters according to your selection. </li>
           <li>Definition lists are formatted by adding spaces in the left side of the definition lines. </li>
           <li>Optionally, centered and right-aligned paragraphs are formatted accordingly by adding space characters in the left side of the lines. </li>
           <li>Optionally, HTMLAsText allows you to add a line under each heading (&lt;h1&gt; - &lt;h6&gt; tags) </li>
           <li>Simple tables can be delimited by spaces, tab characters, commas, or CRLF. </li>
           <li>Preformatted text blocks (&lt;pre&gt; tag) are copied "as is", without formatting the text. </li>
           <li>You can convert multiple HTML files in the same folder at once, by using wildcard. (e.g.: c:\files\*.html) </li>
           <li>You can run the conversion process without displaying any user interface, by using the /run command-line option. </li>
          </ul> <p> <img src="htmlastext.gif"> </p><p> </p><p> </p><h4 class="utilsubject">Versions History</h4>

          </ul> <p> </p><h4 class="utilsubject">Using HTMLAsText</h4> HTMLAsText is a small and standalone executable. You don't need to install anything, and additional DLLs are not required for using this utility. Just copy the executable to any folder you like, and run it. <p> In the main screen of HTMLAsText utility, you are allowed to set the configuration required for converting your HTML files to text files. The following options are available: </p>
          <ul> 
           <li><span class="special3">Single/Multiple:</span> In this option you should specify whether you want to convert a single file or multiple files: 
            <ul> 
             <li><span class="special2">Convert Single File:</span> If you choose this option, the HTML file specified in 'HTML File' field will be converted into the text file specified in the 'Text File' field. </li>
             <li><span class="special2">Convert Multiple Files:</span> If you choose this option, the 'HTML Wildcard' field should contain wildcard for the HTML files you want to convert, with full path. for example: <br> C:\MyFiles\HTML\*.html <br> F:\Projects\HTML\Test1\*.htm <br> The 'Text Wildcard' field should contain wildcard for the text files you want to save, for example: <br> C:\MyFiles\text\*.txt </li>
            </ul> </li>
           <li><span class="special3">Scan Subfolders:</span> If your select this option, HtmlAsText search for html files in all subfolders under the specified folder of 'HTML Wildcard' field. <br> Available only if 'Convert Multiple Files' is selected. </li>
           <li><span class="special3">Characters Per Line:</span> Maximum number of characters per line in the converted text file. </li>
           <li><span class="special3">Unordered List Characters:</span> The characters specified in this field will be used for replacing the bullets of unordered lists. You can specify here up to 5 characters (for nested lists). </li>
           <li><span class="special3">Add line under headings:</span> If this option is checked, a line will be added under each heading. The line will be created by using the 6 characters that you specify - first character for &lt;h1&gt; tag, second character for &lt;h2&gt; tag, and so on. </li>
           <li><span class="special3">Skip the text stored inside &lt;title&gt; tag:</span> Just like it sounds - if this option is checked, the text inside the &lt;title&gt; won't be added to the converted text file. </li>
           <li><span class="special3">Horizontal rule character:</span> The character that will be used for displaying horizontal rule in the text file. </li>
           <li><span class="special3">Definition List spaces:</span> Number of spaces to add to definition line. (&lt;dd&gt; tag) </li>
           <li><span class="special3">Allow centered text:</span> If this option is checked, the text of centered paragraphs is centered by adding spaces in the left side of the lines. </li>
           <li><span class="special3">Allow right-aligned text:</span> If this option is checked, the text of right-aligned paragraphs is aligned to the right by adding space characters. </li>
           <li><span class="special3">separate table cells by... :</span> Determines how the cells of tables are separate: by space, by comma, by tab or CRLF (Enter). </li>
           <li><span class="special3">Enclose bold text with the following characters:</span> If this option is checked, bold text (&lt;b&gt; tag) is enclosed with the start/end characters that you select. </li>
           <li><span class="special3">Links Display Format:</span> Determines how to represent a link (&lt;a&gt; tag) in your text file. You can select one of the predefined formats, or create your own. the '%T' represents the text of the link, the '%L' represents the URL address. </li>
          </ul> <p> After setting the conversion options according to your needs, press the 'Convert' button in order to convert your HTML file(s) to text. You can also use the 'Save Config' option to save the current configuration, and then use it later by loading it again with 'Load Config'. </p><p> </p><h4 class="utilsubject">Converting HTML files from command-line</h4> In order to convert HTML files from command-line, you have to save the desired conversion options to configuration file (by using 'Save Config' option), and then run the HTMLAsText utility with /run option, followed by the configuration filename. <br> For example: <br> htmlastext.exe /run "c:\temp\convert.cfg" <p> Notice: When using /run command-line option, you must use the .cfg file created by the 'Save Config' inside the main window of HTMLAsText. You cannot not use the default config file (htmlastext.cfg) thta is created automatically by HTMLAsText, because this file is not compatible with the 'Save Config' file. </p><p> </p><h4 class="utilsubject">License</h4> This utility is released as freeware.
