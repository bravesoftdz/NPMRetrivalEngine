              <p>Is there a converter that will convert LaTeX to HTML? I have tried several services but the LaTeX diagrams and some of the maths came out weird.</p> 
              <p>I actually converted the Latex to PDF without a problem. Then I used <a href="http://www.pdfonline.com/convert-pdf-to-html/default.aspx" rel="nofollow noreferrer">an online PDF to HTML converter</a> to convert from PDF to HTML. The square root signs did not appear properly and some of the Cartesian coordinate plane diagrams had nothing on them except the grid. </p>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>Using PDF as an intermediate format when converting from LaTeX to HTML is not very good idea. LaTeX and HTML are both mostly structural markup langauages, which means you use them to describe the document structure (sections, emphasize, formulas etc.), whereas PDF is mostly about representation of your document on the screen or paper. When converting LaTeX to PDF, you loose much of the structural information, and it cannot be successfully recovered by conversion from PDF to HTML. </p> 
              <p>It is much better to convert LaTeX directly to HTML. There are <a href="http://web.archive.org/web/20121107120612/http://www.charlietanksley.net/philtex/converting-from-latex/">number of ways</a> (WayBack Archive) how to do that, one I would recommend is by using <code>htlatex</code>. It is probably already part of your TeX distribution, is very powerful and flexible, and its use can be as simple as running</p> 
              <pre><code>htlatex mydocument.tex
</code></pre> 
              <p>If you tell us more about your environment (which operating system do you use, what is your TeX distribution, your text editor/LaTeX IDE, how you generated the PDF file etc.) we may be able to give you more details on how to use <code>htlatex</code>. </p>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <blockquote> 
               <p>Is there a converter that will convert LaTeX to HTML? </p> 
              </blockquote> 
              <p>Yes. <a href="http://www.tex.ac.uk/cgi-bin/texfaq2html?label=LaTeX2HTML" rel="nofollow noreferrer">There</a> <a href="https://tex.stackexchange.com/questions/3079/how-do-i-convert-html-to-latex">are</a> lots. But you already knew this, as evidenced by your second "question":</p> 
              <blockquote> 
               <p>I have tried several services but the LaTeX diagrams and some of the maths came out weird.</p> 
              </blockquote> 
              <p>"turning out weird" could mean any number of things:</p> 
              <ul> 
               <li>The document fails to compile</li> 
               <li>The document compiles, but is empty or corrupted</li> 
               <li>The document compiles and opens, but is somehow faulty.</li> 
              </ul> 
              <p>Each of these categories breaks down further and the solution may well depend on what you used to do the conversion, so as it stands, this question is unanswerable.</p> 
              <p>For diagrams, one word of general advice I would give is that it might be best to compile the diagrams separately and include them into the html by hand with <code>&lt;img&gt;</code> tags. Diagrams are hard. The <a href="http://ctan.org/pkg/standalone" rel="nofollow noreferrer"><code>standalone</code></a> package/class can help with this. See also the <a href="https://tex.stackexchange.com/questions/tagged/tikz-external">pgf/TikZ externalisation tag</a>.</p>

The work-flows are given in the Figure 3 of the article on TUGboat: <em>Online publishing via pdf2htmlEX</em> <a href="http://coolwanglu.github.io/pdf2htmlEX/doc/tb108wang.html" rel="noreferrer">HTML</a> / <a href="http://coolwanglu.github.io/pdf2htmlEX/doc/tb108wang.pdf" rel="noreferrer">PDF</a>.</p>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>As previously observed using PDF format for converting from LaTeX to HTML is very bad idea. Here you will find a comprehensive list of programs that convert LaTeX to HTML (Note not all of those programs are capable of converting pure TeX.)</p> 
              <p><a href="http://www.tex.ac.uk/cgi-bin/texfaq2html?label=LaTeX2HTML" rel="nofollow noreferrer">http://www.tex.ac.uk/cgi-bin/texfaq2html?label=LaTeX2HTML</a></p> 
              <p>I am familiar with three of these programs: LaTeX2HTML, TeX4ht, and Hevea. All of them have their strengths and weaknesses and they use different approaches. TeX4ht is included in TeXLive and MiKTeX unlike other two and from the point of view of theoretical CS is a "correct" way to do conversion (not surprisingly it was written <a href="https://www.ctan.org/author/gurari" rel="nofollow noreferrer">by a (now deceased) member of the CS department at Ohio State</a>). If you have just a few pages to convert, probably the right approach for you would be to chose any of the three and then do some manual editing. Now you have to be careful. For example Hevea will try to use HTML tags for things like square root while TeX4ht will use bitmap images for any math expressions. Since pure HTML has a poor support for mathematics the first approach can quickly get into trouble but when it works the output is really beautiful. </p> 
              <p>As for diagrams and pictures, all of the above programs to my knowledge rely on ImageMagic to do image conversion. Hevea has to be run multiple times (it is actually bunch of scripts) and the proper way to use it is by writing a Makefile (Unix again). I would actually convert images and diagrams manually using GraphicMagic which is IMHO far more stable and better batch image processor than ImageMagic.</p>

            <td class="answercell"> 
             <div class="post-text" itemprop="text"> 
              <p>Three words: <strong>use LaTeX subset</strong></p> 
              <p>LaTeX and its common output formats like PDF are super complicated. Converting all of it to HTML + CSS is close to impossible.</p> 
              <p>But if you restrict yourself only to a small subset of LaTeX features it becomes feasible.</p> 
              <p>First I recommend that you just try to go with a simpler format like Markdown, which can be converted to HTML and PDF by several implementations like pandoc. I have come to believe that having painless HTML is more important than the infinite many features of LaTeX.</p> 
              <p>Then, check out <a href="https://github.com/softcover/softcover" rel="nofollow noreferrer">Softcover</a>: it has a LaTeX subset which it calls Polytex that it can convert to HTML. The exact subset is not well documented, but it already handles 99.9% of what we need to make regular books as shown in their documentation <a href="https://github.com/softcover/softcover_book/tree/master/chapters" rel="nofollow noreferrer">https://github.com/softcover/softcover_book/tree/master/chapters</a> : sections, lists, equations, tables, images, references. Softcover does it's magic through <a href="https://github.com/softcover/tralics" rel="nofollow noreferrer">Tralics</a>, which does LaTeX to XML.</p> 
              <p>As Seamus said, diagrams are hard, and it is unlikely they will be part of the implementable subset any time soon, so use images. Generate the images through a command line tool like gplot or pyplot, and then build the PDF with a Makefile that first builds the images and then the PDF.</p>

             <div class="post-text" itemprop="text"> 
              <p>I'm currently developing a free open source tool that can convert LaTeX to a single HTML file. Its called PDBF (<a href="https://github.com/uds-datalab/PDBF" rel="nofollow">https://github.com/uds-datalab/PDBF</a>) and runs on win/linux/mac.</p> 
              <p>Instructions:</p> 
              <ul> 
               <li>Make sure you have a Java Runtime (version &gt;= 1.7) installed</li> 
               <li>Download latest version of PDBF: <a href="https://github.com/uds-datalab/PDBF/archive/gh-pages.zip" rel="nofollow">https://github.com/uds-datalab/PDBF/archive/gh-pages.zip</a></li> 
               <li>Unpack archive</li> 
               <li>Change current dir to unpacked folder</li> 
               <li>Run: java -jar pdbf.jar /path/to/sometexfile.tex</li> 
              </ul> 
              <p>Resulting HTML file is placed in the same folder as the source tex file with the same name as the source tex file.</p> 
              <p>Technical details: The PDBF compiler basically uses a regular tex engine to compile your pdf and then stores the pdf as base64 encoded javascript string in the HTML and then includes a slightly modified version of pdf.js (The free and open source pdf engine of firefox) into the HTML to display that pdf. </p> 
              <p>Hope you like my project. Feel free to contact me through github if you have suggestions or encounter bugs.</p>
