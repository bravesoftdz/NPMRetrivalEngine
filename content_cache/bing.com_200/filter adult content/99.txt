           <h3 id="N10045" class="ibm-h3">Purpose</h3>
           <p>This document will illustrate how to implement global filters that filter reports in a Multi-page portlet in an IBM Cognos portal page. The core documentation covers the steps to achieve a global filter for reports on one page. This document goes beyond the core documentation to show how to apply the technique when multiple pages are involved.</p>
           <h3 id="N1004B" class="ibm-h3">Applicability</h3>
           <p>This technique was tested using IBM Cognos 8.4.1 and IBM Cognos 10.1 using the GO Data Warehouse (query) samples package.</p>
           <h3 id="N10051" class="ibm-h3">Assumptions</h3>
           <p>This document assumes experience with IBM Cognos Connection and IBM Cognos Report Studio.</p>
           <h2 id="N10057" class="ibm-h2">Overview</h2>
           <p>The IBM Cognos Connection User Guide provides an example of how to create inter-activity between reports with respect to having a prompt report control filtering in other independent reports on the same IBM Cognos portal page.</p>
           <p>For example, on one portal page you can place three Cognos Viewer portlets, each one containing a different report. The first one contains a simple report page that contains a prompt control for Product Lines. The other two reports contain detailed information that include Product Line information and each are filtered on Product Lines. When the prompt value is changed in the first report, the other two reports reflect that change. This is accomplished by using the same parameter name for Product Line in each report and configuring all three portlets to communicate with each other on a common channel.</p>
           <p>To take this technique one step further and allow prompt controls to affect the filters of reports located on multiple pages of a Multi-page portlet, there is an extra step involved. In order to allow communication between portlets, you must configure a common channel name. The Multi-page portlet does not have this configuration since it is a container intended to point to a location containing objects for display on multiple pages; one page for each object in the target location. These objects include reports, report views, shortcuts to reports, and portal pages. The key to this technique is to place each report in a Cognos Viewer portlet in a separate portal page. Each Cognos Viewer portlet would be configured with a common channel name. Consider these pages the detail pages. Finally, create a master portal page that points to the detail portal pages using a Multi-page portlet. This technique will allow communication between all the Cognos Viewer portlets regardless of which multi-page tab they are on since they are all in the same parent container, the master portal page.</p>
           <h2 id="N10061" class="ibm-h2">Create the Reports</h2>

           <p>In Public Folders, create the following folder structure:</p>
           <ul class="ibm-bullet-list">
            <li>Global Filter Dashboard
             <ul class="ibm-bullet-list">
              <li>Detail Pages</li>
              <li>Reports</li>
             </ul></li>
           </ul>

           <ol>
            <li>Open IBM Cognos Report Studio, create a blank report and drag a Value Prompt onto the page.</li>
            <li>In the create a new parameter box, type p_Product_Line. <h5 id="N1008A" class="ibm-h5">Prompt Wizard - Value Prompt</h5><img src="page531_images/page531_figure2.gif" class="ibm-downsize" alt="Prompt Wizard - Value Prompt" height="225" width="496"><p class="ibm-ind-link"><a class="ibm-popup-link" onclick="IBMCore.common.widget.overlay.show('N1008D');return false;" href="#N1008D">View image at full size</a></p>

             </div></li>
            <li>Click Next, and then click the ellipsis (...) beside the Package Item box.</li>
            <li>Navigate to Sales and Marketing (query) &gt; Sales (query) &gt; Product &gt; Codes, select Product line code, and then click OK.</li>
            <li>Select Make the filter optional, and then click Next.<br> By making the filter optional, you can choose to filter on both filters, only one filter, or choose not to filter at all. This choice will be based on your requirements.</li>
            <li>Click the ellipsis (...) beside the Values to Display box and browse to Sales and Marketing (query) &gt; Sales (query) &gt; Product, select Product line, and then click OK. <h5 id="N1009D" class="ibm-h5">Prompt Wizard - Value Prompt</h5><img src="page531_images/page531_figure3.gif" class="ibm-downsize" alt="Prompt Wizard - Value Prompt" height="265" width="498"><p class="ibm-ind-link"><a class="ibm-popup-link" onclick="IBMCore.common.widget.overlay.show('N100A0');return false;" href="#N100A0">View image at full size</a></p>

             </div><br> Using the Product line code as the “use” value makes the query more efficient, while the “display” value provide user friendly values.</li>
            <li>Click Finish.</li>
            <li>Drag another Value Prompt to the page.</li>
            <li>In the Create a new parameter box, type p_Year.</li>
            <li>Click Next, and then click the ellipsis (...) beside the Package Item box.</li>
            <li>Navigate to Sales and Marketing (query) &gt; Sales (query) &gt; Time dimension, select Year, and then click OK.</li>
            <li>Select Make the filter optional, and then click Next.<br> Since the “use” values and “display” values are the same in this instance, there is no need to configure the Values to display box.</li>

            <li>Organize and label the prompts and prompt button using a Table object and Text objects as shown below. <h5 id="N100BB" class="ibm-h5">Prompt Controls organized into a table</h5><img src="page531_images/page531_figure4.gif" class="ibm-downsize" alt="Prompt Controls organized into a table" height="506" width="218"></li>

           <p><strong>Note</strong>: When using just one single-select prompt, you can choose to use Auto-Submit to filter other reports in a dashboard page. However, when using multiple prompts, as in this example, do not use Auto-Submit since all the reports in the dashboard will be rerun for each prompt selection. This will create unnecessary traffic to the BI report servers and the data source. If you use a prompt button to submit your prompt selections, you can submit several prompt choices at once and only rerun the reports once.</p>
