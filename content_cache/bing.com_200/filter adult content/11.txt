         <h1><a href="#react-native-atom-reader" aria-hidden="true" class="anchor" id="user-content-react-native-atom-reader">
           <svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewbox="0 0 16 16" width="16">
            <path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path>
           </svg></a>react-native-atom-reader</h1> 
         <p>This is a demo project with React Native to view and filter an Atom/RSS feed. Currently, it has only been tested with Android smartphones and the Atom feed of <a href="https://heise.de/">heise online</a>, but it should also work on iOS and other standards compliant Atom newsfeeds.</p> 
         <p>This project was created for a tutorial in <a href="https://ct.de/">c't Magazin</a> which will appear in autumn 2017.</p> 
         <p><a href="/wortwart/react-native-atom-reader/blob/master/screenshot.png" target="_blank"><img src="/wortwart/react-native-atom-reader/raw/master/screenshot.png" alt="App Screenshot" style="max-width:100%;"></a></p> 
         <p>The app displays all feed entries, opens them in the browser on touch, and allows entries to be filtered according to the user's text input (not case-sensitive). The button on the top right resets the filter. Pulling refreshes the feed data.</p> 
         <p><strong>Important: You may not publish an app with heise online's newsfeed</strong> (which is used in the example code) <strong>because <a href="https://www.heise.de/news-extern/news.html">they do not permit displaying their images outside their own outlets</a>!</strong></p>

         <p>When starting your project, you'll see something like this for your project URL:</p> 
         <pre><code>exp://192.168.0.2:19000

         <p>The "manifest" at that URL tells the Expo app how to retrieve and load your app's JavaScript bundle, so even if you load it in the app via a URL like <code>exp://localhost:19000</code>, the Expo client app will still try to retrieve your app at the IP address that the start script provides.</p>

</code></pre>

         <pre><code>set REACT_NATIVE_PACKAGER_HOSTNAME='my-custom-ip-address-or-hostname'
npm start
</code></pre> 
         <p>The above example would cause the development server to listen on <code>exp://my-custom-ip-address-or-hostname:19000</code>.</p>

           </svg></a>Adding Flow</h3> 
         <p>Flow is a static type checker that helps you write code with fewer bugs. Check out this <a href="https://medium.com/@preethikasireddy/why-use-static-types-in-javascript-part-1-8382da1e0adb">introduction to using static types in JavaScript</a> if you are new to this concept.</p> 
         <p>React Native works with <a href="http://flowtype.org/">Flow</a> out of the box, as long as your Flow version matches the one used in the version of React Native.</p> 
         <p>To add a local dependency to the correct Flow version to a Create React Native App project, follow these steps:</p> 
         <ol> 
          <li>Find the Flow <code>[version]</code> at the bottom of the included <a href="/wortwart/react-native-atom-reader/blob/master/.flowconfig">.flowconfig</a></li> 
          <li>Run <code>npm install --save-dev flow-bin@x.y.z</code> (or <code>yarn add --dev flow-bin@x.y.z</code>), where <code>x.y.z</code> is the .flowconfig version number.</li> 
          <li>Add <code>"flow": "flow"</code> to the <code>scripts</code> section of your <code>package.json</code>.</li> 
          <li>Add <code>// @flow</code> to any files you want to type check (for example, to <code>App.js</code>).</li> 
         </ol> 
         <p>Now you can run <code>npm run flow</code> (or <code>yarn flow</code>) to check the files for type errors. You can optionally use a <a href="https://flow.org/en/docs/editors/">plugin for your IDE or editor</a> for a better integrated experience.</p> 
         <p>To learn more about Flow, check out <a href="https://flow.org/">its documentation</a>.</p>

         <p>If you have made use of Expo APIs while working on your project, then those API calls will stop working if you eject to a regular React Native project. See the <a href="https://github.com/react-community/create-react-native-app/blob/master/EJECTING.md">ejecting guide</a> for more details about this option.</p>

           </svg></a>Networking</h4> 
         <p>If you're unable to load your app on your phone due to a network timeout or a refused connection, a good first step is to verify that your phone and computer are on the same network and that they can reach each other. Create React Native App needs access to ports 19000 and 19001 so ensure that your network and firewall settings allow access from your device to your computer on both of these ports.</p> 
         <p>Try opening a web browser on your phone and opening the URL that the packager script prints, replacing <code>exp://</code> with <code>http://</code>. So, for example, if underneath the QR code in your terminal you see:</p> 
         <pre><code>exp://192.168.0.1:19000
</code></pre> 
         <p>Try opening Safari or Chrome on your phone and loading</p> 
         <pre><code>http://192.168.0.1:19000
</code></pre> 
         <p>and</p> 
         <pre><code>http://192.168.0.1:19001
</code></pre> 
         <p>If this works, but you're still unable to load your app by scanning the QR code, please open an issue on the <a href="https://github.com/react-community/create-react-native-app">Create React Native App repository</a> with details about these steps and any other error messages you may have received.</p> 
         <p>If you're not able to load the <code>http</code> URL in your phone's web browser, try using the tethering/mobile hotspot feature on your phone (beware of data usage, though), connecting your computer to that WiFi network, and restarting the packager.</p>

           </svg></a>iOS Simulator won't open</h4> 
         <p>If you're on a Mac, there are a few errors that users sometimes see when attempting to <code>npm run ios</code>:</p> 
         <ul> 
          <li>"non-zero exit code: 107"</li> 
          <li>"You may need to install Xcode" but it is already installed</li> 
          <li>and others</li> 
         </ul> 
         <p>There are a few steps you may want to take to troubleshoot these kinds of errors:</p> 
         <ol> 
          <li>Make sure Xcode is installed and open it to accept the license agreement if it prompts you. You can install it from the Mac App Store.</li> 
          <li>Open Xcode's Preferences, the Locations tab, and make sure that the <code>Command Line Tools</code> menu option is set to something. Sometimes when the CLI tools are first installed by Homebrew this option is left blank, which can prevent Apple utilities from finding the simulator. Make sure to re-run <code>npm/yarn run ios</code> after doing so.</li> 
          <li>If that doesn't work, open the Simulator, and under the app menu select <code>Reset Contents and Settings...</code>. After that has finished, quit the Simulator, and re-run <code>npm/yarn run ios</code>.</li> 
         </ol>

           </svg></a>QR Code does not scan</h4> 
         <p>If you're not able to scan the QR code, make sure your phone's camera is focusing correctly, and also make sure that the contrast on the two colors in your terminal is high enough. For example, WebStorm's default themes may <a href="https://github.com/react-community/create-react-native-app/issues/49">not have enough contrast</a> for terminal QR codes to be scannable with the system barcode scanners that the Expo app uses.</p> 
         <p>If this causes problems for you, you may want to try changing your terminal's color theme to have more contrast, or running Create React Native App from a different terminal. You can also manually enter the URL printed by the packager script in the Expo app's search bar to load it manually.</p>
