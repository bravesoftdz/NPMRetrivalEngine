               Java 8 provides excellent features to support filtering of elements in Java Collections. Prior to Java 8, only better way to filter elements is by using 
              <a href="http://javarevisited.blogspot.sg/2014/04/for-each-loop-puzzle-in-java-example.html" target="_blank">foreach loop</a> or iterating over Collection using Iterator and selecting required object, leaving out rest. Though that approach work, it was very difficult to run them in parallel and take advantage of multiple CPU available in modern day servers. Java 8 provides Streams, which not only makes it easy to run any operation parallel but also support 
              <i>lazy loading</i> and 
              <i>lazy evaluation</i>, which means as soon as filtering condition is satisfied, it stooped doing work, doesn't matter how many object collection contains. You can filter Java Collections like 
              <span style="font-family: Courier New, Courier, monospace;">List</span>, 
              <span style="font-family: Courier New, Courier, monospace;">Set </span>or 
              <span style="font-family: Courier New, Courier, monospace;">Map </span>in Java 8 by using 
              <span style="font-family: Courier New, Courier, monospace;">filter()</span> method of Stream class. You first need to obtain stream from Collection by calling 
              <span style="font-family: Courier New, Courier, monospace;">stream()</span> method and than you can use 
              <span style="font-family: Courier New, Courier, monospace;">filter()</span> method, which takes a 
              <span style="font-family: Courier New, Courier, monospace;">Predicate</span> as only argument. Predicate is a functional interface with couple of boolean valued method e.g. 
              <span style="font-family: Courier New, Courier, monospace;">test()</span>, which returns 
              <span style="font-family: Courier New, Courier, monospace;">boolean</span> true or false.
              <i> Java 8 uses this method to filter Collection</i>. Just remember that filter doesn't remove elements which matches the condition given in predicate, instead it selects them in output stream. I agree, little bit counter intuitive, select would have been better name for this method, but once you used it couple of times, you will be Ok. For example, if you have list of String and you want another list which contains only long string say whose length is greater than 20 character, you can use filter method to do that and you don't need a loop. 
              <a href="http://java67.blogspot.sg/2014/04/java-8-stream-api-examples-filter-map.html" rel="nofollow" target="_blank">Java 8 Stream</a> is very efficient replacement of looping both design and performance wise, because it separates What to do from how to do, just like SQL. Leaving the implementation part to platform.

               <h2 style="text-align: left;"> <u>Java 8 Filter Method</u></h2> As I said 
               <span style="font-family: Courier New, Courier, monospace;">filter()</span> method is defined in 
               <span style="font-family: Courier New, Courier, monospace;">Stream</span> class, it takes a 
               <span style="font-family: Courier New, Courier, monospace;">Predicate</span> object and returns a stream consisting of the elements from original stream which matches the given Predicate. It is an intermediate operation, which means you can call any other method of Stream e.g. 
               <span style="font-family: Courier New, Courier, monospace;">count()</span> as stream will not be closed due to filtering. Predicate is applied to each element of Collection to check whether a particular element should be included in filtered stream or not. Predicate should be stateless and non interfering so that if needed filter operation can run in parallel using 
               <i>parallel stream. In Java8</i>, Predicate is a functional interface with couple of 
               <span style="font-family: Courier New, Courier, monospace;">boolean</span> valued method used to test input against the condition. In our case 
               <span style="font-family: Courier New, Courier, monospace;">test(T t)</span> method is used, which evaluate this predicate in given argument. Since 
               <span style="font-family: Courier New, Courier, monospace;">filter()</span> method accept a functional interface, you can also pass 
               <a href="http://javarevisited.blogspot.sg/2014/02/10-example-of-lambda-expressions-in-java8.html" target="_blank">lambda expression</a> to it, which is what we will do. You can pass any condition to filter elements e.g. either by using relational operator e.g. less than, greater than or equal to or by using methods like 
               <span style="font-family: Courier New, Courier, monospace;">equals()</span> and 
               <span style="font-family: Courier New, Courier, monospace;">equalsIgnoreCase()</span> as shown in our sample program.&nbsp;
