       <h1><a href="http://blog.trifork.com/2011/08/23/filtering-specific-exceptions-when-using-log4j/" rel="bookmark" title="Permanent Link to Filtering specific exceptions when using log4j">Filtering specific exceptions when using log4j</a></h1> 
       <small>August 23rd, 2011 by <span class="author"><a href="http://blog.trifork.com/author/jelmer/" title="Posts by Jelmer Kuperus" rel="author">Jelmer Kuperus</a></span><br></small> 
       <small class="edit"> | </small> 
       <div class="entry"> 
        <a id="dd_start"></a>
        <p><a href="http://blog.jteam.nl/wp-content/uploads/2011/08/log4j-logo.jpg"><img src="http://blog.jteam.nl/wp-content/uploads/2011/08/log4j-logo.jpg" alt="" title="log4j-logo" width="207" height="139" class="alignleft size-full wp-image-3703"></a>A while ago I was working on a project where a background job was being triggered every few seconds or so. This job would call a method on a service, that connected to a remote backend, that was regularly unavailable.</p> 
        <p>When the backend was down, the logs would be flooded with long stacktraces every few seconds. Making it all but impossible to spot actual errors&nbsp; amid the carnage.</p> 
        <p>To solve this problem I could have disabled logging for this service completely. But in this case that seemed like a bad idea. This service was, by all means and purposes a kitchen sink. By disabling logging completely I would run the risk of also swallowing legitimate errors. What I really wanted to do was filter out only this “BackendNotAvailableException”</p> 
        <p>Fortunately there is a way to do that. Contained within <a href="http://logging.apache.org/log4j/companions/extras/index.html">Apache Extras Companion for Apache log4j</a> there is a filter called <a href="http://svn.apache.org/repos/asf/logging/site/trunk/docs/log4j/companions/extras/apidocs/org/apache/log4j/filter/ExpressionFilter.html">ExpressionFilter</a> that you can use to do exactly that. Here’s an example of how to configure it for this usecase.</p> 
        <pre class="brush: xml; title: ; notranslate" title="">
&lt;?xml version="1.0" encoding="UTF-8"?&gt;
&lt;!DOCTYPE log4j:configuration SYSTEM "log4j.dtd"&gt;
&lt;log4j:configuration xmlns:log4j="http://jakarta.apache.org/log4j/" debug="false"&gt;
    &lt;appender name="CONSOLE"&gt;
        &lt;param name="Target" value="System.out"/&gt;
        &lt;layout&gt;
            &lt;param name="ConversionPattern" value="%d %p [%c] - %m%n"/&gt;
        &lt;/layout&gt;
        &lt;filter class="org.apache.log4j.filter.ExpressionFilter"&gt;
            &lt;param name="expression" value="EXCEPTION ~= com.company.BackendNotAvailableException" /&gt;
            &lt;param name="acceptOnMatch" value="false"/&gt;
        &lt;/filter&gt;
    &lt;/appender&gt;
    &lt;root&gt;
        &lt;priority value ="INFO" /&gt;
        &lt;appender-ref ref="CONSOLE"/&gt;
    &lt;/root&gt;
&lt;/log4j:configuration&gt;
</pre> 
        <p>There is actually a lot more you can do with this filter and i encourage you to check it out in greater detail</p>
