     <h3><span class="mw-headline" id="What_is_a_squidGuard">What is a squidGuard</span></h3> 
     <p>squidGuard is a URL redirector used to integrate blacklists with the Squid proxy software. There are two big advantages to squidGuard: it is fast and it is free. squidGuard is published under the GNU Public License. </p>
     <p><b>squidGuard can be used to</b> </p> 
     <ul>
      <li> Limit the web access for some users to a list of accepted/well known web servers and/or URLs only. </li> 
      <li> Block access to some listed or blacklisted web servers and/or URLs for some users.</li> 
      <li> Block access to URLs matching a list of regular expressions or words for some users.</li> 
      <li> Enforce the use of domain names/prohibit the use of IP addresses in URLs.</li> 
      <li> Redirect blocked URLs to an info page.</li> 
      <li> Redirect banners to an empty GIF.</li> 
      <li> Have different access rules based on time of day, day of the week, date etc.</li>
     </ul> 
     <h3><span class="mw-headline" id="Installation_Squid2.2F3_.2B_squidGuard_on_pfSense_2.1.x">Installation Squid2/3 + squidGuard on pfSense 2.1.x</span></h3> 
     <ol>
      <li> Open Packages list: click <b>System &gt; Packages</b>, <b>Available Packages</b> tab</li> 
      <li> Install the Squid package if it is not already installed.</li> 
      <li> Install the squidGuard package</li> 
      <li> Configure Squid package.</li> 
      <li> Configure squidGuard package.</li>
     </ol> 
     <h2><span class="mw-headline" id="Configure_the_squidGuard_Package">Configure the squidGuard Package</span></h2> 
     <h3><span class="mw-headline" id="Blacklist">Blacklist</span></h3> 
     <p>Blacklists are optional, but often useful for allowing access to certain types of sites. </p>
     <p>squidGuard comes with a small blacklist basically for testing purposes. They should not be used in production. A better way is to start with one of the blacklist collections listed (alphabetically) below. </p> 
     <ul>
      <li> <a rel="nofollow" class="external text" href="http://squidguard.mesd.k12.or.us/blacklists.tgz">MESD blacklists</a> - They are freely available.</li> 
      <li> <a rel="nofollow" class="external text" href="http://www.shallalist.de/Downloads/shallalist.tar.gz">Shalla's Blacklists</a> - Free for non commercial/private use. (Recommended)</li> 
      <li> <a rel="nofollow" class="external text" href="http://www.squidguard.org/blacklists.html">more..</a></li>
     </ul> 
     <p><b>Downloading blacklist:</b> </p> 
     <ol>
      <li> Open <b>General Settings</b> tab in squidGuard package GUI, found at <b>Services &gt; Proxy Filter</b>.</li> 
      <li> Check <b>Blacklist</b> to enable the use of blacklists</li> 
      <li> Enter blacklist URL in the field <b>Blacklist URL</b>.</li> 
      <li> If the firewall is itself behind a proxy, enter the proxy information in <b>Blacklist proxy</b> (this step is not necessary for most people)</li> 
      <li> Click <b>Save</b></li> 
      <li> Navigate to the <b>Blacklist</b> tab inside of squidGuard</li> 
      <li> Click the <b>Download</b> button.</li> 
      <li> Wait while blacklist will downloaded and prepared to use (10-35 min). Progress will be displayed on that page as the list is downloaded and processed.</li>
     </ol> 
     <h3><span class="mw-headline" id="Basic_configuration">Basic configuration</span></h3> 
     <p>Here describes how to enable and configure squidGuard, and common users access. </p> 
     <ol>
      <li> Open <b>General settings</b> tab. 
       <ol>
        <li> Check the <b>Enable</b> box to activate the package.</li> 
        <li> Set <b>Blacklist</b> options to use blacklist categories. (See above, optional)</li> 
        <li> Click <b>Save</b> button.</li>
       </ol></li> 
      <li> Open <b>Common ACL</b> page. 
       <ol>
        <li> Click <b>Target Rules List</b> to show defined blacklists and target categories 
         <ol>
          <li> Define default user access: select <b>Default access [all]</b> as <i>allow</i> or <i>deny</i>.</li> 
          <li> Define other category actions: 
           <ol>
            <li> Select <i>---</i>, to ignore a category.</li> 
            <li> Select <i>allow</i>, to allow this category for clients.</li> 
            <li> Select <i>deny</i>, to deny this category for clients.</li> 
            <li> Select <i>white</i>, to allow this category without any restrictions. This option is used for exceptions to prohibited categories.</li>
           </ol></li> 
          <li> To prohibit clients from using IP addresses in URLs, check <b>Do Not Allow IP Addresses in URL</b>.</li> 
          <li> Select <b>Redirect mode</b>: 
           <ol>
            <li> <i>Int error page</i>: Use the built-in error page. A custom message may be entered in the <b>Redirect info</b> box below.</li> 
            <li> <i>Int blank page</i>: Redirect to a blank page</li> 
            <li> The other options are various redirects to external error pages, and a URL must be entered in the <b>Redirect info</b> box if they are chosen.</li>
           </ol></li> 
          <li> <b>Use safe search engine</b>: Protect customers from unwanted search results. It is supported by <i>Google, Yandex, Yahoo, MSN, Live Search</i>. Make sure that these search engines are available. If this protection should be strictly enforced, disable access to all other search engines.</li>
         </ol></li>
       </ol></li> 
      <li> After settings are complete, return to the <b>General Settings</b> tab and press <b>Apply</b>.</li>
     </ol>

     <h2><span class="mw-headline" id="HowTo">HowTo</span></h2> 
     <h3><span class="mw-headline" id="Exclude_domain.2FURL_from_blacklist">Exclude domain/URL from blacklist</span></h3> 
     <p>In the squidGuard GUI (<b>Services &gt; Proxy Filter</b>): </p> 
     <ol>
      <li> Open the <b>Target categories</b> page</li> 
      <li> Click <a href="/index.php/File:Icon_plus.png" class="image" title="+"><img alt="&quot;+&quot;" src="/images/0/0c/Icon_plus.png" width="17" height="17"></a> to add a new item</li> 
      <li> Enter a name for the category - <i>myWhitelist</i> for example.</li> 
      <li> Add domains and/or URLs to the lists as needed. Entries should be separated by a space. The examples on the page show how entries should be formatted.</li> 
      <li> As with the Common ACL discussed previously, redirect and logging options specific to this category may be set.</li> 
      <li> Click <b>Save</b></li> 
      <li> Open <b>Common ACL</b> or <b>Groups ACL</b> page (whichever should have an exclusion).</li> 
      <li> Click <b>Target Rule List</b> to expand the list of categories. The newly created category should show alphabetically in the list, above any blacklist categories. Find the <i>MyWhiteList</i> entry in the list and select <i>white</i>.</li> 
      <li> Click <b>Save</b></li> 
      <li> Return to the <b>General Settings</b> tab and press <b>Apply</b>.</li>
     </ol> 
     <h3><span class="mw-headline" id="Block_download_by_Extension">Block download by Extension</span></h3> 
     <p>In the squidGuard GUI (<b>Services &gt; Proxy Filter</b>): </p> 
     <ol>
      <li> Open the <b>Target categories</b> page</li> 
      <li> Click <a href="/index.php/File:Icon_plus.png" class="image" title="+"><img alt="&quot;+&quot;" src="/images/0/0c/Icon_plus.png" width="17" height="17"></a> to add a new item</li> 
      <li> Enter a name for the category - <i>myBlockExt</i> for example.</li> 
      <li> Add Expressions (for example for asf, zip, exe and etc files):</li>
     </ol> 
     <pre>(.*\/.*\.(asf|wm|wma|wmv|zip|rar|cab|mp3|avi|mpg|swf|exe|mpeg|mp.|mpv|mp3|wm.|vpu))
</pre> 
     <ol>
      <li> Click <b>Save</b></li> 
      <li> Open <b>Common ACL</b> or <b>Groups ACL</b> page (whichever should have an exclusion).</li> 
      <li> Click <b>Target Rule List</b> to expand the list of categories. The newly created category should show alphabetically in the list, above any blacklist categories. Find the <i>myBlockExt</i> entry in the list and select <i>deny</i>.</li> 
      <li> Click <b>Save</b></li> 
      <li> Return to the <i>General Settings</i> tab and press <b>Apply</b>.</li>
     </ol> 
     <h2><span class="mw-headline" id="Troubleshooting">Troubleshooting</span></h2> 
     <h3><span class="mw-headline" id="Netflix">Netflix</span></h3> 
     <p>If Netflix will not load while squidGuard is active, it is likely because Netflix requires accessing URLs by IP address. Ensure that ACLs matching clients allowed to reach Netflix also <i>do not have</i> <b>Do not allow IP-Addresses in URL</b> checked. </p> 
     <h3><span class="mw-headline" id="Service_does_not_Start">Service does not Start</span></h3> 
     <p>If the squidGuard service will not start, there are a few possible explanations: </p> 
     <ul>
      <li> On all versions of Squid, if <b>only</b> blacklists have been configured, then at startup some important files/directories may not be set properly. Add at least one Custom Target Category with a site to pass or block and use it along with the blacklist entries to work around the problem.</li> 
      <li> On squid 3.x, the squidGuard service will only start when traffic requires it to run, so it can appear to be stopped even when working properly. Only worry about the service if it appears to not work, don't count on the service status alone.</li>
     </ul>
