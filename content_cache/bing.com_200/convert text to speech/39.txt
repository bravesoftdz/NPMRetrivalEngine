          <h1>Speech Synthesis with ASP.NET and HTML5</h1> 
          <div class="metadata"> 
           <time class="clearfix" datetime="2014-04-08" pubdate>Tuesday, April 8, 2014</time> 
           <div class="tags"> 
            <a class="tag label label-default" rel="tag" href="/ricardoperes/Tags/.NET">.NET</a> 
            <a class="tag label label-default" rel="tag" href="/ricardoperes/Tags/AJAX">AJAX</a> 
            <a class="tag label label-default" rel="tag" href="/ricardoperes/Tags/ASP.NET">ASP.NET</a> 
            <a class="tag label label-default" rel="tag" href="/ricardoperes/Tags/ASP.NET%20AJAX">ASP.NET AJAX</a> 
            <a class="tag label label-default" rel="tag" href="/ricardoperes/Tags/Client%20Callbacks">Client Callbacks</a> 
            <a class="tag label label-default" rel="tag" href="/ricardoperes/Tags/Controls">Controls</a> 
            <a class="tag label label-default" rel="tag" href="/ricardoperes/Tags/HTML">HTML</a> 
            <a class="tag label label-default" rel="tag" href="/ricardoperes/Tags/HTML5">HTML5</a> 
            <a class="tag label label-default" rel="tag" href="/ricardoperes/Tags/JavaScript">JavaScript</a> 
            <a class="tag label label-default" rel="tag" href="/ricardoperes/Tags/Web">Web</a> 
            <a class="tag label label-default" rel="tag" href="/ricardoperes/Tags/Speech">Speech</a> 
           </div> 
          </div> 
         </header> 
         <p align="justify">The .NET framework includes the <a href="http://msdn.microsoft.com/en-us/library/system.speech.synthesis.speechsynthesizer.aspx" target="_blank">SpeechSynthesizer</a> class which can be used to access the Windows speech synthesis engine. The problem with web applications is, of course, this class runs on the server. Because I wanted a mechanism to have speech synthesis (text-to-speech) fired by JavaScript, without requiring any plugin, I decided to implement one myself.</p> 
         <p align="justify"><a href="http://weblogs.asp.net/ricardoperes/archive/tags/AJAX/Client+Callbacks/" target="_blank">Once again</a>, I will be using <a href="http://msdn.microsoft.com/en-us/library/ms178208.aspx" target="_blank">client callbacks</a>, my out-of-the-box ASP.NET favorite AJAX technique. I will also be using HTML5’s <a href="https://developer.mozilla.org/en-US/docs/Web/HTML/Element/audio" target="_blank">AUDIO</a> tag and <a href="https://developer.mozilla.org/en/docs/data_URIs" target="_blank">Data URIs</a>. What I’m going to do is:</p> 
         <ul> 
          <li> 
           <div align="justify">
            Set up a control that renders an 
            <a href="https://developer.mozilla.org/en-US/docs/Web/HTML/Element/audio" target="_blank">AUDIO</a> tag;
           </div> </li> 
          <li> 
           <div align="justify">
            Add to it a JavaScript function that takes a string parameter and causes a client callback;
           </div> </li> 
          <li> 
           <div align="justify">
            Generate a voice sound from the passed text parameter on the server and save it into an in-memory stream;
           </div> </li> 
          <li> 
           <div align="justify">
            Convert the stream’s contents to a 
            <a href="https://developer.mozilla.org/en/docs/data_URIs" target="_blank">Data URI</a>;
           </div> </li> 
          <li> 
           <div align="justify">
            Return the generated 
            <a href="https://developer.mozilla.org/en/docs/data_URIs" target="_blank">Data URI</a> to the client as the response to the client callback.
           </div> </li> 
         </ul> 
         <p align="justify">Of course, all of this in cross-browser style (provided your browser knows the <a href="https://developer.mozilla.org/en-US/docs/Web/HTML/Element/audio" target="_blank">AUDIO</a> tag and <a href="https://developer.mozilla.org/en/docs/data_URIs" target="_blank">Data URIs</a>, which all modern browsers do).</p> 
         <p align="justify">So, first of all, my markup looks like this:</p> 
         <div id="codeSnippetWrapper" style="overflow: auto; cursor: text; font-size: 8pt; font-family: 'Courier New', courier, monospace; width: 97.5%; direction: ltr; text-align: left; margin: 20px 0px 10px; line-height: 12pt; max-height: 200px; background-color: #f4f4f4; border: silver 1px solid; padding: 4px;"> 
          <div id="codeSnippet" style="overflow: visible; font-size: 8pt; font-family: 'Courier New', courier, monospace; width: 100%; color: black; direction: ltr; text-align: left; line-height: 12pt; background-color: #f4f4f4; border-style: none; padding: 0px;"> 
           <pre style="overflow: visible; font-size: 8pt; font-family: 'Courier New', courier, monospace; width: 100%; color: black; direction: ltr; text-align: left; margin: 0em; line-height: 12pt; background-color: white; border-style: none; padding: 0px;"><span id="lnum1" style="color: #606060;">   1:</span> <span style="color: #0000ff;">&lt;</span><span style="color: #800000;">web:SpeechSynthesizer</span> <span style="color: #ff0000;">runat</span><span style="color: #0000ff;">="server"</span> <span style="color: #ff0000;">ID</span><span style="color: #0000ff;">="synthesizer"</span> <span style="color: #ff0000;">Ssml</span><span style="color: #0000ff;">="false" <span style="color: #ff0000;">VoiceName</span><span style="color: #0000ff;">="Microsoft Anna"</span></span> <span style="color: #ff0000;">Age</span><span style="color: #0000ff;">="Adult"</span> <span style="color: #ff0000;">Gender</span><span style="color: #0000ff;">="Male"</span> <span style="color: #ff0000;">Culture</span><span style="color: #0000ff;">="en-US"</span> <span style="color: #ff0000;">Rate</span><span style="color: #0000ff;">="0"</span> <span style="color: #ff0000;">Volume</span><span style="color: #0000ff;">="100"</span> <span style="color: #0000ff;">/&gt;</span></pre> 
           <!--CRLF-->
          </div> 
         </div> 
         <p align="justify">As you can see, the <strong>SpeechSynthesizer</strong> control features a few optional properties:</p> 
         <ul> 
          <li><strong><a href="http://msdn.microsoft.com/en-us/library/system.speech.synthesis.voiceinfo.age.aspx" target="_blank">Age</a></strong>: the age for the generated voice (default is the one of the system’s default language);</li> 
          <li><strong><a href="http://msdn.microsoft.com/en-us/library/system.speech.synthesis.voiceinfo.gender.aspx" target="_blank">Gender</a></strong>: gender of the generated voice (same default as per <strong>Age</strong>);</li> 
          <li><strong><a href="http://msdn.microsoft.com/en-us/library/system.speech.synthesis.voiceinfo.culture.aspx" target="_blank">Culture</a></strong>: the culture of the generated voice (system default);</li> 
          <li><strong><a href="http://msdn.microsoft.com/en-us/library/system.speech.synthesis.speechsynthesizer.rate.aspx" target="_blank">Rate</a></strong>: the speaking rate, from –10 (fastest) to 10 (slowest), where the default is 0 (normal rate);</li> 
          <li><strong>Ssml</strong>: if the text is to be considered <a href="http://www.w3.org/TR/speech-synthesis/" target="_blank">SSML</a> or not (default is false);</li> 
          <li><strong><a href="http://msdn.microsoft.com/en-us/library/system.speech.synthesis.speechsynthesizer.volume.aspx" target="_blank">Volume</a></strong>: the volume %, between 0 and 100 (default);</li> 
          <li><strong>VoiceName</strong>: the name of a voice that is installed on the server machine.</li> 
         </ul> 
         <p align="justify">The <strong>Age</strong>, <strong>Gender</strong> and <strong>Culture</strong> properties and the <strong>VoiceName</strong> are exclusive, you either specify one or the other. If no property is specified, the speech will be synthesized with the operating system’s default (<strong>Microsoft Anna</strong> on Windows 7, <strong>Microsoft Dave</strong>, <strong>Hazel</strong> and <strong>Zira</strong> on Windows 8, etc). By the way, you can get additional voices, either commercially or for free, just look them up in Google.</p>

         <p align="justify">As you can see, the <strong>SpeechSynthesizer</strong> control inherits from <a href="http://msdn.microsoft.com/en-us/library/system.web.ui.htmlcontrols.htmlgenericcontrol.aspx" target="_blank">HtmlGenericControl</a>, this is the simplest out-of-the-box class that will allow me to render my tag of choice (in this case, <a href="https://developer.mozilla.org/en-US/docs/Web/HTML/Element/audio" target="_blank">AUDIO</a>); by the way, this class requires that I decorate it with a <a href="http://msdn.microsoft.com/en-us/library/system.web.ui.constructorneedstagattribute.aspx" target="_blank">ConstructorNeedsTagAttribute</a>, but you don’t have to worry about it. It implements <a href="http://msdn.microsoft.com/en-us/library/system.web.ui.icallbackeventhandler.aspx" target="_blank">ICallbackEventHandler</a> for the client callback mechanism. I make sure that all of <a href="https://developer.mozilla.org/en-US/docs/Web/HTML/Element/audio" target="_blank">AUDIO</a>’s attributes are removed from the output, because I don’t want them around.</p> 
         <p align="justify">Inside of it, I have an instance of the <a href="http://msdn.microsoft.com/en-us/library/system.speech.synthesis.speechsynthesizer.aspx" target="_blank">SpeechSynthesizer</a> class, the one that will be used to do the actual work. Because this class is disposable, I make sure it is disposed at the end of the control’s life cycle. Based on the parameters being supplied, I either call the <a href="http://msdn.microsoft.com/en-us/library/ms586875.aspx" target="_blank">SelectVoiceByHints</a> or the <a href="http://msdn.microsoft.com/en-us/library/system.speech.synthesis.speechsynthesizer.selectvoice.aspx" target="_blank">SelectVoice</a> methods. One thing to note is, we need to set up a synchronization context, because the <a href="http://msdn.microsoft.com/en-us/library/system.speech.synthesis.speechsynthesizer.aspx" target="_blank">SpeechSynthesizer</a> works asynchronously, so that we can wait for its result.</p> 
         <p align="justify">The generated sound will be output to an in-memory buffer and then converted into a <a href="http://en.wikipedia.org/wiki/WAV" target="_blank">WAV</a> <a href="https://developer.mozilla.org/en/docs/data_URIs" target="_blank">Data URI</a>, which is basically a <a href="http://en.wikipedia.org/wiki/Base64" target="_blank">Base64</a> encoded string with an associated mime-type.</p> 
         <p align="justify">Finally, on the client-side, all it takes is to set the returned <a href="https://developer.mozilla.org/en/docs/data_URIs" target="_blank">Data URI</a> as the <a href="https://developer.mozilla.org/en-US/docs/Web/HTML/Element/audio" target="_blank">AUDIO</a> <strong>SRC</strong> property, and that's it.</p>
