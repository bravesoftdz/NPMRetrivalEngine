      <strong>Microsoft Speech API 5.3</strong> 
      <h1>Text-to-Speech Tutorial</h1> 
      <p>This tutorial covers a very basic text-to-speech (TTS) example. The console application is one of the simplest demonstrations of speech. It is the "Hello World" equivalent for TTS. An equivalent sample for a Windows application using a graphical interface (and event pump) is available in <a href="https://msdn.microsoft.com/en-us/library/ms720165(v=vs.85).aspx">Using Events with TTS</a>.</p> 
      <p>The sample builds up from the simplest (though nonfunctional) COM framework to speaking a sentence. Steps are provided for each new function. The sample even goes one step beyond demonstrating the use XML tags to modify speech. The <a href="#ModifyingSpeech">Complete Sample Application</a> is at the bottom of the page.</p> 
      <a href="#SettingUpTheProject">Step 1: Setting Up The Project</a>
      <br> 
      <a href="#InitializeCOM">Step 2: Initialize COM</a>
      <br> 
      <a href="#SettingUpVoices">Step 3: Setting Up Voices</a>
      <br> 
      <a href="#Speak">Step 4: Speak!</a>
      <br> 
      <a href="#ModifyingSpeech">Step 5: Modifying Speech</a>
      <br> 
      <h2><a id="SettingUpTheProject"></a><strong>Step 1: Setting up the project</strong></h2> 
      <p>While it is possible to write an application from scratch, it is easier to start from an existing project. In this case, use Visual Studio's application wizard to create a Win32 console application. Choose "Hello, world" as the sample when asked during the wizard set up. After generating it, open the STDAfx.h file and paste the following code after "#include &lt;stdio.h&gt;" but before the "#endif" statement. This sets up the additional dependencies SAPI requires.</p> 
      <pre><code>
#define _ATL_APARTMENT_THREADED

#include &lt;atlbase.h&gt;
//You may derive a class from CComModule and use it if you want to override something,
//but do not change the name of _Module
extern CComModule _Module;
#include &lt;atlcom.h&gt;
</code></pre> 
      <p>Code Listing 1</p> 
      <p>Next add the paths to SAPI.h and SAPI.lib files. The paths shown are for a standard SAPI SDK install. If the compiler is unable to locate either file, or if a nonstandard install was performed, use the new path to the files. Change the project settings to reflect the paths. Using the Project-&gt;Settings. menu item, set the SAPI.h path. Click the C/C++ tab and select Preprocessor from the Category drop-down list. Enter the following in the "Additional include directories": C:\Program Files\Microsoft Speech SDK 5.3\Include.</p> 
      <p>To set the SAPI.lib path:</p> 
      <ol>
       <li>Select the Link tab from the Same Settings dialog box.</li> 
       <li>Choose Input from the Category drop-down list.</li> 
       <li>Add the following path to the "Additional library path": <br>C:\Program Files\Microsoft Speech SDK 5.3\Lib\i386.</li> 
       <li>Also add "sapi.lib" to the "Object/library modules" line. Be sure that the name is separated by a space. </li>
      </ol> 
      <h2><a id="InitializeCOM"></a><strong>Step 2: Initialize COM</strong></h2> 
      <p>SAPI is a COM-based application, and COM must be initialized both before use and during the time SAPI is active. In most cases, this is for the lifetime of the host application. The following code (from Listing 2) initializes COM. Of course, the application does not do anything beyond initialization, but it does ensure that COM is successfully started.</p> 
      <pre><code>
#include &lt;stdafx.h&gt;
#include &lt;sapi.h&gt;

int main(int argc, char* argv[])
{
    if (FAILED(::CoInitialize(NULL)))
        return FALSE;

    ::CoUninitialize();
    return TRUE;
}
</code></pre> 
      <p>Code Listing 2</p> 
      <h2><a id="SettingUpVoices"></a><strong>Step 3: Setting up voices</strong></h2> 
      <p>Once COM is running, the next step is to create the voice. A voice is simply a COM object. Additionally, SAPI uses intelligent defaults. During initialization of the object, SAPI assigns most values automatically so that the object may be used immediately afterward. This represents an important improvement from earlier versions. The defaults are retrieved from Speech properties in Control Panel and include such information as the voice (if more than one is available on your system), and the language (English, Japanese, etc.). While some defaults are obvious, others are not (speaking rate, pitch, etc.). Nevertheless, all defaults may be changed either programmatically or in Speech properties in Control Panel.</p>
