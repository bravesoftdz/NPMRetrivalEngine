            <h1 class="post-name">SharePoint 2013: Build a Webpart using HTML, JavaScript and the XML Viewer Web Part</h1> 
            <div class="post-content user-defined-markup"> 
             <p>With the rise of SharePoint 2013 and Apps for SharePoint, client side object model (CSOM) and JavaScript object model (JSOM) are becoming the norm for development on SharePoint.&nbsp; In SharePoint 2013 the CSOM and JSOM object models have been greatly expanded and optimized making client side development a joy.&nbsp; Even in SharePoint 2010, these object models allowed for a lot of development opportunities.&nbsp; If you needed something more robust, you could very easily deploy a WCF web service and make AJAX calls to the server.<br> <br> With this thinking in mind, we can stop building custom web parts which are deployed to the server and instead build functionality using JSOM and the out of the box XML View web part.&nbsp; Let's take a look at how to accomplish this.</p> 
             <p>This idea came from a colleague I work with by the name of Mike Etheridge.&nbsp; We were&nbsp;working on a project building custom web parts and this technique was being used in conjunction with JSOM and custom WCF web services. <br> <br> The complete code for this post can be downloaded from the <a href="http://gallery.technet.microsoft.com/sharepoint/Use-the-XML-Viewer-Web-2a36af59" target="_blank"> <span style="color:#336699;">TechNet Gallery here</span></a>.<br> <br> There are&nbsp;only&nbsp;a few components to this: </p> 
             <ul> 
              <li><strong>XML Viewer Web Part</strong> - This is the OOTB XML View web part that comes with SharePoint. </li>
              <li><strong>XML File</strong> - This is a standard XML file that contains HTML markup that will be rendered in the XML Viewer web part. </li>
              <li><strong>JavaScript File</strong> - This is a standard JavaScript file with which to make JSOM and/or jQuery calls from. </li>
              <li><strong>CSS File</strong> - A standard CSS file to applying styling to your HTML. </li>
             </ul> To start off, we'll create a new document library to hold our XML and later CSS and JavaScript files.&nbsp; For this demo it will be a generic document library called "XmlWebParts".
             <br> 
             <br> 
             <br> 
             <div class="separator" style="clear:both;text-align:center;">
              <a href="http://1.bp.blogspot.com/-NwC9bLfxm3k/UkxPemRsTRI/AAAAAAAAA1o/Euseaw9AS_0/s1600/1-Creating-the-library.png" style="margin-right:1em;margin-left:1em;"><img alt=" " width="320" height="166" src="http://1.bp.blogspot.com/-NwC9bLfxm3k/UkxPemRsTRI/AAAAAAAAA1o/Euseaw9AS_0/s320/1-Creating-the-library.png" style="border-width:0px;border-style:solid;"></a>
             </div> 
             <br> 
             <br> Inside this library I've created a folder to hold the first demo files.&nbsp; Inside the folder is an XML file titled "HelloWorld.xml".
             <br> 
             <br> 
             <div class="separator" style="clear:both;text-align:center;">
              <a href="http://3.bp.blogspot.com/-2kETeeIsvfA/UkxP0a6eCxI/AAAAAAAAA1w/-kCkAE__39A/s1600/2-Hello-World-Xml.png" style="margin-right:1em;margin-left:1em;"><img alt=" " width="320" height="157" src="http://3.bp.blogspot.com/-2kETeeIsvfA/UkxP0a6eCxI/AAAAAAAAA1w/-kCkAE__39A/s320/2-Hello-World-Xml.png" style="border-width:0px;border-style:solid;"></a>
             </div> 
             <br> 
             <br> This is a simple XML file that only contains HTML markup.&nbsp; The markup is very simple and only contains two DIVs and a Hello World statement, some formatting, and another line of text.
             <br> 
             <br> 
             <br> 
             <div class="reCodeBlock" style="border:1px solid #7f9db9;font-family:consolas;font-size:12px;"> 
              <div style="background-color:#ffffff;">
               <span><code style="color:#000000;">&lt;</code><code style="color:#006699;font-weight:bold;">div</code><code style="color:#000000;">&gt;Hello &lt;</code><code style="color:#006699;font-weight:bold;">strong</code><code style="color:#000000;">&gt;World!&lt;/</code><code style="color:#006699;font-weight:bold;">strong</code><code style="color:#000000;">&gt;&lt; /</code><code style="color:#006699;font-weight:bold;">div</code><code style="color:#000000;">&gt;&lt; </code><code style="color:#006699;font-weight:bold;">div</code><code style="color:#000000;">&gt; This is another DIV.&lt; /</code><code style="color:#006699;font-weight:bold;">div</code><code style="color:#000000;">&gt;</code></span>
              </div> 
             </div> 
             <br> Now that we have that, we can create a new blank site page, and add an XML Viewer web part to it.&nbsp; Opening the properties pane of the web part, we only need to set the "XML Link" property to point to our new XML file.
             <br> 
             <br> 
             <br> 
             <div class="separator" style="clear:both;text-align:center;">
              <a href="http://1.bp.blogspot.com/-sClQvdvBRZY/UkxRC2VHRLI/AAAAAAAAA18/MrkotcqnmF4/s1600/3-Setting-Path-To-XML.png" style="margin-right:1em;margin-left:1em;"><img alt=" " width="215" height="320" src="http://1.bp.blogspot.com/-sClQvdvBRZY/UkxRC2VHRLI/AAAAAAAAA18/MrkotcqnmF4/s320/3-Setting-Path-To-XML.png" style="border-width:0px;border-style:solid;"></a>
             </div> 
             <br> 
             <br> In this example the file lives at "/sites/blogexamples/XmlWebParts/HelloWorld/HelloWorld.xml", so this is the path placed into the "XML Link" property.&nbsp; Saving this and checking in the page results in the HTML content from the XML file being displayed in the web part.
             <br> 
             <br> 
             <br> 
             <div class="separator" style="clear:both;text-align:center;">
              <a href="http://3.bp.blogspot.com/-mWOM357wDZI/UkxRo3CafcI/AAAAAAAAA2E/lK5iopDBrfs/s1600/4-XML-file-being-displayed.png" style="margin-right:1em;margin-left:1em;"><img alt=" " width="320" height="144" src="http://3.bp.blogspot.com/-mWOM357wDZI/UkxRo3CafcI/AAAAAAAAA2E/lK5iopDBrfs/s320/4-XML-file-being-displayed.png" style="border-width:0px;border-style:solid;"></a>
             </div> 
             <br> Now obviously this demo is pretty simple and not "real world" by any means.&nbsp; So, let's translate this into something real world.&nbsp; Let's say your manager has asked you to develop a web part (SharePoint 2013)&nbsp;for the SharePoint homepage which has the following requirements:
             <br> 
             <ul> 
              <li>Web part must display information for the current user logged in from their user profile and social feeds. </li>
              <li>Web part must display a welcome message to the user above their user profile picture. </li>
              <li>Under the welcome message and profile picture, the web part must display any items the user is following, along with a link to that item for easy access. </li>
             </ul> We can easily produce this web part using HTML, jQuery, JSOM, and an XML Viewer web part.&nbsp; First let's start with building out the files needed for this web part.&nbsp; We'll create another folder in our document library called "UserWelcome".
             <br> 
             <br> 
             <br> 
             <div class="separator" style="clear:both;text-align:center;">
              <a href="http://1.bp.blogspot.com/-_qrB6fbr5Y0/UkxqbFVeWFI/AAAAAAAAA2U/H3tQ_Tvg4Fo/s1600/5-User-Welcome-Folder.png" style="margin-right:1em;margin-left:1em;"><img alt=" " width="320" height="177" src="http://1.bp.blogspot.com/-_qrB6fbr5Y0/UkxqbFVeWFI/AAAAAAAAA2U/H3tQ_Tvg4Fo/s320/5-User-Welcome-Folder.png" style="border-width:0px;border-style:solid;"></a>
             </div> 
             <br> Inside that folder we'll place an XML file and a JavaScript file.
             <br> 
             <br> 
             <div class="separator" style="clear:both;text-align:center;">
              <a href="http://4.bp.blogspot.com/-e9oX7ZpjCcg/UkxqtT7nvwI/AAAAAAAAA2c/bmLC9FGXGb0/s1600/6-User-Welcome-Files.png" style="margin-right:1em;margin-left:1em;"><img alt=" " width="320" height="166" src="http://4.bp.blogspot.com/-e9oX7ZpjCcg/UkxqtT7nvwI/AAAAAAAAA2c/bmLC9FGXGb0/s320/6-User-Welcome-Files.png" style="border-width:0px;border-style:solid;"></a>
             </div> 
             <br> 
             <br> Now we have two files, an XML file for our HTML markup and a JavaScript file.&nbsp; Let's take a look at the HTML markup in the XML file.
             <br> 
             <br> 
             <br> 
             <div class="reCodeBlock" style="border:1px solid #7f9db9;font-family:consolas;font-size:12px;"> 
              <div style="background-color:#ffffff;">
               <span><code style="color:#000000;">&lt;</code><code style="color:#006699;font-weight:bold;">script</code> <code style="color:#808080;">type</code><code style="color:#000000;">=</code><code style="color:blue;">"text/javascript"</code> <code style="color:#808080;">src</code><code style="color:#000000;">=</code><code style="color:blue;">"<a href="http://code.jquery.com/jquery-1.10.1.min.js">http://code.jquery.com/jquery-1.10.1.min.js</a>"</code><code style="color:#000000;">&gt;&lt;/</code><code style="color:#006699;font-weight:bold;">script</code><code style="color:#000000;">&gt;</code></span>
              </div> 
              <div style="background-color:#f8f8f8;">
               <span><code style="color:#000000;">&lt;</code><code style="color:#006699;font-weight:bold;">script</code> <code style="color:#808080;">type</code><code style="color:#000000;">=</code><code style="color:blue;">"text/javascript"</code> <code style="color:#808080;">src</code><code style="color:#000000;">=</code><code style="color:blue;">"/_layouts/15/sp.runtime.js"</code><code style="color:#000000;">&gt;&lt;/</code><code style="color:#006699;font-weight:bold;">script</code><code style="color:#000000;">&gt;</code></span>
              </div> 
              <div style="background-color:#ffffff;">
               <span><code style="color:#000000;">&lt;</code><code style="color:#006699;font-weight:bold;">script</code> <code style="color:#808080;">type</code><code style="color:#000000;">=</code><code style="color:blue;">"text/javascript"</code> <code style="color:#808080;">src</code><code style="color:#000000;">=</code><code style="color:blue;">"/_layouts/15/sp.js"</code><code style="color:#000000;">&gt;&lt;/</code><code style="color:#006699;font-weight:bold;">script</code><code style="color:#000000;">&gt;&lt; </code><code style="color:#006699;font-weight:bold;">script</code> <code style="color:#808080;"> type</code><code style="color:#000000;">=</code><code style="color:blue;">"text/javascript"</code> <code style="color:#808080;">src</code><code style="color:#000000;">=</code><code style="color:blue;">"/_layouts/15/sp.userprofiles.js"</code><code style="color:#000000;">&gt;&lt;/</code><code style="color:#006699;font-weight:bold;">script</code><code style="color:#000000;">&gt;</code></span>
              </div> 
              <div style="background-color:#f8f8f8;">
               <span><code style="color:#000000;">&lt;</code><code style="color:#006699;font-weight:bold;">script</code> <code style="color:#808080;">type</code><code style="color:#000000;">=</code><code style="color:blue;">"text/javascript"</code> <code style="color:#808080;">src</code><code style="color:#000000;">=</code><code style="color:blue;">"/sites/blogexamples/XmlWebParts/UserWelcome/UserWelcome.js"</code><code style="color:#000000;">&gt;&lt;/</code><code style="color:#006699;font-weight:bold;">script</code><code style="color:#000000;">&gt;</code></span>
              </div> 
              <div style="background-color:#ffffff;">
               <span><code style="color:#000000;">&lt;</code><code style="color:#006699;font-weight:bold;">div</code> <code style="color:#808080;">id</code><code style="color:#000000;">=</code><code style="color:blue;">"WelcomeMessage"</code><code style="color:#000000;">&gt;Welcome back &lt;</code><code style="color:#006699;font-weight:bold;">span</code> <code style="color:#808080;">id</code><code style="color:#000000;">=</code><code style="color:blue;">"WelcomeMessageUserName"</code><code style="color:#000000;">&gt;&lt;/</code><code style="color:#006699;font-weight:bold;">span</code><code style="color:#000000;">&gt;&lt;/</code><code style="color:#006699;font-weight:bold;">div</code><code style="color:#000000;">&gt;</code></span>
              </div> 
              <div style="background-color:#f8f8f8;">
               <span><code style="color:#000000;">&lt;</code><code style="color:#006699;font-weight:bold;">div</code> <code style="color:#808080;">id</code><code style="color:#000000;">=</code><code style="color:blue;">"UserProfilePicture"</code><code style="color:#000000;">&gt; &lt;</code><code style="color:#006699;font-weight:bold;">img</code> <code style="color:#808080;">id</code><code style="color:#000000;">=</code><code style="color:blue;">"UserProfileImage"</code> <code style="color:#000000;">/&gt;&lt;/</code><code style="color:#006699;font-weight:bold;">div</code><code style="color:#000000;">&gt;</code></span>
              </div> 
              <div style="background-color:#ffffff;">
               <span><code style="color:#000000;">&lt;</code><code style="color:#006699;font-weight:bold;">div</code> <code style="color:#808080;">id</code><code style="color:#000000;">=</code><code style="color:blue;">"UserFollows"</code><code style="color:#000000;">&gt;&lt;/</code><code style="color:#006699;font-weight:bold;">div</code><code style="color:#000000;">&gt;</code></span>
              </div> 
             </div> 
             <br> 
             <br> In this markup, the first thing we do is reference jQuery from a CDN, then we load in some specific SharePoint JavaScript files needed to pull user profile data.&nbsp; Finally we reference the UserWelcome.js file we uploaded.&nbsp; As you can see, the HTML is very minimal and only includes a few DIVs and an image placeholder.&nbsp; All the content will be loaded from the JavaScript file.&nbsp; Let's take a look at that.
             <br> 
             <em><strong><br> <br> 
               <div class="reCodeBlock" style="border:1px solid #7f9db9;font-family:consolas;font-size:12px;"> 
                <div style="background-color:#ffffff;">
                 <span><code style="color:#008200;">//&lt;![CDATA[</code></span>
                </div> 
                <div style="background-color:#f8f8f8;">
                 <span><code style="color:#000000;">$(</code><code style="color:#006699;font-weight:bold;">function</code> <code style="color:#000000;">() {</code></span>
                </div> 
                <div style="background-color:#ffffff;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#008200;">// Don't fire function until sp.userprofiles.js is loaded</code></span></span>
                </div> 
                <div style="background-color:#f8f8f8;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#000000;">ExecuteOrDelayUntilScriptLoaded(getUserProfileProperties, </code><code style="color:blue;">"sp.userprofiles.js"</code><code style="color:#000000;">);</code></span></span>
                </div> 
                <div style="background-color:#ffffff;">
                 <span><code style="color:#000000;">});</code></span>
                </div> 
                <div style="background-color:#f8f8f8;">
                 <span>&nbsp;</span>
                </div> 
                <div style="background-color:#ffffff;">
                 <span><code style="color:#008200;">// Get the user properties for the current user </code></span>
                </div> 
                <div style="background-color:#f8f8f8;">
                 <span><code style="color:#006699;font-weight:bold;">function</code> <code style="color:#000000;">getUserProfileProperties() { </code></span>
                </div> 
                <div style="background-color:#ffffff;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#008200;">// Create a client context </code></span></span>
                </div> 
                <div style="background-color:#f8f8f8;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#006699;font-weight:bold;">var</code> <code style="color:#000000;">clientContext = </code><code style="color:#006699;font-weight:bold;">new</code> <code style="color:#000000;">SP.ClientContext.get_current(); </code></span></span>
                </div> 
                <div style="background-color:#ffffff;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#008200;">// Get the People Manager and the current user properties </code></span></span>
                </div> 
                <div style="background-color:#f8f8f8;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#006699;font-weight:bold;">var</code> <code style="color:#000000;">peopleManager = </code><code style="color:#006699;font-weight:bold;">new</code> <code style="color:#000000;">SP.UserProfiles.PeopleManager(clientContext); </code> </span></span>
                </div> 
                <div style="background-color:#ffffff;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#000000;">userProperties = peopleManager.getMyProperties(); </code></span></span>
                </div> 
                <div style="background-color:#f8f8f8;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#000000;">clientContext.load(userProperties); </code></span></span>
                </div> 
                <div style="background-color:#ffffff;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#008200;">// Get the following Manager and the items the user is following </code></span></span>
                </div> 
                <div style="background-color:#f8f8f8;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#006699;font-weight:bold;">var</code> <code style="color:#000000;">followingManager = </code><code style="color:#006699;font-weight:bold;">new</code> <code style="color:#000000;">SP.Social.SocialFollowingManager(clientContext); </code> </span></span>
                </div> 
                <div style="background-color:#ffffff;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#000000;">following = followingManager.getFollowed(15); </code></span></span>
                </div> 
                <div style="background-color:#f8f8f8;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#008200;">// Execute the query </code></span></span>
                </div> 
                <div style="background-color:#ffffff;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#000000;">clientContext.executeQueryAsync(onSuccess, onFailure); </code></span></span>
                </div> 
                <div style="background-color:#f8f8f8;">
                 <span><code style="color:#000000;">} </code></span>
                </div> 
                <div style="background-color:#ffffff;">
                 <span>&nbsp;</span>
                </div> 
                <div style="background-color:#f8f8f8;">
                 <span><code style="color:#008200;">// Process the results and populate a DIV </code></span>
                </div> 
                <div style="background-color:#ffffff;">
                 <span><code style="color:#006699;font-weight:bold;">function</code> <code style="color:#000000;">onSuccess() { </code></span>
                </div> 
                <div style="background-color:#f8f8f8;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#000000;">$(</code><code style="color:blue;">"#WelcomeMessageUserName"</code><code style="color:#000000;">).text(userProperties.get_displayName()); </code></span></span>
                </div> 
                <div style="background-color:#ffffff;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#000000;">$(</code><code style="color:blue;">"#UserProfileImage"</code><code style="color:#000000;">).attr(</code><code style="color:blue;">"src"</code><code style="color:#000000;">,userProperties.get_pictureUrl()); </code></span></span>
                </div> 
                <div style="background-color:#f8f8f8;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#006699;font-weight:bold;">var</code> <code style="color:#000000;">followedItems = </code><code style="color:blue;">"Items you are following:&lt;br /&gt;"</code><code style="color:#000000;">; </code></span></span>
                </div> 
                <div style="background-color:#ffffff;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#000000;">$.each(following, </code><code style="color:#006699;font-weight:bold;">function</code><code style="color:#000000;">( index, value ) { </code></span></span>
                </div> 
                <div style="background-color:#f8f8f8;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#000000;">followedItems += </code><code style="color:blue;">"&lt;a href='"</code> <code style="color:#000000;">+ value.get_uri() + </code><code style="color:blue;">"'&gt;"</code> <code style="color:#000000;">+ value.get_name() + </code><code style="color:blue;">"&lt;/a&gt;&lt;br /&gt;"</code><code style="color:#000000;">; </code> </span></span>
                </div> 
                <div style="background-color:#ffffff;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#000000;">});</code></span></span>
                </div> 
                <div style="background-color:#f8f8f8;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#000000;">$(</code><code style="color:blue;">"#UserFollows"</code><code style="color:#000000;">).append(followedItems); </code></span></span>
                </div> 
                <div style="background-color:#ffffff;">
                 <span><code style="color:#000000;">} </code></span>
                </div> 
                <div style="background-color:#f8f8f8;">
                 <span>&nbsp;</span>
                </div> 
                <div style="background-color:#ffffff;">
                 <span><code style="color:#006699;font-weight:bold;">function</code> <code style="color:#000000;">onFailure() { </code></span>
                </div> 
                <div style="background-color:#f8f8f8;">
                 <span><code>&nbsp;&nbsp;&nbsp;&nbsp;</code><span><code style="color:#000000;">alert(</code><code style="color:blue;">"Error occurred retrieving user profile information."</code><code style="color:#000000;">); </code></span></span>
                </div> 
                <div style="background-color:#ffffff;">
                 <span><code style="color:#000000;">} </code></span>
                </div> 
                <div style="background-color:#f8f8f8;">
                 <span><code style="color:#008200;">//]&gt;</code></span>
                </div> 
               </div> <br> <br> NOTE</strong>: The scripts are wrapped in a CDATA container in order to make them render and function properly.&nbsp; In my experience sometimes you do not need this, but it never hurts to have it there.</em>
             <br> 
             <br> This is where all the work takes place.&nbsp; There is one major function here called "getUserProfileProperties()" which is called using the ExecuteOrDelayUntilScriptLoaded function built into SharePoint.&nbsp; Basically we will be accessing user profile information, so we need to make sure the sp.userprofiles.js file is loaded first.
             <br> 
             <br> This function simply creates a PeopleManager object and calls the getMyProperties() method&nbsp;to retrieve all the user profile properties for the current user.&nbsp; Then it creates a SocialFollowingManger object and calls the getFollowed() method to pull all the items the user is following.&nbsp; If you look at the code the method call includes an&nbsp;INT which indicates which type of SocialActors we want to retrieve using the SocialActorTypes enumeration.&nbsp; For instance, Users, Documents, Sites, etc.&nbsp; 15 indicates we want all SocialActors to be returned.&nbsp; For more info on the 
             <a href="http://msdn.microsoft.com/en-us/library/microsoft.sharepoint.client.social.socialactortypes.aspx" target="_blank"> <span style="color:#336699;">SocialActorTypes click here</span></a>.
             <br> 
             <br> If all goes well the onSuccess() function is fired and the user profile properties and following information will be populated in the DIVs in the HTML, as shown below.
             <br> 
             <br> 
             <br> 
             <div class="separator" style="clear:both;text-align:center;">
              <a href="http://2.bp.blogspot.com/-Px2kaMuLf3c/UmV-sXUAvVI/AAAAAAAAA3E/ySXbWOc5NOM/s1600/9-Site-Without-Styling.png" style="margin-right:1em;margin-left:1em;"><img alt=" " width="320" height="293" src="http://2.bp.blogspot.com/-Px2kaMuLf3c/UmV-sXUAvVI/AAAAAAAAA3E/ySXbWOc5NOM/s320/9-Site-Without-Styling.png" style="border-width:0px;border-style:solid;"></a>
             </div>

             <br> Bam!&nbsp; Not too bad, but its basically just some data without any formatting.&nbsp; Let's add some CSS to pretty it up.&nbsp; We'll add a CSS file to the UserWelcome library and call it UserWelcome.css.

             <br> 
             <br> And there you go!&nbsp; A simple demonstration using HTML, JavaScript, JSOM, and the XML Viewer web part to create a "custom" web part for your users.
             <br> 
             <br> Using this approach has several advantages: 
             <ul> 
              <li>It allows for non-C#/Visual Studio development to take place </li>
              <li>It allows for users with only HTML and JavaScript experience to create rich web parts without the need for server side code </li>
              <li>It allows for easy maintenance and upgrades since all the code is stored in a standard document library </li>
              <li>It utilizes JSOM for building web parts, which is a best practice in 2013 (also CSOM) </li>
              <li>You can utilize versioning in the library to ensure new updates that may break functionality can be easily rolled back </li>
             </ul> This of course is not a one size fits all solution, as there will certainly be times you need server side code.&nbsp; However with the expansion of the CSOM/JSOM functionality in SharePoint 2013, many development opportunities can be explored in this fashion.&nbsp; It also makes a great development model for Office 365 as well.
