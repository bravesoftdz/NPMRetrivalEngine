     <section id="commonquestions"> 
      <h2>Common questions </h2> 
      <br> 
      <h4>What about licensing?</h4> 
      <p>All products and services like maps, tiles, APIs and others provided by OpenWeatherMap are distributed under terms of the <a href="http://creativecommons.org/licenses/by-sa/4.0/">Creative Commons Attribution-ShareAlike 4.0 International license (CC BY-SA 4.0).</a></p> 
      <p>Data and database are open and licened by <a href="http://opendatacommons.org/licenses/odbl/">Open Data Commons Open Database License (ODbL).</a></p> 
      <p>You can freely use our products and data with non-commercial or commercial purposes. However, the OpenWeatherMap should be credited as a weather data source in a visible part of your application. </p> 
      <p>For Enterprise account other licenses are possible. Please, <a href="https://openweathermap.desk.com/customer/portal/emails/new?t=535697">contact us</a> for more details.</p> 
      <br> 
      <h4>How to get technical support?</h4> 
      <p>In our Support Center <a href="https://openweathermap.desk.com">openweathermap.desk.com</a> you can search the knowledge base, send us your request or leave a public question to society. On our website <a href="http://openweathermap.org/api">openweathermap.org/api </a>you can find detailed API documentation with examples of API calls and responses.</p> 
      <br> 
      <h4>What if I need in customisation of APIs or new features? </h4> 
      <p>If you need in customization of API, expanded features, bulk downloading of historical data, more than 5 years archive of historical weather, or more flexibility, please, <a href="mailto:info@openweathermap.org?subject=Customisation">contact us.</a></p> 
      <br> 
      <h4>How to work with hourly historical weather API?</h4> 
      <p>Historical data has an hour interval, data is available for 20,000+ cities and for majority of weather stations. Due to considerable volume of data the historical API has limitation of downloading by one API call. Depend on how long period of data you need please divide your request into several time intervals and download data by several API calls accordingly. </p> 
      <br> 
      <h4>Can I work with your API without API key?</h4> 
      <p>To get access to weather API you need to use your API key whatever account you chose from Free to Enterprise. We keep right to not to process API requests without API key. <a href="http://openweathermap.org/appid#get">How to get API key</a></p> 
      <br> 
      <h4>Why Free and Startup accounts has no SSL feature?</h4> 
      <p>There is a simple reason behind limitation of SSL feature by several paid accounts and not to including it into free and Startup accounts at the moments. OpenWeatherMap service is really high loading ones processes dozen of thousands requests per second. Providing SSL for each request will increase loading to the servers significantly and as a result we will not be able to process so many requests in a neat way that we are currently doing. </p>
