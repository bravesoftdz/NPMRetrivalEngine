        <h2>Getting Started</h2> 
        <p>Before you start using the Weather API, it is important to know that</p> 
        <ol> 
         <li>Most of the API features require an API key. <a href="/weather/api/" title="Getting Started">Sign up for a key.</a></li> 
         <li><a href="/weather/api/d/docs?d=data/index" title="Data Features and URL Format">API requests</a> are made over HTTP. Data features return JSON or XML. WunderMap layer features return image files. This documentation is full of examples of how to use API features. See <a href="/weather/api/d/docs?d=resources/code-samples" title="API Code Samples">code samples for several languages</a>, and <a href="/weather/api/d/docs?d=resources/user-generated-code" title="User-generated Code">user-generated code and libraries</a>.</li> 
         <li>Multiple API features can be combined into a single HTTP request. This is an easy way to economize your requests. The <a href="/weather/api/d/docs?d=data/index">Data Features</a> page documents how features can be combined into a single request.</li> 
         <li>Per the <a href="/weather/api/d/terms.html" title="Wunderground Terms of Service">Terms of Service</a> the Weather Underground logo must be included with a credit line where Weather Underground data is displayed. Please see individual Logo variations for all acceptable combinations of layout in the <a href="/weather/api/d/docs?d=resources/logo-usage-guide">Logo Usage Guide</a>.</li> 
        </ol>
