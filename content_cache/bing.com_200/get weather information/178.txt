       <h1 class="title">Add or Remove Packages Offline Using DISM</h1> 
       <p>Updated: October 20, 2013</p>
       <p>Applies To: Windows 8, Windows 8.1, Windows Server 2008 R2, Windows Server 2012, Windows Server 2012 R2</p>
       <div id="mainSection"> 
        <div id="mainBody"> 
         <p>Deployment Image Servicing and Management (DISM.exe) is a command-line tool that is used to update offline Windows® images. There are two ways to install or remove packages offline with DISM. You can either apply an unattend answer file to the offline image, or you can add or remove the package directly from the command prompt.</p> 
         <p>If you are installing multiple packages to a Windows image, and there are dependency requirements, the best way to ensure the correct order of the installation is by using an answer file. You can use DISM to apply the Unattend.xml answer file to the image. When you use DISM to apply an answer file, the unattend settings in the <strong>offlineServicing</strong> configuration pass are applied to the Windows image.</p> 
         <p>You must install the latest version of the Windows Assessment and Deployment Kit (Windows&nbsp;ADK), which contains all of the tools that are required, including DISM. </p> 
         <p>In this topic:</p> 
         <p> <a href="#BKMK_add">To add packages to an offline image by using DISM</a> </p> 
         <p> <a href="#BKMK_remove">To remove packages from an offline image by using DISM</a> </p> 
         <p> <a href="#BKMK_answer">To add or remove packages offline by using DISM and an answer file</a> </p>
