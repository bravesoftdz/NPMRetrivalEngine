        <h1 class="hsbanner">Speech Recognition</h1> 
       </div> 
      </div> 
      <div id="hspagebody"> 
       <div id="mainSection"> 
        <div id="mainBody"> 
         <div>
          <img id="SpeechRecognizer" alt="Speech Recognizer Icon" src="https://i-msdn.sec.s-msft.com/dynimg/IC236255.png" title="Speech Recognizer Icon" xmlns="">
         </div> 
         <p>Speech recognition (SR) converts spoken words to written text and as a result can be used to provide user interfaces that use spoken input. The <strong>Speech Recognizer</strong> service enables you to include speech recognition support for your application. Speech recognition requires a special type of software, called an SR engine. The SR engine may be installed with the operating system or at a later time with other software. Speech-enabled packages such as word processors and web browsers, may install their own engines or they can use existing ones. Additional engines are also available through third party manufacturers. These engines are typically designed to only support a specific language and may also target a certain vocabulary; for example, a vocabulary specializing in medical or legal terminology.</p> 
         <div class="alert"> 
          <div> 
           <table summary="table"> 
            <tbody>
             <tr>
              <td><img id="hs-caution" alt="Dd145257.hs-caution(en-us,MSDN.10).gif" src="https://i-msdn.sec.s-msft.com/dynimg/IC50262.gif" title="Dd145257.hs-caution(en-us,MSDN.10).gif" xmlns=""></td>
              <td> <p>Note that speech recognition is not available on all versions of Windows. Before you attempt using the <strong>Speech Recognizer</strong> service, use the Windows Control Panel on your PC to confirm that a compatible (SAPI) speech recognition engine is installed and make sure that it is properly configured and working by using the Help or user documentation that is provided for it.</p> <p>You will also need a microphone or some other sound input device to receive the sound. In general, the microphone should be a high quality device with noise filters built in. The speech recognition accuracy is directly related to the quality of the input. The recognition rate will be significantly lower or perhaps even unacceptable with a poor microphone.</p> <p>The .NET 3.0 (or later) runtime is also required for using this service (and may be available from the Microsoft website).</p></td>
             </tr>
            </tbody>
           </table>
          </div>
         </div> 
         <p>&nbsp;</p> 
         <h3>Operations</h3> 
         <p>The <strong>Speech Recognizer</strong> services supports the following requests and notifications.</p> 
         <div class="tablediv"> 
          <table responsive="true" summary="table"> 
           <tbody>
            <tr responsive="true">
             <th scope="col">Operation</th>
             <th scope="col">Description</th>
            </tr> 
            <tr>
             <td data-th="Operation">Get</td>
             <td data-th="Description">Returns the entire state of the <strong>Speech Recognizer</strong> service.</td>
            </tr> 
            <tr>
             <td data-th="Operation">InsertGrammarEntry</td>
             <td data-th="Description">Inserts the specified entry (or entries) of the supplied grammar into the current grammar dictionary. If certain entries exist already a <strong>Fault</strong> is returned and the whole operation fails without the current dictionary being modified at all.</td>
            </tr> 
            <tr>
             <td data-th="Operation">UpdateGrammarEntry</td>
             <td data-th="Description">Updates entries that already exist in the current grammar dictionary with the supplied grammar entries. If certain entries in the supplied grammar do not exist in the current dictionary no <strong>Fault</strong> is returned. Instead, only the existing entries are updated.</td>
            </tr> 
            <tr>
             <td data-th="Operation">UpsertGrammarEntry</td>
             <td data-th="Description">Inserts entries from the supplied grammar into the current dictionary if they do not exist yet or updates entries that already exist with entries from the supplied grammar.</td>
            </tr> 
            <tr>
             <td data-th="Operation">DeleteGrammarEntry</td>
             <td data-th="Description">Deletes those entries from the current grammar directory whose keys are equal to one of the supplied grammar entries. If a key from the supplied grammar entries does not exist in the current directory no <strong>Fault</strong> is returned, but any matching entries are deleted.</td>
            </tr> 
            <tr>
             <td data-th="Operation">SetSrgsGrammarFile</td>
             <td data-th="Description">Sets the grammar type to SRGS file and tries to load the specified file, which has to reside inside your application's /store folder (directory). If loading the file fails, a <strong>Fault</strong> is returned and the speech recognizer returns the state it was before it processed this request. <strong>SRGS grammars require Windows 7 and will not work with Windows Server 2003.</strong></td>
            </tr> 
            <tr>
             <td data-th="Operation">EmulateRecognize</td>
             <td data-th="Description">Sets the SR engine to emulate speech input but by using <strong>Text</strong> (string). This is mostly used for testing and debugging.</td>
            </tr> 
            <tr>
             <td data-th="Operation">Replace</td>
             <td data-th="Description">Configures the speech recognizer service, or indicates that the service's configuration has been changed.</td>
            </tr> 
            <tr>
             <td data-th="Operation">SpeechDetected</td>
             <td data-th="Description">Indicates that speech (audio) has been detected and is being processed.</td>
            </tr> 
            <tr>
             <td data-th="Operation">SpeechRecognized</td>
             <td data-th="Description">Indicates that speech has been recognized.</td>
            </tr> 
            <tr>
             <td data-th="Operation">SpeechRecognitionRejected</td>
             <td data-th="Description">Indicates that speech was detected, but not recognized as one of the words or phrases in the current grammar dictionary. The duration of the speech is available as <strong>DurationInTicks</strong>.</td>
            </tr>
           </tbody>
          </table>
         </div> 
         <p>To support SR you define a <em>grammar</em> - the words and phrases to be recognized and then use notifications provided by the service to determine what SR engine recognized as the spoken input. The <strong>Speech Recognizer</strong> service supports usage of simple dictionary-style grammars as well as W3C SRGS grammars.</p> 
         <div class="alert"> 
          <div> 
           <table summary="table"> 
            <tbody>
             <tr>
              <td><img id="hs-caution" alt="Dd145257.hs-caution(en-us,MSDN.10).gif" src="https://i-msdn.sec.s-msft.com/dynimg/IC50262.gif" title="Dd145257.hs-caution(en-us,MSDN.10).gif" xmlns=""></td>
              <td> <p>Note that you also cannot use the service's grammar operations as requests from VPL because they require a special data structure that cannot easily be supported by VPL, but you can receive them as notifications.</p></td>
             </tr>
            </tbody>
           </table>
          </div>
         </div> 
         <p>To define which type of grammar you want the Speech Recognizer service to use, you set the state of this service by either using setting its initial configuration in the Properties window (setting Configuration to <strong>Set initial configuration</strong>) or by using a <strong>Replace</strong> request or a <strong>SetSrgsGrammarFile</strong> request.</p> 
         <p>The service's initial state includes the following properties:</p> 
         <table responsive="true" summary="table"> 
          <tbody>
           <tr responsive="true">
            <th scope="col">Name</th>
            <th scope="col">Type</th>
            <th scope="col">Description</th>
           </tr> 
           <tr>
            <td data-th="Name">IgnoreAudioInput</td>
            <td data-th="Type">Boolean</td>
            <td data-th="Description">Specifies whether the speech service listens for audio (spoken) input (when this is set to <strong>false</strong>). This may useful for turning off the SR engine temporarily(or when using emulation recognition).</td>
           </tr> 
           <tr>
            <td data-th="Name">GrammarType</td>
            <td data-th="Type">GrammarType</td>
            <td data-th="Description">Specifies the type of grammar the SR engine will use, either a simple Dictionary grammar or SRGS grammar.</td>
           </tr> 
           <tr>
            <td data-th="Name">SrgsFileLocation</td>
            <td data-th="Type">string</td>
            <td data-th="Description">Specifies the SRGS grammar file to be loaded (only used if you set GrammarType to SRGS).</td>
           </tr>
          </tbody>
         </table> 
         <p>Setting <strong>GrammarType</strong> to <strong>Dictionary</strong> configures the service to use a simple dictionary-style grammar. A dictionary-style grammar is a list of entries that each consist of a set of words for the speech engine to listen for and an optional corresponding semantic tag that represents that recognition. For example, you might define an entry like, Tell me the time, and call its semantic tag, TimeQuery.</p> 
         <p>To create a simple dictionary-style grammar you define the grammar as part of the service's configuration XML file (SpeechRecognizer.config.xml) for the service. This file is automatically created for your project if you choose the <strong>Set initial configuration</strong> option. If you save your project, and then open this file (using any XML editor including Windows Notepad) you can add entries for your grammar and save it back to this location. The file must be saved to this location. It is loaded when your project runs.</p> 
         <p>For each entry, add a beginning and ending XML Elem tag, and an XML string entry for the words the SR engine should listen for and its for the optional semantic tag. You can also use the <strong>SpeechRecognizerGui</strong> service to generate a Web page that enables you to enter and save a simple dictionary grammar file. For further details, see information on the <strong>SpeechRecognizerGui</strong> service.</p> 
         <p>The following is an example for a simple dictionary-style grammar file:</p> 
         <p>&lt;?xml version="1.0" encoding="utf-8"?&gt;<br>&lt;SpeechRecognizerState xmlns="http://schemas.microsoft.com/robotics/2008/02/speechrecognizer.html"&gt;<br>&nbsp; &lt;DictionaryGrammar&gt;<br>&nbsp;&nbsp;&nbsp; &lt;Elem&gt;<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;string xmlns=""&gt;Backward&lt;/string&gt;<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;string xmlns=""&gt;Backward&lt;/string&gt;<br>&nbsp;&nbsp;&nbsp; &lt;/Elem&gt;<br>&nbsp;&nbsp;&nbsp; &lt;Elem&gt;<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;string xmlns=""&gt;Follow me&lt;/string&gt;<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;string xmlns=""&gt;FollowMe&lt;/string&gt;<br>&nbsp;&nbsp;&nbsp; &lt;/Elem&gt;<br>&nbsp;&nbsp;&nbsp; &lt;Elem&gt;<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;string xmlns=""&gt;Forward&lt;/string&gt;<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;string xmlns=""&gt;Forward&lt;/string&gt;<br>&nbsp;&nbsp;&nbsp; &lt;/Elem&gt;<br>&nbsp;&nbsp;&nbsp; &lt;Elem&gt;<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;string xmlns=""&gt;Left&lt;/string&gt;<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;string xmlns=""&gt;Left&lt;/string&gt;<br>&nbsp;&nbsp;&nbsp; &lt;/Elem&gt;<br>&nbsp;&nbsp;&nbsp; &lt;Elem&gt;<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;string xmlns=""&gt;Right&lt;/string&gt;<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;string xmlns=""&gt;Right&lt;/string&gt;<br>&nbsp;&nbsp;&nbsp; &lt;/Elem&gt;<br>&nbsp;&nbsp;&nbsp; &lt;Elem&gt;<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;string xmlns=""&gt;Stop moving&lt;/string&gt;<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;string xmlns=""&gt;Stop&lt;/string&gt;<br>&nbsp;&nbsp;&nbsp; &lt;/Elem&gt;<br>&nbsp; &lt;/DictionaryGrammar&gt;<br>&nbsp; &lt;IgnoreAudioInput&gt;false&lt;/IgnoreAudioInput&gt;<br>&nbsp; &lt;GrammarType&gt;DictionaryStyle&lt;/GrammarType&gt;<br>&lt;/SpeechRecognizerState&gt;</p> 
         <p>To use an SRGS grammar, set the <strong>GrammarType</strong> to <strong>Srgs</strong> and supply the filename to <strong>SrgsFileLocation</strong>, either by setting the initial configuration properties or using a <strong>Replace</strong> request. Using the <strong>SetSrgsGrammarFile</strong> request automatically sets <strong>GrammarType</strong> and tries to load the specified SRGS grammar file (which must be located in your application's \Store folder).</p> 
         <p>SRGS grammars are also XML files which can be created using a simple editor or from speech tools that generate this format. Details about this format can be found at <a href="http://www.w3.org/TR/speech-grammar/">http://www.w3.org/TR/speech-grammar/</a>.</p> 
         <p>&nbsp;</p> 
         <h3>Service State</h3> 
         <p>You can use a <strong>Get</strong> request to return the general state of the <strong>Speech Recognizer</strong> service. However, the recognition state is provided by the <strong>SpeechDetected</strong>, <strong>SpeechRecognized</strong>, and <strong>SpeechRecognitionRejected</strong> notifications.</p> 
         <p><strong>SpeechDetected</strong> returns <strong>StartTime</strong> (DateTime), which is the time when the SR detects audio input.</p> 
         <p>When the SR recognizes the input a <strong>SpeechRecognized</strong> notification returns the following state:</p> 
         <p></p> 
         <table responsive="true" summary="table"> 
          <tbody>
           <tr responsive="true">
            <th scope="col">Name</th>
            <th scope="col">Type</th>
            <th scope="col">Description</th>
           </tr> 
           <tr>
            <td data-th="Name">Confidence</td>
            <td data-th="Type">float</td>
            <td data-th="Description">Return a value between 0 and 1 indicating the SR engine's rating of the certainty of correct recognition for the phrase information returned (higher is better). However, it is a relative measure of the certainty and therefore may vary for each recognition engine. If -1 is returned the speech engine does not provide confidence information.</td>
           </tr> 
           <tr>
            <td data-th="Name">Text</td>
            <td data-th="Type">string</td>
            <td data-th="Description">Returns the words recognized.</td>
           </tr> 
           <tr>
            <td data-th="Name">Semantics</td>
            <td data-th="Type">RecognizedSemanticValue</td>
            <td data-th="Description">Returns the semantic value object(s), if any, of the recognized words.</td>
           </tr> 
           <tr>
            <td data-th="Name">DurationInTicks</td>
            <td data-th="Type">long integer</td>
            <td data-th="Description">Returns the duration of the utterance recognized. There are 10,000,000 ticks per second.</td>
           </tr>
          </tbody>
         </table> 
         <p></p> 
         <p>If you load an SRGS grammar you can use <strong>Semantics</strong> to access the semantic information which applies to the recognized utterance. It may also include a collection of the child semantic values of the utterance recognized.</p> 
         <table responsive="true" summary="table"> 
          <tbody>
           <tr responsive="true">
            <th scope="col">Name</th>
            <th scope="col">Type</th>
            <th scope="col">Description</th>
           </tr> 
           <tr>
            <td data-th="Name">Children</td>
            <td data-th="Type">DssDictionary</td>
            <td data-th="Description">Returns the collection of (child) semantic value objects.</td>
           </tr> 
           <tr>
            <td data-th="Name">Confidence</td>
            <td data-th="Type">float</td>
            <td data-th="Description">Returns a value between 0 and 1 indicating the SR engine's rating of the certainty of correct recognition for the phrase information returned (higher is better). However, it is a relative measure of the certainty and therefore may vary for each recognition engine. If -1 is returned the speech engine does not provide confidence information.</td>
           </tr> 
           <tr>
            <td data-th="Name">KeyName</td>
            <td data-th="Type">string</td>
            <td data-th="Description">Returns thekey string by which this semantic value can be referenced.</td>
           </tr> 
           <tr>
            <td data-th="Name">TypeOfValue</td>
            <td data-th="Type">RecognizedValueType</td>
            <td data-th="Description">Returns the type of this semantic value.</td>
           </tr> 
           <tr>
            <td data-th="Name">ValueBool</td>
            <td data-th="Type">Boolean</td>
            <td data-th="Description">Returns the Boolean value of the semantic value.</td>
           </tr> 
           <tr>
            <td data-th="Name">ValueFloat</td>
            <td data-th="Type">float</td>
            <td data-th="Description">Returns the float value of the semantic value.</td>
           </tr> 
           <tr>
            <td data-th="Name">ValueInt</td>
            <td data-th="Type">int</td>
            <td data-th="Description">Returns the int value of the semantic value.</td>
           </tr> 
           <tr>
            <td data-th="Name">ValueString</td>
            <td data-th="Type">string</td>
            <td data-th="Description">Returns the string value of the semantic value.</td>
           </tr>
          </tbody>
         </table> 
         <p>To access child semantic values, use the dot notation. For example, you can access the number of child semantic values by using <strong>Semantics.Children.Count</strong>. To access the values of the children, you can use their grammar rule name. For example, if you have a number of cities listed under a rule called "Destination", and the recognition matched a destination, you could access the confidence rating of the destination match using <strong>Semantics.Children["Destination"].Confidence</strong> and its value using <strong>Semantics.Children["Destination"].Value.</strong></p> 
         <p>If the SR engine fails to recognize the input as matching anything in its grammar, then it instead generates a <strong>SpeechRecognitionRejected</strong> notification, returning <strong>StartTime</strong> (DateTime) and <strong>DurationInTicks</strong> (long integer).</p> 
         <p>&nbsp;</p> 
         <h2>Speech Recognizer Gui</h2> 
         <div>
          <img id="SpeechRecognizerGui" alt="Speech Recognizer Gui Icon" src="https://i-msdn.sec.s-msft.com/dynimg/IC236256.png" title="Speech Recognizer Gui Icon" xmlns="">
         </div> 
         <p>The <strong>Speech Recognizer</strong> <strong>Gui</strong> service is a companion service that you can use with the <strong>Speech Recognizer</strong> service. Including the <strong>Speech Recognizer Gui</strong> service in your project enables you to enter a simple dictionary-style grammar or to upload SRGS (Speech Recognition Grammar Specification) grammar files through a HTML page.</p> 
         <p>To access the <strong>Speech Recognizer Gui</strong> service in VPL, drag a copy of the service block into your diagram. It does not require any connections, and it will start up when you run the diagram. You can also optionally start an instance of the <strong>Speech Recognizer Gui</strong> once you have a DSS node running by using a web browser and going to the Control Panel page. Starting the service will automatically attempt to load the default SR engine.</p> 
         <p>Once the <strong>Speech Recognizer Gui</strong> service is running, browse to the page for the service. To do this start your browser and enter in http:/localhost:50000 (50000 is the default port setting. If you run services on a different port use that.) Then click on Service Directory in the left column to display the list of running services. You should find the entry /speechrecognizergui in the list. Click this and you should see a page like the figure below.</p> 
         <div>
          <img id="SpeechRecognizerGuiService" alt="Speech Recognizer" src="https://i-msdn.sec.s-msft.com/dynimg/IC227220.png" title="Speech Recognizer" xmlns=""> 
          <div class="caption"> 
           <p><strong>Speech Recognizer Gui -</strong> Service page</p>
          </div>
         </div> 
         <p>At the bottom of the <strong>Speech Recognizer Gui</strong> page you select either a simple dictionary style grammar or load an SRGS grammar. Click Save to use the grammar. This will create a copy of the grammar file in the \Store folder. If you chose to create a Dictionary grammar, the Save command creates a SpeechRecognizer.config.xml file in the \Store folder. If you chose to load a SRGS grammar file, the file you browse to will be copied to \Store.</p> 
         <p>Note that if you create a grammar using the <strong>Speech Recognizer Gui</strong> service and do not explicitly configure the <strong>Speech Recognizer</strong> service to load a grammar file, the <strong>Speech Recognizer</strong> service will automatically attempt to load and use the grammar file you created.</p> 
         <p>The <strong>Speech Recognizer Gui</strong> service page also displays the notification generated by the SR engine such as <em>speech detected</em> or <em>speech recognized</em> in a scrolling area that can be cleared. Note that the SR engine only recognizes words and phrases that are in its grammar. If the grammar is empty, then nothing will be recognized.</p> 
         <p>The <strong>Speech Recognizer Gui</strong> service is not designed to be sent requests or to issue notifications. It is only intended to be used via a Web browser to build dictionary-style grammars and test speech recognition.</p>
