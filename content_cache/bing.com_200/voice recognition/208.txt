         <td width="82%" valign="top" style="border-top: 1px solid #ccc; padding:15px;"> <h2>CREATING ANDROID IMAGE RECOGNITION APPLICATION USING NETBEANS AND NEUROPH</h2> <p align="justify">by Dalibor Micic</p> <p align="justify">This guide will give you an overview on how to develop simple Android application for image recognition using NetBeans 7, NBAndroid plugin and Neuroph framework version 2.6.</p> <h3>Android SDK installation guide</h3> <p align="justify">Download Android SDK starter package from <a href="http://developer.android.com/sdk/index.html">http://developer.android.com/sdk/index.html</a> (the SDK starter package is not a full development environment-it includes only the core SDK Tools, which you can use to download the rest of the SDK components (such as the latest Android platform).</p> <p align="justify">If you downloaded the Windows installer (.exe file), run it now and it will check whether the proper Java SE Development Kit (JDK) is installed (installing it, if necessary), then install the SDK Tools into a default location (which you can modify). Make a note of the name and location of the SDK directory on your system - you will need to refer to the SDK directory later, when using the SDK tools from the command line.</p> <p align="center"><img src="images/android_sdk_setup1.jpg"></p> <p align="center"><img src="images/android_sdk_setup2.jpg"></p> <p align="justify">After the installation is completed start SDK Manager.</p> <p align="center"><img src="images/android_sdk_setup3.jpg"></p> <p align="justify">You need to download <strong>at least one platform</strong> into your environment, so that you will be able to compile an application and set up an Android Virtual Device (AVD) to run it on (in the emulator). Android 2.1 is supported on approximately 98% of Android phones and this is the platform that is going to be used in this example. You can also download other platforms as well.</p> <p align="center"><img src="images/android_sdk_manager.jpg"></p> <p align="justify"><strong>SDK Platform-tools</strong> includes more tools that are required for application development. These tools are platform-dependent and typically update only when a new SDK platform is made available, in order to support new features in the platform. These tools are always backward compatible with older platforms, but you must be sure that you have the latest version of these tools when you install a new SDK platform.</p> <p align="justify">The <strong>Samples</strong> components give you source code that you can use to learn about Android, load as a project and run, or reuse in your own app. Note that multiple samples components are available - one for each Android platform version. When you are choosing a samples component to download, select the one whose API Level matches the API Level of the Android platform that you plan to use. To be able to run this example, select <strong>Tools -&gt; Android SDK Platform-tools and Android 2.1 (API 7) -&gt; SDK Platform</strong> and install them.</p> <h3>NBAndroid installation guide</h3> <p align="justify">NBAndroid (<a href="http://www.nbandroid.org">http://www.nbandroid.org</a>) is a plugin that provides support for the whole development cycle of Android applications and it will be used in this example.</p> <p align="justify">Installation is very simple. Open plug-in manager in NetBeans (Tools -&gt; Plugins) and select Settings tab. Click on Add and add following URL http://kenai.com/projects/nbandroid/downloads/download/updatecenter/updates.xml</p> <p align="center"><img src="images/nbandroid_setup1.jpg"></p> <p align="justify">Press OK and go to Available plugins. Select Android and Android Test Runner For NetBeans 7.0+ (there is also Android Test Runner For NetBeans 6.9 and NetBeans 6.9.1 for use with older versions of NetBeans) and install them.</p> <p align="justify">Next step is to configure Android SDK path. Go to Tools -&gt; Options, open Miscellaneous tab and specify location of Android SDK.</p> <p align="center"><img src="images/nbandroid_setup2.jpg"></p> <h3>Android test application</h3> <p align="justify">Create new Android project in NetBeans, go to File -&gt; New Project and select Android project. On next screen select target platform (in our case it is Android 2.1). We can rename activity that will automatically be created for us (an activity is an application component that provides a screen with which users can interact in order to do something. Each activity is given a window in which to draw its user interface. More about Activities on <a href="http://developer.android.com/guide/topics/fundamentals/activities.html">http://developer.android.com/guide/topics/fundamentals/activities.html</a>)</p> <p align="center"><img src="images/new_android_application.jpg"></p> <p align="justify">Application is going to be very simple, it will first load neural network and then it will display the main screen. By tapping anywhere on the main screen, it will open a phone's gallery. After selecting an image from gallery, it will show the selected image and display the name of the recognized image.</p> <p align="justify">Layout is defined in layout/main.xml and string values are defined in values/strings.xml</p> <p align="justify">Neural network (.nnet file) should be placed in folder /raw. In order to run the application, AVD (Android Virtual Device) needs to be created first. Go to ANDROID_SDK_ROOT and start AVD Manager.</p> <p align="center"><img src="images/android_virtual_device_manager1.jpg"></p> <p align="justify">Press New and create a new AVD.</p> <p align="center"><img src="images/create_new_avd.jpg"></p> <p align="justify">After creating a new AVD, the created AVD will be shown in list of existing Android Virtual Devices.</p> <p align="center"><img src="images/android_virtual_device_manager2.jpg"></p> <p align="justify">Close this window and go back to NetBeans. In order to start the application, select project and press F6. After a few seconds, the Select device dialog is shown with previously created Neuroph_Emulator AVD already selected. Press OK and wait until the emulator is loaded (this may take some time).</p> <p align="center"><img src="images/select_device.jpg"></p> <p align="center"><img src="images/emulator1.jpg"></p> <p align="center"><img src="images/emulator2.jpg"></p> <p align="center"><img src="images/emulator3.jpg"></p> <p align="justify">Select Neuroph Example from applications list.</p> <p align="center"><img src="images/emulator4.jpg"></p> <p align="justify">Application loads neural network on start.</p> <p align="justify">If you wish to create your own neural network using Neuroph, please read <a href="http://neuroph.sourceforge.net/image_recognition.html">Image recognition with neural networks</a>.</p> <p align="center"><img src="images/example1.jpg"></p> <p align="justify"><strong>Important notice:</strong> When you a load neural network in Android, you need to do it in a separate thread with increased stack size</p>

At this point, gallery is empty so we need to put some images. On Windows, go to Start -&gt; Run, type cmd and press OK. Android Debug Bridge (adb) is a command line tool that lets you communicate with an emulator instance or connected Android-powered device and it is located in ANDROID_SDK_ROOT\platform-tools. To put an image on the phone, use the following command:</p> <p align="center">adb push mypic.jpg /sdcard/</p> <p align="justify">Close the emulator and run the project once again. After loading a neural network, click on "Tap here to select image" and select an image from gallery. Application will try to recognize the image and it will ouput the result.</p> <p align="center"><img src="images/example2.jpg"></p> <p align="center"><img src="images/example3.jpg"></p> <p align="center"><img src="images/example4.jpg"></p><br> <p align="justify"><strong>DOWNLOAD:</strong></p> <p align="justify">Source code for Neuroph Android Example can be downloaded from <a href="neuroph_android_example.zip">here</a>. </p> <p align="justify"><strong>See also:</strong></p> <p align="justify"><a href="http://neuroph.sourceforge.net/image_recognition.html">Image Recognition with Neural Networks How To</a></p> </td>
