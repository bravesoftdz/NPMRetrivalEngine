      <section class="postMain">
       <h3 class="title sp-2"><a href="http://badassjs.com/post/40534144131/the-webspeech-api-enables-voice-recognition-and">The WebSpeech API Enables Voice Recognition and Speech Synthesis in the Browser</a></h3>
       <h4 class="textDate "><span>January 14th 2013</span></h4>
       <div class="mainText">
        <p>The new <a href="http://t.umblr.com/redirect?z=https%3A%2F%2Fdvcs.w3.org%2Fhg%2Fspeech-api%2Fraw-file%2Ftip%2Fspeechapi.html&amp;t=Y2JmZmRlMjVhNWI2NDY5NGJlMmRkODU4ZDA3MGVhNTNmMTQ0NzIzNixsdWxBQ0RxSw%3D%3D&amp;b=t%3AGINb_asixaSGG1PcWl6n7w&amp;p=http%3A%2F%2Fbadassjs.com%2Fpost%2F40534144131%2Fthe-webspeech-api-enables-voice-recognition-and&amp;m=1" target="_blank">WebSpeech API</a>, driven by Google but supported by at least Apple and perhaps other browser vendors is yet another interesting API for JavaScript applications that will enable some pretty amazing stuff in the future. &nbsp;The API adds support for both voice recognition as well as speech synthesis to JavaScript.</p>
        <p>You may remember that Chrome had added support for an additional form input attribute,&nbsp;<span>x-webkit-speech</span>, which enables speech recognition for that form field. &nbsp;The WebSpeech API takes this to a new level, enabling JavaScript to directly interface with the speech recognition engine and do whatever it pleases with the text it receives. &nbsp;This means you could write a Siri-like application in your browser or support voice commands, using the WebSpeech API for speech recognition and synthesis, with your own processing in between. &nbsp;This is super exciting.</p>
        <p><a href="http://t.umblr.com/redirect?z=http%3A%2F%2Fchrome.blogspot.com%2F2013%2F01%2Fhello-browser.html&amp;t=Nzg2YmViMGZmYzk3MTc1NzViMDhhMTJmMTRkZWZiZDhlMGJlNWFjZSxsdWxBQ0RxSw%3D%3D&amp;b=t%3AGINb_asixaSGG1PcWl6n7w&amp;p=http%3A%2F%2Fbadassjs.com%2Fpost%2F40534144131%2Fthe-webspeech-api-enables-voice-recognition-and&amp;m=1" target="_blank">Chrome 25 beta</a>, released today, contains support for the speech recognition portion of the WebSpeech API. &nbsp;There is a small <a href="http://t.umblr.com/redirect?z=https%3A%2F%2Fwww.google.com%2Fintl%2Fen%2Fchrome%2Fdemos%2Fspeech.html&amp;t=NGNiZjY0MWMyZWY4MTNlY2Y3NzAzY2Y1Yjc3MzAwNzZjNTVkZDhiZCxsdWxBQ0RxSw%3D%3D&amp;b=t%3AGINb_asixaSGG1PcWl6n7w&amp;p=http%3A%2F%2Fbadassjs.com%2Fpost%2F40534144131%2Fthe-webspeech-api-enables-voice-recognition-and&amp;m=1" target="_blank">demo</a>&nbsp;showing this at work, which you can try in the beta or Canary builds of Chrome. &nbsp;Over on HTML5 Rocks, there is a <a href="http://t.umblr.com/redirect?z=http%3A%2F%2Fupdates.html5rocks.com%2F2013%2F01%2FVoice-Driven-Web-Apps-Introduction-to-the-Web-Speech-API&amp;t=NTRlZjMxMmVjODM4YWU4OTNkOGNhMGU4MTJjOGZmMjIwNTdmNzhhMCxsdWxBQ0RxSw%3D%3D&amp;b=t%3AGINb_asixaSGG1PcWl6n7w&amp;p=http%3A%2F%2Fbadassjs.com%2Fpost%2F40534144131%2Fthe-webspeech-api-enables-voice-recognition-and&amp;m=1" target="_blank">short article</a> showing how the API can be used.</p>
        <p>It’s pretty simple: create a SpeechRecognition (or currently webkitSpeechRecognition) instance, set some options and configure some events, and you have speech recognition in your application! &nbsp;The API even supports returning results as you speak for instant visual feedback. &nbsp;The text can also update as you talk as more of the context of the sentence you are constructing is known. &nbsp;This is pretty neat to watch, so I’ve embedded a video showing it in use below. &nbsp;Do <a href="http://t.umblr.com/redirect?z=https%3A%2F%2Fwww.google.com%2Fintl%2Fen%2Fchrome%2Fdemos%2Fspeech.html&amp;t=NGNiZjY0MWMyZWY4MTNlY2Y3NzAzY2Y1Yjc3MzAwNzZjNTVkZDhiZCxsdWxBQ0RxSw%3D%3D&amp;b=t%3AGINb_asixaSGG1PcWl6n7w&amp;p=http%3A%2F%2Fbadassjs.com%2Fpost%2F40534144131%2Fthe-webspeech-api-enables-voice-recognition-and&amp;m=1" target="_blank">check it out</a> for yourself though, it’s definitely worth it.</p>
        <p><iframe frameborder="0" height="315" src="http://www.youtube.com/embed/qRolXPWqCSo" width="420"></iframe></p>
        <p>The only downside of the Voice Recognition API is that it requires that you be online, probably because it does the actual recognition on a remote server. &nbsp;I’m not sure why since desktop computers are definitely powerful enough to do it all locally, but perhaps it was easier for Google to use their already existing speech recognition servers for Android instead of adding all of that code to the browser. &nbsp;Perhaps we’ll see an update in the future with support for offline voice recognition.&nbsp;&nbsp;I’m not sure what other browser vendors will use for this when they add support for the API, since not everyone has those servers in their pockets already.</p>
        <p>The <a href="http://t.umblr.com/redirect?z=https%3A%2F%2Fdvcs.w3.org%2Fhg%2Fspeech-api%2Fraw-file%2Ftip%2Fspeechapi.html%23tts-section&amp;t=NTMxYjAwMzA3N2NjMmU3YjUzZWY4NGIxZjBiNzJlYjRhZmYyZDU3ZSxsdWxBQ0RxSw%3D%3D&amp;b=t%3AGINb_asixaSGG1PcWl6n7w&amp;p=http%3A%2F%2Fbadassjs.com%2Fpost%2F40534144131%2Fthe-webspeech-api-enables-voice-recognition-and&amp;m=1" target="_blank">Speech Synthesis</a> part of the API has not yet been implemented in WebKit, but it appears that plans for this are already underway. &nbsp;Chris Fleizach from Apple <a href="http://t.umblr.com/redirect?z=http%3A%2F%2Flists.webkit.org%2Fpipermail%2Fwebkit-dev%2F2013-January%2F023309.html&amp;t=ZmQxYTg0MTk3NDE1ZDdjZjg3NWY0M2Q1MmZhYjMyYTczNDgxZmUzMCxsdWxBQ0RxSw%3D%3D&amp;b=t%3AGINb_asixaSGG1PcWl6n7w&amp;p=http%3A%2F%2Fbadassjs.com%2Fpost%2F40534144131%2Fthe-webspeech-api-enables-voice-recognition-and&amp;m=1" target="_blank">announced</a> just yesterday that he will be implementing the speech synthesis API. &nbsp;It’s good to see a developer from a company other than Google interested in the API, since this means that more browsers than just Chrome will have support for the API. &nbsp;I’m not sure of this, but it looks like <a href="http://t.umblr.com/redirect?z=https%3A%2F%2Fwiki.mozilla.org%2FHTML5_Speech_API&amp;t=ZDAxMTk3ZmU5NzA0OGVhNDZiMzUxZTFmZDBiMjA0MDU2YjBkOGUzYixsdWxBQ0RxSw%3D%3D&amp;b=t%3AGINb_asixaSGG1PcWl6n7w&amp;p=http%3A%2F%2Fbadassjs.com%2Fpost%2F40534144131%2Fthe-webspeech-api-enables-voice-recognition-and&amp;m=1" target="_blank">Mozilla</a> may also be on board with the API, but has not begun implementation yet as far as I am aware. &nbsp;</p>
        <p>The Speech Synthesis API will support multiple voices, adjusting the speed, pitch, and volume of the voice, changing the language, and more. &nbsp;Seems pretty full featured, so I’m looking forward to that being implemented as well.</p>
        <p>Once again, I’m sure you’re getting tired of me saying this, but I think 2013 is going to be a great year for web development. &nbsp;The WebSpeech API will be yet another awesome one to watch this year and beyond!</p>
