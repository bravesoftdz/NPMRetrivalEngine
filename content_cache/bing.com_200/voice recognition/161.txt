       <p>Convert an Audio-File into text file via voice recognition. </p> 
       <table style="background-color:#ddddee;color:#000000;border:1px solid #606060;margin:1em 5% 1em 5%;"> 
        <tbody>
         <tr> 
          <td style="padding:0.5em 0.5em;"> </td> 
          <td style="padding:0.5em 0.5em;"> <b>NOTE:</b> The shell script mentioned below can be used on any Linux-Operating System with some software requirements, because the speech recognition is not performed on the local machine. </td>
         </tr>
        </tbody>
       </table> 
       <p><br> </p>
       <p>Because the performance of your Freerunner is too poor for voice recognition, the Google Voice API can be used to convert a <a rel="nofollow" class="external text" href="http://wiki.openmoko.org/wiki/Applications#Audio">recorded Audio file</a> into a text file. Be aware that the audio file will be transmitted to Google and the recognition is not performed on your FR. This implies, that you need to have Internet access on your freerunner FR to submit the audio file. </p> 
       <table style="background-color:#ddddee;color:#000000;border:1px solid #606060;margin:1em 5% 1em 5%;"> 
        <tbody>
         <tr> 
          <td style="padding:0.5em 0.5em;"> </td> 
          <td style="padding:0.5em 0.5em;"> <b>NOTE:</b> You must be aware of the fact, that the follow script is running on your freerunner but it is not a standalone voice recognition software and so you might not want to use this tool for private audio files. </td>
         </tr>
        </tbody>
       </table>

       <h2> <span class="mw-headline" id="Google_Voice_API">Google Voice API</span></h2> 
       <p>For using the Google Voice API and the script you need to have the following package installed on your freerunner: </p> 
       <ul>
        <li> <b>SoX</b> <a rel="nofollow" class="external free" href="http://sox.sourceforge.net">http://sox.sourceforge.net</a>for converting WAV into FLAC files </li>
        <li> <b>WGET</b> <a rel="nofollow" class="external free" href="http://www.gnu.org/s/wget/">http://www.gnu.org/s/wget/</a> for submitting the FLAC file to the Google Voice API </li>
        <li> <b>SED</b> <a rel="nofollow" class="external free" href="http://sed.sourceforge.net/">http://sed.sourceforge.net/</a> for extracting the recognized text in the returned string of the Google Voice API </li>
       </ul> 
       <p>Install the packages from the repositories of the freerunner <a href="/wiki/Distributions" title="Distributions">Distributions</a>. </p> 
       <h2> <span class="mw-headline" id="Script_Usage">Script Usage</span></h2> 
       <ul>
        <li> The script <tt>googlevoice.sh</tt> uses a the audio file <tt>message.wav</tt> in the directory of the script. All files are stored in the same directory, so you need write permissions for that directory. </li>
        <li> SoX converts <tt>message.wav</tt> into <tt>message.flac</tt> </li>
        <li> wget submits the file <tt>message.flac</tt> to the Google Voice API and writes the return message to <tt>message.ret</tt>. The language variable in the script is set to German by <tt>lang=de-de</tt>. If you want to submit a recorded file in US-English use <tt>lang=en-us</tt> instead. </li>
        <li> SED extracts the recognized text <tt>message.ret</tt> by regular expressions and writes the text into <tt>message.txt</tt>. </li>
        <li> Temporary files <tt>message.flac</tt> and <tt>message.ret</tt> will be deleted after the process. </li>
       </ul> 
       <table style="background-color:#ddddee;color:#000000;border:1px solid #606060;margin:1em 5% 1em 5%;"> 
        <tbody>
         <tr> 
          <td style="padding:0.5em 0.5em;"> </td> 
          <td style="padding:0.5em 0.5em;"> <b>NOTE:</b> The return code of German audio files needs capitalization of nouns, because all words are return in small caps. A <tt>ispell</tt> or <tt>aspell</tt> correction of the <tt>message.txt</tt> might improve the recognized text. </td>
         </tr>
        </tbody>
       </table> 
       <p><br> </p> 
       <h2> <span class="mw-headline" id="Basic_Script_Code">Basic Script Code</span></h2> 
       <p>The script code <tt>googlevoice.sh</tt> can be tested on any Linux machine with SoX, SED, WGET installed. Modifiy the script according to your demands and storage of your audio files </p> 
       <pre> #!/bin/sh
 echo "1 SoX Sound Exchange - Convert WAV to FLAC with 16000" 
 sox message.wav message.flac rate 16k
 echo "2 Submit to Google Voice Recognition"
 wget -q -U "Mozilla/5.0" --post-file message.flac --header="Content-Type: audio/x-flac; rate=16000" -O - "<a rel="nofollow" class="external free" href="http://www.google.com/speech-api/v1/recognize?lang=de-de&amp;client=chromium">http://www.google.com/speech-api/v1/recognize?lang=de-de&amp;client=chromium</a>" &gt; message.ret
 echo "3 SED Extract recognized text" 
 cat message.ret | sed 's/.*utterance":"//' | sed 's/","confidence.*//' &gt; message.txt
 echo "4 Remove Temporary Files"
 rm message.flac
 rm message.ret
 echo "5 Show Text "
 cat message.txt
</pre> 
       <p>The parameter <tt>lang=de-de</tt> is indicating, that the Google Voice API is expecting a German language audio file. Replace <tt>lang=de-de</tt> by <tt>lang=en-us</tt> to submit an audio file in US-English. </p> 
       <h2> <span class="mw-headline" id="Script_with_Language_Setting_and_Command_Line_Parameter"> Script with Language Setting and Command Line Parameter </span></h2> 
       <p>The script <tt>googlevoicepar.sh</tt> with a command line parameter can be used if you want to use multiple input files for batch file recognition. You will call this script with the basename e.g. <tt>message0, message1,...</tt> by </p> 
       <ul>
        <li> <tt>googlevoicepar.sh message0</tt> converts <tt>message0.wav</tt> into <tt>message0.txt</tt> </li>
        <li> <tt>googlevoicepar.sh message1</tt> converts <tt>message1.wav</tt> into <tt>message1.txt</tt> </li>
        <li> <tt>googlevoicepar.sh message2</tt> converts <tt>message2.wav</tt> into <tt>message2.txt</tt> </li>
        <li> .... </li>
       </ul> 
       <pre> #!/bin/sh
 LANGUAGE="de-de"
 echo "1 SoX Sound Exchange - Convert $1.wav to $1.flacC with 16000" 
 sox $1.wav $1.flac rate 16k
 echo "2 Submit to Google Voice Recognition $1.flac"
 wget -q -U "Mozilla/5.0" --post-file $1.flac --header="Content-Type: audio/x-flac; rate=16000" -O - "<a rel="nofollow" class="external free" href="http://www.google.com/speech-api/v1/recognize?lang=${LANGUAGE}&amp;client=chromium">http://www.google.com/speech-api/v1/recognize?lang=${LANGUAGE}&amp;client=chromium</a>" &gt; $1.ret
 echo "3 SED Extract recognized text" 
 cat $1.ret | sed 's/.*utterance":"//' | sed 's/","confidence.*//' &gt; $1.txt
 echo "4 Remove Temporary Files"
 rm $1.flac
 rm $1.ret
 echo "5 Show Text "
 cat $1.txt
</pre> 
       <h2> <span class="mw-headline" id="Compare_Performance_and_Results_of_Google_Voice_API_with_Standalone_Speech_Recognition_on_Linux"> Compare Performance and Results of Google Voice API with Standalone Speech Recognition on Linux </span></h2> 
       <p>As mentioned above is speech recognition will not be performed on your Freerunner, the audio file will be submitted to a remote server and the the remote server will return an XML-File with info on accuracy and the recognized text. To compare the performance with a standalone speech recognition software, you have to test that on a Linux Box (e.g. Ubuntu) </p> 
       <ul>
        <li> as a text environment of a Large Vocabulary Speech Recognition <a rel="nofollow" class="external text" href="https://launchpad.net/ubuntu/maverick/+package/julius">Julius</a> was used with the Julius-VoxForge Accoutic Model for English from the repository on Ubuntu 10.10. </li>
        <li> Install the PPA for Ubuntu 10.10 by </li>
       </ul> 
       <pre> sudo add-apt-repository ppa:grasch-simon-listens/simon
</pre> 
       <ul>
        <li> Update repository data </li>
       </ul> 
       <pre> sudo apt-get update
</pre> 
       <ul>
        <li> Install OpenSource Speech Recognition Software <a rel="nofollow" class="external text" href="http://sourceforge.net/projects/speech2text/">Simon</a> via Ubuntu Software-Center and configure Simon (see <a rel="nofollow" class="external text" href="http://simon.gibolles.com/doc/0.3/simon/en/index.pdf">PDF-Simon-Tutorial</a>). </li>
       </ul> 
       <pre> sudo apt-get install simon
</pre> 
       <ul>
        <li> Start Simon from Commandline and test a few words with your mic and record it to an audio file. </li>
       </ul> 
       <pre> simon
</pre> 
       <ul>
        <li> Compare results with same audio file by Google Voice API and Simon. </li>
       </ul> 
       <table style="background-color:#ddddee;color:#000000;border:1px solid #606060;margin:1em 5% 1em 5%;"> 
        <tbody>
         <tr> 
          <td style="padding:0.5em 0.5em;"> </td> 
          <td style="padding:0.5em 0.5em;"> <b>NOTE:</b> These test might help to create a remote Speech Recognition Solution based on Julius and Simon for usage on a Freerunnner. Simon can run as server. Tests can be used to connect your freerunner to Simon. </td>
         </tr>
        </tbody>
       </table> 
       <p><br> </p> 
       <h2> <span class="mw-headline" id="Future_Development">Future Development</span></h2> 
       <p>Google API will always need internet access like other speech recognition application for smart phones. </p> 
       <table style="background-color:#ddddee;color:#000000;border:1px solid #606060;margin:1em 5% 1em 5%;"> 
        <tbody>
         <tr> 
          <td style="padding:0.5em 0.5em;"> </td> 
          <td style="padding:0.5em 0.5em;"> <b>NOTE:</b> Also other Speech recognition apps on Android (FlexT9) or on the iPhone (Dragon Natural Speaking for iPhone) are performing the transcription of an audio sample on the server and returning the transcript to the client mobile phone. That has to be done because of the limitations of the hardware on a mobile phone. This will improve in the future. </td>
         </tr>
        </tbody>
       </table> 
       <p><br> </p>
       <p>In analogy to the commercial apps for the development of <a rel="nofollow" class="external text" href="http://en.wikipedia.org/wiki/Speech_recognition_in_Linux">OpenSource standalone Software on Linux</a> it might be good to have an OpenSource-Webinterface or an Android app to collect Audio Samples for improving the user independent Speech Recognition Profiles HMM for Speech Recognition of large vocabulary and different languages. </p> 
       <ul>
        <li> Users will get the speech recognition result on the freerunner or any other linux box, </li>
        <li> The can correct the speech recognition result (transcript) and submit the correction back to the server </li>
        <li> By the Audio-File-Submit and Text-File-Return of a server based speech recognition the Open Source Speech Recognition on Linux can be improved. </li>
        <li> Most advanced candidate seems to <a rel="nofollow" class="external text" href="http://www.simon-listens.org">Simon Listens</a> with server based backend (Tested on Ubuntu). </li>
       </ul> 
       <p><b>Google Streaming Remote Engine</b> - look over the chrome <a rel="nofollow" class="external text" href="http://src.chromium.org/viewvc/chrome/trunk/src/content/browser/speech/google_streaming_remote_engine.h?view=markup">source</a>. As you can see in the long introductory comment from the sourcecode from the link, the javascript api work includes a separate streaming feature where there is a full-duplex Http connection. This allows a long running connection for chunking audio data on the submit side. No longer do you have to segment audio clips longer than 15 seconds as mentioned earlier. While there is more data, you just use the Http connection to write another chunk on the upload channel. The download stream returns the json containing hypotheses and utterance elements. </p>
