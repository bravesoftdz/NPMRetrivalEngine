         <h3><a href="#p111719">Re: Speech Recognition with Pocketsphinx</a></h3> 
         <ul class="posts-buttons"> 
          <li> <a href="./posting.php?mode=quote&amp;f=37&amp;p=111719&amp;sid=dc5b2c86ecfe6e61fd1976526a314349" title="Reply with quote"><i class="fa fa-quote-left"></i><span>Quote</span></a> </li> 
         </ul> 
         <p class="author"><a href="./viewtopic.php?p=111719&amp;sid=dc5b2c86ecfe6e61fd1976526a314349#p111719">Fri Jun 29, 2012 2:01 pm</a> </p> 
         <div class="content">
          I like to suggest another alternative: Google Speech API and off-load the recognition to google, here's a quick example:
          <br> 
          <div class="codebox">
           <p>Code: <a href="#" onclick="selectCode(this); return false;">Select all</a></p>
           <pre><code>arecord -D plughw:1,0 -f cd -t wav -d 3 -r 16000 | flac - -f --best --sample-rate 16000 -o out.flac; wget -O - -o /dev/null --post-file out.flac --header="Content-Type: audio/x-flac; rate=16000" http://www.google.com/speech-api/v1/recognize?lang=en | sed -e 's/[{}]/''/g'
</code></pre>
          </div> This came from 
          <a href="http://mikepultz.com/2011/03/accessing-google-speech-api-chrome-11/" class="postlink">here</a>.
          <br> 
          <br> There's a C library 
          <a href="https://github.com/H2CO3/libsprec" class="postlink">libsprec</a> but I'm still struggling with the dependencies on the r-pi. 
          <br> 
          <br> I would like to hear from others on the accuracy of this approach.
         </div> 
        </div>

          Just a tip: If you install the python development headers (python-dev and/or python2.7-dev) before building sphinxbase and pocketsphinx, the Python API module will be installed by default. This way, you can simply:
          <br> 
          <br> import pocketsphinx as ps
          <br> speechRec = ps.Decoder()
          <br> wavFile = file(wavfile,'rb')
          <br> wavFile.seek(44)
          <br> speechRec.decode_raw(wavFile)
          <br> result = speechRec.get_hyp()
          <br> print result[0]
