                    <h3 class="post-title entry-title" itemprop="name"> Mission Package 2 - Voice Recognition for AVA (Part 2) </h3> 
                    <div class="post-header"> 
                     <div class="post-header-line-1"></div> 
                    </div> 
                    <div class="post-body entry-content" id="post-body-8236872918386361514" itemprop="description articleBody"> 
                     <div dir="ltr" style="text-align: left;" trbidi="on"> 
                      <h3 style="text-align: left;"> USB Microphone</h3> 
                      <br> You will of course need a USB microphone in order for the Pi to hear you speak. You can buy a USB adaptor for a standard 3.5mm mic but I went with the&nbsp;SunFounder USB 2.0 Mini Microphone for Raspberry Pi 3, 2 Module B &amp; RPi 1 Model B+/B (figure 1).
                      <br> 
                      <br> 
                      <div class="separator" style="clear: both; text-align: center;"> 
                       <a href="https://3.bp.blogspot.com/-jJxnHZPqMbk/WHXB3FbcsgI/AAAAAAAABw4/GEXwaOmL4Soln-7lhp_mpKD5k_519bxSgCLcB/s1600/6.jpeg" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" height="320" src="https://3.bp.blogspot.com/-jJxnHZPqMbk/WHXB3FbcsgI/AAAAAAAABw4/GEXwaOmL4Soln-7lhp_mpKD5k_519bxSgCLcB/s320/6.jpeg" width="320"></a>
                      </div> 
                      <div class="separator" style="clear: both; text-align: center;"> 
                       <br>
                      </div> 
                      <div class="separator" style="clear: both; text-align: center;">
                        Figure 1.&nbsp;
                       <span style="text-align: left;">SunFounder USB 2.0 Mini Microphone</span>
                      </div> 
                      <br> This is where a Raspberry Pi 3 may be a better choice. With built in WiFi you have an extra free USB port.
                      <br> 
                      <br> 
                      <h3 style="text-align: left;"> Pi AUI Suite</h3> 
                      <div> 
                       <br>
                      </div> 
                      <div class="separator" style="clear: both; text-align: center;"> 
                       <iframe allowfullscreen class="YOUTUBE-iframe-video" data-thumbnail-src="https://i.ytimg.com/vi/SuD42ZPOto8/0.jpg" frameborder="0" height="266" src="https://www.youtube.com/embed/SuD42ZPOto8?feature=player_embedded" width="320"></iframe>
                      </div> 
                      <br> 
                      <br> For the voice recognition package our first test was using&nbsp;
                      <a href="https://github.com/StevenHickson/PiAUISuite" target="_blank">Steven Hickson's Pi AUI Suite</a>. The thing we like best about this software is the ability to create your own word lists and associated actions. The suite is up to version 3 and you can read about it on 
                      <a href="http://stevenhickson.blogspot.com.au/" target="_blank">Steve's blog</a>. The one caveat is that it appears that the suite is no longer being developed or supported. In particular text to speech (i.e. when the Raspberry Pi responds to a command) appears to be problematic. We will develop an alternative approach to get around this.
                      <br> 
                      <br> A quick summary of Pi AUI Suite. The configuration is done in the commands.conf file (more on this later).
                      <br> 
                      <ul style="text-align: left;"> 
                       <li>The format is voice==command (where voice is the word which AUI Suite looks for)</li> 
                       <li>You can use any character except for newlines or ==&nbsp;</li> 
                       <li>If the voice starts with ~, the program looks for the keyword anywhere. Ex: ~weather would pick up on "weather" or "what's the weather"&nbsp;</li> 
                       <li>You can use ... at the end of the command to specify that everything after the given keyword should be options to the command. For example: play==playvideo ... This means that if you say "play Futurama", it will run the command playvideo Futurama&nbsp;</li> 
                       <li>You can use $# (where # is any number 1 to 9) to represent a variable. These should go in order from 1 to 9. For example: $1 season $2 episode $3==playvideo -s $2 -e $3 $1 This means if you say "game of thrones season 1 episode 2", it will run playvideo with the -s flag as 1, the -e flag as 2, and the main argument as game of thrones, i.e. playvideo -s 1 -e 2 game of thrones&nbsp;</li> 
                       <li>Because of these options, it is important that the arguments range from most strict to least strict.&nbsp;</li> 
                       <li>This means that ~ arguments should probably be at the end.&nbsp;</li> 
                       <li>You can also put comments if the line starts with # and special options if the line starts with a !&nbsp;</li> 
                       <li>Default options are shown as follows:&nbsp;</li> 
                       <ul> 
                        <li>!keyword==pi,</li> 
                        <li>!verify==1,</li> 
                        <li>!continuous==1,</li> 
                        <li>!quiet==0,</li> 
                        <li>!ignore==0,</li> 
                        <li>!thresh==0.7,</li> 
                        <li>!maxResponse==-1&nbsp;</li> 
                        <li>api==BLANK,</li> 
                        <li>!filler==FILLER FILL,</li> 
                        <li>!response==Yes Sir?,</li> 
                        <li>!improper==Received improper command:,</li> 
                        <li>!duration==3,</li> 
                        <li>!com_dur==2,</li> 
                        <li>!hardware==plughw:1,0,</li> 
                        <li>!language==en_us&nbsp;</li> 
                       </ul> 
                       <li>Keyword, filler, and response accept strings. verify, continuous, quiet, and ignore except 1 or 0 (true or false respectively). thresh excepts a floating point number. These allow you to set some of the flags as permanent options (If these are set, you can overwrite them with the flag options).&nbsp;</li> 
                       <li>You can set a WolframAlpha API and maxResponse (the number of branches) like !api==XXXXXX-XXXXXXXXXX amd !maxResponse==3&nbsp;</li> 
                       <li>You can now customise the language support for speech recognition and some text to speech with the language flag. Look up your country code and use that. Ex. For US: !language==en_us, for Spain !language==es, for Germany !language==de.</li> 
                      </ul> 
                      <br> 
                      <ul style="text-align: left;"> 
                      </ul> 
                      <h3 style="text-align: left;"> Installing Pi AUI Suite</h3> 
                      <br> The install process is fairly straight forward. Start by installing the dependencies. Open up Terminal or connect via SSH and type the following (all on one line):
                      <br> 
                      <br> 
                      <pre style="background: rgb(242, 241, 240); border: 1px solid rgb(221, 221, 221); line-height: 20px; max-width: 100%; padding: 10px; width: 540px;"><span style="color: #222222; font-family: &quot;courier&quot; , monospace;"><span style="font-size: 14px; white-space: pre-wrap;">sudo apt-get install -y libboost-dev libboost-regex-dev youtube-dl axel curl xterm libcurl4-gnutls-dev mpg123 flac sox</span></span></pre> 
                      <br> Then download the install files:
                      <br> 
                      <br> 
                      <pre style="background: rgb(242, 241, 240); border: 1px solid rgb(221, 221, 221); line-height: 20px; max-width: 100%; padding: 10px; width: 540px;"><span style="color: #222222; font-family: &quot;courier&quot; , monospace;"><span style="font-size: 14px; white-space: pre-wrap;">git clone https://github.com/StevenHickson/PiAUISuite.git</span></span></pre> 
                      <br> Change to the install directory:
                      <br> 
                      <br> 
                      <pre style="background: rgb(242, 241, 240); border: 1px solid rgb(221, 221, 221); color: #222222; font-family: courier, monospace; font-size: 14px; line-height: 20px; max-width: 100%; padding: 10px; white-space: pre-wrap; width: 540px;"><code style="background: transparent; border: 0px; font-family: Courier, monospace; padding: 0px;">cd PiAUISuite/Install</code></pre> 
                      <br> Run the install command:
                      <br> 
                      <br> 
                      <pre style="background: rgb(242, 241, 240); border: 1px solid rgb(221, 221, 221); color: #222222; font-family: courier, monospace; font-size: 14px; line-height: 20px; max-width: 100%; padding: 10px; white-space: pre-wrap; width: 540px;"><code style="background: transparent; border: 0px; font-family: Courier, monospace; padding: 0px;">sudo ./InstallAUISuite.sh</code></pre> 
                      <br> Choose y to install any dependencies required and work your way through the installation wizard. If you encounter an error during this process (I did) then install the Boost C++ library:
                      <br> 
                      <br> 
                      <pre style="background: rgb(242, 241, 240); border: 1px solid rgb(221, 221, 221); color: #222222; font-size: 14px; line-height: 20px; max-width: 100%; padding: 10px; white-space: pre-wrap; width: 540px;"><span style="font-family: &quot;courier&quot; , monospace;">sudo apt-get install libboost-regex1.49.0</span></pre> 
                      <br> After this library has installed, Run&nbsp;
                      <span style="background-color: #f3f3f3;"><span style="color: #222222; font-family: &quot;courier&quot; , monospace; font-size: 14px; white-space: pre-wrap;">sudo ./InstallAUISuite.sh</span>&nbsp;</span>again.
                      <br> 
                      <br> 
                      <h3 style="text-align: left;"> Alternate Text to Speech Solution</h3> 
                      <br> After running the install wizard you can use the setup wizard to generate your command configuration file. To run the setup wizard:
                      <br> 
                      <br> 
                      <pre style="background: rgb(242, 241, 240); border: 1px solid rgb(221, 221, 221); color: #222222; font-family: courier, monospace; font-size: 14px; line-height: 20px; max-width: 100%; padding: 10px; white-space: pre-wrap; width: 540px;"><code style="background: transparent; border: 0px; font-family: Courier, monospace; padding: 0px;">sudo voicecommand -s</code></pre> 
                      <br> As part of this process you have the option to test text to speech (TTS). This appears to be no longer working. The Pi AUI Suite is using the Google TTS API and in early 2016 Google stopped letting automated scripts access the service. You can read about this issue on the 
                      <a href="https://github.com/StevenHickson/PiAUISuite/issues/56" target="_blank">Pi AUI Suite GitHub repository</a>.
                      <br> 
                      <br> One workaround is to install the speech synthesis software called Festival:
                      <br> 
                      <br> 
                      <pre style="background: rgb(242, 241, 240); border: 1px solid rgb(221, 221, 221); color: #222222; font-family: courier, monospace; font-size: 14px; line-height: 20px; max-width: 100%; padding: 10px; white-space: pre-wrap; width: 540px;"><code style="background: transparent; border: 0px; font-family: Courier, monospace; padding: 0px;">sudo apt-get install festival</code></pre> 
                      <br> You can test that this is working by typing the following:
                      <br> 
                      <br> 
                      <pre style="background: rgb(242, 241, 240); border: 1px solid rgb(221, 221, 221); color: #222222; font-family: courier, monospace; font-size: 14px; line-height: 20px; max-width: 100%; padding: 10px; white-space: pre-wrap; width: 540px;">echo "Test message" | festival --tts</pre> 
                      <br> You will obviously need to have some speakers plugged into the 3.5mm audio out plug. You can adjust the volume using:
                      <br> 
                      <br> 
                      <pre style="background: rgb(242, 241, 240); border: 1px solid rgb(221, 221, 221); color: #222222; font-size: 14px; line-height: 20px; max-width: 100%; padding: 10px; white-space: pre-wrap; width: 540px;"><span style="font-family: &quot;courier&quot; , monospace;">alsamixer</span></pre> 
                      <br> Then use the up and down arrows to adjust volume. ESC to save and exit.
                      <br> 
                      <br> 
                      <h3 style="text-align: left;"> Configuring Pi AUI Suite</h3> 
                      <br> You can run voicecommand using the default configuration options detailed above, but you will probably want to tweak the available commands. The first thing we need to do is move the configuration file because we are running the program as the root user. To do this type:
                      <br> 
                      <br> 
                      <pre style="background: rgb(242, 241, 240); border: 1px solid rgb(221, 221, 221); color: #222222; font-size: 14px; line-height: 20px; max-width: 100%; padding: 10px; white-space: pre-wrap; width: 540px;"><span style="font-family: &quot;courier&quot; , monospace;">sudo mv /home/pi/.commands.conf /root/.commands.conf</span></pre> 
                      <br> To edit the configuration file:
                      <br> 
                      <br> 
                      <pre style="background: rgb(242, 241, 240); border: 1px solid rgb(221, 221, 221); color: #222222; font-size: 14px; line-height: 20px; max-width: 100%; padding: 10px; white-space: pre-wrap; width: 540px;"><span style="font-family: &quot;courier&quot; , monospace;">sudo voicecommand -e</span></pre> 
                      <br> or if you prefer using your choice of text editor you can type something like:
                      <br> 
                      <br> 
                      <pre style="background: rgb(242, 241, 240); border: 1px solid rgb(221, 221, 221); color: #222222; font-family: courier, monospace; font-size: 14px; line-height: 20px; max-width: 100%; padding: 10px; white-space: pre-wrap; width: 540px;"><code style="background: transparent; border: 0px; font-family: Courier, monospace; padding: 0px;">sudo nano /root/.commands.conf</code></pre> 
                      <br> You can now change your keyword or add extra commands. For example, if you want the Raspberry Pi to tell you the current time, you could add:
                      <br> 
                      <br> 
                      <pre style="background: rgb(242, 241, 240); border: 1px solid rgb(221, 221, 221); color: #222222; font-size: 14px; line-height: 20px; max-width: 100%; padding: 10px; white-space: pre-wrap; width: 540px;"><span style="font-family: &quot;courier&quot; , monospace;">time==echo "The time is " | festival --tts &amp;&amp; date | festival --tts</span></pre> 
                      <br> CTRL X to exit, select y to save and then return.
                      <br> 
                      <br> 
                      <h3 style="text-align: left;"> Running Pi AUI Suite</h3> 
                      <br> To run voice command:
                      <br> 
                      <br> 
                      <pre style="background: rgb(242, 241, 240); border: 1px solid rgb(221, 221, 221); color: #222222; font-size: 14px; line-height: 20px; max-width: 100%; padding: 10px; white-space: pre-wrap; width: 540px;"><span style="font-family: &quot;courier&quot; , monospace;">sudo voicecommand -c</span></pre> 
                      <br> The program will then listen out for your keyword and attempt to respond.
                      <br> 
                      <br> In the next post we will evaluate performance of Pi AUI Suite and look at alternatives.
                      <br> 
                      <br>
