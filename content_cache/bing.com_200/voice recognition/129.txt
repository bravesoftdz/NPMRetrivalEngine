    <h3>About</h3> 
    <p>This library used to bring speech recognition to your Processing applications. Due to limitations in the involved speech to text API, the library has been replaced by a workaround that involves Google Chrome.</p> 
    <p>As of December 2014, further development has been discontinued. <a href="archive/">Looking for the old library</a>?</p> 
    <h3>Workaround</h3> 
    <p>Using WebSocket, Google Chrome and Processing, you can get unlimited speech recognition results in your Processing sketch.</p> 
    <p>All you need is a running instance of Google Chrome with a bit of JavaScript that will handle the speech recognition. Every transcript will then be sent to Processing via WebSockets.</p> 
    <p>Check out the example below to get started.</p> 
    <h3>Requirements</h3> 
    <p> </p>
    <ul> 
     <li><a href="https://processing.org">Processing IDE</a></li> 
     <li><a href="https://github.com/alexandrainst/processing_websockets">WebSockets for Processing (Library)</a> </li>
     <li><a href="https://www.google.com/chrome">Google Chrome</a></li> 
    </ul> 
    <p></p> 
    <h3>Demo</h3> 
    <p> <iframe src="https://player.vimeo.com/video/122245227" width="500" height="297" frameborder="0" webkitallowfullscreen mozallowfullscreen allowfullscreen></iframe> </p> 
    <h3>Setup</h3> 
    <p>Install the <a href="https://github.com/alexandrainst/processing_websockets">WebSockets Library</a> and create a new sketch with the example <a href="https://gist.githubusercontent.com/getflourish/96b663af0e4418b11e76/raw/4c04d3ffba28e682ddaa28219a076611e193f33f/receiver.pde">source code</a>. Open this <a href="https://codepen.io/getflourish/pen/NpBGqe">CodePen</a> to start the voice recording in your Chrome web browser. Transcribed speech recognition results will now be sent to the running Processing Sketch and printed to the console.</p> 
    <p>If you don’t want to use the CodePen example, you can create your own HTML/JS file and access it from your own server or localhost.</p>
