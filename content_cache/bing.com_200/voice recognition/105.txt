             <h2>Introduction</h2> 
             <p>This is part of a larger project on speech recognition we developed at ORT Braude college. The aim of the project is to activate programs on your desktop or panel by voice. </p> 
             <h2>Motivation</h2> 
             <p>We planned to make some common tasks that every user does on his/her computer (opening/ closing programs, editing texts, calculating) possible not only by mouse/ keyboard, but also by voice. </p> 
             <h2>Background</h2> 
             <p>Every speech recognition application consists of: </p> 
             <ul> 
              <li>An engine that translates waves into text </li> 
              <li>A list of speech commands </li> 
             </ul> 
             <p>Needless to say that as the grammar increases, the probability of misinterpretations grows. We tried to keep the grammar as small as possible without loosing information. The grammar format is explained latter. </p> 
             <h2>Requirements</h2> 
             <ul> 
              <li>We need SAPI5 (ships with XP) </li> 
              <li>Microsoft Engine for English (if not found can be downloaded from Microsoft's site) </li> 
             </ul> 
             <p>The easiest way to check if you have these is to enter your control panel-&gt; speech. Here you should see the "Text to Speech" tab AND the "Speech recognition" tab. If you don't see the "Speech Recognition" tab then you should download it from the Microsoft site. </p> 
             <ul> 
              <li><a href="http://www.microsoft.com/downloads/details.aspx?FamilyID=5e86ec97-40a7-453f-b0ee-6583171b4530&amp;DisplayLang=en">Speech SDK 5.1</a> </li> 
             </ul> 
             <h2>Some Technical Stuff</h2> 
             <ul> 
              <li>The program is designed to run from its source directory. So, while compiling and running it from the .NET, you will need to copy to the <em>\bin\debug\</em> directory the agent file <em>merlin.acs</em>. I've included it in the demo in case you don't have it in your computer. You can still download from Microsoft's site any agent you want. </li> 
              <li>For the same reason, if you changed an XML file you will need to copy it also to the <em>\bin\debug\</em> directory. </li> 
             </ul> 
             <h2>How to Start</h2> 
             <p>The project's interface is shown bellow (Fig 1). </p> 
             <img height="351" src="/KB/audio-video/tambiSR/fig01.jpg" width="294" border="0"> 
             <p>In order to start talking right away, you should do these two steps... </p> 
             <ol> 
              <li>The first (and important) thing to do is adjust the microphone by clicking the right mouse button and choosing the "Mic training wizard." </li> 
              <li>The second (also important) thing to do is training the engine to your voice by choosing "User training wizard." </li> 
             </ol> 
             <p><strong>IMPORTANT:</strong> after these changes, you will need to make the program start listening again by clicking the right mouse button and choosing "Start listen." The more you train the engine, the better it will recognize your voice, although you will see an improvement from the first training. After the program is started, it may be in several "states". In every state, it recognizes a list of specific commands. The list of the commands that the program can identify is shown below. </p> 
             <p>A little explanation of the menu... </p> 
             <ul> 
              <li>"Start listen"/"Stop listen" <p>To enable/disable the mic (it's switched according to what you choose), after disabling the label's becomes red (accuracy and state) indicating our state. </p> </li> 
              <li>"Use agent" <p>Although the agent is used only for giving feedback, it could be useful to know if your command is heard or not. This is so even though you can disable it if you want or if you don't have an agent file (can be downloaded from Microsoft, ACS files) or if it is not working and you still want to use the recognition (there is no connection between the agent and the recognition). This also is being taken care of if the program didn't find the agent file or could not be loaded from any other reason. </p> </li> 
              <li>"Add favorites" <p>In the "activate" state you can say the command "favorites programs" and open a form with your favorites programs and running them by saying the program name. This menu will open a form showing your favorites programs so you can add/delete or edit them as you want.</p> </li> 
              <li>"Change character" <p>This will allow you changing the agent character (can download them from Microsoft site, ACS files). </p> </li> 
              <li>"Change accuracy limit" <p>Every recognition accuracy is displayed in the "Accuracy" label. You can choose this menu and change the accuracy limit that you want the program to respond to the command that he hears with. You should do this to avoid responding to any voice or sound that he hears. you can raise this more every time that you train your computer and increase the recognition. </p> </li> 
              <li>"Change user profile" <p>If the program is being used by several users, you can choose to give each user a profile and train the computer for each one (to add a user profile enter "control panel -&gt; speech." Here you can only choose existing ones). </p> </li> 
              <li>"Mic training wizard..." <p>This is very <strong>important</strong> (as I explained before) for the recognition. The first thing to do in every computer (only at the first time) is to activate this menu and setting up your mic or if you changed your mic to a new one. </p> </li> 
              <li>"User training wizard..." <p>For a better recognition (notice that the training is for the selected user profile). </p> </li> 
             </ul> 
             <h2>How it Works</h2> 
             <p>The initial state is in the "deactivate" state, which means that the program is in a sleepy state... After the command "activate" you will wake up the program ("activate" state) and start recognizes other commands (Fig 2). </p> 
             <img height="261" src="/KB/audio-video/tambiSR/fig02.jpg" width="233" border="0"> 
             <p>For example, use <strong>"start"</strong> to activate the start menu. Then you can say <strong>"programs"</strong> to enter the programs menu. From this point, you can navigate by saying <strong>"down"," up", "right"... "OK"</strong> according the commands list. You can also say <strong>"commands list"</strong> from any point to see a form with the list of the commands that you can say. </p> 
             <p>One of the important states in the program is the "menu" state, meaning that if a program is running (and focused) you can say <strong>"menu"</strong> to hook all menu items and start using them. For example, if you are running Notepad you could open new file by saying <strong>"menu"-&gt;"File"-&gt;"new file"<em>. </em></strong>Every time that you hook menu, you can see how many menus the program hooked so you can start using them as commands. I had a little problem with some menus like "Word" and "Excel" that I couldn't hook, but... I'll check it later. </p> 
             <p>Another nice state is "Numeric state". For example, say the commands <strong>"favorites programs","calculator","enter numeric state", "one","plus","two","equal"</strong> and see the result. Alternatively, you can open a site in "Alphabetic state". For example, say the commands <strong>"favorites programs","internet explorer","enter alphabetic state", "menu","down","down","O K", "enter alphabetic state","c","o","d","e",...,"dot","c","o","m"</strong> and see the result.</p>

             <h2>Code Explanation</h2> 
             <p>The first thing to do is to add reference to the file... <em>C:\Program Files\Common Files\Microsoft Shared\Speech\SAPI.dll</em> so we can use the Speech Library by writing... </p> 
             <pre lang="cs"><span class="code-keyword">using</span> SpeechLib;</pre> 
             <p>When we activate the engine, the initialization step takes place. There are mainly 3 objects involved: </p> 
             <ol> 
              <li>An <code lang="cs">SpSharedRecoContext</code> that starts the recognition process (must be shared so it can apply to all processes). It implements an <code lang="cs">ISpeechRecoContext</code> interface. After this object is created, we add the events we are interested in (in our case AudioLevel and Recognition) </li> 
              <li>A static grammar object that can be loaded from XML file or programmatically implements <code lang="cs">ISpeechRecoGrammar</code> the list of static recognizable words is shown in Fig 2 and attached for downloading dynamic grammar that lets adding rules implement <code lang="cs">ISpeechGrammarRule;. T</code>he rule has two main parts: 
               <ul> 
                <li>The phrase associated </li> 
                <li>The name of the rule </li> 
               </ul> </li> 
             </ol> 
             <p>Three basic functions that we will need... </p> 
             <ul> 
              <li><code lang="cs">initSAPI()</code>: To create grammar interface and activating interested events </li> 
              <li><code lang="cs">SAPIGrammarFromFile(string FileName)</code>: To load grammar from file </li> 
              <li><code lang="cs">SAPIGrammarFromArrayList(ArrayList PhraseList)</code>: To change grammar programmatically </li> 
             </ul> 
             <pre lang="cs"><span class="code-keyword">private</span> <span class="code-keyword">void</span> initSAPI()
{
    <span class="code-keyword">try</span>
    {
        objRecoContext = <span class="code-keyword">new</span> SpeechLib.SpSharedRecoContext();
        objRecoContext.AudioLevel+= 
            <span class="code-keyword">new</span> _ISpeechRecoContextEvents_AudioLevelEventHandler(
            RecoContext_VUMeter);
        objRecoContext.Recognition+= 
            <span class="code-keyword">new</span> _ISpeechRecoContextEvents_RecognitionEventHandler(
            RecoContext_Recognition);
        objRecoContext.EventInterests=
            SpeechLib.SpeechRecoEvents.SRERecognition | 
            SpeechLib.SpeechRecoEvents.SREAudioLevel;
 
         <span class="code-comment">//</span><span class="code-comment">create grammar interface with ID = 0</span>
         grammar=objRecoContext.CreateGrammar(<span class="code-digit">0</span>);
    }
    <span class="code-keyword">catch</span>(Exception ex)
    {
        MessageBox.Show(<span class="code-string">"</span><span class="code-string">Exeption \n"</span>+ex.ToString(),<span class="code-string">"</span><span class="code-string">Error - initSAPI"</span>);
    }
}</pre> 
             <p>After initialization, the engine still will not recognize anything until we load a grammar. There are two ways to do that: loading a grammar from file... </p> 
             <pre lang="cs"><span class="code-keyword">private</span> <span class="code-keyword">void</span> SAPIGrammarFromFile(<span class="code-keyword">string</span> FileName)
{
    <span class="code-keyword">try</span>
    {
        grammar.CmdLoadFromFile(appPath+FileName,
            SpeechLib.SpeechLoadOption.SLODynamic);
        grammar.CmdSetRuleIdState(<span class="code-digit">0</span>,SpeechRuleState.SGDSActive);
    }
    <span class="code-keyword">catch</span>
    {
        MessageBox.Show(<span class="code-string">"</span><span class="code-string">Error loading file "</span>+
            FileName+<span class="code-string">"</span><span class="code-string">\n"</span>,<span class="code-string">"</span><span class="code-string">Error - SAPIGrammarFromFile"</span>);
    }
}</pre> 
             <p>Or we can change the grammar programmatically. The function is getting an <code lang="cs">ArrayList</code> that every item is a structure: </p> 
             <pre lang="cs"><span class="code-keyword">private</span> <span class="code-keyword">struct</span> command
{
    <span class="code-keyword">public</span> <span class="code-keyword">string</span> ruleName;
    <span class="code-keyword">public</span> <span class="code-keyword">string</span> phrase;
}

<span class="code-keyword">private</span> <span class="code-keyword">void</span> SAPIGrammarFromArrayList(ArrayList phraseList)
{
    <span class="code-keyword">object</span> propertyValue=<span class="code-string">"</span><span class="code-string">"</span>;
    command command1;

    <span class="code-keyword">int</span> i;
    <span class="code-keyword">for</span> (i=<span class="code-digit">0</span>;i&lt; phraseList.Count;i++)
    {
        command1=(command)phraseList[i];

        <span class="code-comment">//</span><span class="code-comment">add new rule with ID = i+100</span>
        rule=grammar.Rules.Add(command1.ruleName, 
            SpeechRuleAttributes.SRATopLevel, i+100);

        <span class="code-comment">//</span><span class="code-comment">add new word to the rule</span>
        state=rule.InitialState;
        propertyValue=<span class="code-string">"</span><span class="code-string">"</span>;
        state.AddWordTransition(<span class="code-keyword">null</span>,command1.phrase,<span class="code-string">"</span><span class="code-string"> "</span>, 
            SpeechGrammarWordType.SGLexical, <span class="code-string">"</span><span class="code-string">"</span>, 
            <span class="code-digit">0</span>, <span class="code-keyword">ref</span> propertyValue, 1F);

        <span class="code-comment">//</span><span class="code-comment">commit rules</span>
        grammar.Rules.Commit();

        <span class="code-comment">//</span><span class="code-comment">make rule active (needed for each rule)</span>
        grammar.CmdSetRuleState(command1.ruleName, 
            SpeechRuleState.SGDSActive);
    }
}</pre>

}</pre> 
             <h2>Hooking Menus</h2> 
             <p>When a program is activated, by saying "Menu" its menu is hooked and its commands added to the dynamic grammar. We used some unmanaged functions which we imported from <em>user32.dll</em>. The program also hooks the accelerators that are associated with each menu (that have an &amp; sign before them). The command is simulated with function <code lang="cs">keybd_event</code> and executed. </p> 
             <pre lang="cs"><span class="code-keyword">private</span> <span class="code-keyword">void</span> hookMenu(<span class="code-sdkkeyword">IntPtr</span> hMnu)
{
    <span class="code-comment">//</span><span class="code-comment">reset grammar</span>
    initSAPI();
    SAPIGrammarFromFile(<span class="code-string">"</span><span class="code-string">XMLDeactivate.xml"</span>);

    <span class="code-keyword">int</span> mnuCnt=GetMenuItemCount(hMnu);

    <span class="code-keyword">if</span> (mnuCnt!=<span class="code-digit">0</span>)
    {
        <span class="code-comment">//</span><span class="code-comment">add menu to grammar</span>
        <span class="code-keyword">int</span> i;
        command command1;

        StringBuilder mnuStr=<span class="code-keyword">new</span> StringBuilder(<span class="code-digit">50</span>);

        ArrayList phraseList=<span class="code-keyword">new</span> ArrayList();

        <span class="code-keyword">for</span> (i=<span class="code-digit">0</span>;i <span class="code-keyword">&lt;</span> mnuCnt;i++)
        {
            <span class="code-comment">//</span><span class="code-comment">get sting from menu ... to mnuString</span>
            GetMenuString(hMnu,i,mnuStr,<span class="code-digit">50</span>,-1);

            <span class="code-comment">//</span><span class="code-comment">make sure its not a separator</span>
            <span class="code-keyword">if</span> (mnuStr.ToString()!=<span class="code-string">"</span><span class="code-string">"</span>)
            {
                <span class="code-comment">//</span><span class="code-comment">save in commnd1.ruleName only the underlined letter</span>
                command1.ruleName=mnuStr.ToString();
                command1.ruleName=command1.ruleName[
                    command1.ruleName.IndexOf(<span class="code-string">'</span><span class="code-string">&amp;'</span>)+1].ToString();

                <span class="code-comment">//</span><span class="code-comment">save in command1.phrase the word (without &amp;)</span>
                command1.phrase=mnuStr.ToString();
                command1.phrase=command1.phrase.Remove(
                    command1.phrase.IndexOf(<span class="code-string">'</span><span class="code-string">&amp;'</span>),<span class="code-digit">1</span>);

                phraseList.Add(command1);
            }
        }

        <span class="code-comment">//</span><span class="code-comment">add the phraseList (menu) to grammar</span>
        SAPIGrammarFromArrayList(phraseList);
    }
}</pre>
