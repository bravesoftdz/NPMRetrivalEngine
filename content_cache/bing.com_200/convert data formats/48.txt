                       <h1><a name="Theartofdataconversion-Introduction"></a>Introduction</h1> 
                       <p>Data conversion is not as trivial as it seems at first glance.&nbsp; Many things come into play: local language support, conversion masks, and a lot more.</p> 
                       <p>To solve this once and for all (!) we wrote new conversion algorithms that, while remaining backward compatible with older versions, offer greater ease of use.</p> 
                       <h2><a name="Theartofdataconversion-ValueMeta"></a>ValueMeta</h2> 
                       <p>The ValueMeta class is handling all data conversions in version 3.0.&nbsp; Where data conversion was typically done all over the place in version 2, we have now centralized all the conversion code in this class. <br class="atl-forced-newline"></p> 
                       <h3><a name="Theartofdataconversion-Stringto%3Ctype%3Econversions"></a>String to &lt;type&gt; conversions</h3> 
                       <p><br class="atl-forced-newline"></p> 
                       <p>Typically these conversions happen in "Input" steps such as "Text File Input", "CSV Input", "Fixed Input", "XML Input", etc.</p> 
                       <p>We can handle these one-way conversions by using the follow method in ValueMeta and ValueMetaInterface:</p> 
                       <div class="code panel" style="border-width: 1px;">
                        <div class="codeContent panelContent"> 
                         <pre class="code-java">/**
  * Convert the specified data to the data type specified in <span class="code-keyword">this</span> object.
  * @param meta2 the metadata of the object to be converted
  * @param data2 the data of the object to be converted
  * @<span class="code-keyword">return</span> the object in the data type of <span class="code-keyword">this</span> value metadata object
  * @<span class="code-keyword">throws</span> KettleValueException in <span class="code-keyword">case</span> there is a data conversion error
  */
  <span class="code-keyword">public</span> <span class="code-object">Object</span> convertData(ValueMetaInterface meta2, <span class="code-object">Object</span> data2) <span class="code-keyword">throws</span> KettleValueException;
</pre> 
                        </div>
                       </div> 
                       <h3><a name="Theartofdataconversion-Dateexample"></a>Date example</h3> 
                       <p>Let's take a date String as an example:&nbsp; "999.59:59:23 31/12/2007"</p> 
                       <p>Here is to code to convert this to a Date object: <br class="atl-forced-newline"></p> 
                       <div class="code panel" style="border-width: 1px;">
                        <div class="codeContent panelContent"> 
                         <pre class="code-java">ValueMetaInterface source = <span class="code-keyword">new</span> ValueMeta(<span class="code-quote">"src"</span>, ValueMetaInterface.TYPE_STRING);
source.setConversionMask(<span class="code-quote">"SSS.ss:mm:HH dd/MM/yyyy"</span>);
ValueMetaInterface target = <span class="code-keyword">new</span> ValueMeta(<span class="code-quote">"tgt"</span>, ValueMetaInterface.TYPE_DATE);

Date date = (Date) target.convertData(source, <span class="code-quote">"999.59:59:23 31/12/2007"</span>);
</pre> 
                        </div>
                       </div> 
                       <p>If we want to go the other way around, we need to explain to the target how the Date value should be represented as a String:</p> 
                       <div class="code panel" style="border-width: 1px;">
                        <div class="codeContent panelContent"> 
                         <pre class="code-java">target.setConversionMask(<span class="code-quote">"yy/MM/dd HH:mm"</span>);
<span class="code-object">String</span> string = (<span class="code-object">String</span>) source.convertData(target, date);
</pre> 
                        </div>
                       </div> 
                       <h3><a name="Theartofdataconversion-Stringto%3Ctype%3EtoStringconversions"></a>String to &lt;type&gt; to String conversions</h3> 
                       <p>On occasion you encounter these round trip conversions. For example, in the sorting logic of the TableView (default PDI table editor).&nbsp; The data in TableView is always represented as a String.&nbsp; However, the original data could have certain local language settings or other specific formatting options.&nbsp; Once the data is in a String format, it can be tricky to convert it back to the original data type and back to String.</p> 
                       <p>For that reason we introduced the notion of "conversion meta-data" in ValueMeta.&nbsp; The "conversion meta-data" explains to the ValueMeta object to which it is attached how the conversion should happen in both directions.</p> 
                       <h3><a name="Theartofdataconversion-Dateexample"></a>Date example</h3> 
                       <div class="code panel" style="border-width: 1px;">
                        <div class="codeContent panelContent"> 
                         <pre class="code-java">ValueMetaInterface datValueMeta = <span class="code-keyword">new</span> ValueMeta(<span class="code-quote">"i"</span>, ValueMetaInterface.TYPE_DATE);
datValueMeta.setConversionMask(<span class="code-quote">"yyyy - MM - dd   HH:mm:ss'('SSS')'"</span>);

<span class="code-object">String</span> string = datValueMeta.getString(originalValue); <span class="code-comment">// This string is used in the preview window.
</span>
ValueMetaInterface strValueMeta = <span class="code-keyword">new</span> ValueMeta(<span class="code-quote">"str"</span>, ValueMetaInterface.TYPE_STRING);
strValueMeta.setConversionMetadata(datValueMeta);

Date x = (Date) strValueMeta.convertDataUsingConversionMetaData(string);
</pre> 
                        </div>
                       </div> 
                       <p>&nbsp;This has the distinct advantage that you can now convert the original date format back to the original format without having direct access to the original since you can call getConversionMetaData() on the String object.&nbsp; It guarantees that data conversion will take place as it should in both directions. <br class="atl-forced-newline"> <br class="atl-forced-newline"></p> 
                      </div>
