    <h1>Data types</h1> 
    <p> In this part of the Java tutorial, we will talk about data types. </p> 
    <p> Computer programs, including spreadsheets, text editors, calculators, or chat clients, work with data. Tools to work with various data types are essential part of a modern computer language. A <dfn>data type</dfn> is a set of values and the allowable operations on those values. </p> 
    <div class="big_hor"> 
     <script async src="//pagead2.googlesyndication.com/pagead/js/adsbygoogle.js"></script>

    <p> Java programming language is a statically typed language. It means that every variable and every expression has a type that is known at compile time. Java language is also a strongly typed language because types limit the values that a variable can hold or that an expression can produce, limit the operations supported on those values, and determine the meaning of the operations. Strong static typing helps detect errors at compile time. Variables in dynamically typed languages like Ruby or Python can receive different data types over the time. In Java, once a variable is declared to be of a certain data type, it cannot hold values of other data types. </p> 
    <p> There are two fundamental data types in Java: <em>primitive types</em> and <em>reference types</em>. Primitive types are: </p> 
    <ul> 
     <li>boolean</li> 
     <li>char</li> 
     <li>byte</li> 
     <li>short</li> 
     <li>int</li> 
     <li>long</li> 
     <li>float</li> 
     <li>double</li> 
    </ul> 
    <p> There is a specific keyword for each of these types in Java. Primitive types are not objects in Java. Primitive data types cannot be stored in Java collections which work only with objects. They can be placed into arrays instead. </p> 
    <p> The reference types are: </p> 
    <ul> 
     <li>class types</li> 
     <li>interface types</li> 
     <li>array types</li> 
    </ul> 
    <p> There is also a special <code>null</code> type which represents a non-existing value. </p> 
    <p> In Ruby programming language, everything is an object. Even basic data types. </p> 
    <pre>
#!/usr/bin/ruby

4.times { puts "Ruby" }
</pre> 
    <p> This Ruby script prints four times "Ruby" string to the console. We call a times method on the 4 number. This number is an object in Ruby. </p> 
    <p> Java has a different approach. It has primitive data types and wrapper classes. Wrapper classes transform primitive types into objects. Wrapper classes are covered in the next chapter. </p> 
    <h2>Boolean values</h2> 
    <p> There is a duality built in our world. There is a Heaven and Earth, water and fire, jing and jang, man and woman, love and hatred. In Java the <code>boolean</code> data type is a primitive data type having one of two values: <code>true</code> or <code>false</code>. </p> 
    <p> Happy parents are waiting a child to be born. They have chosen a name for both possibilities. If it is going to be a boy, they have chosen Robert. If it is going to be a girl, they have chosen Victoria. </p> 
    <div class="codehead">
     BooleanType.java
    </div> 
    <pre class="code">
package com.zetcode;

import java.util.Random;

public class BooleanType {

    public static void main(String[] args) {
        
        String name = "";
        Random r = new Random();
        boolean male = r.nextBoolean();

        if (male == true) {
            
            name = "Robert";
        }

        if (male == false) {
            
            name = "Victoria";
        }

        System.out.format("We will use name %s%n", name);

        System.out.println(9 &gt; 8);
    }
}
</pre> 
    <p> The program uses a random number generator to simulate our case. </p> 
    <pre class="explanation">
Random r = new Random();
boolean male = r.nextBoolean();
</pre> 
    <p> The <code>Random</code> class is used to produce random numbers. The <code>nextBoolean()</code> method returns randomly a boolean value. </p> 
    <pre class="explanation">
if (male == true) {
    
    name = "Robert";
}
</pre> 
    <p> If the boolean variable male equals to true, we set the name variable to "Robert". The <code>if</code> keyword works with boolean values. </p> 
    <pre class="explanation">
if (male == false) {
    
    name = "Victoria";
}
</pre> 
    <p> If the random generator chooses false than we set the name variable to "Victoria". </p> 
    <pre class="explanation">
System.out.println(9 &gt; 8);
</pre> 
    <p> Relational operators result in a boolean value. This line prints true to the console. </p> 
    <pre>
$ java com.zetcode.BooleanType 
We will use name Robert
true
$ java com.zetcode.BooleanType 
We will use name Victoria
true
$ java com.zetcode.BooleanType 
We will use name Victoria
true
</pre>

    <h2>Integers</h2> 
    <p> Integers are a subset of the real numbers. They are written without a fraction or a decimal component. Integers fall within a set Z = {..., -2, -1, 0, 1, 2, ...} Integers are infinite. </p> 
    <p> In computer languages, integers are (usually) primitive data types. Computers can practically work only with a subset of integer values, because computers have finite capacity. Integers are used to count discrete entities. We can have 3, 4, or 6 humans, but we cannot have 3.33 humans. We can have 3.33 kilograms, 4.564 days, or 0.4532 kilometers. </p>

    <p> These integer types may be used according to our needs. We can then use the <code>byte</code> type for a variable that stores the number of children a woman gave birth to. The oldest verified person died at 122, therefore we would probably choose at least the <code>short</code> type for the age variable. This will save us some memory. </p> 
    <p> Integer literals may be expressed in decimal, hexadecimal, octal, or binary notations. If a number has an ASCII letter <code>L</code> or <code>l</code> suffix, it is of type <code>long</code>. Otherwise it is of type <code>int</code>. The capital letter L is preferred for specifying long numbers, since lowercase l can be easily confused with number 1. </p> 
    <pre>
int a = 34;        
byte b = 120;
short c = 32000;
long d = 45000;
long e = 320000L;
</pre> 
    <p> We have five assignments. Values 34, 120, 32000, and 45000 are integer literals of type <code>int</code>. There are no integer literals for <code>byte</code> and <code>short</code> types. If the values fit into the destination type, the compiler does not complain and performs a conversion automatically. For <code>long</code> numbers smaller than <code>Integer.MAX_VALUE</code>, the L suffix is optional. </p> 
    <pre>
long x = 2147483648L;
long y = 2147483649L;
</pre> 
    <p> For <code>long</code> numbers larger than <code>Integer.MAX_VALUE</code>, we must add the <code>L</code> suffix. </p> 
    <p> When we work with integers, we deal with discrete items. For instance, we can use integers to count apples. </p> 
    <div class="codehead">
     Apples.java
    </div> 
    <pre class="code">
package com.zetcode;

public class Apples {

    public static void main(String[] args) {

        int baskets = 16;
        int applesInBasket = 24;

        int total = baskets * applesInBasket;

        System.out.format("There are total of %d apples%n", total);
    }
}
</pre> 
    <p> In our program, we count the total amount of apples. We use the multiplication operation. </p> 
    <pre class="explanation">
int baskets = 16;
int applesInBasket = 24;
</pre> 
    <p> The number of baskets and the number of apples in each basket are integer values. </p> 
    <pre class="explanation">
int total = baskets * applesInBasket;
</pre> 
    <p> Multiplying those values we get an integer, too. </p> 
    <pre>
$ java com.zetcode.Apples 
There are total of 384 apples
</pre> 
    <p> This is the output of the program. </p> 
    <p> Integers can be specified in four different <em>notations</em> in Java: decimal, octal, hexadecimal, and binary. The binary notation was introduced in Java 7. Decimal numbers are used normally as we know them. Octal numbers are preceded with a <code>0</code> character and followed by octal numbers. Hexadecimal numbers are preceded with <code>0x</code> characters and followed by hexadecimal numbers. Binary numbers start with <code>0b</code> and are followed by binary numbers (zeroes and ones). </p> 
    <div class="codehead">
     IntegerNotations.java
    </div> 
    <pre class="code">
package com.zetcode;

public class IntegerNotations {

    public static void main(String[] args) {
        
        int n1 = 31;
        int n2 = 0x31;
        int n3 = 031;
        int n4 = 0b1001;
        
        System.out.println(n1);
        System.out.println(n2);
        System.out.println(n3);
        System.out.println(n4);        
    }
}
</pre> 
    <p> We have four integer variables. Each of the variables is assigned a value with a different integer notation. </p> 
    <pre class="explanation">
int n1 = 31;
int n2 = 0x31;
int n3 = 031;
int n4 = 0b1001;
</pre> 
    <p> The first is decimal, the second hexadecimal, the third octal, and the fourth binary. </p> 
    <pre>
$ java com.zetcode.IntegerNotations 
31
49
25
9
</pre> 
    <p> We see the output of the <code>com.zetcode.IntegerNotations</code> program. </p> 
    <p> Big numbers are difficult to read. If we have a number like 245342395423452, we find it difficult to read it quickly. Outside computers, big numbers are separated by spaces or commas. Since Java SE 1.7, it is possible to separate integers with an underscore. </p> 
    <p> The underscore cannot be used at the beginning or end of a number, adjacent to a decimal point in a floating point literal, and prior to an <code>F</code> or <code>L</code> suffix. </p> 
    <div class="codehead">
     UsingUnderscores.java
    </div> 
    <pre class="code">
package com.zetcode;

public class UsingUnderscores {
    
    public static void main(String[] args) {
        
        long a = 23482345629L;
        long b = 23_482_345_629L;

        System.out.println(a == b);
    }    
}
</pre> 
    <p> This code sample demonstrates the usage of underscores in Java. </p> 
    <pre class="explanation">
long a = 23482345629L;
long b = 23_482_345_629L;
</pre> 
    <p> We have two identical long numbers. In the second one we separate every three digits in a number. Comparing these two numbers we receive a boolean true. The <code>L</code> suffix tells the compiler that we have a long number literal. </p> 
    <p> Java <code>byte</code>, <code>short</code>, <code>int</code> and <code>long</code> types are used do represent <em>fixed precision</em> numbers. This means that they can represent a limited amount of integers. The largest integer number that a long type can represent is 9223372036854775807. If we deal with even larger numbers, we have to use the <code>java.math.BigInteger</code> class. It is used to represent immutable <em>arbitrary precision</em> integers. Arbitrary precision integers are only limited by the amount of computer memory available. </p> 
    <div class="codehead">
     VeryLargeIntegers.java
    </div> 
    <pre class="code">
package com.zetcode;

import java.math.BigInteger;

public class VeryLargeIntegers {

    public static void main(String[] args) {
        
        System.out.println(Long.MAX_VALUE);
        
        BigInteger b = new BigInteger("92233720368547758071");
        BigInteger c = new BigInteger("52498235605326345645");
               
        BigInteger a = b.multiply(c);
        
        System.out.println(a);            
    }
}
</pre> 
    <p> With the help of the <code>java.math.BigInteger</code> class, we multiply two very large numbers. </p> 
    <pre class="explanation">
System.out.println(Long.MAX_VALUE);
</pre> 
    <p> We print the largest integer value which can be represented by a <code>long</code> type. </p> 
    <pre class="explanation">
BigInteger b = new BigInteger("92233720368547758071");
BigInteger c = new BigInteger("52498235605326345645");
</pre> 
    <p> We define two <code>BigInteger</code> objects. They both hold larger values that a <code>long</code> type can hold. </p> 
    <pre class="explanation">
BigInteger a = b.multiply(c);
</pre> 
    <p> With the <code>multiply()</code> method, we multiply the two numbers. Note that the <code>BigInteger</code> numbers are immutable. The operation returns a new value which we assign to a new variable. </p> 
    <pre class="explanation">
System.out.println(a);
</pre> 
    <p> The computed integer is printed to the console. </p> 
    <pre>
$ java com.zetcode.VeryLargeIntegers 
9223372036854775807
4842107582663807707870321673775984450795
</pre> 
    <p> This is the example output. </p> 
    <h2>Arithmetic overflow</h2> 
    <p> An <em>arithmetic overflow</em> is a condition that occurs when a calculation produces a result that is greater in magnitude than that which a given register or storage location can store or represent. </p> 
    <div class="codehead">
     Overflow.java
    </div> 
    <pre class="code">
package com.zetcode;

public class Overflow {

    public static void main(String[] args) {
        
        byte a = 126;

        System.out.println(a);
        a++;

        System.out.println(a);
        a++;

        System.out.println(a);
        a++;

        System.out.println(a);
    }
}
</pre> 
    <p> In this example, we try to assign a value beyond the range of a data type. This leads to an arithmetic overflow. </p> 
    <pre>
$ java com.zetcode.Overflow 
126
127
-128
-127
</pre> 
    <p> When an overflow occurs, the variable is reset to negative upper range value. </p> 
    <h2>Floating point numbers</h2> 
    <p> Real numbers measure continuous quantities, like weight, height, or speed. Floating point numbers represent an approximation of real numbers in computing. In Java we have two primitive floating point types: <code>float</code> and <code>double</code>. The <code>float</code> is a single precision type which store numbers in 32 bits. The <code>double</code> is a double precision type which store numbers in 64 bits. These two types have fixed precision and cannot represent exactly all real numbers. In situations where we have to work with precise numbers, we can use the <code>BigDecimal</code> class. </p> 
    <p> Floating point numbers with an <code>F/f</code> suffix are of type <code>float</code>, <code>double</code> numbers have <code>D/d</code> suffix. The suffix for <code>double</code> numbers is optional. </p> 
    <p> Let's say a sprinter for 100m ran 9.87s. What is his speed in km/h? </p> 
    <div class="codehead">
     Sprinter.java
    </div> 
    <pre class="code">
package com.zetcode;

public class Sprinter {

    public static void main(String[] args) {
        
        float distance;
        float time;
        float speed;

        distance = 0.1f;
        
        time = 9.87f / 3600;

        speed = distance / time;

        System.out.format("The average speed of a sprinter is %f km/h%n", speed);
    }
}
</pre> 
    <p> In this example, it is necessary to use floating point values. The low precision of the float data type does not pose a problem in this case. </p> 
    <pre class="explanation">
distance = 0.1f;
</pre> 
    <p> 100m is 0.1km. </p> 
    <pre class="explanation">
time = 9.87f / 3600;
</pre> 
    <p> 9.87s is 9.87/60*60h. </p> 
    <pre class="explanation">
speed = distance / time;
</pre> 
    <p> To get the speed, we divide the distance by the time. </p> 
    <pre>
$ java com.zetcode.Sprinter 
The average speed of a sprinter is 36.474163 km/h
</pre> 
    <p> This is the output of the <code>com.zetcode.Sprinter</code> program. A small rounding error in the number does not affect our understanding of the sprinter's speed. </p> 
    <p> The <code>float</code> and <code>double</code> types are inexact. </p> 
    <div class="codehead">
     FloatingInPrecision.java
    </div> 
    <pre class="code">
package com.zetcode;

public class FloatingInPrecision {

    public static void main(String[] args) {        
                
        double a = 0.1 + 0.1 + 0.1;
        double b = 0.3;
        
        System.out.println(a);
        System.out.println(b);
        
        System.out.println(a == b);        
    }
}
</pre> 
    <p> The code example illustrates the inexact nature of the floating point values. </p> 
    <pre class="explanation">
double a = 0.1 + 0.1 + 0.1;
double b = 0.3;
</pre> 
    <p> We define two <code>double</code> values. The <code>D/d</code> suffix is optional. At first sight, they should be equal. </p> 
    <pre class="explanation">
System.out.println(a);
System.out.println(b);
</pre> 
    <p> Printing them will show a very small difference. </p> 
    <pre class="explanation">
System.out.println(a == b);  
</pre> 
    <p> This line will return false. </p> 
    <pre>
$ java com.zetcode.FloatingInPrecision 
0.30000000000000004
0.3
false
</pre> 
    <p> There is a small margin error. Therefore, the comparison operator returns a boolean false. </p> 
    <p> When we work with money, currency, and generally in business applications, we need to work with precise numbers. The rounding errors of the basic floating point types are not acceptable. </p> 
    <div class="codehead">
     CountingMoney.java
    </div> 
    <pre class="code">
package com.zetcode;

public class CountingMoney {

    public static void main(String[] args) {
        
        float c = 1.46f;
        float sum = 0f;
        
        for (int i=0; i&lt;100_000; i++) {
            
            sum += c;
        }
        
        System.out.println(sum);     
    }
}
</pre> 
    <p> The 1.46f represents 1 euro and 46 cents. We create a sum from 100000 such amounts. </p> 
    <pre class="explanation">
for (int i=0; i&lt;100_000; i++) {
    
    sum += c;
}
</pre> 
    <p> In this loop, we create a sum from 100000 such amounts of money. </p> 
    <pre>
$ java com.zetcode.CountingMoney 
146002.55
</pre> 
    <p> The calculation leads to an error of 2 euros and 55 cents. </p> 
    <p> To avoid this margin error, we utilize the <code>BigDecimal</code> class. It is used to hold immutable, arbitrary precision signed decimal numbers. </p> 
    <div class="codehead">
     CountingMoney2.java
    </div> 
    <pre class="code">
package com.zetcode;

import java.math.BigDecimal;

public class CountingMoney2 {

    public static void main(String[] args) {
    
        BigDecimal c = new BigDecimal("1.46");
        BigDecimal sum = new BigDecimal("0");
        
        for (int i=0; i&lt;100_000; i++) {
            
            sum = sum.add(c);
        }
        
        System.out.println(sum);                
    }
}
</pre> 
    <p> We do the same operation with the same amount of money. </p> 
    <pre class="explanation">
BigDecimal c = new BigDecimal("1.46");
BigDecimal sum = new BigDecimal("0");
</pre> 
    <p> We define two <code>BigDecimal</code> numbers. </p> 
    <pre class="explanation">
for (int i=0; i&lt;100_000; i++) {
    
    sum = sum.add(c);
}
</pre> 
    <p> The <code>BigDecimal</code> number is immutable, therefore a new object is always assigned to the sum variable in every loop. </p> 
    <pre>
$ java com.zetcode.CountingMoney2 
146000.00
</pre> 
    <p> In this example, we get the precise value. </p> 
    <p> Java supports the scientific syntax of the floating point values. Also known as exponential notation, it is a way of writing numbers too large or small to be conveniently written in standard decimal notation. </p> 
    <div class="codehead">
     ScientificNotation.java
    </div> 
    <pre class="code">

    <p> We use the <code>DecimalFormat</code> class to arrange our <code>double</code> value into standard decimal format. </p> 
    <pre class="explanation">

System.out.println(bd.toPlainString());

    <p> The <code>BigDecimal</code> class takes a floating point value in a scientific notation as a parameter. We use two methods of the class to print the value in the engineering and plain strings. </p>

    <h2>Enumerations</h2> 
    <p> An <dfn>enum</dfn> type is a special data type that enables for a variable to be a set of predefined constants. A variable that has been declared as having an enumerated type can be assigned any of the enumerators as a value. Enumerations make the code more readable. Enumerations are useful when we deal with variables that can only take one out of a small set of possible values. </p> 
    <div class="codehead">
     Enumerations.java
    </div> 
    <pre class="code">
package com.zetcode;

public class Enumerations {

    enum Days {

        MONDAY,
        TUESDAY,
        WEDNESDAY,
        THURSDAY,
        FRIDAY,
        SATURDAY,
        SUNDAY
    }

    public static void main(String[] args) {

        Days day = Days.MONDAY;

        if (day == Days.MONDAY) {
            
            System.out.println("It is Monday");
        }

        System.out.println(day);        

        for (Days d : Days.values()) {
            
            System.out.println(d);
        }
    }
}
</pre> 
    <p> In our code example, we create an enumeration for week days. </p> 
    <pre class="explanation">
enum Days {

    MONDAY,
    TUESDAY,
    WEDNESDAY,
    THURSDAY,
    FRIDAY,
    SATURDAY,
    SUNDAY
}
</pre> 
    <p> An enumeration representing the days of a week is created with a <code>enum</code> keyword. Items of an enumeration are constants. By convention, constants are written in uppercase letters. </p> 
    <pre class="explanation">
Days day = Days.MONDAY;
</pre> 
    <p> We have a variable called day which is of enumerated type Days. It is initialized to Monday. </p> 
    <pre class="explanation">
if (day == Days.MONDAY) {
    
    System.out.println("It is Monday");
}
</pre> 
    <p> This code is more readable than if comparing a day variable to some number. </p> 
    <pre class="explanation">
System.out.println(day);
</pre> 
    <p> This line prints Monday to the console. </p> 
    <pre class="explanation">
for (Days d : Days.values()) {
    
    System.out.println(d);
}
</pre> 
    <p> This loop prints all days to the console. The static <code>values()</code> method returns an array containing the constants of this <code>enum</code> type, in the order they are declared. This method may be used to iterate over the constants with the enhanced for statement. The enhanced <code>for</code> goes through the array, element by element, and prints them to the terminal. </p> 
    <pre>
It is Monday
MONDAY
MONDAY
TUESDAY
WEDNESDAY
THURSDAY
FRIDAY
SATURDAY
SUNDAY
</pre> 
    <p> This is the example output. </p> 
    <p> It is possible to give some values to the enumeration constants. </p>

     Enumerations2.java
    </div> 
    <pre class="code">
package com.zetcode;

enum Season {

    SPRING(10),
    SUMMER(20),
    AUTUMN(30),
    WINTER(40);

    private int value;

    private Season(int value) {
        this.value = value;
    }

    public int getValue() {

        return value;
    }
}

public class Enumerations2 {

    public static void main(String[] args) {

        for (Season season : Season.values()) {
            System.out.println(season + " " + season.getValue());
        }
    }
}

    <p> The example contains a <code>Season</code> enumeration which has four constants. </p> 
    <pre class="explanation">
SPRING(10),
SUMMER(20),
AUTUMN(30),
WINTER(40);
</pre> 
    <p> Here we define four constants of the <code>enum</code>. The constants are given specific values. </p> 
    <pre class="explanation">
private int value;

private Season(int value) {
    this.value = value;
}
</pre>

    <h2>Strings and chars</h2> 
    <p> A <code>String</code> is a data type representing textual data in computer programs. A string in Java is a sequence of characters. A <code>char</code> is a single character. Strings are enclosed by double quotes. </p> 
    <p> Since strings are very important in every programming language, we will dedicate a whole chapter to them. Here we only drop a small example. </p> 
    <div class="codehead">
     StringsChars.java
    </div> 
    <pre class="code">
package com.zetcode;

public class StringsChars {

    public static void main(String[] args) {

        String word = "ZetCode";

        char c = word.charAt(0);
        char d = word.charAt(3);

        System.out.println(c);
        System.out.println(d);
    }
}
</pre> 
    <p> The program prints Z character to the terminal. </p> 
    <pre class="explanation">
String word = "ZetCode";
</pre> 
    <p> Here we create a string variable and assign it "ZetCode" value. </p> 
    <pre class="explanation">
char c = word.charAt(0);
</pre> 
    <p> The <code>charAt()</code> method returns the <code>char</code> value at the specified index. The first char value of the sequence is at index 0, the next at index 1, and so on. </p> 
    <pre>
$ java com.zetcode.StringsChars 
Z
C
</pre> 
    <p> The program prints the first and the fourth character of the "ZetCode" string to the console. </p> 
    <h2>Arrays</h2> 
    <p> Array is a complex data type which handles a collection of elements. Each of the elements can be accessed by an index. All the elements of an array must be of the same data type. </p> 
    <p> We dedicate a whole chapter to arrays; here we show only a small example. </p> 
    <div class="codehead">
     ArraysEx.java
    </div> 
    <pre class="code">
package com.zetcode;

public class ArraysEx {

    public static void main(String[] args) {

        int[] numbers = new int[5];

        numbers[0] = 3;
        numbers[1] = 2;
        numbers[2] = 1;
        numbers[3] = 5;
        numbers[4] = 6;

        int len = numbers.length;

        for (int i = 0; i &lt; len; i++) {
            
            System.out.println(numbers[i]);
        }
    }
}
</pre> 
    <p> In this example, we declare an array, fill it with data and then print the contents of the array to the console. </p> 
    <pre class="explanation">
int[] numbers = new int[5];
</pre> 
    <p> We create an integer array which can store up to 5 integers. So we have an array of five elements, with indexes 0..4. </p> 
    <pre class="explanation">
numbers[0] = 3;
numbers[1] = 2;
numbers[2] = 1;
numbers[3] = 5;
numbers[4] = 6;
</pre> 
    <p> Here we assign values to the created array. We can access the elements of an array by the array access notation. It consists of the array name followed by square brackets. Inside the brackets we specify the index to the element that we want. </p> 
    <pre class="explanation">
int len = numbers.length;
</pre> 
    <p> Each array has a <code>length</code> property which returns the number of elements in the array. </p> 
    <pre class="explanation">
for (int i = 0; i &lt; len; i++) {
    
    System.out.println(numbers[i]);
}
</pre> 
    <p> We traverse the array and print the data to the console. </p> 
    <pre>
$ java com.zetcode.ArraysEx
3
2
1
5
6
</pre> 
    <p> This is the output of the <code>com.zetcode.ArraysEx</code> program. </p>
