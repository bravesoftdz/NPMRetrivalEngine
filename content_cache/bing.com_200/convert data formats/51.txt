           <p>Data Pipeline makes it easy to read, transform, and write XML and Excel files. This post demonstrates how to load data from an on-disk XML file, apply transformations on-the-fly, and save the result to an Excel file.</p> 
           <p><span id="more-831"></span></p> 
           <p>I recommend you to take a look at the <a href="https://northconcepts.com/docs/examples/read-a-json-stream/">How to Read a JSON Stream</a> post if you need to read data directly from a URL and skip the disk altogether. If you prefer to write your data to multiple Excel sheets, you can to read the post <a href="https://northconcepts.com/blog/2015/05/30/how-to-create-multiple-sheets-in-a-single-excel-file/#more-817">How to create multiple sheets in a single Excel file</a>.</p>

           <h2>Input XML file</h2> 
           <p>In this example we will process stock quotes available at <a href="http://query.yahooapis.com/v1/public/yql?q=select%20*%20from%20yahoo.finance.quotes%20where%20symbol%20in%20%28%22YHOO%22%2C%22AAPL%22%2C%22GOOG%22%2C%22MSFT%22%29%0A%09%09&amp;diagnostics=false&amp;env=http%3A%2F%2Fdatatables.org%2Falltables.env">Yahoo Finance</a>. We saved this XML data on disk in a file named <em>yahoo_stock_quote_feed.xml</em>.</p>

package com.northconcepts.datapipeline.examples.cookbook.blog;

import java.io.File;

import com.northconcepts.datapipeline.core.DataReader;
import com.northconcepts.datapipeline.core.DataWriter;
import com.northconcepts.datapipeline.core.FieldType;
import com.northconcepts.datapipeline.excel.ExcelDocument;
import com.northconcepts.datapipeline.excel.ExcelWriter;
import com.northconcepts.datapipeline.job.JobTemplate;
import com.northconcepts.datapipeline.transform.BasicFieldTransformer;
import com.northconcepts.datapipeline.transform.MoveFieldBefore;
import com.northconcepts.datapipeline.transform.RemoveFields;
import com.northconcepts.datapipeline.transform.SetCalculatedField;
import com.northconcepts.datapipeline.transform.TransformingReader;
import com.northconcepts.datapipeline.xml.XmlReader;

public class ConvertAnXmlFileToAnExcelFile {

	
	
	public static void main(String[] args) 
	{
		/*
		 * 1) Define a reader from an XML file
		 */
		DataReader xmlReader = new XmlReader(new File("example/data/input/yahoo_stock_quote_feed.xml"))
					.addField("company", "//quote/Name")
					.addField("ticker", "//quote/@symbol")
					.addField("EBITDA", "//quote/EBITDA")
					.addField("last trade date", "//quote/LastTradeDate")
					.addField("currency", "//quote/Currency")
					.addField("ask", "//quote/Ask")
					.addField("bid", "//quote/Bid")
					.addField("day range", "//quote/DaysRange")
					.addRecordBreak("//quote");

		/*
		 * 2) Transform the input data on-the-fly (optional)
		 */
		DataReader transformingReader = new TransformingReader(xmlReader)
					.add(

						new BasicFieldTransformer("EBITDA")
							.replaceAll("M", "E6")
							.replaceAll("B", "E9")
							.stringToDouble()
							.numberToInt(),

						new BasicFieldTransformer("last trade date")
							.stringToDate("M/d/y"),
						
						new BasicFieldTransformer("bid")
							.stringToDouble(),
						
						new BasicFieldTransformer("ask")
							.stringToDouble(),
						
						new SetCalculatedField("bid-ask spread", "bid-ask"),
						
						new RemoveFields("ask"),
						
						new MoveFieldBefore("bid-ask spread", "day range"),
						
						new BasicFieldTransformer("day range")
							.replaceAll(" - ", ", ")
							.prepend("[")
							.append("]")
		);

		/*
		 * 3) Define a writer to an Excel file
		 */
		ExcelDocument excelDocument = new ExcelDocument(); //default is .xlsx format (Excel 2007+)
		DataWriter excelWriter = new ExcelWriter(excelDocument)
						.setSheetName("stocks")
						.setAutofitColumns(true)
						.setStyleFormat(FieldType.INT, "0.00E+00")
						.setStyleFormat(FieldType.DOUBLE, "0.00")
						.setStyleFormat(FieldType.DATE, "yyyy-mmm-dd, ddd");
		

		/*
		 * 4) Transfer the data from the XML file to the Excel file and save the result
		 */
		JobTemplate.DEFAULT.transfer(transformingReader, excelWriter);
		excelDocument.save(new File("example/data/output/stock_quote_summary.xlsx"));
	}

           <h3>1. Define a reader from an XML file</h3> 
           <p><a href="https://northconcepts.com/javadocs/com/northconcepts/datapipeline/xml/XmlReader.html">XmlReader</a> lets you obtain records from any XML stream.&nbsp; The XML stream is broken into separate records by calling <code>addRecordBreak(String locationPath)</code> on your XmlReader instance. &nbsp; Fields are then added to each record using the <code>addField(String name, String locationPath)</code> method. The <em>locationPath</em> expressions used in the above methods are a subset of the <a href="https://northconcepts.com/javadocs/com/northconcepts/datapipeline/xml/XmlReader.html">XPath 1.0 location paths notation</a>. Below we give an example of location paths for selecting a node and an attribute:</p> 
           <ul> 
            <li><code>"//quote/Company"</code>  selects all Company nodes that are children of a quote node no matter where they are in the document;</li> 
            <li><code>"//quote/@symbol"</code>  selects the symbol attribute in all quote nodes no matter where they are in a document.</li> 
           </ul> 
           <h3>2. Transform the input data on-the-fly (optional)</h3> 
           <p>When converting data between formats, we often need to perform transformations. For example, we may read numbers as text from an XML file, but prefer to save them as a number in an Excel file. Or, we may need to adjust date and number formatting, and calculate aggregated values.</p> 
           <p><a href="https://northconcepts.com/javadocs/com/northconcepts/datapipeline/transform/TransformingReader.html">TransformingReader</a> is a proxy that applies transformations to records being passed through it. On-the-fly record transformations are added to a TransformingReader using the method <code>add(Transformer... transformers)</code>. Data Pipeline makes it easy to:</p> 
           <ul> 
            <li>transform field values: <a href="https://northconcepts.com/javadocs/com/northconcepts/datapipeline/transform/BasicFieldTransformer.html">BasicFieldTransformer</a>;</li> 
            <li>add calculated and constant value fields: <a href="https://northconcepts.com/javadocs/com/northconcepts/datapipeline/transform/SetCalculatedField.html">SetCalculatedField</a>, <a href="https://northconcepts.com/javadocs/com/northconcepts/datapipeline/transform/SetField.html">SetField</a>;</li> 
            <li>modify the field list: <a href="https://northconcepts.com/javadocs/com/northconcepts/datapipeline/transform/SelectFields.html">SelectFields</a>, <a href="https://northconcepts.com/javadocs/com/northconcepts/datapipeline/transform/MoveFieldAfter.html">MoveFieldAfter</a>, <a href="https://northconcepts.com/javadocs/com/northconcepts/datapipeline/transform/MoveFieldBefore.html">MoveFieldBefore</a>, <a href="https://northconcepts.com/javadocs/com/northconcepts/datapipeline/transform/RemoveFields.html">RemoveFields</a>, <a href="https://northconcepts.com/javadocs/com/northconcepts/datapipeline/transform/CopyField.html">CopyField</a>, <a href="https://northconcepts.com/javadocs/com/northconcepts/datapipeline/transform/RenameField.html">RenameField</a>;</li> 
            <li>split array fields and lookup data: <a href="https://northconcepts.com/javadocs/com/northconcepts/datapipeline/transform/SplitArrayField.html">SplitArrayField</a>, <a href="https://northconcepts.com/javadocs/com/northconcepts/datapipeline/transform/lookup/LookupTransformer.html">LookupTransformer</a>.</li> 
           </ul> 
           <p>The following posts contain additional on-the-fly data-transformation examples: <a href="https://northconcepts.com/docs/examples/lookup-data-in-any-data-reader/">Lookup Data in any Data Reader</a>, <a href="https://northconcepts.com/docs/examples/search-twitter-for-tweets/">Search Twitter for Tweets</a>, <a href="https://northconcepts.com/docs/examples/rename-a-field/">Rename a Field</a> and <a href="https://northconcepts.com/docs/examples/set-a-field/">Set a Field</a>.</p> 
           <h3>3. Define a writer to a Microsoft Excel file</h3> 
           <p><a href="https://northconcepts.com/javadocs/com/northconcepts/datapipeline/excel/ExcelWriter.html">ExcelWriter</a> lets you write records to a Microsoft Excel document. It is used in combination with <a href="https://northconcepts.com/javadocs/index.html?com/northconcepts/datapipeline/excel/ExcelDocument.html">ExcelDocument</a>, a class that encapsulates an Excel workbook in memory. Data Pipeline supports both the XLS and XSLX file format. By default, the ExcelDocument objects represent an XLSX file. You ca use ExcelWriters <code>setStyleFormat(FieldType fieldType, String pattern)</code> method to set the desired cell format per data type.</p> 
           <h3>4. Transfer the data from the XML file to the Excel file and save the result</h3> 
           <p><a href="https://northconcepts.com/javadocs/index.html?com/northconcepts/datapipeline/job/JobTemplate.html">JobTemplate</a> is a base type for classes that transfer records from a DataReader to a DataWriter. Data Pipeline provides a default JobTemplate implementation accessible at <code>JobTemplate.DEFAULT</code>. A call to the method <code>JobTemplate.DEFAULT.transfer(DataReader reader, DataWriter writer)</code> will do the data transfer.</p> 
           <p>Finally, a call to the ExcelDocuments <code>save(File excelFile)</code> method is needed to write the generated workbook data to an Excel file on disk.</p> 
           <h2>Output file</h2> 
           <p><img src="https://northconcepts.com/blog/wp-content/uploads/2015/06/how-to-convert-an-xml-file-to-an-excel-file-output.png" alt="Excel output"></p> 
           <p>All examples can be found in the <strong>examples/src</strong> folder of the <a href="https://northconcepts.com/downloads/">download</a>.</p>
