        <article data-history-node-id="150" role="article" about="/software/bio-formats" class="node node--type-software node--view-mode-full"> 
         <div class="node__content"> 
          <div class="clearfix text-formatted field field--name-field-software-summary field--type-text-long field--label-hidden field__item">
           <blockquote style="padding-bottom: 0;"> 
            <p style="text-align: center;">This page describes LOCI's role in developing Bio-Formats. For the main web site, see:<br> <span style="font-size: 14px;"><a href="http://openmicroscopy.org/info/bio-formats">http://openmicroscopy.org/info/bio-formats</a></span></p> 
           </blockquote> 
           <p>Bio-Formats is a standalone Java library for reading and writing life sciences image file formats. It is capable of parsing both pixels and metadata for a large number of formats, as well as writing to several formats.</p> 
           <h2>Purpose</h2> 
           <p>Bio-Formats's primary purpose is to convert proprietary microscopy data into an open standard called the <a href="http://www.openmicroscopy.org/site/support/ome-model/">OME data model</a>, particularly into the <a href="http://www.openmicroscopy.org/site/products/ome-tiff">OME-TIFF</a> file format. See our article on <a href="/software/oss">open source software in science</a> for a thorough explanation and rationale.</p> 
           <h2>Authors</h2> 
           <p>Bio-Formats is a collaboration between several partners in the <a href="http://openmicroscopy.org/">Open Microscopy Environment</a> consortium, including LOCI, the <a href="http://www.dundee.ac.uk/">University of Dundee</a> and <a href="http://www.glencoesoftware.com/">Glencoe Software</a>. The library is written by <a href="/people/melissa-linkert">Melissa Linkert</a>, <a href="/people/curtis-rueden">Curtis Rueden</a>, Chris Allan, Roger Leigh and Sébastien Besson, with past contributions from Eric Kjellman, Brian Loranger and Andrew Patterson, as well as <a href="http://www.openmicroscopy.org/site/about/ome-contributors">many contributions from the community</a>.</p> 
           <h2>History</h2> 
           <p>Bio-Formats began to coalesce in 2002, when Curtis Rueden started developing an N-dimensional tool for biological visualization and analysis called <a href="/software/visbio">VisBio</a>. We found ourselves implementing an increasing number of file formats within the <a href="http://visad.ssec.wisc.edu/">VisAD</a> framework used by VisBio.</p> 
           <p>By 2005 we knew we needed a dedicated developer to continue pursuing additional file format support, and Melissa Linkert joined the team as an undergraduate programmer. Melissa made rapid progress, and we soon realized that this code was of value to the worldwide community beyond the context of VisAD. On January 10, 2006, we formally <a href="https://github.com/openmicroscopy/bioformats/commit/9c0c8a9deed81b912a5dd98af0831ed0e7536967">branded the project as Bio-Formats</a>, and on March 31 the project had its <a href="https://list.nih.gov/cgi-bin/wa.exe?A2=IMAGEJ;f032e255.0603">first release</a>, including a plugin for <a href="http://imagej.net/">ImageJ</a>.</p> 
           <p>In late 2006, as part of our collaboration with the <a href="http://www.openmicroscopy.org/">Open Microscopy Environment</a>, we <a href="http://lists.openmicroscopy.org.uk/pipermail/ome-users/2006-December/000515.html">integrated Bio-Formats into the original OME server</a>, and in early 2007, the <a href="http://lists.openmicroscopy.org.uk/pipermail/ome-users/2007-February/000521.html">OMERO project also adopted Bio-Formats</a> to drive its file format I/O. In May 2008, Melissa moved to work for Glencoe Software, continuing as the project's lead developer.</p> 
           <p>In 2012, LOCI launched the <a href="/software/scifio">SCIFIO</a> project to generalize and extend Bio-Formats into N dimensions and beyond the OME-XML data model. LOCI is developing SCIFIO in collaboration with the <a href="http://scijava.org/">SciJava software consortium</a>, in which <a href="/software/fiji">Fiji</a>&nbsp;and OME are key partners.</p> 
           <p>See the <a href="/about-loci/loci-and-ome">LOCI and OME</a> page for more information on our history and relationship with the Open Microscopy Environment consortium.</p> 
           <h2>License</h2> 
           <p>Bio-Formats is open source software. The project consists of two major components:</p> 
           <ul> 
            <li>The <a href="https://github.com/openmicroscopy/bioformats/tree/v5.0.0/components/formats-bsd">core library</a>, released under a permissive <a href="https://github.com/openmicroscopy/bioformats/blob/v5.0.0/components/formats-bsd/LICENSE.txt">BSD-2</a> license. The core components provide the architecture of the library; i.e., all the format-independent parts of the code. It also includes reader and writer implementations for open file formats. The more permissive BSD license enables non-GPL third party software, including closed source commercial software, to read and write open formats such as OME-TIFF. For a complete list of which file formats are included as part of the core, see the "BSD" column of the <a href="http://www.openmicroscopy.org/site/support/bio-formats/supported-formats.html">supported formats table</a>.</li> 
            <li><a href="https://github.com/openmicroscopy/bioformats/tree/v5.0.0/components/formats-gpl">Proprietary file format readers</a>,&nbsp;released under a "copyleft" <a href="https://github.com/openmicroscopy/bioformats/blob/v5.0.0/components/formats-gpl/LICENSE.txt">GPLv2+</a> license. Bio-Formats also provides proprietary file format (PFF) reader implementations for a large number of microscopy formats. Developers of non-GPL software wishing to leverage Bio-Formats may purchase a commercial license (see below).</li> 
           </ul> 
           <p><strong>What is the GPL?</strong> The <a href="http://www.gnu.org/licenses/gpl.html">GNU General Public License</a> (GPL) is a "copyleft" license which requires that any software package linking to Bio-Formats, either directly or indirectly, cannot be distributed unless its source code is also made available under the terms of the GPL. In addition, it forbids distribution if your software package depends on any non-free software other than common operating system components.</p> 
           <p>For more information about the GPL and related licenses, see the <a href="http://www.oss-watch.ac.uk/resources/gpl.xml">GPL Overview on OSS Watch</a> or the <a href="http://en.wikipedia.org/wiki/GNU_General_Public_License">Wikipedia entry on the GPL</a>.</p> 
           <p><strong>Commercial licensing.</strong> If you prefer, you can obtain a commercial Bio-Formats license and/or support package from our commercial partner, <a href="http://www.glencoesoftware.com/">Glencoe Software</a>. Several different licensing arrangements are possible. Glencoe is committed to reaching a licensing solution that meets your needs; please contact them at <a href="mailto:bioformats@glencoesoftware.com">bioformats@glencoesoftware.com</a> to discuss your requirements.</p> 
           <h2 id="questions">Questions</h2> 
           <ul> 
            <li><b>Reporting a bug:</b> To report a bug in Bio-Formats, please see the <a href="http://www.openmicroscopy.org/site/support/bio-formats/about/bug-reporting.html">Bug reporting</a> page.</li> 
            <li><strong>SCIFIO:</strong> For questions about the <a href="/software/scifio">SCIFIO</a> library, contact the SCIFIO team via the <a href="http://forum.imagej.net/tags/scifio">ImageJ Forum</a>, using the <code>scifio</code> tag.</li> 
            <li><b>Commercial inquiries:</b> If you are interested in a commercial license from <a href="http://www.glencoesoftware.com/">Glencoe Software</a>, please see the <a href="/bio-formats/license">License</a> page.</li> 
            <li>For any other feedback, you can contact the Bio-Formats team via the <a href="http://lists.openmicroscopy.org.uk/mailman/listinfo/ome-users/">OME Users mailing list</a>.</li>
