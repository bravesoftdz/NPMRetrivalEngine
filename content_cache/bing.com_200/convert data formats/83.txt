    <h2 id="xml_overview"><a class="link" href="#xml_overview">1. XML Introduction</a></h2> 
    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="xml_overview_s2a"><a class="link" href="#xml_overview_s2a">1.1. XML Overview</a></h3> 
      <div class="paragraph"> 
       <p>XML is the abbreviation for <em>Extensible Markup Language</em> and is an established data exchange format. XML was defined 1998 by the World Wide Web Consortium (W3C).</p> 
      </div> 
      <div class="paragraph"> 
       <p>An XML document consists of elements, each element has a start tag, content and an end tag. An XML document must have exactly one root element (i.e., one tag which encloses the remaining tags). XML differentiates between capital and non-capital letters.</p> 
      </div> 
      <div class="paragraph"> 
       <p>An XML file must be <em>well-formed</em>. This means that it must apply to the following conditions:</p> 
      </div> 
      <div class="ulist"> 
       <ul> 
        <li> <p>An XML document always starts with a prolog (see below for an explanation of what a prolog is)</p> </li> 
        <li> <p>Every opening tag has a closing tag.</p> </li> 
        <li> <p>All tags are completely nested.</p> </li> 
       </ul> 
      </div> 
      <div class="paragraph"> 
       <p>An <em>valid</em> XML file is well-formed and must contain a link to an XML schema and be valid according to that schema.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The following is a valid, well-formed XML file.</p> 
      </div> 
      <div class="listingblock"> 
       <div class="content"> 
        <pre class="CodeRay highlight"><code data-lang="xml"><span class="preprocessor">&lt;?xml version="1.0"?&gt;</span>
<span class="comment">&lt;!-- This is a comment --&gt;</span>
<span class="tag">&lt;address&gt;</span>
    <span class="tag">&lt;name&gt;</span>Lars <span class="tag">&lt;/name&gt;</span>
    <span class="tag">&lt;street&gt;</span> Test <span class="tag">&lt;/street&gt;</span>
    <span class="tag">&lt;telephone</span> <span class="attribute-name">number</span>= <span class="string"><span class="delimiter">"</span><span class="content">0123</span><span class="delimiter">"</span></span><span class="tag">/&gt;</span>
<span class="tag">&lt;/address&gt;</span></code></pre> 
       </div> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="xml_comparisionwithotherformats"><a class="link" href="#xml_comparisionwithotherformats">1.2. Comparison of XML to other formats</a></h3> 
      <div class="paragraph"> 
       <p>It is relatively easy to process an XML document, compared with a binary or unstructured format. This is because of the following characteristics:</p> 
      </div> 
      <div class="ulist"> 
       <ul> 
        <li> <p>XML is plain text</p> </li> 
        <li> <p>XML represents data without defining how the data should be displayed</p> </li> 
        <li> <p>XML can be transformed into other formats via XSL</p> </li> 
        <li> <p>XML can be easily processed via standard parsers</p> </li> 
        <li> <p>XML files are hierarchical</p> </li> 
       </ul> 
      </div> 
      <div class="paragraph"> 
       <p>The XML format is relatively verbose, i.e., if data is represented as XML the size of this data is relatively large compared to other formats. In the Internet JSON or binary formats are frequently used to replace XML if the data throughput is important.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="xml_overview_s2c"><a class="link" href="#xml_overview_s2c">1.3. XML Elements</a></h3> 
      <div class="paragraph"> 
       <p>An XML document always starts with a <code>prolog</code> which describes the XML file. This prolog can be minimal, e.g. <code>&lt;?xml version="1.0"?&gt;</code>. It can also contain other information, e.g. the encoding <code>&lt;?xml version="1.0" encoding="UTF-8" standalone="yes" ?&gt;</code>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>A tag which does not enclose any content is know as an "empty tag", for example <code>&lt;flag/&gt;</code>.</p> 
      </div> 
      <div class="paragraph"> 
       <p>Comments in XML are defined as: &lt;! COMMENT&gt;.</p> 
      </div> 
     </div> 
    </div> 
   </div> 
   <div class="sect1"> 
    <h2 id="javaxml"><a class="link" href="#javaxml">2. Java XML overview</a></h2> 
    <div class="sectionbody"> 
     <div class="paragraph"> 
      <p>The Java programming language contains several methods for processing and writing XML.</p> 
     </div> 
     <div class="paragraph"> 
      <p>Older Java versions supported only the DOM API (Document Object Model) and the SAX (Simple API for XML) API.</p> 
     </div> 
     <div class="paragraph"> 
      <p>In DOM you access the XML document over an object tree. DOM can be used to read and write XML files.</p> 
     </div> 
     <div class="paragraph"> 
      <p>SAX (Simple API for XML) is a Java API for sequential reading of XML files. SAX can only read XML documents. SAX provides an event driven XML Processing following the Push-Parsing model. In this model you register listeners in the form of Handlers to the Parser. These are notified through call-back methods.</p> 
     </div> 
     <div class="paragraph"> 
      <p>Both DOM and Sax are older APIs and I recommend not using them anymore.</p> 
     </div> 
     <div class="paragraph"> 
      <p>Stax (Streaming API for XML) is an API for reading and writing XML Documents. It was introduced in Java 6.0 and is considered superior to SAX and DOM.</p> 
     </div> 
     <div class="paragraph"> 
      <p><a href="http://www.vogella.com/tutorials/JAXB/article.html">Java Architecture for XML Binding (JAXB)</a> is a Java standard that allows to convert Java objects to XML and vice versa. JAXB defines a programmer API for reading and writing Java objects to from XML documents. It also defines a service provider which allows the selection of the JAXB implementation. JAXB applies a lot of defaults thus making reading and writing of XML via Java very easy.</p> 
     </div> 
     <div class="paragraph"> 
      <p>The following explains the Stax interface; for an introduction to JAXB, please see <a href="http://www.vogella.com/tutorials/JAXB/article.html">JAXB tutorial</a>.</p> 
     </div>

    <h2 id="javastax"><a class="link" href="#javastax">3. Streaming API for XML (StaX)</a></h2> 
    <div class="sectionbody"> 
     <div class="sect2"> 
      <h3 id="javastax_overview"><a class="link" href="#javastax_overview">3.1. Overview</a></h3> 
      <div class="paragraph"> 
       <p>Streaming API for XML, called StaX, is an API for reading and writing XML Documents.</p> 
      </div> 
      <div class="paragraph"> 
       <p>StaX is a Pull-Parsing model. Application can take the control over parsing the XML documents by pulling (taking) the events from the parser.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The core StaX API falls into two categories and they are listed below. They are</p> 
      </div> 
      <div class="ulist"> 
       <ul> 
        <li> <p>Cursor API</p> </li> 
        <li> <p>Event Iterator API</p> </li> 
       </ul> 
      </div> 
      <div class="paragraph"> 
       <p>Applications can use any of these two API for parsing XML documents. The following will focus on the event iterator API as I consider it more convenient to use.</p> 
      </div> 
     </div> 
     <div class="sect2"> 
      <h3 id="javastax_api"><a class="link" href="#javastax_api">3.2. Event Iterator API</a></h3> 
      <div class="paragraph"> 
       <p>The event iterator API has two main interfaces: XMLEventReader for parsing XML and XMLEventWriter for generating XML.</p> 
      </div> 
     </div> 
     <div class="sect2">

      </div>

      </div>

    <span class="directive">public</span> <span class="predefined-type">List</span>&lt;Item&gt; readConfig(<span class="predefined-type">String</span> configFile) {
        <span class="predefined-type">List</span>&lt;Item&gt; items = <span class="keyword">new</span> <span class="predefined-type">ArrayList</span>&lt;Item&gt;();
        <span class="keyword">try</span> {
            <span class="comment">// First, create a new XMLInputFactory</span>
            XMLInputFactory inputFactory = XMLInputFactory.newInstance();
            <span class="comment">// Setup a new eventReader</span>
            <span class="predefined-type">InputStream</span> in = <span class="keyword">new</span> <span class="predefined-type">FileInputStream</span>(configFile);
            XMLEventReader eventReader = inputFactory.createXMLEventReader(in);

            Item item = <span class="predefined-constant">null</span>;

                    <span class="comment">// If we have an item element, we create a new item</span>
                    <span class="keyword">if</span> (startElement.getName().getLocalPart().equals(ITEM)) {

                            <span class="predefined-type">Attribute</span> attribute = attributes.next();
                            <span class="keyword">if</span> (attribute.getName().toString().equals(DATE)) {
                                item.setDate(attribute.getValue());
                            }

                        }
                    }

                    <span class="keyword">if</span> (event.isStartElement()) {
                        <span class="keyword">if</span> (event.asStartElement().getName().getLocalPart()
                                .equals(MODE)) {
                            event = eventReader.nextEvent();
                            item.setMode(event.asCharacters().getData());
                            <span class="keyword">continue</span>;
                        }
                    }
                    <span class="keyword">if</span> (event.asStartElement().getName().getLocalPart()
                            .equals(UNIT)) {
                        event = eventReader.nextEvent();
                        item.setUnit(event.asCharacters().getData());
                        <span class="keyword">continue</span>;
                    }

                    <span class="keyword">if</span> (event.asStartElement().getName().getLocalPart()
                            .equals(CURRENT)) {
                        event = eventReader.nextEvent();
                        item.setCurrent(event.asCharacters().getData());
                        <span class="keyword">continue</span>;
                    }

                    <span class="keyword">if</span> (event.asStartElement().getName().getLocalPart()
                            .equals(INTERACTIVE)) {
                        event = eventReader.nextEvent();
                        item.setInteractive(event.asCharacters().getData());
                        <span class="keyword">continue</span>;
                    }
                }
                <span class="comment">// If we reach the end of an item element, we add it to the list</span>
                <span class="keyword">if</span> (event.isEndElement()) {
                    EndElement endElement = event.asEndElement();
                    <span class="keyword">if</span> (endElement.getName().getLocalPart().equals(ITEM)) {
                        items.add(item);
                    }
                }

            }
        } <span class="keyword">catch</span> (<span class="exception">FileNotFoundException</span> e) {
            e.printStackTrace();
        } <span class="keyword">catch</span> (XMLStreamException e) {

        <span class="keyword">return</span> items;
    }

      <h3 id="javaxpath_a"><a class="link" href="#javaxpath_a">4.1. Overview</a></h3> 
      <div class="paragraph"> 
       <p>XPath (XML Path Language) is a language for selecting / searching nodes from an XML document. Java 5 introduced the javax.xml.xpath package which provides a XPath library.</p> 
      </div> 
      <div class="paragraph"> 
       <p>The following explains how to use XPath to query an XML document via Java.</p>
