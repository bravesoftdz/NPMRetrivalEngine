           <p>JSON format is very convenient when dealing with web applications. When you are in more traditional grounds, though, it might not be the most recommended. In this tutorial I’ll show how you can easily convert data pulled out from your MongoDB into a CSV file in Meteor.</p> 
           <p>When I came across this challenge, I&nbsp;took&nbsp;several approaches. MeteorChef has a <a href="http://themeteorchef.com/recipes/exporting-data-from-your-meteor-application/" target="_blank">pretty complete tutorial</a> explaining how you can export your data into different formats. <a href="https://github.com/netanelgilad/meteor-excel/" target="_blank">@netanelgilad wrote a wrapper to export your data directly to xls/xlsx formats</a>. But in the end of the day I was looking for something more straight and <a href="http://papaparse.com/" target="_blank">Papa Parse</a> seemed to do the trick.</p> 
           <p>Parsing (or unparsing, as they called) the data did work but in an inflexible way. For example, when dealing with embed documents you would end up with this:</p> 
           <p><a href="http://rafael.daniels1006.webfactional.com/wp-content/uploads/2015/06/Selection_001.png"><img class=" size-medium wp-image-88 aligncenter" src="http://rafael.daniels1006.webfactional.com/wp-content/uploads/2015/06/Selection_001-300x129.png" alt="Unparse Example" srcset="http://rafaelquintanilha.com/wp-content/uploads/2015/06/Selection_001-300x129.png 300w, http://rafaelquintanilha.com/wp-content/uploads/2015/06/Selection_001.png 430w" sizes="(max-width: 300px) 100vw, 300px"></a></p> 
           <p>So it doesn’t work out-of-the-box, but let’s not blame the poor guys from Papa Parse! There’s a pretty simple workaround for it.&nbsp;In this tutorial we will build a contacts application (I feel like I need to be more creative) that has some data and options&nbsp;to export either all your contacts or a single&nbsp;one in the convenient CSV format.&nbsp;To make things a little tougher we will deal with embed documents and data manipulation.</p> 
           <p><a href="https://github.com/rafaelquintanilha/meteor-json-to-csv" target="_blank">There is a repository for this project in GitHub</a>&nbsp;and an <a href="http://json-to-csv.meteor.com/" target="_blank">online demo</a>&nbsp;if you want to see&nbsp;what we will build.&nbsp;Experiment to git clone&nbsp;and play with the data (or change it directly in the browser’s console) alongside with this tutorial.</p> 
           <h4>1. Creating the Project</h4> 
           <p>To start, go to your terminal and run the following:</p> 
           <div class="terminal-box">
            $ meteor create papa-unparse
            <br> $ cd papa-unparse
            <br> $ meteor add harrison:papa-parse
            <br> $ meteor add momentjs:moment
            <br> $ meteor add iron:router
           </div> 
           <p>You have now successfully created your Meteor project and added the Papa Parse, MomentJS (we will need it to format our date) and Iron Router packages.&nbsp;That should be enough for our purposes today.</p> 
           <p>Remove the files automatically generated by Meteor and create three folders: <strong>client</strong>, <strong>both</strong> and <strong>server</strong>.</p>

           <p> That’s all. Experiment to run your application and see everything is alright. You should see a list with all names in a table and a hyperlink that shows more details about that contact.&nbsp;Sounds like it’s time for doing some exportation.</p> 
           <h4>4. Export!</h4> 
           <p>Papa Parse is client-side so you will need all your JSON data in there. That’s no big deal – you would download it anyway. Nevertheless,&nbsp;you might want data that it’s not in the client. For example, our contacts template only show the name of the contact. Imagine the scenario when we are using subscriptions and that’s the only data available – you would have to request the remaining from the server.</p> 
           <p>With that in mind, our logic becomes: client&nbsp;clicks in export button -&gt; server gathers data and builds JSON -&gt; server handles JSON array to client -&gt; client converts to CSV and downloads file.</p> 
           <p>All good?&nbsp;Now note that apart from the server gathering data, the rest of the process is the same for any export you might do. To avoid&nbsp;code duplication, let’s create a global helper which handles that for you. There’s many ways of achieving and it’s a matter of flavor. In this tutorial, create&nbsp;a file&nbsp;<strong>exporter.js&nbsp;</strong>in <em>client/</em> with the following code:</p>

           <p> The first two functions&nbsp;are pretty alike. They call a Meteor method and wait for the data to unparse them using Papa Parse. Once you have the CSV string a private function is called and the download starts. A note: downloading a file via browser&nbsp;might raise some cross-browser issues. The above solution works well both in Chrome and Firefox (<a href="https://github.com/mholt/PapaParse/issues/175" target="_blank">thanks to @mholt in this post</a>).</p> 
           <p>Alright, now we switch roles and dive deep into the server realm. Make sure you are in server/ and create a file called <b>export.js</b>. Before adding any content, let’s understand how Papa works.</p> 
           <p>There are <a href="http://papaparse.com/docs#json-to-csv" target="_blank">many ways</a> to&nbsp;parse a JSON data to CSV. The one I liked most was the third example in the previous link. This way, you pass an object that contains an array with the fields name and after an array of arrays with the proper data. Using this approach gives us the flexibility to iterate over embed documents and format any data.</p>
