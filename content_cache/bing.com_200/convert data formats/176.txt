                    <p>I've done this on Chemstation version A.09. It SHOULD work with later ones as well.</p> 
                    <p>If your chromatograph has 2 signals, like an FID on the front and an NPD on the back, it loads and saves them together. (If you have not looked in explorer, there is not actually a .d file, but a .d folder that contains several other files. The one your interested in is the .ch file that&nbsp;typically looks something&nbsp;like 010F0101.ch . So long as the directory has a .d file extension Chemstation will open and export any .ch files together.</p> 
                    <p>Heres what to do:</p> 
                    <p>Open a dos box and navigate to that directory with the .d&nbsp;directories your interesting in.&nbsp;Its usually something like C:\HPCHEM\2\DATA (where that number is the number of the instrument.)</p> 
                    <p>The directory name "aiaexp.d" could really be any name you want, so long as it ends with the .d. In later versions it may even be able to be a long filename, but I have not tried it.</p> 
                    <p>run the following commands.</p> 
                    <p>mkdir aiaexp.d</p> 
                    <p>for /D %i in (*.d) do (copy %i\FID1A.ch aiaexp.d\%i)&nbsp; &amp; ren aiaexp.d\*.d *.cf</p> 
                    <p>If you had two injections, 001F0011.d and 002F0021.d ,each containing FID1A.ch and NPD1A.ch,it would take the FID1A.ch file from the 001F0011.d folder, and make a copy of it in aiaexp.d named 001F0011.d and rename it (in fact it will change&nbsp;&nbsp;anything else in the the aiaexp.d directory with a .d extension to a .ch extension) to 001F0011.cf. Similarly it would make a copy of FID1A.ch from 002F0021.d in the aiaexp.d folder named 002D0021.cf.</p> 
                    <p>Now open aiaexp.d in Chemstation. They will come up as stacked chromatograms&nbsp;on your screen. Then simply save as AIA format where the individual signals will be saved as signal1.cdf,signal2.cdf etc.</p> 
                    <p>No special software or difficult to write macros. Just some old school command line action.</p> 
                   </div>
