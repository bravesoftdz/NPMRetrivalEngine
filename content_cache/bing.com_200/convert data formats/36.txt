                   And one other note... JavaScript was created in about 2 months time, in a dead-heat race for a finished language. &nbsp;As a result there are some oddities in JavaScript, to be sure. &nbsp;The "month" number returned by JavaScript functions is zero-based, with zero being January and 11 being December. &nbsp;Nobody writes calendar dates that way (except JavaScript). &nbsp;You do not have to worry about this anomaly if you're using the new Date() notation, but it's worth being aware of it when you retrieve fractional parts of a date value or attempt arithmetic.
