     <h1><a name="to_date-convert-string-to-datetime-oracle-to-sql-server-migration" id="to_date-convert-string-to-datetime-oracle-to-sql-server-migration">TO_DATE - Convert String to Datetime - Oracle to SQL Server Migration</a></h1> 
     <div class="level1"> 
      <p> In Oracle, TO_DATE function converts a string value to DATE data type value using the specified format. In <acronym title="Structured Query Language">SQL</acronym> Server, you can use CONVERT or TRY_CONVERT function with an appropriate datetime style. </p> 
      <p> <strong>Oracle</strong>: </p> 
      <pre class="code sql">  <span class="co1">-- Specify a datetime string and its exact format</span>
  <span class="kw1">SELECT</span> TO_DATE<span class="br0">(</span><span class="st0">'2012-06-05'</span><span class="sy0">,</span> <span class="st0">'YYYY-MM-DD'</span><span class="br0">)</span> <span class="kw1">FROM</span> dual;</pre> 
      <p> <strong><acronym title="Structured Query Language">SQL</acronym> Server</strong>: </p> 
      <pre class="code sql">  <span class="co1">-- Specify a datetime string and style 102 (ANSI format), raises an error if conversion fails</span>
  <span class="kw1">SELECT</span> CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'2012-06-05'</span><span class="sy0">,</span> 102<span class="br0">)</span>;
&nbsp;
  <span class="co1">-- TRY_CONVERT available since SQL Server 2012 (returns NULL if conversion fails)</span>
  <span class="kw1">SELECT</span> TRY_CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'2012-06-05'</span><span class="sy0">,</span> 102<span class="br0">)</span>;</pre> 
     </div>

     <h2><a name="converting-oracle-to_date-to-sql-server" id="converting-oracle-to_date-to-sql-server">Converting Oracle TO_DATE to SQL Server</a></h2> 
     <div class="level2"> 
      <p> Unlike Oracle TO_DATE function that allows you to build any format string using format specifiers (YYYY and MM i.e.), in <acronym title="Structured Query Language">SQL</acronym> Server, you have to use a datetime <strong>style</strong> that defines the format for <strong>the entire</strong> datetime string. </p> 
      <p> Fortunately, most applications use typical datetime formats in Oracle that can be easily mapped to a datetime format style in <acronym title="Structured Query Language">SQL</acronym> Server. </p> 
     </div> 
     <h2><a name="difference-between-convert-and-try_convert-in-sql-server" id="difference-between-convert-and-try_convert-in-sql-server">Difference Between CONVERT and TRY_CONVERT in SQL Server</a></h2> 
     <div class="level2"> 
      <p> You can use both CONVERT and TRY_CONVERT functions to convert a string to a datetime value. </p> 
      <p> CONVERT raises an error when it cannot recognize the format, while TRY_CONVERT returns NULL in this case: </p> 
      <p> <strong><acronym title="Structured Query Language">SQL</acronym> Server</strong>: </p> 
      <pre class="code sql">  <span class="co1">-- Specify not valid datetime string</span>
  <span class="kw1">SELECT</span> CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'ABC'</span><span class="br0">)</span>;
  <span class="co2"># Msg 241, Level 16, State 1, Line 1</span>
  <span class="co2"># Conversion failed when converting date and/or time from character string.</span>
&nbsp;
  <span class="kw1">SELECT</span> TRY_CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'ABC'</span><span class="br0">)</span>;
  <span class="co2"># NULL</span></pre> 
     </div> 
     <h2><a name="convert-and-try_convert-recognize-ansiiso-and-us-formats-by-default" id="convert-and-try_convert-recognize-ansiiso-and-us-formats-by-default">CONVERT and TRY_CONVERT Recognize ANSI/ISO and US Formats by Default</a></h2> 
     <div class="level2"> 
      <p> Note that when converting a string to datetime, both CONVERT and TRY_CONVERT recognize ANSI/<acronym title="International Organization for Standardization">ISO</acronym> datetime formats with various delimiters by default, so you <strong>do not</strong> need to specify a style for them. </p> 
      <p> An ANSI/<acronym title="International Organization for Standardization">ISO</acronym> format is year, month, day, hour, minute, seconds, fractional seconds (YYYY-MM-DD HH24:MI:SS.FFF) where trailing parts can be omitted so you can specify YYYY-MM-DD, or YYYY-MM-DD HH24:MI etc. </p> 
      <p> <strong><acronym title="Structured Query Language">SQL</acronym> Server</strong>: </p> 
      <pre class="code sql">  <span class="co1">-- ISO date formats with various delimiters recognized by default (year, month, day)</span>
  <span class="kw1">SELECT</span> CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'2012-06-30'</span><span class="br0">)</span>;
  <span class="kw1">SELECT</span> CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'2012/06/30'</span><span class="br0">)</span>;
  <span class="kw1">SELECT</span> CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'2012.06.30'</span><span class="br0">)</span>;
&nbsp;
  <span class="kw1">SELECT</span> CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'2012-06-30 11:10'</span><span class="br0">)</span>;
  <span class="kw1">SELECT</span> CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'2012-06-30 11:10:09'</span><span class="br0">)</span>;
  <span class="kw1">SELECT</span> CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'2012-06-30 11:10:09.333'</span><span class="br0">)</span>;
  <span class="kw1">SELECT</span> CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'2012/06/30 11:10:09.333'</span><span class="br0">)</span>;
  <span class="kw1">SELECT</span> CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'2012.06.30 11:10:09.333'</span><span class="br0">)</span>;
&nbsp;
  <span class="co1">-- ISO date without delimiters is also recognized</span>
   <span class="kw1">SELECT</span> CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'20120630'</span><span class="br0">)</span>;</pre> 
      <p> <acronym title="Structured Query Language">SQL</acronym> Server also recognizes United States datetime format (month, day, year and time) by default, so you do not need to specify style 101: </p> 
      <p> <strong><acronym title="Structured Query Language">SQL</acronym> Server</strong>: </p> 
      <pre class="code sql">  <span class="co1">-- United States date formats with various delimiters recognized by default (month, day, year)</span>
  <span class="kw1">SELECT</span> CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'06-30-2012'</span><span class="br0">)</span>;
  <span class="kw1">SELECT</span> CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'06/30/2012'</span><span class="br0">)</span>;
  <span class="kw1">SELECT</span> CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'06.30.2012'</span><span class="br0">)</span>;
&nbsp;
  <span class="kw1">SELECT</span> CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'06-30-2012 11:10'</span><span class="br0">)</span>;
  <span class="kw1">SELECT</span> CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'06/30/2012 11:10:09'</span><span class="br0">)</span>;
  <span class="kw1">SELECT</span> CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'06.30.2012  11:10:09.333'</span><span class="br0">)</span>;</pre> 
      <p> Also <acronym title="Structured Query Language">SQL</acronym> Server recognizes the following formats by default: </p> 
      <p> <strong><acronym title="Structured Query Language">SQL</acronym> Server</strong> </p> 
      <pre class="code sql">  <span class="kw1">SELECT</span> CONVERT<span class="br0">(</span><span class="kw1">DATETIME</span><span class="sy0">,</span> <span class="st0">'17-FEB-2013'</span><span class="br0">)</span>;
  <span class="co2"># 2013-02-17 00:00:00.000</span></pre> 
     </div>
