     <br>
     <hr> 
     <p><code>DateFormat</code> is an abstract class for date/time formatting subclasses which formats and parses dates or time in a language-independent manner. The date/time formatting subclass, such as <code><a href="https://developer.android.com/reference/java/text/SimpleDateFormat.html">SimpleDateFormat</a></code>, allows for formatting (i.e., date ? text), parsing (text ? date), and normalization. The date is represented as a <code>Date</code> object or as the milliseconds since January 1, 1970, 00:00:00 GMT. </p>
     <p><code>DateFormat</code> provides many class methods for obtaining default date/time formatters based on the default or a given locale and a number of formatting styles. The formatting styles include <code><a href="https://developer.android.com/reference/java/text/DateFormat.html#FULL">FULL</a></code>, <code><a href="https://developer.android.com/reference/java/text/DateFormat.html#LONG">LONG</a></code>, <code><a href="https://developer.android.com/reference/java/text/DateFormat.html#MEDIUM">MEDIUM</a></code>, and <code><a href="https://developer.android.com/reference/java/text/DateFormat.html#SHORT">SHORT</a></code>. More detail and examples of using these styles are provided in the method descriptions. </p>
     <p><code>DateFormat</code> helps you to format and parse dates for any locale. Your code can be completely independent of the locale conventions for months, days of the week, or even the calendar format: lunar vs. solar. </p>
     <p>To format a date for the current Locale, use one of the static factory methods: </p>
     <blockquote> 
      <pre><code>myString = DateFormat.getDateInstance().format(myDate);
 </code></pre> 
     </blockquote> 
     <p>If you are formatting multiple dates, it is more efficient to get the format and use it multiple times so that the system doesn't have to fetch the information about the local language and country conventions multiple times. </p>
     <blockquote> 
      <pre><code>DateFormat df = DateFormat.getDateInstance();
 for (int i = 0; i &lt; myDate.length; ++i) {
     output.println(df.format(myDate[i]) + "; ");
 }
 </code></pre> 
     </blockquote> 
     <p>To format a date for a different Locale, specify it in the call to <code><a href="https://developer.android.com/reference/java/text/DateFormat.html#getDateInstance(int, java.util.Locale)">getDateInstance()</a></code>. </p>
     <blockquote> 
      <pre><code>DateFormat df = DateFormat.getDateInstance(DateFormat.LONG, Locale.FRANCE);
 </code></pre> 
     </blockquote> 
     <p>You can use a DateFormat to parse also. </p>
     <blockquote> 
      <pre><code>myDate = df.parse(myString);
 </code></pre> 
     </blockquote> 
     <p>Use <code>getDateInstance</code> to get the normal date format for that country. There are other static factory methods available. Use <code>getTimeInstance</code> to get the time format for that country. Use <code>getDateTimeInstance</code> to get a date and time format. You can pass in different options to these factory methods to control the length of the result; from <code><a href="https://developer.android.com/reference/java/text/DateFormat.html#SHORT">SHORT</a></code> to <code><a href="https://developer.android.com/reference/java/text/DateFormat.html#MEDIUM">MEDIUM</a></code> to <code><a href="https://developer.android.com/reference/java/text/DateFormat.html#LONG">LONG</a></code> to <code><a href="https://developer.android.com/reference/java/text/DateFormat.html#FULL">FULL</a></code>. The exact result depends on the locale, but generally: </p>
     <ul>
      <li><code><a href="https://developer.android.com/reference/java/text/DateFormat.html#SHORT">SHORT</a></code> is completely numeric, such as <code>12.13.52</code> or <code>3:30pm</code> </li>
      <li><code><a href="https://developer.android.com/reference/java/text/DateFormat.html#MEDIUM">MEDIUM</a></code> is longer, such as <code>Jan 12, 1952</code> </li>
      <li><code><a href="https://developer.android.com/reference/java/text/DateFormat.html#LONG">LONG</a></code> is longer, such as <code>January 12, 1952</code> or <code>3:30:32pm</code> </li>
      <li><code><a href="https://developer.android.com/reference/java/text/DateFormat.html#FULL">FULL</a></code> is pretty completely specified, such as <code>Tuesday, April 12, 1952 AD or 3:30:42pm PST</code>. </li>
     </ul> 
     <p>You can also set the time zone on the format if you wish. If you want even more control over the format or parsing, (or want to give your users more control), you can try casting the <code>DateFormat</code> you get from the factory methods to a <code><a href="https://developer.android.com/reference/java/text/SimpleDateFormat.html">SimpleDateFormat</a></code>. This will work for the majority of countries; just remember to put it in a <code>try</code> block in case you encounter an unusual one. </p>
     <p>You can also use forms of the parse and format methods with <code><a href="https://developer.android.com/reference/java/text/ParsePosition.html">ParsePosition</a></code> and <code><a href="https://developer.android.com/reference/java/text/FieldPosition.html">FieldPosition</a></code> to allow you to </p>
     <ul>
      <li>progressively parse through pieces of a string. </li>
      <li>align any particular field, or find out where it is for selection on the screen. </li>
     </ul> 
     <h3><a name="synchronization">Synchronization</a></h3> 
     <p> Date formats are not synchronized. It is recommended to create separate format instances for each thread. If multiple threads access a format concurrently, it must be synchronized externally.</p>

      <p>Parse a date/time string according to the given parse position. For example, a time text <code>"07/10/96 4:5 PM, PDT"</code> will be parsed into a <code>Date</code> that is equivalent to <code>Date(837039900000L)</code>. </p>
      <p> By default, parsing is lenient: If the input is not in the form used by this object's format method but can still be parsed as a date, then the parse succeeds. Clients may insist on strict adherence to the format by calling <code><a href="https://developer.android.com/reference/java/text/DateFormat.html#setLenient(boolean)">setLenient(false)</a></code>. </p>
      <p>This parsing operation uses the <code><a href="https://developer.android.com/reference/java/text/DateFormat.html#calendar">calendar</a></code> to produce a <code>Date</code>. As a result, the <code>calendar</code>'s date-time fields and the <code>TimeZone</code> value may have been overwritten, depending on subclass implementations. Any <code>TimeZone</code> value that has previously been set by a call to <code><a href="https://developer.android.com/reference/java/text/DateFormat.html#setTimeZone(java.util.TimeZone)">setTimeZone</a></code> may need to be restored for further operations.</p>

If parsing succeeds, then the index of <code>pos</code> is updated to the index after the last character used (parsing does not necessarily use all characters up to the end of the string), and the parsed date is returned. The updated <code>pos</code> can be used to indicate the starting point for the next call to this method. If an error occurs, then the index of <code>pos</code> is not changed, the error index of <code>pos</code> is set to the index of the character where the error occurred, and null is returned. </p>
