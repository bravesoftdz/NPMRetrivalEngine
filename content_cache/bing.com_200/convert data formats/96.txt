      <h1>Java SimpleDateFormat</h1>

       <p> The <code>java.text.SimpleDateFormat</code> class is used to both parse and format dates according to a formatting pattern you specify yourself. </p> 
       <p> This text explains how to use the <code>SimpleDateFormat</code> class to format dates. </p> 
       <a name="creating-a-simpledateformat"></a> 
       <h2>Creating a SimpleDateFormat</h2> 
       <p> You create a <code>SimpleDateFormat</code> instance like this: </p> 
       <pre class="codeBox">
String pattern = "yyyy-MM-dd";
SimpleDateFormat simpleDateFormat = new SimpleDateFormat(pattern);
</pre> 
       <p> The <code>pattern</code> String parameter passed to the <code>SimpleDateFormat</code> constructor is the pattern to use for parsing and formatting of dates. The pattern syntax is covered later in this text. </p> 
       <a name="formatting-dates"></a> 
       <h2>Formatting Dates</h2> 
       <p> Once you have created a <code>SimpleDateFormat</code> instance you can format dates using its <code>format()</code> method. Here is an example: </p> 
       <pre class="codeBox">
String pattern = "yyyy-MM-dd";
SimpleDateFormat simpleDateFormat = new SimpleDateFormat(pattern);

String date = simpleDateFormat.format(new Date());
System.out.println(date);
</pre> 
       <p> The <code>Date</code> instance passed to the <code>format()</code> method is a <code>java.util.Date</code> instance. </p> 
       <p> The output printed from this code would be: </p> 
       <pre class="codeBox">
2012-11-03
</pre> 
       <a name="parsing-dates"></a> 
       <h2>Parsing Dates</h2> 
       <p> You can parse a String into a <code>java.util.Date</code> instance using the <code>parse()</code> method of the <code>SimpleDateFormat</code> instance. Here is an example: </p> 
       <pre class="codeBox">
String pattern = "yyyy-MM-dd";
SimpleDateFormat simpleDateFormat = new SimpleDateFormat(pattern);

Date date = simpleDateFormat.parse("2012-12-24");
</pre> 
       <p> Once this code is executed, the <code>date</code> variable points to a <code>Date</code> instance representing december 24th, 2012. </p> 
       <a name="creating-a-simpledateformat"></a> 
       <h2>Creating a SimpleDateFormat For a Specific Locale</h2> 
       <p> You can create a <code>SimpleDateFormat</code> instance targeted at a specific <code>Locale</code>. Doing so will format the dates according to that <code>Locale</code> whenever relevant. For instance, a formatting pattern including the name of the week day will write the week day in the language of the given <code>Locale</code>. Here is an example: </p> 
       <pre class="codeBox">
String pattern = "EEEEE MMMMM yyyy HH:mm:ss.SSSZ";
SimpleDateFormat simpleDateFormat =
        new SimpleDateFormat(pattern, new Locale("da", "DK"));

String date = simpleDateFormat.format(new Date());
System.out.println(date);
</pre> 
       <p> The output printed from this code could be: </p> 
       <pre class="codeBox">
lørdag november 2012 10:38:45.156+0100
</pre> 
       <p> Notice how week day (lø = saturday) and month (november) is written in Danish. </p> 
       <a name="pattern-syntax"></a> 
       <h2>Pattern Syntax</h2> 
       <p> You can use the following symbols in your formatting pattern: </p> 
       <table class="dataTable" cellspacing="0"> 
        <tbody>
         <tr> 
          <td>G</td> 
          <td>Era designator (before christ, after christ)</td> 
         </tr> 
         <tr> 
          <td>y</td> 
          <td>Year (e.g. 12 or 2012). Use either yy or yyyy.</td> 
         </tr> 
         <tr> 
          <td>M</td> 
          <td>Month in year. Number of M's determine length of format (e.g. MM, MMM or MMMMM)</td> 
         </tr> 
         <tr> 
          <td>d</td> 
          <td>Day in month. Number of d's determine length of format (e.g. d or dd)</td> 
         </tr> 
         <tr> 
          <td>h</td> 
          <td>Hour of day, 1-12 (AM / PM) (normally hh)</td> 
         </tr> 
         <tr> 
          <td>H</td> 
          <td>Hour of day, 0-23 (normally HH)</td> 
         </tr> 
         <tr> 
          <td>m</td> 
          <td>Minute in hour, 0-59 (normally mm)</td> 
         </tr> 
         <tr> 
          <td>s</td> 
          <td>Second in minute, 0-59 (normally ss)</td> 
         </tr> 
         <tr> 
          <td>S</td> 
          <td>Millisecond in second, 0-999 (normally SSS)</td> 
         </tr> 
         <tr> 
          <td>E</td> 
          <td>Day in week (e.g Monday, Tuesday etc.)</td> 
         </tr> 
         <tr> 
          <td>D</td> 
          <td>Day in year (1-366)</td> 
         </tr> 
         <tr> 
          <td>F</td> 
          <td>Day of week in month (e.g. 1st Thursday of December)</td> 
         </tr> 
         <tr> 
          <td>w</td> 
          <td>Week in year (1-53)</td> 
         </tr> 
         <tr> 
          <td>W</td> 
          <td>Week in month (0-5)</td> 
         </tr> 
         <tr> 
          <td>a</td> 
          <td>AM / PM marker</td> 
         </tr> 
         <tr> 
          <td>k</td> 
          <td>Hour in day (1-24, unlike HH's 0-23)</td> 
         </tr> 
         <tr> 
          <td>K</td> 
          <td>Hour in day, AM / PM (0-11)</td> 
         </tr> 
         <tr> 
          <td>z</td> 
          <td>Time Zone </td> 
         </tr> 
         <tr> 
          <td>'</td> 
          <td>Escape for text delimiter</td> 
         </tr> 
         <tr> 
          <td>'</td> 
          <td>Single quote</td> 
         </tr> 
        </tbody>
       </table> 
       <p> Characters other than these will be treated as normal text to insert into the pattern, and thus into the formatted dates. </p> 
       <p> Some characters can be used in different numbers. For instance, you can write either <code>yy</code> for a 2-character version of the year (e.g. 12), or you can write <code>yyyy</code> for a 4-character version of the year (e.g. 2012). For more information about the patterns accepted, see the JavaDoc for the <code>SimpleDateFormat</code> class. </p> 
       <a name="pattern-examples"></a> 
       <h2>Pattern Examples</h2> 
       <p> Here are a few patterns examples: </p> 
       <table class="dataTable" cellspacing="0"> 
        <tbody>
         <tr> 
          <th>Pattern</th> 
          <th>Example</th> 
         </tr> 
         <tr> 
          <td>dd-MM-yy</td> 
          <td>31-01-12</td> 
         </tr> 
         <tr> 
          <td>dd-MM-yyyy</td> 
          <td>31-01-2012</td> 
         </tr> 
         <tr> 
          <td>MM-dd-yyyy</td> 
          <td>01-31-2012</td> 
         </tr> 
         <tr> 
          <td>yyyy-MM-dd</td> 
          <td>2012-01-31</td> 
         </tr> 
         <tr> 
          <td>yyyy-MM-dd HH:mm:ss</td> 
          <td>2012-01-31 23:59:59</td> 
         </tr> 
         <tr> 
          <td>yyyy-MM-dd HH:mm:ss.SSS</td> 
          <td>2012-01-31 23:59:59.999</td> 
         </tr> 
         <tr> 
          <td>yyyy-MM-dd HH:mm:ss.SSSZ</td> 
          <td>2012-01-31 23:59:59.999+0100</td> 
         </tr> 
         <tr> 
          <td>EEEEE MMMMM yyyy HH:mm:ss.SSSZ</td> 
          <td>Saturday November 2012 10:45:42.720+0100</td> 
         </tr> 
        </tbody>
       </table> 
       <a name="dateformatsymbols"></a> 
       <h2>DateFormatSymbols</h2> 
       <p> It is possible to customize the date symbols used in the formatted output, for a specific <code>Locale</code>. You do so using a <code>java.text.DateFormatSymbols</code> instance. Here is an example: </p> 
       <pre class="codeBox">
Locale locale = new Locale("en", "UK");
DateFormatSymbols dateFormatSymbols = new DateFormatSymbols(locale);
dateFormatSymbols.setWeekdays(new String[]{
        "Unused",
        "Sad Sunday",
        "Manic Monday",
        "Thriving Tuesday",
        "Wet Wednesday",
        "Total Thursday",
        "Fat Friday",
        "Super Saturday",
});

String pattern = "EEEEE MMMMM yyyy";
SimpleDateFormat simpleDateFormat =
        new SimpleDateFormat(pattern, dateFormatSymbols);

String date = simpleDateFormat.format(new Date());
System.out.println(date);
</pre> 
       <p> First a new <code>DateFormatSymbols</code> instance is created for the UK <code>Locale</code>. </p> 
       <p> Second, a new set of names for week days is set. Notice that the first string <code>"unused"</code> is never used. The indices in this array must start from one, to be indexable by the <code>Calendar.SUNDAY</code>, <code>Calendar.MONDAY</code> etc. constants. The <code>Calendar.SUNDAY</code> constant is 1, <code>Calendar.MONDAY</code> is 2 etc. </p> 
       <p> Third a <code>SimpleDateFormat</code> is created using the <code>DateFormatSymbols</code>, and a date is formatted with it. The output printed from this could would look like this: </p> 
       <pre class="codeBox">
Super Saturday November 2012
</pre> 
       <p> Notice how the custom week day name is used. </p> 
       <p> You can set more date formatting symbols on the <code>DateFormatSymbols</code> instance. Here are the methods you can use to set additional symbols: </p> 
       <pre class="codeBox">
dateFormatSymbols.setWeekdays();
dateFormatSymbols.setAmPmStrings();
dateFormatSymbols.setEras();
dateFormatSymbols.setLocalPatternChars();
dateFormatSymbols.setMonths();
dateFormatSymbols.setShortMonths();
dateFormatSymbols.setShortWeekdays();
dateFormatSymbols.setZoneStrings();
</pre> 
       <p> See the JavaDoc for the <code>java.text.DateFormatSymbols</code> class for more details about these methods and symbols. </p>
