                    <h3 class="post-title entry-title" itemprop="name"> Accessing Package Version Information in Java Code </h3> 
                    <div class="post-header"> 
                     <div class="post-header-line-1"></div> 
                    </div> 
                    <div class="post-body entry-content" id="post-body-887201726139322323" itemprop="description articleBody">
                      The 
                     <a href="http://download.oracle.com/javase/7/docs/api/java/lang/Package.html">Javadoc documentation for the Package class</a> states:
                     <br> 
                     <blockquote>
                      Package objects contain version information about the implementation and specification of a Java package. This versioning information is retrieved and made available by the ClassLoader instance that loaded the class(es). Typically, it is stored in the manifest that is distributed with the classes.
                     </blockquote>In this blog post, I look at how Java code can use the Package class to access version information about a particular Java package's specification and implementation.
                     <br> 
                     <br> The 
                     <a href="http://download.oracle.com/javase/tutorial/">Java Tutorials</a>' 
                     <a href="http://download.oracle.com/javase/tutorial/deployment/TOC.html">Deployment Trail</a> features the 
                     <a href="http://download.oracle.com/javase/tutorial/deployment/jar/">Packaging Programs in JAR Files</a> lesson that includes the relevant section 
                     <a href="http://download.oracle.com/javase/tutorial/deployment/jar/packageman.html">Setting Package Version Information</a>. That 
                     <a href="http://download.oracle.com/javase/tutorial/deployment/jar/packageman.html">short page</a> demonstrates how to 
                     <a href="http://download.oracle.com/javase/tutorial/deployment/jar/packageman.html">provide package version information</a> inside a 
                     <a href="http://download.oracle.com/javase/tutorial/deployment/jar/basicsindex.html">JAR</a>'s 
                     <a href="http://download.oracle.com/javase/tutorial/deployment/jar/manifestindex.html">manifest file</a>.
                     <br> 
                     <br> The 
                     <a href="http://download.oracle.com/javase/tutorial/deployment/jar/packageman.html">Setting Package Version Information page</a> indicates that seven pieces of information (seven lines in the manifest file) can be specified for package version information. The first line of the seven is the name of the package and is indicated by 
                     <code>Name</code>. The other six pieces of version information that can be specified are title, version, and vendor for the specification and the same title, version, and vendor for the implementation. An example of what this might look like is shown next in this example 
                     <code>MANIFEST.MF</code> file.
                     <br> 
                     <br> 
                     <strong>Example MANIFEST.MF File With Package Version Information</strong>
                     <br> 
                     <pre name="code" class="plain" style="font-size: 9pt; overflow: auto;">Name: dustin/examples/
Specification-Title: Dustin's Java/Groovy Examples
Specification-Version: 2011.06.17
Specification-Vendor: Marxian Software
Implementation-Title: Dustin Examples
Implementation-Version: 2011.06.17.02
Implementation-Vendor: Marxian Software
</pre>
                     <br> In the above example, the various version information is provided for the package 
                     <code>dustin.examples</code> because that (
                     <code>dustin/examples/</code>) is listed after 
                     <code>Name: </code>.
                     <br> 
                     <br> These manifest entries can be created in an 
                     <a href="http://ant.apache.org/">Ant</a> build as shown in the next excerpt from an 
                     <a href="http://ant.apache.org/">Ant</a> 
                     <code>build.xml</code> file.
                     <br> 
                     <br> 
                     <strong>Excerpt from build.xml for Building JAR with Manifest with Package Version Information</strong>
                     <br> 
                     <pre name="code" class="xml" style="font-size: 9pt; overflow: auto;">&lt;target name="jar"
        description="Package compiled classes into JAR file"
        depends="compile"&gt;
   &lt;jar destfile="${dist.dir}/${jar.name}"
        basedir="${classes.dir}"
        filesonly="${jar.filesonly}"&gt;
     &lt;manifest&gt;
        &lt;section name="common"&gt;
           &lt;attribute name="Specification-Title" value="Example Java Classes" /&gt;
           &lt;attribute name="Specification-Version" value="1.2" /&gt;
           &lt;attribute name="Specification-Vendor" value="marxsoftware.blogspot.com" /&gt;
           &lt;attribute name="Implementation-Version" value="1.2.1" /&gt;
           &lt;attribute name="Implementation-Vendor" value="Dustin" /&gt;
           &lt;attribute name="Implementation-Title" value="Java Packaging Example" /&gt;
        &lt;/section&gt;
        &lt;section name="dustin/examples/"&gt;
           &lt;attribute name="Specification-Title" value="Dustin's Java/Groovy Examples (Spec)" /&gt;
           &lt;attribute name="Specification-Version" value="2011.06.17" /&gt;
           &lt;attribute name="Specification-Vendor" value="Marxian Software (Spec)" /&gt;
           &lt;attribute name="Implementation-Title" value="Dustin Examples (Impl)" /&gt;
           &lt;attribute name="Implementation-Version" value="2011.06.17.02" /&gt; 
           &lt;attribute name="Implementation-Vendor" value="Marxian Software (Impl)" /&gt;
           &lt;attribute name="Package-Title" value="Dustin Examples (Package)" /&gt;
           &lt;attribute name="Package-Version" value="2011.06.17.05" /&gt;
           &lt;attribute name="Package-Vendor" value="Dustin (Package)" /&gt;
        &lt;/section&gt;
     &lt;/manifest&gt;
   &lt;/jar&gt;
&lt;/target&gt;
</pre>
                     <br> When the above Ant target is executed, it builds a JAR file with a MANIFEST.MF file that looks like that shown next.
                     <br> 
                     <br> 
                     <pre name="code" class="plain" style="font-size: 9pt; overflow: auto;">Manifest-Version: 1.0
Ant-Version: Apache Ant 1.8.0
Created-By: 1.7.0-ea-b134 (Oracle Corporation)

Name: common
Specification-Title: Example Java Classes
Specification-Version: 1.2
Specification-Vendor: marxsoftware.blogspot.com
Implementation-Version: 1.2.1
Implementation-Vendor: Dustin
Implementation-Title: Java Packaging Example

Name: dustin/examples/
Specification-Title: Dustin's Java/Groovy Examples (Spec)
Specification-Version: 2011.06.17
Specification-Vendor: Marxian Software (Spec)
Implementation-Title: Dustin Examples (Impl)
Implementation-Version: 2011.06.17.02
Implementation-Vendor: Marxian Software (Impl)
Package-Title: Dustin Examples (Package)
Package-Version: 2011.06.17.05
Package-Vendor: Dustin (Package)
</pre>
                     <br> The example shows a common section as well as a section specific to the 
                     <code>dustin.examples</code> package. Note that I specified 
                     <code>Package-Title</code>, 
                     <code>Package-Version</code>, and 
                     <code>Package-Vendor</code> in the Ant build file and they were included in the resultant manifest file. I'll return to these later in this post.
                     <br> 
                     <br> It is very easy to access the package information at this point. Use of 
                     <a href="http://download.oracle.com/javase/7/docs/api/java/lang/Package.html#getPackages()">Package.getPackages()</a> to get all packages associated with the current class loader and 
                     <a href="http://download.oracle.com/javase/7/docs/api/java/lang/Package.html#getPackage(java.lang.String)">Package.getPackage(String)</a> to get a particular package by name from the current class loader are demonstrated in the next code listing for 
                     <code>Main.java</code>.
                     <br> 
                     <br> 
                     <strong>Main.java</strong>
                     <br> 
                     <pre name="code" class="java" style="font-size: 9pt; overflow: auto;">package dustin.examples;

import static java.lang.System.out;

/**
 * Simple class demonstrating in-code access of package-specific information as
 * specified in the Manifest file.
 */
public class Main
{
   /**
    * Display (to standard output) package details for provided Package.
    *
    * @param pkg Package whose details need to be printed to standard output.
    */
   private void displayPackageDetails(final Package pkg)
   {
      final String name = pkg.getName();
      out.println(name);
      out.println("\tSpec Title/Version: " + pkg.getSpecificationTitle() + " " + pkg.getSpecificationVersion());
      out.println("\tSpec Vendor: " +  pkg.getSpecificationVendor());
      out.println("\tImplementation: " + pkg.getImplementationTitle() + " " + pkg.getImplementationVersion());
      out.println("\tImplementation Vendor: " + pkg.getImplementationVendor());
   }

   /**
    * Display all packages associated with the class loader that do not start
    * with "sun", "com", "java", or "org".
    */
   private void displayCustomPackage()
   {
      final Package[] packages = Package.getPackages();
      for (final Package pkg : packages)
      {
         final String name = pkg.getName();
         if (   !name.startsWith("sun") &amp;&amp; !name.startsWith("java")
             &amp;&amp; !name.startsWith("com") &amp;&amp; !name.startsWith("org") )
         {
            displayPackageDetails(pkg);
         }
      }
   }

   /**
    * Main executable that demonstrates use of Package.getPackages() and
    * Package.getPackage(String).
    *
    * @param arguments Command-line arguments; none expected.
    */
   public static void main(final String[] arguments)
   {
      final Main me = new Main();
      me.displayCustomPackage();
      me.displayPackageDetails(Package.getPackage("dustin.examples"));
   }
}
</pre>
                     <br> The output from running the above is shown next.
                     <br> 
                     <br> 
                     <div class="separator" style="clear: both; text-align: center;">
                      <a href="http://4.bp.blogspot.com/-TT5BSlUbhM4/TgA1buoPHzI/AAAAAAAACSc/K2wrPbQcvCI/s1600/outputExecutingPackageMethodsOnPackageDescribedVersioningInManifest.png" imageanchor="1" style="margin-left:1em; margin-right:1em"><img border="0" height="104" src="http://4.bp.blogspot.com/-TT5BSlUbhM4/TgA1buoPHzI/AAAAAAAACSc/K2wrPbQcvCI/s320/outputExecutingPackageMethodsOnPackageDescribedVersioningInManifest.png" width="320"></a>
                     </div>
                     <br> The output shows that the package information described in the MANIFEST.MF's file is available to Java code. I intentionally included three attributes in the MANIFEST.MF example above that are not accessible from Java's 
                     <a href="http://download.oracle.com/javase/7/docs/api/java/lang/Package.html">Package</a> class. These are 
                     <code>Package-Title</code>, 
                     <code>Package-Version</code>, and 
                     <code>Package-Vendor</code>. The 
                     <a href="http://download.oracle.com/javase/tutorial/deployment/jar/packageman.html">Setting Package Version Information page</a> links to the 
                     <a href="http://download.oracle.com/javase/6/docs/technotes/guides/versioning/spec/versioning2.html#wp89936">Package Version Specification</a>, which has a section 
                     <a href="http://download.oracle.com/javase/6/docs/technotes/guides/versioning/spec/versioning2.html#wp91706">1.5.8 JAR Manifest Format</a> that shows use of these three attributes beginning with 
                     <code>Package-</code> rather than the three I showed earlier beginning with 
                     <code>Implementation-</code>. My example intentionally included both sets with different values for each to prove that its the three 
                     <code>Implementation-</code> attributes that are available via the 
                     <code>Package</code> class rather than the three 
                     <code>Package-</code> attributes.
                     <br> 
                     <br> 
                     <br> 
                     <strong style="font-size: 125%;">Conclusion</strong>
                     <br> 
                     <br> I demonstrated in this blog post how to use Java's 
                     <a href="http://download.oracle.com/javase/7/docs/api/java/lang/Package.html">Package class</a> to access version information about a particular Java package. To demonstrate this, I used Ant to specify the package version information in a JAR's MANIFEST.MF file as part of the 
                     <a href="http://ant.apache.org/manual/Tasks/javac.html">javac task</a>.
