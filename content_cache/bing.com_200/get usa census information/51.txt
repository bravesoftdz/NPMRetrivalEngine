        <div id="mainBody">
         &nbsp;
         <p>The new home for Visual Studio documentation is <a href="http://docs.microsoft.com/visualstudio">Visual Studio 2017 Documentation</a> on docs.microsoft.com.</p>
         <p>The latest version of this topic can be found at <a href="https://docs.microsoft.com/en-us/visualstudio/">Visual Studio 2017 Documentation</a>. The JavaScript IntelliSense extensibility feature enables you to customize IntelliSense results in the JavaScript editor for third-party libraries. This can improve the experience of developers who use these libraries.</p>
         <p>The JavaScript language service provides IntelliSense features for third-party JavaScript libraries that are added to a project. For most libraries, statement completion is provided automatically by the language service. The following illustration shows an example of statement completion:</p>
         <p><span><img id="88bd20c0-5e2a-4b53-9946-7224466aac56" alt="Example of statement completion" src="https://i-msdn.sec.s-msft.com/dynimg/IC613188.jpeg" title="Example of statement completion" xmlns=""></span></p>
         <p>If your library includes descriptions of variables, functions, and objects in standard JavaScript comment tags (//), you automatically benefit, by default, from IntelliSense extensibility features, which provide descriptive information in a pop-up box that appears to the right of elements in a completion list, or when you type the opening parenthesis in a function call. The comments in the pop-up box contain the description of the member. The following example shows the pop-up box for a completion list.</p>
         <p><span><img id="3b203e4d-a73c-4937-bb41-10a825090680" alt="Example of a Quick Info pop-up box" src="https://i-msdn.sec.s-msft.com/dynimg/IC613189.jpeg" title="Example of a Quick Info pop-up box" xmlns=""></span></p>
         <p>To further improve the developer experience, you might want to provide type information to developers in the pop-up box. You can provide type information by using JavaScript <a href="https://msdn.microsoft.com/en-us/library/hh524453.aspx">XML Documentation Comments</a> instead of standard comment tags. You add XML Documentation Comments by using triple-slash comment tags (///) and a defined set of XML elements.</p>
         <p>Alternatively, you can provide type information by using JavaScript IntelliSense extensibility. This feature enables you to customize IntelliSense results by creating JavaScript extensions and adding them to the script context. In the extension, which is a JavaScript file, you subscribe to events that are exposed by the <code>intellisense</code> object of the language service. JavaScript IntelliSense extensibility is the preferred solution for libraries if a behavior pattern in the library prevents the JavaScript language service from providing the desired level of IntelliSense support, and if an alternative to declarative XML Documentation Comments is also needed. By customizing the IntelliSense results, you can create a first-class IntelliSense experience, regardless of any behavioral patterns that might restrict the language service's default capabilities. For more information, see <a href="https://msdn.microsoft.com/en-us/library/hh395059.aspx">Statement Completion for Identifiers</a>.</p>

          <div class="sectionblock">
           <a id="Adding-an-Extension-to-the-Script-Context"></a>
          </div>
         </div>
         <p>For an IntelliSense extension to be executed, it needs to be added to the current script context. The extension can be automatically added to the script context by the automatic discovery mechanism, or you can add the extension to the script context manually by using reference groups or the reference directive.</p>
         <p>The automatic discovery mechanism enables the language service to automatically find extensions that follow the file naming convention <em>libraryname</em>.intellisense.js, and that are located in the same directory as the library to which the extension applies. For example, a valid extension for the jQuery library would be jQuery.intellisense.js. For more restrictive jQuery extensions, you can use file names such as jQuery-1.7.1.intellisense.js (a version-specific extension) or jQuery.ui.intellisense.js (an extension for a scoped jQuery library). The most restrictive version of the extension is used if more than one extension is found for a given library.</p>
         <p>If you want to use the extension for all your JavaScript project files, you might instead choose to add the extension to a reference group. There are several types of reference groups, either those that include implicit references and those that include dedicated worker references. To add an extension, you typically need to add the file as an implicit reference group, either <strong>Implicit (Windows)</strong>, <strong>Implicit (Web)</strong>. Implicit references are in scope for every .js file opened in the Code Editor. When you use this method, you need to add both the extension and the file that the extension is supplementing.</p>
         <p>Use the <strong>IntelliSense</strong> page of the <strong>Options</strong> dialog box to add an extension as a reference group. You can access the <strong>IntelliSense</strong> page by choosing <strong>Tools</strong>, <strong>Options</strong> on the menu bar, and then choosing <strong>Text Editor</strong>, <strong>JavaScript</strong>, <strong>IntelliSense</strong>, <strong>References</strong>. For more information about reference groups, see <a href="https://msdn.microsoft.com/en-us/library/bb385682.aspx">JavaScript IntelliSense</a> and <a href="https://msdn.microsoft.com/en-us/library/hh362485.aspx">Options, Text Editor, JavaScript, IntelliSense</a>.</p>
         <p>If you want to use the extension for a specific set of files, use a reference directive. When you use this method, you need to reference both the extension and the file the extension is supplementing. For information about using the reference directive, see <a href="https://msdn.microsoft.com/en-us/library/bb385682.aspx">JavaScript IntelliSense</a>.</p>

         <p>The extensibility feature enables you to customize IntelliSense results by subscribing to events such as the <code>statementcompletion</code> event of the language service <code>intellisense</code> object. The following example shows a simple extension that's used by the language service to hide members that begin with an underscore from statement completion. This code is contained in underscorefilter.js and is in the \\<em>Visual Studio installation path</em>\JavaScript\References folder.</p> 
         <div id="code-snippet-1" class="codeSnippetContainer" xmlns=""> 
          <div class="codeSnippetContainerTabs"> 
           <div class="codeSnippetContainerTabSingle" dir="ltr">
            <a>JavaScript</a>

intellisense.addEventListener(<span style="color:#A31515;">'statementcompletion'</span>, <span style="color:Blue;">function</span> (event) {  
    <span style="color:Blue;">if</span> (event.targetName === <span style="color:#A31515;">"this"</span>) <span style="color:Blue;">return</span>;  
  
    <span style="color:Blue;">var</span> filterRegex;  
  
    <span style="color:Blue;">if</span> (event.target === undefined || event.target === window)  
        filterRegex = /^_.*\d{2,}/;  
    <span style="color:Blue;">else</span>

    event.items = event.items.filter(<span style="color:Blue;">function</span> (item) {

If a completion list can be identified, the extension updates the statement completion <a href="#Items">items Property</a> collection by filtering out members that begin with an underscore.</p>

For additional examples, see <a href="#CodeExamples">Code Examples</a>.</p>

         <p>In your extension code, you can create handlers for the following event types by using <code>addEventListener</code>:</p>
         <ul>
          <li><p><code>statementcompletion</code>, which adds a handler for a statement completion event. Statement completion provides a list of members for a particular type that appears after you type a special character such as a period (.), or a list of identifiers that appears while you type or when you press CTRL + J. The handler receives an event object of type <code>CompletionEvent</code>, which supports the following members: <a href="#Items">items Property</a>, <a href="#Target">target Property</a>, <a href="#TargetName">targetName Property</a>, and <a href="#Scope">scope Property</a>.</p></li>
          <li><p><code>signaturehelp</code>, which adds a handler for IntelliSense Parameter Info. Parameter information gives you information about the number, names, and types of parameters required by a function. The handler receives an event object of type <code>SignatureHelpEvent</code>, which supports the following members: <a href="#Target">target Property</a>, <a href="#ParentObject">parentObject Property</a>, <a href="#FunctionComments">functionComments Property</a>, <a href="#FunctionHelp">functionHelp Property</a>.</p></li>
          <li><p><code>statementcompletionhint</code>, which adds a handler for IntelliSense Quick Info. The Quick Info pop-up box shows the complete declaration for identifiers in your code. The handler receives an event object of type <code>CompletionHintEvent</code>, which supports the following members: <a href="#CompletionItem">completionItem Property</a>, and <a href="#SymbolHelp">symbolHelp Property</a>.</p></li>
         </ul>

         <p>The following sections describe the members that are exposed in the event object for the following events: <code>statementcompletion</code>, <code>signaturehelp</code>, and <code>statementcompletionhint</code>.</p>
         <h3 class="subHeading"><a id="completionItem-Property"></a><a id="CompletionItem"></a> completionItem Property</h3>
         <p>Returns the identifier, known as the completion item, for which a Quick Info pop-up box is requested. This property is available for the <code>statementcompletionhint</code> event object and for the <a href="#Items">items Property</a> property of the <code>statementcompletion</code> event object.</p>
         <p>Return value: <code>completionItem</code> object</p>
         <p>Following are the members of the <code>completionItem</code> object:</p>
         <ul>
          <li><p><code>name</code>. Read/write when used in the <code>items</code> collection; otherwise, read-only. Returns a string that identifies the completion item.</p></li>
          <li><p><code>kind</code>. Read/write when used in the <code>items</code> collection; otherwise, read-only. Returns a string that represents the type of completion item. The possible values are method, field, property, parameter, variable, and reserved.</p></li>
          <li><p><code>glyph</code>. Read/write when used in the <code>items</code> collection; otherwise, read-only. Returns a string that represents an icon that's displayed in the completion list. The possible values for <code>glyph</code> use the following format: vs:<em>glyphType</em>, where <em>glyphType</em> corresponds to the language-independent members in the <a href="https://msdn.microsoft.com/en-us/library/microsoft.visualstudio.language.intellisense.standardglyphgroup.aspx">StandardGlyphGroup</a> enumeration. For example, <code>vs:GlyphGroupMethod</code> is one possible value for <code>glyph</code>. When <code>glyph</code> is not set, the <code>kind</code> property determines the default icon.</p></li>
          <li><p><code>parentObject</code>. Read-only. Returns the parent object.</p></li>
          <li><p><code>value</code>. Read-only. Returns an object that represents the value of the completion item.</p></li>
          <li><p><code>comments</code>. Read-only. Returns a string that contains the comments that are above the field or variable.</p></li>
          <li><p><code>scope</code>. Read-only. Returns the scope of the completion item. The possible values are global, local, parameter, and member.</p></li>
         </ul>
         <h3 class="subHeading"><a id="items-Property"></a><a id="Items"></a> items Property</h3>
         <p>Gets or sets the array of statement completion items. Each element in the array is a <a href="#CompletionItem">completionItem Property</a> object. The <code>items</code> property is available for the <code>statementcompletion</code> event object.</p>
         <p>Return value: array</p>
         <h3 class="subHeading"><a id="functionComments-Property"></a><a id="FunctionComments"></a> functionComments Property</h3>
         <p>Returns the comments for the function. This property is available for the <code>signaturehelp</code> event object.</p>
         <p>Return value: <code>comments</code> object</p>
         <p>Following are the members of the <code>comments</code> object:</p>
         <ul>
          <li><p><code>above</code>. Returns the comments above the function.</p></li>
          <li><p><code>inside</code>. Returns the comments inside the function, typically in VSDoc format.</p></li>
          <li><p><code>paramComments</code>. Returns an array representing comments for each parameter in the function. The members of the array include:</p>
           <ul>
            <li><p><code>name</code>. Returns a string representing the parameter name.</p></li>
            <li><p><code>comment</code>. Returns a string that contains the parameter comment.</p></li>
           </ul></li>
         </ul>
         <h3 class="subHeading"><a id="functionHelp-Property"></a><a id="FunctionHelp"></a> functionHelp Property</h3>
         <p>Returns the Help for the function. This property is available for the <code>signaturehelp</code> event object.</p>
         <p>Return value: <code>functionHelp</code> object</p>
         <p>Following are the members of the <code>functionHelp</code> object:</p>
         <ul>
          <li><p><code>functionName</code>. Read/write. Returns a string that contains the function name.</p></li>
          <li><p><code>signatures</code>. Read/write. Gets or sets the array of function signatures. Each element in the array is a <code>signature</code> object. Some <code>signature</code> properties, such as <code>locid</code>, correspond to common <a href="https://msdn.microsoft.com/en-us/library/hh524453.aspx">XML Documentation Comments</a> attributes.</p><p>The members of the <code>signature</code> object include:</p>
           <ul>
            <li><p><code>description</code>. Read/write. Returns a string that describes the function.</p></li>
            <li><p><code>locid</code>. Read/write. Returns a string identifier that contains localization information about the function.</p></li>
            <li><p><code>helpKeyword</code>. Read/write. Returns a string that contains the Help keyword.</p></li>
            <li><p><code>externalFile</code>. Read/write. Returns a string that represents the file that contains the member ID.</p></li>
            <li><p><code>externalid</code>. Read/write. Returns a string that represents the member ID of the function.</p></li>
            <li><p><code>params</code>. Read/write. Gets or sets the array of parameters for the function. Each element in the parameters array is a <code>parameter</code> object that has properties that correspond to the following attributes of the <a href="https://msdn.microsoft.com/en-us/library/hh542720.aspx">&lt;param&gt;</a> element:</p>
             <ul>
              <li><p><code>name</code>. Read/write. Returns a string that represents the parameter name.</p></li>
              <li><p><code>type</code>. Read/write. Returns a string that represents the parameter type.</p></li>
              <li><p><code>elementType</code>. Read/write. If the type is <code>Array</code>, returns a string that represents the type of the elements in the array.</p></li>
              <li><p><code>description</code>. Read/write. Returns a string that describes the parameter.</p></li>
              <li><p><code>locid</code>. Read/write. Returns a string identifier that contains localization information about the function.</p></li>
              <li><p><code>optional</code>. Read/write. Returns a string that indicates whether the parameter is optional. <code>true</code> indicates that the parameter is optional; <code>false</code> indicates that it isn't.</p></li>
             </ul></li>
            <li><p><code>returnValue</code>. Read/write. Gets or sets a return value object with properties that correspond to the following attributes of the <a href="https://msdn.microsoft.com/en-us/library/hh542719.aspx">&lt;returns&gt;</a> element:</p>
             <ul>
              <li><p><code>type</code>. Read/write. Returns a string that represents the return type.</p></li>
              <li><p><code>elementType</code>. Read/write. If the type is <code>Array</code>, returns a string that represents the type of the elements in the array.</p></li>
              <li><p><code>description</code>. Read/write. Returns a string that describes the return value.</p></li>
              <li><p><code>locid</code>. Read/write. Returns a string identifier that contains localization information about the function.</p></li>
              <li><p><code>helpKeyword</code>. Read/write. Returns a string that contains the Help keyword.</p></li>
              <li><p><code>externalFile</code>. Read/write. Returns a string that represents the file that contains the member ID.</p></li>
              <li><p><code>externalid</code>. Read/write. Returns a string that represents the member ID of the function.</p></li>
             </ul></li>
           </ul></li>
         </ul>
         <h3 class="subHeading"><a id="parentObject-Property"></a><a id="ParentObject"></a> parentObject Property</h3>
         <p>Returns the parent object of a member function. For example, for <code>document.getElementByID</code>, <code>parentObject</code> returns the <code>document</code> object. This property is available for the <code>signaturehelp</code> event object.</p>
         <p>Return value: object</p>
         <h3 class="subHeading"><a id="target-Property"></a><a id="Target"></a> target Property</h3>
         <p>Returns an object that represents the item to the left of the trigger character, which is a period (.). For functions, <code>target</code> returns the function for which Parameter Info is requested. This property is available for the <code>statementcompletion</code> and <code>signaturehelp</code> event objects.</p>
         <p>Return value: object</p>
         <h3 class="subHeading"><a id="targetName-Property"></a><a id="TargetName"></a> targetName Property</h3>
         <p>Returns a string that represents the target. For example, for "this.", <code>targetName</code> returns "this". For "A.B" (when the cursor is after "B"), <code>targetName</code> returns "B". This property is available for the <code>statementcompletion</code> event object.</p>
         <p>Return value: string</p>
         <h3 class="subHeading"><a id="symbolHelp-Property"></a><a id="SymbolHelp"></a> symbolHelp Property</h3>
         <p>Returns the completion item for which a Quick Info pop-up box is requested. This property is available for the <code>statementcompletionhint</code> event object.</p>
         <p>Return value: <code>symbolHelp</code> object.</p>
         <p>Some properties of the <code>symbolHelp</code> object, such as <code>locid</code>, correspond to common <a href="https://msdn.microsoft.com/en-us/library/hh524453.aspx">XML Documentation Comments</a> attributes.</p>
         <p>Following are the members of the <code>symbolHelp</code> object:</p>
         <ul>
          <li><p><code>name</code>. Read/write. Returns a string that contains the identifier name.</p></li>
          <li><p><code>symbolType</code>. Read/write. Returns a string that represents the symbol type. Possible values include Unknown, Boolean, Number, String, Object, Function, Array, Date, and Regex.</p></li>
          <li><p><code>symbolDisplayType</code>. Read/write. Returns a string that contains the type name to display. If <code>symbolDisplayType</code> isn't set, <code>symbolType</code> is used.</p></li>
          <li><p><code>elementType</code>. Read/write. If the <code>symbolType</code> is <code>Array</code>, returns a string that represents the type of the elements in the array.</p></li>
          <li><p><code>scope</code>. Read/write. Returns a string that represents the scope of the symbol. Possible values include global, local, parameter, and member.</p></li>
          <li><p><code>description</code>. Read/write. Returns a string that contains a description of the symbol.</p></li>
          <li><p><code>locid</code>. Read/write. Returns a string identifier that contains localization information about the symbol.</p></li>
          <li><p><code>helpKeyword</code>. Read/write. Returns a string that contains the Help keyword.</p></li>
          <li><p><code>externalFile</code>. Read/write. Returns a string that represents the file that contains the member ID.</p></li>
          <li><p><code>externalid</code>. Read/write. Returns a string that represents the member ID of the symbol.</p></li>
          <li><p><code>functionHelp</code>. Read/write. Returns a <a href="#FunctionHelp">functionHelp Property</a>, which might contain information when the <code>symbolType</code> is Function.</p></li>
         </ul>
         <h3 class="subHeading"><a id="scope-Property"></a><a id="Scope"></a> scope Property</h3>
         <p>Returns the completion scope of the event. The possible values for the completion scope are global and members. This property is available for the <code>statementcompletion</code> event object.</p>
         <p>Return value: string</p>

         <h3 class="subHeading"><a id="Avoiding-run-time-effects-on-IntelliSense-results"></a><a id="Overriding"></a> Avoiding run-time effects on IntelliSense results</h3>
         <p>The JavaScript language service runs code to dynamically provide IntelliSense information. As a result, run-time behavior can occasionally interfere with desired results. The following procedure shows how to override IntelliSense results when run-time behavior results in incorrect IntelliSense.</p>
         <p>For this example to work, you need the following JavaScript files in your project:</p>
         <ul>
          <li><p>exampleLib.js, which is a project file that represents a third-party library.</p></li>
          <li><p>exampleLib.intellisense.js, which is the IntelliSense extension. This file doesn't need to be included in the project, but it does need to be in the same folder as exampleLib.js.</p></li>
          <li><p>appCode.js, which is a project file that represents app code.</p></li>
         </ul>
