         <h2><b><i>An example of R in TERR (Package “RinR”) using the Census REST API</i></b></h2>
         <p>In this post we will walk through an example of acquiring data from the US Census RESTful web service API (Application Program Interface) via URL parameters. In simple terms, a RESTful URL allows you to pass URL parameters into an online datasource and returns the data specified by the parameters.&nbsp; Using a RESTful source enables access to data without having to store this information locally in a database or flat file.</p>
         <p>Please note, I will be referring to the Census web service as the “Census API” for much of this post; however, at times I may say “Census URL” particularly when speaking of the parameter’s required in the URL. For clarity, the Census URL is the “Interface” portion of the Census Application Program Interface.</p>
         <p><span style="color: #0000ff;"><a title="REST - Representational State Transfer" href="http://en.wikipedia.org/wiki/Representational_state_transfer" target="_blank"><span style="color: #0000ff;">Click here to learn more about&nbsp;REST</span></a></span></p>
         <p>The full Census API to Spotfire process consists of four parts:</p>
         <p>1)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Installing R alongside TERR in your local machine<br> 2)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Adding new packages to your R instance<br> 3)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Manipulating the Census Data API via R<br> 4)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Learning how to access R within TERR using the TERR package “RinR”</p>
         <p><span style="line-height: 1.5em;">This post will go over parts one and two. A follow-up post will detail parts three and four.</span></p>
         <h2></h2>
         <h2><b><i>TIBCO Enterprise Runtime for R (“TERR”)</i></b></h2>
         <p><span style="color: #0000ff;"><a style="font-size: 14px; line-height: 1.5em;" title="Spotfire TERR" href="http://spotfire.tibco.com/discover-spotfire/what-does-spotfire-do/predictive-analytics/tibco-enterprise-runtime-for-r-terr" target="_blank"><span style="color: #0000ff;">TIBCO Enterprise Runtime for R</span></a></span><span style="font-size: 14px; line-height: 1.5em;"> or TERR is a powerful, and often overlooked, tool that comes standard with Spotfire Analyst. It allows the use of many R packages optimized for use in Spotfire. While the TERR development team has done a tremendous job of migrating R packages, there are thousands of R packages with new ones being added every day, making it impossible to migrate all of them.</span></p>
         <p>Therefore, to maximize your TERR experience, we need to leverage all the packages available for R.</p>
         <p>But first, a warning:&nbsp; This solution is only for a local device running TERR and R. It does not address using Statistics Service or configuring analysis for online consumers.</p>
         <p>With that out of the way, let’s install R!</p>
         <p>&nbsp;</p>
         <h2><b><i>Installing R on your local machine</i></b></h2>
         <p><span style="line-height: 1.5em;">Installing R starts with downloading the installer from CRAN.&nbsp; R is made available through the Comprehensive R Archive Network (CRAN). This is arguably the most confusing step in installing R – finding what you need to download!</span></p>
         <p>To make things easy here’s the <span style="color: #0000ff;"><a title="CRAN Download for Windows Users" href="http://cran.us.r-project.org/bin/windows/base/" target="_blank"><span style="color: #0000ff;">download location for Windows users</span></a></span>. Other users can navigate to the <span style="color: #0000ff;"><a title="CRAN Download" href="http://cran.us.r-project.org/" target="_blank"><span style="color: #0000ff;">download here</span></a></span>.</p>
         <p>While CRAN does provide installation instructions,&nbsp;you’ll find these few short steps most helpful:</p>
         <p>1)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Download and run the R installer<br> 2)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Select the installation directory, the default is <b>C:\Program Files\R\&lt;version number&gt;<br> </b>3)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Select if you want both or just the 32- or 64-bit install (R will figure out which one your need in most cases)<br> 4)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Select start up options (this just customizes how the R default GUI will display and it will not impact performance)<br> 5)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Select your preference of start menu and desktop options<br> 6)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Install</p>
         <p>You now have R installed!</p>
         <p>There are several ways to access R:</p>
         <p>1)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Original Console /command line / terminal interface (<b>R.exe</b>)</p>
         <ul>
          <li>R was initially entered line by line and that option remains today. Assuming you installed R in your default directory you can access R Console here: <b>C:\Program Files\R\R-3.0.2\bin</b>. Simply double click <b>R.exe</b> to open the Console.&nbsp; It looks just like a command line interface in Windows or a Terminal in UNIX.</li>
         </ul>
         <p>2)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; R GUI (<b>Rgui.exe</b>)</p>
         <ul>
          <li>R also comes packaged with a basic GUI application. There are 32- and 64-bit versions, both found in the x32and x64 folders contained in the “bin” folder you found <b>R.exe</b> and both named <b>Rgui.exe</b>. The R GUI is also the application opened when selecting the R application from your start menu or desktop, if you elected to have those shortcuts created when installing R.</li>
         </ul>
         <p>3)&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Third Party Application</p>
         <ul>
          <li>There are many different applications which use R (like Spotfire!); however, creating complex scripts in Spotfire’s interface can be difficult, almost as difficult as using R’s command line interface. Therefore, several third party applications exist to make your R coding experience much easier.&nbsp; We will not go into these here, but in an upcoming post we’ll go in-depth on setting up Rstudio to program in both R and TERR.</li>
         </ul>
         <p>Now you’ve installed R and you can open it, so let’s use it!</p>
         <h2><em><b>Installing Packages in R</b></em></h2>
         <p>Let’s open up our R GUI application.&nbsp; You should be able to find it in your start menu. If it’s not there, check in your R “bin” folder <b>C:\Program Files\R\R-3.0.2\bin\&lt;64 or 32&gt; </b>&nbsp;for the file called <b>Rgui.exe</b>.</p>
         <p>See the image below for some help if this is your first time using R:</p>
         <p><a href="http://syntelli.com/wp-content/uploads/2014/05/1-Installing-Packages-in-R.jpg"><img class="aligncenter size-full wp-image-5130" src="http://syntelli.com/wp-content/uploads/2014/05/1-Installing-Packages-in-R.jpg" alt="Installing Packages in R" width="430" height="349"></a></p>
         <p>All you get initially is a fancy version of the R Console. The red cursor at the bottom is where you enter commands, so let’s enter a command.</p>
         <p>First, to verify R is working, let’s try something simple.&nbsp; Type <span style="color: #ff0000;"><b>2+2</b></span> and hit Enter:</p>
         <p><a href="http://syntelli.com/wp-content/uploads/2014/05/2-Command-in-R.jpg"><img class="aligncenter size-full wp-image-5131" src="http://syntelli.com/wp-content/uploads/2014/05/2-Command-in-R.jpg" alt="Command in R" width="111" height="78"></a></p>
         <p>It should return 4.</p>
         <p><a title="R Functionality and Commands" href="http://www.cyclismo.org/tutorial/R/index.html" target="_blank">Click here </a>for an excellent tutorial and&nbsp;explanation on R functionality and commands.</p>
         <p>Now that we have confirmed R is working, it is time to extend its functionality.&nbsp; As we mentioned before, in order to get the most out of R, we need to install packages. In this example, we are installing the <b>jsonlite</b> package (R is case sensitive) in order to extract and transform JSON data from the Census API.</p>
         <p>Installing packages is a relatively painless process. It is as simple as executing <b>install.packages(&lt;packagename&gt;)</b> if you know the exact name and case of your package. You can also select <b><i>Packages &gt; Install Packages</i></b>… from your R GUI for a list of available packages.</p>
         <p>In this example we will install jsonlite by typing <i>install.packages(“jsonlite”) </i>in the console and hit Enter.</p>
         <p style="text-align: center;"><strong><em><span style="color: #ff9900;">What. Is. Happening!?</span></em></strong><span style="color: #ff9900;"><span style="color: #000000;">&nbsp;</span></span>&nbsp;<a href="http://syntelli.com/wp-content/uploads/2014/05/3-Installing-jsonlite-packing-in-R.jpg"><img class="aligncenter size-full wp-image-5133" src="http://syntelli.com/wp-content/uploads/2014/05/3-Installing-jsonlite-packing-in-R.jpg" alt="Installing jsonlite packing in R" width="473" height="353"></a></p>
         <p>A feature of R GUI is an easy way to select your CRAN mirror. A CRAN mirror is an online repository of R packages. Typically highlighting the location closest to you geographically, then clicking “OK” will work fine.</p>
         <p>Also notice the line on the Console output à Installing package into <b>C:/Users/&lt;user&gt;/Documents/R/win-library/3.0</b> (as ‘lib’ is unspecified)</p>
         <p>This is your default library location. It is where packages are installed. In TERR this will be different; therefore, if you run into issues combining TERR and R, the first step is to define the library location of scripts when loading libraries. There will be additional info on library locations in a future post.</p>
         <p>Once you select a CRAN mirror, jsonlite will begin to install. R may also install other dependent packages required for jsonlite to function properly if they are not already present. So don’t panic if it starts “Installing Dependencies.”</p>
         <p>Finally, confirm jsonlite has been successfully installed by loading it from the package library via <b>library(jsonlite)</b> or <b>requrie(jsonlite)</b>. <b>library(&lt;package name&gt;)</b> is how you tell R to use a particular package in your script. Remember, a library is the location a package is stored in, but in most cases you can use the terms “library” and “package” interchangeably when talking about these things.</p>
         <p>If there are no error messages when calling the library, you have successfully installed jsonlite.</p>
         <p><a href="http://syntelli.com/wp-content/uploads/2014/05/4-Confirming-jsonlite-package-install-in-R.jpg"><img class="aligncenter size-full wp-image-5136" src="http://syntelli.com/wp-content/uploads/2014/05/4-Confirming-jsonlite-package-install-in-R.jpg" alt="Confirming jsonlite package install in R" width="168" height="40"></a></p>
         <p>With R and jsonlite successfully installed our next step is to access data, then manipulate it with Spotfire. Our next post will show you how to use the jsonlite package to access data via the Census REST URL, then use the R jsonlite code within a TERR script to run everything within Spotfire.</p>
         <p>That’s all for now – stay tuned for <a href="http://syntelli.com/tip-of-the-week/use-r-scripts-tibco-spotfire-terr-part-2/" target="_blank">PART 2</a> on <b>How to use R-Scripts in TIBCO Spotfire® TERR.</b></p>
