      <div class="summary clearFix"> 
       <p>Provides members you can use to access and manage Windows Store-related data for the current app. For example, you can use members of this class to get Windows Store listing and license info for the current app, purchase the current app or products that are offered by the app, or download and install package updates for the app.</p>

     <h2 class="accented">Remarks</h2> 
     <div class="NOTE">
      <h5>Note</h5>
      <p>This class and the rest of the <a class="xref" href="windows.services.store" data-linktype="relative-path">Windows.Services.Store</a> namespace was introduced in Windows 10, version 1607. This class can only be used in projects that target <strong>Windows 10 Anniversary Edition (10.0; Build 14393)</strong> or a later release in Visual Studio. If your project targets an earlier version of Windows 10, you must use the <a class="xref" href="windows.applicationmodel.store" data-linktype="relative-path">Windows.ApplicationModel.Store</a> namespace instead of the <a class="xref" href="windows.services.store" data-linktype="relative-path">Windows.Services.Store</a> namespace. For more information, see <a href="https://msdn.microsoft.com/windows/uwp/monetize/in-app-purchases-and-trials" data-linktype="external">In-app purchases and trials</a>.</p> 
     </div> 
     <p>The <a class="xref" href="windows.services.store.storecontext" data-linktype="relative-path">StoreContext</a> class is the main entry point to the <a class="xref" href="windows.services.store" data-linktype="relative-path">Windows.Services.Store</a> namespace. Use members of this class to perform tasks such as getting Windows Store listing and license info for the current app, purchasing the current app or add-ons that are offered by the app, or downloading and installing package updates for the app. Other classes and types in this namespace represent items such as add-ons for the app, licenses for the app and its add-ons, and Windows Store listing info for the app.</p> 
     <p>To get a <a class="xref" href="windows.services.store.storecontext" data-linktype="relative-path">StoreContext</a> object, use one of these static methods:</p> 
     <ul> 
      <li><p><a class="xref" href="windows.services.store.storecontext#Windows_Services_Store_StoreContext_GetDefault" data-linktype="relative-path">GetDefault</a>: Use this method in single-user apps (that is, apps that run only in the context of the user that launched the app). This method gets a <a class="xref" href="windows.services.store.storecontext" data-linktype="relative-path">StoreContext</a> object that you can use to access and manage Windows Store-related data for the user. Most Universal Windows Platform (UWP) apps are single-user apps.</p> <pre><code class="lang-csharp">Windows.Services.Store.StoreContext context = StoreContext.GetDefault();
</code></pre></li> 
      <li><p><a class="xref" href="windows.services.store.storecontext#Windows_Services_Store_StoreContext_GetForUser_Windows_System_User_" data-linktype="relative-path">GetForUser</a>: Use this method in multi-user apps. This method gets a <a class="xref" href="windows.services.store.storecontext" data-linktype="relative-path">StoreContext</a> object that you can use to access and manage Windows Store-related data for a specific user. For more information about multi-user apps, see <a href="https://msdn.microsoft.com/windows/uwp/xbox-apps/multi-user-applications" data-linktype="external">Introduction to multi-user applications</a>.</p> <pre><code class="lang-csharp">var users = await Windows.System.User.FindAllAsync();
Windows.Services.Store.StoreContext context = StoreContext.GetForUser(users[0]);
</code></pre></li> 
     </ul> 
     <div class="NOTE">
      <h5>Note</h5>
      <p>In a Windows desktop application that uses the <a href="https://developer.microsoft.com/windows/bridges/desktop" data-linktype="external">Desktop Bridge</a>, you must add some additional code to configure the <a class="xref" href="windows.services.store.storecontext" data-linktype="relative-path">StoreContext</a> object before your app can use this object. For more information, see <a href="https://msdn.microsoft.com/windows/uwp/monetize/in-app-purchases-and-trials#desktop" data-linktype="external">Using the StoreContext class in a desktop application that uses the Desktop Bridge</a>.</p> 
     </div> 
     <p>After you have a <a class="xref" href="windows.services.store.storecontext" data-linktype="relative-path">StoreContext</a> object, you can start calling methods to purchase an app or add-on for the current user and other tasks. For more information, see the following articles:</p> 
     <ul> 
      <li><a href="https://msdn.microsoft.com/windows/uwp/monetize/in-app-purchases-and-trials" data-linktype="external">In-app purchases and trials</a></li> 
      <li><a href="https://msdn.microsoft.com/windows/uwp/monetize/get-product-info-for-apps-and-add-ons" data-linktype="external">Get product info for apps and add-ons</a></li> 
      <li><a href="https://msdn.microsoft.com/windows/uwp/monetize/get-license-info-for-apps-and-add-ons" data-linktype="external">Get license info for apps and add-ons</a></li> 
      <li><a href="https://msdn.microsoft.com/windows/uwp/monetize/enable-in-app-purchases-of-apps-and-add-ons" data-linktype="external">Enable in-app purchases of apps and add-ons</a></li> 
      <li><a href="https://msdn.microsoft.com/windows/uwp/monetize/enable-consumable-add-on-purchases" data-linktype="external">Enable consumable add-on purchases</a></li> 
      <li><a href="https://msdn.microsoft.com/windows/uwp/monetize/implement-a-trial-version-of-your-app" data-linktype="external">Implement a trial version of your app</a></li> 
     </ul> 
     <p>You can also use a <a class="xref" href="windows.services.store.storecontext" data-linktype="relative-path">StoreContext</a> object to download and install package updates for the app. For more information, see <a href="https://msdn.microsoft.com/windows/uwp/packaging/self-install-package-updates" data-linktype="external">Download and install package updates for your app</a>.</p>

The Store ID for the add-on is available in the Windows Dev Center dashboard, and it is returned by the <a class="xref" href="windows.services.store.storeproduct#Windows_Services_Store_StoreProduct_StoreId" data-linktype="relative-path">StoreId</a> property of the <a class="xref" href="windows.services.store.storeproduct" data-linktype="relative-path">StoreProduct</a> that represents the add-on. For more information about using this method, including a code example, see <a href="https://msdn.microsoft.com/windows/uwp/monetize/enable-consumable-add-on-purchases" data-linktype="external">Enable consumable add-on purchases</a>.</p> 
        <p>The <em>trackingId</em> parameter is used to help ensure that each fulfillment operation is handled exactly once by the Windows Store. For example, consider a scenario where you call <a class="xref" href="windows.services.store.storecontext#Windows_Services_Store_StoreContext_ReportConsumableFulfillmentAsync_System_String_System_UInt32_System_Guid_" data-linktype="relative-path">ReportConsumableFulfillmentAsync</a> with a given pair of <em>productStoreId</em> and <em>trackingId</em> values, the Windows Store successfully marks the consumable as fulfilled, but there is a network connectivity issue that prevents your app from receiving and handling the response. The next time your app retries the same <a class="xref" href="windows.services.store.storecontext#Windows_Services_Store_StoreContext_ReportConsumableFulfillmentAsync_System_String_System_UInt32_System_Guid_" data-linktype="relative-path">ReportConsumableFulfillmentAsync</a> call with the same pair of <em>productStoreId</em> and <em>trackingId</em> values after it regains connectivity, the Windows Store will report the transaction as successful without charging the user again. The <em>trackingId</em> parameter can be retrieved via the <a class="xref" href="windows.services.store.storeconsumableresult#Windows_Services_Store_StoreConsumableResult_TrackingId" data-linktype="relative-path">TrackingId</a> property of the <a class="xref" href="windows.services.store.storeconsumableresult" data-linktype="relative-path">StoreConsumableResult</a> return value.</p>

For more information, see <a href="https://docs.microsoft.com/windows/uwp/monetize/in-app-purchases-and-trials#desktop" data-linktype="external">this article</a>.</p> 
        </div> 
        <h4 class="accented">Remarks</h4> 
        <p>To get the list of packages that have updates available, use the <a class="xref" href="windows.services.store.storecontext#Windows_Services_Store_StoreContext_GetAppAndOptionalStorePackageUpdatesAsync" data-linktype="relative-path">GetAppAndOptionalStorePackageUpdatesAsync</a> method. After you download the packages, you can install packages by using the <a class="xref" href="windows.services.store.storecontext#Windows_Services_Store_StoreContext_RequestDownloadAndInstallStorePackageUpdatesAsync_Windows_Foundation_Collections_IIterable_Windows_Services_Store_StorePackageUpdate__" data-linktype="relative-path">RequestDownloadAndInstallStorePackageUpdatesAsync</a> method. For more information about using this method, including a code example, see <a href="https://msdn.microsoft.com/windows/uwp/packaging/self-install-package-updates" data-linktype="external">Download and install package updates for your app</a>.</p> 
        <p>This operation will not block. The <a class="xref" href="windows.foundation.iasyncoperationwithprogress_tresult_tprogress_" data-linktype="relative-path">IAsyncOperationWithProgress</a> object returned by this method will complete after the update packages are downloaded.</p> 
        <p>When you call this method, the OS displays a dialog that tells the user that an app update is available to download and asks the user's permission to proceed with the download. If the user does not grant permission to start the download, the <a class="xref" href="windows.services.store.storepackageupdateresult#Windows_Services_Store_StorePackageUpdateResult_OverallState" data-linktype="relative-path">OverallState</a> property of the <a class="xref" href="windows.services.store.storepackageupdateresult" data-linktype="relative-path">StorePackageUpdateResult</a> return value has the value <strong>Canceled</strong>.</p> 
        <p>The method that you assign to handle <a class="xref" href="windows.foundation.iasyncoperationwithprogress_tresult_tprogress_#Windows_Foundation_IAsyncOperationWithProgress_2_Progress" data-linktype="relative-path">Progress</a> notifications is called one time for each step in the download process for each package in this request. The <a class="xref" href="windows.foundation.iasyncoperationwithprogress_tresult_tprogress_#Windows_Foundation_IAsyncOperationWithProgress_2_Progress" data-linktype="relative-path">Progress</a> handler receives a <a class="xref" href="windows.services.store.storepackageupdatestatus" data-linktype="relative-path">StorePackageUpdateStatus</a> argument that provides info about the update package that raised the progress notification.</p>
