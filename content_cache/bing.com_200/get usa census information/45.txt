      <h1 class="title">Using JavaScript Along with ASP.NET</h1> 
      <div id="nstext">
       &nbsp; 
       <p>Bill Evjen<br> Reuters</p> 
       <p>February 2004</p> 
       <p>Applies to:<br> &nbsp;&nbsp;&nbsp;&nbsp;Microsoft® ASP.NET<br> &nbsp;&nbsp;&nbsp;&nbsp;Microsoft Visual Basic® .NET<br> &nbsp;&nbsp;&nbsp;&nbsp;Microsoft Visual C#® .NET</p> 
       <p><strong>Summary: </strong>Learn how to apply JavaScript in ASP.NET applications. Also learn ways of doing control focus, button rollovers, alerts and placing text in the browser's status bar. (17 printed pages)</p> 
       <h4 class="dtH1">Contents</h4> 
       <p><a href="#aspnet-usingjavascript_topic01">Introduction</a> <br> <a href="#aspnet-usingjavascript_topic02">Adding JavaScript to a Server Control</a> <br> <a href="#aspnet-usingjavascript_topic03">Performing a Simple Button-rollover</a> <br> <a href="#aspnet-usingjavascript_topic04">Setting Control Focus</a> <br> <a href="#aspnet-usingjavascript_topic05">Changing the Control Focus Dynamically</a> <br> <a href="#aspnet-usingjavascript_topic06">Using Larger JavaScript Functions</a> <br> <a href="#aspnet-usingjavascript_topic07">Keeping JavaScript in a Separate File (.js)</a> <br> <a href="#aspnet-usingjavascript_topic08">Summary</a></p> 
       <h2 class="dtH1"><a id="aspnet-usingjavascript_topic01"></a>Introduction</h2> 
       <p>Web developers have been working around the limitations of the browser for a long time by using a combination of server-side processing and client-side processing. Working logic and application processes on the client-side allows browser based applications to seem more responsive and to have more "snappiness" to them. For client-side development in the past, most developers turned to JavaScript and intermingled this programming language into their classic Microsoft® Active Server Pages along with their server-side Microsoft Visual Basic® Scripting Edition or Microsoft JScript® code. Now with Microsoft ASP.NET and the new models that it provides, Web developers often wonder how to properly work with their JavaScript functions in their ASP.NET pages.</p> 
       <p>This article will take a look at some of the ways past JavaScript functions can now be used. There is more than one way to accomplish this task and this article will take a look at many different possibilities. This article will also take a look at some of the more common uses of JavaScript in ASP.NET pages with some short examples.</p> 
       <h2 class="dtH1"><a id="aspnet-usingjavascript_topic02"></a>Adding JavaScript to a Server Control</h2> 
       <p>It is quite easy to add JavaScript to a specific server control that resides on an ASP.NET page. Let's take a look at the button server control as an example. If you drag and drop a Button HTML server control (<code class="ce">HtmlInputButton</code> Class) onto a page using either Microsoft Visual Studio® .NET or the ASP.NET Web Matrix and run it as a server control, it should have the following construction:</p> 
       <pre class="code">&lt;INPUT type="button" value="Button" runat="server" id="Button1"&gt;
</pre> 
       <p>This is a normal button that can be programmatically manipulated in the code-behind or server-side script of an ASP.NET page. For example, to assign the button text when the page is generated, simply use the <strong>value</strong> property of the button after this element is turned into an HTML server control (right-click on the control and select <strong>Run As Server Control</strong>).</p> 
       <p><strong>Visual Basic .NET</strong></p> 
       <pre class="code">Private Sub Page_Load(ByVal sender As System.Object, _
  ByVal e As System.EventArgs) Handles MyBase.Load
     Button1.Value = DateTime.Now.ToString()
End Sub
</pre> 
       <p><strong>Visual C# .NET</strong></p> 
       <pre class="code">void Page_Load(object sender, EventArgs e) {
   Button1.Value = DateTime.Now.ToString();
}
</pre> 
       <p>This simply provides a button on the page that shows a date and time as the text of the button.</p>

       <h2 class="dtH1"><a id="aspnet-usingjavascript_topic03"></a>Performing a Simple Button-rollover</h2> 
       <p>When it comes to buttons on a Web page, one of the more common functionalities that Web developers want to give their buttons is a rollover effect. The rollover effect experience is when the end user hovers their mouse over a button on a Web page (without clicking the button) and the button itself changes color or shape. This can be especially useful for Web pages that have multiple buttons, and it would be beneficial from a usability standpoint to notify the end user of the button they would be clicking prior to clicking it.</p> 
       <p>This was fairly easy to do before server controls came along and it isn't that difficult now with server controls. The code for performing such an operation is as follows:</p> 
       <p><strong>Visual Basic .NET</strong></p> 
       <pre class="code">&lt;%@ Page Language="VB" %&gt;
&lt;script runat="server"&gt;    
   Sub ImageButton1_Click(sender As Object, e As ImageClickEventArgs)
      Label1.Text = "Postback!"
   End Sub
&lt;/script&gt;
&lt;html&gt;
&lt;head&gt;&lt;/head&gt;
&lt;body&gt;
    &lt;form runat="server"&gt;
        &lt;p&gt;
            &lt;asp:ImageButton id="ImageButton1" 
             onmouseover="this.src='button2.jpg'" 
             onclick="ImageButton1_Click" 
             onmouseout="this.src='button1.jpg'" runat="server" 
             ImageUrl="button1.jpg"&gt;&lt;/asp:ImageButton&gt;
        &lt;/p&gt;
        &lt;p&gt;
            &lt;asp:Label id="Label1" runat="server" /&gt;
        &lt;/p&gt;
    &lt;/form&gt;
&lt;/body&gt;
&lt;/html&gt;
</pre> 
       <p><strong>Visual C# .NET</strong></p> 
       <pre class="code">&lt;%@ Page Language="C#" %&gt;
&lt;script runat="server"&gt;    
    void ImageButton1_Click(object sender, ImageClickEventArgs e) {
       Label1.Text = "Postback!";
    }
&lt;/script&gt;
&lt;html&gt;
&lt;head&gt;&lt;/head&gt;
&lt;body&gt;
    &lt;form runat="server"&gt;
        &lt;p&gt;
            &lt;asp:ImageButton id="ImageButton1" 
             onmouseover="this.src='button2.jpg'" 
             onclick="ImageButton1_Click" 
             onmouseout="this.src='button1.jpg'" runat="server" 
             ImageUrl="button1.jpg"&gt;&lt;/asp:ImageButton&gt;
        &lt;/p&gt;
        &lt;p&gt;
            &lt;asp:Label id="Label1" runat="server" /&gt;
        &lt;/p&gt;
    &lt;/form&gt;
&lt;/body&gt;
&lt;/html&gt;
</pre> 
       <p>Instead of assigning the JavaScript to a server control through the <code class="ce">&lt;body&gt;</code> element, this time we used the <strong>onmouseover</strong> and <strong>onmouseout</strong> events of the control. For each of these events, we assigned a JavaScript value. The <strong>onmouseover</strong> event is when the end user hovers their mouse over the control and the <strong>onmouseout</strong> is for actions when the end user removes their mouse from hovering over the control. In our case, we want to show one image while the mouse hovers over the button and then show the original image from when the page was loaded when the mouse moves away from the button.</p> 
       <p>If you are working directly in the control such as this, instead of specifying the control in the <code class="ce">form</code> as we did when working with JavaScript in the <code class="ce">&lt;body&gt;</code> element, you can use the <strong>this</strong> keyword followed by the property you are trying to change.</p> 
       <h2 class="dtH1"><a id="aspnet-usingjavascript_topic04"></a>Setting Control Focus</h2> 
       <p>In the construction of Web Forms, notice that no property is enabled to set the focus of a Web server control (this will be an added feature in ASP.NET 2.0 Whidbey). Therefore when using the .NET Framework 1.0 or 1.1, you need to employ different methods to accomplish this task. You can do it just as you did before ASP.NET came along—using JavaScript.</p> 
       <p>For example, if your ASP.NET page has multiple text boxes on it, focus can be set to the first TextBox control when the page is loaded by employing the following code in the <code class="ce">&lt;body&gt;</code> tag of the page.</p> 
       <pre class="code">&lt;body onload="document.forms[0]['TextBox1'].focus();"&gt;
</pre> 
       <p>Using this construct, when the page is loaded, the element that contains the ID <code class="ce">TextBox1</code> will employ the focus, and this enables the end user to start entering text directly without the need to use the mouse to position the focus.</p> 
       <h2 class="dtH1"><a id="aspnet-usingjavascript_topic05"></a>Changing the Control Focus Dynamically</h2> 
       <p>To change the focus on the ASP.NET page dynamically, turn the <code class="ce">&lt;body&gt; </code>tag into a HTML server control.</p> 
       <pre class="code">&lt;body id="Body1" runat="server"&gt;
</pre> 
       <p>Next, you can change the focus by placing the following code constructs in your ASP.NET server-side events.</p> 
       <p><strong>Visual Basic .NET</strong></p> 
       <pre class="code">Body1.Attributes("onload") = "document.forms[0]['TextBox2'].focus();"
</pre> 
       <p><strong>Visual C# .NET</strong></p> 
       <pre class="code">Body1.Attributes["onload"] = "document.forms[0]['TextBox2'].focus();";
</pre> 
       <p>Using this technique, you can assign the focus to other elements in the form. You might find this useful when using the <strong>OnTextChanged</strong> event if there are multiple elements in the form. If you don't do this, the end user must refocus on the form again using their mouse after an <strong>OnTextChanged</strong> event is fired due to the page postback.</p>

For this article, we will look at the <strong>RegisterStartupScript</strong> and the <strong>RegisterClientScriptBlock</strong> methods.</p>

       <h3 class="dtH1">RegisterStartupScript Method</h3> 
       <p>One of the first options available is to register script blocks using one of the .NET classes for this purpose. The first is the <strong>RegisterStartupScript</strong> method. This class would be best used when you have a JavaScript function that you want to initiate when the page is loaded. For an example of this, create an ASP.NET page in Visual Studio .NET that contains two buttons. <code class="ce">Button1</code> and <code class="ce">Button2</code> should be the IDs of the two buttons.    "function AlertHello() { alert('Hello ASP.NET'); }&lt;/script&gt;")

Button1.Attributes("onclick") = "AlertHello()"
Button2.Attributes("onclick") = "AlertHello()"
</pre> 
       <p><strong>Visual C# .NET</strong></p> 
       <pre class="code">Page.RegisterStartupScript("MyScript",
   "&lt;script language=javascript&gt;" +
   "function AlertHello() { alert('Hello ASP.NET'); }&lt;/script&gt;");

Button1.Attributes["onclick"] = "AlertHello()";
Button2.Attributes["onclick"] = "AlertHello()";
</pre>

       <pre class="code">&lt;HTML&gt;

      &lt;title&gt;WebForm1&lt;/title&gt;
   &lt;/HEAD&gt;
   &lt;body&gt;

         &lt;P&gt;
            &lt;input type="submit" name="Button1" value="Button" 
             id="Button1" onclick="AlertHello()" /&gt;
            &lt;input type="submit" name="Button2" value="Button" 
             id="Button2" onclick="AlertHello()" /&gt;&lt;/P&gt;
            &lt;script language=javascript&gt;function AlertHello() { 
             alert('Hello'); }&lt;/script&gt;
      &lt;/form&gt;
   &lt;/body&gt;
&lt;/HTML&gt;
</pre> 
       <p>Working with this ASP.NET page, notice that there is one JavaScript function that was placed at the bottom of the page before the close of the form (<code class="ce">&lt;/form&gt;</code>).</p> 
       <p>When working with the <strong>RegisterStartupScript</strong> method, the constructor asks for two parameters—the first being the <em>key</em> of the script and the second being the <em>script</em> itself (represented as a <code class="ce">string</code>).</p> 
       <pre class="code">Page.RegisterStartupScript("MyScript", _
   "&lt;script language=javascript&gt;" &amp; _

</pre> 
       <p>It is important that all the JavaScripts on the page have unique keys given to them. If there is more than one JavaScript with the same key name, only the first one will be placed on the page.</p> 
       <h3 class="dtH1">RegisterClientScriptBlock Method</h3> 
       <p>Now let's create a better version of the button rollover example by using the <strong>RegisterClientScriptBlock</strong> method. The problem with the rollover button example from earlier is that when the end user's mouse hovered over the button image, the rollover image had to be retrieved from the server in a separate request. A better rollover button situation would be where the rollover image of the button is already downloaded and stored in the browser's cache so that when the end user hovers over the button, it is instantaneously displayed. To accomplish this we must build a JavaScript function. The following example shows the JavaScript function as well as the use of the <strong>RegisterClientScriptBlock</strong> method to get the function onto the page. For this example, the code-behind only needs a <strong>Page_Load</strong> event and a <strong>button-click</strong> event for an ImageButton server control.</p>

       <h3 class="dtH1">The Difference Between RegisterStartupScript and RegisterClientScriptBlock</h3> 
       <p>So we have shown two different methods for placing JavaScript functions on an ASP.NET page—so what is the difference? The main difference is that the <strong>RegisterStartupScript</strong> method places the JavaScript at the bottom of the ASP.NET page right before the closing <code class="ce">&lt;/form&gt;</code> element. The <strong>RegisterClientScriptBlock</strong> method places the JavaScript directly after the opening <code class="ce">&lt;form&gt;</code> element in the page. So what difference does this make? It can make quite a bit of difference as we will see.</p> 
       <p>For an example of this, here is a way to put focus on a text box on a page when the page is loaded into the browser—with Visual Basic .NET using the <strong>RegisterStartupScript</strong> method:</p> 
       <pre class="code">Page.RegisterStartupScript("Testing", "&lt;script 
  language=javascript&gt;document.forms[0]['TextBox1'].focus();&lt;/script&gt;")
</pre> 
       <p>This works all well and fine because the textbox on the page is generated and placed on the page by the time the browser gets down to the bottom of the page and gets to this little bit of JavaScript. But if instead it was written like this (using the <strong>RegisterClientScriptBlock</strong> method):</p> 
       <pre class="code">Page.RegisterClientScriptBlock("Testing", "&lt;script 
  language=javascript&gt;document.forms[0]['TextBox1'].focus();&lt;/script&gt;")
</pre> 
       <p>Focus will not get to the textbox control and a JavaScript error will be generated on the page (shown in the Figure 5).</p> 
       <p class="fig"><img id="aspnet-usingjavascript-05" alt="Aa479011.aspnet-usingjavascript-05(en-us,MSDN.10).gif" src="https://i-msdn.sec.s-msft.com/dynimg/IC90495.gif" title="Aa479011.aspnet-usingjavascript-05(en-us,MSDN.10).gif" xmlns=""></p> 
       <p class="label"><strong>Figure 5. Error executing JavaScript</strong></p> 
       <p>The reason for this is that the browser will encounter the JavaScript <em>before</em> the text box is on the page. Therefore, the JavaScript will not be able to find a<code class="ce"> TextBox1</code>.</p> 
       <h2 class="dtH1"><a id="aspnet-usingjavascript_topic07"></a>Keeping JavaScript in a Separate File (.js)</h2> 
       <p>Keeping JavaScript functions in a separate file (a <code class="ce">.js</code> file) is highly recommended. Once they are in a separate file and part of a project, the file can be imported into a page using some of the methods already described.</p> 
       <p>For instance, a <code class="ce">.js</code> file can be included in an ASP.NET page using the following code:</p> 
       <p><strong>Visual Basic .NET</strong></p> 
       <pre class="code">Page.RegisterClientScriptBlock("MyScript", _
   "&lt;script language=javascript src='MyJavaScriptFile.js'&gt;")
</pre> 
       <p><strong>Visual C# .NET</strong></p> 
       <pre class="code">Page.RegisterClientScriptBlock("MyScript",
   "&lt;script language=javascript src='MyJavaScriptFile.js'&gt;");
</pre> 
       <p>Once the <code class="ce">.js</code> file is imported into the ASP.NET page, any of the JavaScript functions can be called as before. This is a great way to manage JavaScript functions and keep them separate from the other logic of ASP.NET pages. It is also an easy way to use the same JavaScript functions on multiple ASP.NET pages.</p> 
       <h2 class="dtH1"><a id="aspnet-usingjavascript_topic08"></a>Summary</h2> 
       <p>This paper took a quick look at some common ways of working with JavaScript in ASP.NET pages and some of the more common uses of JavaScript. Some of the more important tips to take away are to keep JavaScript in a separate <code class="ce">.js</code> file and to make use of the <strong>RegisterStartupScript</strong> and <strong>RegisterClientScriptBlock</strong> methods to get the JavaScript into pages. It is also quite easy to use the capabilities of the<strong> HtmlGenericControl</strong> to deploy control-specific JavaScript to ASP.NET pages.</p>
