           <h1>Soundex System</h1> 
           <!-- The content on this page SLURPs into /publications/general-info-leaflets/55.html--> 
           <div id="shared"> 
            <h1>The Soundex Indexing System</h1> 
            <p> <em><small>Updated May 30, 2007</small></em> </p> 
            <div class="nwidget nimage_caption img-wrap pull-right width40" data-caption-body="" data-version="1"> 
             <img alt="soundex-indexing.jpg" src="/files/research/genealogy/images/soundex-indexing.jpg"> 
            </div> 
            <p>To use the census soundex to locate information about a person, you must know his or her full name and the state or territory in which he or she lived at the time of the census. It is also helpful to know the full name of the <i>head of the household</i> in which the person lived because census takers recorded information under that name.</p> 
            <p>The soundex is a coded surname (last name) index based on the way a surname sounds rather than the way it is spelled. Surnames that sound the same, but are spelled differently, like SMITH and SMYTH, have the same code and are filed together. The soundex coding system was developed so that you can find a surname even though it may have been recorded under various spellings.</p> 
            <p>To search for a particular surname, you must first work out its code.</p> 
            <hr> 
            <h2>Basic Soundex Coding Rule</h2> 
            <p>Every soundex code consists of a letter and three numbers, such as W-252. The letter is always the first letter of the surname. The numbers are assigned to the remaining letters of the surname according to the soundex guide shown below. Zeroes are added at the end if necessary to produce a four-character code. Additional letters are disregarded. Examples:</p> 
            <ul> 
             <li><b>Washington</b> is coded W-252 (W, 2 for the S, 5 for the N, 2 for the G, remaining letters disregarded).</li> 
             <li><b>Lee</b> is coded L-000 (L, 000 added).</li> 
            </ul> 
            <hr> 
            <h2>Soundex Coding Guide</h2> 
            <p class="boxed floatRight width38"> See also this <a class="moreLink" href="http://www.eogn.com/soundex/"><strong>online Soundex Calculator</strong></a> </p> 
            <div class="nwidget ncontainer ntable">
             <div class="table table-responsive table-striped"> 
              <table cellpadding="6" cellspacing="0"> 
               <tbody> 
                <tr> 
                 <td align="right"><b>Number</b> </td> 
                 <td><b>Represents the Letters</b> </td> 
                </tr> 
                <tr> 
                 <td align="right"><b>1</b> </td> 
                 <td>B, F, P, V </td> 
                </tr> 
                <tr> 
                 <td align="right"><b>2</b> </td> 
                 <td>C, G, J, K, Q, S, X, Z </td> 
                </tr> 
                <tr> 
                 <td align="right"><b>3</b> </td> 
                 <td>D, T </td> 
                </tr> 
                <tr> 
                 <td align="right"><b>4</b> </td> 
                 <td>L </td> 
                </tr> 
                <tr> 
                 <td align="right"><b>5</b> </td> 
                 <td>M, N </td> 
                </tr> 
                <tr> 
                 <td align="right"><b>6</b> </td> 
                 <td>R </td> 
                </tr> 
               </tbody> 
              </table>
             </div> 
            </div>
            <br> Disregard the letters A, E, I, O, U, H, W, and Y. 
            <p>&nbsp;</p> 
            <hr> 
            <h2>Additional Soundex Coding Rules</h2> 
            <ol> 
             <li><b>Names With Double Letters </b> <p>If the surname has any double letters, they should be treated as one letter. For example:</p> 
              <ul> 
               <li><b>Gutierrez</b> is coded G-362 (G, 3 for the T, 6 for the first R, second R ignored, 2 for the Z).</li> 
              </ul></li> 
             <li><b>Names with Letters Side-by-Side that have the Same Soundex Code Number</b> <p>If the surname has different letters side-by-side that have the same number in the soundex coding guide, they should be treated as one letter. Examples:</p> 
              <ul> 
               <li><b>Pfister</b> is coded as P-236 (P, F ignored, 2 for the S, 3 for the T, 6 for the R).<br> &nbsp;</li> 
               <li><b>Jackson</b> is coded as J-250 (J, 2 for the C, K ignored, S ignored, 5 for the N, 0 added).<br> &nbsp;</li> 
               <li><b>Tymczak</b> is coded as T-522 (T, 5 for the M, 2 for the C, Z ignored, 2 for the K). Since the vowel "A" separates the Z and K, the K is coded.<br> &nbsp;</li> 
              </ul></li> 
             <li><b>Names with Prefixes</b> <p>If a surname has a prefix, such as Van, Con, De, Di, La, or Le, code both with and without the prefix because the surname might be listed under either code. Note, however, that Mc and Mac are <b>not</b> considered prefixes.</p> 
              <blockquote>
                For example, 
               <b>VanDeusen</b> might be coded two ways: 
               <p>V-532 (V, 5 for N, 3 for D, 2 for S)</p> 
               <blockquote> 
                <blockquote>
                  or
                </blockquote> 
               </blockquote> 
               <p>D-250 (D, 2 for the S, 5 for the N, 0 added).</p> 
              </blockquote> </li> 
             <li><b>Consonant Separators</b> <p>If a vowel (A, E, I, O, U) separates two consonants that have the same soundex code, the consonant to the right of the vowel is coded. Example:</p> 
              <blockquote> 
               <strong>Tymczak</strong> is coded as T-522 (T, 5 for the M, 2 for the C, Z ignored (see "Side-by-Side" rule above), 2 for the K). Since the vowel "A" separates the Z and K, the K is coded.
              </blockquote><br> If "H" or "W" separate two consonants that have the same soundex code, the consonant to the right of the vowel is <b>not</b> coded. Example: 
              <blockquote> 
               <b>Ashcraft</b> is coded A-261 (A, 2 for the S, C ignored, 6 for the R, 1 for the F). It is not coded A-226.
              </blockquote></li> 
            </ol> 
            <p>&nbsp;</p>
