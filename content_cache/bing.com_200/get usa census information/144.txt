             <div class="post-text" itemprop="text"> 
              <p>Normally in java or C# , I would use a following construct when try and catch errors.</p> 
              <p>Bad Exception Handling in C# and Java</p> 
              <pre><code>try
{
  // do something
}
catch(Exception ex)
{
}
</code></pre> 
              <p>Good Exception Handling in Java and C#</p> 
              <pre><code>try
{

}
catch(ArgumentException ex)
{
  log(ex);
  // java throw
  throw ex;
 // in C#, throw only, is better since it keeps stack trace 
 throw;

}
catch(Exception ex)
{
  // this is catch ALL others block
  log(ex);
  // java throw
  throw ex;
 // in C#, throw only, is better since it keeps stack trace 
 throw;
}
</code></pre> 
              <p>In C# and java , I would use logging libraries. java : log4j, see others <a href="https://stackoverflow.com/questions/3042450/what-log4j-alternative-logging-libraries-are-available">https://stackoverflow.com/questions/3042450/what-log4j-alternative-logging-libraries-are-available</a></p> 
              <p>.Net : log4net, Nlog etc</p> 
              <p>In PL/SQL</p> 
              <p>Bad Exception Handling</p> 
              <pre><code>BEGIN
-- Do Something
EXCEPTION  -- exception handlers begin
   WHEN ZERO_DIVIDE THEN  
   null;
   WHEN OTHERS THEN  
   null;
END;
</code></pre> 
              <p>Good Exception Handling</p> 
              <pre><code>BEGIN
-- Do Something
EXCEPTION  -- exception handlers begin
   WHEN ZERO_DIVIDE THEN  
   LOG
   RAISE 

   WHEN OTHERS THEN  
   LOG
   RAISE 
END;
</code></pre> 
              <p>a simple google search brings log4plsql package : <a href="http://log4plsql.sourceforge.net/" rel="nofollow noreferrer">http://log4plsql.sourceforge.net/</a></p> 
              <ul> 
               <li>Are there any other libraries?</li> 
               <li>Are they prevalent in PL/SQL programming?</li> 
              </ul> 
              <p><a href="http://docs.oracle.com/cd/B13789_01/appdev.101/b10807/07_errs.htm" rel="nofollow noreferrer">Oracle Documentation for Error Handling</a> does not give hints about logging.</p> 
              <p>It gives following sentence and uses dbms_output.put_line for logging:</p> 
              <blockquote> 
               <p>"Sometimes, you want to reraise an exception, that is, handle it locally, then pass it to an enclosing block. For example, you might want to roll back a transaction in the current block, then log the error in an enclosing block."</p> 
              </blockquote>

             <div class="post-text" itemprop="text"> 
              <p>Prevalent? No. It's not uncommon, however, to write a procedure like so:</p> 
              <pre><code>create or replace procedure write_to_log
  (p_entry_point in varchar,
   p_log_type    in varchar,
   p_message     in varchar)
as
pragma autonomous_transaction
begin
  insert into my_log_file
    (entry_point, log_type, message, log_timestamp)
  values
    (p_entry_point, p_log_type, p_message, systimestamp);
  commit;
end;
</code></pre> 
              <p>Then you can add calls to <code>write_to_log</code> where needed throughout the code.</p>
