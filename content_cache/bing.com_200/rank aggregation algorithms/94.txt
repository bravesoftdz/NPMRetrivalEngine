     <td width="66%"> 
      <blockquote> 
       <p>Here we consider <i>T</i>,&nbsp; the tightness of lower bounds. This is calculated as:</p> 
       <p>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; <i>T</i> = LowerBoundDist(<i>a</i>,<i>b</i>) / TrueEuclidianDist(<i>a</i>,<i>b</i>)</p> 
       <p>We randomly sample <i>a</i> and <i>b</i> (with replacement) 1,000 times for each combination of&nbsp; parameters. We vary the time series length [480, 960, 1440, 1920] and the number of bytes available per time series [16, 24, 32, 40]. We assume that each real valued representation requires 4 bytes per coefficient thus they use [4, 6, 8, 10] coefficients. For iSAX, we hard code the cardinality to 256, and we thus have [16, 24, 32, 40] words. For each experiment we created a bar chart and table. In addition, we include a link to the dataset if the dataset is not already in the public domain (UCR Archive data is <a href="http://www.cs.ucr.edu/~eamonn/Keogh_Time_Series_CDrom.zip"> here</a>, warning 500meg file, password <i>peggy</i>).</p> 
       <p>Recall that larger values are better. If the value of <i>T</i> is zero, then <i>any</i> indexing technique is condemned to retrieving every object from the disk. If the value of <i>T</i> is one, then there is no search, we could simply retrieve one file from disk and guarantee that we we had the true nearest neighbor. Note that the speedup obtained is generally non-linear in <i>T</i>, that is to say if one representation has a lower bound that is twice as large as another, we can usually expect a much greater than two-fold decrease in disk accesses.</p>

      </ul> <p><b>Notes:</b></p> 
      <ol> 
       <li> <p class="MsoNormal"><font size="2">There is an apparent paradox in the some datasets, including&nbsp; <a href="muscle_activation.bmp">Muscle Activation </a>dataset. The tightest of the lower bounds improves from 480 to 960. How is this possible? The data consist of noisy steps, like this [1, 2, 0, 1, 2, 0, 101, 100, 101, 103, 100, 0, 1, 2, 1, ...]. If a subsequence straddles at least one step, then the representation devotes most of its energy in approximating that step. However, if the subsequence falls within a single step, the z-normalization "expands" the signal to pure noise, which is hard to approximate. In this case, only the &nbsp;subsequences of length 480 can fall within a single step.</font></p></li> 
       <li> <p class="MsoNormal"><font size="2">In general, there is little difference between DCT, DFT, PAA, iPLA and CHEB. This has been noted before (see table 1 of this <a href="http://www.cs.ucr.edu/~eamonn/ICDM_2004.pdf">paper</a>). APCA is slightly worse that the rest if the data is completely stationary, but can be significantly better on datasets that are bursty, see<a href="SyntheticLightningEMP.bmp"> Synthetic Lightning EMP</a> as an example. Note that this dataset exhibits the apparent paradox discussed above. The <a href="foetal_ecg.bmp">Foetal_ecg</a> dataset also exhibits this burstyness and corresponding improvement for APCA.</font></p> </li>

   <li>The first paper on DFT for indexing time series is <a href="http://scholar.google.com/scholar?hl=en&amp;lr=&amp;cluster=14254743641493348509"> here</a>, however the graduate work of <a href="http://www.cs.ualberta.ca/~drafiei/"> Davood Rafiei</a> makes very useful contributions.</li> 
   <li>The first paper on DWT for indexing time series is <a href="http://scholar.google.com/scholar?hl=en&amp;lr=&amp;cluster=15311539954191173587"> here</a>, that paper used Haar wavelets, later papers generalized to other wavelets (see <a href="http://scholar.google.com/scholar?hl=en&amp;lr=&amp;cluster=12070572561008150335"> here</a>), but it makes little difference to the efficiency of indexing.</li> 
   <li>The first paper on Chebyshev polynomials (CHEB) for indexing time series is <a href="http://scholar.google.com/scholar?hl=en&amp;lr=&amp;cluster=3931365586105514100">here</a>, but see <a href="www.cs.ubc.ca-rngpsdepositorychebyReport2.pdf.pdf">this</a>.</li> 
   <li>The first paper on IPLA for indexing time series is <a href="IPLA.pdf"> here</a>, although a lower bound for more general PLA appears <a href="http://scholar.google.com/scholar?hl=en&amp;lr=&amp;cluster=10907439764878335133"> earlier</a>. </li> 
   <li>The first paper on PAA for indexing time series is <a href="http://scholar.google.com/scholar?hl=en&amp;lr=&amp;cluster=11561962358756747085"> here</a> (it was called PCA, but later <a href="http://scholar.google.com/scholar?hl=en&amp;lr=&amp;cluster=9353000770571149358"> renamed</a>), the same representation was independently discovered 6 months later, and named <a href="http://scholar.google.com/scholar?hl=en&amp;lr=&amp;cluster=15456600195255057902"> Segmented Means</a>.</li> 
   <li>The first paper on APCA for indexing time series is <a href="http://scholar.google.com/scholar?hl=en&amp;lr=&amp;cluster=6628959700592916088"> here</a>.</li> 
   <li>The first paper on SVD for indexing time series is <a href="http://scholar.google.com/scholar?hl=en&amp;lr=&amp;cluster=17037833577761002858"> here</a>, the same paper is the first to use DCT (as a strawman comparison). This <a href="http://scholar.google.com/scholar?hl=en&amp;lr=&amp;cluster=9353000770571149358"> paper</a> compares SVD, PAA/DWT and DFT.</li> 
   <li><span style="font-size: 12.0pt; font-family: Times New Roman">Michalis Vlachos and Spiros Papadimitriou present a wonderful </span> <a href="tutorial_ICDM06.ppt">tutorial</a> at ICDM06 that beautifully explains all the above approaches and compares (most of) them on 65 dataset (page 292).</li> 
  </ul>
