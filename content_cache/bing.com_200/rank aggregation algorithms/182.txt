       <h3> <span class="mw-headline" id="Line_of_sight.2C_field_of_vision">Line of sight, field of vision</span></h3> 
       <ul>
        <li> <a href="/index.php?title=Line_of_Sight" title="Line of Sight" class="mw-redirect">Line of Sight</a> is used to determine when a specific destination square is visible from a source square. This can be used to determine whether a player is visible to a monster, the path an arrow takes to a particular enemy, and as a building block for a field of vision algorithm. </li>
        <li> <a href="/index.php?title=Field_of_Vision" title="Field of Vision">Field of Vision</a> determines all squares visible from a particular source. This is useful when determining which squares to show a player, what squares are lit up by a light source, etc. </li>
        <li> <a href="/index.php?title=Complete_Roguelike_Tutorial,_using_python%2Blibtcod,_part_4" title="Complete Roguelike Tutorial, using python+libtcod, part 4">Using the libtcod library for FOV, and implementing fog-of-war</a>, part of the python+libtcod tutorial. </li>
        <li> <a rel="nofollow" class="external text" href="http://www.evilscience.co.uk/?p=225">C# implementation of field of vision using recursive shadow casting</a> </li>
        <li> <a rel="nofollow" class="external text" href="https://gist.github.com/zloedi/9551625">Grid based shadow casting / field of vision</a> </li>
       </ul>
