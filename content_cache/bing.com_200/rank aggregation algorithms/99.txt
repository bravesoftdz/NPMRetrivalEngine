        <div class="section" id="text-indexes"> 
         <span id="index-feature-text"></span>
         <h1>Text Indexes<a class="headerlink" href="#text-indexes" title="Permalink to this headline">¶</a></h1> 
         <div class="contents singlecol local topic" id="on-this-page"> 
          <p class="topic-title first">On this page</p> 
          <ul class="simple"> 
           <li><a class="reference internal" href="#overview" id="id4">Overview</a></li> 
           <li><a class="reference internal" href="#create-text-index" id="id5">Create Text Index</a></li> 
           <li><a class="reference internal" href="#case-insensitivity" id="id6">Case Insensitivity</a></li> 
           <li><a class="reference internal" href="#diacritic-insensitivity" id="id7">Diacritic Insensitivity</a></li> 
           <li><a class="reference internal" href="#tokenization-delimiters" id="id8">Tokenization Delimiters</a></li> 
           <li><a class="reference internal" href="#index-entries" id="id9">Index Entries</a></li> 
           <li><a class="reference internal" href="#supported-languages-and-stop-words" id="id10">Supported Languages and Stop Words</a></li> 
           <li><a class="reference internal" href="#sparse-property" id="id11"><code class="docutils literal"><span class="pre">sparse</span></code> Property</a></li> 
           <li><a class="reference internal" href="#restrictions" id="id12">Restrictions</a></li> 
           <li><a class="reference internal" href="#storage-requirements-and-performance-costs" id="id13">Storage Requirements and Performance Costs</a></li> 
           <li><a class="reference internal" href="#text-search-support" id="id14">Text Search Support</a></li> 
          </ul> 
         </div> 
         <div class="versionchanged"> 
          <p><span class="versionmodified">Changed in version 3.2.</span></p> 
         </div> 
         <p>Starting in MongoDB 3.2, MongoDB introduces a version 3 of the <code class="docutils literal"><span class="pre">text</span></code> index. Key features of the new version of the index are:</p> 
         <ul class="simple"> 
          <li>Improved <a class="reference internal" href="#text-index-case-insensitivity"><span class="std std-ref">case insensitivity</span></a></li> 
          <li><a class="reference internal" href="#text-index-diacritic-insensitivity"><span class="std std-ref">Diacritic insensitivity</span></a></li> 
          <li>Additional <a class="reference internal" href="#text-index-tokenization-delimiters"><span class="std std-ref">delimiters for tokenization</span></a></li> 
         </ul> 
         <p>Starting in MongoDB 3.2, version 3 is the default version for new <code class="docutils literal"><span class="pre">text</span></code> indexes.</p> 
         <div class="section" id="overview"> 
          <h2>Overview<a class="headerlink" href="#overview" title="Permalink to this headline">¶</a></h2> 
          <p>MongoDB provides <a class="reference internal" href="#index-feature-text"><span class="std std-ref">text indexes</span></a> to support text search queries on string content. <code class="docutils literal"><span class="pre">text</span></code> indexes can include any field whose value is a string or an array of string elements.</p> 
         </div> 
         <div class="section" id="create-text-index"> 
          <span id="id1"></span>
          <h2>Create Text Index<a class="headerlink" href="#create-text-index" title="Permalink to this headline">¶</a></h2> 
          <div class="admonition important"> 
           <p class="first admonition-title">Important</p> 
           <p class="last">A collection can have at most <strong>one</strong> <code class="docutils literal"><span class="pre">text</span></code> index.</p> 
          </div> 
          <p>To create a <code class="docutils literal"><span class="pre">text</span></code> index, use the <a class="reference internal" href="../../reference/method/db.collection.createIndex/#db.collection.createIndex" title="db.collection.createIndex()"><code class="xref mongodb mongodb-method docutils literal"><span class="pre">db.collection.createIndex()</span></code></a> method. To index a field that contains a string or an array of string elements, include the field and specify the string literal <code class="docutils literal"><span class="pre">"text"</span></code> in the index document, as in the following example:</p> 
          <div class="highlight-javascript">
           <div class="highlight">
            <pre><span></span><span class="nx">db</span><span class="p">.</span><span class="nx">reviews</span><span class="p">.</span><span class="nx">createIndex</span><span class="p">(</span> <span class="p">{</span> <span class="nx">comments</span><span class="o">:</span> <span class="s2">"text"</span> <span class="p">}</span> <span class="p">)</span>
</pre>
           </div> 
          </div> 
          <p>You can index multiple fields for the <code class="docutils literal"><span class="pre">text</span></code> index. The following example creates a <code class="docutils literal"><span class="pre">text</span></code> index on the fields <code class="docutils literal"><span class="pre">subject</span></code> and <code class="docutils literal"><span class="pre">comments</span></code>:</p> 
          <div class="highlight-javascript">
           <div class="highlight">
            <pre><span></span><span class="nx">db</span><span class="p">.</span><span class="nx">reviews</span><span class="p">.</span><span class="nx">createIndex</span><span class="p">(</span>
   <span class="p">{</span>
     <span class="nx">subject</span><span class="o">:</span> <span class="s2">"text"</span><span class="p">,</span>
     <span class="nx">comments</span><span class="o">:</span> <span class="s2">"text"</span>
   <span class="p">}</span>
 <span class="p">)</span>
</pre>
           </div> 
          </div> 
          <p>A <a class="reference internal" href="../index-compound/"><span class="doc">compound index</span></a> can include <code class="docutils literal"><span class="pre">text</span></code> index keys in combination with ascending/descending index keys. For more information, see <a class="reference internal" href="#text-index-compound"><span class="std std-ref">Compound Index</span></a>.</p> 
          <p>In order to drop a <code class="docutils literal"><span class="pre">text</span></code> index, use the index name. See <a class="reference internal" href="../../tutorial/avoid-text-index-name-limit/#drop-text-index"><span class="std std-ref">Use the Index Name to Drop a text Index</span></a> for more information.</p> 
          <div class="section" id="specify-weights"> 
           <h3>Specify Weights<a class="headerlink" href="#specify-weights" title="Permalink to this headline">¶</a></h3> 
           <p>For a <code class="docutils literal"><span class="pre">text</span></code> index, the <em>weight</em> of an indexed field denotes the significance of the field relative to the other indexed fields in terms of the text search score.</p> 
           <p>For each indexed field in the document, MongoDB multiplies the number of matches by the weight and sums the results. Using this sum, MongoDB then calculates the score for the document. See <a class="reference internal" href="../../reference/operator/projection/meta/#proj._S_meta" title="$meta"><code class="xref mongodb mongodb-projection docutils literal"><span class="pre">$meta</span></code></a> operator for details on returning and sorting by text scores.</p> 
           <p>The default weight is 1 for the indexed fields. To adjust the weights for the indexed fields, include the <code class="docutils literal"><span class="pre">weights</span></code> option in the <a class="reference internal" href="../../reference/method/db.collection.createIndex/#db.collection.createIndex" title="db.collection.createIndex()"><code class="xref mongodb mongodb-method docutils literal"><span class="pre">db.collection.createIndex()</span></code></a> method.</p> 
           <p>For more information using weights to control the results of a text search, see <a class="reference internal" href="../../tutorial/control-results-of-text-search/"><span class="doc">Control Search Results with Weights</span></a>.</p> 
          </div> 
          <div class="section" id="wildcard-text-indexes"> 
           <span id="text-index-wildcard"></span>
           <h3>Wildcard Text Indexes<a class="headerlink" href="#wildcard-text-indexes" title="Permalink to this headline">¶</a></h3> 
           <p>When creating a <code class="docutils literal"><span class="pre">text</span></code> index on multiple fields, you can also use the wildcard specifier (<code class="docutils literal"><span class="pre">$**</span></code>). With a wildcard text index, MongoDB indexes every field that contains string data for each document in the collection. The following example creates a text index using the wildcard specifier:</p> 
           <div class="highlight-javascript">
            <div class="highlight">
             <pre><span></span><span class="nx">db</span><span class="p">.</span><span class="nx">collection</span><span class="p">.</span><span class="nx">createIndex</span><span class="p">(</span> <span class="p">{</span> <span class="s2">"$**"</span><span class="o">:</span> <span class="s2">"text"</span> <span class="p">}</span> <span class="p">)</span>
</pre>
            </div> 
           </div> 
           <p>This index allows for text search on all fields with string content. Such an index can be useful with highly unstructured data if it is unclear which fields to include in the text index or for ad-hoc querying.</p> 
           <p>Wildcard text indexes are <code class="docutils literal"><span class="pre">text</span></code> indexes on multiple fields. As such, you can assign weights to specific fields during index creation to control the ranking of the results. For more information using weights to control the results of a text search, see <a class="reference internal" href="../../tutorial/control-results-of-text-search/"><span class="doc">Control Search Results with Weights</span></a>.</p> 
           <p>Wildcard text indexes, as with all text indexes, can be part of a compound indexes. For example, the following creates a compound index on the field <code class="docutils literal"><span class="pre">a</span></code> as well as the wildcard specifier:</p> 
           <div class="highlight-javascript">
            <div class="highlight">
             <pre><span></span><span class="nx">db</span><span class="p">.</span><span class="nx">collection</span><span class="p">.</span><span class="nx">createIndex</span><span class="p">(</span> <span class="p">{</span> <span class="nx">a</span><span class="o">:</span> <span class="mi">1</span><span class="p">,</span> <span class="s2">"$**"</span><span class="o">:</span> <span class="s2">"text"</span> <span class="p">}</span> <span class="p">)</span>
</pre>
            </div> 
           </div> 
           <p>As with all <a class="reference internal" href="#text-index-compound"><span class="std std-ref">compound text indexes</span></a>, since the <code class="docutils literal"><span class="pre">a</span></code> precedes the text index key, in order to perform a <a class="reference internal" href="../../reference/operator/query/text/#op._S_text" title="$text"><code class="xref mongodb mongodb-query docutils literal"><span class="pre">$text</span></code></a> search with this index, the query predicate must include an equality match conditions <code class="docutils literal"><span class="pre">a</span></code>. For information on compound text indexes, see <a class="reference internal" href="#text-index-compound"><span class="std std-ref">Compound Text Indexes</span></a>.</p> 
          </div> 
         </div> 
         <div class="section" id="case-insensitivity"> 
          <span id="text-index-case-insensitivity"></span>
          <h2>Case Insensitivity<a class="headerlink" href="#case-insensitivity" title="Permalink to this headline">¶</a></h2> 
          <div class="versionchanged"> 
           <p><span class="versionmodified">Changed in version 3.2.</span></p> 
          </div> 
          <p>The version 3 <code class="docutils literal"><span class="pre">text</span></code> index supports the common <code class="docutils literal"><span class="pre">C</span></code>, simple <code class="docutils literal"><span class="pre">S</span></code>, and for Turkish languages, the special <code class="docutils literal"><span class="pre">T</span></code> case foldings as specified in <a class="reference external" href="http://www.unicode.org/Public/8.0.0/ucd/CaseFolding.txt">Unicode 8.0 Character Database Case Folding</a>.</p> 
          <p>The case foldings expands the case insensitivity of the <code class="docutils literal"><span class="pre">text</span></code> index to include characters with diacritics, such as <code class="docutils literal"><span class="pre">é</span></code> and <code class="docutils literal"><span class="pre">É</span></code>, and characters from non-Latin alphabets, such as “?” and “?” in the Cyrillic alphabet.</p> 
          <p>Version 3 of the <code class="docutils literal"><span class="pre">text</span></code> index is also <a class="reference internal" href="#text-index-diacritic-insensitivity"><span class="std std-ref">diacritic insensitive</span></a>. As such, the index also does not distinguish between <code class="docutils literal"><span class="pre">é</span></code>, <code class="docutils literal"><span class="pre">É</span></code>, <code class="docutils literal"><span class="pre">e</span></code>, and <code class="docutils literal"><span class="pre">E</span></code>.</p> 
          <p>Previous versions of the <code class="docutils literal"><span class="pre">text</span></code> index are case insensitive for <code class="docutils literal"><span class="pre">[A-z]</span></code> only; i.e. case insensitive for non-diacritics Latin characters only . For all other characters, earlier versions of the text index treat them as distinct.</p> 
         </div> 
         <div class="section" id="diacritic-insensitivity"> 
          <span id="text-index-diacritic-insensitivity"></span>
          <h2>Diacritic Insensitivity<a class="headerlink" href="#diacritic-insensitivity" title="Permalink to this headline">¶</a></h2> 
          <div class="versionchanged"> 
           <p><span class="versionmodified">Changed in version 3.2.</span></p> 
          </div> 
          <p>With version 3, <code class="docutils literal"><span class="pre">text</span></code> index is diacritic insensitive. That is, the index does not distinguish between characters that contain diacritical marks and their non-marked counterpart, such as <code class="docutils literal"><span class="pre">é</span></code>, <code class="docutils literal"><span class="pre">ê</span></code>, and <code class="docutils literal"><span class="pre">e</span></code>. More specifically, the <code class="docutils literal"><span class="pre">text</span></code> index strips the characters categorized as diacritics in <a class="reference external" href="http://www.unicode.org/Public/8.0.0/ucd/PropList.txt">Unicode 8.0 Character Database Prop List</a>.</p> 
          <p>Version 3 of the <code class="docutils literal"><span class="pre">text</span></code> index is also <a class="reference internal" href="#text-index-case-insensitivity"><span class="std std-ref">case insensitive</span></a> to characters with diacritics. As such, the index also does not distinguish between <code class="docutils literal"><span class="pre">é</span></code>, <code class="docutils literal"><span class="pre">É</span></code>, <code class="docutils literal"><span class="pre">e</span></code>, and <code class="docutils literal"><span class="pre">E</span></code>.</p> 
          <p>Previous versions of the <code class="docutils literal"><span class="pre">text</span></code> index treat characters with diacritics as distinct.</p> 
         </div> 
         <div class="section" id="tokenization-delimiters"> 
          <span id="text-index-tokenization-delimiters"></span>
          <h2>Tokenization Delimiters<a class="headerlink" href="#tokenization-delimiters" title="Permalink to this headline">¶</a></h2> 
          <div class="versionchanged"> 
           <p><span class="versionmodified">Changed in version 3.2.</span></p> 
          </div> 
          <p>For tokenization, version 3 <code class="docutils literal"><span class="pre">text</span></code> index uses the delimiters categorized under <code class="docutils literal"><span class="pre">Dash</span></code>, <code class="docutils literal"><span class="pre">Hyphen</span></code>, <code class="docutils literal"><span class="pre">Pattern_Syntax</span></code>, <code class="docutils literal"><span class="pre">Quotation_Mark</span></code>, <code class="docutils literal"><span class="pre">Terminal_Punctuation</span></code>, and <code class="docutils literal"><span class="pre">White_Space</span></code> in <a class="reference external" href="http://www.unicode.org/Public/8.0.0/ucd/PropList.txt">Unicode 8.0 Character Database Prop List</a>.</p> 
          <p>For example, if given a string <code class="docutils literal"><span class="pre">"Il</span> <span class="pre">a</span> <span class="pre">dit</span> <span class="pre">qu'il</span> <span class="pre">«était</span> <span class="pre">le</span> <span class="pre">meilleur</span> <span class="pre">joueur</span> <span class="pre">du</span> <span class="pre">monde»"</span></code>, the <code class="docutils literal"><span class="pre">text</span></code> index treats <code class="docutils literal"><span class="pre">«</span></code>, <code class="docutils literal"><span class="pre">»</span></code>, and spaces as delimiters.</p> 
          <p>Previous versions of the index treat <code class="docutils literal"><span class="pre">«</span></code> as part of the term <code class="docutils literal"><span class="pre">"«était"</span></code> and <code class="docutils literal"><span class="pre">»</span></code> as part of the term <code class="docutils literal"><span class="pre">"monde»"</span></code>.</p> 
         </div> 
         <div class="section" id="index-entries"> 
          <h2>Index Entries<a class="headerlink" href="#index-entries" title="Permalink to this headline">¶</a></h2> 
          <p><code class="docutils literal"><span class="pre">text</span></code> index tokenizes and stems the terms in the indexed fields for the index entries. <code class="docutils literal"><span class="pre">text</span></code> index stores one index entry for each unique stemmed term in each indexed field for each document in the collection. The index uses simple <a class="reference internal" href="#text-index-supported-languages"><span class="std std-ref">language-specific</span></a> suffix stemming.</p> 
         </div> 
         <div class="section" id="supported-languages-and-stop-words"> 
          <span id="text-index-supported-languages"></span>
          <h2>Supported Languages and Stop Words<a class="headerlink" href="#supported-languages-and-stop-words" title="Permalink to this headline">¶</a></h2> 
          <p>MongoDB supports text search for various languages. <code class="docutils literal"><span class="pre">text</span></code> indexes drop language-specific stop words (e.g. in English, <code class="docutils literal"><span class="pre">the</span></code>, <code class="docutils literal"><span class="pre">an</span></code>, <code class="docutils literal"><span class="pre">a</span></code>, <code class="docutils literal"><span class="pre">and</span></code>, etc.) and use simple language-specific suffix stemming. For a list of the supported languages, see <a class="reference internal" href="../../reference/text-search-languages/#text-search-languages"><span class="std std-ref">Text Search Languages</span></a>.</p> 
          <p>If you specify a language value of <code class="docutils literal"><span class="pre">"none"</span></code>, then the <code class="docutils literal"><span class="pre">text</span></code> index uses simple tokenization with no list of stop words and no stemming.</p> 
          <p>To specify a language for the <code class="docutils literal"><span class="pre">text</span></code> index, see <a class="reference internal" href="../../tutorial/specify-language-for-text-index/"><span class="doc">Specify a Language for Text Index</span></a>.</p> 
         </div> 
         <div class="section" id="sparse-property"> 
          <h2><code class="docutils literal"><span class="pre">sparse</span></code> Property<a class="headerlink" href="#sparse-property" title="Permalink to this headline">¶</a></h2> 
          <p><code class="docutils literal"><span class="pre">text</span></code> indexes are <a class="reference internal" href="../index-sparse/"><span class="doc">sparse</span></a> by default and ignore the <a class="reference internal" href="../index-sparse/"><span class="doc">sparse: true</span></a> option. If a document lacks a <code class="docutils literal"><span class="pre">text</span></code> index field (or the field is <code class="docutils literal"><span class="pre">null</span></code> or an empty array), MongoDB does not add an entry for the document to the <code class="docutils literal"><span class="pre">text</span></code> index. For inserts, MongoDB inserts the document but does not add to the <code class="docutils literal"><span class="pre">text</span></code> index.</p> 
          <p>For a compound index that includes a <code class="docutils literal"><span class="pre">text</span></code> index key along with keys of other types, only the <code class="docutils literal"><span class="pre">text</span></code> index field determines whether the index references a document. The other keys do not determine whether the index references the documents or not.</p> 
         </div> 
         <div class="section" id="restrictions"> 
          <h2>Restrictions<a class="headerlink" href="#restrictions" title="Permalink to this headline">¶</a></h2> 
          <div class="section" id="one-text-index-per-collection"> 
           <h3>One Text Index Per Collection<a class="headerlink" href="#one-text-index-per-collection" title="Permalink to this headline">¶</a></h3> 
           <p>A collection can have at most <strong>one</strong> <code class="docutils literal"><span class="pre">text</span></code> index.</p> 
          </div> 
          <div class="section" id="text-search-and-hints"> 
           <h3>Text Search and Hints<a class="headerlink" href="#text-search-and-hints" title="Permalink to this headline">¶</a></h3> 
           <p>You cannot use <a class="reference internal" href="../../reference/method/cursor.hint/#cursor.hint" title="cursor.hint()"><code class="xref mongodb mongodb-method docutils literal"><span class="pre">hint()</span></code></a> if the query includes a <a class="reference internal" href="../../reference/operator/query/text/#op._S_text" title="$text"><code class="xref mongodb mongodb-query docutils literal"><span class="pre">$text</span></code></a> query expression.</p> 
          </div> 
          <div class="section" id="text-index-and-sort"> 
           <span id="id3"></span>
           <h3>Text Index and Sort<a class="headerlink" href="#text-index-and-sort" title="Permalink to this headline">¶</a></h3> 
           <p>Sort operations cannot obtain sort order from a <code class="docutils literal"><span class="pre">text</span></code> index, even from a <a class="reference internal" href="#text-index-compound"><span class="std std-ref">compound text index</span></a>; i.e. sort operations cannot use the ordering in the text index.</p> 
          </div> 
          <div class="section" id="compound-index"> 
           <span id="text-index-compound"></span>
           <h3>Compound Index<a class="headerlink" href="#compound-index" title="Permalink to this headline">¶</a></h3> 
           <p>A <a class="reference internal" href="../index-compound/"><span class="doc">compound index</span></a> can include a <code class="docutils literal"><span class="pre">text</span></code> index key in combination with ascending/descending index keys. However, these compound indexes have the following restrictions:</p> 
           <ul class="simple"> 
            <li>A compound <code class="docutils literal"><span class="pre">text</span></code> index cannot include any other special index types, such as <a class="reference internal" href="../index-multikey/#index-type-multi-key"><span class="std std-ref">multi-key</span></a> or <a class="reference internal" href="../../geospatial-queries/#index-feature-geospatial"><span class="std std-ref">geospatial</span></a> index fields.</li> 
            <li>If the compound <code class="docutils literal"><span class="pre">text</span></code> index includes keys <strong>preceding</strong> the <code class="docutils literal"><span class="pre">text</span></code> index key, to perform a <a class="reference internal" href="../../reference/operator/query/text/#op._S_text" title="$text"><code class="xref mongodb mongodb-query docutils literal"><span class="pre">$text</span></code></a> search, the query predicate must include <strong>equality match conditions</strong> on the preceding keys.</li> 
           </ul> 
           <p>See also <a class="reference internal" href="#text-index-and-sort"><span class="std std-ref">Text Index and Sort</span></a> for additional limitations.</p> 
           <p>For an example of a compound text index, see <a class="reference internal" href="../../tutorial/limit-number-of-items-scanned-for-text-search/"><span class="doc">Limit the Number of Entries Scanned</span></a>.</p> 
          </div> 
          <div class="section" id="drop-a-text-index"> 
           <span id="text-index-storage-requirements"></span>
           <h3>Drop a Text Index<a class="headerlink" href="#drop-a-text-index" title="Permalink to this headline">¶</a></h3> 
           <p>To drop a <code class="docutils literal"><span class="pre">text</span></code> index, pass the <em>name</em> of the index to the <a class="reference internal" href="../../reference/method/db.collection.dropIndex/#db.collection.dropIndex" title="db.collection.dropIndex()"><code class="xref mongodb mongodb-method docutils literal"><span class="pre">db.collection.dropIndex()</span></code></a> method. To get the name of the index, run the <a class="reference internal" href="../../reference/method/db.collection.getIndexes/#db.collection.getIndexes" title="db.collection.getIndexes()"><code class="xref mongodb mongodb-method docutils literal"><span class="pre">db.collection.getIndexes()</span></code></a> method.</p> 
           <p>For information on the default naming scheme for <code class="docutils literal"><span class="pre">text</span></code> indexes as well as overriding the default name, see <a class="reference internal" href="../../tutorial/avoid-text-index-name-limit/"><span class="doc">Specify Name for text Index</span></a>.</p> 
          </div> 
         </div> 
         <div class="section" id="storage-requirements-and-performance-costs"> 
          <h2>Storage Requirements and Performance Costs<a class="headerlink" href="#storage-requirements-and-performance-costs" title="Permalink to this headline">¶</a></h2> 
          <p><code class="docutils literal"><span class="pre">text</span></code> indexes have the following storage requirements and performance costs:</p> 
          <ul class="simple"> 
           <li><code class="docutils literal"><span class="pre">text</span></code> indexes can be large. They contain one index entry for each unique post-stemmed word in each indexed field for each document inserted.</li> 
           <li>Building a <code class="docutils literal"><span class="pre">text</span></code> index is very similar to building a large multi-key index and will take longer than building a simple ordered (scalar) index on the same data.</li> 
           <li>When building a large <code class="docutils literal"><span class="pre">text</span></code> index on an existing collection, ensure that you have a sufficiently high limit on open file descriptors. See the <a class="reference internal" href="../../reference/ulimit/"><span class="doc">recommended settings</span></a>.</li> 
           <li><code class="docutils literal"><span class="pre">text</span></code> indexes will impact insertion throughput because MongoDB must add an index entry for each unique post-stemmed word in each indexed field of each new source document.</li> 
           <li>Additionally, <code class="docutils literal"><span class="pre">text</span></code> indexes do not store phrases or information about the proximity of words in the documents. As a result, phrase queries will run much more effectively when the entire collection fits in RAM.</li> 
          </ul> 
         </div> 
         <div class="section" id="text-search-support"> 
          <h2>Text Search Support<a class="headerlink" href="#text-search-support" title="Permalink to this headline">¶</a></h2> 
          <p>The <code class="docutils literal"><span class="pre">text</span></code> index supports <a class="reference internal" href="../../reference/operator/query/text/#op._S_text" title="$text"><code class="xref mongodb mongodb-query docutils literal"><span class="pre">$text</span></code></a> query operations. For examples of text search, see the <a class="reference internal" href="../../reference/operator/query/text/#op._S_text" title="$text"><code class="xref mongodb mongodb-query docutils literal"><span class="pre">$text</span> <span class="pre">reference</span> <span class="pre">page</span></code></a>. For examples of <a class="reference internal" href="../../reference/operator/query/text/#op._S_text" title="$text"><code class="xref mongodb mongodb-query docutils literal"><span class="pre">$text</span></code></a> operations in aggregation pipelines, see <a class="reference internal" href="../../tutorial/text-search-in-aggregation/"><span class="doc">Text Search in the Aggregation Pipeline</span></a>.</p>
